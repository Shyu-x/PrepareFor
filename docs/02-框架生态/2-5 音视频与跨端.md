## 一、跨平台组件库架构设计

### 1.1 如何设计一个高性能、可扩展的跨平台组件库？

**架构要点：**
1.  **逻辑与样式分离**：
    *   **Headless 组件层**：封装状态和交互逻辑（React Hooks）。
    *   **适配层**：根据不同端（Web, MiniProgram, RN）分发底层原子组件。
2.  **样式方案选型**：使用 CSS-in-JS 或原子化 CSS (Tailwind) 来抹平不同端对样式的支持差异。
3.  **编译驱动**：利用 AST (抽象语法树) 在打包阶段将逻辑代码转换为特定平台的 DSL。
