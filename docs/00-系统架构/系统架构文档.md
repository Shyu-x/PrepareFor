## 一、项目概览

### 1.1 项目简介

PrepareFor 是一个功能完善的前端面试题库系统，采用前后端分离架构设计。系统提供文档管理、内容搜索、主题切换等丰富的功能，帮助前端开发者高效准备技术面试。

### 1.2 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 前端框架 | Next.js | 16.x (App Router) |
| UI组件库 | Ant Design | 5.x |
| 状态管理 | Zustand | 5.x |
| 动画 | Framer Motion | 12.x |
| Markdown解析 | marked | 15.x |
| 代码高亮 | highlight.js | 11.x |
| 可视化图表 | ECharts | 5.x |
| 后端框架 | NestJS | 10.x |
| 包管理 | Yarn Workspaces | - |
| 部署 | Docker | 24.x |

## 二、后端API接口

### 2.1 基础信息

| 项目 | 值 |
|------|-----|
| 基础URL | `http://localhost:42123` |
| 前缀 | `/api` |
| 框架 | NestJS |
| 文档目录 | `项目根目录/前端面试题汇总` |

### 2.2 接口列表

| 方法 | 路径 | 功能 | 请求参数 | 返回格式 |
|------|------|------|----------|----------|
| GET | `/api` | 健康检查/欢迎 | - | `string` |
| GET | `/api/docs` | 获取文档目录结构 | - | `DocTree[]` |
| GET | `/api/doc` | 获取文档内容 | `filename: string` | `{ content: string }` |
| POST | `/api/plantuml` | 渲染PlantUML图表 | `{ code: string }` | `{ svg: string }` |

### 2.3 详细说明

#### 2.3.1 GET /api - 健康检查

```bash
curl http://localhost:42123/api
```

```json
"API Server is running!"
```

#### 2.3.2 GET /api/docs - 获取文档目录

```bash
curl http://localhost:42123/api/docs
```

```typescript
// 返回值类型
interface DocTree {
  title: string;      // 显示标题
  key: string;        // 唯一标识(路径)
  type: 'directory' | 'file';
  children?: DocTree[];
}

// 示例
[
  {
    "title": "01-基础核心",
    "key": "01-基础核心",
    "type": "directory",
    "children": [
      {
        "title": "1-1 HTMLCSS.md",
        "key": "01-基础核心/1-1 HTMLCSS.md",
        "type": "file"
      }
    ]
  }
]
```

#### 2.3.3 GET /api/doc - 获取文档内容

```bash
curl "http://localhost:42123/api/doc?filename=01-基础核心/1-1 HTMLCSS.md"
```

```typescript
// 请求参数
filename: string  // 文件路径，如 "01-基础核心/1-1 HTMLCSS.md"

// 成功返回
{
  "content": "# 文档内容..."
}

// 错误返回
{
  "error": "File not found: xxx"
}
```

#### 2.3.4 POST /api/plantuml - 渲染PlantUML

```bash
curl -X POST http://localhost:42123/api/plantuml \
  -H "Content-Type: application/json" \
  -d '{"code": "@startuml\nAlice -> Bob: Hello\n@enduml"}'
```

```typescript
// 请求体
{
  "code": "@startuml\nAlice -> Bob: Hello\n@enduml"
}

// 返回值
{
  "svg": "https://www.plantuml.com/plantuml/svg/..."
}
```

## 五、环境配置

### 5.1 开发环境

```bash
# 安装依赖
yarn install

# 启动前端 (端口 33980)
yarn dev:web

# 启动后端 (端口 42123)
yarn dev:api

# 同时启动
yarn dev
```

### 5.2 Docker 部署

```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

### 5.3 端口配置

| 服务 | 端口 | 说明 |
|------|------|------|
| 前端开发 | 33980 | Next.js 开发服务器 |
| 后端开发 | 42123 | NestJS 服务器 |
| 前端生产 | 3000 | Docker 部署 |
| 后端生产 | 42123 | Docker 部署 |

## 八、注意事项

1. 后端 API 固定端口 42123
2. 前端通过 fetch 请求后端 API
3. 文档路径使用正斜杠 `/`
4. PlantUML 使用第三方服务渲染
5. 主题设置保存到 localStorage

## 九、常见问题

### Q1: 前端无法连接到后端
**A**: 确保后端服务运行在 localhost:42123，检查 CORS 配置

### Q2: 文档加载失败
**A**: 确认 `前端面试题汇总` 目录存在且包含 .md 文件

### Q3: 搜索功能不工作
**A**: 检查浏览器控制台是否有错误，确保 API 正常工作
