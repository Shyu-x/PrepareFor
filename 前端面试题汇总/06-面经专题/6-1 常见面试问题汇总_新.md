## 一、HTML/CSS 基础

### 1.1 CSS 盒模型

**面试官问**：CSS 盒模型是什么？标准的和 IE 的有啥不一样？

**标准盒模型**：
- `box-sizing: content-box`（默认值）
- 元素 width/height 只包含内容区域
- 总宽度 = width + padding + border + margin

**IE 盒模型**：
- `box-sizing: border-box`
- 元素 width/height 包含内容 + padding + border
- 总宽度 = width + margin

```css
/* 标准盒模型 */
.box-standard {
  box-sizing: content-box;
  width: 100px;
  padding: 10px;
  border: 5px solid #333;
  /* 实际占用宽度 = 100 + 20 + 10 = 130px */
}

/* IE 盒模型 */
.box-ie {
  box-sizing: border-box;
  width: 100px;
  padding: 10px;
  border: 5px solid #333;
  /* 内容区域 = 100 - 20 - 10 = 70px */
}

/* 全局设置盒模型 */
* {
  box-sizing: border-box;
}
```

### 1.2 CSS 选择器优先级

**面试官问**：CSS 选择器的优先级是怎么计算的？

**优先级计算规则**：

| 选择器 | 权重 |
|--------|------|
| `!important` | 最高 |
| 内联样式 `style=""` | 1000 |
| ID 选择器 `#id` | 100 |
| 类选择器 `.class`、属性选择器 `[attr]`、伪类 `:hover` | 10 |
| 元素选择器 `div`、伪元素 `::before` | 1 |
| 通配符 `*`、组合符 `+`、`>`、`~` | 0 |

```css
/* 优先级计算示例 */
#nav .list li:first-child  /* 1*100 + 1*10 + 1*1 + 1*10 = 121 */
.nav .list .item            /* 10 + 10 + 10 = 30 */
div p                       /* 1 + 1 = 2 */
```

**注意事项**：
- `!important` 会覆盖正常优先级规则
- 相同优先级时，后定义的样式生效
- 继承的属性优先级最低

### 1.3 Flex 布局

**面试官问**：讲讲 flex 属性有什么用？

**Flex 容器属性**：

```css
.container {
  display: flex;

  /* 主轴方向 */
  flex-direction: row | row-reverse | column | column-reverse;

  /* 换行方式 */
  flex-wrap: nowrap | wrap | wrap-reverse;

  /* 主轴对齐 */
  justify-content: flex-start | flex-end | center | space-between | space-around;

  /* 交叉轴对齐 */
  align-items: stretch | flex-start | flex-end | center | baseline;

  /* 多行对齐 */
  align-content: flex-start | flex-end | center | space-between | space-around | stretch;
}
```

**Flex 项目属性**：

```css
.item {
  /* 放大比例 */
  flex-grow: 0 | 1;

  /* 缩小比例 */
  flex-shrink: 0 | 1;

  /* 基础宽度 */
  flex-basis: auto | 100px | 50%;

  /* 简写 */
  flex: 0 1 auto;
}
```

**flex: 1 是什么**：
- `flex: 1` = `flex: 1 1 0%`
- 项目会平均分配剩余空间
- `flex-shrink: 1` 表示空间不足时会缩小
- `flex-basis: 0%` 表示初始不占用空间

### 1.5 BFC 块级格式化上下文

**面试官问**：解释一下 BFC 的原理及其常见应用场景。

**BFC 定义**：
BFC 是页面上一个独立的渲染区域，容器内的元素与外部元素互不影响。

**触发 BFC 的方式**：
- `display: flex` / `display: grid`
- `float` 不为 none
- `position: absolute` / `fixed`
- `overflow` 不为 visible
- 根元素 `<html>`

**BFC 应用场景**：

```css
/* 1. 防止 margin 合并 */
.box1, .box2 {
  margin-bottom: 20px;
}
.parent {
  overflow: hidden;  /* 创建 BFC，防止 margin 合并 */
}

/* 2. 清除浮动 */
.clearfix::after {
  content: '';
  display: block;
  clear: both;
}

/* 3. 自适应两栏布局 */
.left {
  float: left;
  width: 200px;
}
.right {
  overflow: hidden;  /* 创建 BFC，不被浮动影响 */
}
```

### 1.6 移动端适配方案

**面试官问**：移动端不同设备适配有哪些方案？

**方案一：rem 适配**

```javascript
// 动态设置 rem 基准值
function setRem() {
  const doc = document.documentElement;
  const width = doc.getBoundingClientRect().width;
  doc.style.fontSize = width / 7.5 + 'px';  // 750px 设计稿分为 7.5 份
}
setRem();
window.addEventListener('resize', setRem);
```

```css
/* 使用 rem */
.box {
  width: 2rem;  /* 750px 设计稿中 150px */
  padding: 0.2rem;
}
```

**方案二：vw/vh 适配**

```css
/* 100vw = 设备宽度 */
.box {
  width: 20vw;  /* 20% 设备宽度 */
  height: 10vh;
  padding: 2vw;
}
```

**方案三：媒体查询**

```css
/* 小屏幕 */
.container {
  width: 100%;
}

/* 中等屏幕 */
@media (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* 大屏幕 */
@media (min-width: 1200px) {
  .container {
    width: 1170px;
  }
}
```

**方案对比**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| rem | 适配简单 | 需要 JS 动态计算 |
| vw/vh | 纯 CSS | 兼容性稍差 |
| 百分比 | 简单 | 不能解决所有问题 |
| 媒体查询 | 精确控制 | 代码量大 |

### 2.2 原型链与继承

**面试官问**：谈谈你对原型和原型链的理解。

**核心概念**：

```javascript
// 每个对象都有 __proto__ 属性
const obj = {};
obj.__proto__ === Object.prototype  // true

// 每个函数都有 prototype 属性
function Person() {}
Person.prototype.constructor === Person  // true

// 原型链
obj.__proto__ === Object.prototype
Object.prototype.__proto__ === null
```

**原型链图示**：
```
obj -> Object.prototype -> null
arr -> Array.prototype -> Object.prototype -> null
func -> Function.prototype -> Object.prototype -> null
```

**继承实现**：

```javascript
// 1. 原型链继承
function Parent() {
  this.name = 'parent';
}
function Child() {}
Child.prototype = new Parent();

// 2. 构造函数继承
function Parent(name) {
  this.name = name;
}
function Child(name) {
  Parent.call(this, name);
}

// 3. 组合继承
function Parent(name) {
  this.name = name;
}
Parent.prototype.say = function() {};
function Child(name) {
  Parent.call(this, name);
}
Child.prototype = new Parent();
Child.prototype.constructor = Child;

// 4. ES6 class 继承
class Parent {
  constructor(name) {
    this.name = name;
  }
  say() {}
}
class Child extends Parent {
  constructor(name) {
    super(name);
  }
}
```

### 2.3 闭包

**面试官问**：闭包是什么？它的应用场景有哪些？

**定义**：闭包是指能够访问自由变量的函数。

```javascript
function outer() {
  const a = 1;
  function inner() {
    console.log(a);  // inner 访问了 outer 的变量 a
  }
  return inner;
}
const fn = outer();
fn();  // 输出 1
```

**三大作用**：

```javascript
// 1. 封装私有变量
function createCounter() {
  let count = 0;
  return {
    getCount() { return count; },
    increment() { count++; },
    decrement() { count--; }
  };
}

// 2. 函数柯里化
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    }
    return function(...args2) {
      return curried.apply(this, args.concat(args2));
    };
  };
}

// 3. 模拟块级作用域
for (var i = 0; i < 3; i++) {
  (function(j) {
    setTimeout(() => console.log(j), 100);
  })(i);  // 输出 0, 1, 2
}
```

**缺点**：可能导致内存泄漏

### 2.4 事件循环 Event Loop

**面试官问**：介绍一下 Event Loop 机制，区分一下宏任务和微任务。

**执行顺序**：
1. 执行同步代码
2. 执行所有微任务
3. 渲染页面
4. 执行下一个宏任务

**常见宏任务**：
- `setTimeout` / `setInterval`
- I/O 操作
- UI 渲染
- `requestAnimationFrame`

**常见微任务**：
- `Promise.then/catch/finally`
- `MutationObserver`
- `queueMicrotask`

**代码输出题**：

```javascript
console.log('1');  // 同步

setTimeout(() => {
  console.log('2');  // 宏任务
}, 0);

Promise.resolve().then(() => {
  console.log('3');  // 微任务
});

console.log('4');

// 输出顺序: 1 -> 4 -> 3 -> 2
```

**进阶题**：

```javascript
let wait = (ms) => {
  return new Promise(resolve => {
    setTimeout(() => {
      resolve();
      console.log(1);
    }, ms);
    console.log(0);
  });
};

wait(0).then(() => console.log(6));

Promise.resolve().then(() => console.log(2))
  .then(() => console.log(3));

Promise.resolve().then(() => console.log(4));

console.log(5);

// 输出: 0 -> 5 -> 2 -> 4 -> 3 -> 6 -> 1
```

### 2.5 this 指向

**面试官问**：this 的指向在不同场景下有什么不同？

**this 指向场景**：

```javascript
// 1. 普通函数调用
function fn() { console.log(this); }
fn();  // window/undefined (严格模式)

// 2. 对象方法调用
const obj = {
  name: 'obj',
  fn() { console.log(this.name); }
};
obj.fn();  // 'obj'

// 3. 构造函数调用
function Person(name) {
  this.name = name;
}
const p = new Person('Tom');
p.name;  // 'Tom'

// 4. call/apply/bind
function fn() { console.log(this.name); }
const obj = { name: 'obj' };
fn.call(obj);  // 'obj'

// 5. 箭头函数
const fn = () => console.log(this);
fn();  // 定义时的外层 this
```

**改变 this 指向**：

```javascript
function fn(a, b) {
  console.log(this.name, a, b);
}
const obj = { name: 'obj' };

// call: 立即调用，参数展开
fn.call(obj, 1, 2);

// apply: 立即调用，参数数组
fn.apply(obj, [1, 2]);

// bind: 返回新函数，不立即调用
const newFn = fn.bind(obj, 1, 2);
newFn();
```

### 2.6 Promise 与 async/await

**面试官问**：Promise 的原理是什么？

**Promise 三种状态**：
- `pending` - 进行中
- `fulfilled` - 已成功
- `rejected` - 已失败

```javascript
const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve('success'), 1000);
```

### 2.7 深拷贝与浅拷贝

**面试官问**：手写一个深拷贝函数。

**浅拷贝**：

```javascript
// 方法一：扩展运算符
const copy = { ...original };

// 方法二：Object.assign
const copy = Object.assign({}, original);
```

**深拷贝实现**：

```javascript
function deepClone(obj, hash = new WeakMap()) {
  // 基本类型直接返回
  if (obj === null || typeof obj !== 'object') {
    return obj;
  }

  // 防止循环引用
  if (hash.has(obj)) {
    return hash.get(obj);
  }

  // 区分数组和对象
  const cloneObj = Array.isArray(obj) ? [] : {};
  hash.set(obj, cloneObj);

  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      cloneObj[key] = deepClone(obj[key], hash);
    }
  }

  return cloneObj;
}

// 简化版（不考虑循环引用）
function deepCloneSimple(obj) {
  if (obj === null || typeof obj !== 'object') {
    return obj;
  }
  if (Array.isArray(obj)) {
    return obj.map(item => deepCloneSimple(item));
  }
  const clone = {};
  for (const key in obj) {
    clone[key] = deepCloneSimple(obj[key]);
  }
  return clone;
}
```

### 2.8 防抖与节流

**面试官问**：手写一个防抖函数。

**防抖 (Debounce)**：debounce 是在事件被触发 n 秒后再执行回调，如果在这 n 秒内又被触发，则重新计时。

```javascript
function debounce(fn, delay) {
  let timer = null;
  return function(...args) {
    if (timer) clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

// 使用
input.addEventListener('input', debounce((e) => {
  console.log(e.target.value);
}, 300));
```

**节流 (Throttle)**：throttle 是指连续触发事件但是在 n 秒中只执行一次函数。

```javascript
// 方法一：时间戳
function throttle(fn, delay) {
  let lastTime = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastTime >= delay) {
      fn.apply(this, args);
      lastTime = now;
    }
  };
}

// 方法二：定时器
function throttle(fn, delay) {
  let timer = null;
  return function(...args) {
    if (!timer) {
      timer = setTimeout(() => {
        fn.apply(this, args);
        timer = null;
      }, delay);
    }
  };
}

// 使用
window.addEventListener('scroll', throttle(() => {
  console.log('scroll');
}, 200));
```

**应用场景**：
- 防抖：搜索框输入、手机号验证
- 节流：滚动加载、鼠标移动

### 2.9 浮点数精度问题

**面试官问**：为什么 0.1 + 0.2 !== 0.3？

**原因**：JavaScript 使用 IEEE 754 双精度浮点数，0.1 和 0.2 在二进制下是无限循环小数。

```javascript
console.log(0.1 + 0.2); // 0.30000000000000004
console.log(0.1 + 0.2 === 0.3); // false
```

**解决方案**：

```javascript
// 1. 使用 Number.EPSILON
function isEqual(a, b) {
  return Math.abs(a - b) < Number.EPSILON;
}
console.log(isEqual(0.1 + 0.2, 0.3)); // true

// 2. 放大倍数
function add(a, b) {
  const factor = Math.pow(10, Math.max(precision(a), precision(b)));
  return (Math.round(a * factor) + Math.round(b * factor)) / factor;
}

// 3. 使用 toFixed
Number((0.1 + 0.2).toFixed(10)) === 0.3; // true
```

### 3.1 Vue 响应式原理

**面试官问**：Vue3 的响应式原理是什么？

**Vue2 Object.defineProperty**：

```javascript

  Object.defineProperty(obj, key, {
    enumerable: true,
    configurable: true,
    get() {
      return val;
    },
    set(newVal) {

        val = newVal;
        // 通知更新
      }
    }
  });
}
```

**Vue3 Proxy**：

```javascript
function reactive(obj) {
  return new Proxy(obj, {
    get(target, key, receiver) {
      const res = Reflect.get(target, key, receiver);
      // 收集依赖
      track(target, key);
      return res;
    },
    set(target, key, value, receiver) {
      const res = Reflect.set(target, key, value, receiver);
      // 触发更新
      trigger(target, key);
      return res;
    }
  });
}
```

**Vue3 优势**：
- Proxy 能监听对象属性的新增/删除
- 不需要递归遍历，性能更好
- 支持 Map、Set、WeakMap、WeakSet

## 五、浏览器原理

### 5.1 浏览器渲染过程

**面试官问**：从用户在浏览器输入 URL 到页面完整展现出来，整个过程是什么？

**完整过程**：

1. **URL 解析**
   - 检查 URL 合法性
   - 浏览器进程转发给网络进程

2. **DNS 解析**
   - 递归查询：浏览器 → 本地 DNS → 根 DNS → 顶级 DNS → 权威 DNS

3. **建立连接**
   - TCP 三次握手
   - TLS 握手（HTTPS）

4. **发送请求**
   - 浏览器构造 HTTP 请求
   - 网络进程发送请求

5. **响应处理**
   - 服务器返回响应
   - 检查缓存

6. **渲染过程**
   - 浏览器主进程发送 HTML 数据给渲染进程
   - 渲染进程解析 HTML → DOM 树
   - 解析 CSS → CSSOM 树
   - 合并 DOM + CSSOM → 渲染树
   - 布局（Layout）
   - 绘制（Paint）
   - 合成（Composite）

```javascript
  const [scrollTop, setScrollTop] = useState(0);
  const containerHeight = 400;

  const startIndex = Math.floor(scrollTop / itemHeight);
  const endIndex = Math.min(
    items.length,
    Math.ceil((scrollTop + containerHeight) / itemHeight) + 10
  );

          {visibleItems.map((item, i) => (
            <div key={startIndex + i} style={{ height: itemHeight }}>
              {item.content}
            </div>
          ))}
        </div>
      </div>
    </div>
  );
```

### 9.2 手写 Promise

```javascript
class MyPromise {
  constructor(executor) {
    this.state = 'pending';
    this.value = undefined;
    this.onFulfilledCallbacks = [];
    this.onRejectedCallbacks = [];

    const resolve = (value) => {
      if (this.state !== 'pending') return;
      if (value instanceof MyPromise) {
        value.then(resolve, reject);
        return;
      }
      this.state = 'fulfilled';
      this.value = value;
      this.onFulfilledCallbacks.forEach(cb => cb());
    };

    const reject = (reason) => {
      if (this.state !== 'pending') return;
      this.state = 'rejected';
      this.value = reason;
      this.onRejectedCallbacks.forEach(cb => cb());
    };

    try {
      executor(resolve, reject);
    } catch (err) {
      reject(err);
    }
  }

  then(onFulfilled, onRejected) {
    return new MyPromise((resolve, reject) => {
      const fulfilledMicrotask = () => {
        try {
          const result = onFulfilled(this.value);
          resolve(result);
        } catch (err) {
          reject(err);
        }
      };

      const rejectedMicrotask = () => {
        try {
          const result = onRejected(this.value);
          resolve(result);
        } catch (err) {
          reject(err);
        }
      };

      if (this.state === 'fulfilled') {
        queueMicrotask(fulfilledMicrotask);
      } else if (this.state === 'rejected') {
        queueMicrotask(rejectedMicrotask);
      } else {
        this.onFulfilledCallbacks.push(fulfilledMicrotask);
        this.onRejectedCallbacks.push(rejectedMicrotask);
      }
    });
  }

  catch(onRejected) {
    return this.then(null, onRejected);
  }

  finally(onFinally) {
    return this.then(
      value => MyPromise.resolve(onFinally()).then(() => value),
      reason => MyPromise.resolve(onFinally()).then(() => { throw reason; })
    );
  }

  static resolve(value) {
    return new MyPromise(resolve => resolve(value));
  }

  static reject(reason) {
    return new MyPromise((_, reject) => reject(reason));
  }

  static all(promises) {
    return new MyPromise((resolve, reject) => {
      const results = [];
      let count = 0;

      promises.forEach((p, i) => {
        MyPromise.resolve(p).then(
          value => {
            results[i] = value;
            count++;
            if (count === promises.length) {
              resolve(results);
            }
          },
          reject
        );
      });
    });
  }

  static race(promises) {
    return new MyPromise((resolve, reject) => {
      promises.forEach(p => {
        MyPromise.resolve(p).then(resolve, reject);
      });
    });
  }
}
```

#### 13.3 async/await 实现原理

**面试官问**：async/await 原理是什么？

```javascript
// async/await 是 Promise 的语法糖
// 编译后会转换为 Generator + 自动执行器

// 手写 async 函数
async function fn(args) {}

// 等价于
function fn(args) {
  return spawn(function*() {
    // 函数体被转换为 yield
  });
}

// 自动执行器
function spawn(genF) {
  return new Promise((resolve, reject) => {
    const gen = genF();

    function step(nextF) {
      let next;
      try {
        next = nextF();
      } catch (e) {
        return reject(e);
      }

      if (next.done) {
        return resolve(next.value);
      }

      Promise.resolve(next.value).then(
        v => step(() => gen.next(v)),
        e => step(() => gen.throw(e))
      );
    }

    step(() => gen.next(undefined));
  });
}
```

#### 13.4 Proxy 深度监听

```javascript
function reactive(obj) {

  const proxy = new Proxy(obj, {
    get(target, key) {
      const value = target[key];
      return reactive(value);
    },
    set(target, key, value) {
      console.log('set:', key, value);
      target[key] = value;
    }
  });

  return proxy;
}

deepProxy.a.b = 2; // 会触发 set: b 2
```

#### 13.14 归并排序

```javascript
function mergeSort(arr) {
  if (arr.length <= 1) return arr;

  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));

  return merge(left, right);
}

function merge(left, right) {
  const result = [];
  let i = 0, j = 0;

  while (i < left.length && j < right.length) {
    if (left[i] <= right[j]) {
      result.push(left[i++]);
    } else {
      result.push(right[j++]);
    }
  }

  return result.concat(left.slice(i)).concat(right.slice(j));
}
```

```javascript
function lengthOfLongestSubstring(s) {
  const set = new Set();
  let left = 0, maxLen = 0;

  for (let right = 0; right < s.length; right++) {
    while (set.has(s[right])) {
      set.delete(s[left]);
      left++;
    }
    set.add(s[right]);

}
```

#### 13.19 二叉树遍历

```javascript
// 前序遍历
function preorder(root) {
  if (!root) return [];
  return [root.val, ...preorder(root.left), ...preorder(root.right)];
}

// 中序遍历
function inorder(root) {
  if (!root) return [];
  return [...inorder(root.left), root.val, ...inorder(root.right)];
}

// 后序遍历
function postorder(root) {
  if (!root) return [];
  return [...postorder(root.left), ...postorder(root.right), root.val];
}

// 层序遍历
function levelOrder(root) {
  if (!root) return [];
  const result = [];
  const queue = [root];

  while (queue.length) {
    const level = [];
    const size = queue.length;

    for (let i = 0; i < size; i++) {
      const node = queue.shift();
      level.push(node.val);
      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }

    result.push(level);
  }

  return result;
}
```

#### 13.22 LRU 缓存

```javascript
class LRUCache {
  constructor(capacity) {
    this.capacity = capacity;
    this.cache = new Map();
  }

  get(key) {
    if (!this.cache.has(key)) return -1;

    const value = this.cache.get(key);
    this.cache.delete(key);
    this.cache.set(key, value);
    return value;
  }

  put(key, value) {
    if (this.cache.has(key)) {
      this.cache.delete(key);
    } else if (this.cache.size >= this.capacity) {
      const firstKey = this.cache.keys().next().value;
      this.cache.delete(firstKey);
    }
    this.cache.set(key, value);
  }
}
```
