## 一、项目概述与架构设计

### 1.1 WebEnv-OS 桌面系统

**Q1: 请详细描述 WebEnv-OS 桌面系统的整体架构设计你是如何划分模块的？**

WebEnv-OS 是一个基于 Web 技术构建的类桌面操作系统环境。整体架构采用分层设计：

- **核心层（Core Layer）**：处理窗口管理、进程调度、文件系统抽象
- **UI 层（Presentation Layer）**：React 组件树，负责视图渲染
- **状态管理层（State Layer）**：Zustand 全局状态管理
- **通信层（IPC Layer）**：组件间通信机制

窗口管理系统采用“窗口池”模式，每个窗口实例独立管理自己的状态，通过 Context 进行跨窗口通信。

**Q2: WebEnv-OS 与传统桌面应用（如 Electron）的核心区别是什么？**

| 特性 | WebEnv-OS | Electron |
|------|------------|----------|
| 渲染引擎 | 浏览器内核 |  Chromium + Node.js |
| 系统访问 | 受限于浏览器沙箱 | 完整系统权限 |
| 包体积 | 依赖浏览器 | 体积较大 |
| 安全性 | 浏览器安全模型 | 需自行处理安全 |

**Q3: 在 WebEnv-OS 中，你如何实现多窗口管理与窗口通信？**

采用发布-订阅模式结合 Zustand：
- 每个窗口拥有唯一 ID
- 通过 `window.postMessage` 进行跨窗口消息传递
- Zustand store 中维护窗口映射表
- 使用 `BroadcastChannel API` 实现同源窗口通信

## 二、React 核心原理深度

### 2.1 Virtual DOM 与 Diff 算法

**Q4: React 的 Virtual DOM 相比直接操作真实 DOM 有何优势？**

Virtual DOM 的核心优势：

1. **减少重排重绘**：批量更新操作，减少浏览器回流
2. **跨平台渲染**：同一套 Virtual DOM 可渲染到不同平台（Web、Native、SSR）
3. **抽象层解耦**：开发者无需关心平台差异
4. **增量更新**：支持时间切片、并发模式

**Q5: React 的 Diff 算法时间复杂度是多少？请详细解释其核心逻辑**

React Diff 算法复杂度为 **O(n)**，核心策略：

1. **Tree Diff**：只比较同级节点，使用 key 匹配
2. **Component Diff**：相同类型组件继续 Diff，不同类型直接替换
3. **Element Diff**：
   - 相同类型：更新属性
   - 不同类型：删除旧节点、创建新节点

关键优化：
- 设置 key 帮助识别移动节点
- 避免将节点从深层移动到浅层
- 使用 `shouldComponentUpdate` 跳过无需更新的子树

### 2.2 React Hooks 原理

**Q6: useState 的实现原理是什么？请手动实现一个简易版**

```javascript
// 简易 useState 实现
let state;
let queue = [];

function useState(initialValue) {
  if (!state) {
    state = initialValue;
  }
  
  const setState = (newValue) => {
    state = typeof newValue === 'function' 
      ? newValue(state) 
      : newValue;
    // 触发重新渲染
    render();
  };
  
  return [state, setState];
}
```

React 实际实现使用链表存储 hooks，通过 `currentHook` 指针追踪当前执行的 hook。

**Q7: 为什么 useEffect 的依赖数组很重要？错误配置会导致什么问题？**

依赖数组问题会导致：

1. **内存泄漏**：清理函数未正确执行，订阅/定时器未清除
2. **无限循环**：依赖项为对象/数组时，每次渲染创建新引用导致死循环
3. **陈旧闭包**：遗漏依赖导致读取过期值
4. **性能问题**：不必要的重新执行

```javascript
// 错误示例
useEffect(() => {
  fetchData(data); // data 为对象引用
}, [data]); // 每次渲染都触发

// 正确示例
useEffect(() => {
  fetchData(data.id); // 使用稳定值
}, [data.id]);
```

### 2.3 React Fiber 架构

**Q8: 请详细解释 React Fiber 的工作原理及其解决的问题**

Fiber 是 React 16 引入的新协调引擎：

**解决的问题**：
1. **不可中断**：旧版本一旦开始协调，无法中断
2. **优先级不一致**：所有任务同等优先级
3. **同步阻塞**：大组件树会导致卡顿

**Fiber 核心概念**：
- **Fiber 节点**：每个 React 元素对应一个 Fiber 节点
- **双缓冲**：构建工作循环，commit 阶段一次性更新
- **优先级调度**：Lane 模型支持细粒度优先级
- **可中断**：每个工作单元完成后可让出主线程

### 3.1 原理与实现

**Q9: Zustand 与 Redux 的核心区别是什么？为何选择 Zustand？**

**Q10: Zustand 如何实现精准订阅与更新？**

```javascript
// Zustand 核心实现原理
function create(set) {
  const state = {};
  const listeners = new Set();
  
  return {
    getState: () => state,
    setState: (partial) => {
      const nextState = typeof partial === 'function' 
        ? partial(state) 
        : partial;
      Object.assign(state, nextState);
      listeners.forEach(listener => listener(state));
    },
    subscribe: (listener) => {
      listeners.add(listener);
      return () => listeners.delete(listener);
    }
  };
}
```

```javascript
const getDisplayMedia = () => {
  // 优先使用标准 API
  if (navigator.mediaDevices?.getDisplayMedia) {

      video: { cursor: 'always' },
      audio: false

  }
  
  // 兼容旧版 Safari
  return navigator.getDisplayMedia({ video: true });
```

## 十、前后端认证与安全

### 10.1 认证机制对比

**Q25: JWT 与 Session 的核心区别是什么？各自的优缺点？**

| 特性 | JWT | Session |
|------|-----|---------|
| 存储位置 | 客户端 | 服务器端 |
| 状态管理 | 无状态 | 有状态 |
| 跨域支持 | 天然支持 | 需要特殊处理 |

**JWT 劣势**：
- token 泄露风险
- 无法主动失效
- token 体积较大

### 10.2 OAuth 2.0

**Q26: OAuth 2.0 的授权流程是什么？请详细解释四种授权模式**

授权流程（授权码模式）：

### 10.3 前端安全

**Q27: XSS 攻击的原理是什么？如何在前端防御？**

XSS（Cross-Site Scripting）原理：
- 注入恶意脚本到页面
- 窃取用户 Cookie、Token
- 伪造用户操作

防御措施：
1. **内容转义**：对用户输入进行 HTML 转义
2. **CSP（Content Security Policy）**：限制脚本来源
3. **HttpOnly Cookie**：禁止 JS 读取敏感 Cookie
4. **输入验证**：白名单过滤

```javascript
// 转义函数
const escapeHtml = (str) => {
  const map = {
    '&': '&amp;',
    '<': '&lt;',
    '>': '&gt;',
    '"': '&quot;',
    "'": '&#039;'
  };
  return str.replace(/[&<>"']/g, char => map[char]);
};
```

**Q28: CSRF 攻击的原理是什么？防御手段有哪些？**

CSRF（Cross-Site Request Forgery）原理：
- 用户登录受信任网站
- 诱导访问恶意网站
- 恶意网站利用用户身份发起请求

防御手段：
1. **CSRF Token**：服务端生成 token，前端随请求发送
2. **SameSite Cookie**：设置 `SameSite=Strict/Lax`
3. **验证 Referer**：检查请求来源
4. **双重提交 Cookie**：对比 Cookie 与表单 token

### 11.1 浏览器渲染原理

**Q29: 浏览器渲染流程是什么？哪些环节可以优化？**

1. **HTML 解析** → DOM 树

- 减少 DOM 层级
- 使用 `transform` 而非 `left/top` 动画
- 避免强制同步布局
- 使用 `will-change` 提示浏览器

### 25.1 案例一：WebEnv-OS 项目

**常见问题回答**：

   > WebEnv-OS 是一个基于 Web 技术构建的类桌面操作系统环境。我负责了窗口管理、任务栏、文件管理器等核心模块的开发。项目采用 React + TypeScript + Zustand 技术栈，实现了多窗口管理、主题切换、文件系统抽象等功能。

**项目介绍**：

   > 使用 Y.js 作为协作引擎，通过 CRDT 算法解决冲突问题。每个用户的编辑操作都会生成增量更新，通过 WebSocket 同步到其他用户。光标同步使用 Y.js 的 Awareness 协议。

3. **Q: 编辑器是如何选型的**
   > 对比了 Quill、Slate、TipTap 等编辑器。TipTap 基于 ProseMirror，扩展性强，支持协作编辑，社区活跃，因此选择了 TipTap。

## 二十六、简历项目技术细节

### 26.1 React 相关

**Q: React 虚拟 DOM 的原理是什么**
> 虚拟 DOM 是真实 DOM 的 JavaScript 对象表示。React 通过比较新旧虚拟 DOM 的差异来确定最小更新范围。Diff 算法采用同层比较策略，时间复杂度为 O(n)。

**Q: useEffect 的执行时机**
> 不带依赖数组：每次渲染后执行；带空数组：只在挂载时执行一次；带依赖数组：依赖变化时执行。清理函数在组件卸载和下一次 effect 执行前执行。

### 26.2 状态管理相关

**Q: Zustand 与 Redux 的区别**
> Zustand 更加轻量，API 简洁，不需要模板代码。内部使用 Proxy 实现响应式，自动订阅优化。Redux 需要 action、reducer、middleware，样板代码较多，但生态更丰富。

**Q: 如何选择状态管理方案**
> 本地状态用 useState；跨级共享用 Context；中型应用用 Zustand；大型应用用 Redux。需要根据项目规模和团队情况选择。

## 二十七、简历项目网络相关

### 27.1 HTTP 相关

**Q: HTTP 缓存机制**
> 强缓存使用 Cache-Control 和 Expires，不请求服务器直接使用缓存；协商缓存使用 ETag 和 Last-Modified，需要请求服务器确认是否使用缓存。

**Q: HTTPS 握手过程**
> 1. Client Hello：客户端发送支持的加密算法
> 2. Server Hello + 证书：服务端选择算法并发送证书
> 3. 验证证书：客户端验证证书有效性
> 4. 密钥交换：使用公钥加密对称密钥
> 5. 加密通信：使用对称密钥加密数据传输

### 27.2 WebSocket 相关

**Q: WebSocket 与 HTTP 的区别**
> HTTP 是半双工通信，只能客户端发起请求；WebSocket 是全双工通信，服务器可以主动推送。WebSocket 建立连接后不需要每次请求都带 Headers。

**Q: 断线重连如何处理**
> 使用心跳检测连接状态；断线后使用指数退避算法重连；离线消息本地缓存，连接恢复后发送。

## 四十一、简历项目优化案例

### 41.1 首屏优化案例

**问题描述**：
首屏加载时间 3s，用户反馈慢

**原因分析**：
1. Bundle 体积过大
2. 图片未优化
3. 重复请求
4. 缓存策略不佳

**优化措施**：
1. 代码分割 + Tree Shaking
2. 图片压缩 + WebP
3. 请求合并 + 缓存
4. CDN 加速 + 缓存策略

**优化结果**：
首屏加载时间降到 1s 以内

### 41.2 渲染优化案例

**问题描述**：
列表滚动卡顿，帧率低

**原因分析**：
1. 大量 DOM 渲染
2. 不必要的重渲染
3. 复杂计算
4. 内存泄漏

**优化措施**：
1. 虚拟列表
2. React.memo + useMemo
3. Web Worker
4. 清理定时器

**优化结果**：
帧率从 20fps 提升到 60fps

### 41.3 内存优化案例

**问题描述**：
长时间使用页面变卡

**原因分析**：
1. 事件监听未移除
2. 定时器未清理
3. 闭包引用
4. 全局变量

**优化措施**：
1. useEffect 清理
2. clearInterval/clearTimeout
3. WeakMap/WeakSet
4. 减少全局变量

**优化结果**：
内存使用稳定，无泄漏

## 五十七、JavaScript 核心面试题详解

### 57.1 数据类型与判断

#### JavaScript 有哪些数据类型？

JavaScript 共有 8 种数据类型：

**原始类型（7种）**：
1. **undefined**：未定义
2. **null**：空对象
3. **Boolean**：布尔值
4. **Number**：数字（包含整数和浮点数）
5. **String**：字符串
6. **Symbol**：符号（ES6新增，唯一标识符）
7. **BigInt**：大整数（ES2020新增）

**引用类型（1种）**：
- **Object**：对象（包含 Array、Function、Date、RegExp 等）

#### typeof 的结果有哪些？

```javascript
typeof undefined     // "undefined"
typeof null          // "object" - 历史遗留bug
typeof 123          // "number"
typeof 'str'        // "string"
typeof true         // "boolean"
typeof Symbol()     // "symbol"
typeof 123n        // "bigint"
typeof {}           // "object"
typeof []           // "object"
typeof function(){} // "function"
```

#### 如何准确判断数据类型？

**Object.prototype.toString.call()**：
```javascript
Object.prototype.toString.call([])    // "[object Array]"
Object.prototype.toString.call({})    // "[object Object]"
Object.prototype.toString.call(null)  // "[object Null]"
```

**Array.isArray()**：
```javascript
Array.isArray([])    // true
Array.isArray({})    // false
```

#### null vs undefined 的区别？

| 特性 | null | undefined |
|------|------|-----------|
| 含义 | 空对象，主动设置 | 未定义，自动生成 |
| typeof | "object" | "undefined" |
| 转换为数字 | 0 | NaN |
| 场景 | 主动清空对象 | 变量未赋值、函数无返回值 |

### 57.2 原型与继承

#### 什么是原型链？

每个 JavaScript 对象都有一个内部属性 `[[Prototype]]`（浏览器中通过 `__proto__` 访问），指向它的原型对象。当访问对象的属性时，如果对象本身没有该属性，JavaScript 会沿着原型链向上查找，直到找到或到达原型链顶端 `null`。

```
实例 --__proto__--> 原型 --__proto__--> Object.prototype --__proto__--> null
```

#### 原型链顶端是什么？

`Object.prototype.__proto__ === null`

#### 如何实现继承？

**1. 原型链继承**：
```javascript
function Parent() {
  this.name = 'parent';
}
Parent.prototype.say = function() { console.log('hello'); };

function Child() {}
Child.prototype = new Parent(); // 建立原型链
Child.prototype.constructor = Child;

const child = new Child();
child.say(); // "hello" - 继承自 Parent
```

**2. 构造函数继承（call/apply）**：
```javascript
function Parent(name) {
  this.name = name;
}
function Child(name, age) {
  Parent.call(this, name); // 借用父类构造函数
  this.age = age;
}

const child = new Child('Tom', 20);
console.log(child.name); // "Tom"
```

**3. 组合继承（最常用）**：
```javascript
function Parent(name) {
  this.name = name;
}
Parent.prototype.say = function() { console.log(this.name); };

function Child(name, age) {
  Parent.call(this, name); // 构造函数继承
  this.age = age;
}
Child.prototype = new Parent(); // 原型链继承
Child.prototype.constructor = Child;
```

**4. ES6 class 继承**：
```javascript
class Parent {
  constructor(name) { this.name = name; }
  say() { console.log(this.name); }
}

class Child extends Parent {
  constructor(name, age) {
    super(name);
    this.age = age;
  }
}
```

### 57.3 闭包

#### 什么是闭包？

闭包是指有权访问另一个函数作用域中变量的函数。简单来说，就是一个函数能够访问并操作外部函数中的变量。

```javascript
function outer() {
  const x = 10;
  function inner() {
    console.log(x); // 访问外部函数的变量
  }
  return inner;
}

const fn = outer();
fn(); // 10 - 即使 outer 已经执行完毕，inner 仍然可以访问 x
```

#### 闭包的应用场景

1. **私有化数据**：
```javascript
function createCounter() {
  let count = 0;
  return {
    increment() { return ++count; },
    decrement() { return --count; },
    getCount() { return count; }
  };
}

const counter = createCounter();
counter.increment(); // 1
counter.increment(); // 2
console.log(counter.getCount()); // 2
```

2. **函数工厂**：
```javascript
function multiply(factor) {
  return function(number) {
    return number * factor;
  };
}

const double = multiply(2);
console.log(double(5)); // 10
```

#### 闭包会导致内存泄漏吗？

会！常见原因：

1. **循环引用**：两个对象相互引用，形成闭包
2. **未清理的定时器**：setTimeout/setInterval 引用外部变量
3. **DOM 引用**：保存 DOM 元素引用

```javascript
// 内存泄漏示例
function leak() {
  const largeData = new Array(1000000).fill('x');
  const timer = setInterval(() => {
    console.log(largeData[0]);
  }, 1000);
  // leak 函数执行完毕，但 timer 仍在运行，largeData 不会被回收
}
```

### 57.4 事件循环

#### 什么是事件循环？

事件循环是 JavaScript 执行异步代码的机制，它负责协调宏任务和微任务的执行顺序。

**执行顺序**：
1. 执行主线程上的同步代码
2. 清空微任务队列（Promise 回调等）
3. 尝试执行 DOM 渲染
4. 从宏任务队列取出一个任务执行
5. 重复步骤 2-4

#### 宏任务 vs 微任务

**宏任务（Macrotask）**：
- setTimeout / setInterval
- I/O 操作
- UI 渲染
- setImmediate（Node.js）

**微任务（Microtask）**：
- Promise.then / catch / finally
- MutationObserver
- queueMicrotask
- process.nextTick（Node.js）

#### async/await 原理

async 函数返回 Promise，await 等待 Promise resolve。await 后的代码相当于 Promise.then 的回调。

```javascript
async function example() {
  console.log(1);
  await Promise.resolve();
  console.log(2); // 相当于 Promise.then 的回调，在微任务中执行
  console.log(3);
}

example();
console.log(4);
// 输出顺序：1, 4, 2, 3
```

### 57.5 this 指向

#### this 的指向规则

| 场景 | this 指向 |
|------|----------|
| 普通函数调用 | window / undefined（严格模式） |
| 对象方法调用 | 调用方法的对象 |
| 构造函数调用 | 实例对象 |
| call/apply/bind | 第一个参数 |
| 箭头函数 | 定义时的外层 this |

#### 箭头函数的 this

箭头函数没有自己的 this，它会捕获定义时外层函数的 this。

```javascript
const obj = {
  name: 'obj',
  say() {
    setTimeout(() => {
      console.log(this.name); // "obj" - 箭头函数的 this 继承外层 say 的 this
    }, 100);
  }
};
```

### 57.6 深拷贝实现

#### 浅拷贝 vs 深拷贝

**浅拷贝**：只复制对象的第一层属性
```javascript
const shallow = { ...obj };
const shallow2 = Object.assign({}, obj);
```

**深拷贝**：递归复制所有层级
```javascript
function deepClone(obj, hash = new WeakMap()) {
  // 处理基本类型
  if (obj === null || typeof obj !== 'object') return obj;

  // 处理循环引用
  if (hash.has(obj)) return hash.get(obj);

  // 创建新对象或数组
  const clone = Array.isArray(obj) ? [] : {};
  hash.set(obj, clone);

  // 递归复制
  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      clone[key] = deepClone(obj[key], hash);
    }
  }

  return clone;
}
```

### 57.7 防抖与节流

#### 防抖（Debounce）

函数执行后，在指定时间内不再触发才执行。适用于搜索框输入。

```javascript
function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}
```

#### 节流（Throttle）

函数执行后，在指定时间内不再执行。适用于滚动事件、点击事件。

```javascript
function throttle(fn, delay) {
  let last = 0;
  return function(...args) {
    const now = Date.now();
    if (now - last >= delay) {
      last = now;
      fn.apply(this, args);
    }
  };
}
```

### 58.1 盒模型

#### 什么是盒模型？

**两种盒模型**：

1. **content-box（默认）**：

   - 实际占用 = width + padding + border

2. **border-box**：
   - width = 内容 + padding + border

```css
/* 默认 */
```

### 58.2 BFC（块级格式化上下文）

#### 什么是 BFC？

BFC 是 CSS 布局中的一个独立渲染区域，内部的元素不会影响外部的元素。

- position: absolute / fixed
- display: inline-block / flex / grid
- float: 不为 none

**BFC 的应用**：
1. 防止 margin 折叠
2. 清除浮动
3. 自适应两栏布局

```css
/* 防止 margin 折叠 */
.parent {

.child {
```

### 58.7 重排与重绘

#### 什么是重排（Reflow）？

当 DOM 元素的位置、尺寸等几何属性发生变化时，浏览器需要重新计算布局。

**触发重排的属性**：
- width, height, padding, margin
- display
- position
- top, right, bottom, left

#### 什么是重绘（Repaint）？

当元素的外观发生变化但不影响布局时，浏览器只需要重新绘制。

**触发重绘的属性**：
- color, background-color
- border-style, border-radius
- visibility
- box-shadow

#### 性能优化

1. **使用 transform 代替 top/left**：
```css
/* 触发重排 */
.element { top: 100px; }

/* 触发合成 */
.element { transform: translateY(100px); }
```

2. **使用 will-change 提示浏览器**：
```css
.element {
  will-change: transform;
}
```

```javascript
});

useEffect(() => {
  const timer = setInterval(() => {}, 1000);
  return () => clearInterval(timer); // 清理
}, []);
```

### 60.1 响应式原理

#### Vue 2 响应式

```javascript
// 核心：Object.defineProperty
Object.defineProperty(obj, 'name', {
  get() {
    console.log('get');
    return value;
  },
  set(newValue) {
    console.log('set');
    value = newValue;
  }
});
```

**问题**：
- 不能检测数组下标变化
- 不能检测对象属性添加

**解决方案**：
- 重写数组方法
- Vue.set

#### Vue 3 响应式

```javascript
// 核心：Proxy
const handler = {
  get(target, key, receiver) {
    console.log('get:', key);
    return Reflect.get(target, key, receiver);
  },
  set(target, key, value, receiver) {
    console.log('set:', key, value);
    return Reflect.set(target, key, value, receiver);
  }
};

const obj = new Proxy(target, handler);
```

### 60.2 生命周期对比

| Vue 2 | Vue 3 | 说明 |
|-------|-------|------|
| beforeCreate | setup | 实例创建前 |
| created | setup | 实例创建后 |
| beforeMount | onBeforeMount | 挂载前 |
| mounted | onMounted | 挂载后 |
| beforeUpdate | onBeforeUpdate | 更新前 |
| updated | onUpdated | 更新后 |
| beforeDestroy | onBeforeUnmount | 销毁前 |
| destroyed | onUnmounted | 销毁后 |

### 60.3 computed vs watch

- **computed**：计算属性，缓存结果，只有依赖变化才重新计算
- **watch**：监听数据变化，执行副作用

### 61.1 HTTP 状态码

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 204 | 无内容 |
| 301 | 永久重定向 |
| 302 | 临时重定向 |
| 304 | 缓存 |
| 400 | 请求错误 |
| 401 | 未授权 |
| 403 | 禁止 |
| 404 | 未找到 |
| 500 | 服务器错误 |
| 502 | 网关错误 |
| 504 | 超时 |

### 61.2 HTTP vs HTTPS

- HTTP：端口 80，明文传输
- HTTPS：端口 443，SSL/TLS 加密

**HTTPS 握手流程**：
1. Client Hello：客户端发送支持的加密算法
2. Server Hello + 证书：服务端选择算法并发送证书
3. 验证证书：客户端验证证书有效性
4. 密钥交换：使用公钥加密对称密钥
5. 加密通信：使用对称密钥加密数据传输

### 61.3 TCP 三次握手

```
客户端 -> SYN -> 服务器
服务器 -> SYN + ACK -> 客户端
客户端 -> ACK -> 服务器
```

**目的**：建立可靠连接，同步双方序列号

### 61.4 TCP 四次挥手

```
客户端 -> FIN -> 服务器
服务器 -> ACK -> 客户端
服务器 -> FIN -> 客户端
客户端 -> ACK -> 服务器
```

**目的**：关闭连接，确保数据完整传输

### 61.5 缓存策略

| 策略 | 原理 | 字段 |
|------|------|------|
| 强缓存 | 不请求直接用 | Cache-Control, Expires |
| 协商缓存 | 询问服务器 | ETag, Last-Modified |

```javascript
class CountdownTimer {
  constructor() {
    // DOM 元素
    this.daysEl = document.getElementById('days');
    this.hoursEl = document.getElementById('hours');
    this.minutesEl = document.getElementById('minutes');
    this.secondsEl = document.getElementById('seconds');
    this.inputEl = document.getElementById('target-time');
    this.startBtn = document.getElementById('start-btn');
    this.resetBtn = document.getElementById('reset-btn');

    // 状态
    this.targetTime = null;
    this.intervalId = null;
    this.isRunning = false;

    // 绑定事件
    this.bindEvents();

    // 初始化默认时间（7天后）
    this.initDefaultTime();
  }

  bindEvents() {
    this.startBtn.addEventListener('click', () => this.toggle());
    this.resetBtn.addEventListener('click', () => this.reset());
  }

  initDefaultTime() {
    const now = new Date();
    const target = new Date(now.getTime() + 7 * 24 * 60 * 60 * 1000);
    this.inputEl.value = this.formatDateTimeLocal(target);
  }

  formatDateTimeLocal(date) {
    const year = date.getFullYear();
    const month = String(date.getMonth() + 1).padStart(2, '0');
    const day = String(date.getDate()).padStart(2, '0');
    const hours = String(date.getHours()).padStart(2, '0');
    const minutes = String(date.getMinutes()).padStart(2, '0');
    return `${year}-${month}-${day}T${hours}:${minutes}`;
  }

  toggle() {
    if (this.isRunning) {
      this.pause();
    } else {
      this.start();
    }
  }

  start() {
    const inputValue = this.inputEl.value;
    if (!inputValue) {
      alert('请选择倒计时目标时间');
      return;
    }

    this.targetTime = new Date(inputValue).getTime();
    this.isRunning = true;
    this.startBtn.textContent = '暂停';

    this.update();
    this.intervalId = setInterval(() => this.update(), 1000);
  }

  pause() {
    this.isRunning = false;
    this.startBtn.textContent = '继续';
    clearInterval(this.intervalId);
  }

  reset() {
    this.pause();
    this.targetTime = null;
    this.clearDisplay();
    this.startBtn.textContent = '开始';
    this.inputEl.value = '';
    this.initDefaultTime();
  }

  update() {
    const now = Date.now();
    const diff = this.targetTime - now;

    if (diff <= 0) {
      this.complete();
      return;
    }

    const { days, hours, minutes, seconds } = this.calculateTime(diff);
    this.display(days, hours, minutes, seconds);
  }

  calculateTime(ms) {
    const seconds = Math.floor((ms / 1000) % 60);
    const minutes = Math.floor((ms / (1000 * 60)) % 60);
    const hours = Math.floor((ms / (1000 * 60 * 60)) % 24);
    const days = Math.floor(ms / (1000 * 60 * 60 * 24));

    return { days, hours, minutes, seconds };
  }

  display(days, hours, minutes, seconds) {
    this.daysEl.textContent = String(days).padStart(2, '0');
    this.hoursEl.textContent = String(hours).padStart(2, '0');
    this.minutesEl.textContent = String(minutes).padStart(2, '0');
    this.secondsEl.textContent = String(seconds).padStart(2, '0');
  }

  clearDisplay() {
    this.display(0, 0, 0, 0);
  }

  complete() {
    this.pause();
    this.clearDisplay();
    this.startBtn.textContent = '开始';
    alert('倒计时结束！');
  }
}

});
```

```tsx
  const hours = Math.floor((diff / (1000 * 60 * 60)) % 24);
  const days = Math.floor(diff / (1000 * 60 * 60 * 24));

  return { days, hours, minutes, seconds };
}

function TimeBlock({ value, label }: { value: number; label: string }) {
  return (
    <div className="flex flex-col items-center">
      <div className="text-4xl font-bold text-blue-600 bg-gray-100 p-4 rounded-lg min-w-[70px] text-center">
        {String(value).padStart(2, '0')}
      </div>
      <div className="mt-2 text-gray-600 text-sm">{label}</div>
    </div>
  );
}

export default function CountdownAntd() {
  const [targetTime, setTargetTime] = useState<string>('');
  const [isRunning, setIsRunning] = useState(false);
  const [timeLeft, setTimeLeft] = useState<TimeLeft>({
    days: 0,
    hours: 0,
    minutes: 0,
    seconds: 0,
  });

  // 初始化默认时间
  useEffect(() => {
    const now = new Date();
    const defaultTarget = new Date(now.getTime() + 7 * 24 * 60 * 60 * 1000);
    setTargetTime(defaultTarget.toISOString().slice(0, 16));
  }, []);

  // 倒计时逻辑
  useEffect(() => {
    if (!isRunning || !targetTime) return;

    const interval = setInterval(() => {
      setTimeLeft(calculateTimeLeft(targetTime));
    }, 1000);

    return () => clearInterval(interval);
  }, [isRunning, targetTime]);

  return (
    <ConfigProvider
      theme={{
        token: {
          colorPrimary: '#1677ff',
        },
      }}
    >
      <div className="min-h-screen flex items-center justify-center p-4">
        <Card className="max-w-xl w-full shadow-xl">
          <div className="text-center mb-8">
            <Title level={2} className="!mb-2">
              双十一倒计时
            </Title>
            <Text type="secondary">精准计时，实时更新</Text>
          </div>

          {/* 倒计时显示 */}
          <div className="flex justify-center items-center gap-2 mb-8">
            <TimeBlock value={timeLeft.days} label="天" />
            <span className="text-3xl text-blue-600 font-bold">:</span>
            <TimeBlock value={timeLeft.hours} label="时" />
            <span className="text-3xl text-blue-600 font-bold">:</span>
            <TimeBlock value={timeLeft.minutes} label="分" />
            <span className="text-3xl text-blue-600 font-bold">:</span>
            <TimeBlock value={timeLeft.seconds} label="秒" />
          </div>

          {/* 控件 */}
          <Space direction="vertical" className="w-full" size="middle">
            <DatePicker
              showTime
              value={targetTime ? dayjs(targetTime) : null}
              onChange={(date) => setTargetTime(date?.toISOString().slice(0, 16) || '')}
              className="w-full"
              disabled={isRunning}
              placeholder="选择目标时间"
            />

            <Space className="w-full justify-center">
              {!isRunning ? (
                <Button
                  type="primary"
                  size="large"
                  onClick={() => setIsRunning(true)}
                >
                  开始倒计时
                </Button>
              ) : (
                <Button
                  size="large"
                  onClick={() => setIsRunning(false)}
                >
                  暂停
                </Button>
              )}
              <Button
                size="large"
                onClick={() => {
                  setIsRunning(false);
                  const now = new Date();
                  const defaultTarget = new Date(now.getTime() + 7 * 24 * 60 * 60 * 1000);
                  setTargetTime(defaultTarget.toISOString().slice(0, 16));
                }}
              >
                重置
              </Button>
            </Space>
          </Space>
        </Card>
      </div>
    </ConfigProvider>
  );
}
```
