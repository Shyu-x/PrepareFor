### 1.1 z-index管理问题

**面试问题：请解释一下你的项目是如何管理z-index的？**

**参考答案：**

在多窗口系统中，z-index的管理是一个核心问题。我使用Zustand创建了一个全局store来统一管理所有窗口的z-index值。

**实现原理：**

1. **全局状态管理**：维护一个highestZIndex变量，记录当前最高的z-index值
2. **激活窗口**：当窗口被点击时，将其z-index设置为highestZIndex + 1
3. **状态更新**：使用Zustand的set方法更新状态，触发相关组件重新渲染

**代码实现：**

```javascript
// Zustand store 定义
import { create } from 'zustand';

const useWindowStore = create((set, get) => ({
    highestZIndex: 100,
    activeWindowId: null,

    // 将窗口置顶
    bringToFront: (windowId) => {
        set((state) => ({
            highestZIndex: state.highestZIndex + 1,
            activeWindowId: windowId
        }));
    },

    // 获取窗口的z-index
    getWindowZIndex: (windowId, currentZIndex) => {
        const state = get();
        // 激活窗口使用最高z-index
        if (state.activeWindowId === windowId) {
            return state.highestZIndex;
        }
        // 非激活窗口使用原始z-index
        return currentZIndex;
    }
}));

// 窗口组件中使用
function Window({ id, initialZIndex }) {
    const { bringToFront, activeWindowId } = useWindowStore();
    const isActive = activeWindowId === id;
    const zIndex = isActive ? useWindowStore(state => state.highestZIndex) : initialZIndex;

    return (
        <div
            className={`window ${isActive ? 'active' : ''}`}
            style={{ zIndex }}
            onMouseDown={() => bringToFront(id)}
        >
            <WindowHeader>标题栏</WindowHeader>
            <WindowContent>内容区域</WindowContent>
        </div>
    );
}
```

**优化建议：**

1. 使用requestAnimationFrame优化频繁的状态更新
2. 使用CSS transform代替top/left属性，利用GPU加速
3. 考虑使用虚拟列表优化大量窗口的场景

### 1.2 窗口拖拽实现

**面试问题：如何实现窗口的拖拽功能？**

**参考答案：**

窗口拖拽需要监听鼠标事件，计算鼠标移动的距离，并更新窗口的位置。

**实现原理：**

1. **mousedown**：记录鼠标初始位置和窗口初始位置
2. **mousemove**：计算鼠标偏移量，更新窗口位置
3. **mouseup**：移除事件监听，结束拖拽

**代码实现：**

```javascript
import { useState, useCallback, useEffect, useRef } from 'react';

function useWindowDrag(windowRef) {
    const [isDragging, setIsDragging] = useState(false);
    const [offset, setOffset] = useState({ x: 0, y: 0 });
    const dragStartRef = useRef({ x: 0, y: 0, windowX: 0, windowY: 0 });

    // 开始拖拽
    const handleMouseDown = (e) => {
        // 只允许通过标题栏拖拽
        if (e.target.closest('.window-header')) {
            setIsDragging(true);
            dragStartRef.current = {
                x: e.clientX,
                y: e.clientY,
                windowX: windowRef.current.offsetLeft,
                windowY: windowRef.current.offsetTop
            };
        }
    };

    // 拖拽中
    const handleMouseMove = useCallback((e) => {
        if (!isDragging) return;

        const deltaX = e.clientX - dragStartRef.current.x;
        const deltaY = e.clientY - dragStartRef.current.y;

        const newX = dragStartRef.current.windowX + deltaX;
        const newY = dragStartRef.current.windowY + deltaY;

        setOffset({ x: newX, y: newY });
    }, [isDragging]);

    // 结束拖拽
    const handleMouseUp = useCallback(() => {
        setIsDragging(false);
    }, []);

    useEffect(() => {
        if (isDragging) {
            window.addEventListener('mousemove', handleMouseMove);
            window.addEventListener('mouseup', handleMouseUp);
        }

        return () => {
            window.removeEventListener('mousemove', handleMouseMove);
            window.removeEventListener('mouseup', handleMouseUp);
        };
    }, [isDragging, handleMouseMove, handleMouseUp]);

    return {
        offset,
        isDragging,
        handleMouseDown,
        style: {
            transform: `translate(${offset.x}px, ${offset.y}px)`
        }
    };
}

// 使用示例
function DraggableWindow({ id }) {
    const windowRef = useRef(null);
    const { offset, handleMouseDown, style } = useWindowDrag(windowRef);

    return (
        <div
            ref={windowRef}
            className="window"
            style={style}
            onMouseDown={handleMouseDown}
        >
            <div className="window-header">标题栏</div>
            <div className="window-content">内容</div>
        </div>
    );
}
```

**性能优化技巧：**

1. 使用requestAnimationFrame进行位置更新
2. 使用CSS transform代替top/left，利用GPU加速
3. 限制拖拽区域在可视窗口内

### 1.3 Monaco Editor集成问题

**面试问题：Monaco Editor在项目中是如何集成的？**

**参考答案：**

Monaco Editor是一个功能强大的代码编辑器，在WebEnv-OS项目中作为代码编辑窗口使用。

**集成要点：**

1. **安装依赖**：npm install @monaco-editor/react
2. **组件封装**：创建CodeEditor组件封装Monaco
3. **主题适配**：匹配项目的主题风格
4. **自动布局**：处理窗口大小变化

**代码实现：**

```javascript
import Editor from '@monaco-editor/react';

function CodeEditor({ language = 'javascript', value = '', onChange }) {
    const handleEditorChange = (value) => {
        onChange?.(value);
    };

    const handleEditorMount = (editor, monaco) => {
        // 配置编辑器和主题
        monaco.editor.defineTheme('os-theme', {
            base: 'vs-dark',
            inherit: true,
            rules: [],
            colors: {
                'editor.background': '#1e1e1e'
            }
        });
        monaco.editor.setTheme('os-theme');

        // 自动布局
        // automaticLayout: true
        // 当容器大小变化时，编辑器自动调整
        editor.updateOptions({
            automaticLayout: true,
            minimap: { enabled: false },
            fontSize: 14,
            lineNumbers: 'on',
            scrollBeyondLastLine: false,
            automaticLayout: true
        });
    };

    return (
        <Editor
            height="100%"
            language={language}
            value={value}
            onChange={handleEditorChange}
            onMount={handleEditorMount}
            theme="vs-dark"
            options={{
                automaticLayout: true,
                minimap: { enabled: false },
                fontSize: 14,
                lineNumbers: 'on',
                scrollBeyondLastLine: false,
                wordWrap: 'on',
                folding: true,
                lineDecorationsWidth: 10,
                lineNumbersMinChars: 3
            }}
        />
    );
}
```

**常见问题解决：**

1. **自动布局**：设置automaticLayout: true解决窗口大小变化时编辑器不调整的问题
2. **性能问题**：对于大文件，考虑使用Monaco Editor的worker实现
3. **主题适配**：使用defineTheme定义匹配项目的主题

### 1.4 Zustand状态管理

**面试问题：为什么选择Zustand作为状态管理方案？**

**参考答案：**

Zustand是一个轻量级的状态管理库，相比Redux更加简洁，适合中小型项目。

**选择理由：**

1. **简洁API**：没有reducer、action等复杂概念
2. **轻量级**：体积小（约1KB）
3. **TypeScript友好**：类型推断好
4. **无需Provider**：不需要包裹组件树

**代码示例：**

```typescript
import { create } from 'zustand';
import { devtools, persist } from 'zustand/middleware';

// 定义Store类型
interface WindowStore {
    windows: Window[];
    activeWindowId: string | null;
    highestZIndex: number;

    // Actions
    addWindow: (window: Window) => void;
    removeWindow: (id: string) => void;
    bringToFront: (id: string) => void;
    updateWindowPosition: (id: string, position: Position) => void;
}

// 创建Store
const useWindowStore = create<WindowStore>()(
    devtools(
        persist(
            (set, get) => ({
                windows: [],
                activeWindowId: null,
                highestZIndex: 100,

                addWindow: (window) => set((state) => ({
                    windows: [...state.windows, window],
                    highestZIndex: state.highestZIndex + 1
                })),

                removeWindow: (id) => set((state) => ({
                    windows: state.windows.filter(w => w.id !== id)
                })),

                bringToFront: (id) => set((state) => ({
                    activeWindowId: id,
                    highestZIndex: state.highestZIndex + 1
                })),

                updateWindowPosition: (id, position) => set((state) => ({
                    windows: state.windows.map(w =>
                        w.id === id ? { ...w, position } : w
                    )
                }))
            }),
            {
                name: 'window-storage'
            }
        ),
        { name: 'WindowStore' }
    )
);

// 在组件中使用
function WindowManager() {
    const { windows, activeWindowId, bringToFront } = useWindowStore();

    return (
        <div className="window-manager">
            {windows.map(window => (
                <Window
                    key={window.id}
                    isActive={window.id === activeWindowId}
                    onFocus={() => bringToFront(window.id)}
                />
            ))}
        </div>
    );
}
```

### 1.5 React组件通信方式

**面试问题：React中有哪些组件通信方式？**

**参考答案：**

1. **Props**：父子组件通信
2. **Context**：跨组件通信
3. **Zustand/Redux**：全局状态管理
4. **事件总线**：自定义事件

**代码示例：**

```javascript
// 1. Props通信
function Parent() {
    const [count, setCount] = useState(0);
    return <Child count={count} onIncrement={() => setCount(c => c + 1)} />;
}

function Child({ count, onIncrement }) {
    return <button onClick={onIncrement}>{count}</button>;
}

// 2. Context通信
const ThemeContext = React.createContext();

function ThemeProvider({ children }) {
    const [theme, setTheme] = useState('dark');
    return (
        <ThemeContext.Provider value={{ theme, setTheme }}>
            {children}
        </ThemeContext.Provider>
    );
}

function ThemedButton() {
    const { theme, setTheme } = useContext(ThemeContext);
    return <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
        Current: {theme}
    </button>;
}

// 3. Zustand通信
const useStore = create((set) => ({
    count: 0,
    increment: () => set(state => ({ count: state.count + 1 }))
}));

function Counter() {
    const { count, increment } = useStore();
    return <button onClick={increment}>{count}</button>;
}
```

### 1.6 React Hooks面试问题

**面试问题：useEffect的运行时机是什么？**

**参考答案：**

useEffect的运行时机取决于依赖数组：

1. **无依赖数组**：每次渲染后执行
2. **空依赖数组**：仅首次渲染后执行（相当于componentDidMount）
3. **有依赖数组**：依赖变化时执行

**代码示例：**

```javascript
function Example() {
    const [count, setCount] = useState(0);

    // 1. 每次渲染后执行
    useEffect(() => {
        console.log('每次渲染后执行');
    });

    // 2. 仅首次渲染后执行
    useEffect(() => {
        console.log('首次渲染后执行');
        // 清理函数
        return () => {
            console.log('组件卸载时执行');
        };
    }, []);

    // 3. count变化时执行
    useEffect(() => {
        console.log('count变化时执行:', count);
        return () => {
            console.log('count变化前清理');
        };
    }, [count]);

    // 4. 异步effect
    useEffect(() => {
        const fetchData = async () => {
            const data = await fetch('/api/data');
            setCount(data.count);
        };
        fetchData();
    }, []);

    return <div>{count}</div>;
}
```

### 2.2 WebSocket实时通信

**面试问题：WebSocket和HTTP有什么区别？**

**参考答案：**

| 特性 | HTTP | WebSocket |
|------|------|------------|
| 连接方式 | 请求-响应 | 双向通信 |
| 协议 | HTTP | ws/wss |
| 实时性 | 轮询/长轮询 | 实时推送 |
| 资源消耗 | 每次请求header | 只需一次握手 |

**代码示例：**

```javascript
// WebSocket客户端
class WebSocketClient {
    constructor(url) {
        this.url = url;
        this.ws = null;
        this.reconnectAttempts = 0;
        this.maxReconnectAttempts = 5;
    }

    connect() {
        return new Promise((resolve, reject) => {
            this.ws = new WebSocket(this.url);

            this.ws.onopen = () => {
                console.log('WebSocket连接成功');
                this.reconnectAttempts = 0;
                resolve();
            };

            this.ws.onmessage = (event) => {
                const data = JSON.parse(event.data);
                this.handleMessage(data);
            };

            this.ws.onerror = (error) => {
                console.error('WebSocket错误:', error);
                reject(error);
            };

            this.ws.onclose = () => {
                console.log('WebSocket连接关闭');
                this.reconnect();
            };
        });
    }

    send(data) {
        if (this.ws && this.ws.readyState === WebSocket.OPEN) {
            this.ws.send(JSON.stringify(data));
        }
    }

    reconnect() {
        if (this.reconnectAttempts < this.maxReconnectAttempts) {
            this.reconnectAttempts++;
            const delay = Math.min(1000 * Math.pow(2, this.reconnectAttempts), 30000);
            setTimeout(() => {
                console.log(`尝试重连 (${this.reconnectAttempts})...`);
                this.connect();
            }, delay);
        }
    }

    handleMessage(data) {
        // 处理接收到的消息
        console.log('收到消息:', data);
    }
}

// 使用示例
const wsClient = new WebSocketClient('ws://localhost:8080');
wsClient.connect().then(() => {
    wsClient.send({ type: 'join', roomId: 'document-1' });
});
```

### 2.3 富文本编辑器实现

**面试问题：如何实现一个富文本编辑器？**

**参考答案：**

实现富文本编辑器的核心是处理 contenteditable 属性和_selection API。

**代码示例：**

```javascript
class RichTextEditor {
    constructor(element) {
        this.element = element;
        this.editor = element.querySelector('.editor');
        this.init();
    }

    init() {
        // 监听输入事件
        this.editor.addEventListener('input', this.handleInput.bind(this));

        // 监听粘贴事件，处理粘贴内容
        this.editor.addEventListener('paste', this.handlePaste.bind(this));

        // 监听键盘快捷键
        this.editor.addEventListener('keydown', this.handleKeydown.bind(this));

        // 初始化选区
        this.editor.focus();
    }

    handleInput(e) {
        // 获取当前选区
        const selection = window.getSelection();
        console.log('当前选区:', selection.toString());

        // 获取HTML内容
        console.log('HTML内容:', this.editor.innerHTML);

        // 获取纯文本内容
        console.log('纯文本:', this.editor.innerText);
    }

    handlePaste(e) {
        e.preventDefault();

        // 获取剪贴板内容
        const clipboardData = e.clipboardData || window.clipboardData;
        const pastedData = clipboardData.getData('Text');

        // 插入纯文本
        document.execCommand('insertText', false, pastedData);
    }

    handleKeydown(e) {
        // 处理Ctrl+B加粗
        if (e.ctrlKey && e.key === 'b') {
            e.preventDefault();
            document.execCommand('bold', false, null);
        }

        // 处理Ctrl+I斜体
        if (e.ctrlKey && e.key === 'i') {
            e.preventDefault();
            document.execCommand('italic', false, null);
        }

        // 处理Ctrl+U下划线
        if (e.ctrlKey && e.key === 'u') {
            e.preventDefault();
            document.execCommand('underline', false, null);
        }
    }

    // 获取编辑器内容
    getContent() {
        return {
            html: this.editor.innerHTML,
            text: this.editor.innerText,
            json: this.getJSON()
        };
    }

    // 获取JSON格式内容
    getJSON() {
        // 实现JSON序列化
        return Array.from(this.editor.childNodes).map(node => {
            if (node.nodeType === Node.TEXT_NODE) {
                return { type: 'text', content: node.textContent };
            }
            if (node.nodeType === Node.ELEMENT_NODE) {
                return {
                    type: node.tagName.toLowerCase(),
                    children: Array.from(node.childNodes).map(child => ({
                        type: 'text',
                        content: child.textContent
                    }))
                };
            }
        });
    }
}
```

### 2.4 LiveKit视频集成

**面试问题：LiveKit是什么？如何使用？**

**参考答案：**

LiveKit是一个开源的实时音视频平台，提供WebRTC基础设施。

**代码示例：**

```javascript
import LiveKit from 'livekit-client';

class VideoRoom {
    constructor() {
        this.room = null;
        this.localParticipant = null;
    }

    async connect(url, token) {
        // 连接到LiveKit房间
        this.room = await LiveKit.connect(url, token, {
            autoSubscribe: true,
            adaptiveStream: true
        });

        this.localParticipant = this.room.localParticipant;

        // 监听房间事件
        this.room
            .on('participantConnected', this.handleParticipantConnected)
            .on('participantDisconnected', this.handleParticipantDisconnected)
            .on('trackSubscribed', this.handleTrackSubscribed);

        // 创建并发布本地音视频
        await this.createLocalTracks();
    }

    async createLocalTracks() {
        // 获取本地音视频轨道
        const tracks = await LiveKit.createLocalTracks({
            audio: true,
            video: true
        });

        // 发布到房间
        for (const track of tracks) {
            this.localParticipant.publishTrack(track);
        }
    }

    handleParticipantConnected(participant) {
        console.log('参与者加入:', participant.identity);
    }

    handleParticipantDisconnected(participant) {
        console.log('参与者离开:', participant.identity);
    }

    handleTrackSubscribed(track, publication, participant) {
        // 将远程轨道attach到DOM
        const element = track.attach();
        document.getElementById('video-container').appendChild(element);
    }

    // 切换摄像头
    async switchCamera() {
        const videoTrack = this.localParticipant
            .getTrackPublications()
            .find(p => p.kind === LiveKit.TrackKind.Video)?.track;

        if (videoTrack) {
            const newDeviceId = await this.getNextDeviceId('videoinput');
            await videoTrack.switchDevice(newDeviceId);
        }
    }

    async getNextDeviceId(kind) {
        const devices = await navigator.mediaDevices.enumerateDevices();
        return devices.find(d => d.kind === kind)?.deviceId;
    }

    // 离开房间
    disconnect() {
        this.room?.disconnect();
    }
}

// 使用示例
const videoRoom = new VideoRoom();
await videoRoom.connect('wss://your-server.com', 'your-token');
```

### 2.5 OT/CRDT冲突解决

**面试问题：OT和CRDT算法有什么区别？**

**参考答案：**

OT（Operational Transformation）和CRDT（Conflict-free Replicated Data Type）都是用于解决协作编辑中冲突问题的算法。

| 特性 | OT | CRDT |
|------|-----|------|
| 原理 | 操作转换 | 数据结构 |
| 服务器 | 需要 | 不需要 |
| 复杂度 | 高 | 中 |
| 适用场景 | 文本编辑 | 任意数据类型 |

**代码示例：**

```javascript
// 简化的OT实现
class OTTextOperation {
    constructor() {
        this.ops = [];
    }

    // 插入操作
    insert(position, text) {
        this.ops.push({ type: 'insert', position, text });
        return this;
    }

    // 删除操作
    delete(position, length) {
        this.ops.push({ type: 'delete', position, length });
        return this;
    }

    // 应用操作到文本
    apply(text) {
        let result = text;
        // 按位置排序操作
        const sortedOps = [...this.ops].sort((a, b) => a.position - b.position);

        let offset = 0;
        for (const op of sortedOps) {
            const pos = op.position + offset;
            if (op.type === 'insert') {
                result = result.slice(0, pos) + op.text + result.slice(pos);
                offset += op.text.length;
            } else if (op.type === 'delete') {
                result = result.slice(0, pos) + result.slice(pos + op.length);
                offset -= op.length;
            }
        }
        return result;
    }

    // 转换操作（解决冲突）
    transform(op1, op2) {
        // 实现操作转换逻辑
        // 这是一个简化版本
        if (op1.type === 'insert' && op2.type === 'insert') {
            if (op1.position <= op2.position) {
                return [op1, { ...op2, position: op2.position + op1.text.length }];
            } else {
                return [{ ...op1, position: op1.position + op2.text.length }, op2];
            }
        }
        return [op1, op2];
    }
}

// CRDT示例 - 使用Y.js
import * as Y from 'yjs';

function useCRDT() {
    // 创建CRDT文档
    const ydoc = new Y.Doc();

    // 创建文本类型
    const ytext = ydoc.getText('content');

    // 监听变化
    ytext.observe(event => {
        console.log('内容变化:', ytext.toString());
    });

    // 本地插入
    function insert(index, text) {
        ytext.insert(index, text);
    }

    // 本地删除
    function delete_(index, length) {
        ytext.delete(index, length);
    }

    // 获取内容
    function getContent() {
        return ytext.toString();
    }

    return { insert, delete: delete_, getContent, ydoc };
}
```

### 3.1 TCP/IP三次握手

**面试问题：请解释TCP三次握手的过程**

**参考答案：**

三次握手是TCP建立连接的过程：

1. **第一次握手（SYN）**：客户端发送SYN包（SYN=1），进入SYN_SENT状态
2. **第二次握手（SYN-ACK）**：服务器收到SYN包，发送SYN+ACK包，进入SYN_RCVD状态
3. **第三次握手（ACK）**：客户端收到SYN+ACK包，发送ACK包，进入ESTABLISHED状态

**作用**：确保双方都能接收和发送数据。

### 3.2 HTTP缓存机制

**面试问题：请解释HTTP缓存机制**

**参考答案：**

HTTP缓存分为强缓存和协商缓存：

1. **强缓存**：直接从本地读取，不请求服务器
   - Expires：绝对时间
   - Cache-Control：相对时间（max-age）

2. **协商缓存**：需要请求服务器判断是否使用缓存
   - Last-Modified / If-Modified-Since
   - ETag / If-None-Match

**代码示例：**

```javascript
// 服务器端设置缓存头
app.get('/api/data', (req, res) => {
    // 强缓存：1小时
    res.set('Cache-Control', 'public, max-age=3600');

    // 协商缓存：ETag
    const etag = generateETag(data);
    res.set('ETag', etag);

    res.json(data);
});

// 客户端处理
async function fetchData() {
    const response = await fetch('/api/data', {
        headers: {
            'If-None-Match': localStorage.getItem('etag')
        }
    });

    if (response.status === 304) {
        // 使用本地缓存
        return JSON.parse(localStorage.getItem('data'));
    }

    const data = await response.json();
    localStorage.setItem('data', JSON.stringify(data));
    localStorage.setItem('etag', response.headers.get('ETag'));
    return data;
}
```

2. **开闭原则（OCP）**

3. **里氏替换原则**
   - 子组件应该能够替换父组件而不影响功能
   - 保持props接口的稳定性

4. **依赖倒置原则**
   - 组件应该依赖抽象（props），而不是具体实现
   - 使用context和依赖注入解耦

```jsx
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      return initialValue;
    }

      const valueToStore = value instanceof Function
        ? value(storedValue)
        : value;

      window.localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.error(error);
    }
  }, [key, storedValue]);

  return [storedValue, setValue];
}
```

### 9.5 Vue3响应式原理

**面试问题：Vue3的响应式原理是什么？**

**参考答案：**

Vue3使用ES6 Proxy实现响应式系统，相比Vue2的Object.defineProperty有如下优势：

1. **Proxy的优势**
   - 可以监听对象属性的添加和删除
   - 可以监听数组索引变化
   - 不需要遍历对象属性
   - 支持Map、Set等数据结构

2. **核心实现**
   - reactive函数：创建Proxy代理对象
   - ref函数：包装基本类型为响应式对象
   - effect函数：收集依赖并触发更新

3. **依赖收集**
   - 当读取响应式对象时，收集依赖
   - 当修改响应式对象时，触发更新

**代码示例：**

```javascript
// 简化版响应式实现
function reactive(target) {
  return new Proxy(target, {
    get(target, key, receiver) {
      const result = Reflect.get(target, key, receiver);
      // 收集依赖
      track(target, key);
      // 如果是对象，递归包装
      return typeof result === 'object'
        ? reactive(result)
        : result;
    },

    set(target, key, value, receiver) {
      const oldValue = target[key];
      const result = Reflect.set(target, key, value, receiver);
      // 触发更新
      if (oldValue !== value) {
        trigger(target, key);
      }
      return result;
    }
  });
}

// 依赖收集
const targetMap = new WeakMap();
const effectStack = [];

function track(target, key) {
  const effect = effectStack[effectStack.length - 1];
  if (effect) {
    let depsMap = targetMap.get(target);
    if (!depsMap) {
      depsMap = new Map();
      targetMap.set(target, depsMap);
    }
    let deps = depsMap.get(key);
    if (!deps) {
      deps = new Set();
      depsMap.set(key, deps);
    }
    deps.add(effect);
  }
}

function trigger(target, key) {
  const depsMap = targetMap.get(target);
  if (!depsMap) return;
  const deps = depsMap.get(key);
  if (deps) {
    deps.forEach(effect => effect());
  }
}
```

## 第十六部分：算法与数据结构面试问题

### 16.1 常见排序算法

**面试问题：请实现常见的排序算法**

**参考答案：**

1. **冒泡排序**
   - 时间复杂度：O(n²)
   - 空间复杂度：O(1)

2. **快速排序**
   - 平均时间复杂度：O(n log n)
   - 最坏：O(n²)

3. **归并排序**
   - 时间复杂度：O(n log n)
   - 空间复杂度：O(n)

**代码示例：**

```typescript
// 冒泡排序
function bubbleSort(arr: number[]): number[] {
  const n = arr.length;
  for (let i = 0; i < n - 1; i++) {
    for (let j = 0; j < n - i - 1; j++) {
      if (arr[j] > arr[j + 1]) {
        [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
      }
    }
  }
  return arr;
}

// 快速排序
function quickSort(arr: number[]): number[] {
  if (arr.length <= 1) return arr;

  const pivot = arr[Math.floor(arr.length / 2)];
  const left = arr.filter(x => x < pivot);
  const middle = arr.filter(x => x === pivot);
  const right = arr.filter(x => x > pivot);

  return [...quickSort(left), ...middle, ...quickSort(right)];
}

// 归并排序
function mergeSort(arr: number[]): number[] {
  if (arr.length <= 1) return arr;

  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));

  return merge(left, right);
}

function merge(left: number[], right: number[]): number[] {
  const result: number[] = [];
  let i = 0, j = 0;

  while (i < left.length && j < right.length) {
    if (left[i] <= right[j]) {
      result.push(left[i++]);
    } else {
      result.push(right[j++]);
    }
  }

  return result.concat(left.slice(i)).concat(right.slice(j));
}
```

### 16.2 常见搜索算法

**面试问题：请实现常见的搜索算法**

**参考答案：**

1. **二分查找**
   - 时间复杂度：O(log n)
   - 要求：有序数组

2. **深度优先搜索（DFS）**
   - 使用栈或递归
   - 适合遍历树/图

3. **广度优先搜索（BFS）**
   - 使用队列
   - 适合最短路径

**代码示例：**

```typescript
// 二分查找
function binarySearch(arr: number[], target: number): number {
  let left = 0;
  let right = arr.length - 1;

  while (left <= right) {
    const mid = Math.floor((left + right) / 2);
    if (arr[mid] === target) return mid;
    if (arr[mid] < target) left = mid + 1;
    else right = mid - 1;
  }

  return -1;
}

// DFS遍历二叉树
function dfs(root: TreeNode | null, result: number[] = []): number[] {
  if (!root) return result;
  result.push(root.val);
  dfs(root.left, result);
  dfs(root.right, result);
  return result;
}

// BFS遍历二叉树
function bfs(root: TreeNode | null): number[] {
  if (!root) return [];
  const result: number[] = [];
  const queue: TreeNode[] = [root];

  while (queue.length > 0) {
    const node = queue.shift()!;
    result.push(node.val);
    if (node.left) queue.push(node.left);
    if (node.right) queue.push(node.right);
  }

  return result;
}
```

### 16.3 链表操作

**面试问题：请实现链表的基本操作**

**参考答案：**

链表是一种线性数据结构，每个节点包含数据和指向下一个节点的指针。

**代码示例：**

```typescript
class ListNode<T> {
  val: T;
  next: ListNode<T> | null = null;

  constructor(val: T) {
    this.val = val;
  }
}

class LinkedList<T> {
  head: ListNode<T> | null = null;
  size: number = 0;

  // 添加元素
  append(val: T): void {
    const node = new ListNode(val);
    if (!this.head) {
      this.head = node;
    } else {
      let current = this.head;
      while (current.next) {
        current = current.next;
      }
      current.next = node;
    }
    this.size++;
  }

  // 删除元素
  delete(val: T): boolean {
    if (!this.head) return false;
    if (this.head.val === val) {
      this.head = this.head.next;
      this.size--;
      return true;
    }
    let current = this.head;
    while (current.next) {
      if (current.next.val === val) {
        current.next = current.next.next;
        this.size--;
        return true;
      }
      current = current.next;
    }
    return false;
  }

  // 反转链表
  reverse(): void {
    let prev: ListNode<T> | null = null;
    let current = this.head;

    while (current) {
      const next = current.next;
      current.next = prev;
      prev = current;
      current = next;
    }

    this.head = prev;
  }
}
```

### 16.4 栈与队列

**面试问题：请实现栈和队列**

**参考答案：**

栈是后进先出（LIFO），队列是先进先出（FIFO）。

**代码示例：**

```typescript
// 栈实现
class Stack<T> {
  private items: T[] = [];

  push(item: T): void {
    this.items.push(item);
  }

  pop(): T | undefined {
    return this.items.pop();
  }

  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }

  isEmpty(): boolean {
    return this.items.length === 0;
  }

  size(): number {
    return this.items.length;
  }
}

// 队列实现
class Queue<T> {
  private items: T[] = [];

  enqueue(item: T): void {
    this.items.push(item);
  }

  dequeue(): T | undefined {
    return this.items.shift();
  }

  front(): T | undefined {
    return this.items[0];
  }

  isEmpty(): boolean {
    return this.items.length === 0;
  }

  size(): number {
    return this.items.length;
  }
}

// 使用栈实现有效括号
function isValid(s: string): boolean {
  const stack = new Stack<string>();
  const map: Record<string, string> = {
    ')': '(',
    ']': '[',
    '}': '{'
  };

  for (const char of s) {
    if (char in map) {
      if (stack.pop() !== map[char]) return false;
    } else {
      stack.push(char);
    }
  }

  return stack.isEmpty();
}
```
