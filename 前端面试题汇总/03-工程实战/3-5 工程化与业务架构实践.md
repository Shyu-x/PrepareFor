## 一、TOB 与 TOC 业务特点与架构差异

### 1.1 TOB（To Business）业务特点

TOB（面向企业）业务是指为企业客户提供产品或服务的业务模式。其特点如下：

| 特性 | 说明 |
|------|------|
| **用户群体** | 企业员工、管理者、决策者 |
| **使用场景** | 办公、管理、数据分析 |
| **功能复杂度** | 高，功能繁多且相互关联 |
| **交互方式** | 鼠标操作为主，操作密集型 |
| **性能要求** | 强调数据处理能力和稳定性 |
| **安全性** | 高，涉及企业敏感数据 |
| **定制化** | 高，需要支持客户化配置 |
| **上线周期** | 长，测试严格 |

**典型 TOB 产品：**
- 企业管理系统（ERP、CRM、OA）
- 财务系统
- 人力资源管理系统
- 数据分析平台
- SaaS 产品

**TOB 前端架构要点：**

```typescript
// TOB 业务特点的架构设计

// 1. 复杂表单处理
interface FormConfig {
  fields: FormField[];
  layout: 'single' | 'double' | 'triple';

interface DataTableConfig {
  columns: TableColumn[];
  pagination: PaginationConfig;

  resources: Resource[];
  actions: Action[];
  roles: Role[];
  grants: Grant[];
  // 细粒度权限控制
  checkPermission: (resource: string, action: string) => boolean;
}
```

### 1.2 TOC（To Consumer）业务特点

TOC（面向消费者）业务是指为个人用户提供产品或服务的业务模式。其特点如下：

| 特性 | 说明 |
|------|------|
| **用户群体** | 普通消费者 |
| **使用场景** | 娱乐、社交、购物、信息获取 |
| **功能复杂度** | 中等，核心功能突出 |
| **交互方式** | 触摸、滑动、多媒体 |
| **性能要求** | 强调首屏加载和响应速度 |
| **安全性** | 中等，侧重账户安全 |
| **定制化** | 低，标准化体验 |
| **上线周期** | 短，快速迭代 |

**典型 TOC 产品：**
- 电商平台
- 社交应用
- 短视频平台
- 新闻资讯应用
- 在线教育平台

**TOC 前端架构要点：**

### 1.3 TOB vs TOC 架构对比

| 维度 | TOB | TOC |
|------|-----|-----|
| **路由复杂度** | 多级嵌套，简单层级深 | 扁平化，层级少 |
| **状态管理** | 复杂表单状态，多模块联动 | 相对简单，用户状态为主 |
| **数据表格** | 核心组件，高性能要求 | 较少使用 |
| **表单处理** | 核心，大量复杂校验 | 较少，登录注册为主 |
| **缓存策略** | 服务端缓存为主 | 客户端缓存重要 |
| **离线能力** | 非必须 | 重要 |
| **国际化** | 重要，多语言多时区 | 相对简单 |
| **主题定制** | 重要，支持品牌定制 | 固定主题 |

### 2.1 创建型模式

#### 2.1.1 工厂模式（Factory Pattern）

```typescript
// 工厂模式 - 创建不同类型的表单组件
interface FormElement {
  render(): HTMLElement;
  validate(): boolean;
  getValue(): any;
}

class InputElement implements FormElement {
  constructor(private config: InputConfig) {}

  render(): HTMLElement {
    const input = document.createElement('input');
    input.type = this.config.type;
    input.placeholder = this.config.placeholder;
    return input;
  }

  validate(): boolean {
    return this.config.required ? !!this.getValue() : true;
  }

  getValue(): any {
    return (document.getElementById(this.config.id) as HTMLInputElement).value;
  }
}

class SelectElement implements FormElement {
  constructor(private config: SelectConfig) {}

  render(): HTMLElement {
    const select = document.createElement('select');
    this.config.options.forEach(opt => {
      const option = document.createElement('option');
      option.value = opt.value;
      option.textContent = opt.label;
      select.appendChild(option);
    });
    return select;
  }

  validate(): boolean {
    return this.config.required ? !!this.getValue() : true;
  }

  getValue(): any {
    return (document.getElementById(this.config.id) as HTMLSelectElement).value;
  }
}

// 工厂类
class FormElementFactory {
  static create(type: 'input' | 'select' | 'textarea' | 'checkbox', config: any): FormElement {
    switch (type) {
      case 'input':
        return new InputElement(config);
      case 'select':
        return new SelectElement(config);
      // ... 其他类型
      default:
        throw new Error(`Unknown element type: ${type}`);
    }
  }
}

// 使用
const form = FormElementFactory.create('input', {
  id: 'username',
  type: 'text',
  placeholder: '请输入用户名',
  required: true
});
```

#### 2.1.2 单例模式（Singleton Pattern）

```typescript
// 单例模式 - 全局状态管理
class GlobalState {
  private static instance: GlobalState;
  private state: Map<string, any> = new Map();
  private listeners: Map<string, Set<Function>> = new Map();

  private constructor() {}

  static getInstance(): GlobalState {
    if (!GlobalState.instance) {
      GlobalState.instance = new GlobalState();
    }
    return GlobalState.instance;
  }

  get(key: string): any {
    return this.state.get(key);
  }

  set(key: string, value: any): void {
    this.state.set(key, value);
    this.notify(key, value);
  }

  subscribe(key: string, listener: Function): () => void {
    if (!this.listeners.has(key)) {
      this.listeners.set(key, new Set());
    }
    this.listeners.get(key)!.add(listener);

    // 返回取消订阅函数
    return () => {
      this.listeners.get(key)?.delete(listener);
    };
  }

  private notify(key: string, value: any): void {
    this.listeners.get(key)?.forEach(listener => listener(value));
  }
}

// 使用
const state = GlobalState.getInstance();
state.set('user', { name: 'Shyu' });
const unsubscribe = state.subscribe('user', (user) => {
  console.log('User updated:', user);
});
```

#### 2.2.2 代理模式（Proxy Pattern）

```typescript
// 代理模式 - 权限控制、数据懒加载

// 真实主题
class UserService {
  getUser(id: string): User {
    // 实际数据库查询
    return { id, name: '用户' + id };
  }

  deleteUser(id: string): void {
    console.log(`删除用户: ${id}`);
  }
}

// 代理类 - 添加权限控制
class UserServiceProxy {
  private realService: UserService;
  private permissions: Set<string>;

  constructor(permissions: Set<string>) {
    this.realService = new UserService();
    this.permissions = permissions;
  }

  getUser(id: string): User | null {
    if (!this.permissions.has('user:read')) {
      console.log('无权限读取用户');
      return null;
    }
    return this.realService.getUser(id);
  }

  deleteUser(id: string): boolean {
    if (!this.permissions.has('user:delete')) {
      console.log('无权限删除用户');
      return false;
    }
    this.realService.deleteUser(id);
    return true;
  }
}

// 使用
const proxy = new UserServiceProxy(new Set(['user:read']));
proxy.getUser('1'); // 有权限，正常返回
proxy.deleteUser('1'); // 无权限，被拒绝
```

### 2.3 行为型模式

#### 2.3.1 观察者模式（Observer Pattern）

```typescript
// 观察者模式 - 事件系统、状态监听

interface Observer {
  update(data: any): void;
}

interface Subject {
  attach(observer: Observer): void;
  detach(observer: Observer): void;
  notify(): void;
}

class EventEmitter implements Subject {
  private observers: Map<string, Set<Observer>> = new Map();

  attach(event: string, observer: Observer): void {
    if (!this.observers.has(event)) {
      this.observers.set(event, new Set());
    }
    this.observers.get(event)!.add(observer);
  }

  detach(event: string, observer: Observer): void {
    this.observers.get(event)?.delete(observer);
  }

  emit(event: string, data: any): void {
    this.observers.get(event)?.forEach(observer => observer.update(data));
  }

  notify(): void {}
}

// 使用
class UserObserver implements Observer {
  update(data: any): void {
    console.log('用户数据更新:', data);
  }
}

const emitter = new EventEmitter();
const observer = new UserObserver();

emitter.attach('user:update', observer);
emitter.emit('user:update', { name: 'Shyu' });
```

#### 2.3.2 策略模式（Strategy Pattern）

```typescript
// 策略模式 - 多种算法/校验规则切换

interface ValidationStrategy {
  validate(value: any): boolean;
  getErrorMessage(): string;
}

// 策略实现
class EmailValidation implements ValidationStrategy {
  validate(value: any): boolean {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value);
  }

  getErrorMessage(): string {
    return '请输入有效的邮箱地址';
  }
}

class PhoneValidation implements ValidationStrategy {
  validate(value: any): boolean {
    return /^1[3-9]\d{9}$/.test(value);
  }

  getErrorMessage(): string {
    return '请输入有效的手机号码';
  }
}

class LengthValidation implements ValidationStrategy {
  constructor(private min: number, private max: number) {}

  validate(value: any): boolean {
    return value.length >= this.min && value.length <= this.max;
  }

  getErrorMessage(): string {
    return `长度必须在 ${this.min}-${this.max} 之间`;
  }
}

// 策略上下文
class FormField {
  private strategy: ValidationStrategy;

  constructor(private name: string, strategy: ValidationStrategy) {
    this.strategy = strategy;
  }

  setStrategy(strategy: ValidationStrategy): void {
    this.strategy = strategy;
  }

  validate(value: any): { valid: boolean; message: string } {
    const valid = this.strategy.validate(value);
    return {
      valid,
      message: valid ? '' : this.strategy.getErrorMessage()
    };
  }
}

// 使用
const emailField = new FormField('email', new EmailValidation());
console.log(emailField.validate('test@example.com')); // { valid: true, message: '' }
console.log(emailField.validate('invalid')); // { valid: false, message: '请输入有效的邮箱地址' }

// 切换策略
emailField.setStrategy(new LengthValidation(5, 50));
console.log(emailField.validate('abc')); // { valid: false, message: '长度必须在 5-50 之间' }
```

#### 2.3.3 命令模式（Command Pattern）

```typescript
// 命令模式 - 撤销/重做、操作记录

interface Command {
  execute(): void;
  undo(): void;
}

class CommandManager {
  private history: Command[] = [];
  private redoStack: Command[] = [];

  execute(command: Command): void {
    command.execute();
    this.history.push(command);
    this.redoStack = []; // 新命令清除重做栈
  }

  undo(): void {
    const command = this.history.pop();
    if (command) {
      command.undo();
      this.redoStack.push(command);
    }
  }

  redo(): void {
    const command = this.redoStack.pop();
    if (command) {
      command.execute();
      this.history.push(command);
    }
  }
}

// 具体命令
class AddItemCommand implements Command {
  constructor(
    private list: string[],
    private item: string
  ) {}

  execute(): void {
    this.list.push(this.item);
    console.log(`添加: ${this.item}`);
  }

  undo(): void {
    const index = this.list.indexOf(this.item);
    if (index > -1) {
      this.list.splice(index, 1);
      console.log(`撤销添加: ${this.item}`);
    }
  }
}

class RemoveItemCommand implements Command {
  private removedItem: string = '';

  constructor(
    private list: string[],
    private item: string
  ) {}

  execute(): void {
    const index = this.list.indexOf(this.item);
    if (index > -1) {
      this.removedItem = this.list.splice(index, 1)[0];
      console.log(`删除: ${this.removedItem}`);
    }
  }

  undo(): void {
    if (this.removedItem) {
      this.list.push(this.removedItem);
      console.log(`撤销删除: ${this.removedItem}`);
    }
  }
}

// 使用
const list: string[] = [];
const manager = new CommandManager();

manager.execute(new AddItemCommand(list, '项目A'));
manager.execute(new AddItemCommand(list, '项目B'));
manager.execute(new RemoveItemCommand(list, '项目A'));

console.log('当前列表:', list); // ['项目B']

manager.undo();
console.log('撤销后:', list); // ['项目A', '项目B']

manager.redo();
console.log('重做后:', list); // ['项目B']
```

## 四、设计理论与原则

### 4.1 SOLID 原则

| 原则 | 说明 | 违反后果 |
|------|------|----------|
| **S 单一职责** | 每个类/函数只负责一项功能 | 难以维护、难以测试 |
| **O 开闭原则** | 对扩展开放，对修改关闭 | 频繁修改影响稳定性 |
| **L 里氏替换** | 子类能替换父类 | 破坏继承体系 |
| **I 接口隔离** | 使用多个专用接口 | 依赖冗余 |
| **D 依赖倒置** | 依赖抽象而非具体 | 难以替换实现 |

```typescript
class ErrorTracker {
  private endpoint: string;

  constructor(endpoint: string) {
    this.endpoint = endpoint;
    this.setupGlobalHandlers();
  }

  private setupGlobalHandlers(): void {
    // JavaScript 错误
    window.onerror = (message, source, lineno, colno, error) => {
      this.reportError({
        type: 'js',
        message,
        source,
        lineno,
        colno,
        stack: error?.stack,
      });
    };

    // Promise 拒绝
    window.onunhandledrejection = (event) => {
      this.reportError({
        type: 'promise',
        reason: event.reason,
      });
    };

    // 资源加载错误
    window.addEventListener('error', (event) => {
      if (event.target !== window) {
        this.reportError({
          type: 'resource',
          url: (event.target as HTMLScriptElement).src,
        });
      }
    }, true);
  }

  private reportError(error: any): void {
    fetch(this.endpoint, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        ...error,
        timestamp: Date.now(),
        userAgent: navigator.userAgent,
        url: window.location.href,
      }),
    });
  }
}
```

### 7.2 降级与容灾

```typescript
// 降级策略
class DegradationStrategy {
  // 功能降级
  static degradeFeature(
    feature: string,
    primary: () => any,
    fallback: () => any,
    timeout: number = 1000
  ): Promise<any> {
    return new Promise((resolve) => {
      const timer = setTimeout(() => {
        console.warn(`功能 ${feature} 超时，使用降级方案`);
        resolve(fallback());
      }, timeout);

      primary()
        .then((result) => {
          clearTimeout(timer);
          resolve(result);
        })
        .catch((error) => {
          clearTimeout(timer);
          console.error(`功能 ${feature} 失败:`, error);
          resolve(fallback());
        });
    });
  }

  // 渲染降级
  static renderDegrade(
    primary: () => JSX.Element,
    fallback: () => JSX.Element,
    shouldDegrade: () => boolean
  ): JSX.Element {
    if (shouldDegrade()) {
      return fallback();
    }
    return primary();
  }

  // 缓存降级
  static async cacheDegrade<T>(
    key: string,
    fetcher: () => Promise<T>,
    cache: Cache,
    ttl: number = 5 * 60 * 1000
  ): Promise<T> {
    // 尝试从缓存获取
    const cached = await cache.get(key);
    if (cached && Date.now() - cached.timestamp < ttl) {
      return cached.data;
    }

    // 缓存不存在或过期，获取新数据
    try {
      const data = await fetcher();
      await cache.set(key, { data, timestamp: Date.now() });
      return data;
    } catch (error) {
      // 获取失败，返回过期缓存
      if (cached) {
        console.warn('使用过期缓存:', key);
        return cached.data;
      }
      throw error;
    }
  }
}
```

```typescript
// 领域实体
class Order {
  private id: string;
  private items: OrderItem[];
  private status: OrderStatus;
  private createdAt: Date;

  constructor(id: string) {
    this.id = id;
    this.items = [];
    this.status = 'pending';
    this.createdAt = new Date();
  }

  // 领域方法
  addItem(product: Product, quantity: number): void {
    if (this.status !== 'pending') {
      throw new Error('只能对pending状态的订单添加商品');
    }
    this.items.push(new OrderItem(product, quantity));
  }

  submit(): void {
    if (this.items.length === 0) {
      throw new Error('订单不能为空');
    }
    this.status = 'submitted';
  }

  // 值对象
  getTotal(): Money {
    return this.items.reduce(
      (sum, item) => sum.add(item.getSubtotal()),
      Money.zero()
    );
  }
}

// 值对象
class Money {
  constructor(
    private amount: number,
    private currency: string = 'CNY'
  ) {}

  static zero(): Money {
    return new Money(0);
  }

  add(other: Money): Money {
    if (this.currency !== other.currency) {
      throw new Error('货币类型不一致');
    }
    return new Money(this.amount + other.amount, this.currency);
  }
}
```
