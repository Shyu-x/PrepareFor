## 七、核心业务 STAR 法则描述

### 7.1 终端多标签页功能

**背景 (Situation)**
用户需要同时管理多个终端会话，在不同项目间切换工作。

**任务 (Task)**
实现终端多标签页管理，支持创建、关闭、切换终端标签。

**行动 (Action)**
- 使用 `useTerminalStore` 存储标签状态
- 实现 `TerminalTabs` 组件渲染标签栏
- 通过 `TerminalInstance` 管理每个 xterm.js 实例
- 使用 WebSocket 与后端终端服务通信

**结果 (Result)**
- 支持本地、SSH、Docker 三种终端模式
- 最多 10 个标签同时打开
- 标签切换响应时间 < 50ms

### 7.2 代码导航功能

**背景 (Situation)**
用户需要在大型代码库中快速定位函数定义、查找引用。

**任务 (Task)**
实现跳转到定义、查找引用、全局搜索等功能。

**行动 (Action)**
- 集成 Monaco Editor 内置语言服务
- 使用 TypeScript Worker 进行代码分析
- 实现 `useCodeNavigation` Hook 封装导航逻辑
- 创建 ReferencesPanel 显示引用结果

**结果 (Result)**
- 跳转到定义准确率 > 95%
- 全局搜索支持 50+ 文件同时检索
- 引用面板实时显示所有引用位置

### 7.3 多人实时协作

**背景 (Situation)**
多个开发者需要同时编辑同一代码文件，实时同步变更。

**任务 (Task)**
实现基于 WebSocket 的实时协作编辑功能。

**行动 (Action)**
- 使用 Socket.IO 建立 WebSocket 连接
- 集成 Yjs CRDT 处理冲突
- 实现协作者光标和选区同步
- 通过 `CollaborationGateway` 转发编辑事件

**结果 (Result)**
- 延迟 < 100ms 的实时同步
- 支持 10+ 用户同时协作
- 离线编辑自动合并

### 7.4 工作区 Docker 环境隔离

**背景 (Situation)**
每个项目需要独立的开发环境，避免依赖冲突。

**任务 (Task)**
为每个工作区创建独立的 Docker 容器。

**行动 (Action)**
- 后端维护 Docker 连接池
- 为每个工作区创建独立网络和文件系统卷
- 通过 `ensureWorkspaceContainer` 方法创建容器
- 支持动态切换 Docker 镜像

**结果 (Result)**
- 8 种预定义开发环境模板
- 容器启动时间 < 30s
- 独立网络确保安全隔离
