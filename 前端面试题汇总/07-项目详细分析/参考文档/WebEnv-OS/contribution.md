# 贡献指南

感谢你对 webEnvOS 的关注！

## 1. 分支策略
- `main`: 生产稳定分支。
- `develop`: 日常开发分支。
- `feature/*`: 功能开发分支。

## 2. 提交规范
提交信息必须遵循以下格式：
`<type>: <description>` (中文)

常用 Type:
- `feat`: 新功能。
- `fix`: 修复 Bug。
- `docs`: 文档更新。
- `refactor`: 重构。

## 3. Pull Request 流程
1. 提交至 `develop` 分支。
2. 附带必要的本地测试通过截图。
3. 确保没有任何 `eslint` 警告。
