### 2.7 React 虚拟 DOM 原理

**工作流程**：

```
创建虚拟DOM → 更新虚拟DOM → Diff算法比较 → 补丁应用到真实DOM
```

**Diff 算法核心**：
- 同层对比：仅比较 DOM 树同一层级
- key 标识：用唯一 ID 作为 key，避免列表顺序变化导致无效重渲染
- 批量更新：合并多次 DOM 修改

### 2.8 React Fiber 架构

**Fiber 三层含义**：

| 层面 | 定义 |
|------|------|
| **架构** | React 协调引擎重构，从 Stack Reconciler 转为 Fiber Reconciler |
| **静态数据结构** | 每个组件/元素对应 Fiber 节点，链表结构连接 |
| **工作单元** | 任务拆分、优先级调度、增量渲染 |

**核心特性**：
- 异步可中断更新
- 优先级调度（交互>动画>数据请求）
- 双缓存机制（current/workInProgress）

- **创建 Store**：使用 `create` 函数创建，返回一个自定义 Hook
- **组件中使用**：`const { count } = useStore()` - 无需 Provider
- **选择器订阅**：通过 `useStore((state) => state.count)` 只订阅需要的状态片段

### 4.2 Zustand 最佳实践

**推荐实践**：
- 分模块管理：使用切片模式（Slice Pattern）将大型 store 拆分为多个小型 store
- 渐进式状态管理：根据需求选择合适的状态类型
- 异步支持：直接在 store 中编写异步函数
- 合理使用选择器避免不必要的重渲染
- 结合 persist 中间件实现状态持久化

### 4.3 Zustand persist 中间件

**基础用法**：
```typescript
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

const useStore = create(
  persist(
    (set, get) => ({
      userInfo: null,
      setUserInfo: (userInfo) => set({ userInfo }),
    }),
    {
      name: 'user-storage', // localStorage 中的键名
    }
  )
);
```

### 4.4 Zustand TypeScript 类型

**类型定义**：
```typescript
interface CounterState {
  count: number;
  increment: () => void;
  decrement: () => void;
}

const useCounterStore = create<CounterState>((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
}));
```

### 4.5 Zustand 性能优化 useShallow

**为什么需要 useShallow**：
当选择多个状态字段时，默认使用严格相等（===）比较，可能导致不必要的重渲染

**基本用法**：
```typescript
import { useShallow } from 'zustand/shallow';

const { name, age } = useStore(
  useShallow((state) => ({ name: state.name, age: state.age }))
);
```

## 推荐学习路径

### 第一阶段：基础入门（1-2周）

```
HTML5 + CSS3 基础 → JavaScript 基础 → 语义化标签
```

### 第二阶段：React 核心（2-3周）

```
React 基础教程 → Hooks 完全指南 → useState/useEffect 详解
→ 虚拟 DOM 原理 → Fiber 架构
```

### 第三阶段：状态管理（1-2周）

```
Zustand 基础 → persist 中间件 → SWR 数据获取
→ useShallow 性能优化
```

### 第四阶段：Next.js 全栈（2-3周）

```
Next.js 基础 → App Router → 服务端组件 vs 客户端组件
→ 渲染模式 SSR/SSG/ISR → 性能优化
```

### 第五阶段：实时协作（2-3周）

```
WebSocket 基础 → Socket.io → Yjs CRDT
→ 实时协作编辑 → LiveKit 视频会议
```

### 第六阶段：3D 与图形（2-3周）

```
Three.js 基础 → React Three Fiber → WebGL
→ 动画系统 → 性能优化
```

### 第七阶段：编辑器开发（2-3周）

```
Monaco Editor → xterm.js → ProseMirror/TipTap
→ 块编辑器实现 → 实时协作
```

### 第八阶段：AI 与大模型（1-2周）

```
Prompt 工程 → AI Agent → Claude API
→ AI 代码助手 → RAG 知识库
```
