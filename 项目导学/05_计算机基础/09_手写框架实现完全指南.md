# 手写框架实现完全指南

## 一、手写框架的意义与价值

### 1.1 为什么需要手写框架

在前端开发的学习道路上，阅读源码和手写框架是两种相辅相成的重要学习方法。阅读源码让我们理解优秀框架的设计思想和实现细节，而手写框架则让我们真正掌握这些思想和细节，将被动接受知识转变为主动创造。

手写框架的过程是一个深度学习的过程。当我们试图实现一个响应式系统时，我们需要思考如何追踪依赖、如何触发更新、如何避免无限循环等具体问题。这些问题在阅读源码时可能只是匆匆一瞥，但在实现时却必须逐一解决，从而带来更深入的理解。

手写框架也是检验学习效果的有效方式。很多时候，我们觉得自己理解了某个概念，但真正动手实现时才会发现还有许多模糊之处。这种认知偏差只有通过实践才能发现和纠正。

此外，手写框架可以积累宝贵的经验。实现过程中遇到的问题和解决方案，都是独一无二的学习收获。这些经验不仅帮助我们更好地使用成熟框架，也为未来参与开源贡献或自研框架打下基础。

### 1.2 手写框架与阅读源码的关系

手写框架和阅读源码是学习框架原理的两种互补方式，它们相互促进、相互加深理解。

阅读源码是手写框架的基础。通过阅读源码，我们了解成熟框架的设计模式、API设计、代码组织方式等。这些知识为手写实现提供了方向和参照。如果没有阅读源码的基础，手写实现往往只能复刻表面，难以把握精髓。

手写框架是阅读源码的深化。在手写实现过程中，我们会遇到各种边界情况和性能问题，这时重新阅读相关源码，往往能获得更深的理解。手写实现让我们带着问题去读源码，效率大大提高。

建议的学习路径是：先阅读源码了解框架的整体设计和核心概念，然后手写实现一个简化版本，最后再回到源码进行对比分析，查漏补缺。这样的循环学习可以最大化学习效果。

### 1.3 手写框架的学习路径

手写框架的学习应该遵循由简单到复杂、由基础到高级的顺序。

第一个阶段是实现基础功能。以响应式系统为例，可以先实现一个最基础的版本，支持对象的get和set拦截、依赖收集、副作用执行。这个阶段的目标是理解核心原理，不追求完美。

第二个阶段是完善功能。在基础版本上添加数组支持、计算属性、深度响应式等特性。这个阶段的目标是覆盖更多使用场景，理解框架如何处理各种边界情况。

第三个阶段是优化性能。添加批量更新、异步更新、computed缓存等优化。这个阶段的目标是理解框架的性能优化策略。

第四个阶段是重构整理。重新组织代码结构，提取工具函数，优化API设计。这个阶段的目标是学习良好的代码组织和设计模式。

## 二、Mini Vue实现：从原理到实践

### 2.1 Mini Vue设计概述

Mini Vue是一个简化版的Vue.js实现，它保留了Vue3响应式系统的核心特性，包括： reactive函数将对象转换为响应式、ref函数将基本类型转换为响应式、computed计算属性、effect副作用函数自动追踪依赖变化。

Mini Vue的设计目标是：用最少的代码阐明Vue3响应式系统的核心原理，让学习者通过动手实现真正理解Vue3的工作方式。

整个Mini Vue的代码量控制在500行左右，涵盖以下核心模块：

1. reactivity模块：响应式系统的核心实现
2. baseHandlers模块：Proxy处理器对象的定义
3. effect模块：副作用函数的管理
4. computed模块：计算属性的实现

### 2.2 响应式系统基础实现

响应式系统的核心是Proxy。Proxy可以在对象属性的访问和修改时拦截操作，从而实现依赖收集和触发更新。

首先定义基础的响应式对象创建函数：

```typescript
// Mini Vue - 响应式系统核心实现

// 用于存储已代理对象的Map
const rawToProxy = new WeakMap()

// 基础Proxy处理器
const baseHandlers = {
  // get陷阱：依赖收集
  get(target, key, receiver) {
    const result = Reflect.get(target, key, receiver)

    // 追踪依赖：通知当前活跃的effect
    track(target, key)

    // 如果是对象，递归代理实现深度响应式
    if (typeof result === 'object' && result !== null) {
      return reactive(result)
    }

    return result
  },

  // set陷阱：触发更新
  set(target, key, value, receiver) {
    const oldValue = target[key]

    // 设置新值
    const result = Reflect.set(target, key, value, receiver)

    // 只有值发生变化时才触发更新
    if (oldValue !== value) {
      // 触发更新：通知所有依赖此属性的effect
      trigger(target, key, value, oldValue)
    }

    return result
  },

  // deleteProperty陷阱：删除属性
  deleteProperty(target, key) {
    const hadKey = Object.prototype.hasOwnProperty.call(target, key)
    const result = Reflect.deleteProperty(target, key)

    if (hadKey && result) {
      // 触发更新
      trigger(target, key)
    }

    return result
  }
}

// 创建响应式对象的函数
function reactive(target) {
  // 如果已经是代理对象，直接返回
  if (target && target.__v_isReactive) {
    return target
  }

  // 检查是否已经代理过
  const proxy = rawToProxy.get(target)
  if (proxy) {
    return proxy
  }

  // 创建新的代理对象
  const newProxy = new Proxy(target, baseHandlers)

  // 存入Map缓存
  rawToProxy.set(target, newProxy)

  return newProxy
}

// 标记对象为响应式
function isReactive(target) {
  return target && target.__v_isReactive === true
}
```

### 2.3 依赖收集与触发更新实现

依赖收集和触发更新是响应式系统的两个核心机制。

依赖收集发生在读取响应式对象的属性时，通过track函数记录当前活跃的effect函数：

```typescript
// 依赖收集与触发更新

// 存储每个目标对象的依赖关系
// targetMap: WeakMap<target, Map<key, Set<effect>>>
const targetMap = new WeakMap()

// 当前活跃的effect函数
let activeEffect = null

// 依赖收集函数
function track(target, key) {
  // 如果没有活跃的effect，不需要收集依赖
  if (!activeEffect) {
    return
  }

  // 获取目标对象的依赖Map
  let depsMap = targetMap.get(target)
  if (!depsMap) {
    depsMap = new Map()
    targetMap.set(target, depsMap)
  }

  // 获取key对应的effect集合
  let deps = depsMap.get(key)
  if (!deps) {
    deps = new Set()
    depsMap.set(key, deps)
  }

  // 将当前活跃的effect添加到依赖集合
  deps.add(activeEffect)
}

// 触发更新函数
function trigger(target, key, value, oldValue) {
  // 获取目标对象的所有依赖
  const depsMap = targetMap.get(target)
  if (!depsMap) {
    return
  }

  // 获取key对应的effect集合
  const effects = depsMap.get(key)
  if (!effects) {
    return
  }

  // 创建新的Set以避免迭代过程中修改集合
  const effectsToRun = new Set(effects)

  // 执行所有依赖此属性的effect
  effectsToRun.forEach(effect => {
    // 如果effect有调度器，使用调度器
    if (effect.options && effect.options.scheduler) {
      effect.options.scheduler()
    } else {
      // 否则直接执行effect
      effect()
    }
  })
}
```

### 2.4 Effect与WatchEffect实现

Effect是响应式系统的执行单元，它包装了一个函数，并在该函数中访问的响应式数据发生变化时自动重新执行：

```typescript
// Effect副作用函数实现

// effect函数选项
interface EffectOptions {
  lazy?: boolean           // 是否延迟执行
  scheduler?: Function     // 调度器
  flush?: 'pre' | 'post' | 'sync'  // 刷新时机
}

// 创建effect的工厂函数
function effect(fn, options: EffectOptions = {}) {
  // 创建响应式的effect对象
  const effect = function effectRunner() {
    return fn()
  }

  // 保存选项
  effect.options = options

  // 如果不是延迟执行，立即执行
  if (!options.lazy) {
    effect()
  }

  return effect
}

// watchEffect的实现
function watchEffect(fn, options: EffectOptions = {}) {
  const job = () => {
    fn()
  }

  effect(job, {
    scheduler: options.scheduler || job
  })
}

// 清理effect的依赖
function cleanup(effect) {
  // 清除effect的所有依赖关系
  // 这在实现组件卸载时的清理工作很重要
  effect.deps = []
}
```

### 2.5 Ref与Computed实现

Ref用于将基本类型值包装为响应式对象，Computed用于创建计算属性：

```typescript
// Ref与Computed实现

// Ref接口定义
interface Ref<T> {
  value: T
  __v_isRef: boolean
}

// ref函数：将基本类型转换为响应式
function ref<T>(value: T): Ref<T> {
  // 如果已经是ref，直接返回
  if (isRef(value)) {
    return value
  }

  // 创建包含value属性的响应式对象
  const refObject = {
    __v_isRef: true,
    get value() {
      // 收集依赖
      track(refObject, 'value')
      return value
    },
    set value(newValue) {
      // 只有值发生变化时才更新
      if (newValue !== value) {
        const oldValue = value
        value = newValue
        // 触发更新
        trigger(refObject, 'value', newValue, oldValue)
      }
    }
  }

  return refObject
}

// 判断是否为ref
function isRef(value): boolean {
  return value && value.__v_isRef === true
}

// 将对象的所有属性转换为ref
function toRefs(obj) {
  const result = {}
  for (const key in obj) {
    result[key] = toRef(obj, key)
  }
  return result
}

// 转换为ref
function toRef(obj, key) {
  return {
    __v_isRef: true,
    get value() {
      return obj[key]
    },
    set value(newValue) {
      obj[key] = newValue
    }
  }
}

// computed计算属性实现
interface ComputedRef<T> extends Ref<T> {
  readonly value: T
}

function computed<T>(getter): ComputedRef<T> {
  // 计算属性的值
  let value

  // 标记是否需要重新计算
  let dirty = true

  const effect = new Effect(() => {
    // 当依赖变化时，标记需要重新计算
    dirty = true
  })

  return {
    __v_isRef: true,
    get value() {
      if (dirty) {
        // 收集依赖并计算新值
        value = effect.run()
        dirty = false
      }
      return value
    },
    set value(newValue) {
      // computed的值不应被直接设置
      console.warn('computed value cannot be set directly')
    }
  }
}
```

### 3.2 Virtual DOM实现

Virtual DOM是React的核心概念，它是一个描述真实DOM的JavaScript对象。通过对比新旧Virtual DOM的差异，React可以高效地更新真实DOM：

```typescript
// Mini React - Virtual DOM实现

// 创建Virtual DOM的函数
function createElement(type, props, ...children) {
  return {
    type,
    props: {
      ...props,
      children: children.map(child => {
        // 如果是字符串或数字，直接作为文本节点
        if (typeof child === 'string' || typeof child === 'number') {
          return createTextNode(child)
        }
        // 否则递归创建Virtual DOM
        return child
      })
    }
  }
}

// 创建文本节点的Virtual DOM
function createTextNode(text) {
  return {
    type: 'TEXT',
    props: {
      nodeValue: text,
      children: []
    }
  }
}

// 判断两个Virtual DOM节点是否可以复用
function sameNode(a, b) {
  return a && b && a.type === b.type
}

// 将Virtual DOM转换为真实DOM
function createDOM(vnode) {
  if (!vnode) return null

  // 创建文本节点
  if (vnode.type === 'TEXT') {
    return document.createTextNode(vnode.props.nodeValue)
  }

  // 创建普通元素节点
  const dom = document.createElement(vnode.type)

  // 设置属性
  Object.keys(vnode.props)
    .filter(key => key !== 'children')
    .forEach(key => {
      dom[key] = vnode.props[key]
    })

  // 递归创建子节点
  vnode.props.children.forEach(child => {
    dom.appendChild(createDOM(child))
  })

  return dom
}

// 更新DOM的函数
function updateDOM(dom, oldProps, newProps) {
  // 移除或修改旧的属性
  Object.keys(oldProps)
    .filter(key => key !== 'children')
    .filter(key => !(key in newProps))
    .forEach(key => {
      dom[key] = null
    })

  // 添加或修改新的属性
  Object.keys(newProps)
    .filter(key => key !== 'children')
    .forEach(key => {
      if (oldProps[key] !== newProps[key]) {
        dom[key] = newProps[key]
      }
    })
}
```

### 3.3 简易渲染器实现

渲染器负责将Virtual DOM渲染为真实DOM，并处理组件的挂载和更新：

```typescript
// Mini React - 渲染器实现

// 组件渲染函数
function render(element, container) {
  // 将Virtual DOM转换为真实DOM
  const dom = createDOM(element)
  container.appendChild(dom)
}

// 更新函数
function updateElement(container, oldElement, newElement) {
  // 比较新旧Virtual DOM的差异
  const diff = compare(oldElement, newElement)

  // 应用差异到DOM
  applyDiff(container, diff)
}

// 比较两个Virtual DOM，返回差异
function compare(oldElement, newElement) {
  const diff = []

  // 如果新旧节点类型不同，直接替换
  if (!sameNode(oldElement, newElement)) {
    diff.push({
      type: 'REPLACE',
      oldElement,
      newElement
    })
    return diff
  }

  // 如果是文本节点，内容不同则更新
  if (oldElement.type === 'TEXT') {
    if (oldElement.props.nodeValue !== newElement.props.nodeValue) {
      diff.push({
        type: 'TEXT',
        newValue: newElement.props.nodeValue
      })
    }
    return diff
  }

  // 比较属性变化
  const oldProps = oldElement.props
  const newProps = newElement.props
  const propDiff = compareProps(oldProps, newProps)
  if (propDiff.length > 0) {
    diff.push(...propDiff)
  }

  // 递归比较子节点
  const oldChildren = oldProps.children || []
  const newChildren = newProps.children || []
  const minLength = Math.min(oldChildren.length, newChildren.length)

  for (let i = 0; i < minLength; i++) {
    const childDiff = compare(oldChildren[i], newChildren[i])
    diff.push(...childDiff)
  }

  // 如果新节点有更多子节点，需要添加
  if (newChildren.length > oldChildren.length) {
    for (let i = minLength; i < newChildren.length; i++) {
      diff.push({
        type: 'APPEND',
        index: i,
        element: newChildren[i]
      })
    }
  }

  return diff
}

// 比较属性差异
function compareProps(oldProps, newProps) {
  const diff = []
  const allKeys = new Set([...Object.keys(oldProps), ...Object.keys(newProps)])

  allKeys.forEach(key => {
    if (oldProps[key] !== newProps[key]) {
      diff.push({
        type: 'PROPS',
        key,
        newValue: newProps[key],
        oldValue: oldProps[key]
      })
    }
  })

  return diff
}

// 应用差异到DOM
function applyDiff(parentDOM, diff) {
  diff.forEach(action => {
    switch (action.type) {
      case 'REPLACE':
        const newDOM = createDOM(action.newElement)
        parentDOM.replaceChild(newDOM, findDOM(action.oldElement))
        break
      case 'TEXT':
        parentDOM.textContent = action.newValue
        break
      case 'PROPS':
        parentDOM[action.key] = action.newValue
        break
      case 'APPEND':
        parentDOM.appendChild(createDOM(action.element))
        break
    }
  })
}

// 查找Virtual DOM对应的真实DOM（简化实现）
const nodeMap = new WeakMap()

function findDOM(element) {
  return nodeMap.get(element)
}
```

### 3.4 组件系统实现

React组件是UI的抽象单元，组件可以是函数组件或类组件：

```typescript
// Mini React - 组件系统实现

// 函数组件渲染
function renderComponent(component) {
  const { type, props } = component
  let renderedElement

  // 如果是函数组件，调用函数获取Virtual DOM
  if (typeof type === 'function') {
    renderedElement = type(props)
  } else {
    renderedElement = component
  }

  // 递归渲染
  const dom = createDOM(renderedElement)

  // 建立映射关系
  nodeMap.set(component, dom)

  return dom
}

// 类组件基类
class Component {
  constructor(props) {
    this.props = props
    this.state = {}
    this._rendered = null
  }

  // 更新状态
  setState(partialState) {
    // 合并部分状态
    this.state = { ...this.state, ...partialState }

    // 触发重新渲染
    scheduleUpdate(this)
  }

  // 渲染方法（子类实现）
  render() {
    return null
  }
}

// 类组件渲染
function renderClassComponent(component) {
  const instance = new component.type(component.props)

  // 触发挂载生命周期
  if (instance.componentDidMount) {
    instance.componentDidMount()
  }

  // 执行渲染
  const renderedElement = instance.render()
  instance._rendered = renderedElement

  const dom = createDOM(renderedElement)
  nodeMap.set(component, dom)

  return dom
}

// 调度更新
let rootContainer = null
let nextUnitOfWork = null

function scheduleUpdate(component) {
  nextUnitOfWork = {
    component,
    dom: nodeMap.get(component)
  }
}

// 工作循环
function workLoop() {
  while (nextUnitOfWork) {
    nextUnitOfWork = performUnitOfWork(nextUnitOfWork)
  }

  if (!nextUnitOfWork && rootContainer) {
    commitRoot()
  }

  requestIdleCallback(workLoop)
}

// 执行工作单元
function performUnitOfWork(work) {
  // 更新组件
  const { component, dom } = work
  const { type, props } = component._rendered

  // 更新DOM
  if (component._rendered) {
    updateElement(dom, component._rendered, component.render())
  }
}
```

### 3.5 Hooks系统实现

Hooks让函数组件拥有了状态和生命周期能力，是React16.8引入的重要特性：

```typescript
// Mini React - Hooks系统实现

// 当前处理的Fiber节点
let currentlyRenderingFiber = null
// Hook的调用顺序
let currentHookIndex = 0

// useState Hook实现
function useState(initialState) {
  // 获取当前的Fiber节点
  const oldHook = currentlyRenderingFiber?.alternate?.hooks?.[currentHookIndex]

  const hook = {
    state: oldHook ? oldHook.state : initialState,
    queue: oldHook ? oldHook.queue : []
  }

  // 处理队列中的更新
  hook.queue.forEach(action => {
    hook.state = action(hook.state)
  })
  hook.queue = []

  // 创建dispatch函数
  const dispatch = (action) => {
    hook.queue.push(action)

    // 触发重新渲染
    currentlyRenderingFiber.alternate = {
      ...currentlyRenderingFiber,
      hooks: [...currentlyRenderingFiber.hooks]
    }
    currentlyRenderingFiber.hooks[currentHookIndex] = hook
  }

  currentlyRenderingFiber.hooks.push(hook)

  return [hook.state, dispatch]
}

// useEffect Hook实现
function useEffect(effect, deps) {
  const oldHook = currentlyRenderingFiber?.alternate?.hooks?.[currentHookIndex]

  let hasChanged = true

  // 检查依赖是否变化
  if (oldHook && deps) {
    const oldDeps = oldHook.deps
    hasChanged = deps.some((dep, i) => !Object.is(dep, oldDeps[i]))
  }

  // 如果依赖变化或首次渲染，执行effect
  if (hasChanged) {
    // 保存cleanup函数
    const cleanup = effect()
    if (typeof cleanup === 'function') {
      currentlyRenderingFiber.hooks.push({
        deps,
        cleanup
      })
    }
  }
}

// useRef Hook实现
function useRef(initialValue) {
  const hook = {
    current: initialValue
  }

  const oldHook = currentlyRenderingFiber?.alternate?.hooks?.[currentHookIndex]
  if (oldHook) {
    return oldHook
  }

  return hook
}

// useMemo Hook实现
function useMemo(factory, deps) {
  const oldHook = currentlyRenderingFiber?.alternate?.hooks?.[currentHookIndex]

  if (oldHook && deps) {
    const oldDeps = oldHook.deps
    const hasChanged = deps.some((dep, i) => !Object.is(dep, oldDeps[i]))

    if (!hasChanged) {
      return oldHook.value
    }
  }

  const value = factory()
  const hook = { value, deps }

  currentlyRenderingFiber.hooks.push(hook)

  return value
}

// useCallback Hook实现
function useCallback(callback, deps) {
  return useMemo(() => callback, deps)
}
```

### 4.1 手写工具函数库

手写一个工具函数库是很好的编程练习，可以帮助我们理解JavaScript的基础知识和最佳实践。

```typescript
// debounce防抖函数
function debounce<T extends (...args: any[]) => any>(
  func: T,
  wait: number
): (...args: Parameters<T>) => void {
  let timeoutId: ReturnType<typeof setTimeout> | null = null

  return function(this: any, ...args: Parameters<T>) {
    // 如果有待执行的定时器，先清除
    if (timeoutId !== null) {
      clearTimeout(timeoutId)
    }

    // 设置新的定时器
    timeoutId = setTimeout(() => {
      func.apply(this, args)
      timeoutId = null
    }, wait)
  }
}

// throttle节流函数
function throttle<T extends (...args: any[]) => any>(
  func: T,
  limit: number
): (...args: Parameters<T>) => void {
  let inThrottle = false
  let lastResult: ReturnType<T>

  return function(this: any, ...args: Parameters<T>) {
    if (!inThrottle) {
      lastResult = func.apply(this, args)
      inThrottle = true

      setTimeout(() => {
        inThrottle = false
      }, limit)
    }

    return lastResult
  }
}

// deepClone深拷贝
function deepClone<T>(obj: T, map = new WeakMap()): T {
  // 处理基本类型和null
  if (obj === null || typeof obj !== 'object') {
    return obj
  }

  // 处理循环引用
  if (map.has(obj as any)) {
    return map.get(obj as any)
  }

  // 处理日期对象
  if (obj instanceof Date) {
    return new Date(obj.getTime()) as any
  }

  // 处理正则对象
  if (obj instanceof RegExp) {
    return new RegExp(obj.source, obj.flags) as any
  }

  // 处理数组
  if (Array.isArray(obj)) {
    const clonedArr: any[] = []
    map.set(obj, clonedArr)
    obj.forEach((item, index) => {
      clonedArr[index] = deepClone(item, map)
    })
    return clonedArr as any
  }

  // 处理普通对象
  const clonedObj = {} as T
  map.set(obj, clonedObj)

  Object.keys(obj).forEach(key => {
    clonedObj[key] = deepClone((obj as any)[key], map)
  })

  return clonedObj
}

// once只执行一次函数
function once<T extends (...args: any[]) => any>(
  func: T
): (...args: Parameters<T>) => ReturnType<T> | undefined {
  let called = false
  let result: ReturnType<T>

  return function(this: any, ...args: Parameters<T>) {
    if (!called) {
      result = func.apply(this, args)
      called = true
    }
    return result
  }
}

// curry函数柯里化
function curry<T extends (...args: any[]) => any>(
  func: T,
  arity = func.length
): any {
  return function curried(this: any, ...args: any[]): any {
    // 如果参数够了，执行原函数
    if (args.length >= arity) {
      return func.apply(this, args)
    }

    // 否则返回新函数等待更多参数
    return function(this: any, ...nextArgs) {
      return curried.apply(this, [...args, ...nextArgs])
    }
  }
}

// compose函数组合
function compose<T>(...funcs: Array<(arg: T) => T>) {
  if (funcs.length === 0) {
    return (arg: T) => arg
  }

  if (funcs.length === 1) {
    return funcs[0]
  }

  return funcs.reduce((a, b) => (...args: any[]) => a(b(...args)))
}

// pipe函数管道（从左到右执行）
function pipe<T>(...funcs: Array<(arg: T) => T>) {
  return compose(...funcs.reverse())
}

// promiseify将回调函数转为Promise
function promisify<T, A, R>(
  fn: (arg: A, callback: (error: any, result: R) => void) => void
): (arg: A) => Promise<R> {
  return function(arg: A) {
    return new Promise((resolve, reject) => {
      fn(arg, (error, result) => {
        if (error) {
          reject(error)
        } else {
          resolve(result)
        }
      })
    })
  }
}
```

### 4.2 手写Virtual DOM库

Virtual DOM是现代前端框架的核心概念，手写一个简易的Virtual DOM库可以深入理解其原理：

```typescript
// 手写Virtual DOM库

// 虚拟节点类型
type Props = Record<string, any>
type Children = (VNode | string | number)[]
type Key = string | number

// 虚拟节点接口
interface VNode {
  type: string | 'TEXT'
  key?: Key
  props: Props
  children: Children
}

// 创建虚拟节点
function h(type: string, props: Props = {}, ...children: Children): VNode {
  return {
    type,
    key: props.key,
    props,
    children: children.flat().map(child => {
      if (typeof child === 'object') {
        return child
      }
      return { type: 'TEXT', props: { nodeValue: String(child) }, children: [] }
    })
  }
}

// 创建文本节点
function text(str: string | number): VNode {
  return {
    type: 'TEXT',
    props: { nodeValue: String(str) },
    children: []
  }
}

// 创建DOM元素
function createElement(vnode: VNode): Node | null {
  if (!vnode) return null

  if (vnode.type === 'TEXT') {
    return document.createTextNode(vnode.props.nodeValue)
  }

  const dom = document.createElement(vnode.type)

  // 设置属性
  Object.keys(vnode.props)
    .filter(key => key !== 'key')
    .forEach(key => {
      if (key.startsWith('on')) {
        const eventName = key.slice(2).toLowerCase()
        dom.addEventListener(eventName, vnode.props[key])
      } else {
        dom.setAttribute(key, vnode.props[key])
      }
    })

  // 挂载子节点
  vnode.children.forEach(child => {
    const childDom = createElement(child)
    if (childDom) {
      dom.appendChild(childDom)
    }
  })

  return dom
}

// diff算法
interface DiffResult {
  type: 'REPLACE' | 'UPDATE' | 'REMOVE' | 'KEEP'
  vnode?: VNode
  oldVNode?: VNode
}

function diff(oldVNode: VNode, newVNode: VNode): DiffResult[] {
  if (!oldVNode && !newVNode) {
    return []
  }

  if (!oldVNode) {
    return [{ type: 'REPLACE', vnode: newVNode }]
  }

  if (!newVNode) {
    return [{ type: 'REMOVE', oldVNode }]
  }

  if (oldVNode.type !== newVNode.type) {
    return [{ type: 'REPLACE', vnode: newVNode, oldVNode }]
  }

  if (oldVNode.type === 'TEXT') {
    if (oldVNode.props.nodeValue !== newVNode.props.nodeValue) {
      return [{ type: 'UPDATE', vnode: newVNode, oldVNode }]
    }
    return [{ type: 'KEEP' }]
  }

  // 比较属性
  const propDiffs: { key: string; value: any; oldValue: any }[] = []
  const allKeys = new Set([
    ...Object.keys(oldVNode.props),
    ...Object.keys(newVNode.props)
  ])

  allKeys.forEach(key => {
    if (key === 'key') return
    const oldValue = oldVNode.props[key]
    const newValue = newVNode.props[key]
    if (oldValue !== newValue) {
      propDiffs.push({ key, value: newValue, oldValue })
    }
  })

  // 比较子节点
  const childDiffs: DiffResult[] = []
  const maxLen = Math.max(
    oldVNode.children.length,
    newVNode.children.length
  )

  for (let i = 0; i < maxLen; i++) {
    childDiffs.push(...diff(
      oldVNode.children[i] || null,
      newVNode.children[i] || null
    ))
  }

  return [
    ...(propDiffs.length > 0 ? [{ type: 'UPDATE', vnode: newVNode, oldVNode }] : []),
    ...childDiffs
  ]
}

// 打补丁
function patch(dom: Node, diffs: DiffResult[]): Node {
  diffs.forEach(diff => {
    switch (diff.type) {
      case 'REPLACE':
        const newDom = createElement(diff.vnode!)
        dom.parentNode?.replaceChild(newDom, dom)
        return newDom

      case 'UPDATE':
        if (diff.vnode?.type === 'TEXT') {
          dom.textContent = diff.vnode.props.nodeValue
        } else {
          // 更新属性
          Object.keys(diff.oldVNode?.props || {}).forEach(key => {
            if (!diff.vnode?.props[key]) {
              dom.removeAttribute(key)
            }
          })
          Object.keys(diff.vnode?.props || {}).forEach(key => {
            if (key.startsWith('on')) {
              const eventName = key.slice(2).toLowerCase()
              dom.removeEventListener(eventName, diff.oldVNode?.props[key])
              dom.addEventListener(eventName, diff.vnode!.props[key])
            } else {
              dom.setAttribute(key, diff.vnode!.props[key])
            }
          })
        }
        break

      case 'REMOVE':
        dom.parentNode?.removeChild(dom)
        break

      case 'KEEP':
        break
    }
  })

  return dom
}
```

```typescript
// 创建路由实例
class Router {
  private routes: Route[]
  private currentPath: string
  private listeners: Array<(path: string) => void>

  constructor(routes: Route[]) {
    this.routes = routes
    this.currentPath = window.location.pathname
    this.listeners = []

    // 监听浏览器前进后退
    window.addEventListener('popstate', () => {
      this.currentPath = window.location.pathname
      this.notifyListeners()
    })
  }

  // 获取当前路径
  getPath(): string {
    return this.currentPath
  }

  // 导航到指定路径
  push(path: string): void {
    window.history.pushState(null, '', path)
    this.currentPath = path
    this.notifyListeners()
  }

  // 替换当前历史记录
  replace(path: string): void {
    window.history.replaceState(null, '', path)
    this.currentPath = path
    this.notifyListeners()
  }

  // 后退
  back(): void {
    window.history.back()
  }

  // 前进
  forward(): void {
    window.history.forward()
  }

  // 订阅路由变化
  subscribe(listener: (path: string) => void): () => void {
    this.listeners.push(listener)
    return () => {
      this.listeners = this.listeners.filter(l => l !== listener)
    }
  }

  // 通知所有监听器
  private notifyListeners(): void {
    this.listeners.forEach(listener => listener(this.currentPath))
  }

  // 匹配路由
  match(path: string): Route | undefined {
    return this.routes.find(route => route.path === path)
  }

  // 获取路由参数
  getParams(path: string): Record<string, string> {
    const params: Record<string, string> = {}

    this.routes.forEach(route => {
      const paramNames: string[] = []
      const regexPath = route.path.replace(/:([^/]+)/g, (_, name) => {
        paramNames.push(name)
        return '([^/]+)'
      })

      const regex = new RegExp(`^${regexPath}$`)
      const match = path.match(regex)

      if (match) {
        paramNames.forEach((name, index) => {
          params[name] = match[index + 1]
        })
      }
    })

    return params
  }
}

function Route({ path, component }: Route) {
  const router = React.useContext(RouterContext)
  const [content, setContent] = React.useState('')

  React.useEffect(() => {
    // 订阅路由变化
    const unsubscribe = router?.subscribe(currentPath => {
      if (router.match(currentPath)?.path === path) {
        setContent(router.match(currentPath)?.component() || 'Not Found')
      }
    })

    // 初始加载
    const currentPath = router?.getPath() || ''
    if (router?.match(currentPath)?.path === path) {
      setContent(router.match(currentPath)?.component() || 'Not Found')
    }

    return () => unsubscribe?.()
  }, [path, router])

  return <>{content}</>
}

      routerRef.current.push(initialPath)
    }
  }

  return (
    <RouterContext.Provider value={routerRef.current}>
      {routes.map(route => (
        <Route key={route.path} {...route} />
      ))}
    </RouterContext.Provider>
  )
}
```

### 4.4 手写简易Redux

Redux是React生态中最流行的状态管理方案，手写一个简易版Redux可以深入理解其核心概念：

```typescript
// 手写简易Redux

// Action类型
interface Action {
  type: string
  payload?: any
}

// Reducer类型
type Reducer<S, A extends Action = Action> = (state: S, action: A) => S

// Store类型
interface Store<S, A extends Action = Action> {
  getState: () => S
  dispatch: (action: A) => void
  subscribe: (listener: () => void) => () => void
}

// 创建Store
function createStore<S, A extends Action>(
  reducer: Reducer<S, A>,
  preloadedState?: S
): Store<S, A> {
  let state = preloadedState
  const listeners: Array<() => void> = []

  // 获取当前状态
  const getState = () => state

  // 分发action
  const dispatch = (action: A) => {
    state = reducer(state, action)
    // 通知所有订阅者
    listeners.forEach(listener => listener())
  }

  // 订阅状态变化
  const subscribe = (listener: () => void) => {
    listeners.push(listener)
    // 返回取消订阅函数
    return () => {
      const index = listeners.indexOf(listener)
      if (index > -1) {
        listeners.splice(index, 1)
      }
    }
  }

  return { getState, dispatch, subscribe }
}

// 合并Reducer
function combineReducers<S extends Record<string, any>>(
  reducers: { [K in keyof S]: Reducer<S[K]> }
): Reducer<S> {
  return (state = {} as S, action) => {
    const nextState = {} as S

    Object.keys(reducers).forEach(key => {
      nextState[key] = reducers[key](state[key], action)
    })

    return nextState
  }
}

// 中间件类型
type Middleware<S, A extends Action = Action> = (
  store: Store<S, A>
) => (next: (action: A) => void) => (action: A) => void

// applyMiddleware
function applyMiddleware<S, A extends Action = Action>(
  ...middlewares: Array<Middleware<S, A>>
): (createStoreFn: typeof createStore) => (
  reducer: Reducer<S, A>,
  preloadedState?: S
) => Store<S, A> {
  return (createStoreFn) => (reducer, preloadedState) => {
    const store = createStoreFn(reducer, preloadedState)

    // 依次调用中间件
    const chain = middlewares.map(middleware => middleware(store))
    const dispatch = chain.reduce(
      (acc, curr) => curr(acc),
      store.dispatch
    )

    return { ...store, dispatch }
  }
}

// thunk中间件
const thunkMiddleware: Middleware<any> = (store) => (next) => (action) => {
  if (typeof action === 'function') {
    return action(store.dispatch, store.getState)
  }
  return next(action)
}

// logger中间件
const loggerMiddleware: Middleware<any> = (store) => (next) => (action) => {
  console.log('dispatching:', action)
  const result = next(action)
  console.log('next state:', store.getState())
  return result
}

// bindActionCreators
type ActionCreator<A extends Action = Action> = (...args: any[]) => A

function bindActionCreators<
  A extends Action,
  C extends ActionCreator<A>
>(
  actionCreator: C,
  dispatch: (action: A) => void
): (...args: Parameters<C>) => void {
  return (...args) => dispatch(actionCreator(...args))
}
```

### 4.5 手写简易Axios

Axios是目前最流行的HTTP请求库，手写一个简易版Axios可以帮助理解HTTP请求处理的细节：

```typescript
// 手写简易Axios

// 请求配置接口
interface AxiosRequestConfig {
  url?: string
  method?: 'GET' | 'POST' | 'PUT' | 'DELETE' | 'PATCH'
  headers?: Record<string, string>
  params?: Record<string, string>
  data?: any
  timeout?: number
  transformRequest?: (data: any) => string
  transformResponse?: (data: any) => any
}

// 响应接口
interface AxiosResponse<T = any> {
  data: T
  status: number
  statusText: string
  headers: Record<string, string>
  config: AxiosRequestConfig
}

// Axios类
class Axios {
  private defaults: AxiosRequestConfig

  constructor() {
    this.defaults = {
      method: 'GET',
      headers: {},
      timeout: 10000
    }
  }

  // 核心请求方法
  async request<T>(config: AxiosRequestConfig): Promise<AxiosResponse<T>> {
    const finalConfig = { ...this.defaults, ...config }

    // 构建URL
    const url = this.buildURL(finalConfig.url || '', finalConfig.params)

    // 构建请求头
    const headers = { ...this.defaults.headers, ...finalConfig.headers }

    // 转换请求数据
    let data = finalConfig.data
    if (finalConfig.transformRequest && data) {
      data = finalConfig.transformRequest(data)
    }

    // 发送请求
    const response = await this.doRequest<T>({
      ...finalConfig,
      url,
      headers,
      data
    })

    // 转换响应数据
    if (finalConfig.transformResponse) {
      response.data = finalConfig.transformResponse(response.data)
    }

    return response
  }

  // 执行实际请求
  private doRequest<T>(config: AxiosRequestConfig): Promise<AxiosResponse<T>> {
    return new Promise((resolve, reject) => {
      const xhr = new XMLHttpRequest()

      xhr.open(config.method || 'GET', config.url || '', true)

      // 设置请求头
      Object.entries(config.headers || {}).forEach(([key, value]) => {
        xhr.setRequestHeader(key, value)
      })

      // 设置超时
      xhr.timeout = config.timeout || 10000

      xhr.onload = () => {
        const responseHeaders: Record<string, string> = {}
        const headerStr = xhr.getAllResponseHeaders()

        headerStr.split('\r\n').forEach(line => {
          const parts = line.split(': ')
          if (parts.length === 2) {
            responseHeaders[parts[0].toLowerCase()] = parts[1]
          }
        })

        resolve({
          data: xhr.responseText ? JSON.parse(xhr.responseText) : null,
          status: xhr.status,
          statusText: xhr.statusText,
          headers: responseHeaders,
          config
        })
      }

      xhr.onerror = () => {
        reject(new Error('Network Error'))
      }

      xhr.ontimeout = () => {
        reject(new Error('Request Timeout'))
      }

      // 发送请求
      xhr.send(config.data ? JSON.stringify(config.data) : null)
    })
  }

  // 构建URL参数
  private buildURL(url: string, params?: Record<string, string>): string {
    if (!params) return url

    const queryString = Object.entries(params)
      .map(([key, value]) => `${encodeURIComponent(key)}=${encodeURIComponent(value)}`)
      .join('&')

    return `${url}${url.includes('?') ? '&' : '?'}${queryString}`
  }

  // GET请求
  get<T>(url: string, config?: AxiosRequestConfig): Promise<AxiosResponse<T>> {
    return this.request<T>({ ...config, url, method: 'GET' })
  }

  // POST请求
  post<T>(url: string, data?: any, config?: AxiosRequestConfig): Promise<AxiosResponse<T>> {
    return this.request<T>({ ...config, url, method: 'POST', data })
  }

  // PUT请求
  put<T>(url: string, data?: any, config?: AxiosRequestConfig): Promise<AxiosResponse<T>> {
    return this.request<T>({ ...config, url, method: 'PUT', data })
  }

  // DELETE请求
  delete<T>(url: string, config?: AxiosRequestConfig): Promise<AxiosResponse<T>> {
    return this.request<T>({ ...config, url, method: 'DELETE' })
  }

  // PATCH请求
  patch<T>(url: string, data?: any, config?: AxiosRequestConfig): Promise<AxiosResponse<T>> {
    return this.request<T>({ ...config, url, method: 'PATCH', data })
  }
}

// 工厂函数创建axios实例
function createInstance(): Axios {
  return new Axios()
}

// 导出默认实例
const axios = createInstance()

export default axios
```

## 五、框架实现最佳实践

### 5.1 代码组织原则

良好的代码组织是框架实现的基础。以下是一些重要的代码组织原则：

模块化设计是首要原则。将功能划分为独立的模块，每个模块负责单一职责。以响应式系统为例，可以划分为：依赖收集模块、触发更新模块、代理创建模块、工具函数模块等。模块之间通过清晰的接口交互，便于理解、维护和测试。

一致的命名约定可以提高代码可读性。建议使用一致的命名风格，如Vue3源码使用的命名约定：变量和函数使用小驼峰，类名使用大驼峰，常量使用全大写下划线分隔。

合理的文件结构可以方便代码管理。建议按功能模块组织文件，每个文件控制在300行以内，过长的文件应该拆分。

详尽的注释和文档对于框架至关重要。公共API必须有JSDoc注释，说明参数、返回值、可能抛出的异常等。复杂的算法应该添加解释性注释。

### 5.2 性能优化策略

框架实现中需要考虑多种性能优化：

懒计算是常见的优化策略。以computed计算属性为例，只有在值被读取时才会计算，而且一旦依赖没有变化，后续读取直接返回缓存值，避免重复计算。

批量更新可以减少不必要的渲染次数。Vue3中使用scheduler机制将同一个tick内的更新合并为一次更新，避免重复渲染。

依赖省略可以减少不必要的依赖收集。当响应式数据的某个属性没有被读取时，其变化不会触发副作用重新执行。

空间换时间是常见的优化思路。使用WeakMap缓存已代理的对象，可以快速判断对象是否已经被代理，避免重复创建代理。

### 5.3 边界情况处理

框架必须妥善处理各种边界情况：

循环引用可能出现在用户的数据结构中，也可能出现自己内部的实现中。Vue3使用WeakMap处理循环引用，WeakMap的key是弱引用，不会阻止对象被垃圾回收。

null和undefined值在JavaScript中很常见，框架需要优雅地处理这些值，不能因为访问null的属性而抛出异常。

深层嵌套的数据结构需要特殊处理。Vue3默认对深层嵌套的对象进行响应式转换，但如果嵌套太深可能会影响性能，可以通过shallowReactive只做浅层响应式转换来优化。

### 5.4 API设计原则

良好的API设计可以提升框架的易用性：

简洁直观的API让用户容易上手。Axios的API设计就是一个很好的例子，axios.get()、axios.post()等方法直观易懂。

一致性让用户可以用相同的模式使用不同的功能。Vue3的reactive、ref、computed等API都遵循相同的使用模式。

可扩展性让框架能够适应不断变化的需求。Redux的中间件机制、Axios的拦截器机制都是可扩展性的体现。

向后兼容让现有用户可以平滑升级。Vue3虽然有很多breaking change，但官方提供了迁移工具，帮助用户从Vue2升级。

## 六、参与开源贡献

### 6.1 如何选择贡献项目

选择贡献项目时应该考虑以下因素：

项目的活跃度很重要。活跃的项目意味着维护者会及时审查你的PR，问题会被快速修复。可以查看项目的commit历史、issue数量、PR数量等判断活跃度。

自己的兴趣和能力是选择的基础。选择自己感兴趣且有一定理解深度的项目，这样更容易坚持下去。

社区氛围也值得关注。友好的社区可以让贡献过程更加愉快。可以通过查看issue中的交流方式、PR的审查意见等了解社区氛围。

### 6.2 贡献流程详解

贡献开源项目的一般流程：

第一步，在GitHub上fork项目到自己的账户。

第二步，克隆fork后的仓库到本地，创建开发分支。建议使用有意义的分支名，如feature/add-xxx、fix/xxx等。

第三步，在本地进行开发。遵循项目的代码规范，编写测试用例，确保新功能有测试覆盖。

第四步，提交代码并push到远程。提交信息应该清晰描述做了什么，遵循项目的提交信息规范。

第五步，在GitHub上创建PR。详细描述PR的目的、解决的问题、测试情况等。

第六步，等待维护者审查。如果有审查意见，礼貌地进行讨论和修改。

第七步，PR被合并后，可以删除本地分支并同步最新的代码。

### 6.3 贡献注意事项

贡献时需要注意以下事项：

仔细阅读项目的CONTRIBUTING指南。这个文件详细说明了项目接受的贡献类型、代码规范、测试要求等。

遵守代码规范。如果项目使用ESLint、Prettier等工具，确保提交前运行这些工具。

编写测试。完善的测试可以提高代码质量，也是对其他开发者的负责。

描述清楚PR。说明为什么需要这个功能或修改，提供必要的上下文和测试结果。

保持谦逊和耐心。审查者可能提出一些你不同意的意见，但请相信他们的判断是有道理的。保持开放的心态讨论问题。

## 七、总结与展望

### 7.1 手写框架的学习收获

通过手写框架，学习者可以获得多方面的收获：

深入理解框架原理。手写实现让抽象的概念变得具体，帮助真正理解框架的工作方式，而不仅仅是停留在"会用"的层面。

提升编程能力。实现一个框架需要处理各种边界情况，进行性能优化，设计良好的API，这些都可以显著提升编程能力。

学习设计模式。成熟框架中大量使用了设计模式，如观察者模式、发布订阅模式、工厂模式、策略模式等，通过手写实现可以深入理解这些模式的应用。

建立信心。当看到自己实现的功能与成熟框架的行为一致时，会产生强烈的成就感，这种信心会推动你在技术上不断挑战更高的目标。

### 7.2 持续精进之路

手写框架只是学习的起点，持续精进需要：

不断学习新技术。前端技术发展迅速，新的框架、新的工具层出不穷。保持好奇心，持续学习新技术。

深入研究经典技术。对于React、Vue、Webpack等经典技术，可以不断深入研究，从使用到原理，从原理到源码，从源码到贡献。

分享知识。写博客、做演讲是很好的学习方式。教授他人可以加深自己的理解，也可以结识志同道合的开发者。

参与开源。从提交第一个PR开始，逐步参与到更核心的开发中，在社区中成长。

### 7.3 推荐学习路径

以下是推荐的手写框架学习路径：

第一步，从Mini Vue响应式系统开始。响应式系统概念清晰、实现有趣，是入门的好选择。

第二步，实现Mini React的Virtual DOM和组件系统。Virtual DOM是React的核心概念，通过实现可以深入理解其工作原理。

第三步，实现一些基础工具函数，如debounce、throttle、deepClone等，巩固JavaScript基础。

第四步，挑战更复杂的项目，如实现一个简易的Redux、一个简易的Router等。

第五步，阅读成熟框架的源码，如Vue3、React18的响应式系统，理解它们是如何处理各种边界情况的。

第六步，尝试为开源项目贡献代码，将学习成果分享给社区。

手写框架是一个需要长期坚持的过程，不可能一蹴而就。希望本指南能为你的学习之路提供一些帮助。记住，最重要的是开始动手写，并且持续完善。祝你学习愉快！
