## 2. 单元测试 vs E2E测试

### 2.1 单元测试 (Unit Testing)

**定义**: 测试独立的最小代码单元（函数、组件）

**特点**:
- 测试范围小，定位问题快
- 运行速度快
- Mock 外部依赖
- 数量最多

**适用场景**:
- 工具函数
- 业务逻辑
- 组件逻辑

```typescript
// utils/format.ts
export function formatCurrency(amount: number, currency = 'CNY'): string {
  return new Intl.NumberFormat('zh-CN', {
    style: 'currency',
    currency,
  }).format(amount)
}

// utils/format.test.ts
import { describe, it, expect } from 'vitest'
import { formatCurrency } from './format'

describe('formatCurrency', () => {
  it('should format CNY correctly', () => {
    expect(formatCurrency(100)).toBe('¥100.00')
  })

  it('should format USD correctly', () => {
    expect(formatCurrency(100, 'USD')).toBe('$100.00')
  })

  it('should handle decimal amounts', () => {
    expect(formatCurrency(99.99)).toBe('¥99.99')
  })
})
```

### 2.2 集成测试 (Integration Testing)

**定义**: 测试多个单元的协作

**特点**:
- 测试组件交互
- 可能需要 Mock API
- 中等速度

```typescript
// 集成测试示例: 测试组件与 store 的交互
import { render, screen, act } from '@testing-library/react'
import { Provider } from 'react-redux'
import { configureStore } from '@reduxjs/toolkit'
import userReducer from '@/store/userSlice'
import { UserProfile } from './UserProfile'

function renderWithStore(ui: React.ReactElement) {
  const store = configureStore({
    reducer: { user: userReducer },
  })

  return {
    ...render(<Provider store={store}>{ui}</Provider>),
    store,
  }
}

describe('UserProfile', () => {
  it('should display user data from store', async () => {
    // 模拟登录用户
    const { store } = renderWithStore(<UserProfile />)

    await act(async () => {
      store.dispatch(setUser({ name: '张三', email: 'test@example.com' }))
    })

    expect(screen.getByText('张三')).toBeInTheDocument()
  })
})
```

### 2.3 E2E 测试 (End-to-End Testing)

**定义**: 模拟真实用户行为，从头测到尾

**特点**:
- 真实浏览器环境
- 测试完整业务流程
- 运行最慢，最昂贵
- 数量最少

**适用场景**:
- 关键用户路径
- 登录/注册流程
- 购物/支付流程

```typescript
// e2e/login.spec.ts
import { test, expect } from '@playwright/test'

test('用户登录流程', async ({ page }) => {
  // 1. 打开登录页
  await page.goto('http://localhost:33980/login')

  // 2. 输入用户名密码
  await page.fill('[name="username"]', 'testuser')
  await page.fill('[name="password"]', 'password123')

  // 3. 点击登录按钮
  await page.click('button[type="submit"]')

  // 4. 等待跳转并验证
  await expect(page).toHaveURL('http://localhost:33980/dashboard')

  // 5. 验证用户信息显示
  await expect(page.locator('.user-name')).toContainText('testuser')
})
```

### 2.4 测试类型对比

| 特性 | 单元测试 | 集成测试 | E2E 测试 |
|------|---------|---------|----------|
| 范围 | 单个函数/组件 | 多个组件/模块 | 整个应用 |
| 速度 | 快 (ms) | 中 (s) | 慢 (min) |
| 可靠性 | 高 | 中 | 低 |
| Mock | 完全 Mock | 部分 Mock | 无 Mock |
| 数量 | 多 | 中 | 少 |

## 3. Vitest 基础

### 3.1 Vitest 简介

Vitest 是一个基于 Vite 的测试框架，与 Vite 项目无缝集成，支持：
- 极速的 HMR
- 开箱即用的 TypeScript
- Jest 兼容 API
- 内置覆盖率

### 3.2 配置文件

```typescript
// vitest.config.ts
import { defineConfig } from 'vitest/config'

import path from 'path'

  plugins: [react()],

        'src/test/',

    },
  },
  resolve: {
```

### 3.6 Mock 函数

```typescript

// 创建 Mock 函数
const mockFn = vi.fn()

mockFn('hello')

expect(mockFn).toHaveBeenCalledWith('hello') // 用指定参数调用
expect(mockFn).toHaveBeenCalledTimes(1)   // 调用次数

// Mock 实现
const mockReturn = vi.fn(() => 'mocked')
mockReturn() // 'mocked'

// Mock 异步
const mockAsync = vi.fn(() => Promise.resolve('data'))

// Mock getter
const obj = {
  get value() {
    return 'original'
  }
}

vi.spyOn(obj, 'value', 'get').mockReturnValue('mocked')

// 清除 Mock
mockFn.mockClear()   // 清除调用记录
mockFn.mockReset()    // 清除并重置实现
mockFn.mockRestore()  // 恢复到原始实现
```

## 5. 测试组件

### 5.1 测试按钮组件

```typescript
// Button.tsx
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary'
  isLoading?: boolean
}

export function Button({
  variant = 'primary',
  isLoading = false,
  children,
  disabled,
  ...props
}: ButtonProps) {
  return (
    <button
      className={`btn btn-${variant}`}
      disabled={disabled || isLoading}
      {...props}
    >
      {isLoading ? <span className="spinner" /> : children}
    </button>
  )
}

// Button.test.tsx
import { describe, it, expect, vi } from 'vitest'
import { render, screen, fireEvent } from '@testing-library/react'
import { Button } from './Button'

describe('Button', () => {
  it('should render with text', () => {
    render(<Button>Click me</Button>)
    expect(screen.getByRole('button', { name: /click me/i })).toBeInTheDocument()
  })

  it('should handle click events', () => {
    const handleClick = vi.fn()
    render(<Button onClick={handleClick}>Click me</Button>)

    fireEvent.click(screen.getByRole('button'))
    expect(handleClick).toHaveBeenCalledTimes(1)
  })

  it('should show loading state', () => {
    render(<Button isLoading>Submit</Button>)

    expect(screen.queryByText('Submit')).not.toBeInTheDocument()
    expect(screen.getByText('spinner')).toBeInTheDocument()
  })

  it('should be disabled when loading', () => {
    render(<Button isLoading>Submit</Button>)

    const button = screen.getByRole('button')
    expect(button).toBeDisabled()
  })

  it('should apply variant classes', () => {
    const { rerender } = render(<Button variant="primary">Primary</Button>)
    expect(screen.getByRole('button')).toHaveClass('btn-primary')

    rerender(<Button variant="secondary">Secondary</Button>)
    expect(screen.getByRole('button')).toHaveClass('btn-secondary')
  })
})
```

### 5.2 测试表单组件

```typescript
// LoginForm.tsx
interface LoginFormProps {
  onSubmit: (data: { username: string; password: string }) => void
}

export function LoginForm({ onSubmit }: LoginFormProps) {
  const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault()
    const formData = new FormData(e.currentTarget)
    onSubmit({
      username: formData.get('username') as string,
      password: formData.get('password') as string,
    })
  }

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label htmlFor="username">用户名</label>
        <input id="username" name="username" type="text" required />
      </div>
      <div>
        <label htmlFor="password">密码</label>
        <input id="password" name="password" type="password" required />
      </div>
      <button type="submit">登录</button>
    </form>
  )
}

// LoginForm.test.tsx
import { describe, it, expect, vi } from 'vitest'
import { render, screen, fireEvent, waitFor } from '@testing-library/react'
import userEvent from '@testing-library/user-event'
import { LoginForm } from './LoginForm'

describe('LoginForm', () => {
  it('should render form fields', () => {
    render(<LoginForm onSubmit={vi.fn()} />)

    expect(screen.getByLabelText(/用户名/)).toBeInTheDocument()
    expect(screen.getByLabelText(/密码/)).toBeInTheDocument()
    expect(screen.getByRole('button', { name: /登录/ })).toBeInTheDocument()
  })

  it('should call onSubmit with form data', async () => {
    const handleSubmit = vi.fn()
    const user = userEvent.setup()

    render(<LoginForm onSubmit={handleSubmit} />)

    await user.type(screen.getByLabelText(/用户名/), 'testuser')
    await user.type(screen.getByLabelText(/密码/), 'password123')
    await user.click(screen.getByRole('button', { name: /登录/ }))

    await waitFor(() => {
      expect(handleSubmit).toHaveBeenCalledWith({
        username: 'testuser',
        password: 'password123',
      })
    })
  })

  it('should show validation error for empty fields', async () => {
    const user = userEvent.setup()

    render(<LoginForm onSubmit={vi.fn()} />)

    await user.click(screen.getByRole('button', { name: /登录/ }))

    // 浏览器原生验证会阻止提交
    expect(screen.getByLabelText(/用户名/)).toBeInvalid()
  })
})
```

### 5.3 测试异步数据加载

```typescript
// UserList.tsx
import { useEffect, useState } from 'react'

interface User {
  id: number
  name: string
}

export function UserList() {
  const [users, setUsers] = useState<User[]>([])
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState<string | null>(null)

  useEffect(() => {
    fetch('/api/users')
      .then(res => res.json())
      .then(data => {
        setUsers(data)
        setLoading(false)
      })
      .catch(err => {
        setError(err.message)
        setLoading(false)
      })
  }, [])

  if (loading) return <div data-testid="loading">Loading...</div>
  if (error) return <div data-testid="error">{error}</div>

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  )
}

// UserList.test.tsx
import { describe, it, expect, vi } from 'vitest'
import { render, screen, waitFor } from '@testing-library/react'
import { UserList } from './UserList'

// Mock fetch
global.fetch = vi.fn()

const mockUsers = [
  { id: 1, name: '张三' },
  { id: 2, name: '李四' },
]

describe('UserList', () => {
  beforeEach(() => {
    vi.clearAllMocks()
  })

  it('should show loading state initially', () => {
    // 模拟 pending 状态的 fetch
    vi.mocked(fetch).mockImplementation(() => new Promise(() => {}))

    render(<UserList />)
    expect(screen.getByTestId('loading')).toBeInTheDocument()
  })

  it('should render users after loading', async () => {
    vi.mocked(fetch).mockResolvedValue({
      json: () => Promise.resolve(mockUsers),
    } as Response)

    render(<UserList />)

    await waitFor(() => {
      expect(screen.getByText('张三')).toBeInTheDocument()
      expect(screen.getByText('李四')).toBeInTheDocument()
    })
  })

  it('should show error on fetch failure', async () => {
    vi.mocked(fetch).mockRejectedValue(new Error('Network error'))

    render(<UserList />)

    await waitFor(() => {
      expect(screen.getByTestId('error')).toHaveTextContent('Network error')
    })
  })
})
```

## 常见错误和调试技巧

### 错误 1: "Not wrapped in act(...)"

```typescript
// ❌ 错误: 状态更新未在 act 中
import { act } from 'react'

test('should update state', () => {
  render(<Counter />)

  // 需要包裹在 act 中
  fireEvent.click(screen.getByText('Increment'))

  expect(screen.getByText('1')).toBeInTheDocument()
})

// ✅ 正确: 使用 act
test('should update state', async () => {
  render(<Counter />)

  await act(async () => {
    fireEvent.click(screen.getByText('Increment'))
  })

  expect(screen.getByText('1')).toBeInTheDocument()
})
```

### 错误 2: "Cannot find module"

```typescript
// ❌ 错误: 路径别名未配置
import { Button } from '@/components/Button'

// ✅ 正确: 配置 Vitest 路径别名
// vitest.config.ts
export default defineConfig({
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```

### 错误 3: "Timeout - Async callback was not invoked"

```typescript
// ❌ 错误: 异步测试未正确处理
test('should fetch data', () => {
  fetch('/api/data').then(data => {
    expect(data).toBeDefined()
  })
})

// ✅ 正确: 使用 async/await 或返回 Promise
test('should fetch data', async () => {
  const data = await fetch('/api/data')
  expect(data).toBeDefined()
})

// 或返回 Promise
test('should fetch data', () => {
  return fetch('/api/data').then(data => {
    expect(data).toBeDefined()
  })
})
```

### 错误 4: Playwright "element not visible"

```typescript
// ❌ 错误: 元素不可见时操作
await page.click('button#hidden')

// ✅ 正确: 先等待元素可见
await expect(page.locator('button#hidden')).toBeVisible()
await page.click('button#hidden')

// 或使用 force: true
await page.click('button#hidden', { force: true })
```

## 总结

测试是保证代码质量的关键手段：

1. **单元测试** - 快速反馈，定位问题
2. **集成测试** - 验证组件协作
3. **E2E 测试** - 端到端验证用户体验

掌握以下工具：
- **Vitest** - 快速、现代化的单元测试框架
- **React Testing Library** - 以用户为中心的组件测试
- **Playwright** - 强大的 E2E 测试工具

遵循测试最佳实践，让你的代码更可靠！
