# 命令行工具速查

> 本文档是 WebEnv-OS 项目开发中常用命令行工具的快速参考指南，涵盖 Git、npm/yarn、Node.js、Docker 等工具的常用命令。每个工具都配有详细的说明和示例，帮助开发者快速查找所需命令。所有命令都基于项目实际使用场景进行整理，确保实用性。开发过程中遇到问题时，首先查阅本指南可以快速找到解决方案。

## 1. Git 版本控制

### 1.1 基础配置命令

Git 配置是使用 Git 的第一步，合理的配置可以提升开发体验。配置分为系统级（--system）、用户级（--global）、仓库级（--local）三个作用域，优先级依次递增。

配置用户名和邮箱是必须完成的配置，这会影响提交记录的身份信息。执行 `git config --global user.name "Your Name"` 可以设置用户名；执行 `git config --global user.email "your.email@example.com"` 可以设置邮箱。

配置默认分支名可以使用 `git config --global init.defaultBranch master`，将默认分支设置为 master（也可以设置为 main）。

配置别名可以简化常用命令。例如，`git config --global alias.st status` 将 status 命令简化为 st；`git config --global alias.co checkout` 将 checkout 命令简化为 co；`git config --global alias.br branch` 将 branch 命令简化为 br；`git config --global alias.last "log -1 HEAD"` 显示最后一次提交信息。

配置拉取策略可以使用 `git config --global pull.rebase false` 设置 pull 时使用 merge 方式；`git config --global pull.rebase true` 设置 pull 时使用 rebase 方式。

查看所有配置可以使用 `git config --list --show-origin`，这会显示所有配置项及其来源文件。

### 1.2 仓库操作命令

初始化新仓库使用 `git init` 命令，会在当前目录创建 .git 文件夹。如果需要指定目录，可以在命令后加上目录名，如 `git init my-project`。

克隆现有仓库使用 `git clone <仓库地址>` 命令。例如，`git clone https://github.com/user/repo.git` 会克隆远程仓库到本地。如果需要指定本地目录名，可以在仓库地址后加上目录名，如 `git clone https://github.com/user/repo.git my-folder`。

查看仓库状态使用 `git status` 命令，显示工作区、暂存区的文件状态。常用参数包括 `-s` 或 `--short` 以简洁格式显示；`-b` 显示分支信息。

查看远程仓库使用 `git remote -v` 命令，显示远程仓库的详细信息，包括fetch和push的URL。如果需要添加远程仓库，可以使用 `git remote add origin <仓库地址>`。

### 1.3 文件操作命令

将文件添加到暂存区使用 `git add <文件>` 命令。常用变体包括：`git add .` 将所有修改添加到暂存区；`git add -A` 与 `git add .` 类似，但包括删除的文件；`git add -p` 交互式添加，允许选择性地添加文件的特定部分。

提交暂存区的文件使用 `git commit -m "提交信息"` 命令。提交信息应当简洁描述本次修改的内容。常用参数包括 `-a` 自动提交所有已跟踪文件的修改；`--amend` 修改最后一次提交。

查看文件差异使用 `git diff` 命令。常用变体包括：`git diff` 显示工作区与暂存区的差异；`git diff --cached` 显示暂存区与最后一次提交的差异；`git diff HEAD` 显示工作区与最后一次提交的差异；`git diff <分支1> <分支2>` 显示两个分支之间的差异。

查看历史记录使用 `git log` 命令。常用参数包括 `--oneline` 以简洁格式显示；`-n` 显示最近 n 条记录；`-p` 显示每次提交的详细差异；`--graph` 显示分支合并图。

### 1.4 分支操作命令

查看分支使用 `git branch` 命令。常用参数包括 `-a` 显示所有分支（包括远程）；`-r` 显示远程分支；`-v` 显示每个分支的最后一次提交。

创建分支使用 `git branch <分支名>` 命令，这只创建分支，不会切换到该分支。

切换分支使用 `git checkout <分支名>` 命令。Git 2.23 版本后推荐使用 `git switch <分支名>`。创建并切换分支可以使用 `git checkout -b <分支名>` 或 `git switch -c <分支名>`。

合并分支使用 `git merge <分支名>` 命令，将指定分支合并到当前分支。如果有冲突，需要手动解决后提交。

删除分支使用 `git branch -d <分支名>` 命令，这会删除本地分支。如果分支未合并，使用 `-d` 会失败，此时可以使用 `-D` 强制删除。

### 1.5 远程操作命令

从远程仓库拉取最新代码使用 `git fetch` 命令，这只获取数据但不合并。然后可以使用 `git merge` 或 `git rebase` 合并到本地分支。

拉取并合并使用 `git pull` 命令，相当于 `git fetch` 加 `git merge`。如果配置了 rebase，则相当于 `git fetch` 加 `git rebase`。

推送本地分支到远程仓库使用 `git push <远程名> <分支名>` 命令。常用参数包括 `-u` 设置上游分支；`-f` 强制推送（谨慎使用）；`--delete` 删除远程分支。

查看远程仓库详细信息使用 `git remote show origin` 命令，显示远程仓库的 URL、跟踪分支、最新提交等信息。

### 1.6 撤销操作命令

撤销工作区修改使用 `git checkout -- <文件>` 命令，将文件恢复到最后一次提交的状态。Git 2.23 版本后推荐使用 `git restore <文件>`。

撤销暂存区操作使用 `git reset HEAD <文件>` 命令，将文件从暂存区移除，但保留工作区的修改。如果需要同时撤销工作区修改，可以加上 `--hard` 参数。

撤销提交有多种方式。使用 `git revert <提交>` 命令会创建一个新的提交来撤销指定提交的修改，是安全的撤销方式。使用 `git reset --soft HEAD~1` 可以撤销最后一次提交，但保留修改在暂存区；`git reset --mixed HEAD~1`（默认）会保留修改在工作区；`git reset --hard HEAD~1` 会完全撤销最后一次提交，慎用。

### 1.7 储藏操作命令

储藏工作区修改使用 `git stash` 命令，将当前工作区的修改保存起来，常用于临时切换分支。常用参数包括 `save "消息"` 添加储藏消息；`list` 查看所有储藏；`pop` 应用最近的储藏并删除；`apply` 应用储藏但保留；`drop` 删除最近的储藏；`clear` 删除所有储藏。

### 1.8 标签操作命令

创建标签使用 `git tag <标签名>` 命令，创建轻量标签。添加注释标签使用 `git tag -a <标签名> -m "标签消息"` 命令。

查看标签使用 `git tag` 命令，列出所有标签；`git show <标签名>` 显示标签详情。

推送标签使用 `git push origin <标签名>` 命令推送单个标签；`git push origin --tags` 推送所有标签。

删除标签使用 `git tag -d <标签名>` 命令删除本地标签；`git push origin --delete <标签名>` 删除远程标签。

## 2. npm 包管理器

### 2.1 基础命令

初始化项目使用 `npm init` 命令，会在当前目录创建 package.json 文件。常用参数包括 `-y` 使用默认配置快速初始化；`--scope` 创建作用域包。

安装依赖包使用 `npm install <包名>` 命令，会将包安装到 node_modules 目录，并更新 package.json 的 dependencies。常用参数包括 `--save` 或默认将包添加到 dependencies；`--save-dev` 将包添加到 devDependencies；`--save-exact` 精确版本安装；`-g` 全局安装；`--legacy-peer-deps` 解决依赖冲突。

安装 package.json 中定义的所有依赖使用 `npm install` 或 `npm i` 命令。

更新包使用 `npm update <包名>` 命令，会更新到满足 package.json 范围内的新版本。全局包使用 `npm update -g <包名>` 命令。

卸载包使用 `npm uninstall <包名>` 命令，会从 node_modules 中删除并更新 package.json。

### 2.2 查看与搜索命令

查看已安装的包使用 `npm list` 命令。常用参数包括 `--depth=0` 只显示顶层依赖；`--depth=1` 显示一层依赖；`--global` 显示全局安装的包。

查看包信息使用 `npm view <包名>` 命令，显示包的详细信息，包括版本、描述、依赖等。`npm view <包名> versions` 只显示版本列表；`npm view <包名> time` 显示版本发布时间线。

搜索包使用 `npm search <关键词>` 命令，会在 npm 仓库中搜索匹配的包。

查看包的可执行文件使用 `npm bin` 命令，显示 npm 二进制文件的目录路径。

### 2.3 脚本命令

运行 package.json 中定义的脚本使用 `npm run <脚本名>` 命令。npm 提供了 pre 和 post 钩子，例如 `npm run build` 会依次执行 prebuild、build、postbuild。

查看所有可用脚本使用 `npm run` 命令，会列出 package.json 中定义的所有脚本。

常用内置脚本包括：`npm start` 运行 start 脚本（通常启动开发服务器）；`npm test` 运行 test 脚本；`npm run dev` 运行 dev 脚本（通常启动开发服务器）；`npm run build` 运行 build 脚本（通常进行生产构建）；`npm run lint` 运行 lint 脚本（通常运行代码检查）。

### 2.4 配置命令

查看 npm 配置使用 `npm config list` 命令，显示所有配置项。

设置配置使用 `npm config set <key> <value>` 命令。例如，`npm config set registry https://registry.npmjs.org/` 设置 npm 镜像源；`npm config set fund false` 禁用 npm 资助信息。

删除配置使用 `npm config delete <key>` 命令。

### 2.5 常用别名

npm 的常用简写包括：`npm i <包>` 是 `npm install <包>` 的简写；`npm i -S <包>` 是 `npm install --save <包>` 的简写；`npm i -D <包>` 是 `npm install --save-dev <包>` 的简写；`npm i -g <包>` 是 `npm install -g <包>` 的简写；`npm run` 是 `npm run-script` 的简写。

## 3. Yarn 包管理器

### 3.1 基础命令

初始化项目使用 `yarn init` 命令，创建 package.json 文件。

安装依赖使用 `yarn` 或 `yarn install` 命令，会根据 package.json 和 yarn.lock 安装所有依赖。

添加依赖使用 `yarn add <包名>` 命令，将包添加到 dependencies。使用 `--dev` 参数添加到 devDependencies；使用 `--exact` 参数精确版本安装。

移除依赖使用 `yarn remove <包名>` 命令，从项目中移除指定的包。

更新依赖使用 `yarn upgrade <包名>` 命令，更新指定的包到最新版本。

### 3.2 脚本命令

运行脚本使用 `yarn <脚本名>` 命令，与 npm run 类似。

查看所有脚本使用 `yarn run` 命令。

### 3.3 其他常用命令

查看依赖树使用 `yarn tree` 命令，显示依赖包的树形结构。

查看全局安装的包使用 `yarn global list` 命令。

清理缓存使用 `yarn cache clean` 命令。

## 4. Node.js 运行环境

### 4.1 版本管理命令

查看 Node.js 版本使用 `node -v` 或 `--version` 命令。

查看 npm 版本使用 `npm -v` 或 `--version` 命令。

运行 Node.js 脚本使用 `node <文件名>` 命令，例如 `node index.js`。

使用 nvm（Node Version Manager）管理 Node.js 版本：使用 `nvm install <版本>` 安装指定版本；使用 `nvm use <版本>` 切换到指定版本；使用 `nvm list` 列出已安装的版本；使用 `nvm current` 显示当前使用版本。

### 4.2 REPL 交互式环境

Node.js 提供了 REPL（Read-Eval-Print Loop）交互式环境，可以直接执行 JavaScript 代码。进入 REPL 使用 `node` 命令不带参数；退出使用 `.exit` 或按两次 Ctrl+C。

REPL 支持的功能包括：变量声明和使用；函数定义和调用；自动补全（使用 Tab 键）；历史命令（使用上下箭头）；加载外部文件（使用 `.load <文件>`）；清空屏幕（使用 `.clear`）。

### 4.3 全局对象与模块

Node.js 提供了一些全局对象可以直接使用，包括：console（控制台输出）、process（进程信息）、Buffer（缓冲区）、setTimeout / setInterval / setImmediate（定时器）、__dirname（当前文件所在目录）、__filename（当前文件完整路径）。

常用内置模块包括：fs（文件系统操作）、path（路径处理）、http（创建 HTTP 服务器）、url（URL 处理）、querystring（查询字符串解析）、crypto（加密功能）、events（事件处理）、util（工具函数）。

## 5. Docker 容器

### 5.1 基础命令

查看 Docker 版本使用 `docker --version` 命令。

查看 Docker 信息使用 `docker info` 命令，显示 Docker 系统信息，包括容器数量、镜像数量等。

运行容器使用 `docker run <镜像名>` 命令。常用参数包括 `-d` 后台运行；`-p <主机端口>:<容器端口>` 端口映射；`--name <容器名>` 指定容器名称；`-v <主机路径>:<容器路径>` 挂载卷；`-e <环境变量>` 设置环境变量；`--rm` 容器退出时自动删除。

列出容器使用 `docker ps` 命令。`-a` 显示所有容器（包括停止的）；`-q` 只显示容器 ID。

查看容器日志使用 `docker logs <容器名或ID>` 命令。`-f` 实时跟踪日志；`--tail n` 显示最后 n 行日志。

启动、停止、重启容器使用 `docker start <容器名或ID>`、`docker stop <容器名或ID>`、`docker restart <容器名或ID>` 命令。

删除容器使用 `docker rm <容器名或ID>` 命令。`-f` 强制删除运行中的容器。

### 5.2 镜像命令

列出镜像使用 `docker images` 命令。

拉取镜像使用 `docker pull <镜像名>` 命令。

删除镜像使用 `docker rmi <镜像名或ID>` 命令。

构建镜像使用 `docker build -t <镜像名> <Dockerfile目录>` 命令。`-t` 指定镜像名称和标签。

### 5.3 Compose 命令

Docker Compose 用于定义和运行多容器应用。常用命令包括：`docker-compose up` 启动所有服务；`-d` 后台运行；`docker-compose down` 停止并删除所有服务；`docker-compose ps` 列出服务状态；`docker-compose logs` 查看服务日志；`docker-compose build` 构建服务镜像；`docker-compose restart` 重启所有服务。

## 6. 构建与开发工具

### 6.1 Vite 命令

Vite 是 WebEnv-OS 项目使用的构建工具。开发服务器使用 `npm run dev` 或 `yarn dev` 命令，启动开发服务器并监听文件变化。

生产构建使用 `npm run build` 或 `yarn build` 命令，进行生产环境构建。

预览构建结果使用 `npm run preview` 或 `yarn preview` 命令，启动一个本地服务器预览构建结果。

类型检查使用 `npm run typecheck` 或 `yarn typecheck` 命令，进行 TypeScript 类型检查。

### 6.2 Vitest 命令

Vitest 是 WebEnv-OS 项目使用的测试框架。运行所有测试使用 `npm test` 或 `yarn test` 命令。

监听模式运行测试使用 `npm test -- --watch` 或 `yarn test --watch` 命令，文件变化时自动重新运行测试。

运行指定文件的测试使用 `npm test <文件>` 命令。

生成测试覆盖率报告使用 `npm run test:coverage` 命令。

### 6.3 ESLint 命令

代码检查使用 `npm run lint` 或 `yarn lint` 命令，检查代码是否符合规范。

修复可自动修复的问题使用 `npm run lint -- --fix` 命令。

### 6.4 TypeScript 命令

类型检查使用 `npx tsc --noEmit` 命令，检查类型但不生成文件。

编译 TypeScript 使用 `npx tsc` 命令，将 TypeScript 编译为 JavaScript。

## 7. 常用组合命令

### 7.1 项目初始化

创建新项目并安装依赖的完整流程：使用 `npm create vite@latest my-project -- --template react-ts` 创建 Vite 项目；`cd my-project` 进入项目目录；`npm install` 安装依赖；`npm run dev` 启动开发服务器。

克隆并设置项目的完整流程：使用 `git clone <仓库地址>` 克隆仓库；`npm install` 安装依赖；根据项目要求配置环境变量。

### 7.2 日常开发

日常开发中常用的命令组合：运行 `npm run dev` 启动开发服务器；`npm run build` 构建项目；`npm run lint` 检查代码；`npm test` 运行测试。

提交代码的标准流程：使用 `git status` 查看修改；`git add .` 暂存修改；`git commit -m "提交信息"` 提交修改；`git push` 推送到远程。

### 7.3 调试与排查

排查问题时常用的命令：使用 `npm list` 查看依赖安装情况；`npm outdated` 查看过期依赖；`npm doctor` 检查 npm 配置；`docker ps` 查看运行中的容器；`docker logs <容器>` 查看容器日志。

## 8. 环境变量配置

### 8.1 .env 文件

在项目根目录创建 .env 文件可以配置环境变量。常见的预定义变量包括：NODE_ENV（环境类型，development 或 production）；PORT（服务器端口）；API_URL（API 地址）；DATABASE_URL（数据库连接字符串）。

.env 文件不应提交到版本控制系统，应当在 .gitignore 中忽略。

### 8.2 常用环境变量

在 WebEnv-OS 项目中，常用的环境变量包括：NEXT_PUBLIC_API_URL（公开的 API 地址）；NEXT_PUBLIC_WS_URL（WebSocket 地址）；DATABASE_URL（数据库连接）；REDIS_URL（Redis 连接）；DOCKER_HOST（Docker 守护进程地址）。

## 9. 快捷键与别名配置

### 9.1 Git Bash 别名

可以在 ~/.bashrc 或 ~/.zshrc 中配置命令别名简化常用命令。常用的别名包括：`alias gs='git status'`；`alias ga='git add'`；`alias gc='git commit -m'`；`alias gp='git push'`；`alias gl='git pull'`；`alias gb='git branch'`；`alias gd='git diff'`；`alias gco='git checkout'`；`alias gm='git merge'`；`alias gr='git rebase'`。

### 9.2 Windows PowerShell 别名

在 PowerShell 中可以使用 Set-Alias 命令设置别名。例如 `Set-Alias gs git status`。

## 10. 常见问题解决方案

### 10.1 权限问题

遇到权限错误时，Linux/Mac 系统可以使用 `sudo` 提升权限，或者使用 `chmod` 修改文件权限。Windows 系统建议使用管理员权限打开终端。

### 10.2 网络问题

网络问题导致安装失败时，可以尝试更换镜像源。npm 使用 `npm config set registry <镜像地址>`；yarn 使用 `yarn config set registry <镜像地址>`。

### 10.3 缓存问题

遇到奇怪的安装问题时，可以尝试清理缓存。npm 使用 `npm cache clean --force`；yarn 使用 `yarn cache clean`。

### 10.4 端口占用

遇到端口占用问题时，可以使用 `netstat -ano | findstr <端口>` 查找占用端口的进程，然后使用 `taskkill /PID <进程ID> /F` 结束进程。
