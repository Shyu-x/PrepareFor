# 配置文件模板

> 本文档汇总了 WebEnv-OS 项目开发中常用的配置文件模板，涵盖 TypeScript、ESLint、Prettier、Tailwind CSS、Vitest 等工具的配置。每个配置都配有详细的说明和示例，开发者可以根据项目需求进行参考和修改。配置文件是项目工程化的重要组成部分，合理的配置可以提升开发效率和代码质量。在创建新项目或引入新工具时，可以参考本指南创建相应的配置文件。

## 1. TypeScript 配置

### 1.1 tsconfig.json

tsconfig.json 是 TypeScript 项目的核心配置文件，用于指定编译选项和项目结构。以下是适用于 WebEnv-OS 项目的推荐配置：

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "skipLibCheck": true,
    "moduleResolution": "bundler",
    "allowImportingTsExtensions": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "preserve",
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true,
    "allowSyntheticDefaultImports": true,
    "esModuleInterop": true,
    "forceConsistentCasingInFileNames": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"],
      "@/components/*": ["./src/components/*"],
      "@/hooks/*": ["./src/hooks/*"],
      "@/store/*": ["./src/store/*"],
      "@/lib/*": ["./src/lib/*"],
      "@/types/*": ["./src/types/*"]
    }
  },
  "include": ["src"],
  "references": [{ "path": "./tsconfig.node.json" }]
}
```

配置项详细说明：

target 指定编译输出的 JavaScript 版本。ES2020 是一个较好的选择，兼顾兼容性和现代特性。

lib 指定编译时包含的类型声明文件。ES2020 提供 ES2020 标准库类型；DOM 提供浏览器 API 类型；DOM.Iterable 提供可迭代 DOM 集合类型。

module 指定模块系统。ESNext 使用最新的模块语法，支持动态导入。

moduleResolution 指定模块解析策略。bundler 适用于使用 Vite、Webpack 等bundler 的项目。

jsx 指定 JSX 的处理方式。preserve 保留 JSX 源码，由构建工具处理。

strict 启用所有严格类型检查选项。建议保持开启，这有助于发现潜在问题。

noUnusedLocals 和 noUnusedParameters 检查未使用的变量和参数。有助于保持代码整洁。

paths 配置路径别名，简化导入路径。使用 @/ 开头的导入可以使路径更简洁。

### 1.2 tsconfig.node.json

用于配置 Node.js 环境下的 TypeScript 编译，通常用于 Vite 配置文件等：

```json
{
  "compilerOptions": {
    "composite": true,
    "skipLibCheck": true,
    "module": "ESNext",
    "moduleResolution": "bundler",
    "allowSyntheticDefaultImports": true,
    "strict": true
  },
  "include": ["vite.config.ts"]
}
```

composite 启用项目引用，用于增量编译。skipLibCheck 跳过库文件的类型检查，加快编译速度。

### 1.3 类型声明文件

在 src/types 目录下创建全局类型声明文件，增强类型支持。

global.d.ts 用于声明全局类型和命名空间：

```typescript
declare global {
  namespace NodeJS {
    interface ProcessEnv {
      NODE_ENV: 'development' | 'production' | 'test';
      NEXT_PUBLIC_API_URL: string;
      NEXT_PUBLIC_WS_URL: string;
    }
  }
}

export {};
```

next-env.d.ts 是 Next.js 自动生成的文件，用于支持 TypeScript 类型检查。

## 2. ESLint 配置

### 2.1 .eslintrc.json

ESLint 配置文件用于代码检查，发现和修复代码中的问题：

```json
{
  "extends": [
    "next/core-web-vitals",
    "plugin:@typescript-eslint/recommended",
    "plugin:react/recommended",
    "plugin:react-hooks/recommended",
    "prettier"
  ],
  "plugins": ["@typescript-eslint", "react", "react-hooks"],
  "rules": {
    "@typescript-eslint/no-unused-vars": ["warn", { "argsIgnorePattern": "^_" }],
    "@typescript-eslint/no-explicit-any": "warn",
    "react/react-in-jsx-scope": "off",
    "react/prop-types": "off"
  },
  "settings": {
    "react": {
      "version": "detect"
    }
  }
}
```

extends 继承已有配置。next/core-web-vitals 提供 Next.js 推荐规则；@typescript-eslint/recommended 提供 TypeScript 推荐规则；react/recommended 提供 React 推荐规则；react-hooks/recommended 提供 React Hooks 规则；prettier 禁用与 Prettier 冲突的规则。

plugins 加载需要的插件。rules 自定义规则覆盖。

## 3. Prettier 配置

### 3.1 .prettierrc

Prettier 代码格式化配置：

```json
{
  "semi": true,
  "trailingComma": "es5",
  "singleQuote": true,
  "printWidth": 100,
  "tabWidth": 2,
  "useTabs": false,
  "arrowParens": "always",
  "endOfLine": "lf",
  "bracketSpacing": true,
  "jsxSingleQuote": false
}
```

配置项说明：

semi 语句末尾分号。true 添加分号，false 不添加。

trailingComma 多行末尾逗号。es5 在 ES5 合法的位置添加逗号，all 在所有位置添加，none 不添加。

singleQuote 字符串引号。true 使用单引号，false 使用双引号。

printWidth 行宽。超过此宽度会换行。

tabWidth 缩进宽度。

useTabs 使用 Tab 缩进。true 使用 Tab，false 使用空格。

arrowParens 箭头函数参数括号。always 始终添加，avoid 不添加。

endOfLine 行尾换行符。lf 使用 \n，crlf 使用 \r\n，cr 使用 \r。

## 4. Tailwind CSS 配置

### 4.1 tailwind.config.ts

Tailwind CSS 配置文件用于自定义主题、扩展功能：

```typescript
import type { Config } from 'tailwindcss';
import defaultTheme from 'tailwindcss/defaultTheme';

export default {
  content: [
    './src/pages/**/*.{js,ts,jsx,tsx,mdx}',
    './src/components/**/*.{js,ts,jsx,tsx,mdx}',
    './src/app/**/*.{js,ts,jsx,tsx,mdx}',
  ],
  darkMode: 'class',
  theme: {
    extend: {
      colors: {
        primary: {
          50: '#f0f9ff',
          100: '#e0f2fe',
          200: '#bae6fd',
          300: '#7dd3fc',
          400: '#38bdf8',
          500: '#0ea5e9',
          600: '#0284c7',
          700: '#0369a1',
          800: '#075985',
          900: '#0c4a6e',
          950: '#082f49',
        },
      },
      fontFamily: {
        sans: ['var(--font-sans)', ...defaultTheme.fontFamily.sans],
        mono: ['var(--font-mono)', ...defaultTheme.fontFamily.mono],
      },
      animation: {
        'fade-in': 'fadeIn 0.5s ease-out',
        'slide-up': 'slideUp 0.5s ease-out',
      },
      keyframes: {
        fadeIn: {
          '0%': { opacity: '0' },
          '100%': { opacity: '1' },
        },
        slideUp: {
          '0%': { transform: 'translateY(10px)', opacity: '0' },
          '100%': { transform: 'translateY(0)', opacity: '1' },
        },
      },
    },
  },
  plugins: [],
} satisfies Config;
```

content 指定扫描的文件路径，用于生成对应的 CSS。

darkMode 暗色模式支持方式。class 使用 CSS 类控制，media 使用媒体查询控制。

theme.extend 扩展默认主题。可以添加自定义颜色、字体、动画等。

### 4.2 postcss.config.js

PostCSS 配置文件用于配置 PostCSS 插件：

```javascript
module.exports = {

    tailwindcss: {},
    autoprefixer: {},
  },

```

tailwindcss 处理 Tailwind CSS；autoprefixer 自动添加浏览器前缀。

## 5. Vitest 配置

### 5.1 vitest.config.ts

Vitest 测试框架配置文件：

```typescript
import { defineConfig } from 'vitest/config';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  test: {
    globals: true,
    environment: 'jsdom',
    setupFiles: ['./src/test/setup.ts'],
    include: ['src/**/*.{test,spec}.{js,mjs,cjs,ts,mts,cts,jsx,tsx}'],
    coverage: {
      provider: 'v8',
      reporter: ['text', 'json', 'html'],
      exclude: [
        'node_modules/',
        'src/test/',
        '**/*.d.ts',
        '**/*.config.*',
        '**/coverage/**',
      ],
    },
  },
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
      '@/components': path.resolve(__dirname, './src/components'),
      '@/hooks': path.resolve(__dirname, './src/hooks'),
      '@/store': path.resolve(__dirname, './src/store'),
      '@/lib': path.resolve(__dirname, './src/lib'),
      '@/types': path.resolve(__dirname, './src/types'),
    },
  },
});
```

plugins 加载需要的插件。react() 支持 React 测试。

test.globals 全局可用测试 API，如 describe、it、expect 等。

test.environment 测试环境。jsdom 模拟浏览器环境，node 模拟 Node.js 环境。

test.setupFiles 测试前运行的配置文件。

test.include 测试文件匹配模式。

test.coverage 配置测试覆盖率。

resolve.alias 配置路径别名，与 TypeScript 配置保持一致。

### 5.2 src/test/setup.ts

测试配置文件，设置测试环境：

```typescript
import '@testing-library/jest-dom';
import { cleanup } from '@testing-library/react';
import { afterEach } from 'vitest';

afterEach(() => {
  cleanup();
});
```

@testing-library/jest-dom 提供自定义的 Jest 匹配器。cleanup 清理测试后的 DOM，避免测试间相互影响。

## 6. Vite 配置

### 6.1 vite.config.ts

Vite 配置文件用于开发服务器和构建优化：

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
      '@/components': path.resolve(__dirname, './src/components'),
      '@/hooks': path.resolve(__dirname, './src/hooks'),
      '@/store': path.resolve(__dirname, './src/store'),
      '@/lib': path.resolve(__dirname, './src/lib'),
      '@/types': path.resolve(__dirname, './src/types'),
    },
  },
  server: {
    port: 3000,
    open: true,
    proxy: {
      '/api': {
        target: 'http://localhost:3001',
        changeOrigin: true,
      },
    },
  },
  build: {
    sourcemap: false,
    rollupOptions: {
      output: {
        manualChunks: {
          'vendor-react': ['react', 'react-dom'],
          'vendor-utils': ['lodash', 'dayjs'],
        },
      },
    },
  },
});
```

server 开发服务器配置。port 端口号；open 是否自动打开浏览器；proxy 代理配置。

build 生产构建配置。sourcemap 是否生成 sourcemap；rollupOptions.output.manualChunks 代码分割配置。

## 7. Next.js 配置

### 7.1 next.config.js

Next.js 配置文件：

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactStrictMode: true,
  swcMinify: true,
  images: {
    domains: ['example.com'],
    formats: ['image/avif', 'image/webp'],
  },
  env: {
    NEXT_PUBLIC_API_URL: process.env.NEXT_PUBLIC_API_URL,
  },
  async rewrites() {
    return [
      {
        source: '/api/:path*',
        destination: 'http://localhost:3001/api/:path*',
      },
    ];
  },
};

module.exports = nextConfig;
```

reactStrictMode 启用 React 严格模式，帮助发现潜在问题。

swcMinify 使用 SWC 进行压缩，提高构建速度。

images 图片优化配置。domains 允许加载图片的域名；formats 支持的图片格式。

env 环境变量配置，注入到客户端。

rewrites URL 重写配置，用于代理请求。

## 9. Docker 配置

### 9.1 Dockerfile

Docker 镜像构建配置：

```dockerfile
FROM node:20-alpine AS base

# Install dependencies only when needed
FROM base AS deps
WORKDIR /app
COPY package.json package-lock.json* ./
RUN npm ci

# Rebuild the source code only when needed
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# Production image, copy all the files and run next
FROM base AS runner
WORKDIR /app

ENV NODE_ENV production

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs

EXPOSE 3000

ENV PORT 3000

CMD ["node", "server.js"]
```

多阶段构建优化镜像大小。base 基础镜像；deps 依赖安装阶段；builder 构建阶段；runner 运行阶段。
