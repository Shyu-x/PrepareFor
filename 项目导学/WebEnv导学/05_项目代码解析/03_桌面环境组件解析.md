# 桌面环境组件解析

本文档详细解析 WebEnv-OS 项目中桌面环境组件的实现原理，包括 Desktop、WindowManager、Dock、Taskbar 和窗口系统。

## 1. Desktop 桌面组件解析

### 1.1 功能概述

Desktop 组件是桌面环境的核心，负责显示桌面壁纸、桌面图标和右键菜单。

### 1.2 组件接口

```tsx
interface DesktopProps {
  wallpaper?: DesktopWallpaper;  // 壁纸（渐变、图片、视频）
  icons?: DesktopIcon[];        // 桌面图标
  onIconClick?: (id: string) => void;    // 图标点击回调
  onDesktopClick?: () => void;          // 桌面点击回调
  onContextMenu?: (e: React.MouseEvent) => void; // 右键菜单回调
  className?: string;
  style?: React.CSSProperties;
}

interface DesktopIcon {
  id: string;
  label: string;
  icon: ReactNode;
  onClick?: () => void;
  position: { x: number; y: number };
}

type DesktopWallpaper =
  | string  // 渐变字符串
  | {
      type: "gradient" | "image" | "video";
      src: string;
      poster?: string;
    };
```

### 1.3 壁纸系统

Desktop 支持三种壁纸类型：

```tsx

// 视频壁纸
{resolvedWallpaper.type === "video" && (
```

## 2. WindowManager 窗口管理器

### 2.1 功能概述

WindowManager 是桌面环境的核心组件，负责管理所有窗口的创建、关闭、最小化、最大化、激活等操作。

### 2.2 窗口接口定义

```tsx
interface WindowItem {
  id: string;
  appId?: string;              // 应用 ID
  title: string;               // 窗口标题
  content: ReactNode;         // 窗口内容
  visible: boolean;            // 是否可见
  minimized: boolean;         // 是否最小化
  maximized: boolean;         // 是否最大化
  position: { x: number; y: number };  // 位置
  size: { width: number; height: number }; // 尺寸
  minSize?: { width: number; height: number }; // 最小尺寸
  zIndex: number;              // z-index（层叠顺序）
  windowControls?: "windows" | "macos" | "linux"; // 窗口控制按钮风格
}
```

### 2.3 API 接口

```tsx
export interface WindowManagerApi {
  createWindow: (id: string, title: string, content: ReactNode, options?: Partial<WindowItem>) => void;
  closeWindow: (id: string) => void;
  minimizeWindow: (id: string) => void;
  maximizeWindow: (id: string) => void;
  restoreWindow: (id: string) => void;
  hasWindow: (appId: string) => boolean;
  focusWindow: (appId: string) => void;
  closeWindowsByAppId: (appId: string) => void;
  getActiveWindow: () => WindowItem | null;
  getMinimizedWindows: () => WindowItem[];
  getVisibleWindows: () => WindowItem[];
  windowControls: "windows" | "macos" | "linux";
}
```

### 2.4 窗口创建逻辑

```tsx
const createWindow = useCallback((
  id: string,
  title: string,
  content: ReactNode,
  options?: Partial<WindowItem>,
) => {
  setWindows((prev) => {
    const newWindow: WindowItem = {
      id,
      appId: options?.appId,
      title,
      content,
      visible: true,
      minimized: false,
      maximized: false,
      position: options?.position || {
        x: 100 + prev.length * 20,  // 级联偏移
        y: 100 + prev.length * 20,
      },
      size: options?.size || { width: 600, height: 400 },
      zIndex: 100 + prev.length,
      windowControls: options?.windowControls || windowControls,
      ...options,
    };

    // 计算最大 zIndex，确保新窗口在最上层
    const maxZIndex = Math.max(...prev.map((w) => w.zIndex), 100);
    newWindow.zIndex = maxZIndex + 1;

    return [...prev, newWindow];
  });

  setActiveWindowId(id);
}, [windowControls]);
```

### 2.5 窗口激活（置顶）

```tsx
const activateWindow = useCallback((id: string) => {
  setActiveWindowId(id);
  setWindows((prev) => {
    const maxZIndex = Math.max(...prev.map((w) => w.zIndex), 100);

    // 如果已经在最上层，不更新
    const current = prev.find(w => w.id === id);
    if (current && current.zIndex === maxZIndex) return prev;

    // 将被激活的窗口 zIndex 设为最大
    return prev.map((w) =>
      w.id === id ? { ...w, zIndex: maxZIndex + 1 } : w
    );
  });
}, []);
```

### 2.6 窗口状态管理

```tsx
// 最小化窗口
const minimizeWindow = useCallback((id: string) => {
  setWindows((prev) =>
    prev.map((w) => (w.id === id ? { ...w, minimized: true } : w))
  );
}, []);

// 最大化/还原窗口
const maximizeWindow = useCallback((id: string) => {
  setWindows((prev) =>
    prev.map((w) => (w.id === id ? { ...w, maximized: !w.maximized } : w))
  );
}, []);

// 关闭窗口
const closeWindow = useCallback((id: string) => {
  setWindows((prev) => prev.filter((w) => w.id !== id));
}, []);
```

## 3. Window 窗口组件

### 3.1 功能概述

Window 组件是单个窗口的渲染组件，处理拖拽、缩放、窗口控制按钮等功能。

### 3.2 核心功能

- 窗口拖拽移动
- 窗口缩放调整大小
- 窗口控制按钮（关闭、最大化、最小化）
- 窗口内容渲染

### 3.3 窗口控制按钮样式

支持三种系统风格：

```tsx
// Windows 风格：关闭、最大化、最小化（从右到左）
const windowsControls = [
  { icon: <MinusOutlined />, action: onMinimize, title: "最小化" },
  { icon: <FullscreenOutlined />, action: onMaximize, title: "最大化" },
  { icon: <CloseOutlined />, action: onClose, title: "关闭" },
];

// macOS 风格：关闭、最大化、最小化（从左到右，彩色）
const macosControls = [
  { icon: <CloseOutlined />, action: onClose, title: "关闭", color: "#ff5f57" },
  { icon: <MinusOutlined />, action: onMinimize, title: "最小化", color: "#febc2e" },
  { icon: <FullscreenOutlined />, action: onMaximize, title: "最大化", color: "#28c840" },
];

// Linux 风格：类似 Windows
```

## 4. Dock 底部栏

### 4.1 功能概述

Dock 组件是类似 macOS 的 Dock 栏，提供应用快速启动功能，支持鼠标悬停放大动画。

### 4.2 组件接口

```tsx
interface DockApp {
  id: string;
  name: string;
  icon: React.ReactNode;
  onClick: () => void;
  badge?: number;           // 徽章数字
  openCount?: number;      // 打开的窗口数
  isActive?: boolean;      // 是否激活
  onCloseAll?: () => void; // 关闭所有窗口
}

interface DockProps {
  apps: DockApp[];
  position?: "bottom" | "left" | "right";
  className?: string;
}
```

### 4.3 悬停放大效果

```tsx
// 计算缩放效果
const calculateScale = (index: number, hoveredIndex: number | null) => {
  if (hoveredIndex === null) return 1;
  const distance = Math.abs(index - hoveredIndex);
  if (distance === 0) return 1.5;  // 当前项放大 1.5 倍
  if (distance === 1) return 1.2;  // 邻近项放大 1.2 倍
  if (distance === 2) return 1.1;  // 邻近的邻近放大 1.1 倍
  return 1;
};

// 应用缩放
<div
  style={{
    transform: `scale(${currentScale})`,
    transformOrigin: 'bottom center'  // 从底部中心缩放
  }}
>
  {app.icon}
</div>
```

### 4.4 位置配置

```tsx
const getDockPosition = () => {
  switch (position) {
    case "bottom":
      return {
        bottom: 16,
        left: "50%",
        transform: "translateX(-50%)",
        flexDirection: "row",
      };
    case "left":
      return {
        left: 8,
        top: "50%",
        transform: "translateY(-50%)",
        flexDirection: "column",
      };
    case "right":
      return {
        right: 8,
        top: "50%",
        transform: "translateY(-50%)",
        flexDirection: "column",
      };
  }
};
```

### 4.5 状态指示

```tsx
// 打开指示点
{hasOpen && (
```

## 5. Taskbar 任务栏

### 5.1 功能概述

Taskbar 是类似 Windows 的底部任务栏，显示打开的应用程序和系统托盘。

### 5.2 组件接口

```tsx
interface TaskbarApp {
  id: string;
  name: string;
  icon: ReactNode;
}

interface SystemTrayItem {
  id: string;
  icon: ReactNode;
  tooltip?: string;
  onClick?: () => void;
}

interface TaskbarProps {
  className?: string;
  showStartMenu: boolean;
  onStartMenuToggle: () => void;
  activeApps: TaskbarApp[];
  onAppClick: (id: string) => void;
  systemTrayItems: SystemTrayItem[];
  showStartButton: boolean;
  onAppLauncherToggle?: () => void;
}
```

### 5.3 布局结构

```tsx
<div className="glass fixed bottom-0 left-0 right-0 h-12">
  <Space size={12}>
    {/* 开始按钮 */}
    {showStartButton && (
      <Button icon={<WindowsOutlined />} onClick={onStartMenuToggle} />
    )}

    {/* 应用启动器 */}
    <Button onClick={onAppLauncherToggle}>
      <svg>...</svg>  {/* 网格图标 */}
    </Button>

    {/* 打开的应用程序 */}
    {activeApps.map((app) => (
      <Button
        key={app.id}
        icon={app.icon}
        onClick={() => onAppClick(app.id)}
      />
    ))}
  </Space>

  <Space size={8}>
    {/* 系统托盘项 */}
    {systemTrayItems.map((item) => (
      <Button icon={item.icon} onClick={item.onClick} />
    ))}

    {/* 显示桌面按钮 */}
    <div className="w-1.5 h-full hover:bg-white/20" />
  </Space>
</div>
```

## 6. 窗口系统实现原理

### 6.1 窗口渲染流程

```
1. WindowManager 维护 windows 数组状态
2. 遍历 windows，筛选出可见且非最小化的窗口
3. 对每个可见窗口渲染 Window 组件
4. 最小化的窗口显示在底部快捷栏
```

```tsx
// 渲染可见窗口
{getVisibleWindows().map((window) => (
  <Window
    key={window.id}
    title={window.title}
    initialPosition={window.position}
    initialSize={window.size}
    visible={window.visible && !window.minimized}
    onClose={() => closeWindow(window.id)}
    onMinimize={() => minimizeWindow(window.id)}
    onMaximize={() => maximizeWindow(window.id)}
    zIndex={window.zIndex}
    onMouseDown={() => activateWindow(window.id)}
  >
    {window.content}
  </Window>
))}

// 渲染最小化窗口快捷栏
{getMinimizedWindows().length > 0 && (
  <div className="fixed bottom-14 left-2 flex gap-1">
    {getMinimizedWindows().map((window) => (
      <button onClick={() => restoreWindow(window.id)}>
        {window.title}
      </button>
    ))}
  </div>
)}
```

### 6.2 窗口层叠管理

```tsx
// 窗口激活时更新 zIndex
const maxZIndex = Math.max(...prev.map((w) => w.zIndex), 100);
return prev.map((w) =>
  w.id === id ? { ...w, zIndex: maxZIndex + 1 } : w
);
```

### 6.3 窗口操作事件流

```
用户点击窗口标题栏
    ↓
Window 组件捕获 onMouseDown
    ↓
调用 activateWindow(id)
    ↓
更新 windows 状态，设置 zIndex
    ↓
触发重渲染，窗口置顶
```

### 6.4 窗口状态持久化

窗口状态可以通过 WindowManager 的 API 持久化：

```tsx
// 通过事件暴露 API
useEffect(() => {
  const api: WindowManagerApi = {
    createWindow,
    closeWindow,
    minimizeWindow,
    maximizeWindow,
    restoreWindow,
    // ...
  };
  onReady?.(api);

  // 派发事件供其他组件使用
  const event = new CustomEvent("window-manager-ready", { detail: api });
  window.dispatchEvent(event);
}, [...]);
```

## 7. 桌面环境组件协作关系

```
Desktop (桌面容器)
├── Wallpaper (壁纸)
├── Desktop Icons (桌面图标)
├── Context Menu (右键菜单)
│
└── WindowManager (窗口管理器)
    ├── Window 1
    │   ├── Title Bar
    │   ├── Window Controls
    │   └── Content
    ├── Window 2
    └── ...

Dock (底部 Dock 栏)
└── App Icons (应用图标)

Taskbar (任务栏)
├── Start Button
├── App Launcher
├── Active Apps
└── System Tray
```

## 8. 设计模式总结

### 8.1 复合组件模式

Desktop 作为容器，包含 WindowManager、Dock、Taskbar 等子组件，通过 props 传递数据和回调。

### 8.2 状态提升模式

WindowManager 管理所有窗口状态，通过 API 暴露方法给外部调用。

### 8.3 渲染回调模式

Window 的内容通过 `content: ReactNode` 属性传入，实现内容与容器的解耦。

### 8.4 事件驱动模式

通过 `CustomEvent` 和 `onReady` 回调实现组件间通信。
