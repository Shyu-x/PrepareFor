## 1. Vite完整指南

### 1.1 Vite 基础

```bash
# 创建项目
npm create vite@latest my-app -- --template react-ts

# 安装依赖
npm install

# 开发服务器
npm run dev

# 构建生产版本
npm run build

# 预览生产版本
npm run preview
```

### 1.2 Vite 配置

```typescript
// vite.config.ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
    plugins: [react()],

    // 服务器配置
    server: {
        port: 3000,
        proxy: {
            '/api': {
                target: 'http://localhost:8080',
                changeOrigin: true
            }
        }
    },

    // 构建配置
    build: {
        outDir: 'dist',
        sourcemap: true,
        rollupOptions: {
            output: {
                manualChunks: {
                    vendor: ['react', 'react-dom'],
                    utils: ['lodash']
                }
            }
        }
    },

    // 路径别名
    resolve: {
        alias: {
            '@': '/src',
            '@components': '/src/components'
        }
    }
});
```

## 2. Webpack高级配置

### 2.1 Webpack 基础配置

```javascript
// webpack.config.js
const path = require('path');

module.exports = {
    mode: 'production',
    entry: './src/index.js',
    output: {
        path: path.resolve(__dirname, 'dist'),
        filename: '[name].[contenthash].js',
        clean: true
    },
    module: {
        rules: [
            {
                test: /\.tsx?$/,
                use: 'ts-loader',
                exclude: /node_modules/
            },
            {
                test: /\.css$/,
                use: ['style-loader', 'css-loader']
            },
            {
                test: /\.(png|jpg|gif)$/,
                type: 'asset/resource'
            }
        ]
    },
    resolve: {
        extensions: ['.tsx', '.ts', '.js'],
        alias: {
            '@': path.resolve(__dirname, 'src')
        }
    },
    optimization: {
        splitChunks: {
            chunks: 'all'
        }
    }
};
```

## 3. 常见面试问题

**问题1：Vite 和 Webpack 有什么区别？**

答案：
- Vite：开发时原生 ESM，构建使用 Rollup，更快
- Webpack：功能更完整，生态更丰富，配置更复杂
- 生产项目：Webpack 更稳定；新项目：Vite 更快

## 5. 测试

### 5.1 Jest 配置

```javascript
// jest.config.js
module.exports = {
    testEnvironment: 'jsdom',
    moduleFileExtensions: ['ts', 'tsx', 'js', 'jsx'],
    transform: {
        '^.+\\.(ts|tsx)$': 'ts-jest'
    },
    moduleNameMapper: {
        '^@/(.*)$': '<rootDir>/src/$1'
    },
    testMatch: ['**/*.test.(ts|tsx)'],
    collectCoverageFrom: [
        'src/**/*.{ts,tsx}',
        '!src/**/*.d.ts'
    ]
};
```

### 5.2 编写测试

```typescript
// example.test.ts
import { add } from './example';

describe('add', () => {
    test('adds two numbers', () => {
        expect(add(1, 2)).toBe(3);
    });

    test('adds negative numbers', () => {
        expect(add(-1, -1)).toBe(-2);
    });
});
```
