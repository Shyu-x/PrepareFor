# 前端工程化工具对比

## 一、工程化工具概述

### 1.1 为什么需要构建工具

| 功能 | 说明 |
|------|------|
| **代码编译** | TypeScript、JSX、新语法转译 |
| **模块打包** | 将多个模块合并为少量文件 |
| **代码压缩** | 减小文件体积，提升加载速度 |
| **开发服务器** | 热更新、代理、Mock 数据 |
| **代码分割** | 按需加载，优化首屏性能 |
| **资源处理** | 图片、字体、CSS 等资源管理 |

### 1.2 主流构建工具

| 工具 | 语言 | 特点 |
|------|------|------|
| **Vite** | Go + TypeScript | 极速开发、ESM 原生支持 |
| **Webpack** | JavaScript | 生态丰富、配置灵活 |
| **Rspack** | Rust | Webpack 兼容、极速构建 |
| **Turbopack** | Rust | Next.js 内置、增量编译 |
| **esbuild** | Go | 极速打包、Go 语言编写 |
| **Rollup** | JavaScript | 库打包首选、Tree Shaking |

## 二、Vite

### 2.1 简介

Vite 是新一代前端构建工具，特点：

- **极速冷启动**：利用浏览器原生 ESM
- **即时热更新**：毫秒级 HMR
- **按需编译**：只编译当前页面需要的代码
- **优化的生产构建**：基于 Rollup

### 2.2 项目创建

```bash
# 创建项目
npm create vite@latest my-app

# 选择框架
# - React
# - Vue
# - Svelte
# - Vanilla

# 选择变体
# - TypeScript
# - JavaScript

# 安装依赖
cd my-app
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 预览生产版本
npm run preview
```

### 2.3 配置文件

```typescript
// vite.config.ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  // 插件配置
  plugins: [
    react(),
    // 其他插件
  ],
  
  // 路径别名
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
      '@components': path.resolve(__dirname, './src/components'),
      '@utils': path.resolve(__dirname, './src/utils'),
    },
  },
  
  // 开发服务器配置
  server: {
    port: 3000,
    open: true,
    // 代理配置
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, ''),
      },
    },
  },
  
  // 构建配置
  build: {
    outDir: 'dist',
    sourcemap: true,
    // 代码分割策略
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom'],
          utils: ['lodash', 'axios'],
        },
      },
    },
  },
  
  // CSS 配置
  css: {
    modules: {
      localsConvention: 'camelCase',
    },
    preprocessorOptions: {
      scss: {
        additionalData: `@import "@/styles/variables.scss";`,
      },
    },
  },
  
  // 环境变量
  envPrefix: 'VITE_',
});
```

## 三、Webpack

### 3.1 简介

Webpack 是最成熟的模块打包工具，特点：

- **生态丰富**：大量 loader 和 plugin
- **配置灵活**：可定制化程度高
- **代码分割**：强大的代码分割能力
- **模块联邦**：微前端支持

### 3.2 基础配置

```javascript
// webpack.config.js
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
  // 入口
  entry: './src/index.tsx',
  
  // 输出
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: '[name].[contenthash].js',
    clean: true,
  },
  
  // 模式
  mode: process.env.NODE_ENV || 'development',
  
  // 模块规则
  module: {
    rules: [
      // TypeScript
      {
        test: /\.tsx?$/,
        use: 'ts-loader',
        exclude: /node_modules/,
      },
      
      // CSS
      {
        test: /\.css$/,
        use: [
          MiniCssExtractPlugin.loader,
          'css-loader',
          'postcss-loader',
        ],
      },
      
      // SCSS
      {
        test: /\.scss$/,
        use: [
          MiniCssExtractPlugin.loader,
          'css-loader',
          'postcss-loader',
          'sass-loader',
        ],
      },
      
      // 图片
      {
        test: /\.(png|jpg|gif|svg)$/,
        type: 'asset',
        parser: {
          dataUrlCondition: {
            maxSize: 8 * 1024, // 8KB
          },
        },
      },
      
      // 字体
      {
        test: /\.(woff|woff2|eot|ttf|otf)$/,
        type: 'asset/resource',
      },
    ],
  },
  
  // 插件
  plugins: [
    new HtmlWebpackPlugin({
      template: './public/index.html',
    }),
    new MiniCssExtractPlugin({
      filename: '[name].[contenthash].css',
    }),
  ],
  
  // 解析
  resolve: {
    extensions: ['.tsx', '.ts', '.js'],
    alias: {
      '@': path.resolve(__dirname, 'src'),
    },
  },
  
  // 开发服务器
  devServer: {
    static: './dist',
    hot: true,
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
      },
    },
  },
  
  // 优化
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
};
```

## 四、Rspack

### 4.1 简介

Rspack 是字节跳动开发的 Rust 构建工具，特点：

- **Webpack 兼容**：平滑迁移
- **极速构建**：Rust 语言编写
- **内置功能**：减少配置

### 4.2 性能对比

| 指标 | Webpack | Rspack | 提升 |
|------|---------|--------|------|
| 冷启动 | 10s | 0.5s | 20x |
| 热更新 | 1s | 0.1s | 10x |
| 生产构建 | 30s | 3s | 10x |

### 4.3 配置文件

```javascript
// rspack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.tsx',
  
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: '[name].[contenthash].js',
  },
  
  // 内置 TypeScript 支持
  module: {
    rules: [
      {
        test: /\.css$/,
        use: [
          { loader: 'postcss-loader' },
        ],
        type: 'css',
      },
    ],
  },
  
  // 内置 React 支持
  builtins: {
    html: [
      {
        template: './public/index.html',
      },
    ],
    // 自动导入 React
    react: {
      runtime: 'automatic',
    },
    // 定义环境变量
    define: {
      'process.env.NODE_ENV': JSON.stringify(process.env.NODE_ENV),
    },
  },
  
  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src'),
    },
  },
};
```

### 4.4 从 Webpack 迁移

```javascript

  // Webpack 兼容的配置

  // Rspack 特有配置

    // 内置功能替代插件
  },
```

### 5.3 选型建议

## 八、完整构建配置示例

### 8.1 Vite 企业级配置

```typescript
// vite.config.ts - 企业级完整配置
import { defineConfig, loadEnv } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';
import svgr from 'vite-plugin-svgr';
import compression from 'vite-plugin-compression';
import visualizer from 'rollup-plugin-visualizer';
import { createHtmlPlugin } from 'vite-plugin-html';
import viteImagemin from 'vite-plugin-imagemin';
import { viteMockServe } from 'vite-plugin-mock';

export default defineConfig(({ mode }) => {
  // 加载环境变量
  const env = loadEnv(mode, process.cwd());

  return {
    // 基础路径
    base: env.VITE_BASE_URL || '/',

    // 插件配置
    plugins: [
      // React 支持
      react({
        // 使用 React 自动导入
        babel: {
          plugins: [
            ['@babel/plugin-transform-react-jsx', { runtime: 'automatic' }],
          ],
        },
      }),

      // SVG 转 React 组件
      svgr({
        // svgr 选项
        svgrOptions: {
          exportType: 'default',
          ref: true,
          svgo: false,
          titleProp: true,
        },
        include: '**/*.svg',
      }),

      // Gzip 压缩
      compression({
        algorithm: 'gzip',
        ext: '.gz',
        threshold: 10240, // 大于 10KB 才压缩
        deleteOriginFile: false,
      }),

      // Brotli 压缩（压缩率更高）
      compression({
        algorithm: 'brotliCompress',
        ext: '.br',
        threshold: 10240,
      }),

      // 图片压缩
      viteImagemin({
        gifsicle: { optimizationLevel: 3 },
        optipng: { optimizationLevel: 7 },
        mozjpeg: { quality: 80 },
        svgo: {
          plugins: [
            { name: 'removeViewBox', active: false },
            { name: 'removeEmptyAttrs', active: true },
          ],
        },
      }),

      // HTML 模板处理
      createHtmlPlugin({
        minify: true,
        inject: {
          data: {
            title: env.VITE_APP_TITLE || 'React App',
            // 注入环境变量到 HTML
            injectScript: `<script>window.__ENV__ = ${JSON.stringify({
              API_URL: env.VITE_API_URL,
              APP_VERSION: env.VITE_APP_VERSION,
            })}</script>`,
          },
        },
      }),

      // Mock 数据服务（开发环境）
      mode === 'development' && viteMockServe({
        mockPath: 'mock',
        enable: true,
        watchFiles: true,
      }),

      // 打包分析（生产环境）
      mode === 'production' && visualizer({
        open: false,
        gzipSize: true,
        brotliSize: true,
        filename: 'stats/bundle-analysis.html',
      }),
    ].filter(Boolean),

    // 路径别名
    resolve: {
      alias: {
        '@': path.resolve(__dirname, './src'),
        '@components': path.resolve(__dirname, './src/components'),
        '@hooks': path.resolve(__dirname, './src/hooks'),
        '@utils': path.resolve(__dirname, './src/utils'),
        '@services': path.resolve(__dirname, './src/services'),
        '@stores': path.resolve(__dirname, './src/stores'),
        '@types': path.resolve(__dirname, './src/types'),
        '@assets': path.resolve(__dirname, './src/assets'),
        '@styles': path.resolve(__dirname, './src/styles'),
      },
      extensions: ['.mjs', '.js', '.ts', '.jsx', '.tsx', '.json'],
    },

    // 开发服务器配置
    server: {
      host: '0.0.0.0', // 允许局域网访问
      port: Number(env.VITE_PORT) || 3000,
      open: true,
      // 热更新配置
      hmr: {
        overlay: true, // 显示错误覆盖层
      },
      // 代理配置
      proxy: {
        '/api': {
          target: env.VITE_API_URL || 'http://localhost:8080',
          changeOrigin: true,
          rewrite: (path) => path.replace(/^\/api/, ''),
          secure: false,
          configure: (proxy) => {
            proxy.on('error', (err) => {
              console.log('代理错误:', err);
            });
          },
        },
        // WebSocket 代理
        '/ws': {
          target: env.VITE_WS_URL || 'ws://localhost:8080',
          ws: true,
        },
      },
      // CORS 配置
      cors: true,
    },

    // 构建配置
    build: {
      // 输出目录
      outDir: 'dist',
      // 静态资源目录
      assetsDir: 'assets',
      // 生成 sourcemap
      sourcemap: mode === 'development',
      // 压缩配置
      minify: 'terser',
      terserOptions: {
        compress: {
          // 生产环境移除 console
          drop_console: mode === 'production',
          drop_debugger: mode === 'production',
          pure_funcs: mode === 'production' ? ['console.log'] : [],
        },
        format: {
          comments: false, // 移除注释
        },
      },
      // CSS 代码分割
      cssCodeSplit: true,
      // 资源文件名格式
      rollupOptions: {
        output: {
          // 入口文件名
          entryFileNames: 'assets/js/[name]-[hash].js',
          // 代码块文件名
          chunkFileNames: 'assets/js/[name]-[hash].js',
          // 静态资源文件名
          assetFileNames: (assetInfo) => {
            const info = assetInfo.name.split('.');
            const ext = info[info.length - 1];

            // 根据文件类型分类存放
            if (/\.(png|jpe?g|gif|svg|webp|ico)$/i.test(assetInfo.name)) {
              return 'assets/images/[name]-[hash].[ext]';
            } else if (/\.(woff2?|eot|ttf|otf)$/i.test(assetInfo.name)) {
              return 'assets/fonts/[name]-[hash].[ext]';
            } else if (/\.css$/i.test(assetInfo.name)) {
              return 'assets/css/[name]-[hash].[ext]';
            }
            return 'assets/[name]-[hash].[ext]';
          },
          // 手动分割代码块
          manualChunks: (id) => {
            // React 生态
            if (id.includes('node_modules/react/') ||
                id.includes('node_modules/react-dom/') ||
                id.includes('node_modules/scheduler/')) {
              return 'vendor-react';
            }
            // 路由
            if (id.includes('node_modules/react-router/') ||
                id.includes('node_modules/react-router-dom/')) {
              return 'vendor-router';
            }
            // 状态管理
            if (id.includes('node_modules/zustand/') ||
                id.includes('node_modules/immer/')) {
              return 'vendor-state';
            }
            // UI 组件库
            if (id.includes('node_modules/antd/') ||
                id.includes('node_modules/@ant-design/')) {
              return 'vendor-ui';
            }
            // 工具库
            if (id.includes('node_modules/lodash/') ||
                id.includes('node_modules/dayjs/') ||
                id.includes('node_modules/axios/')) {
              return 'vendor-utils';
            }
            // 图表库
            if (id.includes('node_modules/echarts/') ||
                id.includes('node_modules/zrender/')) {
              return 'vendor-charts';
            }
          },
          // 全局变量命名
          globals: {
            react: 'React',
            'react-dom': 'ReactDOM',
          },
        },
      },
      // 块大小警告阈值
      chunkSizeWarningLimit: 1000,
      // 清空输出目录
      emptyOutDir: true,
    },

    // CSS 配置
    css: {
      // CSS Modules 配置
      modules: {
        localsConvention: 'camelCaseOnly',
        generateScopedName: mode === 'production'
          ? '[hash:base64:5]'
          : '[name]__[local]__[hash:base64:5]',
      },
      // CSS 预处理器配置
      preprocessorOptions: {
        scss: {
          additionalData: `@import "@/styles/variables.scss";`,
          api: 'modern-compiler',
        },
        less: {
```

### 10.1 前端监控 SDK

```typescript
// src/monitor/index.ts - 前端监控 SDK
interface MonitorConfig {
  dsn: string; // 数据上报地址
  appId: string; // 应用 ID
  userId?: string; // 用户 ID
  environment: 'development' | 'production';
  sampleRate: number; // 采样率 0-1
  enablePerformance: boolean; // 是否开启性能监控
  enableError: boolean; // 是否开启错误监控
  enableBehavior: boolean; // 是否开启行为追踪
}

interface PerformanceData {
  // 页面加载性能
  dns: number;
  tcp: number;
  ttfb: number;
  domParse: number;
  domComplete: number;
  load: number;
  // Web Vitals
  fcp: number;
  lcp: number;
  fid: number;
  cls: number;
  ttfb_vital: number;
}

interface ErrorData {
  type: 'js' | 'resource' | 'ajax' | 'promise';
  message: string;
  stack?: string;
  url: string;
  line?: number;
  column?: number;
  timestamp: number;
  userAgent: string;
}

class Monitor {
  private config: MonitorConfig;
  private queue: any[] = [];
  private isSending = false;

  constructor(config: MonitorConfig) {
    this.config = config;
    this.init();
  }

  private init() {
    // 初始化性能监控
    if (this.config.enablePerformance) {
      this.initPerformance();
    }

    // 初始化错误监控
    if (this.config.enableError) {
      this.initErrorCapture();
    }

    // 初始化行为追踪
    if (this.config.enableBehavior) {
      this.initBehaviorTracking();
    }

    // 页面卸载时发送数据
    window.addEventListener('unload', () => {
      this.sendBeacon();
    });
  }

  // 性能监控
  private initPerformance() {
    // 等待页面加载完成
    window.addEventListener('load', () => {
      setTimeout(() => {
        this.collectPerformance();
      }, 0);
    });

    // 收集 Web Vitals
    this.collectWebVitals();
  }

  private collectPerformance() {
    const timing = performance.getEntriesByType('navigation')[0] as PerformanceNavigationTiming;

    if (!timing) return;

    const data: PerformanceData = {
      dns: timing.domainLookupEnd - timing.domainLookupStart,
      tcp: timing.connectEnd - timing.connectStart,
      ttfb: timing.responseStart - timing.requestStart,
      domParse: timing.domInteractive - timing.responseEnd,
      domComplete: timing.domComplete - timing.domInteractive,
      load: timing.loadEventEnd - timing.fetchStart,
      fcp: 0,
      lcp: 0,
      fid: 0,
      cls: 0,
      ttfb_vital: 0,
    };

    this.report('performance', data);
  }

  private collectWebVitals() {
    // 使用 web-vitals 库收集指标
    import('web-vitals').then(({ getCLS, getFID, getLCP, getFCP, getTTFB }) => {
      getCLS((metric) => this.report('vitals', { name: 'CLS', ...metric }));
      getFID((metric) => this.report('vitals', { name: 'FID', ...metric }));
      getLCP((metric) => this.report('vitals', { name: 'LCP', ...metric }));
      getFCP((metric) => this.report('vitals', { name: 'FCP', ...metric }));
      getTTFB((metric) => this.report('vitals', { name: 'TTFB', ...metric }));
    });
  }

  // 错误监控
  private initErrorCapture() {
    // JS 错误
    window.addEventListener('error', (event) => {
      const data: ErrorData = {
        type: 'js',
        message: event.message,
        stack: event.error?.stack,
        url: event.filename,
        line: event.lineno,
        column: event.colno,
        timestamp: Date.now(),
        userAgent: navigator.userAgent,
      };
      this.report('error', data);
    });

    // Promise 未捕获错误
    window.addEventListener('unhandledrejection', (event) => {
      const data: ErrorData = {
        type: 'promise',
        message: event.reason?.message || String(event.reason),
        stack: event.reason?.stack,
        url: window.location.href,
        timestamp: Date.now(),
        userAgent: navigator.userAgent,
      };
      this.report('error', data);
    });

    // 资源加载错误
    window.addEventListener('error', (event) => {
      if (event.target !== window) {
        const target = event.target as HTMLElement;
        const data: ErrorData = {
          type: 'resource',
          message: `资源加载失败: ${(target as any).src || (target as any).href}`,
          url: window.location.href,
          timestamp: Date.now(),
          userAgent: navigator.userAgent,
        };
        this.report('error', data);
      }
    }, true);

    // Ajax 错误
    this.interceptAjax();

    // Fetch 错误
    this.interceptFetch();
  }

  private interceptAjax() {
    const originalOpen = XMLHttpRequest.prototype.open;
    const originalSend = XMLHttpRequest.prototype.send;
    const self = this;

    XMLHttpRequest.prototype.open = function(method, url, ...args) {
      (this as any)._url = url;
      (this as any)._method = method;
      return originalOpen.apply(this, [method, url, ...args]);
    };

    XMLHttpRequest.prototype.send = function(...args) {
      const startTime = Date.now();

      this.addEventListener('loadend', function() {
        if (this.status >= 400) {
          const data: ErrorData = {
            type: 'ajax',
            message: `Ajax 请求失败: ${this.status}`,
            url: (this as any)._url,
            timestamp: startTime,
            userAgent: navigator.userAgent,
          };
          self.report('error', data);
        }
      });

      return originalSend.apply(this, args);
    };
  }

  private interceptFetch() {
    const originalFetch = window.fetch;
    const self = this;

    window.fetch = async function(input, init) {
      const startTime = Date.now();
      try {
        const response = await originalFetch.apply(this, [input, init]);
        if (!response.ok) {
          const data: ErrorData = {
            type: 'ajax',
            message: `Fetch 请求失败: ${response.status}`,
            url: typeof input === 'string' ? input : (input as Request).url,
            timestamp: startTime,
            userAgent: navigator.userAgent,
          };
          self.report('error', data);
        }
        return response;
      } catch (error) {
        const data: ErrorData = {
          type: 'ajax',
          message: `Fetch 请求异常: ${(error as Error).message}`,
          url: typeof input === 'string' ? input : (input as Request).url,
          timestamp: startTime,
          userAgent: navigator.userAgent,
        };
        self.report('error', data);
        throw error;
      }
    };
  }

  // 行为追踪
  private initBehaviorTracking() {
    // 页面访问
    this.trackPageView();

    // 点击事件
    document.addEventListener('click', (e) => {
      const target = e.target as HTMLElement;
      this.report('behavior', {
        type: 'click',
        tagName: target.tagName,
        className: target.className,
        id: target.id,
        text: target.textContent?.slice(0, 50),
        timestamp: Date.now(),
      });
    }, true);

    // 路由变化
    this.trackRouteChange();
  }

  private trackPageView() {
    this.report('behavior', {
      type: 'page_view',
      url: window.location.href,
      referrer: document.referrer,
      timestamp: Date.now(),
    });
  }

  private trackRouteChange() {
    // 监听 popstate
    window.addEventListener('popstate', () => {
      this.trackPageView();
    });

    // 监听 pushState 和 replaceState
    const originalPushState = history.pushState;
    const originalReplaceState = history.replaceState;

    history.pushState = function(...args) {
      originalPushState.apply(this, args);
      window.dispatchEvent(new Event('pushstate'));
    };

    history.replaceState = function(...args) {
      originalReplaceState.apply(this, args);
      window.dispatchEvent(new Event('replacestate'));
    };

    window.addEventListener('pushstate', () => this.trackPageView());
    window.addEventListener('replacestate', () => this.trackPageView());
  }

  // 数据上报
  private report(type: string, data: any) {
    // 采样率检查
    if (Math.random() > this.config.sampleRate) return;

    const payload = {
      type,
      appId: this.config.appId,
      userId: this.config.userId,
      environment: this.config.environment,
      timestamp: Date.now(),
      data,
    };

    this.queue.push(payload);

    // 批量发送
    if (this.queue.length >= 10) {
      this.send();
    }
  }

  private send() {
    if (this.isSending || this.queue.length === 0) return;

    this.isSending = true;
    const data = this.queue.splice(0, this.queue.length);

    fetch(this.config.dsn, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data),
      keepalive: true,
    }).finally(() => {
      this.isSending = false;
    });
  }

  private sendBeacon() {
    if (this.queue.length === 0) return;

    const data = this.queue.splice(0, this.queue.length);
    navigator.sendBeacon(this.config.dsn, JSON.stringify(data));
  }

  // 手动上报
  public captureException(error: Error, extra?: Record<string, any>) {
    const data: ErrorData = {
      type: 'js',
      message: error.message,
      stack: error.stack,
      url: window.location.href,
      timestamp: Date.now(),
      userAgent: navigator.userAgent,
      ...extra,
    };
    this.report('error', data);
  }

  // 设置用户信息
  public setUser(userId: string, extra?: Record<string, any>) {
    this.config.userId = userId;
    this.report('user', { userId, ...extra });
  }
}

// 初始化监控
export const monitor = new Monitor({
  dsn: import.meta.env.VITE_MONITOR_DSN,
  appId: import.meta.env.VITE_APP_ID,
  environment: import.meta.env.MODE as 'development' | 'production',
  sampleRate: 0.1, // 10% 采样
  enablePerformance: true,
  enableError: true,
  enableBehavior: true,
});

// 全局错误捕获
window.monitor = monitor;
```

## 十二、面试高频问题

### Q1: Vite 为什么比 Webpack 快？

**答案：**

Vite 的快速主要体现在两个方面：

1. **开发环境**：
   - 利用浏览器原生 ESM 能力，无需打包即可启动
   - 按需编译，只编译当前页面用到的文件
   - 使用 esbuild（Go 语言编写）进行依赖预构建，速度极快

2. **生产环境**：
   - 使用 Rollup 进行打包，Tree Shaking 效果更好
   - 支持原生 ESM，无需额外的模块包装代码

```
Webpack 启动流程：
分析入口 → 递归解析依赖 → 编译所有模块 → 打包输出 → 启动服务器

Vite 启动流程：
启动服务器 → 浏览器请求模块 → 按需编译 → 返回结果
```

### Q2: 什么是 Tree Shaking？如何确保 Tree Shaking 生效？

**答案：**

Tree Shaking 是一种消除死代码的技术，基于 ES Module 的静态分析。

**生效条件：**

1. 使用 ES Module 语法（import/export）
2. 在 package.json 中设置 `sideEffects: false` 或指定有副作用的文件
3. 生产模式构建

```javascript
// ❌ 不会被 Tree Shaking
import { debounce } from 'lodash';
import _ from 'lodash';

// ✅ 会被 Tree Shaking
import { debounce } from 'lodash-es';
import { debounce } from 'lodash/debounce';
```

**配置示例：**

```json
// package.json
{
  "sideEffects": false
}

// 或指定有副作用的文件
{
  "sideEffects": ["*.css", "*.scss"]
}
```

### Q3: 如何优化 Webpack 构建速度？

**答案：**

1. **缓存**：
   - 开启 `cache: { type: 'filesystem' }`
   - 使用 `babel-loader` 的 `cacheDirectory`

2. **缩小搜索范围**：
   - 配置 `resolve.extensions` 减少后缀尝试
   - 配置 `resolve.alias` 减少路径查找
   - 使用 `include/exclude` 排除不需要处理的文件

3. **多进程**：
   - 使用 `thread-loader` 进行多线程编译
   - 使用 `HappyPack`（已不推荐）

4. **DLL 预编译**：
   - 使用 `DllPlugin` 预编译第三方库

5. **externals**：
   - 将大型库通过 CDN 引入，不参与打包

```javascript
// webpack.config.js
module.exports = {
  cache: {
    type: 'filesystem',
  },
  resolve: {
    extensions: ['.ts', '.tsx', '.js'],
    alias: {
      '@': path.resolve(__dirname, 'src'),
    },
  },
  module: {
    rules: [
      {
        test: /\.tsx?$/,
        use: 'babel-loader?cacheDirectory=true',
        exclude: /node_modules/,
      },
    ],
  },
};
```

2. **动态导入**：使用 `import()` 语法

### Q5: 如何实现前端灰度发布？

**答案：**

灰度发布策略：

1. **Nginx 层**：根据 Cookie 或 IP 分流

```nginx
# nginx.conf
upstream backend_v1 {
  server 127.0.0.1:3001;
}

upstream backend_v2 {
  server 127.0.0.1:3002;
}

server {
  location / {
    # 根据灰度标识分流
    if ($http_cookie ~* "gray=true") {
      proxy_pass http://backend_v2;
    }
    proxy_pass http://backend_v1;
  }
}
```

2. **应用层**：根据用户 ID 或百分比分流

```typescript
// 灰度判断
function isGrayUser(userId: string): boolean {
  // 白名单用户
  const whitelist = ['user_001', 'user_002'];
  if (whitelist.includes(userId)) return true;

  // 百分比灰度
  const hash = hashCode(userId);
  return (hash % 100) < grayPercent; // grayPercent = 10 表示 10% 灰度
}

// 加载不同版本
async function loadApp() {
  const version = isGrayUser(userId) ? 'v2' : 'v1';
  const module = await import(`./app/${version}/index.js`);
  module.init();
}
```

3. **特性开关**：通过配置控制功能

```typescript
// feature-flags.ts
const features = {
  newDashboard: {
    enabled: true,
    rollout: 10, // 10% 用户
  },
  newCheckout: {
    enabled: false,
  },
};

function useFeature(name: string): boolean {
  const feature = features[name];
  if (!feature?.enabled) return false;
  if (feature.rollout !== undefined) {
    return Math.random() * 100 < feature.rollout;
  }
  return true;
}
```

### Q7: Webpack 和 Vite 的模块联邦有什么区别？

**答案：**

**Webpack Module Federation：**
- 成熟的微前端解决方案
- 支持运行时动态加载远程模块
- 需要配置 `ModuleFederationPlugin`

```javascript
// webpack.config.js
new ModuleFederationPlugin({
  name: 'app1',
  filename: 'remoteEntry.js',
  exposes: {
    './Button': './src/components/Button',
  },
  shared: {
    react: { singleton: true },
  },
});
```

**Vite Module Federation：**
- 通过 `@originjs/vite-plugin-federation` 插件支持
- 开发体验更好，热更新支持
- 生产环境使用 Rollup 打包

```javascript
// vite.config.ts
import federation from '@originjs/vite-plugin-federation';

export default defineConfig({
  plugins: [
    federation({
      name: 'app1',
      filename: 'remoteEntry.js',
      exposes: {
        './Button': './src/components/Button',
      },
      shared: ['react'],
    }),
  ],
});
```

### Q8: 如何选择构建工具？

**答案：**

| 场景 | 推荐工具 | 理由 |
|------|----------|------|
| 新项目 | Vite | 开发体验好、配置简单、生态成熟 |
| 大型遗留项目 | Webpack | 迁移成本低、生态完善 |
| 追求构建速度 | Rspack | Webpack 兼容、构建最快 |
| 库开发 | Vite (library mode) / Rollup | Tree Shaking 好、输出格式多 |
| 微前端项目 | Webpack | Module Federation 成熟 |
| Next.js 项目 | Turbopack | 内置支持、增量编译 |
