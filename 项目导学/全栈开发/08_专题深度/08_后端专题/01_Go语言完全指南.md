## 1. Go语言基础

### 1.1 Go的定位：系统编程、服务端

Go语言（又称Golang）是Google于2009年正式发布的开源编程语言。它的设计目标非常明确：**简单、高效、可靠**。Go语言的诞生背景值得深入理解——Google在面对C++编译慢、Java过于笨重、Python性能不足的困境时，决定创造一门专门解决服务端问题的语言。

**Go的核心定位：**

| 领域 | 适用程度 | 说明 |
|------|----------|------|
| 服务端开发 | 极佳 | 网络编程、API服务、微服务 |
| 云原生/容器 | 极佳 | Kubernetes、Docker、Terraform都用Go编写 |
| 网络工具 | 极佳 | 代理、网关、负载均衡 |
| 系统编程 | 良好 | 底层工具、命令行程序 |
| Web前端 | 不适用 | 应使用JavaScript/TypeScript |
| 移动开发 | 不推荐 | 应使用Swift/Kotlin |
| 数据科学 | 不推荐 | 应使用Python/R |

Go语言的设计哲学强调**要做一件事就用最好的方式做**，而不是提供所有可能的特性。这种"少即是多"的理念让Go代码易于维护、编译快速、部署简单。

### 1.2 为什么学Go：性能、并发、简洁

**Go语言的三大核心优势：**

**1. 性能卓越**
Go编译成机器码，执行效率接近C语言。根据TechEmpower的基准测试，Go的HTTP服务器性能可以达到每秒处理数十万请求，远超Node.js和Python。

```go
// Go HTTP服务器 - 简洁高效的并发处理
package main

import (
    "fmt"
    "net/http"
    "time"
)

func handler(w http.ResponseWriter, r *http.Request) {
    // 处理请求
    w.Write([]byte("Hello, Go!"))
}

func main() {
    http.HandleFunc("/", handler)

    server := &http.Server{
        Addr:         ":8080",
        ReadTimeout:  10 * time.Second,
        WriteTimeout: 10 * time.Second,
    }

    fmt.Println("Server starting on :8080")
    server.ListenAndServe()
}
```

对比Node.js的HTTP服务器：

```javascript
// Node.js HTTP服务器
const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Hello, Node.js!');
});

server.listen(8080, () => {
    console.log('Server running on port 8080');
});
```

表面上看代码量差不多，但Go的goroutine机制让每个请求的内存开销极低（约2KB），而Node.js每个连接约占用几十KB内存。

**2. 并发原生支持**

Go是第一个将并发作为语言核心特性的主流编程语言。`goroutine`和`channel`让并发编程变得安全而简单。

```go
// Go并发：启动100万个goroutine
package main

import (
    "fmt"
    "sync"
)

func main() {
    var wg sync.WaitGroup

    // 启动100万个goroutine
    for i := 0; i < 1_000_000; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()
            // 模拟工作
            _ = id * 2
        }(i)
    }

    wg.Wait()
    fmt.Println("All 1 million goroutines completed!")
}
```

如果用传统线程实现100万个并发任务，内存会直接爆掉。但Go的goroutine非常轻量，100万个同时运行也只是小意思。

**3. 简洁直观**

Go语法非常简洁，没有冗余的关键字，没有复杂的类型系统，没有繁重的面向对象继承体系。

```go
// Go的结构体组合代替继承
package main

import "fmt"

// 动物结构体
type Animal struct {
    Name string
}

// 动物的方法
func (a *Animal) Speak() {
    fmt.Printf("%s makes a sound\n", a.Name)
}

// Dog组合Animal，不使用继承
type Dog struct {
    Animal    // 嵌入结构体，等同于继承了Animal的所有字段和方法
    Breed     string
}

func main() {
    dog := Dog{
        Animal: Animal{Name: "Buddy"},
        Breed:  "Golden Retriever",
    }

    dog.Speak() // 输出: Buddy makes a sound
    fmt.Printf("Dog's name: %s, Breed: %s\n", dog.Name, dog.Breed)
}
```

### 1.3 Go vs Node.js：性能对比

**核心技术差异：**

### 1.4 Go vs Java：定位对比

**设计哲学差异：**

| 维度 | Go | Java |
|------|-----|------|
| 诞生时间 | 2009年 | 1995年 |
| 运行环境 | 编译产物直接运行 | JVM虚拟机 |
| 泛型 | 2022年才引入 | 1.5引入 |
| 继承 | 无（只有组合） | 类继承 |
| 异常处理 | 错误返回值 | try-catch异常 |
| 启动时间 | <100ms | 1-3秒 |
| 内存占用 | 10-50MB | 100-500MB |

**架构设计哲学：**

Java强调"一切皆对象"，有完整的OOP体系，适合大型企业级应用。Go强调"简单直接"，用接口和组合代替继承，适合云原生微服务。

```go
// Go - 简洁的接口设计
package main

import "fmt"

// 定义接口
type Writer interface {
    Write([]byte) (int, error)
}

// 实现接口 - 只需实现Write方法，无需声明implements
type ConsoleWriter struct{}

func (w *ConsoleWriter) Write(data []byte) (int, error) {
    fmt.Print(string(data))
    return len(data), nil
}

func main() {
    var w Writer = &ConsoleWriter{}
    w.Write([]byte("Hello, Go interface!\n"))
}
```

```java
// Java - 显式接口实现
import java.io.*;

public class InterfaceExample {
    // 定义接口
    interface Writer {
        void write(byte[] data) throws IOException;
    }

    // 实现接口 - 必须显式声明
    static class ConsoleWriter implements Writer {
        @Override
        public void write(byte[] data) throws IOException {
            System.out.write(data);
        }
    }

    public static void main(String[] args) {
        Writer w = new ConsoleWriter();
        w.write("Hello, Java interface!".getBytes());
    }
}
```

### 1.5 我的思考：什么时候该选Go

**选择Go的最佳场景：**

1. **微服务架构**：Go的轻量级、高并发特性非常适合微服务，每个服务可以快速启动、占用资源少。

2. **云原生应用**：Docker、Kubernetes、Prometheus等云原生基础设施都是Go写的，Go是云原生的首选语言。

3. **网络代理/网关**：Go的net/http库性能优秀，加上goroutine，可以轻松处理海量连接。

4. **CLI工具**：Go编译成单一可执行文件，部署方便，非常适合写命令行工具。

5. **数据处理管道**：Go的流式处理能力强，适合ETL、数据清洗等场景。

**不应该选Go的场景：**

1. **GUI桌面应用**：虽然有fyne、gioui等库，但体验远不如Electron或Qt。
2. **游戏开发**：性能要求高的游戏还是选C++/C#更合适。
3. **人工智能/机器学习**：Python有完善的生态，Go这方面还在发展中。
4. **复杂业务系统**：Java的生态和框架更成熟，大型企业级应用Java更合适。

## 2. Go底层原理

### 2.1 内存分配：tcmalloc、mspan

Go的内存分配器借鉴了Google的tcmalloc（Thread-Caching Malloc）设计，是专为高并发设计的内存管理系统。理解内存分配对于写出高效Go程序至关重要。

**tcmalloc的核心思想：**

tcmalloc将内存分为三个层级：线程本地缓存、中心缓存、页堆。每个线程有自己的本地缓存，避免锁竞争。

```
┌─────────────────────────────────────────────────────┐
│                    页堆 (Page Heap)                  │
│   8KB页的集合，管理大对象和小对象的空间               │
└─────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────┐
│                   中心缓存 (Central Cache)           │
│   每个大小级别的自由链表，管理回收的内存              │
└─────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────┐
│                线程本地缓存 (Thread Cache)           │
│   每个线程独立的小对象缓存，避免锁竞争                │
└─────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────┐
│                    应用程序                          │
└─────────────────────────────────────────────────────┘
```

**Go的mspan结构：**

Go将内存分成8KB大小的页（page），每个mspan管理一组连续页。mspan根据对象大小分为不同的size class。

```go
// Go内存分配器的核心概念（源码简化）
type mspan struct {
    // 指向管理内存的起始页
    startAddr uintptr
    // 管理的页数
    npages    uintptr
    // 块大小
    elemsize  uintptr
    // 下一个可用块的索引
    freeindex uintptr
    // 位图，标记哪些块已被分配
    allocBits *gcBits
}
```

**内存分配流程：**

```go
package main

import (
    "fmt"
    "runtime"
)

// 观察Go内存分配行为
func main() {
    // 强制GC，获取干净的内存状态
    runtime.GC()

    // 获取内存统计
    var m runtime.MemStats
    runtime.ReadMemStats(&m)

    fmt.Printf("Alloc: %d KB\n", m.Alloc/1024)
    fmt.Printf("TotalAlloc: %d KB\n", m.TotalAlloc/1024)
    fmt.Printf("Sys: %d KB\n", m.Sys/1024)
    fmt.Printf("NumGC: %d\n", m.NumGC)

    // 分配小对象 - 在mcache中处理
    _ = make([]byte, 100)

    runtime.ReadMemStats(&m)
    fmt.Printf("\nAfter small allocation:\n")
    fmt.Printf("Alloc: %d KB\n", m.Alloc/1024)

    // 分配大对象 - 直接从堆分配
    _ = make([]byte, 10*1024*1024) // 10MB

    runtime.ReadMemStats(&m)
    fmt.Printf("\nAfter large allocation:\n")
    fmt.Printf("Alloc: %d KB\n", m.Alloc/1024)
}
```

**小对象 vs 大对象：**

- **小对象（<32KB）**：优先从线程本地缓存分配，极快
- **大对象（>32KB）**：直接从页堆分配，使用大对象分配器

### 2.2 垃圾回收：逃逸分析、三色标记

Go使用并发三色标记垃圾回收器，这套系统既能保证内存安全，又能最小化暂停时间。

**逃逸分析（Escape Analysis）：**

编译器决定一个变量应该分配在栈上还是堆上。如果一个变量在函数返回后不再被引用，它会被分配在栈上，函数结束自动释放，无需GC。如果变量逃逸到堆上，则需要垃圾回收器管理。

```go
package main

import "fmt"

// 情况1：返回指针 - 变量逃逸到堆
func newInt() *int {
    n := 10  // 这个变量会逃逸到堆
    return &n
}

// 情况2：返回数值 - 变量留在栈上
func newIntValue() int {
    n := 10  // 这个变量不会逃逸，可以优化在栈上分配
    return n
}

// 情况3：打印参数 - 变量逃逸
func printInt(n int) {
    fmt.Println(n) // n会逃逸，因为fmt.Println接收interface{}
}

// 情况4：明确不逃逸
func noEscape() int {
    n := 10  // 明确知道不会逃逸
    return n
}

func main() {
    p := newInt()
    fmt.Println(*p) // p指向堆上的内存

    v := newIntValue() // v在栈上，函数返回即释放

    printInt(5) // 5会逃逸

    // 逃逸分析：使用-gcflags="-m"查看
    // go run -gcflags="-m" escape.go
}
```

**三色标记原理：**

```
初始状态：所有对象都是白色
         ┌───┐    ┌───┐    ┌───┐
白色 ──► │ A │    │ B │    │ C │
         └───┘    └───┘    └───┘

根对象：全局变量、栈上的变量视为灰色
         ┌───┐    ┌───┐    ┌───┐
灰色 ──► │ A │ ──►│ B │    │ C │
         └───┘    └───┘    └───┘
              │
              ▼
         ┌───┐    ┌───┐
黑色 ──► │ B │    │ C │
         └───┘    └───┘

标记结束后：白色对象就是垃圾，可以回收
```

**并发三色标记过程：**

```go
package main

import (
    "fmt"
    "runtime"
    "time"
)

// 模拟对象创建，观察GC行为
func main() {
    // 禁用GC，强制手动触发
    runtime.GC()

    var m1 runtime.MemStats
    runtime.ReadMemStats(&m1)
    fmt.Printf("Initial - Alloc: %d KB, NumGC: %d\n",
        m1.Alloc/1024, m1.NumGC)

    // 创建大量对象
    var objs []*[]byte
    for i := 0; i < 100000; i++ {
        b := make([]byte, 1024) // 每次1KB
        objs = append(objs, &b)
    }

    var m2 runtime.MemStats
    runtime.ReadMemStats(&m2)
    fmt.Printf("After allocation - Alloc: %d KB, NumGC: %d\n",
        m2.Alloc/1024, m2.NumGC)

    // 清空引用，触发GC
    objs = nil

    runtime.GC()
    time.Sleep(100 * time.Millisecond)

    var m3 runtime.MemStats
    runtime.ReadMemStats(&m3)
    fmt.Printf("After GC - Alloc: %d KB, NumGC: %d\n",
        m3.Alloc/1024, m3.NumGC)
}
```

### 2.3 调度器：GMP模型

Go的调度器是Go运行时的核心组件，它让 goroutine 能够高效运行在少量操作系统线程上。GMP模型是Go调度器的精髓。

**GMP模型的三元素：**

| 元素 | 全称 | 说明 |
|------|------|------|
| G | Goroutine | 一个Go代码片段的执行上下文，包含栈、指令指针等 |
| M | Machine | 操作系统线程，运行G的实体 |
| P | Processor | 调度的上下文，管理G队列，决定哪个G在哪个M上运行 |

**GMP架构图：**

```
                        全局运行队列
                           │
                           ▼
    ┌─────────────────────────────────────────────┐
    │                   P0                         │
    │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐            │
    │  │ G1  │ │ G2  │ │ G3  │ │ G4  │  本地队列   │
    │  └─────┘ └─────┘ └─────┘ └─────┘            │
    └─────────────────────────────────────────────┘
              │                       │
              ▼                       ▼
         ┌───────┐               ┌───────┐
         │  M0   │               │  M1   │
         │ (线程)│               │ (线程)│
         └───────┘               └───────┘
              │                       │
              ▼                       ▼
         ┌─────────────────────────────────────────┐
         │              操作系统内核                 │
         │         (负责任务调度、线程管理)          │
         └─────────────────────────────────────────┘
```

**调度器工作原理：**

1. **M获取P**：线程M必须持有P才能执行G
2. **G队列**：每个P维护一个本地G队列
3. **工作窃取**：当P的队列为空时，从其他P或全局队列偷G
4. **系统调用**：M进入系统调用时，会释放P，让其他M可以继续工作

```go
package main

import (
    "fmt"
    "runtime"
    "sync"
    "time"
)

// 观察GOMAXPROCS对性能的影响
func main() {
    // GOMAXPROCS控制同时运行的P数量
    // 默认等于CPU核心数

    cpus := runtime.NumCPU()
    fmt.Printf("Available CPUs: %d\n", cpus)
    fmt.Printf("Current GOMAXPROCS: %d\n", runtime.GOMAXPROCS(0))

    // 设置为1 - 单线程执行
    runtime.GOMAXPROCS(1)

    var wg sync.WaitGroup
    start := time.Now()

    // 模拟CPU密集型任务
    for i := 0; i < 4; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()
            // 做一些计算
            n := 0
            for i := 0; i < 100_000_000; i++ {
                n += i
            }
            fmt.Printf("Goroutine %d finished\n", id)
        }(i)
    }

    wg.Wait()
    singleThread := time.Since(start)
    fmt.Printf("Single thread time: %v\n", singleThread)

    // 设置为所有CPU - 并行执行
    runtime.GOMAXPROCS(cpus)
    start = time.Now()

    for i := 0; i < 4; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()
            n := 0
            for i := 0; i < 100_000_000; i++ {
                n += i
            }
            fmt.Printf("Goroutine %d finished\n", id)
        }(i)
    }

    wg.Wait()
    multiThread := time.Since(start)
    fmt.Printf("Multi thread time: %v\n", multiThread)
    fmt.Printf("Speedup: %.2fx\n", float64(singleThread)/float64(multiThread))
}
```

### 2.4 协程：goroutine原理

goroutine是Go并发模型的核心，它比传统线程更轻量、更高效。

**goroutine vs 线程：**

| 维度 | 线程 | goroutine |
|------|------|-----------|
| 创建成本 | 1-2MB栈 | 2KB栈 |
| 切换成本 | 用户态到内核态 | 用户态切换 |
| 调度 | 内核抢占式 | 协作式+抢占式 |
| 数量级 | 数千个 | 数十万个 |

**goroutine调度时机：**

1. `go` 语句创建新的goroutine
2. 通道操作（send、receive、close）
3. `runtime.Gosched()` 主动让出
4. 阻塞的系统调用（read、write）
5. 垃圾回收暂停后
6. 主动阻塞（mutex、sleep）

```go
package main

import (
    "fmt"
    "runtime"
    "sync"
    "sync/atomic"
)

// 测试goroutine的创建开销
func main() {
    // 测试1：单线程顺序创建
    const count = 100_000
    var counter int64

    // 串行创建
    for i := 0; i < count; i++ {
        counter++
    }
    fmt.Printf("Serial: %d operations done\n", counter)

    // 并发创建 - 使用sync.WaitGroup等待所有goroutine完成
    counter = 0
    var wg sync.WaitGroup
    start := runtime.Now()

    for i := 0; i < count; i++ {
        wg.Add(1)
        go func() {
            atomic.AddInt64(&counter, 1)
            wg.Done()
        }()
    }

    wg.Wait()
    fmt.Printf("Concurrent (100k goroutines): %v\n", runtime.Since(start))
    fmt.Printf("Final counter: %d\n", counter)

    // 测试更大的并发数
    counter = 0
    const bigCount = 1_000_000
    start = runtime.Now()

    for i := 0; i < bigCount; i++ {
        wg.Add(1)
        go func() {
            atomic.AddInt64(&counter, 1)
            wg.Done()
        }()
    }

    wg.Wait()
    fmt.Printf("Concurrent (1M goroutines): %v\n", runtime.Since(start))
    fmt.Printf("Final counter: %d\n", counter)
}
```

### 2.5 我的思考：Go为什么并发这么强

Go的并发强大源于三个核心设计的完美配合：

**1. goroutine的极致轻量**

goroutine初始栈只有2KB，而线程需要1-2MB。这意味着你可以轻松创建数百万个goroutine而不耗尽内存。

```go
// 百万goroutine示例
package main

import "fmt"

func main() {
    const n = 1_000_000
    done := make(chan bool, n)

    for i := 0; i < n; i++ {
        go func(id int) {
            // 每个goroutine只占用2KB栈
            _ = id * 2
            done <- true
        }(i)
    }

    // 等待所有goroutine完成
    for i := 0; i < n; i++ {
        <-done
    }

    fmt.Println("All goroutines completed!")
}
```

**2. 工作窃取调度**

Go的调度器采用工作窃取算法，空闲的P会从其他P的队列中偷取goroutine。这种去中心化的设计避免了单点瓶颈。

**3. M:N模型**

Go采用M:N模型（M个线程运行N个goroutine），既避免了单线程的局限性，又不会为每个goroutine创建线程的开销。

**数组：固定长度，值类型**

### 4.2 Channel：通信顺序进程

channel是Go独特的并发原语，实现了CSP（Communicating Sequential Processes）模型。

```go
package main

import "fmt"

// Channel基础操作
func main() {
    // 创建一个无缓冲channel
    ch1 := make(chan int)

    // 创建一个有缓冲channel
    ch2 := make(chan int, 3)

    // 发送数据 - 阻塞直到有人接收
    go func() {
        ch1 <- 42
    }()

    // 接收数据
    value := <-ch1
    fmt.Printf("Received: %d\n", value)

    // 有缓冲channel - 不阻塞直到缓冲区满
    ch2 <- 1
    ch2 <- 2
    ch2 <- 3
    // ch2 <- 4 // 阻塞，因为缓冲区已满

    fmt.Printf("Bufferred channel len: %d\n", len(ch2))

    // 关闭channel
    close(ch2)

    // 遍历channel - 只能遍历关闭的或空的channel
    for v := range ch2 {
        fmt.Printf("Read from closed channel: %d\n", v)
    }

    // 多返回值接收 - 区分零值和关闭
    ch3 := make(chan int, 1)
    ch3 <- 100
    close(ch3)

    val, ok := <-ch3
    fmt.Printf("Value: %d, Ok: %t\n", val, ok) // Value: 100, Ok: true

    val, ok = <-ch3
    fmt.Printf("Value: %d, Ok: %t\n", val, ok) // Value: 0, Ok: false
}
```

**Channel的高级用法：**

```go
package main

import (
    "fmt"
    "time"
)

// 单向channel - 用于函数参数限制方向
func sender(ch chan<- int) { // 只能发送
    for i := 0; i < 5; i++ {
        ch <- i
    }
    close(ch)
}

func receiver(ch <-chan int) { // 只能接收
    for v := range ch {
        fmt.Printf("Received: %d\n", v)
    }
}

func fanOut() {
    // Fan-out模式：多个goroutine从同一个channel读取
    ch := make(chan int, 10)

    // 启动多个worker
    for i := 0; i < 3; i++ {
        go func(id int) {
            for v := range ch {
                fmt.Printf("Worker %d processed: %d\n", id, v)
            }
        }(i)
    }

    // 发送数据
    for i := 0; i < 9; i++ {
        ch <- i
    }
    close(ch)

    time.Sleep(100 * time.Millisecond)
}

func pipeline() {
    // Pipeline模式：多个channel串联
    // stage 1: 生成数字
    numbers := make(chan int)
    go func() {
        for i := 0; i < 10; i++ {
            numbers <- i
        }
        close(numbers)
    }()

    // stage 2: 平方
    squares := make(chan int)
    go func() {
        for n := range numbers {
            squares <- n * n
        }
        close(squares)
    }()

    // stage 3: 打印
    for s := range squares {
        fmt.Printf("Square: %d\n", s)
    }
}

func main() {
    // 单向channel演示
    ch := make(chan int)
    go sender(ch)
    receiver(ch)

    fmt.Println("\n--- Fan-out ---")
    fanOut()

    fmt.Println("\n--- Pipeline ---")
    pipeline()
}
```

### 4.3 Select：多路复用

select允许同时监听多个channel的操作，就像Linux的select系统调用。

```go
package main

import (
    "fmt"
    "time"
)

// select基础用法
func basicSelect() {
    ch1 := make(chan string, 1)
    ch2 := make(chan string, 1)

    ch1 <- "from channel 1"

    // select会随机选择一个可用的case
    select {
    case msg1 := <-ch1:
        fmt.Println(msg1)
    case msg2 := <-ch2:
        fmt.Println(msg2)
    // 可以添加default分支，当没有channel准备好时执行
    // default:
    //     fmt.Println("no channel ready")
    }
}

// 超时处理
func timeout() {
    ch := make(chan int)

    select {
    case v := <-ch:
        fmt.Printf("Received: %d\n", v)
    case <-time.After(2 * time.Second):
        fmt.Println("Timeout! No message received")
    }
}

// 关闭通知
func closeNotification() {
    done := make(chan struct{})

    go func() {
        time.Sleep(500 * time.Millisecond)
        close(done) // 发送关闭信号
    }()

    select {
    case <-done:
        fmt.Println("Task completed!")
    case <-time.After(1 * time.Second):
        fmt.Println("Timeout!")
    }
}

// 监听多个channel
func multiChannel() {
    ch1 := make(chan int)
    ch2 := make(chan int)
    quit := make(chan struct{})

    go func() {
        for i := 0; i < 5; i++ {
            ch1 <- i
            time.Sleep(100 * time.Millisecond)
        }
    }()

    go func() {
        for i := 10; i < 15; i++ {
            ch2 <- i
            time.Sleep(150 * time.Millisecond)
        }
    }()

    loop := true
    for loop {
        select {
        case v := <-ch1:
            fmt.Printf("ch1: %d\n", v)
        case v := <-ch2:
            fmt.Printf("ch2: %d\n", v)
        case <-time.After(500 * time.Millisecond):
            fmt.Println("timeout, exiting")
            loop = false
        }
    }
}

func main() {
    fmt.Println("--- Basic Select ---")
    basicSelect()

    fmt.Println("\n--- Timeout ---")
    timeout()

    fmt.Println("\n--- Close Notification ---")
    closeNotification()

    fmt.Println("\n--- Multi Channel ---")
    multiChannel()
}
```

### 4.4 Context：上下文传播

context用于在goroutine之间传递请求作用域的取消信号、超时和截止时间。

```go
package main

import (
    "context"
    "fmt"
    "time"
)

// Context基本用法
func basicContext() {
    // 创建带超时的context
    ctx, cancel := context.WithTimeout(context.Background(), 3*time.Second)
    defer cancel()

    select {
    case <-time.After(1 * time.Second):
        fmt.Println("Operation completed before timeout")
    case <-ctx.Done():
        fmt.Println("Context cancelled:", ctx.Err())
    }
}

// 向下传播取消信号
func childContext() {
    // 创建可取消的context
    ctx, cancel := context.WithCancel(context.Background())

    // 创建子context
    childCtx, childCancel := context.WithCancel(ctx)

    go func() {
        select {
        case <-time.After(2 * time.Second):
            fmt.Println("Child operation completed")
            childCancel() // 取消子context
        case <-childCtx.Done():
            fmt.Println("Child context cancelled:", childCtx.Err())
        }
    }()

    go func() {
        select {
        case <-time.After(5 * time.Second):
            fmt.Println("Parent operation completed")
            cancel() // 取消父context
        case <-ctx.Done():
            fmt.Println("Parent context cancelled:", ctx.Err())
        }
    }()

    time.Sleep(3 * time.Second)
}

// Context传递值
func contextWithValue() {
    type userKey string
    const uidKey userKey = "userId"

    ctx := context.WithValue(context.Background(), uidKey, "user123")
```

### 4.5 实战：并发爬虫实现

```go
package main

import (
    "context"
    "fmt"
    "net/http"
    "sync"
    "time"
)

// 并发爬虫示例
type Result struct {
    URL  string
    Size int
    Err  error
}

// 带超时的并发爬虫
func concurrentCrawler(ctx context.Context, urls []string, concurrency int) []Result {
    results := make([]Result, len(urls))

    // 创建带超时的context
    ctx, cancel := context.WithTimeout(ctx, 10*time.Second)
    defer cancel()

    // URL队列
    urlChan := make(chan string, len(urls))
    for i, url := range urls {
        urlChan <- url
        // 立即标记我们开始处理这个URL
        results[i] = Result{URL: url}
    }
    close(urlChan)

    // 结果通道
    resultChan := make(chan Result, concurrency)

    var wg sync.WaitGroup

    // 启动固定数量的worker
    for i := 0; i < concurrency; i++ {
        wg.Add(1)
        go func(workerID int) {
            defer wg.Done()

            for url := range urlChan {
                select {
                case <-ctx.Done():
                    // 超时或被取消
                    resultChan <- Result{URL: url, Err: ctx.Err()}
                    return
                default:
                    // 执行爬取
                    size, err := fetchURL(ctx, url)
                    resultChan <- Result{URL: url, Size: size, Err: err}
                }
            }
        }(i)
    }

    // 等待所有worker完成
    go func() {
        wg.Wait()
        close(resultChan)
    }()

    // 收集结果
    for r := range resultChan {
        for i, res := range results {
            if res.URL == r.URL {
                results[i] = r
                break
            }
        }
    }

    return results
}

func fetchURL(ctx context.Context, url string) (int, error) {
    // 创建请求，带上context
    req, err := http.NewRequestWithContext(ctx, "GET", url, nil)
    if err != nil {
        return 0, err
    }

    // 发送请求
    resp, err := http.DefaultClient.Do(req)
    if err != nil {
        return 0, err
    }
    defer resp.Body.Close()

    return int(resp.StatusCode), nil
}

func main() {
    urls := []string{
        "https://example.com",
        "https://golang.org",
        "https://go.dev",
        "https://pkg.go.dev",
        "https://github.com",
    }

    fmt.Println("Starting concurrent crawler with 3 workers...")
    start := time.Now()

    results := concurrentCrawler(context.Background(), urls, 3)

    fmt.Printf("\nResults (took %v):\n", time.Since(start))
    for _, r := range results {
        if r.Err != nil {
            fmt.Printf("  %s: ERROR - %v\n", r.URL, r.Err)
        } else {
            fmt.Printf("  %s: %d bytes\n", r.URL, r.Size)
        }
    }
}
```

## 5. 接口与多态

### 5.1 接口是隐式实现

Go的接口是隐式实现的，不像Java需要显式声明`implements`关键字。

```go
package main

import "fmt"

// 定义接口
type Writer interface {
    Write([]byte) (int, error)
}

// 定义另一个接口
type Closer interface {
    Close() error
}

// 组合接口
type WriteCloser interface {
    Writer
    Closer
}

// 实现Writer接口 - 只需要实现Write方法
type ConsoleWriter struct{}

func (w *ConsoleWriter) Write(data []byte) (int, error) {
    n, err := fmt.Print(string(data))
    return n, err
}

// 实现Closer接口
func (w *ConsoleWriter) Close() error {
    fmt.Println("\nConsole closed")
    return nil
}

// 函数接受接口
func processData(w Writer, data []byte) {
    w.Write(data)
}

func main() {
    // 创建实现者
    cw := &ConsoleWriter{}

    // 接口是隐式实现 - 无需声明 implements
    var w Writer = cw
    var c Closer = cw
    var wc WriteCloser = cw

    fmt.Printf("Writer interface: %T\n", w)
    fmt.Printf("Closer interface: %T\n", c)
    fmt.Printf("WriteCloser interface: %T\n", wc)

    // 使用接口
    processData(w, []byte("Hello, interface!\n"))

    // 接口组合
    wc.Close()
}
```

### 5.5 实战：策略模式实现

```go
package main

import (
    "encoding/json"
    "fmt"
    "time"
)

// 策略接口
type CompressionStrategy interface {
    Compress(data []byte) ([]byte, error)
    Decompress(data []byte) ([]byte, error)
}

// 具体策略1：JSON压缩（模拟）
type JSONCompression struct{}

func (j *JSONCompression) Compress(data []byte) ([]byte, error) {
    // 模拟压缩：返回原始数据的JSON表示
    return json.Marshal(string(data))
}

func (j *JSONCompression) Decompress(data []byte) ([]byte, error) {
    var s string
    if err := json.Unmarshal(data, &s); err != nil {
        return nil, err
    }
    return []byte(s), nil
}

// 具体策略2：GZIP压缩（模拟）
type GZIPCompression struct {
    level int
}

func (g *GZIPCompression) Compress(data []byte) ([]byte, error) {
    // 模拟压缩
    return []byte(fmt.Sprintf("GZIP(level=%d):%s", g.level, string(data))), nil
}

func (g *GZIPCompression) Decompress(data []byte) ([]byte, error) {
    // 模拟解压缩
    s := string(data)
    if len(s) > 20 && s[:5] == "GZIP(" {
        return []byte(s[20:]), nil
    }
    return data, nil
}

// 具体策略3：不压缩
type NoCompression struct{}

func (n *NoCompression) Compress(data []byte) ([]byte, error) {
    return data, nil
}

func (n *NoCompression) Decompress(data []byte) ([]byte, error) {
    return data, nil
}

// 上下文：使用策略的对象
type Compressor struct {
    strategy CompressionStrategy
}

func (c *Compressor) SetStrategy(strategy CompressionStrategy) {
    c.strategy = strategy
}

func (c *Compressor) Compress(data []byte) ([]byte, error) {
    return c.strategy.Compress(data)
}

func (c *Compressor) Decompress(data []byte) ([]byte, error) {
    return c.strategy.Decompress(data)
}

func main() {
    compressor := &Compressor{}
    original := []byte("Hello, World! This is a test message for compression.")

    strategies := []CompressionStrategy{
        &NoCompression{},
        &JSONCompression{},
        &GZIPCompression{level: 6},
        &GZIPCompression{level: 9},
    }

    for _, strategy := range strategies {
        compressor.SetStrategy(strategy)

        start := time.Now()
        compressed, err := compressor.Compress(original)
        elapsed := time.Since(start)

        if err != nil {
            fmt.Printf("Error: %v\n", err)
            continue
        }

        decompressed, _ := compressor.Decompress(compressed)

        fmt.Printf("Strategy: %T\n", strategy)
        fmt.Printf("  Original size:   %d bytes\n", len(original))
        fmt.Printf("  Compressed size: %d bytes\n", len(compressed))
        fmt.Printf("  Ratio:           %.2f%%\n", float64(len(compressed))/float64(len(original))*100)
        fmt.Printf("  Time:            %v\n", elapsed)
        fmt.Printf("  Data intact:     %t\n", string(decompressed) == string(original))
        fmt.Println()
    }
}
```

### 6.3 Panic vs Recover

panic和recover是Go的异常机制，但应该谨慎使用。

```go
package main

import "fmt"

// panic会停止当前goroutine的执行
// recover可以捕获panic，恢复正常执行

func safeDivision(a, b float64) (result float64) {
    defer func() {
        // recover只能在defer中调用
        if r := recover(); r != nil {
            fmt.Printf("Recovered from panic: %v\n", r)
            result = 0
        }
    }()

    if b == 0 {
        panic("division by zero")
    }
    return a / b
}

func panicExample() {
    fmt.Println("Before panic")

    // panic会立即停止执行
    panic("something went wrong")

    fmt.Println("After panic") // 永远不会执行
}

func recoverExample() {
    fmt.Println("Calling panicExample with recover...")

    defer func() {
        if r := recover(); r != nil {
            fmt.Printf("Caught panic in main: %v\n", r)
        }
    }()

    panicExample()
    fmt.Println("After panicExample") // 执行到这里因为panic被捕获
}

func main() {
    // panic/recover 基本用法
    fmt.Println("=== Safe Division ===")
    fmt.Printf("10 / 2 = %f\n", safeDivision(10, 2))
    fmt.Printf("10 / 0 = %f\n", safeDivision(10, 0))

    fmt.Println("\n=== Recover from Panic ===")
    recoverExample()

    fmt.Println("\nProgram continues normally...")
}
```

```go
    if err := withTimeout(); err != nil {
        fmt.Printf("Timeout error: %v\n", err)
    }

    // 批量错误
    fmt.Println("\n=== Batch Error Handling ===")
    items := []string{"good1", "bad", "good2", "bad", "good3"}
    if err := processAll(items); err != nil {
        fmt.Printf("Batch error: %v\n", err)
        if me, ok := err.(*MultiError); ok {
            for i, e := range me.errors {
                fmt.Printf("  Error %d: %v\n", i+1, e)
            }
        }
    } else {
        fmt.Println("All items processed successfully")
    }

    // 日志记录最佳实践
    fmt.Println("\n=== Logging Best Practices ===")
    err := errors.New("database connection failed")

    // 使用log包记录错误，但仍然返回
    log.Printf("Failed to connect to database: %v", err)

    // 包装后再记录
    wrapped := fmt.Errorf("user service: %w", err)
    log.Printf("Service error: %v", wrapped)
}
```

### 6.5 我的思考：Go错误处理的哲学

Go的错误处理哲学可以概括为：**显式优于隐式，简单优于复杂，信任优于防护**。

**为什么Go选择这种设计？**

1. **性能考虑**：传统的try-catch机制需要编译器插入大量代码来维护异常栈，而Go的错误只是一个返回值。

2. **可读性**：错误处理的逻辑就在眼前，不会跳到几百行外的catch块。

3. **控制流清晰**：函数可能在任何地方出错，不需要理解调用栈就能知道哪里可能出问题。

**什么时候用panic？**

- 程序无法继续运行的极端情况
- 编程错误（如数组越界，这在测试阶段应该被发现）
- 初始化失败（如配置文件缺失）

### 8.3 中间件设计

```go
package main

import (
    "log"
    "time"

    "github.com/gin-gonic/gin"
)

// 自定义中间件
func Logger() gin.HandlerFunc {
    return func(c *gin.Context) {
        start := time.Now()
        path := c.Request.URL.Path

        // 处理请求
        c.Next()

        // 记录日志
        log.Printf("[%s] %s %s %d %v",
            c.Request.Method,
            path,
            c.ClientIP(),
            c.Writer.Status(),
            time.Since(start),
        )
    }
}

// 认证中间件
func AuthRequired() gin.HandlerFunc {
    return func(c *gin.Context) {
        token := c.GetHeader("Authorization")
        if token == "" {
            c.AbortWithStatusJSON(401, gin.H{"error": "Unauthorized"})
            return
        }

        // 验证token...
        c.Set("user_id", "user123")
        c.Next()
    }
}

// 限流中间件
func RateLimiter(maxRequests int, window time.Duration) gin.HandlerFunc {
    requests := make(map[string][]time.Time)

    return func(c *gin.Context) {
        ip := c.ClientIP()
        now := time.Now()

        // 清理旧请求
        var valid []time.Time
        for _, t := range requests[ip] {
            if now.Sub(t) < window {
                valid = append(valid, t)
            }
        }
        requests[ip] = valid

        if len(valid) >= maxRequests {
            c.AbortWithStatusJSON(429, gin.H{"error": "Too Many Requests"})
            return
        }

        requests[ip] = append(requests[ip], now)
        c.Next()
    }
}

// CORS中间件
func CORS() gin.HandlerFunc {
    return func(c *gin.Context) {
        c.Writer.Header().Set("Access-Control-Allow-Origin", "*")
        c.Writer.Header().Set("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE")
        c.Writer.Header().Set("Access-Control-Allow-Headers", "Content-Type, Authorization")

        if c.Request.Method == "OPTIONS" {
            c.AbortWithStatus(http.StatusOK)
            return
        }

        c.Next()
    }
}

func main() {
    r := gin.New()

    // 使用中间件
    r.Use(Logger())
    r.Use(Logger(), RateLimiter(100, time.Minute), CORS())

    r.GET("/public", func(c *gin.Context) {
        c.JSON(200, gin.H{"message": "Public endpoint"})
    })

    r.GET("/private", AuthRequired(), func(c *gin.Context) {
        userID, _ := c.Get("user_id")
        c.JSON(200, gin.H{"user_id": userID})
    })

    r.Run(":8080")
}
```

```go
package main

import (
    "net/http"
    "strconv"
    "sync"
    "time"

    "github.com/gin-gonic/gin"
    "github.com/golang-jwt/jwt/v5"
)

// 模型定义
type User struct {
    ID        int       `json:"id"`
    Username  string    `json:"username"`
    Email     string    `json:"email"`
    CreatedAt time.Time `json:"created_at"`
}

// 内存存储
type UserStore struct {
    mu    sync.RWMutex
    users map[int]*User
    nextID int
}

var userStore = &UserStore{
    users:  make(map[int]*User),
    nextID: 1,
}

// 添加用户
func (s *UserStore) Create(user *User) *User {
    s.mu.Lock()
    defer s.mu.Unlock()

    user.ID = s.nextID
    user.CreatedAt = time.Now()
    s.users[s.nextID] = user
    s.nextID++
    return user
}

// 获取所有用户
func (s *UserStore) GetAll() []*User {
    s.mu.RLock()
    defer s.mu.RUnlock()

    result := make([]*User, 0, len(s.users))
    for _, u := range s.users {
        result = append(result, u)
    }
    return result
}

// 获取单个用户
func (s *UserStore) GetByID(id int) *User {
    s.mu.RLock()
    defer s.mu.RUnlock()
    return s.users[id]
}

// 更新用户
func (s *UserStore) Update(id int, user *User) *User {
    s.mu.Lock()
    defer s.mu.Unlock()

    if _, exists := s.users[id]; !exists {
        return nil
    }
    user.ID = id
    s.users[id] = user
    return user
}

// 删除用户
func (s *UserStore) Delete(id int) bool {
    s.mu.Lock()
    defer s.mu.Unlock()

    if _, exists := s.users[id]; !exists {
        return false
    }
    delete(s.users, id)
    return true
}

// JWT密钥
var jwtSecret = []byte("your-secret-key")

// 生成Token
func generateToken(userID int) (string, error) {
    claims := jwt.MapClaims{
        "user_id": userID,
        "exp":     time.Now().Add(24 * time.Hour).Unix(),
    }

    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    return token.SignedString(jwtSecret)
}

func main() {
    r := gin.Default()

    // 添加初始用户
    userStore.Create(&User{Username: "admin", Email: "admin@example.com"})

    // 中间件
    r.Use(func(c *gin.Context) {
        c.Writer.Header().Set("Access-Control-Allow-Origin", "*")
        c.Writer.Header().Set("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE")
        c.Writer.Header().Set("Access-Control-Allow-Headers", "Content-Type, Authorization")
        c.Next()
    })

    // 认证中间件
    authMiddleware := func(c *gin.Context) {
        token := c.GetHeader("Authorization")
        if token == "" {
            c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{"error": "Token required"})
            return
        }

        _, err := jwt.Parse(token, func(token *jwt.Token) (interface{}, error) {
            return jwtSecret, nil
        })

        if err != nil {
            c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{"error": "Invalid token"})
            return
        }

        c.Next()
    }

    // API路由
    api := r.Group("/api/v1")
    {
        // 公开路由
        api.POST("/login", func(c *gin.Context) {
            var req struct {
                Username string `json:"username"`
                Password string `json:"password"`
            }
            if err := c.ShouldBindJSON(&req); err != nil {
                c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
                return
            }

            // 简化验证
            if req.Username == "admin" && req.Password == "admin" {
                token, _ := generateToken(1)
                c.JSON(http.StatusOK, gin.H{"token": token})
                return
            }

            c.JSON(http.StatusUnauthorized, gin.H{"error": "Invalid credentials"})
        })

        // 需要认证的路由
        protected := api.Group("")
        protected.Use(authMiddleware)
        {
            // 获取所有用户
            protected.GET("/users", func(c *gin.Context) {
                users := userStore.GetAll()
                c.JSON(http.StatusOK, gin.H{"users": users})
            })

            // 获取单个用户
            protected.GET("/users/:id", func(c *gin.Context) {
                id, err := strconv.Atoi(c.Param("id"))
                if err != nil {
                    c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid ID"})
                    return
                }

                user := userStore.GetByID(id)
                if user == nil {
                    c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
                    return
                }

                c.JSON(http.StatusOK, gin.H{"user": user})
            })

            // 创建用户
            protected.POST("/users", func(c *gin.Context) {
                var user User
                if err := c.ShouldBindJSON(&user); err != nil {
                    c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
                    return
                }

                created := userStore.Create(&user)
                c.JSON(http.StatusCreated, gin.H{"user": created})
            })

            // 更新用户
            protected.PUT("/users/:id", func(c *gin.Context) {
                id, err := strconv.Atoi(c.Param("id"))
                if err != nil {
                    c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid ID"})
                    return
                }

                var user User
                if err := c.ShouldBindJSON(&user); err != nil {
                    c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
                    return
```

### 9.4 事务处理

```go
package main

import (
    "gorm.io/gorm"
)

func transactionExample(db *gorm.DB) error {
    // 自动事务
    return db.Transaction(func(tx *gorm.DB) error {
        // 创建用户
        if err := tx.Create(&User{Name: "Alice"}).Error; err != nil {
            return err
        }

        // 创建订单
        if err := tx.Create(&Order{UserID: 1, Amount: 100}).Error; err != nil {
            return err
        }

        return nil
    })
}

// 手动事务
func manualTransaction(db *gorm.DB) error {
    tx := db.Begin()
    defer func() {
        if r := recover(); r != nil {
            tx.Rollback()
        }
    }()

    if err := tx.Create(&User{Name: "Bob"}).Error; err != nil {
        tx.Rollback()
        return err
    }

    if err := tx.Create(&Order{UserID: 2, Amount: 200}).Error; err != nil {
        tx.Rollback()
        return err
    }

    return tx.Commit().Error
}
```

```go
package main

import (
    "context"
    "fmt"
    "log"
    "net"
    "sync"

    "google.golang.org/grpc"
    "google.golang.org/grpc/reflection"

    pb "github.com/myproject/user"
)

// UserStore 内存存储
type UserStore struct {
    mu    sync.RWMutex
    users map[int32]*pb.User
    nextID int32
}

func NewUserStore() *UserStore {
    return &UserStore{
        users:  make(map[int32]*pb.User),
        nextID: 1,
    }
}

func (s *UserStore) Create(user *pb.User) *pb.User {
    s.mu.Lock()
    defer s.mu.Unlock()

    user.Id = s.nextID
    s.users[user.Id] = user
    s.nextID++
    return user
}

func (s *UserStore) GetByID(id int32) (*pb.User, bool) {
    s.mu.RLock()
    defer s.mu.RUnlock()

    user, ok := s.users[id]
    return user, ok
}

func (s *UserStore) GetAll() []*pb.User {
    s.mu.RLock()
    defer s.mu.RUnlock()

    users := make([]*pb.User, 0, len(s.users))
    for _, u := range s.users {
        users = append(users, u)
    }
    return users
}

func (s *UserStore) Update(user *pb.User) bool {
    s.mu.Lock()
    defer s.mu.Unlock()

    if _, ok := s.users[user.Id]; !ok {
        return false
    }
    s.users[user.Id] = user
    return true
}

func (s *UserStore) Delete(id int32) bool {
    s.mu.Lock()
    defer s.mu.Unlock()

    if _, ok := s.users[id]; !ok {
        return false
    }
    delete(s.users, id)
    return true
}

// gRPC服务端实现
type UserServiceServer struct {
    pb.UnimplementedUserServiceServer
    store *UserStore
}

func NewUserServiceServer() *UserServiceServer {
    return &UserServiceServer{
        store: NewUserStore(),
    }
}

func (s *UserServiceServer) CreateUser(ctx context.Context, req *pb.CreateUserRequest) (*pb.CreateUserResponse, error) {
    user := &pb.User{
        Name:  req.Name,
        Email: req.Email,
        Age:   req.Age,
    }
    created := s.store.Create(user)

    log.Printf("Created user: id=%d, name=%s", created.Id, created.Name)

    return &pb.CreateUserResponse{User: created}, nil
}

func (s *UserServiceServer) GetUser(ctx context.Context, req *pb.GetUserRequest) (*pb.User, error) {
    user, ok := s.store.GetByID(req.Id)
    if !ok {
        return nil, fmt.Errorf("user %d not found", req.Id)
    }
    return user, nil
}

func (s *UserServiceServer) ListUsers(ctx context.Context, req *pb.ListUsersRequest) (*pb.ListUsersResponse, error) {
    users := s.store.GetAll()
    return &pb.ListUsersResponse{Users: users}, nil
}

func (s *UserServiceServer) UpdateUser(ctx context.Context, req *pb.UpdateUserRequest) (*pb.UpdateUserResponse, error) {
    user := &pb.User{
        Id:    req.Id,
        Name:  req.Name,
        Email: req.Email,
        Age:   req.Age,
    }

    if !s.store.Update(user) {
        return nil, fmt.Errorf("user %d not found", req.Id)
    }

    return &pb.UpdateUserResponse{User: user}, nil
}

func (s *UserServiceServer) DeleteUser(ctx context.Context, req *pb.DeleteUserRequest) (*pb.DeleteUserResponse, error) {
    if !s.store.Delete(req.Id) {
        return nil, fmt.Errorf("user %d not found", req.Id)
    }
    return &pb.DeleteUserResponse{Success: true}, nil
}

func main() {
    lis, err := net.Listen("tcp", ":50051")
    if err != nil {
        log.Fatal(err)
    }

    s := grpc.NewServer()
    pb.RegisterUserServiceServer(s, NewUserServiceServer())

    // 启用反射，用于grpcurl等工具
    reflection.Register(s)

    log.Println("gRPC UserService starting on :50051")
    if err := s.Serve(lis); err != nil {
        log.Fatal(err)
    }
}
```

### 11.5 实战：性能优化实战

```go
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "strings"
    "testing"
)

// 优化前：字符串拼接
func BadConcat(strs []string) string {
    result := ""
    for _, s := range strs {
        result += s
    }
    return result
}

// 优化后：使用strings.Builder
func GoodConcat(strs []string) string {
    var sb strings.Builder
    for _, s := range strs {
        sb.WriteString(s)
    }
    return sb.String()
}

// 优化JSON编码
type Data struct {
    Name    string  `json:"name"`
    Age     int     `json:"age"`
    Email   string  `json:"email"`
    Address string  `json:"address"`
    Phone   string  `json:"phone"`
}

// 使用json.MarshalIndent的优化版本
func GoodMarshalJSON(d *Data) ([]byte, error) {
    buf := &bytes.Buffer{}
    enc := json.NewEncoder(buf)
    enc.SetIndent("", "  ")
    if err := enc.Encode(d); err != nil {
        return nil, err
    }
    return buf.Bytes(), nil
}

// 批量处理优化
type Item struct {
    ID    int
    Value int
}

// 串行处理
func ProcessItemsSerial(items []Item) []int {
    results := make([]int, len(items))
    for i, item := range items {
        results[i] = item.Value * 2
    }
    return results
}

// 预分配容量
func ProcessItemsPrealloc(items []Item) []int {
    results := make([]int, 0, len(items)) // 预分配容量
    for _, item := range items {
        results = append(results, item.Value*2)
    }
    return results
}

// 基准测试对比
func BenchmarkBadConcat(b *testing.B) {
```

### 12.3 健康检查

实现健康检查接口对于容器编排和服务发现至关重要。

```go
package main

import (
    "database/sql"
    "net/http"
    "time"
)

type HealthChecker interface {
    Check() error
}

type HTTPHealthCheck struct {
    URL string
}

func (h *HTTPHealthCheck) Check() error {
    resp, err := http.Get(h.URL)
    if err != nil {
        return err
    }
    defer resp.Body.Close()

    if resp.StatusCode != http.StatusOK {
        return err
    }
    return nil
}

type DBHealthCheck struct {
    DB *sql.DB
}

func (h *DBHealthCheck) Check() error {
    return h.DB.Ping()
}

// 健康检查处理器
type HealthHandler struct {
    checkers []HealthChecker
}

func NewHealthHandler() *HealthHandler {
    return &HealthHandler{
        checkers: make([]HealthChecker, 0),
    }
}

func (h *HealthHandler) AddChecker(name string, checker HealthChecker) {
    h.checkers = append(h.checkers, checker)
}

func (h *HealthHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    // 健康检查响应结构
    type CheckResult struct {
        Name   string `json:"name"`
        Status string `json:"status"`
        Error  string `json:"error,omitempty"`
    }

    results := make([]CheckResult, len(h.checkers))
    allHealthy := true

    for i, checker := range h.checkers {
        err := checker.Check()
        if err != nil {
            results[i] = CheckResult{
                Name:   fmt.Sprintf("check-%d", i+1),
                Status: "unhealthy",
                Error:  err.Error(),
            }
            allHealthy = false
        } else {
            results[i] = CheckResult{
                Name:   fmt.Sprintf("check-%d", i+1),
                Status: "healthy",
            }
        }
    }

    status := http.StatusOK
    if !allHealthy {
        status = http.StatusServiceUnavailable
    }

    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(status)
    json.NewEncoder(w).Encode(map[string]interface{}{
        "status":    map[bool]string{true: "healthy", false: "unhealthy"}[allHealthy],
        "timestamp": time.Now().UTC().Format(time.RFC3339),
        "checks":    results,
    })
}

func main() {
    handler := NewHealthHandler()

    // 添加健康检查
    handler.AddChecker("http", &HTTPHealthCheck{URL: "http://localhost:8080/ready"})

    // 注意：这里简化了，实际应该传入真实的DB连接
    // handler.AddChecker("database", &DBHealthCheck{DB: db})

    http.HandleFunc("/health", handler.ServeHTTP)
    http.HandleFunc("/ready", func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusOK)
        w.Write([]byte("OK"))
    })

    http.ListenAndServe(":8080", nil)
}
```

### 12.4 部署脚本

```bash
#!/bin/bash
# deploy.sh - 部署脚本

set -e

APP_NAME="myapp"
APP_DIR="/opt/${APP_NAME}"
RELEASE_DIR="/opt/releases/$(date +%Y%m%d_%H%M%S)"
BACKUP_DIR="/opt/backups"
LOG_FILE="/var/log/${APP_NAME}.log"

# 颜色输出
RED='\033[0;31m'
GREEN='\033[0;32m'
NC='\033[0m'

log() {
    echo -e "${GREEN}[$(date '+%Y-%m-%d %H:%M:%S')]${NC} $1"
}

error() {
    echo -e "${RED}[$(date '+%Y-%m-%d %H:%M:%S')] ERROR:${NC} $1"
}

# 创建目录
mkdir -p ${APP_DIR} ${RELEASE_DIR} ${BACKUP_DIR}

# 停止旧进程
stop() {
    log "Stopping ${APP_NAME}..."
    if pgrep -x ${APP_NAME} > /dev/null; then
        pkill ${APP_NAME}
        sleep 2
    fi
    log "Stopped"
}

# 部署新版本
deploy() {
    log "Deploying new version..."

    # 备份旧版本
    if [ -f ${APP_DIR}/${APP_NAME} ]; then
        BACKUP_NAME="${BACKUP_DIR}/${APP_NAME}_$(date +%Y%m%d_%H%M%S)"
        cp ${APP_DIR}/${APP_NAME} ${BACKUP_NAME}
        log "Backed up to ${BACKUP_NAME}"
    fi

    # 复制新版本
    cp ./${APP_NAME} ${RELEASE_DIR}/

    # 创建软链接
    ln -sfn ${RELEASE_DIR}/${APP_NAME} ${APP_DIR}/${APP_NAME}

    log "Deployed to ${RELEASE_DIR}"
}

# 启动应用
start() {
    log "Starting ${APP_NAME}..."

    cd ${APP_DIR}
    ./${APP_NAME} >> ${LOG_FILE} 2>&1 &

    sleep 3

    if pgrep -x ${APP_NAME} > /dev/null; then
        log "Started successfully"
    else
        error "Failed to start"
        exit 1
    fi
}

# 健康检查
healthcheck() {
    log "Running health check..."
    for i in {1..10}; do
        if curl -sf http://localhost:8080/health > /dev/null; then
            log "Health check passed"
            return 0
        fi
        sleep 2
    done
    error "Health check failed"
    return 1
}

# 回滚
rollback() {
    log "Rolling back..."
    if [ -f ${BACKUP_DIR}/${APP_NAME}_* ]; then
        LATEST=$(ls -t ${BACKUP_DIR}/${APP_NAME}_* | head -1)
        cp ${LATEST} ${APP_DIR}/${APP_NAME}
        log "Rolled back to ${LATEST}"
    else
        error "No backup found"
        exit 1
    fi
}

# 主流程
main() {
    stop
    deploy

    if healthcheck; then
        log "Deployment completed successfully"
    else
        error "Deployment failed, rolling back..."
        rollback
        stop
        exit 1
    fi
}

main "$@"
```

### 12.5 我的思考：Go应用的优势

Go语言在云原生和DevOps领域的优势非常明显：

**1. 静态编译，单一二进制文件**

Go应用编译后是一个独立的可执行文件，没有运行时依赖。这对于Docker和Kubernetes部署非常友好。

```dockerfile
# 最终镜像可以非常小
FROM scratch
COPY myapp /myapp
COPY certs /etc/ssl/certs
EXPOSE 8080
ENTRYPOINT ["/myapp"]
```

最终的镜像可以只有10-20MB，而JVM应用通常需要几百MB。

**2. 优秀的性能**

Go的性能接近C语言，远超Python和Ruby，适合处理高并发请求。

**3. 内置并发**

goroutine和channel让并发编程变得简单自然，不需要额外的异步框架。

**4. 快速启动**

Go应用的启动时间在毫秒级，适合serverless和容器环境。

**5. 跨平台编译**

一条命令就可以编译出任何平台的二进制文件。

| 场景 | 代表项目 |
