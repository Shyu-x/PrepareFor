# 微信支付接入完全指南

> 前言：想象你去菜市场买菜，摊主说"用微信付款吧"，你掏出手机扫码，"滴"一声，钱就过去了。你有没有想过这背后发生了什么？微信支付是如何在几秒钟内完成从扫码到扣款的全过程？作为一个开发者，如果要在自己的网站或APP里接入微信支付，需要怎么做？本指南将带你从零开始，全面掌握微信支付接入的技术细节，让你能够独立完成微信支付的对接开发。

## 一、微信支付概述

### 1.1 什么是微信支付

微信支付是腾讯公司于2013年8月推出的移动支付功能，它让用户可以通过微信客户端完成付款。与传统的现金支付和银行卡支付相比，微信支付具有以下优势：

**便捷性**：用户不需要携带现金或银行卡，只需要一部联网的智能手机即可完成支付。对于商户来说，也不需要配备POS机，一台手机就能收钱。

**社交属性**：微信支付与微信社交深度整合。用户可以通过微信好友转账、发送红包等功能，这些功能极大地促进了微信支付的普及。

**生态闭环**：微信支付与公众号、小程序、APP等微信生态内的产品无缝对接，形成了完整的商业闭环。商户可以通过微信获取用户，微信支付提供服务，交易在微信生态内完成。

### 1.2 微信支付的产品体系

微信支付提供了多种支付产品，适用于不同的场景：

**JSAPI支付**：在微信内置浏览器（微信客户端内打开的网页）中调起微信支付。用户通过点击按钮，微信会弹出支付页面，用户输入支付密码完成付款。这种方式常用于微信公众号内购、微信内网页购物等场景。

**Native支付**：商户后台生成一个支付二维码，用户使用微信"扫一扫"功能扫描二维码，完成支付。这种方式适用于PC网站、线下门店等场景。用户打开微信扫描后，会在微信中打开一个H5页面完成支付。

**APP支付**：在手机APP中调起微信支付。用户点击APP内的支付按钮，微信客户端会被调起，用户确认金额后输入密码完成支付。这种方式适用于在APP内购买商品或服务。

**小程序支付**：在小程序内部调起微信支付。小程序是微信推出的一种不需要下载安装即可使用的应用，小程序支付与JSAPI支付非常相似，但有一些细节差异。

**H5支付**：在手机浏览器（不是在微信内置浏览器）中调起微信支付。用户点击支付按钮后，会在微信外部浏览器打开一个微信支付中间页，用户完成支付后跳转回商户页面。

**刷脸支付**：用户在线下门店面对摄像头进行人脸识别，无需掏出手机即可完成支付。这种方式适用于便利店、超市、自助售货机等场景。

### 1.3 微信支付的参与方

微信支付涉及多个参与方，理解这些角色有助于理解整个支付流程：

**商户**：提供商品或服务并收取货款的一方。商户需要入驻微信支付平台，获得商户号（mch_id），才能使用微信支付收钱。

**服务商**：帮助商户接入微信支付的专业公司。有些商户自己没有技术能力接入微信支付，就会找服务商帮忙。服务商可以发展自己的子商户，帮助子商户接入微信支付。

**微信支付平台**：腾讯公司运营的支付平台，负责处理交易、结算资金、提供技术支持等。微信支付的运营主体是"财付通支付科技有限公司"。

**用户**：使用微信支付购买商品或服务的消费者。用户需要绑定银行卡或账户有余额才能使用微信支付。

**银行**：微信支付的资金最终通过银行进行划转。用户绑定的银行卡所属银行负责扣除用户账户的资金。

### 1.4 微信支付的资质要求

接入微信支付需要准备以下资质材料：

**企业资质**：
- 营业执照（大陆境内注册的企业）
- 法人身份证
- 对公银行账户信息

**特殊行业资质**：
- 餐饮行业需要食品经营许可证
- 医疗行业需要医疗机构执业许可证
- 教育行业需要办学许可证等

**网站/APP信息**：
- 已备案的域名（JSAPI支付和Native支付需要）
- APP的应用包名和签名（APP支付需要）

个人开发者无法直接申请微信支付，需要通过企业账户或服务商渠道接入。

## 二、微信支付开发准备

### 2.1 商户注册与认证

接入微信支付的第一步是注册商户账号。整个流程大致如下：

**第一步：注册微信公众号**

微信支付的很多产品（如JSAPI支付、小程序支付）需要先有一个微信公众号。打开微信公众平台官网（https://mp.weixin.qq.com/），选择注册"服务号"或"小程序"。

微信公众号分为几种类型：
- 订阅号：适合媒体、个人
- 服务号：适合企业，支持微信支付
- 小程序：轻量应用

如果你要做微信支付，需要注册服务号或小程序。

**第二步：申请微信支付商户号**

在微信公众号后台，找到"微信支付"入口，申请接入微信支付。提交企业资质材料后，微信支付团队会进行审核。审核通过后，你会获得：
- 商户号（mch_id）：标识你的商户身份，如"1234567890"
- API密钥（api_key）：用于接口签名，长32位的随机字符串
- 证书文件：包括apiclient_cert.p12和apiclient_cert.pem，用于退款等敏感操作

**第三步：配置支付目录**

在商户后台配置支付授权目录。用户发起JSAPI支付时，页面的URL必须在授权目录范围内。

### 2.2 开发环境说明

微信支付开发分为两种环境：

**沙箱环境**：微信支付提供了专门的沙箱环境供开发测试使用。在沙箱环境中，所有API调用都不会真实扣款，使用虚拟资金。沙箱环境的API地址与生产环境不同，且需要使用沙箱专用的密钥。

**生产环境**：使用真实的商户号和密钥，所有交易真实扣款。

```javascript
/**
 * 微信支付环境配置
 * 说明两种环境的区别和使用方法
 */

// 生产环境配置
const productionConfig = {
    // 微信支付的API基础地址
    baseUrl: 'https://api.mch.weixin.qq.com',

    // 商户号
    mchId: '1234567890',

    // 应用AppID（注意：这是微信分配的公众号或小程序的AppID，不是商户自己的）
    appId: 'wx1234567890abcdef',

    // API密钥（32位，在商户后台设置）
    apiKey: 'your_32位_api_key_here',

    // APIv3密钥（用于APIv3接口，32位）
    apiV3Key: 'your_32位_api_v3_key_here',

    // 证书路径（用于退款等敏感操作）
    // 可以使用 p12 格式或 pem 格式
    certPath: '/path/to/apiclient_cert.p12',

    // 证书密码（默认为商户号）
    certPassword: '1234567890'
};

// 沙箱环境配置
const sandboxConfig = {
    // 沙箱环境API地址
    baseUrl: 'https://api.mch.weixin.qq.com/sandboxnew',

    // 商户号不变
    mchId: '1234567890',

    // 应用AppID不变
    appId: 'wx1234567890abcdef',

    // 沙箱专用密钥（需要通过接口获取）
    // 沙箱密钥获取地址：https://api.mch.weixin.qq.com/sandboxnew/pay/getsignkey
    sandboxApiKey: 'your_sandbox_api_key_here'
};

/**
 * 判断当前使用哪个环境
 * 实际项目中可以通过环境变量配置
 */
function getConfig() {
    const isProduction = process.env.NODE_ENV === 'production';

    if (isProduction) {
        console.log('使用生产环境配置');
        return productionConfig;
    } else {
        console.log('使用沙箱环境配置');
        return sandboxConfig;
    }
}
```

## 三、JSAPI支付详解

### 3.1 JSAPI支付原理

JSAPI支付是微信内置浏览器中使用的支付方式。当用户在微信内置浏览器中浏览网页，点击"购买"按钮时，会触发以下流程：

```
用户浏览器 → 商户后台（下单） → 微信支付平台（下单） → 获取支付参数
       ↓
返回支付参数给浏览器
       ↓
浏览器调用微信支付中间页
       ↓
用户输入密码完成支付
       ↓
微信支付平台发送异步通知给商户
       ↓
商户更新订单状态
```

这个流程的核心是"统一下单"接口。商户先调用这个接口告诉微信支付"有一笔订单要收钱"，微信支付返回一个叫prepay_id的东西，然后浏览器用这个prepay_id调起微信支付。

### 3.2 统一下单接口

统一下单是JSAPI支付的核心接口。商户需要把订单信息和用户信息传给微信支付，微信支付返回支付参数。

```javascript
/**
 * JSAPI支付 - 统一下单接口调用
 * 文档: https://pay.weixin.qq.com/wiki/doc/apiv3/apis/chapter3_1_1.shtml
 */

/**
 * 统一下单请求参数
 */
const unifiedOrderRequest = {
    // 应用ID
    appid: 'wx1234567890abcdef',

    // 商户号
    mchid: '1234567890',

    // 商品描述
    description: '腾讯充值中心-QQ会员充值',

    // 商户订单号（我们自己的订单号）
    // 要求：32个字符内，只能是数字、大小写字母以及_-*
    // 这个订单号必须唯一，不能重复
    out_trade_no: 'ORDER_202301011200001',

    // 订单金额
    amount: {
        // 订单总金额，单位为分
        // 注意：必须是整数，不能有小数
        total: 100,

        // 货币类型，默认为人民币
        currency: 'CNY'
    },

    // 支付者信息
    payer: {
        // 用户的OpenID
        // OpenID是用户在公众号下的唯一标识
        // 需要通过微信授权登录获取

    },

    // 通知地址
    // 微信支付完成后会发送POST请求到这个地址
    // 必须是可以公网访问的HTTPS地址
    notify_url: 'https://pay.example.com/notify',

    // 交易类型
    // JSAPI支付固定为"MWEB"
    // 注意：这里跟文档可能有出入，实际以文档为准
    // trade_type: 'JSAPI',

    // 订单优惠标记
    // goods_tag: 'VIP',

    // 场景信息
    scene_info: {
        // 用户端IP

        }
    },

    // 订单失效时间
    // 格式为ISO8601
    time_expire: '2023-01-01T12:00:00+08:00'
};

/**
 * 统一下单响应结果
 */
const unifiedOrderResponse = {
    // 返回状态码
    // SUCCESS表示接口调用成功
    code: 'SUCCESS',

    // 返回信息
    message: '成功',

    // 应用ID
    appid: 'wx1234567890abcdef',

    // 商户号
    mchid: '1234567890',

    // 交易类型
    trade_type: 'JSAPI',

    // 预支付交易会话标识
    // 这是最重要的参数，用于后续调起支付
    prepay_id: 'wx201410272009395522657a690389285100',

    // 时间戳
    // 单位：秒
    timeStamp: '1399520000',

    // 随机字符串
    nonceStr: '5K8264ILTKCH16CQ2502SI8ZNMTM67VS',

    // 签名
    // 用于在浏览器端验证请求的合法性
    package: 'prepay_id=wx201410272009395522657a690389285100',

    // 签名类型
    signType: 'RSA',
}
```

### 3.3 前端调起支付

商户后端完成统一下单后，需要将支付参数返回给前端。前端使用这些参数调起微信支付。

```javascript
/**
 * 前端调起微信支付
 * 适用于微信内置浏览器（H5页面）
 */

/**
 * 后端返回的支付参数
 */
const paymentParams = {
    appId: 'wx1234567890abcdef',
    timeStamp: '1699000000',
    nonceStr: '5K8264ILTKCH16CQ2502SI8ZNMTM67VS',
    package: 'prepay_id=wx201410272009395522657a690389285100',
    signType: 'RSA',
    paySign: 'signature_string_here'  // 如果使用V3签名，这个字段可能不同
};

/**
 * 调起支付函数
 * 需要在微信内置浏览器中执行
 */
function callWechatPay(paymentParams) {
    // 方法1：使用WeixinJSBridge（推荐，兼容性好）
    if (typeof WeixinJSBridge !== 'undefined') {
        WeixinJSBridge.invoke(
            'getBrandWCPayRequest',  // 固定值，调起微信支付
            {
                appId: paymentParams.appId,      // 公众号ID
                timeStamp: paymentParams.timeStamp, // 时间戳
                nonceStr: paymentParams.nonceStr,   // 随机字符串
                package: paymentParams.package,      // 预支付订单号
                signType: paymentParams.signType,   // 签名类型
                paySign: paymentParams.paySign      // 签名
            },
            function(res) {
                // 支付结果回调
                if (res.err_msg === 'get_brand_wcpay_request:ok') {
                    // 支付成功
                    console.log('支付成功', res);
                    // 这里可以跳转到支付成功页面
                    window.location.href = '/order/success';
                } else if (res.err_msg === 'get_brand_wcpay_request:cancel') {
                    // 用户取消支付
                    console.log('用户取消支付', res);
                    // 可以提示用户重新选择支付方式
                } else {
                    // 支付失败
                    console.error('支付失败', res);
                    alert('支付失败，请重试');
                }
            }
        );
    } else {
        // 如果WeixinJSBridge还没加载完成，等待一下
        document.addEventListener('WeixinJSBridgeReady', function() {
            callWechatPay(paymentParams);
        });
    }
}

/**
 * 前端完整支付流程
 */
class WechatPayFrontend {
    constructor() {
        this.orderId = null;
    }

    /**
     * 创建订单并获取支付参数
     */
    async createOrder(productId, quantity) {
        // 1. 调用后端接口创建订单
        const response = await fetch('/api/create-order', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify({
                productId,
                quantity,
                // 支付方式
                paymentMethod: 'WECHAT_JSAPI'
            })
        });

        const result = await response.json();

        if (result.code !== 0) {
            throw new Error(result.message || '创建订单失败');
        }

        // 保存订单ID
        this.orderId = result.data.orderId;

        // 2. 返回支付参数，触发支付
        return result.data.paymentParams;
    }

    /**
     * 执行支付
     */
    async pay(paymentParams) {
        return new Promise((resolve, reject) => {
            // 调用微信支付
            if (typeof WeixinJSBridge !== 'undefined') {
                WeixinJSBridge.invoke(
                    'getBrandWCPayRequest',
                    paymentParams,
                    (res) => {
                        if (res.err_msg === 'get_brand_wcpay_request:ok') {
                            // 支付成功
                            resolve({ success: true, message: '支付成功' });
                        } else if (res.err_msg === 'get_brand_wcpay_request:cancel') {
                            // 用户取消
                            resolve({ success: false, message: '用户取消支付' });
                        } else {
                            // 支付失败
                            reject(new Error(res.err_msg || '支付失败'));
                        }
                    }
                );
            } else {
                document.addEventListener('WeixinJSBridgeReady', () => {
                    this.pay(paymentParams).then(resolve).catch(reject);
                });
            }
        });
    }

    /**
     * 完整支付流程
     */
    async payOrder(productId, quantity) {
        try {
            // 1. 创建订单并获取支付参数
            const paymentParams = await this.createOrder(productId, quantity);

            // 2. 调起支付
            const result = await this.pay(paymentParams);

            // 3. 支付结果处理
            if (result.success) {
                // 跳转到成功页面
                window.location.href = `/order/success?orderId=${this.orderId}`;
            } else {
                // 停留在当前页面，提示用户
                alert(result.message);
            }

            return result;
        } catch (error) {
            console.error('支付流程异常:', error);
            alert('支付异常，请稍后重试');
            throw error;
        }
    }
}

// 使用示例
const wechatPay = new WechatPayFrontend();

// 绑定到购买按钮
document.getElementById('buyBtn').addEventListener('click', () => {
    wechatPay.payOrder('PRODUCT_001', 1);
});
```

```javascript
/**
 * 生成随机字符串
 * 微信支付API要求每次请求都要有唯一的随机字符串
 */
function generateNonceStr(length = 32) {
    const chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789';
    let result = '';
    for (let i = 0; i < length; i++) {
        result += chars.charAt(Math.floor(Math.random() * chars.length));
    }
    return result;
}

/**
 * 生成时间戳（秒）
 */
function generateTimestamp() {
    return Math.floor(Date.now() / 1000).toString();
}

/**
 * 发送HTTP请求的封装
 */
async function request(url, method, data = null, headers = {}) {
    const config = {
        method,
        url,
        headers: {
            'Content-Type': 'application/json',
            ...headers
        }
    };

    if (data) {
        config.data = data;
    }

    try {
        const response = await axios(config);
        return response.data;
    } catch (error) {
        if (error.response) {
            // 服务器返回了错误状态码
            console.error('请求失败:', error.response.data);
            throw new Error(error.response.data.message || '请求失败');
        } else {
            // 网络错误等
            console.error('网络错误:', error.message);
            throw error;
        }
    }
}

/**
 * 获取支付参数（用于前端调起支付）
 * 文档: https://pay.weixin.qq.com/wiki/doc/apiv3/apis/chapter3_1_4.shtml
 */
async function getJsapiPayParams(prepayId) {
    const timestamp = generateTimestamp();
    const nonceStr = generateNonceStr();
    const packageStr = `prepay_id=${prepayId}`;

    // 构建签名串
    // 签名格式：appid\ntimestamp\nnoncestr\npackage\n
    const message = `${wechatConfig.appId}\n${timestamp}\n${nonceStr}\n${packageStr}\n`;

    // 使用私钥进行RSA签名
    const sign = crypto
        .createSign('RSA-SHA256')
        .update(message)
        .sign(wechatConfig.privateKey, 'base64');

    return {
        appId: wechatConfig.appId,
        timeStamp: timestamp,
        nonceStr: nonceStr,
        package: packageStr,
        signType: 'RSA',
        paySign: sign
    };
}

/**
 * 生成授权Token
 * 用于调用微信支付V3接口
 */
async function generateAuthToken(method, url, body) {
    // 获取平台证书...
    // 这里简化处理，实际需要先获取有效的平台证书

    const timestamp = generateTimestamp();
    const nonceStr = generateNonceStr();

    // 构建签名串
    const urlObj = new URL(url);
    const canonicalUrl = `${urlObj.pathname}${urlObj.search}`;

    // 签名内容
    // HTTP方法 + "\n" + URL路径 + "\n" + 时间戳 + "\n" + 随机字符串 + "\n" + 请求正文 + "\n"
    let bodyStr = '';
    if (body) {
        bodyStr = typeof body === 'string' ? body : JSON.stringify(body);
    }

    const signMessage = `${method.toUpperCase()}\n${canonicalUrl}\n${timestamp}\n${nonceStr}\n${bodyStr}\n`;

    // 使用商户私钥签名
    const sign = crypto
        .createSign('RSA-SHA256')
        .update(signMessage)
        .sign(wechatConfig.privateKey, 'base64');

    // 构造Token
    const token = `mchid="${wechatConfig.mchId}",serial_no="${wechatConfig.serialNo}",nonce_str="${nonceStr}",timestamp="${timestamp}",signature="${sign}"`;

    return token;
}

/**
 * 解密支付结果
 * 微信支付的通知内容是加密的，需要使用APIv3密钥解密
 */
function decryptPaymentResult(encryptedData) {
    // 加密算法：AEAD_AES_256_GCM
    // 密钥：APIv3密钥
    // 附加数据：associated_data（对应通知中的 associated_data）
    // 初始向量：nonce（对应通知中的 nonce）
    // 密文：ciphertext（对应通知中的 ciphertext）

    const key = Buffer.from(wechatConfig.apiV3Key, 'utf8');
    const nonce = Buffer.from(encryptedData.nonce, 'utf8');
    const ciphertext = encryptedData.ciphertext;
    const associatedData = Buffer.from(encryptedData.associated_data || '', 'utf8');

    // 使用Node.js内置的crypto模块进行解密
    const decipher = crypto.createDecipheriv(
        'aes-256-gcm',
        key,
        nonce
    );

    decipher.setAuthTag(Buffer.from(encryptedData.tag, 'base64'));
    decipher.setAAD(associatedData);

    let decrypted = decipher.update(ciphertext, 'base64', 'utf8');
    decrypted += decipher.final('utf8');

    return JSON.parse(decrypted);
}
```

## 四、Native支付详解

### 4.1 Native支付原理

Native支付是面向PC网站的支付方式。用户打开电脑上的网站，看到商品介绍，点击"购买"，页面上会显示一个二维码。用户拿出手机，打开微信"扫一扫"，扫描二维码，然后在微信中完成支付。

Native支付的核心是"一码两扫"：

**第一次扫**：商户网站生成一个二维码，微信支付把这个二维码和订单关联起来。

**第二次扫**：用户用微信扫这个二维码，微信支付会识别出这是哪个订单，然后调起微信客户端显示支付页面。

```javascript
/**
 * 前端轮询订单状态
 * 用户扫码支付后，前端需要知道支付结果
 * 可以通过轮询或者WebSocket通知
 */
class NativePayStatusChecker {
    constructor(orderNo, onSuccess, onFail) {
        this.orderNo = orderNo;
        this.onSuccess = onSuccess;
        this.onFail = onFail;
        this.maxAttempts = 60;  // 最多轮询60次
        this.interval = 3000;   // 每3秒轮询一次
        this.attempts = 0;
    }

    /**
     * 开始轮询
     */
    start() {
        console.log(`开始轮询订单状态: ${this.orderNo}`);

        this.timer = setInterval(async () => {
            this.attempts++;

            try {
                // 查询订单状态
                const orderStatus = await queryOrderStatus(this.orderNo);

                if (orderStatus === 'SUCCESS') {
                    this.stop();
                    this.onSuccess && this.onSuccess();
                } else if (orderStatus === 'CLOSED') {
                    this.stop();
                    this.onFail && this.onFail('订单已关闭');
                }
            } catch (error) {
                console.error('查询订单状态失败:', error);
            }

            // 超过最大轮询次数，停止
            if (this.attempts >= this.maxAttempts) {
                this.stop();
                this.onFail && this.onFail('查询超时');
            }
        }, this.interval);
    }

    /**
     * 停止轮询
     */
    stop() {
        if (this.timer) {
            clearInterval(this.timer);
            this.timer = null;
            console.log('停止轮询订单状态');
        }
    }
}

/**
 * 查询订单状态
 */
async function queryOrderStatus(orderNo) {
    const result = await queryOrderByOutTradeNo(orderNo);

    // 交易状态：SUCCESS、REFUND、NOTPAY、CLOSED、PAYERROR
    return result.trade_state;
}
```

## 五、APP支付详解

### 5.1 APP支付原理

APP支付是手机APP内使用的支付方式。当你打开一个APP，点击"充值"或"购买"按钮，选择"微信支付"后，APP会调起微信客户端，用户确认金额并输入支付密码完成支付。

APP支付与JSAPI支付的主要区别在于调起方式：

- **JSAPI支付**：在微信内置浏览器中调起，浏览器通过JSBridge调用微信支付
- **APP支付**：在手机APP中调起，APP通过微信开放平台SDK调起微信支付

## 六、退款操作详解

### 6.1 退款业务场景

退款是支付的反向操作。当用户付款后想要取消订单、商家发现缺货、或者用户投诉商品有问题时，都需要发起退款。

微信支付的退款有几条重要规则：

**退款时限**：默认情况下，付款后可以申请退款，但如果商户设置了"不可退款"，则需要在一定时间内申请。微信支付对退款的时间没有限制，但商户可以自行配置。

**退款金额**：可以全额退款，也可以部分退款。部分退款时，退款金额不能超过实际支付金额。

**手续费处理**：退款时，手续费通常不退给商户。也就是说，如果用户付了100元（其中6角是手续费），商户收到99.4元。退款时，商户只能退给用户99.4元，0.6元手续费不退。

**退款次数**：同一笔交易可以多次部分退款，直到全部退完为止。

## 七、对账与账单

### 7.1 对账概述

微信支付会在每天凌晨生成前一天的账单文件。商户可以下载这些文件，与自己的订单记录进行核对，确保资金安全。

账单文件包括：

**交易账单**：记录所有成功的交易，包括交易时间、订单号、金额、手续费等信息。

**退款账单**：记录所有退款交易。

**充值账单**：如果商户账户有充值操作，还会有充值账单。

## 八、常见问题与解决方案

### 8.1 签名失败

签名失败是最常见的问题之一。排查步骤：

1. **检查签名算法**：微信支付V3版本使用RSA-SHA256签名，确保使用正确的算法。

2. **检查签名内容**：签名串的格式必须严格按文档要求，包括哪些字段、字段之间用换行符分隔等。

3. **检查私钥格式**：私钥必须是PEM格式，以`-----BEGIN PRIVATE KEY-----`开头。

4. **检查证书序列号**：确保使用的是最新的商户证书序列号。

```javascript
/**
 * 签名验证辅助函数
 * 用于排查签名问题
 */

/**
 * 验证签名的正确性
 */
function verifySignature(message, signature, publicKey) {
    const verify = crypto.createVerify('RSA-SHA256');
    verify.update(message);
    return verify.verify(publicKey, signature, 'base64');
}
```

### 8.2 支付回调失败

回调失败可能的原因：

1. **公网不可达**：商户服务器必须能从公网访问，微信支付需要发送HTTP请求到你的回调地址。

2. **SSL证书问题**：回调地址必须是HTTPS，确保SSL证书有效。

3. **签名验证失败**：收到回调后，需要验证签名，确保是微信支付发送的。

```javascript
/**
 * 回调验证
 */

/**
 * 验证回调签名
 */
async function verifyNotifySignature(req) {
    // 从请求头获取签名信息
    const signature = req.headers['wechatpay-signature'];
    const timestamp = req.headers['wechatpay-timestamp'];
    const nonce = req.headers['wechatpay-nonce'];
    const serial = req.headers['wechatpay-serial'];

    // 获取请求体
    const body = req.body;

    // 构建签名串
    // 格式：时间戳\n随机字符串\n请求正文\n
    const signMessage = `${timestamp}\n${nonce}\n${JSON.stringify(body)}\n`;

    // 获取平台证书（用于验证签名）
    const platformCert = await getPlatformCertificate(serial);

    // 验证签名
    const isValid = verifySignature(signMessage, signature, platformCert);

    if (!isValid) {
        throw new Error('回调签名验证失败');
    }

    return true;
}

/**
 * 获取平台证书
 */
async function getPlatformCertificate(serialNo) {
    // 从微信支付平台获取证书
    // 实际项目中应该缓存证书，避免每次都请求
    const url = 'https://api.mch.weixin.qq.com/v3/certificates';

    const token = await generateAuthToken('GET', url);

    const result = await request(url, 'GET', null, {
        'Authorization': `WECHATPAY2-SHA256-RSA2048 ${token}`
    });

    // 证书列表
    const certs = result.data;

    // 根据序列号查找证书
    const cert = certs.find(c => c.serial_no === serialNo);

    if (!cert) {
        throw new Error('未找到对应的平台证书');
    }

    // 解密证书内容
    const decryptedCert = decryptCertificate(cert.encrypt_certificate);

    return decryptedCert;
}
```

### 8.3 订单查询不到

订单查询不到的原因：

1. **订单还没同步**：微信支付处理订单需要几秒钟，如果刚付款就查询，可能还没同步。

2. **商户号不匹配**：查询时需要传入正确的商户号。

3. **订单号错误**：检查传入的订单号是否正确。

### 8.4 退款失败

退款失败的常见原因：

1. **订单已全额退款**：不能再次退款。

2. **退款金额超限**：部分退款时，退款金额不能超过剩余可退金额。

3. **商户余额不足**：退款会从商户账户扣钱，如果余额不足会失败。

- 注册微信公众号或小程序

**关注接口版本**：微信支付有V2和V3两个版本的API。V2正在逐步废弃，建议新接入的项目直接使用V3版本。

**学习风控策略**：微信支付平台本身有强大的风控能力，但你也可以在自己的系统中增加风控逻辑。

**了解合规要求**：作为商户，有一些合规要求需要遵守，比如不得为赌博、博彩等违法业务提供支付服务。
