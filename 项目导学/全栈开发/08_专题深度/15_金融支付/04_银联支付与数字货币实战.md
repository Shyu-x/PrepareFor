# 银联支付与数字货币实战

> 前言：想象你有一张银行卡，无论是国有大银行还是地方商业银行，无论是借记卡还是信用卡，只要带有银联标识，就可以在任何一台贴有银联标识的POS机上刷卡消费，也可以在任何一台ATM机上取款。这一切都离不开银联。银联就像是中国银行卡支付领域的"高速公路"，连接着全国所有的银行网点和商户。本指南将带你深入了解银联支付的运作机制，以及数字人民币这一新兴支付方式的前世今生。

## 一、银联支付体系概述

### 1.1 什么是银联

中国银联股份有限公司（China UnionPay Co., Ltd.）是中国银行卡联合组织，成立于2002年3月，总部位于上海。在银联成立之前，中国的银行卡处于"诸侯割据"状态：工行的卡只能在工行的ATM机上用，建行的卡只能在建行的POS机上刷。银联的成立，打破了这种银行之间的壁垒，实现了全国银行卡的互联互通。

银联的核心职能可以概括为三点：

**跨行清算**：当你在商户刷工行卡消费，商户的账户可能开在建行。这笔钱的划转就需要通过银联的清算网络来完成。银联负责计算各银行之间的应收应付，然后通过央行的大额支付系统完成资金划转。

**标准制定**：银联制定了"银联卡"的标准，包括卡片物理标准、交易流程标准、安全标准等。只有符合银联标准的卡片和终端，才能使用银联网络。

**网络运营**：银联运营着中国最大的银行卡交易处理网络，每天处理数亿笔交易，连接着全国各地的银行、商户和支付机构。

### 1.2 银联的产品体系

银联提供多种支付产品，适用于不同的场景：

**银联卡支付**：最传统的银行卡支付方式。用户刷卡、插卡或挥卡（闪付），商户的POS机处理交易。银联卡包括借记卡和信用卡，以及带有芯片的IC卡。

**云闪付**：银联推出的移动支付产品。用户可以在手机上绑定银行卡，然后使用NFC功能"挥"手机完成支付，或者扫描商户的二维码支付。相比微信和支付宝，云闪付的优势是能够一站式管理多张银行卡。

**无卡快捷支付**：用户在网上购物时，输入银行卡号、姓名、身份证号、预留手机号和短信验证码即可完成支付。这种方式不需要开通网银，适合快速支付场景。

**代收代付**：企业可以通过银联从用户的银行卡中定期扣款，比如缴纳水电煤气费、还信用卡等。

**跨境支付**：银联在国际上也建立了支付网络，支持人民币和外币结算。中国游客在海外很多商户可以直接刷银联卡消费。

### 1.3 银联与第三方支付的关系

很多人搞不清楚银联和微信支付、支付宝之间的关系。简单来说：

**银联是"清算组织"**：银联本身不直接面向消费者或商户，它更像是"银行的银行"。微信支付和支付宝收到的钱，最终也要通过银联进行清算。

**微信支付和支付宝是"收单机构"**：它们帮商户收钱，然后通过银联把钱结算给商户的银行账户。

打个比方：如果把银行卡支付比作寄快递，银联就是负责分拣和转运的物流中心，微信支付和支付宝则是快递公司旗下的收件员。你把快递交给收件员（就像把钱交给微信/支付宝），快递公司再把快递送到物流中心（通过银联清算），最后物流中心把快递送到收件人手中（结算到商户账户）。

```javascript
 *    - 备付金账户 → 商户银行账户
 *    - 通过银联/网联进行清算
 *
 * 注意：自从央行要求备付金集中存管后，
 * 第三方支付机构不能再随意支配用户资金
 */
```

## 二、银联在线支付详解

### 2.1 银联在线支付原理

银联在线支付是指在互联网渠道（PC网站、手机APP等）使用的银联卡支付。相比线下刷卡，在线支付不需要用户拿出实体卡，只需输入卡号等信息即可完成支付。

银联在线支付分为几种模式：

**快捷支付**：用户首次绑卡后，下次支付只需输入支付密码或短信验证码，无需再输入完整的银行卡信息。适合经常使用某张卡的用户。

**网银支付**：用户选择银行后，跳转到银行的网银页面完成支付。适合不经常网络购物或对安全要求较高的用户。

**网关支付**：商户接入银联网关，用户在商户页面输入银行卡信息完成支付。适合没有自己账户体系的商户。

### 2.3 银联支付签名与验签

银联使用PKCS12格式证书进行签名。商户需要从银联商户平台下载签名证书，证书中包含私钥。

```javascript
/**
 * 银联支付签名实现
 */

const crypto = require('crypto');
const fs = require('fs');

/**
 * 银联签名函数
 * @param {Object} params - 待签名参数
 * @param {string} certPath - 签名证书路径
 * @param {string} certPwd - 证书密码
 * @returns {string} 签名字符串
 */
function unionpaySign(params, certPath, certPwd) {
    // 1. 排除空值和signMethod字段
    const filteredParams = {};
    for (const [key, value] of Object.entries(params)) {
        if (value !== '' && value !== null && value !== undefined && key !== 'signMethod') {
            filteredParams[key] = value;
        }
    }

    // 2. 按字典序排序
    const sortedKeys = Object.keys(filteredParams).sort();

    // 3. 拼接参数字符串
    // 格式：key=value&key=value
    const paramString = sortedKeys
        .map(key => `${key}=${filteredParams[key]}`)
        .join('&');

    console.log('签名前的字符串:', paramString);

    // 4. 读取签名证书
    const p12Data = fs.readFileSync(certPath);
    const certObj = crypto.pkcs12.parse(p12Data, certPwd);
    const privateKey = certObj.key;  // 获取私钥

    // 5. 使用SHA1withRSA算法签名
    const sign = crypto.createSign('SHA1withRSA');
    sign.update(paramString);
    const signature = sign.sign(privateKey, 'base64');

    // 6. 返回签名和签名方法
    return {
        signature,
        signMethod: '01'  // RSA
    };
}

/**
 * 银联验签函数
 * @param {Object} params - 原始参数（包含签名）
 * @param {string} verifyCertPath - 银联公钥证书路径
 * @returns {boolean} 验签结果
 */
function unionpayVerify(params, verifyCertPath) {
    // 1. 获取签名
    const signature = params.signature;
    if (!signature) {
        console.error('缺少签名');
        return false;
    }

    // 2. 排除签名和空值字段
    const filteredParams = {};
    for (const [key, value] of Object.entries(params)) {
        if (value !== '' && value !== null && value !== undefined && key !== 'signature') {
            filteredParams[key] = value;
        }
    }

    // 3. 按字典序排序
    const sortedKeys = Object.keys(filteredParams).sort();

    // 4. 拼接参数字符串
    const paramString = sortedKeys
        .map(key => `${key}=${filteredParams[key]}`)
        .join('&');

    // 5. 读取银联公钥证书
    const certData = fs.readFileSync(verifyCertPath);
    const certObj = crypto.x509.certificate(certData);
    const publicKey = certObj.publicKey;

    // 6. 验签
    const verify = crypto.createVerify('SHA1withRSA');
    verify.update(paramString);
    const result = verify.verify(publicKey, signature, 'base64');

    console.log('验签结果:', result);
    return result;
}

/**
 * 构建银联支付请求表单
 * 银联使用古老的HTML表单跳转方式
 */
function buildUnionPayForm(params, gatewayUrl) {
    // 1. 添加签名
    const signedParams = unionpaySign(
        params,
        unionpayConfig.signCertPath,
        unionpayConfig.signCertPwd
    );

    const allParams = { ...params, ...signedParams };

    // 2. 构建HTML表单
    let formHtml = `
        <form id="unionpayForm" action="${gatewayUrl}" method="post">
    `;

    for (const [key, value] of Object.entries(allParams)) {
        formHtml += `
            <input type="hidden" name="${key}" value="${value}" />
        `;
    }

    formHtml += `
            <input type="submit" value="正在跳转至银联支付..." style="display:none" />
        </form>
        <script>document.getElementById('unionpayForm').submit();</script>
    `;

    return formHtml;
}
```

### 2.4 银联支付回调处理

银联的回调机制与微信支付、支付宝类似，分为前端跳转回调和后端异步通知。

```javascript
/**
 * 银联支付回调处理
 */

/**
 * 处理银联前端回调
 * 用户支付完成后，银联会跳转回商户的前端回调地址
 */
function handleUnionPayReturn(query) {
    console.log('收到银联前端回调:', query);

    // 银联前端回调参数
    const {
        version,       // 版本号
        encoding,      // 编码
        certId,        // 证书ID
        signature,     // 签名
        txnType,       // 交易类型
        txnSubType,    // 子交易类型
        bizType,       // 产品类型
        orderId,       // 商户订单号
        queryId,       // 查询索引
        txnTime,       // 订单时间
        txnAmt,        // 交易金额
        currencyCode,  // 币种
        reqReserved,   // 请求保留域
        respCode,      // 响应码
        respMsg        // 响应信息
    } = query;

    // 1. 验证签名
    const isValid = unionpayVerify(query, unionpayConfig.verifyCertPath);

    if (!isValid) {
        console.error('银联回调签名验证失败');
        return {
            success: false,
            message: '签名验证失败'
        };
    }

    // 2. 检查响应码
    // 00表示成功
    if (respCode === '00') {
        return {
            success: true,
            orderId,
            queryId,
            txnAmt,
            message: '支付成功'
        };
    } else {
        return {
            success: false,
            orderId,
            respCode,
            respMsg,
            message: `支付失败: ${respMsg}`
        };
    }
}

/**
 * 处理银联后端异步通知
 */
async function handleUnionPayNotify(postData) {
    console.log('收到银联后端通知:', postData);

    const {
        signature,
        respCode,
        respMsg,
        orderId,
        txnAmt,
        txnTime,
        queryId
    } = postData;

    // 1. 验证签名
    const isValid = unionpayVerify(postData, unionpayConfig.verifyCertPath);

    if (!isValid) {
        console.error('银联通知签名验证失败');
        return 'fail';
    }

    // 2. 检查响应码
    if (respCode === '00') {
        // 支付成功
        await handlePaymentSuccess(orderId, txnAmt, queryId);
        return 'ok';
    } else {
        console.log(`银联通知支付失败: ${respCode} - ${respMsg}`);
        return 'ok';  // 返回ok，避免重复通知
    }
}

/**

 */

    console.log(`处理银联支付成功: 订单号=${orderId}, 金额=${amount}, 查询索引=${queryId}`);

    if (order.status === 'PAID') {

    }

    // 更新订单状态
    await updateOrderStatus(orderId, 'PAID', {
        channel: 'UNIONPAY',
        transactionId: queryId,
        paidAmount: parseInt(txnAmt) / 100,  // 转换为元
        paidAt: new Date()
    });

    console.log(`订单 ${orderId} 处理完成`);
}
```

## 三、云闪付详解

### 3.1 什么是云闪付

云闪付是银联联合各商业银行于2017年12月推出的移动支付产品。相比微信支付和支付宝，云闪付的核心竞争力在于：

**银行账户管理**：用户可以在云闪付APP中一键绑定多家银行的银行卡，统一管理多家银行的账户余额和交易明细。这对于有多张银行卡的用户非常方便。

**跨行转账免费**：云闪付支持绑定银行卡之间的跨行转账，且不收取手续费。相比一些银行APP的跨行转账收费，云闪付有优势。

**商户优惠**：银联会定期推出云闪付专项优惠活动，吸引用户使用。

**海外受理**：银联在全球多个国家和地区建立了商户网络，云闪付在海外也可以使用。

### 3.2 云闪付扫码支付

云闪付支持商户扫码和用户扫码两种模式：

**商户扫用户（被扫）**：商户使用扫码设备扫描用户手机上的云闪付付款码。这种模式与当面付类似。

**用户扫商户（主扫）**：用户使用云闪付APP扫描商户的二维码完成支付。这种模式与Native支付类似。

```javascript
/**
 * 云闪付扫码支付
 */

/**
 * 云闪付商户二维码支付（用户扫商户）
 * 商户生成二维码，用户使用云闪付APP扫描支付
 */
async function createUnionPayQrCode(orderNo, amount) {
    // 1. 调用银联统一MetaCode接口获取二维码
    const url = `${getUnionpayGateway()}gateway/api/frontTrans.do`;

    const params = {
        version: '5.0.0',           // 接口版本
        encoding: 'UTF-8',
        signMethod: '01',          // 签名方式
        txnType: '01',             // 交易类型：01-消费
        txnSubType: '07',          // 二维码支付
        bizType: '000000',         // 产品类型
        channelType: '07',         // 渠道类型：07-PC

        merId: unionpayConfig.merId,
        orderId: orderNo,
        txnTime: new Date().format('yyyyMMddHHmmss'),
        txnAmt: amount.toString(),  // 金额（分）
        currencyCode: '156',        // 币种

        // 二维码生成方式：01-底层生成，02-应用生成
        // 01表示银联生成二维码内容
        qrCodeType: '01',

        // 后端通知地址
        backUrl: unionpayConfig.backUrl
    };

    // 2. 添加签名
    const signedParams = unionpaySign(params, unionpayConfig.signCertPath, unionpayConfig.signCertPwd);
    const allParams = { ...params, ...signedParams };

    // 3. 发送请求
    const response = await axios.post(url, querystring.stringify(allParams), {
        headers: {
            'Content-Type': 'application/x-www-form-urlencoded'
        }
    });

    // 4. 解析响应
    const result = parseUnionPayResponse(response.data);

    if (result.respCode === '00') {
        // 成功，返回二维码内容
        return {
            success: true,
            qrCode: result.qrCode,  // 二维码内容（字符串）
            orderNo: result.orderId
        };
    } else {
        return {
            success: false,
            errorCode: result.respCode,
            errorMessage: result.respMsg
        };
    }
}

        signMethod: '01',
        txnType: '01',             // 消费
        txnSubType: '06',          // 条码支付
        bizType: '000000',
        channelType: '07',

        merId: unionpayConfig.merId,
        orderId: orderNo,
        txnTime: new Date().format('yyyyMMddHHmmss'),
        txnAmt: amount.toString(),
        currencyCode: '156',

        // 用户出示的付款码
        authCode: authCode,

        backUrl: unionpayConfig.backUrl
    };

    // 添加签名
    const signedParams = unionpaySign(params, unionpayConfig.signCertPath, unionpayConfig.signCertPwd);
    const allParams = { ...params, ...signedParams };

    // 发送请求
    const response = await axios.post(url, querystring.stringify(allParams), {
        headers: {
            'Content-Type': 'application/x-www-form-urlencoded'
        }
    });

    const result = parseUnionPayResponse(response.data);

    // 云闪付被扫支付的响应比较特殊
    // 如果respCode=01，表示需要输入密码或密码确认
    // 如果respCode=00，表示支付成功
    // 如果respCode=03/04/05，表示用户正在处理中，需要轮询查询
    if (result.respCode === '00') {
        return {
            success: true,
            orderNo: result.orderId,
            txnId: result.queryId,
            amount: result.txnAmt
        };
    } else if (result.respCode === '01') {
        // 需要用户输入密码
        return {
            success: false,
            needPassword: true,
            orderNo: result.orderId
        };
    } else if (['03', '04', '05'].includes(result.respCode)) {
        // 交易处理中，需要轮询
        return {
            success: false,
            needQuery: true,
            orderNo: result.orderId,
            respMessage: result.respMsg
        };
    } else {
        return {
            success: false,
            errorCode: result.respCode,
            errorMessage: result.respMsg
        };
    }
}
```

## 四、数字人民币详解

### 4.1 什么是数字人民币

数字人民币（e-CNY）是中国人民银行发行的数字形式的法定货币，由指定运营机构参与运营，向公众提供服务。数字人民币于2020年开始在深圳、苏州、成都等地进行试点，随后逐步扩展到全国多个城市。

数字人民币有以下几个显著特点：

**国家信用背书**：数字人民币是法定货币，与纸币人民币等价，具有国家信用背书。相比比特币等虚拟货币，数字人民币不存在价值大幅波动的风险。

**双离线支付**：这是数字人民币最独特的功能。即使在没有网络的情况下，只要两个手机都安装了数字人民币APP，并且碰一碰（使用NFC），就可以完成支付。这在地下商场、飞机上等网络信号不好的地方非常有用。

**可控匿名**：数字人民币遵循"小额匿名、大额可溯"的原则。单笔交易金额较小时可以匿名，保护用户隐私；但如果是洗钱、赌博等违法交易，相关部门可以追溯。

**加载智能合约**：数字人民币可以加载智能合约，实现定向支付、限时支付等功能。比如政府发放的消费券，可以使用数字人民币智能合约，确保专款专用。

### 4.2 数字人民币的使用场景

数字人民币的试点场景非常广泛：

**线下消费**：在商户的POS机上使用碰一碰、扫码等方式支付。支持数字人民币的商户越来越多，包括超市、便利店、餐厅等。

**公共交通**：地铁、公交等公共交通场景可以使用数字人民币支付。部分城市还推出了数字人民币交通卡。

**线上购物**：在电商平台选择数字人民币支付方式完成购物。

**政府补贴**：一些地方政府通过数字人民币发放消费券、补贴等。

**跨境支付**：数字人民币正在探索跨境支付场景，未来可能支持出国旅游时的支付。

### 4.3 数字人民币与支付机构的合作

数字人民币采用"双层运营"架构：

**第一层**：中国人民银行对商业银行和支付机构，发行和兑换数字人民币。

**第二层**：商业银行和支付机构向公众提供服务，包括开立钱包、交易处理等。

这意味着，微信支付、支付宝等支付机构可以作为"第二层"的运营机构，为用户提供数字人民币服务。用户可以在微信支付或支付宝APP中开通数字人民币钱包，使用数字人民币进行支付。

## 五、银行卡支付详解

### 5.1 银行卡支付的原理

银行卡支付是最传统也是最安全的支付方式之一。当用户在线下商户刷卡消费时，整个过程涉及多个参与方：

**持卡人**：使用银行卡进行支付的消费者。

**发卡银行**：发行银行卡的银行，如工商银行、农业银行等。发卡银行负责验证持卡人的身份和账户余额。

**收单机构**：与商户签约，提供POS机具和支付服务的机构。微信支付、支付宝、银联商务等都是常见的收单机构。

**卡组织**：负责不同银行间的交易路由和清算，如中国银联、Visa、MasterCard等。

**清算机构**：负责跨行资金的最终划转，如中国人民银行的大额支付系统。

银行卡支付的基本流程：

```
1. 商户收银员输入交易金额
2. 用户刷卡/插卡/挥卡
3. POS机将交易信息发送给收单机构
4. 收单机构转发给卡组织（银联）
5. 银联将交易路由到发卡银行
6. 发卡银行验证并授权
7. 授权结果逐级返回
8. 交易完成
```

## 六、跨境支付详解

### 6.1 跨境支付的原理

跨境支付是指不同国家和地区之间的支付结算行为。当中国消费者在国外网站购物，或者国外消费者在中国网站购物时，都需要用到跨境支付。

跨境支付的流程比国内支付复杂很多，主要因为涉及：

**汇率转换**：不同货币之间需要按照汇率进行换算。汇率可能是即时汇率，也可能是锁定汇率。

**外汇管制**：中国对外汇有严格的管制，跨境资金流动需要符合国家外汇管理规定。

**清算网络**：跨境支付的清算通常通过国际清算网络完成，如SWIFT（环球银行金融电信协会）。

**合规审查**：跨境交易需要通过反洗钱、反恐怖融资等合规审查。

```javascript
/**
 * 跨境支付
 */

/**
 * 跨境支付配置
 */
const crossBorderConfig = {
    // 跨境支付商户号
    // 需要申请专门的跨境支付资质
    merchantId: 'cross_border_merchant_id',

    // 货币汇率锁定有效期（秒）
    lockDuration: 300,

    // 允许的收款币种
    allowedCurrencies: ['USD', 'EUR', 'GBP', 'JPY', 'KRW', 'HKD'],

    // 允许的付款币种
    allowedPayCurrencies: ['CNY', 'USD']
};

/**
 * 跨境支付订单创建
 */
async function createCrossBorderOrder(params) {
    const {
        orderNo,
        amount,
        payCurrency,      // 付款币种
        receiveCurrency,  // 收款币种
        exchangeRate,     // 汇率（如果需要锁定）
        merchantUrl,      // 商户网站
        goodsInfo         // 商品信息
    } = params;

    // 1. 验证币种是否支持
    if (!crossBorderConfig.allowedPayCurrencies.includes(payCurrency)) {
        throw new Error(`不支持的付款币种: ${payCurrency}`);
    }

    if (!crossBorderConfig.allowedCurrencies.includes(receiveCurrency)) {
        throw new Error(`不支持的收款币种: ${receiveCurrency}`);
    }

    // 2. 如果需要汇率锁定，获取当前汇率
    let lockedRate = exchangeRate;
    if (!exchangeRate) {
        lockedRate = await lockExchangeRate(payCurrency, receiveCurrency);
    }

    // 3. 计算实际收款金额
    const receiveAmount = calculateReceiveAmount(amount, lockedRate);

    // 4. 创建订单
    const order = await createOrder({
        orderNo,
        amount,
        payCurrency,
        receiveCurrency,
        exchangeRate: lockedRate,
        receiveAmount,
        status: 'PENDING'
    });

    // 5. 调用跨境支付接口
    // 实际项目中应该调用银联、Visa或MasterCard的跨境支付接口
    const result = await callCrossBorderPaymentApi(order);

    return result;

/**
 * 锁定汇率
 * 汇率会在一段时间内保持不变
 */
async function lockExchangeRate(fromCurrency, toCurrency) {
    // 实际项目中应该调用银行或汇率服务获取实时汇率
    const exchangeRates = {
        'USD_CNY': 7.25,
        'EUR_CNY': 7.85,
        'GBP_CNY': 9.15,
        'JPY_CNY': 0.048,
        'KRW_CNY': 0.0054,
        'HKD_CNY': 0.93
    };

    const rateKey = `${fromCurrency}_${toCurrency}`;
    const rate = exchangeRates[rateKey];

    if (!rate) {
        throw new Error(`不支持的汇率兑换: ${fromCurrency} -> ${toCurrency}`);
    }

    // 锁定汇率，返回锁定ID和过期时间
    return {
        rate,
        lockId: `RATE_${Date.now()}`,
        expireTime: Date.now() + crossBorderConfig.lockDuration * 1000
    };
}

/**
 * 计算收款金额
 */
function calculateReceiveAmount(amount, rate) {
    // 金额 * 汇率
    const receiveAmount = amount * rate.rate;
    // 四舍五入到分
    return Math.round(receiveAmount * 100) / 100;
}

/**
 * 跨境支付回调处理
 */
async function handleCrossBorderNotify(notifyData) {
    console.log('收到跨境支付通知:', notifyData);

    const {
        orderNo,
        status,
        amount,
        currency,
        transactionId,
        exchangeRate: notifiedRate
    } = notifyData;

    // 验证签名
    if (!verifyCrossBorderSign(notifyData)) {
        throw new Error('签名验证失败');
    }

    if (status === 'SUCCESS') {
        // 验证汇率（防止篡改）
        const order = await findOrderByNo(orderNo);
        if (notifiedRate !== order.exchangeRate) {
            console.error(`汇率不一致: 通知${notifiedRate}, 订单${order.exchangeRate}`);
            // 汇率被篡改，需要人工处理
            await escalateToManualReview(orderNo);
            return;
        }

        await handlePaymentSuccess(orderNo, amount, transactionId);
    }

    return { success: true };
}
```

## 七、银联与数字货币的安全考量

### 7.1 银行卡支付安全

银行卡支付涉及大量的敏感信息，如何保障安全是重中之重。

**密码安全**：银行卡有支付密码，部分卡片还有cvn2码（卡背面3位数字）。这些密码信息必须加密存储，绝不能明文保存。

**卡片验证**：银行卡支付通常需要进行身份验证，如短信验证码、支付密码等。验证过程必须安全可靠。

**交易限额**：银行卡通常有单笔和日累计交易限额。超过限额的交易会被拒绝或需要额外验证。

**风控系统**：银行和支付机构都有风控系统，监测异常交易并及时预警。

```javascript
 * 银行卡支付安全实现

/**
 * CVN2加密存储
 * CVN2是信用卡背面的3位安全码
 */
function encryptCVN2(cvn2) {
    // 使用AES加密存储
    // 实际项目中应该使用安全的密钥管理服务
    const key = Buffer.from('your_32_byte_key_here', 'utf8');
    const iv = Buffer.from('your_16_byte_iv_', 'utf8');

    const cipher = crypto.createCipheriv('aes-256-cbc', key, iv);
    let encrypted = cipher.update(cvn2, 'utf8', 'hex');
    encrypted += cipher.final('hex');

    return encrypted;
}

/**
 * CVN2验证（不存储明文）
 */
function verifyCVN2(inputCvn2, storedEncryptedCvn2) {
    // 解密存储的CVN2
    const key = Buffer.from('your_32_byte_key_here', 'utf8');
    const iv = Buffer.from('your_16_byte_iv_', 'utf8');

    const decipher = crypto.createDecipheriv('aes-256-cbc', key, iv);
    let decrypted = decipher.update(storedEncryptedCvn2, 'hex', 'utf8');
    decrypted += decipher.final('utf8');

    // 恒定时间比较，防止时序攻击
    return crypto.timingSafeEqual(
        Buffer.from(inputCvn2, 'utf8'),
        Buffer.from(decrypted, 'utf8')
    );
}

     */

     * 获取用户限额

        const dailyAmount = await this.getDailyTransactionAmount(userId);

            return {
                allowed: false,

            };

    }

     * 检查交易频率

    async checkAllLimits(userId, cardNo, amount) {
```

### 7.2 数字人民币安全

数字人民币作为新生事物，其安全性也是公众关注的焦点。

**钱包安全**：数字人民币钱包有不同等级的安全措施。低等级钱包可能只需要手机号注册，高等级钱包需要实名认证。钱包的安全还依赖于手机本身的安全（如是否被Root/越狱）。

**双离线安全**：双离线支付虽然方便，但也面临安全挑战。银联和央行设计了多重安全机制，如设备认证、金额限制等。

**隐私保护**：数字人民币采用"可控匿名"设计，普通交易可以匿名，但大额交易和可疑交易可以追溯。这平衡了隐私保护与合规监管的需求。

```javascript
     * 生成离线支付凭证
     * 使用NFC或二维码技术
     */

        // 1. 验证钱包是否允许离线支付
        const walletLevel = queryDcepWalletLevel(walletId);
        if (walletLevel.level < 3) {

        // 凭证包含金额、时间戳、钱包ID等信息

            walletId,
            amount,
            timestamp: Date.now(),
            // 使用钱包私钥签名
            signature: signOfflineToken({ walletId, amount, timestamp: Date.now() })
        };

        return token;

        if (!verifyOfflineTokenSignature(token)) {
            return { success: false, reason: '凭证签名无效' };
        }

        // 2. 检查凭证是否过期（离线凭证有效期通常较短）
        const expireTime = 5 * 60 * 1000;  // 5分钟
        if (Date.now() - token.timestamp > expireTime) {
            return { success: false, reason: '凭证已过期' };
        }

        // 3. 检查离线支付限额
        const dailyOfflineAmount = await getDailyOfflineAmount(token.walletId);
```

## 八、银联与数字货币的展望

### 8.1 银联的发展方向

银联作为中国银行卡支付的核心基础设施，正在积极拥抱数字化转型：

**无卡支付**：银联正在大力推广无卡快捷支付，用户无需掏出银行卡，在APP或网页上即可完成支付。

**跨境支付**：银联在国际上持续扩大受理网络，为中国游客出境消费和境外游客在中国消费提供便利。

**产品创新**：银联推出了云闪付、碰一碰等创新产品，以及各种基于银联卡的金融科技服务。

**开放平台**：银联开放平台正在向更开放的方向发展，为商户和开发者提供更便捷的接入方式。

### 8.2 数字人民币的未来

数字人民币作为央行数字货币（CBDC）的一种，具有巨大的潜力和想象空间：

**全面推广**：数字人民币试点城市和参与机构正在逐步扩大，未来有望在全国范围内推广使用。

**跨境应用**：数字人民币正在探索跨境支付场景，未来可能支持"人民币国际化"的新路径。

**智能合约**：数字人民币加载智能合约的能力，将开启很多新的应用场景，如定向补贴、限时消费、分账支付等。

## 九、总结

### 9.1 关键要点回顾

**银联支付**：

- 银联是中国银行卡联合组织，负责跨行清算和标准制定
- 银联在线支付适用于PC网站和移动APP
- 使用PKCS12证书进行签名
- 回调处理需要验证签名并做好幂等

**云闪付**：

- 云闪付是银联推出的移动支付产品
- 支持扫码支付（主扫）和被扫码支付（被扫）

- 数字人民币是央行发行的法定数字货币

- 涉及汇率转换、外汇管制和合规审查

- 跨境支付流程比国内支付复杂
