# 支付宝支付接入完全指南

> 前言：想象你去超市买东西，结账时你说"用支付宝付"。收银员扫了一下你的付款码，"滴"一声，钱就到超市账户了。你有没有想过这背后发生了什么？作为一个开发者，如果要在自己的网站或APP里接入支付宝支付，需要怎么做？本指南将带你从零开始，全面掌握支付宝支付接入的技术细节，让你能够独立完成支付宝支付的对接开发。比起微信支付的"封闭花园"策略，支付宝更像是一个"开放平台"，接入方式更加灵活多样。

## 一、支付宝支付概述

### 1.1 什么是支付宝支付

支付宝是阿里巴巴集团于2004年推出的第三方支付平台，最初是为了解决淘宝网的交易信任问题。支付宝创造性地推出了"担保交易"模式：买家付款后，钱不是直接给卖家，而是先由支付宝代为保管，等买家确认收到商品并满意后，支付宝才把钱打给卖家。这种模式极大地促进了电商行业的发展，也让支付宝成为了中国最大的第三方支付平台。

与微信支付相比，支付宝有以下特点：

**开放性更强**：支付宝提供了更加开放的接口和更多的配置选项，商户可以根据自己的业务需求进行定制。支付宝的商户可以同时是服务商，发展下级商户。

**产品更丰富**：支付宝不仅提供基础的支付功能，还有分期付款、芝麻信用、蚂蚁花呗、商家贷款等丰富的金融产品。

**国际化程度更高**：支付宝支持多种国际货币，在全球多个国家和地区可以使用，是跨境电商的重要支付渠道。

### 1.2 支付宝支付产品体系

支付宝提供了多种支付产品，适用于不同的场景：

**当面付**：用户出示付款码，商户使用扫描枪或手机扫码收款。这种方式适用于线下门店、便利店、自动售货机等场景。用户打开支付宝APP，点击"付钱"，显示一个付款码，收银员扫码后完成付款。

**手机网站支付**：在手机浏览器中调起支付宝支付。用户点击支付按钮后，浏览器打开支付宝网页完成付款，然后跳转回商户网站。这种方式适用于WAP网站。

**电脑网站支付**：在电脑网站中调起支付宝支付。用户点击支付按钮后，页面跳转到支付宝网关完成付款，然后跳转回商户网站。适用于PC网站购物。

**APP支付**：在手机APP中调起支付宝支付。用户点击支付按钮后，APP调起支付宝客户端完成付款，然后返回APP。这种方式适用于APP内购买。

**小程序支付**：在小程序内部调起支付宝支付。与APP支付类似，但通过小程序容器调起。

### 1.3 支付宝支付的参与方

支付宝支付的参与方与微信支付类似：

**商户**：提供商品或服务并收取货款的一方。商户需要入驻支付宝开放平台，获得AppID和商家UID，才能使用支付宝收款。

**用户**：使用支付宝购买商品或服务的消费者。用户需要绑定银行卡或账户有余额才能使用支付宝。

**支付宝平台**：蚂蚁集团运营的支付平台，负责处理交易、结算资金、提供技术支持等。支付宝的运营主体是"支付宝（中国）网络技术有限公司"。

**银行**：支付宝的资金最终通过银行进行划转。支付宝本身也可以作为一种支付方式，用户可以直接用支付宝余额付款。

### 1.4 支付宝与微信支付的主要区别

| 对比项 | 支付宝 | 微信支付 |
|--------|--------|----------|
| 运营主体 | 蚂蚁集团 | 腾讯公司 |
| 核心入口 | 支付宝APP | 微信APP |
| 开放策略 | 更开放，支持服务商模式 | 相对封闭 |
| 产品丰富度 | 更多金融产品 | 较基础 |
| 技术风格 | 服务端签名为主 | 混合签名 |
| 文档风格 | 详细但较复杂 | 清晰但较简单 |

## 二、支付宝支付开发准备

### 2.1 商户注册与认证

接入支付宝支付的第一步是注册商户账号：

**第一步：注册支付宝开放平台账号**

打开支付宝开放平台官网（https://open.alipay.com/），使用企业支付宝账号登录或注册。

**第二步：创建应用**

在开放平台创建应用，选择需要接入的产品（如手机网站支付、APP支付等）。创建应用后会获得AppID。

**第三步：提交审核**

提交应用审核，支付宝团队会审核应用的功能描述、截图等内容。审核通过后，应用才能使用相应的支付产品。

**第四步：配置密钥**

支付宝使用RSA或RSA2算法进行签名。你需要生成一对RSA密钥，把公钥上传到支付宝后台，私钥保存在自己的服务端。

**第五步：签约产品**

在商户后台签约需要使用的支付产品。每个产品（如当面付、手机网站支付）都需要单独签约。

### 2.2 密钥配置详解

## 三、当面付详解

### 3.1 当面付原理

当面付是线下最常用的支付方式。用户打开支付宝APP，点击"付钱"，显示一个付款码（基于二维码）。商户使用扫描枪或手机扫码，扣除用户账户的资金，完成收款。

当面付的流程如下：

```
商户扫描用户付款码 → 商户系统调用支付宝扣款接口 → 支付宝扣除用户账户资金 → 商户账户增加资金 → 返回支付结果
```

当面付有两种模式：

**扫码支付（POS扫码）**：商户使用扫描枪扫描用户手机上的付款码。这是最常见的线下收款方式。

**被扫码支付（用户扫码）**：商户展示一个二维码，用户使用支付宝扫描并付款。这类似于电商的Native支付。

### 3.2 扫码支付开发

```javascript
/**
 * 当面付 - 扫码支付
 * 用户出示付款码，商户扫码收款
 * 文档: https://opendocs.alipay.com/apis/api_detail?apiCode=10114&codeType=publicApp
 */

const crypto = require('crypto');
const axios = require('axios');

/**
 * 统一收单交易支付接口 - 付款码支付
 * 用于商户扫描用户付款码完成扣款
 */
async function createPayByScanOrder(params) {
    // API方法名
    const method = 'alipay.trade.pay';

    // 构建请求参数
    const bizContent = {
        // 商户订单号 - 必须唯一
        out_trade_no: params.orderNo,

        // 订单总金额 - 单位元，最小0.1元
        total_amount: params.totalAmount,

        // 交易金额 - 必须等于订单总金额
        // 有些场景下可能使用红包等抵扣，实际扣款金额会不同
        // 这里通常与total_amount相同
        total_amount: params.totalAmount,

        // 付款条码 - 用户支付宝APP上的付款码
        // 付款码是用户点击"付钱"后显示的二维码内容
        // 条码格式：25-30位数字
        auth_code: params.authCode,

        // 商品标题
        subject: params.subject,

        // 销售产品码 - 固定值，代表当面付
        product_code: 'FACE_TO_FACE_PAYMENT',

        // 订单描述（可选）
        body: params.body || '',

        // 店铺名称（可选）
        store_name: params.storeName || '',

        // 门店编号（可选）
        store_id: params.storeId || ''
    };

    // 如果需要自定义超时时间
    if (params.timeout) {
        bizContent.timeout_express = params.timeout;  // 如 '5m' 表示5分钟
    }

    // 如果有优惠参数
    if (params.discountableAmount !== undefined) {
        bizContent.discountable_amount = params.discountableAmount;
    }

    // 发送请求
    const result = await executeAlipayApi(method, bizContent);

    return result;
}

/**
 * 扫码支付响应结果
 */
const scanPayResponse = {
    // 响应状态码
    code: '10000',  // 10000表示成功
    msg: 'Success',  // 状态描述

    buyer_logon_id: '159****1234',

    // 交易状态
    // TRADE_SUCCESS：交易成功
    // TRADE_FINISHED：交易完成（不可退款）
    trade_status: 'TRADE_SUCCESS',

    // 订单金额信息
    total_amount: '100.00',       // 订单总金额
    receipt_amount: '98.80',     // 实收金额
    invoice_amount: '100.00',    // 开票金额
    gmt_payment: '2023-01-01 12:00:00',  // 支付时间

    // 支付工具信息
    pay_way: 'ALIPAY_BALANCE',   // 支付方式：余额/银行卡/花呗等
    card_type: ' CREDIT',         // 支付卡类型

    // 商家优惠金额
    merchant_contribution: '0.00',  // 商家优惠
    // ...
};

/**
 * 完整的扫码支付流程
 */
class AlipayScanPay {
    constructor(config) {
        this.config = config;
    }

    /**
     * 执行扫码支付
     */
    async pay(orderNo, totalAmount, authCode, subject) {
        try {
            // 1. 调用支付宝接口发起扣款
            const result = await createPayByScanOrder({
                orderNo,
                totalAmount,
                authCode,
                subject,
                body: '商品描述',
                storeName: '示例门店'
            });

            // 2. 检查返回状态
            if (result.code !== '10000') {
                // 支付失败
                throw new Error(`支付失败: ${result.msg}, 错误码: ${result.code}`);
            }

            // 3. 验证交易状态
            if (result.trade_status !== 'TRADE_SUCCESS'
                && result.trade_status !== 'TRADE_FINISHED') {
                throw new Error(`交易状态异常: ${result.trade_status}`);
            }

            // 4. 返回支付结果
            return {
                success: true,
                tradeNo: result.trade_no,           // 支付宝交易号
                outTradeNo: result.out_trade_no,   // 商户订单号
                totalAmount: result.total_amount,   // 订单金额
                receiptAmount: result.receipt_amount,  // 实收金额
                buyerLogonId: result.buyer_logon_id,  // 买家账号
                tradeStatus: result.trade_status   // 交易状态
            };

        } catch (error) {
            // 处理支付宝返回的业务错误
            // 支付宝的错误码体系比较复杂，需要对照文档处理
            if (error.code) {
                // 根据错误码判断是否需要换码重试
                // 10003：用户授权码超时，需要重新获取
                // 20000：系统繁忙，稍后重试
                // ...
                if (error.code === '10003' || error.code === '20000') {
                    return {
                        success: false,
                        needRetry: true,
                        errorMessage: error.message
                    };
                }
            }

            throw error;
        }
    }

    /**
     * 处理支付结果
     */
    async handlePaymentResult(result, order) {
        // 1. 检查金额是否正确
        if (parseFloat(result.receiptAmount) < parseFloat(order.amount)) {
            console.error('支付金额不足:', result.receiptAmount, order.amount);
            // 金额不足，可能需要报警或人工处理
        }

        // 2. 更新订单状态
        await this.updateOrderStatus(result.outTradeNo, 'PAID', {
            tradeNo: result.tradeNo,
            paidAmount: result.receiptAmount,
            paidAt: new Date()
        });

        // 3. 打印小票（如果有打印机）
        await this.printReceipt(result, order);

        console.log('支付成功处理完成');
    }
}
```

## 四、手机网站支付详解

### 4.1 手机网站支付原理

手机网站支付（H5支付）是在手机浏览器中使用的支付方式。当用户在浏览器中浏览网站，点击购买按钮时，页面会跳转到支付宝网页完成支付，然后跳转回商户网站。

手机网站支付的流程：

```javascript
function handleAlipayReturn() {
    // 获取URL参数
    const urlParams = new URLSearchParams(window.location.search);

    const outTradeNo = urlParams.get('out_trade_no');
    const tradeNo = urlParams.get('trade_no');
    const totalAmount = urlParams.get('total_amount');
    const tradeStatus = urlParams.get('trade_status');

    console.log('支付宝返回参数:', {
        outTradeNo,
        tradeNo,
        totalAmount,
        tradeStatus
    });

    // 重要：验证签名
    // 支付宝跳转回来的参数也包含签名，需要验证
    // 如果不验证签名，可能被伪造

    // 获取签名相关参数
    const sign = urlParams.get('sign');
    const signType = urlParams.get('sign_type');

    // 验证签名
    const isValid = verifyAlipayReturnSign(urlParams, sign, signType);

    if (!isValid) {
        alert('签名验证失败，可能被篡改！');
        return;
    }

    // 检查交易状态
    if (tradeStatus === 'TRADE_SUCCESS' || tradeStatus === 'TRADE_FINISHED') {
        // 支付成功
        showSuccessPage(outTradeNo, totalAmount);
    } else {
        // 支付失败
        showFailPage(outTradeNo);
    }
}

/**
 * 验证支付宝跳转返回的签名
 */
function verifyAlipayReturnSign(urlParams, sign, signType) {
    // 获取所有返回参数
    const params = {};
    urlParams.forEach((value, key) => {
        // 排除sign相关的参数
        if (key !== 'sign' && key !== 'sign_type') {
            params[key] = value;
        }
    });

    // 按字典序排序参数
    const sortedKeys = Object.keys(params).sort();
    const signString = sortedKeys.map(key => `${key}=${params[key]}`).join('&');

    // 使用支付宝公钥验证签名
    const verifier = crypto.createVerify('RSA-SHA256');
    verifier.update(signString);

    return verifier.verify(config.alipayPublicKey, sign, 'base64');
}
```

## 七、账单核对详解

### 7.1 账单概述

支付宝会生成多种账单供商户下载核对：

**交易账单**：包含所有交易的明细，包括交易时间、订单号、金额、手续费等信息。

**退款账单**：包含所有退款的明细。

**账户账单**：包含账户资金变动的明细，包括充值、提现、利息等。

### 7.2 账单获取与解析

```javascript

        // signcustomer：商户基于支付宝担保交易收单的业务账单

        // 格式：yyyy-MM-dd
        bill_date: billDate

/**
 * 解析账单文件
 * 支付宝的账单格式与微信支付略有不同
 */
async function parseAlipayBill(billContent) {
    const lines = billContent.trim().split('\n');
    const transactions = [];

    for (const line of lines) {
        // 支付宝账单字段用制表符分隔
        const fields = line.split('\t');

        // 解析交易记录
        const transaction = parseTransactionLine(fields);
        transactions.push(transaction);
    }

    return transactions;
}

/**
 * 解析单条交易记录
 * 支付宝账单字段顺序：
 * 交易号、商家订单号、买家账号、交易时间、交易地点、商品名称、金额...
 */
function parseTransactionLine(fields) {
    return {
        transactionId: fields[0],           // 支付宝交易号
        orderNo: fields[1],                 // 商家订单号
        buyerAccount: fields[2],             // 买家账号
        transactionTime: fields[3],          // 交易时间
        transactionLocation: fields[4],      // 交易地点
        productName: fields[5],             // 商品名称
        totalAmount: parseFloat(fields[6]),  // 金额

                localAmount: localOrder.amount,
                channelAmount: channelTx.totalAmount,
                description: '金额不一致'
            });
        }
    }

    // 6. 返回结果
    return {
        billDate,
        localCount: localOrders.length,
        channelCount: transactions.length,
        differences
    };
}
```

## 八、回调通知处理

### 8.1 回调通知原理

支付宝的支付结果通知有两种方式：

**同步返回**：用户支付完成后，页面会跳转回商户的return_url，并带上支付结果参数。这种方式用于展示支付结果给用户，但商户不应该以此作为最终判定依据，因为参数可能被篡改。

**异步通知**：用户支付完成后，支付宝会发送POST请求到商户的notify_url，告知支付结果。商户收到通知后，需要返回"success"表示已收到，否则支付宝会重复发送通知。

### 8.2 回调通知开发

```javascript
/**
 * 支付宝异步通知处理
 * 这是支付安全中最关键的环节
 */

const express = require('express');
const crypto = require('crypto');

const app = express();

/**
 * 接收支付宝异步通知
 */
app.post('/api/alipay/notify', express.urlencoded({ extended: true }), async (req, res) => {
    console.log('收到支付宝通知:', req.body);

    try {
        // 1. 验证签名
        // 这是最重要的安全步骤！
        const isValid = verifyAlipayNotifySign(req.body);

        if (!isValid) {
            console.error('签名验证失败！');
            // 返回fail，支付宝会继续发送通知
            res.send('fail');
            return;
        }

        // 2. 获取通知类型
        const notifyType = req.body.notify_type;

        if (notifyType === 'trade_status_sync') {
            // 交易状态同步通知
            await handleTradeStatusSync(req.body);
        }

        // 3. 返回success给支付宝
        // 必须返回success，否则支付宝会重复发送通知
        res.send('success');

    } catch (error) {
        console.error('处理通知失败:', error);
        // 返回fail，让支付宝稍后重试
        res.send('fail');
    }
});

/**
 * 验证异步通知签名
 * 这是支付安全的核心！
 */
function verifyAlipayNotifySign(postData) {
    const sign = postData.sign;
    const signType = postData.sign_type || 'RSA2';

    // 从postData中获取所有参数，排除sign和sign_type
    const params = {};
    for (const [key, value] of Object.entries(postData)) {
        if (key !== 'sign' && key !== 'sign_type') {
            params[key] = value;
        }
    }

    // 按字典序排序参数
    const sortedKeys = Object.keys(params).sort();
    const signString = sortedKeys.map(key => {
        // 参数值需要decodeURIComponent
        // 因为支付宝会对特殊字符进行URL编码
        const value = typeof params[key] === 'string'
            ? decodeURIComponent(params[key])
            : params[key];
        return `${key}=${value}`;
    }).join('&');

    // 使用支付宝公钥验证签名
    const verifier = crypto.createVerify('RSA-SHA256');
    verifier.update(signString);

    const result = verifier.verify(alipayConfig.alipayPublicKey, sign, 'base64');

    console.log('签名验证结果:', result);
    return result;
}
```

## 十、总结

### 10.1 支付宝支付接入要点回顾

**前期准备**：

- 注册支付宝开放平台账号
- 创建应用并获取AppID
- 生成RSA密钥对，上传公钥到支付宝后台
- 签约需要的支付产品

**开发要点**：

- 支付宝使用RSA（非对称）签名，比微信支付更安全
- 所有接口调用都需要在服务端完成签名
- 支付结果以异步通知为准，同步返回仅供展示
- 收到通知后必须验证签名

**安全注意事项**：

- 私钥必须妥善保管，绝对不能泄露
- 收到回调必须验证签名
- 金额必须与本地订单比对，不能直接信任通知中的金额
- 做好幂等性处理，避免重复处理通知

### 10.2 微信支付与支付宝对比

| 对比项 | 微信支付 | 支付宝 |
|--------|----------|--------|
| 签名算法 | HMAC-SHA256（V2）/ RSA-SHA256（V3） | RSA/RSA2 |
| 签名方式 | 混合签名（部分参数参与签名） | 纯签名（排除sign参数） |
| 用户标识 | OpenID | User ID（商家UID下的用户唯一标识） |
| 支付授权 | JSAPI需要OAuth获取OpenID | APP支付需要获取user_id |
| 异步通知 | JSON格式 | FORM表单格式 |
| 账单格式 | CSV格式 | CSV格式 |
| 沙箱环境 | 有独立的沙箱环境 | 有沙箱网关 |

### 10.3 开发建议

**分阶段测试**：

1. 先在沙箱环境完成所有接口的调试
2. 生产环境先用小额交易测试
3. 确认所有功能正常后再放开正常金额

**日志记录**：

- 记录所有API调用的请求和响应
- 记录所有通知的接收和处理结果
- 日志要包含足够的上下文信息，便于排查问题

**监控告警**：

- 监控支付成功率，发现异常及时告警
- 监控异步通知的接收和处理
- 监控对账差异，发现问题及时处理
