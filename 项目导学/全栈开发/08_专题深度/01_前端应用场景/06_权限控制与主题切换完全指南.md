## 一、权限控制完整方案

### 1.1 权限控制概述

权限控制是企业级应用的核心功能，用于控制不同用户可以访问的功能和数据。权限控制的实现需要考虑三个维度：

| 维度 | 说明 | 示例 |
|------|------|------|
| **路由权限** | 控制用户可以访问哪些页面 | 普通用户不能访问管理后台 |
| **按钮权限** | 控制用户可以操作哪些按钮 | 普通用户不能看到删除按钮 |
| **数据权限** | 控制用户可以查看哪些数据 | 员工只能查看自己的数据 |

### 1.2 权限设计方案

#### 1.2.1 RBAC基于角色的权限控制

RBAC（Role-Based Access Control）是最常用的权限设计方案，核心思想是：**用户 -> 角色 -> 权限**。

```
用户（User）  <--n:n-->  角色（Role）  <--n:n-->  权限（Permission）
                              |
                              v
                         包含多个权限
```

**RBAC的优点：**
- 权限分组管理，简化授权流程
- 角色可以继承，形成权限层级
- 支持职责分离原则

**RBAC的缺点：**
- 权限粒度受限，复杂场景需要大量角色
- 不适合需要细粒度权限控制的场景

#### 1.2.2 ACL基于权限的访问控制

ACL（Access Control List）是最直接的权限方案，直接为用户分配权限列表。

```
用户（User）  <--1:n-->  权限列表（Permission List）
```

**ACL的优点：**
- 权限粒度最细，可精确控制每个用户的每个操作
- 权限配置灵活，适合复杂业务场景

**ACL的缺点：**
- 权限管理复杂，需要维护大量权限数据
- 权限继承困难，容易出现权限遗漏或冲突

#### 1.2.3 实际项目中的权限设计

在FastDocument项目中，采用的是**RBAC + ACL混合方案**：

- 大权限使用RBAC（角色）
- 细粒度权限使用ACL（具体操作）

#### 1.3.2 JWT Token存储方案

JWT（JSON Web Token）是现代Web应用最常用的认证方案。Token的存储需要考虑安全性。

```typescript
// =============================================
// Token管理工具
// 提供安全的Token存储和访问机制
// =============================================

import { User, LoginResult } from '../types/auth';

/**
 * Token存储键名常量
 * 使用统一的管理便于后期维护和修改
 */
const TOKEN_KEYS = {
  ACCESS_TOKEN: 'auth_access_token',    // 访问令牌键名
  REFRESH_TOKEN: 'auth_refresh_token', // 刷新令牌键名
  TOKEN_EXPIRES: 'auth_token_expires', // 令牌过期时间键名
} as const;

/**
 * Token存储工具类
 * 提供Token的设置、获取、清除功能
 * 自动处理Token过期检查
 */
class TokenStorage {
  /**
   * 存储访问令牌和相关信息
   * @param result 登录结果，包含Token和用户信息
   * @param rememberMe 是否记住我，影响Token过期时间
   */
  setToken(result: LoginResult, rememberMe: boolean = false): void {
    // 计算Token过期时间
    // 如果记住我，过期时间为7天，否则为1天
    const expiresIn = rememberMe ? 7 * 24 * 60 * 60 : 24 * 60 * 60;

    // 将访问令牌转换为Base64编码后存储
    // 注意：实际项目中不要在前端存储明文Token，这里仅做格式转换示例
    const encodedToken = btoa(JSON.stringify({
      token: result.accessToken,
      timestamp: Date.now(),
    }));

    // 存储访问令牌
    localStorage.setItem(TOKEN_KEYS.ACCESS_TOKEN, encodedToken);
    // 存储刷新令牌（刷新令牌通常过期时间更长）
    localStorage.setItem(TOKEN_KEYS.REFRESH_TOKEN, result.refreshToken);
    // 存储过期时间戳
    localStorage.setItem(
      TOKEN_KEYS.TOKEN_EXPIRES,
      String(Date.now() + expiresIn * 1000)
    );
  }

  /**
   * 获取当前有效的访问令牌
   * 如果Token已过期，返回null
   * @returns 有效的Token或null
   */
  getToken(): string | null {
    const encodedToken = localStorage.getItem(TOKEN_KEYS.ACCESS_TOKEN);
    const expiresAt = localStorage.getItem(TOKEN_KEYS.TOKEN_EXPIRES);

    // 如果没有Token或过期时间，直接返回null
    if (!encodedToken || !expiresAt) {
      return null;
    }

    // 检查Token是否已过期
    if (Date.now() > parseInt(expiresAt, 10)) {
      // Token已过期，清除存储的Token
      this.clearToken();
      return null;
    }

    // 解析并返回Token
    try {
      const decoded = JSON.parse(atob(encodedToken));
      return decoded.token;
    } catch {
      return null;
    }
  }

  /**
   * 获取刷新令牌
   * 用于在Access Token过期时获取新的Token
   * @returns 刷新令牌或null
   */
  getRefreshToken(): string | null {
    return localStorage.getItem(TOKEN_KEYS.REFRESH_TOKEN);
  }

  /**
   * 清除所有Token信息
   * 通常在用户退出登录时调用
   */
  clearToken(): void {
    localStorage.removeItem(TOKEN_KEYS.ACCESS_TOKEN);
    localStorage.removeItem(TOKEN_KEYS.REFRESH_TOKEN);
    localStorage.removeItem(TOKEN_KEYS.TOKEN_EXPIRES);
  }

  /**
   * 检查用户是否已登录
   * 通过检查Token是否存在且未过期来判断
   * @returns 是否已登录
   */
  isAuthenticated(): boolean {
    return this.getToken() !== null;
  }
}

// 导出单例实例
export const tokenStorage = new TokenStorage();
```

 * 用户状态接口

 * 创建用户状态管理Store
 * 使用Zustand进行状态管理，支持localStorage持久化
 */
export const useUserStore = create<UserState>()(

    (set, get) => ({
      // ========== 初始状态 ==========

      isAuthenticated: false,
      isLoading: false,
      permissions: [],
      routePermissions: [],
      buttonPermissions: [],

      // ========== Actions实现 ==========

      /**
       * 设置用户信息

       */

        user,
        isAuthenticated: true,

      /**
       * 设置权限列表
       * 分别设置ACL权限、路由权限、按钮权限
       */
      setPermissions: (permissions, routePermissions, buttonPermissions) => set({
        permissions,
        routePermissions,
        buttonPermissions,

       * 登录成功处理
       * 设置用户信息和对应的权限

      loginSuccess: (user, permissionConfig) => set({

      /**
       * 退出登录
       * 清除所有状态，恢复到初始状态
       */
      logout: () => set({
        user: null,
        isAuthenticated: false,
        isLoading: false,
        permissions: [],
        routePermissions: [],
        buttonPermissions: [],
      }),

      /**
       * 设置加载状态
       */
      setLoading: (loading) => set({ isLoading: loading }),

      /**
       * 检查是否有指定权限
       * 如果权限列表包含'*'，表示拥有所有权限
       * @param permission 权限标识符

      hasPermission: (permission) => {
        const { permissions } = get();
        // 如果权限列表包含'*'，表示拥有所有权限
        if (permissions.includes('*')) {
          return true;
        }
        return permissions.includes(permission);
      },

      /**
       * 检查是否有指定角色
       * @param role 角色类型
       */
      hasRole: (role) => {
        const { user } = get();
        return user?.role === role;
      },
    }),
    {
      // 持久化配置
      name: 'auth-user-storage',           // 存储键名
      storage: createJSONStorage(() => localStorage),  // 使用localStorage
      // 只持久化这些字段
      partialize: (state) => ({
        user: state.user,
        isAuthenticated: state.isAuthenticated,
        permissions: state.permissions,
        routePermissions: state.routePermissions,
        buttonPermissions: state.buttonPermissions,
      }),
    }
  )
);

### 1.4 权限守卫实现

#### 1.4.1 路由守卫实现

路由守卫是权限控制的第一道防线，在路由跳转前进行权限检查。

```typescript
// =============================================
// 路由权限守卫组件
// 用于保护需要特定权限才能访问的路由
// =============================================

import React, { useEffect } from 'react';
import { useRouter, usePathname } from 'next/navigation';  // Next.js路由
import { useUserStore } from '../store/userStore';  // 用户状态管理
import { LoadingSpinner } from './LoadingSpinner';  // 加载组件
import { PermissionDenied } from './PermissionDenied';  // 权限不足页面

/**
 * 路由守卫组件属性
 */
interface ProtectedRouteProps {
  children: React.ReactNode;      // 被保护的子组件
  requiredPermissions?: string[]; // 需要的权限列表（ACL模式）
  requiredRoles?: string[];       // 需要的角色列表
  redirectTo?: string;            // 无权限时重定向的目标路径
  fallback?: React.ReactNode;     // 加载中的备用UI
}

/**
 * 路由权限守卫组件
 *
 * 功能说明：
 * 1. 检查用户是否已登录
 * 2. 检查用户是否有访问该路由的权限
 * 3. 在权限不足时显示无权限页面或重定向
 *
 * 使用方式：
 * ```tsx
 * <ProtectedRoute requiredRoles={['admin']}>
 *   <AdminPanel />
 * </ProtectedRoute>
 * ```
 */
export const ProtectedRoute: React.FC<ProtectedRouteProps> = ({
  children,
  requiredPermissions = [],
  requiredRoles = [],
  redirectTo = '/login',
  fallback,
}) => {
  const router = useRouter();
  const pathname = usePathname();

  // 从用户Store获取状态和权限检查方法
  const {
    isAuthenticated,    // 是否已认证
    isLoading,          // 加载状态
    hasPermission,      // 权限检查方法
    hasRole,            // 角色检查方法
    user,               // 用户信息
  } = useUserStore();

  // 权限检查逻辑
  useEffect(() => {
    // 如果正在加载，等待加载完成
    if (isLoading) {
      return;
    }

    // 1. 检查是否已登录
    if (!isAuthenticated || !user) {
      console.log('[ProtectedRoute] 用户未登录，重定向到登录页');
      router.push(redirectTo);
      return;
    }

    // 2. 检查角色权限（如果指定了需要的角色）
    if (requiredRoles.length > 0) {
      const hasRequiredRole = requiredRoles.some(role =>
        hasRole(role as any)
      );

      if (!hasRequiredRole) {
        console.log('[ProtectedRoute] 用户角色不匹配，显示无权限页面');
        return;
      }
    }

    // 3. 检查ACL权限（如果指定了需要的权限）
    if (requiredPermissions.length > 0) {
      const hasAllPermissions = requiredPermissions.every(permission =>
        hasPermission(permission)
      );

      if (!hasAllPermissions) {
        console.log('[ProtectedRoute] 用户权限不足，显示无权限页面');
        return;
      }
    }

    // 4. 检查路由权限（通用权限检查）
    const { routePermissions } = useUserStore.getState();
    if (routePermissions.length > 0 && !routePermissions.includes('/*')) {
      // 检查当前路由是否在允许的路由列表中
      const hasRoutePermission = routePermissions.some(pattern => {
        // 支持通配符匹配，如 /documents/* 匹配 /documents/123
        if (pattern.endsWith('/*')) {
          const basePath = pattern.slice(0, -2);
          return pathname.startsWith(basePath);
        }
        return pathname === pattern;
      });

      if (!hasRoutePermission) {
        console.log('[ProtectedRoute] 路由不在允许列表中，重定向到首页');
        router.push('/unauthorized');
      }
    }
  }, [
    isAuthenticated,
    isLoading,
    user,
    requiredPermissions,
    requiredRoles,
    pathname,
    router,
    redirectTo,
  ]);

  // 加载中状态
  if (isLoading) {
    return fallback || <LoadingSpinner />;
  }

  // 未登录状态
  if (!isAuthenticated || !user) {
    return fallback || <LoadingSpinner />;
  }

  // 权限不足状态检查
  if (requiredRoles.length > 0) {
    const hasRequiredRole = requiredRoles.some(role =>
      hasRole(role as any)
    );
    if (!hasRequiredRole) {
      return <PermissionDenied />;
    }
  }

  if (requiredPermissions.length > 0) {
    const hasAllPermissions = requiredPermissions.every(permission =>
      hasPermission(permission)
    );
    if (!hasAllPermissions) {
      return <PermissionDenied />;
    }
  }

  // 权限检查通过，渲染受保护的内容
  return <>{children}</>;
};

/**
 * 路由权限配置类型
 * 用于集中管理路由权限
 */
export interface RoutePermission {
  path: string;              // 路由路径
  requiredPermissions: string[];  // 需要的权限
  requiredRoles: string[];        // 需要的角色
}

/**
 * 路由权限配置表
 * 实际项目中可以从后端接口动态获取
 */
export const ROUTE_PERMISSIONS: RoutePermission[] = [
  {
    path: '/admin',
    requiredPermissions: [],
    requiredRoles: ['admin'],
  },
  {
    path: '/dashboard',
    requiredPermissions: [],
    requiredRoles: ['admin', 'editor', 'viewer'],
  },
  {
    path: '/documents',
    requiredPermissions: ['document:read'],
    requiredRoles: [],
  },
  {
    path: '/documents/create',
    requiredPermissions: ['document:write'],
    requiredRoles: [],
  },
  {
    path: '/settings',
    requiredPermissions: [],
    requiredRoles: ['admin'],
  },

#### 1.4.2 按钮级别权限控制

按钮权限控制是路由权限的补充，用于更细粒度的UI控制。

```typescript

// 用于在组件中根据权限显示或隐藏操作按钮
// =============================================

import { useUserStore } from '../store/userStore';
import { Permission } from '../types/auth';

/**
 * usePermission Hook返回值类型
 */
interface UsePermissionReturn {
  /** 是否有指定权限 */
  hasPermission: (permission: string) => boolean;
  /** 是否有所有指定权限 */
  hasAllPermissions: (permissions: string[]) => boolean;
  /** 是否有任一指定权限 */
  hasAnyPermission: (permissions: string[]) => boolean;
  /** 是否有指定角色 */
  hasRole: (role: string) => boolean;
  /** 是否有所有指定角色 */
  hasAllRoles: (roles: string[]) => boolean;
  /** 是否有任一指定角色 */
  hasAnyRole: (roles: string[]) => boolean;
  /** 是否是管理员 */
  isAdmin: boolean;
  /** 当前用户 */
  user: ReturnType<typeof useUserStore.getState>['user'];

/**
 * 权限控制Hook
 * 在组件中使用此Hook进行权限检查
 *
 * @example
 * ```tsx
 * function DeleteButton({ documentId }) {
 *   const { hasPermission } = usePermission();
 *
 *   if (!hasPermission('document:delete')) {
 *     return null;
 *   }
 *
 *   return <button onClick={() => deleteDoc(documentId)}>删除</button>;
 * }
 * ```
 */
export function usePermission(): UsePermissionReturn {
  const store = useUserStore();
  const { hasPermission: storeHasPermission, hasRole: storeHasRole, user } = store;

  /**
   * 检查是否有指定权限
   * @param permission 权限标识符
   */
  const hasPermission = (permission: string): boolean => {
    // 管理员拥有所有权限
    const { permissions } = store;
    if (permissions.includes('*')) {
      return true;
    }
    return storeHasPermission(permission);
  };

  /**
   * 检查是否拥有所有指定权限
   * @param permissions 权限标识符数组
   */
  const hasAllPermissions = (permissions: string[]): boolean => {
    return permissions.every(permission => hasPermission(permission));
  };

  /**
   * 检查是否拥有任一指定权限
   * @param permissions 权限标识符数组
   */
  const hasAnyPermission = (permissions: string[]): boolean => {
    return permissions.some(permission => hasPermission(permission));
  };

  /**
   * 检查是否有指定角色
   * @param role 角色标识符
   */
  const hasRole = (role: string): boolean => {
    return storeHasRole(role as any);
  };

  /**
   * 检查是否拥有所有指定角色
   * @param roles 角色标识符数组
   */
  const hasAllRoles = (roles: string[]): boolean => {
    return roles.every(role => hasRole(role));
  };

  /**
   * 检查是否拥有任一指定角色
   * @param roles 角色标识符数组
   */
  const hasAnyRole = (roles: string[]): boolean => {
    return roles.some(role => hasRole(role));
  };

  /**
   * 是否是管理员
   */
  const isAdmin = hasRole('admin');

  return {
    hasPermission,
    hasAllPermissions,
    hasAnyPermission,
    hasRole,
    hasAllRoles,
    hasAnyRole,
    isAdmin,
    user,
  };
}

/**
 * 权限按钮组件属性

interface PermissionButtonProps {
  children: React.ReactNode;              // 按钮内容
  permission?: string;                     // 需要的权限
  permissions?: string[];                  // 需要的所有权限
  anyPermission?: string[];               // 需要的任一权限
  role?: string;                           // 需要的角色
  roles?: string[];                        // 需要的所有角色
  anyRole?: string[];                      // 需要的任一角色
  onClick?: () => void;                    // 点击事件
  className?: string;                      // CSS类名
  disabled?: boolean;                      // 是否禁用
  variant?: 'primary' | 'secondary' | 'danger';  // 按钮变体
  size?: 'small' | 'medium' | 'large';     // 按钮大小
  /** 权限不足时的替代操作，通常显示提示 */
  onPermissionDenied?: () => void;
}

/**
 * 权限按钮组件
 * 根据用户权限控制按钮的显示和可用性
 *
 * @example
 * ```tsx
 * <PermissionButton
 *   permission="document:delete"
 *   variant="danger"
 *   onClick={handleDelete}
 * >
 *   删除文档
 * </PermissionButton>
 * ```
 */
export const PermissionButton: React.FC<PermissionButtonProps> = ({
  children,
  permission,
  permissions,
  anyPermission,
  role,
  roles,
  anyRole,
  onClick,
  className = '',
  disabled = false,
  variant = 'primary',
  size = 'medium',
  onPermissionDenied,
}) => {
  const {
    hasPermission: checkPermission,
    hasAllPermissions,
    hasAnyPermission,
    hasRole: checkRole,
    hasAllRoles,
    hasAnyRole,
  } = usePermission();

  // 权限检查逻辑
  let hasAccess = true;
  let deniedMessage = '';

  // 1. 检查单一权限
  if (permission) {
    hasAccess = checkPermission(permission);
    deniedMessage = `需要 "${permission}" 权限`;
  }

  // 2. 检查所有权限
  if (permissions && permissions.length > 0) {
    hasAccess = hasAllPermissions(permissions);
    deniedMessage = `需要所有这些权限: ${permissions.join(', ')}`;
  }

  // 3. 检查任一权限
  if (anyPermission && anyPermission.length > 0) {
    hasAccess = hasAnyPermission(anyPermission);
    deniedMessage = `需要任一权限: ${anyPermission.join(', ')}`;
  }

  // 4. 检查单一角色
  if (role) {
    hasAccess = checkRole(role);
    deniedMessage = `需要 "${role}" 角色`;
  }

  // 5. 检查所有角色
  if (roles && roles.length > 0) {
    hasAccess = hasAllRoles(roles);
    deniedMessage = `需要所有这些角色: ${roles.join(', ')}`;
  }

  // 6. 检查任一角色
  if (anyRole && anyRole.length > 0) {
    hasAccess = hasAnyRole(anyRole);
    deniedMessage = `需要任一角色: ${anyRole.join(', ')}`;
  }

  // 权限不足时的处理
  if (!hasAccess) {
    // 如果有权限不足回调，调用它
    if (onPermissionDenied) {
      onPermissionDenied();
      return null;
    }
    // 否则隐藏按钮
    return null;
  }

  // 按钮样式变体映射
  const variantClasses: Record<string, string> = {
    primary: 'bg-blue-600 hover:bg-blue-700 text-white',
    secondary: 'bg-gray-200 hover:bg-gray-300 text-gray-800',
    danger: 'bg-red-600 hover:bg-red-700 text-white',
  };

  // 按钮大小映射
  const sizeClasses: Record<string, string> = {
    small: 'px-3 py-1 text-sm',
    medium: 'px-4 py-2 text-base',
    large: 'px-6 py-3 text-lg',
  };

  return (
    <button
      onClick={onClick}
      disabled={disabled}
      className={`
        ${variantClasses[variant]}
        ${sizeClasses[size]}
        ${disabled ? 'opacity-50 cursor-not-allowed' : 'cursor-pointer'}
        rounded-lg font-medium transition-colors duration-200
        ${className}
      `}
    >
      {children}
    </button>
  );
};

```typescript

import React, { createContext, useContext, useEffect, useState } from 'react';

/**
 * AuthProvider组件
 * 包裹应用根组件，提供认证和权限上下文
 */
export const AuthProvider: React.FC<AuthProviderProps> = ({ children }) => {
  const router = useRouter();

  // 使用Zustand Store
  const {
    user,
    isAuthenticated,
    isLoading: storeLoading,
    loginSuccess,
    logout: storeLogout,
    setLoading,
  } = useUserStore();

  // 本地加载状态（用于初始加载检查）
  const [initialLoading, setInitialLoading] = useState(true);

  // 初始化检查
  // 页面加载时检查本地存储的Token是否有效
  useEffect(() => {
    const checkAuth = async () => {
      try {
        // 检查Token是否有效
        const token = tokenStorage.getToken();
        if (token) {
          // Token有效，尝试获取用户信息
          // 实际项目中调用 /api/auth/me 接口获取用户信息
          const storedUser = localStorage.getItem('auth_user');
          if (storedUser) {
            const userData = JSON.parse(storedUser);
            // 根据用户角色设置权限
            const permissionConfig = ROLE_PERMISSION_MAP[userData.role];
            loginSuccess(userData, permissionConfig);
          }
        }
      } catch (error) {
        console.error('Auth check failed:', error);
        // Token无效，清除存储
        tokenStorage.clearToken();
      } finally {
        setInitialLoading(false);
      }
    };

    checkAuth();
  }, []);

  /**
   * 登录方法
   * 调用登录API，成功后存储Token和用户信息
   */
  const login = async (params: LoginParams): Promise<void> => {
    setLoading(true);

    try {
      // 调用登录API
      const result = await mockLoginApi(params);

      // 存储Token
      tokenStorage.setToken(result, params.rememberMe);

      // 存储用户信息
      localStorage.setItem('auth_user', JSON.stringify(result.user));

      // 根据用户角色获取权限配置
      const permissionConfig = ROLE_PERMISSION_MAP[result.user.role];

      // 更新Store
      loginSuccess(result.user, permissionConfig);

      console.log('[AuthProvider] 登录成功:', result.user.username);
    } catch (error) {
      console.error('[AuthProvider] 登录失败:', error);
      throw error;
    } finally {
      setLoading(false);
    }
  };

  /**
   * 退出登录方法
   * 清除所有存储的认证信息
   */
  const logout = (): void => {
    // 清除Token
    tokenStorage.clearToken();

    // 清除用户信息
    localStorage.removeItem('auth_user');

    // 更新Store
    storeLogout();

    console.log('[AuthProvider] 退出登录');

    // 重定向到登录页
    router.push('/login');
  };

  /**
   * 检查是否有指定权限
   */
  const hasPermission = (permission: string): boolean => {
    if (!isAuthenticated || !user) {
      return false;
    }
    // 如果权限列表包含'*'，表示拥有所有权限
    const { permissions } = useUserStore.getState();
    if (permissions.includes('*')) {
      return true;
    }
    return permissions.includes(permission);
  };

  /**
   * 检查是否有指定角色
   */
  const hasRole = (role: UserRole): boolean => {
    return user?.role === role;
  };

  /**
   * 刷新权限
   * 用于权限变更后重新获取权限列表
   */
  const refreshPermissions = async (): Promise<void> => {
    if (!isAuthenticated || !user) {
      return;
    }

    try {
      // 实际项目中调用 /api/auth/permissions 接口刷新权限
      // const newPermissions = await fetchPermissions();
      // setPermissions(newPermissions);

      // 模拟刷新
      const permissionConfig = ROLE_PERMISSION_MAP[user.role];
      useUserStore.getState().setPermissions(
        permissionConfig.permissions,
        permissionConfig.routePermissions,
        permissionConfig.buttonPermissions
      );

      console.log('[AuthProvider] 权限已刷新');
    } catch (error) {
      console.error('[AuthProvider] 刷新权限失败:', error);
    }
  };

  // 计算最终加载状态
  const isLoading = storeLoading || initialLoading;

  // 构建Context值
  const contextValue: AuthContextType = {
    user,
    isAuthenticated,
    isLoading,
    login,
    logout,
    hasPermission,
    hasRole,
    refreshPermissions,
  };

  return (
    <AuthContext.Provider value={contextValue}>
      {children}
    </AuthContext.Provider>
  );
};

  const [themeMode, setThemeMode] = useState<ThemeMode>('light');

  // 组件挂载时从localStorage读取主题偏好
  useEffect(() => {
    const storedMode = localStorage.getItem('theme-mode') as ThemeMode;
    if (storedMode) {
      setThemeMode(storedMode);
    }

  // 根据主题模式计算是否使用暗黑主题

    if (mode === 'dark') {
      return true;
    }
    if (mode === 'light') {
      return false;

    // 监听系统主题变化
    if (typeof window !== 'undefined') {
      return window.matchMedia('(prefers-color-scheme: dark)').matches;

    return false;

    if (themeMode !== 'system') {
      return;

    const handleChange = (e: MediaQueryListEvent) => {
      setIsDark(e.matches);
    };

    mediaQuery.addEventListener('change', handleChange);

    return () => {
      mediaQuery.removeEventListener('change', handleChange);
    };

  const handleSetThemeMode = (mode: ThemeMode) => {
    setThemeMode(mode);
    localStorage.setItem('theme-mode', mode);

    // 更新isDark状态
    setIsDark(getIsDark(mode));

    // 更新DOM
    if (mode === 'dark') {
      document.documentElement.setAttribute('data-theme', 'dark');
      document.documentElement.classList.add('dark');
    } else if (mode === 'light') {
      document.documentElement.setAttribute('data-theme', 'light');
      document.documentElement.classList.remove('dark');
    } else {
      // system模式
      document.documentElement.removeAttribute('data-theme');
      const systemIsDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
      if (systemIsDark) {
        document.documentElement.classList.add('dark');
      } else {
        document.documentElement.classList.remove('dark');
      }
    }
  };

  // 切换主题
  const toggleTheme = () => {
    const newMode = isDark ? 'light' : 'dark';
    handleSetThemeMode(newMode);
  };

  // 初始加载时应用主题
  useEffect(() => {
    handleSetThemeMode(themeMode);
  }, []);

  // 构建主题配置
  const themeConfig = isDark ? darkThemeConfig : lightThemeConfig;

  // 主题配置数据
  const theme: ThemeConfigData = {
    mode: themeMode,
    primaryColor: themeConfig.token?.colorPrimary || '#3b82f6',
    isDark,
  };

  return (

      value={{
        theme,
        setThemeMode: handleSetThemeMode,
        toggleTheme,
      }}
    >
      <ConfigProvider theme={themeConfig}>
        <App>{children}</App>
      </ConfigProvider>
    </ThemeContext.Provider>
  );
};

### 2.4 主题持久化与自动切换

主题持久化使用localStorage，确保用户偏好设置在页面刷新后保持一致。自动切换基于媒体查询`prefers-color-scheme`。

```typescript
// =============================================
// 主题状态管理
// 使用Zustand进行主题状态管理，支持持久化
// =============================================

import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';

/**
 * 主题模式类型
 */
export type ThemeMode = 'light' | 'dark' | 'system';

/**
 * 主题状态接口
 */
interface ThemeState {
  // 当前主题模式
  mode: ThemeMode;

  // 是否是暗黑模式（计算属性）
  isDark: boolean;

  // 系统主题是否支持暗黑模式
  systemPrefersDark: boolean;

  // 手动设置主题
  setMode: (mode: ThemeMode) => void;

  // 切换主题
  toggleTheme: () => void;

  // 设置暗黑模式
  setDark: (isDark: boolean) => void;
}

/**
 * 监听系统主题变化
 * 返回当前系统是否偏好暗黑模式
 */
const getSystemPrefersDark = (): boolean => {
  if (typeof window === 'undefined') {
    return false;
  }
  return window.matchMedia('(prefers-color-scheme: dark)').matches;
};

/**
 * 主题状态管理Store
 * 使用Zustand进行状态管理，支持localStorage持久化
 */
export const useThemeStore = create<ThemeState>()(
  persist(
    (set, get) => ({
      // 初始状态
      mode: 'system',  // 默认跟随系统
      isDark: getSystemPrefersDark(),  // 根据系统偏好初始化
      systemPrefersDark: getSystemPrefersDark(),

      /**
       * 设置主题模式
       * @param mode 主题模式：light/dark/system
       */
      setMode: (mode) => {
        const isDark = mode === 'dark' ||
          (mode === 'system' && getSystemPrefersDark());

        // 更新DOM
        updateDOM(mode, isDark);

        set({
          mode,
          isDark,
        });

        console.log(`[ThemeStore] 主题模式已设置为: ${mode}, isDark: ${isDark}`);
      },

      /**
       * 切换主题
       * 在亮色和暗黑模式之间切换
       */
      toggleTheme: () => {
        const { mode, isDark } = get();
        const newMode = isDark ? 'light' : 'dark';
        get().setMode(newMode);
      },

      /**
       * 设置暗黑模式
       * @param isDark 是否使用暗黑模式
       */
      setDark: (isDark) => {
        get().setMode(isDark ? 'dark' : 'light');
      },
    }),
    {
      // 持久化配置
      name: 'theme-storage',              // 存储键名
      storage: createJSONStorage(() => localStorage),  // 使用localStorage
      // 只持久化mode字段，isDark是计算属性
      partialize: (state) => ({
        mode: state.mode,
      }),
    }
  )
);

/**
 * 更新DOM以应用主题
 * @param mode 主题模式
 * @param isDark 是否是暗黑模式
 */
function updateDOM(mode: ThemeMode, isDark: boolean): void {
  if (typeof document === 'undefined') {
    return;
  }

  const html = document.documentElement;

  // 设置data-theme属性
  if (mode === 'system') {
    // system模式：根据系统偏好设置
    html.removeAttribute('data-theme');
  } else {
    // light或dark模式：显式设置
    html.setAttribute('data-theme', mode);
  }

  // 设置dark类名
  if (isDark) {
    html.classList.add('dark');
  } else {
    html.classList.remove('dark');
  }
}

/**
 * 初始化主题监听
 * 在应用启动时调用，监听系统主题变化
 */
export function initThemeListener(): () => void {
  if (typeof window === 'undefined') {
    return () => {};
  }

  const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)');

  const handleChange = (e: MediaQueryListEvent) => {
    const { mode } = useThemeStore.getState();

    // 只有在system模式下才响应系统主题变化
    if (mode === 'system') {
      const isDark = e.matches;

      // 更新DOM
      if (isDark) {
        document.documentElement.classList.add('dark');
      } else {
        document.documentElement.classList.remove('dark');
      }

      // 更新状态
      useThemeStore.setState({
        isDark,
        systemPrefersDark: isDark,
      });

      console.log(`[ThemeStore] 系统主题变化: ${isDark ? 'dark' : 'light'}`);
    }
  };

  // 添加监听
  mediaQuery.addEventListener('change', handleChange);

  // 初始设置
  useThemeStore.setState({
    systemPrefersDark: mediaQuery.matches,
  });

  // 返回清理函数
  return () => {
    mediaQuery.removeEventListener('change', handleChange);
  };
}
```

#### 3.1.1 i18next核心模块

| react-i18next | React绑定，提供Hooks和组件 |

| i18next-resources-to-backend | 动态加载翻译资源 |

#### 3.1.2 核心概念

 * 翻译函数类型

 */
export type TranslateFunction = (
  key: string,
  options?: Record<string, unknown>
) => string;

/**
 * 翻译资源结构
 * 第一层key是语言，第二层key是命名空间
 */

  [language: string]: {
    [namespace: string]: {
      [key: string]: string | object;
    };
  };
}

 * i18next配置选项
 */

  /** 默认语言 */

  /** 语言检测配置 */

    /** 语言存储的键名 */

    /** 查询参数的键名 */

    /** localStorage的键名 */
    lookupLocalStorage: string;
    /** Cookie的键名 */
    lookupCookie: string;
  };
}

/**
 * 语言信息类型
 * 包含语言代码、名称、本地化名称

### 3.3 动态语言切换与格式化

```typescript
// =============================================
// 国际化Hook与格式化工具
// 提供类型安全的翻译调用和格式化功能
// =============================================

import { useTranslation as useI18nTranslation } from 'react-i18next';
import { useCallback } from 'react';

/**
 * 翻译Hook返回值类型
 */
interface UseTranslationReturn {
  /** 翻译函数 */
  t: (key: string, options?: Record<string, unknown>) => string;
  /** 当前语言 */
  language: string;
  /** 切换语言 */
  changeLanguage: (lang: string) => Promise<void>;
  /** 格式化数字 */
  formatNumber: (num: number, options?: Intl.NumberFormatOptions) => string;
  /** 格式化日期 */
  formatDate: (date: Date | string | number, options?: Intl.DateTimeFormatOptions) => string;
  /** 格式化时间 */
  formatTime: (date: Date | string | number, options?: Intl.DateTimeFormatOptions) => string;
  /** 格式化日期时间 */
  formatDateTime: (date: Date | string | number, options?: Intl.DateTimeFormatOptions) => string;
  /** 格式化货币 */
  formatCurrency: (amount: number, currency?: string) => string;
  /** 格式化相对时间 */
  formatRelativeTime: (date: Date | string | number) => string;
}

/**
 * 命名空间列表
 */
const NAMESPACES = ['common', 'auth', 'dashboard', 'editor', 'settings'] as const;

/**
 * 支持的语言列表
 */
export const SUPPORTED_LANGUAGES = [
  { code: 'zh-CN', name: '简体中文', nativeName: '简体中文', flag: '🇨🇳', direction: 'ltr' as const },
  { code: 'en-US', name: 'English', nativeName: 'English', flag: '🇺🇸', direction: 'ltr' as const },
];

/**
 * 获取当前语言信息
 */
export function getCurrentLanguageInfo() {
  const currentCode = typeof window !== 'undefined'
    ? localStorage.getItem('i18nextLng') || 'zh-CN'
    : 'zh-CN';

  return SUPPORTED_LANGUAGES.find(lang => lang.code === currentCode) || SUPPORTED_LANGUAGES[0];
}

/**
 * 使用翻译的Hook
 * 提供翻译功能和格式化工具
 *
 * @param namespace 命名空间，默认为'common'
 * @returns 翻译相关功能
 *
 * @example
 * ```tsx
 * function MyComponent() {
 *   const { t, language, changeLanguage, formatNumber } = useTranslation();
 *
 *   return (
 *     <div>
 *       <h1>{t('common.menu.home')}</h1>
 *       <p>{formatNumber(1234567)}</p>
 *       <button onClick={() => changeLanguage('en-US')}>English</button>
 *     </div>
 *   );
 * }
 * ```
 */
export function useTranslation(namespace: string = 'common') {
  // 使用react-i18next的useTranslation
  const { t, i18n } = useI18nTranslation(NAMESPACES as unknown as string[]);

  // 获取当前语言
  const language = i18n.language || 'zh-CN';

  /**
   * 切换语言
   * @param lang 语言代码
   */
  const changeLanguage = useCallback(async (lang: string) => {
    await i18n.changeLanguage(lang);
    // 保存到localStorage
    localStorage.setItem('i18nextLng', lang);
    // 更新文档语言属性
    document.documentElement.lang = lang;
    console.log(`[i18n] 语言已切换至: ${lang}`);
  }, [i18n]);

  /**
   * 格式化数字
   * @param num 数字
   * @param options 格式化选项
   */
  const formatNumber = useCallback((num: number, options?: Intl.NumberFormatOptions): string => {
    return new Intl.NumberFormat(language, options).format(num);
  }, [language]);

  /**
   * 格式化日期
   * @param date 日期
   * @param options 格式化选项
   */
  const formatDate = useCallback((
    date: Date | string | number,
    options?: Intl.DateTimeFormatOptions
  ): string => {
    const dateObj = date instanceof Date ? date : new Date(date);
    return new Intl.DateTimeFormat(language, {
      year: 'numeric',
      month: 'long',
      day: 'numeric',
      ...options,
    }).format(dateObj);
  }, [language]);

  /**
   * 格式化时间
   * @param date 日期
   * @param options 格式化选项
   */
  const formatTime = useCallback((
    date: Date | string | number,
    options?: Intl.DateTimeFormatOptions
  ): string => {
    const dateObj = date instanceof Date ? date : new Date(date);
    return new Intl.DateTimeFormat(language, {
      hour: '2-digit',
      minute: '2-digit',
      second: '2-digit',
      ...options,
    }).format(dateObj);
  }, [language]);

  /**
   * 格式化日期时间
   * @param date 日期
   * @param options 格式化选项
   */
  const formatDateTime = useCallback((
    date: Date | string | number,
    options?: Intl.DateTimeFormatOptions
  ): string => {
    const dateObj = date instanceof Date ? date : new Date(date);
    return new Intl.DateTimeFormat(language, {
      year: 'numeric',
      month: 'long',
      day: 'numeric',
      hour: '2-digit',
      minute: '2-digit',
      ...options,
    }).format(dateObj);
  }, [language]);

  /**
   * 格式化货币
   * @param amount 金额
   * @param currency 货币代码，默认为CNY
   */
  const formatCurrency = useCallback((amount: number, currency: string = 'CNY'): string => {
    return new Intl.NumberFormat(language, {
      style: 'currency',
      currency,
    }).format(amount);
  }, [language]);

  /**
   * 格式化相对时间
   * @param date 日期
   * @returns 相对时间字符串
   */
  const formatRelativeTime = useCallback((date: Date | string | number): string => {
    const dateObj = date instanceof Date ? date : new Date(date);
    const now = new Date();
    const diff = now.getTime() - dateObj.getTime();
    const diffSeconds = Math.floor(diff / 1000);
    const diffMinutes = Math.floor(diffSeconds / 60);
    const diffHours = Math.floor(diffMinutes / 60);
    const diffDays = Math.floor(diffHours / 24);

    // 使用翻译键
    if (diffSeconds < 60) {
      return t('common.datetime.now');
    } else if (diffMinutes < 60) {
      return t('common.datetime.minutesAgo', { count: diffMinutes });
    } else if (diffHours < 24) {
      return t('common.datetime.hoursAgo', { count: diffHours });
    } else if (diffDays < 2) {
      return t('common.datetime.yesterday');
    } else if (diffDays < 7) {
      return t('common.datetime.daysAgo', { count: diffDays });
    } else {
      return formatDate(dateObj);
    }
  }, [language, t]);

  return {
    // 翻译函数
    t: (key: string, options?: Record<string, unknown>) => {
      // 如果没有指定命名空间，使用默认命名空间
      if (!key.includes(':')) {
        key = `${namespace}:${key}`;
      }
      return t(key, options);
    },

    // 当前语言
    language,

    // 切换语言
    changeLanguage,

    // 格式化工具
    formatNumber,
    formatDate,
    formatTime,
    formatDateTime,
    formatCurrency,
    formatRelativeTime,
  };
}
```

### 3.5 i18n高级用法：命名空间动态加载

```typescript
// =============================================
// i18n命名空间动态加载
// 实现按需加载翻译资源，优化应用性能
// =============================================

import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';

/**
 * 命名空间管理器
 * 提供命名空间的动态加载和缓存管理
 */
class NamespaceManager {
  // 已加载的命名空间缓存
  private loadedNamespaces: Set<string> = new Set();

  // 正在加载的命名空间
  private loadingNamespaces: Map<string, Promise<void>> = new Map();

  /**
   * 加载指定命名空间的翻译资源
   * @param namespace 命名空间名称
   * @param language 语言代码
   */
  async loadNamespace(namespace: string, language: string): Promise<void> {
    // 如果已经加载，直接返回
    if (this.loadedNamespaces.has(`${language}/${namespace}`)) {
      return;
    }

    // 如果正在加载，等待加载完成
    const existingLoad = this.loadingNamespaces.get(`${language}/${namespace}`);
    if (existingLoad) {
      return existingLoad;
    }

    // 开始加载
    const loadPromise = this.doLoadNamespace(namespace, language);
    this.loadingNamespaces.set(`${language}/${namespace}`, loadPromise);

    try {
      await loadPromise;
    } finally {
      this.loadingNamespaces.delete(`${language}/${namespace}`);
    }
  }

  /**
   * 执行命名空间加载
   * @param namespace 命名空间名称
   * @param language 语言代码
   */
  private async doLoadNamespace(namespace: string, language: string): Promise<void> {
    try {
      // 动态导入翻译资源
      // 实际项目中从API或CDN加载
      const resources = await import(`./locales/${language}/${namespace}.json`);

      // 添加到i18next
      i18n.addResourceBundle(
        language,
        namespace,
        resources.default || resources,
        true,  // 深合并
        true   // 序列化
      );

      // 标记为已加载
      this.loadedNamespaces.add(`${language}/${namespace}`);

      console.log(`[NamespaceManager] 已加载命名空间: ${namespace} (${language})`);
    } catch (error) {
      console.error(`[NamespaceManager] 加载命名空间失败: ${namespace} (${language})`, error);
      throw error;
    }
  }

  /**
   * 预加载多个命名空间
   * @param namespaces 命名空间列表
   * @param language 语言代码
   */
  async preloadNamespaces(namespaces: string[], language: string): Promise<void> {
    await Promise.all(
      namespaces.map(ns => this.loadNamespace(ns, language))
    );
  }

  /**
   * 检查命名空间是否已加载
   * @param namespace 命名空间名称
   * @param language 语言代码
   */
  isLoaded(namespace: string, language: string): boolean {
    return this.loadedNamespaces.has(`${language}/${namespace}`);
  }

  /**
   * 获取已加载的所有命名空间
   */
  getLoadedNamespaces(): string[] {
    return Array.from(this.loadedNamespaces);
  }
}

// 导出单例实例
export const namespaceManager = new NamespaceManager();

/**
 * 懒加载翻译资源的HOC
 * @param WrappedComponent 被包装的组件
 * @param namespace 命名空间
 */
export function withTranslation(
  WrappedComponent: React.ComponentType,
  namespace: string
): React.FC {
  return function WithTranslationComponent(props: any) {
    const [, setReady] = useState(false);

    useEffect(() => {
      // 懒加载命名空间
      namespaceManager
        .loadNamespace(namespace, i18n.language)
        .then(() => setReady(true));
    }, [namespace]);

    return <WrappedComponent {...props} />;
  };
}

/**
 * // 1. 初始化i18next
 * i18n.use(initReactI18next).init({
 *   lng: 'zh-CN',
 *   fallbackLng: 'zh-CN',
 *   ns: ['common'],
 *   defaultNS: 'common',
 *   interpolation: { escapeValue: false },
 * });
 *
 * // 2. 预加载常用命名空间
 * namespaceManager.preloadNamespaces(['common', 'auth', 'dashboard'], 'zh-CN');
 *
 * // 3. 在组件中使用
 * function MyComponent() {
 *   const { t } = useTranslation('common');
 *   return <h1>{t('common.menu.home')}</h1>;
 * }
 *
 * // 4. 懒加载特定命名空间
