## 一、JavaScript核心基础

### 1.1 闭包（Closure）

**问题：什么是闭包？闭包有什么作用？请举例说明。**

**详细解答：**

闭包是JavaScript中最重要也最容易被误解的概念之一。要理解闭包，我们先从一个简单的例子开始。

```javascript
function outer() {
  // 外部函数的局部变量
  let count = 0;

  // 内部函数，访问了外部函数的变量
  function inner() {
    count++;
    console.log(count);
  }

  // 返回内部函数
  return inner;
}

// 创建闭包实例
const counter = outer();

counter(); // 输出: 1
counter(); // 输出: 2
counter(); // 输出: 3
```

在这个例子中，`outer`函数执行完毕后，按照常理它的局部变量`count`应该被垃圾回收机制回收，内存应该被释放。但是实际上，由于`inner`函数仍然引用着`count`，所以`count`不会被回收，这就形成了一个闭包。

**闭包的定义**：闭包是指函数能够访问其词法作用域之外的变量的能力。简单来说，就是一个函数记住并可以访问它创建时所处的环境。

**闭包的形成条件**：
1. 存在内层函数
2. 内层函数引用了外层函数的变量
3. 外层函数返回内层函数

**闭包的常见应用场景**：

```javascript
// 场景1：数据私有化
function createCache() {
  const cache = {};

  return {
    get: function(key) {
      return cache[key];
    },
    set: function(key, value) {
      cache[key] = value;
    },
    clear: function() {
      cache = {};
    }
  };
}

const myCache = createCache();
myCache.set('name', '张三');
console.log(myCache.get('name')); // '张三'
// console.log(cache); // 报错，cache是私有的
```

```javascript
// 场景2：函数柯里化
function currying(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    } else {
      return function(...args2) {
        return curried.apply(this, [...args, ...args2]);
      };
    }
  };
}

// 示例：计算长方形的面积
function calculateArea(width, height) {
  return width * height;
}

const getAreaWithFixedWidth = currying(calculateArea)(10);
console.log(getAreaWithFixedWidth(5));  // 50
console.log(getAreaWithFixedWidth(10)); // 100
```

```javascript
// 场景3：防抖和节流（后面会详细讲）
function debounce(fn, delay) {
  let timer = null;

  return function(...args) {
    if (timer) clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}
```

**闭包的缺点和注意事项**：

```javascript
// 问题1：内存泄漏
// 闭包会导致引用的变量无法被回收，如果使用不当会造成内存泄漏
function problematic() {
  const largeData = new Array(1000000);

  return function() {
    console.log(largeData.length);
  };
}

// 问题2：this指向问题
const obj = {
  name: '张三',
  getName: function() {
    // 这里的this指向obj
    return function() {
      // 这里的this指向window（严格模式下是undefined）
      console.log(this.name);
    };
  }
};

obj.getName()(); // undefined（除非全局有name变量）

// 正确做法：保存this引用
const obj2 = {
  name: '李四',
  getName: function() {
    const that = this;
    return function() {
      console.log(that.name);
    };
  }
};

obj2.getName()(); // '李四'
```

### 1.2 原型与原型链

**问题：请解释JavaScript中的原型和原型链。什么是原型继承？**

**详细解答：**

JavaScript是一门基于原型的面向对象编程语言，这与Java、C++等基于类的语言有本质区别。在JavaScript中，没有类（ES6的class只是语法糖），对象是通过原型链来继承属性和方法的。

**原型（prototype）基础：**

```javascript
// 创建一个构造函数
function Person(name, age) {
  this.name = name;
  this.age = age;
}

// 在构造函数的原型上添加方法
Person.prototype.sayHello = function() {
  console.log(`你好，我是${this.name}，今年${this.age}岁`);
};

Person.prototype.greet = function() {
  console.log(`Hello, I'm ${this.name}`);
};

// 创建实例
const zhangsan = new Person('张三', 25);
const lisi = new Person('李四', 30);

// 实例可以调用原型上的方法
zhangsan.sayHello(); // 输出: 你好，我是张三，今年25岁
lisi.sayHello();    // 输出: 你好，我是李四，今年30岁

// 实例和构造函数的关系
console.log(zhangsan.__proto__ === Person.prototype); // true
console.log(Person.prototype.constructor === Person); // true
```

**原型链：**

```javascript
// 原型链的查找机制
console.log(zhangsan.__proto__ === Person.prototype);                    // true
console.log(Person.prototype.__proto__ === Object.prototype);             // true
console.log(Object.prototype.__proto__ === null);                         // true

// 当访问 zhangsan.toString() 时：
// 1. 先在 zhangsan 自身查找，没找到
// 2. 在 zhangsan.__proto__ (即 Person.prototype) 查找，没找到
// 3. 在 Person.prototype.__proto__ (即 Object.prototype) 查找，找到了
console.log(zhangsan.toString()); // [object Object]
```

**原型链图示：**

```
zhangsan 实例
    │
    │ __proto__
    ▼
Person.prototype ───────► constructor: Person
    │                          │
    │ __proto__                │ prototype
    ▼                          │
Object.prototype ◄─────────────┘
    │
    │ __proto__
    ▼
   null
```

**原型继承的实现方式：**

```javascript
// 方式1：原型链继承
function Animal(name) {
  this.name = name;
}

Animal.prototype.eat = function() {
  console.log(`${this.name} 正在吃东西`);
};

function Dog(name, color) {
  // 借用构造函数继承属性
  Animal.call(this, name);
  this.color = color;
}

// 原型链继承：让Dog的原型指向Animal的实例
Dog.prototype = new Animal();
Dog.prototype.constructor = Dog; // 修正构造函数指向

Dog.prototype.bark = function() {
  console.log(`${this.name} 在汪汪叫`);
};

const dog = new Dog('旺财', '黄色');
dog.eat();  // 继承自Animal: 旺财 正在吃东西
dog.bark(); // 自己定义: 旺财 在汪汪叫
```

```javascript
// 方式2：组合继承（最常用）
function Student(name, score) {
  Animal.call(this, name); // 继承属性
  this.score = score;
}

Student.prototype = new Animal(); // 继承方法
Student.prototype.constructor = Student;

Student.prototype.study = function() {
  console.log(`${this.name} 正在学习，成绩是${this.score}`);
};
```

```javascript
// 方式3：ES6 Class继承（推荐）
class Animal {
  constructor(name) {
    this.name = name;
  }

  eat() {
    console.log(`${this.name} 正在吃东西`);
  }
}

class Dog extends Animal {
  constructor(name, color) {
    super(name); // 调用父类构造函数
    this.color = color;
  }

  bark() {
    console.log(`${this.name} 在汪汪叫`);
  }
}

const dog = new Dog('旺财', '黄色');
dog.eat();  // 继承自Animal
dog.bark(); // 自己定义
```

**原型链面试要点：**

```javascript
// 面试常问：原型链的尽头是什么？

// 面试常问：Function和Object的关系

console.log(Function instanceof Object); // true

// 面试常问：如何判断属性是自身还是原型上的
function hasOwnPropertyDemo() {
  function A() {}
  A.prototype.x = 10;

  const a = new A();
  a.y = 20;

  console.log(a.hasOwnProperty('x')); // false，原型上的
  console.log(a.hasOwnProperty('y')); // true，自身的
  console.log('x' in a); // true，in操作符会查找整个原型链
}
```

### 1.3 this指向问题

**问题：请详细解释JavaScript中this的指向问题。this在什么情况下指向什么？**

**详细解答：**

`this`是JavaScript中最令人困惑的关键字之一。它的指向取决于函数的调用方式，而不是函数的定义位置。理解`this`的关键是要搞清楚函数的调用上下文。

**this的五大绑定规则：**

```javascript
// 规则1：默认绑定（独立函数调用）
function showThis() {
  console.log(this);
}

showThis(); // 浏览器中输出: window (严格模式下是undefined)
```

```javascript
// 规则2：隐式绑定（对象方法调用）
const person = {
  name: '张三',
  showThis: function() {
    console.log(this);
  }
};

person.showThis(); // 输出: person对象
```

```javascript
// 规则3：显式绑定（call、apply、bind）
function showThis() {
  console.log(this);
}

const obj = { name: '李四' };

showThis.call(obj);  // 输出: obj对象
showThis.apply(obj); // 输出: obj对象

const boundShowThis = showThis.bind(obj);
boundShowThis(); // 输出: obj对象
```

```javascript
// 规则4：new绑定（构造函数）
function Person(name) {
  this.name = name;
  console.log(this);
}

const p = new Person('王五'); // 输出: Person实例对象
```

```javascript
// 规则5：箭头函数（词法作用域）
const obj = {
  name: '赵六',
  showThis: () => {
    console.log(this);
  }
};

obj.showThis(); // 输出: window（箭头函数没有自己的this）

// 箭头函数在普通函数内部
function outer() {
  const that = this;
  return function inner() {
    console.log(that);
  };
}
```

**优先级对比：**

```javascript
// new绑定 > 显式绑定 > 隐式绑定 > 默认绑定

// 例1：new和隐式冲突
function Foo() {}
const obj = { Foo: Foo };
const instance = new Foo();

console.log(instance instanceof Foo); // true
// new绑定的this指向实例，忽略隐式绑定

// 例2：显式和隐式冲突
function show() {
  console.log(this.name);
}

const obj1 = { name: 'obj1', show: show };
const obj2 = { name: 'obj2', show: show };

obj1.show();      // obj1
obj2.show();      // obj2
obj1.show.call(obj2); // obj2（显式绑定优先级更高）
```

**常见面试场景分析：**

```javascript
// 场景1：setTimeout中的this
const obj = {
  name: '测试',
  delayShow: function() {
    setTimeout(function() {
      console.log(this.name); // undefined（严格模式报错）
      // 这里的this指向window，因为setTimeout内部调用函数时是默认绑定
    }, 1000);
  }
};

// 解决方案1：保存this引用
const obj1 = {
  name: '测试',
  delayShow: function() {
    const that = this;
    setTimeout(function() {
      console.log(that.name);
    }, 1000);
  }
};

// 解决方案2：使用箭头函数
const obj2 = {
  name: '测试',
  delayShow: function() {
    setTimeout(() => {
      console.log(this.name); // '测试'
    }, 1000);
  }
};

// 解决方案3：使用bind
const obj3 = {
  name: '测试',
  delayShow: function() {
    setTimeout(function() {
      console.log(this.name);
    }.bind(this), 1000);
  }
};
```

```javascript
// 场景2：DOM事件处理函数中的this
const button = document.querySelector('button');

button.addEventListener('click', function() {
  console.log(this); // button元素
  // 事件监听函数中的this指向当前元素
});
```

```javascript
// 场景3：类方法中的this
class Timer {
  constructor() {
    this.seconds = 0;
  }

  start() {
    setInterval(function() {
      this.seconds++; // 这里的this指向window
      console.log(this.seconds);
    }, 1000);
  }

  // 正确的箭头函数写法
  startFixed() {
    setInterval(() => {
      this.seconds++;
      console.log(this.seconds);
    }, 1000);
  }
}

const timer = new Timer();
// timer.start(); // 错误：this.seconds是NaN
timer.startFixed(); // 正确
```

**面试官想听到什么**：
- 清晰地说出五种this绑定规则及其优先级
- 能够分析具体代码中this的指向
- 知道箭头函数没有自己的this
- 能够针对不同场景提出解决this问题的方案

### 1.4 事件循环（Event Loop）

**问题：请解释JavaScript的事件循环机制。同步任务和异步任务是如何执行的？宏任务和微任务有什么区别？**

**详细解答：**

JavaScript是一门单线程语言，为了处理耗时的异步操作，引入了事件循环机制。理解事件循环对于编写高效的异步代码至关重要。

**为什么JavaScript是单线程？**

JavaScript最初设计是用于浏览器交互的脚本语言，单线程可以避免复杂的同步问题。比如一个线程在某个DOM节点上添加内容，另一个线程删除了这个节点，浏览器就不知道该听谁的。所以JavaScript从设计之初就是单线程的。

**事件循环的基本概念：**

```javascript
// 事件循环的核心机制
console.log('1'); // 同步任务

setTimeout(function() {
  console.log('2'); // 宏任务
}, 0);

Promise.resolve().then(function() {
  console.log('3'); // 微任务
});

console.log('4'); // 同步任务

// 输出顺序: 1, 4, 3, 2
// 原因：同步任务先执行，遇到异步任务会注册
// 同步任务执行完后，先清空微任务队列，再执行宏任务
```

**JavaScript执行上下文：**

```javascript
// 代码执行过程
/*
1. 首先执行全局同步代码
2. 创建执行上下文栈（调用栈）
3. 遇到异步任务，交给Web APIs处理
4. 同步任务执行完毕后
5. 检查微任务队列，执行所有微任务
6. 检查宏任务队列，执行一个宏任务
7. 重复步骤5-6（这就是事件循环）
*/

console.log('script start');

setTimeout(function() {
  console.log('setTimeout');
}, 0);

Promise.resolve().then(function() {
  console.log('promise1');
}).then(function() {
  console.log('promise2');
});

console.log('script end');

// 输出: script start, script end, promise1, promise2, setTimeout
```

**宏任务与微任务的区别：**

```javascript
// 宏任务（Macrotask）- 每次执行一个
/*
包含：setTimeout, setInterval, I/O操作, UI渲染, requestAnimationFrame
*/
// 微任务（Microtask）- 清空队列
/*
包含：Promise.then, MutationObserver, process.nextTick (Node)
*/

// 实战例题
console.log('1');

setTimeout(() => console.log('2'), 0);

new Promise((resolve) => {
  console.log('3');
  resolve();
}).then(() => console.log('4'));

setTimeout(() => console.log('5'), 0);

console.log('6');

// 详细分析：
// 同步任务: 1, 3, 6
// 微任务: 4 (在第一个Promise的then之后)
// 宏任务: 2, 5
// 输出顺序: 1, 3, 6, 4, 2, 5
```

**async/await与事件循环：**

```javascript
// async/await实际上是Promise的语法糖
// await会阻塞后面的代码，但await本身是微任务

async function asyncAwait() {
  console.log('1');

  await Promise.resolve();

  console.log('2');
}

console.log('3');

asyncAwait();

console.log('4');

// 分析：
// 1. console.log('3') 是同步任务，先执行
// 2. asyncAwait() 是同步调用，但async函数内部从await开始是异步
// 3. 打印 '1'（async函数内部同步部分）
// 4. await Promise.resolve() 会暂停async函数执行，让出线程
// 5. console.log('4') 是同步任务，继续执行
// 6. 微任务队列清空，执行await之后的代码，打印 '2'
// 输出: 3, 1, 4, 2
```

**复杂面试题分析：**

```javascript
// 高级例题
function test() {
  console.log('1');

  setTimeout(() => console.log('2'), 0);

  new Promise((resolve) => {
    console.log('3');
    resolve();
  }).then(() => {
    console.log('4');

    new Promise((resolve) => {
      console.log('5');
      resolve();
    }).then(() => console.log('6'));
  });

  setTimeout(() => console.log('7'), 0);

  console.log('8');
}

test();

// 详细分析：
// 同步: 1, 3, 8
// 微任务: 4 (then触发)
// 执行4时，又注册了新的微任务: 6
// 清空微任务: 6
// 宏任务: 2, 7
// 输出: 1, 3, 8, 4, 5, 6, 2, 7
// 注意：5在4的then里面，是同步执行的
```

**浏览器与Node.js事件循环的区别：**

```javascript
// Node.js事件循环阶段（按顺序）：
/*
1. timers - setTimeout, setInterval的回调
2. pending callbacks - 延迟到下一个循环迭代的I/O回调
3. idle, prepare - 仅内部使用
4. poll - 检索新的I/O事件，必要时阻塞
5. check - setImmediate的回调
6. close callbacks - 关闭的回调，如socket.on('close')
*/

// setTimeout vs setImmediate
setTimeout(() => console.log('timeout'), 0);
setImmediate(() => console.log('immediate'));

// 在Node.js中，输出顺序不确定，取决于性能
// 但在I/O回调中，setImmediate总是先执行
```

- 知道宏任务和微任务的区别，以及它们的执行时机
- 能够分析复杂嵌套Promise和setTimeout的输出顺序
- 理解async/await与Promise的关系

### 1.5 深拷贝与浅拷贝

**问题：什么是深拷贝和浅拷贝？如何实现深拷贝？有什么需要注意的地方？**

**详细解答：**

在JavaScript中，基本类型赋值是值拷贝，而引用类型赋值是引用拷贝。这种区别导致了浅拷贝和深拷贝的概念。

**基本概念：**

```javascript
// 值类型（基本类型）：number, string, boolean, null, undefined, symbol, bigint
// 引用类型（对象类型）：object, array, function, Date, RegExp等

// 基本类型赋值 - 值拷贝
let num1 = 10;
let num2 = num1; // 拷贝值
num2 = 20;
console.log(num1); // 10，不受影响

// 引用类型赋值 - 引用拷贝（共享内存）
let obj1 = { name: '张三' };
let obj2 = obj1; // 拷贝引用，指向同一内存
obj2.name = '李四';
console.log(obj1.name); // '李四'，被改变了
```

**浅拷贝的实现：**

```javascript
// 方法1：Object.assign()
```

```javascript
// 方法2：递归实现深拷贝（基础版本）
function deepClone(obj) {
  // 基本类型直接返回
  if (obj === null || typeof obj !== 'object') {
    return obj;
  }

  // 处理Date对象
  if (obj instanceof Date) {
    return new Date(obj.getTime());
  }

  // 处理RegExp对象
  if (obj instanceof RegExp) {
    return new RegExp(obj.source, obj.flags);
  }

  // 处理数组
  if (Array.isArray(obj)) {
    const clonedArr = [];
    for (let i = 0; i < obj.length; i++) {
      clonedArr[i] = deepClone(obj[i]);
    }
    return clonedArr;
  }

  // 处理普通对象
  const clonedObj = {};
  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      clonedObj[key] = deepClone(obj[key]);
    }
  }

  return clonedObj;
}
```

```javascript
// 方法3：完善的深拷贝（处理更多边界情况）
function deepCloneAdvanced(obj, hash = new WeakMap()) {
  // 处理null和基本类型
  if (obj === null || typeof obj !== 'object') {
    return obj;
  }

  // 处理循环引用
  if (hash.has(obj)) {
    return hash.get(obj);
  }

  // 处理Date对象
  if (obj instanceof Date) {
    return new Date(obj.getTime());
  }

  // 处理RegExp对象
  if (obj instanceof RegExp) {
    return new RegExp(obj.source, obj.flags);
  }

  // 处理Map对象
  if (obj instanceof Map) {
    const clonedMap = new Map();
    hash.set(obj, clonedMap);
    obj.forEach((value, key) => {
      clonedMap.set(
        deepCloneAdvanced(key, hash),
        deepCloneAdvanced(value, hash)
      );
    });
    return clonedMap;
  }

  // 处理Set对象
  if (obj instanceof Set) {
    const clonedSet = new Set();
    hash.set(obj, clonedSet);
    obj.forEach(value => {
      clonedSet.add(deepCloneAdvanced(value, hash));
    });
    return clonedSet;
  }

  // 处理函数（通常不需要深拷贝，这里返回原引用）
  if (typeof obj === 'function') {
    return obj;
  }

  // 处理数组和普通对象
  const clonedObj = Array.isArray(obj) ? [] : {};
  hash.set(obj, clonedObj);

  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      clonedObj[key] = deepCloneAdvanced(obj[key], hash);
    }
  }

  return clonedObj;
}
```

```javascript
    return this.listRef?.scrollHeight || 0;

    // 适用于：滚动位置恢复、内容高度计算等
    if (snapshot !== null && this.listRef) {
      this.listRef.scrollTop +=
        this.listRef.scrollHeight - snapshot;
    }

  componentDidMount() {
```

### 2.2 Hooks核心原理

**问题：React Hooks的实现原理是什么？为什么Hooks不能放在条件语句中？useState和useEffect的细节有哪些？**

**详细解答：**

React Hooks是React 16.8引入的新特性，它让我们可以在函数组件中使用state和其他React特性。理解Hooks的实现原理对于深入掌握React至关重要。

**Hooks的基本实现原理：**

```javascript
// React是如何实现Hooks的？
// 答案：React通过链表来保存每个Hook的状态

// React内部大致的数据结构：
let currentComponent = null;
let nextHookIndex = 0;

const hookStates = []; // 存储所有hook的状态

function useState(initialValue) {
  // 获取当前hook的索引
  const hookIndex = nextHookIndex;

  // 如果状态尚未初始化
  if (hookStates[hookIndex] === undefined) {
    hookStates[hookIndex] = initialValue;
  }

  // 获取当前状态
  const state = hookStates[hookIndex];

  // 创建更新函数
  const setState = (newValue) => {
    hookStates[hookIndex] = newValue;
    // 触发组件重新渲染
    render();
  };

  // 更新索引，为下一个hook做准备
  nextHookIndex++;

  return [state, setState];
}

// 使用示例
function Counter() {
  // 第一次渲染
  const [count, setCount] = useState(0);  // hookIndex = 0

  // 第二次渲染
  const [name, setName] = useState('张三'); // hookIndex = 1

  // 第三次渲染
  const [age, setAge] = useState(25); // hookIndex = 2

  // ...
}
```

```javascript
function useForm(initialValues) {

  const [errors, setErrors] = useState({});

    const { name, value } = e.target;
    setValues(prev => ({ ...prev, [name]: value }));
  };

  const reset = () => {
    setValues(initialValues);
    setErrors({});
  };

  return { values, errors, handleChange, reset, setValues, setErrors };
}
```

### 2.3 虚拟DOM与Diff算法

**问题：什么是虚拟DOM？React的Diff算法是如何工作的？为什么需要虚拟DOM？**

**详细解答：**

虚拟DOM是React最核心的概念之一。理解虚拟DOM和Diff算法能够帮助我们理解React的性能优化原理。

**为什么需要虚拟DOM：**

```javascript
// 直接操作真实DOM的问题：
// 1. 真实DOM节点很复杂，创建和销毁很昂贵
// 2. 直接操作DOM会导致浏览器回流和重绘
// 3. 手动操作DOM代码难以维护

// 示例：更新一个列表
const list = document.getElementById('list');

// 原始做法：先清空，再重新创建所有元素
// 浏览器需要：销毁旧DOM、创建新DOM、重新布局、重新绘制
// 涉及多次回流和重绘，性能很差

// 虚拟DOM的解决方案：
// 1. 先在JS内存中构建虚拟DOM树
// 2. 比较新旧虚拟DOM的差异
// 3. 只更新变化的部分到真实DOM
// 4. 最小化DOM操作
```

**虚拟DOM的工作原理：**

```javascript
// 虚拟DOM本质上是一个普通的JS对象
// 它描述了真实DOM的结构

// React元素的结构
const element = {
  type: 'div',
  props: {
    className: 'container',
    children: [
      {
        type: 'h1',
        props: {
          children: '标题'
        }
      },
      {
        type: 'p',
        props: {
          children: '段落内容'
        }
      }
    ]
  }
};

// JSX会被Babel编译成createElement调用
// const element = (
//   <div className="container">
//     <h1>标题</h1>
//     <p>段落内容</p>
//   </div>
// );

// Babel转换后：
const element = React.createElement(
  'div',
  { className: 'container' },
  React.createElement('h1', null, '标题'),
  React.createElement('p', null, '段落内容')
);
```

**Diff算法的核心原则：**

```javascript
// React的Diff算法有三个核心策略：

// 策略1：Tree Diff - 层级对比
/*
   - 两棵树只对比同一层级的节点
   - 跨层级的节点移动，React会当作删除+创建处理
   - 便于算法优化，避免跨层级移动的低效

   示例：
   div                      div
   ├─ span          =>      ├─ p    （删除span，创建p）
   └─ p                    └─ span （创建span，删除p）

   实际上React只会标记为更新，因为同是div的子元素
*/

// 策略2：Component Diff - 组件对比
/*
   - 不同类型的组件产生不同的树
   - 同类型组件才会继续对比
   - 组件类型改变时，React会卸载旧组件、创建新组件

   示例：
   <Header />         <Main />   （完全不同的树）
   <Content />   =>   <Sidebar />
   <Footer />         <Footer />

   Header -> Main: 卸载Header，创建Main
   Footer -> Footer: 可能复用（相同类型）
*/

// 策略3：Element Diff - 元素对比（Key的作用）
/*
   - 同一层级的子元素通过key来区分
   - React通过key来匹配元素

   没有key的情况（列表移动会导致性能问题）：
   A -> B -> C -> D
   B -> A -> D -> C  （React认为所有元素都变了，需要重新创建）

   有key的情况（高效复用）：
   A -> B -> C -> D
   B -> A -> D -> C  （React通过key识别：A不变，C变成A，A变成B...）
*/

// Key的最佳实践：
// 使用稳定的、唯一的ID
const todoList = [
  { id: 'todo-1', text: '学习React' },
  { id: 'todo-2', text: '学习Vue' },
  { id: 'todo-3', text: '学习Angular' }
];

// 错误的key使用：
// 1. 使用数组索引 - 列表重排时会导致bug
// {this.props.todos.map((todo, index) => (
//   <Todo key={index} {...todo} />
// ))}

// 2. 使用随机数 - 每次渲染都不同
// <Todo key={Math.random()} {...todo} />

// 正确的key使用：
// {this.props.todos.map(todo => (
//   <Todo key={todo.id} {...todo} />
// ))}
```

**Diff算法的具体执行流程：**

```javascript
// React对比新旧虚拟DOM的简化实现
function diff(oldTree, newTree) {
  // 1. 节点类型不同 - 替换整个节点及其子节点
  if (oldTree.type !== newTree.type) {
    return createNewTree(newTree);
  }

  // 2. 节点类型相同 - 比较属性
  if (oldTree.type === newTree.type) {
    const changes = [];

    // 比较属性变化
    const oldProps = oldTree.props;
    const newProps = newTree.props;

    for (const key in newProps) {
      if (oldProps[key] !== newProps[key]) {
        changes.push({ type: 'UPDATE_PROP', key, value: newProps[key] });
      }
    }

    for (const key in oldProps) {
      if (!(key in newProps)) {
        changes.push({ type: 'REMOVE_PROP', key });
      }
    }

    // 递归比较子节点
    const oldChildren = oldTree.children;
    const newChildren = newTree.children;

    diffChildren(oldChildren, newChildren);

    return changes;
  }
}

// 子节点Diff算法
function diffChildren(oldChildren, newChildren) {
  // 使用key来匹配元素
  const keyedOld = oldChildren.reduce((acc, child, index) => {
    acc[child.key || index] = child;
    return acc;
  }, {});

  newChildren.forEach((newChild, index) => {
    const oldChild = keyedOld[newChild.key || index];

    if (oldChild) {
      // 元素存在，进行diff
      diff(oldChild, newChild);
    } else {
      // 新元素，创建
      createElement(newChild);
    }
  });

  // 删除不存在的元素
  for (const key in keyedOld) {
    if (!(key in newChildren)) {
      removeElement(keyedOld[key]);
    }
  }
}
```

```javascript
  return { type: TOGGLE_TODO, id };

function todos(state = [], action) {

    case ADD_TODO:

    case TOGGLE_TODO:
      return state.map(todo =>
        todo.id === action.id ? { ...todo, completed: !todo.completed } : todo
      );
```

### 3.1 HTTP缓存

**问题：请详细解释HTTP缓存机制。强缓存和协商缓存有什么区别？Cache-Control有哪些常用值？**

**详细解答：**

HTTP缓存是前端性能优化的重要手段。合理使用缓存可以大幅减少网络请求，提升用户体验，降低服务器负载。

**缓存的分类：**

```javascript
// HTTP缓存分为两大类：
// 1. 强缓存（Cache-Control / Expires）
//    - 不需要发送请求到服务器，直接使用本地缓存
//    - 缓存有效时，请求不会到达服务器
//
// 2. 协商缓存（Last-Modified / ETag）
//    - 需要发送请求到服务器验证
//    - 服务器确认缓存有效则返回304，浏览器使用本地缓存

// 完整的缓存流程：
/*
   浏览器发起请求
         │
         ▼
   检查强缓存 ──────► 命中 ──────► 返回缓存（200 from cache）
         │                      │
         │ 未命中                │
         ▼                      │
   发送请求到服务器                │
         │                      │
         ▼                      │
   检查协商缓存 ──────► 命中 ──────► 返回304（Not Modified）
         │                      │
         │ 未命中                │
         ▼                      │
   返回新内容（200）               │
         │                      │
         ▼                      │
   存入缓存                       │
*/
```

**强缓存详解：**

```javascript
// 1. Cache-Control（HTTP/1.1，现代浏览器首选）
// 服务器响应头：
Response Headers:
  Cache-Control: max-age=3600  // 缓存有效期为3600秒

// 常用指令：
/*
  max-age=seconds        - 缓存有效期（秒）
  s-maxage=seconds       - 共享缓存（如CDN）的有效期
  no-cache               - 不使用强缓存，每次都验证
  no-store               - 不缓存任何内容
  public                 - 可被任何缓存存储（浏览器、CDN）
  private                - 只允许浏览器缓存
  must-revalidate        - 缓存过期后必须验证
  proxy-revalidate       - 代理缓存过期后必须验证
*/

// 实际例子
Cache-Control: public, max-age=31536000, immutable
// public: 可被任何缓存存储
// max-age=31536000: 一年
// immutable: 资源永不变

// 2. Expires（HTTP/1.0，兼容性考虑）
Response Headers:
  Expires: Wed, 21 Oct 2026 07:28:00 GMT
// 指定具体的过期时间
// 缺点：客户端时间和服务器时间可能不一致
```

**协商缓存详解：**

```javascript
// 1. Last-Modified / If-Modified-Since（HTTP/1.0）
// 首次请求：
Response Headers:
  Last-Modified: Wed, 21 Oct 2026 07:28:00 GMT

// 后续请求：
Request Headers:
  If-Modified-Since: Wed, 21 Oct 2026 07:28:00 GMT

// 服务器判断：
// - 资源未变化：返回304 Not Modified，浏览器使用缓存
// - 资源已变化：返回新内容（200 OK）

// 缺点：
// - 只能精确到秒级
// - 有些文件会周期性变化但内容不变

// 2. ETag / If-None-Match（HTTP/1.1，更精确）
// 首次请求：
Response Headers:
  ETag: "33a64df551425fcc55e4d42a148795d9f25f89d4"

// 后续请求：
Request Headers:
  If-None-Match: "33a64df551425fcc55e4d42a148795d9f25f89d4"

// 服务器通过ETag判断：
// - ETag匹配：返回304
// - ETag不匹配：返回新内容（200）

// ETag优先级高于Last-Modified
```

**实际开发中的应用场景：**

```javascript
// 场景1：HTML文件 - 协商缓存（经常变化）
Cache-Control: no-cache
// 每次都验证，但可避免重复下载

// 场景2：静态资源（CSS/JS/图片）- 强缓存+文件名哈希
Cache-Control: max-age=31536000, immutable
// 文件名包含内容哈希，内容变化则URL变化
// <link href="/app.abc123.css"> 哈希变了就是新文件

// 场景3：API数据 - 根据需求选择
// 用户数据（实时性要求高）：
Cache-Control: no-cache

// 可缓存数据（如列表数据）：
Cache-Control: max-age=60
// 1分钟内的请求直接用缓存

// 场景4：Service Worker中的缓存策略
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request).then((response) => {
      // 缓存命中返回缓存，否则网络请求
      return response || fetch(event.request);
    })
  );
});
```

**开发中的缓存问题及解决：**

```javascript
// 问题1：更新资源但用户看到旧版
// 原因：强缓存导致不请求服务器

// 解决方案1：文件名哈希（推荐）
// app.js -> app.abc123.js
// app.css -> app.def456.css
// 内容变化，URL变化，绕过缓存

// 解决方案2：使用版本号或时间戳
// <script src="app.js?v=2.0"></script>

// 解决方案3：配置no-cache
Cache-Control: no-cache, must-revalidate

// 问题2：CDN缓存导致更新不及时
// 解决方案：
Cache-Control: max-age=0, must-revalidate
// 或在CDN配置：缓存刷新时间

// 问题3：用户隐私数据被缓存
// 解决方案：
Cache-Control: no-store, private
// no-store: 完全不缓存
// private: 只允许用户本地缓存
```

- 知道Cache-Control各指令的含义
- 理解ETag和Last-Modified的区别

- 知道如何解决实际开发中的缓存更新问题

### 3.2 CORS与跨域

**问题：什么是CORS？同源策略是什么？跨域请求被拦截后应该如何解决？**

**详细解答：**

跨域问题是前端开发中经常遇到的拦路虎。理解跨域的原理和解决方案对于开发前后端分离的应用至关重要。

**同源策略（Same-Origin Policy）：**

```javascript
// 同源策略是浏览器的安全机制
// 限制了一个源的文档或脚本如何与另一个源的资源交互

// 源（Origin）= 协议 + 域名 + 端口
/*
  URL: https://example.com:443/path
  协议: https
  域名: example.com
  端口: 443

  同源例子：
  https://example.com/page1 和 https://example.com/page2 - 同源

  跨域例子：
  https://example.com 和 http://example.com - 协议不同
  https://example.com 和 https://api.example.com - 域名不同
  https://example.com:8080 和 https://example.com:3000 - 端口不同
*/

// 同源策略限制的行为：
// 1. Cookie、LocalStorage、IndexDB无法读取
```

### 3.3 WebSocket

**问题：WebSocket是什么？它和HTTP有什么区别？什么场景下应该使用WebSocket？**

**详细解答：**

WebSocket是一种在单个TCP连接上提供全双工通信的协议。它解决了HTTP请求-响应模型的局限性，特别适合需要实时数据更新的应用场景。

**HTTP与WebSocket的核心区别：**

```javascript
// HTTP请求-响应模型
/*
  客户端                          服务器
    │                               │
    │─────── HTTP Request ─────────►│
    │◄────── HTTP Response ─────────│
    │                               │
    │   （连接关闭）                  │
    │                               │
    │─────── HTTP Request ─────────►│
    │◄────── HTTP Response ─────────│
    │                               │
*/

// WebSocket全双工通信
/*
  客户端                          服务器
    │                               │
    │─────── HTTP Upgrade ─────────►│  (握手建立连接)
    │◄────── HTTP Response ─────────│
    │                               │
    │   (TCP连接保持)                 │
    │                               │
    │◄══════ WebSocket数据 ═════════►│  (双向通信)
    │◄══════ WebSocket数据 ═════════►│
    │◄══════ WebSocket数据 ═════════►│
*/

// 核心区别：
// 1. HTTP是半双工，WebSocket是全双工
// 2. HTTP每次请求都需要建立新连接，WebSocket一次连接持续通信
// 3. HTTP只能客户端发起请求，WebSocket双方都可以主动发送
// 4. WebSocket头部更小，传输效率更高
```

```javascript
    this.url = url;
    this.ws = null;
    this.timer = null;
    this.isAlive = false;
  }

      this.isAlive = true;
      this.startHeartbeat();
    };

    this.ws.onmessage = (event) => {
      this.isAlive = true; // 收到消息说明还活着

      if (event.data === 'pong') {
        console.log('收到心跳响应');
        return;
      }

      // 处理业务消息
      this.handleMessage(event.data);
    };

    this.ws.onclose = () => {
      this.stopHeartbeat();
    };

    this.ws.onerror = () => {
      console.error('连接错误');
    };
  }

  startHeartbeat() {
    // 每30秒发送一次心跳
    this.timer = setInterval(() => {
      if (!this.isAlive) {
        console.log('心跳超时，断开连接');
        this.ws.close();
        return;
      }

      this.isAlive = false;
      this.ws.send('ping');
    }, 30000);
  }

  stopHeartbeat() {
    if (this.timer) {
      clearInterval(this.timer);
      this.timer = null;
    }
  }

  handleMessage(data) {
    // 业务消息处理
    console.log('收到消息:', data);
  }
}
```

## 四、性能优化

### 4.1 重排与重绘

**问题：什么是重排（Reflow）和重绘（Repaint）？它们有什么区别？如何避免不必要的重排重绘？**

**详细解答：**

浏览器渲染页面的过程涉及多个步骤，理解重排和重绘是进行性能优化的基础。每当我们修改DOM的样式或结构时，浏览器可能需要重新计算布局（重排）或重新绘制像素（重绘），这些操作都会消耗性能。

**浏览器渲染基本原理：**

```javascript
// 浏览器渲染流程：
/*
  1. HTML解析 → DOM树
  2. CSS解析 → CSSOM树
  3. DOM + CSSOM → Render树
  4. Layout（布局）→ 计算每个节点的位置和大小
  5. Paint（绘制）→ 将节点绘制到图层
  6. Composite（合成）→ 合并图层生成最终画面

  重排：步骤4 - 布局改变
  重绘：步骤5 - 外观改变但布局没变
*/
```

**重排（Reflow/Layout）和重绘（Repaint）的区别：**

```javascript
// 重排（Reflow）：
// 当DOM的变化影响了文档的几何属性（宽、高、位置）时
// 浏览器需要重新计算元素的几何信息
// 会触发从根节点到目标节点的整条路径的所有元素

// 导致重排的操作：
/*
  - 添加或删除可见的DOM元素
  - 元素内容改变
  - 元素位置改变
  - 元素尺寸改变（margin、padding、border、width、height）
  - 页面初始渲染
  - 浏览器窗口尺寸改变（resize）
*/

// 重绘（Repaint）：
// 当DOM的变化只影响外观，不影响布局时
// 浏览器只需要重新绘制受影响的元素

// 导致重绘的操作：
/*
  - 颜色改变（color、background-color）
  - visibility改变（visible -> hidden）
  - 边框样式改变（但尺寸不变）
  - 阴影改变
*/

// 示例：颜色变化只触发重绘
element.style.color = 'blue'; // 重绘
element.style.backgroundColor = 'red'; // 重绘

// 示例：尺寸变化触发重排
element.style.width = '200px'; // 重排 + 重绘
element.style.padding = '10px'; // 重排 + 重绘

// 示例：transform和opacity可以避免重排重绘
element.style.transform = 'translateX(100px)'; // 只触发合成
element.style.opacity = '0.5'; // 只触发合成
```

**批量操作以减少重排重绘：**


### 4.2 防抖与节流

**问题：什么是防抖和节流？它们有什么区别？各自适合用在什么场景？请手写实现。**

**详细解答：**

防抖和节流是两种常用的性能优化技术，它们用于控制函数的执行频率。理解这两种技术的区别和应用场景，能够帮助我们优化事件处理、数据获取等高频触发操作的性能。

**防抖（Debounce）：**

```javascript
// 防抖的核心思想：事件触发后，等待一段时间
// 如果这段时间内事件再次触发，则重新计时
// 直到事件停止触发一段时间后，才执行函数

// 场景：搜索框输入 - 用户停止输入后才发送请求
// 场景：窗口调整大小 - 用户停止调整后才执行
// 场景：按钮点击 - 防止用户重复提交

// 防抖实现原理：
function debounce(func, delay) {
  let timer = null;

  return function(...args) {
    // 每次调用都清除之前的定时器
    if (timer) {
      clearTimeout(timer);
    }

    // 设置新的定时器
    timer = setTimeout(() => {
      func.apply(this, args);
    }, delay);
  };
}

// 使用示例
const handleSearch = debounce((keyword) => {
  console.log('搜索:', keyword);
  fetch(`/api/search?q=${keyword}`);
}, 500);

// 搜索框输入时
input.addEventListener('input', (e) => {
  handleSearch(e.target.value);
});
```

```javascript
// 防抖的完整实现（考虑立即执行）
function debounce(func, delay, immediate = false) {
  let timer = null;

  return function(...args) {
    const context = this;

    // 如果有定时器，说明在等待期间又被调用
    if (timer) {
      clearTimeout(timer);
    }

    if (immediate && !timer) {
      // 立即执行模式
      func.apply(context, args);
    }

    timer = setTimeout(() => {
      func.apply(context, args);

      if (immediate) {
        // 重置timer，允许下次立即执行
        timer = null;
      }
    }, delay);
  };
}

// 立即执行版的使用场景：
// 用户输入时立即搜索（不等待输入停止）
// 但用户连续输入时，不要在输入过程中执行
const handleInput = debounce((value) => {
  console.log('发送搜索请求:', value);
}, 1000, true);

input.addEventListener('input', (e) => {
  handleInput(e.target.value);
});
```

**节流（Throttle）：**

```javascript
// 节流的核心思想：事件触发后，立即执行函数
// 然后锁定一段时间，这段时间内不再执行
// 锁定结束后，事件再次触发才执行

// 场景：滚动事件 - 每隔一段时间处理一次
// 场景：鼠标移动事件 - 限制移动时的处理频率
// 场景：resize事件 - 控制处理次数

// 节流实现原理：
function throttle(func, interval) {
  let lastTime = 0; // 上次执行时间

  return function(...args) {
    const now = Date.now();

    // 如果距离上次执行已经超过interval，执行并更新时间戳
    if (now - lastTime >= interval) {
      func.apply(this, args);
      lastTime = now;
    }
  };
}

// 使用示例
const handleScroll = throttle(() => {
  console.log('处理滚动位置:', window.scrollY);
  updateLazyLoadImages();
}, 200);

window.addEventListener('scroll', handleScroll);
```

```javascript
// 节流的另一种实现：使用定时器
function throttle(func, interval) {
  let timer = null;

  return function(...args) {
    // 如果已经在定时器中，说明刚执行过，忽略
    if (timer) {
      return;
    }

    timer = setTimeout(() => {
      func.apply(this, args);
      timer = null; // 执行完毕后清除标记
    }, interval);
  };
}

// 节流带尾部执行（最后一次一定会执行）
function throttleWithTail(func, interval) {
  let lastTime = 0;
  let timer = null;

  return function(...args) {
    const now = Date.now();
    const remaining = interval - (now - lastTime);

    if (remaining <= 0) {
      // 距上次执行已超过interval，立即执行
      if (timer) {
        clearTimeout(timer);
        timer = null;
      }
      lastTime = now;
      func.apply(this, args);
    } else if (!timer) {
      // 不到interval，但在interval后执行一次
      timer = setTimeout(() => {
        lastTime = Date.now();
        timer = null;
        func.apply(this, args);
      }, remaining);
    }
  };
}
```

### 4.3 懒加载

**详细解答：**

懒加载（Lazy Load）是一种性能优化技术，它延迟加载非关键资源，只在需要时才加载。这可以减少页面的初始加载时间，提升用户体验，节省带宽。

```javascript
// 懒加载的核心原理：
// 1. 先用占位图展示
// 2. 真实图片的URL存在data-src属性中
// 3. 元素进入视口时，将data-src赋值给src
// 4. 触发图片加载

// 基础实现
function lazyLoadImage() {
  const images = document.querySelectorAll('img[data-src]');

  const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        const img = entry.target;
        img.src = img.dataset.src;
        img.removeAttribute('data-src');
        observer.unobserve(img);
      }
    });
  }, {
    root: null,          // 使用视口作为根
    rootMargin: '0px',   // 提前触发
    threshold: 0         // 元素可见就触发
  });

}
```

```javascript
class LazyImage {
  constructor(selector) {
    this.images = document.querySelectorAll(selector);
    this.init();
  }

  init() {
    // 使用IntersectionObserver
    this.observer = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          this.loadImage(entry.target);
        }
      });
    }, {
      rootMargin: '50px', // 提前50px加载
      threshold: 0
    });

    this.images.forEach(img => this.observer.observe(img));
  }

  loadImage(img) {
    const src = img.dataset.src;

    // 显示占位图（如果还没有）
    if (!img.src.includes('placeholder')) {
      img.src = img.dataset.placeholder || '/placeholder.jpg';
    }

    // 创建新图片来预加载
    const tempImage = new Image();

    tempImage.onload = () => {
      img.src = src;
      img.classList.add('loaded');
    };

    tempImage.onerror = () => {
      console.error('图片加载失败:', src);
      img.classList.add('error');
    };

    tempImage.src = src;

    // 停止观察
    this.observer.unobserve(img);
  }
}
```

```javascript
// 完善版本：处理构造函数场景
Function.prototype.myBind = function(context, ...args) {
  const originalFunc = this;

  // 如果原函数是箭头函数，直接返回（箭头函数没有自己的this）
  if (originalFunc.prototype === undefined) {
    return originalFunc;
  }

  // 创建空函数作为桥梁
  const EmptyFunc = function() {};

  // 绑定函数
  const boundFunc = function(...innerArgs) {
    // 判断是否作为构造函数调用
    // 如果使用new调用，this会指向新创建的实例
    // 否则this指向context（绑定的对象）
    const thisToUse = this instanceof EmptyFunc ? this : context;

    return originalFunc.apply(thisToUse, [...args, ...innerArgs]);
  };

  // 处理原型链
  // 如果原函数有原型，需要让绑定函数继承原函数的原型
  EmptyFunc.prototype = originalFunc.prototype;
  boundFunc.prototype = new EmptyFunc();

  // 处理函数 name
  boundFunc.prototype.constructor = originalFunc;

  return boundFunc;
};
```

```javascript
// 最终完善版本：更接近原生实现
Function.prototype.myBind = function(thisArg, ...args) {
  if (typeof this !== 'function') {
    throw new TypeError('Function.prototype.bind called on non-function');
  }

  const func = this;
  const context = thisArg === null || thisArg === undefined ? globalThis : Object(thisArg);

  // 剩余参数会在调用时拼接
  const boundFunc = function(...restArgs) {
    // 判断是否为 new 调用
    const isNew = this instanceof boundFunc;

    // 优先使用 new 的 this，否则使用绑定的 context
    const targetThis = isNew ? this : context;

    // 拼接参数：bind时预设的参数 + 调用时传入的参数
    const finalArgs = [...args, ...restArgs];

    // 使用 apply 调用，保持函数调用语义
    return func.apply(targetThis, finalArgs);
  };

  // 安全地处理原型链
  if (func.prototype) {
    // 创建一个中间函数，用它来继承原型的属性
    const Nop = function() {};
    Nop.prototype = func.prototype;
    boundFunc.prototype = new Nop();
    boundFunc.prototype.constructor = boundFunc;
  }

  // 设置 length（函数参数个数）
  // 但由于rest参数的存在，这个只能是一个估算
  const descriptor = Object.getOwnPropertyDescriptor(func, 'length');
  if (descriptor) {
    Object.defineProperty(boundFunc, 'length', {
      ...descriptor,
      configurable: true
    });
  }

  return boundFunc;
};
```

### 5.2 实现Promise

**问题：请手写一个简易的Promise/A+规范的Promise实现。**

**详细解答：**

Promise是现代JavaScript异步编程的基础。理解Promise的实现原理对于深入掌握JavaScript异步机制至关重要。

**Promise基础结构：**

```javascript
// Promise的三种状态
const PENDING = 'pending';
const FULFILLED = 'fulfilled';
const REJECTED = 'rejected';

// 简易Promise实现
class MyPromise {
  constructor(executor) {
    // 初始状态
    this.state = PENDING;
    // 成功值
    this.value = undefined;
    // 失败原因
    this.reason = undefined;
    // 成功回调队列
    this.onFulfilledCallbacks = [];
    // 失败回调队列
    this.onRejectedCallbacks = [];

    // 成功处理
    const resolve = (value) => {
      // 只有pending状态可以转换
      if (this.state === PENDING) {
        this.state = FULFILLED;
        this.value = value;

        // 执行所有待执行的成功回调
        this.onFulfilledCallbacks.forEach(cb => cb(value));
      }
    };

    // 失败处理
    const reject = (reason) => {
      if (this.state === PENDING) {
        this.state = REJECTED;
        this.reason = reason;

        this.onRejectedCallbacks.forEach(cb => cb(reason));
      }
    };

    // 执行器可能会抛出异常，需要捕获
    try {
      executor(resolve, reject);
    } catch (error) {
      reject(error);
    }
  }

  // then方法
  then(onFulfilled, onRejected) {
    // 参数可选处理
    onFulfilled = typeof onFulfilled === 'function' ? onFulfilled : value => value;
    onRejected = typeof onRejected === 'function' ? onRejected : reason => { throw reason; };

    // 返回新Promise，实现链式调用
    const promise2 = new MyPromise((resolve, reject) => {
      // 根据状态处理
      if (this.state === FULFILLED) {
        // 异步执行回调（符合Promise/A+规范）
        setTimeout(() => {
          try {
            const x = onFulfilled(this.value);
            resolvePromise(promise2, x, resolve, reject);
          } catch (error) {
            reject(error);
          }
        }, 0);
      } else if (this.state === REJECTED) {
        setTimeout(() => {
          try {
            const x = onRejected(this.reason);
            resolvePromise(promise2, x, resolve, reject);
          } catch (error) {
            reject(error);
          }
        }, 0);
      } else {
        // pending状态，将回调存入队列
        this.onFulfilledCallbacks.push((value) => {
          setTimeout(() => {
            try {
              const x = onFulfilled(value);
              resolvePromise(promise2, x, resolve, reject);
            } catch (error) {
              reject(error);
            }
          }, 0);
        });

        this.onRejectedCallbacks.push((reason) => {
          setTimeout(() => {
            try {
              const x = onRejected(reason);
              resolvePromise(promise2, x, resolve, reject);
            } catch (error) {
              reject(error);
            }
          }, 0);
        });
      }
    });

    return promise2;
  }
}

// 处理Promise返回值（核心逻辑）
function resolvePromise(promise2, x, resolve, reject) {
  // 如果返回的是自身，跳出循环
  if (promise2 === x) {
    return reject(new TypeError('Chaining cycle detected'));
  }

  // 如果是Promise
  if (x instanceof MyPromise) {
    x.then(
      value => resolve(value),
      reason => reject(reason)
    );
    return;
  }

  // 如果是普通值或函数
  if (x !== null && (typeof x === 'object' || typeof x === 'function')) {
    let then;
    try {
      then = x.then;
    } catch (error) {
      return reject(error);
    }

    if (typeof then === 'function') {
      // 如果是thenable对象（有then方法）
      let called = false; // 防止多次调用
      try {
        then.call(
          x,
          value => {
            if (called) return;
            called = true;
            resolvePromise(promise2, value, resolve, reject);
          },
          reason => {
            if (called) return;
            called = true;
            reject(reason);
          }
        );
      } catch (error) {
        if (called) return;
        reject(error);
      }
    } else {
      resolve(x);
    }
  } else {
    resolve(x);
  }
}
```

```javascript
    const results = [];
    let completedCount = 0;

    if (promises.length === 0) {
      return resolve([]);

    promises.forEach((promise, index) => {
      MyPromise.resolve(promise).then(
        value => {
          results[index] = value;
          completedCount++;
          if (completedCount === promises.length) {
            resolve(results);
          }
        },
        reason => {
          reject(reason);
        }
      );
    });
  });
};

    if (!Array.isArray(promises)) {
      return reject(new TypeError('Arguments must be an array'));
    }

    promises.forEach(promise => {
      MyPromise.resolve(promise).then(
        value => resolve(value),
        reason => reject(reason)
      );
    });
  });
};

MyPromise.allSettled = function(promises) {
  return new MyPromise((resolve) => {
    const results = [];
    let completedCount = 0;

    if (promises.length === 0) {
      return resolve([]);
    }

    promises.forEach((promise, index) => {
      MyPromise.resolve(promise).then(
        value => {
          results[index] = { status: 'fulfilled', value };
        },
        reason => {
          results[index] = { status: 'rejected', reason };
        }
      ).finally(() => {

        if (completedCount === promises.length) {

        }
```

### 5.3 实现深拷贝（复习扩展）

**问题：请手写一个完整的深拷贝函数，要求处理循环引用、Symbol、Date、RegExp、Map、Set等类型。**

**详细解答：**

深拷贝是面试中的高频考点。前面我们已经介绍过深拷贝的基础实现，这里我们给出一个更加完善的版本。

**完整深拷贝实现：**

```javascript
function deepClone(target, hash = new WeakMap()) {
  // 处理null和undefined
  if (target === null) return null;
  if (typeof target !== 'object') return target;

  // 处理Date
  if (target instanceof Date) {
    return new Date(target.getTime());
  }

  // 处理RegExp
  if (target instanceof RegExp) {
    const flags = target.flags || '';
    return new RegExp(target.source, flags);
  }

  // 处理Map
  if (target instanceof Map) {
    const clonedMap = new Map();
    hash.set(target, clonedMap);
    target.forEach((value, key) => {
      clonedMap.set(
        deepClone(key, hash),
        deepClone(value, hash)
      );
    });
    return clonedMap;
  }

  // 处理Set
  if (target instanceof Set) {
    const clonedSet = new Set();
    hash.set(target, clonedSet);
    target.forEach(value => {
      clonedSet.add(deepClone(value, hash));
    });
    return clonedSet;
  }

  // 处理Symbol作为key
  if (typeof target === 'object' && target !== null) {
    // 处理Symbol类型的属性
    if (typeof Symbol !== 'undefined') {
      const symbolKeys = Object.getOwnPropertySymbols(target);
      const clonedObj = Array.isArray(target) ? [] : {};

      // 先复制常规属性
      for (const key in target) {
        if (Object.prototype.hasOwnProperty.call(target, key)) {
          clonedObj[key] = deepClone(target[key], hash);
        }
      }

      // 再复制Symbol属性
      for (const sym of symbolKeys) {
        clonedObj[sym] = deepClone(target[sym], hash);
      }

      return clonedObj;
    }
  }

  // 处理函数
  if (typeof target === 'function') {
    // 普通函数
    if (target.prototype) {
      return function(...args) {
        return target.apply(this, args);
      };
    }
    // 箭头函数（没有prototype）
    return target;
  }

  // 处理Error
  if (target instanceof Error) {
    return new target.constructor(target.message);
  }

  // 防止循环引用导致的无限递归
  if (hash.has(target)) {
    return hash.get(target);
  }

  // 处理数组和普通对象
  const clonedObj = Array.isArray(target) ? [] : {};
  hash.set(target, clonedObj);

  for (const key in target) {
    if (Object.prototype.hasOwnProperty.call(target, key)) {
      clonedObj[key] = deepClone(target[key], hash);
    }
  }

  return clonedObj;
}
```

### 6.2 浏览器渲染原理

**问题：请描述浏览器从输入URL到页面展示的完整过程。**

**详细解答：**

这是面试中最经典的综合题，涉及网络、操作系统、前端等多个领域的知识。完整理解这个过程对于成为优秀的前端工程师至关重要。

**完整渲染流程：**

```javascript
// 1. URL解析
/*
  - 浏览器解析URL（协议、域名、路径、参数）
  - HSTS列表检查（安全传输）
  - DNS查询（域名 -> IP）
*/

// 2. DNS查询详解
/*
  浏览器缓存 -> 系统缓存 -> 路由器缓存 -> ISP DNS缓存
  -> 递归查询 -> 根域名服务器 -> 顶级域名服务器 -> 权威域名服务器

  DNS优化：
  - DNS预解析 <link rel="dns-prefetch" href="//static.example.com">
  - 减少DNS查询（减少域名数量）
  - DNS负载均衡
*/

// 3. TCP连接（三次握手）
/*
  客户端 -> SYN -> 服务器
  客户端 <- SYN-ACK <- 服务器
  客户端 -> ACK -> 服务器
  连接建立！
*/

// 4. TLS/SSL握手（HTTPS）
/*
  1. 客户端发送支持的TLS版本、加密套件、随机数C
  2. 服务器回复证书、服务器随机数S、选择的加密套件
  3. 客户端验证证书，生成预主密钥，用公钥加密后发送
  4. 双方基于随机数和预主密钥生成会话密钥
  后续通信使用会话密钥加密
*/

// 5. 发送HTTP请求
/*
  请求行：GET /index.html HTTP/1.1
  请求头：Host, User-Agent, Accept, Cookie等
  请求体：POST数据（如果有）
*/

// 6. 服务器处理并响应
/*
  状态码：
  1xx - 信息响应
  2xx - 成功（200 OK, 201 Created, 204 No Content）
  3xx - 重定向（301 Moved, 302 Found, 304 Not Modified）
  4xx - 客户端错误（400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found）
  5xx - 服务器错误（500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable）
*/

// 7. 浏览器解析HTML
/*
  字节流 -> 分词器 -> 令牌 -> 树构造器 -> DOM树
*/

// 8. CSS解析和CSSOM
/*
  CSS字节 -> CSS规则 -> CSSOM树
  CSSOM：CSS Object Model
*/

// 9. JavaScript执行
/*
  - HTML解析遇到<script>会暂停DOM构建
  - defer属性：异步下载，等DOM构建完成执行
  - async属性：异步下载，下载完立即执行
  - 现代做法：<script type="module">默认defer
*/

// 10. 构建渲染树
/*
  DOM + CSSOM -> Render Tree
  只包含可见节点
  不包含：display: none, <head>, <script>等
*/

// 11. 布局计算（Layout/Reflow）
/*
  - 计算每个节点的位置和大小
  - 盒模型计算
  - 文档流布局
*/

// 12. 分层绘制
/*
  - 浏览器将页面分成多个图层
  - 触发重排重绘的元素会有自己的图层
  - 使用transform、opacity、will-change等可以创建新图层
*/

// 13. 栅格化（Rasterization）
/*
  将矢量图转换为位图
  可能使用GPU加速
*/

// 14. 合成（Compositing）
/*
  合并所有图层生成最终画面
  通过GPU发送到屏幕
*/

// 完整流程图
/*
  URL输入
     │
     ▼
  DNS解析
     │
     ▼
  TCP连接 ◄─────────── 三次握手
     │
     ▼

     │
     ▼
  HTTP请求 ──────────► 服务器
     │
     ▼

     │
     ▼
  解析HTML ─────► DOM树
```

```javascript
  - 展示知识的广度和深度
*/

// 4. 手写代码题技巧
/*
  - 先思考后动手
  - 先实现核心逻辑
  - 考虑边界情况
  - 写完后主动检查
  - 如果发现错误，勇于承认并修正
*/

// 5. 项目经历的描述
/*
  使用STAR法则：
  Situation（情境）: 项目背景是什么？
  Task（任务）: 你的职责是什么？
  Action（行动）: 你做了哪些具体工作？
  Result（结果）: 取得了什么成果？

  示例：
  "我负责重构了项目的状态管理模块（情境）
  原来使用Redux代码量大，维护困难（任务）
  我引入Zustand，重构了20多个store，统一了状态管理模式（行动）
  代码量减少了40%，团队满意度提升，后续项目都采用了这个方案（结果）"
*/
```
