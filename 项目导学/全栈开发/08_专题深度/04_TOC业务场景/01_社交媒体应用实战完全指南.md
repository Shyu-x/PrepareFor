## 一、社交应用概述

### 1.1 社交应用的核心本质

社交应用的本质是**建立人与人之间的连接**，围绕三个核心要素展开：

| 核心要素 | 说明 | 产品表现 |
|----------|------|----------|
| **关系链** | 用户之间的连接方式 | 关注、好友、粉丝、双向关系 |
| **内容** | 用户产生的信息载体 | 文字、图片、视频、直播 |
| **互动** | 用户之间的行为交互 | 点赞、评论、转发、私信 |

这三个要素相互依存，构成了社交产品的铁三角。关系链决定内容分发范围，内容是用户留存的核心，互动则是产生黏性的关键。

### 1.3 技术挑战全景图

社交应用面临的技术挑战远超一般Web应用：

**1. 实时性挑战**
- Feed更新需要秒级触达用户
- 消息发送延迟要求低于100ms
- 在线状态需要实时同步
- 评论、点赞需要即时反馈

**2. 一致性挑战**
- 点赞数、评论数统计的准确性
- 多端登录时的数据同步
- 分布式环境下的关系链一致性
- 消息投递的可靠性

**3. 海量数据挑战**
- 日活亿级的用户规模
- 每日数十亿级的Feed读取
- 关注关系可能达到千万级
- 历史消息的存储与查询

**4. 性能挑战**
- 首页Feed接口响应时间<50ms
- 图片加载需要CDN加速
- 大V账号的粉丝分发
- 热点的瞬时流量冲击

**从产品角度看**，社交产品需要同时满足：

## 二、Feed流系统设计

### 2.1 推送模式 vs 拉取模式：Push vs Pull

Feed流（信息流）是社交应用的核心功能，用户通过Feed获取关注对象发布的内容。根据数据分发方式，可以分为两种典型架构：

**1. 拉取模式（Pull Model）**

```
用户请求 → 获取关注列表 → 获取关注者内容 → 合并排序 → 返回

优点：
- 数据实时性强
- 存储成本低
- 实现简单

缺点：
- 读取放大：用户关注1000人，需查询1000次数据库
- 响应延迟高：多次数据库查询
- 无法应对大V账号
```

**拉取模式代码实现：**

```typescript
// 后端：Feed拉取服务
// services/feedPullService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository, In } from 'typeorm';
import { Post } from '../entities/post.entity';
import { Follow } from '../entities/follow.entity';

@Injectable()
export class FeedPullService {
  constructor(
    @InjectRepository(Post)
    private readonly postRepository: Repository<Post>,
    @InjectRepository(Follow)
    private readonly followRepository: Repository<Follow>,
  ) {}

  /**
   * 拉取模式的Feed获取
   * 适用于关注数较少、帖子数量适中的场景
   *
   * @param userId 当前用户ID
   * @param cursor 游标分页位置
   * @param limit 每页数量
   */
  async getFeedForUser(
    userId: string,
    cursor: string | null,
    limit: number = 20,
  ): Promise<{ posts: Post[]; nextCursor: string | null }> {
    // 第一步：获取用户关注列表
    const follows = await this.followRepository.find({
      where: { followerId: userId },
      select: ['followingId'],
    });

    if (follows.length === 0) {
      return { posts: [], nextCursor: null };
    }

    // 提取关注者ID列表
    const followingIds = follows.map(f => f.followingId);

    // 第二步：构建查询条件
    // 如果有cursor，使用cursor作为时间戳分页
    const queryBuilder = this.postRepository
      .createQueryBuilder('post')
      .where('post.authorId IN (:...followingIds)', { followingIds })
      .orderBy('post.createdAt', 'DESC')
      .take(limit + 1); // 多取一条用于判断是否有下一页

    // 第三步：应用游标分页
    if (cursor) {
      // cursor格式：Base64编码的时间戳
      const decodedCursor = Buffer.from(cursor, 'base64').toString('utf-8');
      const cursorTime = new Date(decodedCursor);
      queryBuilder.andWhere('post.createdAt < :cursorTime', {
        cursorTime,
      });
    }

    // 第四步：执行查询
    const posts = await queryBuilder.getMany();

    // 第五步：生成分页游标
    const hasNextPage = posts.length > limit;
    if (hasNextPage) {
      posts.pop(); // 移除多余的一条
    }

    const nextCursor = hasNextPage && posts.length > 0
      ? Buffer.from(posts[posts.length - 1].createdAt.toISOString()).toString('base64')
      : null;

    return { posts, nextCursor };
  }

  /**
   * 获取用户首页Feed（融合算法排序）
   * 实际生产环境中会结合推荐算法进行排序
   */
  async getPersonalizedFeed(
    userId: string,
    cursor: string | null,
    limit: number = 20,
  ): Promise<{ posts: Post[]; nextCursor: string | null }> {
    // 1. 获取基础Feed（关注者发布的内容）
    const { posts: followingPosts, nextCursor } = await this.getFeedForUser(
      userId,
      cursor,
      Math.floor(limit * 0.7), // 70%来自关注内容
    );

    // 2. 获取推荐内容（基于用户兴趣）
    const recommendedPosts = await this.getRecommendedPosts(
      userId,
      limit - followingPosts.length,
    );

    // 3. 合并并重新排序
    const mergedPosts = this.mergeAndRankPosts(followingPosts, recommendedPosts, userId);

    return { posts: mergedPosts, nextCursor };
  }

  /**
   * 获取推荐内容
   * 基于协同过滤的简单实现
   */
  private async getRecommendedPosts(
    userId: string,
    limit: number,
  ): Promise<Post[]> {
    // 简化实现：返回热门/最新内容
    return this.postRepository.find({
      where: {
        isPublic: true,
        // 排除已读内容...
      },
      order: { likeCount: 'DESC', createdAt: 'DESC' },
      take: limit,
    });
  }

  /**
   * 合并并排序帖子
   * 结合时间序和互动质量进行综合排序
   */
  private mergeAndRankPosts(
    followingPosts: Post[],
    recommendedPosts: Post[],
    userId: string,
  ): Post[] {
    // 简单加权排序算法
    const scoredPosts = [
      ...followingPosts.map(p => ({
        ...p,
        score: this.calculatePostScore(p, 'following'),
      })),
      ...recommendedPosts.map(p => ({
        ...p,
        score: this.calculatePostScore(p, 'recommended'),
      })),
    ];

    // 按分数降序排列
    scoredPosts.sort((a, b) => b.score - a.score);

    return scoredPosts.map(p => {
      const { score, source, ...post } = p;
      return post;
    });
  }

  /**
   * 计算帖子得分
   * 考虑因素：互动率、时间衰减、内容质量
   */
  private calculatePostScore(post: Post, source: string): number {
    // 互动得分
    const interactionScore =
      post.likeCount * 1 +
      post.commentCount * 3 +
      post.repostCount * 5;

    // 时间衰减因子（24小时前的帖子分数减半）
    const hoursAge = (Date.now() - post.createdAt.getTime()) / (1000 * 60 * 60);
    const timeDecay = Math.pow(0.95, hoursAge);

    // 来源权重
    const sourceWeight = source === 'following' ? 1.2 : 0.8;

    return interactionScore * timeDecay * sourceWeight;
  }
}
```

**2. 推送模式（Push Model）**

```
发布内容 → 写入自己的Feed → 分发给所有粉丝的收件箱

优点：
- 读取性能好：直接读取自己的Feed
- 适合大V账号：写入放大而非读取放大
- 用户体验好：秒级更新

缺点：
- 存储成本高：每条内容需要存储N份
- 分发延迟：分发到大V粉丝需要时间
- 实现复杂：需要解决内容更新问题
```

**推送模式代码实现：**

```typescript
// 后端：Feed推送服务
// services/feedPushService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { RedisService } from '../services/redis.service';
import { Post } from '../entities/post.entity';
import { User } from '../entities/user.entity';

@Injectable()
export class FeedPushService {
  // 收件箱缓存队列的最大长度
  private readonly INBOX_MAX_SIZE = 1000;

  constructor(
    @InjectRepository(Post)
    private readonly postRepository: Repository<Post>,
    @InjectRepository(User)
    private readonly userRepository: Repository<User>,
    private readonly redisService: RedisService,
  ) {}

  /**
   * 发布内容并分发到粉丝收件箱
   * 核心流程：写入帖子 → 分发到关注者收件箱
   *
   * @param post 发布的帖子
   */
  async publishAndDistribute(post: Post): Promise<void> {
    // 第一步：写入帖子内容
    await this.postRepository.save(post);

    // 第二步：获取粉丝列表
    const followers = await this.getFollowers(post.authorId);

    // 第三步：异步分发到粉丝收件箱
    // 注意：这是异步操作，不阻塞主流程
    this.distributeToInboxes(post, followers);

    // 第四步：处理大V账号的特殊分发策略
    if (followers.length > 100000) {
      // 粉丝数超过10万的大V，走异步分发队列
      await this.queueLargeVDistribution(post, followers);
    }
  }

  /**
   * 同步分发到收件箱
   * 适用于中小V账号（粉丝数 < 10万）
   */
  private async distributeToInboxes(post: Post, followers: User[]): Promise<void> {
    const pipeline = this.redisService.getPipeline();
    const postPreview = this.createPostPreview(post);

    for (const follower of followers) {
      // 使用Redis List的LPUSH将新帖子推入粉丝收件箱
      // 收件箱key格式：inbox:{userId}
      const inboxKey = `inbox:${follower.id}`;

      // 添加到收件箱
      pipeline.lpush(inboxKey, JSON.stringify(postPreview));

      // 保持收件箱大小，超出则裁剪
      pipeline.ltrim(inboxKey, 0, this.INBOX_MAX_SIZE - 1);

      // 设置收件箱更新时间
      pipeline.hset(
        `inbox:meta:${follower.id}`,
        'lastUpdate',
        Date.now().toString(),
      );
    }

    await pipeline.exec();
  }

  /**
   * 大V异步分发队列
   * 将分发任务放入消息队列，避免阻塞主流程
   */
  private async queueLargeVDistribution(post: Post, followers: User[]): Promise<void> {
    // 粉丝分批处理，每批10000人
    const BATCH_SIZE = 10000;

    for (let i = 0; i < followers.length; i += BATCH_SIZE) {
      const batch = followers.slice(i, i + BATCH_SIZE);

      // 放入异步任务队列
      await this.redisService.lpush(
        'distribution:queue',
        JSON.stringify({
          postId: post.id,
          postPreview: this.createPostPreview(post),
          batchUserIds: batch.map(u => u.id),
          batchIndex: Math.floor(i / BATCH_SIZE),
        }),
      );
    }
  }

  /**
   * 从收件箱读取Feed
   */
  async getFeedFromInbox(
    userId: string,
    cursor: string | null,
    limit: number = 20,
  ): Promise<{ posts: Post[]; nextCursor: string | null }> {
    const inboxKey = `inbox:${userId}`;

    // 计算起始位置
    let start = 0;
    if (cursor) {
      start = parseInt(cursor, 10);
    }

    // 从收件箱读取
    const rawPosts = await this.redisService.lrange(
      inboxKey,
      start,
      start + limit,
    );

    // 判断是否有下一页
    const hasNextPage = rawPosts.length === limit;
    const nextCursor = hasNextPage ? (start + limit).toString() : null;

    // 解析帖子数据
    const posts = rawPosts.map(raw => JSON.parse(raw));

    return { posts, nextCursor };
  }

  /**
   * 创建帖子预览数据
   * 只存储展示所需的最少信息，减少存储占用
   */
  private createPostPreview(post: Post): PostPreview {
    return {
      id: post.id,
      authorId: post.authorId,
      authorName: post.author?.name || '',
      authorAvatar: post.author?.avatar || '',
      content: post.content,
      images: post.images?.slice(0, 9) || [], // 只保留前9张图
      likeCount: post.likeCount,
      commentCount: post.commentCount,
      createdAt: post.createdAt,
    };
  }

  /**
   * 获取用户的所有粉丝
   */
  private async getFollowers(userId: string): Promise<User[]> {
    return this.userRepository
      .createQueryBuilder('user')
      .innerJoin('user.followers', 'follow')
      .where('follow.followingId = :userId', { userId })
      .getMany();
  }

  /**
   * 收件箱合并（用户同时有关注和推荐内容）
   */
  async getMergedFeed(
    userId: string,
    cursor: string | null,
    limit: number = 20,
  ): Promise<{ posts: Post[]; nextCursor: string | null }> {
    // 从Redis收件箱读取
    const inboxResult = await this.getFeedFromInbox(userId, null, limit * 2);

    // 从推荐服务获取推荐内容
    const recommendedPosts = await this.getRecommendedPosts(userId, limit);

    // 合并并排序
    const merged = this.mergeFeeds(inboxResult.posts, recommendedPosts);

    return {
      posts: merged.slice(0, limit),
      nextCursor: merged.length > limit ? (cursor ? parseInt(cursor) + limit : limit).toString() : null,
    };
  }
}
```

### 2.2 Timeline存储：写入放大 vs 读取放大

**核心矛盾：**

| 模式 | 写入放大 | 读取放大 | 适用场景 |
|------|----------|----------|----------|
| **Pull** | 无 | 高（读取N个关注者的数据） | 关注数少、内容多的场景 |
| **Push** | 高（写入N份到粉丝收件箱） | 无（直接读取自己的收件箱） | 关注数多、粉丝均匀的场景 |
| **混合** | 中等 | 中等 | 大多数通用场景 |

**混合模式设计：**

```typescript
// 后端：混合模式Feed服务
// services/hybridFeedService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { RedisService } from './redis.service';
import { Post } from '../entities/post.entity';

// 大V阈值：粉丝数超过此值采用Push模式
const BIG_V_THRESHOLD = 10000;

@Injectable()
export class HybridFeedService {
  constructor(
    @InjectRepository(Post)
    private readonly postRepository: Repository<Post>,
    private readonly redisService: RedisService,
  ) {}

  /**
   * 混合模式的Feed获取策略
   *
   * 策略逻辑：
   * 1. 普通用户（粉丝 < 1万）：采用Pull模式，读取时聚合
   * 2. 大V用户（粉丝 > 1万）：采用Push模式，预计算收件箱
   * 3. 热点内容：强制Push，确保第一时间触达
   */
  async getFeed(
    userId: string,
    cursor: string | null,
    limit: number = 20,
  ): Promise<{ posts: any[]; nextCursor: string | null; source: string }> {
    // 获取用户的关注列表和他们的粉丝数
    const followings = await this.getFollowingsWithStats(userId);

    // 分离普通账号和大V账号
    const { normalUsers, bigVUsers } = this.categorizeUsers(followings);

    // 并行执行两种模式的查询
    const [normalPosts, bigVInboxPosts] = await Promise.all([
      // 普通账号：Pull模式，实时聚合
      this.pullFeedFromNormalUsers(normalUsers, cursor, Math.floor(limit * 0.6)),
      // 大V账号：从收件箱读取（Push模式）
      this.pullFeedFromBigVInbox(userId, bigVUsers, cursor, Math.floor(limit * 0.4)),
    ]);

    // 合并结果
    const mergedPosts = this.mergeFeedsByTime(normalPosts, bigVInboxPosts);

    return {
      posts: mergedPosts.slice(0, limit),
      nextCursor: this.calculateNextCursor(mergedPosts, limit, cursor),
      source: `normal:${normalPosts.length},bigv:${bigVInboxPosts.length}`,
    };
  }

  /**
   * 获取关注列表及其粉丝统计
   */
  private async getFollowingsWithStats(userId: string): Promise<FollowingWithStats[]> {
    return this.postRepository.query(`
      SELECT
        u.id as userId,
        u.name as userName,
        u.avatar as userAvatar,
        COUNT(f.follower_id) as followerCount
      FROM user_follows uf
      JOIN users u ON u.id = uf.following_id
      LEFT JOIN user_follows f ON f.following_id = u.id
      WHERE uf.follower_id = ?
      GROUP BY u.id, u.name, u.avatar
    `, [userId]);
  }

  /**
   * 分类用户：大V和普通用户
   */
  private categorizeUsers(followings: FollowingWithStats[]): {
    normalUsers: FollowingWithStats[];
    bigVUsers: FollowingWithStats[];
  } {
    const normalUsers: FollowingWithStats[] = [];
    const bigVUsers: FollowingWithStats[] = [];

    for (const user of followings) {
      if (user.followerCount >= BIG_V_THRESHOLD) {
        bigVUsers.push(user);
      } else {
        normalUsers.push(user);
      }
    }

    return { normalUsers, bigVUsers };
  }

  /**
   * 从普通用户Pull模式获取Feed
   */
  private async pullFeedFromNormalUsers(
    users: FollowingWithStats[],
    cursor: string | null,
    limit: number,
  ): Promise<PostWithAuthor[]> {
    if (users.length === 0) {
      return [];
    }

    const userIds = users.map(u => u.userId);

    let queryBuilder = this.postRepository
      .createQueryBuilder('post')
      .innerJoinAndSelect('post.author', 'author')
      .where('post.authorId IN (:...userIds)', { userIds })
      .orderBy('post.createdAt', 'DESC')
      .take(limit);

    if (cursor) {
      const cursorTime = new Date(Buffer.from(cursor, 'base64').toString('utf-8'));
      queryBuilder = queryBuilder.andWhere('post.createdAt < :cursorTime', {
        cursorTime,
      });
    }

    return queryBuilder.getMany();
  }

  /**
   * 从大V收件箱获取Feed（Push模式）
   */
  private async pullFeedFromBigVInbox(
    viewerId: string,
    bigVUsers: FollowingWithStats[],
    cursor: string | null,
    limit: number,
  ): Promise<PostWithAuthor[]> {
    if (bigVUsers.length === 0) {
      return [];
    }

    // 获取所有大V的收件箱key
    const inboxKeys = bigVUsers.map(u => `inbox:${u.userId}`);

    // 从Redis批量获取大V的收件箱内容
    const posts = await this.redisService.getMultiInboxPosts(inboxKeys, cursor, limit);

    return posts.map(p => ({
      ...p,
      author: bigVUsers.find(u => u.userId === p.authorId),
    }));
  }

  /**
   * 按时间合并多个Feed源
   */
  private mergeFeedsByTime(
    feed1: PostWithAuthor[],
    feed2: PostWithAuthor[],
  ): PostWithAuthor[] {
    const merged = [...feed1, ...feed2];

    // 按时间降序排列
    merged.sort((a, b) =>
      new Date(b.createdAt).getTime() - new Date(a.createdAt).getTime()
    );

    return merged;
  }

  /**
   * 计算下一页游标
   */
  private calculateNextCursor(
    posts: PostWithAuthor[],
    limit: number,
    currentCursor: string | null,
  ): string | null {
    if (posts.length <= limit) {
      return null;
    }

    const lastPost = posts[limit - 1];
    return Buffer.from(lastPost.createdAt.toISOString()).toString('base64');
  }
```

### 2.3 分布式ID：雪花算法

在分布式系统中，生成唯一ID是基础能力。社交应用中的帖子ID、消息ID、用户ID都需要唯一且有序。雪花算法（Snowflake）是业界最常用的解决方案。

**雪花算法原理：**

```
┌─────────────────────────────────────────────────────────────────┐
│                        64位ID结构                                │
├──────────────┬────────────────────────┬─────────────┬──────────┤
│   时间戳(41位) │    机器ID(10位)        │  序列号(12位) │ 符号位(1) │
│              │                        │             │          │
│  2024-01-01   │    数据中心5 + 机器7    │    0-4095   │    0     │
│  毫秒级精度   │    共1024个节点         │  每毫秒递增  │  固定0   │
└──────────────┴────────────────────────┴─────────────┴──────────┘

优势：
- 趋势递增：适合作为数据库主键和分页游标
- 分布式友好：无需集中式ID生成服务
- 容量足够：每毫秒可生成4096个ID
- 时间戳可读：便于调试和日志分析
```

**雪花算法实现：**

```typescript
// 后端：雪花ID生成器
// utils/snowflake.ts

/**
 * 雪花算法ID生成器
 *
 * 设计考虑：
 * 1. 使用符号位固定为0，保证正数
 * 2. 时间戳使用相对时间，支持手动校准
 * 3. 机器ID由数据中心ID和机器ID组成，共10位
 * 4. 序列号在同毫秒内递增，支持每毫秒4096个ID
 */
export class SnowflakeIdGenerator {
  // 起始时间戳：2024-01-01 00:00:00 UTC
  private readonly EPOCH = 1704067200000n;

  // 各部分位数
  private readonly TIMESTAMP_BITS = 41n;
  private readonly DATACENTER_BITS = 5n;
  private readonly MACHINE_BITS = 5n;
  private readonly SEQUENCE_BITS = 12n;

  // 各部分最大值
  private readonly MAX_DATACENTER = (1n << this.DATACENTER_BITS) - 1n;
  private readonly MAX_MACHINE = (1n << this.MACHINE_BITS) - 1n;
  private readonly MAX_SEQUENCE = (1n << this.SEQUENCE_BITS) - 1n;

  // 各部分左移位数
  private readonly TIMESTAMP_SHIFT = this.SEQUENCE_BITS + this.MACHINE_BITS + this.DATACENTER_BITS;
  private readonly DATACENTER_SHIFT = this.SEQUENCE_BITS + this.MACHINE_BITS;
  private readonly MACHINE_SHIFT = this.SEQUENCE_BITS;

  // 当前状态
  private datacenterId: bigint;
  private machineId: bigint;
  private sequence: bigint = 0n;
  private lastTimestamp: bigint = 0n;
  private isFallbackToRedis: boolean = false;

  constructor(datacenterId: number, machineId: number) {
    // 校验参数合法性
    if (datacenterId < 0 || datacenterId > Number(this.MAX_DATACENTER)) {
      throw new Error(`数据中心ID必须在0到${this.MAX_DATACENTER}之间`);
    }
    if (machineId < 0 || machineId > Number(this.MAX_MACHINE)) {
      throw new Error(`机器ID必须在0到${this.MAX_MACHINE}之间`);
    }

    this.datacenterId = BigInt(datacenterId);
    this.machineId = BigInt(machineId);
  }

  /**
   * 生成下一个唯一ID
   *
   * @returns 64位唯一ID
   */
  nextId(): bigint {
    // 获取当前时间戳（毫秒）
    let timestamp = this.getCurrentTimestamp();

    // 处理时钟回拨问题
    if (timestamp < this.lastTimestamp) {
      console.warn(`检测到时钟回拨${this.lastTimestamp - timestamp}ms，采取补救措施`);

      // 等待回拨的时间差
      const waitTime = Number(this.lastTimestamp - timestamp);
      timestamp = this.waitForTimestamp(timestamp, waitTime);

      // 如果回拨过多，标记使用Redis备选方案
      if (timestamp < this.lastTimestamp) {
        this.isFallbackToRedis = true;
      }
    }

    // 同一毫秒内，序列号递增
    if (timestamp === this.lastTimestamp) {
      this.sequence = (this.sequence + 1n) & this.MAX_SEQUENCE;

      // 序列号溢出，等待下一毫秒
      if (this.sequence === 0n) {
        timestamp = this.waitForNextMillisecond(timestamp);
      }
    } else {
      // 新毫秒，重置序列号
      this.sequence = 0n;
    }

    this.lastTimestamp = timestamp;

    // 组装ID
    const id =
      ((timestamp - this.EPOCH) << this.TIMESTAMP_SHIFT) |
      (this.datacenterId << this.DATACENTER_SHIFT) |
      (this.machineId << this.MACHINE_SHIFT) |
      this.sequence;

    return id;
  }

  /**
   * 生成字符串形式的ID
   */
  nextIdString(): string {
    return this.nextId().toString();
  }

  /**
   * 解析ID的各部分
   * 用于调试和日志记录
   */
  parseId(id: bigint): {
    timestamp: Date;
    datacenterId: number;
    machineId: number;
    sequence: number;
    date: string;
  } {
    const idBigInt = BigInt(id);

    const timestamp =
      (idBigInt >> this.TIMESTAMP_SHIFT) + this.EPOCH;

    const datacenterId = Number(
      (idBigInt >> this.DATACENTER_SHIFT) & this.MAX_DATACENTER
    );

    const machineId = Number(
      (idBigInt >> this.MACHINE_SHIFT) & this.MAX_MACHINE
    );

    const sequence = Number(idBigInt & this.MAX_SEQUENCE);

    return {
      timestamp: new Date(Number(timestamp)),
      datacenterId,
      machineId,
      sequence,
      date: new Date(Number(timestamp)).toISOString(),
    };
  }

  /**
   * 获取当前时间戳
   * 包含容错处理，支持Node.js和浏览器环境
   */
  private getCurrentTimestamp(): bigint {
    return BigInt(Date.now());
  }

  /**
   * 等待到指定时间戳
   */
  private waitForTimestamp(current: bigint, waitMs: number): bigint {
    // 如果回拨时间小于5ms，等待回拨时间后继续
    if (waitMs < 5) {
      const waitUntil = current + BigInt(waitMs);
      while (this.getCurrentTimestamp() < waitUntil) {
        // 空循环等待
      }
      return waitUntil;
    }

    // 回拨超过5ms，返回lastTimestamp+1
    return this.lastTimestamp + 1n;
  }

  /**
   * 等待下一毫秒
   */
  private waitForNextMillisecond(current: bigint): bigint {
    let timestamp = this.getCurrentTimestamp();
    while (timestamp <= current) {
      timestamp = this.getCurrentTimestamp();
    }
    return timestamp;
  }
}

/**
 * 雪花ID生成器工厂
 * 支持按数据中心和机器ID创建实例
 */
export class SnowflakeFactory {
  private static instances: Map<string, SnowflakeIdGenerator> = new Map();

  /**
   * 获取指定数据中心和机器ID的生成器实例
   */
  static getInstance(datacenterId: number, machineId: number): SnowflakeIdGenerator {
    const key = `${datacenterId}:${machineId}`;

    if (!this.instances.has(key)) {
      this.instances.set(key, new SnowflakeIdGenerator(datacenterId, machineId));
    }

    return this.instances.get(key)!;
  }

  /**
   * 根据当前服务器信息自动创建生成器
   * 从环境变量或自动检测
   */
  static createAuto(): SnowflakeIdGenerator {
    // 尝试从环境变量读取
    const datacenterId = parseInt(process.env.SNOWFLAKE_DATACENTER_ID || '0', 10);
    const machineId = parseInt(process.env.SNOWFLAKE_MACHINE_ID || '0', 10);

    // 如果环境变量未配置，使用进程ID和机器MAC地址混合计算
    const finalDatacenterId = datacenterId || this.getProcessDatacenterId();
    const finalMachineId = machineId || this.getProcessMachineId();

    return this.getInstance(finalDatacenterId, finalMachineId);
  }

  private static getProcessDatacenterId(): number {
    // 使用进程ID取模，避免多进程冲突
    const pid = process.pid || 1;
    return pid % 32; // 5位数据中心ID，最大32
  }

  private static getProcessMachineId(): number {
    // 使用Node.js的cluster模块获取worker ID
    // 或者使用机器名的哈希值
    const os = require('os');
    const hostname = os.hostname();
    let hash = 0;
    for (let i = 0; i < hostname.length; i++) {
      hash = ((hash << 5) - hash) + hostname.charCodeAt(i);
      hash = hash & hash; // 转换为32位整数
    }
    return Math.abs(hash) % 32; // 5位机器ID，最大32
  }
}
```

```typescript
export class SnowflakeDecoder {
  private readonly EPOCH = 1704067200000; // 与后端一致的起始时间

  /**
   * 从雪花ID解析出时间戳
   */
  static parseTimestamp(id: string | number): Date {
    const idBigInt = BigInt(id);
    // 雪花ID结构：时间戳(41位) << 22 | 数据中心(5位) << 17 | 机器(5位) << 12 | 序列号(12位)
    const TIMESTAMP_BITS = 22n; // 序列号(12) + 机器(5) + 数据中心(5) = 22

    const timestamp = Number((idBigInt >> TIMESTAMP_BITS) + BigInt(this.EPOCH));
    return new Date(timestamp);
  }

  /**
   * 判断ID是否在给定时间之后生成
   */
  static isAfter(id: string | number, date: Date): boolean {
    const idTimestamp = this.parseTimestamp(id);
    return idTimestamp > date;
  }

  /**
   * 计算ID生成的大致时间（用于调试展示）
   */
  static getRelativeTime(id: string | number): string {
    const timestamp = this.parseTimestamp(id).getTime();
    const now = Date.now();
    const diff = now - timestamp;

    const seconds = Math.floor(diff / 1000);
    const minutes = Math.floor(seconds / 60);
    const hours = Math.floor(minutes / 60);
    const days = Math.floor(hours / 24);

    if (days > 0) return `${days}天前`;
    if (hours > 0) return `${hours}小时前`;
    if (minutes > 0) return `${minutes}分钟前`;
    if (seconds > 5) return `${seconds}秒前`;
    return '刚刚';
  }
```

### 2.4 分页：游标分页 vs 传统分页

**分页方式对比：**

| 方式 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **偏移量分页** | LIMIT offset, count | 简单直观，可跳页 | 数据量大时性能差，深分页问题 |
| **游标分页** | WHERE cursor > X | 性能稳定，无深分页问题 | 不能跳页，只能顺序翻页 |
| **双向游标** | 同时支持向前向后 | 灵活，可前后翻页 | 实现复杂 |

**游标分页实现：**

```typescript
// 后端：通用游标分页服务
// services/cursorPagination.ts

import { Injectable } from '@nestjs/common';

export interface CursorPaginationOptions<T> {
  /** 每页数量 */
  pageSize: number;
  /** 数据列表 */
  data: T[];
  /** 游标字段名 */
  cursorField: keyof T;
  /** 排序方向 */
  order: 'ASC' | 'DESC';
}

export interface PaginatedResult<T> {
  /** 数据列表 */
  items: T[];
  /** 下一页游标 */
  nextCursor: string | null;
  /** 上一页游标 */
  prevCursor: string | null;
  /** 是否有下一页 */
  hasNext: boolean;
  /** 是否有上一页 */
  hasPrev: boolean;
  /** 分页元信息 */
  meta: {
    pageSize: number;
    itemCount: number;
  };
}

@Injectable()
export class CursorPaginationService {
  /**
   * 通用游标分页方法
   *
   * @param options 分页选项
   * @param cursor 当前游标（Base64编码）
   */
  paginate<T>(
    options: CursorPaginationOptions<T>,
    cursor: string | null,
  ): PaginatedResult<T> {
    const { pageSize, data, cursorField, order } = options;

    // 解码游标获取当前位置
    let currentIndex = 0;

    if (cursor) {
      const decodedCursor = this.decodeCursor(cursor);
      const cursorValue = decodedCursor[cursorField as string];
      currentIndex = data.findIndex(
        item => String(item[cursorField]) === String(cursorValue)
      );

      // 如果找不到游标，从第一页开始
      if (currentIndex === -1) {
        currentIndex = 0;
      }
    }

    // 根据排序方向确定切片方向
    let startIndex: number;
    let endIndex: number;

    if (order === 'DESC') {
      // 倒序：游标表示"在这个之前的数据"
      startIndex = currentIndex;
      endIndex = Math.min(currentIndex + pageSize, data.length);
    } else {
      // 正序：游标表示"在这个之后的数据"
      startIndex = currentIndex;
      endIndex = Math.min(currentIndex + pageSize, data.length);
    }

    // 提取当前页数据
    const items = data.slice(startIndex, endIndex);

    // 计算游标
    const hasNext = endIndex < data.length;
    const hasPrev = currentIndex > 0;

    const nextCursor = hasNext
      ? this.encodeCursor(items[items.length - 1], cursorField)
      : null;

    const prevCursor = hasPrev
      ? this.encodeCursor(items[0], cursorField)
      : null;

    return {
      items,
      nextCursor,
      prevCursor,
      hasNext,
      hasPrev,
      meta: {
        pageSize,
        itemCount: items.length,
      },
    };
  }

  /**
   * 编码游标
   * 将游标数据编码为Base64字符串
   */
  private encodeCursor<T>(item: T, cursorField: keyof T): string {
    const cursorData = {
      [cursorField]: item[cursorField],
      _timestamp: Date.now(), // 添加时间戳防止游标复用
    };

    return Buffer.from(JSON.stringify(cursorData)).toString('base64');
  }

  /**
   * 解码游标
   */
  private decodeCursor(cursor: string): Record<string, any> {
    try {
      const decoded = Buffer.from(cursor, 'base64').toString('utf-8');
      return JSON.parse(decoded);
    } catch {
      throw new Error('无效的游标格式');
    }
  }

  /**
   * 创建基于时间的游标
   * 用于时间序分页（Feed流、消息列表等场景）
   */
  createTimeBasedCursor(timestamp: Date, id?: string): string {
    const cursorData = {
      t: timestamp.getTime(),
      id: id || '',
    };

    return Buffer.from(JSON.stringify(cursorData)).toString('base64');
  }

  /**
   * 解析时间游标
   */
  parseTimeBasedCursor(cursor: string): { timestamp: Date; id: string | null } {
    const decoded = this.decodeCursor(cursor);

    return {
      timestamp: new Date(decoded.t),
      id: decoded.id || null,
    };
  }
}
```

```tsx
    } catch (err) {
      setError('刷新失败，请检查网络');
    } finally {
      setIsRefreshing(false);
    }
  }, [refreshFeed, type]);

  /**
   * 重新加载（用于错误重试）
   */
  const handleRetry = useCallback(() => {
    setError(null);
    refreshFeed({ type, limit: 20 });
  }, [refreshFeed, type]);

  // 渲染加载状态
  if (isLoading && posts.length === 0) {
    return (
      <div className={styles.feedContainer}>
        <FeedSkeleton count={5} />
      </div>
    );
  }

  // 渲染错误状态
  if (error && posts.length === 0) {
    return (
      <div className={styles.errorContainer}>
        <div className={styles.errorContent}>
          <span className={styles.errorIcon}>⚠️</span>
          <p className={styles.errorMessage}>{error}</p>
          <button onClick={handleRetry} className={styles.retryButton}>
            重试
          </button>
        </div>
      </div>
    );
  }

  // 渲染空状态
  if (!isLoading && posts.length === 0) {
    return (
      <div className={styles.emptyContainer}>
        <div className={styles.emptyContent}>
          <span className={styles.emptyIcon}>📭</span>
          <p className={styles.emptyTitle}>暂无内容</p>
          <p className={styles.emptyDescription}>
            {type === 'following'
              ? '关注更多创作者，发现更多精彩内容'
              : '暂无推荐内容，请稍后再试'}
          </p>
        </div>
      </div>
    );
  }

  return (
    <div className={styles.feedContainer}>
      {/* 下拉刷新指示器 */}
      {isRefreshing && (
        <div className={styles.refreshIndicator}>
          <div className={styles.refreshSpinner} />
          <span>刷新中...</span>
        </div>
      )}

      {/* Feed列表 */}
      <div className={styles.feedList}>
        {posts.map((post, index) => (
          <FeedItem
            key={post.id}
            post={post}
            index={index}
            onVisible={() => {
              // 曝光统计算法
              console.log(`Post ${post.id} visible at index ${index}`);
            }}
          />
        ))}
      </div>

      {/* 加载更多指示器 */}
      {hasMore && (
        <div ref={scrollContainerRef} className={styles.loadMoreContainer}>
          {isLoadingMore ? (
            <div className={styles.loadingMore}>
              <div className={styles.loadingSpinner} />
              <span>加载中...</span>
            </div>
          ) : (
            <div className={styles.loadMoreHint}>上滑加载更多</div>
          )}
        </div>
      )}

      {/* 没有更多内容 */}
      {!hasMore && posts.length > 0 && (
        <div className={styles.noMoreContent}>
          <span>— 已加载全部内容 —</span>
        </div>
      )}

      {/* 错误提示（底部） */}
      {error && posts.length > 0 && (
        <div className={styles.bottomError}>
          <button onClick={handleRetry}>{error}，点击重试</button>
        </div>
      )}
    </div>
  );
};
```

### 3.1 关注取关：异步化处理

关注/取关操作看似简单，但在高并发场景下需要考虑性能优化。将同步操作异步化是常见策略。

**异步关注服务：**

```typescript
// 后端：异步关注服务
// services/followAsyncService.ts

import { Injectable, OnModuleInit } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { RedisService } from './redis.service';
import { Follow } from '../entities/follow.entity';
import { User } from '../entities/user.entity';

@Injectable()
export class FollowAsyncService {
  // 关注操作的Redis队列key
  private readonly FOLLOW_QUEUE = 'follow:async:queue';
  // 批量处理的窗口期（毫秒）
  private readonly BATCH_WINDOW = 100;
  // 批量处理的大小阈值
  private readonly BATCH_SIZE = 100;

  constructor(
    @InjectRepository(Follow)
    private readonly followRepository: Repository<Follow>,
    @InjectRepository(User)
    private readonly userRepository: Repository<User>,
    private readonly redisService: RedisService,
  ) {}

  /**
   * 异步关注操作
   * 将关注请求放入队列，异步处理
   *
   * @param followerId 关注者ID
   * @param followingId 被关注者ID
   */
  async followAsync(followerId: string, followingId: string): Promise<void> {
    // 第一步：立即更新Redis缓存（快速响应）
    await this.updateFollowCache(followerId, followingId, 'follow');

    // 第二步：写入异步队列
    await this.redisService.lpush(
      this.FOLLOW_QUEUE,
      JSON.stringify({
        type: 'follow',
        followerId,
        followingId,
        timestamp: Date.now(),
      })
    );

    // 第三步：触发关注计数的乐观更新
    await this.incrementFollowCount(followerId, followingId);
  }

  /**
   * 异步取消关注操作
   */
  async unfollowAsync(followerId: string, followingId: string): Promise<void> {
    // 立即更新Redis缓存
    await this.updateFollowCache(followerId, followingId, 'unfollow');

    // 写入异步队列
    await this.redisService.lpush(
      this.FOLLOW_QUEUE,
      JSON.stringify({
        type: 'unfollow',
        followerId,
        followingId,
        timestamp: Date.now(),
      })
    );

    // 触发关注计数的乐观更新
    await this.decrementFollowCount(followerId, followingId);
  }

  /**
   * 批量处理关注队列
   * 由定时任务调用
   */
  async processFollowQueue(): Promise<{ success: number; failed: number }> {
    let success = 0;
    let failed = 0;

    // 批量取出队列中的任务
    const tasks = await this.redisService.rpopMultiple(
      this.FOLLOW_QUEUE,
      this.BATCH_SIZE
    );

    if (tasks.length === 0) {
      return { success, failed };
    }

    // 按类型分组
    const followTasks = tasks.filter(t => t.type === 'follow');
    const unfollowTasks = tasks.filter(t => t.type === 'unfollow');

    // 批量处理关注
    if (followTasks.length > 0) {
      const results = await this.batchCreateFollows(followTasks);
      success += results.success;
      failed += results.failed;
    }

    // 批量处理取消关注
    if (unfollowTasks.length > 0) {
      const results = await this.batchDeleteFollows(unfollowTasks);
      success += results.success;
      failed += results.failed;
    }

    return { success, failed };
  }

  /**
   * 批量创建关注关系
   */
  private async batchCreateFollows(
    tasks: Array<{ followerId: string; followingId: string; timestamp: number }>
  ): Promise<{ success: number; failed: number }> {
    const queryRunner = this.followRepository.manager.connection.createQueryRunner();
    await queryRunner.connect();
    await queryRunner.startTransaction();

    let success = 0;
    let failed = 0;

    try {
      for (const task of tasks) {
        // 检查是否已存在关注关系
        const existing = await queryRunner.manager.findOne(Follow, {
          where: {
            followerId: task.followerId,
            followingId: task.followingId,
          },
        });

        if (existing) {
          failed++;
          continue;
        }

        // 创建关注关系
        const follow = queryRunner.manager.create(Follow, {
          followerId: task.followerId,
          followingId: task.followingId,
          createdAt: new Date(task.timestamp),
        });

        await queryRunner.manager.save(follow);
        success++;
      }

      await queryRunner.commitTransaction();
    } catch (error) {
      await queryRunner.rollbackTransaction();
      console.error('批量创建关注关系失败:', error);
      failed = tasks.length;
      success = 0;
    } finally {
      await queryRunner.release();
    }

    return { success, failed };
  }

  /**
   * 批量删除关注关系
   */
  private async batchDeleteFollows(
    tasks: Array<{ followerId: string; followingId: string; timestamp: number }>
  ): Promise<{ success: number; failed: number }> {
    const queryRunner = this.followRepository.manager.connection.createQueryRunner();
    await queryRunner.connect();
    await queryRunner.startTransaction();

    let success = 0;
    let failed = 0;

    try {
      for (const task of tasks) {
        const result = await queryRunner.manager.delete(Follow, {
          followerId: task.followerId,
          followingId: task.followingId,
        });

        if (result.affected && result.affected > 0) {
          success++;
        } else {
          failed++;
        }
      }

      await queryRunner.commitTransaction();
    } catch (error) {
      await queryRunner.rollbackTransaction();
      console.error('批量删除关注关系失败:', error);
      failed = tasks.length;
      success = 0;
    } finally {
      await queryRunner.release();
    }

    return { success, failed };
  }

  /**
   * 更新关注缓存
   */
  private async updateFollowCache(
    followerId: string,
    followingId: string,
    action: 'follow' | 'unfollow'
  ): Promise<void> {
    const cacheKey = `user:${followerId}:following`;
    const timestamp = Date.now();

    if (action === 'follow') {
      // 添加到关注列表缓存
      await this.redisService.hset(cacheKey, followingId, timestamp.toString());
    } else {
      // 从关注列表缓存中移除
      await this.redisService.hdel(cacheKey, followingId);
    }
  }

  /**
   * 更新关注计数
   */
  private async incrementFollowCount(
    followerId: string,
    followingId: string
  ): Promise<void> {
    // 增加被关注者的粉丝数
    await this.redisService.hincrby(`user:${followingId}`, 'followerCount', 1);
    // 增加关注者的关注数
    await this.redisService.hincrby(`user:${followerId}`, 'followingCount', 1);
  }

  private async decrementFollowCount(
    followerId: string,
    followingId: string
  ): Promise<void> {
    await this.redisService.hincrby(`user:${followingId}`, 'followerCount', -1);
    await this.redisService.hincrby(`user:${followerId}`, 'followingCount', -1);
  }
}
```

### 3.2 好友关系：双向关注

好友关系是双向的，关注关系是单向的。好友需要双方互相关注才能建立。

```typescript
// 后端：好友关系服务

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { Follow } from '../entities/follow.entity';

  constructor(
    @InjectRepository(Follow)
    private readonly followRepository: Repository<Follow>,
  ) {}

   * 检查是否为好友关系（双向关注）
   */
  async isFriends(userId1: string, userId2: string): Promise<boolean> {
    // 查询两条关注记录
    const [follow1, follow2] = await Promise.all([
      this.followRepository.findOne({
        where: { followerId: userId1, followingId: userId2 },
      }),
      this.followRepository.findOne({
        where: { followerId: userId2, followingId: userId1 },
      }),
    ]);

    // 双方都关注对方才是好友
    return !!follow1 && !!follow2;

   */

  ): Promise<{
    isFollowing: boolean;   // 我是否关注了他
    isFollowed: boolean;   // 他是否关注了我
    isFriends: boolean;    // 是否是好友（双向关注）
  }> {
    const [follow1, follow2] = await Promise.all([
      this.followRepository.findOne({
        where: { followerId: currentUserId, followingId: targetUserId },
      }),
      this.followRepository.findOne({
        where: { followerId: targetUserId, followingId: currentUserId },
      }),
    ]);

    return {
      isFollowing: !!follow1,
      isFollowed: !!follow2,
```

### 3.3 关系存储：邻接表 vs 逆邻接表

**关系存储模型对比：**

| 模型 | 结构 | 查询关注列表 | 查询粉丝列表 | 存储空间 |
|------|------|-------------|-------------|----------|
| **邻接表** | (follower_id, following_id) | 查我关注的 | 查关注我的 | O(E) |
| **逆邻接表** | (user_id, follower_ids[]) | 查我关注的 | 直接读取 | O(V) |
| **混合存储** | 邻接表 + Redis缓存 | 缓存加速 | 缓存加速 | O(E) + O(V) |

**关系存储完整实现：**

```typescript
// 后端：关系存储服务
// services/relationshipStorageService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { RedisService } from './redis.service';
import { Follow } from '../entities/follow.entity';

/**
 * 关系存储优化策略：
 *
 * 1. MySQL邻接表：存储完整的关系数据，支持复杂查询
 * 2. Redis缓存：
 *    - following:{userId} - Set结构，存储我关注的人的ID
 *    - followers:{userId} - Set结构，存储关注我的人的ID
 * 3. 读写分离：
 *    - 写入：先写MySQL，再更新Redis
 *    - 读取：优先读Redis，Redis没有则读MySQL并回填
 */
@Injectable()
export class RelationshipStorageService {
  // Redis缓存key前缀
  private readonly FOLLOWING_KEY = 'following:';
  private readonly FOLLOWERS_KEY = 'followers:';
  // 缓存过期时间（秒）
  private readonly CACHE_TTL = 7 * 24 * 60 * 60; // 7天

  constructor(
    @InjectRepository(Follow)
    private readonly followRepository: Repository<Follow>,
    private readonly redisService: RedisService,
  ) {}

  /**
   * 创建关注关系
   * 写入MySQL并更新Redis缓存
   */
  async createFollow(followerId: string, followingId: string): Promise<void> {
    // 第一步：写入MySQL
    const follow = this.followRepository.create({
      followerId,
      followingId,
      createdAt: new Date(),
    });

    await this.followRepository.save(follow);

    // 第二步：更新Redis缓存
    await Promise.all([
      // 添加到我的关注列表
      this.redisService.sadd(`${this.FOLLOWING_KEY}${followerId}`, followingId),
      // 添加到他的粉丝列表
      this.redisService.sadd(`${this.FOLLOWERS_KEY}${followingId}`, followerId),
      // 更新计数
      this.redisService.hincrby(`user:${followerId}`, 'followingCount', 1),
      this.redisService.hincrby(`user:${followingId}`, 'followerCount', 1),
    ]);
  }

  /**
   * 删除关注关系
   */
  async deleteFollow(followerId: string, followingId: string): Promise<void> {
    // 第一步：删除MySQL记录
    await this.followRepository.delete({ followerId, followingId });

    // 第二步：更新Redis缓存
    await Promise.all([
      this.redisService.srem(`${this.FOLLOWING_KEY}${followerId}`, followingId),
      this.redisService.srem(`${this.FOLLOWERS_KEY}${followingId}`, followerId),
      this.redisService.hincrby(`user:${followerId}`, 'followingCount', -1),
      this.redisService.hincrby(`user:${followingId}`, 'followerCount', -1),
    ]);
  }

  /**
   * 获取用户的关注列表（带缓存）
   */
  async getFollowing(
    userId: string,
    cursor: string | null = null,
    limit: number = 20
  ): Promise<{ userIds: string[]; nextCursor: string | null }> {
    const cacheKey = `${this.FOLLOWING_KEY}${userId}`;

    // 尝试从Redis获取
    let followingSet = await this.redisService.smembers(cacheKey);

    // 缓存不存在，从MySQL加载并回填
    if (followingSet.length === 0) {
      const dbFollowing = await this.followRepository.find({
        where: { followerId: userId },
        order: { createdAt: 'DESC' },
      });

      if (dbFollowing.length > 0) {
        // 回填Redis
        const ids = dbFollowing.map(f => f.followingId);
        await this.redisService.sadd(cacheKey, ...ids);
        await this.redisService.expire(cacheKey, this.CACHE_TTL);
        followingSet = ids;
      }
    }

    // 应用游标分页
    let startIndex = 0;
    if (cursor) {
      const index = followingSet.indexOf(cursor);
      if (index !== -1) {
        startIndex = index + 1;
      }
    }

    const paginatedIds = followingSet.slice(startIndex, startIndex + limit);
    const hasMore = startIndex + limit < followingSet.length;
    const nextCursor = hasMore ? paginatedIds[paginatedIds.length - 1] : null;

    return { userIds: paginatedIds, nextCursor };
  }

  /**
   * 获取用户的粉丝列表（带缓存）
   */
  async getFollowers(
    userId: string,
    cursor: string | null = null,
    limit: number = 20
  ): Promise<{ userIds: string[]; nextCursor: string | null }> {
    const cacheKey = `${this.FOLLOWERS_KEY}${userId}`;

    let followersSet = await this.redisService.smembers(cacheKey);

    if (followersSet.length === 0) {
      const dbFollowers = await this.followRepository.find({
        where: { followingId: userId },
        order: { createdAt: 'DESC' },
      });

      if (dbFollowers.length > 0) {
        const ids = dbFollowers.map(f => f.followerId);
        await this.redisService.sadd(cacheKey, ...ids);
        await this.redisService.expire(cacheKey, this.CACHE_TTL);
        followersSet = ids;
      }
    }

    let startIndex = 0;
    if (cursor) {
      const index = followersSet.indexOf(cursor);
      if (index !== -1) {
        startIndex = index + 1;
      }
    }

    const paginatedIds = followersSet.slice(startIndex, startIndex + limit);
    const hasMore = startIndex + limit < followersSet.length;
    const nextCursor = hasMore ? paginatedIds[paginatedIds.length - 1] : null;

    return { userIds: paginatedIds, nextCursor };
  }

  /**
   * 检查关注状态
   */
  async isFollowing(followerId: string, followingId: string): Promise<boolean> {
    // 优先从Redis检查
    const cached = await this.redisService.sismember(
      `${this.FOLLOWING_KEY}${followerId}`,
      followingId
    );

    if (cached !== null) {
      return cached;
    }

    // 缓存未命中，查询数据库
    const follow = await this.followRepository.findOne({
      where: { followerId, followingId },
    });

    return !!follow;
  }

  /**
   * 批量获取关注状态
   * 用于显示用户列表的关注按钮状态
   */
  async batchIsFollowing(
    viewerId: string,
    targetIds: string[]
  ): Promise<Map<string, boolean>> {
    if (targetIds.length === 0) {
      return new Map();
    }

    // 批量查询Redis
    const pipeline = this.redisService.getPipeline();
    for (const targetId of targetIds) {
      pipeline.sismember(`${this.FOLLOWING_KEY}${viewerId}`, targetId);
    }
    const results = await pipeline.exec();

    const statusMap = new Map<string, boolean>();
    for (let i = 0; i < targetIds.length; i++) {
      const [err, isFollowing] = results[i];
      if (!err && isFollowing !== null) {
        statusMap.set(targetIds[i], !!isFollowing);
      }
    }

    // 缺失的key从数据库查询
    const missingIds = targetIds.filter(id => !statusMap.has(id));
    if (missingIds.length > 0) {
      const dbResults = await this.followRepository.find({
        where: missingIds.map(id => ({ followerId: viewerId, followingId: id })),
      });

      for (const id of missingIds) {
        statusMap.set(id, false);
      }

      for (const follow of dbResults) {
        statusMap.set(follow.followingId, true);
      }
    }

    return statusMap;
  }
}
```

### 3.4 推荐系统基础：协同过滤

**协同过滤原理：**

```
用户相似度计算：
┌─────────────────────────────────────────────────────────┐
│                                                          │
│   用户A的兴趣： [商品1, 商品3, 商品5]                     │
│   用户B的兴趣： [商品1, 商品3, 商品7, 商品9]             │
│   用户C的兴趣： [商品2, 商品4, 商品6]                    │
│                                                          │
│   A和B的相似度： Jaccard = 2/6 = 0.33  (共同商品多)      │
│   A和C的相似度： Jaccard = 0/6 = 0      (无共同商品)      │
│                                                          │
│   结论：向用户A推荐时，更应该推荐B喜欢的商品              │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**协同过滤推荐实现：**

```typescript
// 后端：协同过滤推荐服务
// services/collaborativeFilteringService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { Post } from '../entities/post.entity';
import { Like } from '../entities/like.entity';
import { Follow } from '../entities/follow.entity';

@Injectable()
export class CollaborativeFilteringService {
  constructor(
    @InjectRepository(Post)
    private readonly postRepository: Repository<Post>,
    @InjectRepository(Like)
    private readonly likeRepository: Repository<Like>,
    @InjectRepository(Follow)
    private readonly followRepository: Repository<Follow>,
  ) {}

  /**
   * 基于用户的协同过滤推荐
   *
   * 核心思想：
   * 1. 找到与目标用户兴趣相似的用户（相似用户）
   * 2. 将相似用户喜欢但目标用户未看过的内容推荐给目标用户
   */
  async getRecommendedForUser(
    userId: string,
    limit: number = 20
  ): Promise<RecommendedPost[]> {
    // 第一步：获取目标用户的行为序列（点赞的帖子）
    const userLikes = await this.getUserLikeSet(userId);

    // 第二步：找到相似用户
    const similarUsers = await this.findSimilarUsers(userId, userLikes);

    if (similarUsers.length === 0) {
      // 没有相似用户，返回热门内容
      return this.getHotContent(limit);
    }

    // 第三步：获取相似用户喜欢的内容
    const recommendedPosts = await this.getPostsLikedByUsers(similarUsers, userLikes);

    // 第四步：计算推荐分数并排序
    const scoredPosts = this.calculateRecommendationScore(
      recommendedPosts,
      similarUsers
    );

    // 第五步：返回top N
    return scoredPosts.slice(0, limit);
  }

  /**
   * 获取用户点赞的帖子ID集合
   */
  private async getUserLikeSet(userId: string): Promise<Set<string>> {
    const likes = await this.likeRepository.find({
      where: { userId },
      select: ['postId'],
    });

    return new Set(likes.map(l => l.postId));
  }

  /**
   * 找到与目标用户相似的K个用户
   * 使用Jaccard相似度计算
   */
  private async findSimilarUsers(
    userId: string,
    userLikeSet: Set<string>,
    topK: number = 50
  ): Promise<Array<{ userId: string; similarity: number }>> {
    // 获取目标用户关注的人的ID列表
    const following = await this.followRepository.find({
      where: { followerId: userId },
      select: ['followingId'],
    });

    if (following.length === 0) {
      return [];
    }

    const followingIds = following.map(f => f.followingId);

    // 获取这些用户的点赞列表
    const likes = await this.likeRepository
      .createQueryBuilder('like')
      .where('like.userId IN (:...followingIds)', { followingIds })
      .andWhere('like.postId NOT IN (:...excludePosts)', {
        excludePosts: Array.from(userLikeSet),
      })
      .select(['like.userId', 'like.postId'])
      .getMany();

    // 按用户聚合点赞
    const userLikesMap = new Map<string, Set<string>>();
    for (const like of likes) {
      if (!userLikesMap.has(like.userId)) {
        userLikesMap.set(like.userId, new Set());
      }
      userLikesMap.get(like.userId)!.add(like.postId);
    }

    // 计算每个用户的相似度
    const similarities: Array<{ userId: string; similarity: number }> = [];

    for (const [otherUserId, otherLikeSet] of userLikesMap) {
      // 计算Jaccard相似度：交集/并集
      const intersection = this.setIntersection(userLikeSet, otherLikeSet);
      const union = this.setUnion(userLikeSet, otherLikeSet);

      const similarity =
        union.size > 0 ? intersection.size / union.size : 0;

      if (similarity > 0) {
        similarities.push({ userId: otherUserId, similarity });
      }
    }

    // 按相似度降序排列，返回topK
    similarities.sort((a, b) => b.similarity - a.similarity);

    return similarities.slice(0, topK);
  }

  /**
   * 获取指定用户群体喜欢的帖子
   */
  private async getPostsLikedByUsers(
    users: Array<{ userId: string; similarity: number }>,
    excludePostIds: Set<string>
  ): Promise<Array<{ postId: string; userId: string; similarity: number }>> {
    const userIds = users.map(u => u.userId);
    const userSimilarityMap = new Map(users.map(u => [u.userId, u.similarity]));

    // 获取这些用户最近7天点赞的帖子
    const sevenDaysAgo = new Date();
    sevenDaysAgo.setDate(sevenDaysAgo.getDate() - 7);

    const likes = await this.likeRepository
      .createQueryBuilder('like')
      .where('like.userId IN (:...userIds)', { userIds })
      .andWhere('like.createdAt > :sevenDaysAgo', { sevenDaysAgo })
      .andWhere('like.postId NOT IN (:...excludePostIds)', {
        excludePostIds: Array.from(excludePostIds),
      })
      .select(['like.postId', 'like.userId'])
      .getMany();

    // 添加相似度权重
    return likes.map(like => ({
      postId: like.postId,
      userId: like.userId,
      similarity: userSimilarityMap.get(like.userId) || 0,
    }));
  }

  /**
   * 计算推荐分数
   * 分数 = Σ(用户相似度 × 互动时间衰减)
   */
  private calculateRecommendationScore(
    posts: Array<{ postId: string; userId: string; similarity: number }>,
    _users: Array<{ userId: string; similarity: number }>
  ): RecommendedPost[] {
    // 按帖子聚合分数
    const scoreMap = new Map<string, number>();
    const userMap = new Map<string, { similarity: number; userId: string }>();

    for (const post of posts) {
      // 计算带时间衰减的分数
      const timeDecay = 1.0; // 可根据互动时间计算衰减
      const score = post.similarity * timeDecay;

      const currentScore = scoreMap.get(post.postId) || 0;
      scoreMap.set(post.postId, currentScore + score);

      // 记录推荐来源（相似度最高的用户）
      const existing = userMap.get(post.postId);
      if (!existing || post.similarity > existing.similarity) {
        userMap.set(post.postId, {
          similarity: post.similarity,
          userId: post.userId,
        });
      }
    }

    // 转换为推荐帖子列表
    const recommendedPosts: RecommendedPost[] = [];

    for (const [postId, score] of scoreMap) {
      const source = userMap.get(postId)!;
      recommendedPosts.push({
        postId,
        score,
        sourceUserId: source.userId,
        reason: `因为你关注了@${source.userId}，他们喜欢这个内容`,
      });
    }

    // 按分数降序排列
    recommendedPosts.sort((a, b) => b.score - a.score);

    return recommendedPosts;
  }

  /**
   * 获取热门内容（fallback方案）
   */
  private async getHotContent(limit: number): Promise<RecommendedPost[]> {
    const hotPosts = await this.postRepository
      .createQueryBuilder('post')
      .orderBy('post.likeCount + post.commentCount * 2 + post.repostCount * 3', 'DESC')
      .take(limit)
      .getMany();

    return hotPosts.map(post => ({
      postId: post.id,
      score: post.likeCount + post.commentCount * 2 + post.repostCount * 3,
      sourceUserId: '',
      reason: '热门内容推荐',
    }));
  }

  private setIntersection<T>(setA: Set<T>, setB: Set<T>): Set<T> {
    const result = new Set<T>();
    for (const item of setB) {
      if (setA.has(item)) {
        result.add(item);
      }
    }
    return result;
  }

  private setUnion<T>(setA: Set<T>, setB: Set<T>): Set<T> {
    const result = new Set<T>(setA);
    for (const item of setB) {
      result.add(item);
    }
    return result;
  }
}

interface RecommendedPost {
  postId: string;
  score: number;
  sourceUserId: string;
  reason: string;
}
```

### 3.5 我的思考：关系链的设计

**关系链设计的三个层次：**

```
┌─────────────────────────────────────────────────────────────────┐
│                      关系链设计三层模型                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  第一层：数据存储层                                                │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  MySQL邻接表 │ Redis缓存 │ 消息队列                        │    │
│  │  完整关系数据   高性能缓存   异步一致性保证                    │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  第二层：服务逻辑层                                                │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  关注服务 │ 好友服务 │ 推荐服务 │ 通知服务                    │    │
│  │  核心CRUD   双向判断   协同过滤   触达推送                    │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  第三层：业务应用层                                                │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  Feed分发 │ 私信范围 │ @提及 │ 排行榜                      │    │
│  │  内容过滤   接收人过滤  高亮提醒  热度计算                    │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**关系链设计的核心问题：**

1. **一致性问题**：强一致 vs 最终一致
   - 关注操作需要立即生效（强一致）
   - 推荐列表可以接受延迟（最终一致）

2. **性能问题**：海量关系下的读写
   - 大V账号的粉丝列表可能超过千万
   - 需要分层缓存和异步处理

3. **扩展性问题**：从小到大的演进
   - 单机 → 分库分表 → 分布式数据库
   - 关系存储需要支持平滑扩容

> 经验之谈：在设计关系链系统时，一定要预留足够的扩展空间。一旦用户规模突破预期，扩容成本会非常高。建议在用户量达到百万级别时就考虑分库分表方案。

### 4.1 文字限制：为什么要限制

文字限制是社交平台的基础设计，主要考虑：

| 限制因素 | 说明 | 示例 |
|----------|------|------|
| **产品定位** | 短内容强调即时性 | Twitter 280字、微博 2000字 |
| **存储成本** | 每条内容都是存储成本 | 亿级用户×2000字=海量存储 |
| **阅读体验** | 长文影响滑动体验 | 碎片化消费场景 |
| **互动率** | 短内容更易产生互动 | 研究表明短内容互动率高 |

```typescript
// 前端：文字输入组件
// components/Composer/TextComposer.tsx

'use client';

import React, { useState, useCallback, useRef, useEffect } from 'react';
import { useTextLimit } from '@/hooks/useTextLimit';
import styles from './TextComposer.module.css';

interface TextComposerProps {
  /** 最大字数限制 */
  maxLength?: number;
  /** 初始内容 */
  initialContent?: string;
  /** 占位符 */
  placeholder?: string;
  /** 内容变化回调 */
  onChange?: (content: string) => void;
  /** 发布回调 */
  onSubmit?: (content: string) => Promise<void>;
}

/**
 * 文本输入组件
 *
 * 功能：
 * 1. 实时字数统计与限制
 * 2. 超限警告（颜色变化）
 * 3. 草稿自动保存
 * 4. 键盘快捷键（Ctrl+Enter发送）
 */
export const TextComposer: React.FC<TextComposerProps> = ({
  maxLength = 2000,
  initialContent = '',
  placeholder = '有什么新鲜事？',
  onChange,
  onSubmit,
}) => {
  // 状态
  const [content, setContent] = useState(initialContent);
  const [isComposing, setIsComposing] = useState(false); // 中文输入中
  const [isSubmitting, setIsSubmitting] = useState(false);
  const [showTip, setShowTip] = useState(false);

  // 引用
  const textareaRef = useRef<HTMLTextAreaElement>(null);

  // 字数统计
  const { currentLength, isOverLimit, remainingChars, percentUsed } =
    useTextLimit(content, maxLength);

  // 自动调整高度
  useEffect(() => {
    const textarea = textareaRef.current;
    if (textarea) {
      textarea.style.height = 'auto';
      textarea.style.height = `${Math.min(textarea.scrollHeight, 300)}px`;
    }
  }, [content]);

  // 草稿保存
  useEffect(() => {
    const draftKey = `draft:${window.location.pathname}`;
    const timer = setTimeout(() => {
      if (content.trim()) {
        localStorage.setItem(draftKey, content);
      }
    }, 1000);

    return () => clearTimeout(timer);
  }, [content]);

  // 加载草稿
  useEffect(() => {
    const draftKey = `draft:${window.location.pathname}`;
    const savedDraft = localStorage.getItem(draftKey);
    if (savedDraft && !initialContent) {
      setContent(savedDraft);
    }
  }, []);

  /**
   * 内容变化处理
   */
  const handleChange = useCallback(
    (e: React.ChangeEvent<HTMLTextAreaElement>) => {
      const newContent = e.target.value;
      setContent(newContent);
      onChange?.(newContent);
    },
    [onChange]
  );

  /**
   * 键盘事件处理
   */
  const handleKeyDown = useCallback(
    (e: React.KeyboardEvent<HTMLTextAreaElement>) => {
      // Ctrl+Enter 或 Cmd+Enter 发送
      if ((e.ctrlKey || e.metaKey) && e.key === 'Enter') {
        e.preventDefault();
        if (!isOverLimit && !isSubmitting && content.trim()) {
          handleSubmit();
        }
      }

      // Escape 取消输入
      if (e.key === 'Escape') {
        if (content.trim()) {
          if (confirm('确定要放弃编辑吗？')) {
            setContent('');
            localStorage.removeItem(`draft:${window.location.pathname}`);
          }
        }
      }
    },
    [content, isOverLimit, isSubmitting]
  );

  /**
   * 提交处理
   */
  const handleSubmit = async () => {
    if (!content.trim() || isOverLimit || isSubmitting) {
      return;
    }

    setIsSubmitting(true);

    try {
      await onSubmit?.(content);
      setContent('');
      localStorage.removeItem(`draft:${window.location.pathname}`);
    } catch (error) {
      console.error('发布失败:', error);
      alert('发布失败，请重试');
    } finally {
      setIsSubmitting(false);
    }
  };

  // 计算显示状态
  const getCounterClass = () => {
    if (isOverLimit) return styles.counterError;
    if (remainingChars < 100) return styles.counterWarning;
    return styles.counterNormal;
  };

  return (
    <div className={styles.composer}>
      {/* 输入区域 */}
      <textarea
        ref={textareaRef}
        className={styles.textarea}
        value={content}
        onChange={handleChange}
        onKeyDown={handleKeyDown}
        onCompositionStart={() => setIsComposing(true)}
        onCompositionEnd={() => setIsComposing(false)}
        placeholder={placeholder}
        disabled={isSubmitting}
      />

      {/* 底部工具栏 */}
      <div className={styles.toolbar}>
        {/* 字数统计 */}
        <div className={`${styles.counter} ${getCounterClass()}`}>
          {isOverLimit ? (
            <span>超出 {Math.abs(remainingChars)} 字</span>
          ) : (
            <span>{remainingChars} 字可用</span>
          )}
          <div
            className={styles.progressBar}
            style={{ width: `${Math.min(percentUsed * 100, 100)}%` }}
          />
        </div>

        {/* 操作按钮 */}
        <div className={styles.actions}>
          {showTip && (
            <span className={styles.shortcutTip}>Ctrl+Enter 发送</span>
          )}

          <button
            type="button"
            className={styles.draftButton}
            onClick={() => {
              setShowTip(!showTip);
            }}
          >
            草稿
```

### 4.2 图片上传：九宫格布局

图片是社交内容的重要组成部分。合理的九宫格布局能够：

1. **视觉美观**：对称的网格布局更吸引人
2. **信息密度**：一张图到九张图有不同的展示方式
3. **性能优化**：根据展示位置加载不同尺寸的图片

```tsx
// 前端：九宫格图片组件
// components/ImageGrid/ImageGrid.tsx

'use client';

import React, { memo, useState, useCallback } from 'react';
import Image from 'next/image';
import styles from './ImageGrid.module.css';

interface ImageGridProps {
  /** 图片列表 */
  images: Array<{
    id: string;
    url: string;
    thumbnailUrl?: string;
    width?: number;
    height?: number;
    aspectRatio?: number;
  }>;
  /** 布局类型 */
  layout?: 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9;
  /** 点击回调 */
  onImageClick?: (index: number) => void;
}

/**
 * 九宫格图片组件
 *
 * 布局规则：
 * - 1张图：宽度100%，支持点击放大
 * - 2张图：左右各50%
 * - 3张图：左1右2 或 上1下2
 * - 4张图：2×2网格
 * - 5-6张图：2列布局
 * - 7-9张图：3列网格
 */
export const ImageGrid: React.FC<ImageGridProps> = memo(
  ({ images, layout, onImageClick }) => {
    const [loadedImages, setLoadedImages] = useState<Set<number>>(new Set());

    // 根据图片数量确定布局
    const actualLayout = layout || (images.length <= 1 ? 1 : images.length <= 2 ? 2 : images.length <= 4 ? 4 : images.length <= 6 ? 6 : 9);

    /**
     * 获取网格类名
     */
    const getGridClass = () => {
      switch (actualLayout) {
        case 1: return styles.grid1;
        case 2: return styles.grid2;
        case 3: return styles.grid3;
        case 4: return styles.grid4;
        case 5: return styles.grid5;
        case 6: return styles.grid6;
        case 7: return styles.grid7;
        case 8: return styles.grid8;
        case 9: return styles.grid9;
        default: return styles.grid9;
      }
    };

    /**
     * 计算图片显示尺寸
     */
    const getImageSize = (index: number): { width: number; height: number } => {
      const baseSize = 100 / (actualLayout <= 2 ? 2 : actualLayout <= 4 ? 2 : 3);

      if (actualLayout === 1) {
        // 单图：保持原始比例，最大高度400px
        return { width: 100, height: 400 };
      }

      if (actualLayout === 3 && index === 0) {
        // 3张图时第一张占满一行
        return { width: 100, height: baseSize * 2 };
      }

      return { width: baseSize, height: baseSize };
    };

    /**
     * 图片加载完成处理
     */
    const handleImageLoad = useCallback((index: number) => {
      setLoadedImages(prev => new Set([...prev, index]));
    }, []);

    /**
     * 点击图片处理
     */
    const handleClick = useCallback(
      (index: number) => {
        onImageClick?.(index);
      },
      [onImageClick]
    );

    // 渲染单图模式
    if (actualLayout === 1 && images.length === 1) {
      const image = images[0];
      return (
        <div
          className={styles.singleImageContainer}
          onClick={() => handleClick(0)}
        >
          <Image
            src={image.url}
            alt="图片内容"
            width={600}
            height={400}
            className={styles.singleImage}
          />
        </div>
      );
    }

    // 渲染多图网格
    return (
      <div className={`${styles.imageGrid} ${getGridClass()}`}>
        {images.slice(0, actualLayout).map((image, index) => {
          const { width, height } = getImageSize(index);

          return (
            <div
              key={image.id}
              className={`${styles.imageWrapper} ${
                loadedImages.has(index) ? styles.loaded : styles.loading
              }`}
              style={{ paddingBottom: `${100 / width}%` }}
              onClick={() => handleClick(index)}
            >
              <Image
                src={image.thumbnailUrl || image.url}
                alt={`图片 ${index + 1}`}
                fill
                sizes="(max-width: 768px) 50vw, 33vw"
                className={styles.gridImage}
                onLoad={() => handleImageLoad(index)}
              />

              {/* 加载占位符 */}
              {!loadedImages.has(index) && (
                <div className={styles.skeleton} />
              )}

              {/* 更多图片遮罩（超过9张时） */}
              {index === 8 && images.length > 9 && (
                <div className={styles.moreOverlay}>
                  +{images.length - 9}
                </div>
              )}
            </div>
          );
        })}
      </div>
    );
  },
  (prevProps, nextProps) => {
    // 浅比较：只有图片URL变化时才重新渲染
    return (
      prevProps.images.length === nextProps.images.length &&
      prevProps.images.every((img, i) => img.url === nextProps.images[i]?.url)
    );
  }
);

ImageGrid.displayName = 'ImageGrid';
```

### 4.3 @提及：正则匹配

@提及功能允许用户在任何内容中引用其他用户。

**@提及的正则表达式实现：**

```typescript
// 后端：@提及解析服务
// services/mentionService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository, In } from 'typeorm';
import { User } from '../entities/user.entity';

@Injectable()
export class MentionService {
  constructor(
    @InjectRepository(User)
    private readonly userRepository: Repository<User>,
  ) {}

  /**
   * @提及的正则表达式
   *
   * 支持的格式：
   * - @username （字母、数字、下划线）
   * - @用户昵称 （中文）
   * - @@ （连续@符号）
   *
   * 负向预查排除：
   * - 邮箱地址中的@（如 user@example.com）
   */
  private readonly MENTION_REGEX = /@(?!!)([a-zA-Z0-9_\u4e00-\u9fa5]{1,30})/g;

  /**
   * 解析文本中的@提及
   *
   * @param text 原始文本
   * @returns 提取出的提及列表
   */
  extractMentions(text: string): string[] {
    if (!text) return [];

    const matches = text.matchAll(this.MENTION_REGEX);
    const mentions: string[] = [];

    for (const match of matches) {
      const username = match[1];
      if (!mentions.includes(username)) {
        mentions.push(username);
      }
    }

    return mentions;
  }

  /**
   * 高亮文本中的@提及
   * 用于前端展示
   *
   * @param text 原始文本
   * @param mentions 提及的用户列表
   * @returns 高亮处理后的HTML
   */
  highlightMentions(
    text: string,
    mentions: Array<{ username: string; userId: string }>
  ): string {
    if (!text) return '';

    // 构建用户名到ID的映射
    const mentionMap = new Map(mentions.map(m => [m.username, m.userId]));

    // 替换@提及为可点击链接
    return text.replace(
      this.MENTION_REGEX,
      (match, username) => {
        const userId = mentionMap.get(username);
        if (userId) {
          return `<a href="/user/${userId}" class="mention">@${username}</a>`;
        }
        return match; // 未匹配到的用户保持原样
      }
    );
  }

  /**
   * 解析并验证@提及
   * 返回有效的提及用户信息
   */
  async parseAndValidateMentions(
    text: string
  ): Promise<Array<{ username: string; userId: string; isValid: boolean }>> {
    const usernames = this.extractMentions(text);

    if (usernames.length === 0) {
      return [];
    }

    // 批量查询用户
    const users = await this.userRepository.find({
      where: [
        { name: In(usernames) },
        { handle: In(usernames) },
      ],
      select: ['id', 'name', 'handle'],
    });

    // 构建用户映射
    const userMap = new Map<string, { name: string; handle: string }>();
    for (const user of users) {
      userMap.set(user.name, { name: user.name, handle: user.handle });
      userMap.set(user.handle, { name: user.name, handle: user.handle });
    }

    // 验证并返回结果
    return usernames.map(username => {
      const user = userMap.get(username);
      return {
        username,
        userId: user?.name || '',
        isValid: !!user,
      };
    });
  }

  /**
   * 创建@提及通知
   * 当用户被@提及时，发送通知
   */
  async createMentionNotifications(
    postAuthorId: string,
    postId: string,
    text: string
  ): Promise<void> {
    const mentions = await this.parseAndValidateMentions(text);

    // 过滤掉自己@自己的情况
    const validMentions = mentions.filter(
      m => m.isValid && m.userId !== postAuthorId
    );

    if (validMentions.length === 0) {
      return;
    }

    // 创建通知（实际实现中会写入通知表）

    //     postId,
    //     content: `在帖子中提到了你`,
    //   }))
    // );

    console.log('创建提及通知:', validMentions);
  }
}
```

### 4.4 #话题：话题识别与聚合

话题标签允许用户围绕特定主题组织内容。

```typescript
// 后端：话题标签服务
// services/hashtagService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository, Like } from 'typeorm';
import { Hashtag } from '../entities/hashtag.entity';
import { PostHashtag } from '../entities/post_hashtag.entity';

@Injectable()
export class HashtagService {
  private readonly HASHTAG_REGEX = /#([^\s#.,!?，。！？、；:'"（）【】《》【】]+)#?/g;

  constructor(
    @InjectRepository(Hashtag)
    private readonly hashtagRepository: Repository<Hashtag>,
    @InjectRepository(PostHashtag)
    private readonly postHashtagRepository: Repository<PostHashtag>,
  ) {}

  /**
   * 从文本中提取话题标签
   */
  extractHashtags(text: string): string[] {
    if (!text) return [];

    const matches = text.matchAll(this.HASHTAG_REGEX);
    const hashtags: string[] = [];

    for (const match of matches) {
      const tag = match[1].toLowerCase(); // 统一小写存储
      if (tag.length >= 2 && tag.length <= 30) {
        hashtags.push(tag);
      }
    }

    return [...new Set(hashtags)]; // 去重
  }

  /**
   * 高亮文本中的话题标签
   */
  highlightHashtags(text: string): string {
    if (!text) return '';

    return text.replace(
      this.HASHTAG_REGEX,
      (match, tag) => {
        const encodedTag = encodeURIComponent(tag);
        return `<a href="/topic/${encodedTag}" class="hashtag">#${tag}</a>`;
      }
    );
  }

  /**
   * 获取或创建话题
   */
  async getOrCreateHashtags(tags: string[]): Promise<Hashtag[]> {
    if (tags.length === 0) return [];

    // 查询已存在的话题
    const existingTags = await this.hashtagRepository.find({
      where: tags.map(tag => ({ name: tag })),
    });

    const existingMap = new Map(existingTags.map(t => [t.name, t]));

    // 创建不存在的话题
    const newTags: Hashtag[] = [];
    for (const tag of tags) {
      if (!existingMap.has(tag)) {
        const newTag = this.hashtagRepository.create({
          name: tag,
          postCount: 0,
        });
        newTags.push(newTag);
      }
    }

    // 批量保存新话题
    if (newTags.length > 0) {
      await this.hashtagRepository.save(newTags);
    }

    // 返回所有话题（包含新建的）
    return [...existingTags, ...newTags];
  }

  /**
   * 更新话题的帖子计数
   */
  async incrementPostCount(hashtagIds: string[]): Promise<void> {
    if (hashtagIds.length === 0) return;

    await this.hashtagRepository
      .createQueryBuilder()
      .update()
      .set({ postCount: () => 'post_count + 1' })
      .whereInIds(hashtagIds)
      .execute();
  }

  /**
   * 获取话题详情
   */
  async getHashtagDetail(tag: string): Promise<Hashtag | null> {
    return this.hashtagRepository.findOne({
      where: { name: tag.toLowerCase() },
    });
  }

  /**
   * 获取话题下的热门帖子
   */
  async getHashtagPosts(
    tag: string,
    cursor: string | null,
    limit: number = 20
  ): Promise<{ posts: any[]; nextCursor: string | null }> {
    const hashtag = await this.getHashtagDetail(tag);

    if (!hashtag) {
      return { posts: [], nextCursor: null };
    }

    // 查询该话题下的帖子（通过关联表）
    const queryBuilder = this.postHashtagRepository
      .createQueryBuilder('ph')
      .innerJoinAndSelect('ph.post', 'post')
      .where('ph.hashtagId = :hashtagId', { hashtagId: hashtag.id })
      .orderBy('post.createdAt', 'DESC')
      .take(limit + 1);

    if (cursor) {
      const cursorTime = new Date(Buffer.from(cursor, 'base64').toString('utf-8'));
      queryBuilder.andWhere('post.createdAt < :cursorTime', { cursorTime });
    }

    const results = await queryBuilder.getMany();
    const posts = results.map(r => r.post);

    const hasNextPage = posts.length > limit;
    if (hasNextPage) posts.pop();

    const nextCursor = hasNextPage && posts.length > 0
      ? Buffer.from(posts[posts.length - 1].createdAt.toISOString()).toString('base64')
      : null;

    return { posts, nextCursor };
  }

  /**
   * 搜索话题（自动补全）
   */
  async searchHashtags(query: string, limit: number = 10): Promise<Hashtag[]> {
    return this.hashtagRepository.find({
      where: { name: Like(`%${query.toLowerCase()}%`) },
      order: { postCount: 'DESC' },
      take: limit,
    });
  }

  /**
   * 获取热门话题
   */
  async getHotHashtags(limit: number = 10): Promise<Hashtag[]> {
    return this.hashtagRepository.find({
      where: { postCount: MoreThan(0) },
      order: { postCount: 'DESC' },
      take: limit,
    });
  }
}
```

### 4.5 实战：完整发布流程

```typescript
// 后端：内容发布服务
// services/contentPublishService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { Transactional } from 'typeorm-transactional';
import { Post } from '../entities/post.entity';
import { MentionService } from './mentionService';
import { HashtagService } from './hashtagService';
import { FeedPushService } from './feedPushService';
import { SnowflakeIdGenerator } from '../utils/snowflake';

@Injectable()
export class ContentPublishService {
  private readonly snowflake = SnowflakeIdGenerator.createAuto();

  constructor(
    @InjectRepository(Post)
    private readonly postRepository: Repository<Post>,
    private readonly mentionService: MentionService,
    private readonly hashtagService: HashtagService,
    private readonly feedPushService: FeedPushService,
  ) {}

  /**
   * 发布内容完整流程
   *
   * 流程步骤：
   * 1. 生成唯一ID
   * 2. 解析@提及
   * 3. 解析#话题
   * 4. 保存帖子
   * 5. 更新话题计数
   * 6. 分发到粉丝Feed
   * 7. 发送提及通知
   */
  @Transactional()
  async publishPost(params: {
    authorId: string;
    content: string;
    images?: string[];
    sourcePostId?: string; // 转发来源
    replyToPostId?: string; // 回复目标
    replyToUserId?: string; // 回复目标用户
  }): Promise<Post> {
    const {
      authorId,
      content,
      images = [],
      sourcePostId,
      replyToPostId,
      replyToUserId,
    } = params;

    // 第一步：生成唯一ID
    const postId = this.snowflake.nextIdString();

    // 第二步：解析@提及
    const mentions = await this.mentionService.parseAndValidateMentions(content);
    const validMentions = mentions.filter(m => m.isValid).map(m => ({
      userId: m.userId,
      username: m.username,
    }));

    // 第三步：解析#话题
    const hashtagNames = this.hashtagService.extractHashtags(content);
    const hashtags = await this.hashtagService.getOrCreateHashtags(hashtagNames);

    // 第四步：创建帖子实体
    const post = this.postRepository.create({
      id: postId,
      authorId,
      content,
      images: images.map((url, index) => ({
        url,
        order: index,
      })),
      sourcePostId,
      replyToPostId,
      replyToUserId,
      mentionIds: validMentions.map(m => m.userId),
      hashtagIds: hashtags.map(h => h.id),
      likeCount: 0,
      commentCount: 0,
      repostCount: 0,
      createdAt: new Date(),
      updatedAt: new Date(),
    });

    // 第五步：保存帖子
    await this.postRepository.save(post);

    // 第六步：更新话题计数
    if (hashtags.length > 0) {
      await this.hashtagService.incrementPostCount(hashtags.map(h => h.id));
    }

    // 第七步：分发到粉丝Feed（异步）
    // 注意：这是异步操作，不阻塞主流程
    this.feedPushService.publishAndDistribute(post).catch(err => {
      console.error('Feed分发失败:', err);
    });

    // 第八步：发送@提及通知（异步）
    if (validMentions.length > 0) {
      this.mentionService.createMentionNotifications(
        authorId,
        postId,
        content
      ).catch(err => {
        console.error('提及通知发送失败:', err);
      });
    }

    // 返回创建的帖子
    return post;
  }

  /**
   * 转发内容
   */
  async repost(
    authorId: string,
    sourcePostId: string,
    comment?: string
  ): Promise<Post> {
    // 获取原帖
    const sourcePost = await this.postRepository.findOne({
      where: { id: sourcePostId },
    });

    if (!sourcePost) {
      throw new Error('原帖不存在');
    }

    // 构建转发内容
    let content = '';
    if (comment) {
      content = comment;
    }

    // 发布转发
    return this.publishPost({
      authorId,
      content,
      sourcePostId,
    });
  }

  /**
   * 回复内容
   */
  async reply(
    authorId: string,
    sourcePostId: string,
    content: string,
    images?: string[]
  ): Promise<Post> {
    // 获取原帖
    const sourcePost = await this.postRepository.findOne({
      where: { id: sourcePostId },
    });

    if (!sourcePost) {
      throw new Error('原帖不存在');
    }

    // 发布回复
    const reply = await this.publishPost({
      authorId,
      content,
      images,
      replyToPostId: sourcePostId,
      replyToUserId: sourcePost.authorId,
    });

    // 更新原帖的评论计数
    await this.postRepository.increment(
      { id: sourcePostId },
      'commentCount',
      1
    );

    return reply;
  }
}
```

### 5.1 评论嵌套：多级评论

评论系统需要支持嵌套回复，常见的有两种实现方式：

1. **邻接表模型**：parent_id指向父评论
2. **路径模型**：path字段存储完整路径，如"1/3/5"

```typescript
// 后端：评论服务
// services/commentService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository, MoreThan } from 'typeorm';
import { Comment } from '../entities/comment.entity';

@Injectable()
export class CommentService {
  // 评论最大嵌套层数
  private readonly MAX_NESTING_LEVEL = 3;

  constructor(
    @InjectRepository(Comment)
    private readonly commentRepository: Repository<Comment>,
  ) {}

  /**
   * 创建评论
   */
  @Transactional()
  async createComment(params: {
    postId: string;
    userId: string;
    content: string;
    parentId?: string; // 回复的评论ID
  }): Promise<Comment> {
    const { postId, userId, content, parentId } = params;

    // 计算嵌套层级
    let level = 1;
    let path = '';

    if (parentId) {
      const parentComment = await this.commentRepository.findOne({
        where: { id: parentId },
      });

      if (!parentComment) {
        throw new Error('父评论不存在');
      }

      level = Math.min(parentComment.level + 1, this.MAX_NESTING_LEVEL);
      path = parentComment.path
        ? `${parentComment.path}/${parentComment.id}`
        : parentComment.id;
    }

    // 创建评论
    const comment = this.commentRepository.create({
      id: this.generateCommentId(),
      postId,
      userId,
      content,
      parentId,
      level,
      path,
      likeCount: 0,
      replyCount: 0,
      createdAt: new Date(),
      updatedAt: new Date(),
    });

    await this.commentRepository.save(comment);

    // 更新父评论的回复计数
    if (parentId) {
      await this.commentRepository.increment(
        { id: parentId },
        'replyCount',
        1
      );
    }

    return comment;
  }

  /**
   * 获取帖子的评论列表
   * 支持嵌套展示和分页
   */
  async getCommentsByPost(
    postId: string,
    options: {
      cursor?: string;
      limit?: number;
      sort?: 'time' | 'hot';
      filterLevel?: number; // 只获取某一级别的评论
    } = {}
  ): Promise<{
    comments: Comment[];
    nextCursor: string | null;
  }> {
    const { cursor, limit = 20, sort = 'time', filterLevel } = options;

    let queryBuilder = this.commentRepository
      .createQueryBuilder('comment')
      .innerJoinAndSelect('comment.user', 'user')
      .where('comment.postId = :postId', { postId })
      .andWhere('comment.parentId IS NULL'); // 只获取一级评论

    // 过滤层级
    if (filterLevel !== undefined) {
      queryBuilder = queryBuilder.andWhere('comment.level = :level', {
        level: filterLevel,
      });
    }

    // 排序
    if (sort === 'hot') {
      queryBuilder = queryBuilder.orderBy(
        'comment.likeCount + comment.replyCount * 2',
        'DESC'
      );
    } else {
      queryBuilder = queryBuilder.orderBy('comment.createdAt', 'DESC');
    }

    // 游标分页
    if (cursor) {
      const cursorData = JSON.parse(
        Buffer.from(cursor, 'base64').toString('utf-8')
      );
      queryBuilder = queryBuilder.andWhere(
        sort === 'hot'
          ? 'comment.likeCount < :cursorLikeCount'
          : 'comment.createdAt < :cursorTime',
        cursorData
      );
    }

    queryBuilder = queryBuilder.take(limit + 1);

    const comments = await queryBuilder.getMany();
    const hasNextPage = comments.length > limit;
    if (hasNextPage) comments.pop();

    // 如果需要热门排序，二次查询子评论
    if (sort === 'hot') {
      const commentIds = comments.map(c => c.id);
      const childComments = await this.getChildComments(commentIds);

      // 将子评论附加到父评论
      comments.forEach(parent => {
        parent.children = childComments.filter(
          c => c.parentId === parent.id
        );
      });
    }

    const nextCursor = hasNextPage
      ? Buffer.from(
          JSON.stringify(
            sort === 'hot'
              ? { cursorLikeCount: comments[comments.length - 1].likeCount }
              : { cursorTime: comments[comments.length - 1].createdAt }
          )
        ).toString('base64')
      : null;

    return { comments, nextCursor };
  }

  /**
   * 获取子评论
   */
  private async getChildComments(
    parentIds: string[],
    limit: number = 3
  ): Promise<Comment[]> {
    if (parentIds.length === 0) return [];

    return this.commentRepository
      .createQueryBuilder('comment')
      .innerJoinAndSelect('comment.user', 'user')
      .where('comment.parentId IN (:...parentIds)', { parentIds })
      .andWhere('comment.level <= :maxLevel', {
        maxLevel: this.MAX_NESTING_LEVEL,
      })
      .orderBy('comment.createdAt', 'ASC')
      .take(limit)
      .getMany();
  }

  /**
   * 获取评论的完整回复链
   */
  async getCommentThread(commentId: string): Promise<{
    ancestors: Comment[]; // 祖先评论
    comment: Comment; // 当前评论
    descendants: Comment[]; // 子评论
  }> {
    const comment = await this.commentRepository.findOne({
      where: { id: commentId },
    });

    if (!comment) {
      throw new Error('评论不存在');
    }

    // 获取祖先路径
    const ancestors: Comment[] = [];
    if (comment.path) {
      const ancestorIds = comment.path.split('/');
      const ancestorComments = await this.commentRepository.findByIds(ancestorIds);
      ancestors.push(...ancestorComments);
    }

    // 获取子评论
    const descendants = await this.commentRepository.find({
      where: { path: Like(`${commentId}%`) },
      order: { createdAt: 'ASC' },
    });

    return { ancestors, comment, descendants };
  }

  /**
   * 删除评论（软删除）
   */
  async deleteComment(commentId: string, userId: string): Promise<void> {
    const comment = await this.commentRepository.findOne({
      where: { id: commentId, userId },
    });

    if (!comment) {
      throw new Error('评论不存在或无权删除');
    }

    // 软删除
    comment.isDeleted = true;
    comment.content = '该评论已删除';
    await this.commentRepository.save(comment);

    // 更新父评论的回复计数
    if (comment.parentId) {
      await this.commentRepository.decrement(
        { id: comment.parentId },
        'replyCount',
        1
      );
    }
  }

  /**
   * 生成评论ID（简化版，实际应使用雪花算法）
   */
  private generateCommentId(): string {
    return `c_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`;
  }
}
```

### 5.3 点赞：计数爆炸问题

点赞是高频操作，当点赞数达到百万级时，如何准确存储和显示是一个挑战。

```typescript
// 后端：点赞服务
// services/likeService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { RedisService } from './redis.service';
import { Like } from '../entities/like.entity';

/**
 * 点赞服务
 *
 * 计数爆炸问题解决方案：
 * 1. 分级计数：精确计数 + 模糊展示
 * 2. 异步写入：高峰期的点赞先写缓存
 * 3. 批量聚合：定时任务合并计数到数据库
 */
@Injectable()
export class LikeService {
  // Redis缓存key前缀
  private readonly LIKE_COUNT_KEY = 'like:count:';
  private readonly USER_LIKE_KEY = 'user:like:';

  // 计数阈值（超过此值使用模糊展示）
  private readonly FUZZY_THRESHOLD = 10000;

  constructor(
    @InjectRepository(Like)
    private readonly likeRepository: Repository<Like>,
    private readonly redisService: RedisService,
  ) {}

  /**
   * 点赞/取消点赞
   */
  async toggleLike(
    userId: string,
    targetType: 'post' | 'comment',
    targetId: string
  ): Promise<{ isLiked: boolean; count: number }> {
    const likeKey = `${targetType}:${targetId}`;
    const userLikeKey = `${this.USER_LIKE_KEY}${userId}`;

    // 检查用户是否已点赞
    const isLiked = await this.redisService.sismember(userLikeKey, likeKey);

    if (isLiked) {
      // 取消点赞
      await this.unlike(userId, targetType, targetId);
      return { isLiked: false, count: -1 };
    } else {
      // 点赞
      await this.addLike(userId, targetType, targetId);
      return { isLiked: true, count: 1 };
    }
  }

  /**
   * 添加点赞
   */
  private async addLike(
    userId: string,
    targetType: 'post' | 'comment',
    targetId: string
  ): Promise<void> {
    const likeKey = `${targetType}:${targetId}`;
    const userLikeKey = `${this.USER_LIKE_KEY}${userId}`;

    // 写入用户点赞集合
    await this.redisService.sadd(userLikeKey, likeKey);

    // 增加计数（先在Redis中更新）
    await this.redisService.hincrby(
      this.LIKE_COUNT_KEY + likeKey,
      'count',
      1
    );

    // 异步写入数据库
    await this.queueLikeToDb(userId, targetType, targetId, 'add');
  }

  /**
   * 取消点赞
   */
  private async unlike(
    userId: string,
    targetType: 'post' | 'comment',
    targetId: string
  ): Promise<void> {
    const likeKey = `${targetType}:${targetId}`;
    const userLikeKey = `${this.USER_LIKE_KEY}${userId}`;

    // 从用户点赞集合中移除
    await this.redisService.srem(userLikeKey, likeKey);

    // 减少计数
    await this.redisService.hincrby(
      this.LIKE_COUNT_KEY + likeKey,
      'count',
      -1
    );

    // 异步写入数据库
    await this.queueLikeToDb(userId, targetType, targetId, 'remove');
  }

  /**
   * 获取点赞状态和计数
   */
  async getLikeStatus(
    userId: string,
    targetType: 'post' | 'comment',
    targetId: string
  ): Promise<{
    isLiked: boolean;
    count: number;
    displayCount: string; // 模糊展示的计数
  }> {
    const likeKey = `${targetType}:${targetId}`;
    const userLikeKey = `${this.USER_LIKE_KEY}${userId}`;

    // 并行获取用户点赞状态和计数
    const [isLiked, countData] = await Promise.all([
      this.redisService.sismember(userLikeKey, likeKey),
      this.redisService.hgetall(this.LIKE_COUNT_KEY + likeKey),
    ]);

    const count = parseInt(countData?.count || '0', 10);

    return {
      isLiked: !!isLiked,
      count,
      displayCount: this.formatCount(count),
    };
  }

  /**
   * 批量获取点赞状态
   */
  async batchGetLikeStatus(
    userId: string,
    targets: Array<{ type: 'post' | 'comment'; id: string }>
  ): Promise<Map<string, boolean>> {
    if (targets.length === 0) {
      return new Map();
    }

    const userLikeKey = `${this.USER_LIKE_KEY}${userId}`;
    const pipeline = this.redisService.getPipeline();

    for (const target of targets) {
      const likeKey = `${target.type}:${target.id}`;
      pipeline.sismember(userLikeKey, likeKey);
    }

    const results = await pipeline.exec();
    const statusMap = new Map<string, boolean>();

    targets.forEach((target, index) => {
      const [err, isLiked] = results[index];
      statusMap.set(`${target.type}:${target.id}`, !err && !!isLiked);
    });

    return statusMap;
  }

  /**
   * 将点赞操作写入数据库（异步）
   */
  private async queueLikeToDb(
    userId: string,
    targetType: 'post' | 'comment',
    targetId: string,
    action: 'add' | 'remove'
  ): Promise<void> {
    // 实际实现中会写入消息队列
    // 这里简化处理，直接同步写入
    if (action === 'add') {
      const like = this.likeRepository.create({
        userId,
        targetType,
        targetId,
        createdAt: new Date(),
      });

      try {
        await this.likeRepository.insert(like);
      } catch (error) {
        // 忽略重复点赞的错误
        if (error.code !== '23505') {
          throw error;
        }
      }
    } else {
      await this.likeRepository.delete({ userId, targetType, targetId });
    }
  }

  /**
   * 格式化计数（模糊展示）
   */
  private formatCount(count: number): string {
    if (count >= 100000000) {
      return `${(count / 100000000).toFixed(1)}亿`;
    }
    if (count >= 100000) {
      return `${(count / 10000).toFixed(1)}万`;
    }
    if (count >= 10000) {
      return `${(count / 10000).toFixed(1)}万`;
    }
    return count.toString();
  }

  /**
   * 同步Redis计数到数据库（定时任务）

    // 获取所有点赞计数的key

      const likeKey = key.replace(this.LIKE_COUNT_KEY, '');

      const count = parseInt(countData?.count || '0', 10);

      // 更新数据库

    }
  }
```

### 5.4 @回复：通知聚合

当用户收到大量回复时，需要对通知进行聚合，避免通知爆炸。

```typescript
// 后端：回复通知聚合服务
// services/replyNotificationService.ts

import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { Notification } from '../entities/notification.entity';

@Injectable()
export class ReplyNotificationService {
  // 通知聚合时间窗口（毫秒）
  private readonly AGGREGATION_WINDOW = 30 * 60 * 1000; // 30分钟

  constructor(
    @InjectRepository(Notification)
    private readonly notificationRepository: Repository<Notification>,
  ) {}

  /**
   * 创建回复通知
   *
   * 聚合策略：
   * 1. 检查是否在聚合时间窗口内存在未读通知
   * 2. 如果存在，更新现有通知的计数和摘要
   * 3. 如果不存在，创建新通知
   */
  async createReplyNotification(params: {
    postAuthorId: string; // 被回复的用户ID
    replierId: string; // 回复者ID
    replierName: string;
    postId: string;
    commentId: string;
    commentContent: string;
  }): Promise<void> {
    const { postAuthorId, replierId, replierName, postId, commentId, commentContent } = params;

    // 查找是否在聚合窗口内有未读通知
    const existingNotification = await this.findAggregatableNotification(
      postAuthorId,
      postId
    );

    if (existingNotification) {
      // 聚合到现有通知
      await this.aggregateNotification(
        existingNotification,
        replierId,
        replierName,
        commentContent
      );
    } else {
      // 创建新通知
      await this.createNewNotification(params);
    }
  }

  /**
   * 查找可聚合的通知
   */
  private async findAggregatableNotification(
    userId: string,
    postId: string
  ): Promise<Notification | null> {
    const windowStart = new Date(Date.now() - this.AGGREGATION_WINDOW);

    return this.notificationRepository
      .createQueryBuilder('notification')
      .where('notification.userId = :userId', { userId })
      .andWhere('notification.type = :type', { type: 'reply' })
      .andWhere('notification.postId = :postId', { postId })
      .andWhere('notification.isRead = :isRead', { isRead: false })
      .andWhere('notification.createdAt > :windowStart', { windowStart })
      .orderBy('notification.createdAt', 'DESC')
      .getOne();
  }

  /**
   * 聚合到现有通知
   */
  private async aggregateNotification(
    notification: Notification,
    replierId: string,
    replierName: string,
    commentContent: string
  ): Promise<void> {
    // 解析现有的聚合数据
    const聚合数据 = notification.aggregationData || {
      replyIds: [],
      repliers: [],
      previewContents: [],
    };

    // 添加新的回复者
    if (!聚合数据.repliers.some(r => r.id === replierId)) {
      聚合数据.repliers.push({ id: replierId, name: replierName });
    }

    // 添加回复ID和预览内容
    聚合数据.replyIds.push(replierId);
    聚合数据.previewContents.push(
      commentContent.slice(0, 50)
    );

    // 限制聚合数量（最多聚合10条）
    if (聚合数据.replyIds.length > 10) {
      聚合数据.replyIds = 聚合数据.replyIds.slice(-10);
      聚合数据.previewContents = 聚合数据.previewContents.slice(-10);
    }

    // 更新通知
    notification.aggregationData = 聚合数据;
    notification.replyCount = (notification.replyCount || 0) + 1;
    notification.content = this.generateAggregationContent(聚合数据);
    notification.updatedAt = new Date();

    await this.notificationRepository.save(notification);
  }

  /**
   * 创建新通知
   */
  private async createNewNotification(params: {
    postAuthorId: string;
    replierId: string;
    replierName: string;
    postId: string;
    commentId: string;
    commentContent: string;
  }): Promise<void> {
    const { postAuthorId, replierId, replierName, postId, commentId, commentContent } = params;

    const notification = this.notificationRepository.create({
      userId: postAuthorId,
      type: 'reply',
      postId,
      commentId,
      replierId,
      content: `${replierName} 回复了你的评论：${commentContent.slice(0, 50)}`,
      replyCount: 1,
      aggregationData: {
        replyIds: [replierId],
        repliers: [{ id: replierId, name: replierName }],
        previewContents: [commentContent.slice(0, 50)],
      },
      isRead: false,
      createdAt: new Date(),
      updatedAt: new Date(),
    });

    await this.notificationRepository.save(notification);
  }

  /**
   * 生成聚合后的通知内容
   */
  private generateAggregationContent(聚合数据: {
    repliers: Array<{ id: string; name: string }>;
    replyIds: string[];
    previewContents: string[];
  }): string {
    const count = 聚合数据.replyIds.length;

    if (count === 1) {
      return `${聚合数据.repliers[0].name} 回复了你的评论：${聚合数据.previewContents[0]}`;
    }

    if (聚合数据.repliers.length === 1) {
      return `${聚合数据.repliers[0].name} 等${count}人回复了你的评论`;
    }

    const names = 聚合数据.repliers.slice(0, 2).map(r => r.name).join('、');
    return `${names}等${聚合数据.repliers.length}人回复了你的评论`;
  }

  /**
   * 获取通知列表（包含聚合展示）
   */
  async getNotifications(userId: string): Promise<Notification[]> {
    return this.notificationRepository.find({
      where: { userId },
      order: { updatedAt: 'DESC' },
      take: 50,
    });
  }
}
```

```typescript
  // 已发送消息的确认key前缀

  }): Promise<Message> {

      if (existingMessage) {
        return existingMessage;
      }
    }

    // 获取或创建会话

    // 创建消息

      id: this.generateMessageId(),
      clientMessageId,
      conversationId: conversation.id,
      senderId,
      receiverId,
      content,
      status: 'sent', // sent -> delivered -> read
      createdAt: new Date(),
      updatedAt: new Date(),
    });

    await this.messageRepository.save(message);

    // 增加未读计数
    await this.redisService.hincrby(
      `${this.UNREAD_COUNT_KEY}${receiverId}`,
      conversation.id,
      1
    );

    // 触发实时推送（通过WebSocket）
    this.notifyNewMessage(message);

    return message;
  }

  /**
   * 标记消息已读
   */
  async markAsRead(
    userId: string,
    conversationId: string,
    messageId: string
  ): Promise<void> {
    // 更新消息状态
    await this.messageRepository.update(
      { id: messageId, receiverId: userId },
      {
        status: 'read',
        readAt: new Date(),
      }
    );

    // 清除未读计数
    await this.redisService.hset(
      `${this.UNREAD_COUNT_KEY}${userId}`,
      conversationId,
      '0'
    );

    // 发送已读回执
    this.notifyMessageRead(userId, conversationId, messageId);
  }

  /**
   * 标记会话全部已读
   */
  async markConversationAsRead(userId: string, conversationId: string): Promise<void> {
    // 更新该会话所有未读消息
    await this.messageRepository
      .createQueryBuilder()
      .update()
      .set({ status: 'read', readAt: new Date() })
      .where('conversationId = :conversationId', { conversationId })
      .andWhere('receiverId = :userId', { userId })
      .andWhere('status != :readStatus', { readStatus: 'read' })
      .execute();

    // 清除未读计数
    await this.redisService.hset(
      `${this.UNREAD_COUNT_KEY}${userId}`,
      conversationId,
      '0'
    );
  }

  /**
   * 获取会话列表
   */
  async getConversations(
    userId: string,
    cursor?: string,
    limit: number = 20
  ): Promise<{
    conversations: ConversationWithLastMessage[];
    nextCursor: string | null;
  }> {
    let queryBuilder = this.conversationRepository
      .createQueryBuilder('conv')
      .leftJoinAndSelect('conv.lastMessage', 'lastMessage')
      .leftJoinAndSelect('lastMessage.sender', 'sender')
      .where('conv.participantIds LIKE :userIdPattern', {
        userIdPattern: `%${userId}%`,
      })
      .orderBy('conv.updatedAt', 'DESC');

    if (cursor) {
      const cursorTime = new Date(Buffer.from(cursor, 'base64').toString('utf-8'));
      queryBuilder = queryBuilder.andWhere('conv.updatedAt < :cursorTime', {
        cursorTime,
      });
    }

    const conversations = await queryBuilder.take(limit + 1).getMany();

    // 获取每个会话的未读数
    for (const conv of conversations) {
      const unreadCount = await this.redisService.hget(
        `${this.UNREAD_COUNT_KEY}${userId}`,
        conv.id
      );
      conv.unreadCount = parseInt(unreadCount || '0', 10);
    }

    const hasNextPage = conversations.length > limit;
    if (hasNextPage) conversations.pop();

    const nextCursor = hasNextPage
      ? Buffer.from(conversations[conversations.length - 1].updatedAt.toISOString()).toString('base64')
      : null;

    return { conversations, nextCursor };
  }

  /**
   * 获取会话消息历史
   */
  async getMessages(
    conversationId: string,
    userId: string,
    cursor?: string,
    limit: number = 20
  ): Promise<{
    messages: Message[];
    nextCursor: string | null;
  }> {
    // 先验证用户是否有权访问该会话
    const conversation = await this.conversationRepository.findOne({
      where: { id: conversationId },
    });

    if (!conversation || !conversation.participantIds.includes(userId)) {
      throw new Error('无权访问该会话');
    }

    let queryBuilder = this.messageRepository
      .createQueryBuilder('msg')
      .innerJoinAndSelect('msg.sender', 'sender')
      .where('msg.conversationId = :conversationId', { conversationId })
      .orderBy('msg.createdAt', 'DESC');

    if (cursor) {
      const cursorData = JSON.parse(
        Buffer.from(cursor, 'base64').toString('utf-8')
      );
      queryBuilder = queryBuilder.andWhere('msg.createdAt < :cursorTime', {
        cursorTime: cursorData.cursorTime,
      });
    }

    const messages = await queryBuilder.take(limit + 1).getMany();

    const hasNextPage = messages.length > limit;
    if (hasNextPage) messages.pop();

    const nextCursor = hasNextPage
      ? Buffer.from(
          JSON.stringify({
            cursorTime: messages[messages.length - 1].createdAt,
          })
        ).toString('base64')
      : null;

    return { messages, nextCursor };
  }

  /**
   * 获取会话的未读数
   */
  async getUnreadCount(userId: string): Promise<number> {
    const unreadData = await this.redisService.hgetall(
      `${this.UNREAD_COUNT_KEY}${userId}`
    );

    let total = 0;
    for (const count of Object.values(unreadData || {})) {
      total += parseInt(count || '0', 10);
    }

    return total;
  }

  /**

  private async getOrCreateConversation(
    userId1: string,
    userId2: string
  ): Promise<Conversation> {
```

```typescript
  const [error, setError] = useState<string | null>(null);
```

### 7.5 我的思考：社交应用性能优化的关键

**性能优化的核心原则：**

```
┌─────────────────────────────────────────────────────────────────┐
│                    性能优化四象限法则                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│     高频 + 高耗时        →       重点优化                        │
│   (Feed读取、点赞)         (缓存、预加载)                        │
│                                                                  │
├─────────────────────────────┬───────────────────────────────────┤
│                              │                                   │
│   低频 + 高耗时        →       异步处理                          │
│   (搜索、复杂筛选)         (Web Worker、消息队列)                  │
│                              │                                   │
├─────────────────────────────┼───────────────────────────────────┤
│                              │                                   │
│   低频 + 低耗时        →       暂不关注                          │
│   (设置页、偏好)             (保持简单实现)                       │
│                              │                                   │
├─────────────────────────────┼───────────────────────────────────┤
│                              │                                   │
│   高频 + 低耗时        →       保持现状                          │
│   (心跳检测)                 (无需优化)                           │
│                              │                                   │
└─────────────────────────────┴───────────────────────────────────┘
```

**三个最重要的优化点：**

1. **首屏加载**：控制在1秒内
   - 骨架屏占位
   - 关键数据优先加载
   - 图片懒加载

2. **Feed滑动流畅**：60fps
   - 列表项复用
   - 图片尺寸预计算
   - 减少重渲染

3. **交互响应**：100ms内
   - 乐观更新
   - 本地缓存
   - 防抖节流

> 经验之谈：性能优化的投入产出比很重要。不要过早优化，也不要忽视用户体验。建议通过性能监控工具（如DataDog、Prometheus）识别瓶颈，再针对性优化。

### 8.1 双击点赞：为什么是双击

双击点赞是Twitter/Instagram的经典交互设计：

|----------|------|

| **手势直观** | 与"点赞"的心理预期一致 |

| **动画反馈** | 即时满足感，强化行为 |

```tsx
// 前端：双击点赞手势处理
// hooks/useDoubleTapLike.ts

'use client';

import { useCallback, useRef } from 'react';

interface UseDoubleTapLikeOptions {
  onDoubleTap: () => void;
  onSingleTap?: () => void;
  /** 双击判定时间窗口（毫秒） */
  delay?: number;
}

/**
 * 双击点赞手势Hook
 *
 * 实现逻辑：
 * 1. 记录第一次点击的时间
 * 2. 如果在时间窗口内发生第二次点击，判定为双击
 * 3. 否则在时间窗口结束后触发单击回调
 */
export function useDoubleTapLike({
  onDoubleTap,
  onSingleTap,
  delay = 300,
}: UseDoubleTapLikeOptions) {
  const lastTapRef = useRef<number>(0);
  const timeoutRef = useRef<NodeJS.Timeout | null>(null);

  const handleTap = useCallback(() => {
    const now = Date.now();
    const timeSinceLastTap = now - lastTapRef.current;

    if (timeSinceLastTap < delay && timeSinceLastTap > 0) {
      // 第二次点击，且在时间窗口内 → 双击
      if (timeoutRef.current) {
        clearTimeout(timeoutRef.current);
        timeoutRef.current = null;
      }

      onDoubleTap();
      lastTapRef.current = 0;
    } else {
      // 第一次点击，或超出时间窗口 → 可能是单击
      lastTapRef.current = now;

      if (onSingleTap) {
        timeoutRef.current = setTimeout(() => {
          onSingleTap();
          lastTapRef.current = 0;
        }, delay);
      }
    }
  }, [delay, onDoubleTap, onSingleTap]);

  return { handleTap };
}
```

### 8.3 草稿保存：防止丢失

```typescript
// 前端：草稿管理Hook
// hooks/useDraft.ts

'use client';

import { useEffect, useCallback, useRef } from 'react';

interface UseDraftOptions {
  /** 草稿存储的key */
  key: string;
  /** 内容变化回调 */
  onChange?: (content: string) => void;
  /** 自动保存间隔（毫秒） */
  autoSaveInterval?: number;
  /** 最大保存数量 */
  maxDrafts?: number;
}

/**
 * 草稿管理Hook
 *
 * 功能：
 * 1. 自动保存草稿到localStorage
 * 2. 防止编辑内容意外丢失
 * 3. 多份草稿管理
 */
export function useDraft({
  key,
  onChange,
  autoSaveInterval = 1000,
  maxDrafts = 10,
}: UseDraftOptions) {
  // 草稿列表
  const [drafts, setDrafts] = useState<Draft[]>([]);
  // 当前草稿
  const [currentDraft, setCurrentDraft] = useState<Draft | null>(null);

  const timerRef = useRef<NodeJS.Timeout | null>(null);
  const lastContentRef = useRef<string>('');

  /**
   * 加载草稿列表
   */
  useEffect(() => {
    const savedDrafts = localStorage.getItem(`drafts:${key}`);
    if (savedDrafts) {
      setDrafts(JSON.parse(savedDrafts));
    }
  }, [key]);

  /**
   * 保存草稿列表
   */
  const saveDraftsList = useCallback(
    (newDrafts: Draft[]) => {
      // 限制数量
      const trimmedDrafts = newDrafts.slice(0, maxDrafts);
      localStorage.setItem(`drafts:${key}`, JSON.stringify(trimmedDrafts));
      setDrafts(trimmedDrafts);
    },
    [key, maxDrafts]
  );

  /**
   * 保存当前草稿
   */
  const saveDraft = useCallback(
    (content: string) => {
      if (!content.trim()) {
        return;
      }

      const now = Date.now();
      const draft: Draft = {
        id: currentDraft?.id || `draft_${now}`,
        content,
        savedAt: now,
        preview: content.slice(0, 100),
      };

      // 更新当前草稿
      setCurrentDraft(draft);

      // 更新草稿列表
      const existingIndex = drafts.findIndex(d => d.id === draft.id);
      let newDrafts: Draft[];

      if (existingIndex >= 0) {
        newDrafts = drafts.map((d, i) => (i === existingIndex ? draft : d));
      } else {
        newDrafts = [draft, ...drafts];
      }

      saveDraftsList(newDrafts);
      onChange?.(content);
    },
    [drafts, currentDraft, saveDraftsList, onChange]
  );

  /**
   * 删除草稿
   */
  const deleteDraft = useCallback(
    (draftId: string) => {
      const newDrafts = drafts.filter(d => d.id !== draftId);
      saveDraftsList(newDrafts);

      if (currentDraft?.id === draftId) {
        setCurrentDraft(null);
      }
    },
    [drafts, currentDraft, saveDraftsList]
  );

  /**
   * 加载草稿
   */
  const loadDraft = useCallback(
    (draftId: string) => {
      const draft = drafts.find(d => d.id === draftId);
      if (draft) {
        setCurrentDraft(draft);
        onChange?.(draft.content);
      }
    },
    [drafts, onChange]
  );

  /**
   * 创建新草稿
   */
  const createNewDraft = useCallback(() => {
    setCurrentDraft({
      id: `draft_${Date.now()}`,
      content: '',
      savedAt: Date.now(),
      preview: '',
    });
    onChange?.('');
  }, [onChange]);

  /**
   * 自动保存
   */
  useEffect(() => {
    if (timerRef.current) {
      clearInterval(timerRef.current);
    }

    timerRef.current = setInterval(() => {
      if (lastContentRef.current) {
        saveDraft(lastContentRef.current);
      }
    }, autoSaveInterval);

    return () => {
      if (timerRef.current) {
        clearInterval(timerRef.current);
      }
    };
  }, [autoSaveInterval, saveDraft]);

  return {
    drafts,
    currentDraft,
    saveDraft: (content: string) => {
      lastContentRef.current = content;
    },
    deleteDraft,
    loadDraft,
    createNewDraft,
  };
}

interface Draft {
  id: string;
  content: string;
  savedAt: number;
  preview: string;
}
```

### 9.2 敏感词过滤：DFA算法

```typescript
// 后端：DFA敏感词过滤
// utils/sensitiveWordFilter.ts

/**
 * DFA（确定有限状态自动机）敏感词过滤算法
 *
 * 原理：
 * - 将敏感词构建为状态转移图
 * - 匹配时沿状态转移图遍历
 * - 时间复杂度O(n)，n为文本长度
 */

export class SensitiveWordFilter {
  // 状态节点
  private root: TrieNode = new TrieNode();

  /**
   * 添加敏感词到词典
   */
  addWord(word: string): void {
    let current = this.root;

    for (const char of word) {
      if (!current.children.has(char)) {
        current.children.set(char, new TrieNode());
      }
      current = current.children.get(char)!;
    }

    current.isEnd = true;
    current.word = word;
  }

  /**
   * 批量添加敏感词
   */
  addWords(words: string[]): void {
    for (const word of words) {
      this.addWord(word);
    }
  }

  /**
   * 检测文本中是否包含敏感词
   */
  contains(text: string): boolean {
    return this.findAny(text) !== null;
  }

  /**
   * 查找第一个敏感词
   */
  findAny(text: string): string | null {
    for (let i = 0; i < text.length; i++) {
      const result = this.matchFrom(text, i);
      if (result) {
        return result;
      }
    }
    return null;
  }

  /**
   * 查找所有敏感词
   */
  findAll(text: string): string[] {
    const found: string[] = [];

    for (let i = 0; i < text.length; i++) {
      const result = this.matchFrom(text, i);
      if (result) {
        found.push(result);
        i += result.length - 1; // 跳过已匹配的字符
      }
    }

    return [...new Set(found)]; // 去重
  }

  /**
   * 替换敏感词为指定字符
   */
  replace(text: string, replacement: string = '*'): string {
    let result = text;

    for (let i = 0; i < text.length; i++) {
      const match = this.matchFrom(text, i);
      if (match) {
        result = result.replace(match, replacement.repeat(match.length));
        i += match.length - 1;
      }
    }

    return result;
  }

  /**
   * 从指定位置开始匹配
   */
  private matchFrom(text: string, startIndex: number): string | null {
    let current = this.root;

    for (let i = startIndex; i < text.length; i++) {
      const char = text[i];
      const node = current.children.get(char);

      if (!node) {
        return null;
      }

      if (node.isEnd) {
        return node.word!;
      }

      current = node;
    }

    return null;
  }

  /**
   * 获取过滤器大小（词数）
   */
  getSize(): number {
    let count = 0;
    const traverse = (node: TrieNode) => {
      if (node.isEnd) count++;
      for (const child of node.children.values()) {
        traverse(child);
      }
    };
    traverse(this.root);
    return count;
  }
}

class TrieNode {
  children: Map<string, TrieNode> = new Map();
  isEnd: boolean = false;
  word: string | null = null;
}

// 使用示例

  '敏感词1',

  '违禁内容',
  '不当言论',

console.log(filter.contains('这是一段包含敏感词1的文本')); // true
```

### 9.4 隐私保护：数据脱敏

```typescript
// 后端：数据脱敏服务
// services/dataMaskingService.ts

import { Injectable } from '@nestjs/common';

/**
 * 数据脱敏工具
 *
 * 应用场景：
 * 1. 日志中的用户信息保护
 * 2. API返回的用户敏感信息
 * 3. 数据导出时的隐私保护
 */
@Injectable()
export class DataMaskingService {
  /**
   * 手机号脱敏：138****5678
   */
  maskPhone(phone: string): string {
    if (!phone || phone.length < 11) {
      return phone;
    }
    return phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2');
  }

  /**
   * 邮箱脱敏：t***@example.com
   */
  maskEmail(email: string): string {
    if (!email || !email.includes('@')) {
      return email;
    }

    const [name, domain] = email.split('@');
    if (name.length <= 2) {
      return `${name[0]}***@${domain}`;
    }

    return `${name[0]}***${name[name.length - 1]}@${domain}`;
  }

  /**
   * 身份证脱敏：110***********1234
   */
  maskIdCard(idCard: string): string {
    if (!idCard || idCard.length < 18) {
      return idCard;
    }
    return idCard.replace(/(\d{3})\d{12}(\d{4})/, '$1**********$2');
  }

  /**
   * 姓名脱敏
   */
  maskName(name: string): string {
    if (!name) return name;

    if (name.length === 2) {
      return `${name[0]}*`;
    }

    return `${name[0]}${'*'.repeat(name.length - 2)}${name[name.length - 1]}`;
  }

  /**
   * 批量脱敏用户对象
   */
  maskUserProfile(user: {
    phone?: string;
    email?: string;
    name?: string;
    idCard?: string;
    [key: string]: any;
  }): any {
    const masked = { ...user };

    if (masked.phone) masked.phone = this.maskPhone(masked.phone);
    if (masked.email) masked.email = this.maskEmail(masked.email);
    if (masked.name) masked.name = this.maskName(masked.name);
    if (masked.idCard) masked.idCard = this.maskIdCard(masked.idCard);

    return masked;
  }
}
```

**内容安全的挑战：**

1. **对抗性**：恶意用户会不断变换手法绕过检测
2. **规模性**：日均亿级内容，全量审核成本极高
3. **及时性**：热点事件需要在分钟级响应
4. **准确性**：误杀会影响用户体验，漏放会产生风险

> 经验之谈：内容安全不能只依赖技术手段，必须配合人工审核和用户举报机制。技术手段的正确率在95%以上已经很高，但5%的错误率在海量数据下仍然是巨大的绝对数量。

```typescript
async getFeedWithCache(userId: string): Promise<FeedData> {
  // 第一级：本地内存缓存（LRU）
  const localCached = this.localCache.get(userId);
  if (localCached) {
    return localCached;
  }

  // 第二级：Redis缓存
  const redisCached = await this.redis.get(`feed:${userId}`);
  if (redisCached) {
    this.localCache.set(userId, redisCached);
    return redisCached;
  }

  // 第三级：数据库
  const feedData = await this.fetchFeedFromDb(userId);

  // 回填缓存
  await this.redis.setex(`feed:${userId}`, 300, feedData);
  this.localCache.set(userId, feedData);

  return feedData;
}
```

### 10.5 我的思考：优化要有的放矢

**优化方法论：**

```
1. 测量先行
   - 不要猜测性能问题
   - 使用APM工具定位瓶颈
   - 收集真实用户数据（RUM）

2. 二八原则
   - 80%的时间花在20%的代码上
   - 优化投入产出比最高的部分

3. 权衡取舍
   - 缓存 vs 一致性
   - 预加载 vs 流量消耗
   - 复杂度 vs 可维护性

4. 持续监控
   - 性能回归检测
   - 异常告警
   - 用户体验追踪
```

**常见的优化陷阱：**

| 陷阱 | 后果 |
|------|------|
| 过度优化 | 代码复杂，难以维护 |
| 过早优化 | 浪费精力，可能猜错 |
| 局部优化 | 全局效果不明显 |
| 不测量就优化 | 无法验证效果 |

> 经验之谈：性能优化的最佳实践是先有监控，再有优化。通过数据驱动的方式，找到真正的瓶颈，然后针对性地解决。不要为了"技术炫技"而优化，用户体验才是最终目标。
