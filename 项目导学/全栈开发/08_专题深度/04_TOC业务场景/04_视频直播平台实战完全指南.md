# 视频直播平台实战完全指南

> 本文档将从零开始，详细讲解视频直播平台的技术架构、核心功能实现、实战代码编写。通过本指南，你将掌握视频平台开发的核心技术栈，包括视频上传、转码、播放、CDN分发、短视频系统、直播系统、互动系统、审核系统和数据埋点等关键技术点。每个技术点都配有完整的代码示例，帮助你从理论到实践全面掌握视频平台开发。

## 一、视频平台概述

### 1.1 视频平台类型

视频平台根据内容和用户场景的不同，主要分为三种类型：长视频平台、短视频平台和直播平台。

**长视频平台**以爱奇艺、优酷、腾讯视频为代表，主要提供电影、电视剧、综艺节目等长时间内容。这类平台的特点是内容时长通常在30分钟以上，用户观看行为呈现"发现-观看-离开"的单次消费模式。长视频平台的技术重点在于内容的版权管理、高清转码、多字幕支持、倍速播放等播放体验优化，以及会员订阅、点播付费等商业化能力。

**短视频平台**以抖音、快手、视频号为代表，主要提供15秒到5分钟的竖屏内容。这类平台的特点是内容消费呈现"刷-停-继续"的循环模式，用户粘性极高。短视频平台的技术重点在于内容推荐算法、无限滑动体验、预加载优化、缓存管理、社交分享等。内容生产门槛低，普通用户即可成为创作者，因此内容爆发式增长。

**直播平台**以斗鱼、虎牙、B站直播为代表，提供实时视频流内容。直播平台的特点是内容具有时效性，用户与主播实时互动。技术重点在于推流稳定性、低延迟播放、实时弹幕、礼物特效、连麦互动等。由于直播内容无法预先转码，对实时性和稳定性要求极高。

### 1.2 核心功能架构

视频平台的核心功能可以归纳为四个主要环节：**上传、转码、播放、分发**。

上传环节负责接收用户生成的视频内容。由于视频文件通常较大（手机拍摄的视频往往达到数百MB甚至数GB），直接上传不仅耗时长，而且一旦中断需要重新开始。因此分片上传和断点续传成为必须具备的能力。分片上传将大文件分割成小块独立上传，每块可以并行请求，失败时只重试失败的块。断点续传记录已上传的分片信息，中断后可以从断点继续。

转码环节将用户上传的原始视频转换为多种格式和分辨率，以适应不同网络条件和设备能力。原始视频通常是用户设备拍摄的高码率原始文件，直接用于分发会导致播放卡顿和流量浪费。转码后生成多个分辨率档位（如流畅360P、高清720P、超清1080P、4K2160P），以及多种编码格式（H.264兼容性好、H.265压缩率高、AV1新一代标准）。播放器根据当前网络状况自动选择最合适的清晰度。

播放环节是用户感知最直接的部分。播放器需要支持多种视频格式（MP4、WebM、HLS、DASH）、多种播放控制（播放/暂停、进度拖动、倍速、画中画）、清晰度切换、字幕加载等。播放器性能直接影响用户体验，首帧时间、卡顿率、Seek响应时间都是关键指标。

分发环节通过CDN将视频内容高效送达用户。CDN构建了覆盖全国甚至全球的边缘节点网络，用户请求就近访问边缘节点，减少跨运营商、跨地域传输带来的延迟。CDN还承担着防盗链、缓存管理、回源控制等重要职责。

### 1.3 技术挑战分析

视频平台面临的技术挑战主要来自三个方面：海量存储、高并发分发、成本控制。

**海量存储**是视频平台的基础挑战。以抖音为例，每天有数千万条视频上传，每个视频需要存储原始文件和多版本转码文件，存储规模达到EB级别。如何设计存储架构、如何冷热数据分离、如何保证数据可靠性都是必须考虑的问题。通常采用分布式文件系统（如HDFS、Ceph）配合对象存储（如MinIO、阿里云OSS）实现海量存储。原始文件和转码文件分开存储，转码文件可以采用更节省空间的存储层级。

**高并发分发**是视频平台的性能挑战。当热门视频发布或知名主播开播时，瞬间流量可能达到数百万并发。CDN是解决分发的核心架构，但CDN的容量规划、流量调度、故障应对都需要精心设计。同时，为了保证观看体验，需要在CDN层面做缓存预热、热点隔离、防盗链等处理。

**成本控制**是视频平台的商业挑战。视频分发是带宽密集型业务，带宽成本占运营成本的很大比例。优化编码效率可以降低码率从而节省带宽，但过度压缩会损害画质影响体验。CDN费用与流量成正比，通过智能调度、缓存命中率优化可以降低回源流量。存储成本同样不可忽视，合理的生命周期管理和冷数据归档可以降低存储费用。

### 1.4 视频平台技术栈

构建一个完整的视频平台需要以下技术栈。

前端技术方面，播放器是最核心的组件。可以基于HTML5 Video标签封装原生播放器，也可以基于video.js、plyr等开源播放器二次开发。商业平台通常自研播放器以掌握核心技术。播放器需要支持DRM数字版权管理（Widevine、PlayReady、FairPlay）以保护版权内容。弹幕系统通常使用Canvas渲染以获得更好的性能。短视频场景下，无限滑动列表需要使用虚拟列表技术优化内存占用。

后端技术方面，视频上传接口需要支持大文件分片，通常使用Socket.io或WebSocket实现上传进度实时推送。视频转码任务调度需要消息队列（如RabbitMQ、Kafka）解耦上传和转码。直播推流服务器可以使用SRS、nginx-rtmp、LiveFlow等开源方案。存储方面，对象存储是首选，配合CDN实现分发。

基础设施方面，CDN是视频分发的必备设施，可以选用阿里云CDN、腾讯云CDN、AWS CloudFront等商业方案，也可以自建CDN（如基于Squid/Varnish）。视频处理依赖FFmpeg这一强大的多媒体处理工具，几乎所有视频平台的后端都离不开FFmpeg。监控方面，需要实时监控播放质量（卡顿率、加载时间）和系统状态（CPU、内存、网络）。

### 1.5 我的思考：视频平台的技术壁垒

视频平台看似是内容平台的竞争，但其本质是技术能力的竞争。相同的内容，在不同的平台上播放体验可能天差地别。用户对视频平台的忠诚度很大程度上取决于播放体验——加载快不快、卡不卡顿、清晰度切换是否流畅。

技术壁垒主要体现在三个层面。第一层是基础设施层，包括CDN覆盖范围和调度能力、存储系统的可靠性和成本控制、转码效率和多格式支持。这些是硬实力，需要大量资金投入。第二层是体验优化层，包括首帧时间优化、码率自适应算法、弱网环境下的流畅播放。这些需要深入的技术研发和持续迭代。第三层是智能化层，包括内容推荐算法、智能封面提取、AI审核能力。这些是平台差异化的关键。

对于开发者来说，理解视频平台的技术全貌非常重要。即使你只负责其中的一个模块，也需要了解上下游的交互关系。例如，只负责播放器开发的工程师，如果不了解CDN的分发机制和转码的输出格式，就无法有效优化播放体验。

## 二、视频上传

### 2.1 分片上传原理

分片上传是将大文件分割成多个小块，分别上传，每个小块独立传输和确认。这种方式带来了三个重要优势：并行上传提高速度、断点续传节省时间、按需上传节省资源。

并行上传可以充分利用用户的带宽资源。假设用户上传速度是10Mbps，将一个1GB的文件分割成100个10MB的块，可以同时上传10个块，理论上可以将上传时间缩短到原来的十分之一。当然，实际上传速度还受服务器处理能力、网络波动等因素影响。

断点续传对于大文件上传至关重要。用户可能在上传过程中因为网络不稳定、应用闪退、切换网络等原因中断连接。没有断点续传，中断后需要从头开始，用户体验极差。断点续传通过记录已上传的分片信息，中断后只需要从断点继续，而不是重新上传所有内容。

按需上传允许用户先上传部分内容进行预览。例如，直播平台通常允许用户先上传封面图和基本信息，视频内容可以在后台慢慢上传。这降低了用户等待时间，提高了内容发布效率。

分片上传的核心流程如下。首先，客户端向服务器发起初始化请求，服务器返回一个唯一的上传ID和分片大小建议。然后，客户端按照指定大小将文件分割成多个分片，并行上传每个分片，同时记录上传进度。接着，服务器接收分片并存储在临时目录，同时响应每个分片的确认。最后，当所有分片上传完成后，客户端发起合并请求，服务器将所有分片合并为完整文件。

### 2.2 断点续传实现

断点续传的核心是记录分片的上传状态。通常将状态存储在客户端（localStorage或IndexedDB），同时在服务器端也记录状态以支持跨设备续传。

```typescript
// utils/uploadStorage.ts - 本地存储管理
// 本地存储管理模块，负责保存和读取上传分片状态
// 使用 localStorage 存储元数据，实际分片数据存储在 IndexedDB 中

import { UploadMetadata } from '../types/upload';

/**
 * 存储服务类
 * 管理上传任务的本地持久化存储
 */
class UploadStorage {
  // localStorage 键名前缀
  private readonly PREFIX = 'video_upload_';
  // localStorage 中元数据列表的键名
  private readonly META_LIST_KEY = 'video_upload_list';
  // 元数据过期时间（7天）
  private readonly EXPIRE_TIME = 7 * 24 * 60 * 60 * 1000;

  /**
   * 保存上传元数据到本地存储
   * @param metadata 上传元数据对象
   */
  saveMetadata(metadata: UploadMetadata): void {
    // 更新最后访问时间
    metadata.updatedAt = Date.now();

    // 获取现有元数据列表
    const metaList = this.getMetaList();

    // 查找是否已存在该文件的元数据
    const existingIndex = metaList.findIndex(m => m.fileHash === metadata.fileHash);

    if (existingIndex >= 0) {
      // 如果已存在，更新现有条目
      metaList[existingIndex] = metadata;
    } else {
      // 如果不存在，添加新条目
      metaList.push(metadata);
    }

    // 保存更新后的元数据列表
    localStorage.setItem(this.META_LIST_KEY, JSON.stringify(metaList));

    // 保存单独的元数据对象
    localStorage.setItem(
      this.getMetaKey(metadata.fileHash),
      JSON.stringify(metadata)
    );
  }

  /**
   * 获取上传元数据列表
   * @returns 元数据列表
   */
  getMetaList(): UploadMetadata[] {
    const data = localStorage.getItem(this.META_LIST_KEY);
    if (!data) return [];

    try {
      const list: UploadMetadata[] = JSON.parse(data);
      // 清理过期元数据
      const now = Date.now();
      const validList = list.filter(m => now - m.updatedAt < this.EXPIRE_TIME);

      if (validList.length !== list.length) {
        localStorage.setItem(this.META_LIST_KEY, JSON.stringify(validList));
      }

      return validList;
    } catch {
      return [];
    }
  }

  /**
   * 根据文件哈希获取上传元数据
   * @param fileHash 文件唯一标识
   * @returns 上传元数据，如果不存在或已过期返回 null
   */
  getMetadata(fileHash: string): UploadMetadata | null {
    const data = localStorage.getItem(this.getMetaKey(fileHash));
    if (!data) return null;

    try {
      const metadata: UploadMetadata = JSON.parse(data);

      // 检查是否过期
      if (Date.now() - metadata.updatedAt > this.EXPIRE_TIME) {
        this.removeMetadata(fileHash);
        return null;
      }

      return metadata;
    } catch {
      return null;
    }
  }

  /**
   * 移除上传元数据
   * @param fileHash 文件唯一标识
   */
  removeMetadata(fileHash: string): void {
    // 从元数据列表中移除
    const metaList = this.getMetaList();
    const filteredList = metaList.filter(m => m.fileHash !== fileHash);
    localStorage.setItem(this.META_LIST_KEY, JSON.stringify(filteredList));

    // 删除单独的元数据
    localStorage.removeItem(this.getMetaKey(fileHash));
  }

  /**
   * 获取元数据存储键名
   * @param fileHash 文件唯一标识
   * @returns localStorage 键名
   */
  private getMetaKey(fileHash: string): string {
    return `${this.PREFIX}${fileHash}`;
  }

  /**
   * 清理所有过期的上传任务
   */
  clearExpired(): void {
    const metaList = this.getMetaList();
    const now = Date.now();

    const validList = metaList.filter(m => now - m.updatedAt < this.EXPIRE_TIME);
    const expiredList = metaList.filter(m => now - m.updatedAt >= this.EXPIRE_TIME);

    // 更新有效列表
    localStorage.setItem(this.META_LIST_KEY, JSON.stringify(validList));

    // 清理过期条目的独立存储
    expiredList.forEach(m => {
      localStorage.removeItem(this.getMetaKey(m.fileHash));
    });
  }
}

// 导出单例实例
export const uploadStorage = new UploadStorage();
```

```typescript
// utils/chunkUploader.ts - 分片上传核心实现
// 分片上传核心模块，处理文件分割、分片上传、断点续传
// 支持并行上传、进度追踪、错误重试

import { uploadStorage } from './uploadStorage';
import { UploadMetadata, UploadProgress, ChunkConfirm, UploadResult } from '../types/upload';

// 并行上传配置
const PARALLEL_UPLOAD_COUNT = 3;  // 同时上传的分片数量
const CHUNK_SIZE = 2 * 1024 * 1024;  // 分片大小 2MB
const MAX_RETRY_COUNT = 3;  // 单个分片最大重试次数
const RETRY_DELAY = 1000;  // 重试间隔 1秒

/**
 * 分片上传器类
 * 核心功能：文件分片、并行上传、断点续传、进度追踪
 */
export class ChunkUploader {
  // 文件对象
  private file: File;
  // 文件唯一标识（哈希）
  private fileHash: string;
  // 上传元数据
  private metadata: UploadMetadata;
  // 上传状态
  private isPaused: boolean = false;
  // 上传取消标记
  private isCancelled: boolean = false;
  // 当前进行中的上传请求
  private uploadingChunks: Map<number, AbortController> = new Map();
  // 进度回调函数
  private progressCallback?: UploadProgressCallback;
  // 完成回调函数
  private completeCallback?: (result: UploadResult) => void;
  // 错误回调函数
  private errorCallback?: (error: Error) => void;

  /**
   * 构造函数
   * @param file 要上传的文件对象
   * @param fileHash 文件唯一标识（由调用方计算）
   * @param progressCallback 进度回调函数
   */
  constructor(
    file: File,
    fileHash: string,
    progressCallback?: UploadProgressCallback
  ) {
    this.file = file;
    this.fileHash = fileHash;
    this.progressCallback = progressCallback;

    // 计算分片数量
    const totalChunks = Math.ceil(file.size / CHUNK_SIZE);

    // 初始化或恢复元数据
    const existingMeta = uploadStorage.getMetadata(fileHash);
    if (existingMeta && existingMeta.fileSize === file.size) {
      // 恢复已有上传任务的元数据
      this.metadata = existingMeta;
    } else {
      // 创建新的上传任务元数据
      this.metadata = {
        fileHash,
        fileName: file.name,
        fileSize: file.size,
        chunkSize: CHUNK_SIZE,
        totalChunks,
        uploadedChunks: [],
        createdAt: Date.now(),
        updatedAt: Date.now(),
      };
    }
  }

  /**
   * 开始上传
   * @param completeCallback 上传完成回调
   * @param errorCallback 上传失败回调
   */
  async start(
    completeCallback: (result: UploadResult) => void,
    errorCallback?: (error: Error) => void
  ): Promise<void> {
    this.completeCallback = completeCallback;
    this.errorCallback = errorCallback;

    try {
      // 初始化上传任务（向服务器获取上传ID）
      const uploadId = await this.initializeUpload();

      // 获取待上传的分片列表
      const pendingChunks = this.getPendingChunks();

      if (pendingChunks.length === 0) {
        // 所有分片已上传，直接触发完成
        await this.mergeChunks(uploadId);
        return;
      }

      // 并行上传分片
      await this.uploadChunksInParallel(pendingChunks, uploadId);

      // 所有分片上传完成，触发合并
      await this.mergeChunks(uploadId);

    } catch (error) {
      this.errorCallback?.(error instanceof Error ? error : new Error(String(error)));
    }
  }

  /**
   * 暂停上传
   * 暂停后可以调用 start 方法从断点继续
   */
  pause(): void {
    this.isPaused = true;
    // 取消所有进行中的请求
    this.uploadingChunks.forEach(controller => controller.abort());
    this.uploadingChunks.clear();
  }

  /**
   * 恢复上传
   */
  resume(): void {
    this.isPaused = false;
    this.start(this.completeCallback!, this.errorCallback);
  }

  /**
   * 取消上传
   * 取消后会清理本地记录，但已上传的分片需要手动清理
   */
  cancel(): void {
    this.isCancelled = true;
    this.isPaused = false;
    // 取消所有进行中的请求
    this.uploadingChunks.forEach(controller => controller.abort());
    this.uploadingChunks.clear();
    // 清理本地记录
    uploadStorage.removeMetadata(this.fileHash);
  }

  /**
   * 获取上传进度
   * @returns 当前上传进度
   */
  getProgress(): UploadProgress {
    const uploadedBytes = this.metadata.uploadedChunks.length * CHUNK_SIZE;
    const percentage = Math.round((uploadedBytes / this.file.size) * 100);

    return {
      fileHash: this.fileHash,
      uploadedBytes,
      totalBytes: this.file.size,
      percentage,
      currentChunk: this.metadata.uploadedChunks.length,
      speed: 0,  // 实际计算需要记录时间
      remainingTime: 0,
    };
  }

  /**
   * 初始化上传任务
   * @returns 上传ID
   */
  private async initializeUpload(): Promise<string> {
    // 调用后端 API 初始化上传任务
    const response = await fetch('/api/upload/init', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        fileHash: this.fileHash,
        fileName: this.file.name,
        fileSize: this.file.size,
        mimeType: this.file.type,
        totalChunks: this.metadata.totalChunks,
      }),
    });

    if (!response.ok) {
      throw new Error(`初始化上传失败: ${response.status}`);
    }

    const data = await response.json();
    return data.uploadId;
  }

  /**
   * 获取待上传的分片索引列表
   * @returns 待上传的分片索引数组
   */
  private getPendingChunks(): number[] {
    const pending: number[] = [];
    for (let i = 0; i < this.metadata.totalChunks; i++) {
      // 只上传未上传的分片
      if (!this.metadata.uploadedChunks.includes(i)) {
        pending.push(i);
      }
    }
    return pending;
  }

  /**
   * 并行上传分片
   * @param chunks 待上传的分片索引列表
   * @param uploadId 上传ID
   */
  private async uploadChunksInParallel(
    chunks: number[],
    uploadId: string
  ): Promise<void> {
    // 将分片分组，每组并行上传
    for (let i = 0; i < chunks.length; i += PARALLEL_UPLOAD_COUNT) {
      if (this.isCancelled) break;

      // 等待暂停
      while (this.isPaused && !this.isCancelled) {
        await new Promise(resolve => setTimeout(resolve, 100));
      }

      // 获取当前批次的分片
      const batch = chunks.slice(i, i + PARALLEL_UPLOAD_COUNT);

      // 并行上传当前批次
      await Promise.all(
        batch.map(chunkIndex => this.uploadChunk(chunkIndex, uploadId))
      );
    }
  }

  /**
   * 上传单个分片
   * @param chunkIndex 分片索引
   * @param uploadId 上传ID
   * @returns 分片确认结果
   */
  private async uploadChunk(
    chunkIndex: number,
    uploadId: string
  ): Promise<ChunkConfirm> {
    // 计算分片的起始位置和结束位置
    const start = chunkIndex * CHUNK_SIZE;
    const end = Math.min(start + CHUNK_SIZE, this.file.size);

    // 提取分片数据
    const chunk = this.file.slice(start, end);

    // 创建 AbortController 用于取消请求
    const controller = new AbortController();
    this.uploadingChunks.set(chunkIndex, controller);

    let lastError: Error | null = null;

    // 重试机制
    for (let retry = 0; retry < MAX_RETRY_COUNT; retry++) {
      if (this.isCancelled) break;

      try {
        // 构建 FormData
        const formData = new FormData();
        formData.append('uploadId', uploadId);
        formData.append('chunkIndex', String(chunkIndex));
        formData.append('chunk', chunk);

        // 上传分片
        const response = await fetch('/api/upload/chunk', {
          method: 'POST',
          body: formData,
          signal: controller.signal,
        });

        if (!response.ok) {
          throw new Error(`分片上传失败: ${response.status}`);
        }

        const result: ChunkConfirm = await response.json();

        // 上传成功，更新本地记录
        if (result.success && !this.metadata.uploadedChunks.includes(chunkIndex)) {
          this.metadata.uploadedChunks.push(chunkIndex);
          uploadStorage.saveMetadata(this.metadata);
        }

        // 通知进度更新
        this.notifyProgress();

        return result;

      } catch (error) {
        // 如果是取消请求，不重试
        if (error instanceof Error && error.name === 'AbortError') {
          throw error;
        }

        lastError = error instanceof Error ? error : new Error(String(error));

        // 等待后重试
        await new Promise(resolve => setTimeout(resolve, RETRY_DELAY));
      }
    }

    throw lastError || new Error('分片上传失败');
  }

  /**
   * 通知进度更新
   */
  private notifyProgress(): void {
    const progress = this.getProgress();
    this.progressCallback?.(progress);
  }

  /**
   * 合并分片
   * @param uploadId 上传ID
   */
  private async mergeChunks(uploadId: string): Promise<void> {
    // 调用后端 API 合并分片
    const response = await fetch('/api/upload/merge', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        uploadId,
        fileHash: this.fileHash,
        fileName: this.file.name,
      }),
    });

    if (!response.ok) {
      throw new Error(`合并分片失败: ${response.status}`);
    }

    const result: UploadResult = await response.json();

    // 清理本地记录
    uploadStorage.removeMetadata(this.fileHash);

    // 触发完成回调
    this.completeCallback?.(result);
  }
}
```

### 2.3 上传进度实时反馈

上传进度反馈是用户体验的关键。良好的进度反馈应该包含已上传百分比、当前分片、上传速度、预计剩余时间等信息。

```typescript
// components/UploadProgress.tsx - 上传进度展示组件
// 使用 React Hooks 管理上传状态和进度展示
// 支持暂停、恢复、取消操作

"use client";

import React, { useState, useEffect, useCallback, useRef } from 'react';
import { ChunkUploader } from '../utils/chunkUploader';
import { UploadProgress } from '../types/upload';

/**
 * 上传进度条组件属性
 */
interface UploadProgressProps {
  /** 要上传的文件 */
  file: File;
  /** 上传完成回调 */
  onUploadComplete: (url: string) => void;
  /** 上传失败回调 */
  onUploadError?: (error: Error) => void;
  /** 是否自动开始上传 */
  autoStart?: boolean;
}

/**
 * 上传进度状态
 */
type UploadState = 'idle' | 'uploading' | 'paused' | 'completed' | 'error';

/**
 * 上传进度条组件
 * 支持分片上传、暂停、恢复、取消
 */
export const UploadProgressBar: React.FC<UploadProgressProps> = ({
  file,
  onUploadComplete,
  onUploadError,
  autoStart = true,
}) => {
  // 上传器实例
  const uploaderRef = useRef<ChunkUploader | null>(null);

  // 上传状态
  const [state, setState] = useState<UploadState>('idle');
  // 进度数据
  const [progress, setProgress] = useState<UploadProgress | null>(null);
  // 错误信息
  const [error, setError] = useState<string | null>(null);
  // 上传速度历史（用于计算平均值）
  const speedHistoryRef = useRef<number[]>([]);
  // 上传开始时间
  const startTimeRef = useRef<number>(0);

  /**
   * 计算格式化后的上传速度
   * @param speed 速度（字节/秒）
   * @returns 格式化后的速度字符串
   */
  const formatSpeed = useCallback((speed: number): string => {
    if (speed < 1024) {
      return `${speed} B/s`;
    } else if (speed < 1024 * 1024) {
      return `${(speed / 1024).toFixed(1)} KB/s`;
    } else {
      return `${(speed / (1024 * 1024)).toFixed(1)} MB/s`;
    }
  }, []);

  /**
   * 计算格式化后的剩余时间
   * @param remainingTime 剩余时间（秒）
   * @returns 格式化后的时间字符串
   */
  const formatRemainingTime = useCallback((remainingTime: number): string => {
    if (remainingTime < 60) {
      return `${Math.round(remainingTime)} 秒`;
    } else if (remainingTime < 3600) {
      const minutes = Math.floor(remainingTime / 60);
      const seconds = Math.round(remainingTime % 60);
      return `${minutes} 分 ${seconds} 秒`;
    } else {
      const hours = Math.floor(remainingTime / 3600);
      const minutes = Math.floor((remainingTime % 3600) / 60);
      return `${hours} 小时 ${minutes} 分`;
    }
  }, []);

  /**
   * 格式化文件大小
   * @param bytes 字节数
   * @returns 格式化后的文件大小
   */
  const formatFileSize = useCallback((bytes: number): string => {
    if (bytes < 1024) {
      return `${bytes} B`;
    } else if (bytes < 1024 * 1024) {
      return `${(bytes / 1024).toFixed(1)} KB`;
    } else if (bytes < 1024 * 1024 * 1024) {
      return `${(bytes / (1024 * 1024)).toFixed(1)} MB`;
    } else {
      return `${(bytes / (1024 * 1024 * 1024)).toFixed(2)} GB`;
    }
  }, []);

  /**
   * 计算上传速度和剩余时间
   * @param uploadedBytes 已上传字节数
   */
  const calculateSpeedAndTime = useCallback((uploadedBytes: number): void => {
    if (progress && startTimeRef.current > 0) {
      const elapsedTime = (Date.now() - startTimeRef.current) / 1000;
      const currentSpeed = uploadedBytes / elapsedTime;

      // 记录速度历史
      speedHistoryRef.current.push(currentSpeed);
      if (speedHistoryRef.current.length > 10) {
        speedHistoryRef.current.shift();
      }

      // 计算平均速度
      const avgSpeed = speedHistoryRef.current.reduce((a, b) => a + b, 0) /
                       speedHistoryRef.current.length;

      // 计算剩余时间
      const remainingBytes = progress.totalBytes - uploadedBytes;
      const remainingTime = remainingBytes / avgSpeed;

      // 更新进度
      setProgress(prev => prev ? {
        ...prev,
        speed: avgSpeed,
        remainingTime,
      } : null);
    }
  }, [progress]);

  /**
   * 开始上传
   */
  const handleStart = useCallback(async () => {
    if (uploaderRef.current) {
      return;
    }

    // 计算文件哈希（实际应用中应该先计算哈希，用于断点续传）
    const fileHash = await calculateFileHash(file);

    // 创建上传器
    const uploader = new ChunkUploader(file, fileHash, (progressData) => {
      setProgress(progressData);
      calculateSpeedAndTime(progressData.uploadedBytes);
    });

    uploaderRef.current = uploader;
    startTimeRef.current = Date.now();
    speedHistoryRef.current = [];

    setState('uploading');
    setError(null);

    // 开始上传
    uploader.start(
      (result) => {
        // 上传完成
        setState('completed');
        onUploadComplete(result.url);
      },
      (error) => {
        // 上传失败
        setState('error');
        setError(error.message);
        onUploadError?.(error);
      }
    );
  }, [file, onUploadComplete, onUploadError, calculateSpeedAndTime]);

  /**
   * 暂停上传
   */
  const handlePause = useCallback(() => {
    uploaderRef.current?.pause();
    setState('paused');
  }, []);

  /**
   * 恢复上传
   */
  const handleResume = useCallback(() => {
    uploaderRef.current?.resume();
    setState('uploading');
  }, []);

  /**
   * 取消上传
   */
  const handleCancel = useCallback(() => {
    uploaderRef.current?.cancel();
    uploaderRef.current = null;
    setState('idle');
    setProgress(null);
    setError(null);
  }, []);

  /**
   * 自动开始上传
   */
  useEffect(() => {
    if (autoStart && state === 'idle') {
      handleStart();
    }
  }, [autoStart, state, handleStart]);

  /**
   * 清理函数
   */
  useEffect(() => {
    return () => {
      if (uploaderRef.current && state === 'uploading') {
        uploaderRef.current.pause();
      }
    };
  }, []);

  // 渲染状态图标
  const renderStateIcon = () => {
    switch (state) {
      case 'uploading':
        return <span className="animate-spin">⟳</span>;
      case 'paused':
        return <span>⏸</span>;
      case 'completed':
        return <span className="text-green-500">✓</span>;
      case 'error':
        return <span className="text-red-500">✗</span>;
      default:
        return <span>○</span>;
    }
  };

  return (
    <div className="w-full max-w-md p-4 bg-white rounded-lg shadow">
      {/* 文件信息 */}
      <div className="flex items-center gap-3 mb-4">
        <div className="flex-1">
          <p className="font-medium text-gray-900 truncate">{file.name}</p>
          <p className="text-sm text-gray-500">{formatFileSize(file.size)}</p>
        </div>
        {renderStateIcon()}
      </div>

      {/* 进度条 */}
      <div className="mb-4">
        <div className="h-2 bg-gray-200 rounded-full overflow-hidden">
          <div
            className={`h-full transition-all duration-300 ${
              state === 'error' ? 'bg-red-500' :
              state === 'completed' ? 'bg-green-500' : 'bg-blue-500'
            }`}
            style={{ width: `${progress?.percentage || 0}%` }}
          />
        </div>
      </div>

      {/* 进度信息 */}
      <div className="flex justify-between text-sm text-gray-600 mb-4">
        <span>{progress?.percentage || 0}%</span>
        {state === 'uploading' && progress && (
          <>
            <span>{formatSpeed(progress.speed)}</span>
            <span>剩余: {formatRemainingTime(progress.remainingTime)}</span>
          </>
        )}
      </div>

      {/* 操作按钮 */}
      <div className="flex gap-2">
        {state === 'idle' && (
          <button
            onClick={handleStart}
            className="flex-1 px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600"
          >
            开始上传
          </button>
        )}
        {state === 'uploading' && (
          <>
            <button
              onClick={handlePause}
              className="flex-1 px-4 py-2 bg-yellow-500 text-white rounded hover:bg-yellow-600"
            >
              暂停
            </button>
            <button
              onClick={handleCancel}
              className="px-4 py-2 bg-red-500 text-white rounded hover:bg-red-600"
            >
              取消
            </button>
          </>
        )}
        {state === 'paused' && (
          <>
            <button
              onClick={handleResume}
              className="flex-1 px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600"
            >
              继续
            </button>
            <button
              onClick={handleCancel}
              className="px-4 py-2 bg-red-500 text-white rounded hover:bg-red-600"
            >
              取消
            </button>
          </>
        )}
        {state === 'error' && (
          <>
            <button
              onClick={handleStart}
              className="flex-1 px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600"
            >
              重试
            </button>
            <button
              onClick={handleCancel}
              className="px-4 py-2 bg-gray-500 text-white rounded hover:bg-gray-600"
            >
              取消
            </button>
          </>
        )}
      </div>

      {/* 错误信息 */}
      {error && (
        <p className="mt-3 text-sm text-red-500">{error}</p>
      )}
    </div>
  );
};

/**
 * 计算文件的哈希值
 * 使用 File API 读取文件内容并计算 SHA-256 哈希
 * @param file 文件对象
 * @returns 文件哈希值
 */
async function calculateFileHash(file: File): Promise<string> {
  // 使用 Web Crypto API 计算 SHA-256 哈希
  const hashBuffer = await crypto.subtle.digest(
    'SHA-256',
    await file.arrayBuffer()
  );

  // 将 ArrayBuffer 转换为十六进制字符串
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  const hashHex = hashArray.map(b => b.toString(16).padStart(2, '0')).join('');

  return hashHex;
}
```

### 2.4 客户端格式检查

在上传前进行客户端格式检查可以提前发现无效文件，避免浪费带宽。检查内容包括文件类型、文件大小、时长、分辨率等。

```typescript
// utils/videoValidator.ts - 视频格式验证工具
// 客户端视频格式验证，支持文件类型、大小、时长、分辨率等检查

export interface VideoValidationResult {
  // 是否通过验证
  valid: boolean;
  // 错误信息（如果有）
  error?: string;
  // 视频信息
  videoInfo?: VideoInfo;
}

/**
 * 视频信息
 */
export interface VideoInfo {
  // 宽度（像素）
  width: number;
  // 高度（像素）
  height: number;
  // 时长（秒）
  duration: number;
  // 文件大小（字节）
  size: number;
  // 文件类型
  type: string;
  // 帧率
  frameRate: number;
  // 码率（bps）
  bitrate: number;
  // 是否有视频轨道
  hasVideo: boolean;
  // 是否有音频轨道
  hasAudio: boolean;
}

// 支持的视频格式
const SUPPORTED_VIDEO_TYPES = [
  'video/mp4',
  'video/webm',
  'video/quicktime',
  'video/x-msvideo',
  'video/x-matroska',
];

// 支持的视频扩展名
const SUPPORTED_EXTENSIONS = [
  '.mp4', '.webm', '.mov', '.avi', '.mkv', '.m4v', '.3gp'
];

// 最大文件大小（2GB）
const MAX_FILE_SIZE = 2 * 1024 * 1024 * 1024;

// 最小文件大小（100KB）
const MIN_FILE_SIZE = 100 * 1024;

// 最小视频时长（1秒）
const MIN_DURATION = 1;

// 最大视频时长（4小时）
const MAX_DURATION = 4 * 60 * 60;

// 最小分辨率
const MIN_WIDTH = 128;
const MIN_HEIGHT = 128;

/**
 * 验证视频文件
 * @param file 要验证的文件
 * @returns 验证结果
 */
export async function validateVideo(file: File): Promise<VideoValidationResult> {
  // 检查文件是否存在
  if (!file) {
    return { valid: false, error: '请选择文件' };
  }

  // 检查文件大小
  if (file.size < MIN_FILE_SIZE) {
    return { valid: false, error: `文件太小（最小 ${formatFileSize(MIN_FILE_SIZE)}）` };
  }
  if (file.size > MAX_FILE_SIZE) {
    return { valid: false, error: `文件太大（最大 ${formatFileSize(MAX_FILE_SIZE)}）` };
  }

  // 检查文件类型
  const isValidType = SUPPORTED_VIDEO_TYPES.includes(file.type) ||
                      SUPPORTED_EXTENSIONS.some(ext => file.name.toLowerCase().endsWith(ext));
  if (!isValidType) {
    return {
      valid: false,
      error: `不支持的格式（支持：${SUPPORTED_EXTENSIONS.join('、')}）`
    };
  }

  // 获取视频信息
  const videoInfo = await getVideoInfo(file);
  if (!videoInfo) {
    return { valid: false, error: '无法读取视频信息' };
  }

  // 检查是否有视频轨道
  if (!videoInfo.hasVideo) {
    return { valid: false, error: '文件不包含视频轨道' };
  }

  // 检查分辨率
  if (videoInfo.width < MIN_WIDTH || videoInfo.height < MIN_HEIGHT) {
    return {
      valid: false,
      error: `分辨率太低（最小 ${MIN_WIDTH}x${MIN_HEIGHT}）`
    };
  }

  // 检查时长
  if (videoInfo.duration < MIN_DURATION) {
    return { valid: false, error: `视频太短（最短 ${MIN_DURATION} 秒）` };
  }
  if (videoInfo.duration > MAX_DURATION) {
    return { valid: false, error: `视频太长（最长 ${formatDuration(MAX_DURATION)}）` };
  }

  return {
    valid: true,
    videoInfo,
  };
}

/**
 * 获取视频信息
 * @param file 视频文件
 * @returns 视频信息，如果获取失败返回 null
 */
export async function getVideoInfo(file: File): Promise<VideoInfo | null> {
  return new Promise((resolve) => {
    // 创建视频元素
    const video = document.createElement('video');
    video.preload = 'metadata';

    // 创建对象 URL
    const url = URL.createObjectURL(file);

    // 监听元数据加载完成
    video.onloadedmetadata = () => {
      // 获取视频宽高
      const width = video.videoWidth;
      const height = video.videoHeight;
      const duration = video.duration;

      // 释放对象 URL
      URL.revokeObjectURL(url);

      // 尝试获取更多信息（通过 canvas 分析）
      getDetailedVideoInfo(file, url).then(detailedInfo => {
        resolve({
          width,
          height,
          duration,
          size: file.size,
          type: file.type,
          frameRate: detailedInfo.frameRate || 30,
          bitrate: detailedInfo.bitrate || Math.round(file.size * 8 / duration),
          hasVideo: width > 0 && height > 0,
          hasAudio: detailedInfo.hasAudio,
        });
      });
    };

    // 监听错误
    video.onerror = () => {
      URL.revokeObjectURL(url);
      resolve(null);
    };

    // 设置视频源
    video.src = url;
  });
}

/**
 * 获取详细视频信息
 * @param file 视频文件
 * @param url 视频对象 URL
 * @returns 详细视频信息
 */
async function getDetailedVideoInfo(
  file: File,
  url: string
): Promise<{ frameRate: number; bitrate: number; hasAudio: boolean }> {
  return new Promise((resolve) => {
    // 尝试通过 AudioContext 检测音频
    const audioContext = new AudioContext();
    let hasAudio = false;

    // 尝试通过 MediaSource 获取更多信息
    fetch(url).then(response => response.arrayBuffer()).then(buffer => {
      // 简化的检测逻辑
      // 实际应用中可以使用 mp4box.js 等库解析容器格式
      hasAudio = file.type.includes('mp4') || file.type.includes('quicktime');

      resolve({
        frameRate: 30, // 默认值
        bitrate: Math.round(file.size * 8 / (video.duration || 1)),
        hasAudio,
      });

      audioContext.close();
    }).catch(() => {
      resolve({
        frameRate: 30,
        bitrate: 0,
        hasAudio: false,
      });
      audioContext.close();
    });
  });
}

/**
 * 格式化文件大小
 * @param bytes 字节数
 * @returns 格式化后的文件大小
 */
function formatFileSize(bytes: number): string {
  if (bytes < 1024 * 1024) {
    return `${(bytes / 1024).toFixed(1)} KB`;
  } else if (bytes < 1024 * 1024 * 1024) {
    return `${(bytes / (1024 * 1024)).toFixed(1)} MB`;
  } else {
    return `${(bytes / (1024 * 1024 * 1024)).toFixed(2)} GB`;
  }
}

/**
 * 格式化时长
 * @param seconds 秒数
 * @returns 格式化后的时长字符串
 */
function formatDuration(seconds: number): string {
  const hours = Math.floor(seconds / 3600);
  const minutes = Math.floor((seconds % 3600) / 60);
  const secs = Math.floor(seconds % 60);

  if (hours > 0) {
    return `${hours} 小时 ${minutes} 分`;
  } else if (minutes > 0) {
    return `${minutes} 分 ${secs} 秒`;
  } else {
    return `${secs} 秒`;
  }
}

/**
 * 生成视频封面
 * @param file 视频文件
 * @param captureTime 截取时间（秒），默认视频的 1/4 处
 * @param quality 压缩质量（0-1）
 * @returns 封面图片的 Blob 对象
 */
export async function generateVideoCover(
  file: File,
  captureTime?: number,
  quality: number = 0.8
): Promise<Blob | null> {
  return new Promise((resolve) => {
    const video = document.createElement('video');
    video.preload = 'metadata';
    video.muted = true;

    const url = URL.createObjectURL(file);

    video.onloadeddata = () => {
      // 跳转到指定时间
      const seekTime = captureTime ?? video.duration * 0.25;
      video.currentTime = seekTime;
    };

    video.onseeked = () => {
      // 创建 Canvas 绘制当前帧
      const canvas = document.createElement('canvas');
      canvas.width = video.videoWidth;
      canvas.height = video.videoHeight;

      const ctx = canvas.getContext('2d');
      if (ctx) {
        ctx.drawImage(video, 0, 0, canvas.width, canvas.height);

        // 转换为 Blob
        canvas.toBlob(
          (blob) => {
            URL.revokeObjectURL(url);
            resolve(blob);
          },
          'image/jpeg',
          quality
        );
      } else {
        URL.revokeObjectURL(url);
        resolve(null);
      }
    };

    video.onerror = () => {
      URL.revokeObjectURL(url);
      resolve(null);
    };

    video.src = url;
  });
}
```

### 2.5 我的思考：上传体验是用户留存的第一关

视频上传是用户与平台的第一次深度交互。上传体验的好坏直接影响用户是否愿意再次使用平台。一个好的上传体验应该做到以下几点。

首先，无感知的断点续传。用户不应该意识到上传曾经中断过。当网络恢复后，上传应该自动继续，用户甚至不需要点击任何按钮。这需要精心设计的断点续传机制和智能的重试策略。

其次，准确的进度反馈。进度条应该真实反映上传进度，不能出现"进度条走完了还在转圈"的情况。上传速度和剩余时间的预估应该尽可能准确，让用户心里有数。

再次，智能的带宽管理。当用户在上传视频时浏览其他内容，上传不应该严重影响页面加载速度。可以通过动态调整并发数或上传优先级来实现。

最后，贴心的提醒和帮助。当上传失败时，应该给出明确的错误原因和解决方案，而不是简单的"上传失败"。当用户长时间没有操作时，可以提醒用户检查网络。

## 三、视频转码

### 3.1 转码流程与架构

视频转码是视频平台最核心的技术环节之一。转码将用户上传的原始视频转换为适合分发的多种格式和分辨率。转码流程可以概括为：上传完成后触发转码任务，任务进入队列等待处理，处理完成后生成多个清晰度版本，最后更新数据库和CDN缓存。

一个完整的转码流水线包含以下步骤。首先是格式检测，识别原始视频的编码格式、分辨率、码率等信息。然后是质量分析，评估原始视频的质量，确定是否需要降码或增强处理。接着是输出规划，根据目标平台（移动端、PC端、TV端）和网络条件，确定输出哪些分辨率档位。然后是逐个转码，为每个目标分辨率执行编码操作，生成对应文件。完成后是元数据提取，提取关键帧生成缩略图，提取音视频信息入库。最后是分发更新，将转码完成状态通知到CDN，准备提供服务。

转码架构通常采用分布式集群。转码任务由任务调度器分发到各个转码节点。任务调度器根据节点负载情况进行负载均衡，实现任务的合理分配。为了保证转码的实时性，通常会设置多个优先级队列，直播转码优先于普通上传转码，VIP用户优先于普通用户。转码节点通常是CPU和GPU混合配置。CPU适合处理音频和部分通用转码任务，GPU则用于硬编码，可以大幅提升转码速度。

### 3.2 视频编码格式深度解析

视频编码格式决定了视频的压缩效率和画质。主流的视频编码格式有H.264、H.265（HEVC）、AV1三种。

H.264（AVC）是最广泛使用的视频编码标准，兼容性最好，几乎所有设备和浏览器都支持。H.264的压缩效率相对于原始视频可以做到100:1，这意味着一个1GB的原始视频可以压缩到10MB左右。H.264支持多种档位（Profile），从Baseline到High，用于平衡画质和兼容性。

H.265（HEVC）是H.264的继任者，同等画质下可以节省约50%的码率。这意味着在相同带宽下可以传输更高质量的视频，或者在相同画质下使用更少的带宽。H.265的缺点是编码复杂度更高，需要更强的计算能力。不过随着硬件编码的支持，H.265的实用性越来越高。目前主流平台都已支持H.265硬件解码。

AV1是新一代开源视频编码标准，由Alliance for Open Media开发。AV1的压缩效率比H.265再提升约30%，且不收专利费。AV1的缺点是编码速度非常慢，软件编码效率低。不过随着硬件编码器的支持，AV1的实用性正在快速提升。YouTube、Netflix等平台已经开始提供AV1格式。

在实际应用中，平台通常会同时提供多种编码格式的输出，让播放器根据设备能力选择最合适的格式。对于不支持新格式的老设备，降级到H.264保证可用性。

### 3.3 分辨率档位设计

分辨率档位的设计需要考虑用户体验、带宽成本和设备覆盖三个因素。常见的分辨率档位包括以下几种。

流畅（240P或360P），适合网络条件差或移动流量场景。分辨率为426x240或640x360，码率约为500-800kbps。这种档位画质较差，但可以保证基本的可看性。

标清（480P），适合一般网络条件。分辨率为854x480，码率约为1-2Mbps。这是最基本的"清晰"标准。

高清（720P），适合宽带网络，是目前的主流标准。分辨率为1280x720，码率约为2-4Mbps。这个档位可以提供良好的观看体验，大多数用户在此档位已经满意。

超清（1080P），适合高速网络和高质量内容。分辨率为1920x1080，码率约为4-8Mbps。这个档位对带宽要求较高，但画面更加细腻。

2K（1440P），适合大屏幕和高质量内容。分辨率为2560x1440，码率约为8-16Mbps。这个档位主要面向大屏用户和高质量内容。

4K（2160P），适合超大屏幕和旗舰内容。分辨率为3840x2160，码率约为20-40Mbps甚至更高。这个档位带宽成本很高，通常只提供给会员用户或热门内容。

除了固定档位，还需要考虑竖屏视频的特殊性。竖屏视频通常是9:16比例，需要独立的档位设计。竖屏的宽度固定（如720、1080），高度按比例计算（如1280x720对应竖屏720x1280）。

### 3.4 转码队列与调度策略

转码任务调度是保证平台服务质量的关键。合理的调度策略可以提高资源利用率、缩短等待时间、保证关键任务的及时处理。

**队列优先级设计**。通常设置三个优先级：紧急、普通、低优。紧急队列用于直播转码和热点内容，这类内容需要在最短时间内完成。普通队列用于普通用户上传的视频，这是最常见的场景。低优队列用于后台转码和重新转码任务，这类任务可以在系统空闲时处理。

**任务分发策略**。主流的分发策略有两种：负载均衡和任务轮询。负载均衡根据各节点的当前负载（CPU、内存、GPU利用率）分配任务，确保没有节点过载。任务轮询则按顺序将任务分配给各节点，实现负载分散。在实际应用中，通常结合两种策略，先选择负载较低的节点，再进行轮询分配。

**任务亲和性**。对于同一个视频的多个档位转码任务，如果分配到同一个节点，可以共享解码结果，避免重复解码，从而提高效率。这就是任务亲和性的概念。实现上，可以将同一个视频的所有转码任务打包，分配给同一个节点处理。

**超时和重试机制**。每个转码任务都应该有超时时间限制。如果任务超时未完成，说明节点可能出现问题，需要将任务重新分配给其他节点。同时需要记录失败次数，超过阈值的任务标记为失败并通知用户。

```typescript
  '240p': {
    width: 426,
    height: 240,
    videoBitrate: '500k',
    audioBitrate: '64k',

  },

/**
 * 转码服务类
 * 核心功能：视频转码、封面提取、媒体信息获取
 */
@Injectable()
export class TranscodeService {
  private readonly logger = new Logger(TranscodeService.name);

  // 正在运行的任务进程
  private runningProcesses: Map<string, ChildProcess> = new Map();

  // 最大并发转码数
  private readonly MAX_CONCURRENT = 3;

  constructor(
    private readonly configService: ConfigService,
    private readonly queueService: QueueService,
    private readonly storageService: StorageService,
  ) {}

  /**
   * 创建转码任务
   * @param videoId 视频ID
   * @param inputPath 输入文件路径
   * @param resolutions 目标分辨率列表
   * @returns 转码任务列表
   */
  async createTranscodeTasks(
    videoId: string,
    inputPath: string,
    resolutions: string[] = ['480p', '720p', '1080p']
  ): Promise<TranscodeTask[]> {
    const tasks: TranscodeTask[] = [];

    // 获取原始视频信息
    const mediaInfo = await this.getMediaInfo(inputPath);

    // 为每个分辨率创建任务
    for (const resolution of resolutions) {
      const profile = RESOLUTION_PROFILES[resolution];
      if (!profile) {
        this.logger.warn(`未知的分辨率: ${resolution}`);
        continue;
      }

      // 计算实际输出的分辨率（保持宽高比）
      const { width, height } = this.calculateOutputResolution(
        mediaInfo.width,
        mediaInfo.height,
        profile.width,
        profile.height
      );

      // 生成输出路径
      const outputPath = this.generateOutputPath(videoId, resolution);

      const task: TranscodeTask = {
        taskId: `${videoId}_${resolution}`,
        videoId,
        inputPath,
        outputPath,
        resolution: `${width}x${height}`,
        bitrate: profile.videoBitrate,
        codec: 'libx264',
        status: TranscodeStatus.PENDING,
        progress: 0,
        createdAt: new Date(),
      };

      tasks.push(task);

      // 保存任务到数据库
      await this.saveTask(task);
    }

    // 将任务添加到队列
    for (const task of tasks) {
      await this.queueService.addTranscodeTask(task);
    }

    return tasks;
  }

  /**
   * 执行转码任务
   * @param task 转码任务
   */
  async processTask(task: TranscodeTask): Promise<TranscodeResult> {
    this.logger.log(`开始处理转码任务: ${task.taskId}`);

    // 更新任务状态为处理中
    task.status = TranscodeStatus.PROCESSING;
    task.startedAt = new Date();
    await this.updateTask(task);

    // 确保输出目录存在
    const outputDir = path.dirname(task.outputPath);
    await fs.mkdir(outputDir, { recursive: true });

    try {
      // 构建 FFmpeg 命令
      const ffmpegArgs = this.buildFFmpegCommand(task);

      // 执行转码
      const result = await this.executeFFmpeg(task, ffmpegArgs);

      // 获取输出文件信息
      const outputInfo = await this.getMediaInfo(task.outputPath);

      // 提取封面
      const coverPath = await this.extractCover(
        task.videoId,
        task.outputPath
      );

      // 更新任务状态为已完成
      task.status = TranscodeStatus.COMPLETED;
      task.progress = 100;
      task.completedAt = new Date();
      await this.updateTask(task);

      return {
        taskId: task.taskId,
        videoId: task.videoId,
        outputPath: task.outputPath,
        fileSize: outputInfo.size,
        actualBitrate: outputInfo.bitrate,
        actualResolution: { width: outputInfo.width, height: outputInfo.height },
        coverPath,
        status: TranscodeStatus.COMPLETED,
        duration: (task.completedAt.getTime() - task.startedAt.getTime()) / 1000,
      };

    } catch (error) {
      // 更新任务状态为失败
      task.status = TranscodeStatus.FAILED;
      task.error = error instanceof Error ? error.message : String(error);
      task.completedAt = new Date();
      await this.updateTask(task);

      throw error;
    } finally {
      // 清理进程
      this.runningProcesses.delete(task.taskId);
    }
  }

  /**
   * 取消转码任务
   * @param taskId 任务ID
   */
  async cancelTask(taskId: string): Promise<void> {
    const process = this.runningProcesses.get(taskId);
    if (process) {
      process.kill('SIGTERM');
      this.runningProcesses.delete(taskId);
    }

    const task = await this.getTask(taskId);
    if (task) {
      task.status = TranscodeStatus.FAILED;
      task.error = '任务被取消';
      await this.updateTask(task);
    }
  }

  /**
   * 获取媒体信息
   * @param filePath 文件路径
   * @returns 媒体信息
   */
  async getMediaInfo(filePath: string): Promise<{
    width: number;
    height: number;
    duration: number;
    bitrate: number;
    size: number;
    format: string;
  }> {
    return new Promise((resolve, reject) => {
      const ffprobe = spawn('ffprobe', [
        '-v', 'quiet',
        '-print_format', 'json',
        '-show_format',
        '-show_streams',
        filePath,
      ]);

      let output = '';

      ffprobe.stdout.on('data', (data) => {
        output += data.toString();
      });

      ffprobe.on('close', (code) => {
        if (code !== 0) {
          reject(new Error(`ffprobe 失败，退出码: ${code}`));
          return;
        }

        try {
          const info = JSON.parse(output);
          const videoStream = info.streams?.find((s: any) => s.codec_type === 'video');

          if (!videoStream) {
            reject(new Error('未找到视频流'));
            return;
          }

          resolve({
            width: videoStream.width || 0,
            height: videoStream.height || 0,
            duration: parseFloat(info.format?.duration || '0'),
            bitrate: parseInt(info.format?.bit_rate || '0', 10),
            size: parseInt(info.format?.size || '0', 10),
            format: info.format?.format_name || '',
          });
        } catch (error) {
          reject(new Error('解析媒体信息失败'));
        }
      });

      ffprobe.on('error', (error) => {
        reject(error);
      });
    });
  }

  /**
   * 构建 FFmpeg 命令
   * @param task 转码任务
   * @returns FFmpeg 参数数组
   */
  private buildFFmpegCommand(task: TranscodeTask): string[] {
    const profile = Object.values(RESOLUTION_PROFILES).find(
      p => p.width.toString() === task.resolution.split('x')[0]
    ) || RESOLUTION_PROFILES['720p'];

    const args: string[] = [
      '-y',  // 覆盖输出文件
      '-i', task.inputPath,  // 输入文件
      '-c:v', task.codec,  // 视频编码器
      '-b:v', task.bitrate,  // 视频码率
      '-maxrate', `${parseInt(task.bitrate) * 1.5}k`,  // 最大码率
      '-bufsize', task.bitrate,  // 缓冲区大小
      '-s', task.resolution,  // 输出分辨率
      '-c:a', 'aac',  // 音频编码器
      '-b:a', profile.audioBitrate,  // 音频码率
      '-ar', '44100',  // 采样率
      '-movflags', '+faststart',  // 移动优化（moov atom前置）
      '-progress', 'pipe:1',  // 输出进度到 stdout
      task.outputPath,  // 输出文件
    ];

    return args;
  }

  /**
   * 执行 FFmpeg 转码
   * @param task 转码任务
   * @param args FFmpeg 参数
   * @returns 转码结果
   */
  private executeFFmpeg(
    task: TranscodeTask,
    args: string[]
  ): Promise<void> {
    return new Promise((resolve, reject) => {
      this.logger.debug(`执行 FFmpeg: ffmpeg ${args.join(' ')}`);

      const ffmpeg = spawn('ffmpeg', args);
      this.runningProcesses.set(task.taskId, ffmpeg);

      // 处理进度输出
      ffmpeg.stdout.on('data', (data) => {
        const output = data.toString();
        // 解析进度信息
        // 格式: out_time_ms=1234567
        const timeMatch = output.match(/out_time_ms=(\d+)/);
        if (timeMatch && task.startedAt) {
          const currentTime = parseInt(timeMatch[1], 10) / 1000000; // 转换为秒
          const duration = 0; // 需要从 ffprobe 获取
          if (duration > 0) {
            const progress = Math.min(99, Math.round((currentTime / duration) * 100));
            task.progress = progress;
          }
        }
      });

      // 处理错误输出
      ffmpeg.stderr.on('data', (data) => {
        this.logger.debug(`FFmpeg: ${data.toString().trim()}`);
      });

      ffmpeg.on('close', (code) => {
        this.runningProcesses.delete(task.taskId);

        if (code === 0) {
          resolve();
        } else {
          reject(new Error(`FFmpeg 失败，退出码: ${code}`));
        }
      });

      ffmpeg.on('error', (error) => {
        this.runningProcesses.delete(task.taskId);
        reject(error);
      });
    });
  }

  /**
   * 提取视频封面
   * @param videoId 视频ID
   * @param videoPath 视频路径
   * @returns 封面路径
   */
  private async extractCover(
    videoId: string,
    videoPath: string
  ): Promise<string | undefined> {
    const coverDir = path.join(this.storageService.getCoverDir(), videoId);
    await fs.mkdir(coverDir, { recursive: true });

    const coverPath = path.join(coverDir, 'cover.jpg');

    return new Promise((resolve) => {
      const ffmpeg = spawn('ffprobe', [
        '-y',
        '-ss', '00:00:01',  // 从第1秒提取
        '-i', videoPath,
        '-vframes', '1',  // 只提取一帧
        '-q:v', '2',  // 输出质量
        '-f', 'image2',
        coverPath,
      ]);

      ffmpeg.on('close', (code) => {
        if (code === 0) {
          resolve(coverPath);
        } else {
          this.logger.warn(`封面提取失败: ${videoId}`);
          resolve(undefined);
        }
      });

      ffmpeg.on('error', () => {
        resolve(undefined);
      });
    });
  }

  /**
   * 计算输出分辨率（保持宽高比）
   */
  private calculateOutputResolution(
    srcWidth: number,
    srcHeight: number,
    maxWidth: number,
    maxHeight: number
  ): { width: number; height: number } {
    // 如果源分辨率小于目标分辨率，直接返回源分辨率
    if (srcWidth <= maxWidth && srcHeight <= maxHeight) {
      // 确保是偶数（编码要求）
      return {
        width: srcWidth % 2 === 0 ? srcWidth : srcWidth - 1,
        height: srcHeight % 2 === 0 ? srcHeight : srcHeight - 1,
      };
    }

    // 计算缩放比例
    const ratio = Math.min(maxWidth / srcWidth, maxHeight / srcHeight);

    // 计算目标分辨率
    let targetWidth = Math.round(srcWidth * ratio);
    let targetHeight = Math.round(srcHeight * ratio);

    // 确保是偶数
    targetWidth = targetWidth % 2 === 0 ? targetWidth : targetWidth - 1;
    targetHeight = targetHeight % 2 === 0 ? targetHeight : targetHeight - 1;

    return { width: targetWidth, height: targetHeight };
  }

  /**
   * 生成输出文件路径
   */
  private generateOutputPath(videoId: string, resolution: string): string {
    const outputDir = path.join(
      this.storageService.getTranscodeDir(),
      videoId
    );
    return path.join(outputDir, `${resolution}.mp4`);
  }
```

### 3.6 我的思考：转码是视频平台的核心竞争力

转码能力是视频平台最重要的技术壁垒之一。一个好的转码系统应该具备以下特点。

**高效率**。转码是计算密集型任务，转码速度直接影响内容发布速度。通过GPU加速、分布式并行、智能调度等技术手段，可以大幅提高转码效率。同时，合理的档位设计可以减少不必要的转码，只转码用户实际需要的清晰度。

**高质量**。转码后的视频应该在保证画质的前提下尽可能压缩。编码参数的选择（码率、帧率、GOP大小等）需要根据内容特点动态调整。对于运动较快的视频（如体育赛事），需要更高的码率；对于静态内容（如演讲），可以适当降低码率。

**低功耗**。硬件编码器（如NVIDIA NVENC、Intel QuickSync、AMD VCE）比软件编码速度快10倍以上，能耗也更低。但硬件编码器有编码质量上限，对于追求最高质量的内容，可能需要软件编码。

**多格式**。除了H.264，还需要支持H.265、AV1等新格式。AV1的压缩效率最高，但编码速度最慢。对于4K及以上内容，AV1可以显著节省带宽成本。

## 四、视频播放

### 4.1 播放器架构设计

视频播放器是用户与视频内容交互的桥梁。一个优秀的播放器不仅要有出色的播放性能，还要具备良好的扩展性以支持各种业务需求。

播放器架构分为三层：**视频层**、**控制层**和**业务层**。

视频层是整个播放器的核心，基于HTML5 Video标签或WebRTC实现。Video标签提供了基础的视频播放能力，包括播放、暂停、进度控制、音量控制等。WebRTC则用于直播场景的低延迟推拉流。视频层负责与浏览器或系统交互，处理视频解码和渲染。

控制层封装了播放器的核心逻辑，包括播放状态管理、缓冲管理、码率自适应、网络适配等。控制层是播放器的"大脑"，决定了播放器如何应对各种网络状况和用户操作。这一层通常用JavaScript实现，对上屏蔽底层细节，对下调用视频层API。

业务层处理平台特定的业务逻辑，包括广告加载、字幕显示、互动功能（弹幕、礼物等）、数据分析上报等。业务层是平台差异化的关键，不同平台可以在业务层实现各自的特色功能。

以下是一个简化但完整的播放器核心实现。

```typescript
// components/VideoPlayer.tsx - 视频播放器核心组件
// 基于 HTML5 Video 标签构建的播放器核心组件
// 支持多种播放协议、清晰度切换、码率自适应等核心功能

"use client";

import React, { useRef, useState, useEffect, useCallback } from 'react';

/**
 * 播放器状态枚举
 */
export enum PlayerState {
  IDLE = 'idle',           // 空闲状态
  LOADING = 'loading',     // 加载中
  READY = 'ready',         // 准备就绪
  PLAYING = 'playing',     // 播放中
  PAUSED = 'paused',       // 已暂停
  BUFFERING = 'buffering', // 缓冲中
  ENDED = 'ended',         // 播放结束
  ERROR = 'error',         // 错误状态
}

/**
 * 清晰度信息
 */
export interface QualityLevel {
  // 清晰度标识（如 '1080p', '720p', '480p'）
  level: string;
  // 分辨率宽度
  width: number;
  // 分辨率高度
  height: number;
  // 码率（bps）
  bitrate: number;
  // 视频URL
  url: string;
}

/**
 * 播放器配置
 */
export interface PlayerConfig {
  // 视频URL
  src: string;
  // 封面图URL
  poster?: string;
  // 是否自动播放
  autoplay?: boolean;
  // 是否循环播放
  loop?: boolean;
  // 初始音量（0-1）
  volume?: number;
  // 是否静音
  muted?: boolean;
  // 是否显示控制栏
  controls?: boolean;
  // 初始播放位置（秒）
  startTime?: number;
  // 直播模式
  isLive?: boolean;
  // 启用手势
  enableGesture?: boolean;
  // 自适应码率
  enableABR?: boolean;
}

/**
 * 播放器事件回调类型
 */
export interface PlayerCallbacks {
  onStateChange?: (state: PlayerState) => void;
  onProgress?: (currentTime: number, duration: number) => void;
  onBufferChange?: (buffered: number, duration: number) => void;
  onQualityChange?: (quality: QualityLevel) => void;
  onError?: (error: Error) => void;
  onEnded?: () => void;
  onReady?: () => void;
}

/**
 * 播放器核心组件
 * 提供完整的视频播放功能，包括播放控制、清晰度切换、码率自适应等
 */
export const VideoPlayer: React.FC<PlayerConfig & PlayerCallbacks> = ({
  src,
  poster,
  autoplay = false,
  loop = false,
  volume = 1,
  muted = false,
  controls = true,
  startTime = 0,
  isLive = false,
  enableGesture = true,
  enableABR = true,
  onStateChange,
  onProgress,
  onBufferChange,
  onQualityChange,
  onError,
  onEnded,
  onReady,
}) => {
  // 播放器引用
  const containerRef = useRef<HTMLDivElement>(null);
  const videoRef = useRef<HTMLVideoElement>(null);

  // 播放状态
  const [state, setState] = useState(PlayerState.IDLE);
  // 当前时间
  const [currentTime, setCurrentTime] = useState(0);
  // 总时长
  const [duration, setDuration] = useState(0);
  // 缓冲时间
  const [buffered, setBuffered] = useState(0);
  // 音量
  const [volumeState, setVolumeState] = useState(volume);
  // 是否静音
  const [mutedState, setMutedState] = useState(muted);
  // 是否全屏
  const [isFullscreen, setIsFullscreen] = useState(false);
  // 当前清晰度
  const [currentQuality, setCurrentQuality] = useState<QualityLevel | null>(null);
  // 可用清晰度列表
  const [qualityLevels, setQualityLevels] = useState<QualityLevel[]>([]);
  // 是否显示控制栏
  const [showControls, setShowControls] = useState(true);
  // 控制栏自动隐藏定时器
  const controlsTimerRef = useRef<NodeJS.Timeout | null>(null);
  // 码率自适应相关
  const abrRef = useRef<{
    lastTime: number;
    bandwidth: number;
    qualityIndex: number;
  }>({ lastTime: 0, bandwidth: 0, qualityIndex: 0 });

  /**
   * 更新播放器状态
   */
  const updateState = useCallback((newState: PlayerState) => {
    setState(newState);
    onStateChange?.(newState);
  }, [onStateChange]);

  /**
   * 初始化播放器
   */
  useEffect(() => {
    const video = videoRef.current;
    if (!video) return;

    // 监听视频事件
    const handleLoadedMetadata = () => {
      setDuration(video.duration);
      updateState(PlayerState.READY);

      // 如果设置了起始播放位置
      if (startTime > 0) {
        video.currentTime = startTime;
      }

      onReady?.();
    };

    const handleCanPlay = () => {
      updateState(PlayerState.READY);
      if (autoplay) {
        video.play().catch(() => {});
      }
    };

    const handlePlay = () => {
      updateState(PlayerState.PLAYING);
    };

    const handlePause = () => {
      if (state !== PlayerState.ENDED) {
        updateState(PlayerState.PAUSED);
      }
    };

    const handleWaiting = () => {
      updateState(PlayerState.BUFFERING);
    };

    const handlePlaying = () => {
      updateState(PlayerState.PLAYING);
    };

    const handleEnded = () => {
      updateState(PlayerState.ENDED);
      onEnded?.();
    };

    const handleError = () => {
      updateState(PlayerState.ERROR);
      onError?.(new Error('视频加载失败'));
    };

    const handleTimeUpdate = () => {
      setCurrentTime(video.currentTime);
      onProgress?.(video.currentTime, video.duration);
    };

    const handleProgress = () => {
      // 获取缓冲进度
      if (video.buffered.length > 0) {
        const bufferedEnd = video.buffered.end(video.buffered.length - 1);
        setBuffered(bufferedEnd);
        onBufferChange?.(bufferedEnd, video.duration);
      }
    };

    const handleVolumeChange = () => {
      setVolumeState(video.volume);
      setMutedState(video.muted);
    };

    video.addEventListener('loadedmetadata', handleLoadedMetadata);
    video.addEventListener('canplay', handleCanPlay);
    video.addEventListener('play', handlePlay);
    video.addEventListener('pause', handlePause);
    video.addEventListener('waiting', handleWaiting);
    video.addEventListener('playing', handlePlaying);
    video.addEventListener('ended', handleEnded);
    video.addEventListener('error', handleError);
    video.addEventListener('timeupdate', handleTimeUpdate);
    video.addEventListener('progress', handleProgress);
    video.addEventListener('volumechange', handleVolumeChange);

    return () => {
      video.removeEventListener('loadedmetadata', handleLoadedMetadata);
      video.removeEventListener('canplay', handleCanPlay);
      video.removeEventListener('play', handlePlay);
      video.removeEventListener('pause', handlePause);
      video.removeEventListener('waiting', handleWaiting);
      video.removeEventListener('playing', handlePlaying);
      video.removeEventListener('ended', handleEnded);
      video.removeEventListener('error', handleError);
      video.removeEventListener('timeupdate', handleTimeUpdate);
      video.removeEventListener('progress', handleProgress);
      video.removeEventListener('volumechange', handleVolumeChange);
    };
  }, [startTime, autoplay, updateState, onReady, onEnded, onError, onProgress, onBufferChange, state]);

  /**
   * 播放
   */
  const play = useCallback(() => {
    videoRef.current?.play();
  }, []);

  /**
   * 暂停
   */
  const pause = useCallback(() => {
    videoRef.current?.pause();
  }, []);

  /**
   * 跳转
   * @param time 目标时间（秒）
   */
  const seek = useCallback((time: number) => {
    if (videoRef.current) {
      videoRef.current.currentTime = time;
      setCurrentTime(time);
    }
  }, []);

  /**
   * 设置音量
   * @param value 音量值（0-1）
   */
  const setVolume = useCallback((value: number) => {
    if (videoRef.current) {
      videoRef.current.volume = Math.max(0, Math.min(1, value));
      videoRef.current.muted = value === 0;
      setVolumeState(value);
    }
  }, []);

  /**
   * 切换静音
   */
  const toggleMute = useCallback(() => {
    if (videoRef.current) {
      videoRef.current.muted = !videoRef.current.muted;
      setMutedState(videoRef.current.muted);
    }
  }, []);

  /**
   * 全屏切换
   */
  const toggleFullscreen = useCallback(() => {
    const container = containerRef.current;
    if (!container) return;

    if (!document.fullscreenElement) {
      container.requestFullscreen().then(() => {
        setIsFullscreen(true);
      });
    } else {
      document.exitFullscreen().then(() => {
        setIsFullscreen(false);
      });
    }
  }, []);

  /**
   * 切换清晰度
   * @param quality 清晰度对象
   */
  const switchQuality = useCallback((quality: QualityLevel) => {
    if (!videoRef.current) return;

    // 保存当前播放位置
    const currentTime = videoRef.current.currentTime;

    // 切换到新清晰度
    videoRef.current.src = quality.url;
    videoRef.current.currentTime = currentTime;
    videoRef.current.play();

    setCurrentQuality(quality);
    onQualityChange?.(quality);
  }, [onQualityChange]);

  /**
   * 手势处理 - 双击左右区域
   */
  const handleDoubleClick = useCallback((e: React.MouseEvent) => {
    if (!enableGesture || !videoRef.current) return;

    const rect = e.currentTarget.getBoundingClientRect();
    const x = e.clientX - rect.left;
    const isLeftSide = x < rect.width / 2;

    if (isLeftSide) {
      // 左半边双击 - 后退 10 秒
      videoRef.current.currentTime = Math.max(0, videoRef.current.currentTime - 10);
    } else {
      // 右半边双击 - 前进 10 秒
      videoRef.current.currentTime = Math.min(
        duration,
        videoRef.current.currentTime + 10
      );
    }
  }, [enableGesture, duration]);

  /**
   * 控制栏自动隐藏
   */
  const showControlsTemporarily = useCallback(() => {
    setShowControls(true);

    if (controlsTimerRef.current) {
      clearTimeout(controlsTimerRef.current);
    }

    // 3秒后自动隐藏
    controlsTimerRef.current = setTimeout(() => {
      if (state === PlayerState.PLAYING) {
        setShowControls(false);
      }
    }, 3000);
  }, [state]);

  /**
   * 格式化时间显示
   * @param seconds 秒数
   * @returns 格式化的时间字符串 (HH:MM:SS 或 MM:SS)
   */
  const formatTime = useCallback((seconds: number): string => {
    if (!isFinite(seconds) || isNaN(seconds)) return '00:00';

    const hours = Math.floor(seconds / 3600);
    const minutes = Math.floor((seconds % 3600) / 60);
    const secs = Math.floor(seconds % 60);

    if (hours > 0) {
      return `${hours.toString().padStart(2, '0')}:${minutes.toString().padStart(2, '0')}:${secs.toString().padStart(2, '0')}`;
    }
    return `${minutes.toString().padStart(2, '0')}:${secs.toString().padStart(2, '0')}`;
  }, []);

  return (

      <video
        ref={videoRef}
```

### 4.2 清晰度切换实现

清晰度切换是视频平台的核心功能之一。用户可以根据网络条件或个人偏好选择不同的清晰度。好的清晰度切换应该做到无缝过渡，切换过程中不出现明显卡顿或花屏。

实现清晰度切换有几种策略。第一种是直接切换，即暂停播放、切换URL、重新播放。这种方式简单但会有明显的停顿感。第二种是预加载切换，即在新清晰度加载完成前保持旧清晰度播放，加载完成后再切换。这种方式可以实现平滑过渡，但需要播放器支持多Source切换。第三种是基于MSE（Media Source Extensions）的自适应切换，将视频切分成小段，每段独立URL，可以动态调整码率。这种方式是HLS和DASH协议的核心。

以下是基于MSE的清晰度切换实现。

```typescript
// utils/qualitySwitcher.ts - 清晰度切换器
// 基于 Media Source Extensions 的清晰度切换实现
// 支持平滑的无缝清晰度切换

/**
 * 清晰度切换器类
 * 使用 MSE 实现无缝清晰度切换
 * 核心原理：预先加载下一个清晰度的视频分段，下载完成后平滑切换
 */
export class QualitySwitcher {
  // 媒体源扩展
  private mediaSource: MediaSource | null = null;
  // 视频元素
  private video: HTMLVideoElement;
  // 源缓冲区
  private sourceBuffer: SourceBuffer | null = null;
  // 当前清晰度等级
  private currentLevel: number = -1;
  // 可用清晰度列表
  private levels: QualityLevel[] = [];
  // 清晰度切换回调
  private onLevelChange: ((level: QualityLevel) => void) | null = null;
  // 视频分段 URL 生成器
  private segmentUrlGenerator: ((level: number, segmentIndex: number) => string) | null = null;
  // 当前分段索引
  private currentSegmentIndex: number = 0;
  // 是否正在加载
  private isLoading: boolean = false;
  // 预加载的下一个分段数据
  private nextSegmentData: ArrayBuffer | null = null;
  // 预加载的下一个清晰度等级
  private nextLevel: number = -1;

  /**
   * 构造函数
   * @param video 视频元素
   */
  constructor(video: HTMLVideoElement) {
    this.video = video;
  }

  /**
   * 初始化清晰度切换器
   * @param levels 清晰度等级列表
   * @param onLevelChange 清晰度切换回调
   */
  async initialize(
    levels: QualityLevel[],
    onLevelChange: (level: QualityLevel) => void
  ): Promise<void> {
    this.levels = levels;
    this.onLevelChange = onLevelChange;

    // 创建 MediaSource
    if ('MediaSource' in window) {
      this.mediaSource = new MediaSource();
      const url = URL.createObjectURL(this.mediaSource);

      // 监听 MediaSource 事件
      this.mediaSource.addEventListener('sourceopen', () => {
        // 创建 SourceBuffer
        const mimeType = this.getMimeType(levels[0]?.bitrate || 2500000);
        if (MediaSource.isTypeSupported(mimeType)) {
          this.sourceBuffer = this.mediaSource!.addSourceBuffer(mimeType);
          this.setupBufferEvents();
        }
      });

      // 设置视频源
      this.video.src = url;
      this.video.currentLevel = -1; // 使用手动清晰度切换
    }
  }

  /**
   * 获取 MIME 类型
   * @param bitrate 码率
   * @returns MIME 类型字符串
   */
  private getMimeType(bitrate: number): string {
    // 根据码率选择合适的配置文件
    const profile = bitrate > 4000000 ? 'High' : bitrate > 2000000 ? 'Main' : 'Baseline';
    return `video/mp4; codecs="avc1.42E0${profile === 'High' ? '1F' : profile === 'Main' ? '1E' : '1E'}"; profiles=sss; level=30`;
  }

  /**
   * 设置缓冲区事件
   */
  private setupBufferEvents(): void {
    if (!this.sourceBuffer) return;

    // 更新完成事件
    this.sourceBuffer.addEventListener('updateend', () => {
      this.isLoading = false;
      this.loadNextSegment();
    });

    // 错误事件
    this.sourceBuffer.addEventListener('error', (error) => {
      console.error('SourceBuffer 错误:', error);
      this.isLoading = false;
    });
  }

  /**
   * 切换到指定清晰度
   * @param levelIndex 清晰度索引
   */
  async switchToLevel(levelIndex: number): Promise<void> {
    if (levelIndex < 0 || levelIndex >= this.levels.length) {
      throw new Error(`无效的清晰度等级: ${levelIndex}`);
    }

    if (levelIndex === this.currentLevel) {
      return; // 已经是要切换的清晰度
    }

    const targetLevel = this.levels[levelIndex];
    console.log(`切换清晰度: ${this.currentLevel} -> ${levelIndex} (${targetLevel.level})`);

    // 预加载目标清晰度的分段
    await this.preloadNextSegment(levelIndex);

    // 等待当前播放点到达切换边界
    // 实际实现中需要监听 video 的时间更新，找到合适的切换点
    this.performSwitch(levelIndex);
  }

  /**
   * 预加载下一个分段
   * @param levelIndex 清晰度索引
   */
  private async preloadNextSegment(levelIndex: number): Promise<void> {
    if (this.isLoading) return;

    this.isLoading = true;
    this.nextLevel = levelIndex;

    try {
      const segmentIndex = this.currentSegmentIndex;
      const url = this.getSegmentUrl(levelIndex, segmentIndex);

      const response = await fetch(url);
      if (!response.ok) {
        throw new Error(`分段加载失败: ${response.status}`);
      }

      this.nextSegmentData = await response.arrayBuffer();
    } catch (error) {
      console.error('预加载失败:', error);
      this.nextSegmentData = null;
      this.nextLevel = -1;
    }

    this.isLoading = false;
  }

  /**
   * 执行切换
   * @param levelIndex 清晰度索引
   */
  private async performSwitch(levelIndex: number): Promise<void> {
    if (!this.sourceBuffer || !this.nextSegmentData) {
      // 如果没有预加载数据，直接切换（可能有卡顿）
      this.currentLevel = levelIndex;
      const newUrl = this.getSegmentUrl(levelIndex, this.currentSegmentIndex);
      this.video.src = newUrl;
      this.onLevelChange?.(this.levels[levelIndex]);
      return;
    }

    // 将预加载的分段数据追加到缓冲区
    try {
      this.sourceBuffer.appendBuffer(this.nextSegmentData);

      // 切换完成后更新状态
      this.currentLevel = levelIndex;
      this.currentSegmentIndex++;
      this.nextSegmentData = null;

      this.onLevelChange?.(this.levels[levelIndex]);
    } catch (error) {
      console.error('切换失败:', error);
      // 切换失败，尝试直接切换
      this.directSwitch(levelIndex);
    }
  }

  /**
   * 直接切换（无预加载）
   * @param levelIndex 清晰度索引
   */
  private directSwitch(levelIndex: number): void {
    this.currentLevel = levelIndex;
    const newUrl = this.getSegmentUrl(levelIndex, this.currentSegmentIndex);
    this.video.src = newUrl;
    this.onLevelChange?.(this.levels[levelIndex]);
  }

  /**
   * 获取分段 URL
   * @param levelIndex 清晰度索引
   * @param segmentIndex 分段索引
   * @returns 分段 URL
   */
  private getSegmentUrl(levelIndex: number, segmentIndex: number): string {
    const level = this.levels[levelIndex];
    if (!level) {
      throw new Error(`清晰度不存在: ${levelIndex}`);
    }

    // 使用 CDN URL 模板
    // 实际实现中，URL 模板应该由服务端提供
    const template = level.url.replace(/\$segment\$/g, String(segmentIndex));
    return template;
  }

  /**
   * 加载下一个分段
   */
  private async loadNextSegment(): Promise<void> {
    if (!this.sourceBuffer || this.isLoading) return;

    // 如果有预加载数据，先使用预加载数据
    if (this.nextSegmentData && this.nextLevel !== -1) {
      return; // 切换正在进行
    }

    this.isLoading = true;
    this.currentSegmentIndex++;

    try {
      const url = this.getSegmentUrl(this.currentLevel, this.currentSegmentIndex);
      const response = await fetch(url);

      if (!response.ok) {
        throw new Error(`分段加载失败: ${response.status}`);
      }

      const data = await response.arrayBuffer();
      this.sourceBuffer.appendBuffer(data);
    } catch (error) {
      console.error('加载分段失败:', error);
      this.isLoading = false;
    }
  }

  /**
   * 获取当前清晰度索引
   */
  getCurrentLevel(): number {
    return this.currentLevel;
  }

  /**
   * 获取可用清晰度列表
   */
  getAvailableLevels(): QualityLevel[] {
    return this.levels;
  }

  /**
   * 销毁清晰度切换器
   */
  destroy(): void {
    if (this.mediaSource) {
      this.mediaSource.close();
    }
    if (this.mediaSource) {
      URL.revokeObjectURL(this.video.src);
    }
    this.sourceBuffer = null;
    this.mediaSource = null;
  }
}

/**
 * 清晰度信息接口
 */
interface QualityLevel {
  level: string;
  width: number;
  height: number;
  bitrate: number;
  url: string;
}
```

### 4.3 码率自适应算法

码率自适应（Adaptive Bitrate Streaming，ABR）是视频播放体验的核心技术。ABR算法根据当前网络状况动态调整视频清晰度，在画质和流畅度之间取得平衡。

常见的ABR算法有三种：**基于带宽的ABR（BBA-0）**、**基于缓冲的ABR（Buffer-based）**和**基于预测的ABR（Neural ABR）**。

BBA-0是最简单的ABR算法，只根据当前带宽选择清晰度。带宽高时提升清晰度，带宽低时降低清晰度。这种算法响应快，但容易震荡。

基于缓冲的ABR以缓冲健康度为主要决策因素。缓冲充足时可以提升清晰度，缓冲不足时降低清晰度。这种算法更稳定，但响应较慢。

基于预测的ABR使用机器学习模型预测未来带宽，选择合适的清晰度。这种算法效果最好，但实现复杂，需要大量数据训练模型。

以下是一个实用的码率自适应实现。

```typescript
// utils/adaptiveBitrate.ts - 码率自适应控制器
// 基于缓冲和带宽的混合码率自适应算法
// 核心原理：通过监控带宽和缓冲状态，动态选择最合适的清晰度

/**
 * 带宽采样数据
 */
interface BandwidthSample {
  // 采样时间戳
  timestamp: number;
  // 采样大小（字节）
  bytes: number;
  // 下载时长（秒）
  duration: number;
}

/**
 * 码率自适应配置
 */
interface ABRConfig {
  // 初始缓冲目标（秒）
  initialBufferTarget: number;
  // 稳态缓冲目标（秒）
  steadyBufferTarget: number;
  // 缓冲低水位（秒）
  bufferLowWatermark: number;
  // 缓冲高水位（秒）
  bufferHighWatermark: number;
  // 最小清晰度索引
  minLevelIndex: number;
  // 最大清晰度索引
  maxLevelIndex: number;
  // 平滑系数（用于带宽估计）
  bandwidthSmoothingFactor: number;
  // 切换惩罚系数
  switchPenaltyFactor: number;
}

/**
 * 默认配置
 */
const DEFAULT_CONFIG: ABRConfig = {
  initialBufferTarget: 5,      // 初始缓冲 5 秒
  steadyBufferTarget: 30,      // 稳态缓冲 30 秒
  bufferLowWatermark: 2,       // 低水位 2 秒
  bufferHighWatermark: 60,     // 高水位 60 秒
  minLevelIndex: 0,            // 最低清晰度（流畅）
  maxLevelIndex: 5,            // 最高清晰度（4K）
  bandwidthSmoothingFactor: 0.7, // 平滑系数
  switchPenaltyFactor: 1.5,    // 切换惩罚
};

/**
 * 码率自适应控制器
 *
 * 算法核心思想：
 * 1. 维护一个带宽估计器，通过采样最近的网络吞吐量来估计可用带宽
 * 2. 维护一个缓冲管理器，监控当前缓冲健康状况
 * 3. 综合带宽和缓冲状态，选择合适的清晰度
 * 4. 避免频繁切换清晰度，减少视觉抖动
 */
export class AdaptiveBitrateController {
  // 视频元素
  private video: HTMLVideoElement;
  // 清晰度列表
  private levels: QualityLevel[] = [];
  // 当前清晰度索引
  private currentLevelIndex: number = 0;
  // 配置
  private config: ABRConfig;
  // 带宽采样数据
  private bandwidthSamples: BandwidthSample[] = [];
  // 估计带宽（字节/秒）
  private estimatedBandwidth: number = 0;
  // 回调函数
  private onLevelChange: ((level: QualityLevel) => void) | null = null;
  // 最后一次切换时间
  private lastSwitchTime: number = 0;
  // 是否已初始化
  private initialized: boolean = false;
  // 定时器 ID
  private timerId: number | null = null;

  /**
   * 构造函数
   * @param video 视频元素
   * @param config 配置（可选）
   */
  constructor(video: HTMLVideoElement, config?: Partial<ABRConfig>) {
    this.video = video;
    this.config = { ...DEFAULT_CONFIG, ...config };
  }

  /**
   * 初始化 ABR 控制器
   * @param levels 清晰度列表
   * @param onLevelChange 清晰度切换回调
   */
  initialize(
    levels: QualityLevel[],
    onLevelChange: (level: QualityLevel) => void
  ): void {
    this.levels = levels;
    this.onLevelChange = onLevelChange;
    this.currentLevelIndex = 0;
    this.initialized = true;

    // 设置初始清晰度（最低清晰度）
    this.switchToLevel(this.config.minLevelIndex);

    // 监听下载进度
    this.setupProgressListener();

    // 启动自适应循环
    this.startAdaptationLoop();
  }

  /**
   * 设置下载进度监听
   */
  private setupProgressListener(): void {
    // 通过监控 video 的 download 进度来采样带宽
    // 实际实现中可能需要拦截 fetch 或使用 Performance API

    // 使用 requestAnimationFrame 定期检查带宽
    let lastLoaded = 0;
    let lastTime = performance.now();

    const measureBandwidth = () => {
      if (!this.video.src) return;

      const buffered = this.video.buffered;
      if (buffered.length > 0) {
        const currentBufferedEnd = buffered.end(buffered.length - 1);
        const currentTime = performance.now();

        // 简单估算：取最近一次分段下载的速度
        // 实际实现中需要更精确的测量
      }

      requestAnimationFrame(measureBandwidth);
    };

    requestAnimationFrame(measureBandwidth);
  }

  /**
   * 启动自适应循环
   */
  private startAdaptationLoop(): void {
    // 每秒评估一次是否需要切换清晰度
    this.timerId = window.setInterval(() => {
      this.evaluate();
    }, 1000);
  }

  /**
   * 停止自适应循环
   */
  stop(): void {
    if (this.timerId !== null) {
      clearInterval(this.timerId);
      this.timerId = null;
    }
  }

  /**
   * 评估是否需要切换清晰度
   */
  private evaluate(): void {
    if (!this.initialized || this.levels.length === 0) return;

    const bufferLevel = this.getBufferLevel();
    const bandwidth = this.estimateBandwidth();

    // 决定目标清晰度
    let targetLevel = this.decideTargetLevel(bufferLevel, bandwidth);

    // 应用切换惩罚
    targetLevel = this.applySwitchPenalty(targetLevel);

    // 如果与当前清晰度不同，执行切换
    if (targetLevel !== this.currentLevelIndex) {
      this.switchToLevel(targetLevel);
    }
  }

  /**
   * 获取当前缓冲水平（秒）
   */
  private getBufferLevel(): number {
    const buffered = this.video.buffered;
    if (buffered.length === 0) return 0;

    // 计算从当前播放位置到缓冲结束的时间
    const currentTime = this.video.currentTime;
    const bufferedEnd = buffered.end(buffered.length - 1);

    return Math.max(0, bufferedEnd - currentTime);
  }

  /**
   * 估计当前带宽（字节/秒）
   */
  private estimateBandwidth(): number {
    // 如果没有采样数据，返回默认值
    if (this.bandwidthSamples.length === 0) {
      // 假设默认带宽为 2Mbps
      return 250000;
    }

    // 只使用最近 5 秒的采样数据
    const now = Date.now();
    const recentSamples = this.bandwidthSamples.filter(
      s => now - s.timestamp < 5000
    );

    if (recentSamples.length === 0) {
      return this.estimatedBandwidth;
    }

    // 计算加权平均带宽（最近的数据权重更大）
    let totalBandwidth = 0;
    let totalWeight = 0;

    recentSamples.forEach((sample, index) => {
      const weight = index + 1; // 越新的数据权重越大
      totalBandwidth += (sample.bytes / sample.duration) * weight;
      totalWeight += weight;
    });

    const bandwidth = totalBandwidth / totalWeight;

    // 使用平滑系数更新估计带宽
    this.estimatedBandwidth =
      this.config.bandwidthSmoothingFactor * this.estimatedBandwidth +
      (1 - this.config.bandwidthSmoothingFactor) * bandwidth;

    return this.estimatedBandwidth;
  }

  /**
   * 添加带宽采样数据
   * @param bytes 下载字节数
   * @param duration 下载耗时（秒）
   */
  addBandwidthSample(bytes: number, duration: number): void {
    const sample: BandwidthSample = {
      timestamp: Date.now(),
      bytes,
      duration: Math.max(0.1, duration), // 最少 100ms
    };

    this.bandwidthSamples.push(sample);

    // 只保留最近 30 秒的采样
    const cutoff = Date.now() - 30000;
    this.bandwidthSamples = this.bandwidthSamples.filter(s => s.timestamp > cutoff);

    // 更新带宽估计
    this.estimateBandwidth();
  }

  /**
   * 决定目标清晰度
   * @param bufferLevel 当前缓冲水平（秒）
   * @param bandwidth 当前带宽（字节/秒）
   * @returns 目标清晰度索引
   */
  private decideTargetLevel(
    bufferLevel: number,
    bandwidth: number
  ): number {
    // 根据码率列表计算每个清晰度需要的带宽（字节/秒）
    // 带宽需求 = 码率 / 8 * 安全系数
    const bandwidthSafetyFactor = 0.9; // 安全系数，留 10% 余量

    // 从高到低找到第一个带宽足够的清晰度
    for (let i = this.config.maxLevelIndex; i >= this.config.minLevelIndex; i--) {
      const level = this.levels[i];
      if (!level) continue;

      // 带宽需求 = 码率 / 8
      const requiredBandwidth = level.bitrate / 8;

      if (bandwidth >= requiredBandwidth * (1 / bandwidthSafetyFactor)) {
        // 带宽足够，检查缓冲是否允许
        if (bufferLevel > this.config.bufferLowWatermark) {
          return i;
        }
      }
    }

    // 带宽不足或缓冲不足，使用最低清晰度
    return this.config.minLevelIndex;
  }

  /**
   * 应用切换惩罚
   * @param targetLevel 目标清晰度索引
   * @returns 调整后的目标清晰度
   */
  private applySwitchPenalty(targetLevel: number): number {
    const now = Date.now();
    const timeSinceLastSwitch = now - this.lastSwitchTime;

    // 如果距离上次切换不足 10 秒
    if (timeSinceLastSwitch < 10000) {
      // 降清晰度时惩罚更重
      if (targetLevel < this.currentLevelIndex) {
        // 只允许切换到次低的清晰度
        return Math.max(targetLevel, this.currentLevelIndex - 1);
      }
    }

    return targetLevel;
  }

  /**
   * 切换到指定清晰度
   * @param levelIndex 清晰度索引
   */
  private switchToLevel(levelIndex: number): void {
    if (levelIndex < 0 || levelIndex >= this.levels.length) {
      console.warn(`无效的清晰度索引: ${levelIndex}`);
      return;
    }

    if (levelIndex === this.currentLevelIndex) {
      return;
    }

    const newLevel = this.levels[levelIndex];
    console.log(`ABR 切换清晰度: ${this.currentLevelIndex} -> ${levelIndex} (${newLevel.level})`);

    this.currentLevelIndex = levelIndex;
    this.lastSwitchTime = Date.now();

    // 触发清晰度切换回调
    this.onLevelChange?.(newLevel);
  }

  /**
   * 获取当前清晰度索引
   */
  getCurrentLevelIndex(): number {
    return this.currentLevelIndex;
  }

  /**
   * 获取当前清晰度信息
   */
  getCurrentLevel(): QualityLevel | null {
    return this.levels[this.currentLevelIndex] || null;
  }

  /**
   * 获取估计带宽（bps）
   */
  getEstimatedBandwidth(): number {
    return this.estimatedBandwidth * 8; // 转换为 bps
  }

  /**
   * 销毁控制器
   */
  destroy(): void {
    this.stop();
    this.bandwidthSamples = [];
    this.initialized = false;
  }
}

/**
 * 清晰度信息
 */
interface QualityLevel {
  level: string;
  width: number;
  height: number;
  bitrate: number;
  url: string;
}
```

### 4.4 我的思考：播放体验是留住用户的关键

视频播放体验是视频平台最核心的竞争力之一。用户对平台的忠诚度很大程度上取决于播放体验——加载快不快、画面清不清晰、卡不卡顿。

播放体验的优化是一个系统工程，需要从多个层面入手。

**首帧时间**。从用户点击播放到看到第一帧画面的时间，这是用户最敏感的等待时刻。优化首帧时间的关键在于减少DNS解析、TCP连接、视频下载的时间。预加载、预连接、CDN优化都能有效缩短首帧时间。

**卡顿率**。播放过程中出现卡顿的频率，直接影响观看体验。卡顿的主要原因是带宽不足以支撑当前码率，或者缓冲不足。通过码率自适应算法，可以在网络波动时及时调整码率，避免卡顿。

**画质稳定性**。频繁的清晰度切换会让用户感到不适。好的ABR算法应该尽量保持清晰度稳定，只在必要时才切换。切换时也要尽量平滑过渡，减少视觉抖动。

**Seek响应**。用户拖动进度条后多久能继续播放。Seek操作需要下载指定位置的视频数据，如果缓冲不足就需要等待加载。通过预加载和智能缓冲策略可以优化Seek响应。

## 五、CDN分发

### 5.1 CDN架构解析

CDN（Content Delivery Network，内容分发网络）是视频分发的核心技术基础设施。CDN通过在全球部署边缘节点，将内容缓存到离用户最近的地方，从而减少传输延迟和带宽成本。

CDN的架构分为三层：**中心节点**、**边缘节点**和**用户请求路由**。

中心节点是CDN的核心，通常位于一线城市数据中心，存储内容的完整副本。中心节点负责内容的注入和更新，以及协调边缘节点。当边缘节点缓存未命中时，会回源到中心节点获取内容。

边缘节点是CDN的分发前端，部署在各地ISP机房或数据中心。边缘节点数量众多，通常有数百甚至数千个。用户请求被路由到最近的边缘节点，如果该节点有缓存内容就直接返回，否则回源到更近的上级节点或中心节点获取。

用户请求路由是CDN的"大脑"，负责将用户请求导向最优的边缘节点。传统的DNS解析通过用户IP判断地理位置，但精度有限。Anycast技术让多个节点使用同一个IP，用户请求会被路由到最近的路由器，从而实现更精确的路由。HTTP DNS（如阿里云HTTPDNS、腾讯云HTTPDNS）通过HTTP接口返回最优节点IP，精度更高，还能避免DNS劫持。

CDN的工作流程如下。用户发起视频请求，首先进行DNS解析或HTTPDNS查询，获取最优边缘节点IP。然后用户直接请求边缘节点。如果边缘节点有缓存内容（命中），直接返回。如果边缘节点没有缓存（未命中），则回源到上级节点或源站获取内容，同时将内容缓存到本地。最后，后续的用户请求可以直接从边缘节点获取，无需回源。

### 5.2 缓存策略设计

缓存策略是CDN性能的关键。好的缓存策略可以提高缓存命中率，减少回源流量，加快内容分发速度。

**缓存时间设计**。不同类型的视频内容应该有不同的缓存策略。热门视频（热点内容）缓存时间应该较长，因为会有大量用户访问。冷门视频缓存时间可以较短，避免占用过多缓存空间。用户上传的新视频缓存时间应该较短或不缓存，确保内容尽快更新。转码完成的视频可以设置较长的缓存时间。

**缓存粒度设计**。缓存可以按URL、清晰度、用户等维度进行隔离。同一视频的不同清晰度是独立的URL，应该独立缓存。用户个性化内容（如付费视频、会员专属）不应该被公共缓存，应该设置私有缓存或使用no-cache策略。

**缓存刷新策略**。当视频内容更新时，需要及时刷新CDN缓存，否则用户可能看到旧内容。被动刷新通过设置较短的max-age实现缓存过期。主动刷新在内容更新时主动调用CDN的刷新接口清除缓存。批量刷新用于大规模内容变更场景。

**缓存层次设计**。边缘节点缓存时间较短（如1小时），中心节点缓存时间较长（如24小时）。这样的分层设计可以让热点内容尽可能留在边缘，减少回源压力。

### 5.3 防盗链机制

防盗链是保护视频内容不被非法盗用的重要手段。常见的防盗链方案有三种：**Referer防盗链**、**URL鉴权**和**DRM数字版权管理**。

Referer防盗链是最简单的方式，通过检查HTTP请求的Referer头来判断请求来源是否合法。这种方式实现简单，但Referer头可以被伪造或被浏览器strip，安全性较低。Referer防盗链适合作为基础防护，配合其他方案使用。

URL鉴权是目前最常用的防盗链方案。用户在请求视频URL时，服务器生成一个带签名和过期时间的特殊URL。CDN节点验证签名后才返回内容，即使攻击者获取了URL也无法传播，因为URL会在短时间内过期。URL鉴权有多种算法，常见的有_md5_算法和_aes_算法。

DRM是最高级别的版权保护，用于高价值内容如电影、电视剧、会员专享内容。DRM通过加密视频内容，只有授权的客户端才能解密播放。主流的DRM方案有Google的Widevine（用于Chrome、Android）、Apple的FairPlay（用于Safari、iOS）和Microsoft的PlayReady（用于Edge）。DRM需要内容提供商和设备厂商的许可，实施成本较高。

```typescript
  private async getFairPlayCertificate(): Promise<string> {
    if (!this.config.fairplayCertificateUrl) {
      throw new Error('未配置 FairPlay 证书 URL');
    }

    const response = await fetch(this.config.fairplayCertificateUrl);
    if (!response.ok) {
      throw new Error(`证书获取失败: ${response.status}`);
    }

    const buffer = await response.arrayBuffer();
    return btoa(String.fromCharCode(...new Uint8Array(buffer)));
  }

  /**
   * 加载外部 DRM 密钥
   * 用于测试或离线许可证场景
   */
  async loadExternalKey(keyInfo: DRMKeyInfo): Promise<void> {
    if (!this.mediaKeys) {
      throw new Error('MediaKeys 未初始化');
    }

    const session = this.mediaKeys.createSession();

    // 构建 PSSH（Protection System Specific Header）
    const pssh = this.buildPSSH(keyInfo.keyId);

    await session.generateRequest('cenc', pssh);

    // 监听消息并更新密钥
    session.addEventListener('message', async (event) => {
      // 使用外部密钥数据直接更新会话
      const key = Uint8Array.from(keyInfo.key, h => parseInt(h, 16));
      await session.update(key.buffer);
    });
  }

  /**
   * 构建 PSSH 数据
   */
  private buildPSSH(keyId: string): ArrayBuffer {
    // 简化的 PSSH 构建
    // 实际实现应该使用标准的 PSSH 格式
    const systemId = new Uint8Array([
      0x10, 0x77, 0xEF, 0xEC, 0xC0, 0xB2, 0x4D, 0x02,
      0xAC, 0xE3, 0x3C, 0x1E, 0x52, 0xE2, 0xFB, 0x2B
    ]);

    const keyIdBytes = Uint8Array.from(keyId.replace(/-/g, ''), h => parseInt(h, 16));

    const dataSize = 4 + systemId.length + 4 + 4 + keyIdBytes.length;
    const buffer = new ArrayBuffer(4 + dataSize);
    const view = new DataView(buffer);

    // PSSH box
    view.setUint32(0, buffer.byteLength);
    view.setUint32(4, 0x70737368); // 'pssh'
    view.setUint32(8, 0x01000000); // version 1
    view.setUint32(12, 0); // flags

    // System ID
    new Uint8Array(buffer, 16).set(systemId);

    // Key ID count
    view.setUint32(32, 1);

    // Key ID length (16)
    view.setUint32(36, 16);

    // Key ID
    new Uint8Array(buffer, 40).set(keyIdBytes);

    return buffer;
  }

  /**
   * 销毁 DRM 管理器
   */
  async destroy(): Promise<void> {
    if (this.videoElement && this.mediaKeys) {
      await this.videoElement.setMediaKeys(null);
    }
    this.mediaKeys = null;
    this.videoElement = null;
  }
}
```

### 5.4 我的思考：CDN是视频分发的基石

CDN是视频平台最重要的基础设施之一。一个视频平台可以不自己开发播放器、不自己实现转码，但不可能不依赖CDN。CDN的质量直接影响用户的播放体验。

选择CDN需要考虑几个因素。**节点覆盖**决定了用户能否就近访问。CDN节点应该覆盖平台主要用户群体所在的地区。**带宽成本**是视频平台的主要成本之一。不同CDN提供商的计费模式和价格差异很大，需要根据实际流量模式选择。**稳定性**决定了服务可用性。CDN是7x24小时的服务，任何故障都会直接影响用户体验。**功能支持**如防盗链、URL鉴权、HLS/DASH支持、HTTP/2支持等，都是实际运营中需要的。

对于大型视频平台，通常会采用多CDN策略，同时使用多家CDN服务商，通过智能DNS或负载均衡设备进行流量调度。这样可以避免单一CDN故障导致的服务中断，也可以根据不同CDN的优势分配不同类型的流量。

## 六、短视频系统

### 6.1 竖屏沉浸式体验

短视频的核心体验是竖屏沉浸式滑动。用户打开视频后，视频占据整个屏幕，通过上下滑动切换到下一个视频。这种体验模仿了现实中翻阅杂志或卡片的感受，非常适合手机用户的单手操作。

竖屏短视频有几个设计要点。全屏显示是基础，视频应该填满整个屏幕，不留黑边。自动播放是必须的，用户进入页面后应该立即开始播放，无需点击。声音默认播放或静音取决于产品策略，抖音默认静音但会提示用户点击开启声音，快手则默认有声音。滑动切换通过监听触摸手势实现，上滑切换到下一个视频，下滑或点击返回按钮回到上一级。

性能优化是短视频系统的关键。由于需要预加载下一个甚至多个视频，内存管理变得尤为重要。当用户快速滑动时，需要确保视频切换流畅，不出现卡顿或黑屏。首帧时间是另一个关键指标，用户应该几乎感觉不到视频加载的过程。

### 6.2 无限滑动列表实现

无限滑动是短视频App的核心交互。用户可以连续滑动数十甚至数百个视频，列表永远不会穷尽。这种体验需要特殊的技术实现。

传统的无限滚动（Infinite Scroll）通过监听滚动事件，当用户滚动到底部时加载更多数据。但短视频场景不同，用户可能快速连续滑动，需要预加载多个视频。

更好的方案是使用虚拟列表（Virtual List）。虚拟列表只渲染可见区域的视频，当用户滑动时动态更新可见区域。对于短视频场景，可见区域通常只有1-2个视频，内存占用极低。

```typescript
      setHasMore(result.hasMore);

      setIsLoading(false);

      // 点击行为
      return;

      // 上滑 - 下一个视频

        setCurrentIndex(prev => prev + 1);
        preloadIndexRef.current = currentIndex + 2;
      } else if (hasMore) {
        // 加载更多
        loadVideos();
      }
    } else {
      // 下滑 - 上一个视频
      if (currentIndex > 0) {
        setCurrentIndex(prev => prev - 1);
      }
    }

    touchEndY.current = e.touches[0].clientY;

   * 处理触摸移动

    touchEndY.current = e.touches[0].clientY;

   * 视频播放结束

  const handleVideoEnded = useCallback(() => {
    // 自动播放下一个

      setCurrentIndex(prev => prev + 1);
    } else if (hasMore) {
      loadVideos();
    }
  }, [currentIndex, videos.length, hasMore, loadVideos]);

  useEffect(() => {
    loadVideos();

   * 预加载下一个视频
```

### 6.3 缓存管理策略

短视频场景下的缓存管理非常重要。用户滑动浏览视频时，已经看过的视频可能被再次观看（回看），这些视频应该被缓存以提供即时播放体验。同时，缓存空间有限，需要淘汰策略来管理存储。

短视频缓存策略的核心是LRU（Least Recently Used，最近最少使用）。最近播放的视频保留在缓存中，长时间未访问的视频被淘汰。同时需要考虑视频大小、时长、清晰度等因素。

```typescript
// utils/videoCache.ts - 视频缓存管理器
// 基于 LRU 的视频缓存管理，支持预加载、容量控制、持久化

/**
 * 缓存条目
 */
interface CacheEntry {
  // 视频 ID
  videoId: string;
  // 视频 URL
  url: string;
  // 缓存数据
  blob: Blob;
  // 大小（字节）
  size: number;
  // 创建时间
  createdAt: number;
  // 最后访问时间
  lastAccessedAt: number;
  // 访问次数
  accessCount: number;
}

/**
 * 缓存配置
 */
interface CacheConfig {
  // 最大缓存大小（字节），默认 500MB
  maxSize: number;
  // 最大缓存条目数
  maxEntries: number;
  // 单个视频最大缓存大小（字节），默认 100MB
  maxEntrySize: number;
  // 缓存过期时间（毫秒），默认 24 小时
  expireTime: number;
  // 是否持久化到 IndexedDB
  persist: boolean;
}

/**
 * 默认配置
 */
const DEFAULT_CONFIG: CacheConfig = {
  maxSize: 500 * 1024 * 1024, // 500MB
  maxEntries: 100,
  maxEntrySize: 100 * 1024 * 1024, // 100MB
  expireTime: 24 * 60 * 60 * 1000, // 24小时
  persist: true,
};

/**
 * 视频缓存管理器
 * 使用 LRU 策略管理视频缓存，支持预加载和持久化
 */
class VideoCacheManager {
  private cache: Map<string, CacheEntry> = new Map();
  private config: CacheConfig;
  private currentSize: number = 0;
  private db: IDBDatabase | null = null;

  constructor(config: Partial<CacheConfig> = {}) {
    this.config = { ...DEFAULT_CONFIG, ...config };
    this.initIndexedDB();
  }

  /**
   * 初始化 IndexedDB 用于持久化
   */
  private async initIndexedDB(): Promise<void> {
    if (!this.config.persist) return;

    return new Promise((resolve, reject) => {
      const request = indexedDB.open('VideoCacheDB', 1);

      request.onerror = () => reject(request.error);

      request.onsuccess = () => {
        this.db = request.result;
        this.loadFromDB();
        resolve();
      };

      request.onupgradeneeded = (event) => {
        const db = (event.target as IDBOpenDBRequest).result;

        // 创建对象存储
        if (!db.objectStoreNames.contains('videoCache')) {
          const store = db.createObjectStore('videoCache', { keyPath: 'videoId' });
          store.createIndex('lastAccessedAt', 'lastAccessedAt', { unique: false });
          store.createIndex('createdAt', 'createdAt', { unique: false });
        }
      };
    });
  }

  /**
   * 从 IndexedDB 加载缓存
   */
  private async loadFromDB(): Promise<void> {
    if (!this.db) return;

    return new Promise((resolve) => {
      const transaction = this.db!.transaction(['videoCache'], 'readonly');
      const store = transaction.objectStore('videoCache');
      const request = store.getAll();

      request.onsuccess = () => {
        const entries = request.result as CacheEntry[];

        entries.forEach(entry => {
          // 检查是否过期
          if (Date.now() - entry.lastAccessedAt > this.config.expireTime) {
            // 删除过期条目
            this.deleteFromDB(entry.videoId);
          } else {
            // 恢复缓存
            this.cache.set(entry.videoId, entry);
            this.currentSize += entry.size;
          }
        });

        // 如果超过最大容量，清理一部分
        this.evictIfNeeded();
        resolve();
      };

      request.onerror = () => resolve();
    });
  }

  /**
   * 获取缓存的blob
   * @param videoId 视频 ID
   * @returns 缓存的 Blob 对象
   */
  async get(videoId: string): Promise<Blob | null> {
    const entry = this.cache.get(videoId);

    if (!entry) {
      // 尝试从 IndexedDB 获取
      const blob = await this.getFromDB(videoId);
      if (blob) {
        // 重新添加到内存缓存
        await this.set(videoId, blob);
        return blob;
      }
      return null;
    }

    // 更新访问时间和次数
    entry.lastAccessedAt = Date.now();
    entry.accessCount++;

    return entry.blob;
  }

  /**
   * 设置缓存
   * @param videoId 视频 ID
   * @param blob 视频 Blob
   * @param url 原始 URL
   */
  async set(videoId: string, blob: Blob, url?: string): Promise<void> {
    // 检查大小是否超过单个条目限制
    if (blob.size > this.config.maxEntrySize) {
      console.warn(`视频 ${videoId} 大小超过限制，跳过缓存`);
      return;
    }

    // 如果已存在，先删除
    if (this.cache.has(videoId)) {
      const existing = this.cache.get(videoId)!;
      this.currentSize -= existing.size;
      this.cache.delete(videoId);
    }

    // 检查是否需要淘汰
    await this.evictIfNeeded();

    // 创建新条目
    const entry: CacheEntry = {
      videoId,
      url: url || '',
      blob,
      size: blob.size,
      createdAt: Date.now(),
      lastAccessedAt: Date.now(),
      accessCount: 1,
    };

    // 添加到缓存
    this.cache.set(videoId, entry);
    this.currentSize += blob.size;

    // 持久化到 IndexedDB
    await this.saveToDB(entry);
  }

  /**
   * 预加载视频
   * @param videoId 视频 ID
   * @param url 视频 URL
   */
  async preload(videoId: string, url: string): Promise<void> {
    // 如果已经在缓存中，跳过
    if (this.cache.has(videoId)) {
      return;
    }

    try {
      const response = await fetch(url, {
        method: 'GET',
        headers: { 'Range': 'bytes=0-1048576' } // 只下载前 1MB 用于快速预览
      });

      if (!response.ok && response.status !== 206) {
        throw new Error(`预加载失败: ${response.status}`);
      }

      // 全量下载（实际应该只预加载一部分）
      const fullResponse = await fetch(url);
      const blob = await fullResponse.blob();

      await this.set(videoId, blob, url);
    } catch (error) {
      console.error(`预加载视频失败 ${videoId}:`, error);
    }
  }

  /**
   * 检查是否已缓存
   * @param videoId 视频 ID
   */
  has(videoId: string): boolean {
    const entry = this.cache.get(videoId);

    if (!entry) return false;

    // 检查是否过期
    if (Date.now() - entry.lastAccessedAt > this.config.expireTime) {
      this.delete(videoId);
      return false;
    }

    return true;
  }

  /**
   * 删除缓存条目
   * @param videoId 视频 ID
   */
  delete(videoId: string): void {
    const entry = this.cache.get(videoId);

    if (entry) {
      this.currentSize -= entry.size;
      this.cache.delete(videoId);
      this.deleteFromDB(videoId);
    }
  }

  /**
   * 清空所有缓存
   */
  clear(): void {
    this.cache.forEach(entry => {
      this.deleteFromDB(entry.videoId);
    });
    this.cache.clear();
    this.currentSize = 0;
  }

  /**
   * 获取缓存统计信息
   */
  getStats(): {
    entryCount: number;
    totalSize: number;
    maxSize: number;
    usagePercent: number;
  } {
    return {
      entryCount: this.cache.size,
      totalSize: this.currentSize,
      maxSize: this.config.maxSize,
      usagePercent: Math.round((this.currentSize / this.config.maxSize) * 100),
    };
  }

  /**
   * 如果需要，淘汰缓存条目
   */
  private async evictIfNeeded(): Promise<void> {
    // 淘汰超过容量的条目
    while (this.currentSize > this.config.maxSize && this.cache.size > 0) {
      await this.evictLRU();
    }

    // 淘汰超过条目数限制的
    while (this.cache.size > this.config.maxEntries && this.cache.size > 0) {
      await this.evictLRU();
    }
  }

  /**
   * 淘汰最近最少使用的条目
   */
  private async evictLRU(): Promise<void> {
    let oldest: CacheEntry | null = null;
    let oldestId: string | null = null;

    this.cache.forEach((entry, id) => {
      if (!oldest || entry.lastAccessedAt < oldest.lastAccessedAt) {
        oldest = entry;
        oldestId = id;
      }
    });

    if (oldestId) {
      await this.evict(oldestId);
    }
  }

  /**
   * 淘汰指定 ID 的条目
   */
  private async evict(videoId: string): Promise<void> {
    const entry = this.cache.get(videoId);

    if (entry) {
      this.currentSize -= entry.size;
      this.cache.delete(videoId);
      await this.deleteFromDB(videoId);
    }
  }

  /**
   * 保存到 IndexedDB
   */
  private async saveToDB(entry: CacheEntry): Promise<void> {
    if (!this.db) return;

    return new Promise((resolve) => {
      const transaction = this.db!.transaction(['videoCache'], 'readwrite');
      const store = transaction.objectStore('videoCache');
      store.put(entry);
      transaction.oncomplete = () => resolve();
      transaction.onerror = () => resolve();
    });
  }

  /**
   * 从 IndexedDB 获取
   */
  private async getFromDB(videoId: string): Promise<Blob | null> {
    if (!this.db) return null;

    return new Promise((resolve) => {
      const transaction = this.db!.transaction(['videoCache'], 'readonly');
      const store = transaction.objectStore('videoCache');
      const request = store.get(videoId);

      request.onsuccess = () => {
        const entry = request.result as CacheEntry | undefined;
        resolve(entry?.blob || null);
      };

      request.onerror = () => resolve(null);
    });
  }

  /**
   * 从 IndexedDB 删除
   */
  private async deleteFromDB(videoId: string): Promise<void> {
    if (!this.db) return;

    return new Promise((resolve) => {
      const transaction = this.db!.transaction(['videoCache'], 'readwrite');
      const store = transaction.objectStore('videoCache');
      store.delete(videoId);
      transaction.oncomplete = () => resolve();
      transaction.onerror = () => resolve();
    });
  }
}
```

### 6.4 我的思考：短视频体验的核心是流畅

短视频产品的核心竞争力是"刷"的体验——流畅、无停顿、无感知加载。要做到这一点，需要在多个层面进行优化。

首先是预加载。用户还没滑到下一个视频时，后台已经在下载了。预加载的范围需要根据用户滑动方向和速度动态调整。快速下滑时多预加载几个，暂停滑动时减少预加载以节省带宽。

其次是首帧时间。视频的首帧应该几乎立即显示。这需要视频元数据（moov atom）前置，CDN缓存热点内容，以及适当的预连接。

再次是内存管理。短视频列表可能很长，如果不加控制内存会爆炸。虚拟列表确保只渲染可见区域的视频，已播放视频的Blob URL需要及时释放。

最后是缓存策略。用户回看是常见行为，已看过的视频应该被缓存。但缓存空间有限，需要智能的淘汰策略。可以结合访问频率、时长、清晰度等因素综合判断。

## 七、直播系统

### 7.1 推拉流协议

直播系统涉及两套核心协议：**推流协议**和**拉流协议**。

推流协议用于将主播端采集的视频流发送到服务器。RTMP（Real-Time Messaging Protocol）是目前最主流的推流协议，基于TCP实现，稳定可靠，延迟约2-5秒。RTMP的优势是成熟稳定、兼容性好，几乎所有推流软件和硬件都支持。WebRTC是新一代协议，基于UDP实现，延迟可以低到500ms以内，但复杂度高，主要用于连麦等低延迟场景。SRT（Secure Reliable Transport）是新兴协议，基于UD设计，在不稳定网络下表现更好。

拉流协议用于将直播流分发给观众。HLS（HTTP Live Streaming）是苹果主推的协议，将视频切分成小TS分段，通过HTTP传输。HLS延迟较高，通常10-30秒，但兼容性极好，任何设备都可以直接播放。DASH（Dynamic Adaptive Streaming over HTTP）是类似HLS的协议，标准化程度更高。FLV（Flash Video）基于HTTP-FLV，延迟约3-8秒，在PC浏览器端有较好支持。LL-HLS（Low Latency HLS）是HLS的低延迟版本，可以做到2-5秒延迟，是苹果主推的方向。

协议选择需要根据场景决定。秀场直播、电商直播对延迟要求不高（5-10秒可接受），可以选择HLS保证兼容性。游戏直播、电竞直播需要低延迟（1-3秒），可以选择HTTP-FLV或LL-HLS。连麦互动需要极低延迟（500ms以内），必须使用WebRTC。

```typescript
// utils/LiveSDK.ts - 直播 SDK
// 支持推流（RTMP/WebRTC）和拉流（HLS/FLV/WebRTC）

/**
 * 直播 SDK 配置
 */
interface LiveSDKConfig {
  // SDK 类型：'publisher'（推流）或 'player'（拉流）
  mode: 'publisher' | 'player';
  // 推流地址（推流模式）
  publishUrl?: string;
  // 拉流地址（拉流模式）
  playbackUrl?: string;
  // 视频元素
  videoElement?: HTMLVideoElement;
  // 直播类型
  protocol?: 'rtmp' | 'hls' | 'flv' | 'webrtc';
  // 自动重连
  autoReconnect?: boolean;
  // 最大重连次数
  maxReconnectAttempts?: number;
  // 重连间隔（毫秒）
  reconnectInterval?: number;
}

/**
 * 直播状态
 */
enum LiveState {
  IDLE = 'idle',
  CONNECTING = 'connecting',
  CONNECTED = 'connected',
  PLAYING = 'playing',
  PUBLISHING = 'publishing',
  RECONNECTING = 'reconnecting',
  ERROR = 'error',
  CLOSED = 'closed',
}

/**
 * 直播质量信息
 */
interface QualityInfo {
  // 视频码率（bps）
  videoBitrate: number;
  // 音频码率（bps）
  audioBitrate: number;
  // 帧率
  frameRate: number;
  // 分辨率
  resolution: { width: number; height: number };
  // 网络延迟（毫秒）
  latency: number;
  // 丢包率
  packetLoss: number;
}

/**
 * 直播 SDK 事件类型
 */
type LiveEventType =
  | 'stateChange'
  | 'qualityUpdate'
  | 'error'
  | 'reconnecting'
  | 'reconnected'
  | 'streamEnd';

/**
 * 直播 SDK 事件回调
 */
interface LiveEvents {
  onStateChange?: (state: LiveState) => void;
  onQualityUpdate?: (quality: QualityInfo) => void;
  onError?: (error: Error) => void;
  onReconnecting?: (attempt: number) => void;
  onReconnected?: () => void;
  onStreamEnd?: () => void;
}

/**
 * 直播 SDK
 *
 * 支持推流和拉流两种模式
 * 推流：使用 RTMP 或 WebRTC 将视频流推送到服务器
 * 拉流：使用 HLS、FLV 或 WebRTC 拉取直播流并播放
 */
export class LiveSDK {
  private config: LiveSDKConfig;
  private state: LiveState = LiveState.IDLE;
  private events: LiveEvents = {};
  private reconnectAttempts: number = 0;
  private reconnectTimer: NodeJS.Timeout | null = null;
  private qualityTimer: NodeJS.Timeout | null = null;
  // WebRTC 相关
  private peerConnection: RTCPeerConnection | null = null;
  private webSocket: WebSocket | null = null;
  // HLS.js 实例
  private hls: any = null;
  // FLV.js 实例
  private flvPlayer: any = null;

  constructor(config: LiveSDKConfig) {
    this.config = {
      autoReconnect: true,
      maxReconnectAttempts: 5,
      reconnectInterval: 3000,
      protocol: 'hls',
      ...config,
    };
  }

  /**
   * 注册事件回调
   */
  on(event: LiveEventType, callback: (...args: any[]) => void): void {
    switch (event) {
      case 'stateChange':
        this.events.onStateChange = callback;
        break;
      case 'qualityUpdate':
        this.events.onQualityUpdate = callback;
        break;
      case 'error':
        this.events.onError = callback;
        break;
      case 'reconnecting':
        this.events.onReconnecting = callback;
        break;
      case 'reconnected':
        this.events.onReconnected = callback;
        break;
      case 'streamEnd':
        this.events.onStreamEnd = callback;
        break;
    }
  }

  /**
   * 开始推流
   * @param stream 视频流（来自 getUserMedia）
   */
  async startPublish(stream: MediaStream): Promise<void> {
    if (this.config.mode !== 'publisher') {
      throw new Error('SDK 模式不是推流模式');
    }

    this.updateState(LiveState.CONNECTING);

    try {
      switch (this.config.protocol) {
        case 'webrtc':
          await this.startWebRTCPublish(stream);
          break;
        case 'rtmp':
        default:
          await this.startRTMPPublish(stream);
          break;
      }

      this.updateState(LiveState.PUBLISHING);
    } catch (error) {
      this.updateState(LiveState.ERROR);
      this.events.onError?.(error instanceof Error ? error : new Error(String(error)));
      this.handleReconnect();
    }
  }

  /**
   * WebRTC 推流
   */
  private async startWebRTCPublish(stream: MediaStream): Promise<void> {
    // 创建 WebRTC PeerConnection
    this.peerConnection = new RTCPeerConnection({
      iceServers: [
        { urls: 'stun:stun.l.google.com:19302' },
      ],
    });

    // 添加视频轨道
    stream.getVideoTracks().forEach(track => {
      this.peerConnection!.addTrack(track, stream);
    });

    // 添加音频轨道
    stream.getAudioTracks().forEach(track => {
      this.peerConnection!.addTrack(track, stream);
    });

    // 创建 WebSocket 连接
    const wsUrl = this.config.publishUrl!.replace('rtmp://', 'wss://').replace('rtmps://', 'wss://');
    this.webSocket = new WebSocket(wsUrl);

    this.webSocket.onopen = async () => {
      // 创建 SDP offer
      const offer = await this.peerConnection!.createOffer();
      await this.peerConnection!.setLocalDescription(offer);

      // 发送 offer 到服务器
      this.webSocket!.send(JSON.stringify({
        type: 'publish',
        sdp: offer.sdp,
      }));
    };

    this.webSocket.onmessage = async (event) => {
      const data = JSON.parse(event.data);

      if (data.type === 'answer') {
        await this.peerConnection!.setRemoteDescription(
          new RTCSessionDescription({ type: 'answer', sdp: data.sdp })
        );
      } else if (data.type === 'ice-candidate') {
        await this.peerConnection!.addIceCandidate(
          new RTCIceCandidate(data.candidate)
        );
      }
    };

    this.webSocket.onerror = () => {
      this.handleError(new Error('WebSocket 连接错误'));
    };
  }

  /**
   * RTMP 推流（模拟实现，实际需要使用 flv.js 的推流功能或自研）
   */
  private async startRTMPPublish(stream: MediaStream): Promise<void> {
    // RTMP 推流在浏览器中需要借助 Flash 或 MediaRecorder
    // 这里使用 MediaRecorder 模拟，实际生产环境应该使用专业推流库

    const mediaRecorder = new MediaRecorder(stream, {
      mimeType: 'video/webm;codecs=vp8,opus',
    });

    mediaRecorder.ondataavailable = async (event) => {
      if (event.data.size > 0 && this.webSocket) {
        // 发送到服务器
        this.webSocket.send(event.data);
      }
    };

    mediaRecorder.start(1000); // 每秒发送一次
  }

  /**
   * 开始拉流播放
   */
  async startPlayback(): Promise<void> {
    if (this.config.mode !== 'player') {
      throw new Error('SDK 模式不是拉流模式');
    }

    if (!this.config.videoElement) {
      throw new Error('未提供视频元素');
    }

    this.updateState(LiveState.CONNECTING);

    try {
      switch (this.config.protocol) {
        case 'hls':
          await this.startHLSPlayback();
          break;
        case 'flv':
          await this.startFLVPlayback();
          break;
        case 'webrtc':
          await this.startWebRTCPlayback();
          break;
        default:
          await this.startHLSPlayback();
      }

      this.updateState(LiveState.PLAYING);
      this.startQualityMonitoring();
    } catch (error) {
      this.updateState(LiveState.ERROR);
      this.events.onError?.(error instanceof Error ? error : new Error(String(error)));
      this.handleReconnect();
    }
  }

  /**
   * HLS 拉流
   */
  private async startHLSPlayback(): Promise<void> {
    const video = this.config.videoElement!;
    const url = this.config.playbackUrl!;

    // 检查浏览器原生 HLS 支持（Safari）
    if (video.canPlayType('application/vnd.apple.mpegurl')) {
      video.src = url;
      await video.play();
      return;
    }

    // 使用 HLS.js
    const Hls = (await import('hls.js')).default;

    this.hls = new Hls({
      enableWorker: true,
      lowLatencyMode: true,
      backBufferLength: 30,
    });

    this.hls.loadSource(url);
    this.hls.attachMedia(video);

    this.hls.on(Hls.Events.MANIFEST_PARSED, () => {
      video.play();
    });

    this.hls.on(Hls.Events.ERROR, (_, data) => {
      if (data.fatal) {
        switch (data.type) {
          case Hls.ErrorTypes.NETWORK_ERROR:
            this.hls.startLoad();
            this.handleReconnect();
            break;
          case Hls.ErrorTypes.MEDIA_ERROR:
            this.hls.recoverMediaError();
            break;
          default:
            this.handleError(new Error(`HLS 错误: ${data.details}`));
            break;
        }
      }
    });
  }

  /**
   * FLV 拉流
   */
  private async startFLVPlayback(): Promise<void> {
    const video = this.config.videoElement!;
    const url = this.config.playbackUrl!;

    // 使用 FLV.js
    const flvjs = (await import('flv.js')).default;

    if (flvjs.isSupported()) {
      this.flvPlayer = flvjs.createPlayer({
        type: 'flv',
        url: url,
      });

      this.flvPlayer.attachMediaElement(video);
      this.flvPlayer.load();
      await video.play();

      this.flvPlayer.on(flvjs.Events.ERROR, (_, data) => {
        this.handleError(new Error(`FLV 错误: ${data.details}`));
      });
    } else {
      throw new Error('浏览器不支持 FLV');
    }
  }

  /**
   * WebRTC 拉流
   */
  private async startWebRTCPlayback(): Promise<void> {
    const video = this.config.videoElement!;
    const wsUrl = this.config.playbackUrl!.replace('http://', 'ws://').replace('https://', 'wss://');

    this.webSocket = new WebSocket(wsUrl);

    this.webSocket.onopen = async () => {
      // 创建 PeerConnection
      this.peerConnection = new RTCPeerConnection({
        iceServers: [{ urls: 'stun:stun.l.google.com:19302' }],
      });

      // 接收轨道
      this.peerConnection.ontrack = (event) => {
        video.srcObject = event.streams[0];
      };

      // 生成 offer
      const offer = await this.peerConnection.createOffer({
        offerToReceiveAudio: true,
        offerToReceiveVideo: true
      });
      await this.peerConnection.setLocalDescription(offer);

      // 发送 offer
      this.webSocket.send(JSON.stringify({
        type: 'play',
        sdp: offer.sdp,
      }));
    };

    this.webSocket.onmessage = async (event) => {
      const data = JSON.parse(event.data);

      if (data.type === 'answer') {
        await this.peerConnection!.setRemoteDescription(
          new RTCSessionDescription({ type: 'answer', sdp: data.sdp })
        );
      } else if (data.type === 'ice-candidate') {
        await this.peerConnection!.addIceCandidate(
          new RTCIceCandidate(data.candidate)
        );
      }
    };
  }

  /**
   * 停止直播
   */
  stop(): void {
    // 清理定时器
    if (this.qualityTimer) {
      clearInterval(this.qualityTimer);
      this.qualityTimer = null;
    }

    if (this.reconnectTimer) {
      clearTimeout(this.reconnectTimer);
      this.reconnectTimer = null;
    }

    // 清理 WebRTC
    if (this.peerConnection) {
      this.peerConnection.close();
      this.peerConnection = null;
    }

    // 清理 WebSocket
    if (this.webSocket) {
      this.webSocket.close();
      this.webSocket = null;
    }

    // 清理 HLS
    if (this.hls) {
      this.hls.destroy();
      this.hls = null;
    }

    // 清理 FLV
    if (this.flvPlayer) {
      this.flvPlayer.destroy();
      this.flvPlayer = null;
    }

    this.updateState(LiveState.CLOSED);
    this.reconnectAttempts = 0;
  }

  /**
   * 获取当前状态
   */
  getState(): LiveState {
    return this.state;
  }

  /**
   * 获取直播质量信息
   */
  getQuality(): QualityInfo | null {
    if (this.state !== LiveState.PLAYING && this.state !== LiveState.PUBLISHING) {
      return null;
    }

    if (this.peerConnection) {
      const stats = this.peerConnection.getStats();
      // 实际实现需要从 stats 中提取质量信息
      return {
        videoBitrate: 2500000,
        audioBitrate: 128000,
        frameRate: 30,
        resolution: { width: 1920, height: 1080 },
        latency: 50,
        packetLoss: 0.01,
      };
    }

    return null;
  }

  /**
   * 开始质量监控
   */
  private startQualityMonitoring(): void {
    this.qualityTimer = setInterval(() => {
      const quality = this.getQuality();
      if (quality) {
        this.events.onQualityUpdate?.(quality);
      }
    }, 2000);
  }

  /**
   * 处理错误
   */
  private handleError(error: Error): void {
    this.updateState(LiveState.ERROR);
    this.events.onError?.(error);
  }

  /**
   * 处理重连
   */
  private handleReconnect(): void {
    if (!this.config.autoReconnect) {
      this.stop();
      return;
    }

    if (this.reconnectAttempts >= (this.config.maxReconnectAttempts || 5)) {
      this.events.onError?.(new Error('重连次数已达上限'));
      this.stop();
      return;
    }

    this.reconnectAttempts++;
    this.updateState(LiveState.RECONNECTING);
    this.events.onReconnecting?.(this.reconnectAttempts);

    this.reconnectTimer = setTimeout(async () => {
      if (this.config.mode === 'player') {
        await this.startPlayback();
      }
    }, this.config.reconnectInterval || 3000);
  }

  /**
   * 更新状态
   */
  private updateState(state: LiveState): void {
    this.state = state;
    this.events.onStateChange?.(state);
  }
}
```

### 7.3 我的思考：直播的技术核心是稳定和低延迟

直播系统的技术挑战主要集中在两个方面：**稳定性**和**低延迟**。

稳定性是指直播过程中不能出现频繁的断流、卡顿。对于一场几小时的直播，任何一次断流都会影响用户体验。提高稳定性的关键是做好重连机制，包括网络状态监测、自动重连、状态恢复等。

低延迟是指从主播端发生事件到观众端看到画面的时间间隔。延迟越低，互动越及时。对于游戏直播、电商直播等场景，5秒以上的延迟会严重影响互动体验。

要在这两者之间取得平衡，需要根据实际场景选择合适的协议。RTMP稳定但延迟高，WebRTC延迟低但复杂度高。在实际产品中，往往会采用混合策略，普通观众使用低延迟协议，连麦互动使用WebRTC。

## 八、互动系统

### 8.1 弹幕系统架构

弹幕是直播和视频平台的核心互动功能之一。用户发送的弹幕以飞字幕的形式从视频画面上飘过，既有信息传递功能，又有社交氛围营造功能。

弹幕系统的技术挑战在于**高并发处理**。热门直播可能有数百万观众同时在线，同一时刻可能有大量弹幕涌入。如何接收、过滤、分发、渲染这些弹幕是核心问题。

弹幕系统的架构分为四层：**接入层**、**处理层**、**存储层**和**客户端**。

接入层负责接收用户发送的弹幕和分发弹幕给用户。通常使用WebSocket连接保持长连接，实现实时推送。接入层需要处理海量并发连接，通常需要负载均衡和连接管理。

处理层负责弹幕的过滤、安全检查、格式转换等。弹幕需要经过敏感词过滤、广告识别、格式校验等处理。违规弹幕应该被拦截，不会分发到其他用户。

存储层负责弹幕的持久化存储。弹幕数据量大但价值随时间衰减，需要设计合理的存储策略。热数据（最近几小时）存储在Redis等内存数据库，冷数据存储在HBase等NoSQL数据库。

客户端负责弹幕的接收和渲染。弹幕渲染通常使用Canvas实现，比DOM元素性能更好。弹幕密度控制、速度控制、样式定制等功能都在客户端实现。

### 8.2 高性能弹幕渲染

以下是高性能弹幕渲染系统的实现。

```typescript
// utils/DanmuRenderer.ts - 高性能弹幕渲染器
// 使用 Canvas 实现高性能弹幕渲染，支持多种样式和动画效果

/**
 * 弹幕条目
 */
interface DanmuItem {
  // 弹幕 ID
  id: string;
  // 弹幕内容
  text: string;
  // 发送者昵称
  sender: string;
  // 弹幕类型：scroll（滚动）、top（顶部固定）、bottom（底部固定）
  type: 'scroll' | 'top' | 'bottom';
  // 颜色（十六进制）
  color: string;
  // 字体大小
  fontSize: number;
  // 发送时间
  timestamp: number;
  // 用户等级（影响样式）
  userLevel: number;
  // 是否是 VIP
  isVIP: boolean;
}

/**
 * 弹幕渲染配置
 */
interface DanmuConfig {
  // 画布宽度
  width: number;
  // 画布高度
  height: number;
  // 最大弹幕数量
  maxCount: number;
  // 滚动速度（像素/秒）
  scrollSpeed: number;
  // 弹幕轨道高度
  trackHeight: number;
  // 是否启用遮罩（半透明层）
  enableMask: boolean;
}

/**
 * 弹幕渲染器
 * 使用 Canvas 2D 实现高性能弹幕渲染
 *
 * 核心优化策略：
 * 1. 使用 Canvas 而非 DOM 元素，避免大量 DOM 操作
 * 2. 使用 requestAnimationFrame 实现流畅动画
 * 3. 对象池复用弹幕对象，减少 GC 压力
 * 4. 分层渲染，将静态弹幕和滚动弹幕分离
 * 5. 限制同屏弹幕数量，避免性能问题
 */
export class DanmuRenderer {
  private canvas: HTMLCanvasElement;
  private ctx: CanvasRenderingContext2D;
  private config: DanmuConfig;

  // 弹幕数据
  private danmuQueue: DanmuItem[] = [];  // 待显示弹幕队列
  private activeDanmu: Map<string, DanmuItem> = new Map();  // 当前活跃弹幕
  private trackUsage: Map<number, number> = new Map();  // 轨道使用情况

  // 弹幕对象池
  private danmuPool: DanmuItem[] = [];

  // 状态
  private isRunning: boolean = false;
  private lastFrameTime: number = 0;
  private animationFrameId: number | null = null;

  // 回调
  private onDanmuFiltered: ((danmu: DanmuItem) => boolean) | null = null;

  constructor(canvas: HTMLCanvasElement, config: Partial<DanmuConfig>) {
    this.canvas = canvas;
    this.ctx = canvas.getContext('2d', { alpha: true })!;

    this.config = {
      width: canvas.width || 800,
      height: canvas.height || 600,
      maxCount: 100,
      scrollSpeed: 150,  // 150 像素/秒
      trackHeight: 30,
      enableMask: true,
      ...config,
    };

    // 设置画布尺寸
    this.resize(this.config.width, this.config.height);

    // 初始化轨道
    this.initTracks();
  }

  /**
   * 初始化弹幕轨道
   */
  private initTracks(): void {
    const trackCount = Math.floor(this.config.height / this.config.trackHeight);
    for (let i = 0; i < trackCount; i++) {
      this.trackUsage.set(i, 0);
    }
  }

  /**
   * 设置弹幕过滤回调
   */
  setFilterCallback(callback: (danmu: DanmuItem) => boolean): void {
    this.onDanmuFiltered = callback;
  }

  /**
   * 调整画布尺寸
   */
  resize(width: number, height: number): void {
    this.canvas.width = width;
    this.canvas.height = height;
    this.config.width = width;
    this.config.height = height;
    this.initTracks();
  }

  /**
   * 添加弹幕到队列
   */
  addDanmu(danmu: DanmuItem): void {
    // 过滤弹幕
    if (this.onDanmuFiltered && this.onDanmuFiltered(danmu)) {
      return;
    }

    // 限制队列长度
    if (this.danmuQueue.length >= this.config.maxCount * 2) {
      return;
    }

    this.danmuQueue.push(danmu);
  }

  /**
   * 批量添加弹幕
   */
  addDanmuBatch(danmus: DanmuItem[]): void {
    danmus.forEach(d => this.addDanmu(d));
  }

  /**
   * 开始渲染
   */
  start(): void {
    if (this.isRunning) return;

    this.isRunning = true;
    this.lastFrameTime = performance.now();
    this.render();
  }

  /**
   * 停止渲染
   */
  stop(): void {
    this.isRunning = false;
    if (this.animationFrameId !== null) {
      cancelAnimationFrame(this.animationFrameId);
      this.animationFrameId = null;
    }
  }

  /**
   * 清空所有弹幕
   */
  clear(): void {
    this.danmuQueue = [];
    this.activeDanmu.clear();
    this.trackUsage.forEach((_, key) => this.trackUsage.set(key, 0));
    this.clearCanvas();
  }

  /**
   * 渲染循环
   */
  private render = (): void => {
    if (!this.isRunning) return;

    const currentTime = performance.now();
    const deltaTime = (currentTime - this.lastFrameTime) / 1000;  // 转换为秒
    this.lastFrameTime = currentTime;

    // 清空画布
    this.clearCanvas();

    // 绘制背景遮罩
    if (this.config.enableMask) {
      this.drawMask();
    }

    // 处理新弹幕
    this.processNewDanmu();

    // 更新活跃弹幕位置
    this.updateActiveDanmu(deltaTime);

    // 渲染活跃弹幕
    this.renderActiveDanmu();

    // 回收离开屏幕的弹幕
    this.recycleDanmu();

    // 继续下一帧
    this.animationFrameId = requestAnimationFrame(this.render);
  };

  /**
   * 清空画布
   */
  private clearCanvas(): void {
    this.ctx.clearRect(0, 0, this.config.width, this.config.height);
  }

  /**
   * 绘制遮罩
   */
  private drawMask(): void {
    // 在底部绘制半透明区域，减少底部弹幕对内容的遮挡
    const maskHeight = this.config.height * 0.2;
    const gradient = this.ctx.createLinearGradient(0, this.config.height - maskHeight, 0, this.config.height);
    gradient.addColorStop(0, 'rgba(0, 0, 0, 0)');
    gradient.addColorStop(1, 'rgba(0, 0, 0, 0.5)');

    this.ctx.fillStyle = gradient;
    this.ctx.fillRect(0, this.config.height - maskHeight, this.config.width, maskHeight);
  }

  /**
   * 处理新弹幕
   */
  private processNewDanmu(): void {
    // 如果活跃弹幕数量已满，跳过
    if (this.activeDanmu.size >= this.config.maxCount) {
      return;
    }

    // 从队列取出弹幕
    while (this.danmuQueue.length > 0 && this.activeDanmu.size < this.config.maxCount) {
      const danmu = this.danmuQueue.shift()!;

      // 获取可用轨道
      const track = this.findAvailableTrack(danmu);
      if (track === -1) {
        // 没有可用轨道，将弹幕放回队列
        this.danmuQueue.unshift(danmu);
        break;
      }

      // 设置弹幕位置
      danmu.track = track;
      danmu.x = this.config.width;
      danmu.y = track * this.config.trackHeight + this.config.trackHeight / 2;

      this.activeDanmu.set(danmu.id, danmu);
    }
  }

  /**
   * 查找可用轨道
   */
  private findAvailableTrack(danmu: DanmuItem): number {
    // 固定弹幕类型直接返回固定轨道
    if (danmu.type === 'top') {
      return 0;
    }
    if (danmu.type === 'bottom') {
      return Math.floor(this.config.height / this.config.trackHeight) - 1;
    }

    // 滚动弹幕需要检查碰撞
    const trackCount = Math.floor(this.config.height / this.config.trackHeight);
    let bestTrack = -1;
    let minOverlap = Infinity;

    // 计算弹幕宽度（估算）
    this.ctx.font = `${danmu.fontSize}px Arial`;
    const danmuWidth = this.ctx.measureText(danmu.text).width + 20;

    for (let i = 1; i < trackCount - 1; i++) {
      // 检查轨道使用情况
      const usage = this.trackUsage.get(i) || 0;

      // 跳过使用率太高的轨道
      if (usage > 0.8) continue;

      // 检查与活跃弹幕的碰撞
      let hasCollision = false;
      for (const [_, active] of this.activeDanmu) {
        if (active.track === i) {
          // 检查是否会碰撞
          const activeWidth = this.ctx.measureText(active.text).width + 20;
          const safeDistance = danmuWidth + activeWidth;

          if (active.x + safeDistance > this.config.width) {
            hasCollision = true;
            break;
          }
        }
      }

      if (!hasCollision && usage < minOverlap) {
        bestTrack = i;
        minOverlap = usage;
      }
    }

    if (bestTrack !== -1) {
      // 更新轨道使用率
      this.trackUsage.set(bestTrack, (this.trackUsage.get(bestTrack) || 0) + danmuWidth / this.config.width);
    }

    return bestTrack;
  }

  /**
   * 更新活跃弹幕位置
   */
  private updateActiveDanmu(deltaTime: number): void {
    for (const [id, danmu] of this.activeDanmu) {
      if (danmu.type === 'scroll') {
        // 滚动弹幕向左移动
        danmu.x -= this.config.scrollSpeed * deltaTime;

        // 回收离开屏幕的弹幕
        if (danmu.x < -200) {
          this.recycleDanmuById(id);
        }
      }
      // top 和 bottom 弹幕位置固定
    }
  }

  /**
   * 渲染活跃弹幕
   */
  private renderActiveDanmu(): void {
    for (const danmu of this.activeDanmu.values()) {
      this.renderDanmu(danmu);
    }
  }

  /**
   * 渲染单条弹幕
   */
  private renderDanmu(danmu: DanmuItem): void {
    const ctx = this.ctx;

    // 设置字体
    ctx.font = `${danmu.fontSize}px Arial`;
    ctx.textAlign = 'left';
    ctx.textBaseline = 'middle';

    // 根据用户等级和 VIP 状态设置样式
    let fillColor = danmu.color;
    let strokeColor = '#000000';
    let strokeWidth = 0;

    if (danmu.isVIP) {
      // VIP 弹幕加描边
      strokeWidth = 2;
      fillColor = '#FFD700';  // 金色
    }

    if (danmu.userLevel >= 10) {
      // 高等级用户弹幕加粗
      ctx.font = `bold ${danmu.fontSize}px Arial`;
    }

    // 绘制文字描边
    if (strokeWidth > 0) {
      ctx.strokeStyle = strokeColor;
      ctx.lineWidth = strokeWidth;
      ctx.strokeText(danmu.text, danmu.x, danmu.y);
    }

    // 绘制文字
    ctx.fillStyle = fillColor;
    ctx.fillText(danmu.text, danmu.x, danmu.y);

    // 绘制发送者名称（可选）
    if (danmu.type === 'top') {
      ctx.font = `${Math.max(10, danmu.fontSize - 4)}px Arial`;
      ctx.fillStyle = 'rgba(255, 255, 255, 0.7)';
      ctx.fillText(`@${danmu.sender}`, danmu.x, danmu.y + danmu.fontSize);
    }
  }

  /**
   * 回收离开屏幕的弹幕
   */
  private recycleDanmu(): void {
    const toRemove: string[] = [];

    for (const [id, danmu] of this.activeDanmu) {
      if (danmu.x < -200) {
        toRemove.push(id);
      }
    }

    toRemove.forEach(id => this.recycleDanmuById(id));
  }

  /**
   * 根据 ID 回收弹幕
   */
  private recycleDanmuById(id: string): void {
    const danmu = this.activeDanmu.get(id);
    if (danmu) {
      // 减少轨道使用率
      if (danmu.track !== undefined) {
        this.ctx.font = `${danmu.fontSize}px Arial`;
        const danmuWidth = this.ctx.measureText(danmu.text).width + 20;
        const currentUsage = this.trackUsage.get(danmu.track) || 0;
        this.trackUsage.set(danmu.track, Math.max(0, currentUsage - danmuWidth / this.config.width));
      }

      // 放入对象池
      this.danmuPool.push(danmu);
      this.activeDanmu.delete(id);
    }
  }

  /**
   * 获取对象池中的弹幕对象
   */
  private getDanmuFromPool(): DanmuItem | undefined {
    return this.danmuPool.pop();
  }

  /**
   * 获取统计信息
   */
  getStats(): {
    activeCount: number;
    queueLength: number;
    fps: number;
  } {
    return {
      activeCount: this.activeDanmu.size,
      queueLength: this.danmuQueue.length,
      fps: 60,  // 简化，实际应该计算
    };
  }
}
```

### 8.3 我的思考：互动是直播的灵魂

直播不同于点播的核心在于"互动"。主播和观众之间、观众和观众之间通过互动形成社区氛围。弹幕、礼物、评论等功能都是为互动服务的。

弹幕是最低门槛的互动方式。用户不需要停下来做什么，只要看一眼屏幕就能感受到"有人在看"的氛围。但弹幕太多也会影响观看体验，所以弹幕密度控制非常重要。好的弹幕系统应该做到"有存在感但不打扰"。

礼物系统是高价值互动。用户花钱购买礼物打赏主播，既有经济激励又有社交满足。礼物特效是直播平台营收的重要来源，但也需要平衡对普通用户的体验——不能让直播间变成"有钱人的秀场"。

连麦是深度互动。观众可以申请与主播连麦实时对话，这种互动层级更深。技术实现上，连麦需要使用WebRTC等低延迟协议，对网络要求更高。

## 九、审核系统

### 9.1 AI审核技术

视频审核是平台内容安全的核心保障。传统的人工审核成本高、效率低，难以应对海量内容。AI审核通过计算机视觉、自然语言处理等技术，可以自动识别违规内容，大幅提高审核效率。

图像识别是AI审核的基础。可以识别画面中的违规内容，如暴恐元素、色情内容、敏感人物等。深度学习模型（如YOLO、ResNet）在图像分类和目标检测任务上已经达到甚至超过人类水平。但AI识别也有局限性，对于模糊、变形、暗示性内容识别效果较差。

语音识别将音频转换为文本，然后对文本进行敏感词检测。这对于涉政、涉暴内容的审核非常重要。语音识别需要处理各种口音、方言、背景噪声等问题。

内容理解是更高层次的审核。不仅识别表面内容，还理解内容的语义和意图。例如，同样是"炸弹"一词，在科普视频中提到和恐吓视频中提到的性质完全不同。这需要结合视频标题、描述、上下文等综合判断。

AI审核通常采用多级过滤策略。第一级是AI全量扫描，所有上传的内容都经过AI审核。第二级是AI标记可疑内容，交人工复核。第三级是人工审核处理疑难问题。这种组合策略可以在效率和准确率之间取得平衡。

```typescript
class SensitiveWordLibrary {

    // 涉政词汇
    '敏感词1', '敏感词2',

    '暴力词1', '暴力词2',
    // 色情词汇

   * 检查是否包含敏感词

    const lowerText = text.toLowerCase();

    for (const word of this.sensitiveWords) {

      while ((index = lowerText.indexOf(word.toLowerCase(), index)) !== -1) {
        positions.push(index);
        index += word.length;
      }
    }

    return { contains: positions.length > 0, positions };
  }

    this.sensitiveWords.add(word);

   * 批量添加敏感词

  addWords(words: string[]): void {
    words.forEach(w => this.sensitiveWords.add(w));
  }
}

class AdDetector {
  // 广告关键词
  private adKeywords: string[] = [
    '加我', '私聊', '联系方式', '扫码', '二维码',
    '点击链接', '网址', '网站', '优惠', '折扣',
  ];

  // 可疑模式
  private adPatterns: RegExp[] = [
    /\d{7,}/g,  // 7位以上连续数字（可能是电话号码）
    /[a-zA-Z0-9]{10,}@[a-zA-Z0-9]+\.[a-zA-Z]+/g,  // 邮箱
    /https?:\/\/[^\s]+/g,  // URL
  ];

  /**
   * 检测广告
   */
  detect(text: string): FilterIssue[] {
    const issues: FilterIssue[] = [];
    const lowerText = text.toLowerCase();

    // 检查关键词
    for (const keyword of this.adKeywords) {
      const index = lowerText.indexOf(keyword.toLowerCase());
      if (index !== -1) {
        issues.push({
          type: 'ad',
          keyword: keyword,
          position: index,
          length: keyword.length,
          description: '包含广告关键词',
        });
      }
    }

    // 检查模式
    for (const pattern of this.adPatterns) {
      let match;
      while ((match = pattern.exec(text)) !== null) {
        issues.push({
          type: 'ad',
          keyword: match[0].substring(0, 10) + '...',
          position: match.index,
          length: match[0].length,
          description: '包含可疑模式',
        });
      }
    }

    return issues;
  }
}

/**
 * 恶意链接检测器
 */
class MaliciousLinkDetector {
  // 已知的恶意域名
  private maliciousDomains: Set<string> = new Set([
    'malicious.com',
    'phishing.com',
  ]);

  // 可疑域名模式
  private suspiciousPatterns: RegExp[] = [
    /\.(xyz|top|work|click|link)\s*$/gi,
    /[a-zA-Z0-9]+\.[a-zA-Z0-9]+\.[a-zA-Z0-9]+/g,
  ];

  /**
   * 检测恶意链接
   */
  detect(text: string): FilterIssue[] {
    const issues: FilterIssue[] = [];
    const urlPattern = /https?:\/\/[^\s]+/gi;

    let match;
    while ((match = urlPattern.exec(text)) !== null) {
      const url = match[0];

      try {
        const urlObj = new URL(url);
        const domain = urlObj.hostname.replace(/^www\./, '');

        // 检查是否是恶意域名
        if (this.maliciousDomains.has(domain)) {
          issues.push({
            type: 'malicious_link',
            keyword: domain,
            position: match.index,
            length: url.length,
            description: '包含已知恶意链接',
          });
        }

        // 检查可疑模式
        for (const pattern of this.suspiciousPatterns) {
          if (pattern.test(domain)) {
            issues.push({
              type: 'malicious_link',
              keyword: domain.substring(0, 20) + '...',
              position: match.index,
              length: url.length,
              description: '包含可疑域名模式',
            });
            break;
          }
        }
      } catch {
        // URL 解析失败，可能是恶意构造的字符串
      }
    }

    return issues;
  }
}

/**
 * 内容过滤器
 * 综合运用敏感词、广告、恶意链接检测
 */
class ContentFilter {
  private config: FilterConfig;
  private sensitiveLibrary: SensitiveWordLibrary;
  private adDetector: AdDetector;
  private maliciousLinkDetector: MaliciousLinkDetector;

  // 缓存
  private cache: Map<string, FilterResult> = new Map();
  private readonly CACHE_SIZE = 1000;

  constructor(config: Partial<FilterConfig> = {}) {
    this.config = {
      enableSensitiveFilter: true,
      enableAdFilter: true,
      enableMaliciousLinkFilter: true,
      minMatchLength: 2,
      filterMode: 'replace',
      ...config,
    };

    this.sensitiveLibrary = new SensitiveWordLibrary();
    this.adDetector = new AdDetector();
    this.maliciousLinkDetector = new MaliciousLinkDetector();
  }

  /**
   * 过滤内容
   */
  filter(content: string): FilterResult {
    // 检查缓存
    const cacheKey = content.substring(0, 100);
    const cached = this.cache.get(cacheKey);
    if (cached) {
      return cached;
    }

    const issues: FilterIssue[] = [];
    let filteredContent = content;
    let riskLevel: 'low' | 'medium' | 'high' = 'low';

    // 敏感词过滤
    if (this.config.enableSensitiveFilter) {
      const sensitiveResult = this.sensitiveLibrary.contains(filteredContent);
      if (sensitiveResult.contains) {
        issues.push({
          type: 'sensitive',
          keyword: '[敏感词]',
          position: sensitiveResult.positions[0],
          length: 3,
          description: '包含敏感内容',
        });

        filteredContent = this.maskSensitiveWords(filteredContent);
        riskLevel = 'high';
      }
    }

    // 广告过滤
    if (this.config.enableAdFilter) {
      const adIssues = this.adDetector.detect(filteredContent);
      issues.push(...adIssues);

      if (adIssues.length > 0) {
        filteredContent = this.maskContent(filteredContent, adIssues);
        riskLevel = riskLevel === 'high' ? 'high' : 'medium';
      }
    }

    // 恶意链接过滤
    if (this.config.enableMaliciousLinkFilter) {
      const linkIssues = this.maliciousLinkDetector.detect(filteredContent);
      issues.push(...linkIssues);

      if (linkIssues.length > 0) {
        filteredContent = this.maskContent(filteredContent, linkIssues);
        riskLevel = 'high';
      }
    }

    // 判断是否通过
    const passed = issues.filter(i => i.type !== 'other').length === 0;

    const result: FilterResult = {
      passed,
      filteredContent,
      issues,
      riskLevel,
    };

    // 更新缓存
    if (this.cache.size >= this.CACHE_SIZE) {
      // 清理一半缓存
      const keysToDelete = Array.from(this.cache.keys()).slice(0, this.CACHE_SIZE / 2);
      keysToDelete.forEach(k => this.cache.delete(k));
    }
    this.cache.set(cacheKey, result);

    return result;
  }

  /**
   * 遮罩敏感词
   */
  private maskSensitiveWords(content: string): string {
    // 简单实现，实际应该使用更复杂的算法
    return content.replace(/[\u4e00-\u9fa5a-zA-Z0-9]{2,}/g, (match) => {
      if (this.sensitiveLibrary.contains(match).contains) {
        return '*'.repeat(match.length);
      }
      return match;
    });
  }

  /**
   * 遮罩指定位置的内容
   */
  private maskContent(content: string, issues: FilterIssue[]): string {
    // 按位置倒序处理，避免位置偏移
    const sortedIssues = [...issues].sort((a, b) => b.position - a.position);

    for (const issue of sortedIssues) {
      const before = content.substring(0, issue.position);
      const masked = '*'.repeat(issue.length);
      const after = content.substring(issue.position + issue.length);
      content = before + masked + after;
    }

    return content;
  }

  /**
   * 添加敏感词
   */
  addSensitiveWords(words: string[]): void {
    this.sensitiveLibrary.addWords(words);
    // 清除缓存
    this.cache.clear();
  }

  /**
   * 批量过滤
   */
  filterBatch(contents: string[]): FilterResult[] {
    return contents.map(c => this.filter(c));
  }
}
```

### 9.3 我的思考：审核是平台的责任

视频平台承担着内容安全的责任。平台上的违规内容不仅可能造成法律风险，还会影响用户体验、损害平台声誉。

审核的核心挑战在于平衡**效率**和**准确率**。纯AI审核速度快但准确率有限，可能漏掉或误判内容。纯人工审核准确率高但成本高、速度慢。好的审核策略是AI+人工的组合。AI负责全量扫描和初步分类，人工负责复核和判断疑难问题。

审核的另一个挑战是**实时性**。对于直播内容，弹幕、评论等需要实时审核，稍有延迟违规内容就可能被大量用户看到。这对审核系统的性能要求很高。

最后，审核也是一个**持续对抗**的过程。违规用户会不断变换方式绕过审核，审核系统需要持续更新和迭代。这需要技术投入，也需要运营配合。

## 十、数据埋点

### 10.1 播放数据埋点

数据埋点是视频平台运营和优化的基础。通过埋点可以了解用户行为、评估内容质量、优化产品体验。视频平台的埋点主要分为三类：**播放数据**、**行为数据**和**质量数据**。

播放数据反映用户观看视频的情况。核心指标包括播放量（VV）、播放人数（UV）、播放时长、完播率、播放完成率等。播放量是最基础的指标，反映内容的热度。完播率是衡量内容质量的重要指标，完播率高说明内容吸引人。播放时长反映用户的沉浸程度。

播放数据埋点需要在播放开始、播放中、播放结束等关键时刻触发。开始播放时记录video_id、user_id、source（来源）、autoplay（是否自动播放）等信息。播放过程中定期上报当前位置、已播放时长、缓冲次数等。播放结束时记录total_duration（总时长）、watched_duration（观看时长）、complete（是否看完）等信息。

以下是播放数据埋点的实现。

```typescript
// utils/analytics.ts - 视频播放数据分析埋点
// 视频播放相关数据采集和上报

/**
 * 播放事件类型
 */
type PlayEventType =
  | 'play_start'
  | 'play_progress'
  | 'play_pause'
  | 'play_resume'
  | 'play_complete'
  | 'play_error'
  | 'quality_change'
  | 'seek'
  | 'buffer_start'
  | 'buffer_end';

/**
 * 播放事件数据
 */
interface PlayEventData {
  // 事件类型
  event: PlayEventType;
  // 视频 ID
  videoId: string;
  // 用户 ID（未登录为 null）
  userId: string | null;
  // 会话 ID
  sessionId: string;
  // 事件时间戳
  timestamp: number;
  // 事件相关数据
  data: Record<string, any>;
}

/**
 * 播放质量数据
 */
interface QualityMetrics {
  // 首帧时间（毫秒）
  firstFrameTime: number;
  // 总缓冲次数
  bufferCount: number;
  // 总缓冲时长（毫秒）
  totalBufferDuration: number;
  // 播放卡顿次数
  stallCount: number;
  // 播放卡顿总时长（毫秒）
  totalStallDuration: number;
  // 视频码率（bps）
  videoBitrate: number;
  // 音频码率（bps）
  audioBitrate: number;
  // 视频分辨率
  resolution: { width: number; height: number };
  // 帧率
  frameRate: number;
}

/**
 * 播放分析器
 */
class VideoAnalytics {
  // 当前视频 ID
  private currentVideoId: string | null = null;
  // 会话 ID
  private sessionId: string;
  // 用户 ID
  private userId: string | null = null;
  // 事件队列
  private eventQueue: PlayEventData[] = [];
  // 上报定时器
  private flushTimer: NodeJS.Timeout | null = null;
  // 上报间隔（毫秒）
  private flushInterval: number = 5000;
  // 最大队列长度
  private maxQueueSize: number = 50;

  // 播放质量指标
  private qualityMetrics: QualityMetrics = {
    firstFrameTime: 0,
    bufferCount: 0,
    totalBufferDuration: 0,
    stallCount: 0,
    totalStallDuration: 0,
    videoBitrate: 0,
    audioBitrate: 0,
    resolution: { width: 0, height: 0 },
    frameRate: 0,
  };

  // 播放状态
  private playStartTime: number = 0;
  private totalWatchedTime: number = 0;
  private lastReportTime: number = 0;
  private isPaused: boolean = false;
  private pauseStartTime: number = 0;

  constructor() {
    this.sessionId = this.generateSessionId();
    this.startFlushTimer();

    // 页面关闭时上报
    window.addEventListener('beforeunload', () => {
      this.flush();
    });
  }

  /**
   * 设置用户 ID
   */
  setUserId(userId: string | null): void {
    this.userId = userId;
  }

  /**
   * 开始播放
   */
  startPlay(videoId: string, data: Record<string, any> = {}): void {
    this.currentVideoId = videoId;
    this.playStartTime = Date.now();
    this.lastReportTime = this.playStartTime;
    this.totalWatchedTime = 0;
    this.resetQualityMetrics();

    this.trackEvent('play_start', {
      videoId,
      source: data.source || 'unknown',
      autoplay: data.autoplay || false,
      quality: data.quality || 'auto',
      referrer: document.referrer,
    });
  }

  /**
   * 播放进度
   * @param currentTime 当前播放位置（秒）
   * @param duration 视频总时长（秒）
   */
  reportProgress(currentTime: number, duration: number): void {
    if (!this.currentVideoId) return;

    const now = Date.now();
    const elapsed = now - this.lastReportTime;

    // 累计观看时间（排除暂停时间）
    if (!this.isPaused) {
      this.totalWatchedTime += elapsed / 1000;
    }
    this.lastReportTime = now;

    // 每 10 秒上报一次进度
    if (elapsed >= 10000) {
      this.trackEvent('play_progress', {
        videoId: this.currentVideoId,
        currentTime,
        duration,
        watchedTime: this.totalWatchedTime,
        progressPercent: duration > 0 ? (currentTime / duration) * 100 : 0,
      });
      this.lastReportTime = now;
    }
  }

  /**
   * 暂停播放
   */
  pause(currentTime: number): void {
    if (this.isPaused) return;

    this.isPaused = true;
    this.pauseStartTime = Date.now();

    this.trackEvent('play_pause', {
      videoId: this.currentVideoId,
      currentTime,
      pausedDuration: this.totalWatchedTime,
    });
  }

  /**
   * 恢复播放
   */
  resume(currentTime: number): void {
    if (!this.isPaused) return;

    this.isPaused = false;

    this.trackEvent('play_resume', {
      videoId: this.currentVideoId,
      currentTime,
    });
  }

  /**
   * 播放完成
   */
  complete(): void {
    if (!this.currentVideoId) return;

    const duration = this.qualityMetrics.resolution.width > 0
      ? (Date.now() - this.playStartTime) / 1000
      : 0;

    this.trackEvent('play_complete', {
      videoId: this.currentVideoId,
      totalDuration: duration,
      watchedDuration: this.totalWatchedTime,
      completeRate: duration > 0 ? (this.totalWatchedTime / duration) * 100 : 0,
      qualityMetrics: { ...this.qualityMetrics },
    });

    this.currentVideoId = null;
  }

  /**
   * 播放错误
   */
  error(errorCode: string, errorMessage: string): void {
    this.trackEvent('play_error', {
      videoId: this.currentVideoId,
      errorCode,
      errorMessage,
    });
  }

  /**
   * 清晰度切换
   */
  qualityChange(fromQuality: string, toQuality: string): void {
    this.trackEvent('quality_change', {
      videoId: this.currentVideoId,
      fromQuality,
      toQuality,
    });
  }

  /**
   * 跳转
   */
  seek(fromTime: number, toTime: number): void {
    this.trackEvent('seek', {
      videoId: this.currentVideoId,
      fromTime,
      toTime,
    });
  }

  /**
   * 缓冲开始
   */
  bufferStart(): void {
    this.qualityMetrics.bufferCount++;

    this.trackEvent('buffer_start', {
      videoId: this.currentVideoId,
      bufferCount: this.qualityMetrics.bufferCount,
    });
  }

  /**
   * 缓冲结束
   * @param duration 缓冲耗时（毫秒）
   */
  bufferEnd(duration: number): void {
    this.qualityMetrics.totalBufferDuration += duration;

    this.trackEvent('buffer_end', {
      videoId: this.currentVideoId,
      bufferDuration: duration,
      totalBufferDuration: this.qualityMetrics.totalBufferDuration,
    });
  }

  /**
   * 设置首帧时间
   */
  setFirstFrameTime(time: number): void {
    this.qualityMetrics.firstFrameTime = time;
  }

  /**
   * 设置质量指标
   */
  setQualityMetrics(metrics: Partial<QualityMetrics>): void {
    Object.assign(this.qualityMetrics, metrics);
  }

  /**
   * 记录事件
   */
  private trackEvent(event: PlayEventType, data: Record<string, any>): void {
    const eventData: PlayEventData = {
      event,
      videoId: this.currentVideoId || '',
      userId: this.userId,
      sessionId: this.sessionId,
      timestamp: Date.now(),
      data,
    };

    this.eventQueue.push(eventData);

    // 如果队列过长，立即上报
    if (this.eventQueue.length >= this.maxQueueSize) {
      this.flush();
    }
  }

  /**
   * 上报数据
   */
  private flush(): void {
    if (this.eventQueue.length === 0) return;

    const events = [...this.eventQueue];
    this.eventQueue = [];

    // 实际实现中发送到后端
    this.sendToServer(events);
  }

  /**
   * 发送数据到服务器
   */
  private async sendToServer(events: PlayEventData[]): Promise<void> {
    try {
      await fetch('/api/analytics/play', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          sessionId: this.sessionId,
          userId: this.userId,
          events,
        }),
      });
    } catch (error) {
      // 上报失败，放回队列
      console.error('Analytics上报失败:', error);
      this.eventQueue.unshift(...events);
    }
  }

  /**
   * 启动定时上报
   */
  private startFlushTimer(): void {
    this.flushTimer = setInterval(() => {
      this.flush();
    }, this.flushInterval);
  }

  /**
   * 重置质量指标
   */
  private resetQualityMetrics(): void {
    this.qualityMetrics = {
      firstFrameTime: 0,
      bufferCount: 0,
      totalBufferDuration: 0,
      stallCount: 0,
      totalStallDuration: 0,
      videoBitrate: 0,
      audioBitrate: 0,
      resolution: { width: 0, height: 0 },
      frameRate: 0,
    };
  }

  /**
   * 生成会话 ID
   */
  private generateSessionId(): string {
    return `${Date.now()}-${Math.random().toString(36).substring(2, 11)}`;
  }

  /**
   * 获取质量报告
   */
  getQualityReport(): QualityMetrics & {
    totalWatchedTime: number;
    completeRate: number;
  } {
    return {
      ...this.qualityMetrics,
      totalWatchedTime: this.totalWatchedTime,
      completeRate: this.qualityMetrics.resolution.width > 0
        ? (this.totalWatchedTime / ((Date.now() - this.playStartTime) / 1000)) * 100
        : 0,
    };
  }

  /**
   * 销毁
   */
  destroy(): void {
    if (this.flushTimer) {
      clearInterval(this.flushTimer);
      this.flushTimer = null;
    }
    this.flush();
  }
}
```

### 10.2 我的思考：数据是优化的基础

数据是视频平台持续优化的基础。没有数据支撑的优化是盲目的，不知道效果好不好，不知道问题在哪里。

视频平台需要关注的数据很多。**播放数据**是最核心的，包括播放量、完播率、播放时长等，反映内容的吸引力和用户粘性。**质量数据**反映技术体验，包括首帧时间、卡顿率、加载时间等。**行为数据**反映用户互动，包括点赞、评论、分享、收藏等。**商业数据**反映变现能力，包括广告点击、付费转化等。

数据埋点需要平衡**详尽**和**效率**。埋点太少，数据价值有限；埋点太多，数据量爆炸且影响性能。好的埋点策略是只埋关键事件和关键维度，次要信息通过计算得出。

数据也需要**治理**。不同来源的数据可能不一致，需要清洗和校验。数据也需要分层，实时数据用于实时监控和告警，离线数据用于深度分析和挖掘。

## 十一、案例：首屏优化

### 11.1 问题分析

短视频首屏时间是用户体验的关键指标。首屏时间指从用户点击播放到看到第一帧画面的时间。行业平均水平是2-3秒，但优秀的产品可以做到300ms以内。每减少100ms的首屏时间，用户留存率可以提升约5%。

首屏时间可以分解为以下几个部分。DNS解析时间，用户需要将域名解析为IP地址。TCP连接时间，需要完成TCP三次握手（如果是HTTPS还需要TLS握手）。SSL握手时间（如果是HTTPS），HTTPS建立安全连接的时间。TTFB（Time To First Byte），服务器开始返回数据的时间。内容下载时间，浏览器下载视频元数据和解码第一帧的时间。首帧渲染时间，浏览器解析视频数据并渲染第一帧的时间。

针对每个环节都有优化手段。DNS预解析可以使用户在播放前就完成DNS查询。预连接可以提前建立TCP连接，甚至TLS连接。CDN就近访问可以减少网络延迟。视频moov原子前置可以让浏览器快速获取视频元数据。预加载可以让浏览器提前下载视频开头部分。

### 11.2 优化方案

以下是首屏优化的完整实现方案。

```typescript
// utils/videoOptimizer.ts - 视频首屏优化工具
// 综合运用预解析、预连接、预加载等技术优化首屏时间

/**
 * 优化配置
 */
interface OptimizationConfig {
  // 启用 DNS 预解析
  enableDnsPrefetch: boolean;
  // 启用预连接
  enablePreconnect: boolean;
  // 启用预加载
  enablePreload: boolean;
  // 预加载范围（字节）
  preloadRange: number;
  // 视频域名列表
  domains: string[];
}

/**
 * 首屏优化器
 */
class VideoOptimizer {
  private config: OptimizationConfig;
  // 已预解析的域名
  private dnsPrefetchedDomains: Set<string> = new Set();
  // 已预连接的域名
  private preconnectedDomains: Set<string> = new Set();
  // 活跃的预加载请求
  private activePrefetches: Map<string, AbortController> = new Map();

  constructor(config: Partial<OptimizationConfig> = {}) {
    this.config = {
      enableDnsPrefetch: true,
      enablePreconnect: true,
      enablePreload: true,
      preloadRange: 2 * 1024 * 1024, // 默认预加载前 2MB
      domains: [],
      ...config,
    };

    // 页面空闲时进行预优化
    this.scheduleOptimization();
  }

  /**
   * 页面空闲时执行预优化
   */
  private scheduleOptimization(): void {
    if ('requestIdleCallback' in window) {
      (window as any).requestIdleCallback(() => {
        this.optimize();
      }, { timeout: 3000 });
    } else {
      setTimeout(() => this.optimize(), 1);
    }
  }

  /**
   * 执行优化
   */
  optimize(): void {
    // 添加 DNS 预解析标签
    if (this.config.enableDnsPrefetch) {
      this.addDnsPrefetchLinks();
    }

    // 添加预连接标签
    if (this.config.enablePreconnect) {
      this.addPreconnectLinks();
    }
  }

  /**
   * 添加 DNS 预解析链接
   */
  private addDnsPrefetchLinks(): void {
    const existingLinks = document.querySelectorAll('link[rel="dns-prefetch"]');
    const existingDomains = new Set(
      Array.from(existingLinks).map(link => (link as HTMLLinkElement).href)
    );

    for (const domain of this.config.domains) {
      if (existingDomains.has(domain)) continue;
      if (this.dnsPrefetchedDomains.has(domain)) continue;

      const link = document.createElement('link');
      link.rel = 'dns-prefetch';
      link.href = domain;
      link.crossOrigin = 'anonymous';
      document.head.appendChild(link);

      this.dnsPrefetchedDomains.add(domain);
    }
  }

  /**
   * 添加预连接链接
   */
  private addPreconnectLinks(): void {
    const existingLinks = document.querySelectorAll('link[rel="preconnect"]');
    const existingDomains = new Set(
      Array.from(existingLinks).map(link => (link as HTMLLinkElement).href)
    );

    for (const domain of this.config.domains) {
      if (existingDomains.has(domain)) continue;
      if (this.preconnectedDomains.has(domain)) continue;

      // 添加预连接
      const link = document.createElement('link');
      link.rel = 'preconnect';
      link.href = domain;
      link.crossOrigin = 'anonymous';
      document.head.appendChild(link);

      // 添加 DNS 预解析
      const dnsLink = document.createElement('link');
      dnsLink.rel = 'dns-prefetch';
      dnsLink.href = domain;
      dnsLink.crossOrigin = 'anonymous';
      document.head.appendChild(dnsLink);

      this.preconnectedDomains.add(domain);
    }
  }

  /**
   * 预加载视频
   * @param url 视频 URL
   * @param range 预加载范围
   */
  preloadVideo(url: string, range?: number): void {
    if (!this.config.enablePreload) return;

    // 如果已经有该 URL 的预加载，跳过
    if (this.activePrefetches.has(url)) return;

    const controller = new AbortController();
    this.activePrefetches.set(url, controller);

    const bytes = range || this.config.preloadRange;

    fetch(url, {
      method: 'HEAD',
      signal: controller.signal,
    }).then(response => {
      // 验证视频内容
      const contentType = response.headers.get('content-type');
      if (contentType && contentType.includes('video')) {
        // 创建范围请求预加载
        return fetch(url, {
          headers: { Range: `bytes=0-${bytes}` },
          signal: controller.signal,
        });
      }
    }).then(response => {
      if (response) {
        // 预加载完成，缓存响应
        return response.blob();
      }
    }).then(blob => {
      if (blob) {
        // 将预加载的数据存入缓存
        this.cachePreloadedData(url, blob);
      }
    }).catch(error => {
      if (error.name !== 'AbortError') {
        console.warn('视频预加载失败:', error);
      }
    }).finally(() => {
      this.activePrefetches.delete(url);
    });
  }

  /**
   * 缓存预加载的数据
   */
  private cachePreloadedData(url: string, blob: Blob): void {
    // 使用 Cache API 缓存响应
    if ('caches' in window) {
      caches.open('video-preload').then(cache => {
        cache.put(url, new Response(blob, {
          headers: {
            'Content-Type': 'video/mp4',
            'Content-Length': blob.size.toString(),
          },
        }));
      });
    }
  }

  /**
   * 取消预加载
   */
  cancelPreload(url: string): void {
    const controller = this.activePrefetches.get(url);
    if (controller) {
      controller.abort();
      this.activePrefetches.delete(url);
    }
  }

  /**
   * 获取视频开始播放
   * 优先使用缓存数据，否则直接播放
   * @param video 视频元素
   * @param url 视频 URL
   */
  async startVideoWithCache(video: HTMLVideoElement, url: string): Promise<void> {
    // 尝试从缓存获取
    if ('caches' in window) {
      const cache = await caches.open('video-preload');
      const cachedResponse = await cache.match(url);

      if (cachedResponse) {
        // 使用缓存数据
        const blob = await cachedResponse.blob();
        const blobUrl = URL.createObjectURL(blob);

        video.src = blobUrl;
        video.play().catch(() => {
          // 缓存数据播放失败，尝试原始 URL
          video.src = url;
        });

        return;
      }
    }

    // 没有缓存，直接播放
    video.src = url;
  }

  /**
   * 获取优化统计
   */
  getStats(): {
    dnsPrefetchCount: number;
    preconnectCount: number;
    activePrefetchCount: number;
  } {
    return {
      dnsPrefetchCount: this.dnsPrefetchedDomains.size,
      preconnectCount: this.preconnectedDomains.size,
      activePrefetchCount: this.activePrefetches.size,
    };
  }
}

/**

 */
export class FirstFrameMonitor {
  private video: HTMLVideoElement | null = null;
  private startTime: number = 0;
  private firstFrameTime: number = 0;
  private metrics: {
    loadTime: number;
    decodeTime: number;
    renderTime: number;
  } = {
    loadTime: 0,
    decodeTime: 0,
    renderTime: 0,
  };

  /**
   * 开始监控
   */
  start(video: HTMLVideoElement): void {
    this.video = video;
    this.startTime = performance.now();
    this.firstFrameTime = 0;

    // 监听 loadedmetadata 事件（元数据加载完成）
    video.addEventListener('loadedmetadata', this.onMetadataLoaded);

    // 监听 canplay 事件（可以开始播放）
    video.addEventListener('canplay', this.onCanPlay);

    // 监听 firstFrame 回调
    video.requestVideoFrameCallback?.(this.onFirstFrame);
  }

  /**
   * 元数据加载完成
   */
  private onMetadataLoaded = (): void => {
    this.metrics.loadTime = performance.now() - this.startTime;
  };

  /**
   * 可以开始播放
   */
  private onCanPlay = (): void => {
    this.metrics.decodeTime = performance.now() - this.startTime;
  };

  /**
   * 首帧渲染
   */
  private onFirstFrame = (now: number, metadata: any): void => {
    if (this.firstFrameTime === 0) {
      this.firstFrameTime = now - this.startTime;
      this.metrics.renderTime = this.firstFrameTime;

      // 上报数据
      this.reportMetrics();
    }
  };

  /**
   * 上报指标
   */
  private reportMetrics(): void {
    const data = {
      firstFrameTime: this.firstFrameTime,
      loadTime: this.metrics.loadTime,
      decodeTime: this.metrics.decodeTime,
      renderTime: this.metrics.renderTime,
      url: this.video?.src,
      timestamp: Date.now(),
    };

    // 发送到监控服务器
    console.log('首屏指标:', data);

    // 实际实现中发送到服务器
    navigator.sendBeacon?.('/api/metrics/firstframe', JSON.stringify(data));
  }

  /**
   * 停止监控
   */
  stop(): void {
    if (this.video) {
      this.video.removeEventListener('loadedmetadata', this.onMetadataLoaded);
      this.video.removeEventListener('canplay', this.onCanPlay);
    }
    this.video = null;
  }
}
```

通过以上优化措施，短视频首屏时间可以从2秒降低到300ms以内，优化效果达到85%以上。

具体优化效果分析。DNS预解析可以节省约50-100ms的DNS查询时间。TCP预连接可以节省约50-150ms的连接建立时间。视频数据预加载可以节省约200-500ms的首帧时间，因为浏览器已经缓存了视频开头的数据。

综合运用这些技术，可以将首屏时间分解为：DNS+连接时间约100ms，内容下载时间约200ms，总计约300ms的首帧时间。

### 11.4 我的思考：体验是留存的关键

视频平台的用户留存很大程度上取决于播放体验。用户容忍不了频繁的卡顿和缓慢的加载，一旦体验不好就会流失到其他平台。

首屏时间是用户体验的第一道门槛。用户点击播放后，如果几秒钟内都没有画面，很多用户会选择退出。优化首屏时间是提升留存的关键抓手。

体验优化需要数据支撑。需要通过埋点监控真实的播放体验数据，发现问题后才能有针对性地优化。数据驱动的优化才是有效的优化。

体验优化也需要权衡。更好的体验往往意味着更高的成本——更快的CDN、更高的码率、更多的预加载。需要在体验和成本之间找到平衡点。
