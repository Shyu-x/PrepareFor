# 定时任务与消息队列完全指南

## 概述

在后端开发中，定时任务和消息队列是处理异步任务的核心基础设施。本指南将深入讲解 Node.js 生态中的定时任务方案（node-cron、node-schedule）、消息队列系统（BullMQ）以及缓存系统（Redis），帮助你构建高可靠性的后端服务。

---

## 第一部分：定时任务详解

### 1.1 为什么需要定时任务

定时任务是指在特定时间或固定间隔自动执行的计算任务。在后端开发中，定时任务有广泛的应用场景：

| 场景 | 说明 | 示例 |
|------|------|------|
| **数据备份** | 定期备份数据库或文件 | 每天凌晨2点备份用户数据 |
| **报表生成** | 定时生成统计报表 | 每周一生成上周运营报表 |
| **缓存清理** | 清理过期缓存数据 | 每小时清理30天前的缓存 |
| **状态同步** | 同步外部系统状态 | 每5分钟检查订单状态 |
| **定期清理** | 清理临时文件和日志 | 每天清理7天前的日志 |
| **通知推送** | 定时发送提醒通知 | 每天上午9点推送日报 |

### 1.2 node-cron 的使用

node-cron 是 Node.js 中最流行的基于 Cron 表达式的定时任务库。它轻量、简单，适合大多数定时任务场景。

#### 1.2.1 安装与基础用法

```bash
npm install node-cron
```

```typescript
// 引入 node-cron 库，用于创建基于 Cron 表达式的定时任务
import cron from 'node-cron';

// 示例1：每分钟执行一次任务
// Cron表达式由5个字段组成：分 时 日 月 周
// '* * * * *' 表示每分钟的第0秒执行
cron.schedule('* * * * *', () => {
  console.log('每分钟执行一次任务:', new Date().toISOString());
});

// 示例2：每小时执行一次
// 第1个*表示第0分钟，第2个*表示每小时
cron.schedule('0 * * * *', () => {
  console.log('每小时执行一次:', new Date().toISOString());
});

// 示例3：每天凌晨2点执行
// 0 2 * * * 表示每天第2小时的第0分钟执行
cron.schedule('0 2 * * *', () => {
  console.log('每天凌晨2点执行数据备份');
});
```

#### 1.2.2 Cron 表达式详解

Cron 表达式是 Linux/Unix 系统中常见的定时任务表达方式，由5到7个字段组成。

```
┌───────────── 分钟 (0-59)
│ ┌───────────── 小时 (0-23)
│ │ ┌───────────── 日期 (1-31)
│ │ │ ┌───────────── 月份 (1-12)
│ │ │ │ ┌───────────── 星期 (0-7，0和7都是周日)
│ │ │ │ │
* * * * *
```

**常用 Cron 表达式示例：**

```typescript
// 基础表达式示例
'* * * * *'           // 每分钟
'0 * * * *'           // 每小时整点
'0 0 * * *'           // 每天午夜
'0 0 * * 0'           // 每周日午夜
'0 0 1 * *'           // 每月第一天的午夜
'0 0 1 1 *'           // 每年1月1日午夜

// 带间隔的表达式
'*/5 * * * *'         // 每5分钟
'*/10 * * * *'        // 每10分钟
'0 */2 * * *'         // 每2小时

// 具体时间点
'30 9 * * 1-5'        // 周一到周五上午9:30
'0 9 * * *'           // 每天上午9点
'0 18 * * *'          // 每天下午6点

// 范围和列表
'0 9-17 * * 1-5'     // 工作时间每小时整点（9点到17点）
'0 9,12,18 * * *'     // 每天9点、12点、18点执行
'0 9-11,13-17 * * 1-5' // 工作日9-11点和13-17点每小时执行

// 特殊字符
'0 * * * *'           // 每小时 - * 表示任意值
'0 0 * * 0'           // 每周日 - 0 表示周日
'0 0 1 * *'           // 每月1日
'0 0 * * 6,0'         // 周六和周日
```

#### 1.2.3 node-cron 高级用法

```typescript
import cron, { ScheduledTask } from 'node-cron';

// 定义一个复杂的定时任务配置
interface TaskConfig {
  name: string;           // 任务名称，用于日志标识
  expression: string;     // Cron 表达式
  timezone?: string;      // 时区设置
  immediate?: boolean;    // 是否立即执行一次
}

// 任务配置：每天凌晨2点执行数据备份，使用东京时区
const backupTaskConfig: TaskConfig = {
  name: 'database-backup',
  expression: '0 2 * * *',
  timezone: 'Asia/Tokyo',
  immediate: false,
};

// 创建定时任务，返回任务实例用于后续控制
const backupTask: ScheduledTask = cron.schedule(
  backupTaskConfig.expression,
  async () => {
    const startTime = Date.now();
    console.log(`[${backupTaskConfig.name}] 任务开始执行`);

    try {
      // 模拟执行数据库备份操作
      await performDatabaseBackup();

      const duration = Date.now() - startTime;
      console.log(`[${backupTaskConfig.name}] 任务执行完成，耗时: ${duration}ms`);
    } catch (error) {
      // 记录错误日志，但不让任务崩溃
      console.error(`[${backupTaskConfig.name}] 任务执行失败:`, error);
    }
  },
  {
    timezone: backupTaskConfig.timezone,  // 设置任务执行的时区
    scheduled: true,                        // 是否自动调度，false则不自动执行
  }
);

// 模拟数据库备份函数
async function performDatabaseBackup(): Promise<void> {
  // 实际项目中这里会连接数据库并执行备份
  console.log('正在执行数据库备份...');
  await new Promise(resolve => setTimeout(resolve, 1000));
  console.log('数据库备份完成');
}

// 动态管理任务：启动和停止
function manageTask(task: ScheduledTask, action: 'start' | 'stop'): void {
  if (action === 'start') {
    task.start();
    console.log('任务已启动');
  } else {
    task.stop();
    console.log('任务已停止');
  }
}

// 带错误处理的定时任务包装器
function createSafeTask(
  expression: string,      // Cron 表达式
  handler: () => Promise<void>,  // 异步任务处理函数
  options?: { name?: string; timezone?: string }
): ScheduledTask {
  return cron.schedule(expression, async () => {
    const taskName = options?.name || 'unnamed-task';

    try {
      console.log(`[${taskName}] 任务开始: ${new Date().toISOString()}`);
      await handler();  // 执行实际任务
      console.log(`[${taskName}] 任务完成: ${new Date().toISOString()}`);
    } catch (error) {
      // 捕获所有错误，防止任务崩溃导致调度器停止
      console.error(`[${taskName}] 任务异常:`, error);
    }
  }, {
    timezone: options?.timezone,
    scheduled: true,
  });
}

// 使用包装器创建带错误处理的任务
const reportTask = createSafeTask(
  '0 9 * * 1',  // 每周一上午9点
  async () => {
    console.log('正在生成周报...');
    await generateWeeklyReport();
  },
  { name: 'weekly-report', timezone: 'Asia/Shanghai' }
);
```

### 1.3 node-schedule 更精确的定时

node-schedule 相比 node-cron 提供了更精确的时间控制，支持基于日期的调度，适合需要特定日期时间的任务。

#### 1.3.1 安装与基础用法

```bash
npm install node-schedule
```

```typescript
// 引入 node-schedule 库，支持更精确的时间调度

// 示例1：在特定日期执行一次任务
```

```typescript
function createRecurringTask(): ScheduleJob {

  const rule = '0 0 * * *';

  // 使用递归规则

// 条件取消：在特定条件下取消任务
function createConditionalTask(): ScheduleJob {
  let executionCount = 0;  // 记录执行次数
  const maxExecutions = 5; // 最大执行次数

  return schedule.scheduleJob('0 * * * *', function() {
    executionCount++;
    console.log(`任务执行第 ${executionCount} 次`);

    // 执行5次后自动取消
    if (executionCount >= maxExecutions) {
      console.log('达到最大执行次数，任务将自动取消');
      (this as ScheduleJob).cancel();  // this 指向当前任务
    }
  });

  nextInvocation: Date | null;
```

#### 1.4.2 缓存清理场景

```typescript
import cron from 'node-cron';
import Redis from 'ioredis';

// 创建 Redis 客户端连接
const redis = new Redis({
  host: process.env.REDIS_HOST || 'localhost',
  port: parseInt(process.env.REDIS_PORT || '6379'),
  password: process.env.REDIS_PASSWORD,
});

interface CacheStats {
  // 缓存统计信息接口
  totalKeys: number;       // 总键数量
  expiredKeys: number;     // 过期的键数量
  memoryUsed: string;      // 内存使用量
}

interface CleanupResult {
  // 清理结果接口
  scannedKeys: number;     // 扫描的键数量
  deletedKeys: number;     // 删除的键数量
  executionTime: number;   // 执行时间（毫秒）
}

// 缓存健康检查定时任务
function setupCacheHealthCheck(): void {
  // 每10分钟检查一次缓存状态
  cron.schedule('*/10 * * * *', async () => {
    console.log('[缓存健康检查] 开始检查...');

    try {
      // 获取缓存统计信息
      const stats = await getCacheStats();
      console.log('[缓存健康检查] 统计信息:', stats);

      // 检查缓存内存使用情况
      const info = await redis.info('memory');
      console.log('[缓存健康检查] 内存信息:', info);
    } catch (error) {
      console.error('[缓存健康检查] 检查失败:', error);
    }
  });
}

// 获取缓存统计信息
async function getCacheStats(): Promise<CacheStats> {
  // 使用 Redis DBSIZE 命令获取键数量
  const keyCount = await redis.dbsize();

  // 获取内存使用信息
  const memoryInfo = await redis.info('memory');
  const memoryMatch = memoryInfo.match(/used_memory_human:(\S+)/);

  return {
    totalKeys: keyCount,
    expiredKeys: 0,  // 需要使用 SCAN 和 TTL 命令计算
    memoryUsed: memoryMatch ? memoryMatch[1] : 'unknown',
  };
}

// 定期清理过期缓存的任务
function setupExpiredCacheCleanup(): void {
  // 每小时清理一次过期缓存
  cron.schedule('0 * * * *', async () => {
    const startTime = Date.now();
    console.log('[过期缓存清理] 开始清理...');

    try {
      const result = await cleanupExpiredKeys();
      console.log(`[过期缓存清理] 完成: ${JSON.stringify(result)}`);
    } catch (error) {
      console.error('[过期缓存清理] 清理失败:', error);
    }
  });
}

// 清理过期键的具体实现
async function cleanupExpiredKeys(): Promise<CleanupResult> {
  let scannedKeys = 0;
  let deletedKeys = 0;
  const cursor = '0';  // SCAN 游标

  // 使用 SCAN 命令遍历所有键（避免阻塞）
  do {
    // SCAN 命令返回 [nextCursor, keys] 元组
    const [nextCursor, keys] = await redis.scan(
      cursor,           // 游标位置
      'COUNT', 1000,    // 每次扫描数量
      'MATCH', 'cache:*',  // 只扫描 cache: 开头的键
    ) as [string, string[]];

    scannedKeys += keys.length;

    // 检查每个键的 TTL，只删除已过期的键
    for (const key of keys) {
      // TTL 返回 -2 表示键不存在，-1 表示没有设置过期时间
      const ttl = await redis.ttl(key);
      if (ttl === -2 || ttl === -1) {
        // 键已过期或无过期时间，但 Redis 会自动清理，这里可以记录日志
      }
    }

    // 更新游标
    cursor = nextCursor;
  } while (cursor !== '0');  // 游标为0表示遍历完成

  return {
    scannedKeys,
    deletedKeys,
    executionTime: Date.now() - startTime,
  };
}
```

### 1.5 分布式定时任务问题

在单机环境下，定时任务实现简单。但在分布式环境下，多个实例同时执行同一个定时任务会导致重复处理、数据不一致等问题。

#### 1.5.1 分布式问题分析

```
┌─────────────────────────────────────────────────────────────┐
│                      分布式定时任务问题                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   实例 A (Node 1)          实例 B (Node 2)                  │
│       │                         │                           │
│       ▼                         ▼                           │
│   ┌─────────┐               ┌─────────┐                     │
│   │ 定时任务 │               │ 定时任务 │    ← 同一任务重复执行 │
│   │ 2:00 AM │               │ 2:00 AM │                     │
│   └─────────┘               └─────────┘                     │
│       │                         │                           │
│       ▼                         ▼                           │
│   ┌─────────────────────────────┐                           │
│   │        数据库操作            │                           │
│   │   INSERT / UPDATE / DELETE │    ← 数据重复或冲突        │
│   └─────────────────────────────┘                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 1.5.2 分布式锁解决方案

```typescript
import Redis from 'ioredis';

const redis = new Redis({
  host: process.env.REDIS_HOST || 'localhost',
  port: parseInt(process.env.REDIS_PORT || '6379'),
});

// 分布式锁键名前缀
const LOCK_KEY_PREFIX = 'distributed:lock:';
// 锁的默认过期时间（秒）
const DEFAULT_LOCK_TTL = 60;
// 锁等待超时时间（毫秒）
const LOCK_WAIT_TIMEOUT = 5000;

interface LockResult {
  // 获取锁的结果
  success: boolean;      // 是否成功获取锁
  lockValue: string;     // 锁的值，用于释放锁时验证
}

/**
 * 尝试获取分布式锁
 * 使用 Redis SET NX EX 组合命令保证原子性
 */
async function acquireLock(
  lockName: string,      // 锁名称
  ttlSeconds: number = DEFAULT_LOCK_TTL,  // 过期时间
): Promise<LockResult> {
  // 生成唯一的锁值，防止误释放他人的锁
  const lockValue = `${process.pid}-${Date.now()}-${Math.random()}`;

  // 使用 SET 命令的 NX 和 EX 选项
  // NX: 仅当键不存在时设置（保证互斥）
  // EX: 设置过期时间（防止死锁）
  const result = await redis.set(
    `${LOCK_KEY_PREFIX}${lockName}`,
    lockValue,
    'EX',      // 设置过期时间
    ttlSeconds,
    'NX',     // 仅当不存在时设置
  );

  return {
    success: result === 'OK',
    lockValue: result === 'OK' ? lockValue : '',
  };
}

/**
 * 释放分布式锁
 * 使用 Lua 脚本保证释放锁的原子性
 */
async function releaseLock(
  lockName: string,      // 锁名称
  lockValue: string,    // 锁的值，用于验证
): Promise<boolean> {
  // Lua 脚本：只有锁的值匹配时才删除
  // 保证释放锁的原子性，避免误删他人的锁
  const luaScript = `
    if redis.call("get", KEYS[1]) == ARGV[1] then
      return redis.call("del", KEYS[1])
    else
      return 0
    end
  `;

  const result = await redis.eval(
    luaScript,
    1,  // 键的数量
    `${LOCK_KEY_PREFIX}${lockName}`,
    lockValue,
  );

  return result === 1;
}

/**
 * 执行分布式任务
 * 包装函数，确保同一时刻只有一个实例执行任务
 */
async function executeDistributedTask<T>(
  taskName: string,           // 任务名称
  taskHandler: () => Promise<T>,  // 任务处理函数
  options?: {
    ttlSeconds?: number;      // 锁过期时间
    waitTimeout?: number;     // 等待超时
    onLockFailed?: () => void; // 获取锁失败时的回调
  }
): Promise<{ executed: boolean; result?: T; error?: unknown }> {
  const { ttlSeconds = DEFAULT_LOCK_TTL, waitTimeout = LOCK_WAIT_TIMEOUT, onLockFailed } = options || {};

  // 1. 尝试获取分布式锁
  const lockResult = await acquireLock(taskName, ttlSeconds);

  if (!lockResult.success) {
    // 获取锁失败，说明其他实例正在执行
    console.log(`[${taskName}] 获取锁失败，跳过本次执行`);
    onLockFailed?.();
    return { executed: false };
  }

  console.log(`[${taskName}] 获取锁成功，开始执行任务`);

  try {
    // 2. 执行实际任务
    const result = await taskHandler();
    console.log(`[${taskName}] 任务执行完成`);
    return { executed: true, result };
  } catch (error) {
    console.error(`[${taskName}] 任务执行异常:`, error);
    return { executed: true, error };
  } finally {
    // 3. 无论成功与否，都要释放锁
    const released = await releaseLock(taskName, lockResult.lockValue);
    console.log(`[${taskName}] 锁释放${released ? '成功' : '失败'}`);
  }
}

function setupDistributedBackupTask(): void {
```

```typescript
import cron from 'node-cron';
import Redis from 'ioredis';
import { Pool } from 'pg';

// 数据库连接池配置
const dbPool = new Pool({
  host: process.env.DB_HOST || 'localhost',
  port: parseInt(process.env.DB_PORT || '5432'),
  database: process.env.DB_NAME || 'mydb',
  user: process.env.DB_USER || 'postgres',
  password: process.env.DB_PASSWORD,
  max: 20,  // 最大连接数
});

// Redis 客户端
const redis = new Redis({
  host: process.env.REDIS_HOST || 'localhost',
  port: parseInt(process.env.REDIS_PORT || '6379'),
});

/**
 * 数据清理定时任务服务
 * 负责清理过期的临时文件、缓存数据和数据库记录
 */
class DataCleanupService {
  // 服务配置
  private config = {
    // 各清理任务的 Cron 表达式
    cleanupSchedule: {
      tempFiles: '0 3 * * *',      // 每天凌晨3点清理临时文件
      expiredSessions: '0 4 * * *', // 每天凌晨4点清理过期会话
      oldLogs: '0 5 * * 0',        // 每周日凌晨5点清理旧日志
      inactiveUsers: '0 6 1 * *', // 每月1日凌晨6点清理不活跃用户
      cacheData: '0 */4 * * *',   // 每4小时清理一次缓存
    },
    // 保留期限配置（天数）
    retention: {
      tempFiles: 7,         // 临时文件保留7天
      sessions: 30,         // 会话保留30天
      logs: 90,              // 日志保留90天
      inactiveUsers: 365,   // 不活跃用户保留365天
    },
  };

  /**
   * 初始化所有定时任务
   */
  initTasks(): void {
    console.log('初始化数据清理定时任务...');

    // 1. 临时文件清理任务
    cron.schedule(this.config.cleanupSchedule.tempFiles, async () => {
      console.log('[临时文件清理] 开始执行...');
      await this.cleanupTempFiles();
    });

    // 2. 过期会话清理任务
    cron.schedule(this.config.cleanupSchedule.expiredSessions, async () => {
      console.log('[过期会话清理] 开始执行...');
      await this.cleanupExpiredSessions();
    });

    // 3. 旧日志清理任务
    cron.schedule(this.config.cleanupSchedule.oldLogs, async () => {
      console.log('[旧日志清理] 开始执行...');
      await this.cleanupOldLogs();
    });

    // 4. 不活跃用户清理任务
    cron.schedule(this.config.cleanupSchedule.inactiveUsers, async () => {
      console.log('[不活跃用户清理] 开始执行...');
      await this.cleanupInactiveUsers();
    });

    // 5. 缓存数据清理任务
    cron.schedule(this.config.cleanupSchedule.cacheData, async () => {
      console.log('[缓存清理] 开始执行...');
      await this.cleanupCacheData();
    });

    console.log('数据清理定时任务已全部启动');
  }

  /**
   * 清理临时文件
   */
  async cleanupTempFiles(): Promise<{ deleted: number; errors: number }> {
    const result = { deleted: 0, errors: 0 };
    const fs = require('fs').promises;
    const path = require('path');

    // 临时文件目录
    const tempDir = process.env.TEMP_DIR || '/tmp';
    const maxAge = this.config.retention.tempFiles * 24 * 60 * 60 * 1000;  // 转换为毫秒
    const now = Date.now();

    try {
      // 读取临时目录下的所有文件
      const files = await fs.readdir(tempDir);

      for (const file of files) {
        try {
          const filePath = path.join(tempDir, file);
          const stats = await fs.stat(filePath);

          // 检查文件是否过期
          const fileAge = now - stats.mtimeMs;
          if (fileAge > maxAge) {
            await fs.unlink(filePath);
            result.deleted++;
            console.log(`[临时文件清理] 删除文件: ${file}`);
          }
        } catch (error) {
          result.errors++;
          console.error(`[临时文件清理] 删除文件失败: ${file}`, error);
        }
      }
    } catch (error) {
      console.error('[临时文件清理] 扫描目录失败:', error);
    }

    console.log(`[临时文件清理] 完成: 删除 ${result.deleted} 个文件`);
    return result;
  }

  /**
   * 清理过期会话
   */
  async cleanupExpiredSessions(): Promise<{ deleted: number }> {
    const result = { deleted: 0 };
    const thirtyDaysAgo = new Date();
    thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - this.config.retention.sessions);

    try {
      // 从数据库删除过期的会话记录
      const query = `
        DELETE FROM sessions
        WHERE last_accessed_at < $1
        AND created_at < $2
      `;

      const res = await dbPool.query(query, [thirtyDaysAgo, thirtyDaysAgo]);
      result.deleted = res.rowCount || 0;

      console.log(`[过期会话清理] 从数据库删除 ${result.deleted} 条会话记录`);

      // 同时清理 Redis 中的会话数据
      const sessionKeys = await redis.keys('session:*');
      for (const key of sessionKeys) {
        const ttl = await redis.ttl(key);
        // 如果会话已过期或剩余 TTL 过短
        if (ttl < 0) {
          await redis.del(key);
          result.deleted++;
        }
      }

      console.log(`[过期会话清理] 完成: 共清理 ${result.deleted} 个会话`);
    } catch (error) {
      console.error('[过期会话清理] 清理失败:', error);
    }

    return result;
  }

  /**
   * 清理旧日志
   */
  async cleanupOldLogs(): Promise<{ deleted: number; freedSpace: number }> {
    const result = { deleted: 0, freedSpace: 0 };
    const fs = require('fs').promises;
    const path = require('path');

    const logsDir = process.env.LOGS_DIR || './logs';
    const maxAge = this.config.retention.logs;

    try {
      // 计算截止日期
      const cutoffDate = new Date();
      cutoffDate.setDate(cutoffDate.getDate() - maxAge);
      const cutoffStr = cutoffDate.toISOString().split('T')[0];  // YYYY-MM-DD 格式

      const files = await fs.readdir(logsDir);

      for (const file of files) {
        // 只处理 .log 文件
        if (!file.endsWith('.log')) continue;

        // 从文件名中提取日期（假设格式为 app-YYYY-MM-DD.log）
        const dateMatch = file.match(/(\d{4}-\d{2}-\d{2})/);
        if (!dateMatch) continue;

        const fileDate = dateMatch[1];

        // 比较日期字符串
        if (fileDate < cutoffStr) {
          const filePath = path.join(logsDir, file);
          const stats = await fs.stat(filePath);

          await fs.unlink(filePath);
          result.deleted++;
          result.freedSpace += stats.size;

          console.log(`[旧日志清理] 删除日志: ${file} (${(stats.size / 1024 / 1024).toFixed(2)} MB)`);
        }
      }

      console.log(`[旧日志清理] 完成: 删除 ${result.deleted} 个文件，释放 ${(result.freedSpace / 1024 / 1024).toFixed(2)} MB`);
    } catch (error) {
      console.error('[旧日志清理] 清理失败:', error);
    }

    return result;
  }

  /**
   * 清理不活跃用户
   */
  async cleanupInactiveUsers(): Promise<{ archived: number; deleted: number }> {
    const result = { archived: 0, deleted: 0 };

    try {
      const oneYearAgo = new Date();
      oneYearAgo.setFullYear(oneYearAgo.getFullYear() - this.config.retention.inactiveUsers);

      // 将不活跃用户移至归档表（保留数据）
      const archiveQuery = `
        INSERT INTO users_archive
        SELECT * FROM users
        WHERE last_login_at < $1
        AND created_at < $1
      `;

      await dbPool.query(archiveQuery, [oneYearAgo]);

      // 从原表删除
      const deleteQuery = `
        DELETE FROM users
        WHERE last_login_at < $1
        AND created_at < $1
      `;

      const res = await dbPool.query(deleteQuery, [oneYearAgo]);
      result.archived = result.deleted = res.rowCount || 0;

      console.log(`[不活跃用户清理] 归档并删除 ${result.deleted} 个不活跃用户`);
    } catch (error) {
      console.error('[不活跃用户清理] 清理失败:', error);
    }

    return result;
  }

  /**
   * 清理缓存数据
   */
  async cleanupCacheData(): Promise<{ deleted: number }> {
    const result = { deleted: 0 };

    try {
      // 扫描所有缓存键
      const cacheKeys = await redis.keys('cache:*');

      for (const key of cacheKeys) {
        // 获取键的元数据（如果有）
        const metaKey = `${key}:meta`;
        const meta = await redis.get(metaKey);

        if (meta) {
          const { expiresAt } = JSON.parse(meta);
          // 检查是否已过期
          if (new Date(expiresAt) < new Date()) {
            await redis.del(key, metaKey);
            result.deleted++;
          }
        } else {
          // 如果没有元数据，检查 Redis 内置 TTL
          const ttl = await redis.ttl(key);
          if (ttl === -1) {
            // 永不过期的缓存，需要人工判断是否删除
            // 这里我们保留，只删除明确设置过期的
          }
        }
      }

      console.log(`[缓存清理] 完成: 删除 ${result.deleted} 个过期缓存项`);
    } catch (error) {
      console.error('[缓存清理] 清理失败:', error);
    }

    return result;
  }
}

// 创建并启动清理服务
const cleanupService = new DataCleanupService();
cleanupService.initTasks();

console.log('数据清理定时任务服务已启动');
```

## 第二部分：消息队列详解

### 2.1 什么是消息队列

消息队列（Message Queue）是一种进程间通信机制，用于在不同服务之间异步传递消息。它是现代分布式系统架构的核心组件之一。

```
┌──────────────────────────────────────────────────────────────────┐
│                        消息队列架构图                              │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌────────┐         ┌──────────────┐         ┌────────┐         │
│   │ 生产者 │ ──────► │   消息队列    │ ──────► │ 消费者 │         │
│   │Producer│         │Message Queue │         │Consumer│         │
│   └────────┘         └──────────────┘         └────────┘         │
│                                                                  │
│   - 订单服务          - RabbitMQ            - 库存服务             │
│   - 支付服务          - BullMQ              - 通知服务             │
│   - 用户服务          - Kafka               - 日志服务             │
│                                                                  │
│                         │                                       │
│                         ▼                                       │
│                  ┌──────────────┐                                │
│                  │    消息      │                                 │
│                  │   持久化     │                                 │
│                  └──────────────┘                                │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 2.2 为什么需要消息队列

| 特性 | 说明 | 好处 |
|------|------|------|
| **异步处理** | 生产者发送消息后立即返回，不等待处理完成 | 提高系统响应速度 |
| **解耦** | 生产者和消费者独立存在，通过队列通信 | 提高系统可维护性 |
| **削峰填谷** | 缓解瞬时流量压力，平滑处理 | 保护系统不被冲垮 |
| **可靠性** | 消息持久化 + 重试机制 | 确保消息不丢失 |
| **顺序保证** | 队列中的消息按顺序处理 | 保证业务逻辑正确性 |

```typescript
  'notification',
  async (job) => {
    const { userId, type, content } = job.data;

    console.log(`[通知 ${type}] 发送给用户 ${userId}，尝试次数: ${job.attemptsMade + 1}`);

    try {
      // 根据通知类型调用不同的发送服务
      switch (type) {
        case 'email':
          await sendEmail(userId, content);
          break;
        case 'sms':
          await sendSMS(userId, content);
          break;
        case 'push':
          await sendPushNotification(userId, content);
          break;
        default:
          console.warn(`[通知] 未知类型: ${type}`);
      }

      return { success: true, sentAt: new Date().toISOString() };
    } catch (error) {
      // 判断错误类型，决定是否重试
      const errorMessage = error instanceof Error ? error.message : String(error);

      // 网络错误可以重试，业务错误不重试
      if (isNetworkError(error)) {

      } else if (isBusinessError(error)) {

        // 标记为失败但不重试
        throw new Error('SKIP_RETRY');  // 使用特殊标记跳过重试
      } else {
        throw error;
      }
    }
  },
  {
    connection: redisConnection,
    // 并发处理数量
    concurrency: 20,
```

### 2.5 实战：实现订单超时取消功能

```typescript
/**
 * 创建订单服务
 * 模拟用户下单流程
 */
class OrderService {
  /**
   * 创建新订单
   */
  async createOrder(
    userId: string,
    items: Array<{ productId: string; productName: string; quantity: number; price: number }>,
    paymentMethod: string,
    timeoutMinutes: number = 30
  ): Promise<Order> {
    // 生成订单ID
    const orderId = `ORD-${Date.now()}-${Math.random().toString(36).slice(2, 6).toUpperCase()}`;

    // 计算订单总金额
    const totalAmount = items.reduce((sum, item) => sum + item.quantity * item.price, 0);

    // 检查并扣减库存
    for (const item of items) {
      const currentInventory = inventoryDb.get(item.productId) || 0;
      if (currentInventory < item.quantity) {
        throw new Error(`商品 ${item.productName} 库存不足`);
      }
    }

    // 创建订单
    const order: Order = {
      orderId,
      userId,
      items: items.map(item => ({
        ...item,
        inventory: inventoryDb.get(item.productId) || 0,
      })),
      totalAmount,
      status: 'pending_payment',
      paymentMethod,
      createdAt: new Date(),
    };

    // 保存到数据库
    ordersDb.set(orderId, order);

    // 扣减库存
    for (const item of items) {
      inventoryDb.set(item.productId, (inventoryDb.get(item.productId) || 0) - item.quantity);
    }

    console.log(`[订单创建] 订单 ${orderId} 已创建，总金额: ${totalAmount}，待支付`);

    // 创建超时检查任务
    await this.createTimeoutJob(orderId, userId, timeoutMinutes);

    return order;
  }

   * 创建超时检查延时任务
   */

    orderId: string,
    userId: string,
    timeoutMinutes: number
  ): Promise<void> {

    await orderTimeoutQueue.add(
      'check-order-timeout',
      {
        orderId,
        userId,
        createdAt: new Date().toISOString(),
        timeoutMinutes,
      },
      {
        jobId: `timeout-${orderId}`,  // 唯一ID，防止重复创建
        delay: timeoutMs,              // 延时执行
        priority: 1,                   // 较高优先级
      }
    );

    console.log(`[超时任务] 已创建订单 ${orderId} 的超时检查任务，${timeoutMinutes} 分钟后执行`);
  }

   * 取消订单
   */
  async cancelOrder(orderId: string, reason: string): Promise<boolean> {
    const order = ordersDb.get(orderId);

    if (!order) {
      console.error(`[取消订单] 订单不存在: ${orderId}`);
      return false;
    }

    if (order.status !== 'pending_payment') {
      console.error(`[取消订单] 订单状态不允许取消: ${order.status}`);
      return false;
    }

    // 更新订单状态
    order.status = 'canceled';
    order.canceledAt = new Date();
    order.cancelReason = reason;

    // 恢复库存
    await this.restoreInventory(order);

    console.log(`[取消订单] 订单 ${orderId} 已取消，原因: ${reason}`);

    return true;
  }

  /**
   * 恢复库存
   */
  async restoreInventory(order: Order): Promise<void> {
    for (const item of order.items) {
      const currentInventory = inventoryDb.get(item.productId) || 0;
      inventoryDb.set(item.productId, currentInventory + item.quantity);
      console.log(`[库存恢复] 商品 ${item.productName} 库存 +${item.quantity}，当前: ${inventoryDb.get(item.productId)}`);
    }
  }

   * 获取订单状态
   */

    return ordersDb.get(orderId);
  }

  /**
   * 标记订单已支付
   */
  async markOrderPaid(orderId: string): Promise<boolean> {
    const order = ordersDb.get(orderId);

    if (!order) {
      return false;
    }

    if (order.status !== 'pending_payment') {
      console.log(`[支付订单] 订单 ${orderId} 状态不是待支付，当前状态: ${order.status}`);
      return false;
    }

    order.status = 'paid';
    order.paidAt = new Date();

    console.log(`[支付订单] 订单 ${orderId} 已支付`);

    return true;
  }
```

#### 3.2.1 ioredis 客户端详解

```typescript
import Redis from 'ioredis';

/**
 * ioredis 是 Node.js 中最流行的 Redis 客户端
 * 支持哨兵模式、集群模式、Lua 脚本、发布订阅等
 */

// 创建普通 Redis 客户端
const redis = new Redis({
  host: process.env.REDIS_HOST || 'localhost',
  port: parseInt(process.env.REDIS_PORT || '6379'),
  password: process.env.REDIS_PASSWORD,

  // 连接池配置
  maxRetriesPerRequest: 3,    // 每个请求最大重试次数
  enableReadyCheck: true,    // 连接就绪检查
  connectTimeout: 10000,      // 连接超时（毫秒）
  commandTimeout: 5000,        // 命令超时（毫秒）

  // 重连配置
  retryStrategy: (times) => {
    const delay = Math.min(times * 50, 2000);  // 指数退避，最大2秒
    return delay;
  },

  // Lua 脚本配置
  luaScript: {
    numberBigKeys: 10,        // 大 key 阈值
  },
});

// 连接事件
redis.on('connect', () => {
  console.log('Redis 连接成功');
});

redis.on('error', (err) => {
  console.error('Redis 连接错误:', err);
});

redis.on('close', () => {
  console.log('Redis 连接已关闭');
});

// 基础操作封装示例
class RedisService {
  /**
   * 通用缓存获取方法（Cache-Aside 模式）
   */
  async cacheAside<T>(
    key: string,                    // 缓存键
    fetchFn: () => Promise<T>,      // 数据获取函数
    options?: {
      ttl?: number;                 // 过期时间（秒）
      prefix?: string;              // 键前缀
    }
  ): Promise<T> {
    const { ttl = 3600, prefix = 'cache' } = options || {};
    const cacheKey = `${prefix}:${key}`;

    // 1. 先从缓存获取
    const cached = await redis.get(cacheKey);
    if (cached) {
      console.log(`[缓存命中] ${cacheKey}`);
      return JSON.parse(cached) as T;
    }

    // 2. 缓存未命中，从数据源获取
    console.log(`[缓存未命中] ${cacheKey}，从数据源获取`);
    const data = await fetchFn();

    // 3. 写入缓存
    await redis.setex(cacheKey, ttl, JSON.stringify(data));

    return data;
  }

  /**
   * 批量缓存获取（减少网络往返）
   */
  async mgetCacheAside<T>(
    keys: string[],                 // 缓存键数组
    fetchFn: (missingKeys: string[]) => Promise<Map<string, T>>,  // 批量获取函数
    options?: { ttl?: number; prefix?: string }
  ): Promise<Map<string, T>> {
    const { ttl = 3600, prefix = 'cache' } = options || {};
    const cacheKeys = keys.map(k => `${prefix}:${k}`);

    // 1. 批量从缓存获取
    const cachedValues = await redis.mget(...cacheKeys);

    const result = new Map<string, T>();
    const missingKeys: string[] = [];

    // 2. 分析哪些键未命中
    keys.forEach((key, index) => {
      if (cachedValues[index]) {
        result.set(key, JSON.parse(cachedValues[index]) as T);
      } else {
        missingKeys.push(key);
      }
    });

    // 3. 获取缺失的数据
    if (missingKeys.length > 0) {
      console.log(`[缓存批量] 未命中 ${missingKeys.length} 个键`);
      const fetchedData = await fetchFn(missingKeys);

      // 4. 写入缓存
      const pipeline = redis.pipeline();
      fetchedData.forEach((value, key) => {
        pipeline.setex(`${prefix}:${key}`, ttl, JSON.stringify(value));
      });
      await pipeline.exec();

      // 5. 合并结果
      fetchedData.forEach((value, key) => {
        result.set(key, value);
      });
    }

    return result;
  }

  /**
   * 缓存删除（支持通配符）
   */
  async clearCache(pattern: string): Promise<number> {
    const keys = await redis.keys(pattern);
    if (keys.length === 0) {
      return 0;
    }

    const deleted = await redis.del(...keys);
    console.log(`[缓存清理] 已删除 ${deleted} 个键，模式: ${pattern}`);

    return deleted;
  }

  /**
   * 缓存更新（删除旧缓存，下次访问时自动重新加载）
   */
  async invalidateCache(key: string, prefix: string = 'cache'): Promise<void> {
    await redis.del(`${prefix}:${key}`);
    console.log(`[缓存失效] ${prefix}:${key}`);
  }
}
```

#### 3.3.2 三大缓存问题

```
┌─────────────────────────────────────────────────────────────────┐
│                        缓存三大问题                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 缓存穿透（Cache Penetration）                                 │
│  ─────────────────────────────────────────────────────           │
│  请求查询一个不存在的数据，缓存和数据库都没有                      │
│  解决方案：                                                       │
│  • 布隆过滤器（Bloom Filter）                                     │
│  • 空值缓存（设置短过期时间）                                      │
│  • 参数合法性校验                                                 │
│                                                                 │
│  2. 缓存雪崩（Cache Avalanche）                                   │
│  ─────────────────────────────────────────────────────           │
│  大量缓存同时过期，导致大量请求直接打到数据库                       │
│  解决方案：                                                       │
│  • 过期时间加随机值（避免同时过期）                                │
│  • 多级缓存架构                                                   │
│  • 热点数据永不过期                                               │
│                                                                 │
│  3. 缓存击穿（Cache Breakdown）                                   │
│  ─────────────────────────────────────────────────────           │
│  热点 key 过期瞬间，大量并发请求涌入查询数据库                      │
│  解决方案：                                                       │
│  • 互斥锁（分布式锁）                                             │
│  • 热点数据永不过期 + 后台异步更新                                │
│  • 逻辑过期时间（不实际删除，只标记过期）                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```typescript
// 缓存问题解决方案实现

/**
 * 布隆过滤器 - 解决缓存穿透
 * 原理：使用多个哈希函数将数据映射到一个位数组
 * 特点：宁可误判也不漏判（存在不一定存在，不存在一定不存在）
 */
import Redis from 'ioredis';

const redis = new Redis();

class BloomFilter {
  // 布隆过滤器的 Redis Key
  private key: string;
  // 位数组大小
  private size: number;
  // 哈希函数数量
  private hashCount: number;

  constructor(key: string, size: number = 100000, hashCount: number = 7) {
    this.key = key;
    this.size = size;
    this.hashCount = hashCount;
  }

  /**
   * MurmurHash3 算法实现
   * 这是一个快速的非加密哈希函数，适合布隆过滤器使用
   */
  private murmurhash3(key: string, seed: number): number {
    const c1 = 0xcc9e2d51;
    const c2 = 0x1b873593;
    const r1 = 15;
    const r2 = 13;
    const m = 5;
    const n = 0xe6546b64;

    let h = seed;
    const k = key;
    const bytes = Buffer.from(k, 'utf8');
    const len = bytes.length;
    const nblocks = Math.floor(len / 4);

    // 处理每个4字节块
    for (let i = 0; i < nblocks; i++) {
      let kk = (
        (bytes[i * 4] & 0xff) |
        ((bytes[i * 4 + 1] & 0xff) << 8) |
        ((bytes[i * 4 + 2] & 0xff) << 16) |
        ((bytes[i * 4 + 3] & 0xff) << 24)
      );

      kk = Math.imul(kk, c1);
      kk = (kk << r1) | (kk >>> (32 - r1));
      kk = Math.imul(kk, c2);

      h ^= kk;
      h = (h << r2) | (h >>> (32 - r2));
      h = Math.imul(h, m) + n;
    }

    // 处理剩余字节
    const remainder = len % 4;
    let kk = 0;
    if (remainder >= 3) kk ^= (bytes[len - 3] & 0xff) << 16;
    if (remainder >= 2) kk ^= (bytes[len - 2] & 0xff) << 8;
    if (remainder >= 1) kk ^= bytes[len - 1];

    if (remainder > 0) {
      kk = Math.imul(kk, c1);
      kk = (kk << r1) | (kk >>> (32 - r1));
      kk = Math.imul(kk, c2);
      h ^= kk;
    }

    // 最终混合
    h ^= len;
    h ^= h >>> 16;
    h = Math.imul(h, 0x85ebca6b);
    h ^= h >>> 13;
    h = Math.imul(h, 0xc2b2ae35);
    h ^= h >>> 16;

    return h >>> 0;  // 返回无符号整数
  }

  /**
   * 计算元素在位数组中的多个位置
   */
  private getHashPositions(value: string): number[] {
    const positions: number[] = [];

    for (let i = 0; i < this.hashCount; i++) {
      // 使用不同的种子生成不同的哈希值
      const hash = this.murmurhash3(value, i * 1000);
      const position = hash % this.size;
      positions.push(position);
    }

    return positions;
  }

  /**
   * 添加元素到布隆过滤器
   */
  async add(value: string): Promise<void> {
    const positions = this.getHashPositions(value);

    // 使用 Lua 脚本一次性设置多个位
    const luaScript = `
      for _, pos in ipairs(ARGV) do
        redis.call('SETBIT', KEYS[1], pos, 1)
      end
      return 1
    `;

    await redis.eval(
      luaScript,
      1,
      this.key,
      ...positions.map(p => p.toString())
    );
  }

  /**
   * 检查元素是否可能存在
   */
  async mightContain(value: string): Promise<boolean> {
    const positions = this.getHashPositions(value);

    // 使用 Lua 脚本一次性检查多个位
    const luaScript = `
      for _, pos in ipairs(ARGV) do
        if redis.call('GETBIT', KEYS[1], pos) == 0 then
          return 0
        end
      end
      return 1
    `;

    const result = await redis.eval(
      luaScript,
      1,
      this.key,
      ...positions.map(p => p.toString())
    );

    return result === 1;
  }

  /**
   * 批量添加
   */
  async addMany(values: string[]): Promise<void> {
    for (const value of values) {
      await this.add(value);
    }
  }
}

/**
 * 互斥锁 + 逻辑过期 - 解决缓存击穿
 */
class CacheBreakerProtection {
  private redis: Redis;
  private lockKey: string;
  private lockValue: string;

  constructor(redis: Redis) {
    this.redis = redis;
    this.lockKey = 'lock:cache:';
    this.lockValue = `${process.pid}-${Date.now()}-${Math.random()}`;
  }

  /**
   * 获取数据（带击穿保护）
   * 特点：
   * 1. 如果缓存命中，直接返回
   * 2. 如果缓存未命中，尝试获取锁
   * 3. 获取锁成功后，后台更新缓存
   * 4. 未获取锁时，返回旧数据（如果有）
   */
  async getWithBreakdownProtection<T>(
    key: string,
    fetchFn: () => Promise<T>,
    options?: { ttl?: number; lockTtl?: number }
  ): Promise<T | null> {
    const { ttl = 3600, lockTtl = 10 } = options || {};

    // 1. 尝试从缓存获取
    const cached = await this.redis.get(key);

    if (cached) {
      // 缓存命中
      const data = JSON.parse(cached) as T & { __expiresAt__?: number };

      // 检查是否逻辑过期
      if (data.__expiresAt__ && Date.now() < data.__expiresAt__) {
        return data;
      }

      // 已过期，尝试更新（击穿保护核心）
      if (data.__expiresAt__ && Date.now() >= data.__expiresAt__) {
        // 尝试获取锁
        const lockAcquired = await this.acquireLock(`${this.lockKey}${key}`, lockTtl);

        if (lockAcquired) {
          // 获取锁成功，异步更新缓存
          console.log(`[击穿保护] 获取锁成功，异步更新缓存: ${key}`);
          this.refreshCache(key, fetchFn, ttl).catch(err => {
            console.error(`[击穿保护] 更新缓存失败:`, err);
          });
        } else {
          console.log(`[击穿保护] 获取锁失败，返回旧数据: ${key}`);
        }
      }

      return data;
    }

    // 2. 缓存完全未命中，必须从数据库获取
    console.log(`[击穿保护] 缓存未命中，从数据库获取: ${key}`);
    const data = await fetchFn();

    // 写入缓存
    await this.redis.setex(key, ttl, JSON.stringify(data));

    return data;
  }

  /**
   * 异步刷新缓存
   */
  private async refreshCache<T>(
    key: string,
    fetchFn: () => Promise<T>,
    ttl: number
  ): Promise<void> {
    try {
      const data = await fetchFn();
      await this.redis.setex(key, ttl, JSON.stringify(data));
      console.log(`[击穿保护] 缓存已更新: ${key}`);
    } finally {
      // 释放锁
      await this.releaseLock(key);
    }
  }

  /**
   * 获取分布式锁
   */
  private async acquireLock(lockKey: string, ttlSeconds: number): Promise<boolean> {
    const result = await this.redis.set(lockKey, this.lockValue, 'EX', ttlSeconds, 'NX');
    return result === 'OK';
  }

  /**
   * 释放分布式锁
   */
  private async releaseLock(key: string): Promise<void> {
    const lockKey = `${this.lockKey}${key}`;
    const luaScript = `
      if redis.call("get", KEYS[1]) == ARGV[1] then
        return redis.call("del", KEYS[1])
      else
        return 0
      end
    `;
    await this.redis.eval(luaScript, 1, lockKey, this.lockValue);
  }
}
```

```typescript
  private slaves: Redis[];

    this.slaves = slaves;

    return slave.get(key);

    operation: 'read' | 'write',
    key: string,
    value?: string
  ): Promise<string | null | 'OK'> {
    if (operation === 'write') {
      return this.write(key, value!);
    }
    return this.read(key);
  }
```

### 4.3 分布式锁

```typescript
/**
 * 分布式锁实现
 *
 * 分布式锁的应用场景：
 * 1. 防止缓存击穿（热点 key 并发重建）
 * 2. 幂等性控制（防止重复提交）
 * 3. 任务调度（保证任务只被执行一次）
 * 4. 分布式事务（乐观锁）
 */

import Redis from 'ioredis';

const redis = new Redis({
  host: process.env.REDIS_HOST || 'localhost',
  port: parseInt(process.env.REDIS_PORT || '6379'),
});

/**
 * 分布式锁选项
 */
interface DistributedLockOptions {
  // 锁的过期时间（秒）
  ttl: number;
  // 获取锁的重试次数
  retries?: number;
  // 重试间隔（毫秒）
  retryDelay?: number;
  // 锁值前缀
  prefix?: string;
}

/**
 * 分布式锁结果
 */
interface LockResult {
  // 是否成功获取锁
  acquired: boolean;
  // 锁的标识值（释放锁时需要）
  token: string;
}

/**
 * 分布式锁类
 */
class DistributedLock {
  private redis: Redis;
  private prefix: string;

  constructor(redis: Redis, prefix: string = 'lock:') {
    this.redis = redis;
    this.prefix = prefix;
  }

  /**
   * 尝试获取锁
   *
   * 使用 SET key value NX EX ttl 命令保证：
   * 1. 原子性：NX 保证只有键不存在时才能设置成功
   * 2. 过期时间：EX 保证锁不会永久占用
   */
  async acquire(
    resource: string,           // 资源标识
    options: DistributedLockOptions
  ): Promise<LockResult> {
    const {
      ttl,
      retries = 0,
      retryDelay = 100,
    } = options;

    // 生成唯一的锁值，防止误释放他人的锁
    const token = `${process.pid}-${Date.now()}-${Math.random().toString(36).slice(2)}`;
    const lockKey = `${this.prefix}${resource}`;

    // 尝试获取锁
    for (let attempt = 0; attempt <= retries; attempt++) {
      // SET NX EX 原子操作
      const result = await this.redis.set(lockKey, token, 'EX', ttl, 'NX');

      if (result === 'OK') {
        return { acquired: true, token };
      }

      // 获取锁失败，等待后重试
      if (attempt < retries) {
        await this.sleep(retryDelay * (attempt + 1));  // 递增等待时间
      }
    }

    return { acquired: false, token: '' };
  }

  /**
   * 释放锁
   *
   * 使用 Lua 脚本保证原子性：
   * 只有锁的值与当前持有的值相同时才删除
   * 防止误删他人持有的锁
   */
  async release(resource: string, token: string): Promise<boolean> {
    const lockKey = `${this.prefix}${resource}`;

    // Lua 脚本：验证锁值后删除
    const luaScript = `
      if redis.call("get", KEYS[1]) == ARGV[1] then
        return redis.call("del", KEYS[1])
      else
        return 0
      end
    `;

    const result = await this.redis.eval(luaScript, 1, lockKey, token);
    return result === 1;
  }

  /**
   * 延长锁的过期时间
   *
   * 适用场景：
   * 执行任务时间不确定，可能超过锁的 TTL
   * 需要在任务执行期间续期锁
   */
  async extend(
    resource: string,
    token: string,
    additionalTtl: number
  ): Promise<boolean> {
    const lockKey = `${this.prefix}${resource}`;

    // Lua 脚本：验证锁值后延长过期时间
    const luaScript = `
      if redis.call("get", KEYS[1]) == ARGV[1] then
        return redis.call("expire", KEYS[1], ARGV[2])
      else
        return 0
      end
    `;

    const result = await this.redis.eval(
      luaScript,
      1,
      lockKey,
      token,
      additionalTtl.toString()
    );

    return result === 1;
  }

  /**
   * 执行带锁的任务
   *
   * 封装获取锁、执行任务、释放锁的完整流程
   */
  async withLock<T>(
    resource: string,
    task: () => Promise<T>,
    options: DistributedLockOptions
  ): Promise<{ executed: boolean; result?: T; error?: unknown }> {
    // 1. 获取锁
    const lockResult = await this.acquire(resource, options);

    if (!lockResult.acquired) {
      console.log(`[分布式锁] 获取锁失败: ${resource}`);
      return { executed: false };
    }

    console.log(`[分布式锁] 获取锁成功: ${resource}`);

    try {
      // 2. 执行任务
      const result = await task();
      return { executed: true, result };
    } catch (error) {
      return { executed: true, error };
    } finally {
      // 3. 释放锁
      const released = await this.release(resource, lockResult.token);
      console.log(`[分布式锁] 释放锁 ${released ? '成功' : '失败'}: ${resource}`);
    }
  }

  /**
   * 睡眠函数
   */
  private sleep(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
}

// 创建分布式锁实例
const distributedLock = new DistributedLock(redis, 'lock:');

/**
 * 分布式锁使用示例：防止缓存击穿
 */
async function getDataWithLock<T>(
  key: string,
  fetchFn: () => Promise<T>,
  options?: { ttl?: number; lockTtl?: number }
): Promise<T> {
  const { ttl = 3600, lockTtl = 10 } = options || {};

  // 1. 先从缓存获取
  const cached = await redis.get(`cache:${key}`);
  if (cached) {
    return JSON.parse(cached);
  }

  // 2. 缓存未命中，尝试获取锁
  const lockResult = await distributedLock.acquire(`cache:${key}`, {
    ttl: lockTtl,
    retries: 3,
    retryDelay: 100,
  });

  if (!lockResult.acquired) {
    // 获取锁失败，短暂等待后重试获取缓存
    await new Promise(resolve => setTimeout(resolve, 100));
    const retryCached = await redis.get(`cache:${key}`);
    if (retryCached) {
      return JSON.parse(retryCached);
    }
    throw new Error('获取数据失败，请稍后重试');
  }

  try {
    // 3. 获取锁成功，查询数据库
    const data = await fetchFn();

    // 4. 写入缓存
    await redis.setex(`cache:${key}`, ttl, JSON.stringify(data));

    return data;
  } finally {
    // 5. 释放锁
    await distributedLock.release(`cache:${key}`, lockResult.token);
  }
}
```

### 4.4 布隆过滤器完整实现

```typescript
/**
 * 布隆过滤器（Bloom Filter）实现
 *
 * 原理：
 * 使用一个大的位数组和多个哈希函数
 * 将元素映射到位数组的多个位置
 *
 * 特点：
 * 1. 空间效率高：每个元素只占用几位
 * 2. 查询效率高：O(k)，k 为哈希函数数量
 * 3. 不存在漏报（false negative）：如果说不存在，就一定不存在
 * 4. 存在误报（false positive）：如果说不存在，可能存在
 *
 * 应用场景：
 * 1. 缓存穿透防护
 * 2. 垃圾邮件过滤
 * 3. URL 去重
 * 4. 用户id 存在性判断
 */

class BloomFilter {
  // Redis key 前缀
  private key: string;
  // 位数组大小
  private size: number;
  // 哈希函数数量
  private hashCount: number;
  // Redis 客户端
  private redis: Redis;

  constructor(
    redis: Redis,
    key: string,
    size: number = 1000000,  // 默认 100 万位，约 120KB
    hashCount: number = 7    // 默认 7 个哈希函数
  ) {
    this.redis = redis;
    this.key = `bloom:${key}`;
    this.size = size;
    this.hashCount = hashCount;
  }

  /**
   * MurmurHash3 实现
   * 这是一个非加密哈希函数，性能好，分布均匀
   */
  private murmurhash3(key: string, seed: number): number {
    const c1 = 0xcc9e2d51;
    const c2 = 0x1b873593;
    const r1 = 15;
    const r2 = 13;
    const m = 5;
    const n = 0xe6546b64;

    let h = seed;
    const bytes = Buffer.from(key, 'utf8');
    const len = bytes.length;
    const nblocks = Math.floor(len / 4);

    // 处理每个4字节块
    for (let i = 0; i < nblocks; i++) {
      let kk = (
        (bytes[i * 4] & 0xff) |
        ((bytes[i * 4 + 1] & 0xff) << 8) |
        ((bytes[i * 4 + 2] & 0xff) << 16) |
        ((bytes[i * 4 + 3] & 0xff) << 24)
      );

      kk = Math.imul(kk, c1);
      kk = (kk << r1) | (kk >>> (32 - r1));
      kk = Math.imul(kk, c2);

      h ^= kk;
      h = (h << r2) | (h >>> (32 - r2));
      h = Math.imul(h, m) + n;
    }

    // 处理剩余字节
    const remainder = len % 4;
    let kk = 0;
    if (remainder >= 3) kk ^= (bytes[len - 3] & 0xff) << 16;
    if (remainder >= 2) kk ^= (bytes[len - 2] & 0xff) << 8;
    if (remainder >= 1) kk ^= bytes[len - 1];

    if (remainder > 0) {
      kk = Math.imul(kk, c1);
      kk = (kk << r1) | (kk >>> (32 - r1));
      kk = Math.imul(kk, c2);
      h ^= kk;
    }

    // 最终混合
    h ^= len;
    h ^= h >>> 16;
    h = Math.imul(h, 0x85ebca6b);
    h ^= h >>> 13;
    h = Math.imul(h, 0xc2b2ae35);
    h ^= h >>> 16;

    return h >>> 0;
  }

  /**
   * 计算元素对应的所有哈希位置
   */
  private getHashPositions(value: string): number[] {
    const positions: number[] = [];

    for (let i = 0; i < this.hashCount; i++) {
      // 使用不同的种子生成不同的哈希值
      const hash = this.murmurhash3(value, i * 1000);
      const position = hash % this.size;
      positions.push(position);
    }

    return positions;
  }

  /**
   * 添加元素到布隆过滤器
   *
   * 时间复杂度：O(k)，k 为哈希函数数量
   */
  async add(value: string): Promise<void> {
    const positions = this.getHashPositions(value);

    // 使用 Lua 脚本一次性设置多个位（保证原子性）
    const luaScript = `
      for _, pos in ipairs(ARGV) do
        redis.call('SETBIT', KEYS[1], pos, 1)
      end
      return 1
    `;

    await this.redis.eval(
      luaScript,
      1,
      this.key,
      ...positions.map(p => p.toString())
    );
  }

  /**
   * 批量添加元素
   */
  async addMany(values: string[]): Promise<void> {
    // 使用 Pipeline 优化性能
    const pipeline = this.redis.pipeline();

    for (const value of values) {
      const positions = this.getHashPositions(value);
      for (const pos of positions) {
        pipeline.setbit(this.key, pos, 1);
      }
    }

    await pipeline.exec();
  }

  /**
   * 检查元素是否可能存在于集合中
   *
   * 返回值：
   * - true：元素可能存在（有一定误报率）
   * - false：元素一定不存在
   *
   * 时间复杂度：O(k)
   */
  async mightContain(value: string): Promise<boolean> {
    const positions = this.getHashPositions(value);

    // 使用 Lua 脚本一次性检查多个位
    const luaScript = `
      for _, pos in ipairs(ARGV) do
        if redis.call('GETBIT', KEYS[1], pos) == 0 then
          return 0
        end
      end
      return 1
    `;

    const result = await this.redis.eval(
      luaScript,
      1,
      this.key,
      ...positions.map(p => p.toString())
    );

    return result === 1;
  }

  /**
   * 重置布隆过滤器
   */
  async reset(): Promise<void> {
    await this.redis.del(this.key);
  }

  /**
   * 获取当前使用的位数
   * 用于估算误报率
   */
  async getUsedBits(): Promise<number> {
    // 统计被设置为1的位数
    let usedBits = 0;

    // 分段检查（Redis GETBIT 只能检查单个位）
    // 这里简化处理，实际应用中可以用 Lua 脚本优化
    for (let i = 0; i < this.size; i += 1000) {
      const end = Math.min(i + 1000, this.size);
      for (let j = i; j < end; j++) {
        const bit = await this.redis.getbit(this.key, j);
        if (bit === 1) usedBits++;
      }
    }

    return usedBits;
  }

  /**
   * 估算误报率
   * 基于已使用的位数计算
   */
  async estimateFalsePositiveRate(): Promise<number> {
    const usedBits = await this.getUsedBits();
    const loadFactor = usedBits / this.size;

    // 误报率估算公式：(1 - e^(-kn/m))^k
    // 其中 k=哈希函数数量，n=已添加元素数，m=位数组大小
    // 简化估算：loadFactor 的 k 次方
    return Math.pow(loadFactor, this.hashCount);
  }
}

// 使用示例：用户 ID 去重系统
async function bloomFilterUsageDemo(): Promise<void> {
  const bloom = new BloomFilter(redis, 'user:ids', 1000000, 7);

  // 添加已存在的用户 ID
  await bloom.add('USER-001');
  await bloom.add('USER-002');
  await bloom.add('USER-003');

  // 检查用户 ID 是否可能存在
  console.log('USER-001 可能存在:', await bloom.mightContain('USER-001'));
  console.log('USER-999 可能存在:', await bloom.mightContain('USER-999'));

  // 批量添加
  const userIds = Array.from({ length: 1000 }, (_, i) => `USER-${i}`);
  await bloom.addMany(userIds);

  console.log('USER-500 可能存在:', await bloom.mightContain('USER-500'));
}
```

## 总结

本指南详细介绍了 Node.js 后端开发中的三大核心基础设施：

### 定时任务
- **node-cron**：轻量级基于 Cron 表达式的定时任务库
- **node-schedule**：支持更精确日期调度的定时任务库
- **分布式定时任务**：通过分布式锁解决多实例执行问题
- **实战应用**：数据备份、缓存清理、日志管理

### 消息队列
- **BullMQ**：基于 Redis 的高性能任务队列
- **核心概念**：队列、任务、处理器、重试机制
- **实战应用**：订单超时取消、异步任务处理
- **优先级与延时**：灵活的任务调度能力

### 缓存系统
- **Redis 数据类型**：String、Hash、List、Set、Sorted Set
- **ioredis 客户端**：Node.js 中最流行的 Redis 客户端
- **缓存策略**：Cache-Aside、Read-Through、Write-Through
- **三大问题**：缓存穿透、缓存雪崩、缓存击穿
- **高级特性**：主从复制、哨兵模式、集群、分布式锁、布隆过滤器

这些技术组合使用，可以构建出高可用、高性能的后端系统。建议在实际项目中根据业务需求选择合适的方案，并注意处理各种边界情况和异常状态。
