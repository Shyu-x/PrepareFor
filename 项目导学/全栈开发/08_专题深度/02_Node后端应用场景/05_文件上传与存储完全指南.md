## 一、文件上传基础与Multer中间件

### 1.1 文件上传原理

文件上传是Web应用中最常见的功能之一，其底层基于HTTP协议的多部分表单数据（multipart/form-data）。当用户提交包含文件的表单时，浏览器将文件数据编码为多部分格式，每个部分包含文件名、文件类型和文件内容。

**HTTP多部分表单数据的结构：**

```
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW

------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="file"; filename="avatar.png"
Content-Type: image/png

[文件二进制数据...]
------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="username"

张三
------WebKitFormBoundary7MA4YWxkTrZu0gW--
```

**Node.js文件上传发展历程：**

- **早期方案**：使用`busboy`、`formidable`等流式解析器手动处理
- **Express时代**：`multer`成为标准，整合了文件存储、字段解析、错误处理
- **现代方案**：结合TypeScript类型、内存限制、流式处理、云存储

### 1.2 Multer核心概念

Multer是Express生态中最流行的文件上传中间件，它基于busboy构建，提供了简洁的API来处理`multipart/form-data`格式的请求。

**Multer的核心设计：**

```typescript
// multer核心架构设计
interface MulterOptions {
  // 存储引擎配置
  storage: StorageEngine;
  // 文件大小限制（字节）
  limits: {
    fileSize: number;        // 单个文件最大大小
    files: number;           // 最大文件数量
    fieldSize: number;        // 字段值最大长度
    fieldNameSize: number;    // 字段名最大长度
    parts: number;            // 最大部分数量
  };
  // 文件过滤器
  fileFilter: FileFilterFn;
}

// 存储引擎接口
interface StorageEngine {
  // 处理文件上传
  _handleFile: (req: Request, file: File, cb: (error?: Error | null, info?: Partial<File>) => void) => void;
  // 删除文件
  _removeFile: (req: Request, file: File, cb: (error: Error | null) => void) => void;
}
```

### 1.3 Multer基础使用

**安装依赖：**

```bash
npm install multer @types/multer
```

**Express集成Multer：**

```typescript
// 导入Express和Multer
import express, { Express, Request, Response, NextFunction } from 'express';
import multer, { Multer, FileFilterCallback } from 'multer';
import path from 'path';
import fs from 'fs';

// 创建Express应用实例
const app: Express = express();

// =============================================
// 第一步：配置Multer存储引擎
// =============================================

// 配置磁盘存储引擎 - 将文件保存到本地磁盘
const storage = multer.diskStorage({
  // destination: 指定文件保存的目录
  // req: HTTP请求对象，包含请求头、查询参数等信息
  // file: 上传的文件对象，包含原始文件名、MIME类型等
  // callback: 回调函数，接收错误和目标目录路径
  destination: function (
    req: Request,
    file: Express.Multer.File,
    callback: (error: Error | null, destination: string) => void
  ) {
    // 定义上传目录的基本路径
    const uploadDir = path.join(process.cwd(), 'uploads');

    // 检查目录是否存在，如果不存在则创建
    // recursive: true 表示递归创建所有父级目录
    if (!fs.existsSync(uploadDir)) {
      fs.mkdirSync(uploadDir, { recursive: true });
    }

    // 将目录路径传递给回调函数
    callback(null, uploadDir);
  },

  // filename: 指定保存到磁盘时的文件名
  // 避免中文文件名和特殊字符导致的兼容性问题
  filename: function (
    req: Request,
    file: Express.Multer.File,
    callback: (error: Error | null, filename: string) => void
  ) {
    // 获取文件扩展名（包含点号）
    const extname = path.extname(file.originalname);

    // 生成唯一文件名：时间戳 + 随机数 + 扩展名
    // 这样可以避免文件名冲突，并便于排序
    const uniqueSuffix = Date.now() + '-' + Math.round(Math.random() * 1E9);
    const filename = file.fieldname + '-' + uniqueSuffix + extname;

    callback(null, filename);
  }
});

// =============================================
// 第二步：配置文件过滤器
// =============================================

// 文件过滤器 - 控制哪些文件可以被接受
const fileFilter = function (
  req: Request,
  file: Express.Multer.File,
  callback: FileFilterCallback
) {
  // 定义允许的MIME类型
  // 图片类型通常包括：jpeg、png、gif、webp、svg+xml
  const allowedMimeTypes = [
    'image/jpeg',
    'image/png',
    'image/gif',
    'image/webp',
    'image/svg+xml'
  ];

  // 检查文件的MIME类型是否在允许列表中
  if (allowedMimeTypes.includes(file.mimetype)) {
    // 接受文件：第二个参数为true
    callback(null, true);
  } else {
    // 拒绝文件：第二个参数为false，并传递错误信息
    callback(new Error('不支持的文件类型，仅允许上传图片文件'), false);
  }
};

// =============================================
// 第三步：创建Multer中间件实例
// =============================================

// 创建Multer实例，配置存储引擎和限制
const upload = multer({
  storage: storage,                    // 使用磁盘存储引擎
  limits: {
    fileSize: 5 * 1024 * 1024,         // 单个文件最大5MB
    files: 1,                          // 最多同时上传1个文件
    fieldSize: 1024 * 1024             // 字段值最大1MB
  },
  fileFilter: fileFilter              // 使用文件过滤器
});

// =============================================
// 第四步：定义文件上传路由
// =============================================

// 单文件上传路由
// upload.single('avatar') 会处理名为'avatar'的表单字段
// 'avatar' 必须与前端表单中的 name 属性一致
app.post('/api/upload/single', upload.single('avatar'), function (
  req: Request,
  res: Response
) {
  // 文件上传成功后，req.file包含文件信息
  // 如果上传失败，错误会在中间件链中传播

  // 返回上传成功信息
  res.json({
    success: true,
    message: '文件上传成功',
    file: {
      // 原始文件名（用户电脑上的文件名）
      originalname: req.file?.originalname,
      // 编码格式（通常是7bit）
      encoding: req.file?.encoding,
      // MIME类型
      mimetype: req.file?.mimetype,
      // 文件大小（字节）
      size: req.file?.size,
      // 保存的文件名
      filename: req.file?.filename,
      // 保存的完整路径
      path: req.file?.path,
      // 文件相对于上传目录的路径
      destination: req.file?.destination,
      // 文件扩展名
      ext: path.extname(req.file?.originalname || '')
    }
  });
});

// 多文件上传路由
// upload.array('photos', 5) 处理名为'photos'的多个文件字段，最多5个
app.post('/api/upload/multiple', upload.array('photos', 5), function (
  req: Request,
  res: Response
) {
  // 多文件上传成功后，req.files是一个文件数组
  const files = req.files as Express.Multer.File[];

  res.json({
    success: true,
    message: `成功上传 ${files.length} 个文件`,
    files: files.map(file => ({
      originalname: file.originalname,
      size: file.size,
      mimetype: file.mimetype,
      path: file.path
    }))
  });
});

// 混合字段上传路由
// upload.fields() 处理多个不同名称的文件字段
app.post('/api/upload/mixed', upload.fields([
  { name: 'avatar', maxCount: 1 },     // 头像字段，最多1个
  { name: 'gallery', maxCount: 10 }    // 相册字段，最多10个
]), function (req: Request, res: Response) {
  // 使用fields时，req.files是一个对象，键名是字段名
  const files = req.files as { [fieldname: string]: Express.Multer.File[] };

  res.json({
    success: true,
    avatar: files.avatar?.[0] ? {
      originalname: files.avatar[0].originalname,
      size: files.avatar[0].size
    } : null,
    gallery: files.gallery?.map(f => ({
      originalname: f.originalname,
      size: f.size
    })) || []
  });
});

// =============================================
// 第五步：错误处理中间件
// =============================================

// Multer错误处理中间件
// 需要放在路由定义之后，所有路由的最后
app.use((err: Error, req: Request, res: Response, next: NextFunction) => {
  // 处理Multer特定错误
  if (err instanceof multer.MulterError) {
    // MulterError包含多种错误类型
    switch (err.code) {
      case 'LIMIT_FILE_SIZE':
        // 文件大小超出限制
        return res.status(400).json({
          success: false,
          message: '文件大小超出限制，最大允许5MB'
        });
      case 'LIMIT_FILE_COUNT':
        // 文件数量超出限制
        return res.status(400).json({
          success: false,
          message: '文件数量超出限制'
        });
      case 'LIMIT_UNEXPECTED_FILE':
        // 意外的表单字段（字段名不匹配）
        return res.status(400).json({
          success: false,
          message: '意外的表单字段'
        });
      default:
        return res.status(400).json({
          success: false,
          message: `上传错误：${err.message}`
        });
    }
  }

  // 处理我们自己抛出的错误（如文件类型验证失败）
  if (err.message === '不支持的文件类型，仅允许上传图片文件') {
    return res.status(415).json({
      success: false,
      message: err.message
    });
  }

  // 处理未知错误
  next(err);
});

// 启动服务器
app.listen(3000, () => {
  console.log('文件上传服务已启动，监听端口3000');
});
```

## 二、文件类型验证与安全检查

### 2.1 为什么要验证文件类型

文件上传是Web应用中最常见的安全风险点之一。攻击者可能通过上传恶意文件来执行任意代码、获取服务器权限或窃取数据。常见的攻击方式包括：

- **恶意脚本**：上传包含恶意JavaScript代码的HTML文件
- **WebShell**：上传PHP、ASP、JSP等可执行脚本
- **文件名攻击**：使用特殊字符或超长文件名导致路径遍历
- **文件内容伪装**：修改文件扩展名和MIME类型欺骗服务器

### 2.2 多层验证策略

```typescript
import crypto from 'crypto';
import { Request } from 'express';

// =============================================
// 第一层：MIME类型验证（最基础）
// =============================================

// MIME类型是浏览器根据文件内容自动设置的
// 但这可以被轻易伪造，所以不能作为唯一依据
const allowedMimeTypes = [
  'image/jpeg',
  'image/png',
  'image/gif',
  'image/webp',
  'application/pdf',
  'application/msword',
  'application/vnd.openxmlformats-officedocument.wordprocessingml.document'
];

// =============================================
// 第二层：文件扩展名验证（配合MIME）
// =============================================

// 定义允许的扩展名映射到正确的MIME类型
const extensionMimeMap: Record<string, string> = {
  '.jpg': 'image/jpeg',
  '.jpeg': 'image/jpeg',
  '.png': 'image/png',
  '.gif': 'image/gif',
  '.webp': 'image/webp',
  '.pdf': 'application/pdf',
  '.doc': 'application/msword',
  '.docx': 'application/vnd.openxmlformats-officedocument.wordprocessingml.document'
};

// 验证扩展名是否与MIME类型匹配
function validateExtension(filename: string, mimetype: string): boolean {
  // 获取文件扩展名（转小写）
  const ext = path.extname(filename).toLowerCase();

  // 检查扩展名是否在允许列表中
  if (!extensionMimeMap[ext]) {
    return false;
  }

  // 检查扩展名对应的MIME类型是否与实际MIME类型匹配
  return extensionMimeMap[ext] === mimetype;
}

// =============================================
// 第三层：文件魔数验证（Magic Number）
// =============================================

// 文件魔数是文件开头的几个字节，标识文件的真实类型
// 即使修改文件扩展名或MIME类型，也无法伪造魔数
const magicNumbers: Record<string, string[]> = {
  // JPEG文件以FFD8FF开头
  'image/jpeg': ['FF', 'D8', 'FF'],
  // PNG文件以89504E47开头
  'image/png': ['89', '50', '4E', '47'],
  // GIF文件以47494638开头
  'image/gif': ['47', '49', '46', '38'],
  // PDF文件以25504446开头
  'application/pdf': ['25', '50', '44', '46']
};

// 验证文件魔数
async function validateMagicNumber(buffer: Buffer, mimetype: string): Promise<boolean> {
  const expectedMagic = magicNumbers[mimetype];

  if (!expectedMagic) {
    // 如果没有定义魔数验证规则，跳过
    return true;
  }

  // 读取文件开头的魔数字节
  const fileMagic = expectedMagic.map(hex =>
    buffer.slice(0, expectedMagic.length).toString('hex').substring(0, 2)
  );

  // 检查是否匹配
  return expectedMagic.every((hex, index) =>
    buffer[index].toString(16).toUpperCase().padStart(2, '0') === hex
  );
}

// =============================================
// 第四层：文件内容安全扫描
// =============================================

// 检查文件是否包含可执行的脚本内容
function scanForMaliciousContent(buffer: Buffer): { safe: boolean; threats: string[] } {
  const threats: string[] = [];

  // 将缓冲区转为字符串进行模式匹配
  const content = buffer.toString('utf8', 0, Math.min(buffer.length, 10000));

  // 检查常见的恶意模式
  const maliciousPatterns = [
    { pattern: /<script/i, name: 'JavaScript标签' },
    { pattern: /<iframe/i, name: 'iframe标签' },
    { pattern: /javascript:/i, name: 'JavaScript协议' },
    { pattern: /on\w+\s*=/i, name: '内联事件处理器' },
    { pattern: /<\?php/i, name: 'PHP标签' },
    { pattern: /<%@?\s*page/i, name: 'ASP标签' },
    { pattern: /\{\{.*?\}\}/i, name: '模板注入' },
    { pattern: /\$\{.*?\}/i, name: '模板字符串注入' }
  ];

  for (const { pattern, name } of maliciousPatterns) {
    if (pattern.test(content)) {
      threats.push(name);
    }
  }

  return {
    safe: threats.length === 0,
    threats
  };
}

// =============================================
// 第五层：文件名安全处理
// =============================================

// 清理文件名，防止路径遍历攻击
function sanitizeFilename(filename: string): string {
  // 移除路径分隔符，防止../攻击
  let sanitized = filename.replace(/[/\\?%*:|"<>]/g, '-');

  // 移除开头和结尾的点、空格
  sanitized = sanitized.replace(/^\.+|\.+$/g, '').trim();

  // 如果文件名为空，生成随机名称
  if (!sanitized) {
    sanitized = 'unnamed-' + Date.now();
  }

  // 限制文件名长度
  const maxLength = 100;
  if (sanitized.length > maxLength) {
    // 保留扩展名，截断主体部分
    const ext = path.extname(sanitized);
    const nameWithoutExt = path.basename(sanitized, ext);
    sanitized = nameWithoutExt.substring(0, maxLength - ext.length) + ext;
  }

  return sanitized;
}

// =============================================
// 综合文件验证中间件
// =============================================

// 创建综合验证的文件过滤器
function createSecureFileFilter() {
  return async function (
    req: Request,
    file: Express.Multer.File,
    callback: (error: Error | null, acceptFile: boolean) => void
  ) {
    try {
      // 第一步：验证MIME类型
      if (!allowedMimeTypes.includes(file.mimetype)) {
        callback(new Error(`MIME类型 ${file.mimetype} 不允许`), false);
        return;
      }

      // 第二步：验证扩展名与MIME类型匹配
      if (!validateExtension(file.originalname, file.mimetype)) {
        callback(new Error('文件扩展名与MIME类型不匹配'), false);
        return;
      }

      // 第三步：验证文件魔数（需要读取文件内容）
      // 注意：这需要使用内存存储来获取文件buffer
      if (file.buffer) {
        // 验证魔数
        const magicValid = await validateMagicNumber(file.buffer, file.mimetype);
        if (!magicValid) {
          callback(new Error('文件内容与声明类型不匹配'), false);
          return;
        }

        // 扫描恶意内容
        const scanResult = scanForMaliciousContent(file.buffer);
        if (!scanResult.safe) {
          callback(new Error(`文件包含潜在威胁: ${scanResult.threats.join(', ')}`), false);
          return;
        }
      }

      // 第四步：验证文件名
      const sanitizedName = sanitizeFilename(file.originalname);
      // 如果需要修改文件名，可以通过req传递
      (file as any).sanitizedFilename = sanitizedName;

      // 所有验证通过
      callback(null, true);

    } catch (error) {
      console.error('文件验证过程出错:', error);
      callback(new Error('文件验证失败'), false);
    }
  };
}

// 使用综合验证的Multer配置
const secureUpload = multer({
  storage: multer.memoryStorage(),  // 使用内存存储以读取文件内容
  limits: {
    fileSize: 10 * 1024 * 1024,      // 10MB
    files: 1
  },
  fileFilter: createSecureFileFilter()
});

// 安全上传路由
app.post('/api/upload/secure', secureUpload.single('file'), (
  req: Request,
  res: Response
) => {
  res.json({
    success: true,
    message: '安全上传成功',
    file: {
      originalname: req.file?.sanitizedFilename || req.file?.originalname,
      size: req.file?.size,
      mimetype: req.file?.mimetype
    }
  });
});
```

### 2.3 文件哈希校验

为确保文件完整性，计算并存储文件哈希值。

```typescript
import crypto from 'crypto';

// 计算文件的MD5、SHA1、SHA256哈希值
function calculateFileHashes(buffer: Buffer): {
  md5: string;
  sha1: string;
  sha256: string;
} {
  // MD5：快速，但不安全（用于快速比对）
  const md5 = crypto.createHash('md5').update(buffer).digest('hex');

  // SHA1：比MD5更安全，但已被发现碰撞（兼容性场景）
  const sha1 = crypto.createHash('sha1').update(buffer).digest('hex');

  // SHA256：目前推荐的安全哈希算法
  const sha256 = crypto.createHash('sha256').update(buffer).digest('hex');

  return { md5, sha1, sha256 };
}

// 文件哈希中间件
const hashUpload = multer({
```

## 三、存储方案：本地存储与云存储

### 3.1 本地存储方案

本地存储是最简单的方案，适合小型应用和开发环境。

```typescript
import express from 'express';
import multer from 'multer';
import path from 'path';
import fs from 'fs';
import crypto from 'crypto';

const app = express();

// =============================================
// 本地存储配置 - 按日期分目录
// =============================================

// 创建带日期目录的存储引擎
const dateBasedStorage = multer.diskStorage({
  // 动态设置保存路径，按年月分目录
  destination: function (req, file, callback) {
    // 获取当前日期
    const now = new Date();
    const year = now.getFullYear();
    const month = String(now.getMonth() + 1).padStart(2, '0');

    // 构建目录路径：uploads/2024/03
    const uploadPath = path.join(process.cwd(), 'uploads', String(year), month);

    // 递归创建目录
    fs.mkdirSync(uploadPath, { recursive: true });

    callback(null, uploadPath);
  },

  // 生成唯一的文件名
  filename: function (req, file, callback) {
    // 生成随机字符串作为文件名的前缀
    const randomString = crypto.randomBytes(8).toString('hex');

    // 获取文件扩展名
    const ext = path.extname(file.originalname).toLowerCase();

    // 清理原始文件名中的特殊字符
    const cleanName = file.originalname
      .replace(ext, '')  // 移除扩展名
      .replace(/[^a-zA-Z0-9\u4e00-\u9fa5]/g, '_')  // 将非中文 alphanumeric 替换为下划线
      .substring(0, 50);  // 限制长度

    // 组装最终文件名：原始名_random.ext
    const filename = `${cleanName}_${randomString}${ext}`;

    callback(null, filename);
  }
});

// 创建上传中间件
const localUpload = multer({
  storage: dateBasedStorage,
  limits: {
    fileSize: 10 * 1024 * 1024,  // 10MB
    files: 5
  }
});

// 本地文件上传路由
app.post('/api/upload/local', localUpload.single('file'), (
  req: Request,
  res: Response
) => {
  const file = req.file;

  if (!file) {
    return res.status(400).json({ success: false, message: '上传失败' });
  }

  // 构建访问URL
  // 注意：生产环境需要配置静态文件服务或Nginx
  const fileUrl = `/uploads/${path.basename(path.dirname(file.path))}/${file.filename}`;

  res.json({
    success: true,
    message: '上传成功',
    url: fileUrl,  // 返回可访问的URL
    file: {
      path: file.path,      // 服务器存储路径
      url: fileUrl,         // 访问URL
      size: file.size,
      originalname: file.originalname
    }
  });
});

// 配置静态文件服务（用于开发环境）
// 访问 http://localhost:3000/uploads/xxx.jpg 会直接返回文件
app.use('/uploads', express.static(path.join(process.cwd(), 'uploads')));

// =============================================
// 本地文件删除接口
// =============================================

app.delete('/api/file/:filename(*)', async (req, res) => {
  const { filename } = req.params;

  // 防止路径遍历攻击：确保文件名不包含目录
  if (filename.includes('..') || filename.includes('/') || filename.includes('\\')) {
    return res.status(400).json({ success: false, message: '无效的文件名' });
  }

  // 查找文件（搜索所有日期目录）
  const searchDir = path.join(process.cwd(), 'uploads');
  const filePath = await findFileRecursive(searchDir, filename);

  if (!filePath) {
    return res.status(404).json({ success: false, message: '文件不存在' });
  }

  // 删除文件
  fs.unlinkSync(filePath);

  res.json({ success: true, message: '文件已删除' });
});

// 递归搜索文件
async function findFileRecursive(dir: string, filename: string): Promise<string | null> {
  if (!fs.existsSync(dir)) return null;

  const entries = fs.readdirSync(dir, { withFileTypes: true });

  for (const entry of entries) {
    const fullPath = path.join(dir, entry.name);

    if (entry.isDirectory()) {
      // 递归搜索子目录
      const found = await findFileRecursive(fullPath, filename);
      if (found) return found;
    } else if (entry.name === filename) {
      return fullPath;
    }
  }

  return null;
}
```

### 3.2 阿里云OSS存储

阿里云OSS（Object Storage Service）是企业级云存储解决方案，提供高可用、高可靠、低成本的存储服务。

```typescript
import OSS from 'ali-oss';
import path from 'path';

// =============================================
// 阿里云OSS客户端配置
// =============================================

// 创建OSS客户端实例
const ossClient = new OSS({
  // 区域地址，可在控制台获取
  region: 'oss-cn-hangzhou',
  // 访问凭证（AccessKey）
  accessKeyId: process.env.OSS_ACCESS_KEY_ID || '',
  accessKeySecret: process.env.OSS_ACCESS_KEY_SECRET || '',
  // 存储空间名称
  bucket: 'my-app-storage',
  // 可选：自定义域名（启用CDN加速时使用）
  cname: 'https://cdn.example.com',
  // 可选：超时时间（毫秒）
  timeout: 60000
});

// =============================================
// OSS文件上传服务
// =============================================

class OSSUploadService {
  // 上传文件到OSS
  async uploadFile(
    localFilePath: string,      // 本地文件路径
    targetName: string,         // OSS上的目标文件名
    options?: {
      headers?: Record<string, string>;  // 自定义HTTP头
      mime?: string;                     // MIME类型
    }
  ): Promise<{ url: string; name: string; size: number }> {
    try {
      // 上传文件
      // put方法会将本地文件上传到OSS
      const result = await ossClient.put(targetName, localFilePath, {
        // 设置文件的MIME类型
        mime: options?.mime || 'application/octet-stream',
        // 自定义元数据
        meta: {
          uploadedAt: Date.now().toString()
        }
      });

      // 返回文件信息
      return {
        url: result.url,        // 文件的访问URL
        name: targetName,       // OSS上的文件名
        size: result.res?.headers['content-length'] as number || 0
      };

    } catch (error) {
      console.error('OSS上传失败:', error);
      throw new Error(`文件上传失败: ${error}`);
    }
  }

  // 上传Buffer数据到OSS
  async uploadBuffer(
    buffer: Buffer,
    targetName: string,
    mimeType: string = 'application/octet-stream'
  ): Promise<{ url: string; name: string; size: number }> {
    try {
      // putStream方法上传流/Buffer数据
      const result = await ossClient.put(
        targetName,
        buffer,
        { mime: mimeType }
      );

      return {
        url: result.url,
        name: targetName,
        size: buffer.length
      };

    } catch (error) {
      console.error('OSS Buffer上传失败:', error);
      throw new Error(`Buffer上传失败: ${error}`);
    }
  }

  // 生成带签名的下载URL（私有文件访问）
  async getSignedUrl(targetName: string, expires: number = 3600): Promise<string> {
    try {
      // 签名URL用于临时访问私有文件
      // expires：URL有效期（秒）
      const url = await ossClient.signatureUrl(targetName, {
        expires
      });

      return url;

    } catch (error) {
      console.error('生成签名URL失败:', error);
      throw new Error(`签名URL生成失败: ${error}`);
    }
  }

  // 删除OSS上的文件
  async deleteFile(targetName: string): Promise<void> {
    try {
      await ossClient.delete(targetName);
    } catch (error) {
      console.error('OSS文件删除失败:', error);
      throw new Error(`文件删除失败: ${error}`);
    }
  }

  // 批量删除文件
  async deleteFiles(targetNames: string[]): Promise<void> {
    try {
      // deleteMulti支持批量删除
      const result = await ossClient.deleteMulti(targetNames);
      console.log('批量删除结果:', result);
    } catch (error) {
      console.error('OSS批量删除失败:', error);
      throw new Error(`批量删除失败: ${error}`);
    }
  }

  // 检查文件是否存在
  async fileExists(targetName: string): Promise<boolean> {
    try {
      await ossClient.head(targetName);
      return true;
    } catch (error: any) {
      if (error.code === 'NoSuchKey') {
        return false;
      }
      throw error;
    }
  }
}

// 创建OSS服务实例
const ossService = new OSSUploadService();

// =============================================
// OSS与Multer集成上传路由
// =============================================

  storage: multer.memoryStorage(),  // 先存到内存，然后上传到OSS
  limits: { fileSize: 20 * 1024 * 1024 }  // 20MB

    res.json({

        mimetype: file.mimetype
      }
    });

  } catch (error) {

});

// 获取私有文件的签名下载URL
app.get('/api/file/signed-url/:filename(*)', async (req, res) => {
```

### 3.3 AWS S3存储

AWS S3（Simple Storage Service）是亚马逊云的对象存储服务，全球使用最广泛的云存储之一。

```typescript
import {
  S3Client,
  PutObjectCommand,
  GetObjectCommand,
  DeleteObjectCommand,
  DeleteObjectsCommand,
  HeadObjectCommand
} from '@aws-sdk/client-s3';
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';
import crypto from 'crypto';

// =============================================
// AWS S3客户端配置
// =============================================

// 创建S3客户端实例
const s3Client = new S3Client({
  region: process.env.AWS_REGION || 'ap-east-1',
  credentials: {
    accessKeyId: process.env.AWS_ACCESS_KEY_ID || '',
    secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY || ''
  },
  // 可选：使用自定义端点（兼容MinIO、本地S3等）
  // endpoint: 'http://localhost:9000',
  // forcePathStyle: true  // MinIO需要开启
});

// S3存储桶配置
const S3_BUCKET = process.env.AWS_S3_BUCKET || 'my-app-storage';
const S3_REGION = process.env.AWS_REGION || 'ap-east-1';

// =============================================
// S3文件上传服务
// =============================================

class S3UploadService {
  // 上传本地文件到S3
  async uploadFile(
    localFilePath: string,
    key: string,
    options?: {
      contentType?: string;
      metadata?: Record<string, string>;
      acl?: 'private' | 'public-read' | 'authenticated-read';
    }
  ): Promise<{ key: string; url: string; size: number }> {
    const fs = await import('fs');
    const fileBuffer = fs.readFileSync(localFilePath);

    return this.uploadBuffer(fileBuffer, key, options);
  }

  // 上传Buffer到S3
  async uploadBuffer(
    buffer: Buffer,
    key: string,
    options?: {
      contentType?: string;
      metadata?: Record<string, string>;
      acl?: 'private' | 'public-read' | 'authenticated-read';
    }
  ): Promise<{ key: string; url: string; size: number }> {
    const command = new PutObjectCommand({
      Bucket: S3_BUCKET,
      Key: key,
      Body: buffer,
      ContentType: options?.contentType || 'application/octet-stream',
      // 自定义元数据
      Metadata: options?.metadata || {},
      // 访问控制列表
      ACL: options?.acl || 'private',
      // 可选：设置存储类
      // StorageClass: 'STANDARD' | 'REDUCED_REDUNDANCY' | 'GLACIER'
    });

    await s3Client.send(command);

    // 构建文件URL
    const url = `https://${S3_BUCKET}.s3.${S3_REGION}.amazonaws.com/${key}`;

    return {
      key,
      url,
      size: buffer.length
    };
  }

  // 生成预签名上传URL（允许客户端直接上传到S3）
  async getPresignedUploadUrl(
    key: string,
    contentType: string,
    expiresIn: number = 3600
  ): Promise<{ url: string; fields: Record<string, string> }> {
    const command = new PutObjectCommand({
      Bucket: S3_BUCKET,
      Key: key,
      ContentType: contentType
    });

    // 生成预签名URL
    const url = await getSignedUrl(s3Client, command, { expiresIn });

    return {
      url,  // 预签名URL
      fields: {
        'Content-Type': contentType  // 客户端需要包含此字段
      }
    };
  }

  // 生成预签名下载URL
  async getPresignedDownloadUrl(key: string, expiresIn: number = 3600): Promise<string> {
    const command = new GetObjectCommand({
      Bucket: S3_BUCKET,
      Key: key
    });

    const url = await getSignedUrl(s3Client, command, { expiresIn });

    return url;
  }

  // 删除单个文件
  async deleteFile(key: string): Promise<void> {
    const command = new DeleteObjectCommand({
      Bucket: S3_BUCKET,
      Key: key
    });

    await s3Client.send(command);
  }

  // 批量删除文件
  async deleteFiles(keys: string[]): Promise<void> {
    const command = new DeleteObjectsCommand({
      Bucket: S3_BUCKET,
      Delete: {
        Objects: keys.map(key => ({ Key: key }))
      }
    });

    await s3Client.send(command);
  }

  // 检查文件是否存在
  async fileExists(key: string): Promise<boolean> {
    try {
      const command = new HeadObjectCommand({
        Bucket: S3_BUCKET,
        Key: key
      });

      await s3Client.send(command);
      return true;
    } catch (error: any) {
      if (error.name === 'NotFound') {
        return false;
      }
      throw error;
    }
  }

  // 获取文件元信息
  async getFileMetadata(key: string): Promise<{
    size: number;
    lastModified: Date;
    contentType: string;
    metadata: Record<string, string>;
  }> {
    const command = new HeadObjectCommand({
      Bucket: S3_BUCKET,
      Key: key
    });

    const response = await s3Client.send(command);

    return {
      size: response.ContentLength || 0,
      lastModified: response.LastModified || new Date(),
      contentType: response.ContentType || 'application/octet-stream',
      metadata: response.Metadata || {}
    };
  }
}

// 创建S3服务实例
const s3Service = new S3UploadService();

// =============================================
// S3与Multer集成上传路由
// =============================================

// 直接上传到S3（服务器中转）

      key,

        contentType: file.mimetype,

      }
    );

    res.json({

      message: '上传成功',
      url: result.url,
      file: {
        key: result.key,
        size: result.size,
        mimetype: file.mimetype
      }
    });

  } catch (error) {
    console.error('S3上传失败:', error);
    res.status(500).json({ success: false, message: '上传失败' });
```

### 4.2 基础图像处理

```typescript
import sharp from 'sharp';
import path from 'path';
import fs from 'fs';

// =============================================
// 图像处理服务类
// =============================================

class ImageProcessingService {
  // 输入目录
  private inputDir: string;
  // 输出目录
  private outputDir: string;

  constructor(inputDir: string = './uploads', outputDir: string = './processed') {
    this.inputDir = inputDir;
    this.outputDir = outputDir;

    // 确保输出目录存在
    if (!fs.existsSync(this.outputDir)) {
      fs.mkdirSync(this.outputDir, { recursive: true });
    }
  }

  // =============================================
  // 获取图像元信息
  // =============================================

  // 获取图片的宽度、高度、格式等信息
  async getMetadata(imagePath: string): Promise<{
    width: number;
    height: number;
    format: string;
    size: number;
    space: string;
    channels: number;
    depth: string;
    hasAlpha: boolean;
  }> {
    const metadata = await sharp(imagePath).metadata();

    return {
      width: metadata.width || 0,
      height: metadata.height || 0,
      format: metadata.format || 'unknown',
      size: fs.statSync(imagePath).size,
      space: metadata.space || '',
      channels: metadata.channels || 0,
      depth: metadata.depth || '',
      hasAlpha: metadata.hasAlpha || false
    };
  }

  // =============================================
  // 格式转换
  // =============================================

  // 将图片转换为指定格式
  async convertFormat(
    inputPath: string,
    outputFormat: 'jpeg' | 'png' | 'webp' | 'avif' | 'tiff' | 'gif',
    options?: {
      quality?: number;       // 输出质量（1-100）
      compressionLevel?: number;  // 压缩级别
    }
  ): Promise<string> {
    // 生成输出文件名
    const inputName = path.basename(inputPath, path.extname(inputPath));
    const outputName = `${inputName}_${Date.now()}.${outputFormat}`;
    const outputPath = path.join(this.outputDir, outputName);

    // 创建转换管道
    let pipeline = sharp(inputPath);

    // 根据格式设置不同的输出选项
    switch (outputFormat) {
      case 'jpeg':
        // JPEG支持质量压缩
        pipeline = pipeline.jpeg({
          quality: options?.quality || 80,
          mozjpeg: true  // 使用MozJPEG编码器，更好的压缩比
        });
        break;

      case 'png':
        // PNG支持压缩级别
        pipeline = pipeline.png({
          compressionLevel: options?.compressionLevel || 6,
          adaptiveFiltering: true
        });
        break;

      case 'webp':
        // WebP支持质量和无损模式
        pipeline = pipeline.webp({
          quality: options?.quality || 80,
          lossless: false
        });
        break;

      case 'avif':
        // AVIF是最新的高效编码格式
        pipeline = pipeline.avif({
          quality: options?.quality || 50  // AVIF压缩率高，低质量也清晰
        });
        break;

      case 'gif':
        pipeline = pipeline.gif();
        break;

      case 'tiff':
        pipeline = pipeline.tiff({
          compression: 'lzw'  // 使用LZW压缩
        });
        break;
    }

    // 执行转换并保存
    await pipeline.toFile(outputPath);

    console.log(`格式转换完成: ${inputPath} -> ${outputPath}`);

    return outputPath;
  }

  // =============================================
  // 缩略图生成

  // 生成多种尺寸的缩略图
  async generateThumbnails(
    inputPath: string,
    sizes: { name: string; width: number; height?: number }[]
  ): Promise<{ name: string; path: string }[]> {
    const inputName = path.basename(inputPath, path.extname(inputPath));
    const results: { name: string; path: string }[] = [];

    for (const size of sizes) {
      // 生成缩略图文件名
      const outputName = `${inputName}_${size.name}_${size.width}x${size.height || 'auto'}${path.extname(inputPath)}`;
      const outputPath = path.join(this.outputDir, outputName);

      // 创建缩略图
      // resize的fit选项控制如何处理原始宽高比
      // 'cover'：裁剪到指定尺寸，可能截断部分画面
      // 'contain'：保留完整画面，可能有空白
      // 'fill'：拉伸到指定尺寸（可能变形）
      let pipeline = sharp(inputPath)
        .resize(size.width, size.height, {
          fit: 'cover',  // 裁剪模式
          position: 'center',  // 裁剪位置：中心
          // 可选：background用于contain模式的背景色
          // background: { r: 255, g: 255, b: 255, alpha: 1 }
        });

      // JPEG质量设置
      if (['jpg', 'jpeg'].some(ext => inputPath.endsWith(ext))) {
        pipeline = pipeline.jpeg({ quality: 80 });
      } else if (inputPath.endsWith('.png')) {
        pipeline = pipeline.png({ compressionLevel: 6 });
      } else if (inputPath.endsWith('.webp')) {
        pipeline = pipeline.webp({ quality: 80 });
      }

      await pipeline.toFile(outputPath);

      results.push({ name: size.name, path: outputPath });

      console.log(`缩略图生成完成: ${outputName}`);
    }

    return results;
  }

    inputPath: string,

    height: number

    const outputName = `${inputName}_crop_${width}x${height}${path.extname(inputPath)}`;

    const left = Math.max(0, Math.floor((metadata.width - width) / 2));
    const top = Math.max(0, Math.floor((metadata.height - height) / 2));

    await sharp(inputPath)
      .extract({
        left,
        top,
        width: Math.min(width, metadata.width - left),
        height: Math.min(height, metadata.height - top)
      })
      .toFile(outputPath);

    return outputPath;

  // 智能裁剪（基于人脸检测区域）
  // 注意：需要配合face-detection库使用
  async smartCrop(
    inputPath: string,
    width: number,
    height: number,
    faceRegions?: { x: number; y: number; width: number; height: number }[]
  ): Promise<string> {
    const inputName = path.basename(inputPath, path.extname(inputPath));
    const outputName = `${inputName}_smart_crop${path.extname(inputPath)}`;
    const outputPath = path.join(this.outputDir, outputName);

    if (faceRegions && faceRegions.length > 0) {
      // 优先裁剪包含人脸的区域
      // 计算所有人脸区域的边界框
      const bounds = faceRegions.reduce(
        (acc, region) => ({
          minX: Math.min(acc.minX, region.x),
          minY: Math.min(acc.minY, region.y),
          maxX: Math.max(acc.maxX, region.x + region.width),
          maxY: Math.max(acc.maxY, region.y + region.height)
        }),
        { minX: Infinity, minY: Infinity, maxX: 0, maxY: 0 }
      );

      // 在人脸区域基础上扩展边距
      const padding = 50;
      const cropArea = {
        left: Math.max(0, bounds.minX - padding),
        top: Math.max(0, bounds.minY - padding),
        width: Math.min(width, bounds.maxX - bounds.minX + padding * 2),
        height: Math.min(height, bounds.maxY - bounds.minY + padding * 2)
      };

      await sharp(inputPath)
        .extract(cropArea)
        .resize(width, height)
        .toFile(outputPath);
    } else {
      // 无人脸信息，使用焦点检测
      await sharp(inputPath)
        .resize(width * 2, height * 2, { fit: 'cover' })
        .linearStain([{ default: { r: 128, g: 128, b: 128, alpha: 0 } }])  // 占位
        .resize(width, height, { fit: 'cover' })
        .toFile(outputPath);
    }

    return outputPath;
  }

  // =============================================
  // 图片合成（加水印）

  // 添加文字水印
  async addTextWatermark(
    inputPath: string,
    text: string,
    options?: {
      fontSize?: number;
      color?: string;
      opacity?: number;
      position?: 'center' | 'top-left' | 'top-right' | 'bottom-left' | 'bottom-right';
    }
  ): Promise<string> {
    const inputName = path.basename(inputPath, path.extname(inputPath));
    const outputName = `${inputName}_watermarked${path.extname(inputPath)}`;
    const outputPath = path.join(this.outputDir, outputName);

    // 获取图片元信息
    const metadata = await this.getMetadata(inputPath);

    // 创建SVG文字水印
    const fontSize = options?.fontSize || 24;
    const color = options?.color || 'white';
    const opacity = options?.opacity ?? 0.5;

    // 计算水印位置
    const positions = {
      'top-left': { x: 20, y: 20 + fontSize },
      'top-right': { x: metadata.width - 20 - text.length * fontSize * 0.6, y: 20 + fontSize },
      'bottom-left': { x: 20, y: metadata.height - 20 },
      'bottom-right': { x: metadata.width - 20 - text.length * fontSize * 0.6, y: metadata.height - 20 },
      'center': { x: (metadata.width - text.length * fontSize * 0.6) / 2, y: metadata.height / 2 }
    };

    const pos = positions[options?.position || 'bottom-right'];

    // 构建SVG
    const svg = `
      <svg width="${metadata.width}" height="${metadata.height}">
        <style>
          .watermark {
            font-size: ${fontSize}px;
            font-weight: bold;
            font-family: Arial, sans-serif;
            fill: ${color};
            opacity: ${opacity};
          }
        </style>
        <text x="${pos.x}" y="${pos.y}" class="watermark">${text}</text>
      </svg>
    `;

    // 合成水印
    await sharp(inputPath)
      .composite([{
        input: Buffer.from(svg),
        top: 0,
        left: 0
      }])
      .toFile(outputPath);

    return outputPath;
  }

  async addImageWatermark(
    inputPath: string,
    watermarkPath: string,
    options?: {
      opacity?: number;
      position?: 'center' | 'top-left' | 'top-right' | 'bottom-left' | 'bottom-right';
      scale?: number;  // 水印相对于原图的比例

  ): Promise<string> {
    const inputName = path.basename(inputPath, path.extname(inputPath));
    const outputName = `${inputName}_image_watermarked${path.extname(inputPath)}`;

    const wmWidth = Math.floor(metadata.width * scale);

    const resizedWatermark = await sharp(watermarkPath)

      .modulate({
        brightness: 1,
        saturation: 1
      })
      .toBuffer();

    // 计算水印位置
    const padding = 20;
    const positions: Record<string, { top: number; left: number }> = {
      'top-left': { top: padding, left: padding },
      'top-right': { top: padding, left: metadata.width - wmWidth - padding },
      'bottom-left': { top: metadata.height - wmHeight - padding, left: padding },
      'bottom-right': { top: metadata.height - wmHeight - padding, left: metadata.width - wmWidth - padding },
      'center': { top: (metadata.height - wmHeight) / 2, left: (metadata.width - wmWidth) / 2 }
    };

    const pos = positions[options?.position || 'bottom-right'];

      .composite([{

        top: pos.top,
        left: pos.left,

      }])

  // 组合多种处理操作
  async processImage(
    inputPath: string,
    operations: {
      resize?: { width: number; height: number; fit?: 'cover' | 'contain' | 'fill' };
      rotate?: number;
      sharpen?: boolean;
      grayscale?: boolean;
      negate?: boolean;
      normalize?: boolean;  // 自动对比度
      format?: 'jpeg' | 'png' | 'webp';
      quality?: number;
    }
  ): Promise<Buffer> {
    let pipeline = sharp(inputPath);

    // 按顺序执行操作
    if (operations.resize) {
      pipeline = pipeline.resize(
        operations.resize.width,
        operations.resize.height,
        { fit: operations.resize.fit || 'cover' }
      );
    }

    if (operations.rotate) {
      pipeline = pipeline.rotate(operations.rotate);
    }

    if (operations.sharpen) {
      // 锐化，增强边缘
      pipeline = pipeline.sharpen({
        sigma: 1,
        m1: 0,
        m2: 3,
        x1: 3,
        y2: 5,
        y3: 2
      });
    }

    if (operations.grayscale) {
      pipeline = pipeline.grayscale();
    }

    if (operations.negate) {
      pipeline = pipeline.negate();
    }

    if (operations.normalize) {
      // 归一化，增强对比度
      pipeline = pipeline.normalize();
    }

    // 格式转换
    if (operations.format) {
      switch (operations.format) {
        case 'jpeg':
          pipeline = pipeline.jpeg({ quality: operations.quality || 80 });
          break;
        case 'png':
          pipeline = pipeline.png({ compressionLevel: 6 });
          break;
        case 'webp':
          pipeline = pipeline.webp({ quality: operations.quality || 80 });
          break;
      }
    }

    return pipeline.toBuffer();
  }
```

### 5.2 视频处理实战

```typescript
import ffmpeg from 'fluent-ffmpeg';
import path from 'path';
import fs from 'fs';
import crypto from 'crypto';

// 设置ffmpeg路径（Windows需要）
import ffmpegPath from '@ffmpeg-installer/ffmpeg';
ffmpeg.setFfmpegPath(ffmpegPath.path);

// =============================================
// 视频处理服务类
// =============================================

class VideoProcessingService {
  private outputDir: string;
  private tempDir: string;

  constructor(outputDir: string = './processed/videos', tempDir: string = './temp') {
    this.outputDir = outputDir;
    this.tempDir = tempDir;

    // 确保目录存在
    [this.outputDir, this.tempDir].forEach(dir => {
      if (!fs.existsSync(dir)) {
        fs.mkdirSync(dir, { recursive: true });
      }
    });
  }

  // =============================================
  // 获取视频信息
  // =============================================

  // 获取视频的时长、分辨率、编码格式等信息
  async getVideoInfo(videoPath: string): Promise<{
    duration: number;       // 时长（秒）
    width: number;          // 宽度
    height: number;         // 高度
    codec: string;          // 视频编码
    audioCodec: string;    // 音频编码
    bitrate: number;       // 码率
    fps: number;           // 帧率
    size: number;          // 文件大小（字节）
  }> {
    return new Promise((resolve, reject) => {
      ffmpeg.ffprobe(videoPath, (err, metadata) => {
        if (err) {
          return reject(err);
        }

        const videoStream = metadata.streams.find(s => s.codec_type === 'video');
        const audioStream = metadata.streams.find(s => s.codec_type === 'audio');

        resolve({
          duration: metadata.format.duration || 0,
          width: videoStream?.width || 0,
          height: videoStream?.height || 0,
          codec: videoStream?.codec_name || 'unknown',
          audioCodec: audioStream?.codec_name || 'none',
          bitrate: parseInt(metadata.format.bit_rate || '0'),
          fps: this.parseFps(videoStream?.r_frame_rate || '0/1'),
          size: fs.statSync(videoPath).size
        });
      });
    });
  }

  // 解析帧率字符串（如 "30/1" -> 30）
  private parseFps(fpsStr: string): number {
    const parts = fpsStr.split('/');
    if (parts.length === 2) {
      return Math.round(parseInt(parts[0]) / parseInt(parts[1]));
    }
    return parseFloat(fpsStr) || 0;
  }

  // =============================================
  // 视频格式转换
  // =============================================

  // 转换视频格式
  async convertFormat(
    inputPath: string,
    outputFormat: 'mp4' | 'webm' | 'avi' | 'mkv' | 'mov',
    options?: {
      videoCodec?: string;    // 视频编码
      audioCodec?: string;     // 音频编码
      videoBitrate?: string;  // 视频码率（如 '1M'）
      audioBitrate?: string;   // 音频码率（如 '128k'）
      resolution?: string;     // 分辨率（如 '1920x1080'）
      fps?: number;           // 帧率
    }
  ): Promise<string> {
    const inputName = path.basename(inputPath, path.extname(inputPath));
    const outputName = `${inputName}_${Date.now()}.${outputFormat}`;
    const outputPath = path.join(this.outputDir, outputName);

    return new Promise((resolve, reject) => {
      let command = ffmpeg(inputPath);

      // 设置视频编码
      if (options?.videoCodec) {
        command = command.videoCodec(options.videoCodec);
      }

      // 设置音频编码
      if (options?.audioCodec) {
        command = command.audioCodec(options.audioCodec);
      }

      // 设置视频码率
      if (options?.videoBitrate) {
        command = command.videoBitrate(options.videoBitrate);
      }

      // 设置音频码率
      if (options?.audioBitrate) {
        command = command.audioBitrate(options.audioBitrate);
      }

      // 设置分辨率
      if (options?.resolution) {
        const [width, height] = options.resolution.split('x').map(Number);
        command = command.size(`${width}x${height}`).autopad();
      }

      // 设置帧率
      if (options?.fps) {
        command = command.fps(options.fps);
      }

      // MP4格式需要特别处理
      if (outputFormat === 'mp4') {
        command = command.format('mp4')
          .outputOptions([
            '-movflags', 'faststart'  // 优化Web播放
          ]);
      }

      command
        .on('start', (cmd) => {
          console.log('ffmpeg开始执行:', cmd);
        })
        .on('progress', (progress) => {
          // 进度回调
          if (progress.percent) {

      width?: number;        // 缩略图宽度

      format?: 'jpg' | 'png';

    const inputName = path.basename(videoPath, path.extname(videoPath));
    const format = options?.format || 'jpg';
    const outputName = `${inputName}_thumb.${format}`;
    const outputPath = path.join(this.outputDir, outputName);

    // 获取视频时长，确定截图时间点
    const info = await this.getVideoInfo(videoPath);

    return new Promise((resolve, reject) => {
      let command = ffmpeg(videoPath);

      if (options?.count && options.count > 1) {
        // 生成多张缩略图
        const interval = info.duration / (options.count + 1);
        const timestamps = Array.from(
          { length: options.count },
          (_, i) => interval * (i + 1)
        );

        // 为每个时间点生成缩略图
        const outputs = timestamps.map((t, i) => {
          const name = `${inputName}_thumb_${i + 1}.${format}`;
          return path.join(this.outputDir, name);
        });

        // 使用FPS筛选器每隔一定时间截取一帧
        command = command
          .fps(options.count)
          .size(options?.width ? `${options.width}x${options.height || '?'}` : '320x?')
          .output(outputs[0].replace(/\.\w+$/, '_%d.$1'));

        // 简化处理：只生成一张
        const singleOutput = outputs[0];

        ffmpeg(videoPath)
          .seekInput(timestamps[0])
          .frames(1)
          .size(options?.width ? `${options.width}x${options.height || '?'}` : '320x?')
          .output(singleOutput)
          .on('error', reject)
          .on('end', () => resolve(singleOutput))
          .run();
      } else {
        // 生成单张缩略图
        const timestamp = options?.timestamp || info.duration * 0.1;

        command = command
          .seekInput(timestamp)
          .frames(1)
          .size(options?.width ? `${options.width}x${options.height || '?'}` : '320x?');

        command
          .on('error', reject)
          .on('end', () => {
            console.log('缩略图生成完成:', outputPath);
            resolve(outputPath);
          })
          .save(outputPath);
      }
    });
```

```typescript
async function validateMagicNumber(buffer: Buffer, mimetype: string): Promise<boolean> {
  const magic = MAGIC_NUMBERS[mimetype];

      }

      const magicValid = await validateMagicNumber(file.buffer, file.mimetype);
      if (!magicValid) {
        return callback(new Error('文件内容与声明类型不匹配'));
      }

      // 第四层：恶意内容扫描
      const scanResult = scanMaliciousContent(file.buffer);
      if (!scanResult.safe) {
        return callback(new Error(`文件包含危险内容: ${scanResult.threats.join(', ')}`));
      }
    }

    // 验证通过
    callback(null, true);

  } catch (error) {
    console.error('文件验证失败:', error);
    callback(new Error('文件验证过程出错'));
  }
};

// 本地存储服务
class LocalStorageService implements StorageService {
  private baseDir: string;

  constructor(baseDir: string) {
    this.baseDir = baseDir;
  }

  // 上传文件到本地磁盘
  async upload(buffer: Buffer, key: string, mimetype: string): Promise<{ url: string; key: string; size: number }> {
    // 构建完整路径
    const fullPath = path.join(this.baseDir, key);

    // 确保目录存在
    const dir = path.dirname(fullPath);
    if (!fs.existsSync(dir)) {
      fs.mkdirSync(dir, { recursive: true });
    }

    // 写入文件
    await fs.promises.writeFile(fullPath, buffer);

    // 生成访问URL（假设配置了静态文件服务）
    const url = `/static/${key}`;

    return {
      url,
      key,
      size: buffer.length
    };
  }

  // 获取文件URL（本地文件直接返回路径）
  async getUrl(key: string, expires?: number): Promise<string> {
    return `/static/${key}`;
  }

  // 删除文件
  async delete(key: string): Promise<void> {
    const fullPath = path.join(this.baseDir, key);
    if (fs.existsSync(fullPath)) {
      await fs.promises.unlink(fullPath);
    }
  }
}

// S3存储服务
class S3StorageService implements StorageService {
  private client: S3Client;
  private bucket: string;
  private region: string;

  constructor(config: { region: string; accessKeyId: string; secretAccessKey: string; bucket: string }) {
    this.client = new S3Client({
      region: config.region,
      credentials: {
        accessKeyId: config.accessKeyId,
        secretAccessKey: config.secretAccessKey
      }
    });
    this.bucket = config.bucket;
    this.region = config.region;
  }

  // 上传到S3
  async upload(buffer: Buffer, key: string, mimetype: string): Promise<{ url: string; key: string; size: number }> {
    const command = new PutObjectCommand({
      Bucket: this.bucket,
      Key: key,
      Body: buffer,
      ContentType: mimetype,
      ACL: 'public-read' as any
    });

    await this.client.send(command);

    const url = `https://${this.bucket}.s3.${this.region}.amazonaws.com/${key}`;

    return { url, key, size: buffer.length };
  }

  // 获取签名URL
  async getUrl(key: string, expires: number = 3600): Promise<string> {
    const command = new GetObjectCommand({
      Bucket: this.bucket,
      Key: key
    });

    return await getSignedUrl(this.client, command, { expiresIn: expires });
  }

  // 删除S3对象
  async delete(key: string): Promise<void> {
    const command = new DeleteObjectCommand({
      Bucket: this.bucket,
      Key: key
    });

    await this.client.send(command);
  }
}

class ImageService {
  private outputDir: string;

  constructor(outputDir: string) {
    this.outputDir = outputDir;
  }

  // 处理图片：生成缩略图、水印、格式转换
  async processImage(
    buffer: Buffer,
    options: {
      // 缩略图尺寸配置
      thumbnails?: { name: string; width: number; height: number }[];
      // 水印文字
      watermarkText?: string;
      // 输出格式
      format?: 'jpeg' | 'png' | 'webp';
      // 质量
      quality?: number;
      // 调整尺寸
      resize?: { width: number; height: number };
    }
  ): Promise<{ original: Buffer; thumbnails?: { name: string; buffer: Buffer }[] }> {
    let processedBuffer = buffer;

    // 1. 调整尺寸
    if (options.resize) {
      processedBuffer = await sharp(processedBuffer)
        .resize(options.resize.width, options.resize.height, { fit: 'inside' })
        .toBuffer();
    }

    // 2. 格式转换和质量调整
    if (options.format) {
      switch (options.format) {
        case 'jpeg':
          processedBuffer = await sharp(processedBuffer)
            .jpeg({ quality: options.quality || 80 })
            .toBuffer();
          break;
        case 'png':
          processedBuffer = await sharp(processedBuffer)
            .png({ compressionLevel: 6 })
            .toBuffer();
          break;
        case 'webp':
          processedBuffer = await sharp(processedBuffer)
            .webp({ quality: options.quality || 80 })
            .toBuffer();
          break;
      }
    }

    // 3. 生成缩略图
    let thumbnails: { name: string; buffer: Buffer }[] | undefined;

    if (options.thumbnails && options.thumbnails.length > 0) {
      thumbnails = [];

      for (const thumb of options.thumbnails) {
        const thumbBuffer = await sharp(buffer)
          .resize(thumb.width, thumb.height, { fit: 'cover', position: 'center' })
          .jpeg({ quality: 80 })
          .toBuffer();

        thumbnails.push({ name: thumb.name, buffer: thumbBuffer });
      }
    }

    return {
      original: processedBuffer,
      thumbnails
    };
  }

  // 添加文字水印
  async addWatermark(buffer: Buffer, text: string, options?: {
    fontSize?: number;
    color?: string;
    opacity?: number;
    position?: 'center' | 'top-left' | 'top-right' | 'bottom-left' | 'bottom-right';
  }): Promise<Buffer> {
    const metadata = await sharp(buffer).metadata();
    const width = metadata.width || 800;
    const height = metadata.height || 600;

    const fontSize = options?.fontSize || 24;
    const color = options?.color || 'white';
    const opacity = options?.opacity ?? 0.5;
    const position = options?.position || 'bottom-right';

    // 水印位置计算
    const positions: Record<string, { x: number; y: number }> = {
      'top-left': { x: 20, y: 40 },
      'top-right': { x: width - text.length * fontSize * 0.6 - 20, y: 40 },
      'bottom-left': { x: 20, y: height - 20 },
      'bottom-right': { x: width - text.length * fontSize * 0.6 - 20, y: height - 20 },
      'center': { x: (width - text.length * fontSize * 0.6) / 2, y: height / 2 }
    };

    const pos = positions[position];

    // 创建SVG水印
    const svg = `
      <svg width="${width}" height="${height}">
        <style>
          .watermark {
            font-size: ${fontSize}px;
            font-weight: bold;
            font-family: Arial, sans-serif;
            fill: ${color};
            opacity: ${opacity};
          }
        </style>
        <text x="${pos.x}" y="${pos.y}" class="watermark">${text}</text>
      </svg>
    `;

    // 合成水印
    return await sharp(buffer)
      .composite([{
        input: Buffer.from(svg),
        top: 0,
        left: 0
      }])
      .toBuffer();
  }

  // 获取图片元信息
  async getMetadata(buffer: Buffer): Promise<{
    width: number;
    height: number;
    format: string;
    size: number;
    hasAlpha: boolean;
  }> {
    const metadata = await sharp(buffer).metadata();

    return {
      width: metadata.width || 0,
      height: metadata.height || 0,
      format: metadata.format || 'unknown',
      size: buffer.length,
      hasAlpha: metadata.hasAlpha || false
    };
  }
}
```

### 7.2 SMTP协议基础

SMTP（Simple Mail Transfer Protocol）是发送邮件的基础协议，工作在TCP的25端口（加密时使用465或587）。

**SMTP工作流程：**

```
客户端                                  服务器
   |                                     |
   |-------- TCP连接建立 -------------->|
   |<------- 220 服务就绪 -------------|
   |
   |-------- HELO/EHLO --------------- -->|
   |<------- 250 OK --------------------|
   |
   |-------- AUTH LOGIN -------------->|
   |<------- 334 Base64(用户名) --------|
   |-------- Base64(用户名) ----------->|
   |<------- 334 Base64(密码) ---------|
   |-------- Base64(密码) ------------->|
   |<------- 235 认证成功 --------------|
   |
   |-------- MAIL FROM -------------->|
   |<------- 250 OK --------------------|
   |
   |-------- RCPT TO ----------------->|
   |<------- 250 OK --------------------|
   |
   |-------- DATA -------------------->|
   |<------- 354 开始发送邮件内容 ------|
   |
   |-------- 邮件内容（Headers+Body） ->|
   |<------- 250 OK --------------------|
   |
   |-------- QUIT -------------------->|
   |<------- 221 关闭连接 --------------|
```

## 八、邮件队列：BullMQ

### 8.1 消息队列简介

消息队列（Message Queue）是一种异步通信机制，用于解耦生产者和消费者。在邮件发送场景中，消息队列可以帮助我们：

- **异步处理**：将邮件发送请求放入队列，立即返回响应，提升用户体验
- **流量削峰**：高峰期将请求排队，平滑处理，避免SMTP服务器过载
- **失败重试**：发送失败时自动重试，提高可靠性
- **定时发送**：支持延迟发送、定时发送

### 8.2 BullMQ简介

BullMQ是基于Redis的消息队列库，是Bull队列的下一代版本，支持优先队列、延迟任务、重复任务、限流等高级功能。

**安装BullMQ：**

```bash
npm install bullmq ioredis
```

```typescript
/**
 * 基于队列的邮件服务
 * 将邮件发送请求加入队列，而非直接发送
 */
class QueueEmailService {
  private queue: Queue;

  constructor(queueName: string = 'email:send') {
    this.queue = new Queue(queueName, {
      connection: myRedis,
      defaultJobOptions: {
        attempts: 3,
        backoff: {
          type: 'exponential',
          delay: 2000
        },
        removeOnComplete: 100,
        removeOnFail: 1000
      }
    });
  }

  /**
   * 异步发送邮件（加入队列）
   */
  async send(options: {
    to: string | string[];
    subject: string;
    html: string;
    text?: string;
    attachments?: any[];
    priority?: number;
  }): Promise<string> {
    const job = await this.queue.add('send', {
      to: options.to,
      subject: options.subject,
      html: options.html,
      text: options.text,
      attachments: options.attachments
    }, {
      priority: options.priority || 2
    });

    return job.id!;
  }

  /**
   * 发送验证码（自动加入队列）
   */
  async sendVerificationCode(email: string, code: string): Promise<string> {
    const job = await this.queue.add(
      'verification',
      { email, code },
      {
        delay: 1000,  // 延迟1秒
        attempts: 5
      }
    );

    return job.id!;
  }

  /**
   * 获取队列状态
   */
  async getStatus() {
    return getQueueStatus('email:send');
  }

  /**
   * 获取任务状态
   */
  async getJobStatus(jobId: string) {
    return getJobDetails('email:send', jobId);
  }

  /**
   * 等待任务完成
   * 适用于需要知道发送结果的场景
   */
  async waitForCompletion(jobId: string, timeout: number = 30000): Promise<any> {
    const job = await this.queue.getJob(jobId);

    if (!job) {
      throw new Error('任务不存在');
    }

    // 等待任务完成或超时
    return job.waitUntilFinished(
      new QueueEvents('email:send', { connection: myRedis }),
      timeout
    );
  }
}
```

```typescript
class VerificationService {
  /**
   * 生成验证码
   */
  private generateCode(): string {
    const min = Math.pow(10, config.verification.codeLength - 1);
    const max = Math.pow(10, config.verification.codeLength) - 1;
    return String(Math.floor(Math.random() * (max - min + 1)) + min);
  }

  /**
   * 检查冷却时间
   */
  async checkCooldown(email: string): Promise<{ allowed: boolean; remainingSeconds: number }> {
    const cooldownKey = REDIS_KEYS.sendCooldown(email);
    const ttl = await redis.ttl(cooldownKey);

    if (ttl > 0) {
      return { allowed: false, remainingSeconds: ttl };
    }

    return { allowed: true, remainingSeconds: 0 };
  }

  /**
   * 设置冷却时间
   */
  private async setCooldown(email: string): Promise<void> {
    const cooldownKey = REDIS_KEYS.sendCooldown(email);
    await redis.setex(cooldownKey, config.verification.resendCooldown, '1');
  }

  /**
   * 存储验证码
   */
  private async storeCode(email: string, code: string): Promise<void> {
    const codeKey = REDIS_KEYS.verificationCode(email);
    // 验证码5分钟过期
    await redis.setex(codeKey, config.verification.expiresMinutes * 60, code);

    // 记录尝试次数
    const attemptsKey = REDIS_KEYS.verificationAttempts(email);
    await redis.setex(attemptsKey, config.verification.expiresMinutes * 60, '0');
  }

  /**
   * 获取验证码
   */
  async getCode(email: string): Promise<string | null> {
    const codeKey = REDIS_KEYS.verificationCode(email);
    return await redis.get(codeKey);
  }

  /**
   * 验证验证码（并检查尝试次数）
   */
  async verifyCode(email: string, code: string): Promise<{
    valid: boolean;
    reason?: string;
  }> {
    const attemptsKey = REDIS_KEYS.verificationAttempts(email);

    // 检查尝试次数
    const attemptsStr = await redis.get(attemptsKey);
    const attempts = parseInt(attemptsStr || '0');

    if (attempts >= config.verification.maxAttempts) {
      return {
        valid: false,
        reason: '验证尝试次数过多，请重新获取验证码'
      };
    }

    // 增加尝试次数
    await redis.incr(attemptsKey);

    // 获取正确验证码
    const storedCode = await this.getCode(email);

    if (!storedCode) {
      return {
        valid: false,
        reason: '验证码已过期，请重新获取'
      };
    }

    if (storedCode !== code) {
      return {
        valid: false,
        reason: '验证码错误'
      };
    }

    // 验证成功，删除验证码
    await redis.del(REDIS_KEYS.verificationCode(email));
    await redis.del(attemptsKey);

    return { valid: true };
  }

  /**
   * 发送验证码邮件
   */
  async sendVerification(email: string, username?: string): Promise<{
    success: boolean;
    message: string;
    jobId?: string;
    previewUrl?: string;
  }> {
    // 检查冷却时间
    const { allowed, remainingSeconds } = await this.checkCooldown(email);

    if (!allowed) {
      return {
        success: false,
        message: `请 ${remainingSeconds} 秒后再试`
      };
    }

    // 生成验证码
    const code = this.generateCode();

    // 存储验证码
    await this.storeCode(email, code);

    // 设置冷却时间
    await this.setCooldown(email);

    // 加入邮件队列
    const job = await emailQueue.add('verification', {
      to: email,
      code,
      username
    });

    // 开发环境返回预览链接
    const previewUrl = config.nodeEnv === 'development'
      ? `http://localhost:${config.port}/api/debug/preview-email?email=${encodeURIComponent(email)}`
      : undefined;

    console.log(`[Verification] 验证码已加入队列: ${job.id}, 邮箱: ${email}, 验证码: ${code}`);

    return {
      success: true,
      message: '验证码已发送',
      jobId: job.id,
      previewUrl
    };
  }

  /**
   * 注册用户（伪实现）
   */
  async registerUser(email: string, password: string, username: string): Promise<{
    success: boolean;
    userId?: string;
    token?: string;
  }> {
    // 实际应用中这里会：
    // 1. 检查邮箱是否已注册
    // 2. 哈希密码
    // 3. 创建用户记录
    // 4. 生成JWT token

    const userId = crypto.randomBytes(16).toString('hex');
    const token = crypto.randomBytes(32).toString('hex');

    // 存储注册token
    const tokenKey = REDIS_KEYS.registerToken(token);
    await redis.setex(tokenKey, 3600, JSON.stringify({ userId, email }));

    return {
      success: true,
      userId,
      token
    };
  }
}
```

## 十、核心概念与协议详解

### 10.1 流式文件处理

流（Stream）是Node.js最强大的特性之一，允许以流式方式处理数据，无需将整个文件加载到内存中。

**流的概念：**

```
普通方式（一次性读取）：
┌─────────────────────────────────┐
│         全部文件数据              │
│    [====================]       │
└─────────────────────────────────┘
           ↓ 全部加载
    ┌─────────────────┐
    │   内存缓冲区      │ ← 文件越大，内存占用越高
    └─────────────────┘

流式方式（分块读取）：
┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐
│块1 │→│块2 │→│块3 │→│块4 │→│块5 │ → 处理/存储
└────┘ └────┘ └────┘ └────┘ └────┘
  ↓     ↓     ↓     ↓     ↓
[内存] [内存] [内存] [内存] [内存] ← 只保留当前块，内存占用恒定
```

**Node.js流类型：**

| 类型 | 方向 | 说明 |
|------|------|------|
| Readable | 可读 | 数据来源（如文件读取、网络下载） |
| Writable | 可写 | 数据目标（如文件写入、网络上传） |
| Duplex | 双工 | 同时可读可写（如TCP Socket） |
| Transform | 转换 | 读写过程中转换数据（如压缩、解压） |

**流式文件处理示例：**

```typescript
import { createReadStream, createWriteStream } from 'fs';
import { pipeline } from 'stream/promises';
import { Transform, zlib } from 'zlib';
import csvParser from 'csv-parser';

// Transform流示例：处理CSV文件
class CSVProcessor extends Transform {
  private headers: string[] = [];
  private rowCount = 0;

  constructor() {
    // 对象模式（处理对象而非Buffer）
    super({ objectMode: true });
  }

  _transform(row: any, encoding: string, callback: Function) {
    // 处理每一行数据
    if (this.headers.length === 0) {
      this.headers = Object.keys(row);
    }

    this.rowCount++;

    // 添加行号
    row._rowNumber = this.rowCount;

    // 可以在这里添加数据转换逻辑
    // row.total = row.quantity * row.price;

    // 推送处理后的行
    this.push(row);

    callback();
  }
}

    readStream,
    // 可选：添加转换流
    // zlib.createGzip(),  // 压缩
    writeStream
  );

  console.log('文件复制完成');
}

  const s3Client = new S3Client({});

  // S3 Upload Manager 需要Readable流
  const upload = new Upload({
    client: s3Client,
    params: {
      Bucket: 'my-bucket',
      Key: key,
      Body: readStream,  // 直接传入流，无需完整加载到内存
      ContentType: 'application/octet-stream'
    },
    // 分片大小（每个分片独立上传）
    partSize: 5 * 1024 * 1024,  // 5MB
    // 并发上传分片数
    queueSize: 4
  });

  // 监听进度
  upload.on('httpUploadProgress', (progress) => {
    if (progress.loaded && progress.total) {
      const percent = Math.round((progress.loaded / progress.total) * 100);
      console.log(`上传进度: ${percent}%`);
    }
  });

  await upload.done();

  return { key };
}

// Multer与流结合：边接收边处理
function createStreamingUploadHandler() {
  return async (req: Request, res: Response) => {
    const chunks: Buffer[] = [];

    // 收集数据块
    req.on('data', (chunk: Buffer) => {
      chunks.push(chunk);
    });

    req.on('end', async () => {
      const buffer = Buffer.concat(chunks);

      // 处理文件...
      const processed = await sharp(buffer)
        .resize(800, 600)
        .jpeg({ quality: 80 })
        .toBuffer();

      // 上传到云存储
      await streamUploadToS3(
        require('stream').Readable.from(processed),
        `uploads/${Date.now()}.jpg`
      );

      res.json({ success: true });
    });
  };
}
```

### 10.2 断点续传原理

断点续传允许在网络中断后，从上次中断的位置继续传输，而无需重新开始。这对于大文件上传至关重要。

**断点续传原理：**

```
普通上传：
客户端 ────────[文件]────────────────> 服务器
              (网络中断)
客户端 ────────[文件]────────────────> 服务器  ← 重新开始

断点续传：
客户端 ────────[部分1]──────> 服务器 (已保存)
              (网络中断)
客户端 ────────[部分2]──────> 服务器 (追加保存)
客户端 ────────[完成]───────> 服务器

实现方式：
1. 前端将文件分片
2. 每片记录已上传状态
3. 服务端记录已接收的分片
4. 中断后，前端查询已上传部分，跳过继续上传
5. 服务端将分片合并为完整文件
```

**分片上传实现：**

```typescript
import express, { Request, Response } from 'express';
import multer from 'multer';
import fs from 'fs';
import path from 'path';
import crypto from 'crypto';

// 分片上传配置
interface ChunkConfig {
  // 分片大小（字节）
  chunkSize: number;
  // 临时目录
  tempDir: string;
  // 合并后目录
  uploadDir: string;
}

const chunkConfig: ChunkConfig = {
  chunkSize: 2 * 1024 * 1024,  // 2MB per chunk
  tempDir: './temp_chunks',
  uploadDir: './uploads'
};

// 确保目录存在
[chunkConfig.tempDir, chunkConfig.uploadDir].forEach(dir => {
  if (!fs.existsSync(dir)) {
    fs.mkdirSync(dir, { recursive: true });
  }
});

// =============================================
// 分片上传服务
// =============================================

class ChunkUploadService {
  /**
   * 获取文件分片信息
   */
  calculateChunks(fileSize: number): number {
    return Math.ceil(fileSize / chunkConfig.chunkSize);
  }

  /**
   * 获取分片文件路径
   */
  getChunkPath(identifier: string, chunkIndex: number): string {
    return path.join(chunkConfig.tempDir, `${identifier}_${chunkIndex}`);
  }

  /**
   * 获取文件合并后的路径
   */
  getMergedPath(identifier: string, filename: string): string {
    return path.join(chunkConfig.uploadDir, `${identifier}_${filename}`);
  }

  /**
   * 检查分片是否存在
   */
  chunkExists(identifier: string, chunkIndex: number): boolean {
    const chunkPath = this.getChunkPath(identifier, chunkIndex);
    return fs.existsSync(chunkPath);
  }

  /**
   * 获取已上传的分片列表
   */
  getUploadedChunks(identifier: string, totalChunks: number): number[] {
    const uploaded: number[] = [];

    for (let i = 0; i < totalChunks; i++) {
      if (this.chunkExists(identifier, i)) {
        uploaded.push(i);
      }
    }

    return uploaded;
  }

  /**
   * 保存分片
   */
  async saveChunk(identifier: string, chunkIndex: number, data: Buffer): Promise<void> {
    const chunkPath = this.getChunkPath(identifier, chunkIndex);
    await fs.promises.writeFile(chunkPath, data);
  }

  /**
   * 合并分片
   */
  async mergeChunks(identifier: string, filename: string, totalChunks: number): Promise<string> {
    const mergedPath = this.getMergedPath(identifier, filename);
    const writeStream = fs.createWriteStream(mergedPath);

    return new Promise((resolve, reject) => {
      let chunkIndex = 0;

      const writeNextChunk = () => {
        if (chunkIndex >= totalChunks) {
          writeStream.end();
          return;
        }

        const chunkPath = this.getChunkPath(identifier, chunkIndex);
        const readStream = fs.createReadStream(chunkPath);

        readStream.pipe(writeStream, { end: false });

        readStream.on('end', () => {
          chunkIndex++;
          writeNextChunk();
        });

        readStream.on('error', reject);
      };

      writeStream.on('finish', () => {
        // 合并完成后删除分片
        this.cleanupChunks(identifier, totalChunks);
        resolve(mergedPath);
      });

      writeStream.on('error', reject);

      writeNextChunk();
    });
  }

  /**
   * 清理分片文件
   */
  async cleanupChunks(identifier: string, totalChunks: number): Promise<void> {
    for (let i = 0; i < totalChunks; i++) {
      const chunkPath = this.getChunkPath(identifier, i);
      if (fs.existsSync(chunkPath)) {
        await fs.promises.unlink(chunkPath);
      }
    }
  }

  /**
   * 获取文件MD5（通过合并后的文件）
   */
  async calculateFileHash(filePath: string): Promise<string> {
    const buffer = await fs.promises.readFile(filePath);
    return crypto.createHash('md5').update(buffer).digest('hex');
  }
}

// 创建分片上传服务实例
const chunkService = new ChunkUploadService();

// Multer配置（内存存储，用于接收分片）
const memoryStorage = multer.memoryStorage();
const upload = multer({ storage: memoryStorage });

// =============================================
// API路由
// =============================================

// ----------------------------------------
// API 1: 初始化上传，获取上传标识
// ----------------------------------------
/**
 * POST /api/upload/init
 * 请求体: { filename: string, fileSize: number, md5?: string }
 */
app.post('/api/upload/init', async (req: Request, res: Response) => {
  const { filename, fileSize, md5 } = req.body;

  if (!filename || !fileSize) {
    return res.status(400).json({
      success: false,
      message: '缺少filename或fileSize'
    });
  }

  // 生成唯一标识符（通常用文件的MD5，这样相同文件只上传一次）
  const identifier = md5 || crypto.randomBytes(16).toString('hex');
  const totalChunks = chunkService.calculateChunks(fileSize);

  // 检查是否已存在完成的上传（文件MD5匹配）
  const existingPath = chunkService.getMergedPath(identifier, filename);
  if (fs.existsSync(existingPath)) {
    return res.json({
      success: true,
      alreadyUploaded: true,
      url: `/uploads/${path.basename(existingPath)}`,
      message: '文件已存在'
    });
  }

  // 获取已上传的分片
  const uploadedChunks = chunkService.getUploadedChunks(identifier, totalChunks);

  res.json({
    success: true,
    identifier,
    chunkSize: chunkConfig.chunkSize,
    totalChunks,
    uploadedChunks,
    message: uploadedChunks.length > 0
      ? `检测到 ${uploadedChunks.length} 个已上传的分片`
      : '上传初始化完成'
  });
});

// ----------------------------------------
// API 2: 上传分片
// ----------------------------------------
/**
 * POST /api/upload/chunk
 * 请求体: FormData { chunk: File, identifier: string, chunkIndex: number, filename: string, totalChunks: number }
 */
app.post('/api/upload/chunk', upload.single('chunk'), async (req: Request, res: Response) => {
  const { identifier, chunkIndex, filename, totalChunks } = req.body;
  const chunk = req.file;

  if (!chunk) {
    return res.status(400).json({
      success: false,
      message: '没有分片数据'
    });
  }

  if (identifier === undefined || chunkIndex === undefined || !filename) {
    return res.status(400).json({
      success: false,
      message: '缺少必要参数'
    });
  }

  const index = parseInt(chunkIndex);
  const total = parseInt(totalChunks);

  try {
    // 保存分片
    await chunkService.saveChunk(identifier, index, chunk.buffer);

    console.log(`分片 ${index + 1}/${total} 已保存`);

    // 检查是否所有分片都已上传
    const uploadedChunks = chunkService.getUploadedChunks(identifier, total);

    if (uploadedChunks.length === total) {
      // 所有分片已上传，开始合并
      console.log('所有分片已上传，开始合并...');

      const mergedPath = await chunkService.mergeChunks(identifier, filename, total);

      return res.json({
        success: true,
        complete: true,
        url: `/uploads/${path.basename(mergedPath)}`,
        message: '文件上传完成'
      });
    }

    res.json({
      success: true,
      complete: false,
      uploadedChunks,
      progress: `${uploadedChunks.length}/${total}`
    });

  } catch (error) {
    console.error('分片保存失败:', error);
    res.status(500).json({
      success: false,
      message: '分片保存失败'
    });
  }
});

// ----------------------------------------
// API 3: 查询上传状态
// ----------------------------------------
/**
 * GET /api/upload/status/:identifier
 * 查询: identifier, totalChunks, filename
 */
app.get('/api/upload/status/:identifier', async (req: Request, res: Response) => {
  const { identifier } = req.params;
  const { totalChunks, filename } = req.query;

  if (!totalChunks || !filename) {
    return res.status(400).json({
      success: false,
      message: '缺少totalChunks或filename参数'
    });
  }

  const total = parseInt(totalChunks as string);
  const uploadedChunks = chunkService.getUploadedChunks(identifier, total);

  // 检查是否已存在完成的上传
  const existingPath = chunkService.getMergedPath(identifier, filename as string);
  const alreadyMerged = fs.existsSync(existingPath);

  res.json({
    success: true,
    uploadedChunks,
    progress: `${uploadedChunks.length}/${total}`,
    complete: uploadedChunks.length === total || alreadyMerged,
    alreadyMerged
  });
});

// ----------------------------------------
// API 4: 取消上传
// ----------------------------------------
/**
 * DELETE /api/upload/cancel/:identifier
 * 查询: totalChunks
 */
app.delete('/api/upload/cancel/:identifier', async (req: Request, res: Response) => {
  const { identifier } = req.params;
  const { totalChunks } = req.query;

  if (!totalChunks) {
    return res.status(400).json({
      success: false,
      message: '缺少totalChunks参数'
    });
  }

  const total = parseInt(totalChunks as string);

  await chunkService.cleanupChunks(identifier, total);

  res.json({
    success: true,
    message: '上传已取消'
  });
});
```

### 10.3 CDN加速

CDN（Content Delivery Network）内容分发网络，通过在全球部署边缘节点，缓存静态资源，使用户能够从最近的节点获取内容，减少延迟，提高访问速度。

**CDN工作原理：**

```
用户请求资源：
┌──────────┐                    ┌─────────────┐                    ┌──────────────┐
│   用户   │ ────────请求──────→│   CDN边缘节点 │───缓存未命中────→│   源站服务器  │
│          │ ←──────响应───────│              │←─────────────────│              │
└──────────┘                    └─────────────┘                    └──────────────┘
     │                                │
     │                                │ 缓存命中时直接返回
     │                                ↓
     │                          ┌─────────────┐
     └──────────────────────────│   缓存文件   │
                                └─────────────┘
```

**CDN缓存策略配置：**

```typescript
// HTTP缓存控制响应头
const cacheHeaders = {
  // 公共资源（CDN可缓存）
  // 适用于：CSS、JS、图片、字体等静态资源
  'Cache-Control': 'public, max-age=31536000, immutable',

  // 私有资源（CDN不可缓存，仅浏览器缓存）
  // 适用于：用户个性化内容、API响应
  'Cache-Control': 'private, no-cache',

  // 协商缓存（需要验证）
  // 适用于：可能变化的静态资源
  'Cache-Control': 'public, max-age=0, must-revalidate'
};

// 静态资源服务（带CDN缓存头）
app.use('/static', express.static(path.join(__dirname, 'public'), {
  // 设置Cache-Control
  setHeaders: (res, filePath) => {
    // 根据文件类型设置不同的缓存策略
    if (filePath.endsWith('.html')) {
      // HTML文件不缓存或短期缓存
      res.set('Cache-Control', 'no-cache, no-store, must-revalidate');
    } else if (filePath.match(/\.(css|js|jpg|jpeg|png|gif|ico|svg|woff|woff2|ttf|eot)$/)) {
      // 静态资源长期缓存
      // 配合构建时的文件哈希，实现内容更新时自动换缓存
      res.set('Cache-Control', 'public, max-age=31536000, immutable');
    } else {
      // 其他资源默认缓存1天
      res.set('Cache-Control', 'public, max-age=86400');
    }
  }
}));

// ----------------------------------------
// CDN URL生成（使用CDN域名替代源站域名）
// ----------------------------------------

interface CDNConfig {
  baseUrl: string;      // CDN域名
  sourceDir: string;    // 源站目录
  signingKey?: string;  // 签名密钥（用于私有CDN）
}

// CDN服务类
class CDNService {
  private baseUrl: string;
  private sourceDir: string;

  constructor(config: CDNConfig) {
    this.baseUrl = config.baseUrl;
    this.sourceDir = config.sourceDir;
  }

  /**
   * 生成CDN URL
   */
  getUrl(filePath: string): string {
    // 移除源站目录前缀
    const relativePath = path.relative(this.sourceDir, filePath);
    // 拼接CDN域名
    return `${this.baseUrl}/${relativePath.replace(/\\/g, '/')}`;
  }

  /**
   * 生成带签名的CDN URL（私有CDN）
   */
  getSignedUrl(filePath: string, expiresIn: number = 3600): string {
    const relativePath = path.relative(this.sourceDir, filePath).replace(/\\/g, '/');
    const expires = Math.floor(Date.now() / 1000) + expiresIn;

    // 签名字符串：路径 + 过期时间
    const signature = crypto
      .createHmac('sha256', 'your-signing-key')
      .update(`${relativePath}${expires}`)
      .digest('hex');

    return `${this.baseUrl}/${relativePath}?expires=${expires}&signature=${signature}`;
  }

  /**
   * 生成图片处理URL（图片CDN支持实时处理）
   */
  getImageUrl(filePath: string, options: {
    width?: number;
    height?: number;
    format?: 'webp' | 'avif' | 'jpg';
    quality?: number;
  }): string {
    const relativePath = path.relative(this.sourceDir, filePath).replace(/\\/g, '/');

    // 构建图片处理参数（不同CDN格式不同）
    const params = new URLSearchParams();

    if (options.width) params.set('w', options.width.toString());
    if (options.height) params.set('h', options.height.toString());
    if (options.format) params.set('f', options.format);
    if (options.quality) params.set('q', options.quality.toString());

    return `${this.baseUrl}/${relativePath}?${params.toString()}`;
  }
}
```

### 10.4 邮件协议详解

**SMTP协议（发送）：**

SMTP（Simple Mail Transfer Protocol）是发送邮件的基础协议。

```
主要命令：
HELO/EHLO <domain>     - 开始会话，声明域名
AUTH PLAIN/CRAM-MD5     - 认证方式
MAIL FROM:<sender>      - 发件人
RCPT TO:<recipient>     - 收件人
DATA                   - 开始邮件内容
QUIT                   - 关闭会话

状态码：
220 - 服务就绪
250 - 请求完成
354 - 开始发送内容
221 - 关闭连接
500 - 语法错误
550 - 邮箱不可用
```

**IMAP协议（接收/管理）：**

IMAP（Internet Message Access Protocol）允许客户端在服务器上管理和阅读邮件。

```
主要命令：
LOGIN <user> <pass>    - 登录
SELECT <mailbox>        - 选择邮箱（如INBOX）
LIST "" *              - 列出所有邮箱
FETCH <id> <items>     - 获取邮件内容
STORE <id> <flags>      - 修改邮件标志（已读/未读等）
SEARCH <criteria>       - 搜索邮件
LOGOUT                 - 退出

标志：
\Seen      - 已读
\Answered  - 已回复
\Flagged   - 标记
\Deleted   - 删除
\Draft     - 草稿
```

**POP3协议（接收）：**

POP3（Post Office Protocol v3）是另一种接收协议，与IMAP的区别在于通常会下载邮件到本地。

```
主要命令：
USER <username>        - 用户名
PASS <password>        - 密码
STAT                   - 状态（邮件数/大小）
LIST                   - 列出邮件
RETR <id>              - 获取邮件
DELE <id>              - 删除邮件
QUIT                   - 退出

特点：
- 简单，但功能有限
- 通常下载后删除服务器邮件
- 不支持服务器端文件夹同步
```

**IMAP vs POP3：**

| 特性 | IMAP | POP3 |
|------|------|------|
| 多设备同步 | 支持 | 不支持 |
| 服务器存储 | 保留邮件 | 通常下载后删除 |
| 文件夹支持 | 支持 | 不支持 |
| 邮件状态同步 | 同步 | 本地 |
| 离线支持 | 受限 | 完整 |
| 复杂度 | 较高 | 较低 |
