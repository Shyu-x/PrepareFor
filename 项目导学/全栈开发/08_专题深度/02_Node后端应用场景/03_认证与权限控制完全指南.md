# 认证与权限控制完全指南

## 一、从故事开始：为什么需要认证和授权？

### 1.1 生活中的认证与授权

想象你去一家高档餐厅：

- **认证**：服务员要你出示身份证，证明"你确实是你"——这就是**认证（Authentication）**
- **授权**：服务员查了你的会员卡，发现你是VIP，可以进入包厢就餐——这就是**授权（Authorization）**

用一个更直观的例子：

| 场景 | 认证 | 授权 |
|------|------|------|
| 进入小区 | 门禁刷卡（证明你是业主） | 业主可以进所有楼，访客只能进客厅 |
| 使用银行APP | 登录密码（证明你是你） | 你只能操作自己的账户 |
| 公司门禁 | 工卡刷卡（证明你是员工） | 员工只能进自己部门，经理可以进所有区域 |

### 1.2 技术世界的认证与授权

在Web开发中：

```
用户浏览器                    服务器                      数据库
    |                          |                           |
    |-- 请求登录（用户名密码） -->                           |
    |                          |-- 验证密码 ----------------->|
    |                          |<-- 用户信息 ----------------|
    |                          |-- 生成登录凭证              |
    |<-- 返回凭证（Cookie/Token)--                            |
    |                          |                            |
    |-- 请求文章列表（带凭证） -->                            |
    |                          |-- 验证凭证                  |
    |                          |-- 查询我的文章 ------------->|
    |<-- 返回文章列表 ---------|                            |
```

**认证（Authentication）**：验证"你是谁"
- 登录：输入用户名和密码
- 注册：创建新账户
- 找回密码：通过邮箱验证身份

**授权（Authorization）**：验证"你能做什么"
- 普通用户：只能看自己的文章
- 管理员：可以看所有人的文章、删除任何文章
- 编辑：可以编辑所有人的文章

### 1.3 为什么需要认证？

**原因一：个性化服务**
- 购物车：记住你加入了什么商品
- 阅读历史：推荐你可能喜欢的文章
- 个人设置：记住你的偏好

**原因二：数据保护**
- 你的银行账户只有你能操作
- 你的聊天记录只有你能看到
- 你的隐私文章只有你能访问

**原因三：追踪与问责**
- 谁在什么时候做了什么操作
- 出了问题可以追溯
- 恶意行为可以追踪到人

## 二、认证方式大比拼：Session vs JWT

### 2.1 Session 是什么？

**Session（会话）**：像一张游乐园的通行证

```
工作原理：
1. 你买票入场（登录成功）
2. 工作人员给你一张手环（Session ID）
3. 你凭手环可以玩各种项目（访问受保护资源）
4. 手环关联着你的信息（用户数据存在服务器）
5. 离开时归还手环（退出登录，手环失效）

优点：
- 服务器控制，可以随时失效
- 数据存在服务器，更安全
- 实现简单，生态成熟

缺点：
- 需要存储空间（内存或数据库）
- 分布式部署麻烦（需要共享Session）
- 扩展性较差
```

### 2.2 JWT 是什么？

**JWT（JSON Web Token）**：像一张 laminated（过胶）的会员卡

```
工作原理：
1. 你买票入场（登录成功）
2. 工作人员给你一张 laminated 卡（JWT）
3. 卡上直接印着你的信息（用户名、会员等级）
4. 你凭卡可以玩各种项目（访问受保护资源）
5. 任何人看到卡都知道你是谁（令牌自包含信息）

优点：
- 无状态，服务器不需要存储
- 天然适合分布式系统
- 扩展性好，轻量

缺点：
- 一旦签发，无法撤销（除非加入黑名单）
- 数据存在令牌里，有大小限制
- 令牌泄露风险
```

### 2.3 直观对比

| 特性 | Session | JWT |
|------|---------|-----|
| 存储位置 | 服务器（内存/Redis/数据库） | 客户端（Cookie/LocalStorage） |
| 扩展性 | 需要Session共享 | 天然支持分布式 |
| 安全性 | 相对安全（数据在服务器） | 需要注意令牌保护 |
| 性能 | 需要查询存储 | 验证快（无需查询） |
| 撤销 | 可以立即失效 | 需要黑名单机制 |
| 大小 | 小（只有Session ID） | 较大（包含用户信息） |
| 使用场景 | 传统Web应用 | 移动应用、微服务、API |

### 2.4 什么时候用哪个？

```
用 Session 的场景：
- 传统Web应用（后端渲染）
- 需要频繁修改权限
- 安全性要求很高
- 用户量不大的企业应用

用 JWT 的场景：
- 移动应用（App、小程序）
- 微服务架构
- 前后端分离的API
- 需要高性能的分布式系统
```

## 三、密码安全：bcrypt 加密详解

### 3.1 为什么不能明文存储密码？

```
错误做法：
数据库存储：admin / 123456

如果数据库泄露：
- 黑客直接看到所有用户的密码
- 用户往往在多个网站使用相同密码
- 你的密码可能也被用来访问其他网站

正确做法：
数据库存储：admin / $2b$12$LQv3c1yqBWVHxkd0LHAkCOYz6TtxMQJqhN8/LewY5IsJ1r6G.isWe

这样即使用户拿到这个字符串，也看不懂是什么意思
```

### 3.2 bcrypt 是什么？

**bcrypt**：一种专门为密码设计的哈希算法

```
bcrypt 的特点：

1. 加盐（Salt）
   - 自动生成随机字符串
   - 相同密码每次加密结果不同
   - 防止彩虹表攻击

2. 可调节难度
   - 通过 cost 参数控制计算时间
   - 硬件变强了，可以提高难度
   - 保持安全性

3. 专为密码设计
   - 计算速度慢（故意设计）
   - 适合密码验证场景
   - 多年验证，安全性高
```

### 3.3 bcrypt 实战代码

```typescript
// src/utils/password.ts
import bcrypt from 'bcrypt';

// bcrypt 的工作原理：
// 1. 生成随机盐（salt）
// 2. 将盐和密码一起哈希
// 3. 返回 "盐$哈希值" 的格式

/**
 * 密码服务
 * 负责密码的加密和验证
 */
class PasswordService {
  // 加密轮数，数值越高越安全但越慢
  // 12 是一个比较好的平衡点（2024年的推荐值）
  private readonly SALT_ROUNDS = 12;

  /**
   * 加密密码
   * 将用户输入的明文密码转换为哈希值
   *
   * @param password - 用户输入的明文密码
   * @returns 加密后的哈希字符串
   *
   * 示例：输入 "MyPass123" -> 输出 "$2b$12$..."
   */
  async hash(password: string): Promise<string> {
    // bcrypt.hash() 会自动：
    // 1. 生成随机盐（16个随机字节）
    // 2. 将盐和密码一起哈希
    // 3. 返回格式：$2b$版本$轮数$22位的盐$31位的哈希
    return bcrypt.hash(password, this.SALT_ROUNDS);
  }

  /**
   * 验证密码
   * 将用户输入的密码和存储的哈希值比对
   *
   * @param password - 用户输入的明文密码
   * @param hashedPassword - 数据库中存储的哈希值
   * @returns 密码是否匹配
   */
  async compare(password: string, hashedPassword: string): Promise<boolean> {
    // bcrypt.compare() 会自动：
    // 1. 从哈希值中提取盐
    // 2. 用同样的盐加密输入的密码
    // 3. 比较两个哈希值是否相同
    return bcrypt.compare(password, hashedPassword);
  }

  /**
   * 验证密码强度
   * 帮助用户设置一个安全的密码
   */
  validateStrength(password: string): { valid: boolean; errors: string[] } {
    const errors: string[] = [];

    // 检查长度
    if (password.length < 8) {
      errors.push('密码长度至少8位');
    }
    if (password.length < 12) {
      errors.push('建议使用12位以上的密码');
    }

    // 检查是否包含小写字母
    if (!/[a-z]/.test(password)) {
      errors.push('密码需包含小写字母');
    }

    // 检查是否包含大写字母
    if (!/[A-Z]/.test(password)) {
      errors.push('密码需包含大写字母');
    }

    // 检查是否包含数字
    if (!/[0-9]/.test(password)) {
      errors.push('密码需包含数字');
    }

    // 检查是否包含特殊字符
    if (!/[!@#$%^&*(),.?":{}|<>]/.test(password)) {
      errors.push('密码需包含特殊字符');
    }

    // 检查常见密码
    const commonPasswords = [
      'password', '123456', 'qwerty', 'abc123',
      'letmein', 'welcome', 'monkey', 'dragon'
    ];
    if (commonPasswords.includes(password.toLowerCase())) {
      errors.push('不能使用常见密码');
    }

    // 检查连续重复字符
    if (/(.)\1{2,}/.test(password)) {
      errors.push('不能包含连续重复字符');
    }

    // 检查键盘序列
    const keyboardSequences = ['123456789', '987654321', 'qwertyuiop', 'asdfghjkl'];
    if (keyboardSequences.some(seq => password.toLowerCase().includes(seq))) {
      errors.push('不能包含键盘序列');
    }

    return {
      valid: errors.length === 0,
      errors,
    };
  }

  /**
   * 生成随机密码
   * 用于自动生成临时密码或重置密码
   */
  generate(length: number = 16): string {
    // 定义字符集
    const lowercase = 'abcdefghijklmnopqrstuvwxyz';
    const uppercase = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ';
    const numbers = '0123456789';
    const specials = '!@#$%^&*()_+-=[]{}|;:,.<>?';

    const allChars = lowercase + uppercase + numbers + specials;

    // 确保每种字符至少有一个
    let password = '';
    password += lowercase[Math.floor(Math.random() * lowercase.length)];
    password += uppercase[Math.floor(Math.random() * uppercase.length)];
    password += numbers[Math.floor(Math.random() * numbers.length)];
    password += specials[Math.floor(Math.random() * specials.length)];

    // 填充剩余长度
    for (let i = 4; i < length; i++) {
      password += allChars[Math.floor(Math.random() * allChars.length)];
    }

    // 打乱顺序
    return password.split('').sort(() => Math.random() - 0.5).join('');
  }
}

// 导出单例
export const passwordService = new PasswordService();
```

## 四、JWT 认证实战

### 4.1 JWT 的结构

```
一个 JWT 长这样：
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c

分成三部分：
┌─────────────────────────────────────────────────────────────────┐
│ Header.Payload.Signature                                        │
├─────────────────────────────────────────────────────────────────┤
│ Header（头部）         │ eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9   │
│ （告诉别人这是什么）   │ Base64URL 编码的 JSON                   │
├─────────────────────────────────────────────────────────────────┤
│ Payload（载荷）        │ eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6...│
│ （存放要传的数据）     │ Base64URL 编码的 JSON                   │
├─────────────────────────────────────────────────────────────────┤
│ Signature（签名）      │ SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_ad..│
│ （防止数据被篡改）     │ HMACSHA256 加密结果                     │
└─────────────────────────────────────────────────────────────────┘

解码后的内容：
┌──────────────────┐  ┌─────────────────────────────┐
│ Header           │  │ {                          │
│ {                │  │   "sub": "1234567890",     │ ← 用户ID
│   "alg": "HS256",│  │   "name": "John Doe",      │ ← 用户名
│   "typ": "JWT"   │  │   "iat": 1516239022,       │ ← 签发时间
│ }                │  │   "exp": 1516242622        │ ← 过期时间
└──────────────────┘  └─────────────────────────────┘
```

### 4.2 JWT 服务封装

```typescript
// src/utils/jwt.ts
import jwt from 'jsonwebtoken';
import crypto from 'crypto';

// JWT 配置接口
interface JwtConfig {
  secret: string;           // 密钥
  expiresIn: string;        // 访问令牌过期时间
  refreshExpiresIn: string;  // 刷新令牌过期时间
}

// Token 载荷：存放在 JWT 中的用户信息
interface TokenPayload {
  userId: string;   // 用户ID
  email: string;    // 用户邮箱
  role: string;     // 用户角色
}

// Token 响应：登录成功后返回的令牌信息
interface TokenResponse {
  accessToken: string;   // 访问令牌（短期）
  refreshToken: string; // 刷新令牌（长期）
  expiresIn: number;     // 过期时间（秒）
  tokenType: string;     // 令牌类型
}

/**
 * JWT 服务
 * 负责生成、验证、刷新令牌
 */
class JwtService {
  // 默认配置
  private config: JwtConfig = {
    // 密钥：生产环境一定要使用复杂的安全密钥
    // 可以用 crypto.randomBytes(64).toString('hex') 生成
    secret: process.env.JWT_SECRET || 'your-super-secret-key-change-in-production',

    // 访问令牌：15分钟
    // 为什么短？因为访问令牌丢失风险大
    expiresIn: '15m',

    // 刷新令牌：7天
    // 用户可以不用重新登录，用刷新令牌获取新访问令牌
    refreshExpiresIn: '7d',
  };

  /**
   * 生成访问令牌
   * 访问令牌包含用户信息，用于API调用
   *
   * @param payload - 用户信息
   * @returns 访问令牌字符串
   */
  generateAccessToken(payload: TokenPayload): string {
    return jwt.sign(
      {
        ...payload,        // 用户ID、邮箱、角色
        type: 'access',    // 标记这是访问令牌
      },
      this.config.secret,  // 用密钥签名
      {
        expiresIn: this.config.expiresIn,  // 设置过期时间
        issuer: 'blog-api',               // 发行者
        subject: payload.userId,           // 主题（用户ID）
        jwtid: crypto.randomBytes(16).toString('hex'),  // 唯一ID，用于撤销
      }
    );
  }

  /**
   * 生成刷新令牌
   * 刷新令牌只包含用户ID，用于获取新的访问令牌
   *
   * @param payload - 用户信息
   * @returns 刷新令牌字符串
   */
  generateRefreshToken(payload: TokenPayload): string {
    return jwt.sign(
      {
        userId: payload.userId,  // 只存用户ID
        type: 'refresh',         // 标记这是刷新令牌
      },
      this.config.secret,
      {
        expiresIn: this.config.refreshExpiresIn,
        issuer: 'blog-api',
        subject: payload.userId,
      }
    );
  }

  /**
   * 生成令牌对
   * 登录时同时返回访问令牌和刷新令牌
   *
   * @param payload - 用户信息
   * @returns 包含两种令牌的响应
   */
  generateTokenPair(payload: TokenPayload): TokenResponse {
    const accessToken = this.generateAccessToken(payload);
    const refreshToken = this.generateRefreshToken(payload);

    // 解码访问令牌获取过期时间
    const decoded = jwt.decode(accessToken) as any;
    const expiresIn = decoded.exp - decoded.iat;

    return {
      accessToken,
      refreshToken,
      expiresIn,
      tokenType: 'Bearer',  // OAuth 2.0 标准格式
    };
  }

  /**
   * 验证令牌
   * 验证令牌是否有效、是否过期
   *
   * @param token - 令牌字符串
   * @returns 有效返回载荷，无效返回 null
   */
  verify(token: string): TokenPayload | null {
    try {
      const decoded = jwt.verify(token, this.config.secret) as any;

      return {
        userId: decoded.userId,
        email: decoded.email,
        role: decoded.role,
      };
    } catch (error) {
      // 验证失败：可能是过期、伪造、格式错误
      console.error('JWT 验证失败:', error.message);
      return null;
    }
  }

  /**
   * 验证刷新令牌
   * 刷新令牌验证失败不能换访问令牌
   */
  verifyRefreshToken(token: string): { userId: string } | null {
    try {
      const decoded = jwt.verify(token, this.config.secret) as any;

      // 确保是刷新令牌
      if (decoded.type !== 'refresh') {
        console.error('令牌类型错误：期望 refresh');
        return null;
      }

      return {
        userId: decoded.userId,
      };
    } catch (error) {
      console.error('刷新令牌验证失败:', error.message);
      return null;
    }
  }

  /**
   * 解码令牌（不验证）
   * 只解析内容，不验证签名
   * 用于查看令牌内容，调试用
   */
  decode(token: string): any {
    return jwt.decode(token);
  }

  /**
   * 检查令牌是否即将过期
   * 用于前端判断是否需要刷新令牌
   *
   * @param token - 令牌字符串
   * @param thresholdSeconds - 阈值，默认5分钟
   * @returns 是否即将过期
   */
  isTokenExpiringSoon(token: string, thresholdSeconds: number = 300): boolean {
    try {
      const decoded = jwt.decode(token) as any;
      if (!decoded || !decoded.exp) return true;

      const expiresAt = decoded.exp * 1000;  // 转换为毫秒
      const now = Date.now();
      const timeUntilExpiry = expiresAt - now;

      // 如果剩余时间小于阈值，返回 true
      return timeUntilExpiry < thresholdSeconds * 1000;
    } catch {
      return true;
    }
  }

  /**
   * 获取令牌剩余有效时间（秒）
   */
  getTimeToExpiry(token: string): number {
    const decoded = jwt.decode(token) as any;
    if (!decoded?.exp) return 0;
    return Math.max(0, decoded.exp - Math.floor(Date.now() / 1000));
  }
}

// 导出单例
export const jwtService = new JwtService();
```

### 4.3 Session 认证实战

```typescript
// src/utils/session.ts
import session from 'express-session';
import RedisStore from 'connect-redis';
import Redis from 'ioredis';
import crypto from 'crypto';

// Redis 客户端
const redisClient = new Redis(process.env.REDIS_URL || 'redis://localhost:6379');

/**
 * Session 配置
 *
 * Session 的工作原理：
 * 1. 用户登录成功后，服务器创建 Session
 * 2. 服务器生成唯一的 Session ID
 * 3. Session 数据存在服务器（这里是 Redis）
 * 4. 通过 Cookie 将 Session ID 发给浏览器
 * 5. 后续请求浏览器自动带上 Cookie
 * 6. 服务器通过 Session ID 获取用户数据
 */
const sessionConfig = {
  // 会话存储：使用 Redis
  store: new RedisStore({
    client: redisClient,           // Redis 客户端
    prefix: 'session:',            // Key 前缀
    ttl: 7 * 24 * 60 * 60,        // 7天过期（秒）
  }),

  // 密钥：用于签名 Cookie
  secret: process.env.SESSION_SECRET || 'your-session-secret-change-in-production',

  // 会话 ID 名称
  name: 'session_id',

  // 是否每次请求都重置会话过期时间
  // 设为 true 可以实现"活跃用户续期"
  rolling: true,

  // 是否强制保存未修改的会话
  resave: false,

  // 是否在尚未初始化时保存会话
  saveUninitialized: false,

  // Cookie 配置
  cookie: {
    // 仅 HTTPS 传输（生产环境必须设为 true）
    secure: process.env.NODE_ENV === 'production',

    // 防止 XSS 攻击（JavaScript 无法访问）
    httpOnly: true,

    // CSRF 防护
    sameSite: 'strict',

    // 最大存活时间
    maxAge: 7 * 24 * 60 * 60 * 1000,  // 7天
  },
};

 * 创建会话中间件

export function createSessionMiddleware() {
  return session(sessionConfig);
}

  interface SessionData {
    // 用户 ID
    userId: string;

    // 用户邮箱
    email: string;

    // 用户角色
    role: string;

    // 登录时间
    loginAt: Date;

    // 登录 IP
    ip: string;

    // 用户代理

/**
 * Session 服务
 * 提供 Session 相关的操作

class SessionService {
  /**
   * 创建会话
   */
  createSession(
    req: any,
    userData: { userId: string; email: string; role: string }
  ): void {
    req.session.userId = userData.userId;
    req.session.email = userData.email;
    req.session.role = userData.role;
    req.session.loginAt = new Date();
    req.session.ip = req.ip;
    req.session.userAgent = req.headers['user-agent'];
  }

  /**
   * 销毁会话（登出）
   */
  destroySession(req: any): Promise<void> {
    return new Promise((resolve, reject) => {
      req.session.destroy((err: Error) => {
        if (err) reject(err);
        else resolve();
      });
    });
  }

  /**
   * 重新生成会话 ID（安全登出后再登录）
   */
  regenerateSession(req: any): Promise<void> {
    return new Promise((resolve, reject) => {
      req.session.regenerate((err: Error) => {
        if (err) reject(err);
        else resolve();
      });
    });
  }

  /**
   * 获取当前会话用户
   */
  getCurrentUser(req: any): { userId: string; email: string; role: string } | null {
    if (!req.session || !req.session.userId) {
      return null;
    }
    return {
      userId: req.session.userId,
      email: req.session.email,
      role: req.session.role,
    };
  }

  /**
   * 检查用户是否已登录
   */
  isAuthenticated(req: any): boolean {
    return !!(req.session && req.session.userId);
  }
}
```

### 5.1 中间件是什么？

```
中间件：请求处理流水线上的"关卡"

请求流程：
客户端 --> [中间件A] --> [中间件B] --> [路由处理] --> 客户端
              |              |
          检查请求1        检查请求2
          （认证）         （权限）

中间件的特点：
1. 按顺序执行
2. 可以修改请求和响应
3. 可以决定是否继续传递
4. 可以处理错误
```

```typescript
// src/middleware/auth.ts
import { Request, Response, NextFunction } from 'express';
import { jwtService } from '../utils/jwt';

// 扩展 Express 的 Request 类型
// 添加用户信息到请求对象
declare global {
  namespace Express {
    interface Request {
      // 经过认证中间件后，用户信息会附加到这里
      user?: {
        userId: string;
        email: string;
        role: string;
      };
    }
  }
}

/**
 * JWT 认证中间件
 * 验证请求中的 JWT 令牌
 *
 * 工作流程：
 * 1. 从请求头获取 Authorization
 * 2. 提取 Bearer Token
 * 3. 验证 Token
 * 4. 验证成功则附加用户信息到 req.user
 */
export function authenticate(
  req: Request,
  res: Response,
  next: NextFunction
): void {
  // 第一步：获取 Authorization 头
  const authHeader = req.headers.authorization;

  // 检查是否存在 Authorization 头
  if (!authHeader) {
    res.status(401).json({
      success: false,
      error: {
        code: 'MISSING_TOKEN',
        message: '未提供认证令牌，请先登录',
      },
    });
    return;
  }

  // 第二步：检查是否是 Bearer Token 格式
  // 格式应该是：Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
  if (!authHeader.startsWith('Bearer ')) {
    res.status(401).json({
      success: false,
      error: {
        code: 'INVALID_TOKEN_FORMAT',
        message: '令牌格式错误，应该是：Bearer <token>',
      },
    });
    return;
  }

  // 第三步：提取 Token
  const token = authHeader.split(' ')[1];

  // 第四步：验证 Token
  const payload = jwtService.verify(token);

  if (!payload) {
    res.status(401).json({
      success: false,
      error: {
        code: 'INVALID_TOKEN',
        message: '令牌无效或已过期，请重新登录',
      },
    });
    return;
  }

  // 第五步：附加用户信息到请求对象
  req.user = payload;

  // 第六步：继续处理
  next();
}

/**
 * 可选认证中间件
 * 如果有令牌就验证，没有就继续
 * 用于需要认证但不强制的地方
 */
export function optionalAuthenticate(
  req: Request,
  res: Response,
  next: NextFunction
): void {
  const authHeader = req.headers.authorization;

  // 如果没有 Authorization 头，直接继续
  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    next();
    return;
  }

  // 有令牌就尝试验证
  const token = authHeader.split(' ')[1];
  const payload = jwtService.verify(token);

  // 验证成功才附加用户信息
  if (payload) {
    req.user = payload;
  }

  next();
}

/**
 * 角色授权中间件
 * 检查用户是否有指定角色
 *
 * @param allowedRoles - 允许访问的角色列表
 *
 * 示例：
 * router.delete('/articles/:id', authenticate, requireRole('admin'), deleteArticle);
 */
export function requireRole(...allowedRoles: string[]) {
  // 返回一个中间件函数
  return (req: Request, res: Response, next: NextFunction): void => {
    // 第一步：检查是否已认证
    if (!req.user) {
      res.status(401).json({
        success: false,
        error: {
          code: 'UNAUTHORIZED',
          message: '请先登录',
        },
      });
      return;
    }

    // 第二步：检查用户角色
    // 管理员（admin）可以访问所有资源
    if (req.user.role === 'admin' || allowedRoles.includes(req.user.role)) {
      next();
      return;
    }

    // 第三步：角色不匹配
    res.status(403).json({
      success: false,
      error: {
        code: 'FORBIDDEN',
        message: '您没有权限执行此操作',
      },
    });
  };
}

/**
 * 资源所有权验证中间件
 * 检查用户是否拥有该资源
 *
 * @param getResourceId - 从请求中提取资源 ID 的函数
 *
 * 示例：
 * router.put('/articles/:id', authenticate, requireOwnership(
 *   (req) => req.params.id
 * ), updateArticle);
 */
export function requireOwnership(getResourceId: (req: Request) => string) {
  return (req: Request, res: Response, next: NextFunction): void => {
    // 检查是否已认证
    if (!req.user) {
      res.status(401).json({
        success: false,
        error: {
          code: 'UNAUTHORIZED',
          message: '请先登录',
        },
      });
      return;
    }

    // 管理员可以访问所有资源
    if (req.user.role === 'admin') {
      next();
      return;
    }

    // 获取资源 ID
    const resourceId = getResourceId(req);

    // 获取当前用户 ID
    const currentUserId = req.user.userId;

    // 检查是否是资源所有者
    // 实际项目中，这里可能需要查询数据库
    // 为了简化，我们假设资源 ID 就是用户 ID
    if (currentUserId !== resourceId) {
      res.status(403).json({
        success: false,
        error: {
          code: 'FORBIDDEN',
          message: '您无权访问此资源',
        },
      });
      return;
    }

    next();
  };
}
```

## 六、RBAC 权限模型

### 6.1 什么是 RBAC？

**RBAC（Role-Based Access Control）**：基于角色的访问控制

```
核心概念：
┌─────────┐      ┌─────────┐      ┌─────────┐
│   用户   │ ───> │   角色   │ ───> │  权限   │
└─────────┘      └─────────┘      └─────────┘
     │                │                │
     │   多对一        │   多对一        │
     └── 属于 ────────└── 拥有 ────────└── 可以做

示例：
用户（小明）──属于──> 角色（编辑）──拥有──> 权限（创建文章、编辑文章、删除自己的文章）
用户（小红）──属于──> 角色（管理员）──拥有──> 权限（所有权限）
用户（小刚）──属于──> 角色（普通用户）──拥有──> 权限（查看文章、评论）
```

### 6.2 RBAC 的优势

```
为什么用 RBAC？

1. 简化权限管理
   - 不用给每个用户单独分配权限
   - 只需要给角色分配权限
   - 用户通过角色获得权限

2. 便于审计
   - 可以看到每个角色有什么权限
   - 可以看到每个用户属于什么角色

3. 最小权限原则
   - 用户只有完成工作所需的最小权限
   - 减少安全风险

4. 易于扩展
   - 新增角色容易
   - 修改权限只需要改角色
```

### 6.4 RBAC 权限检查服务

```typescript
// src/services/rbac.service.ts
import { Permission, RolePermissions, RoleHierarchy } from '../models/role.model';

/**
 * RBAC 权限服务
 * 提供基于角色的权限检查功能
 */
class RbacService {
  /**
   * 检查用户是否拥有指定权限
   *
   * @param userRole - 用户的角色
   * @param permission - 要检查的权限
   * @returns 是否拥有该权限
   */
  hasPermission(userRole: string, permission: Permission): boolean {
    // 获取角色对应的权限列表
    const permissions = RolePermissions[userRole];

    // 如果角色不存在，返回 false
    if (!permissions) {
      return false;
    }

    // 检查权限列表中是否包含要检查的权限
    return permissions.includes(permission);
  }

  /**
   * 检查用户是否拥有所有指定权限
   */
  hasAllPermissions(userRole: string, permissions: Permission[]): boolean {
    return permissions.every(p => this.hasPermission(userRole, p));
  }

  /**
   * 检查用户是否拥有任意一个指定权限
   */
  hasAnyPermission(userRole: string, permissions: Permission[]): boolean {
    return permissions.some(p => this.hasPermission(userRole, p));
  }

  /**
   * 检查用户角色是否大于等于目标角色
   * 用于判断是否有更高权限
   *
   * @param userRole - 用户的角色
   * @param targetRole - 目标角色
   * @returns 是否大于等于目标角色
   */
  isRoleHigherOrEqual(userRole: string, targetRole: string): boolean {
    const userLevel = RoleHierarchy[userRole] || 0;
    const targetLevel = RoleHierarchy[targetRole] || 0;
    return userLevel >= targetLevel;
  }

  /**
   * 获取角色的所有权限
   */
  getRolePermissions(role: string): Permission[] {
    return RolePermissions[role] || [];
  }

  /**
   * 获取用户的所有权限
   * 支持角色继承
   */
  getUserPermissions(userRole: string): Permission[] {
    const permissions = new Set<Permission>();

    // 获取该角色的所有下级角色的权限
    for (const [role, level] of Object.entries(RoleHierarchy)) {
      const userLevel = RoleHierarchy[userRole] || 0;
      if (level <= userLevel) {
        const rolePerms = RolePermissions[role] || [];
        rolePerms.forEach(p => permissions.add(p));
      }
    }

    return Array.from(permissions);
  }
}

export const rbacService = new RbacService();
```

## 七、Token 刷新机制

### 7.1 为什么要刷新 Token？

```
问题：
访问令牌有效期15分钟，太短了用户需要频繁登录
访问令牌有效期太长，泄露后风险大

解决方案：双令牌机制

┌─────────────────────────────────────────────────────────────────┐
│                         双令牌流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   登录成功                                                       │
│       │                                                         │
│       ├──> 访问令牌（15分钟）───────> 存在内存中，API调用使用      │
│       │                                                         │
│       └──> 刷新令牌（7天）────────> 存在 localStorage，用于刷新   │
│                                                                 │
│   访问令牌即将过期（前端检测）                                    │
│       │                                                         │
│       ├── 发送刷新令牌到 /auth/refresh-token                     │
│       │                                                         │
│       ├──> 验证刷新令牌                                         │
│       │                                                         │

│       │                                                         │

│                                                                 │
│   刷新令牌也过期了                                               │
│       │                                                         │
```

### 7.2 刷新 Token 中间件

```typescript
// src/middleware/tokenRefresh.ts
import { Request, Response, NextFunction } from 'express';
import { jwtService } from '../utils/jwt';

/**
 * 自动刷新 Token 中间件
 * 当访问令牌即将过期时，自动使用刷新令牌获取新令牌
 *
 * 注意：这个中间件需要在 authenticate 中间件之后使用
 */
export function autoRefreshToken(
  req: Request,
  res: Response,
  next: NextFunction
) {
  // 如果没有 Authorization 头，直接继续
  const authHeader = req.headers.authorization;
  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return next();
  }

  const token = authHeader.split(' ')[1];

  // 检查令牌是否即将过期（5分钟内）
  if (!jwtService.isTokenExpiringSoon(token, 300)) {
    return next();
  }

  // 获取刷新令牌
  // 刷新令牌可以从请求头、Cookie 或 body 中获取
  const refreshToken = req.headers['x-refresh-token'] as string ||
                       req.cookies?.refreshToken ||
                       req.body?.refreshToken;

  if (!refreshToken) {
    // 没有刷新令牌，继续处理（让后续的认证中间件处理）
    return next();
  }

  // 验证刷新令牌
  const payload = jwtService.verifyRefreshToken(refreshToken);

  if (!payload) {
    // 刷新令牌无效，继续处理
    return next();
  }

  // 生成新令牌（这里先存起来，在响应时返回）
  // 实际项目中可以使用响应拦截器来处理
  const newTokens = jwtService.generateTokenPair({
    userId: payload.userId,
    email: req.user?.email || '',
    role: req.user?.role || 'user',
  });

  // 在响应头中返回新令牌
  // 前端需要拦截响应，提取新令牌并更新存储
  res.setHeader('X-New-Access-Token', newTokens.accessToken);
  res.setHeader('X-New-Refresh-Token', newTokens.refreshToken);
  res.setHeader('X-Token-Expiring-Soon', 'true');

  next();
}
```

### 7.3 前端 Token 刷新处理

```typescript
// src/utils/api.ts
import axios, { AxiosInstance, AxiosError } from 'axios';

// 创建 API 实例
const api: AxiosInstance = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3000',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
});

// 正在刷新的标记，防止多个请求同时刷新
let isRefreshing = false;

// 刷新失败时等待的请求队列
let failedQueue: Array<{
  resolve: (token: string) => void;
  reject: (error: Error) => void;
}> = [];

/**
 * 处理令牌刷新
 * 将失败的请求放入队列，等待刷新完成后重试
 */
const processQueue = (error: Error | null, token: string | null = null) => {
  failedQueue.forEach(({ resolve, reject }) => {
    if (error) {
      reject(error);
    } else if (token) {
      resolve(token);
    }
  });
  failedQueue = [];
};

/**
 * 保存令牌到存储
 */
const saveTokens = (accessToken: string, refreshToken: string) => {
  localStorage.setItem('accessToken', accessToken);
  localStorage.setItem('refreshToken', refreshToken);
};

/**
 * 获取存储的令牌
 */
const getTokens = () => {
  return {
    accessToken: localStorage.getItem('accessToken'),
    refreshToken: localStorage.getItem('refreshToken'),
  };
};

/**
 * 清除令牌
 */
const clearTokens = () => {
  localStorage.removeItem('accessToken');
  localStorage.removeItem('refreshToken');
};

// 请求拦截器：添加令牌
api.interceptors.request.use(
  (config) => {
    const { accessToken } = getTokens();

    if (accessToken) {
      // 将令牌添加到请求头
      config.headers.Authorization = `Bearer ${accessToken}`;
    }

    // 添加刷新令牌（用于服务端验证）
    const { refreshToken } = getTokens();
    if (refreshToken) {
      config.headers['X-Refresh-Token'] = refreshToken;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

// 响应拦截器：处理令牌刷新
api.interceptors.response.use(
  (response) => response,

  // 响应错误处理
  async (error: AxiosError) => {
    const originalRequest = error.config as any;

    // 检查是否是401错误（令牌过期）
    if (error.response?.status === 401 && !originalRequest._retry) {
      // 如果正在刷新，等待刷新完成
      if (isRefreshing) {
        return new Promise((resolve, reject) => {
          failedQueue.push({
            resolve: (token: string) => {
              originalRequest.headers.Authorization = `Bearer ${token}`;
              resolve(api(originalRequest));
            },
            reject: (err: Error) => {
              reject(err);
            },
          });
        });
      }

      originalRequest._retry = true;
      isRefreshing = true;

      const { refreshToken } = getTokens();

      // 没有刷新令牌，跳转到登录页
      if (!refreshToken) {
        clearTokens();
        window.location.href = '/login';
        return Promise.reject(error);
      }

      try {
        // 调用刷新接口
        const response = await axios.post(
          `${process.env.NEXT_PUBLIC_API_URL}/auth/refresh-token`,
          { refreshToken }
        );

        const { accessToken, refreshToken: newRefreshToken } = response.data.data;

        // 保存新令牌
        saveTokens(accessToken, newRefreshToken);

        // 处理等待队列
        processQueue(null, accessToken);

        // 重试原始请求
        originalRequest.headers.Authorization = `Bearer ${accessToken}`;
        return api(originalRequest);
      } catch (refreshError) {
        // 刷新失败，处理等待队列
        processQueue(refreshError as Error, null);

        // 清除令牌，跳转到登录页
        clearTokens();
        window.location.href = '/login';
        return Promise.reject(refreshError);
      } finally {
        isRefreshing = false;
      }
    }

    return Promise.reject(error);
  }
);

// 检查响应中的新令牌（中间件自动刷新时会返回新令牌）
api.interceptors.response.use(
  (response) => {
    // 检查响应头中是否有新令牌
    const newAccessToken = response.headers['x-new-access-token'];
    const newRefreshToken = response.headers['x-new-refresh-token'];

    if (newAccessToken && newRefreshToken) {
      // 保存新令牌
      saveTokens(newAccessToken, newRefreshToken);
    }

    return response;
  },
  (error) => Promise.reject(error)
);

export default api;
```

### 8.3 认证服务

```typescript
// src/services/auth.service.ts
import { User } from '../models/user.model';
import { jwtService } from '../utils/jwt';
import { passwordService } from '../utils/password';

/**
 * 认证服务
 * 处理用户注册、登录、登出等认证相关的业务逻辑
 */
class AuthService {
  /**
   * 用户注册
   *
   * @param data - 注册信息
   * @returns 注册结果
   */
  async register(data: {
    name: string;
    email: string;
    password: string;
  }): Promise<{
    success: boolean;
    message: string;
    user?: any;
    tokens?: any;
  }> {
    // 检查邮箱是否已存在
    const existingUser = await User.findOne({
      where: { email: data.email },
    });

    if (existingUser) {
      return {
        success: false,
        message: '该邮箱已被注册',
      };
    }

    // 验证密码强度
    const validation = passwordService.validateStrength(data.password);
    if (!validation.valid) {
      return {
        success: false,
        message: `密码强度不足：${validation.errors.join('、')}`,
      };
    }

    // 创建用户
    const user = await User.create({
      name: data.name,
      email: data.email,
      password: data.password,
      role: 'user',
      status: 'active',
    });

    // 生成令牌
    const tokens = jwtService.generateTokenPair({
      userId: user.id,
      email: user.email,
      role: user.role,
    });

    return {
      success: true,
      message: '注册成功',
      user: user.toJSON(),
      tokens,
    };
  }

  /**
   * 用户登录
   */
  async login(data: { email: string; password: string }): Promise<{
    success: boolean;
    message: string;
    user?: any;
    tokens?: any;
  }> {
    // 查找用户
    const user = await User.findOne({
      where: { email: data.email },
    });

    if (!user) {
      return {
        success: false,
        message: '邮箱或密码错误',
      };
    }

    // 检查账户状态
    if (user.status !== 'active') {
      return {
        success: false,
        message: '账户已被禁用，请联系管理员',
      };
    }

    // 验证密码
    const isValid = await passwordService.compare(data.password, user.password);
    if (!isValid) {
      return {
        success: false,
        message: '邮箱或密码错误',
      };
    }

    // 生成令牌
    const tokens = jwtService.generateTokenPair({
      userId: user.id,
      email: user.email,
      role: user.role,
    });

    return {
      success: true,
      message: '登录成功',
      user: user.toJSON(),
      tokens,
    };
  }

  /**
   * 刷新令牌
   */
  async refreshToken(refreshToken: string): Promise<{
    success: boolean;
    message: string;
    tokens?: any;
  }> {
    // 验证刷新令牌
    const payload = jwtService.verifyRefreshToken(refreshToken);
    if (!payload) {
      return {
        success: false,
        message: '刷新令牌无效或已过期',
      };
    }

    // 查找用户
    const user = await User.findByPk(payload.userId);
    if (!user || user.status !== 'active') {
      return {
        success: false,
        message: '用户不存在或已被禁用',
      };
    }

    // 生成新令牌
    const tokens = jwtService.generateTokenPair({
      userId: user.id,
      email: user.email,
      role: user.role,
    });

    return {
      success: true,
      message: '令牌刷新成功',
      tokens,
    };
  }

  /**
   * 获取当前用户信息
   */
  async getCurrentUser(userId: string): Promise<any> {
    const user = await User.findByPk(userId, {
      attributes: { exclude: ['password'] },
    });

    return user;
  }

  /**
   * 修改密码
   */
  async changePassword(
    userId: string,
    currentPassword: string,
    newPassword: string
  ): Promise<{ success: boolean; message: string }> {
    // 获取用户
    const user = await User.findByPk(userId);
    if (!user) {
      return {
        success: false,
        message: '用户不存在',
      };
    }

    // 验证当前密码
    const isValid = await passwordService.compare(currentPassword, user.password);
    if (!isValid) {
      return {
        success: false,
        message: '当前密码错误',
      };
    }

    // 验证新密码强度
    const validation = passwordService.validateStrength(newPassword);
    if (!validation.valid) {
      return {
        success: false,
        message: `新密码强度不足：${validation.errors.join('、')}`,
      };
    }

    // 更新密码
    user.password = newPassword;
    await user.save();

    return {
      success: true,
      message: '密码修改成功',
    };
  }
}

export const authService = new AuthService();
```

```typescript
// src/services/article.service.ts
import { Article } from '../models/article.model';

/**

class ArticleService {

   * 创建文章
   */
  async create(
    authorId: string,
    data: {
      title: string;
      content: string;
      slug?: string;

    }
  ): Promise<Article> {

    // 创建文章

      ...data,
      slug,
      authorId,
      status: 'draft',
    });

    return article;
  }

  /**
   * 获取文章列表
   */
  async findAll(options: {
    page?: number;
    pageSize?: number;
    status?: string;
    authorId?: string;
    keyword?: string;
  }): Promise<{ rows: Article[]; count: number }> {
    const page = options.page || 1;
    const pageSize = options.pageSize || 10;
    const offset = (page - 1) * pageSize;

    // 构建查询条件
    const where: any = {};

    // 只有已发布的文章才能被所有人看到
    if (options.status) {
      where.status = options.status;
    } else {
      // 默认只显示已发布的文章
      where.status = 'published';
    }

    if (options.authorId) {
      where.authorId = options.authorId;
    }

    if (options.keyword) {
      where[Op.or] = [
        { title: { [Op.like]: `%${options.keyword}%` } },
        { content: { [Op.like]: `%${options.keyword}%` } },
      ];
    }

    const { rows, count } = await Article.findAndCountAll({
      where,
      include: [
        {
          model: User,
          as: 'author',
          attributes: ['id', 'name', 'email'],
        },
      ],
      order: [['createdAt', 'DESC']],
      limit: pageSize,
      offset,
    });

    return { rows, count };
  }

  /**
   * 根据 ID 获取文章
   */
  async findById(id: string): Promise<Article | null> {
    return Article.findByPk(id, {
      include: [
        {
          model: User,
          as: 'author',
          attributes: ['id', 'name', 'email'],
        },
      ],
    });
  }

  /**
   * 根据 slug 获取文章
   */
  async findBySlug(slug: string): Promise<Article | null> {
    return Article.findOne({
      where: { slug },
      include: [
        {
          model: User,
          as: 'author',
          attributes: ['id', 'name', 'email'],
        },
      ],
    });
  }

  /**
   * 更新文章
   * 需要检查权限：作者可以更新自己的文章，编辑可以更新任何文章
   */
  async update(
    id: string,
    userId: string,
    userRole: string,
    data: Partial<{
      title: string;
      content: string;
      slug: string;
      excerpt: string;
      coverImage: string;
      status: string;
    }>
  ): Promise<{ success: boolean; message: string; article?: Article }> {
    // 获取文章
    const article = await Article.findByPk(id);
    if (!article) {
      return {
        success: false,
        message: '文章不存在',
      };
    }

    // 检查权限：作者本人或编辑/管理员可以更新
    const isOwner = article.authorId === userId;
    const isEditorOrAbove = rbacService.isRoleHigherOrEqual(userRole, 'editor');

    if (!isOwner && !isEditorOrAbove) {
      return {
        success: false,
        message: '您没有权限更新这篇文章',
      };
    }

    // 更新文章
    await article.update(data);

    return {
      success: true,
      message: '文章更新成功',
      article,
    };
  }

  /**
   * 删除文章
   * 需要检查权限：作者可以删除自己的文章，编辑/管理员可以删除任何文章
   */
  async delete(
    id: string,
    userId: string,
    userRole: string
  ): Promise<{ success: boolean; message: string }> {
    const article = await Article.findByPk(id);
    if (!article) {
      return {
        success: false,
        message: '文章不存在',
      };
    }

    // 检查权限
    const isOwner = article.authorId === userId;
    const isEditorOrAbove = rbacService.isRoleHigherOrEqual(userRole, 'editor');

    if (!isOwner && !isEditorOrAbove) {
      return {
        success: false,
        message: '您没有权限删除这篇文章',
      };
    }

    // 删除文章
    await article.destroy();

    return {
      success: true,
      message: '文章删除成功',
    };
  }

  /**
   * 发布文章
   */
  async publish(
    id: string,
    userId: string,
    userRole: string
  ): Promise<{ success: boolean; message: string; article?: Article }> {
    const result = await this.update(id, userId, userRole, {
      status: 'published',
      publishedAt: new Date(),
    });

    return result;
  }

  /**
   * 生成 URL 友好的 slug
   */
  private generateSlug(title: string): string {
    // 转小写，替换空格为连字符
    let slug = title
      .toLowerCase()
      .replace(/[^a-z0-9\u4e00-\u9fa5]+/g, '-')
      .replace(/^-|-$/g, '');

    // 添加随机后缀确保唯一
    const suffix = Math.random().toString(36).substring(2, 8);
    return `${slug}-${suffix}`;
  }
```

## 十、安全最佳实践总结

### 10.1 密码安全

```
1. 永远不要明文存储密码
2. 使用 bcrypt 或 Argon2 加密
3. 使用足够的 salt rounds（bcrypt 建议 12）
4. 验证密码强度
5. 强制要求复杂密码
6. 实现登录失败锁定
7. 敏感操作要求二次验证
```

### 10.2 令牌安全

```
1. 使用 HTTPS
2. 设置短期访问令牌（15分钟内）
3. 使用长期刷新令牌（7天）
4. 令牌存储在安全位置
5. 实现令牌黑名单
6. 定期轮换密钥
7. 记录令牌使用日志
```

### 10.3 传输安全

```
1. 始终使用 HTTPS
2. 使用 HttpOnly Cookie 防止 XSS
3. 使用 SameSite Cookie 防止 CSRF
4. 不要在 URL 中传递敏感信息
5. 实现 CSRF 令牌
```

### 10.4 访问控制

```
1. 默认拒绝：未明确允许的都拒绝
2. 最小权限：只授予必要的权限
```
