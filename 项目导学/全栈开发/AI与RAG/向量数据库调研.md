# 向量数据库调研报告

> 调研日期：2026年3月22日

## 一、概述

向量数据库是一类专门用于存储和检索高维向量（Embeddings）的数据库系统，在 AI 应用中扮演着核心角色，尤其适用于：

- **RAG（检索增强生成）**：为 LLM 提供外部知识库
- **语义搜索**：基于语义相似度的全文检索
- **图像/视频检索**：以图搜图、以视频搜视频
- **推荐系统**：基于向量相似度的个性化推荐
- **异常检测**：识别与正常模式显著偏离的数据

## 三、技术选型建议

### 3.1 按规模选择

| 规模 | 推荐方案 | 理由 |
|------|----------|------|
| **个人项目 / 原型** | ChromaDB / sqlite-vec | 零部署，集成简单 |
| **中小企业（百万级）** | Qdrant / Weaviate | 平衡性能与易用性 |
| **大型企业（千万+级）** | Milvus | 分布式架构，成熟生态 |
| **超大规模（亿级）** | Milvus + FAISS | 云原生 + 算法深度优化 |

### 3.2 按场景选择

| 场景 | 推荐方案 |
|------|----------|
| **RAG 应用** | ChromaDB（简单）或 Qdrant（生产） |
| **混合搜索（语义+关键词）** | Weaviate |
| **图 + 向量融合** | Kùzu |
| **需要本地/边缘部署** | sqlite-vec |
| **已有 PostgreSQL，想扩展向量** | pgvector（补充方案）|

### 3.3 技术栈集成建议

| 你的技术栈 | 推荐向量数据库 |
|------------|---------------|
| **Python / LangChain / LlamaIndex** | ChromaDB、Milvus、Qdrant |
| **Node.js / TypeScript** | Qdrant（gRPC）、Weaviate |

| **已有关系型数据库** | pgvector（PostgreSQL 扩展）|
| **嵌入式 / 移动端** | sqlite-vec |

## 四、pgvector 补充方案

除专门的向量数据库外，**pgvector** 作为 PostgreSQL 扩展值得特别关注：

| 特性 | 说明 |
|------|------|
| **集成成本** | 零，几乎所有 PostgreSQL 项目可直接使用 |
| **向量维度** | 支持最高 16,000 维度 |
| **索引类型** | IVFFlat、HNSW |
| **查询方式** | SQL，可与业务数据 join |
| **适用规模** | 百万级向量以内效果最佳 |

```sql
-- pgvector 示例
CREATE EXTENSION vector;
CREATE TABLE documents (
  id SERIAL PRIMARY KEY,
  content TEXT,
  embedding vector(1536)
);

-- 相似性检索
SELECT content, (embedding <=> $1) AS distance
FROM documents
ORDER BY embedding <=> $1
LIMIT 5;
```

## 五、趋势与展望

1. **云原生化**：Milvus Cloud、Qdrant Cloud 等全托管服务日益成熟
2. **混合架构**：向量数据库与传统数据库融合（如 Kùzu 的图+向量、Weaviate 的全文+向量）
3. **边缘计算**：sqlite-vec 等轻量方案支持端侧部署
4. **AI 原生**：专门为 LLM/RAG 场景优化的数据库（如 ChromaDB）
5. **多模态支持**：未来将支持图像、视频、音频等多种向量化数据的统一检索
