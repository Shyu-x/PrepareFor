## 1. 语义化标签

### 1.1 什么是语义化标签

语义化标签是 HTML5 引入的一种新的标签体系，它能够清晰地表达文档的结构和内容的含义。与传统的 `<div>` 和 `<span>` 这种无语义标签不同，语义化标签能够让浏览器、搜索引擎和开发者更清晰地理解网页内容的结构和意义。

**为什么需要语义化标签？**

在 HTML5 之前，开发者通常使用大量的 `<div>` 标签来构建页面结构，导致代码难以阅读和维护。语义化标签的出现解决了这个问题，它们具有明确的语义含义：

| 标签 | 语义含义 | 使用场景 |
|------|----------|----------|
| `<header>` | 页面或区块的头部 | 页面顶部导航、标题区域 |
| `<nav>` | 导航链接区域 | 主导航菜单、面包屑导航 |
| `<main>` | 文档的主要内容 | 文章主体、内容区域 |
| `<article>` | 独立的文章内容 | 博客文章、新闻报道 |
| `<section>` | 文档中的章节 | 页面分区、主题分组 |
| `<aside>` | 与主内容相关但可独立的侧边栏 | 侧边栏、广告区域 |
| `<footer>` | 页面或区块的底部 | 版权信息、联系方式 |

### 1.2 文档结构标签详解

#### header 标签

`<header>` 标签用于定义页面或区块的头部区域，通常包含标题 logo、导航链接或搜索框等元素。一个页面可以有多个 `<header>` 标签，分别用于不同的区块。

```html
<!-- 页面级别的 header -->
<header>
    <h1>网站标题</h1>
    <nav>
        <ul>
            <li><a href="/">首页</a></li>
            <li><a href="/about">关于</a></li>
            <li><a href="/contact">联系</a></li>
        </ul>
    </nav>
</header>

<!-- 文章级别的 header -->
<article>
    <header>
        <h1>文章标题</h1>
        <p>发布时间：2024-01-15</p>
        <p>作者：张三</p>
    </header>
    <p>文章内容...</p>
</article>
```

**最佳实践**：
- 一个页面可以有多个 `<header>` 标签，但不应嵌套在 `<footer>`、`<address>` 或另一个 `<header>` 中
- `<header>` 通常包含标题元素（h1-h6）或 hgroup
- 保持语义清晰，不要将所有内容都放在 header 中

#### nav 标签

`<nav>` 标签用于定义导航链接区域，专门用于放置主要的导航链接。一个页面可以包含多个 `<nav>` 标签，分别用于主导航和侧边栏导航等。

```html
<!-- 主导航 -->
<nav aria-label="主导航">
    <ul>
        <li><a href="/">首页</a></li>
        <li><a href="/products">产品</a></li>
        <li><a href="/services">服务</a></li>
        <li><a href="/blog">博客</a></li>
    </ul>
</nav>

<!-- 面包屑导航 -->
<nav aria-label="面包屑导航">
    <ol>
        <li><a href="/">首页</a></li>
        <li><a href="/products">产品</a></li>
        <li><a href="/products/electronics">电子产品</a></li>
        <li>智能手机</li>
    </ol>
</nav>

<!-- 侧边栏导航 -->
<aside>
    <nav aria-label="侧边栏导航">
        <h2>相关链接</h2>
        <ul>
            <li><a href="#">最新文章</a></li>
            <li><a href="#">热门文章</a></li>
            <li><a href="#">推荐阅读</a></li>
        </ul>
    </nav>
</aside>
```

**最佳实践**：
- 并非所有链接组都需要使用 `<nav>` 标签，只有主要的导航区域才应该使用
- 使用 `aria-label` 属性为导航添加描述性标签，提高可访问性
- 搜索引擎会优先抓取 `<nav>` 中的链接

#### main 标签

`<main>` 标签用于定义文档的主要内容区域。一个页面应该只有一个 `<main>` 标签，且不应包含在 `<article>`、`aside>`、`<footer>`、`<header>` 或 `<nav>` 中。

```html
<body>
    <header>
        <h1>我的博客</h1>
    </header>

    <main>
        <article>
            <h2>文章标题</h2>
            <p>文章内容...</p>
        </article>

        <article>
            <h2>另一篇文章</h2>
            <p>内容...</p>
        </article>
    </main>

    <footer>
        <p>&copy; 2024 我的博客</p>
    </footer>
</body>
```

**最佳实践**：
- 每个页面应该只有一个 `<main>` 标签
- `<main>` 的内容应该是页面的核心功能，不要将导航、侧边栏等内容放进去
- 搜索引擎会优先索引 `<main>` 中的内容

#### article 标签

`<article>` 标签用于定义独立的、完整的内容单元，如博客文章、新闻报道、论坛帖子等。每个 `<article>` 应该能够独立存在和分发。

```html
<article>
    <header>
        <h1>JavaScript闭包详解</h1>
        <time datetime="2024-01-15">2024年1月15日</time>
    </header>

    <section>
        <h2>什么是闭包</h2>
        <p>闭包是指...</p>
    </section>

    <section>
        <h2>闭包的用途</h2>
        <p>闭包的主要用途包括...</p>
    </section>

    <footer>
        <p>作者：张三</p>
        <p>标签：JavaScript</p>
    </footer>
</article>

<!-- article 嵌套使用：评论是文章的一部分 -->
<article>
    <h1>文章标题</h1>
    <p>文章内容...</p>

    <section class="comments">
        <h2>评论</h2>
        <article>
            <p>评论内容...</p>
            <footer>
                <p>评论者：李四</p>
            </footer>
        </article>
    </section>
</article>
```

**最佳实践**：
- `<article>` 应该是独立的、可分发的内容
- 如果内容脱离当前页面仍然有意义，就适合使用 `<article>`
- 可以在 `<article>` 内部嵌套 `<article>`，如评论列表中的单条评论

#### section 标签

`<section>` 标签用于将文档分成不同的主题区域或章节。通常与标题（h1-h6）配合使用，表示文档的一个逻辑部分。

```html
<!-- 文档章节划分 -->
<article>
    <h1>JavaScript 教程</h1>

    <section>
        <h2>第一章：基础知识</h2>
        <p>本章介绍 JavaScript 的基本语法...</p>
    </section>

    <section>
        <h2>第二章：数据类型</h2>
        <p>本章介绍 JavaScript 的数据类型...</p>
    </section>

    <section>
        <h2>第三章：函数</h2>
        <p>本章介绍 JavaScript 的函数...</p>
    </section>
</article>

<!-- 功能区块划分 -->
<section>
    <h2>搜索功能</h2>
    <form action="/search" method="get">
        <input type="search" name="q" placeholder="搜索...">
        <button type="submit">搜索</button>
    </form>
</section>
```

**最佳实践**：
- `<section>` 通常应该包含一个标题（h1-h6）
- 不要将 `<section>` 仅用于添加样式，如果只是需要一个容器，使用 `<div>`
- 如果内容可以独立存在并分发，使用 `<article>` 而不是 `<section>`

#### aside 标签

`<aside>` 标签用于定义与主内容相关但可以独立的辅助信息，如侧边栏、广告区域、相关链接等。

```html
<article>
    <h1>JavaScript 闭包</h1>
    <p>闭包是 JavaScript 中一个重要的概念...</p>

    <aside>
        <h3>相关概念</h3>
        <ul>
            <li><a href="#">作用域</a></li>
            <li><a href="#">原型链</a></li>
            <li><a href="#">this 绑定</a></li>
        </ul>
    </aside>
</article>

<!-- 页面级别的 aside -->
<body>
    <main>
        <p>主要内容...</p>
    </main>

    <aside>
        <h2>热门文章</h2>
        <ul>
            <li><a href="#">文章1</a></li>
            <li><a href="#">文章2</a></li>
        </ul>
    </aside>
</body>
```

**最佳实践**：
- `<aside>` 的内容应该与主内容相关
- 不要将无关内容放在 `<aside>` 中
- `<aside>` 可以放在 `<article>` 内部或外部

#### footer 标签

`<footer>` 标签用于定义页面或区块的底部区域，通常包含版权信息、作者联系方式、相关文档链接等。

```html
<!-- 页面级别的 footer -->
<footer>
    <p>&copy; 2024 公司名称. 保留所有权利.</p>
    <nav>
        <a href="/privacy">隐私政策</a>
        <a href="/terms">服务条款</a>
        <a href="/contact">联系我们</a>
    </nav>
</footer>

<!-- article 的 footer -->
<article>
    <h1>文章标题</h1>
    <p>文章内容...</p>

    <footer>
        <p>作者：张三</p>
        <address>
            <a href="mailto:zhangsan@example.com">zhangsan@example.com</a>
        </address>
    </footer>
</article>

<!-- section 的 footer -->
<section>
    <h2>读者评论</h2>
    <p>评论列表...</p>

    <footer>
        <button>加载更多评论</button>
    </footer>
</section>
```

### 1.3 其他语义化标签

#### figure 和 figcaption

`<figure>` 标签用于定义独立的流内容（如图像、图表、代码块），`<figcaption>` 用于定义 `<figure>` 的标题。

```html
<!-- 图片示例 -->
<figure>
    <img src="chart.png" alt="销售趋势图">
    <figcaption>图1：2023年季度销售趋势</figcaption>
</figure>

<!-- 代码块示例 -->
<figure>
    <figcaption>JavaScript 闭包示例代码</figcaption>
    <pre><code>
function createCounter() {
    let count = 0;
    return function() {
        return ++count;
    };
}
const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
    </code></pre>
</figure>

<!-- 引用示例 -->
<figure>
    <blockquote>
        <p>代码是写给人看的，顺便能在机器上运行。</p>
    </blockquote>
    <figcaption>—— Donald Knuth</figcaption>
</figure>
```

#### time 标签

`<time>` 标签用于定义机器可读的时间和日期。

```html
<!-- 日期 -->
<p>发布日期：<time datetime="2024-01-15">2024年1月15日</time></p>

<!-- 时间和时区 -->
<p>会议时间：<time datetime="2024-01-15T14:30:00+08:00">2024年1月15日 14:30</time></p>

<!-- 持续时间 -->
<p>视频时长：<time datetime="PT1H30M">1小时30分钟</time></p>

<!-- 相对时间（配合 JavaScript 使用） -->
<time class="relative-time" datetime="2024-01-15T10:00:00">
    2024年1月15日 10:00
</time>
```

#### mark 标签

`<mark>` 标签用于标记文本中的重点或高亮内容。

```html
<p>搜索结果：找到 <mark>JavaScript</mark> 相关文章 15 篇。</p>

<p>在学习 <mark>闭包</mark> 之前，需要先了解作用域的概念。</p>
```

#### progress 和 meter

`<progress>` 标签用于显示任务的进度，`<meter>` 标签用于显示已知范围内的标量值。

```html
<!-- 进度条 -->
<progress value="70" max="100">70%</progress>
<progress indeterminate>加载中...</progress>

<!-- 计量器 -->
<meter value="2" min="0" max="10">2 out of 10</meter>
<meter value="0.6">60%</meter>

<!-- 实际应用场景 -->
<label for="file">下载进度：</label>
<progress id="file" value="45" max="100">45%</progress>

<label for="disk">磁盘使用：</label>
<meter id="disk" value="0.8" low="0.2" high="0.8" optimum="0.5">80%</meter>
```

#### details 和 summary

`<details>` 标签用于创建一个可展开/收起的 Widget，`<summary>` 用于定义 `<details>` 的可见标题。

```html
<details>
    <summary>点击展开更多选项</summary>
    <p>这里是隐藏的详细内容...</p>
    <ul>
        <li>选项1</li>
        <li>选项2</li>
        <li>选项3</li>
    </ul>
</details>

<!-- 默认展开 -->
<details open>
    <summary>默认展开的说明</summary>
    <p>这部分内容默认是可见的。</p>
</details>

<!-- 常见使用场景：FAQ -->
<details>
    <summary>什么是闭包？</summary>
    <p>闭包是指...</p>
</details>

<details>
    <summary>闭包有什么用途？</summary>
    <p>闭包的主要用途包括...</p>
</details>
```

#### wbr 标签

`<wbr>` 标签用于指定在长单词或 URL 中的换行位置。

```html
<p>学习地址：https://developer.mozilla.org/<wbr>zh-CN/<wbr>docs/<wbr>Web/<wbr>HTML</p>

<p>SuperLongWordCanBreakAtAnyPointButMostlyAtW<wbr>ill.</p>
```

### 1.5 常见面试问题

**问题1：语义化标签有什么好处？**

答案：
1. **搜索引擎优化（SEO）**：搜索引擎能够更好地理解页面结构，提高内容索引的准确性
2. **可访问性（Accessibility）**：屏幕阅读器能够更准确地识别内容区域，帮助视障用户理解页面
3. **代码可读性和可维护性**：开发者能够更清晰地理解页面结构，代码更容易维护
4. **浏览器支持**：现代浏览器能够为语义化标签提供更好的默认样式

**问题2：div、section、article 有什么区别？**

答案：
- `<div>`：无语义容器，仅用于样式和脚本
- `<section>`：文档中的独立章节，通常需要包含标题，表示文档的一个逻辑部分
- `<article>`：独立的、可分发的内容单元，如博客文章、新闻报道，可以脱离页面独立存在

**问题3：什么时候使用 section 而不是 div？**

答案：当内容需要一个明确的标题（h1-h6）时，应该使用 `<section>`。如果只是需要一个容器来添加样式或脚本，应该使用 `<div>`。

## 2. 表单验证

### 2.1 HTML5 原生表单验证

HTML5 引入了强大的原生表单验证功能，无需编写 JavaScript 即可实现基本的客户端验证。

#### 验证属性

```html
<form novalidate>
    <!-- required：必填字段 -->
    <label>用户名：<input type="text" required></label>

    <!-- minlength/maxlength：字符长度限制 -->
    <label>密码：<input type="password" minlength="6" maxlength="20"></label>

    <!-- min/max：数值范围 -->
    <label>年龄：<input type="number" min="18" max="100"></label>

    <!-- pattern：正则表达式验证 -->
    <label>手机号：<input type="tel" pattern="1[3-9]\d{9}"></label>

    <!-- type：类型自动验证 -->
    <label>邮箱：<input type="email"></label>
    <label>网址：<input type="url"></label>

    <!-- step：数值步长 -->
    <label>价格：<input type="number" min="0" step="0.01"></label>

    <!-- 自定义验证消息 -->
    <input type="text" id="username" required
           oninvalid="this.setCustomValidity('用户名不能为空')"
           oninput="this.setCustomValidity('')">

    <button type="submit">提交</button>
</form>
```

#### 表单验证伪类

CSS 提供了多个表单验证相关的伪类，可以用于自定义验证状态的样式：

```css
/* 输入有效的样式 */
input:valid {
    border-color: green;
}

/* 输入无效的样式 */
input:invalid {
    border-color: red;
}

/* 必填字段的样式 */
input:required {
    border-left: 3px solid orange;
}

/* 可选字段的样式 */
input:optional {
    border-left: 3px solid gray;
}

/* 在用户输入后验证 */
input:user-invalid {
    background-color: #fff0f0;
}

/* 范围内/外的值 */
input:in-range {
    background-color: lightgreen;
}

input:out-of-range {
    background-color: lightcoral;
}
```

```html
        }

        .form-group {
            margin-bottom: 15px;
        }

        label {
            display: block;
            margin-bottom: 5px;
            font-weight: bold;
        }

        input, select, textarea {
```

### 2.3 自定义表单验证组件

```javascript
// 创建自定义验证器
class FormValidator {
    constructor(formId, rules) {
        this.form = document.getElementById(formId);
        this.rules = rules;
        this.init();
    }

    init() {
        // 为每个规则绑定验证事件
        for (const [fieldName, fieldRules] of Object.entries(this.rules)) {
            const input = this.form.querySelector(`[name="${fieldName}"]`);
            if (!input) continue;

            // blur 事件验证
            input.addEventListener('blur', () => this.validateField(fieldName));

            // input 事件清除错误
            input.addEventListener('input', () => {
                if (input.validity.valid) {
                    this.clearError(fieldName);
                }
            });
        }

        // 表单提交验证
        this.form.addEventListener('submit', (e) => {
            if (!this.validateAll()) {
                e.preventDefault();
            }
        });
    }

    validateField(fieldName) {
        const input = this.form.querySelector(`[name="${fieldName}"]`);
        const fieldRules = this.rules[fieldName];
        let errorMessage = '';

        for (const rule of fieldRules) {
            const result = rule.validate(input.value, this.form);
            if (!result.valid) {
                errorMessage = result.message;
                break;
            }
        }

        if (errorMessage) {
            this.showError(fieldName, errorMessage);
            return false;
        } else {
            this.clearError(fieldName);
            return true;
        }
    }

    validateAll() {
        let isValid = true;
        for (const fieldName of Object.keys(this.rules)) {
            if (!this.validateField(fieldName)) {
                isValid = false;
            }
        }
        return isValid;
    }

    showError(fieldName, message) {
        const input = this.form.querySelector(`[name="${fieldName}"]`);
        let errorEl = input.parentElement.querySelector('.error-text');

        if (!errorEl) {
            errorEl = document.createElement('span');
            errorEl.className = 'error-text';
            input.parentElement.appendChild(errorEl);
        }

        errorEl.textContent = message;
        input.classList.add('is-invalid');
    }

    clearError(fieldName) {
        const input = this.form.querySelector(`[name="${fieldName}"]`);
        const errorEl = input.parentElement.querySelector('.error-text');

        if (errorEl) {
            errorEl.remove();
        }
        input.classList.remove('is-invalid');
    }
}

// 定义验证规则
const validator = new FormValidator('registerForm', {
    username: [
        {

                valid: value.length >= 3,
                message: '用户名至少需要3个字符'

            })
        }
    ],
    email: [
        {
            validate: (value) => ({
                valid: /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value),
                message: '请输入有效的邮箱地址'
            })
        }
    ],
    password: [

            validate: (value) => ({

                message: '密码至少需要8个字符'
            })
        },
```

### 2.4 常见面试问题

**问题1：HTML5 原生表单验证有哪些属性？**

答案：
- `required`：必填字段
- `minlength` / `maxlength`：字符长度限制
- `min` / `max`：数值范围限制
- `pattern`：正则表达式验证
- `type`：自动类型验证（email、url、tel 等）
- `step`：数值步长
- `novalidate`：禁用表单验证

**问题2：如何自定义表单验证消息？**

答案：
1. 使用 `oninvalid` 和 `oninput` 事件
2. 使用 `setCustomValidity()` 方法
3. 使用 Constraint Validation API 的 `validationMessage` 属性
4. 使用 CSS 伪类 `:valid` 和 `:invalid`

**问题3：表单验证应该在客户端还是服务端进行？**

答案：客户端验证主要用于提升用户体验，服务端验证是数据安全的最后保障。两者必须同时进行：
- 客户端验证：减少服务器负载，提升用户体验
- 服务端验证：防止恶意请求，确保数据安全

```javascript
// 1. Object.assign polyfill
if (typeof Object.assign !== 'function') {
    Object.assign = function(target, ...sources) {
        if (target === null || target === undefined) {
            throw new TypeError('Cannot convert undefined or null to object');
        }

        const to = Object(target);

        for (const source of sources) {
            if (source !== null && source !== undefined) {
                for (const key of Object.keys(source)) {
                    to[key] = source[key];
                }
            }
        }

        return to;
    };
}

// 2. Array.prototype.includes polyfill
if (!Array.prototype.includes) {
    Array.prototype.includes = function(searchElement, fromIndex) {
        const O = Object(this);
        const len = Math.floor(O.length) || 0;

        if (len === 0) return false;

        const n = fromIndex === undefined ? 0 : Math.floor(fromIndex);
        let k = n >= 0 ? n : len + n;

        while (k < len) {
            if (O[k] === searchElement) return true;
            k++;
        }

        return false;
    };
}

// 3. String.prototype.startsWith polyfill
if (!String.prototype.startsWith) {
    String.prototype.startsWith = function(searchString, position) {
        return this.substr(position || 0, searchString.length) === searchString;
    };
}

// 4. Promise.allSettled polyfill
if (!Promise.allSettled) {
    Promise.allSettled = function(promises) {
        return Promise.all(promises.map(p => {
            return Promise.resolve(p).then(
                value => ({ status: 'fulfilled', value }),
                reason => ({ status: 'rejected', reason })
            );
        }));
    };
}

// 5. Array.prototype.flat polyfill
if (!Array.prototype.flat) {
    Array.prototype.flat = function(depth = 1) {
        const arr = [];
        const flatten = (arr, depth) => {
            for (const item of arr) {
                if (Array.isArray(item) && depth > 0) {
                    flatten(item, depth - 1);
                } else {
                    arr.push(item);
                }
            }
        };
        flatten(this, depth);
        return arr;
    };
}
```

### 3.6 常见面试问题

**问题1：什么是 polyfill？**

答案：Polyfill 是一段 JavaScript 代码，用于在不支持新特性的旧浏览器中模拟这些特性的功能。例如，使用 JavaScript 在旧浏览器中实现 `Promise` 或 `fetch` 的功能。

**问题2：渐进增强和优雅降级有什么区别？**

答案：
- **渐进增强**：先为所有浏览器提供基础功能，然后为支持新特性的浏览器添加增强功能
- **优雅降级**：先为现代浏览器实现完整功能，然后为旧浏览器提供降级方案

**问题3：如何检测浏览器是否支持某个 CSS 特性？**

答案：
1. 使用 `@supports` 规则
2. 使用 JavaScript 的 `CSS.supports()` 方法
3. 使用 Modernizr 库进行特性检测

### 4.4 常见面试问题

**问题1：如何优化网页的 SEO？**

答案：
1. **内容优化**：使用语义化标签、高质量内容、合理关键词密度
2. **结构优化**：合理的标题层级、清晰的 URL 结构、内部链接
3. **技术优化**：页面加载速度、移动端适配、HTTPS
4. **Meta 标签**：title、description、Open Graph
5. **结构化数据**：JSON-LD 标记
6. **图片优化**：alt 属性、压缩、懒加载

**问题2：什么是 ARIA？它与 HTML 语义化标签有什么区别？**

答案：
- ARIA（Accessible Rich Internet Applications）是一套用于增强 Web 应用可访问性的属性
- HTML 语义化标签本身具有语义含义，而 ARIA 只是添加额外的语义信息
- 两者可以配合使用，但应优先使用原生 HTML 语义化标签

**问题3：如何实现键盘可访问的组件？**

答案：
1. 使用语义化 HTML 元素（button、a 等）
2. 添加适当的 tabindex
3. 实现焦点管理（focus/blur）
4. 监听键盘事件（Enter、Space、ESC）
5. 使用 ARIA 属性提供状态信息
6. 确保焦点可见性

### 7.1 常见陷阱

#### 陷阱1：过度使用 div


## 9. 面试高频问题汇总

### 9.1 基础问题

**问题1：HTML5 新增了哪些语义化标签？**

答案：
- 结构标签：`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`
- 文本标签：`<figure>`, `<figcaption>`, `<time>`, `<mark>`, `<wbr>`
- 交互标签：`<details>`, `<summary>`, `<dialog>`
- 表单标签：`<datalist>`, `<output>`, `<progress>`, `<meter>`
- 多媒体标签：`<audio>`, `<video>`, `<source>`, `<track>`, `<embed>`

**问题2：section 和 article 有什么区别？**

答案：
- `<section>`：用于对页面内容进行分区，通常包含一个标题，表示文档的一个逻辑部分
- `<article>`：用于独立的、可分发的内容单元，如博客文章、新闻报道，可以脱离页面独立存在

判断标准：如果内容可以单独发布（如 RSS 订阅），使用 `<article>`；否则使用 `<section>`

**问题3：什么是渐进增强和优雅降级？**

答案：
- **渐进增强**：先为所有浏览器提供基础功能，然后为支持新特性的浏览器添加增强功能
- **优雅降级**：先为现代浏览器实现完整功能，然后为旧浏览器提供降级方案

推荐使用渐进增强，因为它确保所有用户都能获得基本功能。

### 9.2 进阶问题

**问题4：如何优化图片加载性能？**

答案：
1. 使用现代图片格式（WebP、AVIF）
2. 实现懒加载（loading="lazy"）
3. 使用响应式图片（srcset、sizes）
4. 添加图片尺寸属性避免布局偏移
5. 使用 CDN 加速图片分发
6. 压缩图片质量
7. 使用 CSS Sprites 或图标字体

**问题5：什么是 Core Web Vitals？如何优化？**

答案：
Core Web Vitals 是 Google 定义的核心网页指标：

| 指标 | 含义 | 目标值 | 优化方法 |
|------|------|--------|----------|
| LCP | 最大内容绘制时间 | < 2.5s | 优化服务器响应、预加载关键资源、优化图片 |
| FID | 首次输入延迟 | < 100ms | 减少 JavaScript 执行时间、使用 Web Worker |
| CLS | 累积布局偏移 | < 0.1 | 设置图片尺寸、预留广告位、避免动态插入内容 |

**问题6：如何实现一个可访问的模态框？**

答案：
1. 使用 `role="dialog"` 和 `aria-modal="true"`
2. 添加 `aria-labelledby` 和 `aria-describedby`
3. 打开时聚焦到模态框
4. 实现焦点陷阱（Tab 键只在模态框内循环）
5. ESC 键关闭模态框
6. 关闭时恢复之前的焦点
7. 禁止背景滚动

### 9.3 实战问题

**问题7：如何为 SPA 应用做 SEO 优化？**

答案：
1. 使用服务端渲染（SSR）或静态站点生成（SSG）
2. 实现预渲染（Prerender）
3. 使用动态渲染（检测爬虫返回预渲染页面）
4. 正确设置 meta 标签和结构化数据
5. 实现合理的路由和导航结构
6. 提交 sitemap.xml

**问题8：如何处理 HTML5 新特性的浏览器兼容性？**

答案：
1. 使用特性检测（Modernizr、@supports）
2. 加载 Polyfill（core-js、polyfill.io）
3. 使用渐进增强策略
4. 添加浏览器前缀（Autoprefixer）
5. 设置合理的浏览器支持范围
6. 使用 Babel 转译 JavaScript

## 11. Web Components 入门

### 11.1 什么是 Web Components

Web Components 是一套用于创建可复用自定义元素的 Web 技术，它允许开发者创建封装的、可重用的 HTML 组件。Web Components 由三个主要技术组成：

| 技术 | 说明 |
|------|------|
| **Custom Elements** | 自定义元素，允许创建新的 HTML 标签 |
| **Shadow DOM** | 影子 DOM，提供样式和脚本的封装 |
| **HTML Templates** | HTML 模板，定义可复用的 HTML 结构 |

### 11.4 Shadow DOM 样式封装

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>Shadow DOM 样式封装</title>
    <style>
        /* 全局样式 */
        p {
            color: red !important; /* 即使使用 !important 也无法穿透 Shadow DOM */
        }
    </style>
</head>
<body>
    <!-- 全局段落 - 红色 -->
    <p>这是全局段落，应该是红色的</p>

    <!-- Shadow DOM 内部段落 - 不受全局样式影响 -->
    <shadow-demo></shadow-demo>

    <script>
        class ShadowDemo extends HTMLElement {
            constructor() {
                super();
                
                // open 模式：外部可以访问 shadowRoot
                // closed 模式：外部无法访问 shadowRoot
                this.attachShadow({ mode: 'open' });
                
                this.shadowRoot.innerHTML = `
                    <style>
                        /* Shadow DOM 内部样式 */
                        :host {
                            display: block;
                            padding: 20px;
                            background: #f0f0f0;
                            border-radius: 8px;
                            margin: 10px 0;
                        }
                        
                        /* :host 选择器 - 选择组件本身 */
                        :host(.highlighted) {
                            background: #fff3cd;
                        }
                        
                        /* :host-context - 根据祖先元素选择 */
                        :host-context(.dark-theme) {
                            background: #333;
                            color: white;
                        }
                        
                        p {
                            color: blue;
                            margin: 0;
                        }
                        
                        /* ::slotted - 选择插槽内容 */
                        ::slotted(.special) {
                            font-weight: bold;
                            color: green;
                        }
                    </style>
                    <p>这是 Shadow DOM 内部的段落，不受全局样式影响</p>
                    <slot></slot>
                `;
            }
        }

        customElements.define('shadow-demo', ShadowDemo);
    </script>

    <!-- 使用 highlighted 类 -->
    <shadow-demo class="highlighted">
        <span class="special">这是插槽内容，可以通过 ::slotted 设置样式</span>
    </shadow-demo>
```

### 11.5 实战：封装一个选项卡组件

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>选项卡组件</title>
</head>
<body>
    <!-- 使用选项卡组件 -->
    <tab-container>
        <tab-panel title="首页">
            <h2>欢迎来到首页</h2>
            <p>这是首页的内容...</p>
        </tab-panel>
        <tab-panel title="产品">
            <h2>产品列表</h2>
            <ul>
                <li>产品 A</li>
                <li>产品 B</li>
                <li>产品 C</li>
            </ul>
        </tab-panel>
        <tab-panel title="关于">
            <h2>关于我们</h2>
            <p>这是关于页面的内容...</p>
        </tab-panel>
    </tab-container>

    <script>
        // 选项卡容器组件
        class TabContainer extends HTMLElement {
            constructor() {
                super();
                this.attachShadow({ mode: 'open' });
                this.activeIndex = 0;
            }

            connectedCallback() {
                this.render();
            }

            switchTab(index) {
                this.activeIndex = index;
                this.render();
                
                // 触发自定义事件
                this.dispatchEvent(new CustomEvent('tab-change', {
                    detail: { index, title: this.panels[index].title }
                }));
            }

            render() {
                // 获取所有面板
                this.panels = Array.from(this.querySelectorAll('tab-panel'));
                const titles = this.panels.map((panel, index) => `
                    <button 
                        class="tab-btn ${index === this.activeIndex ? 'active' : ''}"
                        data-index="${index}"
                        role="tab"
                        aria-selected="${index === this.activeIndex}"
                        aria-controls="panel-${index}"
                    >
                        ${panel.title}
                    </button>
                `).join('');

                const contents = this.panels.map((panel, index) => `
                    <div 
                        class="tab-content ${index === this.activeIndex ? 'active' : ''}"
                        id="panel-${index}"
                        role="tabpanel"
                        aria-labelledby="tab-${index}"
                        ${index !== this.activeIndex ? 'hidden' : ''}
                    >
                        <slot name="panel-${index}"></slot>
                    </div>
                `).join('');

                this.shadowRoot.innerHTML = `
                    <style>
                        :host {
                            display: block;
                            font-family: system-ui, sans-serif;
                        }
                        .tabs-header {
                            display: flex;
                            border-bottom: 2px solid #e0e0e0;
                            margin-bottom: 16px;
                        }
                        .tab-btn {
                            padding: 12px 24px;
                            background: none;
                            border: none;
                            cursor: pointer;
                            font-size: 14px;
                            color: #666;
                            position: relative;
                            transition: color 0.3s;
                        }
                        .tab-btn:hover {
                            color: #333;
                        }
                        .tab-btn.active {
                            color: #667eea;
                            font-weight: 600;
                        }
                        .tab-btn.active::after {
                            content: '';
                            position: absolute;
                            bottom: -2px;
                            left: 0;
                            right: 0;
                            height: 2px;
                            background: #667eea;
                        }
                        .tab-content {
                            display: none;
                        }
                        .tab-content.active {
                            display: block;
                        }
                    </style>
                    <div class="tabs-header" role="tablist">
                        ${titles}
                    </div>
                    <div class="tabs-body">
                        ${contents}
                    </div>
                `;

                // 为每个面板分配 slot
                this.panels.forEach((panel, index) => {
                    panel.setAttribute('slot', `panel-${index}`);
                });

                // 绑定点击事件
                this.shadowRoot.querySelectorAll('.tab-btn').forEach(btn => {
                    btn.addEventListener('click', () => {
                        this.switchTab(Number(btn.dataset.index));
                    });
                });
            }
        }

        // 面板组件
        class TabPanel extends HTMLElement {
            static get observedAttributes() {
                return ['title'];
            }

            constructor() {
                super();
            }
        }

        customElements.define('tab-container', TabContainer);
        customElements.define('tab-panel', TabPanel);

        // 监听选项卡切换事件
        document.querySelector('tab-container').addEventListener('tab-change', (e) => {
            console.log('切换到选项卡:', e.detail);
        });
    </script>
</body>
</html>
```

## 12. Web Storage 和 IndexedDB

### 12.1 Web Storage 概述

Web Storage 提供了在浏览器中存储键值对数据的机制，包括 `localStorage` 和 `sessionStorage`。

| 特性 | localStorage | sessionStorage |
|------|--------------|----------------|
| 生命周期 | 永久存储，除非手动清除 | 会话期间有效，关闭标签页后清除 |
| 存储大小 | 约 5MB | 约 5MB |
| 作用域 | 同源的所有标签页共享 | 仅当前标签页可用 |
| 使用场景 | 用户偏好、主题设置 | 临时表单数据、页面状态 |

### 12.3 IndexedDB 详解

IndexedDB 是一个事务型数据库系统，允许在浏览器中存储大量结构化数据。

```html

<html lang="zh-CN">

    <meta charset="UTF-8">

    <script>
        // ==================== IndexedDB 封装类 ====================
        
        class IndexedDBUtil {
            constructor(dbName, version = 1) {
                this.dbName = dbName;
                this.version = version;
                this.db = null;
            }

            // 打开数据库
            async open(stores = []) {
                return new Promise((resolve, reject) => {
                    const request = indexedDB.open(this.dbName, this.version);

                    // 数据库升级（创建或修改对象存储）
                    request.onupgradeneeded = (event) => {
                        const db = event.target.result;
                        
                        stores.forEach(storeConfig => {
                            if (!db.objectStoreNames.contains(storeConfig.name)) {
                                const store = db.createObjectStore(storeConfig.name, {
                                    keyPath: storeConfig.keyPath || 'id',
                                    autoIncrement: storeConfig.autoIncrement !== false
                                });

                                // 创建索引
                                if (storeConfig.indexes) {
                                    storeConfig.indexes.forEach(index => {
                                        store.createIndex(index.name, index.keyPath, {
                                            unique: index.unique || false
                                        });
                                    });
                                }
                            }
                        });
                    };

                    request.onsuccess = (event) => {
                        this.db = event.target.result;
                        resolve(this.db);
                    };

                    request.onerror = (event) => {
                        reject(event.target.error);
                    };
                });
            }

            // 添加数据
            async add(storeName, data) {
                return new Promise((resolve, reject) => {
                    const transaction = this.db.transaction([storeName], 'readwrite');
                    const store = transaction.objectStore(storeName);
                    const request = store.add(data);

                    request.onsuccess = () => resolve(request.result);
                    request.onerror = () => reject(request.error);
                });
            }

            // 更新数据（如果存在则更新，不存在则添加）
            async put(storeName, data) {
                return new Promise((resolve, reject) => {
                    const transaction = this.db.transaction([storeName], 'readwrite');
                    const store = transaction.objectStore(storeName);
                    const request = store.put(data);

                    request.onsuccess = () => resolve(request.result);
                    request.onerror = () => reject(request.error);
                });
            }

            // 获取单条数据
            async get(storeName, key) {
                return new Promise((resolve, reject) => {
                    const transaction = this.db.transaction([storeName], 'readonly');
                    const store = transaction.objectStore(storeName);
                    const request = store.get(key);

                    request.onsuccess = () => resolve(request.result);
                    request.onerror = () => reject(request.error);
                });
            }

            // 获取所有数据
            async getAll(storeName) {
                return new Promise((resolve, reject) => {
                    const transaction = this.db.transaction([storeName], 'readonly');
                    const store = transaction.objectStore(storeName);
                    const request = store.getAll();

                    request.onsuccess = () => resolve(request.result);
                    request.onerror = () => reject(request.error);
                });
            }

            // 通过索引查询
            async getByIndex(storeName, indexName, value) {
                return new Promise((resolve, reject) => {
                    const transaction = this.db.transaction([storeName], 'readonly');
                    const store = transaction.objectStore(storeName);
                    const index = store.index(indexName);
                    const request = index.getAll(value);

                    request.onsuccess = () => resolve(request.result);
                    request.onerror = () => reject(request.error);
                });
            }

            // 删除数据
            async delete(storeName, key) {
                return new Promise((resolve, reject) => {
                    const transaction = this.db.transaction([storeName], 'readwrite');
                    const store = transaction.objectStore(storeName);
                    const request = store.delete(key);

                    request.onsuccess = () => resolve();
                    request.onerror = () => reject(request.error);
                });
            }

            // 清空对象存储
            async clear(storeName) {
                return new Promise((resolve, reject) => {
                    const transaction = this.db.transaction([storeName], 'readwrite');
                    const store = transaction.objectStore(storeName);
                    const request = store.clear();

                    request.onsuccess = () => resolve();
                    request.onerror = () => reject(request.error);
                });
            }

            // 使用游标遍历
            async cursor(storeName, callback) {
                return new Promise((resolve, reject) => {
                    const transaction = this.db.transaction([storeName], 'readonly');
                    const store = transaction.objectStore(storeName);
                    const request = store.openCursor();

                    request.onsuccess = (event) => {
                        const cursor = event.target.result;
                        if (cursor) {
                            callback(cursor.value);
                            cursor.continue();
                        } else {
                            resolve();
                        }
                    };

                    request.onerror = () => reject(request.error);
                });
            }

            // 关闭数据库
            close() {
                if (this.db) {
                    this.db.close();
                    this.db = null;
                }
            }
        }

        // ==================== 使用示例 ====================
        
        async function demo() {

            // 打开数据库并创建对象存储

                },

                    name: 'articles',
```

```html
                // 监听主线程消息
                self.onmessage = function(e) {
                    const { type, data } = e.data;
                    
                    switch (type) {
                        case 'calculate':
                            // 执行复杂计算
                            const result = heavyCalculation(data);
                            // 发送结果回主线程
                            self.postMessage({ type: 'result', data: result });
                            break;
                            
                        case 'progress':
                            // 模拟带进度的计算
                            calculateWithProgress(data);
                            break;
                    }
                };

                // 复杂计算函数
                function heavyCalculation(n) {
                    let result = 0;
                    for (let i = 0; i < n; i++) {
                        result += Math.sqrt(i) * Math.sin(i);
                    }
                    return result;
                }

                // 带进度的计算
                function calculateWithProgress(n) {
                    let result = 0;
                    const chunk = Math.floor(n / 100);
                    
                    for (let i = 0; i < n; i++) {
                        result += Math.sqrt(i) * Math.sin(i);
                        
                        // 每 1% 发送一次进度
                        if (i % chunk === 0) {
                            self.postMessage({
                                type: 'progress',
                                progress: Math.floor((i / n) * 100)
                            });
                        }
                    }
                    
                    self.postMessage({ type: 'result', data: result });
                }
            `;

            const blob = new Blob([workerCode], { type: 'application/javascript' });
            const workerUrl = URL.createObjectURL(blob);
            worker = new Worker(workerUrl);

            // 监听 Worker 消息
            worker.onmessage = function(e) {
                const { type, data, progress } = e.data;
                
                switch (type) {
                    case 'result':
                        document.getElementById('result').textContent = 
                            `计算结果: ${data.toFixed(2)}`;
                        document.getElementById('progressBar').style.width = '100%';
                        break;
                        
                    case 'progress':
                        document.getElementById('progressBar').style.width = `${progress}%`;
                        break;
                }
            };

            // 监听 Worker 错误
            worker.onerror = function(error) {
                console.error('Worker 错误:', error);
                document.getElementById('result').textContent = 
                    `错误: ${error.message}`;
            };

            return worker;
        }

        // ==================== 同步计算（阻塞主线程）====================
        
        document.getElementById('startSync').addEventListener('click', () => {
            const startTime = performance.now();
            
            // 阻塞主线程的计算
```

### 13.3 Service Workers 详解

Service Worker 是一种特殊的 Web Worker，用于实现离线缓存、推送通知、后台同步等功能。

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>Service Worker 示例</title>
</head>
<body>
    <h2>Service Worker 缓存示例</h2>
    <button id="register">注册 Service Worker</button>
    <button id="cache">缓存资源</button>
    <button id="fetch">测试离线</button>
    <div id="status"></div>

    <script>
        // ==================== Service Worker 注册 ====================
        
        const statusEl = document.getElementById('status');

        document.getElementById('register').addEventListener('click', async () => {
            if ('serviceWorker' in navigator) {
                try {
                    // 注册 Service Worker
                    const registration = await navigator.serviceWorker.register('/sw.js');
                    
                    statusEl.textContent = 'Service Worker 注册成功';
                    console.log('注册成功:', registration.scope);

                    // 监听更新
                    registration.addEventListener('updatefound', () => {
                        const newWorker = registration.installing;
                        newWorker.addEventListener('statechange', () => {
                            if (newWorker.state === 'installed' && navigator.serviceWorker.controller) {
                                statusEl.textContent = '新版本可用，请刷新页面';
                            }
                        });
                    });

                } catch (error) {
                    statusEl.textContent = '注册失败: ' + error.message;
                    console.error('注册失败:', error);
                }
            } else {
                statusEl.textContent = '浏览器不支持 Service Worker';
            }
        });

        // ==================== Service Worker 文件内容 ====================
        // 注意：以下代码应保存为单独的 sw.js 文件

        const serviceWorkerCode = `
            // sw.js - Service Worker 文件

            const CACHE_NAME = 'my-app-v1';
            const CACHE_URLS = [
                '/',
                '/index.html',
                '/styles.css',
                '/main.js',
                '/images/logo.png'
            ];

            // 安装事件 - 预缓存资源
            self.addEventListener('install', (event) => {
                console.log('Service Worker 安装中...');
                
                event.waitUntil(
                    caches.open(CACHE_NAME)
                        .then((cache) => {
                            console.log('缓存已打开');
                            return cache.addAll(CACHE_URLS);
                        })
                        .then(() => {
                            // 跳过等待，立即激活
                            return self.skipWaiting();
                        })
                );
            });

            // 激活事件 - 清理旧缓存
            self.addEventListener('activate', (event) => {
                console.log('Service Worker 已激活');
                
                event.waitUntil(
                    caches.keys().then((cacheNames) => {
                        return Promise.all(
                            cacheNames.map((cacheName) => {
                                if (cacheName !== CACHE_NAME) {
                                    console.log('删除旧缓存:', cacheName);
                                    return caches.delete(cacheName);
                                }
                            })
                        );
                    }).then(() => {
                        // 立即控制所有客户端
                        return self.clients.claim();
                    })
                );
            });

            // 请求拦截 - 缓存优先策略
            self.addEventListener('fetch', (event) => {
                event.respondWith(
                    caches.match(event.request)
                        .then((cachedResponse) => {
                            // 如果有缓存，返回缓存
                            if (cachedResponse) {
                                // 同时更新缓存（后台更新）
                                fetch(event.request).then((response) => {
                                    caches.open(CACHE_NAME).then((cache) => {
                                        cache.put(event.request, response);
                                    });
                                });
                                return cachedResponse;
                            }

                            // 没有缓存，从网络获取
                            return fetch(event.request)
                                .then((response) => {
                                    // 检查是否是有效响应
                                    if (!response || response.status !== 200 || response.type !== 'basic') {
                                        return response;
                                    }

                                    // 克隆响应并缓存
                                    const responseToCache = response.clone();
                                    caches.open(CACHE_NAME).then((cache) => {
                                        cache.put(event.request, responseToCache);
                                    });

                                    return response;
                                })
                                .catch(() => {
                                    // 网络失败，返回离线页面
                                    if (event.request.mode === 'navigate') {
                                        return caches.match('/offline.html');
                                    }
                                });
                        })
                );
            });

            // 后台同步
            self.addEventListener('sync', (event) => {
                if (event.tag === 'sync-data') {
                    event.waitUntil(syncData());
                }
            });

            async function syncData() {
                // 从 IndexedDB 获取待同步数据
                // 发送到服务器
                console.log('后台同步中...');
            }

            // 推送通知
            self.addEventListener('push', (event) => {
                const data = event.data.json();
                
                const options = {
                    body: data.body,
                    icon: '/images/icon.png',
                    badge: '/images/badge.png',
                    vibrate: [100, 50, 100],
                    data: {
                        url: data.url
                    }
                };

                event.waitUntil(
                    self.registration.showNotification(data.title, options)
                );
            });

            // 通知点击
            self.addEventListener('notificationclick', (event) => {
                event.notification.close();
                
                event.waitUntil(
                    clients.openWindow(event.notification.data.url)
                );
            });
        `;

        // ==================== 缓存策略 ====================

        // 1. 缓存优先（Cache First）
        async function cacheFirst(request) {
            const cached = await caches.match(request);
            if (cached) return cached;
            return fetch(request);
        }

        // 2. 网络优先（Network First）
        async function networkFirst(request) {
            try {
                const response = await fetch(request);
                const cache = await caches.open('dynamic');
                cache.put(request, response.clone());
                return response;
            } catch (error) {
                return caches.match(request);
            }
        }

        // 3. 网络优先带超时
        async function networkFirstWithTimeout(request, timeout = 3000) {
            try {
                const response = await Promise.race([
                    fetch(request),
                    new Promise((_, reject) => 
                        setTimeout(() => reject(new Error('超时')), timeout)
                    )
                ]);
                return response;
            } catch (error) {
                return caches.match(request);
            }
        }

        // 4. 缓存优先，后台更新（Stale While Revalidate）
        async function staleWhileRevalidate(request) {
            const cache = await caches.open('dynamic');
            const cached = await cache.match(request);

            const fetchPromise = fetch(request).then((response) => {
                cache.put(request, response.clone());
                return response;
            });

            return cached || fetchPromise;
        }

        // ==================== 实用功能 ====================

        // 检查更新
        async function checkForUpdates() {
            const registration = await navigator.serviceWorker.ready;
            await registration.update();
        }

        // 清除所有缓存
        async function clearAllCaches() {
            const cacheNames = await caches.keys();
            await Promise.all(cacheNames.map(name => caches.delete(name)));
            console.log('所有缓存已清除');
        }

        // 获取缓存大小
        async function getCacheSize() {
            if ('storage' in navigator && 'estimate' in navigator.storage) {
                const estimate = await navigator.storage.estimate();
                console.log(`已使用: ${(estimate.usage / 1024 / 1024).toFixed(2)} MB`);
                console.log(`可用: ${(estimate.quota / 1024 / 1024).toFixed(2)} MB`);
            }
        }

        // 请求持久化存储
        async function requestPersistentStorage() {
            if (navigator.storage && navigator.storage.persist) {
                const isPersisted = await navigator.storage.persist();
                console.log('持久化存储:', isPersisted ? '已授权' : '被拒绝');
            }
        }
    </script>
</body>
</html>
```

```html
        function updateOnlineStatus() {
            const indicator = document.getElementById('offlineIndicator');
            if (navigator.onLine) {
                indicator.classList.remove('show');
                console.log('已连接网络');
                // 同步离线数据

            } else {
                indicator.classList.add('show');
                console.log('已断开网络');
            }
        }

        window.addEventListener('offline', updateOnlineStatus);

        updateOnlineStatus();

        // ==================== 离线数据同步 ====================
        
        // 保存离线操作
        async function saveOfflineAction(action) {
            const db = await openDB();
            const tx = db.transaction('pendingActions', 'readwrite');
            const store = tx.objectStore('pendingActions');
            await store.add({
                ...action,
                timestamp: Date.now()
            });
        }

        // 同步离线数据
        async function syncOfflineData() {
            const db = await openDB();
            const tx = db.transaction('pendingActions', 'readwrite');
            const store = tx.objectStore('pendingActions');
            const actions = await store.getAll();

            for (const action of actions) {
                try {
                    const response = await fetch(action.url, {
                        method: action.method,
                        headers: action.headers,
                        body: JSON.stringify(action.data)
                    });

                    if (response.ok) {
                        await store.delete(action.id);
                        console.log('同步成功:', action);
                    }
                } catch (error) {
                    console.error('同步失败:', error);
                }
            }
        }

        // 简单的 IndexedDB 封装
        function openDB() {
            return new Promise((resolve, reject) => {
                const request = indexedDB.open('OfflineAppDB', 1);
                request.onupgradeneeded = (e) => {
                    const db = e.target.result;
                    if (!db.objectStoreNames.contains('pendingActions')) {
                        db.createObjectStore('pendingActions', { keyPath: 'id', autoIncrement: true });
                    }
                };
                request.onsuccess = (e) => resolve(e.target.result);
                request.onerror = (e) => reject(e.target.error);
            });
        }

        // ==================== 网络请求封装 ====================
        
        async function fetchWithOfflineSupport(url, options = {}) {
            if (navigator.onLine) {
                // 在线：直接请求
                return fetch(url, options);
            } else {
                // 离线：保存操作
                if (options.method !== 'GET') {
                    await saveOfflineAction({
                        url,
                        method: options.method || 'GET',
                        headers: options.headers,
                        data: options.body
                    });
                    return new Response(JSON.stringify({ 
                        success: true, 
                        message: '操作已保存，将在恢复网络后同步' 
                    }), {
                        headers: { 'Content-Type': 'application/json' }
                    });
                } else {
                    // GET 请求：尝试从缓存获取
                    const cache = await caches.open('offline-cache');
                    const cached = await cache.match(url);
                    if (cached) return cached;
                    throw new Error('离线且无缓存');
                }
            }
        }

        // 使用示例
        async function submitForm(data) {
            const response = await fetchWithOfflineSupport('/api/submit', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify(data)
            });
            return response.json();
        }
    </script>
</body>
```

```html
        // ==================== 图像处理 ====================

        img.src = 'https://picsum.photos/400/300';

        function addFilterButtons() {
            const buttons = [
                { text: '原图', filter: null },
                { text: '灰度', filter: 'grayscale' },
                { text: '反色', filter: 'invert' },
                { text: '模糊', filter: 'blur' }
            ];

            buttons.forEach(btn => {
                const button = document.createElement('button');
                button.textContent = btn.text;
                button.onclick = () => applyFilter(btn.filter);
                document.body.appendChild(button);
            });
        }

        function applyFilter(filter) {
            // 清除画布
            imageCtx.clearRect(0, 0, imageCanvas.width, imageCanvas.height);

            // 重绘原图
            imageCtx.drawImage(img, 0, 0);

            if (!filter) return;

            // 获取图像数据
            const imageData = imageCtx.getImageData(0, 0, imageCanvas.width, imageCanvas.height);
            const data = imageData.data;

            switch (filter) {
                case 'grayscale':
                    // 灰度处理
                    for (let i = 0; i < data.length; i += 4) {
                        const avg = (data[i] + data[i + 1] + data[i + 2]) / 3;
                        data[i] = avg;     // R
                        data[i + 1] = avg; // G
                        data[i + 2] = avg; // B
                    }
                    break;

                case 'invert':
                    // 反色处理
                    for (let i = 0; i < data.length; i += 4) {
                        data[i] = 255 - data[i];         // R
                        data[i + 1] = 255 - data[i + 1]; // G
                        data[i + 2] = 255 - data[i + 2]; // B
                    }
                    break;

                case 'blur':
                    // 简单模糊（实际应用使用卷积核）
                    imageCtx.filter = 'blur(3px)';
                    imageCtx.drawImage(img, 0, 0);
                    imageCtx.filter = 'none';
                    return;
            }

            // 放回处理后的图像数据
            imageCtx.putImageData(imageData, 0, 0);
        }
```

## 16. 面试高频问题补充

### 16.1 HTML5 新特性问题

**问题1：HTML5 新增了哪些 API？**

答案：
- **存储 API**：localStorage、sessionStorage、IndexedDB
- **通信 API**：WebSocket、Server-Sent Events、WebRTC
- **图形 API**：Canvas 2D、WebGL、SVG
- **多媒体 API**：Audio API、Video API、MediaStream
- **设备访问 API**：Geolocation、DeviceOrientation、Vibration
- **离线 API**：Service Worker、Application Cache（已废弃）
- **性能 API**：Web Workers、requestAnimationFrame、Performance API
- **其他 API**：Drag and Drop、File API、Fullscreen、Notification

**问题2：什么是 Web Worker？有什么使用场景？**

答案：
Web Worker 允许 JavaScript 在后台线程中运行，避免复杂计算阻塞主线程。

使用场景：
- 复杂数学计算
- 大数据处理
- 图像/视频处理
- 加密/解密操作
- 长时间轮询

注意事项：
- 无法访问 DOM
- 无法访问 window 对象
- 通过 postMessage 通信
- 需要单独的 JS 文件或 Blob URL

**问题3：Service Worker 的作用是什么？**

答案：
Service Worker 是一种特殊的 Web Worker，主要用于：
1. **离线缓存**：拦截网络请求，返回缓存内容
2. **推送通知**：接收服务器推送消息
3. **后台同步**：在网络恢复时同步数据
4. **性能优化**：预缓存关键资源

生命周期：
1. 注册（register）
2. 安装（install）
3. 激活（activate）
4. 空闲（idle）
5. 终止（terminated）

### 16.2 存储相关问题

**问题4：localStorage、sessionStorage、Cookie、IndexedDB 的区别？**

答案：

| 特性 | Cookie | localStorage | sessionStorage | IndexedDB |

| 大小 | 4KB | ~5MB | ~5MB | 无限制 |
| 过期时间 | 可设置 | 永久 | 会话结束 | 永久 |
| 发送到服务器 | 是 | 否 | 否 | 否 |
| 数据类型 | 字符串 | 字符串 | 字符串 | 结构化数据 |
| 跨标签页 | 是 | 是 | 否 | 是 |
| 事务支持 | 否 | 否 | 否 | 是 |
| 索引查询 | 否 | 否 | 否 | 是 |

### 16.3 Canvas 和 SVG 问题

**问题6：Canvas 和 SVG 有什么区别？**

答案：

| 特性 | Canvas | SVG |
|------|--------|-----|
| 渲染方式 | 位图（像素） | 矢量（数学描述） |
| 缩放 | 失真 | 不失真 |
| DOM 结构 | 无 | 有 |

| 性能 | 大量元素更好 | 少量元素更好 |
| 动画 | 需要重绘 | CSS/SMIL 动画 |
| 适用场景 | 游戏、图表、图像处理 | 图标、Logo、简单动画 |

### 16.4 响应式设计问题

**问题8：什么是移动优先设计？**

答案：
移动优先设计是一种响应式设计策略，先为移动设备设计和开发，然后逐步增强到更大的屏幕。

优点：
1. 专注于核心内容
2. 更好的性能（移动端资源有限）
3. 更简单的代码结构
4. 更好的用户体验

实现方式：
```css
/* 移动优先：基础样式针对移动端 */
.container {
    width: 100%;
    padding: 10px;
}

/* 逐步增强：使用 min-width 媒体查询 */
@media (min-width: 768px) {
    .container {
        max-width: 720px;
        margin: 0 auto;
    }
}

@media (min-width: 1024px) {
    .container {
        max-width: 960px;
    }
}
```

**问题10：什么是容器查询？与媒体查询有什么区别？**

答案：
容器查询（Container Queries）是 CSS 的新特性，允许根据容器的大小而不是视口大小来应用样式。

区别：
- **媒体查询**：基于视口大小，适合页面级响应式

```css
/* 容器查询 */
```

### 17.1 popover API 正式稳定

`popover` 属性从实验性特性正式进入稳定阶段，成为替代 `dialog` 元素显示叠加内容的推荐方案。

```html
<!-- 最简用法：只需一个属性即可 -->
<button popovertarget="my-tooltip">悬停查看</button>
<div popover id="my-tooltip">
  这是一个原生提示框，无需 JavaScript！
</div>
```

```css
/* popover 默认隐藏，显示时有动画效果 */
[popover] {
  margin: 0;
  padding: 10px;
  border: 1px solid #ccc;
  border-radius: 4px;
  background: white;
}

/* 显示状态的过渡动画 */
[popover]:popover-open {
  animation: fade-in 0.2s ease-out;
}

@keyframes fade-in {
  from { opacity: 0; transform: scale(0.95); }
  to { opacity: 1; transform: scale(1); }
}
```

**与 dialog 的核心区别：**

| 特性 | popover | dialog |
|------|---------|--------|
| 模态背景 | 通过 `popover` 伪类自动生成 | 需手动实现 |
| 层级管理 | 浏览器自动处理顶层叠加 | 需 z-index 管理 |
| 焦点陷阱 | 自动启用 | 需 `showModal()` |
| 点击外部关闭 | `hint` 模式自动关闭 | 需手动实现 |
| 事件冒泡 | 不冒泡到底层文档 | 冒泡 |

**三种显示模式：**

```javascript
// hint 模式：轻量提示，点击外部自动关闭
element.setAttribute('popover', 'hint');

// manual 模式：完全手动控制，不会自动关闭
element.setAttribute('popover', 'manual');

// 无属性默认为 auto，等同于 hint
element.setAttribute('popover', 'auto');

// JavaScript API
element.showPopover();    // 显示
element.hidePopover();    // 隐藏
element.togglePopover();  // 切换
```

### 17.2 View Transitions API 全面支持

页面视图过渡动画现已在所有主流浏览器中得到完整支持，实现了单页应用内的丝滑过渡效果。

```javascript
// 基础使用：两幅视图之间的过渡
async function navigateTo(url) {
  // 启动视图过渡
  const transition = document.startViewTransition(() => {
    // 更新 DOM 的回调函数
    updateDOMContent(url);
  });

  // 等待过渡动画完成
  await transition.finished;
}

// 命名的视图过渡：精确控制特定元素
document.startViewTransition(() => {
  // 旧 DOM → 新 DOM
}, {
  updateDOM: (oldState, newState) => {
    // 可以手动指定哪些元素参与过渡
  }
});
```

```css
/* 自定义过渡动画 */
::view-transition-old(root) {
  /* 旧页面的退出动画 */

::view-transition-new(root) {
  /* 新页面的进入动画 */
  animation: slide-in 0.3s ease-in;
}

/* 命名视图过渡：只过渡特定元素 */
.header {

/* 禁用过渡 */
.no-transition::view-transition-none(*) {
```

### 17.5 Temporal API 生产可用

`Temporal` 是现代 JavaScript 中 `Date` 的替代品，提供了完整、不可变且易于使用的时间与日期 API。

```javascript
// 获取当前时间（无需时区歧义）
const now = Temporal.Now.instant(); // UTC 时间
const dateTime = Temporal.Now.zonedDateTimeISO(); // 带时区的本地时间

// 创建日期时间
const birthday = Temporal.ZonedDateTime.from({
  year: 2026,
  month: 3,
  day: 18,
  timeZone: 'Asia/Shanghai',
  hour: 9,
  minute: 0,
});

// 日期计算（简单直观）
const oneWeekLater = birthday.add({ days: 7 });
const threeHoursAgo = Temporal.Now.instant().subtract({ hours: 3 });

// 日期比较
const dur = later.difference(earlier); // 返回 Duration 对象
console.log(`${dur.days} 天 ${dur.hours} 小时`);

// 不同时区转换
const tokyoTime = dateTime.withTimeZone('Asia/Tokyo');
```

### 17.6 声明式 Shadow DOM

无需 JavaScript，直接在 HTML 中声明 Shadow DOM：

```html
<!-- 直接在模板中声明 Shadow DOM -->
<template id="my-card">
  <style>
    :host {
      display: block;
      border: 1px solid #ddd;
      border-radius: 8px;
      padding: 16px;
    }
    h2 { color: var(--card-title-color, #333); }
  </style>
  <h2 part="title"></h2>
  <slot></slot>
</template>

<!-- 使用 -->
<my-card>
  <span slot="title">卡片标题</span>
  <p>卡片内容...</p>
</my-card>
```

### 18.1 HTML 代码质量检查清单

在团队协作中，HTML 代码质量直接影响可维护性、可访问性和性能。以下是完整的检查清单：

```html
<!-- 结构与语义 -->
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <!-- 字符编码：必须是首行 -->
  <meta charset="UTF-8">
  <!-- 视口设置：移动端必需 -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <!-- 描述标签：SEO 必需 -->
  <meta name="description" content="页面核心描述，不超过160字符">
</head>
<body>
  <!-- 语义化标签 -->
  <header>顶栏</header>
  <nav>导航</nav>
  <main id="main-content">
    <article>
      <section>
        <h1>主标题</h1>
        <!-- 内容 -->
      </section>
    </article>
    <aside>侧边栏</aside>
  </main>
  <footer>底栏</footer>
</body>
</html>
```

**可访问性（A11Y）检查要点：**

```html
<!-- 图片必需有 alt 属性 -->
```

### 18.3 HTML 安全防护清单

