## 2. 渲染引擎工作流程

### 2.1 完整渲染流程

渲染引擎解析HTML和CSS，将它们转换成屏幕上可见的像素：

```
┌─────────────────────────────────────────────────────────────────────┐
│                       渲染引擎工作流程                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. HTML解析                                                       │
│     HTML ──────► DOM树                                             │
│                                                                     │
│     ┌─────────────────────────────────────────────────────────┐   │
│     │ <html>                                                   │   │
│     │   <head>...</head>                                       │   │
│     │   <body>                                                 │   │
│     │     <div class="container">                              │   │
│     │       <h1>标题</h1>                                     │   │
│     │       <p>内容</p>                                        │   │
│     │     </div>                                                │   │
│     │   </body>                                                 │   │
│     │ </html>                                                   │   │
│     └─────────────────────────────────────────────────────────┘   │
│                                                                     │
│  2. CSS解析                                                       │
│     CSS ──────► CSSOM树                                           │
│                                                                     │
│     ┌─────────────────────────────────────────────────────────┐   │
│     │ document                                                 │   │
│     │   ├─ html { ... }                                       │   │
│     │   ├─ body { ... }                                      │   │
│     │   ├─ div.container { color: blue; }                     │   │
│     │   ├─ h1 { font-size: 24px; }                           │   │
│     │   └─ p { margin: 10px; }                               │   │
│     └─────────────────────────────────────────────────────────┘   │
│                                                                     │
│  3. 渲染树构建                                                    │
│     DOM + CSSOM ──────► 渲染树（Render Tree）                      │
│                                                                     │
│     ┌─────────────────────────────────────────────────────────┐   │
│     │ 渲染树节点                                                 │   │
│     │   ├─ html (可忽略，仅结构性)                              │   │
│     │   ├─ body                                                │   │
│     │   │   └─ div.container (可见)                            │   │
│     │   │       ├─ h1 (可见，内容)                             │   │
│     │   │       └─ p (可见，内容)                              │   │
│     │   └─ head (不可见，不加入渲染树)                          │   │
│     └─────────────────────────────────────────────────────────┘   │
│                                                                     │
│  4. 布局（Layout）                                                │
│     计算每个元素的精确位置和大小                                      │
│                                                                     │
│  5. 绘制（Paint）                                                 │
│     将渲染树绘制到多个图层                                           │
│                                                                     │
│  6. 合成（Composite）                                             │
│     将所有图层合成为最终画面                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 渲染流水线详解

```javascript
// 渲染流水线的JavaScript监控

function analyzeRenderingPipeline() {
  const pipeline = {
    // 1. 解析HTML
    parseHTML: {
      描述: '将HTML文本解析为DOM节点',
      性能影响: 'HTML越大、嵌套越深，解析越慢',
      优化: '减少HTML体积、扁平化DOM结构',
    },

    // 2. 解析CSS
    parseCSS: {
      描述: '将CSS规则解析为CSSOM',
      性能影响: '选择器越复杂，匹配越慢',
      优化: '简化CSS选择器、避免通配符',
    },

    // 3. 构建渲染树
    buildRenderTree: {
      描述: '合并DOM和CSSOM为渲染树',
      性能影响: '不可见元素不加入渲染树',
      优化: 'display:none的元素不参与渲染',
    },

    // 4. 布局计算
    layout: {
      描述: '计算元素的几何属性',
      性能影响: '元素越多、嵌套越深，布局越慢',
      优化: '使用flexbox/grid代替浮动、减少DOM深度',
    },

    // 5. 绘制
    paint: {
      描述: '将渲染树绘制到图层',
      性能影响: '复杂绘图操作、渐变、阴影等',
      优化: '使用transform、opacity代替位置/外观变化',
    },

    // 6. 合成
    composite: {
      描述: '将多个图层合成为最终画面',
      性能影响: '过多图层会消耗GPU内存',
      优化: '使用will-change提示、避免过多合成层',
    },
  };

  return pipeline;
}

// 使用Performance API分析渲染阶段
function measureRenderingPhases() {
  const entries = performance.getEntriesByType('paint');

  entries.forEach(entry => {
    console.log(`绘制事件: ${entry.name}`, {
      时间: entry.startTime.toFixed(2) + 'ms',
    });
  });
}
```

## 3. 渲染树构建

### 3.1 DOM与CSSOM的关系

渲染树是DOM树和CSSOM树合并的结果：

```javascript
// DOM树结构
const domTree = {
  type: 'Document',
  children: [
    {
      type: 'html',
      children: [
        { type: 'head' },
        {
          type: 'body',
          children: [
            {
              type: 'div',
              className: 'container',
              children: [
                { type: 'h1', textContent: '标题' },
                { type: 'p', textContent: '内容' },
              ],
            },
          ],
        },
      ],
    },
  ],
};

// CSSOM树结构
const cssomTree = {
  rules: [
    { selector: 'body', declarations: { margin: '0', padding: '0' } },
    { selector: '.container', declarations: { color: 'blue' } },
    { selector: 'h1', declarations: { fontSize: '24px' } },
  ],
};

// 渲染树结构
const renderTree = [
  {
    type: 'body',
    style: { margin: '0', padding: '0' },
    layout: { width: 800, height: 600, x: 0, y: 0 },
    children: [
      {
        type: 'div',
        style: { color: 'blue' },
        layout: { width: 800, height: 100, x: 0, y: 0 },
        children: [
          {
            type: 'h1',
            style: { fontSize: '24px' },
            layout: { width: 800, height: 30, x: 0, y: 10 },
          },
          {
            type: 'p',
            style: {},
            layout: { width: 800, height: 20, x: 0, y: 50 },
          },
        ],
      },
    ],
  },
];
```

### 3.2 可见性与渲染树

```javascript
// 哪些元素不会加入渲染树

const invisibleElements = {
  // 1. display: none
  displayNone: `
    <div style="display: none">不会显示</div>
    // 完全不参与渲染树
  `,

  // 2. visibility: hidden
  visibilityHidden: `
    <div style="visibility: hidden">看不见</div>
    // 仍占用空间，但不显示
    // 会加入渲染树
  `,

  // 3. 带有hidden属性的元素
  hiddenAttribute: `
    <input type="hidden" />
    // 表单数据，会加入渲染树但不显示
  `,

  // 4. <head>元素
  headElement: `
    <head>...</head>
    // 不显示，不加入渲染树
  `,

  // 5. 注释和Processing Instructions
  comments: `
    <!-- 注释 -->
    // 不加入渲染树
  `,

  // 6. script和style元素
  scriptStyle: `
    <script>...</script>
    <style>...</style>
    // 不直接显示，但<style>的规则会影响渲染
  `,
};

// 与display:none的区别
function visibilityVsDisplay() {
  // display: none
  // - 不占用任何空间
  // - 不参与布局计算
  // - 不加入渲染树

  // visibility: hidden
  // - 占用空间（但不可见）
  // - 参与布局计算
  // - 加入渲染树但不可见
}
```

## 5. 合成层管理

### 5.1 合成层原理

合成层（Compositing Layer）是浏览器为实现高性能渲染而创建的独立图层：

```
┌─────────────────────────────────────────────────────────────────────┐
│                       合成层示意图                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  主文档流                                                           │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ <body>                                                      │   │
│  │   <div class="normal">Normal Content</div>                 │   │
│  │                                                             │   │
│  │   ┌─────────────────────────────────────────────────┐     │   │
│  │   │         合成层1（transform: translateZ）          │     │   │
│  │   │         <div class="layer-1">Layer 1</div>       │     │   │
│  │   └─────────────────────────────────────────────────┘     │   │
│  │                                                             │   │
│  │   ┌─────────────────────────────────────────────────┐     │   │
│  │   │         合成层2（will-change: transform）        │     │   │
│  │   │         <div class="layer-2">Layer 2</div>       │     │   │
│  │   └─────────────────────────────────────────────────┘     │   │
│  │                                                             │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  合成结果                                                           │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                        最终画面                              │   │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐                    │   │
│  │  │  合成层1  │  │  合成层2  │  │  主文档层  │  ───► GPU合成   │   │
│  │  └─────────┘  └─────────┘  └─────────┘                    │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 5.3 合成层优化

```javascript
// 合成层优化策略

// 1. 合理使用will-change
function optimizeWillChange() {
  // ❌ 过度使用
  // 不要给大量元素添加will-change
  document.querySelectorAll('.item').forEach(el => {
    el.style.willChange = 'transform'; // 糟糕！
  });

  // ✅ 正确使用
  // 只对需要动画的元素添加
  const animatedElement = document.getElementById('animated');
  animatedElement.style.willChange = 'transform';

  // 动画结束后移除
  animatedElement.addEventListener('animationend', () => {
    animatedElement.style.willChange = 'auto';
  });
}

// 2. 减少合成层数量
function reduceLayerCount() {
  // 问题：过多合成层消耗GPU内存
  // 解决：合并可以合并的图层

  // 方案1：使用transform合并
  // ❌ 多个分离的动画
  const elements = document.querySelectorAll('.particle');
  elements.forEach(el => {
    el.style.willChange = 'transform';
  });

  // ✅ 一个容器动画
  const container = document.getElementById('particles-container');
  container.style.willChange = 'transform';
  // 所有粒子在容器内移动，容器整体动画

  // 方案2：使用CSS containment
  document.querySelectorAll('.component').forEach(el => {
    el.style.contain = 'layout paint';
    // 限制组件内部变化对外部的影响
  });
}

// 3. 使用CSS containment
const containmentValues = {
  // 布局隔离
  layout: 'contain: layout',
```

## 7. 面试高频问题

### 问题1：浏览器渲染流程是什么？

**答案：**

1. **解析HTML** → 构建DOM树
2. **解析CSS** → 构建CSSOM树
3. **合并DOM+CSSOM** → 构建渲染树
4. **布局计算** → 计算元素几何属性
5. **绘制** → 将渲染树绘制到图层
6. **合成** → 将图层合成为最终画面

### 问题2：什么是重排和重绘？如何避免？

**答案：**

**重排（Reflow）：**
- 几何属性变化时触发
- 需要重新计算布局
- 成本较高

**重绘（Repaint）：**
- 外观属性变化时触发
- 不需要重新布局
- 成本比重排低

**避免方法：**
- 使用transform/opacity代替其他属性
- 批量DOM操作
- 使用display:none隐藏元素后修改
- 避免读取布局属性

### 问题3：什么是合成层？如何创建？

**答案：**

**合成层：**浏览器为实现高性能渲染而创建的独立图层

**创建方式：**
- 3D变换：`transform: translateZ(0)`
- will-change属性
- CSS动画（transform/opacity）
- `<video>`、`<canvas>`元素
- position: fixed

**优化建议：**
- 不要过度创建合成层
- 动画结束后移除will-change
- 使用contain隔离布局

### 问题4：如何优化动画性能？

**答案：**

1. **使用高性能属性**
   - 优先使用transform和opacity
   - 避免动画width、height等布局属性

2. **启用GPU加速**
   - 使用transform: translateZ(0)
   - 或will-change属性

3. **避免强制同步布局**
   - 不要在动画中读取布局属性
   - 先读后写

4. **使用requestAnimationFrame**
   - 确保动画在最佳时机执行
   - 避免使用setTimeout/setInterval

### 问题5：CSS选择器性能有差异吗？

**答案：**

**从快到慢的选择器：**
1. ID选择器（最快）
2. 类选择器
3. 标签选择器
4. 属性选择器
5. 伪类选择器
6. 后代选择器（较慢）

**优化建议：**
- 避免使用通配符
- 减少选择器嵌套
- 使用BEM等命名规范
- 保持选择器简洁
