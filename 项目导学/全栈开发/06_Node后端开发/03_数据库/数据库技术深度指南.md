### 2.4 PostgreSQL核心原理

#### 2.4.1 MVCC多版本并发控制

**什么是MVCC？**

MVCC（Multi-Version Concurrency Control）是PostgreSQL实现事务隔离的核心机制。它允许同一数据的多个版本同时存在，每个事务看到的是特定时间点的数据快照，从而实现读不加锁、读写不冲突的高并发特性。

**核心概念：**

```
事务ID（xmin/xmax）：
- xmin：创建此行版本的事务ID
- xmax：删除/更新此行版本的事务ID（如果被标记删除）

可见性判断规则：
1. 事务ID < 当前事务ID，且状态为已提交 → 可见
2. 事务ID = 当前事务ID → 可见（自身操作）
3. 事务ID > 当前事务ID → 不可见
4. 事务状态为回滚中/回滚完成 → 不可见
```

**实践验证：**

```sql
-- 查看表的行版本信息
SELECT xmin, xmax, * FROM users;

-- 开启两个事务窗口测试MVCC
-- 事务A：
BEGIN;
SELECT * FROM users WHERE id = 1;  -- 看到旧数据

-- 事务B（在事务A之外）：
UPDATE users SET name = '新名字' WHERE id = 1;
COMMIT;

-- 事务A（继续）：
SELECT * FROM users WHERE id = 1;  -- 仍然看到旧数据（快照）
UPDATE users SET name = '另一个名字' WHERE id = 1;  -- 会基于旧数据进行更新
COMMIT;  -- 最终看到什么？
```

**MVCC优缺点分析：**

| 优点 | 说明 |
|------|------|
| 读不加锁 | SELECT操作不会阻塞其他读写操作 |
| 读写不冲突 | 读和写可以并发执行 |
| 事务隔离 | 每个事务看到一致的数据快照 |

| 缺点 | 说明 |
|------|------|
| 版本膨胀 | 更新产生垃圾，需要定期VACUUM清理 |
| 事务ID回绕 | 长事务会阻塞VACUUM |
| 存储开销 | 每个行版本都需要存储 |

**避免长事务：**

```sql
-- 查看长时间运行的事务
SELECT pid, now() - xact_start AS duration, state, query
FROM pg_stat_activity
WHERE state != 'idle' AND xact_start IS NOT NULL
ORDER BY duration DESC;

-- 查看最小活动事务ID
SELECT min(xid) FROM pg_database;

-- 终止长时间运行的事务
SELECT pg_terminate_backend(pid) FROM pg_stat_activity
WHERE pid = 12345;
```

#### 2.4.2 WAL预写日志机制

**WAL工作原理：**

预写日志（Write-Ahead Logging）是PostgreSQL实现持久性和原子性的核心机制。所有对数据的修改首先写入WAL日志，然后才写入数据文件。这确保了即使系统崩溃，也能通过重放WAL恢复数据。

```
写入流程：
1. 事务开始
2. 修改数据页（内存中的Buffer）
3. 写入WAL日志（同时写入磁盘）
4. 提交事务时，WAL已持久化
5. 后台进程异步刷写数据页到磁盘

崩溃恢复流程：
1. 系统启动
2. 读取WAL日志
3. 重放所有已提交事务的修改
4. 回滚未提交事务的修改
```

**WAL配置参数：**

```sql
-- 查看WAL相关配置
SHOW wal_level;           -- minimal, replica, logical
SHOW fsync;               -- 是否强制同步
SHOW synchronous_commit;  -- 提交时同步方式
SHOW wal_buffers;         -- WAL缓冲区大小

-- 推荐的WAL配置
ALTER SYSTEM SET wal_level = 'replica';
ALTER SYSTEM SET max_wal_size = '1GB';
ALTER SYSTEM SET min_wal_size = '80MB';
ALTER SYSTEM SET synchronous_commit = 'on';  -- 平衡安全性和性能
```

**WAL级别对比：**

| 级别 | 说明 | 用途 |
|------|------|------|
| minimal | 只记录保证恢复的最少信息 | 最高性能，丢失最后1个事务 |
| replica | 记录支持复制和归档的信息 | 推荐，用于主从复制 |
| logical | 记录支持逻辑复制的详细信息 | 用于逻辑复制、CDC |

**WAL维护命令：**

```sql
-- 手动检查点（强制刷写WAL）
CHECKPOINT;

-- 强制切换WAL段文件
SELECT pg_switch_wal();

-- 查看WAL生成速度
SELECT * FROM pg_stat_database
WHERE datname = 'myapp';

-- 估算WAL保留时间
SELECT
    pg_size_pretty(
        pg_wal_lsn_diff(
            pg_current_wal_lsn(),
            '0/0'::pg_lsn
        )
    ) as wal_size;
```

#### 2.4.3 B-tree索引原理

**B-tree索引结构：**

PostgreSQL的默认索引类型是B-tree（平衡多路搜索树）。它是一种自平衡的树形数据结构，保持数据有序，支持等值查询和范围查询。

```
B-tree索引结构示意：
                    [50]
                   /    \
           [25, 30]      [75, 80]
           /    \        /      \
      [10,20] [30,40] [60,70] [90,100]
```

**索引创建详解：**

```sql
-- 基础B-tree索引
CREATE INDEX idx_users_email ON users(email);

-- 复合索引（最左前缀原则）
CREATE INDEX idx_orders_user_status ON orders(user_id, status);

-- 带排序的索引（支持DESC排序查询）
CREATE INDEX idx_products_price_desc ON products(price DESC);

-- 表达式索引
CREATE INDEX idx_users_email_lower ON users(LOWER(email));

-- 部分索引（只索引活跃用户）
CREATE INDEX idx_users_active_email ON users(email)
WHERE status = 'active';

-- 唯一索引
CREATE UNIQUE INDEX idx_users_email_unique ON users(email);

-- 查看索引大小
SELECT
    indexname,
    pg_size_pretty(pg_relation_size(indexrelid)) as index_size
FROM pg_stat_user_indexes
WHERE schemaname = 'public';

-- 查看索引的使用情况
SELECT
    indexrelname,
    idx_scan,       -- 索引扫描次数
    idx_tup_read,   -- 读取的索引行数
    idx_tup_fetch   -- 通过索引获取的行数
FROM pg_stat_user_indexes
ORDER BY idx_scan DESC;
```

**索引失效场景：**

```sql
-- ❌ 索引失效的情况

-- 1. 在索引列上使用函数
SELECT * FROM users WHERE LOWER(email) = 'test@example.com';
-- 解决：创建表达式索引 CREATE INDEX idx_users_email_lower ON users(LOWER(email));

-- 2. 类型转换
SELECT * FROM users WHERE id = '123';  -- id是INTEGER类型
-- 解决：使用正确的类型 SELECT * FROM users WHERE id = 123;

-- 3. 使用LIKE以通配符开头
SELECT * FROM users WHERE email LIKE '%@example.com';
-- 解决：使用全文索引或反转索引

-- 4. OR条件
SELECT * FROM users WHERE name = '张三' OR email = 'zhangsan@example.com';
-- 解决：使用UNION ALL或复合索引

-- 5. 对索引列进行运算
SELECT * FROM orders WHERE total / 2 > 100;
-- 解决：改写为 WHERE total > 200;
```

**索引优化策略：**

```sql
-- 1. 选择性高的列优先创建索引
-- 验证列的选择性
SELECT
    count(DISTINCT column_name) / count(*)::float as selectivity
FROM table_name;

-- 2. 复合索引列顺序
-- 查询条件包含 (a, b, c) 时，考虑创建 idx(a, b, c)
-- 最左前缀原则：查询必须包含a才能使用索引

-- 3. 覆盖索引（索引覆盖查询所有列）
CREATE INDEX idx_orders_covering ON orders(user_id, status)
INCLUDE (total_amount, created_at);
-- 解释：查询只需访问索引，无需回表

-- 4. 重建膨胀的索引
REINDEX INDEX CONCURRENTLY idx_users_email;

-- 5. 查看索引建议
-- 使用 pg_stat_user_indexes 识别未使用的索引
SELECT schemaname, tablename, indexname, idx_scan
FROM pg_stat_user_indexes
WHERE idx_scan = 0 AND indexname NOT LIKE '%pkey'
ORDER BY pg_relation_size(indexrelid) DESC;
```

#### 2.4.4 连接池管理

**为什么需要连接池？**

PostgreSQL每个连接是一个独立的进程，创建和销毁连接开销大（几百毫秒）。连接池复用连接，大幅提升性能。

**PgBouncer连接池配置：**

```ini
# pgbouncer.ini
[databases]
myapp = host=127.0.0.1 port=5432 dbname=myapp

[pgbouncer]
listen_addr = 127.0.0.1
listen_port = 6432
auth_type = md5
auth_file = /etc/pgbouncer/userlist.txt
pool_mode = transaction  # 事务模式
max_client_conn = 1000
default_pool_size = 20
min_pool_size = 5
reserve_pool_size = 5
reserve_pool_timeout = 5
max_db_connections = 100
log_connections = 0
log_disconnections = 0
log_pooler_errors = 1
```

**连接池模式对比：**

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| session | 整个会话复用同一连接 | 需要用SET命令、长连接 |
| transaction | 每个事务获取连接 | 大多数应用，推荐 |
| statement | 每个语句获取连接 | 高并发短查询 |

**Node.js连接池最佳实践：**

```typescript
// 连接池配置优化
import { Pool } from 'pg';

const pool = new Pool({
  host: 'localhost',
  port: 5432,
  database: 'myapp',
  user: 'postgres',
  password: 'password',

  // 连接池大小配置
  max: 30,           // 最大连接数（每个Node.js进程）
  idleTimeoutMillis: 30000,  // 空闲连接超时（30秒）
  connectionTimeoutMillis: 5000,  // 获取连接超时（5秒）

  // 性能优化
  statement_timeout: 30000,  // 语句执行超时
  query_timeout: 30000,
});

// 监控连接池状态
setInterval(async () => {
  const { total, idle, waiting } = pool;
  console.log(`连接池: 总数=${total}, 空闲=${idle}, 等待=${waiting}`);
}, 10000);

// 优雅关闭
process.on('SIGTERM', async () => {
  await pool.end();
  process.exit(0);
});

// 推荐的事务写法
async function withTransaction<T>(
  callback: (client: PoolClient) => Promise<T>
): Promise<T> {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    const result = await callback(client);
    await client.query('COMMIT');
    return result;
  } catch (error) {
    await client.query('ROLLBACK');
    throw error;
  } finally {
    client.release();  // 重要：必须释放
  }
}

// 使用示例
const user = await withTransaction(async (client) => {
  await client.query(
    'UPDATE accounts SET balance = balance - $1 WHERE id = $2',
    [amount, fromId]
  );
  await client.query(
    'UPDATE accounts SET balance = balance + $1 WHERE id = $2',
    [amount, toId]
  );
  const result = await client.query(
    'SELECT * FROM accounts WHERE id = $1',
    [fromId]
  );
  return result.rows[0];
});
```

#### 2.4.5 分区表详解

**分区表原理：**

分区表将大表逻辑上拆分为多个物理子表，查询时只扫描相关分区，大幅提升性能。

```sql
-- 创建分区表
CREATE TABLE orders (
    id SERIAL,
    user_id INTEGER NOT NULL,
    total_amount DECIMAL(10, 2) NOT NULL,
    status VARCHAR(20),
    created_at DATE NOT NULL
) PARTITION BY RANGE (created_at);  -- 按范围分区

-- 创建月度分区
CREATE TABLE orders_2024_01 PARTITION OF orders
    FOR VALUES FROM ('2024-01-01') TO ('2024-02-01');

CREATE TABLE orders_2024_02 PARTITION OF orders
    FOR VALUES FROM ('2024-02-01') TO ('2024-03-01');

CREATE TABLE orders_2024_03 PARTITION OF orders
    FOR VALUES FROM ('2024-03-01') TO ('2024-04-01');

-- 创建默认分区（捕获未匹配的数据）
CREATE TABLE orders_default PARTITION OF orders DEFAULT;

-- 在分区表上创建索引（自动应用到所有分区）
CREATE INDEX idx_orders_user_id ON orders(user_id);
CREATE INDEX idx_orders_created_at ON orders(created_at);

-- 查询优化：PostgreSQL自动只扫描相关分区
EXPLAIN SELECT * FROM orders
WHERE created_at BETWEEN '2024-01-15' AND '2024-01-31';

-- 批量插入数据
INSERT INTO orders (user_id, total_amount, status, created_at)
SELECT
    floor(random() * 1000 + 1)::integer,
    (random() * 1000)::decimal(10, 2),
    (ARRAY['pending', 'paid', 'shipped'])[floor(random() * 3 + 1)],
    '2024-01-01'::date + (random() * 90)::integer
FROM generate_series(1, 100000);

-- 查看分区信息
SELECT
    parent.relname as parent_table,
    child.relname as partition,
    pg_size_pretty(pg_relation_size(child.oid)) as size
FROM pg_inherits
JOIN pg_class parent ON pg_inherits.inhparent = parent.oid
JOIN pg_class child ON pg_inherits.inhrelid = child.oid
ORDER BY parent.relname, child.relname;

-- 分离分区（将分区变为独立表）
ALTER TABLE orders DETACH PARTITION orders_2024_01;

-- 附加分区
ALTER TABLE orders ATTACH PARTITION orders_2024_04
    FOR VALUES FROM ('2024-04-01') TO ('2024-05-01');
```

**自动分区管理：**

```sql
-- 创建自动创建分区的函数
CREATE OR REPLACE FUNCTION create_monthly_partition()
RETURNS void AS $$
DECLARE
    partition_date DATE;
    partition_name TEXT;
    start_date TEXT;
    end_date TEXT;
BEGIN
    -- 创建下个月的分区
    partition_date := date_trunc('month', CURRENT_DATE + INTERVAL '1 month')::date;
    partition_name := 'orders_' || to_char(partition_date, 'YYYY_MM');
    start_date := to_char(partition_date, 'YYYY-MM-DD');
    end_date := to_char(partition_date + INTERVAL '1 month', 'YYYY-MM-DD');

    -- 检查分区是否已存在
    IF NOT EXISTS (
        SELECT 1 FROM pg_tables
        WHERE tablename = partition_name
    ) THEN
        EXECUTE format(
            'CREATE TABLE %I PARTITION OF orders FOR VALUES FROM (%L) TO (%L)',
            partition_name, start_date, end_date
        );
        RAISE NOTICE 'Created partition: %', partition_name;
    END IF;
END;
$$ LANGUAGE plpgsql;

-- 创建定时任务自动调用
SELECT cron.schedule(
    'create-monthly-partition',
    '0 0 25 * *',  -- 每月25号执行
    'SELECT create_monthly_partition()'
);
```

```typescript
    where: {
      OR: [
        { name: { contains: query } },
        { email: { contains: query } },
      ],
    },
    orderBy: { createdAt: 'desc' },
    take: 10,
  });

    }),
    prisma.post.create({
      data: { title: postTitle, authorId: 1 },
    }),
  ]);
```

## 三、MongoDB

### 3.1 简介

MongoDB 是最流行的文档型数据库，特点：

- **灵活的模式**：无需预定义表结构
- **水平扩展**：支持分片

### 3.2 基础操作

```javascript
// 创建数据库（自动创建）
use myapp;

// 创建集合（自动创建）
```

### 3.5 MongoDB核心原理

#### 3.5.1 文档模型设计原则

**内嵌文档 vs 引用文档：**

MongoDB的核心设计决策是选择内嵌还是引用。内嵌适合一对一/一对少的关系，引用适合多对多或数据量大的场景。

```javascript
// 场景1：用户收藏的文章（适合内嵌，数量少）
// users集合
{
  "_id": ObjectId("..."),
  "username": "zhangsan",
  "favorites": [
    { "articleId": ObjectId("..."), "title": "MongoDB最佳实践", "savedAt": ISODate("...") },
    { "articleId": ObjectId("..."), "title": "性能优化技巧", "savedAt": ISODate("...") }
  ]  // 收藏文章直接内嵌，避免关联查询
}

// 场景2：订单包含的产品（适合引用，数量大且需要频繁更新）
// orders集合
{
  "_id": ObjectId("..."),
  "orderNumber": "ORD-20240115-001",
  "userId": ObjectId("..."),  // 引用用户
  "items": [
    {
      "productId": ObjectId("..."),  // 引用产品
      "productSnapshot": {  // 快照：保存下单时的产品信息
        "name": "iPhone 15",
        "price": 6999,
        "image": "..."
      },
      "quantity": 1
    }
  ]
}

// 场景3：博客文章与评论（适合引用，但评论需要分页）
// articles集合
{
  "_id": ObjectId("..."),
  "title": "MongoDB性能优化",
  "content": "...",
  "commentCount": 156,  // 评论数冗余，方便快速显示
  "lastCommentAt": ISODate("...")
}

// comments集合（独立集合，支持分页）
{
  "_id": ObjectId("..."),
  "articleId": ObjectId("..."),  // 引用文章
  "authorId": ObjectId("..."),
  "content": "...",
  "createdAt": ISODate("...")
}
```

**设计模式：**


#### 3.5.2 BSON格式详解

BSON（Binary JSON）是MongoDB使用的二进制序列化格式，相比JSON增加了更多数据类型：

```javascript
// BSON支持的数据类型
{
  // 常用类型
  "_id": ObjectId("507f1f77bcf86cd799439011"),  // ObjectId
  "string": "Hello World",                         // UTF-8字符串
  "number": 123.45,                               // Double（64位浮点）
  "integer": 42,                                  // Integer（32/64位）
  "long": NumberLong("123456789"),                // Long
  "decimal": NumberDecimal("123.45"),             // 高精度Decimal
  "boolean": true,                               // Boolean
  "null": null,                                  // Null
  "array": [1, 2, 3],                           // Array
  "object": { "key": "value" },                 // Object

  // 日期和时间
  "date": ISODate("2024-01-15T10:30:00Z"),     // Date（UTC）
  "timestamp": Timestamp(1234567890, 1),         // Timestamp（内部使用）

  // 二进制数据
  "binary": BinData(0, "base64encoded..."),     // Binary

  // 正则表达式
  "regex": /pattern/i,                          // Regular Expression

  // JavaScript代码
  "jsCode": Code("function() { return 1; }"),  // Code（不推荐）

  // MongoDB特定类型
  "minKey": MinKey,                            // 最小值
  "maxKey": MaxKey,                            // 最大值
  "dbRef": DBRef("collection", ObjectId("..."))  // 数据库引用
}

// ObjectId结构（12字节）
// 0-3: 时间戳（4字节）
// 4-6: 机器标识（3字节）
// 7-8: PID（2字节）
// 9-11: 计数器（3字节）

// 生成ObjectId
ObjectId()           // 随机生成
```

#### 3.5.3 WiredTiger存储引擎

WiredTiger是MongoDB 3.2之后的默认存储引擎，提供了MVCC、压缩、缓存等特性：

```javascript
// WiredTiger配置
// mongod.conf
storage:
  dbPath: /var/lib/mongodb
  journal:
    enabled: true  // 启用日志，保证持久性

    engineConfig:
      cacheSizeGB: 4  # 缓存大小，通常为物理内存的50%
      journalCompressor: snappy  # 日志压缩算法
    collectionConfig:
      blockCompressor: snappy  # 数据压缩算法（snappy/zlib/zstd）
    indexConfig:
      prefixCompression: true  # 索引前缀压缩
```

**MVCC机制：**

```javascript
  }},

  // 计算平均订单价值
  { $addFields: {

  }},

  // 排序
  { $sort: { totalRevenue: -1 } },

  // 限制结果
  { $limit: 10 },

  // 投影（只保留需要的字段）
  { $project: {
    _id: 0,
    productId: "$_id",
    productName: 1,
    totalQuantity: 1,
    totalRevenue: { $round: ["$totalRevenue", 2] },
    avgOrderValue: { $round: ["$avgOrderValue", 2] },
    buyerCount: 1
  }}
]);

  { $match: { createdAt: { $gte: ISODate("2024-01-01") } } },

  // 使用$cond进行条件计算
  { $project: {
    orderNumber: 1,
    total: 1,
    discountApplied: {
      $cond: {
        if: { $gt: ["$discount", 0] },
        then: true,
        else: false
      }
    },
    finalAmount: {
      $cond: {
        if: { $gt: ["$discount", 0] },
        then: { $subtract: ["$total", "$discount"] },

      }
    }

    from: "users",

      { $match: { $expr: { $eq: ["$_id", "$$userId"] } } },
      { $project: { username: 1, email: 1 } }

  // 关联产品信息

    from: "products",
    let: { productIds: "$items.productId" },

      { $match: { $expr: { $in: ["$_id", "$$productIds"] } } },
      { $project: { name: 1, category: 1 } }
    ],
    as: "productInfo"
  }},

  // 展开用户信息
  { $unwind: {
    path: "$userInfo",
    preserveNullAndEmptyArrays: true
  }}
]);

// 4. $facet多面查询（一次查询获取多种聚合结果）
db.products.aggregate([
  { $match: { status: "active" } },

  { $facet: {

        _id: "$category",

      { $sort: { count: -1 } }

      { $skip: 0 },
      { $limit: 20 },

      }}
```

```typescript
async function cacheWrapper<T>(
  key: string,
  fetcher: () => Promise<T>,
  ttl: number = 3600
): Promise<T> {
  // 尝试从缓存获取
  const cached = await client.get(key);
```

#### 4.5.4 分布式锁进阶

```typescript
// 完整的分布式锁实现
import { createClient, RedisClientType } from 'redis';

class DistributedLock {
  private client: RedisClientType;
  private lockKey: string;
  private lockValue: string;
  private ttl: number;

  constructor(client: RedisClientType, lockKey: string, ttl: number = 10000) {
    this.client = client;
    this.lockKey = lockKey;
    this.lockValue = `${process.pid}-${Date.now()}-${Math.random()}`;
    this.ttl = ttl;
  }

  // 获取锁（带重试）
  async acquire(maxRetries: number = 3, retryDelay: number = 200): Promise<boolean> {
    for (let i = 0; i < maxRetries; i++) {
      const result = await this.client.set(this.lockKey, this.lockValue, {
        NX: true,  // 只在键不存在时设置
        PX: this.ttl  // 过期时间（毫秒）
      });

      if (result === 'OK') {
        return true;
      }

      // 重试前等待
      await new Promise(resolve => setTimeout(resolve, retryDelay));
    }
    return false;
  }

  // 释放锁（使用Lua脚本保证原子性）
  async release(): Promise<boolean> {
    // Lua脚本：只有锁的值匹配时才删除（防止误删其他客户端的锁）
    const script = `
      if redis.call("get", KEYS[1]) == ARGV[1] then
        return redis.call("del", KEYS[1])
      else
        return 0
      end
    `;

    const result = await this.client.eval(script, {
      keys: [this.lockKey],
      arguments: [this.lockValue]
    });

    return result === 1;
  }

  // 延长锁的过期时间
  async extend(additionalTtl: number): Promise<boolean> {
    // Lua脚本：只有锁的值匹配时才延长过期时间
    const script = `
      if redis.call("get", KEYS[1]) == ARGV[1] then
        return redis.call("pexpire", KEYS[1], ARGV[2])
      else
        return 0
      end
    `;

    const result = await this.client.eval(script, {
      keys: [this.lockKey],
      arguments: [this.lockValue, additionalTtl.toString()]
    });

    return result === 1;
  }
}

// 使用示例
async function processOrder(orderId: string) {
  const lock = new DistributedLock(client, `lock:order:${orderId}`, 30000);

  try {
    // 获取锁
    const acquired = await lock.acquire(3, 200);
    if (!acquired) {
      throw new Error('无法获取锁，订单正在被处理');
    }

    // 执行业务逻辑
    await doProcessOrder(orderId);

    // 延长锁时间（如果处理时间很长）
    await lock.extend(60000);

  } finally {
    // 释放锁
    await lock.release();
  }
}
```

#### 4.5.5 延迟队列实现

```typescript
// 基于Redis有序集合的延迟队列
import { createClient, RedisClientType } from 'redis';

class DelayQueue {
  private client: RedisClientType;
  private queueName: string;

  constructor(client: RedisClientType, queueName: string) {
    this.client = client;
    this.queueName = queueName;
  }

  // 添加延迟任务
  async add<T>(data: T, delayMs: number): Promise<void> {
    const executeAt = Date.now() + delayMs;
    const taskId = `${Date.now()}-${Math.random()}`;

    await this.client.zAdd(this.queueName, {
      score: executeAt,
      value: JSON.stringify({ taskId, data, executeAt })
    });
  }

  // 拉取并执行到期的任务
  async poll<T>(callback: (task: T) => Promise<void>): Promise<number> {
    const now = Date.now();

    // 获取所有到期的任务
    const tasks = await this.client.zRangeByScore(this.queueName, 0, now);

    if (tasks.length === 0) {
      return 0;
    }

    let processed = 0;

    for (const taskStr of tasks) {
      try {
        const { taskId, data } = JSON.parse(taskStr);

        // 原子性地删除任务（使用Lua脚本防止重复处理）
        const script = `
          if redis.call("zscore", KEYS[1], ARGV[1]) then
            return redis.call("zrem", KEYS[1], ARGV[1])
          else
            return 0
          end
        `;

        const removed = await this.client.eval(script, {
          keys: [this.queueName],
          arguments: [taskStr]
        });

        if (removed === 1) {
          await callback(data as T);
          processed++;
        }
      } catch (error) {
        console.error('处理任务失败:', error);
      }
    }

    return processed;
  }

  // 获取队列长度
  async size(): Promise<number> {
    return this.client.zCard(this.queueName);
  }

  // 清理过期任务（物理删除已删除的任务）
  async cleanup(): Promise<void> {
    const now = Date.now();
    await this.client.zRemRangeByScore(this.queueName, '-inf', now - 86400000);
  }
}

// 使用示例

  await delayQueue.add({ type: 'sms', to: '13800138000' }, 10000); // 10秒后执行

  // 轮询处理

    const count = await delayQueue.poll(async (task) => {
      console.log('执行任务:', task);

}
```

```typescript
        { MKSTREAM: true }  // 流不存在时创建
      );
    } catch (error: any) {
      // 消费者组已存在，忽略错误
      if (!error.message.includes('BUSYGROUP')) {
        throw error;
      }
    }
  }

  // 生产消息
  async produce(message: Record<string, string>): Promise<string> {
    const entries = Object.entries(message).map(([k, v]) => [k, v] as [string, string]);
    const messageId = await this.client.xAdd(this.streamName, '*', ...entries);
    return messageId;
  }

  // 批量生产
  async produceBatch(messages: Record<string, string>[]): Promise<string[]> {
    const pipeline = this.client.multi();
    for (const message of messages) {
      const entries = Object.entries(message).flat();
      pipeline.xAdd(this.streamName, '*', ...entries);
    }
    const results = await pipeline.exec();
    return results?.map(r => r[1] as string) || [];
  }

  // 消费消息（自动ACK）
  async consume(count: number = 10, blockMs: number = 5000): Promise<void> {
    // 读取新消息
    const result = await this.client.xReadGroup(
      this.consumerGroup,
      this.consumerName,
      { key: this.streamName, id: '>' },  // '>' 表示新消息
      { count, block: blockMs }
    );

    if (!result) return;

    for (const [stream, messages] of result) {
      for (const [id, fields] of messages) {
        try {
          const message = Object.fromEntries(fields);
          console.log(`处理消息 ${id}:`, message);

          // 业务逻辑
          await processMessage(message);

          // ACK消息
          await this.client.xAck(this.streamName, this.consumerGroup, id);
        } catch (error) {
          console.error(`处理消息 ${id} 失败:`, error);
          // 可选：NACK消息，重新入队
          // await this.client.xNack(this.streamName, this.consumerGroup, id);
        }
      }
    }
  }

  // 获取待处理消息（Pending）
  async getPending(): Promise<number> {
    const info = await this.client.xPending(this.streamName, this.consumerGroup);
    return info.count;
  }

  // claimpending消息（处理超时的消息）
  async claimStaleMessages(minIdleTimeMs: number = 30000): Promise<void> {
    // 获取待处理消息
    const pending = await this.client.xPending(
      this.streamName,
      this.consumerGroup
    );

    if (pending.count === 0) return;

    // 获取具体消息
    const messages = await this.client.xRange(
      this.streamName,
      pending.firstId,
      pending.lastId
    );

    for (const [id, fields] of messages) {
      const idleTime = Date.now() - parseInt(id.split('-')[0]);

      if (idleTime >= minIdleTimeMs) {
        // 认领消息
        const claimed = await this.client.xClaim(
          this.streamName,
          this.consumerGroup,
          this.consumerName,
          minIdleTimeMs,
          id
        );

        if (claimed && claimed.length > 0) {
          console.log(`认领超时消息 ${id}`);
          // 重新处理
        }
      }
    }
  }

  // 查看队列长度
  async size(): Promise<number> {
    const info = await this.client.xLen(this.streamName);
    return info;
  }
}
```

```typescript
// 多种限流算法实现

// 1. 固定窗口限流
async function fixedWindowRateLimit(
  key: string,
  limit: number,
  windowMs: number
): Promise<{ allowed: boolean; remaining: number; resetAt: number }> {
  const now = Date.now();
  const windowKey = `ratelimit:${key}:${Math.floor(now / windowMs)}`;

  const count = await client.incr(windowKey);

  if (count === 1) {
    // 首次访问，设置过期时间
    await client.pExpire(windowKey, windowMs);
  }

  const resetAt = Math.floor(now / windowMs) * windowMs + windowMs;
  const remaining = Math.max(0, limit - count);

  return {
    allowed: count <= limit,
    remaining,
    resetAt
  };
}

// 2. 滑动窗口限流
async function slidingWindowRateLimit(
  key: string,
  limit: number,
  windowMs: number
): Promise<{ allowed: boolean; remaining: number }> {
  const now = Date.now();
  const windowKey = `ratelimit:${key}`;
  const oneMinuteAgo = now - windowMs;

  // 使用Lua脚本保证原子性
  const script = `
    redis.call('ZREMRANGEBYSCORE', KEYS[1], '-inf', ARGV[1])
    local count = redis.call('ZCARD', KEYS[1])
    if count < tonumber(ARGV[3]) then
      redis.call('ZADD', KEYS[1], ARGV[2], ARGV[2] .. ':' .. math.random())
      redis.call('PEXPIRE', KEYS[1], ARGV[4])
      return {1, tonumber(ARGV[3]) - count - 1}
    else
      return {0, 0}
    end
  `;

  const result = await client.eval(script, {
    keys: [windowKey],
    arguments: [oneMinuteAgo.toString(), now.toString(), limit.toString(), windowMs.toString()]
  });

  return {
    allowed: result[0] === 1,
    remaining: result[1] as number
  };
}

// 3. 令牌桶限流
class TokenBucket {
  private client: RedisClientType;
  private key: string;
  private capacity: number;
  private refillRate: number; // 每秒补充的令牌数

  constructor(
    client: RedisClientType,
    key: string,
    capacity: number,
    refillRate: number
  ) {
    this.client = client;
    this.key = `tokenbucket:${key}`;
    this.capacity = capacity;
    this.refillRate = refillRate;
  }

  async allow(tokens: number = 1): Promise<boolean> {
    const script = `
      local key = KEYS[1]
      local capacity = tonumber(ARGV[1])
      local refillRate = tonumber(ARGV[2])
      local now = tonumber(ARGV[3])
      local requested = tonumber(ARGV[4])

      local data = redis.call('HMGET', key, 'tokens', 'lastRefill')
      local tokens = tonumber(data[1]) or capacity
      local lastRefill = tonumber(data[2]) or now

      -- 补充令牌
      local elapsed = (now - lastRefill) / 1000
      local refill = math.floor(elapsed * refillRate)
      tokens = math.min(capacity, tokens + refill)

      if refill > 0 then
        redis.call('HMSET', key, 'tokens', tokens, 'lastRefill', now)
        redis.call('PEXPIRE', key, 3600000)
      end

      -- 消耗令牌
      if tokens >= requested then
        redis.call('HSET', key, 'tokens', tokens - requested)
        return 1
      else
        return 0
      end
    `;

    const result = await this.client.eval(script, {
      keys: [this.key],
      arguments: [
        this.capacity.toString(),
        this.refillRate.toString(),
        Date.now().toString(),
        tokens.toString()
      ]
    });

    return result === 1;
  }
}

// 使用示例
async function rateLimitMiddleware(ctx: any, next: () => Promise<void>) {
  const userId = ctx.user?.id || ctx.ip;

  }

  await next();
}
```

```typescript
// 多级缓存架构
import { createClient, RedisClientType } from 'redis';

interface CacheConfig {
  // L1: 本地内存缓存
  localTtl: number;      // 本地缓存TTL
  localMaxSize: number;  // 本地缓存最大条目数

  // L2: Redis缓存
  redisTtl: number;      // Redis缓存TTL

  // L3: 数据库
  dbQuery: () => Promise<any>;
}

class MultiLevelCache {
  private localCache: Map<string, { value: any; expireAt: number }>;
  private redis: RedisClientType;
  private config: CacheConfig;

  constructor(redis: RedisClientType, config: CacheConfig) {
    this.redis = redis;
    this.config = config;
    this.localCache = new Map();

    // 定期清理过期本地缓存
    setInterval(() => this.cleanupLocalCache(), 60000);
  }

  // 从多级缓存获取数据
  async get<T>(key: string): Promise<T | null> {
    // L1: 本地内存缓存
    const local = this.localCache.get(key);
    if (local && local.expireAt > Date.now()) {
      console.log(`[Cache HIT L1] ${key}`);
      return local.value as T;
    }

    // L2: Redis缓存
    const redisValue = await this.redis.get(`cache:${key}`);
    if (redisValue) {
      console.log(`[Cache HIT L2] ${key}`);
      const data = JSON.parse(redisValue);

      // 回填L1
      this.localCache.set(key, {
        value: data,
        expireAt: Date.now() + this.config.localTtl
      });

      return data as T;
    }

    // L3: 数据库
    console.log(`[Cache MISS] ${key}`);
    const dbData = await this.config.dbQuery();

    if (dbData !== null) {
      // 回填L2
      await this.redis.setEx(
        `cache:${key}`,
        this.config.redisTtl,
        JSON.stringify(dbData)
      );

      // 回填L1
      this.localCache.set(key, {
        value: dbData,
        expireAt: Date.now() + this.config.localTtl
      });
    }

    return dbData;
  }

  // 写入缓存
  async set(key: string, value: any): Promise<void> {
    // 写入L2
    await this.redis.setEx(
      `cache:${key}`,
      this.config.redisTtl,
      JSON.stringify(value)
    );

    // 写入L1
    this.localCache.set(key, {
      value,
      expireAt: Date.now() + this.config.localTtl
    });
  }

  // 删除缓存
  async invalidate(key: string): Promise<void> {
    await this.redis.del(`cache:${key}`);
    this.localCache.delete(key);
  }

  // 批量失效（通配符）
  async invalidatePattern(pattern: string): Promise<void> {
    const keys = await this.redis.keys(`cache:${pattern}`);
    if (keys.length > 0) {
      await this.redis.del(keys);
    }

    // 清理本地缓存
    for (const key of this.localCache.keys()) {
      if (key.includes(pattern.replace('*', ''))) {
        this.localCache.delete(key);
      }
    }
  }

  // 清理过期本地缓存
  private cleanupLocalCache(): void {
    const now = Date.now();
    for (const [key, { expireAt }] of this.localCache.entries()) {
      if (expireAt <= now) {
        this.localCache.delete(key);
      }
    }

    // 如果缓存过大，删除最老的条目
    if (this.localCache.size > this.config.localMaxSize) {
      const entries = Array.from(this.localCache.entries());
      entries.sort((a, b) => a[1].expireAt - b[1].expireAt);

      const toDelete = entries.slice(0, Math.floor(this.config.localMaxSize * 0.2));
      for (const [key] of toDelete) {
        this.localCache.delete(key);
      }
    }
  }
}

// 使用示例
const cache = new MultiLevelCache(redisClient, {
  localTtl: 60000,     // 本地缓存1分钟
  localMaxSize: 1000,  // 最多1000条
  redisTtl: 300,      // Redis缓存5分钟
  dbQuery: async () => {
    // 数据库查询
    return db.users.findFirst({ where: { id: 1 } });
  }
});

const user = await cache.get('user:1');
```

```typescript
// Redis Session存储
import { createClient, RedisClientType } from 'redis';
import crypto from 'crypto';

interface SessionData {
  userId: number;
  username: string;
  email: string;
  roles: string[];
  loginAt: Date;
  [key: string]: any;
}

class RedisSessionStore {
  private client: RedisClientType;
  private prefix: string;
  private ttl: number;

  constructor(client: RedisClientType, ttl: number = 86400) {
    this.client = client;
    this.prefix = 'session:';
    this.ttl = ttl;
  }

  // 创建Session
  async createSession(data: SessionData): Promise<string> {
    const sessionId = this.generateSessionId();
    const key = `${this.prefix}${sessionId}`;

    await this.client.hSet(key, {
      data: JSON.stringify(data),
      createdAt: Date.now().toString()
    });

    await this.client.expire(key, this.ttl);

    return sessionId;
  }

  // 获取Session
  async getSession(sessionId: string): Promise<SessionData | null> {
    const key = `${this.prefix}${sessionId}`;

    const data = await this.client.hGet(key, 'data');
    if (!data) return null;

    return JSON.parse(data);
  }

  // 更新Session
  async updateSession(sessionId: string, data: Partial<SessionData>): Promise<void> {
    const key = `${this.prefix}${sessionId}`;
    const exists = await this.client.exists(key);

    if (!exists) {
      throw new Error('Session不存在');
    }

    const currentData = await this.getSession(sessionId);
    const newData = { ...currentData, ...data };

    await this.client.hSet(key, {
      data: JSON.stringify(newData),
      updatedAt: Date.now().toString()
    });

    // 刷新TTL
    await this.client.expire(key, this.ttl);
  }

  // 删除Session
  async destroySession(sessionId: string): Promise<void> {
    const key = `${this.prefix}${sessionId}`;
    await this.client.del(key);
  }

  // Session续期
  async refreshSession(sessionId: string): Promise<void> {
    const key = `${this.prefix}${sessionId}`;
    await this.client.expire(key, this.ttl);
  }

  // 获取所有活跃Session（管理员功能）
  async getActiveSessions(userId?: number): Promise<SessionData[]> {
    const keys = await this.client.keys(`${this.prefix}*`);
    const sessions: SessionData[] = [];

    for (const key of keys) {
      const data = await this.client.hGet(key, 'data');
      if (data) {
        const session = JSON.parse(data);
        if (!userId || session.userId === userId) {
          sessions.push(session);
        }
      }
    }

    return sessions;
  }

  // 强制登出用户的所有Session
  async logoutUser(userId: number): Promise<number> {
    const sessions = await this.getActiveSessions(userId);
    let count = 0;

    for (const session of sessions) {
      // 通过扫描找到sessionId
      const keys = await this.client.keys(`${this.prefix}*`);
      for (const key of keys) {
        const data = await this.client.hGet(key, 'data');
        if (data) {
          const sessionData = JSON.parse(data);
          if (sessionData.userId === userId) {
            await this.client.del(key);
            count++;
          }
        }
      }
    }

    return count;
  }

  // 生成Session ID
  private generateSessionId(): string {
    return crypto.randomBytes(32).toString('hex');
  }
}

// Express中间件集成
function sessionMiddleware(store: RedisSessionStore) {
  return async (req: any, res: any, next: any) => {
    const sessionId = req.cookies?.sessionId || req.headers['x-session-id'];

    if (sessionId) {
      try {
        const session = await store.getSession(sessionId);
        if (session) {
          req.session = session;
          req.sessionId = sessionId;

          // 自动续期（访问时刷新TTL）
          await store.refreshSession(sessionId);
        } else {
          req.session = null;
        }
      } catch (error) {
        console.error('Session验证失败:', error);
        req.session = null;
      }
    } else {
      req.session = null;
    }

    next();
  };
}
```

```typescript
interface Player {

class Leaderboard {

  private key: string;

  constructor(client: RedisClientType, leaderboardName: string) {
    this.client = client;
    this.key = `leaderboard:${leaderboardName}`;
  }

  // 更新玩家分数
  async updateScore(playerId: string, score: number): Promise<void> {
    await this.client.zAdd(this.key, {
      score,
      value: playerId
    });
  }

  // 增加分数
  async incrementScore(playerId: string, increment: number): Promise<number> {
    const newScore = await this.client.zIncrBy(this.key, increment, playerId);
    return newScore;
  }

  // 获取玩家排名（0-based）
  async getRank(playerId: string): Promise<number | null> {
    const rank = await this.client.zRevRank(this.key, playerId);
    return rank; // null表示玩家不存在
  }

  // 获取玩家排名（1-based）
  async getRankWithScore(playerId: string): Promise<{ rank: number; score: number } | null> {
    const [rank, score] = await Promise.all([
      this.client.zRevRank(this.key, playerId),
      this.client.zScore(this.key, playerId)
    ]);

    if (rank === null || score === null) return null;

    return {
      rank: rank + 1,
      score
    };
  }

  // 获取Top N玩家
  async getTopPlayers(count: number = 10): Promise<Player[]> {
    const results = await this.client.zRangeWithScores(this.key, 0, count - 1, {
      REV: true
    });

    return results.map((item, index) => ({
      id: item.value,
      score: item.score,
      rank: index + 1
    }));
  }

  // 获取玩家的邻居（前后N名）
  async getPlayerNeighbors(
    playerId: string,
    range: number = 2
  ): Promise<{ above: Player[]; below: Player[] }> {
    const rank = await this.getRank(playerId);
    if (rank === null) {
      return { above: [], below: [] };
    }

    const [above, below] = await Promise.all([
      // 排名更高的玩家（分数更高）
      this.client.zRangeWithScores(
        this.key,
        Math.max(0, rank - range),
        rank - 1,
        { REV: true }
      ),
      // 排名更低的玩家（分数更低）
      this.client.zRangeWithScores(
        this.key,
        rank + 1,
        rank + range,
        { REV: true }
      )
    ]);

    return {
      above: above.map((item, index) => ({
        id: item.value,
        score: item.score,
        rank: rank - range + index
      })),
      below: below.map((item, index) => ({
        id: item.value,
        score: item.score,
        rank: rank + 2 + index
      }))
    };
  }

  // 获取指定排名范围的玩家
  async getPlayersByRankRange(
    startRank: number,
    endRank: number
  ): Promise<Player[]> {
    const results = await this.client.zRangeWithScores(
      this.key,
      startRank - 1,
      endRank - 1,
      { REV: true }
    );

    return results.map((item, index) => ({
      id: item.value,
      score: item.score,
      rank: startRank + index
    }));
  }

  // 获取玩家总数
  async getTotalPlayers(): Promise<number> {
    return this.client.zCard(this.key);
  }

  // 获取指定分数段的玩家
  async getPlayersByScoreRange(
    minScore: number,
    maxScore: number
  ): Promise<Player[]> {
    const results = await this.client.zRangeWithScores(
      this.key,
      minScore,
      maxScore,
      { REV: true }
    );

    return results.map(item => ({
      id: item.value,
      score: item.score
    }));
  }

  // 删除玩家
  async removePlayer(playerId: string): Promise<void> {
    await this.client.zRem(this.key, playerId);
  }

  // 清空排行榜
  async clear(): Promise<void> {
    await this.client.del(this.key);
  }

  // 获取玩家排名百分比（百比分）
  async getPercentile(playerId: string): Promise<number | null> {
    const [rank, total] = await Promise.all([
      this.getRank(playerId),
      this.getTotalPlayers()
    ]);

    if (rank === null || total === 0) return null;

    return Math.round((1 - rank / total) * 100);
  }
}
```
