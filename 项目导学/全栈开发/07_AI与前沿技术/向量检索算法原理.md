# 向量检索算法原理

## 一、ANN（近似最近邻）搜索概述

### 1.1 什么是向量检索

向量检索是一种在高维空间中查找与查询向量最相似向量的技术。在 AI 领域，文本、图像、音频等数据通常被转换为高维向量表示（嵌入向量/Embeddings），维度从 128 维到 1536 维甚至更高。

```
传统数据库查询 vs 向量检索：

传统查询：WHERE age > 25 AND city = 'Beijing'
向量查询：找到与向量 [0.12, -0.34, 0.78, ...] 最相似的 Top-K 个向量
```

### 1.2 精确检索的困境

对于精确最近邻搜索（KNN），时间复杂度为 O(n)，意味着需要遍历所有向量进行距离计算。当向量数量达到百万级时，精确搜索变得不切实际：

| 向量规模 | 精确搜索耗时 | ANN 搜索耗时（ms） |
|----------|-------------|-------------------|
| 1M | 1000ms | 5-10ms |
| 10M | 10000ms | 10-20ms |
| 100M | 100000ms | 20-50ms |

### 1.3 ANN 核心思想

近似最近邻（Approximate Nearest Neighbor）通过牺牲少量精度换取巨大的性能提升。核心思想包括：

- **空间划分**：将高维空间划分为多个子空间/簇
- **索引结构**：构建高效的数据结构加速检索
- **剪枝策略**：跳过不可能包含最近邻的区域

### 1.4 距离度量

向量检索中常用的距离度量：

| 度量方式 | 公式 | 适用场景 |
|----------|------|----------|
| **余弦相似度** | cos(θ) = A·B/(\|A\|\|B\|) | 文本嵌入、推荐系统 |
| **欧氏距离** | √Σ(Ai-Bi)² | 图像特征、图像检索 |
| **点积** | A·B = ΣAiBi | 需要考虑向量长度的场景 |

## 二、HNSW 算法原理

### 2.1 算法概述

HNSW（Hierarchical Navigable Small World）是一种基于图的近似最近邻搜索算法，被广泛应用于 Milvus、Pinecone 等向量数据库中。其核心思想是构建一个**多层跳表结构**，实现高效的近邻检索。

**核心特点：**
- 查询时间复杂度：O(log n)
- 支持动态插入
- 无需额外内存用于量化压缩
- 检索精度高

### 2.2 跳表结构

跳表（Skip List）是一种多层链表结构，每一层都是下层的"快速通道"：

```
跳表结构示意：

Layer 2:  [A] ----------------------- [K]
          |                             |
Layer 1:  [A] ------- [E] ----- [J] --- [K]
          |           |         |       |
Layer 0:  [A] - [C] - [E] - [G] - [J] - [K] - [L] - [M]

Layer 0: 完整链表，包含所有节点
Layer 1: 中间层节点，提供快速跳过
Layer 2: 顶层节点，最大跨度跳跃
```

HNSW 将跳表的思想应用于高维向量空间：
- **Layer 0**：包含所有向量，精度最高
- **Layer L**：仅包含少量向量，跨度最大
- **越高层**：节点越稀疏，搜索范围越大

### 2.3 层级构建过程

HNSW 的构建过程是**自顶向下**的，每插入一个向量，按照以下步骤分配层级：

**步骤 1：随机选择最大层级**

使用指数衰减概率分布随机决定向量的最大层级 L：

```
P(L = l) = exp(-l / λ) / (1 - exp(-1/λ))

例如：λ = 1/M（L=0 的概率约为 0.37，L=1 的概率约为 0.37，L=2 的概率约为 0.18...）
```

**实际实现中**：通常使用 `max_l = ceil(log(n))`，其中 n 为向量总数

**步骤 2：分配向量层级**

```
伪代码：
max_level = random_level()  // 随机选择最大层级
for level in range(max_level, 0, -1):
    找到第 level 层中距离最近的邻居
    在该位置插入向量
```

**步骤 3：构建连接（边）**

每个节点在每一层维护一个**邻居列表**（ef_construction 个最近邻），形成图结构：

```
层级连接示意：

Layer 1:
Node A  ------> Node E  ------> Node J
  |               |               |
  v               v               v
Layer 0: [A] - [C] - [E] - [G] - [J] - [K] - [L]
```

### 2.4 搜索过程

HNSW 的搜索过程是**自顶向下**贪婪遍历：

```
搜索算法流程：

1. 从最高层开始，在该层的图中找到距离查询向量最近的节点

2. 以该节点为当前节点，搜索下一层：
   - 在下一层中，找到比当前节点更近的邻居
   - 重复直到最底层（Layer 0）

3. 在 Layer 0 的候选集中进行最终距离计算
   - 返回最近的 K 个结果
```

**搜索过程图解：**

```
查询向量 Q 的搜索路径：

Layer 2:    [E] Q ──────────发现最近邻居 [E]
               │
               ▼
Layer 1:    [E] ───── [F]    贪婪遍历找到 [F]
               │       │
               ▼       ▼
Layer 0: [A]-[C]-[E]-[F]-[G]-[H]-[J]-[K]  最终找到最近邻 [K]
                   ↑
                Q 在此区域
```

### 2.5 关键参数

| 参数 | 含义 | 推荐值 | 影响 |
|------|------|--------|------|
| **M** | 每层邻居数量 | 16-64 | 影响召回率和内存 |
| **efConstruction** | 构建时搜索范围 | 100-400 | 影响索引质量和构建时间 |
| **efSearch** | 搜索时搜索范围 | 50-1000 | 影响召回率和搜索速度 |
| **maxLayer** | 最大层数 | log(n) | 自动计算 |

### 2.6 HNSW 优点与局限

**优点：**
- 查询速度快（O(log n)）
- 索引构建简单，支持增量插入
- 不需要额外的训练过程
- 检索精度高（召回率可达 95%+）

**局限：**
- 内存占用较高（每个向量需要存储多层图结构）
- 不支持带过滤条件的查询
- 索引构建时间较长

## 三、IVF 原理

### 3.1 算法概述

IVF（Inverted File Index，倒排索引）是一种基于聚类的向量检索方法。其核心思想是将向量空间划分为多个聚类（Voronoi 细胞），搜索时只需在少数几个相关聚类中进行精确搜索。

### 3.2 聚类过程

IVF 首先使用 K-Means 等聚类算法将向量集合划分为 K 个聚类：

```
聚类结果示意（2维空间）：

        ● Cluster 0
       / \
      /   \
   ● ●     ● ●
   / Cluster 1
  ●
 /           ● Cluster 2
● ●         / \
             ● ● ●

每个 ● 代表一个向量
每个 Cluster 中心用 ★ 标记
```

**聚类中心（Centroid）** 存储在一个向量列表中，每个向量被分配到其最近的聚类中心。

### 3.3 倒排索引结构

IVF 构建一个**倒排索引**，记录每个聚类包含的所有向量：

```
倒排索引结构：

聚类中心列表: [C0, C1, C2, C3, C4, C5, C6, C7]

倒排列表:
┌─────────┬──────────────────────────────┐
│ Cluster │ 向量ID列表                    │
├─────────┼──────────────────────────────┤
│ C0      │ [v1, v5, v12, v23, ...]      │
│ C1      │ [v2, v8, v15, v34, ...]      │
│ C2      │ [v3, v7, v19, v45, ...]      │
│ C3      │ [v4, v11, v28, v56, ...]     │
│ ...     │ ...                          │
└─────────┴──────────────────────────────┘
```

### 3.4 搜索过程

**步骤 1：找到查询向量所属的聚类**

计算查询向量 Q 与所有聚类中心的距离，找出最近的 N 个聚类（N 为搜索宽度）：

```
nprobe = 4  // 搜索 4 个最近的聚类

查询向量 Q：
距离 Q 最近的聚类：C3, C1, C6, C0（按距离排序）
```

**步骤 2：在选中的聚类中进行精确搜索**

```
对 Cluster C3、C1、C6、C0 中的所有向量
计算与 Q 的精确距离
返回最近的 K 个结果
```

**搜索过程图解：**

```
                ★ C0
            ● ● ● ● ●
         ● ●
        ★ C1 ★ C2
       ● ● ● ● ● ● ●
      ● ● ● Q ● ● ● ●
        ★ C3 ★ C4
       ● ● ● ● ● ●
          ★ C5
         ● ● ●

阴影区域为 nprobe=2 时搜索的聚类
```

### 3.5 关键参数

| 参数 | 含义 | 推荐值 | 影响 |
|------|------|--------|------|
| **nlist** | 聚类数量 | 1000-10000 | 影响召回率和内存 |
| **nprobe** | 搜索聚类数 | 1-256 | 越大召回率越高，速度越慢 |
| **distance_type** | 距离类型 | l2/cosine/ip | 影响距离计算方式 |

### 3.6 IVF 优点与局限

**优点：**
- 支持带过滤条件的查询
- 内存占用相对较低
- 可与 PQ 结合进一步压缩

**局限：**
- 需要预先聚类，不支持动态更新
- 聚类质量影响检索精度
- 高维空间中聚类效果可能下降

## 四、PQ 原理

### 4.1 算法概述

PQ（Product Quantization，乘积量化）是一种向量压缩技术，用于将高维向量压缩为低存储格式。广泛应用于向量数据库的存储优化和加速检索。

### 4.2 向量分块

PQ 首先将原始向量划分为 M 个子向量（sub-vectors）：

```
原始向量 V（128维）划分为 M=8 个子向量：

V = [v0, v1, v2, v3, v4, v5, v6, v7, ..., v127]
    ├───┬───┬───┬───┬───┬───┬───┬───┐
    │ v0 │ v1 │ v2 │ v3 │ v4 │ v5 │ v6 │ v7 │   <- 8 个子向量
    │    │    │    │    │    │    │    │    │
    │16维│16维│16维│16维│16维│16维│16维│16维│   <- 每个子向量 16 维
    └───┴───┴───┴───┴───┴───┴───┴───┘
```

每个子向量独立进行量化。

### 4.3 聚类与编码

对每个子向量空间分别进行 K-Means 聚类：

```
子向量空间 0：
聚类中心: C0 = [c00, c01, ..., c0k]  (K 个中心，每个 16 维)

子向量空间 1：
聚类中心: C1 = [c10, c11, ..., c1k]

...

子向量空间 7：
聚类中心: C7 = [c70, c71, ..., c7k]
```

**编码过程：**

```
原始向量 -> 分块 -> 每个子向量找到最近中心 -> 用中心 ID 替换

向量 V 的第 i 个子向量 vi:
1. 找到最近的聚类中心 Cj
2. 用 j（0 到 K-1 的整数）替换 vi

压缩结果: [j0, j1, j2, j3, j4, j5, j6, j7]  (8 个中心 ID)
```

### 4.4 存储空间对比

| 表示方式 | 每个向量存储空间 | 说明 |
|----------|-----------------|------|
| 原始 float32 | 128 × 4 = 512 bytes | 完全精确 |
| PQ (M=8, K=256) | 8 × 1 = 8 bytes | 压缩 64 倍 |
| PQ (M=16, K=256) | 16 × 1 = 16 bytes | 压缩 32 倍 |
| PQ (M=8, K=65536) | 8 × 2 = 16 bytes | 压缩 32 倍 |

### 4.5 距离计算

由于 PQ 压缩后的向量是聚类中心的 ID，无法精确计算距离。使用**查表+求和**策略：

**SDC（Symmetric Distance Computation）**：仅使用聚类中心计算距离

```
d(V1, V2) ≈ Σ d_sub(C1[i], C2[i])

其中 d_sub 为两个子向量中心之间的距离
```

**ADC（Asymmetric Distance Computation）**：查询向量用原始表示，数据库向量用压缩表示

```
d(V1, V2) ≈ Σ d_sub(V1[i], C2[i])
```

**距离查表：**

```
预计算距离表 D[i][j] = d_sub(查询子向量 Qi, 中心 Cj)

查询时直接查表求和：
距离 ≈ Σ D[i][编码[i]]
```

### 4.6 PQ 优点与局限

**优点：**
- 压缩率高，显著降低内存占用
- 支持快速距离计算（查表）
- 可与 IVF 结合（IVF-PQ）

**局限：**
- 有精度损失
- 需要额外计算距离表
- 不支持动态更新（需要重建索引）

## 五、ANNOY 算法原理

### 5.1 算法概述

ANNOY（Approximate Nearest Neighbors Oh Yeah）是一种基于随机投影树的向量检索算法，由 Spotify 开发。其核心思想是使用多棵随机二叉树来划分向量空间，实现高效的近似最近邻搜索。

### 5.2 随机投影树构建

ANNOY 使用随机超平面来划分向量空间：

**步骤 1：随机选择投影向量**

在向量空间中随机选择一个投影向量 v（通常从训练向量中随机采样或使用高斯随机向量）。

**步骤 2：使用超平面划分**

```
超平面定义：所有满足 v·x = 0 的点 x 组成的集合

划分结果：
- v·x > 0  -> 右子树
- v·x < 0  -> 左子树
```

**树构建过程图解：**

```
原始向量集合
     │
     ▼
随机选择超平面 H1 [v1·x = 0]
     │
     ├──► 左子树（v1·x < 0）──► 随机选择超平面 H2 [v2·x = 0]
     │                              ├──► 左...
     │                              └──► 右...
     │
     └──► 右子树（v1·x > 0）──► 随机选择超平面 H3 [v3·x = 0]
                                    ├──► 左...
                                    └──► 右...
```

**步骤 3：递归构建二叉树**

- 递归地在每个子空间中选择随机超平面
- 直到每个叶子节点包含的向量数量 <= leaf_size
- 构建 N 棵树（通常 N = 50-100）

### 5.3 森林结构

ANNOY 构建多棵树形成森林：

```
ANNOY 森林结构（3棵树示例）：

Tree 0:           Tree 1:           Tree 2:
   ○                 ○                 ○
  / \               / \               / \
 ○   ○             ○   ○             ○   ○
/ \ / \           / \ / \           / \ / \
○  ○ ○  ○        ○  ○ ○  ○        ○  ○ ○  ○

每棵树都是完整的二叉树
所有树的叶子节点包含相同的向量集合
```

### 5.4 搜索过程

ANNOY 的搜索过程是多棵树的**优先级遍历**：

**步骤 1：从所有树的根节点开始**

**步骤 2：使用优先级队列选择最有可能的分支**

```
优先级队列按距离上界排序：
- 计算当前节点到查询向量的最小可能距离
- 优先搜索距离更小的分支
```

**步骤 3：收集候选向量**

```
从每棵树的优先级队列中取出最近的叶子节点
合并所有叶子节点中的向量作为候选集
```

**步骤 4：精确距离计算**

```
在候选集中计算与查询向量的精确距离
返回最近的 K 个结果
```

**搜索过程图解：**

```
查询向量 Q 的搜索路径（单棵树）：

     ○ 距离 Q: 5.2
    / \
   /   \
  ○ 3.1 ○ 6.8        <- 选择左子树
 / \
○ 2.3 ○ 4.5          <- 选择左子节点
 / \
■   ■                 <- 叶子节点

搜索顺序：根节点 -> 左子节点 -> 左左叶子 -> 左右叶子
```

### 5.5 关键参数

| 参数 | 含义 | 推荐值 | 影响 |
|------|------|--------|------|
| **n_trees** | 树的数量 | 50-200 | 越多精度越高，内存越大 |
| **search_k** | 搜索节点数 | n_trees × n_leaves / 2 | 影响召回率和速度 |
| **leaf_size** | 叶子节点大小 | 100-1000 | 影响树的深度和精度 |
| **distance_type** | 距离类型 | euclidean/cosine/manhattan | 影响距离计算方式 |

### 5.6 ANNOY 优点与局限

**优点：**
- 构建速度快，支持动态插入和删除
- 内存占用相对较低
- 支持多种距离度量
- 支持磁盘存储，适合超大规模数据

**局限：**
- 召回率相对较低
- 搜索精度对参数敏感
- 不支持内积（IP）距离的精确优化

## 六、算法对比与适用场景

### 6.1 核心算法对比

| 特性 | HNSW | IVF | PQ | ANNOY |
|------|------|-----|-----|-------|
| **数据结构** | 分层图 | 倒排索引 | 量化编码 | 随机投影树 |
| **查询复杂度** | O(log n) | O(n/K + K) | O(M + K) | O(log n) |
| **内存占用** | 高 | 中 | 低 | 中 |
| **构建速度** | 慢 | 中 | 慢 | 快 |
| **支持更新** | 是 | 否 | 否 | 是 |
| **召回率** | 95%+ | 80-95% | 70-90% | 60-85% |
| **搜索速度** | 最快 | 快 | 最快 | 快 |

### 6.2 组合使用

实际生产环境中，经常将多种算法组合使用：

**IVF-PQ（最常用）：**

```
IVF-PQ 结合了倒排索引和乘积量化的优点：

1. IVF 聚类：将向量空间划分为多个聚类
2. PQ 量化：每个向量压缩为紧凑编码
3. 搜索时：在少数聚类中搜索 PQ 编码

优势：
- 内存占用极低（可压缩到原始大小的 1%
- 搜索速度快
- 支持大规模向量检索
```

**HNSW-PQ：**

```
HNSW 提供高精度检索
PQ 提供向量压缩

适用场景：
- 对召回率要求极高
- 内存资源有限
```

### 6.3 场景选择指南

```
选择向量检索算法的决策树：

你的场景需要支持动态更新吗？
    │
    ├── 是 -> HNSW 或 ANNOY
    │
    └── 否 -> 继续判断

    你的数据规模是多少？
    │
    ├── < 100万 -> HNSW 或 IVF-PQ
    │
    └── > 1000万 -> IVF-PQ 或 ANNOY

    你的内存资源情况？
    │
    ├── 内存充足 -> HNSW（高精度）
    │
    └── 内存有限 -> IVF-PQ（高压缩率）
```

### 6.4 主流向量数据库选用

| 数据库 | 默认算法 | 适用场景 |
|--------|---------|----------|
| **Milvus** | HNSW + IVF-PQ | 企业级大规模检索 |
| **Pinecone** | HNSW | 云原生、高性能 |
| **Qdrant** | HNSW + IVF | 过滤查询支持好 |
| **Weaviate** | HNSW + BM25 | 混合搜索（向量+关键词） |
| **Chroma** | HNSW（可选 IVF-PQ） | 轻量级、嵌入式 |

### 6.5 性能优化建议

**HNSW 调优：**
- 增加 `efSearch` 提高召回率（代价是延迟增加）
- 调整 `M` 平衡精度和内存
- 使用 `normalize` 预处理向量

**IVF-PQ 调优：**
- `nlist` 通常设为 `4 × sqrt(n)`
- `nprobe` 在精度和速度间权衡
- `M` 和 `K` 决定压缩率和精度

**ANNOY 调优：**
- 增加 `n_trees` 提高精度
- 增大 `search_k` 提高召回率

## 七、总结

向量检索是现代 AI 应用的核心技术，HNSW、IVF、PQ、ANNOY 等算法各有优劣：

- **HNSW**：综合性能最佳，是当前主流选择
- **IVF-PQ**：适合超大规模、低内存场景
- **ANNOY**：构建速度快，支持动态更新
- **组合使用**：IVF-PQ 结合了多种算法的优点

在实际应用中，需要根据数据规模、精度要求、内存限制、实时性需求等因素综合选择合适的算法和参数。
