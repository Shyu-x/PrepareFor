## 一、设计原则：SOLID架构基石

SOLID原则是面向对象设计的五大基本原则，由Robert C. Martin（ Uncle Bob）提出，是所有设计模式的基础。

### 1.1 单一职责原则（SRP）

**定义**：一个类应该只有一个引起它变化的原因。

**为什么要遵循SRP**：
- 降低类的复杂度，提高可读性
- 提高类的可维护性
- 减少变更带来的风险

**违反SRP的典型案例**：

```typescript
// ❌ 违反SRP：UserService同时处理用户数据和文件操作
class UserService {
  getUser(id: number) { /* 获取用户 */ }
  saveUser(user: User) { /* 保存用户 */ }
  exportToPDF(user: User) { /* 导出PDF */ }      // 文件操作不应该在这里
  sendEmail(user: User) { /* 发送邮件 */ }       // 通信操作不应该在这里
  generateReport(user: User) { /* 生成报表 */ }   // 报表生成不应该在这里
}

// ✅ 遵循SRP：职责分离
class UserService {
  getUser(id: number) { /* 获取用户 */ }
  saveUser(user: User) { /* 保存用户 */ }
}

class PDFExporter {
  exportToPDF(data: any): Blob { /* 导出PDF */ }
}

class EmailService {
  sendEmail(to: string, content: string) { /* 发送邮件 */ }
}

class ReportGenerator {
  generateReport(data: any): Report { /* 生成报表 */ }
}
```

**面试题：如何判断是否违反SRP？**

当一个类需要因为多个不同的原因而修改时，就违反了SRP。可以通过问自己这个问题来判断："这个类的职责是什么？"如果答案中包含"和"、"以及"，说明类有多个职责。

### 1.2 开闭原则（OCP）

**定义**：软件实体应该对扩展开放，对修改关闭。

**核心思想**：在不修改现有代码的情况下，通过继承、组合等方式扩展功能。

**策略模式实现OCP**：

```typescript
// ❌ 违反OCP：每次添加新折扣都需要修改Order类
class Order {
  calculateDiscount(type: 'student' | 'vip' | 'holiday' | 'newType') {
    if (type === 'student') return 0.1;
    if (type === 'vip') return 0.2;
    if (type === 'holiday') return 0.15;
    if (type === 'newType') return 0.25; // 需要修改原有代码
    return 0;
  }
}

// ✅ 遵循OCP：使用策略模式
interface DiscountStrategy {
  calculate(amount: number): number;
}

class StudentDiscount implements DiscountStrategy {
  calculate(amount: number): number {
    return amount * 0.1;
  }
}

class VIPDiscount implements DiscountStrategy {
  calculate(amount: number): number {
    return amount * 0.2;
  }
}

class HolidayDiscount implements DiscountStrategy {
  calculate(amount: number): number {
    return amount * 0.15;
  }
}

// 添加新折扣无需修改Order类
class Order {
  private discountStrategy: DiscountStrategy;

  constructor(discountStrategy: DiscountStrategy) {
    this.discountStrategy = discountStrategy;
  }

  setDiscountStrategy(strategy: DiscountStrategy) {
    this.discountStrategy = strategy;
  }

  calculateTotal(amount: number): number {
    return amount - this.discountStrategy.calculate(amount);
  }
}

// 使用
const order = new Order(new StudentDiscount());
const discounted = order.calculateTotal(1000);
```

```typescript
  if (!ChartComponent) {
    throw new Error(`Unknown chart type: ${type}`);
  }
  return <ChartComponent data={data} {...props} />;
}
```

### 1.3 里氏替换原则（LSP）

**定义**：子类必须能够替换其基类（父类），而不影响程序的正确性。

**违反LSP的例子**：

```typescript
// ❌ 违反LSP：子类改变了父类的行为契约
class Rectangle {
  constructor(protected width: number, protected height: number) {}

  setWidth(w: number) { this.width = w; }
  setHeight(h: number) { this.height = h; }

  getArea(): number {
    return this.width * this.height;
  }
}

class Square extends Rectangle {
  // 违反LSP：setWidth和setHeight的行为与父类期望不一致
  setWidth(w: number) {
    this.width = w;
    this.height = w; // 改变了正方形的性质
  }

  setHeight(h: number) {
    this.width = h;
    this.height = h;
  }
}

function increaseWidth(rect: Rectangle) {
  rect.setWidth(rect.width + 10);
}

// 如果传入Square，行为将与预期不符
const square = new Square(5, 5);
increaseWidth(square);
// 预期：宽变成15，高仍是5，面积=75
// 实际：宽变成15，高也变成15，面积=225
```

**遵循LSP的设计**：

```typescript
// ✅ 正确的设计：使用接口隔离
interface Shape {
  getArea(): number;
}

interface Scalable {
  scale(factor: number): void;
}

class Rectangle implements Shape, Scalable {
  constructor(public width: number, public height: number) {}

  getArea(): number {
    return this.width * this.height;
  }

  scale(factor: number): void {
    this.width *= factor;
    this.height *= factor;
  }
}

class Square implements Shape, Scalable {
  constructor(public side: number) {}

  getArea(): number {
    return this.side * this.side;
  }

  scale(factor: number): void {
    this.side *= factor;
  }
}

function printArea(shape: Shape) {
  console.log(`Area: ${shape.getArea()}`);
}
```

### 1.4 接口隔离原则（ISP）

**定义**：客户端不应该依赖它不需要的接口。一个类对另一个类的依赖应该建立在最小的接口上。

**违反ISP的例子**：

```typescript
// ❌ 违反ISP：打印机接口过于臃肿
interface IPrinter {
  print(content: string): void;
  scan(content: string): void;      // 有些打印机不支持扫描
  fax(content: string): void;      // 有些打印机不支持传真
  copy(content: string): void;      // 有些打印机不支持复印
  staple(): void;                   // 有些打印机不支持装订
}

class SimplePrinter implements IPrinter {
  print(content: string) { /* 打印 */ }

  // 被迫实现不需要的方法
  scan(content: string) {
    throw new Error('Scan not supported');
  }

  fax(content: string) {
    throw new Error('Fax not supported');
  }

  copy(content: string) {
    throw new Error('Copy not supported');
  }

  staple() {
    throw new Error('Staple not supported');
  }
}
```

**遵循ISP的设计**：

```typescript
// ✅ 遵循ISP：接口细分
interface IPrint {
  print(content: string): void;
}

interface IScan {
  scan(content: string): void;
}

interface IFax {
  fax(content: string): void;
}

interface ICopy {
  copy(content: string): void;
}

interface IStaple {
  staple(): void;
}

// 简单打印机只需要打印功能
class SimplePrinter implements IPrint {
  print(content: string) { /* 打印逻辑 */ }
}

// 多功能打印机组合多个接口
class AdvancedPrinter implements IPrint, IScan, ICopy {
  print(content: string) { /* 打印逻辑 */ }
  scan(content: string) { /* 扫描逻辑 */ }
  copy(content: string) { /* 复印逻辑 */ }
}
```

### 1.5 依赖倒置原则（DIP）

**定义**：高层模块不应该依赖低层模块，两者都应该依赖抽象。

**核心要点**：
- 抽象不应该依赖细节，细节应该依赖抽象
- 针对接口编程，而非针对实现编程

```typescript
// ❌ 违反DIP：高层模块依赖低层模块
class MySQLDatabase {
  connect() { /* MySQL连接 */ }
  query(sql: string) { /* MySQL查询 */ }
}

class UserService {
  private db: MySQLDatabase; // 直接依赖具体实现

  constructor() {
    this.db = new MySQLDatabase();
  }

  getUser(id: number) {
    return this.db.query(`SELECT * FROM users WHERE id = ${id}`);
  }
}
```

**遵循DIP的设计**：

```typescript
// ✅ 遵循DIP：依赖抽象
// 定义数据库抽象接口
interface IDatabase {
  connect(): void;
  query(sql: string): any;
}

// 低层模块实现接口
class MySQLDatabase implements IDatabase {
  connect() { /* MySQL连接 */ }
  query(sql: string) { /* MySQL查询 */ }
}

class PostgreSQLDatabase implements IDatabase {
  connect() { /* PostgreSQL连接 */ }
  query(sql: string) { /* PostgreSQL查询 */ }
}

class MongoDBDatabase implements IDatabase {
  connect() { /* MongoDB连接 */ }
  query(sql: string) { /* MongoDB查询 */ }
}

// 高层模块依赖抽象
class UserService {
  private db: IDatabase; // 依赖抽象，而非具体实现

  constructor(database: IDatabase) {
    this.db = database;
  }

  getUser(id: number) {
    return this.db.query(`SELECT * FROM users WHERE id = ${id}`);
  }
}

// 使用依赖注入容器或手动注入
const userService = new UserService(new MySQLDatabase());
// 轻松切换到其他数据库
const userService2 = new UserService(new PostgreSQLDatabase());
```

**React中的DIP实践**：

## 二、创建型模式：对象的创建艺术

创建型模式关注如何创建对象，封装创建过程，使系统独立于具体类型的创建方式。

### 2.1 单例模式（Singleton）

**定义**：确保一个类只有一个实例，并提供一个全局访问点。

**应用场景**：
- 数据库连接池
- 日志记录器
- 全局配置管理器
- 缓存管理器

**TypeScript单例模式实现**：

```typescript
// 方式一：懒汉式（延迟初始化）
class Database {
  private static instance: Database;
  private connection: any;

  // 私有构造函数，防止直接实例化
  private constructor() {
    this.connection = '数据库连接已建立';
    console.log('数据库连接创建');
  }

  // 获取唯一实例的静态方法
  static getInstance(): Database {
    if (!Database.instance) {
      Database.instance = new Database();
    }
    return Database.instance;
  }

  // 数据库操作方法
  query(sql: string) {
    console.log(`执行SQL: ${sql}`);
    return [{ id: 1, name: '用户' }];
  }
}

// 使用
const db1 = Database.getInstance();
const db2 = Database.getInstance();
console.log(db1 === db2); // true

// 方式二：饿汉式（立即初始化）
class ConfigManager {
  private static instance = new ConfigManager();
  private config: Record<string, any>;

  private constructor() {
    this.config = {
      apiUrl: process.env.API_URL,
      timeout: 5000,
      maxRetries: 3
    };
  }

  static getInstance(): ConfigManager {
    return ConfigManager.instance;
  }

  get(key: string) {
    return this.config[key];
  }
}

// 方式三：TypeScript实现单例的最佳方式（利用ES6模块特性）
class Logger {
  private static instance: Logger;

  private constructor() {}

  static getInstance(): Logger {
    if (!Logger.instance) {
      Logger.instance = new Logger();
    }
    return Logger.instance;
  }

  log(message: string, level: 'info' | 'warn' | 'error' = 'info') {
    const timestamp = new Date().toISOString();
    console.log(`[${timestamp}] [${level.toUpperCase()}] ${message}`);
  }
}

// 方式四：使用Proxy实现单例（代理模式+单例模式）
function singleton<T extends { constructor: new (...args: any[]) => T }>(TargetClass: T): T {
  let instance: T | null = null;

  return new Proxy(TargetClass, {
    construct(target, args) {
      if (!instance) {
        instance = new target(...args);
      }
      return instance;
    }
  }) as any;
}

class CacheManager {
  private cache = new Map();

  constructor() {
    console.log('CacheManager初始化');
  }

  set(key: string, value: any) {
    this.cache.set(key, value);
  }

  get(key: string) {
    return this.cache.get(key);
  }
}

const cachedManager = singleton(CacheManager);
const manager1 = new cachedManager();
const manager2 = new cachedManager();
console.log(manager1 === manager2); // true
```

**React中的单例模式应用**：

```typescript
// EventBus事件总线（单例）
class EventBus {
  private static instance: EventBus;
  private events: Map<string, Set<Function>> = new Map();

  private constructor() {}

  static getInstance(): EventBus {
    if (!EventBus.instance) {
      EventBus.instance = new EventBus();
    }
    return EventBus.instance;
  }

  // 订阅事件
  on(event: string, callback: Function, context?: any) {
    if (!this.events.has(event)) {
      this.events.set(event, new Set());
    }
    // 绑定上下文，防止this丢失
    const boundCallback = context ? callback.bind(context) : callback;
    this.events.get(event)!.add(boundCallback);
  }

  // 发布事件
  emit(event: string, data?: any) {
    this.events.get(event)?.forEach(callback => callback(data));
  }

  // 取消订阅
  off(event: string, callback: Function) {
    this.events.get(event)?.delete(callback);
  }
}

// 使用EventBus
const eventBus = EventBus.getInstance();

// 组件A订阅事件
eventBus.on('user:login', (user) => {
  console.log('用户登录:', user);
});

// 组件B发布事件
eventBus.emit('user:login', { id: 1, name: '张三' });

// 全局状态管理器（类Redux）
class GlobalState {
  private static instance: GlobalState;
  private state: Record<string, any> = {};
  private listeners: Set<() => void> = new Set();

  private constructor() {}

  static getInstance(): GlobalState {
    if (!GlobalState.instance) {
      GlobalState.instance = new GlobalState();
    }
    return GlobalState.instance;
  }

  getState() {
    return { ...this.state };
  }

  setState(newState: Partial<Record<string, any>>) {
    this.state = { ...this.state, ...newState };
    this.notify();
  }

  subscribe(listener: () => void) {
    this.listeners.add(listener);
    return () => this.listeners.delete(listener);
  }

  private notify() {
    this.listeners.forEach(listener => listener());
  }
}
```

```typescript
// 问题：多线程环境下可能创建多个实例
// 解决方案一：双重检查锁定（DCL）
class ThreadSafeSingleton {
  private static instance: ThreadSafeSingleton | null = null;
  private static lock = new Object();

  private constructor() {}

  static getInstance(): ThreadSafeSingleton {
    // 第一重检查
    if (this.instance === null) {
      // 加锁
      synchronized (this.lock) {
        // 第二重检查
        if (this.instance === null) {
          this.instance = new ThreadSafeSingleton();
        }
      }
    }
    return this.instance;
  }
}
```

### 2.2 工厂模式（Factory）

**定义**：定义一个创建对象的接口，让子类决定实例化哪个类。

**三种工厂模式**：

```typescript
// 1. 简单工厂（Simple Factory）
// 不属于GoF设计模式，但实践中非常常用
class UserFactory {
  // 静态方法创建用户
  static create(type: 'admin' | 'user' | 'guest'): User {
    switch (type) {
      case 'admin':
        return new AdminUser();
      case 'user':
        return new NormalUser();
      case 'guest':
        return new GuestUser();
      default:
        throw new Error(`Unknown user type: ${type}`);
    }
  }
}

// 使用简单工厂
const admin = UserFactory.create('admin');
const user = UserFactory.create('user');

// 2. 工厂方法（Factory Method）
// 定义创建对象的接口，让子类决定创建哪个类
```

```typescript
        return cached.value;
      }

      const result = originalMethod.apply(this, args);
      cache.set(key, { value: result, timestamp: Date.now() });
      return result;
    };

    return descriptor;
  };
}
```

## 四、行为型模式：对象间的交互模式

### 4.1 观察者模式（Observer）

**定义**：定义对象间的一种一对多依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知。

**核心组成**：
- Subject（目标/主题）：维护观察者列表，提供添加/删除观察者的方法
- Observer（观察者）：定义更新接口

```typescript
// 1. 简单观察者实现
class EventEmitter {
  private events: Map<string, Set<Function>> = new Map();

  // 订阅事件
  on(event: string, callback: Function, context?: any): () => void {
    if (!this.events.has(event)) {
      this.events.set(event, new Set());
    }

    // 绑定上下文，防止this丢失
    const boundCallback = context
      ? callback.bind(context)
      : callback;

    this.events.get(event)!.add(boundCallback);

    // 返回取消订阅的函数
    return () => this.off(event, boundCallback);
  }

  // 取消订阅
  off(event: string, callback: Function) {
    this.events.get(event)?.delete(callback);
  }

  // 触发事件（单播）
  emit(event: string, data?: any) {
    this.events.get(event)?.forEach(callback => callback(data));
  }

  // 触发事件（广播给所有事件）
  broadcast(event: string, data?: any) {
    this.events.forEach((callbacks, e) => {
      if (e.startsWith(event)) {
        callbacks.forEach(cb => cb(data));
      }
    });
  }

  // 订阅一次
  once(event: string, callback: Function, context?: any): () => void {
    const wrappedCallback = (data: any) => {
      callback.call(context, data);
      this.off(event, wrappedCallback);
    };
    return this.on(event, wrappedCallback);
  }
}

// 2. 可观察状态对象
class ObservableStore<T extends object> {
  private state: T;
  private listeners: Set<(state: T) => void> = new Set();
  private equalityFn: (a: T, b: T) => boolean;

  constructor(initialState: T, equalityFn?: (a: T, b: T) => boolean) {
    this.state = initialState;
    // 默认使用浅比较
    this.equalityFn = equalityFn || ((a, b) => a === b);
  }

  // 获取当前状态
  getState(): T {
    return { ...this.state } as T;
  }

  // 更新状态
  setState(updater: T | ((prev: T) => T)) {
    const prevState = this.state;
    const newState = typeof updater === 'function'
      ? (updater as Function)(prevState)
      : updater;

    // 使用相等函数比较，避免不必要的更新
    if (!this.equalityFn(prevState, newState)) {
      this.state = newState;
      this.notify();
    }
  }

  // 订阅状态变化
  subscribe(listener: (state: T) => void): () => void {
    this.listeners.add(listener);
    // 返回取消订阅函数
    return () => this.listeners.delete(listener);
  }

  // 通知所有监听器
  private notify() {
    this.listeners.forEach(listener => listener(this.state));
  }
}

// 使用ObservableStore

  user: User | null;
  isLoading: boolean;
  error: string | null;
```

```typescript
// 自定义Hooks实现观察者
function useObservable<T>(
  observable: ObservableStore<T>,
  selector: (state: T) => any = (state) => state
) {
  const [value, setValue] = React.useState(() => selector(observable.getState()));

  React.useEffect(() => {
    const unsubscribe = observable.subscribe((state) => {
      const selectedValue = selector(state);
      setValue(prev => {
        // 使用Object.is进行比较，避免不必要的更新
        return Object.is(prev, selectedValue) ? prev : selectedValue;
      });
    });

    return unsubscribe;
  }, [observable, selector]);

  return value;
}
```

```typescript
  message?: string;

class RequiredValidator implements ValidationStrategy {
  validate(value: any): ValidationResult {
    const valid = value !== null && value !== undefined && value !== '';
    return {
      valid,
      message: valid ? undefined : '此字段必填'
    };
  }
}

class EmailValidator implements ValidationStrategy {
  validate(value: string): ValidationResult {
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    const valid = emailRegex.test(value);
    return {
      valid,
      message: valid ? undefined : '请输入有效的邮箱地址'
    };
  }

class LengthValidator implements ValidationStrategy {
  constructor(
    private min: number,
    private max: number
  ) {}

  validate(value: string): ValidationResult {
    const valid = value.length >= this.min && value.length <= this.max;
    return {
      valid,
      message: valid
        ? undefined
        : `长度必须在${this.min}到${this.max}之间`
    };
  }
}

class PatternValidator implements ValidationStrategy {
  constructor(private regex: RegExp, private message: string) {}

  validate(value: string): ValidationResult {
    const valid = this.regex.test(value);
    return { valid, message: valid ? undefined : this.message };
  }
}

// 表单验证器
class FormValidator {
  private validators: Map<string, ValidationStrategy[]> = new Map();

  addValidator(field: string, validator: ValidationStrategy) {
    if (!this.validators.has(field)) {
      this.validators.set(field, []);
    }
    this.validators.get(field)!.push(validator);
  }

  validate(data: Record<string, any>): Record<string, string> {
    const errors: Record<string, string> = {};

    this.validators.forEach((validators, field) => {
      for (const validator of validators) {
        const result = validator.validate(data[field]);
        if (!result.valid) {
          errors[field] = result.message!;
          break; // 遇到第一个错误就停止
        }
      }
    });

    return errors;
  }
}
```

### 4.3 命令模式（Command）

**定义**：将请求封装成对象，从而允许你用不同的请求参数化客户，对请求排队或记录请求日志。

**核心价值**：
- 解耦请求发送者和接收者
- 支持撤销/重做操作
- 支持命令队列和日志记录

```typescript
// 1. 命令接口
interface Command {
  execute(): void;
  undo(): void;
  redo(): void;
}

// 2. 具体命令
class AddItemCommand implements Command {
  constructor(
    private list: string[],
    private item: string
  ) {}

  execute() {
    this.list.push(this.item);
  }

  undo() {
    const index = this.list.indexOf(this.item);
    if (index > -1) {
      this.list.splice(index, 1);
    }
  }

  redo() {
    this.execute();
  }
}

class RemoveItemCommand implements Command {
  private removedItem?: string;
  private removedIndex?: number;

  constructor(
    private list: string[],
    private item: string
  ) {}

  execute() {
    const index = this.list.indexOf(this.item);
    if (index > -1) {
      this.removedItem = this.list[index];
      this.removedIndex = index;
      this.list.splice(index, 1);
    }
  }

  undo() {
    if (this.removedIndex !== undefined && this.removedItem !== undefined) {
      this.list.splice(this.removedIndex, 0, this.removedItem);
    }
  }

  redo() {
    this.execute();
  }
}

class UpdateItemCommand implements Command {
  private oldValue?: string;

  constructor(
    private list: string[],
    private index: number,
    private newValue: string
  ) {}

  execute() {
    if (this.index >= 0 && this.index < this.list.length) {
      this.oldValue = this.list[this.index];
      this.list[this.index] = this.newValue;
    }
  }

  undo() {
    if (this.oldValue !== undefined) {
      this.list[this.index] = this.oldValue;
    }
  }

  redo() {
    this.execute();
  }
}

// 3. 命令管理器（调用者）
class CommandManager {
  private history: Command[] = [];
  private redoStack: Command[] = [];
  private maxHistorySize: number;

  constructor(maxHistorySize: number = 100) {
    this.maxHistorySize = maxHistorySize;
  }

  executeCommand(command: Command) {
    command.execute();
    this.history.push(command);

    // 限制历史记录数量
    if (this.history.length > this.maxHistorySize) {
      this.history.shift();
    }

    // 新命令清空重做栈
    this.redoStack = [];
  }

  undo() {
    const command = this.history.pop();
    if (command) {
      command.undo();
      this.redoStack.push(command);
    }
  }

  redo() {
    const command = this.redoStack.pop();
    if (command) {
      command.redo();
      this.history.push(command);
    }
  }

  canUndo(): boolean {
    return this.history.length > 0;
  }

  canRedo(): boolean {
    return this.redoStack.length > 0;
  }

  getHistory(): Command[] {
    return [...this.history];
  }
}

// 使用
const todoList = ['学习React', '学习TypeScript'];
const manager = new CommandManager();

// 添加命令
const addCommand = new AddItemCommand(todoList, '学习Node.js');
manager.executeCommand(addCommand);
console.log(todoList); // ['学习React', '学习TypeScript', '学习Node.js']

// 撤销
manager.undo();
console.log(todoList); // ['学习React', '学习TypeScript']

// 重做
manager.redo();
console.log(todoList); // ['学习React', '学习TypeScript', '学习Node.js']

// 删除命令
const removeCommand = new RemoveItemCommand(todoList, '学习TypeScript');
manager.executeCommand(removeCommand);
console.log(todoList); // ['学习React', '学习Node.js']

// 修改命令

manager.executeCommand(updateCommand);
console.log(todoList); // ['精通React', '学习Node.js']
```

```typescript
// 1. useLocalStorage - 本地存储Hook
function useLocalStorage<T>(key: string, initialValue: T) {
  const [storedValue, setStoredValue] = useState<T>(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      console.error(error);
      return initialValue;
    }
  });

  const setValue = (value: T | ((val: T) => T)) => {
    try {
      const valueToStore = value instanceof Function ? value(storedValue) : value;
      setStoredValue(valueToStore);
      window.localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.error(error);
    }
  };

  return [storedValue, setValue] as const;
}

// 2. useDebounce - 防抖Hook
function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState<T>(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => clearTimeout(handler);
  }, [value, delay]);

  return debouncedValue;
}

// 搜索示例
function SearchComponent() {
  const [query, setQuery] = useState('');
  const debouncedQuery = useDebounce(query, 300); // 300ms防抖

  useEffect(() => {
    if (debouncedQuery) {
      searchAPI(debouncedQuery);
    }
  }, [debouncedQuery]);

  return <input value={query} onChange={e => setQuery(e.target.value)} />;
}

// 3. useAsync - 异步数据获取Hook
interface AsyncState<T> {
  data: T | null;
  loading: boolean;
  error: Error | null;
}

function useAsync<T>(
  asyncFn: () => Promise<T>,
  immediate: boolean = true
): AsyncState<T> & { execute: () => void } {
  const [state, setState] = useState<AsyncState<T>>({
    data: null,
    loading: immediate,
    error: null
  });

  const execute = useCallback(async () => {
    setState(prev => ({ ...prev, loading: true, error: null }));

    try {
      const data = await asyncFn();
      setState({ data, loading: false, error: null });
    } catch (error) {
      setState({ data: null, loading: false, error: error as Error });
    }
  }, [asyncFn]);

  useEffect(() => {
    if (immediate) {
      execute();
    }
  }, [execute, immediate]);

  return { ...state, execute };
}

// 使用
function UserProfile({ userId }: { userId: string }) {
  const { data: user, loading, error, execute } = useAsync(
    () => fetch(`/api/users/${userId}`).then(r => r.json()),
    true
  );

  if (loading) return <div>加载中...</div>;
  if (error) return <div>错误: {error.message}</div>;

  return (
    <div>
      <h1>{user.name}</h1>
      <button onClick={execute}>刷新</button>
    </div>
  );
}
```

### 6.1 中间件模式

```typescript
// Express中间件模式
interface Middleware {
  (req: Request, res: Response, next: NextFunction): void;
}

// 日志中间件
function loggerMiddleware(req: Request, res: Response, next: NextFunction) {
  const start = Date.now();

  res.on('finish', () => {
    const duration = Date.now() - start;
    console.log(`${req.method} ${req.path} - ${res.statusCode} (${duration}ms)`);
  });

  next();
}

// 认证中间件
function authMiddleware(req: Request, res: Response, next: NextFunction) {
  const token = req.headers.authorization?.split(' ')[1];

  if (!token) {
    return res.status(401).json({ error: '未授权' });
  }

  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET!);
    (req as any).user = decoded;
    next();
  } catch (error) {
    return res.status(401).json({ error: 'Token无效' });
  }
}

// 错误处理中间件
function errorHandler(
  err: Error,
  req: Request,
  res: Response,
  next: NextFunction
) {
  console.error('错误:', err);

  if (err instanceof ValidationError) {
    return res.status(400).json({ error: err.message });
  }

  if (err instanceof NotFoundError) {
    return res.status(404).json({ error: err.message });
  }

  res.status(500).json({ error: '服务器内部错误' });
}

// 中间件组合
function compose(...middlewares: Middleware[]): Middleware {
  return (req, res, next) => {
    let index = 0;

    function run() {
      if (index >= middlewares.length) {
        return next();
      }
      middlewares[index++](req, res, run);
    }

    run();
  };
}

// 使用
const app = express();

app.use(loggerMiddleware);
app.use(compose(authMiddleware, rateLimitMiddleware));

app.get('/api/protected', authMiddleware, (req, res) => {
  res.json({ message: '受保护的路由', user: (req as any).user });
});

app.use(errorHandler);
```

### 6.2 管道模式

```typescript
// 数据处理管道
interface PipelineContext {
  data: any;
  errors: string[];
  metadata: Record<string, any>;
}

type PipeFunction = (ctx: PipelineContext) => PipelineContext;

// 管道辅助函数
function createPipeline(...pipes: PipeFunction[]) {
  return (initialData: any) => {
    const ctx: PipelineContext = {
      data: initialData,
      errors: [],
      metadata: {}
    };

    for (const pipe of pipes) {
      try {
        pipe(ctx);
      } catch (error) {
        ctx.errors.push((error as Error).message);
      }
    }

    return ctx;
  };
}

// 数据处理管道函数
const validateInput: PipeFunction = (ctx) => {
  if (!ctx.data) {
    ctx.errors.push('数据不能为空');
  }
  return ctx;
};

const normalizeData: PipeFunction = (ctx) => {
  if (ctx.data.name) {
    ctx.data.name = ctx.data.name.trim().toLowerCase();
  }
  return ctx;
};

const enrichData: PipeFunction = (ctx) => {
  ctx.data.createdAt = new Date().toISOString();
  ctx.metadata.processed = true;
  return ctx;
};

const saveToDatabase: PipeFunction = (ctx) => {
  // 模拟数据库保存
  console.log('保存数据到数据库:', ctx.data);
  ctx.metadata.saved = true;
  return ctx;
};

// 使用管道
const pipeline = createPipeline(
  validateInput,
  normalizeData,
  enrichData,
  saveToDatabase
);

const result = pipeline({ name: '  张三  ', email: 'zhang@example.com' });

console.log('处理结果:', result.data);
console.log('元数据:', result.metadata);
console.log('错误:', result.errors);
```

```typescript
// CQRS - 命令查询职责分离

// 命令端（写入）
interface Command {
  type: string;
  payload: any;
}

class CreateUserCommand implements Command {
  type = 'CREATE_USER';
  constructor(public payload: { name: string; email: string }) {}
}

class UpdateUserCommand implements Command {
  type = 'UPDATE_USER';
  constructor(public payload: { id: string; name: string }) {}
}

class CommandHandler {
  handle(command: Command) {
    switch (command.type) {
      case 'CREATE_USER':
        return this.createUser(command.payload);
      case 'UPDATE_USER':
        return this.updateUser(command.payload);
    }
  }

  private createUser(data: { name: string; email: string }) {
    console.log('创建用户:', data);
    return { id: crypto.randomUUID(), ...data, createdAt: new Date() };
  }

  private updateUser(data: { id: string; name: string }) {
    console.log('更新用户:', data);
    return data;
  }
}

// 查询端（读取）
interface Query {
  type: string;
  payload: any;
}

class GetUserByIdQuery implements Query {
  type = 'GET_USER_BY_ID';
  constructor(public payload: { id: string }) {}
}

class SearchUsersQuery implements Query {
  type = 'SEARCH_USERS';
  constructor(public payload: { keyword: string }) {}
}

class QueryHandler {
  private readModels: Map<string, any> = new Map();

  handle(query: Query) {
    switch (query.type) {
      case 'GET_USER_BY_ID':
        return this.getUserById(query.payload);
      case 'SEARCH_USERS':
        return this.searchUsers(query.payload);
    }
  }

  private getUserById(data: { id: string }) {
    return this.readModels.get(`user:${data.id}`);
  }

  private searchUsers(data: { keyword: string }) {
    const all = Array.from(this.readModels.values());
    return all.filter(u =>
      u.name.toLowerCase().includes(data.keyword.toLowerCase())
    );
  }

  // 更新读取模型
  updateReadModel(id: string, data: any) {
    this.readModels.set(`user:${id}`, data);
  }
}
```

## 八、面试高频问题

### 问题1：单例模式的线程安全问题

**回答要点**：
- Java、C#等语言需要考虑线程安全
- 解决方案：双重检查锁定、静态初始化、饿汉式
- TypeScript/JavaScript是单线程语言，无需考虑

### 问题2：装饰器和继承的区别

| 特性 | 装饰器模式 | 继承 |
|------|-----------|------|
| 灵活性 | 运行时动态添加 | 编译时静态确定 |
| 复用性 | 可叠加多个装饰器 | 单一继承链 |
| 侵入性 | 不改变原类 | 改变类的结构 |
| 透明度 | 对客户端透明 | 可能改变父类行为 |

### 问题3：策略模式和状态模式的区别

| 特性 | 策略模式 | 状态模式 |
|------|---------|---------|
| 意图 | 算法可互换 | 状态决定行为 |
| 上下文角色 | 不感知状态变化 | 持有状态对象 |
| 状态关系 | 策略间无关联 | 状态间可能有转换关系 |
| 修改影响 | 上下文不变 | 可能影响状态转换 |

### 问题4：观察者模式和发布订阅的区别

| 特性 | 观察者模式 | 发布订阅模式 |
|------|-----------|-------------|
| 耦合度 | 松耦合 | 更松耦合 |
| 通信方式 | 发布者直接通知订阅者 | 通过Broker中转 |
| 复杂度 | 简单 | 较复杂 |
| 适用场景 | 紧密关联的对象 | 需要解耦的系统 |

设计模式是软件工程中的重要知识体系，掌握这些模式不仅能提升代码质量，更能帮助我们在面试中展示深厚的技术功底。记住以下关键点：
