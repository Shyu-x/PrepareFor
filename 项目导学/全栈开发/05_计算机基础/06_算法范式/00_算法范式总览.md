# 算法范式总览

## 一、算法范式概述

算法范式（Algorithm Paradigm）是指解决问题的一种通用方法或思想框架，它提供了一套解决问题的策略和模式。掌握算法范式能够让我们在面对新问题时快速找到思路，将未知问题转化为已知模式来求解。

### 1.1 为什么需要算法范式？

```
问题场景                           적용 算法范式
─────────────────────────────────────────────────────
在解空间中寻找所有可能的解          暴力搜索、回溯
寻找最优解                         贪心、动态规划、分支限界
分解问题为子问题                   分治
在有序数据中查找                   二分搜索
在数组/字符串中双方向遍历           双指针、滑动窗口
```

### 1.2 六大核心算法范式

| 范式 | 核心思想 | 适用问题类型 | 典型应用 |
|------|----------|--------------|----------|
| **暴力搜索** | 枚举所有可能性 | 解空间较小 | 字符串匹配、简单遍历 |
| **贪心算法** | 局部最优推全局最优 | 最优子结构、无后效性 | 活动选择、Huffman编码 |
| **分治算法** | 分而治之、递归合并 | 子问题独立、可合并 | 归并排序、快速排序 |
| **动态规划** | 记录子问题、避免重复计算 | 最优子结构、重叠子问题 | 背包问题、LIS、LCS |
| **回溯算法** | 试探+回退+剪枝 | 组合优化、排列问题 | 八皇后、全排列 |
| **分支限界** | 搜索+剪枝+上界/下界 | 组合优化、离散问题 | 0-1背包、旅行商 |

## 二、算法范式分类详解

### 2.1 暴力搜索（Brute Force）

**核心思想**：枚举所有可能的解，逐一验证找出正确答案。

**特点**：
- 最简单、最直接的解法
- 时间复杂度通常较高（指数级或阶乘级）
- 作为其他算法的 baseline

**代表算法**：
- 简单字符串匹配：BF算法 O(mn)
- 数组全排列：O(n!)
- 线性扫描：O(n)

```typescript
// 暴力搜索示例：在数组中寻找目标值
function bruteForceSearch(arr: number[], target: number): number {
  // 遍历数组的每一个元素
  for (let i = 0; i < arr.length; i++) {
    // 如果当前元素等于目标值，返回索引
    if (arr[i] === target) {
      return i;
    }
  }
  // 未找到返回-1
  return -1;
}
```

### 2.2 贪心算法（Greedy）

**核心思想**：每一步都做出当前最优的选择，寄希望于局部最优能导致全局最优。

**特点**：
- 不回退、不后悔
- 需要证明贪心策略的正确性
- 通常是最优解或近似最优解

**代表算法**：
- 活动选择问题
- 哈夫曼编码
- 最小生成树（Prim、Kruskal）
- 单源最短路（Dijkstra）

```typescript
// 贪心算法示例：活动选择问题
function activitySelection(activities: [number, number][]): number {
  // 按结束时间排序
  activities.sort((a, b) => a[1] - b[1]);

  let count = 1; // 选择第一个活动
  let lastEnd = activities[0][1];

  // 选择下一个不冲突且结束时间最早的活动
  for (let i = 1; i < activities.length; i++) {
    if (activities[i][0] >= lastEnd) {
      count++;
      lastEnd = activities[i][1];
    }
  }

  return count;
}
```

### 2.3 分治算法（Divide and Conquer）

**核心思想**：将原问题分解为若干个规模较小的相同问题，递归解决后合并。

**特点**：
- 递归结构
- 子问题相互独立
- 需要合并步骤

**代表算法**：
- 二分查找
- 归并排序
- 快速排序
- 大整数乘法（Karatsuba）

```typescript
// 分治算法示例：归并排序
function mergeSort(arr: number[]): number[] {
  // 基准情形：数组长度小于等于1时直接返回
  if (arr.length <= 1) return arr;

  // 分：将数组从中间分成两半
  const mid = Math.floor(arr.length / 2);
  const left = arr.slice(0, mid);
  const right = arr.slice(mid);

  // 治：递归排序左右两半
  const sortedLeft = mergeSort(left);
  const sortedRight = mergeSort(right);

  // 合：合并两个有序数组
  return merge(sortedLeft, sortedRight);
}

function merge(left: number[], right: number[]): number[] {
  const result: number[] = [];
  let i = 0, j = 0;

  // 比较两数组元素，按从小到大放入结果数组
  while (i < left.length && j < right.length) {
    if (left[i] <= right[j]) {
      result.push(left[i++]);
    } else {
      result.push(right[j++]);
    }
  }

  // 将剩余元素直接加入结果数组
  return result.concat(left.slice(i)).concat(right.slice(j));
}
```

### 2.4 动态规划（Dynamic Programming）

**核心思想**：将问题分解为重叠子问题，通过存储子问题避免重复计算。

**特点**：
- 最优子结构：全局最优包含局部最优
- 重叠子问题：子问题会被多次求解
- 状态转移：通过已求得的子问题求解更大问题

**代表算法**：
- 背包问题（0-1背包、完全背包、多重背包）
- 最长递增子序列（LIS）
- 最长公共子序列（LCS）
- 编辑距离

```typescript
// 动态规划示例：斐波那契数列
// 1. 递归版本（存在重复计算）
function fibRecursive(n: number): number {
  if (n <= 1) return n;
  return fibRecursive(n - 1) + fibRecursive(n - 2);
  // 问题：fib(5) = fib(4) + fib(3) = fib(3) + fib(2) + fib(2) + fib(1)
  // fib(3)被计算了2次，fib(2)被计算了3次，存在大量重复
}

// 2. 动态规划版本（自底向上）
function fibDP(n: number): number {
  if (n <= 1) return n;

  // 创建dp数组，dp[i]表示第i个斐波那契数
  const dp: number[] = new Array(n + 1).fill(0);
  dp[1] = 1; // 基础情况：第1个斐波那契数是1

  // 自底向上递推：依次计算dp[2], dp[3], ..., dp[n]
  for (let i = 2; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2]; // 状态转移方程
  }

  return dp[n];
}

// 3. 空间优化版本
function fibOptimized(n: number): number {
  if (n <= 1) return n;

  let prev = 0, curr = 1; // 只保留前两个状态
  for (let i = 2; i <= n; i++) {
    const next = prev + curr;
    prev = curr;
    curr = next;
  }

  return curr;
}
```

### 2.5 回溯算法（Backtracking）

**核心思想**：深度优先搜索解空间，通过尝试和回退找到所有解。

**特点**：
- 系统性搜索
- 包含"走不通就回退"的思想
- 常配合剪枝提升效率

**代表问题**：
- 八皇后问题
- 全排列、组合
- 数独求解
- 图的着色问题

```typescript
// 回溯算法示例：全排列
function permute(nums: number[]): number[][] {
  const result: number[][] = [];

  // 决策树遍历
  function backtrack(path: number[], used: boolean[]) {
    // 终止条件：路径长度等于数组长度，找到一个完整排列
    if (path.length === nums.length) {
      result.push([...path]); // 保存当前排列的副本
      return;
    }

    // 遍历所有选择
    for (let i = 0; i < nums.length; i++) {
      // 跳过已使用的元素（同一路径中不能重复）
      if (used[i]) continue;

      // 做选择
      path.push(nums[i]);
      used[i] = true;

      // 递归进入下一层决策树
      backtrack(path, used);

      // 撤销选择（回溯）
      path.pop();
      used[i] = false;
    }
  }

  backtrack([], new Array(nums.length).fill(false));
  return result;
}
```

### 2.6 分支限界算法（Branch and Bound）

**核心思想**：在解空间树中搜索，通过上界/下界剪枝减少搜索范围。

**特点**：
- 搜索策略：广度优先或最佳优先
- 剪枝条件：利用上界/下界判断
- 通常用优先队列维护待搜索节点

**代表问题**：
- 0-1背包问题
- 旅行商问题（TSP）
- 装载问题
- 旅行商问题的分支限界解法

```typescript
// 分支限界示例：0-1背包问题（优先队列实现）
interface Item {
  weight: number;  // 物品重量
  value: number;    // 物品价值
  ratio: number;    // 价值重量比
}

interface Node {
  level: number;      // 当前层数
  weight: number;      // 当前总重量
  value: number;       // 当前总价值
  bound: number;       // 该节点的价值上界
}

function knapsackBranchBound(items: Item[], capacity: number): number {
  // 按价值重量比降序排序
  items.sort((a, b) => b.ratio - a.ratio);

  const queue: Node[] = []; // 使用队列存储活节点
  let maxValue = 0;

  // 从根节点开始搜索
  queue.push({ level: 0, weight: 0, value: 0, bound: 0 });

  while (queue.length > 0) {
    // 取出当前节点（这里简化为FIFO，实际可用优先队列按bound排序）
    const node = queue.shift()!;

    // 计算上界
    node.bound = calculateBound(node, items, capacity);

    // 如果上界小于当前最大价值，剪枝
    if (node.bound < maxValue) continue;

    // 扩展节点：尝试包含下一个物品
    const nextLevel = node.level + 1;
    if (nextLevel <= items.length) {
      const item = items[nextLevel - 1];

      // 左分支：包含该物品
      if (node.weight + item.weight <= capacity) {
        maxValue = Math.max(maxValue, node.value + item.value);
        queue.push({
          level: nextLevel,
          weight: node.weight + item.weight,
          value: node.value + item.value,
          bound: 0 // 实际应重新计算
        });
      }

      // 右分支：不包含该物品
      queue.push({
        level: nextLevel,
        weight: node.weight,
        value: node.value,
        bound: 0 // 实际应重新计算
      });
    }
  }

  return maxValue;
}
```

## 三、算法范式决策树

面对一个问题，如何选择合适的算法范式？

```
问题分析
    │
    ├── 解空间是否较小？ ──YES──> 暴力搜索 / 回溯
    │
    ├── 是否需要找出所有解？ ──YES──> 回溯算法
    │
    ├── 是否具有最优子结构？
    │       │
    │       ├── 贪心选择性质是否成立？ ──YES──> 贪心算法
    │       │
    │       └── 是否有重叠子问题？
    │               │
    │               ├──YES──> 动态规划
    │               │
    │               └──NO──> 分治算法
    │
    └── 是否需要求最优解且解空间大？
            │
            └─YES──> 分支限界 / 近似算法
```

## 四、范式对比总结

| 范式 | 时间复杂度 | 空间复杂度 | 关键特征 | 经典问题 |
|------|-----------|-----------|----------|----------|
| 暴力搜索 | O(n!) ~ O(2^n) | O(n) | 枚举所有可能 | 简单遍历、全排列 |
| 贪心算法 | O(nlogn) ~ O(n^2) | O(n) | 局部最优 | 活动选择、Huffman |
| 分治算法 | O(nlogn) | O(logn) ~ O(n) | 分而治之 | 归并排序、二分查找 |
| 动态规划 | O(n) ~ O(n^2) | O(n) ~ O(n^2) | 存储子问题 | 背包、LIS、LCS |
| 回溯算法 | O(n!) ~ O(2^n) | O(n) | 试探+回退 | 八皇后、全排列 |
| 分支限界 | 最坏指数级 | O(节点数) | 上界/下界剪枝 | TSP、0-1背包 |

## 七、本章小结

本章我们概述了六种核心算法范式：
1. **暴力搜索**：简单枚举，适合解空间小的情况
2. **贪心算法**：局部最优推导全局最优，需要证明正确性
3. **分治算法**：分解问题、递归解决、合并结果
4. **动态规划**：存储重叠子问题，避免重复计算
5. **回溯算法**：深度优先搜索 + 尝试回退
6. **分支限界**：广度优先搜索 + 界限剪枝
