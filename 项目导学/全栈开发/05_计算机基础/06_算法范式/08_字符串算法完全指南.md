# 字符串算法完全指南

## 一、字符串基础

### 1.1 字符串的特性

字符串是由字符组成的有序序列。字符串算法是计算机科学中最常见的问题类型之一。

**类比理解——字符串就像火车：**
- 每个字符是一节车厢
- 字符串有方向性（从头到尾）
- 子字符串是火车的一段连续车厢
- 子序列是火车中按顺序抽取的一些车厢（不一定是连续的）

### 1.2 字符串操作的时间复杂度

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| 访问字符 | O(1) | 数组索引 |
| 长度 | O(1) | 存储的长度 |
| 拼接 | O(n) | 需要创建新字符串 |
| 子串 | O(k) | k为子串长度 |
| 比较 | O(min(n,m)) | 最坏情况O(n+m) |
| 搜索子串 | O(n×m) 或 O(n) | 取决于算法 |

## 二、字符串匹配算法

### 2.1 KMP算法

**KMP（Knuth-Morris-Pratt）算法** 是最经典的字符串匹配算法。

**核心思想：** 当出现不匹配时，利用已匹配的前缀信息，避免重复比较。

**类比理解：** 想象你在阅读一本书寻找某个词语。当你发现不匹配时，你不会从词语的开头重新开始，而是利用已经匹配的部分，跳过已知不可能匹配的位置。

```typescript
/**
 * KMP算法
 *
 * 时间复杂度：O(n + m)
 * 空间复杂度：O(m)
 *
 * 核心概念：
 * 1. 前缀函数（π数组）：最长相等前后缀长度
 * 2. 匹配失败时，利用前缀函数跳转
 */

/**
 * 计算前缀函数
 * prefix[i] = 模式串[0..i]的最长相等前后缀长度
 *
 * 例子：模式串 "ABAB"
 * - A：前后缀为空集，长度=0
 * - AB：没有相等前后缀，长度=0
 * - ABA：前缀"A"和后缀"A"相等，长度=1
 * - ABAB：前缀"AB"和后缀"AB"相等，长度=2
 * prefix = [0, 0, 1, 2]
 */
function computePrefixFunction(pattern: string): number[] {
  const m = pattern.length;
  const prefix = new Array(m).fill(0);

  // j 表示当前正在匹配的前缀长度
  for (let i = 1; i < m; i++) {
    let j = prefix[i - 1];

    // 当不匹配时，利用前缀函数回退
    while (j > 0 && pattern[i] !== pattern[j]) {
      j = prefix[j - 1];
    }

    // 如果匹配，长度+1
    if (pattern[i] === pattern[j]) {
      j++;
    }

    prefix[i] = j;
  }

  return prefix;
}

/**
 * KMP匹配
 */
function kmpSearch(text: string, pattern: string): number {
  const n = text.length;
  const m = pattern.length;

  if (m === 0) return 0;
  if (m > n) return -1;

  // 计算前缀函数
  const prefix = computePrefixFunction(pattern);

  // j 表示已匹配的字符数
  let j = 0;

  for (let i = 0; i < n; i++) {
    // 当不匹配时，利用前缀函数回退
    while (j > 0 && text[i] !== pattern[j]) {
      j = prefix[j - 1];
    }

    // 如果匹配，继续
    if (text[i] === pattern[j]) {
      j++;
    }

    // 如果完全匹配，返回起始位置
    if (j === m) {
      // 起始位置 = i - m + 1
      return i - m + 1;
    }
  }

  return -1;
}

/**
 * KMP匹配（返回所有匹配位置）
 */
function kmpSearchAll(text: string, pattern: string): number[] {
  const n = text.length;
  const m = pattern.length;
  const result: number[] = [];

  if (m === 0) {
    for (let i = 0; i <= n; i++) result.push(i);
    return result;
  }
  if (m > n) return result;

  const prefix = computePrefixFunction(pattern);
  let j = 0;

  for (let i = 0; i < n; i++) {
    while (j > 0 && text[i] !== pattern[j]) {
      j = prefix[j - 1];
    }

    if (text[i] === pattern[j]) {
      j++;
    }

    if (j === m) {
      result.push(i - m + 1);
      j = prefix[j - 1];
    }
  }

  return result;
}

/**
 * KMP应用：最长重复子串
 */
function longestRepeatedSubstring(s: string): string {
  const n = s.length;
  const prefix = computePrefixFunction(s);

  let maxLen = 0;
  let maxIndex = 0;

  for (let i = 1; i < n; i++) {
    if (prefix[i] > maxLen) {
      maxLen = prefix[i];
      maxIndex = i - maxLen + 1;
    }
  }

  return s.substring(maxIndex, maxIndex + maxLen);
}

/**
 * KMP应用：字符串循环左旋转
 */
function leftRotate(s: string, k: number): string {
  const n = s.length;
  k = k % n;

  if (k === 0) return s;

  // 使用KMP找模式
  const doubled = s + s;
  const pattern = doubled.substring(0, n + k);

  const prefix = computePrefixFunction(s);
  let j = 0;

  for (let i = 0; i < pattern.length; i++) {
    while (j > 0 && pattern[i] !== s[j]) {
      j = prefix[j - 1];
    }
    if (pattern[i] === s[j]) j++;
    if (j === n) return doubled.substring(i - n + 1, i + 1);
  }

  return s;
}
```

### 2.2 Boyer-Moore算法

**Boyer-Moore算法** 从右向左匹配，利用坏字符和好后缀规则。

**核心思想：** 从模式串的末尾开始匹配，当出现不匹配时，利用字符信息跳过多个位置。

```typescript
/**
 * Boyer-Moore算法
 *
 * 时间复杂度：最坏O(n×m)，实际接近O(n)
 * 空间复杂度：O(m)
 *
 * 两个启发式规则：
 * 1. 坏字符规则：从模式串末尾开始，遇到不匹配的字符
 * 2. 好后缀规则：匹配的后缀可以用于下一步匹配
 */

/**
 * 坏字符规则：
 * 当字符不匹配时，将模式串向右移动，使其与文本中坏字符最近的出现对齐
 *
 * 坏字符表：
 * badChar[char] = 模式串中char最靠右的位置
 */

/**
 * 好后缀规则：
 * 当匹配失败时，已匹配的后缀可能在模式串的其他位置出现
 * 好后缀表：
 * goodSuffix[i] = 如果在位置i匹配失败，已匹配后缀对应的最长相同前缀的位置
 */

/**
 * 坏字符启发式
 */
function buildBadCharTable(pattern: string): Map<string, number> {
  const badChar = new Map<string, number>();

  for (let i = 0; i < pattern.length; i++) {
    badChar.set(pattern[i], i);
  }

  return badChar;
}

/**
 * Boyer-Moore搜索
 */
function boyerMooreSearch(text: string, pattern: string): number {
  const n = text.length;
  const m = pattern.length;

  if (m === 0) return 0;
  if (m > n) return -1;

  const badChar = buildBadCharTable(pattern);

  // 从模式串末尾开始匹配
  let shift = 0;  // 文本中当前匹配的起始位置

  while (shift <= n - m) {
    let j = m - 1;  // 从模式串末尾开始

    // 从后向前匹配
    while (j >= 0 && pattern[j] === text[shift + j]) {
      j--;
    }

    if (j < 0) {
      // 完全匹配
      return shift;
    }

    // 坏字符规则：获取文本中坏字符在模式串中的位置
    const badCharPos = badChar.get(text[shift + j]) ?? -1;

    // 计算移动距离：坏字符位置和当前位置之差
    // 如果坏字符不在模式串中，badCharPos = -1
    // 移动量 = j - (-1) = j + 1
    // 如果坏字符在模式串中，移动量 = j - badCharPos
    const move = j - badCharPos;

    shift += Math.max(1, move);
  }

  return -1;
}

/**
 * 完整的Boyer-Moore（包含好后缀规则）
 * 实现较复杂，这里给出简化版本
 */
function boyerMooreFull(text: string, pattern: string): number {
  const n = text.length;
  const m = pattern.length;

  if (m === 0) return 0;
  if (m > n) return -1;

  // 构建坏字符表
  const badChar = new Map<string, number>();
  for (let i = 0; i < pattern.length; i++) {
    badChar.set(pattern[i], i);
  }

  // 构建好后缀表（简化版）
  const goodSuffix = new Array(m).fill(0);
  let longestPrefixSuffix = new Array(m).fill(0);

  // 计算每个位置的最长相等前后缀
  for (let i = 1; i < m; i++) {
    let j = longestPrefixSuffix[i - 1];
    while (j > 0 && pattern[i] !== pattern[j]) {
      j = longestPrefixSuffix[j - 1];
    }
    if (pattern[i] === pattern[j]) j++;
    longestPrefixSuffix[i] = j;
  }

  let shift = 0;

  while (shift <= n - m) {
    let j = m - 1;

    while (j >= 0 && pattern[j] === text[shift + j]) {
      j--;
    }

    if (j < 0) {
      return shift;
    }

    // 坏字符移动
    const badCharPos = badChar.get(text[shift + j]) ?? -1;
    const badCharShift = j - badCharPos;

    // 好后缀移动
    let goodSuffixShift = 0;
    if (j < m - 1) {
      goodSuffixShift = m - 1 - longestPrefixSuffix[m - 1];
    }

    shift += Math.max(badCharShift, goodSuffixShift);
  }

  return -1;
}
```

### 2.3 Rabin-Karp算法

**Rabin-Karp算法** 使用哈希来实现字符串匹配。

```typescript
/**
 * Rabin-Karp算法
 *
 * 时间复杂度：平均O(n+m)，最坏O(n×m)
 * 空间复杂度：O(1)
 *
 * 使用滚动哈希来快速比较子串
 */

function rabinKarpSearch(text: string, pattern: string): number {
  const n = text.length;
  const m = pattern.length;

  if (m === 0) return 0;
  if (m > n) return -1;

  const BASE = 31;
  const MOD = 10 ** 9 + 7;

  // 计算pattern的哈希值
  let patternHash = 0;
  for (let i = 0; i < m; i++) {
    patternHash = (patternHash * BASE + pattern.charCodeAt(i)) % MOD;
  }

  // 计算text第一个窗口的哈希值
  let textHash = 0;
  for (let i = 0; i < m; i++) {
    textHash = (textHash * BASE + text.charCodeAt(i)) % MOD;
  }

  // 计算BASE^(m-1) % MOD
  let basePower = 1;
  for (let i = 0; i < m - 1; i++) {
    basePower = (basePower * BASE) % MOD;
  }

  for (let i = 0; i <= n - m; i++) {
    // 如果哈希值匹配，再验证子串（避免哈希冲突）
    if (textHash === patternHash) {
      let match = true;
      for (let j = 0; j < m; j++) {
        if (text[i + j] !== pattern[j]) {
          match = false;
          break;
        }
      }
      if (match) return i;
    }

    // 计算下一个窗口的哈希值
    if (i < n - m) {
      textHash = ((textHash - text.charCodeAt(i) * basePower) * BASE +
                  text.charCodeAt(i + m)) % MOD;

      // 处理负数
      if (textHash < 0) textHash += MOD;
    }
  }

  return -1;
}

/**
 * Rabin-Karp（多个模式串）
 */
function multiPatternRabinKarp(text: string, patterns: string[]): Map<string, number> {
  const BASE = 31;
  const MOD = 10 ** 9 + 7;

  // 计算所有模式串的哈希值
  const patternHashes = new Map<string, number>();
  for (const p of patterns) {
    let hash = 0;
    for (const c of p) {
      hash = (hash * BASE + c.charCodeAt(0)) % MOD;
    }
    patternHashes.set(p, hash);
  }

  const results = new Map<string, number>();
  let textHash = 0;

  // 对每个可能的子串长度计算
  for (const pattern of patterns) {
    const m = pattern.length;
    const patternHash = patternHashes.get(pattern)!;

    // 重置textHash
    textHash = 0;
    for (let i = 0; i < m; i++) {
      textHash = (textHash * BASE + text.charCodeAt(i)) % MOD;
    }

    let basePower = 1;
    for (let i = 0; i < m - 1; i++) {
      basePower = (basePower * BASE) % MOD;
    }

    for (let i = 0; i <= text.length - m; i++) {
      if (textHash === patternHash) {
        let match = true;
        for (let j = 0; j < m; j++) {
          if (text[i + j] !== pattern[j]) {
            match = false;
            break;
          }
        }
        if (match) {
          results.set(pattern, i);
          break;
        }
      }

      if (i < text.length - m) {
        textHash = ((textHash - text.charCodeAt(i) * basePower) * BASE +
                    text.charCodeAt(i + m)) % MOD;
        if (textHash < 0) textHash += MOD;
      }
    }
  }

  return results;
}
```

## 三、Trie树（前缀树）

### 3.1 Trie树基础

**Trie树（前缀树/字典树）** 是用于快速检索字符串的数据结构。

**类比理解：** 想象一本字典，所有的单词都按照字母顺序排列。但Trie树更进一步，把相同前缀的单词共享同一条路径，而不是每个单词单独占一行。

```typescript
/**
 * Trie树实现
 *
 * 时间复杂度：
 *   - 插入：O(m)，m为字符串长度
 *   - 搜索：O(m)
 *   - 前缀搜索：O(m)
 *
 * 空间复杂度：O(m × n)，其中m为平均长度，n为字符串数量
 */

interface TrieNode {
  children: Map<string, TrieNode>;
  isEnd: boolean;      // 是否是单词结尾
  count: number;       // 以该节点为结尾的单词数量
  prefixCount: number; // 以该节点为前缀的单词数量
}

class Trie {
  private root: TrieNode;

  constructor() {
    this.root = this.createNode();
  }

  private createNode(): TrieNode {
    return {
      children: new Map(),
      isEnd: false,
      count: 0,
      prefixCount: 0
    };
  }

  /**
   * 插入一个单词
   */
  insert(word: string): void {
    let node = this.root;

    for (const char of word) {
      if (!node.children.has(char)) {
        node.children.set(char, this.createNode());
      }
      node = node.children.get(char)!;
      node.prefixCount++;
    }

    node.isEnd = true;
    node.count++;
  }

  /**
   * 搜索完整单词
   */
  search(word: string): boolean {
    const node = this.searchPrefix(word);
    return node !== null && node.isEnd;
  }

  /**
   * 搜索前缀
   */
  startsWith(prefix: string): boolean {
    return this.searchPrefix(prefix) !== null;
  }

  private searchPrefix(prefix: string): TrieNode | null {
    let node = this.root;

    for (const char of prefix) {
      if (!node.children.has(char)) {
        return null;
      }
      node = node.children.get(char)!;
    }

    return node;
  }

  /**
   * 返回以prefix为前缀的所有单词
   */
  wordsStartingWith(prefix: string): string[] {
    const node = this.searchPrefix(prefix);
    if (!node) return [];

    const result: string[] = [];
    this.collectWords(node, prefix, result);
    return result;
  }

  private collectWords(node: TrieNode, prefix: string, result: string[]): void {
    if (node.isEnd) {
      result.push(prefix);
    }

    for (const [char, child] of node.children) {
      this.collectWords(child, prefix + char, result);
    }
  }

  /**
   * 返回出现次数>=k的单词
   */
  findWordsWithCountAtLeast(k: number): string[] {
    const result: string[] = [];
    this.collectByCount(this.root, '', result, k);
    return result;
  }

  private collectByCount(
    node: TrieNode,
    word: string,
    result: string[],
    k: number
  ): void {
    if (node.isEnd && node.count >= k) {
      result.push(word);
    }

    for (const [char, child] of node.children) {
      this.collectByCount(child, word + char, result, k);
    }
  }

  /**
   * 最长公共前缀
   */
  longestCommonPrefix(): string {
    let node = this.root;
    let prefix = '';

    while (node.children.size === 1 && !node.isEnd) {
      const char = node.children.keys().next().value;
      prefix += char;
      node = node.children.get(char)!;
    }

    return prefix;
  }
}
```

### 3.2 Trie树应用

```typescript
/**
 * Trie树应用：字符串检索与自动补全
 */

/**
 * 搜索引擎自动补全
 */
class AutoComplete {
  private trie: Trie;

  constructor() {
    this.trie = new Trie();
  }

  addQuery(query: string): void {
    // 将查询转换为小写并插入
    const normalized = query.toLowerCase().trim();
    if (normalized) {
      this.trie.insert(normalized);
    }
  }

  suggest(prefix: string, limit: number = 10): string[] {
    const normalized = prefix.toLowerCase().trim();
    const words = this.trie.wordsStartingWith(normalized);
    return words.slice(0, limit);
  }
}

/**
 * Trie树应用：最长公共前缀（多个字符串）
 */
function longestCommonPrefix(strs: string[]): string {
  if (strs.length === 0) return '';

  // 找到最短的字符串
  const minLen = Math.min(...strs.map(s => s.length));

  // 从第一个字符串的前缀开始，逐步缩短
  let prefix = strs[0].substring(0, minLen);

  for (const str of strs) {
    while (!str.startsWith(prefix)) {
      prefix = prefix.substring(0, prefix.length - 1);
    }
  }

  return prefix;
}

/**
 * Trie树应用：单词搜索II
 * 在二维字符网格中找所有在字典中的单词
 */
function findWords(board: string[][], words: string[]): string[] {
  const result: string[] = [];
  const m = board.length;
  const n = board[0].length;

  // 构建Trie
  const trie = new Trie();
  for (const word of words) {
    trie.insert(word);
  }

  // DFS遍历网格
  function dfs(i: number, j: number, node: TrieNode, path: string): void {
    // 越界检查
    if (i < 0 || i >= m || j < 0 || j >= n) return;

    const char = board[i][j];
    const child = node.children.get(char);

    if (!child) return;

    path += char;

    if (child.isEnd) {
      result.push(path);
      child.isEnd = false;  // 避免重复
    }

    // 标记访问
    board[i][j] = '#';

    // 四个方向继续搜索
    dfs(i + 1, j, child, path);
    dfs(i - 1, j, child, path);
    dfs(i, j + 1, child, path);
    dfs(i, j - 1, child, path);

    // 恢复
    board[i][j] = char;
  }

  for (let i = 0; i < m; i++) {
    for (let j = 0; j < n; j++) {
      dfs(i, j, trie.root, '');
    }
  }

  return result;
}
```

## 四、AC自动机

### 4.1 AC自动机原理

**AC自动机（Aho-Corasick自动机）** 是多模式字符串匹配算法，结合了Trie和KMP的思想。

**类比理解：** 想象你要在一本书中找出所有出现的单词（可能有几百个）。一个一个搜索太低效了。AC自动机让你一次性构建所有单词的"Trie树"，然后像KMP一样，在匹配过程中利用"失败链接"跳过不可能匹配的位置。

```typescript
/**
 * AC自动机
 *
 * 用于：多模式字符串匹配
 * 时间复杂度：O(n + m + z)，其中z为匹配结果数量
 * 空间复杂度：O(m × Σ)
 *
 * 核心概念：
 * 1. Trie树：存储所有模式串
 * 2. 失败链接（fail）：KMP中的next指针，用于失配时跳转
 * 3. 输出链接（output）：快速找到匹配的模式串
 */

interface ACNode {
  children: Map<string, ACNode>;
  fail: ACNode | null;      // 失败链接
  output: ACNode | null;     // 输出链接
  isEnd: boolean;
  pattern: string | null;     // 存储匹配的模式串
  depth: number;
}

class AhoCorasick {
  private root: ACNode;

  constructor() {
    this.root = this.createNode('');
  }

  private createNode(char: string): ACNode {
    return {
      children: new Map(),
      fail: null,
      output: null,
      isEnd: false,
      pattern: null,
      depth: char.length
    };
  }

  /**
   * 插入模式串
   */
  insert(pattern: string): void {
    let node = this.root;

    for (const char of pattern) {
      if (!node.children.has(char)) {
        node.children.set(char, this.createNode(char));
      }
      node = node.children.get(char)!;
    }

    node.isEnd = true;
    node.pattern = pattern;
  }

  /**
   * 构建失败链接
   */
  build(): void {
    const queue: ACNode[] = [];

    // 第一层节点的失败链接指向根
    for (const child of this.root.children.values()) {
      child.fail = this.root;
      queue.push(child);
    }

    // BFS构建失败链接
    while (queue.length > 0) {
      const current = queue.shift()!;

      for (const [char, child] of current.children) {
        // 计算失败链接
        let fail = current.fail;
        while (fail !== null && !fail.children.has(char)) {
          fail = fail.fail;
        }

        if (fail === null) {
          child.fail = this.root;
        } else {
          child.fail = fail.children.get(char)!;
          // 设置输出链接：指向最近的输出节点
          if (child.fail.isEnd) {
            child.output = child.fail;
          } else if (child.fail.output !== null) {
            child.output = child.fail.output;
          }
        }

        queue.push(child);
      }
    }
  }

  /**
   * 在文本中搜索所有模式串
   */
  search(text: string): Array<{pattern: string, index: number}> {
    const results: Array<{pattern: string, index: number}> = [];
    let current = this.root;

    for (let i = 0; i < text.length; i++) {
      const char = text[i];

      // 沿着失败链接找匹配的边
      while (current !== this.root && !current.children.has(char)) {
        current = current.fail!;
      }

      if (current.children.has(char)) {
        current = current.children.get(char)!;
      }

      // 检查输出
      let temp: ACNode | null = current;
      while (temp !== null && temp.isEnd) {
        results.push({
          pattern: temp.pattern!,
          index: i - temp.pattern!.length + 1
        });
        temp = temp.output;
      }
    }

    return results;
  }

  /**
   * 搜索所有匹配（包含重复的模式）
   */
  searchAll(text: string): Map<string, number[]> {
    const results = new Map<string, number[]>();
    const matches = this.search(text);

    for (const {pattern, index} of matches) {
      if (!results.has(pattern)) {
        results.set(pattern, []);
      }
      results.get(pattern)!.push(index);
    }

    return results;
  }
}

/**
 * AC自动机应用示例
 */
function acAutoMaticExample(): void {
  const patterns = ['he', 'she', 'his', 'hers', 'her'];
  const text = 'ahishers';

  const ac = new AhoCorasick();
  for (const p of patterns) {
    ac.insert(p);
  }
  ac.build();

  const matches = ac.search(text);

  console.log(`文本: "${text}"`);
  console.log('匹配结果:');
  for (const {pattern, index} of matches) {
    console.log(`  "${pattern}" 出现在位置 ${index}`);
  }
}
```

### 5.1 编辑距离

```typescript
/**
 * 编辑距离（Levenshtein Distance）
 * 使用动态规划计算两个字符串之间的最小编辑距离
 */

function minDistance(word1: string, word2: string): number {
  const m = word1.length;
  const n = word2.length;

  // dp[i][j] = word1[0..i-1] 和 word2[0..j-1] 的最小编辑距离
  const dp = Array.from({length: m + 1}, () => new Array(n + 1).fill(0));

  // 初始化
  for (let i = 0; i <= m; i++) dp[i][0] = i;  // 删除i个字符
  for (let j = 0; j <= n; j++) dp[0][j] = j;  // 插入j个字符

  // DP
  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (word1[i - 1] === word2[j - 1]) {
        dp[i][j] = dp[i - 1][j - 1];
      } else {
        dp[i][j] = 1 + Math.min(
          dp[i - 1][j],      // 删除
          dp[i][j - 1],      // 插入
          dp[i - 1][j - 1]   // 替换
        );
      }
    }
  }

  return dp[m][n];
}

/**
 * 编辑距离（空间优化）
 */
function minDistanceOptimized(word1: string, word2: string): number {
  const m = word1.length;
  const n = word2.length;

  // 确保m <= n，用较短的作为行
  if (m > n) {
    [word1, word2] = [word2, word1];
    [m, n] = [n, m];
  }

  const dp = new Array(m + 1).fill(0);

  for (let j = 0; j <= m; j++) {
    dp[j] = j;
  }

  for (let i = 1; i <= n; i++) {
    let prev = dp[0];
    dp[0] = i;

    for (let j = 1; j <= m; j++) {
      const temp = dp[j];
      if (word1[j - 1] === word2[i - 1]) {
        dp[j] = prev;
      } else {
        dp[j] = 1 + Math.min(prev, dp[j], dp[j - 1]);
      }
      prev = temp;
    }
  }

  return dp[m];
}
```

### 5.2 最长回文子串

```typescript
/**
 * 最长回文子串
 *
 * 方法1：中心扩展法
 * 时间复杂度：O(n²)
 * 空间复杂度：O(1)
 */
function longestPalindrome(s: string): string {
  if (s.length <= 1) return s;

  let start = 0;
  let maxLen = 1;

  function expand(left: number, right: number): void {
    while (left >= 0 && right < s.length && s[left] === s[right]) {
      const len = right - left + 1;
      if (len > maxLen) {
        maxLen = len;
        start = left;
      }
      left--;
      right++;
    }
  }

  for (let i = 0; i < s.length; i++) {
    // 奇数长度回文
    expand(i, i);
    // 偶数长度回文
    expand(i, i + 1);
  }

  return s.substring(start, start + maxLen);
}

/**
 * 最长回文子串（Manacher算法）
 * 时间复杂度：O(n)
 * 空间复杂度：O(n)
 */
function longestPalindromeManacher(s: string): string {
  // 预处理：将原字符串用特殊字符分隔
  const t = '#' + s.split('').join('#') + '#';
  const n = t.length;

  const p = new Array(n).fill(0);  // p[i] = 以i为中心的回文半径
  let center = 0;
  let right = 0;
  let maxLen = 0;
  let maxCenter = 0;

  for (let i = 0; i < n; i++) {
    // 计算初始半径
    const mirror = 2 * center - i;
    if (i < right) {
      p[i] = Math.min(right - i, p[mirror]);
    }

    // 中心扩展
    while (i - p[i] - 1 >= 0 && i + p[i] + 1 < n &&
           t[i - p[i] - 1] === t[i + p[i] + 1]) {
      p[i]++;
    }

    // 更新center和right
    if (i + p[i] > right) {
      center = i;
      right = i + p[i];
    }

    // 记录最长回文
    if (p[i] > maxLen) {
      maxLen = p[i];
      maxCenter = i;
    }
  }

  // 还原原始字符串
  const start = (maxCenter - maxLen) / 2;
  return s.substring(start, start + maxLen);
}
```

```typescript
/**
 * 不同的子序列
 * 计算子序列出现的次数
 */

function numDistinct(s: string, t: string): number {
  const m = s.length;
  const n = t.length;

  if (n > m) return 0;

  // dp[i][j] = s[0..i-1]中t[0..j-1]出现的次数
  const dp = Array.from({length: m + 1}, () => new Array(n + 1).fill(0));

  // 初始化：空字符串t在任意s的前缀中出现1次
  for (let i = 0; i <= m; i++) {
    dp[i][0] = 1;
  }

  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (s[i - 1] === t[j - 1]) {
        dp[i][j] = dp[i - 1][j - 1] + dp[i - 1][j];
      } else {
        dp[i][j] = dp[i - 1][j];
      }
    }
  }

  return dp[m][n];
}

/**
 * 不同的子序列（空间优化）
 */
function numDistinctOptimized(s: string, t: string): number {
  const m = s.length;
  const n = t.length;

  if (n > m) return 0;

  const dp = new Array(n + 1).fill(0);
  dp[0] = 1;

  for (let i = 1; i <= m; i++) {
    for (let j = n; j >= 1; j--) {
      if (s[i - 1] === t[j - 1]) {
        dp[j] = dp[j - 1] + dp[j];
      }
    }
  }

  return dp[n];
}
```

## 七、总结

### 7.1 字符串算法核心要点

```
字符串算法要点

字符串匹配：
- KMP：O(n+m)，利用前缀函数
- Boyer-Moore：O(n/m)，从后向前匹配
- Rabin-Karp：O(n+m)，使用哈希
- AC自动机：多模式匹配，O(n+m)

前缀树（Trie）：
- 插入/搜索：O(m)
- 空间换时间
- 适合前缀查询、自动补全

后缀相关：
- 最长回文：中心扩展O(n²)，Manacher O(n)
- 编辑距离：DP O(mn)

字符串DP：
- 状态定义要清晰
- 注意空间优化
```

### 7.2 算法选择指南

```
字符串问题算法选择

单模式匹配：
- KMP：稳定高效
- Boyer-Moore：实际应用效果好
- Rabin-Karp：需要验证哈希冲突

多模式匹配：
- AC自动机：最优选择

前缀问题：
- Trie树：最优选择

子序列/子串问题：
- DP：LCS、LPS、编辑距离
- 滑动窗口：定长子串

回文问题：
- 中心扩展：简单直观
- Manacher：最优O(n)
```
