# 算法复杂度分析

## 一、时间复杂度基础

### 1.1 什么是时间复杂度？

```
时间复杂度：算法执行时间随输入规模增长的趋势

我们通常关心的是：
1. 当输入规模 n 趋于无穷大时，算法运行时间增长多快
2. 使用大 O 表示法描述这个增长率

为什么关注增长趋势？
- 因为实际运行时间受硬件、编译器等因素影响
- 增长趋势能反映算法本身的效率高低
```

### 1.2 大 O 表示法

```typescript
/**
 * 大 O 表示法（渐近上界）
 *
 * 定义：T(n) = O(f(n)) 当且仅当存在正常数 c 和 n₀
 *       使得当 n ≥ n₀ 时，T(n) ≤ c × f(n)
 *
 * 理解：f(n) 是 T(n) 的上界
 *       T(n) 的增长不会超过 f(n) 的某个倍数
 */

/**
 * 常见时间复杂度排序（从快到慢）：
 *
 * O(1)        <  O(log n)  <  O(n)  <  O(n log n)
 * 常数时间    对数时间     线性时间   线性对数时间
 *
 * O(n)       <  O(n²)    <  O(n³)  <  O(2^n)
 * 平方时间    立方时间    指数时间
 */
```

### 1.3 代码复杂度分析示例

```typescript
/**
 * O(1) - 常数时间
 */
function getFirstElement(arr: number[]): number {
  return arr[0]; // 只执行一次
}

/**
 * O(1) - 常数时间（但常数可能大于1）
 */
function sumTwoElements(arr: number[]): number {
  return arr[0] + arr[1]; // 执行3次（两次取值 + 一次加法）
}

/**
 * O(n) - 线性时间
 */
function sumArray(arr: number[]): number {
  let sum = 0;
  for (let i = 0; i < arr.length; i++) {
    sum += arr[i]; // 执行 n 次
  }
  return sum;
}

/**
 * O(n²) - 平方时间
 */
function bubbleSort(arr: number[]): number[] {
  const n = arr.length;
  for (let i = 0; i < n; i++) {
    for (let j = 0; j < n - i - 1; j++) {
      if (arr[j] > arr[j + 1]) {
        [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
      }
    }
  }
  return arr;
}

/**
 * O(log n) - 对数时间
 */
function binarySearch(arr: number[], target: number): number {
  let left = 0;
  let right = arr.length - 1;

  while (left <= right) {
    const mid = left + Math.floor((right - left) / 2);
    if (arr[mid] === target) return mid;
    else if (arr[mid] < target) left = mid + 1;
    else right = mid - 1;
  }

  return -1;
}

/**
 * O(2^n) - 指数时间（斐波那契暴力递归）
 */
function fibRecursive(n: number): number {
  if (n <= 1) return n;
  return fibRecursive(n - 1) + fibRecursive(n - 2);
}

/**
 * O(n!) - 阶乘时间（全排列）
 */
function allPermutations(arr: number[]): number[][] {
  const result: number[][] = [];

  function permute(path: number[], remaining: number[]) {
    if (remaining.length === 0) {
      result.push(path);
      return;
    }
    for (let i = 0; i < remaining.length; i++) {
      permute([...path, remaining[i]], remaining.filter((_, idx) => idx !== i));
    }
  }

  permute([], arr);
  return result;
}
```

## 二、空间复杂度分析

### 2.1 什么是空间复杂度？

```
空间复杂度：算法占用内存空间随输入规模增长的趋势

包括：
1. 固定部分：代码本身、常量、简单变量
2. 可变部分：根据输入规模分配的空间（数组、递归栈等）
```

### 2.2 代码空间分析示例

```typescript
/**
 * O(1) - 原地算法，只用常量额外空间
 */
function findMax(arr: number[]): number {
  let max = arr[0]; // O(1)
  for (let i = 1; i < arr.length; i++) {
    if (arr[i] > max) max = arr[i];
  }
  return max;
}

/**
 * O(n) - 使用了长度为 n 的数组
 */
function createPrefixSum(arr: number[]): number[] {
  const prefix = new Array(arr.length); // O(n)
  prefix[0] = arr[0];
  for (let i = 1; i < arr.length; i++) {
    prefix[i] = prefix[i - 1] + arr[i];
  }
  return prefix;
}

/**
 * O(n) - 递归栈深度为 n
 */
function sumRecursive(arr: number[], index: number = 0): number {
  if (index >= arr.length) return 0;
  return arr[index] + sumRecursive(arr, index + 1); // 递归深度 O(n)
}

/**
 * O(log n) - 递归栈深度为 log n（二分递归）
 */
function binarySearchRecursive(
  arr: number[],
  target: number,
  left: number = 0,
  right: number = arr.length - 1
): number {
  if (left > right) return -1;
  const mid = left + Math.floor((right - left) / 2);
  if (arr[mid] === target) return mid;
  else if (arr[mid] < target) return binarySearchRecursive(arr, target, mid + 1, right);
  else return binarySearchRecursive(arr, target, left, mid - 1);
}

/**
 * O(n²) - 二维矩阵
 */
function createMatrix(n: number): number[][] {
  const matrix: number[][] = [];
  for (let i = 0; i < n; i++) {
    matrix[i] = new Array(n).fill(0); // O(n²) 空间
  }
  return matrix;
}
```

## 三、主定理（Master Theorem）

### 3.1 主定理定义

```
主定理用于分析分治算法的时间复杂度

如果一个递归算法将问题规模为 n 的问题分解为 a 个子问题，
每个子问题规模为 n/b，分解和合并需要 O(n^d) 时间，
则总时间复杂度 T(n) 为：

         ┌ a × T(n/b) + O(n^d)          如果 n > 1
T(n) =  ┤
         └ O(1)                        如果 n = 1

主定理的解：

1. 如果 d > log_b(a)：T(n) = O(n^d)
2. 如果 d = log_b(a)：T(n) = O(n^d × log n)
3. 如果 d < log_b(a)：T(n) = O(n^(log_b(a)))
```

### 3.2 主定理应用示例

```typescript
/**
 * 示例1：归并排序
 *
 * 分解为 2 个子问题 (a = 2)
 * 每个子问题规模 n/2 (b = 2)
 * 合并时间 O(n) (d = 1)
 *
 * 因为 d = 1 = log_2(2)
 * 所以 T(n) = O(n^1 × log n) = O(n log n)
 */

/**
 * 示例2：二分查找
 *
 * 分解为 1 个子问题 (a = 1)
 * 每个子问题规模 n/2 (b = 2)
 * 合并时间 O(1) (d = 0)
 *
 * 因为 d = 0 < log_2(1) = 0
 * 所以 T(n) = O(n^0 × log n) = O(log n)
 */

/**
 * 示例3：大整数乘法（普通）
 *
 * 分解为 4 个子问题 (a = 4)
 * 每个子问题规模 n/2 (b = 2)
 * 合并时间 O(n) (d = 1)
 *
 * 因为 d = 1 < log_2(4) = 2
 * 所以 T(n) = O(n^2)
 */

/**
 * 示例4：Karatsuba 大整数乘法
 *
 * 分解为 3 个子问题 (a = 3)
 * 每个子问题规模 n/2 (b = 2)
 * 合并时间 O(n) (d = 1)
 *
 * 因为 d = 1 < log_2(3) ≈ 1.585
 * 所以 T(n) = O(n^log_2(3)) ≈ O(n^1.585)
 */

/**
 * 示例5：
 *
 * 分解为 2 个子问题 (a = 2)
 * 每个子问题规模 n/4 (b = 4)
 * 合并时间 O(n²) (d = 2)
 *
 * 因为 d = 2 = log_4(2) = 0.5？不对
 *
 * log_b(a) = log_4(2) = 0.5
 * d = 2 > 0.5
 * 所以 T(n) = O(n^2)
 */
```

## 四、递归复杂度分析

### 4.1 递归树方法

```typescript
/**
 * 递归复杂度分析 - 递归树方法
 *
 * 方法：
 * 1. 画出递归树
 * 2. 计算每层的成本
 * 3. 计算总层数
 * 4. 求和
 */

/**
 * 示例：T(n) = 2T(n/2) + n
 *
 * 递归树：
 *                    n
 *                  /    \
 *                n/2     n/2
 *               /  \    /  \
 *             n/4  n/4 n/4  n/4
 *              ...
 *
 * 每层成本：n, n, n/2, n/4, ...
 *          （第1层：n）
 *          （第2层：2 × n/2 = n）
 *          （第3层：4 × n/4 = n）
 *          （第log n层：n × 1 = n）
 *
 * 层数：log n + 1
 * 总成本：n × (log n + 1) = O(n log n)
 */

/**
 * 示例：T(n) = T(n/3) + T(2n/3) + O(n)
 *
 * 递归树（不均匀分支）：
 *                       n
 *                    /      \
 *                  n/3      2n/3
 *                 /  \      /    \
 *               n/9  2n/9  2n/9  4n/9
 *                ...
 *
 * 每层成本：n, n, n, n, ... （直到子问题规模变为1）
 * 层数：最坏情况是 log_1.5(n) ≈ O(log n)
 * 总成本：O(n log n)
 */

/**
 * 示例：T(n) = 2T(n-1) + O(1)
 *
 * 递归树：
 *         n
 *        /
 *      n-1
 *      /
 *    n-2
 *    ...
 *    2
 *    |
 *    1
 *
 * 每层成本：O(1)
 * 层数：n
 * 总成本：O(n)
 */

/**
 * 示例：T(n) = T(n-1) + O(n)
 *
 * 递归树：
 *         n
 *        /
 *      n-1
 *      /
 *    n-2
 *    ...
 *    2
 *    |
 *    1
 *
 * 每层成本：n, n-1, n-2, ..., 1
 * 总成本：n + (n-1) + ... + 1 = n(n+1)/2 = O(n²)
 */
```

### 4.2 扩展递归（Substitution Method）

```typescript
/**
 * 扩展递归方法
 *
 * 方法：
 * 1. 猜测解的形式
 * 2. 用数学归纳法证明
 */

/**
 * 示例1：T(n) = 2T(n/2) + n
 * 猜测：T(n) = O(n log n)
 *
 * 证明：
 * 假设 T(k) ≤ c × k × log k 对所有 k < n 成立
 * T(n) = 2T(n/2) + n
 *      ≤ 2 × c × (n/2) × log(n/2) + n
 *      = c × n × (log n - log 2) + n
 *      = c × n × log n - c × n + n
 *      ≤ c × n × log n  （当 c ≥ 1 时）
 *
 * 成立！
 */

/**
 * 示例2：T(n) = T(n-1) + O(n)
 * 猜测：T(n) = O(n²)
 *
 * 证明：
 * 假设 T(k) ≤ c × k² 对所有 k < n 成立
 * T(n) = T(n-1) + n
 *      ≤ c × (n-1)² + n
 *      = c × (n² - 2n + 1) + n
 *      = c × n² - 2c × n + c + n
 *      ≤ c × n²  （当 c ≥ 1 时，且 -2cn + n + c ≤ 0）
 *      即 c ≥ 1 且 n ≥ c/(2c-1)
 *
 * 成立！
 */
```

## 五、复杂度分析进阶

### 5.1 平摊分析

```typescript
/**
 * 平摊分析（Amortized Analysis）
 *
 * 定义：分析操作序列的总成本除以操作数，得到每个操作的平均成本
 *
 * 方法：
 * 1. 聚集方法（Gathering Method）：总成本 / 操作数
 * 2. 会计方法（Accounting Method）：给每个操作分配"存款"
 * 3. 势能方法（Potential Method）：用势能函数分析
 */

/**
 * 示例：动态数组（ArrayList）的平摊分析
 *
 * 操作：
 * - push: O(1) 大多数时候
 * - resize: O(n) 当数组满时
 *
 * 分析：
 * - 假设数组容量以 2 倍增长：1, 2, 4, 8, 16, ...
 * - n 次 push 操作，总成本：
 *   1 + 2 + 4 + 8 + ... + n ≤ 2n
 * - 平摊成本：2n / n = O(1)
 */

/**
 * 示例：栈的平摊分析
 *
 * 问题：栈支持 push、pop、multipop（弹出 k 个元素）
 *
 * 聚集方法：
 * - n 次操作，最多 n 次 push
 * - 每次 pop 或 multipop 最多 n 次
 * - 总成本：O(n)
 * - 平摊成本：O(1)
 */

/**
 * 势能方法示例：动态数组
 */
class DynamicArray<T> {
  private arr: (T | undefined)[] = [];
  private length = 0;
  private capacity = 1;
  private potential = 0; // 势能 = length - capacity（或 2*length - capacity）

  push(item: T): void {
    if (this.length === this.capacity) {
      // 扩容：复制 2n 个元素
      this.capacity *= 2;
      this.potential += this.length; // 势能增加
    }

    this.arr[this.length] = item;
    this.length++;
  }

  pop(): T | undefined {
    if (this.length === 0) return undefined;

    const item = this.arr[--this.length];
    this.arr[this.length] = undefined;

    // 当使用量小于容量/4时，收缩到容量/2
    if (this.length < this.capacity / 4 && this.capacity > 1) {
      this.capacity /= 2;
      this.potential -= this.length; // 势能减少
    }

    return item;
  }

  // 平摊成本 = 实际成本 + 势能变化
  // push 的平摊成本：1 + (length - (length-1)) = 2 = O(1)
  // pop 的平摊成本：1 + (length - (length+1)) = 0 = O(1)
}
```

### 5.2 最好、最坏、平均情况

```typescript
/**
 * 时间复杂度的不同情况
 *
 * 最好情况：Ω (Omega)
 * 最坏情况：O (Big O)
 * 平均情况：Θ (Theta) 或 Ω
 */

/**
 * 示例：线性查找
 *
 * 最好情况：目标在第一个位置 → Ω(1)
 * 最坏情况：目标在最后一个位置 → O(n)
 * 平均情况：目标在中间位置 → Θ(n/2) = Θ(n)
 */

/**
 * 示例：快速排序
 *
 * 最好情况：pivot 总是中间值 → Θ(n log n)
 * 最坏情况：pivot 总是最大或最小值 → O(n²)
 * 平均情况：随机数据 → Θ(n log n)
 */

/**
 * 示例：哈希查找
 *
 * 最好情况：无冲突 → Ω(1)
 * 最坏情况：所有键冲突 → O(n)
 * 平均情况：冲突均匀分布 → Θ(1)
 */

/**
 * 示例：冒泡排序
 *
 * 最好情况：已有序 → O(n)（优化版本）
 * 最坏情况：逆序 → O(n²)
 * 平均情况：随机数据 → Θ(n²)
 */
```

## 六、复杂度分析技巧

### 6.1 常见规则

```typescript
/**
 * 复杂度分析规则
 *
 * 1. 忽略常数系数
 *    O(2n) = O(n)
 *    O(n/2) = O(n)
 *
 * 2. 忽略低阶项
 *    O(n² + n) = O(n²)
 *    O(n³ + n² + n) = O(n³)
 *
 * 3. 多个循环：加法规则
 *    for (...) { O(n) } + for (...) { O(n) } = O(2n) = O(n)
 *
 * 4. 嵌套循环：乘法规则
 *    for (...) { for (...) { O(1) } } = O(n × n) = O(n²)
 *
 * 5. 分支语句：取最大值
 *    if (...) { O(n) } else { O(log n) } = O(n)
 *
 * 6. 递归：考虑递归树或主定理
 */
```

### 6.2 分析示例

```typescript
/**
 * 示例1：嵌套循环
 */
function example1(n: number): void {
  for (let i = 0; i < n; i++) {           // n 次
    for (let j = 0; j < n; j++) {         // n 次
      console.log(i, j);                   // O(1)
    }
  }
}
// 复杂度：O(n²)

/**
 * 示例2：增量循环
 */
function example2(n: number): void {
  for (let i = 0; i < n; i++) {           // n 次
    for (let j = i; j < n; j++) {         // i 到 n-1
      console.log(i, j);
    }
  }
}
// 复杂度：n + (n-1) + (n-2) + ... + 1 = n(n+1)/2 = O(n²)

/**
 * 示例3：递归循环
 */
function example3(n: number): void {
  for (let i = 1; i < n; i *= 2) {        // log₂(n) 次
    console.log(i);
  }
}
// 复杂度：O(log n)

/**
 * 示例4：递归调用
 */
function example4(n: number): void {
  if (n <= 1) return;
  example4(n / 2);                        // T(n/2)
  console.log(n);
}
// 复杂度：O(log n)

/**
 * 示例5：递归 + 循环
 */
function example5(n: number): void {
  if (n <= 1) return;
  for (let i = 0; i < n; i++) {           // n 次
    console.log(i);
  }
  example5(n - 1);                        // T(n-1)
}
// 复杂度：n + (n-1) + (n-2) + ... + 1 = O(n²)

/**
 * 示例6：双重递归
 */
function example6(n: number): number {
  if (n <= 1) return 1;
  return example6(n - 1) + example6(n - 1);
}
// 复杂度：2^0 + 2^1 + 2^2 + ... + 2^n = O(2^n)
```

## 七、LeetCode 复杂度要求

```
常见 LeetCode 题目复杂度要求：

简单题目：
- 通常要求 O(n) 或 O(n log n)
- 空间 O(1) 或 O(n)

中等题目：
- 通常要求 O(n log n)
- 空间 O(1) 或 O(n)

困难题目：
- 可能要求 O(n log n) 或 O(n)
- 空间 O(1) 或 O(n)
- 常见技巧：二分 + DP、单调栈等

面试中的复杂度要求：
- O(n²) 在 n ≤ 1000 时可接受
- O(n log n) 是比较好的解法
- O(n) 是最优解
- O(2^n) 或 O(n!) 通常不可接受
```

## 八、本章小结

本章我们学习了算法复杂度分析：

1. **时间复杂度**：
   - 大 O 表示法：渐近上界
   - 常见复杂度：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2^n)

2. **空间复杂度**：
   - 算法额外占用的内存空间
   - 包括：辅助数组、递归栈等

3. **主定理**：
   - T(n) = aT(n/b) + O(n^d)
   - 根据 d 与 log_b(a) 的关系确定复杂度

4. **递归复杂度分析**：
   - 递归树方法
   - 扩展递归方法

5. **平摊分析**：
   - 聚集方法、会计方法、势能方法
   - 用于分析动态数据结构的平均成本

6. **分析技巧**：
   - 忽略常数和低阶项
   - 加法规则、乘法规则
   - 取分支的最大值
