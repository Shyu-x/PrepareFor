# 暴力搜索BF与回溯法完全指南

## 一、暴力搜索基础

### 1.1 什么是暴力搜索？

**暴力搜索（Brute Force）** 是最简单、最直接的算法设计方法——穷举所有可能的解，从中找到正确答案。

**类比生活场景：** 想象你在一堆乱码密码中尝试破解密码，你从一个一个尝试，直到找到正确的密码。这就是暴力搜索的基本思想——枚举所有可能性。

**暴力搜索的三个特点：**
1. **简单粗暴：** 不需要什么巧妙的思路，就是一个一个试
2. **一定能找到解：** 如果解存在，暴力搜索一定能找到
3. **效率通常不高：** 时间复杂度往往是指数级别

### 1.2 暴力搜索的适用场景

**什么时候适合用暴力搜索？**

| 条件 | 说明 |
|------|------|
| **问题规模小** | n通常小于20-30，否则会超时 |
| **没有明显规律** | 无法用数学公式或贪心策略直接求解 |
| **需要验证所有情况** | 比如密码破解、全排列验证 |
| **作为基准算法** | 先实现暴力版本，再优化 |

### 1.3 暴力搜索的基本模板

```typescript
/**
 * 暴力搜索基本模板
 * @param nums 可选元素数组
 * @param target 目标值
 */

// 模板一：线性枚举
function linearSearch(nums: number[], target: number): number {
  // 遍历数组，逐个检查
  for (let i = 0; i < nums.length; i++) {
    if (nums[i] === target) {
      return i; // 找到目标，返回索引
    }
  }
  return -1; // 未找到
}

// 模板二：嵌套循环枚举（双指针基础形式）
function twoSum暴力(nums: number[], target: number): [number, number] | null {
  const n = nums.length;
  // 枚举所有可能的配对
  for (let i = 0; i < n; i++) {
    for (let j = i + 1; j < n; j++) {
      if (nums[i] + nums[j] === target) {
        return [i, j]; // 找到配对
      }
    }
  }
  return null; // 未找到
}

// 模板三：组合枚举（从n个元素中选k个）
function combine暴力(n: number, k: number): number[][] {
  const result: number[][] = [];

  // 使用回溯生成组合
  function backtrack(start: number, path: number[]) {
    // 终止条件：选够了k个数
    if (path.length === k) {
      result.push([...path]);
      return;
    }

    // 枚举可选元素
    for (let i = start; i <= n; i++) {
      path.push(i);       // 选择i
      backtrack(i + 1, path);  // 递归选择下一个
      path.pop();         // 撤销选择
    }
  }

  backtrack(1, []);
  return result;
}
```

## 二、回溯法深入解析

### 2.1 回溯法的核心思想

**回溯法（Backtracking）** 是暴力搜索的进化版，它比普通暴力搜索多了一个关键能力——**剪枝（Pruning）**。

**类比理解：** 想象你在走迷宫。当你在一个岔路口选择一条路走下去时，如果走到死胡同了，你会怎么办？正常人会**退回上一个岔路口**，尝试另一条路。这正是回溯法的思想——搜索过程中，如果发现当前路径不可能产生有效解，就退回去尝试其他路径。

**回溯法三要素：**
1. **选择（Choice）：** 在每一步做出选择
2. **探索（Explore）：** 递归进入下一层
3. **撤销（Undo）：** 回退当前选择，尝试其他选择

### 2.2 回溯法的算法框架

```typescript
/**
 * 回溯法算法框架
 */

// 结果类型定义
type Result = number[][];

// 主回溯函数
function backtrack(
  // 当前状态
  state: number[],
  // 可选列表
  choices: number[],
  // 深度/阶段
  depth: number,
  // 结果集
  result: Result
): void {
  // 终止条件：找到有效解
  if (isGoal(state)) {
    result.push([...state]);
    return;
  }

  // 剪枝条件：确定无解，不必继续搜索
  if (isDeadEnd(state)) {
    return;
  }

  // 遍历所有选择
  for (const choice of choices) {
    // 做出选择
    makeChoice(state, choice);

    // 递归探索（更新可选列表和深度）
    backtrack(
      state,
      getUpdatedChoices(choices, choice),
      depth + 1,
      result
    );

    // 撤销选择（回溯）
    undoChoice(state, choice);
  }
}

/**
 * 回溯法关键函数说明：
 *
 * isGoal(state): 判断当前状态是否已达到目标
 *   - 例如：组合之和等于目标值、排列长度等于k等
 *
 * isDeadEnd(state): 剪枝函数，判断是否应该剪枝
 *   - 例如：组合之和已超过目标值、长度已超过限制等
 *
 * makeChoice(state, choice): 将选择加入当前状态
 *   - 例如：state.push(choice)
 *
 * getUpdatedChoices(choices, choice): 获取更新后的选择列表
 *   - 例如：choices.filter(c => c !== choice)（排列）
 *   - 例如：choices.slice(choices.indexOf(choice))（组合）
 *
 * undoChoice(state, choice): 撤销选择，恢复状态
 *   - 例如：state.pop()
 */
```

## 三、经典问题实战

### 3.1 全排列问题（Permutation）

**问题描述：** 给定一个数组，生成所有可能的排列。

**问题分析：**
- 这是一个经典的全排列问题
- 对于n个元素，有n!种排列方式
- 使用回溯法，每个位置可以选择任何一个未使用的元素

**算法步骤：**
1. 创建一个空结果数组
2. 维护一个used数组标记元素是否使用
3. 对每个位置，尝试所有未使用的元素
4. 递归处理下一个位置
5. 回溯撤销选择

**TypeScript实现：**

```typescript
/**
 * 全排列问题
 *
 * 算法思想：
 * - 将问题看作：按顺序填写n个位置
 * - 每个位置可以从所有未使用的元素中选择
 * - 使用used数组标记哪些元素已被使用
 *
 * 时间复杂度：O(n × n!)
 * 空间复杂度：O(n)（递归栈+used数组）
 */

function permute(nums: number[]): number[][] {
  const result: number[][] = [];  // 存储所有排列结果
  const path: number[] = [];       // 当前正在构建的排列
  const used = new Array(nums.length).fill(false);  // 标记元素是否已使用

  /**
   * 回溯函数
   * @param path 当前已选择的排列
   * @param used 元素使用标记
   */
  function backtrack(path: number[], used: boolean[]): void {
    // 终止条件：排列长度等于数组长度，说明一个完整的排列已构建完成
    if (path.length === nums.length) {
      result.push([...path]);  // 保存当前排列的副本
      return;
    }

    // 遍历所有元素，尝试放在当前位置
    for (let i = 0; i < nums.length; i++) {
      // 如果元素已经被使用，跳过
      if (used[i]) {
        continue;
      }

      // 选择当前元素
      path.push(nums[i]);
      used[i] = true;

      // 递归处理下一个位置
      backtrack(path, used);

      // 回溯：撤销选择，尝试其他元素
      path.pop();
      used[i] = false;
    }
  }

  // 开始回溯
  backtrack(path, used);
  return result;
}

/**
 * 全排列（含重复元素）
 *
 * 当数组中含有重复元素时，需要去重
 * 思路：先排序，然后相同元素只使用一次
 */
function permuteUnique(nums: number[]): number[][] {
  const result: number[][] = [];
  const path: number[] = [];
  const used = new Array(nums.length).fill(false);

  // 先排序，便于去重
  nums.sort((a, b) => a - b);

  function backtrack(path: number[], used: boolean[]): void {
    if (path.length === nums.length) {
      result.push([...path]);
      return;
    }

    for (let i = 0; i < nums.length; i++) {
      // 去重条件：如果当前元素与上一个元素相同，且上一个元素未被使用，则跳过
      // 原理：相同元素的排列，上一个已经枚举过了，这里不需要重复枚举
      if (i > 0 && nums[i] === nums[i - 1] && !used[i - 1]) {
        continue;
      }

      if (used[i]) {
        continue;
      }

      path.push(nums[i]);
      used[i] = true;
      backtrack(path, used);
      path.pop();
      used[i] = false;
    }
  }

  backtrack(path, used);
  return result;
}
```

**复杂度分析：**
- 时间复杂度：O(n × n!)
  - 总共生成n!个排列
  - 每个排列需要O(n)时间复制到结果数组
- 空间复杂度：O(n)
  - 递归深度最多为n
  - used数组占用O(n)

### 3.2 组合问题（Combination）

**问题描述：** 从n个数中选出k个数的所有组合。

**问题分析：**
- 组合与排列的区别：组合不考虑顺序，[1,2]和[2,1]是同一个组合
- 为了避免重复枚举，采用升序选择策略
- 从start位置开始选择，保证选择的元素是递增的

```typescript
/**
 * 组合问题
 *
 * 算法思想：
 * - 组合不考虑顺序，所以按索引顺序选择
 * - 每层递归从start开始选择，避免重复
 * - 选够k个元素就保存结果
 *
 * 时间复杂度：O(C(n,k) × k)
 * 空间复杂度：O(k)
 */

function combine(n: number, k: number): number[][] {
  const result: number[][] = [];
  const path: number[] = [];

  /**
   * 回溯函数
   * @param start 起始位置（本层可选择的起始索引）
   * @param depth 当前已选元素数量
   */
  function backtrack(start: number, depth: number): void {
    // 终止条件：选够了k个数
    if (depth === k) {
      result.push([...path]);
      return;
    }

    // 剪枝优化：计算还需要选多少个元素
    // 剩余元素数量 = n - start + 1
    // 还需要选 = k - depth
    // 如果剩余元素不够，直接返回

    // 优化：控制搜索范围
    // i最大只能到 n - (k - path.length) + 1
    for (let i = start; i <= n - (k - path.length) + 1; i++) {
      // 选择i
      path.push(i);

      // 递归选择下一个（从i+1开始，保证递增）
      backtrack(i + 1, depth + 1);

      // 回溯
      path.pop();
    }
  }

  backtrack(1, 0);
  return result;
}

/**
 * 组合总和问题（可重复使用元素）
 *
 * 问题：从数组中选若干数，使和等于目标值，元素可以重复使用
 */
function combinationSum(candidates: number[], target: number): number[][] {
  const result: number[][] = [];
  const path: number[] = [];

  // 排序以便剪枝
  candidates.sort((a, b) => a - b);

  function backtrack(start: number, sum: number): void {
    // 终止条件：和等于目标值
    if (sum === target) {
      result.push([...path]);
      return;
    }

    // 剪枝：和已超过目标值
    if (sum > target) {
      return;
    }

    // 遍历可选元素（可以从start开始，说明元素可以重复使用）
    for (let i = start; i < candidates.length; i++) {
      // 剪枝：已排序，如果sum + candidates[i] > target，后面的也不用试了
      if (sum + candidates[i] > target) {
        break;
      }

      path.push(candidates[i]);
      backtrack(i, sum + candidates[i]);  // 注意：start还是i，不是i+1（可重复）
      path.pop();
    }
  }

  backtrack(0, 0);
  return result;
}

/**
 * 组合总和问题（不可重复使用元素）
 */
function combinationSum2(
  candidates: number[],
  target: number
): number[][] {
  const result: number[][] = [];
  const path: number[] = [];
  const used = new Array(candidates.length).fill(false);

  // 排序以便去重
  candidates.sort((a, b) => a - b);

  function backtrack(start: number, sum: number): void {
    if (sum === target) {
      result.push([...path]);
      return;
    }

    if (sum > target) {
      return;
    }

    for (let i = start; i < candidates.length; i++) {
      // 去重：如果当前元素与上一个元素相同，跳过
      if (i > 0 && candidates[i] === candidates[i - 1] && !used[i - 1]) {
        continue;
      }

      if (sum + candidates[i] > target) {
        break;
      }

      path.push(candidates[i]);
      used[i] = true;
      backtrack(i + 1, sum + candidates[i]);  // i+1，不可重复使用
      path.pop();
      used[i] = false;
    }
  }

  backtrack(0, 0);
  return result;
}
```

### 3.3 子集问题（Subset / 幂集）

**问题描述：** 生成一个数组的所有子集（幂集）。

**问题分析：**
- 子集问题是组合问题的特例（从n个元素中选0到n个的所有组合）
- 对于n个元素，每个元素都有"选"和"不选"两种状态
- 所以总共有2^n个子集

```typescript
/**
 * 子集问题（幂集）
 *
 * 算法思想：
 * - 每个元素都有"选"和"不选"两种状态
 * - 共2^n种状态（n个元素）
 *
 * 时间复杂度：O(n × 2^n)
 * 空间复杂度：O(n × 2^n)（结果存储）
 */

function subsets(nums: number[]): number[][] {
  const result: number[][] = [];
  const path: number[] = [];

  /**
   * 回溯函数
   * @param start 当前可选择的起始索引
   */
  function backtrack(start: number): void {
    // 收集结果：每个状态都是有效的子集
    result.push([...path]);

    // 终止条件：已遍历完所有元素（实际上这里不需要显式终止）
    // 因为当start === nums.length时，循环不会执行
    if (start === nums.length) {
      return;
    }

    // 遍历选择：选或不选
    for (let i = start; i < nums.length; i++) {
      path.push(nums[i]);      // 选择nums[i]
      backtrack(i + 1);        // 递归处理后面的元素
      path.pop();              // 回溯：不选nums[i]
    }
  }

  backtrack(0);
  return result;
}

/**
 * 子集问题（含重复元素）- 幂集II
 *
 * 去重思路：
 * - 先排序
 * - 如果当前元素与上一个元素相同，只从上一个相同元素的下一位置开始选
 * - 避免重复枚举相同元素的子集
 */
function subsetsWithDup(nums: number[]): number[][] {
  const result: number[][] = [];
  const path: number[] = [];

  // 排序以便去重
  nums.sort((a, b) => a - b);

  function backtrack(start: number): void {
    result.push([...path]);

    if (start === nums.length) {
      return;
    }

    for (let i = start; i < nums.length; i++) {
      // 去重：如果当前元素与上一个元素相同，跳过
      // 条件：i > start 确保是同一层递归的去重，不是跨层
      if (i > start && nums[i] === nums[i - 1]) {
        continue;
      }

      path.push(nums[i]);
      backtrack(i + 1);
      path.pop();
    }
  }

  backtrack(0);
  return result;
}
```

### 3.4 N皇后问题（N-Queens）

**问题描述：** 在n×n的棋盘上放置n个皇后，使得它们互不攻击（任意两个皇后不在同一行、同一列、同一对角线）。

**问题分析：**
- 这是一个经典的回溯法问题
- 每行必须且只能放一个皇后
- 对角线条件：主对角线 (row - col) 相同，副对角线 (row + col) 相同

```typescript
/**
 * N皇后问题
 *
 * 算法思想：
 * - 逐行放置皇后
 * - 每行尝试所有列，检查是否与已放置的皇后冲突
 * - 冲突条件：同列、同主对角线、同副对角线
 *
 * 时间复杂度：O(n!)
 * 空间复杂度：O(n)
 */

function solveNQueens(n: number): string[][] {
  const result: string[][] = [];  // 存储最终结果
  const board: string[] = new Array(n).fill('').map(() => '.'.repeat(n));  // 棋盘

  /**
   * 检查在(row, col)位置放置皇后是否安全
   * @param board 当前棋盘
   * @param row 行索引
   * @param col 列索引
   * @returns 是否安全
   */
  function isValid(board: string[], row: number, col: number): boolean {
    // 检查同列是否有皇后（只需要检查上方，因为我们是逐行放置的）
    for (let i = 0; i < row; i++) {
      if (board[i][col] === 'Q') {
        return false;
      }
    }

    // 检查主对角线（左上到右下）
    // 主对角线特点：row - col 是常数
    for (let i = row - 1, j = col - 1; i >= 0 && j >= 0; i--, j--) {
      if (board[i][j] === 'Q') {
        return false;
      }
    }

    // 检查副对角线（右上到左下）
    // 副对角线特点：row + col 是常数
    for (let i = row - 1, j = col + 1; i >= 0 && j < n; i--, j++) {
      if (board[i][j] === 'Q') {
        return false;
      }
    }

    return true;
  }

  /**
   * 回溯函数
   * @param row 当前处理的行
   */
  function backtrack(row: number): void {
    // 终止条件：所有行都放置了皇后
    if (row === n) {
      result.push([...board]);  // 保存当前解
      return;
    }

    // 遍历当前行的所有列
    for (let col = 0; col < n; col++) {
      // 检查是否可以放置皇后
      if (isValid(board, row, col)) {
        // 放置皇后
        const chars = board[row].split('');
        chars[col] = 'Q';
        board[row] = chars.join('');

        // 递归处理下一行
        backtrack(row + 1);

        // 回溯：移除皇后
        chars[col] = '.';
        board[row] = chars.join('');
      }
    }
  }

  backtrack(0);
  return result;
}

/**
 * N皇后问题II - 只需要返回解的数量
 * 使用位运算优化
 */
function totalNQueens(n: number): number {
  let count = 0;

  /**
   * 位运算回溯
   * @param row 当前行
   * @param cols 列占用情况
   * @param diag1 主对角线占用情况
   * @param diag2 副对角线占用情况
   */
  function backtrack(row: number, cols: number, diag1: number, diag2: number): void {
    if (row === n) {
      count++;
      return;
    }

    // 计算当前行所有可用的位置
    // bits = 1 的位置表示可以放置皇后
    let bits = (~(cols | diag1 | diag2)) & ((1 << n) - 1);

    while (bits !== 0) {
      // 取最低位的1（可用位置）
      const position = bits & (-bits);

      // 放置皇后，更新状态
      backtrack(
        row + 1,
        cols | position,
        (diag1 | position) << 1,
        (diag2 | position) >> 1
      );

      // 移除最低位的1，继续尝试下一个位置
      bits = bits & (bits - 1);
    }
  }

  backtrack(0, 0, 0, 0);
  return count;
}
```

### 3.5 括号生成问题

**问题描述：** 生成所有有效的括号组合。

**问题分析：**
- 必须保持括号的有效性：左边括号数量 >= 右边括号数量
- 总共n对括号，2n个位置
- 左括号可以随时添加，右括号只能在左括号数量 > 右括号数量时添加

```typescript
/**
 * 括号生成
 *
 * 算法思想：
 * - 维护左右括号的数量
 * - 左括号可以随时添加
 * - 右括号只能在左括号数量 > 右括号数量时添加
 *
 * 时间复杂度：O(4^n / sqrt(n)) 卡特兰数
 * 空间复杂度：O(n)
 */

function generateParenthesis(n: number): string[] {
  const result: string[] = [];
  const path: string[] = [];

  /**
   * 回溯函数
   * @param left 剩余左括号数量
   * @param right 剩余右括号数量
   */
  function backtrack(left: number, right: number): void {
    // 终止条件：括号用完了
    if (left === 0 && right === 0) {
      result.push(path.join(''));
      return;
    }

    // 可以添加左括号（如果有剩余）
    if (left > 0) {
      path.push('(');
      backtrack(left - 1, right);
      path.pop();
    }

    // 可以添加右括号（如果左括号数量 > 右括号数量）
    if (left < right) {
      path.push(')');
      backtrack(left, right - 1);
      path.pop();
    }
  }

  backtrack(n, n);
  return result;
}
```

## 四、剪枝优化技巧

### 4.1 剪枝的基本思想

**剪枝（Pruning）** 是在回溯过程中，提前判断某个分支不可能产生有效解，从而跳过该分支的搜索。

**类比理解：** 想象你在寻找走出迷宫的最短路径。如果已经知道你当前所在位置距离出口的曼哈顿距离大于已知的最小路径长度，那么即使继续走下去也不可能找到更短的解这时就可以"剪枝"——不再探索这条路径。

### 4.2 剪枝的常见策略

```typescript
/**
 * 剪枝策略总结
 */

// 策略一：可行性剪枝
// 在组合总和问题中，如果当前和已超过目标值，剪枝
function 策略一_可行性剪枝(candidates: number[], target: number): number[][] {
  const result: number[][] = [];
  const path: number[] = [];

  candidates.sort((a, b) => a - b);

  function backtrack(start: number, sum: number): void {
    if (sum === target) {
      result.push([...path]);
      return;
    }

    // 可行性剪枝：和已超过目标值，不可能得到解
    if (sum > target) {
      return;
    }

    for (let i = start; i < candidates.length; i++) {
      if (sum + candidates[i] > target) {
        break;  // 剪枝：后面更大的也不用试了
      }
      path.push(candidates[i]);
      backtrack(i, sum + candidates[i]);
      path.pop();
    }
  }

  backtrack(0, 0);
  return result;
}

// 策略二：去重剪枝
// 在子集/组合问题中，如果元素重复，跳过重复选择
function 策略二_去重剪枝(nums: number[]): number[][] {
  const result: number[][] = [];
  const path: number[] = [];

  nums.sort((a, b) => a - b);

  function backtrack(start: number): void {
    result.push([...path]);

    if (start === nums.length) {
      return;
    }

    for (let i = start; i < nums.length; i++) {
      // 去重剪枝：相同元素，只选择第一个
      if (i > start && nums[i] === nums[i - 1]) {
        continue;
      }
      path.push(nums[i]);
      backtrack(i + 1);
      path.pop();
    }
  }

  backtrack(0);
  return result;
}

// 策略三：交换剪枝
// 在排列问题中，如果当前元素与已选择的后面的元素相同，且后面的元素未被使用，跳过
function 策略三_交换剪枝(nums: number[]): number[][] {
  const result: number[][] = [];
  const used = new Array(nums.length).fill(false);

  nums.sort((a, b) => a - b);

  function backtrack(path: number[]): void {
    if (path.length === nums.length) {
      result.push([...path]);
      return;
    }

    for (let i = 0; i < nums.length; i++) {
      // 去重剪枝：相同值的元素，上一个相同元素未使用则跳过
      if (used[i] || (i > 0 && nums[i] === nums[i-1] && !used[i-1])) {
        continue;
      }

      used[i] = true;
      path.push(nums[i]);
      backtrack(path);
      path.pop();
      used[i] = false;
    }
  }

  backtrack([]);
  return result;
}

// 策略四：界限剪枝
// 在组合总和问题中，如果剩余元素和小于目标值，剪枝
function 策略四_界限剪枝(nums: number[], target: number): number[][] {
  const result: number[][] = [];
  const path: number[] = [];

  nums.sort((a, b) => a - b);

  function backtrack(start: number, sum: number, restSum: number): void {
    if (sum === target) {
      result.push([...path]);
      return;
    }

    for (let i = start; i < nums.length; i++) {
      // 界限剪枝：如果 sum + nums[i] > target，剪枝
      if (sum + nums[i] > target) {
        break;
      }

      // 进一步剪枝：即使选择当前最小值，剩余和也不够
      // 如果 sum + nums[i] + 剩余最小值 > target，跳过
      // （需要预处理剩余元素的最小值）

      path.push(nums[i]);
      backtrack(i + 1, sum + nums[i], restSum - nums[i]);
      path.pop();
    }
  }

  backtrack(0, 0, nums.reduce((a, b) => a + b, 0));
  return result;
}
```

## 五、复杂度分析

### 5.1 时间复杂度分析方法

| 问题类型 | 状态空间 | 时间复杂度 | 说明 |
|----------|----------|------------|------|
| 全排列 | n! | O(n × n!) | n!个排列，每个O(n)复制 |
| 组合 | C(n,k) | O(C(n,k) × k) | C(n,k)个组合，每个O(k)复制 |
| 子集 | 2^n | O(n × 2^n) | 2^n个子集，每个O(n)复制 |
| N皇后 | n^n（实际剪枝后少得多） | O(n!) | 每行有n种选择 |

### 5.2 空间复杂度分析

```
空间复杂度 = 递归栈深度 + 路径存储 + 辅助空间
           = O(n) + O(n) + O(n)
           = O(n)

其中：
- 递归栈深度：最多n层
- 路径存储：最多n个元素
- 辅助空间：used数组、结果存储（结果本身不计入辅助空间）
```

## 七、总结

### 7.1 回溯法模板总结

```typescript
/**
 * 回溯法模板（Python版便于记忆）
 *
 * result = []
 * def backtrack(路径, 选择列表):
 *     if 满足终止条件:
 *         result.add(路径)
 *         return
 *
 *     for 选择 in 选择列表:
 *         做选择
 *         backtrack(路径, 新的选择列表)
 *         撤销选择
 */

// 核心要点：
// 1. 路径：已经做出的选择
// 2. 选择列表：当前可以做出的选择
// 3. 终止条件：满足目标，找到有效解
// 4. 剪枝：在循环中判断是否应该剪枝
```

### 7.2 回溯法 vs 动态规划

| 特性 | 回溯法 | 动态规划 |
|------|--------|----------|
| **搜索策略** | 深度优先 | 自底向上 |
| **子问题** | 独立 | 重叠 |
| **记忆化** | 不需要 | 需要 |
| **时间复杂度** | 指数级（无剪枝） | 多项式级 |
| **空间复杂度** | O(n)（递归栈） | O(n)（状态表） |

### 7.3 学习建议

1. **从简单开始：** 先掌握全排列、组合、子集三个基础问题
2. **理解去重：** 学会排序+used数组或起始索引去重
3. **掌握剪枝：** 理解剪枝条件，提高算法效率
4. **多做练习：** 回溯法题型固定，多刷题自然就熟练
