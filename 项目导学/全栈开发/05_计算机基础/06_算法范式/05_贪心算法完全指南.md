# 贪心算法完全指南

## 一、贪心算法基础

### 1.1 什么是贪心算法？

**贪心算法（Greedy Algorithm）** 是一种在每一步选择中都采取当前状态下最优（即局部最优）选择的算法策略。

**类比理解——购物时选最新鲜的菜：**

想象你去超市买菜，要做一顿大餐。你面临很多选择：番茄、黄瓜、鱼、肉...你会怎么做？

**贪心策略：** 每次去货架前，你都选最新鲜的那个，不考虑这顿饭的整体搭配。

**结果分析：**
- 优点：简单快速，不用想太多
- 缺点：可能不是最优的一桌菜
- 但有时候这样也能做出不错的一桌菜

**贪心算法的核心思想：** "目光短浅"但有效——每步只考虑眼前最好的，不回头比较。

### 1.2 贪心算法的特征

| 特征 | 说明 | 判断方法 |
|------|------|----------|
| **最优子结构** | 问题的最优解包含子问题的最优解 | 全局最优能由局部最优导出 |
| **贪心选择性** | 每步选择都是当前最优，不回溯 | 选择之后不需要考虑之后的选择 |
| **无后效性** | 一个状态的历史不会影响后续决策 | 选择序列确定后，不受之前选择影响 |

**如何判断一个问题是否可以用贪心：**
1. **证明**：证明每一步的最优解能够导致全局最优解
2. **反例**：尝试找反例，如果找不到反例，大概率可以用贪心
3. **经验**：某些经典问题（区间调度、区间覆盖等）天然适合贪心

### 1.3 贪心算法 vs 动态规划

| 特性 | 贪心算法 | 动态规划 |
|------|----------|----------|
| **选择策略** | 只考虑当前最优 | 考虑所有子问题 |
| **最优性** | 不保证全局最优 | 保证全局最优 |
| **效率** | 通常O(n) | 通常O(n²)或更高 |
| **自底向上** | 不需要 | 需要 |
| **记忆化** | 不需要 | 需要 |
| **适用场景** | 局部最优能导出全局最优 | 需要全局比较 |

```
贪心选择 vs 动态规划

贪心：只看眼前
┌─────────────────────────────────────────┐
│ 当前最优 ─────────────────────> 全局最优 │
│    ↓                                      │
│   每步选择局部最优                         │
│   不考虑其他分支                           │
└─────────────────────────────────────────┘

动态规划：全局比较
┌─────────────────────────────────────────┐
│    ┌──> 子问题1 ──┐                      │
│   ┌┴────────────┐│                      │
│ 全局 ← 子问题2 ← ─ ┘ ← 多个选择取最优   │
│   │└────────────┐│                      │
│    └──> 子问题3 ──┘                      │
└─────────────────────────────────────────┘
```

## 二、区间调度问题

### 2.1 基本区间调度

**问题描述：** 给定 n 个区间，每个区间有开始时间和结束时间，找出最多能安排多少个不重叠的区间。

**贪心策略：** 每次选择结束时间最早的区间。

**证明：** 选择结束时间最早的区间，能为后续区间留出更多空间。

```typescript
/**
 * 区间调度问题（无重叠区间最多数量）
 *
 * 贪心策略：按结束时间排序，选择最早结束的区间
 *
 * 时间复杂度：O(n log n)
 * 空间复杂度：O(1)
 */

interface Interval {
  start: number;
  end: number;
}

function eraseOverlapIntervals(intervals: Interval[]): number {
  if (intervals.length === 0) return 0;

  // 按结束时间排序
  intervals.sort((a, b) => a.end - b.end);

  // count记录选择的区间数量
  // end记录当前选择的区间的结束时间
  let count = 1;  // 至少选择一个区间
  let end = intervals[0].end;

  for (let i = 1; i < intervals.length; i++) {
    // 如果当前区间的开始时间 >= 上一个选择的区间的结束时间
    // 说明两个区间不重叠，可以选择当前区间
    if (intervals[i].start >= end) {
      count++;
      end = intervals[i].end;
    }
  }

  return intervals.length - count;  // 需要移除的区间数
}

/**
 * 区间调度（输出具体选择的区间）
 */
function intervalScheduling(intervals: Interval[]): Interval[] {
  if (intervals.length === 0) return [];

  // 按结束时间排序
  intervals.sort((a, b) => a.end - b.end);

  const result: Interval[] = [];
  let end = -Infinity;

  for (const interval of intervals) {
    if (interval.start >= end) {
      result.push(interval);
      end = interval.end;
    }
  }

  return result;
}
```

### 2.2 合并区间

**问题描述：** 给定一些区间，将所有重叠的区间合并。

```typescript
/**
 * 合并区间
 *
 * 贪心策略：按起点排序，依次合并重叠区间
 *
 * 时间复杂度：O(n log n)
 * 空间复杂度：O(n)
 */

function mergeIntervals(intervals: Interval[]): Interval[] {
  if (intervals.length <= 1) return intervals;

  // 按起点排序
  intervals.sort((a, b) => a.start - b.start);

  const result: Interval[] = [];
  let current = intervals[0];

  for (let i = 1; i < intervals.length; i++) {
    const interval = intervals[i];

    // 如果当前区间与上一个区间重叠（当前.start <= current.end）
    // 合并：更新current.end为两者的最大值
    if (interval.start <= current.end) {
      current.end = Math.max(current.end, interval.end);
    } else {
      // 不重叠，将current加入结果，开始新的current
      result.push(current);
      current = interval;
    }
  }

  // 别忘了最后一个区间
  result.push(current);

  return result;
}

/**
 * 合并区间（变体：合并所有重叠区间，返回重叠组）
 */
function mergeOverlappingGroups(intervals: Interval[]): Interval[][] {
  if (intervals.length <= 1) return [intervals];

  intervals.sort((a, b) => a.start - b.start);

  const groups: Interval[][] = [];
  let currentGroup: Interval[] = [intervals[0]];

  for (let i = 1; i < intervals.length; i++) {
    if (intervals[i].start <= currentGroup[currentGroup.length - 1].end) {
      // 与当前组重叠，加入当前组
      currentGroup.push(intervals[i]);
      // 更新当前组最后一个元素的end
      const last = currentGroup[currentGroup.length - 1];
      const prev = currentGroup[currentGroup.length - 2];
      last.start = Math.min(last.start, prev.start);
      last.end = Math.max(last.end, prev.end);
    } else {
      // 不重叠，开始新组
      groups.push(currentGroup);
      currentGroup = [intervals[i]];
    }
  }

  groups.push(currentGroup);
  return groups;
}
```

### 2.3 区间覆盖问题

**问题描述：** 给定目标区间和一组区间，选择最少的区间来覆盖目标区间。

```typescript
/**
 * 用最少的区间覆盖目标区间
 *
 * 贪心策略：
 * 1. 按起点排序
 * 2. 从目标区间的起始位置开始
 * 3. 每次选择能覆盖当前位置、且终点最远的区间
 * 4. 更新当前位置，继续选择
 *
 * 时间复杂度：O(n log n)
 */

function minIntervalsToCover(target: Interval, intervals: Interval[]): number {
  // 过滤掉无法覆盖目标起点的区间
  intervals = intervals.filter(
    i => i.end > target.start && i.start <= target.end
  );

  if (intervals.length === 0) return -1;

  intervals.sort((a, b) => a.start - b.start);

  let count = 0;
  let current = target.start;
  let i = 0;

  while (current < target.end && i < intervals.length) {
    let farthest = current;
    let farthestIndex = -1;

    // 在所有起点 <= current的区间中，选择终点最远的
    while (i < intervals.length && intervals[i].start <= current) {
      if (intervals[i].end > farthest) {
        farthest = intervals[i].end;
        farthestIndex = i;
      }
      i++;
    }

    if (farthestIndex === -1) {
      // 没有区间能覆盖当前位置，无法完全覆盖
      return -1;
    }

    count++;
    current = farthest;
  }

  return current >= target.end ? count : -1;
}
```

## 三、活动选择问题

### 3.1 基本活动选择

**问题描述：** 有 n 个活动，每个活动都有一个开始时间和结束时间。问最多能参加多少个活动（同一时间只能参加一个）。

**贪心策略：** 每次选择结束时间最早且与已选择活动不冲突的活动。

```typescript
/**
 * 活动选择问题
 *
 * 贪心策略：按结束时间排序，选择最早结束且不冲突的活动
 *
 * 时间复杂度：O(n log n)
 */

interface Activity {
  start: number;
  end: number;
}

function maxActivities(activitys: Activity[]): number {
  if (activitys.length === 0) return 0;

  // 按结束时间排序
  activitys.sort((a, b) => a.end - b.end);

  let count = 1;  // 选择第一个活动
  let lastEnd = activitys[0].end;

  for (let i = 1; i < activitys.length; i++) {
    if (activitys[i].start >= lastEnd) {
      count++;
      lastEnd = activitys[i].end;
    }
  }

  return count;
}

/**
 * 活动选择（输出具体选择）
 */
function selectActivities(activitys: Activity[]): Activity[] {
  if (activitys.length === 0) return [];

  activitys.sort((a, b) => a.end - b.end);

  const selected: Activity[] = [activitys[0]];
  let lastEnd = activitys[0].end;

  for (let i = 1; i < activitys.length; i++) {
    if (activitys[i].start >= lastEnd) {
      selected.push(activitys[i]);
      lastEnd = activitys[i].end;
    }
  }

  return selected;
}
```

### 3.2 加油站问题

**问题描述：** 一辆车要绕一圈，路上有加油站。已知每个加油站能加的油量和相邻加油站之间的距离，问能否绕一圈。

```typescript
/**
 * 加油站问题
 *
 * 方法1：贪心
 * 思路：如果从站点i无法到达站点j，那么从i到j之间的任何站点出发都无法到达j
 * 因此跳过这些站点，从j+1重新开始
 *
 * 时间复杂度：O(n)
 * 空间复杂度：O(1)
 */

function canCompleteCircuit(gas: number[], cost: number[]): number {
  const n = gas.length;

  // 总油量必须 >= 总消耗
  let totalTank = 0;
  for (let i = 0; i < n; i++) {
    totalTank += gas[i] - cost[i];
  }
  if (totalTank < 0) return -1;  // 总油量不够

  // 找到起始站点
  let tank = 0;       // 当前油量
  let start = 0;      // 起始站点

  for (let i = 0; i < n; i++) {
    tank += gas[i] - cost[i];

    // 如果当前油量变为负数，无法从start到达i+1
    if (tank < 0) {
      start = i + 1;  // 从下一个站点重新开始
      tank = 0;       // 重置油量
    }
  }

  return start === n ? 0 : start;
}

/**
 * 加油站问题（输出具体路线）
 */
function completeCircuitWithRoute(
  gas: number[],
  cost: number[]
): {can: boolean, start: number, route: number[]} {
  const n = gas.length;

  const totalTank = gas.reduce((sum, g, i) => sum + g - cost[i], 0);
  if (totalTank < 0) {
    return {can: false, start: -1, route: []};
  }

  let tank = 0;
  let start = 0;

  for (let i = 0; i < n; i++) {
    tank += gas[i] - cost[i];
    if (tank < 0) {
      start = i + 1;
      tank = 0;
    }
  }

  // 构建路线
  const route: number[] = [];
  let pos = start;
  do {
    route.push(pos);
    pos = (pos + 1) % n;
  } while (pos !== start);

  return {can: true, start, route};
}
```

## 四、跳跃游戏

### 4.1 跳跃游戏 I

**问题描述：** 给定一个数组 nums，nums[i] 表示从位置 i 最多可以跳多远。问能否到达最后一个位置。

```typescript
/**
 * 跳跃游戏I
 *
 * 贪心策略：
 * 维护一个变量reach表示当前能到达的最远位置
 * 遍历数组，更新reach
 * 如果reach >= n-1，说明能到达末尾
 *
 * 时间复杂度：O(n)
 * 空间复杂度：O(1)
 */

function canJump(nums: number[]): boolean {
  let reach = 0;  // 当前能到达的最远位置

  for (let i = 0; i < nums.length; i++) {
    // 如果当前位置超出reach，说明无法到达
    if (i > reach) return false;

    // 更新reach
    reach = Math.max(reach, i + nums[i]);

    // 如果reach已经能到达末尾
    if (reach >= nums.length - 1) return true;
  }

  return true;
}

/**
 * 跳跃游戏I（找出跳跃路径）
 */
function jumpPath(nums: number[]): number[] | null {
  const n = nums.length;
  if (n === 0) return null;

  // 如果无法到达，返回null
  let reach = 0;
  for (let i = 0; i < n - 1; i++) {
    if (i > reach) return null;
    reach = Math.max(reach, i + nums[i]);
  }
  if (reach < n - 1) return null;

  // 找出具体跳跃路径（使用最少步数）
  const path: number[] = [0];
  let lastReach = 0;
  let currentPos = 0;

  while (currentPos < n - 1) {
    let farthest = currentPos;
    let nextPos = currentPos;

    for (let i = currentPos + 1; i <= Math.min(currentPos + nums[currentPos], n - 1); i++) {
      if (i + nums[i] > farthest) {
        farthest = i + nums[i];
        nextPos = i;
      }
    }

    path.push(nextPos);
    currentPos = nextPos;
  }

  return path;
}
```

### 4.2 跳跃游戏 II

**问题描述：** 同一个数组，问最少需要跳多少次才能到达最后一个位置。

```typescript
/**
 * 跳跃游戏II（最少跳跃次数）
 *
 * 贪心策略：
 * 在当前可达范围内，选择能跳到最远位置的点
 * 每当到达当前范围的边界时，步数+1，并更新边界
 *
 * 时间复杂度：O(n)
 * 空间复杂度：O(1)
 */

function jump(nums: number[]): number {
  const n = nums.length;
  if (n <= 1) return 0;

  let jumps = 0;      // 跳跃次数
  let currentEnd = 0; // 当前跳跃能到达的边界
  let farthest = 0;   // 在当前边界内能跳到的最远位置

  // 注意：遍历到 n-2，因为到达 n-1 时不需要再跳
  for (let i = 0; i < n - 1; i++) {
    // 更新能跳到的最远位置
    farthest = Math.max(farthest, i + nums[i]);

    // 到达当前边界，需要再跳一次
    if (i === currentEnd) {
      jumps++;
      currentEnd = farthest;

      // 如果当前边界已经能到达或超过末尾
      if (currentEnd >= n - 1) break;
    }
  }

  return jumps;
}

/**
 * 跳跃游戏II（输出具体跳跃路径）
 */
function jumpWithPath(nums: number[]): {minJumps: number, path: number[]} {
  const n = nums.length;
  if (n <= 1) return {minJumps: 0, path: [0]};

  const path: number[] = [0];
  let jumps = 0;
  let currentEnd = 0;
  let farthest = 0;

  for (let i = 0; i < n - 1; i++) {
    farthest = Math.max(farthest, i + nums[i]);

    if (i === currentEnd) {
      jumps++;

      // 选择下一步跳到哪里
      let nextPos = currentEnd;
      for (let j = currentEnd + 1; j <= Math.min(farthest, n - 1); j++) {
        if (j + nums[j] >= farthest) {
          nextPos = j;
          break;
        }
      }

      path.push(nextPos);
      currentEnd = farthest;

      if (currentEnd >= n - 1) break;
    }
  }

  return {minJumps: jumps, path};
}
```

## 五、霍夫曼编码

### 5.1 霍夫曼编码原理

**霍夫曼编码（Huffman Coding）** 是一种用于无损数据压缩的贪心算法。

**核心思想：** 出现频率高的字符用较短的编码，出现频率低的字符用较长的编码。

**编码过程：**
1. 统计每个字符的出现频率
2. 用频率作为权重，构建霍夫曼树
3. 从根到叶子节点的路径，左边为0，右边为1
4. 叶子节点的编码就是该字符的霍夫曼编码

```typescript
/**
 * 霍夫曼编码
 *
 * 贪心策略：每次选择频率最小的两个节点合并
 *
 * 时间复杂度：O(n log n)
 * 空间复杂度：O(n)
 */

interface HuffmanNode {
  char?: string;      // 字符（叶子节点有）
  freq: number;       // 频率
  left?: HuffmanNode;
  right?: HuffmanNode;
}

class HuffmanCoding {
  private root: HuffmanNode | null = null;
  private codes: Map<string, string> = new Map();
  private reverseCodes: Map<string, string> = new Map();

  /**
   * 构建霍夫曼树
   */
  build(frequencies: Map<string, number>): HuffmanNode {
    // 创建优先队列（最小堆）
    const pq: HuffmanNode[] = [];

    for (const [char, freq] of frequencies) {
      pq.push({char, freq});
    }

    // 按频率排序（实际上应该用优先队列）
    pq.sort((a, b) => a.freq - b.freq);

    // 构建霍夫曼树
    while (pq.length > 1) {
      // 取出频率最小的两个节点
      const left = pq.shift()!;
      const right = pq.shift()!;

      // 合并节点
      const parent: HuffmanNode = {
        freq: left.freq + right.freq,
        left,
        right
      };

      // 重新插入
      pq.push(parent);
      pq.sort((a, b) => a.freq - b.freq);
    }

    this.root = pq[0];
    this.generateCodes(this.root, '');

    return this.root;
  }

  /**
   * 生成编码表
   */
  private generateCodes(node: HuffmanNode, code: string): void {
    if (!node.char) {
      // 非叶子节点
      if (node.left) this.generateCodes(node.left, code + '0');
      if (node.right) this.generateCodes(node.right, code + '1');
    } else {
      // 叶子节点
      this.codes.set(node.char, code || '0');  // 只有一个字符时编码为0
      this.reverseCodes.set(code || '0', node.char);
    }
  }

  /**
   * 编码
   */
  encode(text: string): string {
    let result = '';
    for (const char of text) {
      const code = this.codes.get(char);
      if (code) {
        result += code;
      }
    }
    return result;
  }

  /**
   * 解码
   */
  decode(encoded: string): string {
    let result = '';
    let current = '';

    for (const bit of encoded) {
      current += bit;
      const char = this.reverseCodes.get(current);
      if (char) {
        result += char;
        current = '';
      }
    }

    return result;
  }

  /**
   * 获取编码表
   */
  getCodes(): Map<string, string> {
    return new Map(this.codes);
  }
}

/**
 * 霍夫曼编码使用示例
 */
function huffmanExample() {
  const text = "this is an example for huffman encoding";

  // 统计频率
  const frequencies = new Map<string, number>();
  for (const char of text) {
    frequencies.set(char, (frequencies.get(char) || 0) + 1);
  }

  // 构建霍夫曼编码
  const huffman = new HuffmanCoding();
  huffman.build(frequencies);

  // 编码
  const encoded = huffman.encode(text);
  const codes = huffman.getCodes();

  // 输出编码表
  console.log("编码表:");
  for (const [char, code] of codes) {
    console.log(`  '${char}': ${code}`);
  }

  // 解码验证
  const decoded = huffman.decode(encoded);
  console.log("\n原文:", text);
  console.log("编码:", encoded);
  console.log("解码:", decoded);
  console.log("验证:", text === decoded ? "成功" : "失败");
}
```

## 六、分发饼干

### 6.1 基本问题

**问题描述：** 假设你是一位家长，有一群孩子和一堆饼干。每个孩子有一个"贪心指数"（对饼干大小的最小需求），每个饼干有一个大小。你给每个孩子最多一块饼干，让孩子满意。问最多能让多少个孩子满意？

```typescript
/**
 * 分发饼干
 *
 * 贪心策略：
 * 将孩子的贪心指数和饼干大小分别排序
 * 用双指针遍历，优先满足贪心指数小的孩子
 *
 * 时间复杂度：O(n log n)
 * 空间复杂度：O(1)
 */

function findContentChildren(children: number[], cookies: number[]): number {
  // 排序
  children.sort((a, b) => a - b);
  cookies.sort((a, b) => a - b);

  let child = 0;  // 孩子指针
  let cookie = 0; // 饼干指针

  while (child < children.length && cookie < cookies.length) {
    // 如果当前饼干能满足当前孩子的需求
    if (cookies[cookie] >= children[child]) {
      child++;  // 孩子满意，移动到下一个孩子
    }
    // 不管是否满足，都移动到下一块饼干
    cookie++;
  }

  return child;  // 满意的孩子数量
}

/**
 * 分发饼干（输出具体分配方案）
 */
function findContentChildrenWithAssignment(
  children: number[],
  cookies: number[]
): {count: number, assignments: [number, number][]} {
  children.sort((a, b) => a - b);
  cookies.sort((a, b) => a - b);

  const assignments: [number, number][] = [];
  let child = 0;
  let cookie = 0;

  while (child < children.length && cookie < cookies.length) {
    if (cookies[cookie] >= children[child]) {
      assignments.push([children[child], cookies[cookie]]);
      child++;
    }
    cookie++;
  }

  return {count: child, assignments};
}
```

## 七、Trie树与贪心

### 7.1 前缀Trie树

**Trie树（前缀树/字典树）** 虽然本身不是贪心算法，但它经常与贪心策略结合使用，比如实现前缀匹配、单词搜索等功能。

```typescript
/**
 * Trie树（前缀树）
 *
 * 时间复杂度：
 *   - 插入：O(m)，m为字符串长度
 *   - 搜索：O(m)，m为字符串长度
 *   - 前缀搜索：O(m)
 *
 * 空间复杂度：O(ALPHABET_SIZE × m × n)，其中n为字符串数量，m为平均长度
 */

class TrieNode {
  children: Map<string, TrieNode>;
  isEnd: boolean;  // 是否是单词结尾
  frequency: number;  // 词频（用于贪心相关问题）

  constructor() {
    this.children = new Map();
    this.isEnd = false;
    this.frequency = 0;
  }
}

class Trie {
  private root: TrieNode;

  constructor() {
    this.root = new TrieNode();
  }

  /**
   * 插入一个单词
   */
  insert(word: string): void {
    let node = this.root;

    for (const char of word) {
      if (!node.children.has(char)) {
        node.children.set(char, new TrieNode());
      }
      node = node.children.get(char)!;
    }

    node.isEnd = true;
    node.frequency++;
  }

  /**
   * 搜索一个单词
   */
  search(word: string): boolean {
    let node = this.searchPrefix(word);
    return node !== null && node.isEnd;
  }

  /**
   * 搜索前缀
   */
  startsWith(prefix: string): boolean {
    return this.searchPrefix(prefix) !== null;
  }

  private searchPrefix(prefix: string): TrieNode | null {
    let node = this.root;

    for (const char of prefix) {
      if (!node.children.has(char)) {
        return null;
      }
      node = node.children.get(char)!;
    }

    return node;
  }

  /**
   * 找到最长前缀（用于IP路由等）
   */
  findLongestPrefix(prefix: string): string {
    let node = this.root;
    let longestPrefix = '';

    for (const char of prefix) {
      if (!node.children.has(char)) {
        break;
      }
      node = node.children.get(char)!;
      longestPrefix += char;
      if (!node.isEnd) {
        // 可以继续找更长的前缀
      }
    }

    return longestPrefix;
  }

  /**
   * 找到以给定前缀开头的所有单词（需要结合DFS）
   */
  findWordsWithPrefix(prefix: string): string[] {
    const node = this.searchPrefix(prefix);
    if (!node) return [];

    const words: string[] = [];
    this.collectWords(node, prefix, words);
    return words;
  }

  private collectWords(node: TrieNode, prefix: string, words: string[]): void {
    if (node.isEnd) {
      words.push(prefix);
    }

    for (const [char, child] of node.children) {
      this.collectWords(child, prefix + char, words);
    }
  }
}
```

## 八、其他经典贪心问题

### 8.1 柠檬水找零

```typescript
/**
 * 柠檬水找零
 *
 * 贪心策略：
 * 收到20美元时，优先找零10+5（因为5美元更万能）
 *
 * 时间复杂度：O(n)
 * 空间复杂度：O(1)
 */

function lemonadeChange(bills: number[]): boolean {
  // five表示5美元的数量，ten表示10美元的数量
  // 不需要20美元的找零，因为没有需要找零20的情况
  let five = 0, ten = 0;

  for (const bill of bills) {
    if (bill === 5) {
      // 收到5美元，直接收下
      five++;
    } else if (bill === 10) {
      // 收到10美元，需要找零5美元
      if (five === 0) return false;
      five--;
      ten++;
    } else {
      // 收到20美元，需要找零15美元
      // 贪心：优先使用10+5，如果没有10，则使用3个5
      if (ten > 0 && five > 0) {
        ten--;
        five--;
      } else if (five >= 3) {
        five -= 3;
      } else {
        return false;
      }
    }
  }

  return true;
}
```

### 8.2 摆动序列

```typescript
/**
 * 摆动序列
 *
 * 问题：找到一个序列的子序列，使得序列呈上升-下降-上升-下降...形式
 *
 * 贪心策略：
 * 统计"峰值"的数量
 *
 * 时间复杂度：O(n)
 * 空间复杂度：O(1)
 */

function wiggleMaxLength(nums: number[]): number {
  const n = nums.length;
  if (n <= 1) return n;

  // up表示到当前位置的最长上升摆动序列长度
  // down表示到当前位置的最长下降摆动序列长度
  let up = 1, down = 1;

  for (let i = 1; i < n; i++) {
    if (nums[i] > nums[i - 1]) {
      // 呈上升趋势
      up = down + 1;
    } else if (nums[i] < nums[i - 1]) {
      // 呈下降趋势
      down = up + 1;
    }
    // 如果相等，不改变长度（摆动序列需要变化）
  }

  return Math.max(up, down);
}
```

### 8.3 任务调度器

```typescript
/**
 * 任务调度器
 *
 * 问题：给定任务类型数组和冷却时间n，问完成所有任务的最短时间
 *
 * 贪心策略：
 * 计算最多出现的任务数量maxExec
 * 这些任务之间需要冷却
 * 构建形如：AAA...BBB...CCC... 的序列
 *
 * 时间复杂度：O(n)
 * 空间复杂度：O(1)
 */

function leastInterval(tasks: string[], n: number): number {
  // 统计每个任务的出现次数
  const count = new Map<string, number>();
  for (const task of tasks) {
    count.set(task, (count.get(task) || 0) + 1);
  }

  // 找到最大出现次数
  let maxCount = 0;
  let maxCountTasks = 0;
  for (const c of count.values()) {
    if (c > maxCount) {
      maxCount = c;
      maxCountTasks = 1;
    } else if (c === maxCount) {
      maxCountTasks++;
    }
  }

  // 计算最小时间
  // 公式：max(len, (maxCount - 1) * (n + 1) + maxCountTasks)
  const partCount = maxCount - 1;
  const partLen = n - (maxCountTasks - 1);
  const emptySlots = partCount * partLen;
  const remainingTasks = tasks.length - maxCount * maxCountTasks;

  // 如果有剩余任务填充空槽位
  return Math.max(tasks.length, emptySlots + remainingTasks + maxCount * maxCountTasks);
}
```

## 十、总结

### 10.1 贪心算法核心要点

```
贪心算法要点

核心思想：每步选择当前最优，寄希望于这样能得到全局最优

判断标准：
1. 最优子结构：局部最优能导出全局最优
2. 贪心选择性：选择之后不回溯
3. 无后效性：历史选择不影响后续

解题步骤：
1. 排序相关：先排序
2. 优先队列：用优先队列选择当前最优
3. 双指针：左右指针配合贪心
4. 数学证明：证明贪心策略的正确性

常见贪心问题：
- 区间调度
- 跳跃游戏
- 加油站
- 霍夫曼编码
- 活动选择
- 分发饼干
```
