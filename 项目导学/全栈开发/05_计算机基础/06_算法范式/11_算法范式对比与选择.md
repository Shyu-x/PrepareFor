# 算法范式对比与选择

## 一、算法范式总览

### 1.1 六大算法范式对比

```
┌──────────────┬────────────────┬─────────────┬─────────────────┐
│   范式       │    核心思想    │  时间复杂度  │     空间复杂度   │
├──────────────┼────────────────┼─────────────┼─────────────────┤
│  暴力搜索    │  枚举所有可能  │ O(2^n) ~ O(n!)│    O(n)        │
│  贪心算法    │  局部最优贪心  │ O(n log n)  │    O(n)         │
│  分治算法    │  分而治之     │ O(n log n)  │    O(log n)     │
│  动态规划    │  存储子问题    │ O(n) ~ O(n²)│    O(n) ~ O(n²) │
│  回溯算法    │  试探+回退     │ O(n!) ~ O(2^n)│   O(n)         │
│  分支限界    │  搜索+界限剪枝 │ 最坏指数级   │    O(节点数)    │
└──────────────┴────────────────┴─────────────┴─────────────────┘
```

### 1.2 各范式特点总结

```
暴力搜索：
✓ 最简单直接的解法
✓ 可作为 baseline
✗ 时间复杂度高
适用：解空间小、无明显规律

贪心算法：
✓ 高效、简单
✓ 不需要存储状态
✗ 需要证明正确性
✗ 可能不是全局最优
适用：具有贪心选择性质的问题

分治算法：
✓ 分解独立子问题
✓ 递归合并
✗ 需要合并步骤
适用：子问题独立、可合并

动态规划：
✓ 避免重复计算
✓ 处理重叠子问题
✗ 需要定义状态和转移
适用：最优子结构 + 重叠子问题

回溯算法：
✓ 可找出所有解
✓ 系统性搜索
✗ 指数级时间
适用：组合优化、排列组合

分支限界：
✓ 找最优解
✓ 界限剪枝
✗ 界限计算复杂
✗ 需要大量存储
适用：组合优化问题的最优解
```

## 二、问题类型与算法选择

### 2.1 根据问题特征选择

```
特征检测清单：

1. 问题是否要求找出最优解？
   ├─ 是 → 贪心（需要证明）/ 动态规划 / 分支限界
   └─ 否 → 暴力搜索 / 回溯

2. 问题是否具有最优子结构？
   ├─ 是 → 考虑动态规划
   └─ 否 → 考虑贪心或暴力

3. 子问题是否重叠？
   ├─ 是 → 动态规划（记忆化）
   └─ 否 → 分治算法

4. 是否可以用贪心选择？
   ├─ 是 → 贪心算法
   └─ 否 → 动态规划

5. 是否需要找出所有解？
   ├─ 是 → 回溯算法
   └─ 否 → 贪心/DP/分支限界

6. 解空间是否很小？
   ├─ 是 → 暴力搜索/回溯
   └─ 否 → 需要剪枝或更聪明算法
```

### 2.2 决策树

```
问题分析流程：

开始
  │
  ├─ 是否是优化问题？（最大/最小/最多/最少）
  │     │
  │     ├─ 是 → 是否有最优子结构？
  │     │     │
  │     │     ├─ 是 → 贪心选择性质成立？
  │     │     │     │
  │     │     │     ├─ 是 → 贪心算法
  │     │     │     └─ 否 → 动态规划
  │     │     │
  │     │     └─ 否 → 分支限界 / 近似算法
  │     │
  │     └─ 否 → 需要找出所有解？
  │           │
  │           ├─ 是 → 回溯算法
  │           └─ 否 → 暴力搜索
  │
  └─ 是否是查找问题？
        │
        ├─ 是 → 数据有序？
        │     │
        │     ├─ 是 → 二分搜索 / 三分搜索
        │     └─ 否 → 双指针 / 哈希
        │
        └─ 否 → 分析具体约束条件
```

## 四、动态规划问题分类

### 4.1 DP 问题识别

```
DP 问题识别特征：

1. 计数问题
   - 有多少种方式到达目标？
   - 例：爬楼梯、背包问题

2. 最值问题
   - 最大化/最小化某值？
   - 例：最大子数组和、最长递增子序列

3. 存在性问题
   - 是否存在某解？
   - 例：分割等和子集、博弈问题

4. 路径问题
   - 网格/树的路径
   - 例：不同路径、最小路径和
```

### 4.2 DP 状态定义技巧

```typescript
/**
 * DP 状态定义常见模式
 */

/**
 * 1. 线性 DP（一维）
 * 状态：dp[i] = 以 i 结尾的最佳解
 * 问题：爬楼梯、LIS、斐波那契

/**
 * 2. 区间 DP（二维）
 * 状态：dp[i][j] = 区间 [i,j] 的最佳解
 * 问题：矩阵链乘、戳气球、回文分割

/**
 * 3. 背包 DP（二维）
 * 状态：dp[i][j] = 前 i 个物品、容量 j 的最佳解
 * 问题：0-1 背包、完全背包、多重背包

/**
 * 4. 字符串 DP（二维）
 * 状态：dp[i][j] = text1 前 i 个和 text2 前 j 个的关系
 * 问题：LCS、编辑距离、正则匹配

/**
 * 5. 树形 DP
 * 状态：在树上的子树最优解
 * 问题：树的重心、树的直径

/**
 * 6. 状态压缩 DP
 * 状态：用位掩码表示状态
 * 问题：旅行商、棋盘覆盖、集合幂
 */
```

## 五、贪心问题识别

### 5.1 贪心问题特征

```
贪心问题识别特征：

1. 活动选择类
   - 选择不重叠的最大集合
   - 关键：按结束时间排序

2. 分配问题
   - 将物品分配给某人
   - 关键：按某个指标排序

3. 区间调度
   - 选择最大数量的不重叠区间
   - 关键：按结束时间排序

4. Huffman 编码
   - 最优前缀码
   - 关键：按频率构建哈夫曼树

5. 最小生成树
   - 连通图的最小边权和
   - 关键：Prim 或 Kruskal

6. 单源最短路径
   - 从某点到其他点的最短路径
   - 关键： Dijkstra
```

### 5.2 贪心正确性证明

```typescript
/**
 * 贪心正确性证明方法
 */

/**
 * 1. 交换论证（Exchange Argument）
 *
 * 步骤：
 * 1. 假设存在一个最优解 O
 * 2. 比较 O 和贪心解 G 的第一个不同选择
 * 3. 证明将 O 的选择替换为 G 的选择不会变差
 * 4. 通过迭代将 O 转化为 G
 *
 * 例子：活动选择问题
 * - 贪心选择结束时间最早的活动
 * - 设 O 和 G 的第一个不同是 O 选了 a，G 选了 b（b 结束更早）
 * - 将 O 中的 a 替换为 b，因为 b 结束更早，后面有更多空间
 * - 所以替换后仍是可行解
 */

/**
 * 2. 归纳论证（Induction）
 *
 * 步骤：
 * 1. 证明贪心对基本情况成立
 * 2. 假设贪心对规模 n-1 成立
 * 3. 证明对规模 n 也成立
 *
 * 例子：找零钱问题（对于某些硬币系统）
 */

/**
 * 3. 界论证（Bound Argument）
 *
 * 步骤：
 * 1. 证明贪心解的下界
 * 2. 证明任何解的上界
 * 3. 证明上下界相等
 */
```

## 六、面试中的算法选择策略

### 6.1 面试思考流程

```typescript
/**
 * 面试中面对算法问题的思考流程：
 *
 * Step 1: 理解问题
 * - 输入是什么？输出是什么？
 * - 有没有示例？
 * - 有没有特殊约束？
 *
 * Step 2: 确定问题类型
 * - 是查找问题？计数问题？最值问题？
 * - 数据是否有序？
 * - 是否需要所有解？
 *
 * Step 3: 初步思路
 * - 暴力解法是什么？能优化吗？
 * - 有没有明显的规律可以利用？
 * - 可以用什么数据结构？
 *
 * Step 4: 确定算法
 * - 套用已知模式
 * - 验证复杂度是否可接受
 *
 * Step 5: 编码实现
 * - 先实现基本版本
 * - 考虑边界情况
 * - 优化（如需要）
 *
 * Step 6: 检查
 * - 复杂度分析
 * - 测试用例
 */
```

### 6.2 复杂度选择参考

```
复杂度可接受范围（面试）：

数据规模 n          可接受复杂度
──────────────────────────────────
n ≤ 10              O(n!), O(2^n)
n ≤ 20              O(2^n), O(n×2^n)
n ≤ 100             O(n³)
n ≤ 1000            O(n²), O(n² log n)
n ≤ 10^5            O(n log n)
n ≤ 10^6            O(n), O(n log k)
n ≤ 10^7            O(n), O(log n)

面试中的目标：
1. 先给出暴力解（但要说明复杂度问题）
2. 优化到 O(n²) 或 O(n log n)
3. 进一步优化到 O(n)（如果可能）
```

## 八、本章小结

### 8.1 选择总结

```
算法选择快速参考：

1. 需要最优解且有最优子结构？
   ├─ 贪心选择性质成立 → 贪心
   ├─ 重叠子问题 → 动态规划
   └─ 离散组合优化 → 分支限界

2. 有序数据查找？
   └─ 二分搜索 / 三分搜索

3. 子串/子数组问题？
   └─ 双指针 / 滑动窗口

4. 枚举所有解？
   └─ 回溯算法

5. 无明显规律？
   ├─ 解空间小 → 暴力搜索
   └─ 其他 → 考虑动态规划
```

### 8.2 学习建议

```
算法学习路径：

1. 基础扎实
   - 数组、链表、树基础操作
   - 排序和搜索基础
   - 基本数据结构

2. 模式掌握
   - 双指针模式
   - 滑动窗口模式
   - 二分搜索模式
   - 回溯模式

3. 高级技巧
   - 动态规划
   - 贪心证明
   - 分支限界
   - 图论算法

4. 持续练习
   - 按类别刷题
   - 总结归纳
   - 理解本质
```
