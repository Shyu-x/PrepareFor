# 分治法算法完全指南

## 一、分治法基础概念

### 1.1 分治法的定义

**分治法（Divide and Conquer）** 是一种极其重要的算法设计范式。它的核心思想可以概括为三个步骤：**分（Divide）**、**治（Conquer）**、**合（Merge）**。

**类比理解：** 想象你要整理一个大型图书馆的书籍。你可以这样做：
1. **分：** 先把书按照学科分成大类（如：理工类、文史类、艺术类）
2. **治：** 每个大类再细分成小类（如：理工类分成数学、物理、化学等）
3. **合：** 最后把分好类的书籍摆放到对应的书架上

这就是分治法的思想——大问题分解成小问题，分别解决后再合并。

### 1.2 分治法的三步走

```
分治法三步走

第一步：分（Divide）
┌─────────────────────────┐
│      原问题规模n         │
│    ┌───────────────┐    │
│    │ 分成 a 个子问题│    │
│    │ 规模 n/b      │    │
│    └───────────────┘    │
└─────────────────────────┘
           ↓
第二步：治（Conquer）
┌───┐ ┌───┐ ┌───┐ ┌───┐
│小1│ │小2│ │小3│ │...│
│n/b│ │n/b│ │n/b│ │   │
└───┘ └───┘ └───┘ └───┘
   ↓      ↓      ↓
递归解决每个子问题

第三步：合（Merge）
┌─────────────────────────┐
│   合并 a 个子问题的解     │
│   得到原问题的解         │
└─────────────────────────┘
```

### 1.3 分治法的必要条件

| 条件 | 说明 | 示例 |
|------|------|------|
| **可分解性** | 问题可以分解成若干个相似的子问题 | 排序可以分成左右两半分别排序 |
| **独立性** | 子问题之间相互独立，不共享状态 | 左右子数组之间没有依赖 |
| **可合并性** | 子问题的解可以合并成原问题的解 | 合并两个有序数组 |
| **基本情况** | 有直接可解的基本情况 | 子问题规模为1时直接返回 |

## 二、二分查找及其扩展

### 2.1 二分查找基础

**二分查找（Binary Search）** 是分治法最经典的应用之一。

**核心思想：** 在有序数组中，每次将搜索范围缩小一半，快速定位目标元素。

**类比理解：** 想象你在翻字典找一个单词。你不会从头一页一页找，而是：
1. 翻开字典中间页
2. 判断目标单词在左半部分还是右半部分
3. 只在目标所在的半部分继续搜索
4. 重复直到找到目标

```typescript
/**
 * 二分查找基础版本
 *
 * 算法思想：
 * - 在有序数组中查找目标值
 * - 每次将搜索范围缩小一半
 * - 如果中间元素等于目标，找到
 * - 如果中间元素大于目标，在左半部分继续查找
 * - 如果中间元素小于目标，在右半部分继续查找
 *
 * 时间复杂度：O(log n)
 * 空间复杂度：O(1)
 */

function binarySearch(nums: number[], target: number): number {
  // 定义搜索范围：[left, right]
  let left = 0;
  let right = nums.length - 1;

  // 循环条件：搜索范围不为空
  while (left <= right) {
    // 防止整数溢出，使用 left + (right - left) / 2
    // 而不是 (left + right) / 2
    const mid = left + Math.floor((right - left) / 2);

    if (nums[mid] === target) {
      // 找到目标，返回索引
      return mid;
    } else if (nums[mid] < target) {
      // 目标在右半部分，缩小左边界
      left = mid + 1;
    } else {
      // 目标在左半部分，缩小右边界
      right = mid - 1;
    }
  }

  // 未找到目标
  return -1;
}

/**
 * 二分查找（递归版本）
 */
function binarySearchRecursive(
  nums: number[],
  target: number,
  left: number = 0,
  right: number = nums.length - 1
): number {
  // 基本情况：搜索范围为空
  if (left > right) {
    return -1;
  }

  const mid = left + Math.floor((right - left) / 2);

  if (nums[mid] === target) {
    return mid;
  } else if (nums[mid] < target) {
    return binarySearchRecursive(nums, target, mid + 1, right);
  } else {
    return binarySearchRecursive(nums, target, left, mid - 1);
  }
}
```

### 2.2 二分查找的变体

```typescript
/**
 * 二分查找变体
 */

// 变体1：查找左边界（第一个 >= target 的位置）
function lowerBound(nums: number[], target: number): number {
  let left = 0;
  let right = nums.length;  // 注意：right = n，不是 n-1

  while (left < right) {  // 注意：left < right，不是 <=
    const mid = left + Math.floor((right - left) / 2);

    if (nums[mid] < target) {
      // 目标在右半部分
      left = mid + 1;
    } else {
      // nums[mid] >= target，左边界可能在左半部分或就是mid
      // 继续向左搜索
      right = mid;
    }
  }

  // left 就是左边界
  return left;
}

// 变体2：查找右边界（最后一个 <= target 的位置）
function upperBound(nums: number[], target: number): number {
  let left = 0;
  let right = nums.length;  // 注意：right = n

  while (left < right) {
    const mid = left + Math.floor((right - left) / 2);

    if (nums[mid] <= target) {
      // nums[mid] <= target，右边界可能在右半部分或就是mid
      // 继续向右搜索
      left = mid + 1;
    } else {
      // nums[mid] > target，目标在左半部分
      right = mid;
    }
  }

  // left - 1 就是右边界
  return left - 1;
}

// 变体3：查找旋转排序数组中的最小值
/**
 * 例如：[4, 5, 6, 7, 0, 1, 2] 旋转自 [0, 1, 2, 4, 5, 6, 7]
 * 最小值是 0
 */
function findMin(nums: number[]): number {
  let left = 0;
  let right = nums.length - 1;

  // 如果数组没有旋转，直接返回第一个元素
  if (nums[left] <= nums[right]) {
    return nums[left];
  }

  while (left < right) {
    const mid = left + Math.floor((right - left) / 2);

    if (nums[mid] > nums[right]) {
      // 最小值在右半部分
      left = mid + 1;
    } else {
      // nums[mid] <= nums[right]，最小值在左半部分或就是mid
      right = mid;
    }
  }

  return nums[left];
}

// 变体4：在旋转排序数组中查找目标值
function searchRotated(nums: number[], target: number): number {
  let left = 0;
  let right = nums.length - 1;

  while (left <= right) {
    const mid = left + Math.floor((right - left) / 2);

    if (nums[mid] === target) {
      return mid;
    }

    // 判断哪半部分是排好序的
    if (nums[left] <= nums[mid]) {
      // 左半部分是排好序的
      if (nums[left] <= target && target < nums[mid]) {
        // 目标在左半部分
        right = mid - 1;
      } else {
        // 目标在右半部分
        left = mid + 1;
      }
    } else {
      // 右半部分是排好序的
      if (nums[mid] < target && target <= nums[right]) {
        // 目标在右半部分
        left = mid + 1;
      } else {
        // 目标在左半部分
        right = mid - 1;
      }
    }
  }

  return -1;
}
```

### 2.3 二分查找的应用场景

| 应用场景 | 问题描述 | 解法要点 |
|----------|----------|----------|
| **普通查找** | 在有序数组中查找目标 | 基础二分 |
| **左边界** | 查找第一个 >= target 的位置 | left < right, right = mid |
| **右边界** | 查找最后一个 <= target 的位置 | left < right, left = mid + 1 |
| **插入位置** | 查找插入目标的位置 | lowerBound |
| **搜索旋转数组** | 在旋转排序数组中查找 | 判断哪半部分有序 |
| **寻找峰值** | 找到峰值元素 | 比较mid与mid+1 |
| **找最小值** | 在旋转数组中找最小值 | nums[mid] > nums[right] |

## 三、归并排序

### 3.1 归并排序原理

**归并排序（Merge Sort）** 是分治法的经典应用，也是稳定的排序算法。

**算法步骤：**
1. **分：** 将数组从中间分成两半
2. **治：** 递归地对两个子数组进行归并排序
3. **合：** 合并两个有序子数组

```typescript
/**
 * 归并排序
 *
 * 时间复杂度：O(n log n)
 * 空间复杂度：O(n)（合并时需要额外空间）
 * 稳定性：稳定
 */

function mergeSort(nums: number[]): number[] {
  const n = nums.length;

  // 基本情况：数组长度小于等于1，不需要排序
  if (n <= 1) {
    return nums;
  }

  // 第一步：分 - 从中间分成两半
  const mid = Math.floor(n / 2);
  const left = nums.slice(0, mid);
  const right = nums.slice(mid);

  // 第二步：治 - 递归排序左右两半
  const sortedLeft = mergeSort(left);
  const sortedRight = mergeSort(right);

  // 第三步：合 - 合并两个有序数组
  return merge(sortedLeft, sortedRight);
}

/**
 * 合并两个有序数组
 * @param left 左侧有序数组
 * @param right 右侧有序数组
 * @returns 合并后的有序数组
 */
function merge(left: number[], right: number[]): number[] {
  const result: number[] = [];
  let i = 0;  // left的指针
  let j = 0;  // right的指针

  // 逐个比较两个数组的元素，按从小到大放入结果数组
  while (i < left.length && j < right.length) {
    if (left[i] <= right[j]) {
      // left[i] 更小或相等，放入result
      // 注意：使用 <= 而不是 <，保证稳定性（相等元素保持相对顺序）
      result.push(left[i]);
      i++;
    } else {
      result.push(right[j]);
      j++;
    }
  }

  // 处理剩余元素（最多只有一个数组会有剩余）
  // 直接将剩余元素加入result
  while (i < left.length) {
    result.push(left[i]);
    i++;
  }

  while (j < right.length) {
    result.push(right[j]);
    j++;
  }

  return result;
}

/**
 * 归并排序（原地版本 - 使用额外空间但更高效）
 */
function mergeSortInPlace(nums: number[]): number[] {
  const n = nums.length;

  function sort(left: number, right: number): void {
    // 基本情况：子数组只有一个或零个元素
    if (left >= right) {
      return;
    }

    // 分：找到中间位置
    const mid = left + Math.floor((right - left) / 2);

    // 治：递归排序左右两半
    sort(left, mid);
    sort(mid + 1, right);

    // 合：合并两个有序子数组
    mergeInPlace(left, mid, right);
  }

  function mergeInPlace(left: number, mid: number, right: number): void {
    // 创建临时数组存储合并结果
    const temp: number[] = [];
    let i = left;
    let j = mid + 1;

    while (i <= mid && j <= right) {
      if (nums[i] <= nums[j]) {
        temp.push(nums[i++]);
      } else {
        temp.push(nums[j++]);
      }
    }

    while (i <= mid) {
      temp.push(nums[i++]);
    }

    while (j <= right) {
      temp.push(nums[j++]);
    }

    // 将合并结果复制回原数组
    for (let k = 0; k < temp.length; k++) {
      nums[left + k] = temp[k];
    }
  }

  sort(0, n - 1);
  return nums;
}
```

```

                [0,7] O(n)
               /      \
         [0,3]         [4,7] O(n)
         /    \         /    \
      [0,1]  [2,3]   [4,5]   [6,7] O(n)
```

|------|------|
| **时间复杂度** | O(n log n)（每层合并O(n)，共log n层） |
| **空间复杂度** | O(n)（合并时需要额外空间） |
| **稳定性** | 稳定（相等元素保持相对顺序） |
| **最佳/最差/平均** | 都是O(n log n) |

## 四、快速排序

### 4.1 快速排序原理

**快速排序（Quick Sort）** 是最常用的排序算法之一，平均性能非常好。

**算法思想：**
1. **选择基准：** 从数组中选择一个元素作为"基准"（pivot）
2. **分区：** 将数组分成两部分，所有小于基准的元素在左边，所有大于基准的元素在右边
3. **递归：** 递归地对左右两部分进行快速排序

**类比理解：** 想象你有一副扑克牌要按花色分组。你先选一张牌作为基准，然后把所有黑桃放到它左边，所有红心放到它右边。接着对左右两堆分别重复这个过程，直到每堆只有一张牌。

```typescript
/**
 * 快速排序
 *
 * 时间复杂度：
 *   - 最佳：O(n log n)
 *   - 平均：O(n log n)
 *   - 最差：O(n²)（数组已有序）
 *
 * 空间复杂度：O(log n)（递归栈）
 * 稳定性：不稳定
 */

function quickSort(nums: number[]): number[] {
  const n = nums.length;

  /**
   * 分区函数
   * @param left 起始索引
   * @param right 结束索引
   * @returns pivot的最终位置
   */
  function partition(left: number, right: number): number {
    // 选择最右边的元素作为基准
    const pivot = nums[right];

    // i 表示小于基准的元素的最后位置
    let i = left - 1;

    // 遍历[left, right-1]范围内的元素
    for (let j = left; j < right; j++) {
      if (nums[j] < pivot) {
        // nums[j]小于基准，移到左边
        i++;
        [nums[i], nums[j]] = [nums[j], nums[i]];
      }
    }

    // 将基准放到正确位置
    // i+1 就是基准的正确位置
    [nums[i + 1], nums[right]] = [nums[right], nums[i + 1]];

    return i + 1;
  }

  /**
   * 快速排序递归函数
   */
  function sort(left: number, right: number): void {
    // 基本情况：子数组有一个或零个元素
    if (left >= right) {
      return;
    }

    // 分区，返回基准位置
    const pivotIndex = partition(left, right);

    // 递归排序基准左右两边
    sort(left, pivotIndex - 1);
    sort(pivotIndex + 1, right);
  }

  sort(0, n - 1);
  return nums;
}

/**
 * 快速排序（随机基准优化）
 * 解决数组已有序时退化为O(n²)的问题
 */
function quickSortRandom(nums: number[]): number[] {
  const n = nums.length;

  function partition(left: number, right: number): number {
    // 随机选择一个元素作为基准，与最后一个元素交换
    const randomIndex = left + Math.floor(Math.random() * (right - left + 1));
    [nums[randomIndex], nums[right]] = [nums[right], nums[randomIndex]];

    const pivot = nums[right];
    let i = left - 1;

    for (let j = left; j < right; j++) {
      if (nums[j] < pivot) {
        i++;
        [nums[i], nums[j]] = [nums[j], nums[i]];
      }
    }

    [nums[i + 1], nums[right]] = [nums[right], nums[i + 1]];
    return i + 1;
  }

  function sort(left: number, right: number): void {
    if (left >= right) {
      return;
    }

    const pivotIndex = partition(left, right);
    sort(left, pivotIndex - 1);
    sort(pivotIndex + 1, right);
  }

  sort(0, n - 1);
  return nums;
}

/**
 * 快速排序（三数取中优化）
 * 选择左中右三个数的中位数作为基准
 */
function quickSortMedian(nums: number[]): number[] {
  const n = nums.length;

  function medianOfThree(a: number, b: number, c: number): number {
    // 返回三个数的中位数
    if ((nums[a] - nums[b]) * (nums[a] - nums[c]) <= 0) {
      return a;
    } else if ((nums[b] - nums[a]) * (nums[b] - nums[c]) <= 0) {
      return b;
    } else {
      return c;
    }
  }

  function partition(left: number, right: number): number {
    const mid = Math.floor((left + right) / 2);

    // 三数取中
    const pivotIndex = medianOfThree(left, mid, right);
    [nums[pivotIndex], nums[right]] = [nums[right], nums[pivotIndex]];

    const pivot = nums[right];
    let i = left - 1;

    for (let j = left; j < right; j++) {
      if (nums[j] < pivot) {
        i++;
        [nums[i], nums[j]] = [nums[j], nums[i]];
      }
    }

    [nums[i + 1], nums[right]] = [nums[right], nums[i + 1]];
    return i + 1;
  }

  function sort(left: number, right: number): void {
    if (left >= right) {
      return;
    }

    const pivotIndex = partition(left, right);
    sort(left, pivotIndex - 1);
    sort(pivotIndex + 1, right);
  }

  sort(0, n - 1);
  return nums;
}
```

### 4.2 快速排序 vs 归并排序

| 特性 | 快速排序 | 归并排序 |
|------|----------|----------|
| **核心操作** | 分区（partition） | 合并（merge） |
| **分区点选择** | 任意元素 | 总是中间位置 |
| **时间复杂度（平均）** | O(n log n) | O(n log n) |
| **时间复杂度（最差）** | O(n²) | O(n log n) |
| **空间复杂度** | O(log n)（栈空间） | O(n)（合并数组） |
| **稳定性** | 不稳定 | 稳定 |
| **适用场景** | 内排序，原地排序 | 外排序，需要稳定排序 |

## 五、汉诺塔问题

### 5.1 问题描述

**汉诺塔（Tower of Hanoi）** 是一个经典的递归问题：

> 有三根柱子A、B、C，A柱子上有n个圆盘，圆盘大小不同，大的在下，小的在上。现在要把所有圆盘从A柱子移动到C柱子，规则是：
> 1. 每次只能移动一个圆盘
> 2. 大圆盘不能放在小圆盘上面
> 3. 可以使用B柱子作为辅助

### 5.2 算法分析

```typescript
/**
 * 汉诺塔问题
 *
 * 算法思想（递归）：
 * 1. 如果只有一个圆盘，直接从源柱子移动到目标柱子
 * 2. 如果有多个圆盘：
 *    a. 先将n-1个圆盘从源柱子移动到辅助柱子（以目标柱子为辅助）
 *    b. 将最大的圆盘从源柱子移动到目标柱子
 *    c. 将n-1个圆盘从辅助柱子移动到目标柱子（以源柱子为辅助）
 *
 * 时间复杂度：O(2^n - 1) ≈ O(2^n)
 * 空间复杂度：O(n)（递归栈）
 */

/**
 * 汉诺塔移动记录
 * @param n 圆盘数量
 * @param source 源柱子（A/B/C）
 * @param target 目标柱子（A/B/C）
 * @param auxiliary 辅助柱子（A/B/C）
 * @returns 移动步骤列表
 */
function hanoi(
  n: number,
  source: string = 'A',
  target: string = 'C',
  auxiliary: string = 'B'
): string[] {
  const moves: string[] = [];

  function move(n: number, from: string, to: string, aux: string): void {
    // 基本情况：只有一个圆盘
    if (n === 1) {
      moves.push(`将第${n}个圆盘从 ${from} 柱子移动到 ${to} 柱子`);
      return;
    }

    // 第一步：将n-1个圆盘从源柱子移动到辅助柱子
    move(n - 1, from, aux, to);

    // 第二步：将最大的圆盘从源柱子移动到目标柱子
    moves.push(`将第${n}个圆盘从 ${from} 柱子移动到 ${to} 柱子`);

    // 第三步：将n-1个圆盘从辅助柱子移动到目标柱子
    move(n - 1, aux, to, from);
  }

  move(n, source, target, auxiliary);
  return moves;
}

/**
 * 汉诺塔（计算移动次数）
 */
function hanoiCount(n: number): number {
  // 移动次数公式：2^n - 1
  return Math.pow(2, n) - 1;
}

/**
 * 汉诺塔（返回移动步骤及编号）
 */
function hanoiWithIndex(n: number): Array<{step: number, from: string, to: string}> {
  const moves: Array<{step: number, from: string, to: string}> = [];
  let step = 0;

  function move(n: number, from: string, to: string, aux: string): void {
    if (n === 1) {
      step++;
      moves.push({step, from, to});
      return;
    }

    move(n - 1, from, aux, to);
    step++;
    moves.push({step, from, to});
    move(n - 1, aux, to, from);
  }

  move(n, 'A', 'C', 'B');
  return moves;
}
```

### 5.3 汉诺塔的递归理解

```
汉诺塔递归分解（以3个圆盘为例）

hanoi(3, 'A', 'C', 'B')
├── hanoi(2, 'A', 'B', 'C')
│   ├── hanoi(1, 'A', 'C', 'B') → A→C
│   ├── A→B (移动第2个盘子)
│   └── hanoi(1, 'C', 'B', 'A') → C→B
├── A→C (移动第3个盘子)
└── hanoi(2, 'B', 'C', 'A')
    ├── hanoi(1, 'B', 'A', 'C') → B→A
    ├── B→C (移动第2个盘子)
    └── hanoi(1, 'A', 'C', 'B') → A→C

移动序列：
1. A→C
2. A→B
3. C→B
4. A→C  ← 第3个盘子（最大的）
5. B→A
6. B→C
7. A→C
```

## 六、欧几里得算法（最大公约数）

### 6.1 算法原理

**欧几里得算法（Euclidean Algorithm）** 用于计算两个正整数的最大公约数（GCD）。

**核心原理：** gcd(a, b) = gcd(b, a mod b)，当b=0时，gcd(a, 0) = a

**证明：**
设 a = qb + r，其中 0 ≤ r < b
则 r = a - qb

设 d = gcd(a, b)，则 d | a 且 d | b
因此 d | (a - qb) = r
所以 d 也是 b 和 r 的公约数

设 d' = gcd(b, r)，则 d' | b 且 d' | r
因此 d' | (qb + r) = a
所以 d' 也是 a 和 b 的公约数

因此 gcd(a, b) = gcd(b, r)

```typescript
/**
 * 欧几里得算法 - 迭代版本
 *
 * 时间复杂度：O(log min(a, b))
 * 空间复杂度：O(1)
 */
function gcd(a: number, b: number): number {
  // 确保 a >= b
  if (a < b) {
    [a, b] = [b, a];
  }

  // 辗转相除
  while (b !== 0) {
    const temp = b;
    b = a % b;
    a = temp;
  }

  return a;
}

/**
 * 欧几里得算法 - 递归版本
 */
function gcdRecursive(a: number, b: number): number {
  if (a < b) {
    [a, b] = [b, a];
  }

  if (b === 0) {
    return a;
  }

  return gcdRecursive(b, a % b);
}

/**
 * 扩展欧几里得算法
 * 计算 gcd(a, b) = ax + by 中的 x 和 y
 */
function extendedGcd(a: number, b: number): {gcd: number, x: number, y: number} {
  if (a < b) {
    [a, b] = [b, a];
  }

  if (b === 0) {
    return {gcd: a, x: 1, y: 0};
  }

  const result = extendedGcd(b, a % b);

  // x1 = y2
  // y1 = x2 - floor(a/b) * y2
  return {
    gcd: result.gcd,
    x: result.y,
    y: result.x - Math.floor(a / b) * result.y
  };
}

/**
 * 最小公倍数（LCM）
 */
function lcm(a: number, b: number): number {
  return Math.abs(a * b) / gcd(a, b);
}
```

## 七、主定理（Master Theorem）

### 7.1 主定理定义

主定理用于分析分治算法的时间复杂度。

```
T(n) = aT(n/b) + f(n)

其中：
- n：问题规模
- a：子问题数量（a ≥ 1）
- b：子问题规模缩减因子（b > 1）
- f(n)：分解和合并问题的成本
```

### 7.2 主定理的三种情况

| 情况 | 条件 | 结果 | 解释 |
|------|------|------|------|
| **情况1** | f(n) = O(n^(log_b(a) - ε)) | T(n) = Θ(n^(log_b(a))) | 合并成本小于子问题成本 |
| **情况2** | f(n) = Θ(n^(log_b(a)) × log^k(n)) | T(n) = Θ(n^(log_b(a)) × log^(k+1)(n)) | 合并成本与子问题成本相当 |
| **情况3** | f(n) = Ω(n^(log_b(a) + ε)) | T(n) = Θ(f(n)) | 合并成本大于子问题成本 |

### 7.3 主定理应用示例

```typescript
/**
 * 主定理应用示例
 */

// 示例1：归并排序
// T(n) = 2T(n/2) + O(n)
// a = 2, b = 2, f(n) = O(n)
// n^(log_b(a)) = n^(log_2(2)) = n^1 = n
// f(n) = Θ(n)，属于情况2（k=0）
// 结果：T(n) = Θ(n log n)

// 示例2：二分查找
// T(n) = T(n/2) + O(1)
// a = 1, b = 2, f(n) = O(1)
// n^(log_b(a)) = n^(log_2(1)) = n^0 = 1
// f(n) = Θ(1) = Θ(n^0)，属于情况2（k=0）
// 结果：T(n) = Θ(log n)

// 示例3：原始快速排序（所有元素都划分到一边）
// T(n) = T(n-1) + O(n)
// 这不满足主定理形式，需要用其他方法分析
// 结果：T(n) = Θ(n²)

// 示例4：大整数乘法（Karatsuba）
// T(n) = 3T(n/2) + O(n)
// a = 3, b = 2, f(n) = O(n)
// n^(log_b(a)) = n^(log_2(3)) ≈ n^1.585
// f(n) = O(n) = O(n^1)
// 由于 n^1.585 > n，所以属于情况1
// 结果：T(n) = Θ(n^1.585)
```

## 八、其他分治算法

### 8.1 最近点对问题

```typescript
/**
 * 最近点对问题
 * 在平面中找到距离最近的两个点
 *
 * 时间复杂度：O(n log n)
 */

// 定义点类型
interface Point {
  x: number;
  y: number;
}

function closestPair(points: Point[]): {p1: Point, p2: Point, distance: number} {
  const n = points.length;

  // 按x坐标排序
  points.sort((a, b) => a.x - b.x);

  function distance(p1: Point, p2: Point): number {
    return Math.sqrt((p1.x - p2.x) ** 2 + (p1.y - p2.y) ** 2);
  }

  // 暴力法计算最近距离（用于基本情况）
  function bruteForce(left: number, right: number): number {
    let minDist = Infinity;
    for (let i = left; i <= right; i++) {
      for (let j = i + 1; j <= right; j++) {
        minDist = Math.min(minDist, distance(points[i], points[j]));
      }
    }
    return minDist;
  }

  function closestUtil(left: number, right: number): number {
    // 基本情况：点数少于等于3时使用暴力法
    if (right - left <= 3) {
      return bruteForce(left, right);
    }

    const mid = Math.floor((left + right) / 2);
    const midX = points[mid].x;

    // 递归计算左右两半的最近距离
    const dl = closestUtil(left, mid);
    const dr = closestUtil(mid + 1, right);
    const d = Math.min(dl, dr);

    // 创建一个数组存储可能形成更近距离的点（位于分割线d范围内）
    const temp: Point[] = [];
    for (let i = left; i <= right; i++) {
      if (Math.abs(points[i].x - midX) < d) {
        temp.push(points[i]);
      }
    }

    // 按y坐标排序
    temp.sort((a, b) => a.y - b.y);

    // 检查temp中相邻点的距离
    for (let i = 0; i < temp.length; i++) {
      for (let j = i + 1; j < temp.length && (temp[j].y - temp[i].y) < d; j++) {
        const dist = distance(temp[i], temp[j]);
        if (dist < d) {
          d = dist;
        }
      }
    }

    return d;
  }

  const minDist = closestUtil(0, n - 1);

  // 找到具体是哪两个点
  let p1 = points[0], p2 = points[1];
  let minD = distance(p1, p2);

  for (let i = 0; i < n; i++) {
    for (let j = i + 1; j < n; j++) {
      const d = distance(points[i], points[j]);
      if (d < minD) {
        minD = d;
        p1 = points[i];
        p2 = points[j];
      }
    }
  }

  return {p1, p2, distance: minD};
}
```

### 8.2 大整数乘法（Karatsuba）

```typescript
/**
 * Karatsuba大整数乘法
 *
 * 传统乘法：O(n²)
 * Karatsuba：O(n^log_2(3)) ≈ O(n^1.585)
 *
 * 算法思想：
 * 将两个n位整数a和b分解：
 * a = a1 * 10^(n/2) + a0
 * b = b1 * 10^(n/2) + b0
 *
 * a * b = a1b1 * 10^n + (a1b0 + a0b1) * 10^(n/2) + a0b0
 *
 * Karatsuba优化：
 * 只需计算3个乘法（而不是4个）：
 * - a1b1
 * - a0b0
 * - (a1+a0)(b1+b0) - a1b1 - a0b0
 */

function multiply(a: string, b: string): string {
  // 去除前导零和符号
  const isNegative = (a[0] === '-' && b[0] !== '-') || (a[0] !== '-' && b[0] === '-');
  a = a.replace(/^-/, '');
  b = b.replace(/^-/, '');

  // 补齐长度一致
  const n = Math.max(a.length, b.length);
  a = a.padStart(n, '0');
  b = b.padStart(n, '0');

  function karatsuba(x: string, y: string): string {
    const len = x.length;

    // 基本情况：个位数相乘
    if (len <= 1) {
      const result = parseInt(x) * parseInt(y);
      return result.toString();
    }

    const half = Math.floor(len / 2);

    // 分解
    const a1 = x.substring(0, len - half);
    const a0 = x.substring(len - half);
    const b1 = y.substring(0, len - half);
    const b0 = y.substring(len - half);

    // 递归计算
    const z2 = karatsuba(a1, b1);  // a1b1
    const z0 = karatsuba(a0, b0);  // a0b0
    const z1 = karatsuba(
      (parseInt(a1) + parseInt(a0)).toString(),
      (parseInt(b1) + parseInt(b0)).toString()
    );

    // z1 = z1 - z2 - z0
    const z1Calc = (
      parseInt(z1) -
      parseInt(z2) -
      parseInt(z0)
    ).toString();

    // 合并结果
    // result = z2 * 10^(2*half) + z1 * 10^half + z0
    const result =
      parseInt(z2) * Math.pow(10, 2 * half) +
      parseInt(z1Calc) * Math.pow(10, half) +
      parseInt(z0);

    return result.toString();
  }

  const result = karatsuba(a, b);

  // 去除前导零
  const finalResult = result.replace(/^0+/, '') || '0';

  return isNegative ? '-' + finalResult : finalResult;
}
```

## 十、总结

### 10.1 分治法核心要点

```
分治法三步走：分 → 治 → 合

适用条件：
1. 可分解：问题可分成相似的子问题
2. 独立：子问题之间相互独立
3. 可合并：子问题的解可以合并
4. 有基例：存在可直接求解的基本情况

主定理（时间复杂度）：
T(n) = aT(n/b) + f(n)
```

### 10.2 常见分治算法复杂度汇总

| 算法 | 时间复杂度 | 空间复杂度 | 稳定性 |
|------|------------|------------|--------|
| 二分查找 | O(log n) | O(1) | 稳定 |
| 归并排序 | O(n log n) | O(n) | 稳定 |
| 快速排序 | O(n log n) | O(log n) | 不稳定 |
| 汉诺塔 | O(2^n) | O(n) | N/A |
| Karatsuba乘法 | O(n^1.585) | O(n) | N/A |
