## 1. 操作系统基础

### 1.1 进程与程序的区别

#### 什么是程序

**程序（Program）** 是存储在磁盘上的可执行文件，是静态的代码和数据集合。它由一系列指令和数据组成，以文件形式存在于文件系统中。程序本身不占用系统资源（CPU 时间、内存），只是等待被执行。

```
// 程序示例：保存在磁盘上的 hello.js 文件
// 文件路径：/usr/local/bin/hello.js
// 文件大小：156 字节
// 文件权限：rwxr-xr-x

console.log('Hello, World!');
```

上述文件就是程序，它是静态的。当我们不运行它时，它只是磁盘上的一串字节。

#### 什么是进程

**进程（Process）** 是程序的一次执行实例，是操作系统分配资源的基本单位。当程序被加载到内存并开始执行时，操作系统创建一个进程来管理其执行。进程拥有独立的地址空间、文件描述符、寄存器状态等。

```
进程 = 程序 + 执行状态 + 系统资源

// 进程的组成部分
进程 {
    pid: 12345,                    // 进程ID，唯一标识
    state: 'running',             // 进程状态
    memory: {                      // 进程的内存空间
        text: 0x00400000,          // 代码区
        data: 0x00600000,          // 数据区
        heap: 0x00610000,          // 堆区
        stack: 0x7fff0000          // 栈区
    },
    files: [                       // 打开的文件描述符
        stdin: 0,                  // 标准输入
        stdout: 1,                 // 标准输出
        stderr: 2                  // 标准错误
    ],
    cpu_state: {                   // CPU 寄存器状态
        pc: 0x00400000,            // 程序计数器
        sp: 0x7fff0000,            // 栈指针
        registers: [...]           // 通用寄存器
    },
    parent: 12340,                // 父进程ID
    children: []                   // 子进程列表
}
```

#### 核心区别对比

| 维度 | 程序 | 进程 |
|------|------|------|
| **存在状态** | 静态，存储在磁盘上 | 动态，存在于内存中 |
| **生命周期** | 从创建到删除，跨越多个运行周期 | 一次执行，从创建到终止 |
| **资源占用** | 不占用系统资源 | 占用 CPU、内存、I/O 等资源 |
| **唯一标识** | 没有唯一标识 | PID（进程标识符） |
| **并发性** | 可以同时运行多个实例 | 同一程序的多个进程可以并发执行 |
| **地址空间** | 无独立地址空间 | 有独立的虚拟地址空间 |
| **系统调用** | 不能直接执行系统调用 | 可以执行系统调用 |

#### 进程的创建过程

当你在终端执行 `node server.js` 时，操作系统经历了以下步骤创建进程：

```
1. 文件系统读取
   ├── 打开 /usr/local/bin/node 可执行文件
   ├── 检查文件格式（ELF/Mach-O/PE）
   └── 读取文件头部信息

2. 创建进程控制块（PCB）
   ├── 分配唯一的 PID
   ├── 初始化进程状态为 NEW
   └── 创建进程表项

3. 加载程序
   ├── 将代码段加载到内存（text segment）
   ├── 初始化数据段（data segment）
   ├── 设置 BSS 段为零
   └── 分配初始堆内存

4. 设置执行环境
   ├── 设置程序计数器（PC）指向入口点
   ├── 初始化栈（argc, argv, envp）
   ├── 初始化文件描述符（继承或重定向）
   └── 设置信号处理

5. 进程就绪
   ├── 状态设为 READY
   └── 加入就绪队列，等待调度器选中
```

### 1.2 进程状态

#### 五状态进程模型

操作系统中的进程并非只有"运行"和"停止"两种状态，而是一个复杂的状态机。以下是经典的五状态模型：

```
                    ┌─────────────┐
                    │   创建态     │
                    │   (New)     │
                    └──────┬──────┘
                           │
                           │ fork/exec
                           ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   终止态     │     │   就绪态     │     │   运行态     │
│ (Terminated)│     │   (Ready)   │◄───►│  (Running)  │
└─────────────┘     └──────┬──────┘     └──────┬──────┘
                           │                   │
                           │ 调度选中          │ 时间片用完/阻塞
                           │                   ▼
                           │            ┌─────────────┐
                           │            │   阻塞态     │
                           │            │ (Blocked)   │
                           └───────────►└─────────────┘
                                         等待I/O/信号
```

#### 各状态详解

**1. 创建态（New）**

进程正在被创建，操作系统完成创建进程的所有准备工作。此阶段完成的工作包括：

```c
// 操作系统内核中进程创建的简化伪代码
process = allocate_process_table_entry();  // 分配PCB
process.pid = next_pid++;                  // 分配唯一PID
process.state = NEW;                        // 设置为创建态
process.parent = current_process;           // 记录父进程

// 复制或继承资源
process.files = duplicate_file_table(parent);  // 复制文件描述符
process.memory = allocate_page_tables();       // 分配页表
process.threads = create_initial_thread();     // 创建初始线程

// 加载程序
load_program_into_memory(process, program_file);

// 设置初始状态
process.state = READY;                     // 转入就绪态
enqueue_to_ready_queue(process);           // 加入就绪队列
```

**2. 就绪态（Ready）**

进程已做好运行准备，等待操作系统调度器选中它。所有处于就绪态的进程组成就绪队列，等待 CPU 时间片。

```python
# 就绪队列示意图
ready_queue = [
    Process(pid=101, priority=5, name="node"),
    Process(pid=102, priority=3, name="nginx"),
    Process(pid=103, priority=5, name="python"),
    Process(pid=104, priority=1, name="bash"),
    # ...
]
```

就绪态的特点：

- 进程已分配了所有必要的资源（内存、I/O设备等）
- 进程只等待 CPU 时间片
- 调度器根据算法从就绪队列中选择进程运行

**3. 运行态（Running）**

进程正在 CPU 上执行。处于运行态的进程拥有 CPU 的控制权，可以执行指令。

```c
// 运行态进程的结构示意
running_process = {
    pid: 1043,
    state: RUNNING,
    cpu_time_used: 45,        // 已使用CPU时间（毫秒）
    time_slice_remaining: 5, // 剩余时间片（毫秒）
    pcb: 0xffff9000,         // 进程控制块地址
    kernel_stack: 0xffff8000 // 内核栈地址
}
```

**4. 阻塞态（Blocked / Waiting）**

进程因等待某些事件（如 I/O 完成、信号量、资源）而暂停执行。进程进入阻塞态后，调度器会选择其他就绪态进程运行。

```
阻塞场景示例：

场景1：等待I/O完成
process.state = BLOCKED
process.wait_event = "disk_read_completion"
process.wait_queue = disk_io_queue

场景2：等待文件锁
process.state = BLOCKED
process.wait_event = "file_lock_release"
process.wait_queue = file_lock_queue

场景3：等待子进程
process.state = BLOCKED
process.wait_event = "child_exit"
process.wait_queue = wait_children_queue
```

**5. 终止态（Terminated）**

进程执行完毕或被异常终止，操作系统回收其资源。

```c
// 进程终止的处理流程
void terminate_process(Process *p) {
    // 1. 关闭所有打开的文件
    for (fd = 0; fd < process.max_fd; fd++) {
        close(fd);
    }

    // 2. 释放内存空间
    free_page_tables(p->mm);
    free_physical_pages(p->memory);

    // 3. 通知父进程（通过wait系统调用）
    if (p->parent->wait_pid == p->pid) {
        p->parent->exit_status = p->exit_code;
        wake_up(p->parent);  // 唤醒父进程
    }

    // 4. 将进程状态设为终止
    p->state = TERMINATED;

    // 5. 保留PCB信息（用于父进程获取退出状态）
    p->zombie = true;

    // 6. 最终由父进程释放PCB
}
```

#### 进程状态转换的代码示例

以下是一个模拟进程状态转换的 Python 示例，帮助理解状态机：

```python
#!/usr/bin/env python3
"""
进程状态转换模拟器
本程序演示操作系统中进程的五种状态及其转换关系
"""

from enum import Enum
from dataclasses import dataclass, field
from typing import Optional, List
import time
import random


class ProcessState(Enum):
    """进程状态的枚举类型，对应操作系统的进程状态"""
    NEW = "新建"           # 进程正在被创建
    READY = "就绪"         # 进程已准备好，等待CPU
    RUNNING = "运行"       # 进程正在CPU上执行
    BLOCKED = "阻塞"       # 进程因等待事件而阻塞
    TERMINATED = "终止"    # 进程执行完毕


@dataclass
class Process:
    """
    进程控制块（PCB）的模拟类
    实际操作系统中，这对应内核中的进程控制块结构体
    """
    pid: int                           # 进程ID，唯一标识
    name: str                          # 进程名称
    state: ProcessState = ProcessState.NEW  # 当前状态
    priority: int = 5                  # 进程优先级（1-10）
    cpu_burst: int = 0                 # CPU执行时间需求（毫秒）
    io_burst: int = 0                  # I/O阻塞时间（毫秒）
    arrival_time: int = 0              # 到达时间
    exit_status: Optional[int] = None  # 退出状态

    # 统计信息
    start_time: Optional[int] = None  # 开始运行时间
    end_time: Optional[int] = None    # 结束时间
    waiting_time: int = 0              # 累计等待时间
    turnaround_time: int = 0            # 周转时间（结束-到达）

    # 用于状态追踪
    state_history: List[tuple] = field(default_factory=list)

    def record_state(self, time_ms: int, new_state: ProcessState):
        """记录状态转换历史"""
        self.state_history.append((time_ms, self.state, new_state))
        self.state = new_state

    def __repr__(self):
        return f"Process(pid={self.pid}, name={self.name}, state={self.state.value})"


class ProcessStateSimulator:
    """
    进程状态转换模拟器
    演示操作系统如何管理进程状态转换
    """

    def __init__(self):
        self.current_time = 0          # 模拟时间（毫秒）
        self.ready_queue: List[Process] = []  # 就绪队列
        self.blocked_queue: List[Process] = []  # 阻塞队列
        self.running_process: Optional[Process] = None  # 当前运行进程
        self.terminated_processes: List[Process] = []  # 已终止进程
        self.next_pid = 1              # 下一个可用PID

    def create_process(self, name: str, cpu_burst: int, io_burst: int = 0, priority: int = 5) -> Process:
        """
        创建新进程，对应操作系统的进程创建操作
        1. 分配PID
        2. 初始化PCB
        3. 设置初始状态为NEW
        4. 加载程序（这里省略）
        5. 状态转为READY，加入就绪队列
        """
        # 创建进程对象（模拟PCB分配）
        process = Process(
            pid=self.next_pid,
            name=name,
            cpu_burst=cpu_burst,
            io_burst=io_burst,
            priority=priority,
            arrival_time=self.current_time
        )
        self.next_pid += 1

        # 模拟从NEW状态转到READY状态
        process.record_state(self.current_time, ProcessState.READY)
        self.ready_queue.append(process)

        print(f"[时刻 {self.current_time}ms] 创建进程: {process.name} (PID={process.pid})")
        print(f"           状态转换: NEW -> READY")
        print(f"           加入就绪队列（优先级={priority}）")

        return process

    def schedule(self):
        """
        调度器函数，选择下一个运行的进程
        这里使用简单的优先级调度
        """
        if not self.ready_queue or self.running_process:
            return

        # 按优先级排序（优先级数值越小优先级越高）
        self.ready_queue.sort(key=lambda p: (p.priority, p.arrival_time))

        # 选出最高优先级的进程
        process = self.ready_queue.pop(0)

        # 状态转换：READY -> RUNNING
        process.record_state(self.current_time, ProcessState.RUNNING)
        process.start_time = self.current_time
        self.running_process = process

        print(f"\n[时刻 {self.current_time}ms] 调度器选中进程: {process.name} (PID={process.pid})")
        print(f"           状态转换: READY -> RUNNING")

    def simulate_io_event(self, process: Process):
        """
        模拟I/O事件：RUNNING -> BLOCKED
        当进程需要等待I/O操作时，会进入阻塞状态
        """
        process.record_state(self.current_time, ProcessState.BLOCKED)
        self.running_process = None
        self.blocked_queue.append(process)

        print(f"[时刻 {self.current_time}ms] 进程 {process.name} 发起I/O请求")
        print(f"           状态转换: RUNNING -> BLOCKED")
        print(f"           进入阻塞队列，等待I/O完成")

    def io_complete(self, process: Process):
        """
        模拟I/O完成事件：BLOCKED -> READY
        当I/O操作完成时，进程重新进入就绪队列
        """
        process.record_state(self.current_time, ProcessState.READY)
        self.blocked_queue.remove(process)
        self.ready_queue.append(process)

        print(f"[时刻 {self.current_time}ms] I/O完成，进程 {process.name} 唤醒")
        print(f"           状态转换: BLOCKED -> READY")
        print(f"           加入就绪队列")

    def time_slice_expire(self, process: Process):
        """
        时间片用完：RUNNING -> READY
        适用于时间片轮转调度算法
        """
        process.record_state(self.current_time, ProcessState.READY)
        self.running_process = None
        self.ready_queue.append(process)

        print(f"[时刻 {self.current_time}ms] 时间片用完: {process.name}")
        print(f"           状态转换: RUNNING -> READY")
        print(f"           加入就绪队列尾部")

    def terminate_process(self, process: Process, exit_code: int = 0):
        """
        终止进程：RUNNING -> TERMINATED
        """
        process.record_state(self.current_time, ProcessState.TERMINATED)
        process.exit_status = exit_code
        process.end_time = self.current_time
        process.turnaround_time = process.end_time - process.arrival_time
        process.waiting_time = process.turnaround_time - process.cpu_burst - process.io_burst

        if self.running_process == process:
            self.running_process = None

        self.terminated_processes.append(process)

        print(f"[时刻 {self.current_time}ms] 进程 {process.name} 执行完毕")
        print(f"           状态转换: RUNNING -> TERMINATED")
        print(f"           退出码: {exit_code}")
        print(f"           周转时间: {process.turnaround_time}ms, 等待时间: {process.waiting_time}ms")

    def run_time_slice(self, duration: int):
        """
        运行一段时间片
        """
        if not self.running_process:
            return

        process = self.running_process
        process.cpu_burst -= duration

        print(f"[时刻 {self.current_time}ms] 进程 {process.name} 运行中...")
        print(f"           剩余CPU需求: {max(0, process.cpu_burst)}ms")

        self.current_time += duration

    def print_summary(self):
        """打印调度结果汇总"""
        print("\n" + "=" * 70)
        print("调度结果汇总")
        print("=" * 70)

        for p in self.terminated_processes:
            print(f"PID={p.pid:3d} | {p.name:10s} | "
                  f"周转时间={p.turnaround_time:4d}ms | "
                  f"等待时间={p.waiting_time:4d}ms | "
                  f"优先级={p.priority}")

        avg_turnaround = sum(p.turnaround_time for p in self.terminated_processes) / len(self.terminated_processes)
        avg_waiting = sum(p.waiting_time for p in self.terminated_processes) / len(self.terminated_processes)

        print("-" * 70)
        print(f"平均周转时间: {avg_turnaround:.2f}ms")
        print(f"平均等待时间: {avg_waiting:.2f}ms")
        print("=" * 70)


def demo_process_states():
    """
    演示进程状态转换的完整流程
    """
    print("\n" + "=" * 70)
    print("进程状态转换模拟演示")
    print("=" * 70)

    simulator = ProcessStateSimulator()

    # 时刻0：创建三个进程
    p1 = simulator.create_process("WebServer", cpu_burst=100, io_burst=50, priority=5)
    p2 = simulator.create_process("Database", cpu_burst=80, io_burst=30, priority=3)
    p3 = simulator.create_process("Logger", cpu_burst=40, io_burst=20, priority=7)

    simulator.current_time += 10

    # 时刻10：调度器选择进程运行
    simulator.schedule()

    # 运行一段时间片
    simulator.run_time_slice(20)

    # 模拟I/O事件
    simulator.simulate_io_event(simulator.running_process)

    # 模拟I/O完成
    simulator.current_time += 30  # 模拟I/O等待时间
    simulator.io_complete(p1)

    # 继续调度
    simulator.schedule()
    simulator.run_time_slice(30)
    simulator.terminate_process(simulator.running_process)

    simulator.current_time += 10
    simulator.schedule()
    simulator.run_time_slice(40)
    simulator.terminate_process(simulator.running_process)

    # 打印汇总
    simulator.print_summary()


if __name__ == "__main__":
    demo_process_states()
```

```
[时刻 10ms] 进程 Database 运行中...
           剩余CPU需求: 60ms
[时刻 30ms] 进程 Database 发起I/O请求
           状态转换: RUNNING -> BLOCKED
           进入阻塞队列，等待I/O完成
[时刻 60ms] I/O完成，进程 WebServer 唤醒
           状态转换: BLOCKED -> READY
           加入就绪队列
```

### 1.3 进程调度算法

#### 调度算法的评价指标

```
┌─────────────────────────────────────────────────────────────────┐
│                     调度算法评价指标                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. CPU利用率 (CPU Utilization)                                  │
│     - CPU忙碌时间占总时间的比例                                    │
│     - 目标：最大化 CPU 利用率                                     │
│                                                                  │
│  2. 吞吐量 (Throughput)                                          │
│     - 单位时间内完成的进程数量                                    │
│     - 目标：最大化吞吐量                                          │
│                                                                  │
│  3. 周转时间 (Turnaround Time)                                   │
│     - 进程从提交到完成的总时间                                     │
│     - T = 结束时间 - 到达时间                                     │
│     - 目标：最小化平均周转时间                                     │
│                                                                  │
│  4. 等待时间 (Waiting Time)                                      │
│     - 进程在就绪队列中等待CPU的总时间                              │
│     - W = 周转时间 - 运行时间 - I/O时间                           │
│     - 目标：最小化平均等待时间                                     │
│                                                                  │
│  5. 响应时间 (Response Time)                                      │
│     - 从提交请求到首次响应的时间                                   │
│     - 目标：最小化响应时间                                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### 1. FCFS（先来先服务）

**原理**：按进程到达顺序服务，最先到达的进程最先获得 CPU。这是**非抢占式**算法。

```python
def fcfs_scheduling(processes: list) -> dict:
    """
    FCFS 调度算法实现
    First Come First Serve - 按到达顺序服务

    参数：进程列表，每个进程包含 pid, arrival_time, burst_time
    返回：调度结果统计
    """
    # 按到达时间排序
    sorted_processes = sorted(processes, key=lambda p: p['arrival_time'])

    current_time = 0
    completed = []
    wait_time_total = 0
    turnaround_time_total = 0

    print("=" * 60)
    print("FCFS 调度 - 执行顺序")
    print("=" * 60)

    for i, proc in enumerate(sorted_processes):
        pid = proc['pid']
        arrival = proc['arrival_time']
        burst = proc['burst_time']

        # 如果当前时间小于进程到达时间，CPU空闲等待
        if current_time < arrival:
            idle_time = arrival - current_time
            print(f"[时刻 {current_time}ms] CPU空闲，等待 {idle_time}ms")
            current_time = arrival

        # 记录等待时间
        wait_time = current_time - arrival
        wait_time_total += wait_time

        # 执行进程
        print(f"[时刻 {current_time}ms] 开始执行 P{pid} (需要 {burst}ms CPU时间)")
        current_time += burst

        # 计算周转时间
        turnaround = current_time - arrival
        turnaround_time_total += turnaround

        completed.append({
            'pid': pid,
            'completion_time': current_time,
            'turnaround_time': turnaround,
            'wait_time': wait_time,
            'burst_time': burst
        })

        print(f"[时刻 {current_time}ms] P{pid} 执行完毕，周转时间={turnaround}ms")

    # 打印统计结果
    n = len(processes)
    avg_wait = wait_time_total / n
    avg_turnaround = turnaround_time_total / n

    print("\n" + "=" * 60)
    print("FCFS 调度 - 统计结果")
    print("=" * 60)
    print(f"{'PID':<8}{'到达时间':<12}{'执行时间':<12}{'等待时间':<12}{'周转时间':<12}")
    print("-" * 60)

    for p in completed:
        print(f"P{p['pid']:<6}{p['burst_time']+p['wait_time']:<12}"
              f"{p['burst_time']:<12}{p['wait_time']:<12}{p['turnaround_time']:<12}")

    print("-" * 60)
    print(f"平均等待时间: {avg_wait:.2f}ms")
    print(f"平均周转时间: {avg_turnaround:.2f}ms")

    return {
        'avg_wait_time': avg_wait,
        'avg_turnaround_time': avg_turnaround,
        'schedule': completed
    }
```

#### 2. SJF（最短作业优先）

**原理**：选择下一个执行时间最短的进程。SJF 可以最小化平均等待时间，但需要已知每个进程的执行时间。

```python
def sjf_scheduling(processes: list) -> dict:
    """
    SJF 调度算法实现（非抢占式）
    Shortest Job First - 最短作业优先

    核心思想：优先执行短作业，减少在就绪队列中的等待时间
    优点：平均等待时间最小
    缺点：可能导致长作业饥饿（Starvation）
    """
    # 复制进程列表，避免修改原数据
    processes = [{'pid': p['pid'], 'arrival_time': p['arrival_time'],
                   'burst_time': p['burst_time']} for p in processes]

    current_time = 0
    completed = []
    ready_queue = []
    wait_time_total = 0
    turnaround_time_total = 0

    print("=" * 60)
    print("SJF 调度 - 执行顺序")
    print("=" * 60)

    while completed_len < len(processes) or ready_queue:
        # 将所有已到达的进程加入就绪队列
        while processes and processes[0]['arrival_time'] <= current_time:
            proc = processes.pop(0)
            ready_queue.append(proc)
            print(f"[时刻 {current_time}ms] P{proc['pid']} 到达，加入就绪队列")

        if not ready_queue:
            # 没有就绪进程，跳转到下一个进程到达时间
            if processes:
                next_arrival = processes[0]['arrival_time']
                print(f"[时刻 {current_time}ms] CPU空闲，跳转至 {next_arrival}ms")
                current_time = next_arrival
                continue

        # 按短作业优先排序（SJF 核心）
        ready_queue.sort(key=lambda p: p['burst_time'])

        # 取出最短作业
        proc = ready_queue.pop(0)
        pid = proc['pid']
        burst = proc['burst_time']
        arrival = proc['arrival_time']

        # 计算等待时间
        wait_time = current_time - arrival
        wait_time_total += wait_time

        # 执行进程
        print(f"[时刻 {current_time}ms] 开始执行 P{pid} (需要 {burst}ms)")
        current_time += burst

        # 计算周转时间
        turnaround = current_time - arrival
        turnaround_time_total += turnaround

        completed.append({
            'pid': pid,
            'completion_time': current_time,
            'turnaround_time': turnaround,
            'wait_time': wait_time,
            'burst_time': burst
        })

        print(f"[时刻 {current_time}ms] P{pid} 执行完毕，周转时间={turnaround}ms")

    # 打印统计结果
    n = len(processes)
    avg_wait = wait_time_total / n
    avg_turnaround = turnaround_time_total / n

    print("\n" + "=" * 60)
    print("SJF 调度 - 统计结果")
    print("=" * 60)
    print(f"{'PID':<8}{'到达时间':<12}{'执行时间':<12}{'等待时间':<12}{'周转时间':<12}")
    print("-" * 60)

    for p in completed:
        print(f"P{p['pid']:<6}{p['wait_time']+p['burst_time']:<12}"
              f"{p['burst_time']:<12}{p['wait_time']:<12}{p['turnaround_time']:<12}")

    print("-" * 60)
    print(f"平均等待时间: {avg_wait:.2f}ms")
    print(f"平均周转时间: {avg_turnaround:.2f}ms")

    return {
        'avg_wait_time': avg_wait,
        'avg_turnaround_time': avg_turnaround
    }
```

#### 3. RR（时间片轮转）

**原理**：为每个进程分配固定大小的时间片，进程使用完时间片后被放回就绪队列尾部。

```python
def round_robin_scheduling(processes: list, time_quantum: int) -> dict:
    """
    RR 时间片轮转调度算法实现
    Round Robin - 经典的时间片轮转调度

    参数：
        processes: 进程列表
        time_quantum: 时间片大小（毫秒）

    特点：
        - 抢占式调度
        - 每个进程都有机会执行
        - 响应时间可预测
        - 适用于交互式系统
    """
    # 复制进程列表
    processes = [{'pid': p['pid'], 'arrival_time': p['arrival_time'],
                   'burst_time': p['burst_time']} for p in processes]

    # 按到达时间排序
    processes.sort(key=lambda p: p['arrival_time'])

    current_time = 0
    ready_queue = []  # 就绪队列 (pid, 剩余执行时间, 到达时间)
    completed = []
    timeline = []  # 时间线记录

    print("=" * 70)
    print(f"RR 时间片轮转调度 (时间片 = {time_quantum}ms)")
    print("=" * 70)

    idx = 0  # 下一个要检查的进程

    while ready_queue or idx < len(processes):
        # 将所有已到达的进程加入就绪队列
        while idx < len(processes) and processes[idx]['arrival_time'] <= current_time:
            proc = processes[idx]
            ready_queue.append({
                'pid': proc['pid'],
                'remaining_time': proc['burst_time'],
                'arrival_time': proc['arrival_time']
            })
            print(f"[时刻 {current_time}ms] P{proc['pid']} 到达，加入就绪队列")
            idx += 1

        if not ready_queue:
            # CPU 空闲，跳转到下一个进程到达
            if idx < len(processes):
                next_arrival = processes[idx]['arrival_time']
                print(f"[时刻 {current_time}ms] CPU空闲，等待至 {next_arrival}ms")
                current_time = next_arrival
                continue

        # 从就绪队列取出队首进程
        current_proc = ready_queue.pop(0)

        # 确定本次执行的时间
        exec_time = min(time_quantum, current_proc['remaining_time'])

        print(f"[时刻 {current_time}ms] 开始执行 P{current_proc['pid']} "
              f"(执行 {exec_time}ms, 剩余 {current_proc['remaining_time'] - exec_time}ms)")

        # 更新时间
        current_time += exec_time
        current_proc['remaining_time'] -= exec_time

        # 记录时间线
        timeline.append({
            'time': current_time,
            'pid': current_proc['pid'],
            'executed': exec_time
        })

        # 检查是否有新进程在此期间到达
        while idx < len(processes) and processes[idx]['arrival_time'] <= current_time:
            proc = processes[idx]
            ready_queue.append({
                'pid': proc['pid'],
                'remaining_time': proc['burst_time'],
                'arrival_time': proc['arrival_time']
            })
            print(f"[时刻 {current_time}ms] P{proc['pid']} 到达，加入就绪队列")
            idx += 1

        # 检查进程是否完成
        if current_proc['remaining_time'] > 0:
            # 进程未完成，放回就绪队列
            ready_queue.append(current_proc)
        else:
            # 进程完成
            turnaround = current_time - current_proc['arrival_time']
            wait_time = turnaround - (processes[idx-1]['burst_time'] if idx > 0 else 0)

            completed.append({
                'pid': current_proc['pid'],
                'completion_time': current_time,
                'turnaround_time': turnaround,
                'wait_time': wait_time,
                'burst_time': processes[idx-1]['burst_time']
            })
            print(f"[时刻 {current_time}ms] P{current_proc['pid']} 执行完毕，"
                  f"周转时间={turnaround}ms, 等待时间={wait_time}ms")

    # 统计结果
    n = len(completed)
    avg_wait = sum(p['wait_time'] for p in completed) / n
    avg_turnaround = sum(p['turnaround_time'] for p in completed) / n

    print("\n" + "=" * 70)
    print("RR 调度 - 统计结果")
    print("=" * 70)
    print(f"{'PID':<8}{'执行时间':<12}{'等待时间':<12}{'周转时间':<12}{'完成时间':<12}")
    print("-" * 70)

    for p in completed:
        print(f"P{p['pid']:<6}{p['burst_time']:<12}{p['wait_time']:<12}"
              f"{p['turnaround_time']:<12}{p['completion_time']:<12}")

    print("-" * 70)
    print(f"平均等待时间: {avg_wait:.2f}ms")
    print(f"平均周转时间: {avg_turnaround:.2f}ms")

    return {
        'avg_wait_time': avg_wait,
        'avg_turnaround_time': avg_turnaround,
        'timeline': timeline
    }
```

#### 调度算法对比总结

| 算法 | 类型 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **FCFS** | 非抢占 | 简单、公平 | 对短作业不公，平均等待时间长 | 批处理系统 |
| **SJF** | 非抢占/抢占 | 平均等待时间最小 | 需要预估执行时间，可能饥饿 | 批处理系统 |
| **RR** | 抢占 | 响应时间公平、可预测 | 时间片选择困难 | 交互式系统 |

#### 进程与线程的层次关系

```c
    struct files_struct *files;     // 文件描述符表

    struct signal_struct *signal;    // 信号处理函数表

    struct task_struct *task;        // 内核态 PCB
```

### 1.5 上下文切换

#### 什么是上下文切换

**上下文切换（Context Switch）** 是 CPU 从一个进程（或线程）切换到另一个进程（或线程）的过程。在切换过程中，操作系统需要保存当前进程的状态，并恢复另一个进程的状态。

```

│                      上下文切换过程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  进程 A (运行中)          操作系统           进程 B (就绪)      │
│       │                       │                    │            │
│       │                       │                    │            │
│       ▼                       ▼                    ▼            │
│  ┌─────────┐            ┌───────────┐           ┌─────────┐     │
│  │ CPU 正在 │ ────────► │ 触发中断/ │ ─────────►│ 恢复进程 │     │
│  │ 执行 A   │            │ 系统调用   │           │ B 上下文 │     │
│  └─────────┘            └───────────┘           └─────────┘     │
│       │                       │                    │            │
```

## 2. Node.js 多进程架构

### 2.1 child_process 模块

Node.js 的 `child_process` 模块提供了创建子进程的能力，使我们能够实现多进程架构。以下是四种创建进程的方法及其详细对比：

```javascript
/**
 * Node.js child_process 模块详解
 * 本文件演示如何使用 spawn、exec、execFile、fork 四种方式创建子进程
 * 所有代码均带有中文注释，便于理解每个API的用法和区别
 */

// child_process 模块的所有导出
const {
    spawn,       // 创建流式子进程，适合大数据量
    exec,        // 创建带shell的命令执行，适合小数据量
    execFile,    // 直接执行文件，不使用shell，更安全
    fork,        // 创建Node.js子进程，用于运行JavaScript模块
} = require('child_process');


// ================================================================
// 2.1.1 spawn - 流式创建子进程
// ================================================================

/**
 * spawn 用于创建流式子进程，适合处理大数据量的场景
 * 特点：
 * - 通过流（Stream）进行数据传递，内存效率高
 * - 支持 stdout、stderr 的流式读取
 * - 可以处理大量输出而不会导致内存溢出
 *
 * @param command - 要执行的命令
 * @param args - 命令参数数组
 * @param options - 配置选项
 *
 * 返回值：ChildProcess 对象
 */
function demoSpawn() {
    console.log('='.repeat(60));
    console.log('spawn - 流式创建子进程');
    console.log('='.repeat(60));

    // 示例1：流式读取文件内容（模拟）
    // spawn('grep', ['-r', 'pattern', '.'])
    const lsProcess = spawn('ls', ['-la'], {
        // 工作目录
        cwd: '/tmp',
        // 环境变量
        env: { ...process.env, NODE_ENV: 'development' },
        // 不使用shell（更安全，但不支持shell内置命令）
        shell: false,
        // 设置 stdio 配置
        // 'pipe' - 通过 child.stdout/child.stderr 访问
        // 'inherit' - 继承父进程的 stdin/stdout/stderr
        // 'ignore' - 忽略子进程的 stdio
        stdio: ['pipe', 'pipe', 'pipe']
    });

    // 收集 stdout 数据
    let stdoutData = '';
    let stderrData = '';

    // on('data') 用于流式读取大量数据
    lsProcess.stdout.on('data', (chunk) => {
        // chunk 是 Buffer 对象
        stdoutData += chunk.toString();
    });

    // 监听 stderr
    lsProcess.stderr.on('data', (chunk) => {
        stderrData += chunk.toString();
    });

    // 进程退出事件
    lsProcess.on('close', (code, signal) => {
        console.log(`\n子进程退出，退出码: ${code}, 信号: ${signal}`);
        console.log('stdout 输出：');
        console.log(stdoutData.slice(0, 500)); // 只打印前500字符
    });

    // 错误处理
    lsProcess.on('error', (err) => {
        console.error('启动子进程失败:', err.message);
    });
}


// ================================================================
// 2.1.2 exec - 执行shell命令
// ================================================================

/**
 * exec 用于执行shell命令，适合小数据量的场景
 * 特点：
 * - 通过回调函数一次性接收所有输出
 * - 支持shell内置命令（管道、重定向等）
 * - 输出有最大限制（默认 200KB），超出会被截断
 * - 返回一个 child_process 实例，可以手动终止
 *
 * @param command - 要执行的shell命令
 * @param options - 配置选项（包含回调函数）
 *
 * options 常用选项：
 * - cwd: 工作目录
 * - env: 环境变量
 * - shell: 使用的shell程序
 * - timeout: 超时时间（毫秒）
 * - maxBuffer: stdout/stderr 的最大缓冲区（字节）
 * - killSignal: 终止信号
 */
function demoExec() {
    console.log('\n' + '='.repeat(60));
    console.log('exec - 执行shell命令');
    console.log('='.repeat(60));

    // 示例：查找Node进程
    exec('ps aux | grep node | head -5', {
        // 超时时间 5 秒
        timeout: 5000,
        // 最大缓冲区 1MB
        maxBuffer: 1024 * 1024
    }, (error, stdout, stderr) => {
        if (error) {
            // 如果进程超时或出错
            console.error('exec 错误:', error.message);
            console.error('错误码:', error.code);
            console.error('被终止的信号:', error.signal);
            return;
        }

        console.log('执行成功!');
        console.log('stdout:');
        console.log(stdout);

        if (stderr) {
            console.log('stderr:');
            console.log(stderr);
        }
    });
}


// ================================================================
// 2.1.3 execFile - 直接执行可执行文件
// ================================================================

/**
 * execFile 与 exec 类似，但不通过shell执行
 * 特点：
 * - 直接调用指定的执行文件，不启动shell解析
 * - 比 exec 更安全（避免了shell注入攻击）
 * - 性能稍好（省去了shell解析开销）
 * - 不支持shell内置特性（管道、重定向等）
 *
 * @param file - 可执行文件的路径
 * @param args - 命令行参数数组
 * @param options - 配置选项
 */
function demoExecFile() {
    console.log('\n' + '='.repeat(60));
    console.log('execFile - 直接执行可执行文件');
    console.log('='.repeat(60));

    // 示例：执行 ls 命令
    // 注意：execFile 直接执行 ls，不像 exec 那样通过 shell 执行
    // 所以 'ls -la' 会被当作一个命令名，而不是 'ls' 和 ['-la']
    execFile('ls', ['-la', '/tmp'], {
        // 环境变量
        env: { ...process.env },
        // 工作目录
        cwd: '/tmp'
    }, (error, stdout, stderr) => {
        if (error) {
            console.error('execFile 错误:', error.message);
            return;
        }
        console.log('执行成功!');
        console.log(stdout);
    });
}


// ================================================================
// 2.1.4 fork - 创建Node.js子进程
// ================================================================

/**
 * fork 用于创建Node.js子进程，专门用于运行JavaScript模块
 * 特点：
 * - 创建一个新的V8实例，但共享部分资源
 * - 父子进程之间可以通过 IPC（进程间通信）传递消息
 * - 可以使用 process.send() 发送消息
 * - 父子进程共享模块缓存
 *
 * @param modulePath - 要执行的JavaScript模块路径
 * @param args - 命令行参数数组
 * @param options - 配置选项
 *
 * 特殊选项：
 * - execArgv: 传递给 Node.js 可执行文件的参数
 * - stdio: 配置父子进程间的通信通道
 */
function demoFork() {
    console.log('\n' + '='.repeat(60));

    const child = fork(tempFile, [], {
        // 传递给 Node.js 的参数
        execArgv: ['--max-old-space-size=512'],
        // 工作目录
        cwd: '/tmp',
        // silent 模式可以让子进程的输出通过 IPC 传递
        silent: false

    console.log('[父进程] 创建子进程 PID:', child.pid);

    // 监听子进程发送的消息

        console.log('[父进程] 收到消息:', msg);

        if (msg.type === 'ready') {
            console.log('[父进程] 子进程已准备好，发送计算任务...');

            // 向子进程发送消息
            child.send({ type: 'compute', data: 42 });
        }

        if (msg.type === 'result') {
            console.log('[父进程] 计算结果:', msg.output);
            console.log('[父进程] 完成，终止子进程');

            // 终止子进程
            child.kill('SIGTERM');
        }
    });

    // 监听子进程退出
    child.on('exit', (code, signal) => {
        console.log('[父进程] 子进程已退出，退出码:', code, '信号:', signal);

        // 清理临时文件
        fs.unlinkSync(tempFile);
    });

    // 监听错误
    child.on('error', (err) => {
        console.error('[父进程] 子进程错误:', err.message);
    });
}
```

### 2.2 cluster 模块实现负载均衡

Node.js 的 `cluster` 模块让你能够轻松创建共享服务器端口的子进程，实现负载均衡。以下是完整的解析：

```javascript
/**
 * Node.js cluster 模块详解
 * 实现多进程负载均衡，处理 HTTP 请求
 *
 * 核心概念：
 * 1. master 进程 - 负责管理 worker 进程，不处理请求
 * 2. worker 进程 - 处理实际请求，每个都是独立的 Node.js 进程
 * 3. 负载均衡 - 由操作系统内核完成（Linux 使用 SO_REUSEPORT）
 */

const cluster = require('cluster');
const http = require('http');
const os = require('os');

// 获取 CPU 核心数（建议设置为 CPU 核心数或核心数的两倍）
const numCPUs = os.cpus().length;

console.log('='.repeat(60));
console.log('Node.js Cluster 模块详解');
console.log('='.repeat(60));
console.log(`CPU 核心数: ${numCPUs}`);
console.log(`当前进程类型: ${cluster.isMaster ? 'Master' : 'Worker'}`);
console.log(`当前进程 PID: ${process.pid}`);
console.log('='.repeat(60));


// ================================================================
// 2.2.1 cluster 基本使用
// ================================================================

/**
 * cluster 基本工作流程：
 *
 *                    ┌──────────────────┐
 *                    │   Master 进程    │
 *                    │                  │
 *                    │  - 创建 worker   │
 *                    │  - 管理进程      │
 *                    │  - 监听端口      │
 *                    │  - 负载均衡      │
 *                    └────────┬─────────┘
 *                             │
 *              ┌──────────────┼──────────────┐
 *              │              │              │
 *              ▼              ▼              ▼
 *        ┌──────────┐   ┌──────────┐   ┌──────────┐
 *        │ Worker 1 │   │ Worker 2 │   │ Worker 3 │
 *        │ PID: xxx │   │ PID: xxx │   │ PID: xxx │
 *        └──────────┘   └──────────┘   └──────────┘
 *        处理 33% 请求   处理 33% 请求   处理 34% 请求
 *
 * 在 Linux 上，Node.js cluster 使用内核级负载均衡
 * 使用 SO_REUSEPORT socket 选项，多个 worker 可以绑定到同一端口
 * 内核根据连接分布算法（通常是 least connections）分配请求
 */

// 定义 HTTP 服务器配置
const SERVER_CONFIG = {
    PORT: 3000,
    HOST: '0.0.0.0',
    // 每个 worker 处理请求前的延迟（模拟处理时间）
    PROCESS_DELAY: 100,
    // 最大请求数，超过后 worker 将自行退出（演示用）
    MAX_REQUESTS: 10
};

// worker 请求计数
let workerRequestCount = new Map();

/**
 * 创建 HTTP 服务器的函数
 * 这个函数会在每个 worker 进程中执行
 */
function createHttpServer() {
    const server = http.createServer((req, res) => {
        // 获取当前 worker 的请求计数
        const pid = process.pid;
        const count = workerRequestCount.get(pid) || 0;

        // 记录请求
        workerRequestCount.set(pid, count + 1);

        console.log(`[Worker ${pid}] 收到请求 #${count + 1}: ${req.method} ${req.url}`);

        // 模拟处理延迟
        setTimeout(() => {
            // 设置响应头
            res.writeHead(200, {
                'Content-Type': 'application/json',
                'X-Powered-By': 'Node.js Cluster',
                'X-Worker-PID': pid.toString(),
                'X-Request-Count': (count + 1).toString()
            });

            // 发送响应
            res.end(JSON.stringify({
                success: true,
                message: 'Hello from Worker',
                pid: pid,
                requestCount: count + 1,
                uptime: process.uptime().toFixed(2) + 's'
            }, null, 2));

            console.log(`[Worker ${pid}] 响应请求 #${count + 1} 完成`);

            // 超过最大请求数后，worker 退出（演示用）
            if (count + 1 >= SERVER_CONFIG.MAX_REQUESTS) {
                console.log(`[Worker ${pid}] 达到最大请求数 ${SERVER_CONFIG.MAX_REQUESTS}，即将退出`);
                server.close(() => {
                    console.log(`[Worker ${pid}] 服务器已关闭`);
                    process.exit(0);
                });
            }
        }, SERVER_CONFIG.PROCESS_DELAY);
    });

    return server;
}


// ================================================================
// 2.2.2 Master 进程逻辑
// ================================================================

/**
 * Master 进程负责：
 * 1. 创建 worker 进程
 * 2. 监听 worker 的生命周期事件
 * 3. 管理 worker（重启、扩缩容）
 * 4. 不处理实际的 HTTP 请求
 */
function setupMasterProcess() {
    console.log('\n' + '-'.repeat(60));
    console.log('Master 进程启动');
    console.log('-'.repeat(60));

    // 创建 worker 进程
    const numWorkers = Math.min(numCPUs, 4);  // 限制最多 4 个 worker

    console.log(`\n准备创建 ${numWorkers} 个 worker 进程...\n`);

    // fork 是 cluster.fork() 的简写
    for (let i = 0; i < numWorkers; i++) {
        const worker = cluster.fork();

        // 监听单个 worker 的事件
        worker.on('online', () => {
            console.log(`[Master] Worker ${worker.process.pid} 已启动（在线）`);
        });

        worker.on('listening', (address) => {
            console.log(`[Master] Worker ${worker.process.pid} 正在监听 ${address.address}:${address.port}`);
        });

        worker.on('disconnect', () => {
            console.log(`[Master] Worker ${worker.process.pid} 已断开连接`);
        });

        worker.on('exit', (code, signal) => {
            console.log(`[Master] Worker ${worker.process.pid} 已退出，`
                + `退出码: ${code}, 信号: ${signal}`);

            // 如果 worker 不是正常退出（退出码为 0），则重新启动
            if (code !== 0) {
                console.log(`[Master] Worker 非正常退出，3秒后重新启动...`);

                setTimeout(() => {
                    const newWorker = cluster.fork();
                    newWorker.on('online', () => {
                        console.log(`[Master] 替代 Worker ${newWorker.process.pid} 已启动`);
                    });
                }, 3000);
            }
        });

        // 监听来自 worker 的消息
        worker.on('message', (message) => {
            console.log(`[Master] 收到来自 Worker ${worker.process.pid} 的消息:`, message);
        });
    }

    // 监听 cluster 的事件
    cluster.on('fork', (worker) => {
        console.log(`[Master] 正在 fork Worker ${worker.process.pid}`);
    });

    cluster.on('online', (worker) => {
        console.log(`[Master] Worker ${worker.process.pid} 已在线`);
    });

    cluster.on('listening', (worker, address) => {
        console.log(`[Master] Worker ${worker.process.pid} 开始监听端口 ${address.port}`);
    });

    cluster.on('disconnect', (worker) => {
        console.log(`[Master] Worker ${worker.process.pid} 断开连接`);
    });

    // 当所有 worker 都退出时，master 退出
    cluster.on('exit', (worker, code, signal) => {
        console.log(`\n[Master] 所有 worker 已退出`);
        console.log(`[Master] 最后退出码: ${code}`);
    });

    // 向所有 worker 发送消息
    setInterval(() => {
        const workerList = Object.values(cluster.workers);
        if (workerList.length > 0) {
            workerList.forEach(worker => {
                worker.send({
                    type: 'heartbeat',
                    from: 'master',
                    time: Date.now()
                });
            });
        }
    }, 5000);
}


// ================================================================
// 2.2.3 Worker 进程逻辑
// ================================================================

/**
 * Worker 进程负责：
 * 1. 处理实际的 HTTP 请求
 * 2. 与 master 进程通信（IPC）
 * 3. 管理自己的生命周期
 */
function setupWorkerProcess() {
    console.log('\n' + '-'.repeat(60));
    console.log(`Worker ${process.pid} 启动`);
    console.log('-'.repeat(60));

    // 创建 HTTP 服务器
    const server = createHttpServer();

    // worker 进程通过 listen() 监听端口，但不需要指定 host:port
    // 因为 master 进程已经绑定了端口
    server.listen(SERVER_CONFIG.PORT, SERVER_CONFIG.HOST, () => {
        console.log(`[Worker ${process.pid}] HTTP 服务器已启动`);
        console.log(`[Worker ${process.pid}] 监听 ${SERVER_CONFIG.HOST}:${SERVER_CONFIG.PORT}`);

        // 通知 master 进程，worker 已准备好
        // 注意：process.send() 在 cluster 模式下会通过 IPC 发送
        if (process.send) {
            process.send({
                type: 'worker_ready',
                pid: process.pid,
                port: SERVER_CONFIG.PORT
            });
        }
    });

    // 处理错误
    server.on('error', (err) => {
        console.error(`[Worker ${process.pid}] 服务器错误:`, err.message);
    });

    // 处理连接
    server.on('connection', (socket) => {
        console.log(`[Worker ${process.pid}] 新连接，本地地址: ${socket.localAddress}`);
    });

    // 处理进程消息
    process.on('message', (msg) => {
        console.log(`[Worker ${process.pid}] 收到 master 消息:`, msg);

        if (msg.type === 'shutdown') {
            console.log(`[Worker ${process.pid}] 收到关闭信号，准备退出...`);
            server.close(() => {
                process.exit(0);
            });
        }
    });

    // 处理进程退出
    process.on('SIGTERM', () => {
        console.log(`[Worker ${process.pid}] 收到 SIGTERM 信号`);
        server.close(() => {
            process.exit(0);
        });
    });

    process.on('SIGINT', () => {
        console.log(`[Worker ${process.pid}] 收到 SIGINT 信号`);
        server.close(() => {
            process.exit(0);
        });
    });
}

// 2.2.4 负载均衡策略
// ================================================================

/**
 * Node.js cluster 的负载均衡策略：
 *
 * 1. 轮询（Round Robin）- 默认策略
 *    - Linux/Mac：操作系统内核按顺序将连接分配给 worker
 *    - 每个新连接分配给最近最少使用的 worker

 * 2. 共享socket（Shared Socket）
 *    - 所有 worker 监听同一端口
 *    - 内核自动进行负载均衡
 *    - 使用 SO_REUSEPORT（Linux 3.9+）或 SO_REUSEADDR（macOS）
 *
 * 3. 进程间负载不均衡问题
 *    - 长时间运行的连接会导致负载不均
 *    - HTTP Keep-Alive 连接会占用 worker
 *    - 建议：合理设置 keep-alive timeout
 */

// 获取当前使用的调度算法（仅 Linux）
if (cluster.isMaster && process.platform === 'linux') {
    // Linux 3.9+ 支持 SCHED_* 常量
    console.log('\n' + '-'.repeat(60));
    console.log('负载均衡调度算法');
    console.log('-'.repeat(60));

    try {

            [cluster.SCHED_NONE]: 'SCHED_NONE - 不使用负载均衡，由操作系统决定',

        };

    }
}


// ================================================================
// 2.2.5 优雅关闭
// ================================================================

/**
 * 优雅关闭（Graceful Shutdown）流程：
 *
 * 1. master 进程收到关闭信号（SIGTERM/SIGINT）
 * 2. master 停止接受新连接
 * 3. master 向所有 worker 发送关闭信号
 * 4. worker 停止接受新请求
 * 5. worker 等待现有请求处理完成（设置超时）
 * 6. worker 关闭服务器
 * 7. worker 进程退出
 * 8. master 进程退出
 */

function setupGracefulShutdown() {
    if (!cluster.isMaster) {
        // worker 的优雅关闭逻辑已在 setupWorkerProcess() 中实现
        return;
    }

    console.log('\n' + '-'.repeat(60));
    console.log('设置优雅关闭处理');
    console.log('-'.repeat(60));

    // 优雅关闭函数
    const shutdown = (signal) => {
        console.log(`\n[Master] 收到 ${signal} 信号，开始优雅关闭...`);

        // 停止接受新的 worker
        cluster.disconnect(() => {
            console.log('[Master] 所有 worker 已断开连接');

            // 退出 master 进程
            process.exit(0);
        });

        // 设置超时，强制退出
        setTimeout(() => {
            console.error('[Master] 优雅关闭超时，强制退出');
            process.exit(1);
        }, 10000);
    };

    // 监听关闭信号
    process.on('SIGTERM', () => shutdown('SIGTERM'));
    process.on('SIGINT', () => shutdown('SIGINT'));
}


// ================================================================
// 2.2.6 完整的 Cluster HTTP 服务器
// ================================================================

/**
 * 完整的 Cluster HTTP 服务器实现
 * 包含：自动重启、负载报告、状态监控
 */
class ClusterServer {
    constructor(options = {}) {
        this.port = options.port || 3000;
        this.host = options.host || '0.0.0.0';
        this.workers = new Map();
        this.isShuttingDown = false;
    }

    /**
     * 启动集群服务器
     */
    start() {
        if (!cluster.isMaster) {
            this.startWorker();
            return;
        }

        console.log('='.repeat(60));
        console.log('ClusterServer 启动');
        console.log(`监听端口: ${this.port}`);
        console.log(`Worker 数量: ${require('os').cpus().length}`);
        console.log('='.repeat(60));

        // 设置 master 进程
        this.setupMaster();

        // 启动 workers
        this.spawnWorkers();

        // 设置优雅关闭
        this.setupGracefulShutdown();
    }

    /**
     * 设置 master 进程逻辑
     */
    setupMaster() {
        // 记录启动时间
        this.startTime = Date.now();

        // fork workers
        cluster.on('fork', (worker) => {
            this.workers.set(worker.id, {
                pid: worker.process.pid,
                state: 'starting',
                startTime: Date.now(),
                requestCount: 0
            });

            console.log(`[Master] Fork Worker ${worker.id} (PID: ${worker.process.pid})`);
        });

        // Worker 上线
        cluster.on('online', (worker) => {
            const info = this.workers.get(worker.id);
            if (info) {
                info.state = 'online';
            }
            console.log(`[Master] Worker ${worker.id} is online`);
        });

        // Worker 监听端口
        cluster.on('listening', (worker, address) => {
            console.log(`[Master] Worker ${worker.id} listening on ${address.address}:${address.port}`);
        });

        // Worker 断开
        cluster.on('disconnect', (worker) => {
            const info = this.workers.get(worker.id);
            if (info) {
                info.state = 'disconnected';
            }
            console.log(`[Master] Worker ${worker.id} disconnected`);
        });

        // Worker 退出
        cluster.on('exit', (worker, code, signal) => {
            const info = this.workers.get(worker.id);
            console.log(`[Master] Worker ${worker.id} exited (code: ${code}, signal: ${signal})`);

            if (this.workers.has(worker.id)) {
                this.workers.delete(worker.id);
            }

            // 非正常退出时重启
            if (code !== 0 && !this.isShuttingDown) {
                console.log(`[Master] Restarting worker ${worker.id}...`);
                const newWorker = cluster.fork();
                newWorker.on('online', () => {
                    console.log(`[Master] Replacement worker ${newWorker.id} is online`);
                });
            }
        });

        // 定期报告状态
        setInterval(() => {
            this.printStatus();
        }, 10000);
    }

    /**
     * 启动 worker 进程
     */
    startWorker() {
        const server = createHttpServer();

        server.listen(this.port, this.host, () => {
            console.log(`[Worker ${process.pid}] Server started`);
        });

        // 监听消息
        process.on('message', (msg) => {
            if (msg.type === 'status_request') {
                process.send({
                    type: 'status_response',
                    pid: process.pid,
                    uptime: process.uptime(),
                    memory: process.memoryUsage()
                });
            }
        });
    }

    /**
     * 创建 workers
     */
    spawnWorkers() {
        const numWorkers = Math.min(os.cpus().length, 4);

        for (let i = 0; i < numWorkers; i++) {
            cluster.fork();
        }

        console.log(`[Master] Created ${numWorkers} workers`);
    }

    /**
     * 打印集群状态
     */
    printStatus() {
        if (this.isShuttingDown) return;

        console.log('\n' + '-'.repeat(60));
        console.log('Cluster Status Report');
        console.log('-'.repeat(60));
        console.log(`Uptime: ${((Date.now() - this.startTime) / 1000).toFixed(0)}s`);
        console.log(`Workers: ${this.workers.size}`);
        console.log(`Worker States:`);

        for (const [id, info] of this.workers) {
            console.log(`  [${id}] PID ${info.pid}: ${info.state} (uptime: ${info.startTime})`);
        }
    }

    /**
     * 设置优雅关闭
     */
    setupGracefulShutdown() {
        const shutdown = () => {
            console.log('\n[Master] Shutting down gracefully...');
            this.isShuttingDown = true;

            // 通知所有 worker 关闭
            for (const worker of Object.values(cluster.workers)) {
                process.emit('message', { type: 'shutdown' });
            }

            // 断开连接，等待 worker 退出
            cluster.disconnect(() => {
                console.log('[Master] All workers disconnected');
                process.exit(0);
            });

            // 强制退出超时
            setTimeout(() => {
                console.error('[Master] Forced shutdown after timeout');
                process.exit(1);
            }, 10000);
        };

        process.on('SIGTERM', shutdown);
        process.on('SIGINT', shutdown);
    }
}


// ================================================================
// 主程序入口
// ================================================================

if (cluster.isMaster) {
    // Master 进程执行
    setupMasterProcess();
    setupGracefulShutdown();

    // 创建集群服务器
```

### 2.4 IPC 进程间通信

Node.js 中父子进程之间的 IPC 通信机制：

```javascript
/**
 * Node.js IPC（进程间通信）详解
 * 演示父子进程之间如何通过消息传递进行通信
 */

const { fork } = require('child_process');
const path = require('path');

// ================================================================
// 2.4.1 IPC 基本通信
// ================================================================

/**
 * IPC 通信原理
 *
 *                    ┌─────────────────────┐
 *                    │     父进程           │
 *                    │                     │
 *                    │  process.on()      │
 *                    │         ▲           │
 *                    │         │           │
 *                    │    [Channel]        │
 *                    │         │           │
 *                    │         ▼           │
 *                    │  process.send()    │
 *                    └─────────────────────┘
 *                              │
 *                    IPC Channel (管道/Unix Socket)
 *                              │
 *                    ┌─────────────────────┐
 *                    │     子进程           │
 *                    │                     │
 *                    │  process.send()    │
 *                    │         ▲           │
 *                    │         │           │
 *                    │    [Channel]        │
 *                    │         │           │
 *                    │         ▼           │
 *                    │  process.on()      │
 *                    └─────────────────────┘
 *
 * 底层实现：
 * - Unix Domain Socket（Linux/macOS）
 * - Named Pipe（Windows）
 * - 使用同样的文件描述符进行读写
 */

/**
 * IPC 消息类型
 *
 * Node.js 支持多种类型的 IPC 消息：
 * 1. 普通对象 - 自动 JSON 序列化/反序列化
 * 2. 原始数据 - 通过 sendHandle() 发送
 * 3. TCP 句柄 - 发送网络连接
 * 4. 文件描述符 - 发送打开的文件
 */

/**
 * 创建子进程并设置 IPC 通信
 */
function setupIPCCommunication() {
    console.log('='.repeat(60));
    console.log('IPC 进程间通信演示');
    console.log('='.repeat(60));

    // 创建子进程脚本
    const childScript = `
        const { parentPort } = require('worker_threads');

        // worker_threads 使用 parentPort 进行通信
        if (parentPort) {
            parentPort.on('message', (msg) => {
                console.log('[子进程] 收到消息:', msg);

                // 回复消息
                parentPort.postMessage({
                    type: 'response',
                    original: msg,
                    processed: true,
                    timestamp: Date.now()
                });
            });
        }
    `;

    // 使用 fork 创建子进程（与上面的 worker_threads 不同）
    // fork 的子进程使用 process.send/process.on 进行通信
    const child = fork(path.join(__dirname, 'ipc-child.js'), [], {
        // silent 模式允许我们访问子进程的 stdout/stderr
        silent: true
    });

    console.log(`[父进程] 创建子进程 PID: ${child.pid}`);

    // 父进程监听子进程的消息
    child.on('message', (msg) => {
        console.log('[父进程] 收到子进程消息:', msg);

        if (msg.type === 'ready') {
            console.log('[父进程] 子进程已就绪，开始通信...');

            // 发送消息给子进程
            child.send({
                type: 'greeting',
                content: 'Hello from parent!',
                timestamp: Date.now()
            });
        }
    });

    // 监听子进程的 stdout
    child.stdout.on('data', (data) => {
        console.log('[父进程] 子进程 stdout:', data.toString().trim());
    });

    // 监听子进程的 stderr
    child.stderr.on('data', (data) => {
        console.error('[父进程] 子进程 stderr:', data.toString().trim());
    });

    // 模拟多次消息交换
    let count = 0;
    const interval = setInterval(() => {
        count++;
        child.send({
            type: 'ping',
            count: count,
            pid: process.pid
        });

        if (count >= 5) {
            clearInterval(interval);
            // 发送关闭信号
            child.send({ type: 'shutdown' });

            setTimeout(() => {
                child.kill('SIGTERM');
            }, 500);
        }
    }, 1000);

    // 处理子进程退出
    child.on('exit', (code, signal) => {
        console.log(`[父进程] 子进程退出，退出码: ${code}，信号: ${signal}`);
    });

    return child;
}

/**
 * process.send() 的高级用法：sendHandle
 *
 * 可以通过 IPC 传递以下句柄：
 * - net.Socket - TCP 套接字
 * - net.Server - TCP 服务器
 * - dgram.Socket - UDP 套接字

/**
 * 使用 IPC 传递 TCP 服务器句柄
 * 实现多进程共享同一端口
 */
function shareTCPServerWithIPC() {
    const net = require('net');
    const { fork } = require('child_process');

    console.log('\n' + '='.repeat(60));

    console.log('='.repeat(60));

    // 创建 TCP 服务器
    const server = net.createServer((socket) => {
        console.log(`[主服务器] 收到连接 from ${socket.remoteAddress}`);
        socket.write('Hello from main server!\n');
        socket.end();
    });

    server.listen(3000, '0.0.0.0', () => {

 * 如果需要共享状态，需要使用：

/**

 */
function shareConfigBetweenProcesses() {
    console.log('\n' + '='.repeat(60));
    console.log('进程间共享配置');
    console.log('='.repeat(60));

    // 配置对象
    const config = {
        app: {
            name: 'my-app',
            version: '1.0.0',
            port: 3000,
            env: process.env.NODE_ENV || 'development'

 * 实现进程间心跳检测机制

function heartbeatExample() {
    console.log('\n' + '='.repeat(60));
    console.log('进程间心跳检测');
    console.log('='.repeat(60));

    const { fork } = require('child_process');

    // 创建子进程
    const child = fork(path.join(__dirname, 'ipc-child.js'), [], {
        silent: true
    });

    // 心跳状态
    let heartbeatCount = 0;
    const HEARTBEAT_INTERVAL = 1000;  // 1秒
    const HEARTBEAT_TIMEOUT = 5000;    // 5秒超时
    let lastHeartbeat = Date.now();

    console.log('[父进程] 启动心跳检测...');

    // 定时发送心跳
    const heartbeatTimer = setInterval(() => {
        heartbeatCount++;

        child.send({
            type: 'heartbeat',
            count: heartbeatCount,
            timestamp: Date.now()
        });
    }, HEARTBEAT_INTERVAL);

    // 监听子进程的心跳回复
    child.on('message', (msg) => {
        if (msg.type === 'heartbeat_ack') {
            lastHeartbeat = Date.now();
            console.log(`[父进程] 收到心跳回复 #${msg.count}，延迟: ${Date.now() - msg.timestamp}ms`);
        }
    });

    // 检查心跳超时
    const timeoutChecker = setInterval(() => {
        if (Date.now() - lastHeartbeat > HEARTBEAT_TIMEOUT) {
            console.error('[父进程] 心跳超时，子进程可能已死锁');
            child.kill('SIGKILL');
            clearInterval(heartbeatTimer);
            clearInterval(timeoutChecker);
        }
    }, 1000);

    // 清理函数
    const cleanup = () => {
        clearInterval(heartbeatTimer);
        clearInterval(timeoutChecker);
        child.kill('SIGTERM');
    };

    // 10秒后自动停止
    setTimeout(() => {
        console.log('[父进程] 心跳检测完成');
        cleanup();
    }, 10000);

    return child;
}

// 2.4.5 进程池模式

/**
 * 进程池模式
 * 预先创建一组 worker 进程，复用处理任务
 */

class ProcessPool {
    /**
     * @param {string} scriptPath - Worker 脚本路径
     * @param {number} poolSize - 进程池大小
     */
    constructor(scriptPath, poolSize) {
        this.scriptPath = scriptPath;
        this.poolSize = poolSize;
        this.workers = new Map();      // worker id -> worker info
        this.idleWorkers = [];         // 空闲的 worker 队列
        this.taskQueue = [];          // 任务队列
        this.activeTasks = new Map(); // 进行中的任务

        this.initialize();
    }

    /**
     * 初始化进程池
     */
    initialize() {
        console.log(`[进程池] 初始化，创建 ${this.poolSize} 个 worker`);

        for (let i = 0; i < this.poolSize; i++) {
            this.createWorker();
        }
    }

    /**
     * 创建单个 worker
     */
    createWorker() {
        const worker = fork(this.scriptPath, [], {
            silent: true
        });

        const id = `worker-${worker.pid}`;

        this.workers.set(id, {
            worker: worker,
            status: 'idle',
            currentTask: null
        });

        this.idleWorkers.push(id);

        // 监听 worker 消息
        worker.on('message', (msg) => {
            this.handleWorkerMessage(id, msg);
        });

        // 监听 worker 退出
        worker.on('exit', (code, signal) => {
            console.log(`[进程池] Worker ${id} 退出`);
            this.handleWorkerExit(id);
        });

        console.log(`[进程池] Worker ${id} 已创建`);
    }

    /**
     * 处理 worker 消息
     */
    handleWorkerMessage(workerId, msg) {
        const workerInfo = this.workers.get(workerId);

        if (msg.type === 'task_complete') {
            // 任务完成
            const task = workerInfo.currentTask;
            if (task && this.activeTasks.has(task.id)) {
                this.activeTasks.get(task.id).resolve(msg.result);
                this.activeTasks.delete(task.id);
            }

            // 重置 worker 状态
            workerInfo.status = 'idle';
            workerInfo.currentTask = null;

            // 如果有等待的任务，立即处理
            if (this.taskQueue.length > 0) {
                const nextTask = this.taskQueue.shift();
                this.assignTask(workerId, nextTask);
            } else {
                this.idleWorkers.push(workerId);
            }
        } else if (msg.type === 'task_error') {
            // 任务出错
            const task = workerInfo.currentTask;
            if (task && this.activeTasks.has(task.id)) {
                this.activeTasks.get(task.id).reject(new Error(msg.error));
                this.activeTasks.delete(task.id);
            }

            workerInfo.status = 'idle';
            workerInfo.currentTask = null;
            this.idleWorkers.push(workerId);
        }
    }

    /**
     * 处理 worker 退出
     */
    handleWorkerExit(workerId) {
        this.workers.delete(workerId);
        this.idleWorkers = this.idleWorkers.filter(id => id !== workerId);

        // 创建新的 worker 替换
        setTimeout(() => {
            console.log(`[进程池] 重新创建 worker`);
            this.createWorker();
        }, 1000);
    }

    /**
     * 分配任务给 worker
     */
    assignTask(workerId, task) {
        const workerInfo = this.workers.get(workerId);
        if (!workerInfo) return;

        workerInfo.status = 'busy';
        workerInfo.currentTask = task;
        workerInfo.worker.send({ type: 'task', data: task.data });

        this.idleWorkers = this.idleWorkers.filter(id => id !== workerId);
    }

    /**
     * 执行任务
     * @param {any} data - 任务数据
     * @returns {Promise} 任务结果
     */
    async execute(data) {
        return new Promise((resolve, reject) => {
            const task = {
                id: `task-${Date.now()}-${Math.random()}`,
                data: data,
                resolve: resolve,
                reject: reject
            };

            this.activeTasks.set(task.id, task);

            // 如果有空闲 worker，立即处理
            if (this.idleWorkers.length > 0) {
                const workerId = this.idleWorkers.shift();
                this.assignTask(workerId, task);
            } else {
                // 没有空闲 worker，加入队列
                this.taskQueue.push(task);
                console.log(`[进程池] 任务队列已满，任务 ${task.id} 加入队列`);
            }
        });
    }

    /**
     * 关闭进程池
     */
    async destroy() {
        console.log('[进程池] 关闭中...');

        for (const [id, info] of this.workers) {
            info.worker.kill('SIGTERM');
        }

        this.workers.clear();
        this.idleWorkers = [];
        this.taskQueue = [];
        this.activeTasks.clear();
    }

    /**
     * 获取状态信息
     */
    getStatus() {
        return {
            total: this.poolSize,
            active: this.workers.size - this.idleWorkers.length,
            idle: this.idleWorkers.length,
            queued: this.taskQueue.length,
            activeTasks: this.activeTasks.size
        };
    }
}
```

```javascript
class HealthCheck {
    constructor() {
        this.lastCheck = Date.now();
        this.checkInterval = null;
    }

    /**
     * 启动健康检查
     */
    start() {
        this.checkInterval = setInterval(() => {
            this.performHealthCheck();
        }, CONFIG.HEALTH_CHECK_INTERVAL);
    }

    /**
     * 停止健康检查
     */
    stop() {
        if (this.checkInterval) {
            clearInterval(this.checkInterval);
        }
    }

    /**
     * 执行健康检查
     */
    performHealthCheck() {
        const workers = cluster.workers;
        const now = Date.now();

        for (const id in workers) {
            const worker = workers[id];
            const stats = STATE.workerStats.get(worker.id);

            if (!stats) continue;

            // 检查 Worker 内存使用
            const memoryUsage = worker.process.memoryUsage();
            const memoryMB = memoryUsage.heapUsed / 1024 / 1024;

            if (memoryMB > CONFIG.MAX_MEMORY_MB) {
                logger.warn(`Worker ${worker.id} 内存使用过高: ${memoryMB.toFixed(2)}MB`);
                this.restartWorker(worker, 'memory_exceeded');
                continue;
            }

            // 检查 Worker 处理请求数
            if (stats.requestCount >= CONFIG.MAX_REQUEST_PER_WORKER) {
                logger.info(`Worker ${worker.id} 达到最大请求数，优雅重启`);
                this.restartWorker(worker, 'max_requests');
                continue;
            }

            // 检查 Worker 是否响应
            if (now - stats.lastActivity > CONFIG.REQUEST_TIMEOUT * 2) {
                logger.warn(`Worker ${worker.id} 长时间无响应，可能已死锁`);
                this.restartWorker(worker, 'timeout');
            }
        }
    }

    /**
     * 重启 Worker
     */
    restartWorker(worker, reason) {
        logger.info(`重启 Worker ${worker.id}，原因: ${reason}`);

        // 发送优雅关闭信号
        worker.send({ type: 'shutdown', reason });

        // 延迟强制关闭
        setTimeout(() => {
            if (worker.isDead() === false) {
                worker.kill('SIGTERM');
            }
        }, 5000);
    }
}

function startWorker() {
    const worker = cluster.worker;
    const workerLogger = new Logger(`Worker-${worker.id}`);

    workerLogger.info(`Worker ${worker.id} (PID: ${process.pid}) 启动`);

    // 初始化 Worker 统计
    STATE.workerStats.set(worker.id, {
        id: worker.id,
        pid: process.pid,
        startTime: Date.now(),
        requestCount: 0,
        lastActivity: Date.now(),
        memoryUsage: process.memoryUsage()
    });

    // 定期更新内存统计
    setInterval(() => {
        const stats = STATE.workerStats.get(worker.id);
        if (stats) {
            stats.memoryUsage = process.memoryUsage();
            stats.lastActivity = Date.now();
        }
    }, 5000);

    // 创建 HTTP 服务器
    const server = http.createServer((req, res) => {
        RequestHandler.handle(req, res);
    });

    // 设置超时
    server.keepAliveTimeout = CONFIG.KEEP_ALIVE_TIMEOUT;
    server.requestTimeout = CONFIG.REQUEST_TIMEOUT;

    // 监听请求事件
    server.on('request', (req, res) => {
        workerLogger.debug(`收到请求: ${req.method} ${req.url}`);
    });

    // 监听连接事件
    server.on('connection', (socket) => {
        socket.setKeepAlive(true, CONFIG.KEEP_ALIVE_TIMEOUT);
    });

    // 监听错误
    server.on('error', (err) => {
        workerLogger.error(`服务器错误: ${err.message}`);
    });

    // 监听关闭事件
    server.on('close', () => {
        workerLogger.info('服务器已关闭');
    });

    // 监听进程消息
    process.on('message', (msg) => {
        if (msg.type === 'shutdown') {
            workerLogger.info(`收到关闭信号，原因: ${msg.reason || 'unknown'}`);
            server.close(() => {
                workerLogger.info('优雅关闭完成');
                process.exit(0);
            });

            // 强制关闭超时
            setTimeout(() => {
                workerLogger.warn('优雅关闭超时，强制退出');
                process.exit(1);
            }, 5000);
        } else if (msg.type === 'status_request') {
            process.send({
                type: 'status_response',
                pid: process.pid,
                uptime: process.uptime(),
                memory: process.memoryUsage(),
                requestCount: STATE.workerStats.get(worker.id)?.requestCount || 0
            });
        }
    });

    // 启动服务器
    server.listen(CONFIG.PORT, CONFIG.HOST, () => {
        workerLogger.info(`HTTP 服务器监听 ${CONFIG.HOST}:${CONFIG.PORT}`);
    });
}

            createWorker();
        }
    }

    function createWorker() {
        const worker = cluster.fork();

        worker.on('online', () => {
            masterLogger.info(`Worker ${worker.id} 上线 (PID: ${worker.process.pid})`);
        });

        worker.on('listening', (address) => {
            masterLogger.info(`Worker ${worker.id} 监听 ${address.address}:${address.port}`);
        });

        worker.on('disconnect', () => {
            masterLogger.warn(`Worker ${worker.id} 断开连接`);
        });

        worker.on('exit', (code, signal) => {
            masterLogger.warn(`Worker ${worker.id} 退出 (code: ${code}, signal: ${signal})`);

            // 非正常退出时重启
            if (code !== 0 && !isShuttingDown) {
                masterLogger.info(`Worker ${worker.id} 非正常退出，${CONFIG.WORKER_RESTART_DELAY}ms 后重启`);
                setTimeout(createWorker, CONFIG.WORKER_RESTART_DELAY);
            }
        });

        worker.on('message', (msg) => {
            if (msg.type === 'status_response') {
                masterLogger.debug(`Worker ${worker.id} 状态: ${JSON.stringify(msg)}`);
            }
        });
    }

    // 优雅关闭
    let isShuttingDown = false;

    async function gracefulShutdown(signal) {
        if (isShuttingDown) return;
        isShuttingDown = true;

        masterLogger.info(`收到 ${signal} 信号，开始优雅关闭...`);

        // 停止健康检查
        healthCheck.stop();

        // 停止接受新连接
        cluster.disconnect(() => {
            masterLogger.info('所有 Worker 已断开连接');

            // 打印最终统计
            masterLogger.info('最终统计:');
            masterLogger.info(`  总请求数: ${STATE.totalRequests}`);
            masterLogger.info(`  总错误数: ${STATE.totalErrors}`);
            masterLogger.info(`  运行时间: ${Date.now() - STATE.startTime}ms`);

            process.exit(0);
        });

        // 强制关闭超时
        setTimeout(() => {
            masterLogger.error('优雅关闭超时，强制退出');
            process.exit(1);
        }, 30000);
    }

    // 监听关闭信号
    process.on('SIGTERM', () => gracefulShutdown('SIGTERM'));
    process.on('SIGINT', () => gracefulShutdown('SIGINT'));

    // 启动健康检查
    healthCheck.start();

    // 启动 Workers
    spawnWorkers();

    // 定期报告状态
    setInterval(() => {
        const uptime = Date.now() - STATE.startTime;
        const workerCount = Object.keys(cluster.workers).length;

        masterLogger.info(`[状态] 运行时间: ${Math.floor(uptime / 1000)}s | `
            + `Workers: ${workerCount} | `
            + `总请求: ${STATE.totalRequests} | `
            + `错误: ${STATE.totalErrors}`);
    }, 60000);
}


// ================================================================
// 主程序入口
// ================================================================

if (cluster.isMaster) {
    startMaster();
} else {
    startWorker();
}
```

### 3.3 垃圾回收算法

```javascript
/**
 * 垃圾回收算法详解
 * 演示各种垃圾回收算法的原理和实现
 */

// ================================================================
// 3.3.1 引用计数（Reference Counting）
// ================================================================

/**
 * 引用计数算法原理：
 *
 * 每个对象有一个引用计数器，当有新的引用指向对象时，计数器+1
 * 当引用被删除时，计数器-1，当计数器为0时，对象被回收
 *
 * 优点：回收及时，可以立即回收无用对象
 * 缺点：无法处理循环引用
 */

class ReferenceCountingGC {
    constructor() {
        // 对象表，存储所有对象
        this.objects = new Map();
        // 对象ID计数器
        this.nextId = 1;
    }

    /**
     * 创建新对象
     */
    createObject(name, data) {
        const id = this.nextId++;
        const obj = {
            id,
            name,
            data,
            // 引用计数初始为0
            refCount: 0,
            // 记录谁引用了这个对象
            references: []
        };
        this.objects.set(id, obj);
        console.log(`[创建对象] ${name} (ID=${id}), 初始引用计数: 0`);
        return obj;
    }

    /**
     * 添加引用
     * @param {Object} fromObj - 引用者对象
     * @param {Object} toObj - 被引用对象
     */
    addReference(fromObj, toObj) {
        if (!fromObj || !toObj) return;

        // 添加到引用列表
        fromObj.references.push(toObj.id);
        // 引用计数+1
        toObj.refCount++;

        console.log(`[添加引用] ${fromObj.name} -> ${toObj.name}, `
            + `引用计数: ${toObj.refCount}`);
    }

    /**
     * 移除引用
     * @param {Object} fromObj - 引用者对象
     * @param {Object} toObj - 被引用对象
     */
    removeReference(fromObj, toObj) {
        if (!fromObj || !toObj) return;

        // 从引用列表中移除
        const index = fromObj.references.indexOf(toObj.id);
        if (index !== -1) {
            fromObj.references.splice(index, 1);
            toObj.refCount--;

            console.log(`[移除引用] ${fromObj.name} -> ${toObj.name}, `
                + `引用计数: ${toObj.refCount}`);

            // 如果引用计数为0，回收对象
            if (toObj.refCount <= 0) {
                this.collect(toObj);
            }
        }
    }

    /**
     * 垃圾回收
     */
    collect(obj) {
        console.log(`[GC回收] ${obj.name} (ID=${obj.id}), 引用计数: ${obj.refCount}`);

        // 递归处理这个对象引用的其他对象
        for (const refId of obj.references) {
            const refObj = this.objects.get(refId);
            if (refObj) {
                refObj.refCount--;
                if (refObj.refCount <= 0) {
                    this.collect(refObj);
                }
            }
        }

        // 从对象表中删除
        this.objects.delete(obj.id);
        console.log(`[GC回收] ${obj.name} 已从内存中移除`);
    }

    /**
     * 显示所有对象状态
     */
    status() {
        console.log('\n--- 对象状态 ---');
        for (const [id, obj] of this.objects) {
            console.log(`  ID=${id}: ${obj.name}, refCount=${obj.refCount}, refs=[${obj.references.join(', ')}]`);
        }
        console.log(`总共 ${this.objects.size} 个对象\n`);
    }
}


/**
 * 引用计数无法处理循环引用
 */
function referenceCountingDemo() {
    console.log('='.repeat(60));
    console.log('引用计数算法演示');
    console.log('='.repeat(60));

    const gc = new ReferenceCountingGC();

    // 创建对象
    const objA = gc.createObject('A', { value: 1 });
    const objB = gc.createObject('B', { value: 2 });
    const objC = gc.createObject('C', { value: 3 });

    // 正常引用
    console.log('\n--- 正常引用 ---');
    gc.addReference(objA, objB);
    gc.addReference(objB, objC);

    gc.status();

    // 移除引用
    console.log('--- 移除引用 ---');
    gc.removeReference(objA, objB);
    gc.status();

    // 循环引用示例
    console.log('\n--- 循环引用 ---');
    const objX = gc.createObject('X', { value: 10 });
    const objY = gc.createObject('Y', { value: 20 });

    gc.addReference(objX, objY);
    gc.addReference(objY, objX);

    gc.status();

    console.log('问题：即使删除 X 和 Y 的外部引用，它们的引用计数仍为1，');
    console.log('导致内存泄漏（无法被回收）。这就是引用计数算法的缺陷。');
}


// ================================================================
// 3.3.2 标记清除（Mark and Sweep）
// ================================================================

/**
 * 标记清除算法原理：
 *
 * 1. 标记阶段：从根对象（GC Roots）开始，遍历所有可达对象
 * 2. 清除阶段：遍历所有对象，回收未被标记的对象
 *
 * 优点：可以处理循环引用
 * 缺点：需要 STW（Stop The World）暂停
 */

class MarkAndSweepGC {
    constructor() {
        // 所有对象
        this.objects = new Map();
        // GC Roots（全局对象、栈对象等）
        this.roots = new Set();
        this.nextId = 1;
    }

    /**
     * 创建新对象
     */
    createObject(name, data) {
        const id = this.nextId++;
        const obj = {
            id,
            name,
            data,
            // 是否被标记
            marked: false,
            // 引用的其他对象
            references: []
        };
        this.objects.set(id, obj);
        console.log(`[创建对象] ${name} (ID=${id})`);
        return obj;
    }

    /**
     * 添加根对象
     */
    addRoot(name, obj) {
        if (!obj) return;
        this.roots.add(obj);
        console.log(`[添加Root] ${name}`);
    }

    /**
     * 添加引用
     */
    addReference(fromObj, toObj) {
        if (!fromObj || !toObj) return;
        fromObj.references.push(toObj.id);
        console.log(`[添加引用] ${fromObj.name} -> ${toObj.name}`);
    }

    /**
     * 移除引用
     */
    removeReference(fromObj, toObj) {
        if (!fromObj || !toObj) return;
        const index = fromObj.references.indexOf(toObj.id);
        if (index !== -1) {
            fromObj.references.splice(index, 1);
            console.log(`[移除引用] ${fromObj.name} -> ${toObj.name}`);
        }
    }

    /**
     * 标记阶段：从根开始遍历所有可达对象
     */
    mark() {
        console.log('\n--- 标记阶段 ---');

        // 清除所有对象的标记
        for (const obj of this.objects.values()) {
            obj.marked = false;
        }

        // 从根开始深度优先搜索
        let markedCount = 0;

        const dfs = (obj) => {
            if (!obj || obj.marked) return;

            // 标记对象
            obj.marked = true;
            markedCount++;

            console.log(`[标记] ${obj.name}`);

            // 递归标记引用的对象
            for (const refId of obj.references) {
                const refObj = this.objects.get(refId);
                if (refObj) {
                    dfs(refObj);
                }
            }
        };

        // 遍历所有根
        for (const root of this.roots) {
            dfs(root);
        }

        console.log(`标记完成，共标记 ${markedCount} 个对象`);
    }

    /**
     * 清除阶段：回收未标记的对象
     */
    sweep() {
        console.log('\n--- 清除阶段 ---');

        let sweepCount = 0;

        for (const [id, obj] of this.objects) {
            if (!obj.marked) {
                console.log(`[清除] ${obj.name} (ID=${id})`);
                this.objects.delete(id);
                sweepCount++;
            }
        }

        console.log(`清除完成，共回收 ${sweepCount} 个对象`);
    }

    /**
     * 执行完整的 GC
     */
    garbageCollect() {
        console.log('\n' + '='.repeat(40));
        console.log('执行垃圾回收');
        console.log('='.repeat(40));

        this.mark();
        this.sweep();
        this.status();
    }

    /**
     * 显示状态
     */
    status() {
        console.log('\n--- 对象状态 ---');
        for (const [id, obj] of this.objects) {
            const mark = obj.marked ? '✓' : '✗';
            console.log(`  [${mark}] ID=${id}: ${obj.name}, refs=[${obj.references.map(r => this.objects.get(r)?.name || '?').join(', ')}]`);
        }
        console.log(`总共 ${this.objects.size} 个对象\n`);
    }
}

/**
 * 分代回收原理：
 *
 * 1. 观察：大多数对象是"朝生夕死"的（生命周期很短）
 * 2. 策略：将堆分为年轻代和老年代
 * 3. 年轻代：回收频繁（Minor GC）
 * 4. 老年代：回收较少（Major GC）
 *
 * 堆布局：
 *
 * ┌─────────────────────────────────────────────────────┐
 │                    Old Generation                     │
 │  ┌─────────────────────────────────────────────────┐ │
 │  │              Old Space                          │ │
 │  │  - 经历过多次 Minor GC 的对象                   │ │
 │  │  - 长期存活的对象                               │ │
 │  └─────────────────────────────────────────────────┘ │
 │  ┌─────────────────────────────────────────────────┐ │
 │  │           Large Object Space                    │ │
 │  │  - 大对象直接分配在这里                         │ │
 │  └─────────────────────────────────────────────────┘ │
 └─────────────────────────────────────────────────────┘
 │
 ▼
┌─────────────────────────────────────────────────────┐
│                   Young Generation                   │
│  ┌───────────────────┐  ┌───────────────────┐      │
│  │    To Space       │  │   From Space      │      │
│  │  ( Survivor 1 )   │  │  ( Eden + S0 )    │      │
│  │                   │  │                   │      │
│  │  复制完成后        │  │  正在使用          │      │
│  │  成为新的 Eden    │  │                   │      │
│  └───────────────────┘  └───────────────────┘      │
│                              ▲                       │
│                              │  Minor GC            │
│                              │  (From -> To)        │
└─────────────────────────────────────────────────────┘
 */

function generationalGCDemo() {
    console.log('\n' + '='.repeat(60));
    console.log('分代回收算法演示');
    console.log('='.repeat(60));

    console.log(`
分代回收策略：

┌─────────────────────────────────────────────────────────────────┐
│                     Old Generation (老年代)                      │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  - 经历过多次 Minor GC 的对象                              │  │
│  │  - 晋升阈值：默认 8 次 Minor GC                           │  │
│  │  - Major GC (Full GC) 时回收                              │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              ▲
                              │ 晋升

┌─────────────────────────────────────────────────────────────────┐

│                                                                  │
│  ┌─────────────────────┐      ┌─────────────────────┐          │
│  │    To Space         │      │    From Space       │          │
│  │  (Survivor 1)       │      │  (Eden + Survivor 0)│          │
│  │                     │      │                     │          │
│  │  Minor GC 目标空间   │ ──►  │  当前使用的空间      │          │

回收流程：

5. Old Generation 满时，触发 Major GC

分代回收的优点：
- 频繁回收新对象区（大多数对象生命周期短）
- 减少 Major GC 的频率
- 提高整体回收效率
```

```javascript
/**
 * 实时内存监控示例
 */
class MemoryMonitor {
    constructor(interval = 5000) {
        this.interval = interval;
        this.timer = null;
        this.history = [];
        this.maxHistory = 100;
        this.baseline = null;
    }

    /**
     * 启动监控
     */
    start() {
        console.log('='.repeat(60));
        console.log('内存监控启动');
        console.log('='.repeat(60));

        // 获取基线
        this.baseline = this.getMemorySnapshot();

        this.timer = setInterval(() => {
            this.check();
        }, this.interval);
    }

    /**
     * 停止监控
     */
    stop() {
        if (this.timer) {
            clearInterval(this.timer);
            this.timer = null;
        }
    }

    /**
     * 获取内存快照
     */
    getMemorySnapshot() {
        const usage = process.memoryUsage();
        const heapStats = v8.getHeapStatistics();

        return {
            timestamp: Date.now(),
            heapUsed: usage.heapUsed,
            heapTotal: usage.heapTotal,
            external: usage.external,
            rss: usage.rss,
            heapLimit: heapStats.heap_size_limit,
            available: heapStats.total_available_size
        };
    }

    /**
     * 检查内存
     */
    check() {
        const current = this.getMemorySnapshot();
        this.history.push(current);

        // 限制历史记录长度
        if (this.history.length > this.maxHistory) {
            this.history.shift();
        }

        // 计算增长
        const growth = this.baseline
            ? ((current.heapUsed - this.baseline.heapUsed) / this.baseline.heapUsed * 100).toFixed(2)
            : 0;

        // 输出状态
        const heapUsedMB = (current.heapUsed / 1024 / 1024).toFixed(2);
        const heapTotalMB = (current.heapTotal / 1024 / 1024).toFixed(2);
        const heapLimitMB = (current.heapLimit / 1024 / 1024).toFixed(2);

        console.log(`[${new Date().toISOString()}] `
            + `堆: ${heapUsedMB}/${heapTotalMB}MB (${heapLimitMB}MB) `
            + `增长: ${growth}%`);

        // 检测异常增长
        if (growth > 50) {
            console.warn('⚠️  内存增长超过 50%，可能存在内存泄漏！');
        }
    }

    /**
     * 获取历史数据
     */
    getHistory() {
        return this.history;
    }

    /**
     * 生成内存报告
     */
    generateReport() {
        if (this.history.length < 2) {
            return '历史数据不足';
        }

        const first = this.history[0];
        const last = this.history[this.history.length - 1];
        const growth = last.heapUsed - first.heapUsed;
        const growthPercent = ((growth / first.heapUsed) * 100).toFixed(2);

        return {
            startTime: new Date(first.timestamp).toISOString(),
            endTime: new Date(last.timestamp).toISOString(),
            initialHeap: (first.heapUsed / 1024 / 1024).toFixed(2) + ' MB',
            finalHeap: (last.heapUsed / 1024 / 1024).toFixed(2) + ' MB',
            growth: (growth / 1024 / 1024).toFixed(2) + ' MB',
            growthPercent: growthPercent + '%',
            dataPoints: this.history.length
        };
    }
}
```

```bash
#!/bin/bash

PROCESS_NAME="node"
```

```bash
MAX_MEM=80

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1"
}

# 检查进程是否存在
check_process() {
    if pgrep -x "$PROCESS_NAME" > /dev/null; then
        return 0
    else
        return 1
    fi
}

# 获取进程统计
get_stats() {
    local pid=$1
    ps -p $pid -o %cpu,%mem,cmd --no-headers
}

# 发送告警
send_alert() {
    local subject=$1
    local message=$2
    echo "$message" | mail -s "$subject" "$ALERT_EMAIL"
}

# 监控循环
monitor() {
    log "开始监控 $PROCESS_NAME"

    while true; do
        if ! check_process; then
            log "进程不存在，启动中..."
            systemctl start $PROCESS_NAME
            send_alert "[警告] $PROCESS_NAME 进程已重启" "进程意外退出，已自动重启"

            continue

            mem=$(echo $stats | awk '{print $2}')

            fi
        done

        sleep 5
    done
}
```
