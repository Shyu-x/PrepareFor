# 向量数据库与 AI 搜索实战指南

> 前言：你有没有想过，为什么你跟朋友说"我想看那种暖暖的、治愈系的电影"，搜索引擎居然能给你推荐《海蒂和爷爷》、《小森林》这样的影片？传统数据库只会精确匹配"治愈"这个关键词，而现代 AI 搜索能理解"暖暖的、治愈系"背后的含义。这就是向量数据库的魔力——它把文字、图片、声音都转换成"向量"（一组数字），然后通过计算向量之间的"距离"来找相似的内容。

## 一、向量数据库基础：你需要知道的核心概念

### 1.1 什么是向量？

**向量（Vector）**：在数学中，向量是有方向和大小的量。在计算机中，向量是一组有序的浮点数，可以理解为多维空间中的一个点。

```
向量的直观理解：

1 维向量：就是一个数字
    例如：[3.14]  →  数轴上的一个点

2 维向量：平面上一个点
    例如：[3.14, 2.71]  →  二维坐标系中的点

3 维向量：空间中一个点
    例如：[1.0, 2.0, 3.0]  →  三维空间中的点

N 维向量：多维空间中的点（AI 常用）
    例如：一张图片的向量可能是 [0.12, 0.34, 0.56, ..., 0.98]（768 维）
    一段文字的向量可能是 [0.01, 0.23, 0.45, ..., 0.77]（768 维）
```

**向量的特性：**

```
向量的两个核心特性：

1. 相似的向量在空间中是"接近"的
   "温暖的阳光"  → [0.12, 0.45, 0.67, ...]
   "热辣的阳光"  → [0.13, 0.46, 0.68, ...]  ← 与上面相似
   "冰冷的雪花"  → [0.89, 0.12, 0.05, ...]  ← 与上面不相似

2. 可以计算向量之间的距离
   - 欧氏距离：直线距离，类似我们理解的几何距离
   - 余弦相似度：方向相似度，取值 -1 到 1
   - 点积：向量相乘的综合指标
```

### 1.2 什么是向量数据库？

**向量数据库（Vector Database）**：专门用于存储和检索向量数据的数据库，支持高维向量的相似性搜索。

```
向量数据库 vs 传统数据库：

传统数据库：
├── 存储结构化数据（表、行列）
├── 精确匹配（WHERE name = '张三'）
├── 等于查询（找到完全相等的内容）
└── 不擅长语义理解

向量数据库：
├── 存储高维向量（768 维、1536 维等）
├── 近似匹配（找到"差不多"的内容）
├── 相似性搜索（找到最相似的 K 个）
└── 天然支持语义理解
```

**向量数据库的工作原理：**

```
向量检索的三个步骤：

1. 向量化（Embedding）
   文本 → "温暖的治愈系电影" → [0.12, 0.34, 0.56, ...]（768维）
   图片 → 猫咪图片 → [0.23, 0.45, 0.67, ...]（768维）

2. 存储向量
   将向量存入向量数据库
   同时存储原始数据作为 metadata

3. 相似性搜索
   查询："温馨感人的日本电影"
   查询向量：[0.11, 0.33, 0.55, ...]
   在数据库中找最近的 N 个向量
   返回对应的原始数据
```

### 1.3 为什么要用向量数据库？


## 二、Pinecone：云原生向量数据库

### 2.1 Pinecone 核心概念

```python
# Pinecone 是完全托管的向量数据库服务
# 优点：免运维、自动扩缩容、高可用
# 缺点：需要付费（但有免费 tier）

# 安装客户端
# pip install pinecone-client

import pinecone

# 1. 初始化 Pinecone
# 连接到 Pinecone 服务（需要 API Key）
pinecone.init(
    api_key="your-api-key",
    environment="us-west1"  # 区域
)

# 2. 创建索引（Index）
# 索引是存储向量的容器
pinecone.create_index(
    name="movie-recommender",      # 索引名称
    dimension=1536,                # 向量维度（OpenAI text-embedding-3 的维度）
    metric="cosine"                # 距离度量方式：cosine/euclidean/dotproduct
)

# 3. 连接到索引
index = pinecone.Index("movie-recommender")

# 4. 插入向量
# 每条数据包含：id、vector、metadata
vectors = [
    {
        "id": "movie_001",
        "values": [0.12, 0.34, 0.56, ...],  # 1536 维向量
        "metadata": {
            "title": "海蒂和爷爷",
            "genre": "剧情",
            "country": "德国",
            "tags": ["治愈", "温情", "自然"]
        }
    },
    {
        "id": "movie_002",
        "values": [0.23, 0.45, 0.67, ...],
        "metadata": {
            "title": "千与千寻",
            "genre": "动画",
            "country": "日本",
            "tags": ["奇幻", "成长", "温情"]
        }
    }
]

# 批量插入
index.upsert(vectors)

# 5. 查询向量
# 找到最相似的 5 部电影
query_vector = [0.11, 0.33, 0.55, ...]  # "温馨感人的日本电影" 的向量

results = index.query(
    vector=query_vector,
    top_k=5,
    include_metadata=True  # 返回 metadata
)

# results 结构：
# {'matches': [
#     {'id': 'movie_002', 'score': 0.95, 'metadata': {...}},
#     {'id': 'movie_001', 'score': 0.89, 'metadata': {...}},
#     ...
# ]}
```

### 2.2 Pinecone 的距离度量

```python
# Pinecone 支持三种距离度量方式：

# 1. 余弦相似度（Cosine Similarity）
# 最常用的度量方式，适合文本相似度
# 取值范围：-1 到 1（1 表示完全相似）
# 向量方向越接近，余弦相似度越高

# 适用场景：
# - 文本语义相似度
# - 推荐系统
# - 文档聚类

# 2. 欧氏距离（Euclidean / L2）
# 直线距离，类似我们理解的几何距离
# 取值范围：0 到 正无穷（0 表示完全相同）

# 适用场景：
# - 图像相似度
# - 特征匹配
# - 推荐系统

# 3. 点积（Dot Product / IP）
# 向量对应元素相乘后求和
# 取值范围：负无穷到正无穷

# 适用场景：
# - 需要考虑向量长度的场景
# - 神经网络产生的向量

# 选择建议：
# 文本任务 → 余弦相似度（最常用）
# 图像任务 → 欧氏距离或余弦相似度
# 神经网络特征 → 点积
```

### 2.3 Pinecone 的过滤查询

```python
# Pinecone 支持在向量检索时添加 metadata 过滤

# 场景：用户想找"温馨的日本电影"
# 使用向量搜索 + 过滤器

results = index.query(
    vector=query_vector,
    top_k=10,
    filter={  # 元数据过滤器
        "country": {"$eq": "日本"},      # 必须是日本
        "genre": {"$in": ["剧情", "动画"]}  # 剧情或动画
    },
    include_metadata=True
)

# 过滤操作符：
# $eq      → 等于
# $ne      → 不等于
# $in      → 在列表中
# $nin     → 不在列表中
# $gt/$gte → 大于/大于等于
# $lt/$lte → 小于/小于等于
# $and     → 逻辑与
# $or      → 逻辑或
# $not     → 逻辑非

# 复杂过滤示例
filter = {
    "$and": [
        {"country": {"$in": ["日本", "韩国"]}},  # 日本或韩国
        {"year": {"$gte": 2000}},                 # 2000 年后
        {"rating": {"$gte": 8.0}},                # 评分 >= 8.0
        {"tags": {"$contains": "治愈"}}            # 包含"治愈"标签
    ]
}
```

### 3.2 Milvus 的表结构设计

```python
# Milvus 使用 Collection（集合）来组织数据
# 类似关系型数据库的表

from pymilvus import connections, Collection, FieldSchema, CollectionSchema, DataType

# 1. 连接到 Milvus
connections.connect(
    alias="default",
    host="localhost",
    port="19530"
)

# 2. 定义 Collection 的 Schema
# 定义每个字段的类型

fields = [
    # 主键字段
    FieldSchema(name="movie_id", dtype=DataType.INT64, is_primary=True),

    # 向量字段（必填，指定维度）
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=1536),

    # 标量字段（可过滤）
    FieldSchema(name="title", dtype=DataType.VARCHAR, max_length=200),
    FieldSchema(name="genre", dtype=DataType.VARCHAR, max_length=50),
    FieldSchema(name="country", dtype=DataType.VARCHAR, max_length=50),
    FieldSchema(name="year", dtype=DataType.INT64),
    FieldSchema(name="rating", dtype=DataType.FLOAT),
    FieldSchema(name="tags", dtype=DataType.ARRAY, element_type=DataType.VARCHAR, max_length=100, max_capacity=10)
]

schema = CollectionSchema(
    fields=fields,
    description="电影向量数据库"
)

# 3. 创建 Collection
collection = Collection(name="movies", schema=schema)

# 4. 创建索引（加速查询）
index_params = {
    "index_type": "IVF_FLAT",      # 索引类型
    "metric_type": "L2",          # 距离度量：L2（欧氏距离）/ IP（点积）/ COSINE（余弦）
    "params": {"nlist": 128}      # 聚类数量
}

collection.create_index(
    field_name="embedding",      # 为向量字段创建索引
    index_params=index_params
)

# 5. 加载 Collection 到内存（查询前必须）
collection.load()
```

### 3.4 Milvus 的索引类型

```python
# Milvus 支持多种索引类型，适用于不同场景

# 1. FLAT（暴力搜索）
# 遍历所有向量计算距离，精度最高，性能最差
# 适用：小数据集（< 1万），需要 100% 精度

index_params = {"index_type": "FLAT", "metric_type": "L2"}

# 2. IVF_FLAT（倒排索引）
# 先聚类，查询时只搜索最近的几个聚类
# 精度和性能的平衡

index_params = {
    "index_type": "IVF_FLAT",
    "metric_type": "L2",
    "params": {"nlist": 128}  # 聚类数量
}

# 3. IVF_SQ8（量化索引）
# 对向量进行量化压缩，大幅减少内存占用
# 适用：内存受限的大规模数据

index_params = {
    "index_type": "IVF_SQ8",
    "metric_type": "L2",
    "params": {"nlist": 128}
}

# 4. HNSW（分层可导航小世界图）
# 基于图的索引，查询速度最快，但内存占用高
# 适用：要求低延迟的场景

index_params = {
    "index_type": "HNSW",
    "metric_type": "L2",
    "params": {
        "M": 16,      # 节点连接数，越大越精确但越慢
        "efConstruction": 200  # 构建时的搜索深度
    }
}

# 5. ANNOY（随机投影树）
# Google 开源的索引，适合磁盘存储
# 适用：无法放入内存的大规模数据

index_params = {
    "index_type": "ANNOY",
    "metric_type": "L2",
    "params": {"n_trees": 100}
}

# 选择建议：
# 小数据量（< 10万）→ FLAT 或 IVF_FLAT
# 中等数据量（10万~1000万）→ IVF_SQ8 或 HNSW
# 超大规模数据（> 1000万）→ HNSW 或分布式部署
```

## 四、向量索引原理：理解近似最近邻搜索

### 4.1 为什么要近似搜索？

```
精确搜索 vs 近似搜索：

精确搜索（KNN - K Nearest Neighbors）：
├── 遍历所有向量
├── 计算待查询向量与每个向量的距离
├── 排序返回最近的 K 个
└── 时间复杂度：O(N)，N 是向量总数

近似搜索（ANN - Approximate Nearest Neighbors）：
├── 索引结构加速搜索
├── 不保证返回精确的 K 个最近邻
├── 牺牲少量精度换取大幅性能提升
└── 时间复杂度：O(log N) 或更快

性能对比（100 万向量，维度 768）：
- KNN 精确搜索：10-60 秒
- ANN 近似搜索：10-50 毫秒
- 性能提升：100-6000 倍！
```

### 4.2 几种主流索引算法

```python
# 1. IVF（倒排文件索引）
# Inverted File Index

# 原理：
# 1. 对所有向量进行聚类（如 128 个聚类）
# 2. 每个聚类有一个中心点
# 3. 倒排表：记录每个向量属于哪个聚类
# 4. 查询时，先找最近的几个聚类，再在这些聚类中找最近向量

# 优点：实现简单，效果好
# 缺点：需要内存存储倒排表

# 图示：
#     聚类中心点
#         ●
#        /|\
#       / | \
#      ●  ●  ●
#     /|   |\  （向量只属于最近的中心点）
#    ● ●   ● ●

# 2. HNSW（分层可导航小世界图）
# Hierarchical Navigable Small World

# 原理：
# 1. 构建多层图，上层稀疏、下层密集
# 2. 查询时从上层开始，快速定位大致位置
# 3. 逐层下探直到最底层

# 优点：查询速度极快，精度高
# 缺点：内存占用大，构建时间长

# 图示：
#  第 1 层（稀疏）    ●───────●
#                     │       │
#  第 2 层（中等）   ●─●─────●─●
#                   /│\│     │/│\
#  第 3 层（密集）  ●●●●●●●●●●●●●
#                   ↑ 从上层快速定位
#                   ↓ 逐层下探

# 3. PQ（乘积量化）
# Product Quantization

# 原理：
# 1. 将高维向量分成多个子向量
# 2. 对每个子向量空间进行聚类（如 256 个）
# 3. 用聚类 ID 近似表示子向量
# 4. 查询时比较近似表示

# 优点：压缩率高，内存占用极低
# 缺点：精度损失较大

# 图示：
# 原始向量：[0.12, 0.34, 0.56, 0.78]（4维）
# 分成 2 组：[[0.12, 0.34], [0.56, 0.78]]
# 每组量化：每组用一个聚类 ID 表示（8 bits）
# 压缩后：2 个 ID，16 bits（原来是 32 bytes）
```

### 4.3 距离度量详解

```python
# 1. 欧氏距离（L2 Distance）
# 直线距离，最直观

import numpy as np

def euclidean_distance(v1, v2):
    """计算欧氏距离"""
    return np.sqrt(np.sum((v1 - v2) ** 2))

# 例子：
# v1 = [0, 0], v2 = [3, 4]
# 距离 = sqrt((3-0)^2 + (4-0)^2) = sqrt(9+16) = 5

# 2. 余弦相似度（Cosine Similarity）
# 方向相似度，取值 -1 到 1

def cosine_similarity(v1, v2):
    """计算余弦相似度"""
    dot_product = np.dot(v1, v2)
    norm_v1 = np.linalg.norm(v1)
    norm_v2 = np.linalg.norm(v2)
    return dot_product / (norm_v1 * norm_v2)

# 例子：
# v1 = [1, 0], v2 = [1, 0]  → 相似度 = 1（完全相同方向）
# v1 = [1, 0], v2 = [-1, 0] → 相似度 = -1（完全相反方向）
# v1 = [1, 0], v2 = [0, 1]  → 相似度 = 0（垂直）

# 3. 点积（Dot Product / IP）
# 向量对应元素相乘后求和

def dot_product(v1, v2):
    """计算点积"""
    return np.dot(v1, v2)

# 点积与余弦相似度的关系：
# cos_sim = dot_product(v1, v2) / (|v1| * |v2|)
# 如果向量已经归一化（|v1|=|v2|=1），则 cos_sim = dot_product

# 选择建议：
# 文本语义相似度 → 余弦相似度（最常用）
# 推荐系统中 → 余弦相似度或点积
# 图像特征匹配 → 欧氏距离
# 神经网络特征 → 点积或余弦相似度
```

### 5.3 多模态向量搜索

```python
# 多模态向量搜索：文本搜图片、图片搜图片、文本搜文本

# 使用 CLIP 实现多模态搜索

import torch
from PIL import Image
from transformers import CLIPProcessor, CLIPModel
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

class MultiModalSearcher:
    def __init__(self):
        # 加载模型
        self.model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
        self.processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")
        self.images = []  # 存储图片
        self.image_embeddings = []  # 存储图片向量

    def add_image(self, image_path, image_id):
        """添加图片到索引"""
        image = Image.open(image_path)
        inputs = self.processor(images=image, return_tensors="pt")
        with torch.no_grad():
            embedding = self.model.get_image_features(**inputs)
        self.images.append({"id": image_id, "path": image_path})
        self.image_embeddings.append(embedding[0].numpy())

    def text_search(self, query_text, top_k=5):
        """文本搜索图片"""
        # 将查询文本转为向量
        inputs = self.processor(text=[query_text], return_tensors="pt", padding=True)
        with torch.no_grad():
            query_embedding = self.model.get_text_features(**inputs)[0].numpy()

        # 计算相似度
        similarities = cosine_similarity([query_embedding], self.image_embeddings)[0]

        # 返回 top_k 结果
        top_indices = np.argsort(similarities)[-top_k:][::-1]
        results = []
        for idx in top_indices:
            results.append({
                "id": self.images[idx]["id"],
                "path": self.images[idx]["path"],
                "score": float(similarities[idx])
            })
        return results

    def image_search(self, query_image_path, top_k=5):
        """图片搜索图片"""
        # 将查询图片转为向量
        query_image = Image.open(query_image_path)
        inputs = self.processor(images=query_image, return_tensors="pt")
        with torch.no_grad():
            query_embedding = self.model.get_image_features(**inputs)[0].numpy()

        # 计算相似度
        similarities = cosine_similarity([query_embedding], self.image_embeddings)[0]

        # 返回 top_k 结果
        top_indices = np.argsort(similarities)[-top_k:][::-1]
        results = []
        for idx in top_indices:
            results.append({
                "id": self.images[idx]["id"],
                "path": self.images[idx]["path"],
                "score": float(similarities[idx])
            })
        return results
```

```python
from pydantic import BaseModel
from typing import List, Optional

class Document(BaseModel):
    """文档模型"""
    doc_id: str
    title: str
    content: str
    category: str
    tags: List[str]
    created_at: str

class SearchQuery(BaseModel):
    """搜索请求"""
    query: str
    top_k: int = 10
    category: Optional[str] = None
    tags: Optional[List[str]] = None

class SearchResult(BaseModel):
    """搜索结果"""
    doc_id: str
    title: str
    content: str
    category: str
    tags: List[str]
    score: float

# 3. 初始化向量化和数据库
from pymilvus import connections, Collection
from sentence_transformers import SentenceTransformer
import numpy as np

class VectorSearchEngine:
    def __init__(self):
        # 加载向量化模型
        self.model = SentenceTransformer('shibing624/text2vec-base-chinese')

        # 连接 Milvus
        connections.connect(host='localhost', port='19530', alias='default')

        # 获取 Collection
        self.collection = Collection('knowledge_base')
        self.collection.load()

    def encode_text(self, text: str) -> List[float]:
        """将文本转为向量"""
        embedding = self.model.encode(text)
        return embedding.tolist()

    def search(self, query: SearchQuery) -> List[SearchResult]:
        """执行搜索"""
        # 1. 将查询文本向量化
        query_vector = self.encode_text(query.query)

        # 2. 构建过滤表达式
        filter_expr = self._build_filter(query.category, query.tags)

        # 3. 执行向量搜索
        search_params = {"metric_type": "L2", "params": {"nprobe": 10}}

        results = self.collection.search(
            data=[query_vector],
            anns_field="embedding",
            param=search_params,
            limit=query.top_k,
            expr=filter_expr,
            output_fields=["doc_id", "title", "content", "category", "tags"]
        )

        # 4. 转换结果
        search_results = []
        for hit in results[0]:
            search_results.append(SearchResult(
                doc_id=hit.entity["doc_id"],
                title=hit.entity["title"],
                content=hit.entity["content"],
                category=hit.entity["category"],
                tags=hit.entity["tags"],
                score=hit.distance
            ))

        return search_results

    def _build_filter(self, category: str, tags: List[str]) -> Optional[str]:
        """构建过滤表达式"""
        conditions = []

        if category:
            conditions.append(f'category == "{category}"')

        if tags:
            for tag in tags:
                conditions.append(f'tags contains "{tag}"')

        if not conditions:
            return None

        return " and ".join(conditions)
```

### 6.4 混合搜索：向量 + 关键词

```python
# 混合搜索：结合向量相似度和 BM25 关键词评分

# 1. 安装关键词搜索库
# pip install rank_bm25

from rank_bm25 import BM25Okapi
import numpy as np

class HybridSearchEngine:
    def __init__(self):
        self.collection = Collection('knowledge_base')
        self.collection.load()

        # BM25 相关
        self.documents = []  # 存储文档内容
        self.bm25 = None     # BM25 索引

    def build_bm25_index(self, docs: List[Document]):
        """构建 BM25 索引"""
        # 分词（简单按空格分词，实际可用 jieba 等中文分词器）
        tokenized_docs = [doc.content.split() for doc in docs]
        self.bm25 = BM25Okapi(tokenized_docs)
        self.documents = docs

    def hybrid_search(self, query: str, top_k: int = 10, alpha: float = 0.7):
        """
        混合搜索
        alpha: 向量搜索权重（1-alpha 为 BM25 权重）
        """
        # 1. 向量搜索
        query_vector = self.encode_text(query)
        vector_results = self.vector_search(query_vector, top_k * 2)

        # 2. BM25 搜索
        bm25_scores = self.bm25.get_scores(query.split())
        bm25_results = sorted(
            range(len(bm25_scores)),
            key=lambda i: bm25_scores[i],
            reverse=True
        )[:top_k * 2]

        # 3. 融合评分
        fused_scores = {}
        for i, result in enumerate(vector_results):
            doc_id = result["doc_id"]
            # 向量得分（归一化到 0-1）
            vector_score = 1 / (1 + result["distance"])
            fused_scores[doc_id] = alpha * vector_score

        for i, doc_id in enumerate(bm25_results):
            doc_id = self.documents[doc_id].doc_id
            # BM25 得分（归一化）
            bm25_score = bm25_scores[i] / max(bm25_scores) if max(bm25_scores) > 0 else 0
            if doc_id in fused_scores:
                fused_scores[doc_id] += (1 - alpha) * bm25_score
            else:
                fused_scores[doc_id] = (1 - alpha) * bm25_score

        # 4. 排序返回
        sorted_results = sorted(
            fused_scores.items(),
            key=lambda x: x[1],
            reverse=True
        )[:top_k]

        return sorted_results
```

## 八、总结

```
向量数据库核心要点：

1. 向量是什么？
   - 一组有序的浮点数，代表高维空间中的一个点
   - 相似的对象在空间中距离更近

2. 为什么要用向量数据库？
   - 支持语义搜索（理解含义，不只是匹配关键词）
   - 支持多模态（图片、音频、视频搜索）
   - 搜索性能远超传统数据库

3. 主流向量数据库：
   - Pinecone：云服务，免运维
   - Milvus：开源，功能全面
   - Qdrant：高性能，Rust 实现
   - ChromaDB：轻量，适合入门

4. 索引算法：
   - IVF：倒排索引，精度和性能平衡
   - HNSW：图索引，查询最快
   - PQ：量化索引，压缩率高

5. 距离度量：
   - 余弦相似度：最常用，适合文本
   - 欧氏距离：几何距离，适合图像
   - 点积：适合神经网络特征

6. 向量化模型：
   - 英文：OpenAI Embeddings / Sentence-BERT
   - 中文：text2vec-base-chinese / m3e-base
```

> **未来展望：** 向量数据库正在成为 AI 时代的基础设施。随着多模态大模型的发展，向量数据库的应用场景会越来越广泛。想象一下，未来你可以用一段视频搜视频、用一段音乐搜音乐、用一句话搜任何相关内容。向量数据库正是实现这些应用的关键技术。
