# 代码执行工具设计实战

## 概述

代码执行工具是Agent系统中与外部世界交互的关键组件，它使Agent能够执行代码、运行命令、访问系统资源。本文档详细讨论沙箱环境设计、安全隔离机制和工具抽象标准化。

## 沙箱环境设计

### 1. 沙箱核心概念

沙箱是一种安全隔离环境，用于运行不受信任的代码，防止其对系统造成损害。

```
┌─────────────────────────────────────────────────────────┐
│                    主系统                                │
│  ┌─────────────────────────────────────────────────┐   │
│  │                  沙箱环境                          │   │
│  │  ┌─────────────────────────────────────────────┐ │   │
│  │  │              隔离层                          │ │   │
│  │  │  ┌─────────────────────────────────────┐    │ │   │
│  │  │  │           执行的代码                  │    │ │   │
│  │  │  └─────────────────────────────────────┘    │ │   │
│  │  └─────────────────────────────────────────────┘ │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 2. 沙箱实现技术

| 技术 | 原理 | 隔离级别 |
|------|------|---------|
| **Docker容器** | OS级虚拟化 | 高 |
| **gVisor** | 用户态内核 | 中高 |
| **WebAssembly** | 编译时隔离 | 高 |
| **V8 Isolate** | 进程隔离 | 中 |
| **Native Cgroups** | Linux命名空间 | 高 |

### 3. Docker沙箱实现

```typescript
// Docker沙箱执行器
import Docker from 'dockerode';

class DockerSandbox {
  private docker: Docker;
  private image: string;
  private networkEnabled: boolean;

  constructor(options: {
    image?: string;
    networkEnabled?: boolean;
  }) {
    this.docker = new Docker();
    this.image = options.image || 'code-runner: latest';
    this.networkEnabled = options.networkEnabled ?? false;
  }

  async execute(
    code: string,
    language: 'python' | 'javascript' | 'bash',
    options: {
      timeout?: number;
      memoryLimit?: string;
      cpuLimit?: number;
    } = {}
  ): Promise<ExecutionResult> {
    // 1. 准备代码文件
    const codeFile = this.prepareCodeFile(code, language);

    // 2. 构建容器配置
    const containerConfig = this.buildContainerConfig(
      codeFile,
      language,
      options
    );

    // 3. 创建并启动容器
    const container = await this.docker.createContainer(containerConfig);

    try {
      // 4. 等待执行完成
      const result = await this.waitForExecution(
        container,
        options.timeout || 30000
      );

      return result;
    } finally {
      // 5. 清理容器
      await container.remove({ force: true });
    }
  }

  private buildContainerConfig(
    codeFile: CodeFile,
    language: string,
    options: ExecutionOptions
  ): ContainerConfig {
    return {
      Image: this.image,
      Cmd: this.getExecutionCommand(language, codeFile.path),
      AttachStdout: true,
      AttachStderr: true,
      Tty: false,
      HostConfig: {
        // 资源限制
        Memory: parseMemory(options.memoryLimit || '256m'),
        NanoCpus: Math.floor((options.cpuLimit || 0.5) * 1e9),

        // 网络隔离
        NetworkMode: this.networkEnabled ? 'bridge' : 'none',

        // 文件系统只读（除了临时目录）
        ReadonlyRootfs: true,
        Binds: [`${codeFile.dir}:/tmp/code:rw`],

        // 禁用特权模式
        Privileged: false,

        // 自动清理
        AutoRemove: true
      },
      Env: [
        'PATH=/usr/local/bin:/usr/bin:/bin',
        'HOME=/tmp'
      ]
    };
  }
}
```

```typescript
// WebAssembly沙箱 - 用于安全的JavaScript执行
class WASMSandbox {
  private memory: WebAssembly.Memory;
  private exports: WebAssembly.Exports;

  async initialize(wasmBytes: ArrayBuffer): Promise<void> {
    this.memory = new WebAssembly.Memory({
      initial: 1,  // 1页 = 64KB
      maximum: 10  // 最大10页
    });

    const importObject = {
      env: {
        memory: this.memory,
        // 安全的环境函数
        print: (ptr: number, len: number) => {
          const buffer = new Uint8Array(this.memory.buffer, ptr, len);
          console.log(new TextDecoder().decode(buffer));
        }
      }
    };

    const result = await WebAssembly.instantiate(
      wasmBytes,
      importObject
    );

    this.exports = result.instance.exports;
  }

  async execute(code: string): Promise<string> {
    // 编译用户代码
    const compiled = this.compile(code);

    // 写入内存
    const codePtr = this.allocate(code.length);
    new Uint8Array(this.memory.buffer, codePtr, code.length)
      .set(new TextEncoder().encode(code));

    // 执行
    const mainFn = this.exports.main as (ptr: number, len: number) => number;
    const resultPtr = mainFunc(codePtr, code.length);

    // 读取结果
    return this.readString(resultPtr);
  }
}
```

## 安全隔离机制

### 1. 多层安全模型

```typescript
// 多层安全隔离
class SecurityIsolation {
  // 第一层：语法检查
  private syntaxCheck(code: string, language: string): void {
    const dangerous = this.detectDangerousPatterns(code);
    if (dangerous.length > 0) {
      throw new SecurityError(`危险语法: ${dangerous.join(', ')}`);
    }
  }

  // 第二层：权限检查
  private permissionCheck(
    requested: Permission[],
    allowed: Permission[]
  ): void {
    for (const req of requested) {
      if (!allowed.includes(req)) {
        throw new SecurityError(`权限不足: ${req}`);
      }
    }
  }

  // 第三层：运行时监控
  private setupRuntimeMonitor(
    container: Container
  ): RuntimeMonitor {
    return new RuntimeMonitor(container, {
      onMemorySpike: () => container.kill(),
      onCpuExceed: () => container.kill(),
      onNetworkViolation: () => container.disconnect(),
      onFileAccessViolation: () => container.kill()
    });
  }

  // 危险模式检测
  private detectDangerousPatterns(code: string): string[] {
    const patterns = [
      { regex: /import.*os/, danger: 'os模块导入' },
      { regex: /import.*sys/, danger: 'sys模块导入' },
      { regex: /subprocess/, danger: 'subprocess调用' },
      { regex: /eval\s*\(/, danger: 'eval使用' },
      { regex: /exec\s*\(/, danger: 'exec使用' },
      { regex: /open\s*\([^)]*["']\/etc/, danger: '尝试访问系统文件' },
      { regex: /__import__/, danger: '动态导入' },
      { regex: /ctypes/, danger: 'ctypes调用' },
      { regex: /ptrace/, danger: 'ptrace调用' }
    ];

    return patterns
      .filter(p => p.regex.test(code))
      .map(p => p.danger);
  }
}
```

### 2. 系统资源限制

```typescript
// 资源限制执行器
class ResourceLimitedExecutor {
  private limits: ResourceLimits = {
    maxMemoryMB: 512,
    maxCpuPercent: 50,
    maxExecutionTimeMs: 30000,
    maxFileSizeMB: 10,
    maxDiskUsageMB: 100,
    maxNetworkBandwidthMB: 50,
    maxProcesses: 10
  };

  async executeWithLimits(
    code: string,
    resources: Partial<ResourceLimits>
  ): Promise<ExecutionResult> {
    const effectiveLimits = { ...this.limits, ...resources };

    // 1. 检查代码复杂度
    const complexity = this.analyzeComplexity(code);
    if (complexity.loopDepth > 10) {
      throw new SecurityError('代码复杂度超限');
    }

    // 2. 设置cgroup限制
    await this.applyCgroupLimits(effectiveLimits);

    // 3. 执行代码
    try {
      const result = await this.execute(code);

      // 4. 验证资源使用
      this.validateResourceUsage(result.usage, effectiveLimits);

      return result;
    } finally {
      await this.cleanup();
    }
  }

  private async applyCgroupLimits(
    limits: ResourceLimits
  ): Promise<void> {
    // 内存限制
    await exec(`mkdir -p /sys/fs/cgroup/memory/agent`);
    await exec(`echo ${limits.maxMemoryMB * 1024 * 1024} > /sys/fs/cgroup/memory/agent/memory.limit_in_bytes`);

    // CPU限制
    await exec(`mkdir -p /sys/fs/cgroup/cpu/agent`);
    await exec(`echo ${limits.maxCpuPercent * 100} > /sys/fs/cgroup/cpu/agent/cpu.cfs_quota_us`);

    // 进程数限制
    await exec(`mkdir -p /sys/fs/cgroup/pids/agent`);
    await exec(`echo ${limits.maxProcesses} > /sys/fs/cgroup/pids/agent/pids.max`);
  }
}
```

### 3. 文件系统隔离

```typescript
// 文件系统隔离
class FilesystemIsolation {
  private allowedPaths: string[];
  private tempDir: string;

  constructor(projectRoot: string) {
    this.allowedPaths = [
      projectRoot,
      '/tmp'
    ];
    this.tempDir = `/tmp/agent-${generateId()}`;
  }

  // 路径安全检查
  checkPath(path: string): SecurityResult {
    const resolved = path.resolve(path);

    // 禁止路径遍历
    if (path.includes('..')) {
      return { allowed: false, reason: '路径遍历禁止' };
    }

    // 检查是否在允许目录内
    const isAllowed = this.allowedPaths.some(
      allowed => resolved.startsWith(allowed)
    );

    if (!isAllowed) {
      return { allowed: false, reason: `路径不在允许列表内: ${path}` };
    }

    return { allowed: true };
  }

  // 创建受限文件系统
  async createRestrictedFS(): Promise<OverlayFS> {
    const workDir = `${this.tempDir}/work`;
    const upperDir = `${this.tempDir}/upper`;
    const lowerDir = '/';  // 只读底层

    await exec(`mkdir -p ${workDir} ${upperDir}`);

    return new OverlayFS({
      type: 'overlay',
      options: {
        lowerDirs: lowerDir,
        upperDir: upperDir,
        workDir: workDir
      }
    });
  }

  // 监控文件访问
  setupFileMonitor(): FileMonitor {
    return new FileMonitor({
      allowedPaths: this.allowedPaths,
      onViolation: (path) => {
        throw new SecurityError(`禁止访问: ${path}`);
      }
    });
  }
}
```

### 4. 网络隔离

```typescript
// 网络访问控制
class NetworkIsolation {
  private allowedDomains: Set<string>;
  private allowedPorts: Set<number>;

  constructor(options: {
    allowedDomains?: string[];
    allowedPorts?: number[];
  }) {
    this.allowedDomains = new Set(options.allowedDomains || []);
    this.allowedPorts = new Set(options.allowedPorts || [80, 443]);
  }

  // 创建网络沙箱
  async createNetworkSandbox(): Promise<NetworkNamespace> {
    const ns = await NetworkNamespace.create();

    // 禁用网络（默认）
    await ns.setNetworkMode('none');

    return ns;
  }

  // 白名单网络访问
  async allowDomain(domain: string): Promise<void> {
    // 添加到白名单
    this.allowedDomains.add(domain);

    // 更新iptables规则
    await exec(`
      iptables -A OUTPUT -p tcp -d ${domain} -j ACCEPT
    `);
  }

  // HTTP请求拦截
  interceptHttpRequest(
    request: Request
  ): InterceptResult | null {
    try {
      const url = new URL(request.url);

      // 检查域名
      if (!this.allowedDomains.has(url.hostname)) {
        return {
          allowed: false,
          status: 403,
          body: 'Domain not allowed'
        };
      }

      // 检查端口
      if (!this.allowedPorts.has(url.port)) {
        return {
          allowed: false,
          status: 403,
          body: 'Port not allowed'
        };
      }

      return { allowed: true };
    } catch {
      return { allowed: false, status: 400, body: 'Invalid URL' };
    }
  }
}
```

## 工具抽象与标准化

### 1. 统一工具接口

```typescript
// 标准化工具接口
interface Tool {
  // 工具唯一标识
  name: string;

  // 工具描述（用于模型理解）
  description: string;

  // 参数Schema（JSON Schema格式）
  parameters: JSONSchema7;

  // 执行函数
  execute(params: unknown): Promise<ToolResult>;

  // 可选：验证函数
  validate?(params: unknown): ValidationResult;

  // 可选：权限需求
  requiredPermissions?: Permission[];
}

// 工具结果
interface ToolResult {
  success: boolean;
  content?: string;
  error?: string;
  metadata?: {
    executionTime: number;
    memoryUsed?: number;
    [key: string]: unknown;
  };
}

// Python执行工具
class PythonExecutor implements Tool {
  name = 'python';
  description = '执行Python代码，返回stdout输出';

  parameters = {
    type: 'object',
    properties: {
      code: {
        type: 'string',
        description: '要执行的Python代码'
      },
      timeout: {
        type: 'number',
        description: '超时时间(毫秒)',
        default: 30000
      }
    },
    required: ['code']
  };

  async execute(params: { code: string; timeout?: number }): Promise<ToolResult> {
    const start = Date.now();

    try {
      const result = await this.sandbox.execute(
        params.code,
        'python',
        { timeout: params.timeout || 30000 }
      );

      return {
        success: result.success,
        content: result.stdout + result.stderr,
        metadata: {
          executionTime: Date.now() - start,
          memoryUsed: result.memoryUsed
        }
      };
    } catch (error) {
      return {
        success: false,
        error: error.message,
        metadata: { executionTime: Date.now() - start }
      };
    }
  }
}
```

### 2. 工具注册与管理

```typescript
// 工具注册表
class ToolRegistry {
  private tools = new Map<string, Tool>();
  private middleware: ToolMiddleware[] = [];

  // 注册工具
  register(tool: Tool): void {
    // 验证工具接口
    this.validateTool(tool);

    this.tools.set(tool.name, tool);
  }

  // 批量注册
  registerAll(tools: Tool[]): void {
    for (const tool of tools) {
      this.register(tool);
    }
  }

  // 获取工具
  get(name: string): Tool | undefined {
    return this.tools.get(name);
  }

  // 获取所有工具描述（用于Prompt）
  getToolDescriptions(): ToolDescription[] {
    return Array.from(this.tools.values()).map(tool => ({
      name: tool.name,
      description: tool.description,
      parameters: tool.parameters
    }));
  }

  // 执行工具（带中间件）
  async execute(
    name: string,
    params: unknown
  ): Promise<ToolResult> {
    const tool = this.tools.get(name);
    if (!tool) {
      throw new Error(`Tool not found: ${name}`);
    }

    // 应用中间件链
    let result: ToolResult;
    for (const mw of this.middleware) {
      params = await mw.beforeExecute(name, params);
    }

    result = await tool.execute(params);

    for (const mw of this.middleware) {
      result = await mw.afterExecute(name, params, result);
    }

    return result;
  }

  private validateTool(tool: Tool): void {
    if (!tool.name) throw new Error('Tool must have a name');
    if (!tool.description) throw new Error('Tool must have a description');
    if (!tool.execute) throw new Error('Tool must have execute method');
  }
}

// 工具执行中间件
interface ToolMiddleware {
  beforeExecute(
    toolName: string,
    params: unknown
  ): Promise<unknown>;

  afterExecute(
    toolName: string,
    params: unknown,
    result: ToolResult
  ): Promise<ToolResult>;
}

// 日志中间件
class LoggingMiddleware implements ToolMiddleware {
  async beforeExecute(toolName: string, params: unknown): Promise<unknown> {
    console.log(`[Tool] Executing ${toolName} with params:`, params);
    return params;
  }

  async afterExecute(
    toolName: string,
    params: unknown,
    result: ToolResult
  ): Promise<ToolResult> {
    console.log(`[Tool] ${toolName} completed:`, result.success ? 'OK' : 'FAILED');
    return result;
  }
}

// 缓存中间件
class CacheMiddleware implements ToolMiddleware {
  private cache = new Map<string, { result: ToolResult; expiry: number }>();
  private ttl: number;

  constructor(ttlMs: number = 60000) {
    this.ttl = ttlMs;
  }

  async beforeExecute(toolName: string, params: unknown): Promise<unknown> {
    const key = this.getCacheKey(toolName, params);
    const cached = this.cache.get(key);

    if (cached && cached.expiry > Date.now()) {
      return { __cached: true, ...cached.result };
    }

    return params;
  }

  async afterExecute(
    toolName: string,
    params: unknown,
    result: ToolResult
  ): Promise<ToolResult> {
    const key = this.getCacheKey(toolName, params);
    this.cache.set(key, { result, expiry: Date.now() + this.ttl });
    return result;
  }

  private getCacheKey(toolName: string, params: unknown): string {
    return `${toolName}:${JSON.stringify(params)}`;
  }
}
```

```typescript
// 通用工具执行器
class ToolExecutor {
  private registry: ToolRegistry;
  private security: SecurityChecker;

  async executeFromLLM(
    llmResponse: LLMResponse
  ): Promise<ToolExecutionResult[]> {
    const results: ToolExecutionResult[] = [];

    for (const toolCall of llmResponse.toolCalls) {
      // 安全检查
      const securityCheck = await this.security.checkTool(
        toolCall.name,
        toolCall.params
      );

      if (!securityCheck.allowed) {
        results.push({
          toolCall,
          success: false,
          error: securityCheck.reason
        });
        continue;
      }

      try {
        const result = await this.registry.execute(
          toolCall.name,
          toolCall.params
        );

        results.push({
          toolCall,
          success: result.success,
          content: result.content,
          metadata: result.metadata
        });
      } catch (error) {
        results.push({
          toolCall,
          success: false,
          error: error.message
        });
      }
    }

    return results;
  }
}
```

## 总结

代码执行工具设计的核心要点：

1. **沙箱环境** - Docker、 WebAssembly等多层隔离技术
2. **安全隔离** - 资源限制、权限控制、危险模式检测
3. **工具抽象** - 统一Tool接口、注册表、中间件机制
4. **标准化** - JSON Schema参数、ToolResult统一格式

设计良好的代码执行工具是构建安全可靠Agent系统的基础。
