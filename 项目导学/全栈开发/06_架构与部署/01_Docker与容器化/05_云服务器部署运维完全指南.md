### 2.2 阿里云安全组配置

阿里云的安全组就像是服务器的"防火墙"，需要正确配置才能让服务正常访问：

```bash
# ============================================
# 阿里云安全组配置（通过控制台或CLI）
# ============================================

# 安全组规则示例
# ----------------------------------------

# 入口规则（控制入站流量）

# SSH（22端口）- 限制来源IP更安全
协议：TCP
端口：22
来源：你的IP（如 123.45.67.89/32）
策略：允许
备注：SSH管理

# HTTP（80端口）- 对所有开放
协议：TCP
端口：80
来源：0.0.0.0/0
策略：允许
备注：Web服务

# HTTPS（443端口）- 对所有开放
协议：TCP
端口：443
来源：0.0.0.0/0
策略：允许
备注：HTTPS服务

# 自定义端口（如Node.js的3000）
协议：TCP
端口：3000
来源：100.100.100.0/24  # 只允许内网访问
策略：允许
备注：后端API（不对外开放）

# 出口规则（控制出站流量）
# 通常默认允许所有出站

# ============================================
# 使用阿里云CLI配置安全组
# ============================================

# 安装阿里云CLI
# curl -fsSL https://get.aliyun.com/AliyunCLI.sh | sh

# 配置凭证
aliyun configure

# 添加安全组规则
aliyun ecs AuthorizeSecurityGroup \
  --RegionId cn-hangzhou \
  --SecurityGroupId sg-xxxxxxxxx \
  --IpProtocol tcp \
  --PortRange 22/22 \
  --SourceCidrIp 123.45.67.89/32 \
  --Policy accept

# 开放HTTP/HTTPS
aliyun ecs AuthorizeSecurityGroup \
  --RegionId cn-hangzhou \
  --SecurityGroupId sg-xxxxxxxxx \
  --IpProtocol tcp \
  --PortRange 80/80 \
  --SourceCidrIp 0.0.0.0/0 \
  --Policy accept

aliyun ecs AuthorizeSecurityGroup \
  --RegionId cn-hangzhou \
  --SecurityGroupId sg-xxxxxxxxx \
  --Protocol tcp \
  --PortRange 443/443 \
  --SourceCidrIp 0.0.0.0/0 \
  --Policy accept
```

### 2.3 阿里云服务器环境配置

```bash
# ============================================
# 阿里云服务器初始化配置
# ============================================

# 1. 连接服务器（使用SSH密钥登录更安全）
ssh -i ~/.ssh/your-key.pem root@your-server-ip

# 2. 创建新用户（不用root直接操作）
useradd -m -s /bin/bash deploy
usermod -aG sudo deploy
mkdir -p /home/deploy/.ssh
chmod 700 /home/deploy/.ssh

# 3. 配置SSH密钥登录（免密码+更安全）
# 在本机执行，将公钥复制到服务器
ssh-copy-id -i ~/.ssh/your-key.pub deploy@your-server-ip

# 4. 禁用密码登录和root登录（增强安全性）
cat >> /etc/ssh/sshd_config << 'EOF'

# 安全配置
PermitRootLogin no           # 禁止root登录
PasswordAuthentication no    # 禁止密码登录
PubkeyAuthentication yes     # 允许密钥登录
EOF

systemctl restart sshd

# 5. 更新系统
apt update && apt upgrade -y

# 6. 安装基础软件
apt install -y curl wget git vim htop net-tools unzip certbot

# 7. 安装Docker
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

# 把deploy用户加入docker组
usermod -aG docker deploy

# 8. 配置防火墙（Ubuntu使用ufw）
ufw allow 22/tcp     # SSH
ufw allow 80/tcp     # HTTP
ufw allow 443/tcp    # HTTPS
ufw enable
```

### 2.4 阿里云域名配置

```bash
# ============================================
# 阿里云域名解析配置
# ============================================

# 方法1：通过控制台配置（图形界面）
# ----------------------------------------
# 登录阿里云控制台 -> 域名与网站 -> 域名
# 选择你的域名 -> 基础配置 -> 域名解析

# 添加记录：
# ----------------------------------------
# 主机记录    记录类型    记录值              TTL
# ----------------------------------------
# @          A          你的服务器IP        600
# www        A          你的服务器IP        600
# api        CNAME      your-app.aliyuncs.com 600
# *          A          你的服务器IP        600  # 泛解析

# ============================================
# 方法2：使用阿里云CLI配置
# ============================================

# 安装aliyun-cli后配置域名解析
aliyun alidns AddDomainRecord \
  --DomainName example.com \
  --RR '@' \
  --Type A \
  --Value 123.45.67.89 \
  --TTL 600

# 添加www记录
aliyun alidns AddDomainRecord \
  --DomainName example.com \
  --RR 'www' \
  --Type A \
  --Value 123.45.67.89 \
  --TTL 600

# 添加API子域名
aliyun alidns AddDomainRecord \
  --DomainName example.com \
  --RR 'api' \
  --Type A \
  --Value 123.45.67.89 \
  --TTL 600

# ============================================
# SSL证书申请与安装
# ============================================

# 方法1：使用阿里云免费证书
# 控制台 -> 安全 -> SSL证书 -> 免费证书
# 按提示申请后下载Nginx版本

# 方法2：使用Let's Encrypt（certbot）
certbot --nginx -d example.com -d www.example.com

# 方法3：使用阿里云CLI自动申请
aliyun aliyungo_cert CreateCertificate \
  --DomainName example.com \
  --ChallengeType http-01
```

### 4.2 Vercel部署Next.js应用

```bash
# ============================================
# Vercel部署Next.js应用
# ============================================

# 方法1：使用Vercel CLI部署（最简单）
# ----------------------------------------

# 安装Vercel CLI
npm install -g vercel

# 登录Vercel
vercel login

# 进入项目目录
cd your-nextjs-project

# 部署到预览环境
vercel

# 交互式配置：
# - Set up and deploy? Yes
# - Which scope? 选择你的Team或个人账号
# - Link to existing project? No (新建) 或 Yes (已有)
# - Project name? your-app
# - Directory? ./
# - Override settings? No

# 部署到生产环境
vercel --prod

# 方法2：通过Git集成部署（推荐团队使用）
# ----------------------------------------
# 1. 把代码推送到GitHub/GitLab
# 2. 登录 Vercel Dashboard
# 3. 点击 "Import Project"
# 4. 选择你的Git仓库
# 5. 配置构建命令和输出目录
# 6. 点击 Deploy

# 方法3：使用GitHub Actions自动化部署
# ----------------------------------------
# .github/workflows/vercel-deploy.yml

name: Deploy to Vercel

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: '--prod'
```

// next.config.js
// Next.js配置文件

/** @type {import('next').NextConfig} */
const nextConfig = {
  // 生产环境关闭source map（安全）
  productionBrowserSourceMaps: false,

  // 图片优化配置
  images: {
    domains: ['example.com', 'cdn.example.com'],
    formats: ['image/avif', 'image/webp'],
  },

  // 环境变量（Vercel会自动注入process.env）
  env: {

  },

  // 重定向规则
  async redirects() {
    return [
      // 非www跳转到www
      {
        source: '/((?!www).*)',
        has: [{ type: 'host', value: 'example.com' }],
        destination: 'https://www.example.com/:1',
        permanent: true,
      },
    ];
  },

  // 响应头配置
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          { key: 'X-DNS-Prefetch-Control', value: 'on' },
          { key: 'Strict-Transport-Security', value: 'max-age=63072000; includeSubDomains; preload' },
        ],
      },
    ];
  },
};

## 八、生产环境安全配置

### 8.1 服务器安全加固

```bash
# ============================================
# 服务器安全加固脚本
# ============================================

#!/bin/bash
# 服务器安全加固脚本

set -e

echo "========== 服务器安全加固 =========="

# 1. 更新系统
apt update && apt upgrade -y

# 2. 安装必要的安全工具
apt install -y unattended-upgrades fail2ban ufw

# 3. 配置Fail2Ban（防止SSH暴力破解）
cat > /etc/fail2ban/jail.local << 'EOF'
[DEFAULT]
bantime = 3600           # 封禁1小时
findtime = 600           # 10分钟内
maxretry = 5             # 最多尝试5次

[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 86400          # 封禁24小时
EOF

systemctl enable fail2ban
systemctl start fail2ban

# 4. 配置UFW防火墙
ufw default deny incoming   # 拒绝所有入站
ufw default allow outgoing  # 允许所有出站

# 开放必要端口
ufw allow ssh               # SSH
ufw allow http              # HTTP
ufw allow https             # HTTPS

# 启用防火墙
echo "y" | ufw enable

# 5. SSH安全配置
cat >> /etc/ssh/sshd_config << 'EOF'

# SSH安全配置
PermitRootLogin no           # 禁止root登录
PasswordAuthentication no    # 禁止密码登录
PubkeyAuthentication yes      # 允许密钥登录
MaxAuthTries 3               # 最大尝试次数
ClientAliveInterval 300       # 5分钟无活动断开
ClientAliveCountMax 2
EOF

systemctl restart sshd

# 6. 设置系统参数
# 防止SYN洪水攻击
cat >> /etc/sysctl.conf << 'EOF'

# 防止SYN洪水攻击
net.ipv4.tcp_syncookies = 1
net.ipv4.tcp_syn_retries = 2
net.ipv4.tcp_synack_retries = 2
net.ipv4.tcp_max_syn_backlog = 4096

# 防止IP欺骗
net.ipv4.conf.all.rp_filter = 1
net.ipv4.conf.default.rp_filter = 1

# 禁止ICMP重定向
net.ipv4.conf.all.accept_redirects = 0
net.ipv4.conf.default.accept_redirects = 0
net.ipv6.conf.all.accept_redirects = 0
net.ipv6.conf.default.accept_redirects = 0

# 开启TCP Keepalive
net.ipv4.tcp_keepalive_time = 300
net.ipv4.tcp_keepalive_intvl = 30
net.ipv4.tcp_keepalive_probes = 5
EOF

# 应用配置
sysctl -p

echo "========== 安全加固完成 =========="
```
