## 1. 行为型模式概述

### 1.1 什么是行为型模式

行为型设计模式（Behavioral Design Patterns）是一类专注于对象之间通信和职责分配的设计模式。它们描述了对象之间的交互方式，帮助我们设计出更加灵活、可维护的系统架构。

### 1.2 行为型模式分类

```
行为型模式（11种）

通信类
├── 观察者模式（Observer）      - 一对多依赖关系
├── 中介者模式（Mediator）      - 集中化通信
├── 迭代器模式（Iterator）      - 顺序访问元素
└── 访问者模式（Visitor）       - 操作与数据结构分离

算法与流程控制类
├── 策略模式（Strategy）        - 算法家族
├── 命令模式（Command）         - 请求封装为对象
├── 模板方法模式（Template Method） - 算法骨架
├── 职责链模式（Chain of Responsibility） - 链式处理
└── 解释器模式（Interpreter）   - 语法解释

状态管理类
├── 状态模式（State）           - 状态机
└── 备忘录模式（Memento）       - 快照与恢复
```

### 1.3 设计原则在行为型模式中的应用

```
SOLID原则在行为型模式中的体现

单一职责原则（SRP）
└── 每个类只负责一项行为
    例：迭代器模式中，遍历逻辑与业务逻辑分离

开闭原则（OCP）
└── 对扩展开放，对修改关闭
    例：策略模式可以轻松添加新算法

里氏替换原则（LSP）
└── 子类可以替换父类
    例：观察者模式中，任何观察者实现都能替换

依赖倒置原则（DIP）
└── 依赖抽象，不依赖具体
    例：命令模式依赖抽象命令接口
```

## 2. 观察者模式（Observer）

### 2.1 生活类比：社交媒体关注系统

想象一下社交媒体平台上的"关注"功能：

- **微信公众号订阅**：用户关注公众号后，每当公众号发布新文章，所有关注者都会收到更新通知
- **YouTube频道订阅**：订阅频道后，新视频发布时自动通知所有订阅者
- **邮件订阅列表**：订阅 newsletters 后，新内容会自动推送到邮箱

**核心思想**：发布者（Subject）维护一个观察者列表，当状态变化时，自动通知所有观察者。

### 2.2 解决的问题

```
观察者模式解决的核心问题

问题场景：
1. 一个对象的状态变化需要通知其他多个对象
2. 希望建立对象之间的动态关系，无需提前编译
3. 需要实现广播通信机制

传统方案的问题：
- 硬编码：对象直接调用其他对象，造成紧耦合
- 维护困难：新增/删除观察者需要修改发布者代码
- 无法运行时订阅：无法动态添加观察者

观察者模式的优势：
- 松耦合：发布者和观察者之间解耦
- 动态订阅：可以随时添加/移除观察者
- 广播通信：一个事件可以通知多个对象
```

### 2.3 代码示例：完整的 EventEmitter 实现

```javascript
/**
 * 观察者模式 - 完整实现
 *
 * 设计原理：
 * 1. 使用 Map 存储事件类型到回调函数的映射
 * 2. 支持同步和异步事件触发
 * 3. 提供 once（单次订阅）功能
 * 4. 支持 wildcard 通配符事件
 */

class EventEmitter {
    constructor() {
        // 存储事件处理器，使用 Map 提高查询效率
        this.events = new Map();

        // 存储 once 事件的处理器（执行后自动移除）
        this.onceEvents = new Map();
    }

    /**
     * 订阅事件
     * @param {string} event - 事件名称
     * @param {Function} callback - 回调函数
     * @returns {Function} 取消订阅的函数
     */
    on(event, callback) {
        if (!this.events.has(event)) {
            this.events.set(event, []);
        }

        // 将回调函数添加到事件列表
        this.events.get(event).push(callback);

        // 返回取消订阅的函数（记忆模式）
        const unsubscribe = () => {
            this.off(event, callback);
        };

        return unsubscribe;
    }

    /**
     * 订阅事件，但只触发一次
     * @param {string} event - 事件名称
     * @param {Function} callback - 回调函数
     */
    once(event, callback) {
        if (!this.onceEvents.has(event)) {
            this.onceEvents.set(event, []);
        }

        // 创建包装函数，执行后自动取消订阅
        const onceWrapper = (...args) => {
            callback(...args);
            this.off(event, onceWrapper);
        };

        // 存储包装函数
        this.onceEvents.get(event).push(onceWrapper);

        // 同时添加到主事件列表
        this.on(event, onceWrapper);
    }

    /**
     * 触发事件
     * @param {string} event - 事件名称
     * @param {...any} args - 传递给回调的参数
     */
    emit(event, ...args) {
        // 获取事件处理器列表
        const handlers = this.events.get(event);

        if (handlers && handlers.length > 0) {
            // 同步执行所有处理器
            handlers.forEach(handler => {
                try {
                    handler(...args);
                } catch (error) {
                    // 捕获错误但不中断其他处理器
                    console.error(`事件 ${event} 处理出错:`, error);
                }
            });
        }

        // 处理 once 事件（触发后清除）
        const onceHandlers = this.onceEvents.get(event);
        if (onceHandlers && onceHandlers.length > 0) {
            onceHandlers.forEach(handler => {
                this.off(event, handler);
            });
            this.onceEvents.delete(event);
        }

        // 支持通配符事件（*匹配所有）
        if (event !== '*') {
            this.emit('*', event, ...args);
        }
    }

    /**
     * 取消订阅
     * @param {string} event - 事件名称
     * @param {Function} callback - 回调函数
     */
    off(event, callback) {
        const handlers = this.events.get(event);

        if (handlers) {
            // 过滤掉要移除的处理器
            const filteredHandlers = handlers.filter(h => h !== callback);
            this.events.set(event, filteredHandlers);
        }
    }

    /**
     * 订阅多个事件（all）
     * @param {string[]} events - 事件数组
     * @param {Function} callback - 回调函数
     */
    onAll(events, callback) {
        const unsubscribes = events.map(event => this.on(event, callback));
        return () => unsubscribes.forEach(unsub => unsub());
    }
}

// 使用示例：用户通知系统
const userNotifier = new EventEmitter();

// 用户登录事件
userNotifier.on('user:login', (user) => {
    console.log(`用户 ${user.name} 登录了`);
});

// 用户下单事件（带多个处理器）
userNotifier.on('user:order', (order) => {
    console.log(`收到订单: ${order.id}`);
});

userNotifier.on('user:order', (order) => {
    // 发送邮件通知
    console.log(`发送订单确认邮件到: ${order.email}`);
});

userNotifier.on('user:order', (order) => {
    // 更新统计数据
    console.log(`更新销售统计`);
});

// once 示例：首次访问欢迎
userNotifier.once('user:firstVisit', (user) => {
    console.log(`欢迎 ${user.name}！这是您首次访问。`);
});

// 触发事件
userNotifier.emit('user:login', { name: '张三', id: 1 });
userNotifier.emit('user:order', { id: 'ORD001', email: 'zhangsan@example.com' });
userNotifier.emit('user:firstVisit', { name: '李四' });
userNotifier.emit('user:firstVisit', { name: '王五' }); // 不会被触发
```

### 2.4 Promise 中的观察者模式

Promise 实际上是一种特殊的观察者模式实现，它观察异步操作的状态变化。

```javascript
/**
 * Promise 的观察者模式实现分析
 *
 * Promise 相当于一个 Subject（发布者）
 * .then() 注册的回调相当于 Observer（观察者）
 *
 * 设计原理：
 * 1. Promise 内部维护状态（pending/fulfilled/rejected）
 * 2. 当状态改变时，自动通知所有注册的回调
 * 3. 回调函数按注册顺序执行
 */

// 手写一个简易 Promise 来理解观察者模式
class MyPromise {
    constructor(executor) {
        // 初始状态为 pending
        this.state = 'pending';
        this.value = undefined;

        // 存储成功和失败的回调（观察者列表）
        this.onFulfilledCallbacks = [];
        this.onRejectedCallbacks = [];

        // 改变状态并通知所有观察者
        const resolve = (value) => {
            if (this.state === 'pending') {
                this.state = 'fulfilled';
                this.value = value;
                this.onFulfilledCallbacks.forEach(callback => callback(value));
            }
        };

        const reject = (reason) => {
            if (this.state === 'pending') {
                this.state = 'rejected';
                this.value = reason;
                this.onRejectedCallbacks.forEach(callback => callback(reason));
            }
        };

        try {
            executor(resolve, reject);
        } catch (error) {
            reject(error);
        }
    }

    /**
     * 注册观察者（类似 EventEmitter 的 on）
     */
    then(onFulfilled, onRejected) {
        return new MyPromise((resolve, reject) => {
            // 创建包装函数，处理返回值
            const handleFulfilled = (value) => {
                try {
                    const result = onFulfilled ? onFulfilled(value) : value;
                    resolve(result);
                } catch (error) {
                    reject(error);
                }
            };

            const handleRejected = (reason) => {
                try {
                    const result = onRejected ? onRejected(reason) : reason;
                    resolve(result);
                } catch (error) {
                    reject(error);
                }
            };

            // 根据状态决定立即执行或加入队列
            if (this.state === 'fulfilled') {
                setTimeout(() => handleFulfilled(this.value), 0);
            } else if (this.state === 'rejected') {
                setTimeout(() => handleRejected(this.value), 0);
            } else {
                // 状态为 pending，将观察者加入队列
                this.onFulfilledCallbacks.push(handleFulfilled);
                this.onRejectedCallbacks.push(handleRejected);
            }
        });
    }

    /**
     * 捕获错误（观察者的错误处理）
     */
    catch(onRejected) {
        return this.then(null, onRejected);
    }

    /**
     * 最终执行（无论成功失败都执行）
     */
    finally(callback) {
        return this.then(
            value => {
                callback();
                return value;
            },
            reason => {
                callback();
                throw reason;
            }
        );
    }
}

// 使用示例
new MyPromise((resolve, reject) => {
    setTimeout(() => resolve('数据加载成功'), 1000);
})
    .then(data => {
        console.log(data); // 1秒后输出：数据加载成功
        return '处理后的数据';
    })
    .then(data => {
        console.log(data); // 输出：处理后的数据
    })
    .catch(error => {
        console.error('发生错误:', error);
    });
```

### 2.5 React 中的观察者模式

React 的核心机制就是观察者模式的体现：

```javascript
/**
 * React 观察者模式分析
 *
 * 1. 组件订阅状态变化（useState 的观察者机制）
 * 2. 状态管理库（Redux/Zustand）的订阅机制
 * 3. 事件系统
 */

// 1. useState 的简易实现（观察者模式）
function useState(initialValue) {
    let value = initialValue;

    // 存储观察者（setter 函数）
    const setValue = (newValue) => {
        if (value !== newValue) {
            value = newValue;
            // 通知所有依赖这个状态的观察者重新渲染
            render();
        }
    };

    return [value, setValue];
}

// 2. Zustand 状态管理的订阅机制（完整观察者模式）
// 模拟 Zustand 的实现
function createStore(initialState) {
    // 内部状态
    let state = initialState;

    // 观察者列表（订阅者）
    const listeners = new Set();

    // 获取状态
    const getState = () => state;

    // 更新状态并通知观察者
    const setState = (partial) => {
        const nextState = partial instanceof Function
            ? partial(state)
            : partial;

        if (nextState !== state) {
            state = { ...state, ...nextState };
            // 通知所有订阅者
            listeners.forEach(listener => listener(state));
        }
    };

    // 订阅状态变化（观察者注册）
    const subscribe = (listener) => {
        listeners.add(listener);

        // 返回取消订阅的函数
        return () => {
            listeners.delete(listener);
        };
    };

    return { getState, setState, subscribe };
}

// 使用示例：创建文档状态管理
const useDocumentStore = createStore({
    documents: [],
    currentDoc: null,
    isLoading: false
});

// 组件订阅状态
function DocumentList() {
    const [documents, setDocuments] = useState([]);
    const [isLoading, setIsLoading] = useState(false);

    useEffect(() => {
        // 订阅 store 变化
        const unsubscribe = useDocumentStore.subscribe((state) => {
            setDocuments(state.documents);
            setIsLoading(state.isLoading);
        });

        // 初始化获取数据
        useDocumentStore.setState({ isLoading: true });
        fetchDocuments().then(data => {
            useDocumentStore.setState({
                documents: data,
                isLoading: false
            });
        });

        // 组件卸载时取消订阅
        return unsubscribe;
    }, []);

    return isLoading
        ? <div>加载中...</div>
        : <div>{documents.length} 个文档</div>;
}
```

### 2.6 Koa 中的观察者模式

Koa 的中间件系统本质上是观察者模式的应用：

```javascript
/**
 * Koa 中间件系统的观察者模式分析
 *
 * 核心思想：
 * 1. 中间件函数相当于观察者
 * 2. ctx 对象携带请求/响应状态
 * 3. next() 触发下一个中间件执行
 * 4. 洋葱模型体现了观察者的层层深入
 */

// Koa 应用的简易实现
class Koa {
    constructor() {
        // 中间件数组（观察者列表）
        this.middlewares = [];
    }

    /**
     * 注册中间件（订阅）
     */
    use(middleware) {
        this.middlewares.push(middleware);
        return this; // 支持链式调用
    }

    /**
     * 启动服务器（发布事件）
     */
    listen(port, callback) {
        // 组合中间件（核心！将多个函数组合成一个）
        const composed = this.compose(this.middlewares);

        // 返回请求处理函数
        return () => {
            const server = require('http').createServer(async (req, res) => {
                // 创建上下文对象（携带状态）
                const ctx = {
                    req,
                    res,
                    query: require('url').parse(req.url, true).query,
                    body: null,
                    status: 200
                };

                try {
                    // 执行中间件链
                    await composed(ctx);
                } catch (error) {
                    ctx.status = 500;
                    ctx.body = 'Internal Server Error';
                    console.error('服务器错误:', error);
                }

                res.statusCode = ctx.status;
                res.end(ctx.body);
            });

            return server.listen(port, callback);
        };
    }

    /**
     * 组合中间件 - 核心算法
     * 实现洋葱模型：外层中间件 -> 内层中间件 -> 内层中间件 -> 外层中间件
     */
    compose(middlewares) {
        return function (ctx) {
            let index = -1;

            // 递归执行中间件
            const dispatch = (i) => {
                if (i <= index) {
                    throw new Error('next() 被调用多次');
                }

                index = i;

                // 所有中间件执行完毕
                if (i >= middlewares.length) {
                    return Promise.resolve();
                }

                // 获取当前中间件
                const middleware = middlewares[i];

                // 执行中间件，传入 next 函数
                try {
                    return Promise.resolve(
                        middleware(ctx, () => dispatch(i + 1))
                    );
                } catch (error) {
                    return Promise.reject(error);
                }
            };

            return dispatch(0);
        };
    }
}

// 使用示例：创建 Koa 应用
const app = new Koa();

// 日志中间件（观察者1）
app.use(async (ctx, next) => {
    console.log(`[${new Date().toISOString()}] ${ctx.req.method} ${ctx.req.url}`);
    const start = Date.now();
    await next(); // 继续执行下一个中间件
    const ms = Date.now() - start;
    console.log(`处理耗时: ${ms}ms`);
});

// 认证中间件（观察者2）
app.use(async (ctx, next) => {
    if (ctx.query.token !== 'secret') {
        ctx.status = 401;
        ctx.body = '未授权';
        return;
    }
    await next();
});

// 业务逻辑中间件（观察者3）
app.use(async (ctx, next) => {
    if (ctx.req.url === '/api/users') {
        ctx.body = JSON.stringify([
            { id: 1, name: '张三' },
            { id: 2, name: '李四' }
        ]);
    }
    await next();
});

// 错误处理中间件（观察者4，最后执行）
app.use(async (ctx, next) => {
    try {
        await next();
    } catch (error) {
        ctx.status = 500;
        ctx.body = '服务器错误: ' + error.message;
    }
});

app.listen(3000, () => {
    console.log('服务器启动在 http://localhost:3000');
});
```

### 2.7 观察者模式在 Node.js 事件驱动架构中的应用

```javascript
/**
 * Node.js EventEmitter 完整实现
 *
 * Node.js 的核心就是事件驱动架构
 * EventEmitter 是所有事件驱动对象的基类
 */

// 完整的 EventEmitter 实现
const { EventEmitter } = require('events');

class DocumentEditor extends EventEmitter {
    constructor() {
        super();
        this.content = '';
        this.undoStack = [];
        this.redoStack = [];
    }

    /**
     * 编辑内容
     */
    write(text) {
        // 保存当前状态到历史记录
        this.undoStack.push(this.content);

        // 执行写入
        this.content += text;

        // 触发事件通知所有观察者
        this.emit('change', {
            type: 'write',
            text: text,
            content: this.content
        });

        // 写入操作后清空重做栈
        this.redoStack = [];
    }

    /**
     * 撤销操作
     */
    undo() {
        if (this.undoStack.length === 0) {
            return;
        }

        const previousContent = this.undoStack.pop();
        this.redoStack.push(this.content);
        this.content = previousContent;

        this.emit('change', {
            type: 'undo',
            content: this.content
        });
    }

    /**
     * 重做操作
     */
    redo() {
        if (this.redoStack.length === 0) {
            return;
        }

        const nextContent = this.redoStack.pop();
        this.undoStack.push(this.content);
        this.content = nextContent;

        this.emit('change', {
            type: 'redo',
            content: this.content
        });
    }
}

// 使用示例
const editor = new DocumentEditor();

// 观察者1：实时预览组件
const previewComponent = (data) => {
    console.log('[预览组件] 内容已更新:', data.content);
};

// 观察者2：自动保存组件
const autoSaveComponent = (data) => {
    console.log('[自动保存] 正在保存...');
    // 实际项目中会调用 API 保存
};

// 观察者3：字数统计组件
const wordCountComponent = (data) => {
    const count = data.content.length;
    console.log('[字数统计] 当前字数:', count);
};

// 注册观察者
editor.on('change', previewComponent);
editor.on('change', autoSaveComponent);
editor.on('change', wordCountComponent);

// 触发事件
editor.write('Hello ');
editor.write('World!');
editor.undo();
editor.redo();

// 移除观察者
editor.off('change', autoSaveComponent);
```

## 3. 策略模式（Strategy）

### 3.1 生活类比：出行方式选择

想象周末出游时选择交通方式：

- **步行**：适合短距离，环保，但速度慢
- **自行车**：适合中等距离，比步行快
- **公交车**：适合远距离，成本低，但可能不准时
- **出租车/滴滴**：随时叫车，速度快，但成本高
- **自己开车**：自由灵活，但需要找停车位

**核心思想**：根据不同情况选择不同的算法/策略，但最终目的相同（到达目的地）。

### 3.2 解决的问题

```
策略模式解决的核心问题

问题场景：
1. 需要在不同算法/行为之间切换
2. 使用大量条件语句选择算法
3. 希望在不修改主体代码的情况下扩展新算法

传统方案的问题：
- 大量 if-else 或 switch-case
- 新增算法需要修改原有代码
- 代码臃肿，难以维护

策略模式的优势：
- 算法可以自由切换
- 扩展性好，新增算法不影响现有代码
- 将算法与业务逻辑分离
```

```javascript
        this.paymentHistory = [];

     * 切换支付策略
     * @param {PaymentStrategy} strategy - 支付策略

            throw new Error('策略必须是 PaymentStrategy 的实例');
        }
        this.strategy = strategy;
    }

    /**
     * 执行支付
     * @param {number} amount - 金额
     * @param {object} params - 支付参数
     */
    async executePayment(amount, params = {}) {
        // 验证参数
        if (!this.strategy.validate(params)) {
            throw new Error(`${this.strategy.name} 参数验证失败`);
        }

        try {
            // 执行支付
            const result = await this.strategy.pay(amount);

            // 记录支付历史
            this.paymentHistory.push(result);

            return result;
        } catch (error) {
            console.error('支付失败:', error);
            return {
                success: false,
                error: error.message
            };
        }
    }

    /**
     * 获取支付历史
     */
    getHistory() {
        return [...this.paymentHistory];
    }
}
```

```javascript
/**
 * 策略模式 - 表单验证系统

 * 这个示例展示如何用策略模式实现灵活的数据验证

// ==================== 验证策略接口 ====================

class ValidationStrategy {
    validate(value, ...args) {
        throw new Error('子类必须实现 validate 方法');
    }

    getErrorMessage(fieldName) {
        throw new Error('子类必须实现 getErrorMessage 方法');
    }
}

// ==================== 具体验证策略 ====================

class RequiredValidation extends ValidationStrategy {
    validate(value) {
        return value !== null && value !== undefined && value !== '';
    }

    getErrorMessage(fieldName) {
        return `${fieldName} 是必填项`;
    }
}

class MinLengthValidation extends ValidationStrategy {
    constructor(minLength) {
        super();
        this.minLength = minLength;
    }

    validate(value) {
        return value && value.length >= this.minLength;
    }

    getErrorMessage(fieldName) {
        return `${fieldName} 长度不能少于 ${this.minLength} 个字符`;
    }
}

class MaxLengthValidation extends ValidationStrategy {
    constructor(maxLength) {
        super();
        this.maxLength = maxLength;
    }

    validate(value) {
        return !value || value.length <= this.maxLength;
    }

    getErrorMessage(fieldName) {
        return `${fieldName} 长度不能超过 ${this.maxLength} 个字符`;
    }
}

class EmailValidation extends ValidationStrategy {
    validate(value) {
        if (!value) return false;
        const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
        return emailRegex.test(value);
    }

    getErrorMessage(fieldName) {
        return `${fieldName} 格式不正确`;
    }
}

class PatternValidation extends ValidationStrategy {
    constructor(regex, errorMessage) {
        super();
        this.regex = regex;
        this.customErrorMessage = errorMessage;
    }

    validate(value) {
        if (!value) return true;
        return this.regex.test(value);
    }

    getErrorMessage(fieldName) {
        return this.customErrorMessage || `${fieldName} 格式不正确`;
    }
}

class RangeValidation extends ValidationStrategy {
    constructor(min, max) {
        super();
        this.min = min;
        this.max = max;
    }

    validate(value) {
        if (value === null || value === undefined || value === '') return true;
        const num = Number(value);
        return num >= this.min && num <= this.max;
    }

    getErrorMessage(fieldName) {
        return `${fieldName} 必须在 ${this.min} 到 ${this.max} 之间`;
    }
}

// ==================== 验证器上下文 ====================

class Validator {
    constructor() {
        this.fieldRules = new Map();
    }

    /**
     * 添加字段验证规则
     */
    addRule(fieldName, strategy) {
        if (!this.fieldRules.has(fieldName)) {
            this.fieldRules.set(fieldName, []);
        }
        this.fieldRules.get(fieldName).push(strategy);
        return this; // 支持链式调用
    }

    /**
     * 验证所有字段
     */
    validate(data) {
        const errors = [];

        this.fieldRules.forEach((rules, fieldName) => {
            const value = data[fieldName];

            rules.forEach(rule => {
                if (!rule.validate(value)) {
                    errors.push({
                        field: fieldName,
                        message: rule.getErrorMessage(fieldName)
                    });
                }
            });
        });

        return {
            valid: errors.length === 0,
            errors: errors
        };
    }
}

// ==================== 使用示例 ====================

// 创建验证器实例
const validator = new Validator();

// 配置验证规则
validator
    .addRule('username', new RequiredValidation())
    .addRule('username', new MinLengthValidation(3))
    .addRule('username', new MaxLengthValidation(20))
    .addRule('email', new RequiredValidation())
    .addRule('email', new EmailValidation())
    .addRule('password', new RequiredValidation())
    .addRule('password', new MinLengthValidation(8))
    .addRule('password', new PatternValidation(/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)/, '密码必须包含大小写字母和数字'))
    .addRule('age', new RangeValidation(0, 150))
    .addRule('phone', new PatternValidation(/^1[3-9]\d{9}$/, '请输入正确的手机号'));

// 测试数据
const testData = {
    username: 'john',
    email: 'john@example.com',
    password: 'Password123',
    age: 25,
    phone: '13800138000'
};

// 执行验证
const result = validator.validate(testData);

console.log('验证结果:', result);
```

### 3.5 排序策略的实现

```javascript
/**
 * 策略模式 - 排序算法系统
 *
 * 不同数据类型适合不同的排序算法
 */

class SortStrategy {
    sort(array) {
        throw new Error('子类必须实现 sort 方法');
    }

    getName() {
        throw new Error('子类必须实现 getName 方法');
    }
}

class BubbleSort extends SortStrategy {
    sort(array) {
        const arr = [...array];
        const n = arr.length;

        for (let i = 0; i < n - 1; i++) {
            for (let j = 0; j < n - i - 1; j++) {
                if (arr[j] > arr[j + 1]) {
                    [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
                }
            }
        }

        return arr;
    }

    getName() {
        return '冒泡排序';
    }
}

class QuickSort extends SortStrategy {
    sort(array) {
        if (array.length <= 1) return array;

        const arr = [...array];
        const pivot = arr[Math.floor(arr.length / 2)];
        const left = arr.filter(x => x < pivot);
        const middle = arr.filter(x => x === pivot);
        const right = arr.filter(x => x > pivot);

        return [...this.sort(left), ...middle, ...this.sort(right)];
    }

    getName() {
        return '快速排序';
    }
}

class MergeSort extends SortStrategy {
    sort(array) {
        if (array.length <= 1) return array;

        const mid = Math.floor(array.length / 2);
        const left = this.sort(array.slice(0, mid));
        const right = this.sort(array.slice(mid));

        return this.merge(left, right);
    }

    merge(left, right) {
        const result = [];
        let i = 0, j = 0;

        while (i < left.length && j < right.length) {
            if (left[i] <= right[j]) {
                result.push(left[i++]);
            } else {
                result.push(right[j++]);
            }
        }

        return [...result, ...left.slice(i), ...right.slice(j)];
    }

    getName() {
        return '归并排序';
    }
}

class Sorter {
    constructor(strategy) {
        this.setStrategy(strategy);
    }

    setStrategy(strategy) {
        if (!(strategy instanceof SortStrategy)) {
            throw new Error('必须是 SortStrategy 的实例');
        }
        this.strategy = strategy;
    }

    sort(array) {
        console.log(`使用 ${this.strategy.getName()} 排序`);
        return this.strategy.sort(array);
    }
}

// 使用示例
const numbers = [64, 34, 25, 12, 22, 11, 90];

const sorter = new Sorter(new BubbleSort());
console.log('原始数组:', numbers);
console.log('排序结果:', sorter.sort(numbers));

sorter.setStrategy(new QuickSort());
console.log('排序结果:', sorter.sort(numbers));
```

## 4. 命令模式（Command）

### 4.1 生活类比：电视遥控器

电视遥控器是命令模式的经典例子：

- **遥控器按钮**：每个按钮（如开/关、音量+/-、频道+/-）都是一个命令
- **电视**：接收命令并执行相应操作
- **遥控器存储命令**：不需要知道电视如何实现，只需发送命令

**另一个例子**：餐厅点餐系统

- 将命令写入订单单
- 厨师根据订单执行烹饪

### 4.2 解决的问题

```
命令模式解决的核心问题

问题场景：
1. 需要将请求封装为对象
2. 需要支持撤销/重做操作
3. 需要记录操作日志
4. 需要延迟执行命令

传统方案的问题：
- 操作与逻辑紧耦合
- 无法撤销/重做
- 难以记录操作历史

命令模式的优势：
- 解耦请求发送者和接收者
- 支持撤销/重做
- 支持命令队列和延迟执行
```

### 4.3 代码示例：文档编辑器

```javascript
/**
 * 命令模式 - 文档编辑器实现
 *
 * 设计原理：
 * 1. 命令封装了操作（execute）和撤销（undo）
 * 2. 调用者（Invoker）触发命令执行
 * 3. 接收者（Receiver）执行实际操作
 * 4. 支持命令队列、日志记录、撤销重做
 */

// ==================== 命令接口 ====================

/**
 * 命令接口
 * 所有命令必须实现 execute 和 undo 方法
 */
class Command {
    execute() {
        throw new Error('子类必须实现 execute 方法');
    }

    undo() {
        throw new Error('子类必须实现 undo 方法');
    }

    /**
     * 获取命令描述
     */
    getDescription() {
        return '未命名的命令';
    }
}

// ==================== 接收者 ====================

/**
 * 文档接收者
 * 实际执行文档操作
 */
class Document {
    constructor() {
        this.content = '';
        this.clipboard = '';
    }

    insertText(position, text) {
        this.content =
            this.content.slice(0, position) +
            text +
            this.content.slice(position);
    }

    deleteText(position, count) {
        const deleted = this.content.slice(position, position + count);
        this.content =
            this.content.slice(0, position) +
            this.content.slice(position + count);
        return deleted;
    }

    getText() {
        return this.content;
    }

    setText(text) {
        this.content = text;
    }
}

// ==================== 具体命令 ====================

/**
 * 插入文本命令
 */
class InsertTextCommand extends Command {
    constructor(document, position, text) {
        super();
        this.document = document;
        this.position = position;
        this.text = text;
    }

    execute() {
        this.document.insertText(this.position, this.text);
    }

    undo() {
        this.document.deleteText(this.position, this.text.length);
    }

    getDescription() {
        return `插入文本: "${this.text}"`;
    }
}

/**
 * 删除文本命令
 */
class DeleteTextCommand extends Command {
    constructor(document, position, count) {
        super();
        this.document = document;
        this.position = position;
        this.count = count;
        this.deletedText = '';
    }

    execute() {
        this.deletedText = this.document.deleteText(this.position, this.count);
    }

    undo() {
        this.document.insertText(this.position, this.deletedText);
    }

    getDescription() {
        return `删除文本: "${this.deletedText}"`;
    }
}

/**
 * 替换文本命令
 */
class ReplaceTextCommand extends Command {
    constructor(document, position, count, newText) {
        super();
        this.document = document;
        this.position = position;
        this.count = count;
        this.newText = newText;
        this.oldText = '';
    }

    execute() {
        this.oldText = this.document.deleteText(this.position, this.count);
        this.document.insertText(this.position, this.newText);
    }

    undo() {
        this.document.deleteText(this.position, this.newText.length);
        this.document.insertText(this.position, this.oldText);
    }

    getDescription() {
        return `替换: "${this.oldText}" -> "${this.newText}"`;
    }
}

/**
 * 格式化命令
 */
class FormatCommand extends Command {
    constructor(document, formatType, startPos, endPos) {
        super();
        this.document = document;
        this.formatType = formatType;
        this.startPos = startPos;
        this.endPos = endPos;
        this.originalContent = '';
    }

    execute() {
        this.originalContent = this.document.getText();
        const content = this.document.getText();
        const before = content.slice(0, this.startPos);
        const selected = content.slice(this.startPos, this.endPos);
        const after = content.slice(this.endPos);

        // 添加格式标记（简化处理）
        this.document.setText(`${before}[${this.formatType}:${selected}]${after}`);
    }

    undo() {
        this.document.setText(this.originalContent);
    }

    getDescription() {
        return `格式化: ${this.formatType}`;
    }
}

// ==================== 调用者（命令管理器）====================

/**
 * 命令管理器
 * 负责命令的执行、撤销、重做
 */
class CommandManager {
    constructor() {
        this.history = [];      // 执行历史
        this.redoStack = [];    // 重做栈
        this.executedCommands = [];
    }

    /**
     * 执行命令
     */
    executeCommand(command) {
        command.execute();
        this.history.push(command);
        this.executedCommands.push(command);

        // 清空重做栈（执行新命令后不能重做）
        this.redoStack = [];

        console.log(`[执行] ${command.getDescription()}`);
        console.log(`[文档内容] "${command.document.getText()}"`);
    }

    /**
     * 撤销上一次命令
     */
    undo() {
        if (this.history.length === 0) {
            console.log('没有可撤销的命令');
            return;
        }

        const command = this.history.pop();
        command.undo();
        this.redoStack.push(command);

        console.log(`[撤销] ${command.getDescription()}`);
        console.log(`[文档内容] "${command.document.getText()}"`);
    }

    /**
     * 重做上一次撤销的命令
     */
    redo() {
        if (this.redoStack.length === 0) {
            console.log('没有可重做的命令');
            return;
        }

        const command = this.redoStack.pop();
        command.execute();
        this.history.push(command);

        console.log(`[重做] ${command.getDescription()}`);
        console.log(`[文档内容] "${command.document.getText()}"`);
    }

    /**
     * 批量执行命令（宏命令）
     */
    executeMacro(commands) {
        commands.forEach(command => this.executeCommand(command));
    }

    /**
     * 获取执行历史
     */
    getHistory() {
        return this.history.map((cmd, index) =>
            `${index + 1}. ${cmd.getDescription()}`
        ).join('\n');
    }
}

// ==================== 使用示例 ====================

function main() {
    console.log('========== 命令模式 - 文档编辑器示例 ==========\n');

    // 创建文档和命令管理器
    const document = new Document();
    const commandManager = new CommandManager();

    // 执行一系列命令
    commandManager.executeCommand(
        new InsertTextCommand(document, 0, 'Hello')
    );

    commandManager.executeCommand(
        new InsertTextCommand(document, 5, ' World')
    );

    commandManager.executeCommand(
        new InsertTextCommand(document, 11, '!')
    );

    console.log('\n-------- 当前文档 --------');
    console.log(document.getText());

    // 撤销操作
    console.log('\n-------- 撤销操作 --------');
    commandManager.undo();
    commandManager.undo();

    console.log('\n-------- 重做操作 --------');
    commandManager.redo();

    console.log('\n-------- 执行宏命令 --------');
    // 宏命令：同时执行多个操作
    commandManager.executeMacro([
        new InsertTextCommand(document, 0, 'Welcome to '),
        new InsertTextCommand(document, 0, '[Title:'),
        new InsertTextCommand(document, 0, ']'),
    ]);

    console.log('\n-------- 最终文档 --------');
    console.log(document.getText());

    console.log('\n-------- 执行历史 --------');
    console.log(commandManager.getHistory());
}

main();
```

```javascript
class DownloadCommand extends AsyncCommand {
    constructor(url, savePath) {
        super(
            async () => {
                console.log(`[下载] 开始下载: ${url}`);
                // 模拟下载过程
                await new Promise(resolve => setTimeout(resolve, 1000));
                console.log(`[下载] 保存到: ${savePath}`);
                this.completed = true;
            },

            }
        );
        this.url = url;
        this.savePath = savePath;
    }

class TaskQueue {
    constructor() {
        this.pendingTasks = [];
        this.completedTasks = [];
        this.isRunning = false;
    }

    addTask(command) {
        this.pendingTasks.push(command);
        console.log(`[队列] 添加任务，当前待执行: ${this.pendingTasks.length}`);
    }

    async run() {
        if (this.isRunning) return;
        this.isRunning = true;

        while (this.pendingTasks.length > 0) {
            const task = this.pendingTasks.shift();
            await task.execute();
            this.completedTasks.push(task);
        }

        this.isRunning = false;
    }

    async undoLast() {
        if (this.completedTasks.length === 0) return;

        const lastTask = this.completedTasks.pop();
        await lastTask.undo();
        console.log('[队列] 已撤销最后一个任务');
    }
}

// 使用示例
const queue = new TaskQueue();

queue.addTask(new DownloadCommand('https://example.com/file2.zip', '/downloads/file2.zip'));
queue.run();
```

## 5. 迭代器模式（Iterator）

### 5.1 生活类比：图书馆图书遍历

想象在图书馆找书：

- **书架遍历**：从第一排开始，一本一本往后找
- **按类别找**：先分类，再在每个类别中遍历
- **按作者找**：先按作者分组，再遍历
- **搜索**：按条件过滤后遍历

**核心思想**：提供一种方法顺序访问集合中的元素，而不暴露集合的内部结构。

### 5.2 解决的问题

```
迭代器模式解决的核心问题

问题场景：
1. 需要遍历复杂数据结构
2. 希望统一遍历接口
3. 需要支持多种遍历方式

传统方案的问题：
- 直接暴露集合内部结构
- 遍历逻辑与业务逻辑耦合
- 难以支持多种遍历方式

迭代器模式的优势：
- 封装遍历逻辑
- 支持多种迭代方式
- 解耦集合与遍历算法
```

### 5.3 代码示例

```javascript
/**
 * 迭代器模式 - 实现自定义迭代器
 *
 * 设计原理：
 * 1. Iterator 提供 next() 方法获取下一个元素
 * 2. Iterable 对象实现 [Symbol.iterator]() 方法返回迭代器
 * 3. 支持同步和异步迭代
 */

// ==================== 基础迭代器 ====================

/**
 * 数组迭代器
 */
class ArrayIterator {
    constructor(collection) {
        this.collection = collection;
        this.index = 0;
    }

    hasNext() {
        return this.index < this.collection.length;
    }

    next() {
        if (!this.hasNext()) {
            return { done: true, value: undefined };
        }
        return { done: false, value: this.collection[this.index++] };
    }

    reset() {
        this.index = 0;
    }
}

/**
 * 链表节点
 */
class ListNode {
    constructor(value) {
        this.value = value;
        this.next = null;
    }
}

/**
 * 链表迭代器
 */
class LinkedListIterator {
    constructor(head) {
        this.current = head;
    }

    hasNext() {
        return this.current !== null;
    }

    next() {
        if (!this.hasNext()) {
            return { done: true, value: undefined };
        }
        const value = this.current.value;
        this.current = this.current.next;
        return { done: false, value: value };
    }
}

// ==================== 可迭代集合 ====================

/**
 * 树节点
 */
class TreeNode {
    constructor(value) {
        this.value = value;
        this.left = null;
        this.right = null;
    }
}

/**
 * 二叉树 - 支持前序、中序、后序遍历
 */
class BinaryTree {
    constructor(root = null) {
        this.root = root;
    }

    /**
     * 前序遍历迭代器（根-左-右）
     */
    *[Symbol.iterator]() {
        yield* this._preorder(this.root);
    }

    *_preorder(node) {
        if (node === null) return;
        yield node.value;
        yield* this._preorder(node.left);
        yield* this._preorder(node.right);
    }

    /**
     * 中序遍历迭代器（左-根-右）
     */
    *inOrder() {
        yield* this._inOrder(this.root);
    }

    *_inOrder(node) {
        if (node === null) return;
        yield* this._inOrder(node.left);
        yield node.value;
        yield* this._inOrder(node.right);
    }

    /**
     * 后序遍历迭代器（左-右-根）
     */
    *postOrder() {
        yield* this._postOrder(this.root);
    }

    *_postOrder(node) {
        if (node === null) return;
        yield* this._postOrder(node.left);
        yield* this._postOrder(node.right);
        yield node.value;
    }

    /**
     * 层序遍历
     */
    *levelOrder() {
        if (this.root === null) return;

        const queue = [this.root];

        while (queue.length > 0) {
            const node = queue.shift();
            yield node.value;

            if (node.left) queue.push(node.left);
            if (node.right) queue.push(node.right);
        }
    }
}

// ==================== 生成器实现迭代器 ====================

/**
 * 斐波那契数列迭代器
 */
function* fibonacciIterator(count) {
    let current = 0;
    let next = 1;

    for (let i = 0; i < count; i++) {
        yield current;
        [current, next] = [next, current + next];
    }
}

/**
 * range 迭代器（类似 Python）
 */
function* rangeIterator(start, end, step = 1) {
    for (let i = start; i < end; i += step) {
        yield i;
    }
}

/**
 * 组合迭代器
 */
function* zipIterator(...iterables) {
    const iterators = iterables.map(it => it[Symbol.iterator]());

    while (true) {
        const values = iterators.map(it => it.next());

        if (values.some(v => v.done)) break;

        yield values.map(v => v.value);
    }
}

// ==================== 使用示例 ====================

function main() {
    console.log('========== 迭代器模式示例 ==========\n');

    // 示例1：二叉树遍历
    console.log('-------- 二叉树遍历 --------');

    // 构建二叉树
    //       1
    //      / \
    //     2   3
    //    / \   \
    //   4   5   6
    const tree = new BinaryTree(
        new TreeNode(1)
    );
    tree.root.left = new TreeNode(2);
    tree.root.right = new TreeNode(3);
    tree.root.left.left = new TreeNode(4);
    tree.root.left.right = new TreeNode(5);
    tree.root.right.right = new TreeNode(6);

    console.log('前序遍历 (根-左-右):', [...tree]);
    console.log('中序遍历 (左-根-右):', [...tree.inOrder()]);
    console.log('后序遍历 (左-右-根):', [...tree.postOrder()]);
    console.log('层序遍历:', [...tree.levelOrder()]);

    // 示例2：斐波那契数列
    console.log('\n-------- 斐波那契数列 --------');
    const fibIterator = fibonacciIterator(10);
    console.log('前10个斐波那契数:', [...fibIterator]);

    // 示例3：range 迭代器
    console.log('\n-------- Range 迭代器 --------');
    console.log('1-10 偶数:', [...rangeIterator(0, 10, 2)]);

    // 示例4：组合迭代器
    console.log('\n-------- 组合迭代器 --------');
    const zipped = zipIterator([1, 2, 3], ['a', 'b', 'c'], [true, false, true]);
    console.log('组合结果:', [...zipped]);
}

main();
```

## 6. 模板方法模式（Template Method）

### 6.1 生活类比：泡茶和泡咖啡

泡茶和泡咖啡的步骤非常相似：

```
泡茶：
1. 烧开水
2. 泡茶叶
3. 倒入杯中
4. 添加调料（可选）

泡咖啡：
1. 烧开水
2. 冲泡咖啡
3. 倒入杯中
4. 添加糖和奶（可选）
```

**共同点**：步骤 1、3 相同，步骤 2、4 不同

**核心思想**：定义算法骨架，将某些步骤的实现延迟到子类。

### 6.2 解决的问题

```
模板方法模式解决的核心问题

问题场景：
1. 多个类有相似的算法结构
2. 需要统一算法框架，同时允许个性化实现
3. 需要代码复用，减少重复

传统方案的问题：
- 代码重复
- 修改困难（改动需要修改多个类）

模板方法模式的优势：
- 代码复用
- 框架统一
- 扩展性好（新增步骤不影响其他类）
```

### 6.3 代码示例

```javascript
/**
 * 模板方法模式 - 数据处理框架
 *
 * 设计原理：
 * 1. 抽象基类定义算法骨架（模板方法）
 * 2. 子类实现具体步骤
 * 3. 钩子方法允许子类覆盖默认行为
 */

// ==================== 抽象基类 ====================

/**
 * 数据处理模板
 */
class DataProcessor {
    /**
     * 模板方法 - 定义算法骨架
     * 这个方法通常不允许重写
     */
    process(data) {
        // 1. 验证数据
        const validatedData = this.validate(data);
        if (!validatedData) {
            throw new Error('数据验证失败');
        }

        // 2. 预处理
        const preprocessedData = this.preprocess(validatedData);

        // 3. 核心处理（由子类实现）
        const processedData = this.doProcess(preprocessedData);

        // 4. 后处理
        const postprocessedData = this.postprocess(processedData);

        // 5. 格式化输出
        const output = this.format(postprocessedData);

        // 6. 发送通知（钩子）
        this.onComplete(output);

        return output;
    }

    /**
     * 验证数据（默认实现）
     */
    validate(data) {
        return data !== null && data !== undefined;
    }

    /**
     * 预处理
     */
    preprocess(data) {
        console.log(`[${this.constructor.name}] 预处理数据`);
        return data;
    }

    /**
     * 核心处理 - 子类必须实现
     */
    doProcess(data) {
        throw new Error('子类必须实现 doProcess 方法');
    }

    /**
     * 后处理
     */
    postprocess(data) {
        console.log(`[${this.constructor.name}] 后处理数据`);
        return data;
    }

    /**
     * 格式化输出
     */
    format(data) {
        return JSON.stringify(data);
    }

    /**
     * 钩子方法 - 可选覆盖
     */
    onComplete(result) {
        // 默认空实现
    }
}

// ==================== 具体实现 ====================

}

/**
 * 销售数据处理器

        return data.filter(item => item.amount > 0);

    doProcess(data) {
        console.log('[SalesDataProcessor] 计算销售统计');

        const summary = {
            totalRevenue: 0,
            totalOrders: data.length,
            averageOrderValue: 0,
            topProducts: []
        };

        data.forEach(item => {
            summary.totalRevenue += item.amount;

        summary.averageOrderValue = summary.totalRevenue / summary.totalOrders;

        // 获取畅销产品
        summary.topProducts = [...data]
            .sort((a, b) => b.amount - a.amount)
            .slice(0, 3)
            .map(item => item.product);

    }

 * 日志数据处理器
 */
```

## 7. 状态模式（State）

### 7.1 生活类比：手机状态

手机在不同状态下行为不同：

- **锁屏状态**：按下电源键 -> 解锁
- **解锁状态**：滑动屏幕 -> 打开应用
- **应用中**：按下 Home 键 -> 返回桌面
- **低电量模式**：部分功能受限

**核心思想**：对象的行为取决于其内部状态，且状态可以切换。

### 7.2 解决的问题

```
状态模式解决的核心问题

问题场景：
1. 对象行为取决于状态
2. 有大量条件语句判断状态
3. 状态转换复杂

传统方案的问题：
- 大量 if-else 或 switch-case
- 状态逻辑分散，难以维护
- 新增状态需要修改多处代码

状态模式的优势：
- 状态逻辑封装到独立类
- 状态转换自动管理
- 易于扩展新状态
```

## 8. 职责链模式（Chain of Responsibility）

### 8.1 生活类比：客户服务升级系统

客户投诉的处理流程：

- **客服专员**：处理一般咨询，无法解决 -> 升级
- **主管**：处理复杂投诉，无法解决 -> 升级
- **经理**：处理重大问题，无法解决 -> 升级
- **总监**：处理极端情况

**核心思想**：将请求沿着处理者链传递，直到有一个处理者处理它。

### 8.2 解决的问题

```
职责链模式解决的核心问题

问题场景：
1. 有多个对象可以处理请求
2. 不确定哪个对象处理
3. 需要动态指定处理顺序

传统方案的问题：
- 一个对象处理所有逻辑
- 使用 if-else 硬编码处理顺序
- 紧耦合

职责链模式的优势：
- 解耦发送者和接收者
- 可以动态调整处理顺序
- 单一职责，每个处理者只管自己那一部分
```

```javascript
/**
 * Koa 中间件的职责链实现
 *
 * Koa 的中间件系统本质上是职责链模式的应用
 */

// 简化版 Koa 实现
class Koa {
    constructor() {
        this.middlewares = [];
    }

    use(middleware) {
        this.middlewares.push(middleware);
        return this;
    }

    // 关键：compose 函数将中间件串成链
    compose(middlewares) {
        return (ctx) => {
            let index = -1;

            const dispatch = (i) => {
                if (i <= index) {
                    return Promise.reject(new Error('next() 被调用多次'));
                }
                index = i;

                if (i >= middlewares.length) {
                    return Promise.resolve();
                }

                const middleware = middlewares[i];
                return middleware(ctx, () => dispatch(i + 1));
            };

            return dispatch(0);
        };
    }

    handleRequest(ctx) {
        const fn = this.compose(this.middlewares);
        return fn(ctx).catch(err => {
            console.error('请求处理错误:', err);
        });
    }
}

// 使用示例
```

## 9. 中介者模式（Mediator）

### 9.1 生活类比：机场塔台

飞机之间不直接通信，而是通过塔台协调：

- **没有塔台**：每架飞机需要和其他所有飞机通信，复杂
- **有塔台**：飞机只需和塔台通信，塔台协调起飞降落

**核心思想**：用一个中介对象封装一组对象交互，使对象不需要显式相互引用。

### 9.2 解决的问题

```
中介者模式解决的核心问题

问题场景：
1. 一组对象之间通信关系复杂
2. 难以维护对象之间的引用
3. 需要集中管理交互逻辑

传统方案的问题：
- 对象之间紧耦合
- 引用关系混乱
- 修改一个对象可能影响多个对象

中介者模式的优势：
- 解耦对象
- 集中管理交互
- 减少子类生成
```

### 9.3 代码示例：聊天室

```javascript
/**
 * 中介者模式 - 聊天室实现
 *
 * 设计原理：
 * 1. Chatroom 作为中介者
 * 2. User 对象只与中介者通信
 * 3. 中介者处理消息路由
 */

// ==================== 中介者接口 ====================

/**
 * 聊天室中介者接口
 */
class ChatMediator {
    sendMessage(message, fromUser, toUser) {
        throw new Error('子类必须实现 sendMessage');
    }

    addUser(user) {
        throw new Error('子类必须实现 addUser');
    }
}

// ==================== 具体中介者 ====================

/**
 * 聊天室中介者
 */
class ChatRoom extends ChatMediator {
    constructor(roomName) {
        super();
        this.roomName = roomName;
        this.users = new Map();
        this.messageHistory = [];
    }

    /**
     * 添加用户到聊天室
     */
    addUser(user) {
        this.users.set(user.id, user);
        user.setMediator(this);
        console.log(`[${this.roomName}] ${user.name} 加入了聊天室`);

        // 通知其他用户
        this.broadcast({
            type: 'system',
            content: `${user.name} 加入了聊天室`,
            from: '系统'
        }, null);
    }

    /**
     * 移除用户
     */
    removeUser(user) {
        this.users.delete(user.id);
        console.log(`[${this.roomName}] ${user.name} 离开了聊天室`);

        this.broadcast({
            type: 'system',
            content: `${user.name} 离开了聊天室`,
            from: '系统'
        }, null);
    }

    /**
     * 发送消息（公共消息）
     */
    broadcast(message, fromUser) {
        this.messageHistory.push({
            ...message,
            room: this.roomName,
            time: new Date().toISOString()
        });

        this.users.forEach(user => {
            if (user !== fromUser) {
                user.receive(message, fromUser);
            }
        });
    }

    /**
     * 发送私信
     */
    sendMessage(message, fromUser, toUserId) {
        const toUser = this.users.get(toUserId);

        if (!toUser) {
            console.log(`[${this.roomName}] 用户 ${toUserId} 不存在`);
            return;
        }

        // 记录消息
        this.messageHistory.push({
            ...message,
            room: this.roomName,
            to: toUserId,
            time: new Date().toISOString()
        });

        // 发送消息
        fromUser.sendPrivate(message, toUser);
        toUser.receive(message, fromUser);
    }

    /**
     * 获取消息历史
     */
    getHistory() {
        return this.messageHistory;
    }

    /**
     * 获取在线用户列表
     */
    getOnlineUsers() {
        return Array.from(this.users.values()).map(u => ({
            id: u.id,
            name: u.name
        }));
    }
}

// ==================== 同事类 ====================

/**
 * 聊天室用户
 */
class ChatUser {
    static userIdCounter = 0;

    constructor(name) {
        this.id = ++ChatUser.userIdCounter;
        this.name = name;
        this.mediator = null;
    }

    /**
     * 设置中介者
     */
    setMediator(mediator) {
        this.mediator = mediator;
    }

    /**
     * 发送公共消息
     */
    send(content) {
        if (!this.mediator) {
            console.log('未加入聊天室');
            return;
        }

        const message = {
            type: 'public',
            content: content,
            from: this.name
        };

        console.log(`[${this.name}] 发送消息: ${content}`);
        this.mediator.broadcast(message, this);
    }

    /**
     * 发送私信
     */
    sendPrivate(content, toUser) {
        if (!this.mediator) {
            console.log('未加入聊天室');
            return;
        }

        const message = {
            type: 'private',
            content: content,
            from: this.name
        };

        console.log(`[${this.name}] 对 ${toUser.name} 说: ${content}`);
        this.mediator.sendMessage(message, this, toUser.id);
    }

    /**
     * 接收消息
     */
    receive(message, fromUser) {
        const prefix = message.type === 'private' ? '[私信] ' : '';

        if (message.type === 'system') {
            console.log(`${prefix}[系统通知] ${message.content}`);
        } else {
            console.log(`${prefix}[${fromUser.name}] -> [${this.name}]: ${message.content}`);
        }
    }
}
```

## 10. 备忘录模式（Memento）

### 10.1 生活类比：游戏存档

玩游戏时的存档功能：

- **存档**：保存当前游戏状态
- **读取存档**：恢复到之前的状态
- **多个存档位**：可以保存多个版本

**核心思想**：在不破坏封装性的情况下，捕获对象内部状态并保存，便于恢复。

### 10.2 解决的问题

```
备忘录模式解决的核心问题

问题场景：
1. 需要保存/恢复对象状态
2. 需要撤销/恢复功能
3. 需要快照功能

传统方案的问题：
- 直接暴露内部状态
- 破坏封装性

备忘录模式的优势：
- 保持封装性
- 支持状态恢复
- 可以保存多个版本
```

### 10.3 代码示例

```javascript
/**
 * 备忘录模式 - 文档编辑器与撤销系统
 *
 * 设计原理：
 * 1. Originator 创建备忘录，保存自身状态
 * 2. Memento 存储状态（对外部不可见）
 * 3. Caretaker 负责管理备忘录
 */

// ==================== 备忘录类 ====================

/**
 * 备忘录类
 * 存储对象状态的快照
 */
class DocumentMemento {
    constructor(content, cursorPosition, lastSavedAt) {
        this._content = content;
        this._cursorPosition = cursorPosition;
        this._lastSavedAt = lastSavedAt;
        this._createdAt = new Date();
    }

    // 提供只读访问
    getContent() {
        return this._content;
    }

    getCursorPosition() {
        return this._cursorPosition;
    }

    getLastSavedAt() {
        return this._lastSavedAt;
    }

    getCreatedAt() {
        return this._createdAt;
    }
}

// ==================== 原发器 ====================

/**
 * 文档编辑器（原文器）
 * 创建和使用备忘录
 */
class DocumentEditor {
    constructor() {
        this.content = '';
        this.cursorPosition = 0;
        this.lastSavedAt = null;
    }

    /**
     * 输入文本
     */
    type(text) {
        this.content =
            this.content.slice(0, this.cursorPosition) +
            text +
            this.content.slice(this.cursorPosition);
        this.cursorPosition += text.length;
    }

    /**
     * 删除文本
     */
    delete(count) {
        const start = Math.max(0, this.cursorPosition - count);
        const deleted = this.content.slice(start, this.cursorPosition);

        this.content =
            this.content.slice(0, start) +
            this.content.slice(this.cursorPosition);
        this.cursorPosition = start;

        return deleted;
    }

    /**
     * 移动光标
     */
    moveCursor(position) {
        this.cursorPosition = Math.max(0, Math.min(position, this.content.length));
    }

    /**
     * 创建备忘录（保存当前状态）
     */
    save() {
        return new DocumentMemento(
            this.content,
            this.cursorPosition,
            this.lastSavedAt
        );
    }

    /**
     * 恢复状态（从备忘录）
     */
    restore(memento) {
        this.content = memento.getContent();
        this.cursorPosition = memento.getCursorPosition();
        this.lastSavedAt = memento.getLastSavedAt();
    }

    /**
     * 标记为已保存
     */
    markAsSaved() {
        this.lastSavedAt = new Date();
    }

    /**
     * 获取文档信息
     */
    getInfo() {
        return {
            content: this.content,
            cursorPosition: this.cursorPosition,
            lastSavedAt: this.lastSavedAt,
            length: this.content.length,
            words: this.content.split(/\s+/).filter(w => w).length
        };
    }
}

// ==================== 管理者 ====================

/**
 * 文档历史管理器
 * 管理多个备忘录
 */
class DocumentHistory {
    constructor() {
        this.savedStates = [];  // 已保存的版本
        this.autoSaveSlots = []; // 自动保存槽
        this.maxAutoSave = 10;
    }

    /**
     * 添加备忘录
     */
    push(memento) {
        this.savedStates.push(memento);
        console.log(`[历史] 保存快照，当前历史记录数: ${this.savedStates.length}`);
    }

    /**
     * 获取最后一个备忘录
     */
    pop() {
        if (this.savedStates.length === 0) {
            console.log('[历史] 没有可撤销的状态');
            return null;
        }
        const memento = this.savedStates.pop();
        console.log(`[历史] 撤销操作，当前历史记录数: ${this.savedStates.length}`);
        return memento;
    }

    /**
     * 获取所有历史
     */
    getAll() {
        return this.savedStates.map((m, i) => ({
            index: i,
            content: m.getContent(),
            createdAt: m.getCreatedAt()
        }));
    }

    /**
     * 保存到自动保存槽
     */
    autoSave(memento) {
        this.autoSaveSlots.push(memento);

        // 限制自动保存数量
        if (this.autoSaveSlots.length > this.maxAutoSave) {
            this.autoSaveSlots.shift();
        }
    }

    /**
     * 获取最后一个自动保存
     */
    getLastAutoSave() {
        if (this.autoSaveSlots.length === 0) {
            return null;
        }
        return this.autoSaveSlots[this.autoSaveSlots.length - 1];
    }
}

// ==================== 使用示例 ====================

function main() {
    console.log('========== 备忘录模式 - 文档编辑器示例 ==========\n');

    const editor = new DocumentEditor();
    const history = new DocumentHistory();

    // 初始输入
    console.log('-------- 初始编辑 --------');
    editor.type('Hello World!');
    console.log('当前内容:', editor.getInfo());

    // 保存版本1
    console.log('\n-------- 保存版本1 --------');
    history.push(editor.save());
    editor.markAsSaved();

    // 继续编辑
    console.log('\n-------- 继续编辑 --------');
    editor.type(' Welcome to the document editor.');
    console.log('当前内容:', editor.getInfo());

    // 保存版本2
    console.log('\n-------- 保存版本2 --------');
    history.push(editor.save());

    // 模拟意外情况需要恢复
    console.log('\n-------- 模拟崩溃后恢复 --------');
    editor.restore(history.pop());
    console.log('恢复后内容:', editor.getInfo());

    // 再次编辑（使用自动保存恢复）
    console.log('\n-------- 继续编辑后需要恢复 --------');
    editor.type(' More text added.');
    console.log('当前内容:', editor.getInfo());

    // 自动保存
    history.autoSave(editor.save());

    // 继续编辑
    editor.type(' Oops, a mistake!');
    console.log('编辑后内容:', editor.getInfo());

    // 从自动保存恢复
    console.log('\n-------- 从自动保存恢复 --------');
    const autoSave = history.getLastAutoSave();
    if (autoSave) {
        editor.restore(autoSave);
        console.log('恢复后内容:', editor.getInfo());
    }
}
```

## 11. 访问者模式（Visitor）

### 11.1 生活类比：医生和药店

- **医生（访问者）**：只负责诊断和开药方
- **药店（元素）**：提供药品
- **病人（数据结构）**：携带病历

医生到不同药店（元素）访问，开出不同的药方。

### 11.2 解决的问题

```
访问者模式解决的核心问题

问题场景：
1. 数据结构稳定，但操作经常变化
2. 需要对集合执行多种操作
3. 希望将操作与数据结构分离

传统方案的问题：
- 在数据结构中添加操作会破坏封装
- 修改操作需要修改数据结构

访问者模式的优势：
- 操作与数据结构分离
- 添加新操作容易
- 适合对集合执行多种操作
```

```javascript
/**
 * 访问者模式 - 文档导出系统
 *
 * 设计原理：
 * 1. Element 定义接受访问者的接口
 * 2. ConcreteElement 实现具体元素
 * 3. Visitor 定义对每个元素的访问操作
 * 4. Client 创建元素结构并触发访问

// ==================== 访问者接口 ====================

/**
 * HTML 导出访问者

class HtmlExportVisitor extends DocumentVisitor {
    visitHeading(heading) {
        return `<h${heading.level}>${heading.text}</h${heading.level}>`;
    }

    visitParagraph(paragraph) {
        return `<p>${paragraph.text}</p>`;
    }

    visitImage(image) {
        return `<img src="${image.url}" alt="${image.alt}" />`;
    }

    visitTable(table) {
        let html = '<table border="1">';

        // 表头
        html += '<thead><tr>';
        table.headers.forEach(h => {
            html += `<th>${h}</th>`;
        });
        html += '</tr></thead>';

        // 表体
        html += '<tbody>';
        table.rows.forEach(row => {
            html += '<tr>';
            row.forEach(cell => {
                html += `<td>${cell}</td>`;
            });
            html += '</tr>';
        });
        html += '</tbody></table>';

        return html;
    }
}

/**
 * Markdown 导出访问者
 */
class MarkdownExportVisitor extends DocumentVisitor {
    visitHeading(heading) {
        const prefix = '#'.repeat(heading.level);
        return `${prefix} ${heading.text}`;
    }

    visitParagraph(paragraph) {
        return paragraph.text;
    }

    visitImage(image) {
        return `![${image.alt}](${image.url})`;
    }

    visitTable(table) {
        let md = '';

        // 表头
        md += '| ' + table.headers.join(' | ') + ' |\n';
        md += '| ' + table.headers.map(() => '---').join(' | ') + ' |\n';

        // 表体
        table.rows.forEach(row => {
            md += '| ' + row.join(' | ') + ' |\n';
        });

        return md;
    }
}

 * JSON 导出访问者

    constructor() {
        super();
        this.elements = [];
    }

    visitHeading(heading) {
        this.elements.push({
            type: 'heading',
            level: heading.level,
            text: heading.text
        });

    visitParagraph(paragraph) {
        this.elements.push({
            type: 'paragraph',
            text: paragraph.text
        });
    }

    visitImage(image) {
        this.elements.push({
            type: 'image',
            url: image.url,
            alt: image.alt
        });
    }

    visitTable(table) {
        this.elements.push({
            type: 'table',
            headers: table.headers,
            rows: table.rows
        });
    }

    getResult() {
        return JSON.stringify(this.elements, null, 2);

class Document {

        this.elements = [];

        this.elements.push(element);

     * 接受访问者遍历
     */
    accept(visitor) {
        const results = [];

        this.elements.forEach(element => {
            results.push(element.accept(visitor));
        });

        return results;
    }
}
```

## 12. 解释器模式（Interpreter）

### 12.1 生活类比：翻译官

翻译官解释不同语言：

- **中文 -> 英文**：听到中文，理解意思，用英文表达
- **编程语言解释器**：读取代码，理解语法，执行结果

**核心思想**：定义语言的语法表示，解释该语法表示的含义。

### 12.2 解决的问题

```
解释器模式解决的核心问题

问题场景：
1. 需要解释特定语言
2. 语法规则相对简单
3. 需要执行表达式

传统方案的问题：
- 使用复杂的状态机
- 代码难以维护

解释器模式的优势：
- 语法规则清晰
- 易于修改和扩展
- 适合简单语法
```

### 12.3 代码示例：计算器

```javascript
/**
 * 解释器模式 - 简单计算器
 *
 * 设计原理：
 * 1. Expression 定义解释接口
 * 2. NumberExpression 和各运算表达式实现具体解释
 * 3. 递归组合形成复杂表达式
 */

// ==================== 表达式接口 ====================

/**
 * 表达式接口
 */
class Expression {
    /**
     * 解释（求值）
     */
    interpret(context) {
        throw new Error('子类必须实现 interpret');
    }
}

// ==================== 终结符表达式 ====================

/**
 * 数字表达式（终结符）
 */
class NumberExpression extends Expression {
    constructor(value) {
        super();
        this.value = Number(value);
    }

    interpret(context) {
        return this.value;
    }
}

// ==================== 非终结符表达式 ====================

/**
 * 加法表达式
 */
class AddExpression extends Expression {
    constructor(left, right) {
        super();
        this.left = left;
        this.right = right;
    }

    interpret(context) {
        return this.left.interpret(context) + this.right.interpret(context);
    }
}

/**
 * 减法表达式
 */
class SubtractExpression extends Expression {
    constructor(left, right) {
        super();
        this.left = left;
        this.right = right;
    }

    interpret(context) {
        return this.left.interpret(context) - this.right.interpret(context);
    }
}

/**
 * 乘法表达式
 */
class MultiplyExpression extends Expression {
    constructor(left, right) {
        super();
        this.left = left;
        this.right = right;
    }

    interpret(context) {
        return this.left.interpret(context) * this.right.interpret(context);
    }
}

/**
 * 除法表达式
 */
class DivideExpression extends Expression {
    constructor(left, right) {
        super();
        this.left = left;
        this.right = right;
    }

    interpret(context) {
        const rightValue = this.right.interpret(context);
        if (rightValue === 0) {
            throw new Error('除数不能为零');
        }
        return this.left.interpret(context) / rightValue;
    }
}

/**
 * 变量表达式
 */
class VariableExpression extends Expression {
    constructor(name) {
        super();
        this.name = name;
    }

    interpret(context) {
        if (context.has(this.name)) {
            return context.get(this.name);
        }
        throw new Error(`变量 ${this.name} 未定义`);
    }
}

// ==================== 表达式解析器 ====================

/**
 * 表达式解析器
 * 将中缀表达式转换为表达式对象树
 */
class ExpressionParser {
    /**
     * 解析表达式字符串
     * 支持：+、-、*、/、括号、变量
     */
    parse(expression) {
        // 移除空格
        expression = expression.replace(/\s+/g, '');

        return this.parseExpression(expression, 0).expression;
    }

    /**
     * 解析表达式（处理 + 和 -）
     */
    parseExpression(str, pos) {
        let result;
        let currentPos = pos;

        // 先解析乘除
        const result2 = this.parseTerm(str, currentPos);
        result = result2.expression;
        currentPos = result2.pos;

        // 解析加减
        while (currentPos < str.length && (str[currentPos] === '+' || str[currentPos] === '-')) {
            const operator = str[currentPos];
            currentPos++;

            const right = this.parseTerm(str, currentPos);
            currentPos = right.pos;

            if (operator === '+') {
                result = new AddExpression(result, right.expression);
            } else {
                result = new SubtractExpression(result, right.expression);
            }
        }

        return { expression: result, pos: currentPos };
    }

    /**
     * 解析项（处理 * 和 /）
     */
    parseTerm(str, pos) {
        let result;
        let currentPos = pos;

        // 先解析因子
        result = this.parseFactor(str, currentPos);
        currentPos = result.pos;

        // 解析乘除
        while (currentPos < str.length && (str[currentPos] === '*' || str[currentPos] === '/')) {
            const operator = str[currentPos];
            currentPos++;

            const right = this.parseFactor(str, currentPos);
            currentPos = right.pos;

            if (operator === '*') {
                result = new MultiplyExpression(result, right.expression);
            } else {
                result = new DivideExpression(result, right.expression);
            }
        }

        return { expression: result, pos: currentPos };
    }

    /**
     * 解析因子（数字、变量、括号）
     */
    parseFactor(str, pos) {
        let currentPos = pos;

        // 跳过括号
        if (str[currentPos] === '(') {
            currentPos++;
            const result = this.parseExpression(str, currentPos);
            currentPos = result.pos;

            if (str[currentPos] !== ')') {
                throw new Error('缺少右括号');
            }
            currentPos++;

            return { expression: result.expression, pos: currentPos };
        }

        // 解析变量或数字
        let value = '';
        while (currentPos < str.length && /[a-zA-Z0-9_]/.test(str[currentPos])) {
            value += str[currentPos];
            currentPos++;
        }

        if (value === '') {
            throw new Error('意外的字符: ' + str[currentPos]);
        }

        // 判断是数字还是变量
        if (/^\d+$/.test(value)) {
            return { expression: new NumberExpression(value), pos: currentPos };
        } else {
            return { expression: new VariableExpression(value), pos: currentPos };
        }
    }
}

// ==================== 使用示例 ====================

function main() {
    console.log('========== 解释器模式 - 计算器示例 ==========\n');

    const parser = new ExpressionParser();

    // 示例1：简单计算
    console.log('-------- 简单计算 --------');
    const expr1 = parser.parse('10 + 20');
    console.log('10 + 20 =', expr1.interpret(new Map()));

    const expr2 = parser.parse('10 + 20 * 30');
    console.log('10 + 20 * 30 =', expr2.interpret(new Map()));

    const expr3 = parser.parse('(10 + 20) * 30');
    console.log('(10 + 20) * 30 =', expr3.interpret(new Map()));

    // 示例2：带变量
    console.log('\n-------- 带变量 --------');
    const expr4 = parser.parse('x + y * z');
    const context = new Map([
        ['x', 10],
        ['y', 20],
        ['z', 30]
    ]);
    console.log('x=10, y=20, z=30 时，x + y * z =', expr4.interpret(context));

    // 示例3：复杂表达式
    console.log('\n-------- 复杂表达式 --------');
    const expr5 = parser.parse('((a + b) * c - d) / e');
    const context2 = new Map([
        ['a', 100],
        ['b', 200],
        ['c', 3],
        ['d', 50],
        ['e', 10]
    ]);
    console.log('((a + b) * c - d) / e =', expr5.interpret(context2));

    // 示例4：函数式计算
    console.log('\n-------- 验证计算器 --------');
    const tests = [
        ['1 + 2 + 3', 6],
        ['10 - 5 - 2', 3],
        ['100 / 10 / 2', 5],
        ['2 * 3 * 4', 24],
        ['(1 + 2) * (3 + 4)', 21],
        ['1 + 2 * 3 + 4', 11]
    ];

    tests.forEach(([expr, expected]) => {
        const parsed = parser.parse(expr);
        const result = parsed.interpret(new Map());
        const status = result === expected ? '✓' : '✗';
        console.log(`${status} ${expr} = ${result} (期望: ${expected})`);
    });
}

main();
```

### 13.4 最佳实践总结

```
行为型模式最佳实践

1. 观察者模式
   - 使用 EventEmitter 管理事件
   - 注意内存泄漏，及时取消订阅
   - 优先使用 once 处理单次事件

2. 策略模式
   - 策略类应该无状态
   - 使用工厂模式管理策略创建
   - 避免过大量级的策略类

3. 命令模式
   - 命令应该幂等（可重复执行）
   - 合理设计命令粒度
   - 使用宏命令组合简单命令

4. 状态模式
   - 状态转换逻辑放在状态类中
   - 使用状态机可视化状态转换
   - 注意状态类数量控制

5. 职责链模式
   - 链不要太长（影响性能）
   - 每个处理者只处理自己负责的部分
   - 提供默认处理（防止请求丢失）

6. 迭代器模式
   - 支持同步和异步迭代
   - 使用生成器简化实现
   - 注意迭代器的可重入性

7. 模板方法模式
   - 使用钩子方法增加灵活性
   - 模板方法应该是 final 的
   - 抽象方法应该尽量少

8. 中介者模式
   - 中介者不应该成为上帝对象
   - 解耦要有度，避免过度使用
   - 考虑使用事件替代直接调用

9. 备忘录模式
   - 注意备忘录的内存占用
   - 可以使用增量存储节省空间
   - 考虑序列化/反序列化

10. 访问者模式
    - 元素类应该稳定
    - 使用双分派减少类型检查
    - 考虑使用反射替代

11. 解释器模式
    - 仅用于简单语法
    - 复杂语法使用语法解析器
    - 考虑性能优化（缓存）
```
