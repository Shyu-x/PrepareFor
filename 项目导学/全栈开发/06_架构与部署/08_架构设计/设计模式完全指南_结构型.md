## 1. 设计原则回顾

### 1.1 SOLID原则速览

```
SOLID原则

S - 单一职责原则（Single Responsibility Principle）
    一个类只负责一件事，当需要修改某个类时，原因只有一个

O - 开闭原则（Open-Closed Principle）
    对扩展开放，对修改关闭
    允许新增代码来扩展功能，但不应修改现有代码

L - 里氏替换原则（Liskov Substitution Principle）
    子类可以替换父类，而不影响程序的正确性

I - 接口隔离原则（Interface Segregation Principle）
    客户端不应该依赖它不需要的接口
    接口应该小而专一

D - 依赖倒置原则（Dependency Inversion Principle）
    依赖抽象，不依赖具体实现
    高层模块不应该依赖低层模块
```

### 1.2 结构型模式的核心使命

**结构型模式关注的是如何组合类和对象，以形成更大的结构。**

```
结构型模式的核心问题：

类继承 vs 对象组合
├── 类继承：静态的、编译时确定
├── 对象组合：动态的、运行时决定
└── 优先使用对象组合，而不是类继承

结构型模式的7种模式：
├── 适配器模式：接口转换，让不兼容的接口能够协作
├── 桥接模式：分离抽象与实现，让两者独立变化
├── 组合模式：树形结构，整体与部分一致处理
├── 装饰器模式：动态添加职责，替代继承扩展
├── 外观模式：统一接口，简化复杂子系统
├── 享元模式：共享细粒度对象，减少内存开销
└── 代理模式：控制对原对象的访问
```

## 2. 适配器模式（Adapter Pattern）

### 2.1 生活类比：电源适配器

```
生活场景：出国旅行时的电源适配器

问题：
- 你的笔记本需要220V交流电
- 国外插座提供110V或不同规格的接口
- 直接插上去无法工作

解决方案：使用电源适配器
- 适配器将110V转换为220V
- 适配器将不兼容的插头转换为标准插头
- 笔记本无需任何改变

适配器模式的核心思想：
将一个类的接口转换成客户期望的另一个接口，使原本接口不兼容的类可以一起工作
```

### 2.2 解决的问题

```
适配器模式解决的核心问题：接口不兼容

典型场景：
1. 集成第三方库
   - 第三方库有自己的接口规范
   - 你的应用期望统一的接口
   - 适配器来转换接口

2. 遗留代码迁移
   - 旧系统接口已经稳定
   - 新系统需要对接
   - 不想修改旧代码

3. 多数据源整合
   - 不同数据源格式不同
   - 统一的数据访问接口
   - 适配器转换数据格式

4. API版本迭代
   - 旧版API仍在使用
   - 新版API接口改变
   - 适配器保持兼容性
```

### 2.3 代码示例

#### 2.3.1 类适配器（使用继承）

```typescript
// ========== 场景：支付系统集成 ==========

// 旧版支付接口（已存在）
interface LegacyPayment {
    // 旧接口使用美元金额
    processPayment(amountInCents: number, currency: string): boolean;
    refundPayment(transactionId: string): boolean;
}

// 新版支付接口（第三方SDK）
class StripeSDK {
    // 新接口使用小数金额
    charge(amount: number, currency: string): { success: boolean; id: string } {
        console.log(`Stripe扣款: ${amount} ${currency}`);
        return { success: true, id: 'stripe_' + Date.now() };
    }

    refund(chargeId: string): boolean {
        console.log(`Stripe退款: ${chargeId}`);
        return true;
    }
}

// 支付宝SDK
class AlipaySDK {
    // 支付宝使用分为单位
    pay(amountYuan: number): { code: number; outTradeNo: string } {
        console.log(`支付宝收款: ${amountYuan}元`);
        return { code: 200, outTradeNo: 'alipay_' + Date.now() };
    }

    refundRequest(outTradeNo: string): number {
        console.log(`支付宝退款: ${outTradeNo}`);
        return 200;
    }
}

// 适配器类（使用继承）
class StripeAdapter extends StripeSDK implements LegacyPayment {
    // 实现旧接口
    processPayment(amountInCents: number, currency: string): boolean {
        // 转换：从美分转为美元小数
        const amount = amountInCents / 100;
        const result = this.charge(amount, currency);
        return result.success;
    }

    refundPayment(transactionId: string): boolean {
        return this.refund(transactionId);
    }
}

// 支付宝适配器（使用继承）
class AlipayAdapter extends AlipaySDK implements LegacyPayment {
    processPayment(amountInCents: number, currency: string): boolean {
        if (currency !== 'CNY') {
            // 支付宝只支持人民币，转换失败
            return false;
        }
        // 转换：从美分转为元
        const amountYuan = amountInCents / 100;
        const result = this.pay(amountYuan);
        return result.code === 200;
    }

    refundPayment(transactionId: string): boolean {
        return this.refundRequest(transactionId) === 200;
    }
}

// 使用示例
function processUserPayment(payment: LegacyPayment, amountCents: number) {
    console.log('开始处理支付...');
    const success = payment.processPayment(amountCents, 'CNY');
    console.log(`支付结果: ${success ? '成功' : '失败'}`);
    return success;
}

// 统一使用旧接口调用
const stripePayment = new StripeAdapter();
const alipayPayment = new AlipayAdapter();

processUserPayment(stripePayment, 9999);   // Stripe: 99.99元
processUserPayment(alipayPayment, 9999);   // 支付宝: 99.99元
```

```typescript
// ========== 场景：数据格式适配 ==========

// 期望的统一数据格式
interface UserData {
    id: string;
    name: string;
    email: string;
    avatar: string;
    createdAt: Date;
}

// 第三方API返回的格式A（XML格式）
class ThirdPartyAPI_A {
    getUser(userId: string): string {
        // 模拟返回XML格式
        return `
            <user>
                <uid>${userId}</uid>
                <full_name>张三</full_name>
                <email_address>zhangsan@example.com</email_address>
                <profile_pic>https://example.com/avatar/1.jpg</profile_pic>
                <register_time>2024-01-15T10:30:00Z</register_time>
            </user>
        `;
    }
}

// 第三方API返回的格式B（自定义格式）
class ThirdPartyAPI_B {
    getUser(userId: string): object {
        return {
            user_id: userId,
            nickname: '李四',
            mail: 'lisi@example.com',
           头像: 'https://example.com/avatar/2.jpg',
            注册时间: 1705315800000,  // 时间戳
        };
    }
}

// XML解析器
function parseXML(xmlString: string): Record<string, string> {
    const result: Record<string, string> = {};
    // 简化解析，实际项目使用DOMParser
    const regex = /<(\w+)>([^<]*)<\/\1>/g;
    let match;
    while ((match = regex.exec(xmlString)) !== null) {
        result[match[1]] = match[2];
    }
    return result;
}

// 对象适配器：API A适配器
class UserAdapter_A implements UserData {
    id: string;
    name: string;
    email: string;
    avatar: string;
    createdAt: Date;

    constructor(apiA: ThirdPartyAPI_A, userId: string) {
        // 从API获取原始数据
        const rawData = apiA.getUser(userId);
        const xml = parseXML(rawData);

        // 转换为统一格式
        this.id = xml.uid;
        this.name = xml.full_name;
        this.email = xml.email_address;
        this.avatar = xml.profile_pic;
        this.createdAt = new Date(xml.register_time);
    }
}

// 对象适配器：API B适配器
class UserAdapter_B implements UserData {
    id: string;
    name: string;
    email: string;
    avatar: string;
    createdAt: Date;

    constructor(apiB: ThirdPartyAPI_B, userId: string) {
        // 从API获取原始数据
        const rawData = apiB.getUser(userId) as Record<string, any>;

        // 转换为统一格式
        this.id = String(rawData.user_id);
        this.name = rawData.nickname;
        this.email = rawData.mail;
        this.avatar = rawData['头像'];
        this.createdAt = new Date(rawData['注册时间']);
    }
}

// 工厂函数创建适配器
function createUserAdapter(apiType: 'A' | 'B', userId: string): UserData {
    if (apiType === 'A') {
        const apiA = new ThirdPartyAPI_A();
        return new UserAdapter_A(apiA, userId);
    } else {
        const apiB = new ThirdPartyAPI_B();
        return new UserAdapter_B(apiB, userId);
    }
}

// 使用示例
const userA = createUserAdapter('A', 'user_001');
const userB = createUserAdapter('B', 'user_002');

console.log('用户A:', userA);
console.log('用户B:', userB);
```

#### 2.3.3 双向适配器

```typescript
// ========== 场景：双向数据同步 ==========

// 内部系统格式
interface InternalFormat {
    userId: string;
    userName: string;
    userEmail: string;
}

// 外部API格式
interface ExternalFormat {
    id: string;
    name: string;
    email: string;
    status: 'active' | 'inactive';
}

// 双向适配器
class UserDataAdapter {
    // 将内部格式转为外部格式
    static toExternal(internal: InternalFormat): ExternalFormat {
        return {
            id: internal.userId,
            name: internal.userName,
            email: internal.userEmail,
            status: 'active',  // 内部格式没有状态字段，默认active
        };
    }

    // 将外部格式转为内部格式
    static toInternal(external: ExternalFormat): InternalFormat {
        return {
            userId: external.id,
            userName: external.name,
            userEmail: external.email,
        };
    }

    // 批量转换
    static toExternalList(internalList: InternalFormat[]): ExternalFormat[] {
        return internalList.map(item => this.toExternal(item));
    }

    static toInternalList(externalList: ExternalFormat[]): InternalFormat[] {
        return externalList.map(item => this.toInternal(item));
    }
}

// 使用示例
const internalUser: InternalFormat = {
    userId: 'U001',
    userName: '王五',
    userEmail: 'wangwu@example.com',
};

const externalUser = UserDataAdapter.toExternal(internalUser);
console.log('外部格式:', externalUser);

// 模拟外部API返回
const apiResponse: ExternalFormat = {
    id: 'U002',
    name: '赵六',
    email: 'zhaoliu@example.com',
    status: 'inactive',
};

const internalUser2 = UserDataAdapter.toInternal(apiResponse);
console.log('内部格式:', internalUser2);
```

```typescript
import React from 'react';

// 统一的图表配置接口
interface ChartConfig {
    title: string;
    data: number[];
    colors?: string[];

// 不同图表库的接口

interface RechartsProps {
    name: string;
    data: Array<{ name: string; value: number }>;
}

// 图表适配器接口
interface ChartAdapter {
    render(config: ChartConfig): React.ReactNode;
}

// ECharts适配器
class EChartsAdapter implements ChartAdapter {
    render(config: ChartConfig): React.ReactNode {
        // 假设这里使用echarts-for-react组件
        const option: EChartsOption = {
            title: { text: config.title },
            xAxis: { data: config.data.map((_, i) => `项${i + 1}`) },
            series: [{ data: config.data }],
        };

        // 返回ECharts组件
        return (
            <div className="echarts-container">
                {/* 实际项目中这里是 <ReactECharts option={option} /> */}
                <pre>{JSON.stringify(option, null, 2)}</pre>
            </div>
        );
    }
}

// Recharts适配器
class RechartsAdapter implements ChartAdapter {
    render(config: ChartConfig): React.ReactNode {
        const data = config.data.map((value, index) => ({
            name: `项${index + 1}`,
            value,
        }));

        // 返回Recharts组件
        return (
            <div className="recharts-container">
                {/* 实际项目中这里是 <BarChart data={data} /> */}
                <pre>{JSON.stringify(data, null, 2)}</pre>
            </div>
        );
    }
}

// 图表工厂
class ChartFactory {
    private static adapters = {
        echarts: new EChartsAdapter(),
        recharts: new RechartsAdapter(),
    };

    static createChart(type: 'echarts' | 'recharts'): ChartAdapter {
        return this.adapters[type];
    }
}

// 统一图表组件
interface ChartProps {
    type: 'echarts' | 'recharts';
    config: ChartConfig;
}

function UniversalChart({ type, config }: ChartProps) {
    const adapter = ChartFactory.createChart(type);
    return adapter.render(config);
}

// 使用示例
function Dashboard() {

        data: [120, 200, 150, 80, 70, 110, 130],
        colors: ['#1890ff', '#52c41a', '#faad14'],

    return (
        <div>
            <h2>销售报表</h2>
            {/* 可以自由切换图表库 */}
            <UniversalChart type="echarts" config={config} />
            <UniversalChart type="recharts" config={config} />
        </div>
    );
}
```

### 2.5 Express中间件中的适配器

```typescript
// ========== 场景：适配不同格式的中间件 ==========

import { Request, Response, NextFunction } from 'express';

// 不同的认证服务接口
interface AuthService {
    validateToken(token: string): Promise<{ valid: boolean; userId?: string }>;
}

class JWTService implements AuthService {
    async validateToken(token: string) {
        // JWT验证逻辑
        return { valid: true, userId: 'jwt_user_123' };
    }
}

class OAuthService implements AuthService {
    async validateToken(token: string) {
        // OAuth token验证
        return { valid: true, userId: 'oauth_user_456' };
    }
}

class LegacyAuthService implements AuthService {
    async validateToken(token: string) {
        // 遗留系统验证
        return { valid: true, userId: 'legacy_user_789' };
    }
}

// 适配器包装不同服务为统一接口
function authAdapter(service: AuthService) {
    return async (req: Request, res: Response, next: NextFunction) => {
        const token = req.headers.authorization?.replace('Bearer ', '');

        if (!token) {
            return res.status(401).json({ error: '未提供认证令牌' });
        }

        try {
            const result = await service.validateToken(token);

            if (!result.valid) {
                return res.status(401).json({ error: '无效的令牌' });
            }

            // 将用户ID附加到请求对象
            (req as any).userId = result.userId;
            next();
        } catch (error) {
            return res.status(500).json({ error: '认证服务错误' });
        }
    };
}

// Express路由使用
function createAuthRoutes(app: any) {
    // 根据配置选择认证服务
    const authServiceType = process.env.AUTH_SERVICE || 'jwt';

    let authService: AuthService;
    switch (authServiceType) {
        case 'oauth':
            authService = new OAuthService();
            break;
        case 'legacy':
            authService = new LegacyAuthService();
            break;
        default:
            authService = new JWTService();
    }

    // 所有路由使用统一的认证中间件
    app.use('/api', authAdapter(authService), (req: Request, res: Response) => {
        res.json({ message: '已认证访问', userId: (req as any).userId });
    });
}
```

## 3. 桥接模式（Bridge Pattern）

### 3.1 生活类比：遥控器与电器

```
生活场景：电视遥控器

问题分析：
- 遥控器需要控制不同品牌的电视（Sony、LG、Samsung）
- 遥控器也需要支持不同类型的设备（电视、音响、空调）

错误设计（类爆炸）：
├── Sony电视遥控器
├── LG电视遥控器
├── Samsung电视遥控器
├── Sony音响遥控器
├── LG音响遥控器
└── ...  数量爆炸

正确设计（桥接模式）：
├── 抽象层：遥控器（按钮布局）
│   ├── 电视遥控器
│   └── 音响遥控器
│
└── 实现层：设备控制协议
    ├── Sony控制协议
    ├── LG控制协议
    └── Samsung控制协议

桥接模式核心思想：
将抽象部分与实现部分分离，使它们可以独立变化
"抽象"指的是业务层面的高层逻辑，"实现"指的是底层的具体实现
```

### 3.2 解决的问题

```
桥接模式解决的核心问题：两个维度独立变化，避免类爆炸

典型场景：
1. 跨平台应用
   - 维度1：不同的应用功能（画图、文档、游戏）
   - 维度2：不同的操作系统（Windows、Mac、Linux）
   - 组合数量：3 x 3 = 9种

2. 多维度变化
   - 消息通知：维度1（类型：短信、邮件、推送）
   - 通知渠道：维度2（运营商：阿里云、腾讯云、自建）
   - 组合数量爆炸

3. 数据库驱动
   - 维度1：不同的ORM框架
   - 维度2：不同的数据库
   - 独立开发和测试

4. 支付系统
   - 维度1：支付场景（电商、线下、退款）
   - 维度2：支付渠道（支付宝、微信、银行卡）
```

### 3.3 代码示例

#### 3.3.1 基础桥接模式

```typescript
// ========== 场景：消息通知系统 ==========

// 实现者接口：消息发送渠道
interface MessageSender {
    send(content: string, recipient: string): boolean;
    getName(): string;
}

// 具体实现者：短信发送
class SMSSender implements MessageSender {
    private apiKey: string;
    private endpoint: string;

    constructor(apiKey: string, endpoint: string) {
        this.apiKey = apiKey;
        this.endpoint = endpoint;
    }

    send(content: string, recipient: string): boolean {
        console.log(`[SMS] 发送至 ${recipient}: ${content}`);
        console.log(`[SMS] 使用API密钥: ${this.apiKey.substring(0, 8)}...`);
        return true;
    }

    getName(): string {
        return '短信';
    }
}

// 具体实现者：邮件发送
class EmailSender implements MessageSender {
    private smtpServer: string;

    constructor(smtpServer: string) {
        this.smtpServer = smtpServer;
    }

    send(content: string, recipient: string): boolean {
        console.log(`[Email] 发送至 ${recipient}: ${content}`);
        console.log(`[Email] SMTP服务器: ${this.smtpServer}`);
        return true;
    }

    getName(): string {
        return '邮件';
    }
}

// 具体实现者：微信推送
class WechatSender implements MessageSender {
    private appId: string;
    private templateId: string;

    constructor(appId: string, templateId: string) {
        this.appId = appId;
        this.templateId = templateId;
    }

    send(content: string, recipient: string): boolean {
        console.log(`[Wechat] 发送至 ${recipient}: ${content}`);
        console.log(`[Wechat] AppID: ${this.appId}, 模板: ${this.templateId}`);
        return true;
    }

    getName(): string {
        return '微信';
    }
}

// 抽象者：消息通知
abstract class Notification {
    // 桥接引用：组合实现者
    protected sender: MessageSender;

    constructor(sender: MessageSender) {
        this.sender = sender;
    }

    // 抽象方法：发送通知（由子类实现具体逻辑）
    abstract send(content: string, recipient: string): void;

    // 获取渠道名称
    protected getChannelName(): string {
```

```typescript
// 第二维度抽象：支付场景
abstract class PaymentProcessor {
    // 桥接：组合支付渠道
    protected gateway: PaymentGateway;

    constructor(gateway: PaymentGateway) {
        this.gateway = gateway;
    }

    // 模板方法：定义支付流程骨架
    async pay(amount: number, currency: string, metadata?: Record<string, any>): Promise<PaymentResult> {
        // 1. 验证金额
        if (!this.validateAmount(amount)) {
            return { success: false, transactionId: '', message: '金额无效' };
        }

        // 2. 记录日志
        this.logPaymentAttempt(amount, currency);

        // 3. 调用渠道支付
        const result = await this.gateway.processPayment(amount, currency);

        // 4. 后置处理
        if (result.success) {
            this.onPaymentSuccess(result.transactionId, metadata);
        }

        return result;
    }

    async refundPayment(transactionId: string): Promise<RefundResult> {
        return await this.gateway.refund(transactionId);
    }

    // 抽象方法：子类实现
    abstract validateAmount(amount: number): boolean;
    abstract logPaymentAttempt(amount: number, currency: string): void;
    abstract onPaymentSuccess(transactionId: string, metadata?: Record<string, any>): void;
}
```

## 4. 组合模式（Composite Pattern）

### 4.1 生活类比：文件系统

```
生活场景：Windows资源管理器

问题分析：
- 文件和文件夹在界面上看起来是一样的
- 文件夹可以包含文件和子文件夹
- 对文件夹的操作（复制、删除、移动）需要递归处理内部所有元素

组合模式的核心思想：
将对象组合成树形结构以表示"部分-整体"的层次结构
让客户端统一处理单个对象和组合对象

关键特性：
├── 叶节点：没有子节点的元素（如文件）
├── 组合节点：包含子节点的元素（如文件夹）
└── 统一接口：客户端可以一致地处理叶节点和组合节点
```

### 4.2 解决的问题

```
组合模式解决的核心问题：树形结构的统一处理

典型场景：
1. 文件系统
   - 文件和文件夹使用相同接口
   - 递归遍历、复制、删除

2. 组织架构
   - 公司 → 部门 → 团队 → 员工
   - 统一处理个人和团队

3. UI组件树
   - 容器组件 → 子组件
   - 批量操作（显示/隐藏/禁用）

4. 菜单系统
   - 菜单项和子菜单
   - 统一渲染和交互

5. 文档结构
   - 章节 → 段落 → 句子
   - 批量格式化和操作
```

```typescript
// ========== 场景：文件管理系统 ==========

// 统一接口：文件系统节点
interface FileSystemNode {
    name: string;
    getSize(): number;
    print(indent?: string): void;
    // 新增方法
    add(node: FileSystemNode): void;
    remove(node: FileSystemNode): void;
    findByName(name: string): FileSystemNode | null;
}

// 叶节点：文件
class File implements FileSystemNode {
    name: string;
    private size: number;
    private content: string;

    constructor(name: string, size: number, content: string = '') {
        this.name = name;
        this.size = size;
        this.content = content;
    }

    getSize(): number {
        return this.size;
    }

    print(indent: string = ''): void {
        console.log(`${indent}- 文件: ${this.name} (${this.size}KB)`);
    }

    // 叶节点不支持添加子节点
    add(node: FileSystemNode): void {
        throw new Error('文件不能添加子节点');
    }

    remove(node: FileSystemNode): void {
        throw new Error('文件不能删除子节点');
    }

    findByName(name: string): FileSystemNode | null {
        return this.name === name ? this : null;
    }
}

// 组合节点：文件夹
class Folder implements FileSystemNode {
    name: string;
    private children: FileSystemNode[] = [];

    constructor(name: string) {
        this.name = name;
    }

    getSize(): number {
        // 递归计算总大小
        return this.children.reduce((total, child) => total + child.getSize(), 0);
    }

    print(indent: string = ''): void {
        console.log(`${indent}+ 文件夹: ${this.name} (${this.getSize()}KB)`);
        // 递归打印子节点
        this.children.forEach(child => child.print(indent + '  '));
    }

    add(node: FileSystemNode): void {
        this.children.push(node);
    }

    remove(node: FileSystemNode): void {
        const index = this.children.indexOf(node);
        if (index > -1) {
            this.children.splice(index, 1);
        }
    }

    findByName(name: string): FileSystemNode | null {
        // 检查当前节点
        if (this.name === name) {
            return this;
        }
        // 递归查找子节点
        for (const child of this.children) {
            const found = child.findByName(name);
            if (found) {
                return found;
            }
        }
        return null;
    }

    // 获取所有文件
    getAllFiles(): File[] {
        const files: File[] = [];
        for (const child of this.children) {
            if (child instanceof File) {
                files.push(child);
            } else if (child instanceof Folder) {
                files.push(...child.getAllFiles());
            }
        }
        return files;
    }
}

// 使用示例
function demoCompositePattern() {
    console.log('=== 组合模式：文件系统 ===\n');

    // 创建文件
    const readme = new File('README.md', 5, '# 项目文档');
    const packageJson = new File('package.json', 2, '{"name": "demo"}');
    const indexTs = new File('index.ts', 10, 'console.log("Hello")');
    const utilsTs = new File('utils.ts', 8, 'export const util = () => {}');
    const appTs = new File('app.ts', 15, 'const app = express()');

    // 创建源代码文件夹
    const srcFolder = new Folder('src');
    srcFolder.add(indexTs);
    srcFolder.add(utilsTs);
    srcFolder.add(appTs);

    // 创建配置文件夹
    const configFolder = new Folder('config');
    configFolder.add(packageJson);

    // 创建docs文件夹
    const docsFolder = new Folder('docs');
    docsFolder.add(readme);

    // 创建根文件夹
    const rootFolder = new Folder('project');
    rootFolder.add(srcFolder);
    rootFolder.add(configFolder);
    rootFolder.add(docsFolder);

    // 打印整棵树
    console.log('项目结构:');
    rootFolder.print();
    console.log('');

    // 计算总大小
    console.log(`项目总大小: ${rootFolder.getSize()}KB`);

    // 查找文件
    console.log('\n查找文件:');
    const found = rootFolder.findByName('index.ts');
    if (found) {
        console.log(`找到: ${found.name}, 大小: ${found.getSize()}KB`);
    }

    // 获取所有文件
    console.log('\n所有文件:');
    const allFiles = rootFolder.getAllFiles();
    allFiles.forEach(file => console.log(`- ${file.name}`));

    // 动态添加文件
    console.log('\n添加新文件:');
    const testFile = new File('test.ts', 3, 'describe("test", () => {})');
    srcFolder.add(testFile);
    rootFolder.print();

    // 删除文件夹
    console.log('\n删除docs文件夹:');
    rootFolder.remove(docsFolder);
    rootFolder.print();
}
```

```typescript
interface MenuItemConfig {

    label: string;
    icon?: string;

}

// 组合节点：菜单组
interface MenuComposite {
    type: 'menu' | 'item';
    key: string;
    label: string;
    icon?: string;
    children?: MenuComposite[];
    action?: () => void;
}

// 菜单上下文
interface MenuContextValue {
    selectedKey: string;
    onSelect: (key: string) => void;
    openKeys: string[];
    onToggle: (key: string) => void;
}

const MenuContext = createContext<MenuContextValue | null>(null);

// 菜单Provider
function MenuProvider({ children, mode = 'vertical' }: { children: ReactNode; mode?: 'vertical' | 'horizontal' }) {
    const [selectedKey, setSelectedKey] = React.useState('');
    const [openKeys, setOpenKeys] = React.useState<string[]>([]);

    const onSelect = (key: string) => {
        setSelectedKey(key);
    };

    const onToggle = (key: string) => {
        setOpenKeys(current => {
            if (current.includes(key)) {
                return current.filter(k => k !== key);
            }
            return [...current, key];
        });
    };

    return (
        <MenuContext.Provider value={{ selectedKey, onSelect, openKeys, onToggle }}>
            <nav style={{
                display: 'flex',
                flexDirection: mode === 'horizontal' ? 'row' : 'column',
                gap: 4,
                padding: 16,
                background: '#f5f5f5',
            }}>
                {children}
            </nav>
        </MenuContext.Provider>
    );
}

// 菜单项组件
function MenuItem({ config }: { config: MenuItemConfig }) {
    const context = useContext(MenuContext);
    if (!context) return null;

    const isSelected = context.selectedKey === config.key;
    const hasChildren = config.children && config.children.length > 0;
    const isOpen = context.openKeys.includes(config.key);

    return (
        <div style={{ position: 'relative' }}>
            <div
                onClick={() => {
                    context.onSelect(config.key);
                    config.action?.();
                }}
                style={{
                    padding: '8px 16px',
                    background: isSelected ? '#1890ff' : 'transparent',
                    color: isSelected ? '#fff' : '#000',
                    borderRadius: 4,
                    cursor: 'pointer',
                    display: 'flex',
                    alignItems: 'center',
                    gap: 8,
                }}
            >
                {config.icon && <span>{config.icon}</span>}
                <span>{config.label}</span>
                {hasChildren && <span>{isOpen ? '▼' : '▶'}</span>}
            </div>

            {/* 子菜单 */}
            {hasChildren && isOpen && (
                <div style={{
                    position: 'absolute',
                    left: '100%',
                    top: 0,
```

```typescript
class ParagraphNode implements DocumentNode {
    type: NodeType = 'paragraph';
    id: string;
    private children: DocumentNode[] = [];

    constructor(id: string) {
        this.id = id;
    }

    render(): string {
        const content = this.children.map(child => child.render()).join('');
        return `<p>${content}</p>`;
    }

    getText(): string {
        return this.children.map(child => child.getText()).join('');
    }

    setText(text: string): void {
        // 设置第一个文本节点的内容
        const textNode = this.children.find(c => c.type === 'text') as TextNode;
        if (textNode) {
            textNode.setText(text);
        }
    }

    addChild(node: DocumentNode): void {
        this.children.push(node);
    }

    removeChild(node: DocumentNode): void {
        const index = this.children.indexOf(node);
        if (index > -1) {
            this.children.splice(index, 1);
        }
    }

    getChildren(): DocumentNode[] {
        return [...this.children];
    }

    traverse(visitor: NodeVisitor): void {
        visitor.visit(this);
        this.children.forEach(child => child.traverse(visitor));
    }

    clone(): ParagraphNode {
        const clone = new ParagraphNode(this.id + '_copy');
        this.children.forEach(child => clone.addChild(child.clone()));
        return clone;
    }

class WordCountVisitor implements NodeVisitor {
    private wordCount = 0;

    visit(node: DocumentNode): void {
        if (node.type === 'text') {
            const text = node.getText();
            this.wordCount += text.split(/\s+/).filter(w => w.length > 0).length;
        }
    }

    getCount(): number {
        return this.wordCount;
    }
}

// 访问者实现：查找替换
class FindReplaceVisitor implements NodeVisitor {
    private find: string;
    private replace: string;
    private count = 0;

    constructor(find: string, replace: string) {
        this.find = find;
        this.replace = replace;
    }

    visit(node: DocumentNode): void {
        if (node.type === 'text') {
            const text = node.getText();
            if (text.includes(this.find)) {
                node.setText(text.replace(new RegExp(this.find, 'g'), this.replace));
                this.count++;
            }
        }
    }

    getReplaceCount(): number {
        return this.count;
    }
}

// 访问者实现：批量格式化
class FormatVisitor implements NodeVisitor {
    private formatType: 'bold' | 'italic' | 'uppercase';

    constructor(formatType: 'bold' | 'italic' | 'uppercase') {
        this.formatType = formatType;
    }

    visit(node: DocumentNode): void {
        if (node.type === 'text') {
            const textNode = node as TextNode;
            switch (this.formatType) {
                case 'bold':
                    textNode.makeBold();
                    break;
                case 'italic':
                    textNode.makeItalic();
                    break;
            }
        }
    }
}
```

## 5. 装饰器模式（Decorator Pattern）

### 5.1 生活类比：咖啡加料

```
生活场景：咖啡店点单

问题分析：
- 基础咖啡价格固定
- 客户需要添加各种配料：牛奶、糖、摩卡、奶油等
- 不同配料组合产生不同价格

方案对比：
├── 继承方案：每种组合创建一个类
│   ├── 咖啡+牛奶
│   ├── 咖啡+糖
│   ├── 咖啡+牛奶+糖
│   └── ...  组合爆炸
│
└── 装饰器方案：动态组合
    ├── 基础咖啡
    ├── + 牛奶装饰器
    ├── + 糖装饰器
    └── 运行时自由组合

装饰器模式核心思想：
动态地给对象添加一些额外的职责
比继承更加灵活，组合大于继承
```

### 5.2 解决的问题

```
装饰器模式解决的核心问题：动态添加职责，避免类爆炸

典型场景：
1. 功能叠加
   - 日志记录 + 性能监控 + 事务管理
   - 缓存 + 序列化 + 压缩

2. 包装已有组件
   - 增强原有功能
   - 不修改原对象

3. 职责链动态组合
   - HTTP中间件
   - Express/Koa中间件

4. 类功能扩展
   - TypeScript装饰器
   - 类方法增强
```

```typescript
// ========== 场景：TypeScript装饰器实现 ==========

// 类装饰器
function Sealed(constructor: Function) {
    Object.seal(constructor);
    Object.seal(constructor.prototype);
}

function Frozen(value: boolean) {
    return function (_target: Function) {
        Object.freeze(_target);
    };
}

// 装饰器工厂
function Logger(prefix: string) {
    return function (target: any, propertyKey: string, descriptor: PropertyDescriptor) {
        const original = descriptor.value;

        descriptor.value = function (...args: any[]) {
            console.log(`${prefix} [${propertyKey}] 调用参数:`, args);
            const result = original.apply(this, args);
            console.log(`${prefix} [${propertyKey}] 返回值:`, result);
            return result;
        };

        return descriptor;
    };
}

// 方法装饰器
function Readonly(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    descriptor.writable = false;
    return descriptor;
}

// 属性装饰器
function Non enumerable(target: any, propertyKey: string) {
    Object.defineProperty(target, propertyKey, {
        enumerable: false,
    });
}

// 参数装饰器
function MinLength(min: number) {
    return function (target: any, propertyKey: string, parameterIndex: number) {
        // 存储验证规则到原型
        const existingRequiredParams: any[] = Reflect.getMetadata('requiredParams', target, propertyKey) || [];
        existingRequiredParams.push({ index: parameterIndex, min });
        Reflect.defineMetadata('requiredParams', existingRequiredParams, target, propertyKey);
    };
}

// 元数据装饰器
const METADATA_KEY = 'design:paramtypes';

function Required(target: any, propertyKey: string, parameterIndex: number) {
    const existingParams: any[] = Reflect.getMetadata(METADATA_KEY, target, propertyKey) || [];
    existingParams[parameterIndex] = true;
    Reflect.defineMetadata(METADATA_KEY, existingParams, target, propertyKey);
}

// 类使用装饰器
@Frozen(true)
@Sealed
class UserService {
    @Non enumerable
    private internalId = 0;

    @Logger('[UserService]')
    private static dbConnection: string = 'mysql://localhost:3306';

    constructor(
        @Required public name: string,
        @Required public email: string
    ) {}

    @Readonly
    greet(@MinLength(2) name: string): string {
        return `Hello, ${name}!`;

        // 模拟获取用户

    }

    @Logger('[UserService]')
    updateUser(id: number, data: Partial<{ name: string; email: string }>) {
        console.log(`更新用户 ${id}:`, data);
        return { success: true };
    }
}
```

```typescript
    WrappedComponent: ComponentType<Props>

    function WithLoadingComponent({ isLoading = false, ...props }: Props & { isLoading?: boolean }) {
        if (isLoading) {
            return (
                <div style={{ padding: 20, textAlign: 'center' }}>
                    <div className="loading-spinner" />
                    <p>加载中...</p>
                </div>
            );
        }
        return <WrappedComponent {...props as Props} />;
    }

    WithLoadingComponent.displayName = `WithLoading(${getDisplayName(WrappedComponent)})`;
    return WithLoadingComponent;
}

// 2. withErrorBoundary装饰器
function withErrorBoundary<Props extends object>(
    WrappedComponent: ComponentType<Props>,
    errorFallback?: ComponentType<{ error: Error; retry: () => void }>
): ComponentType<Props> {
    function WithErrorBoundaryComponent(props: Props) {
        const [error, setError] = useState<Error | null>(null);

        useEffect(() => {
            const handler = (event: ErrorEvent) => {
                setError(event.error);
            };
            window.addEventListener('error', handler);
            return () => window.removeEventListener('error', handler);
        }, []);

        if (error) {
            if (errorFallback) {
                return React.createElement(errorFallback, {
                    error,
                    retry: () => setError(null),
                });
            }
            return (
                <div style={{ padding: 20, color: 'red' }}>
                    <h2>出错了</h2>
                    <p>{error.message}</p>
                    <button onClick={() => setError(null)}>重试</button>
                </div>
            );
        }

        return <WrappedComponent {...props} />;
    }

    WithErrorBoundaryComponent.displayName = `WithErrorBoundary(${getDisplayName(WrappedComponent)})`;
    return WithErrorBoundaryComponent;
}

// 3. withAuth装饰器
interface AuthProps {

    isLoading?: boolean;

function withAuth<Props extends AuthProps>(

): ComponentType<Omit<Props, keyof AuthProps>> {
    function WithAuthComponent(props: Omit<Props, keyof AuthProps>) {

    }

}

// 4. withLogger装饰器工厂
function withLogger(prefix: string) {
    return function <Props extends object>(WrappedComponent: ComponentType<Props>) {
        function WithLoggerComponent(props: Props) {
            useEffect(() => {

            }, [props]);

        }

        WithLoggerComponent.displayName = `WithLogger(${getDisplayName(WrappedComponent)})`;
        return WithLoggerComponent;
    };
}

// 装饰器组合
function compose<Props extends object>(
    ...hocs: Array<(component: ComponentType<any>) => ComponentType<any>>
): (component: ComponentType<Props>) => ComponentType<Props> {
    return (component: ComponentType<Props>) => {
        return hocs.reduce((acc, hoc) => hoc(acc), component);
    };
}
```

```typescript
// 装饰器工厂：日志记录
function Log(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    const original = descriptor.value;

    descriptor.value = function (...args: any[]) {
        const req: Request = args[0];
        const start = Date.now();

        console.log(`[${new Date().toISOString()}] ${req.method} ${req.path} 开始`);

        const result = original.apply(this, args);

        // 如果返回Promise
        if (result && typeof result.then === 'function') {
            result.then(() => {
                const duration = Date.now() - start;
                console.log(`[${new Date().toISOString()}] ${req.method} ${req.path} 耗时: ${duration}ms`);
            }).catch((err: Error) => {
                console.error(`[${new Date().toISOString()}] ${req.method} ${req.path} 错误:`, err.message);
            });
        } else {
            const duration = Date.now() - start;
            console.log(`[${new Date().toISOString()}] ${req.method} ${req.path} 耗时: ${duration}ms`);
        }

        return result;
    };

    return descriptor;
}

// 装饰器工厂：认证检查
function Auth(required: boolean = true) {
    return function (target: any, propertyKey: string, descriptor: PropertyDescriptor) {
        const original = descriptor.value;

        descriptor.value = async function (...args: any[]) {
            const req: Request = args[0];
            const res: Response = args[1];

            // 检查token
            const token = req.headers.authorization?.replace('Bearer ', '');

            if (required && !token) {
                return res.status(401).json({ error: '未授权访问' });
            }

            if (token) {
                // 验证token（实际项目中调用JWT验证）
                const user = await verifyToken(token);
                if (!user && required) {
                    return res.status(403).json({ error: '无效的令牌' });
                }
                (req as any).user = user;
            }

            return original.apply(this, args);
        };

        return descriptor;
    };
}

// 装饰器工厂：参数验证
function Validate(schema: { body?: any; query?: any; params?: any }) {
    return function (target: any, propertyKey: string, descriptor: PropertyDescriptor) {
        const original = descriptor.value;

        descriptor.value = function (...args: any[]) {
            const req: Request = args[0];
            const res: Response = args[1];

            // 验证请求体
            if (schema.body) {
                const errors = validateObject(schema.body, req.body);
                if (errors.length > 0) {
                    return res.status(400).json({ errors });
                }
            }

            // 验证查询参数
            if (schema.query) {
                const errors = validateObject(schema.query, req.query);
                if (errors.length > 0) {
                    return res.status(400).json({ errors });
                }
            }

            // 验证路由参数
            if (schema.params) {
                const errors = validateObject(schema.params, req.params);
                if (errors.length > 0) {
                    return res.status(400).json({ errors });
                }
            }

            return original.apply(this, args);
        };

        return descriptor;
    };
}

// 简单的对象验证函数
function validateObject(schema: any, data: any): string[] {
    const errors: string[] = [];
    for (const key in schema) {
        if (schema[key].required && !data[key]) {
            errors.push(`${key} 是必填字段`);
        }
        if (schema[key].type && typeof data[key] !== schema[key].type) {
            errors.push(`${key} 类型错误，期望${schema[key].type}`);
        }
    }
    return errors;
}

// 模拟token验证

    if (token === 'valid_token') {
        return { id: '1', name: '张三' };
    }
    return null;
}

            data: { id: 'new_user_id', name, email },
        });
    }

    @Log
    @Auth(false) // 可选认证
    static async getUser(req: Request, res: Response) {

        console.log(`获取用户信息, 当前用户: ${user?.name || '游客'}`);

    }
```

## 6. 外观模式（Facade Pattern）

### 6.1 生活类比：汽车启动

```
生活场景：启动汽车

问题分析：
- 启动汽车需要操作多个子系统
  - 发动机控制
  - 变速箱控制
  - 燃油喷射
  - 点火系统
  - 电池管理

错误设计：
- 让驾驶员直接操作每个子系统
- 驾驶员需要了解所有子系统的工作原理

正确设计（外观模式）：
- 提供一键启动按钮
- 驾驶员只需按下启动键
- 内部自动完成所有复杂操作

外观模式核心思想：
为复杂的子系统提供一个统一的接口
让客户端更容易使用子系统，而不需要了解子系统的复杂性
```

### 6.2 解决的问题

```
外观模式解决的核心问题：简化复杂系统的接口

典型场景：
1. 复杂库/框架封装
   - THREE.js 的复杂3D API
   - 提供简化的组件接口

2. 微服务聚合
   - 聚合多个微服务的调用
   - 提供统一的API网关

3. 第三方SDK封装
   - 封装复杂的第三方支付
   - 提供简单的业务接口

4. 系统初始化
   - 封装复杂初始化流程
   - 提供一键启动接口
```

```typescript
class OrderFacade {
    private inventory: InventoryService;
    private payment: PaymentService;
    private shipping: ShippingService;
    private notification: NotificationService;
    private orderRepo: OrderRepository;

    constructor() {
        // 初始化所有子系统
        this.inventory = new InventoryService();
        this.payment = new PaymentService();
        this.shipping = new ShippingService();
        this.notification = new NotificationService();
        this.orderRepo = new OrderRepository();
    }

    // 统一的下单接口
    async createOrder(params: {
        userId: string;
        productId: string;
        quantity: number;
        price: number;
        paymentMethod: string;
        shippingAddress: string;
        email: string;
        phone: string;
    }): Promise<{ success: boolean; orderId?: string; message: string }> {
        console.log('\n========== 开始处理订单 ==========');
        console.log(`商品: ${params.productId}, 数量: ${params.quantity}, 总价: ${params.price}`);

        try {
            // 1. 检查库存
            const hasStock = await this.inventory.checkStock(params.productId, params.quantity);
            if (!hasStock) {
                return { success: false, message: '库存不足' };
            }

            // 2. 创建订单记录
            const orderId = `ORD_${Date.now()}`;
            const order = {
                id: orderId,
                userId: params.userId,
                productId: params.productId,
                quantity: params.quantity,
                price: params.price,
                status: 'pending',
            };
            await this.orderRepo.save(order);

            // 3. 预留库存
            const reservationId = await this.inventory.reserveStock(params.productId, params.quantity);

            // 4. 处理支付
            const paymentResult = await this.payment.processPayment(
                params.userId,
                params.price,
                params.paymentMethod
            );

            if (!paymentResult.success) {
                // 支付失败，释放库存
                await this.inventory.releaseStock(reservationId);
                return { success: false, message: '支付失败' };
            }

            // 5. 更新订单状态
            order.status = 'paid';
            await this.orderRepo.update(order);

            // 6. 创建物流
            const trackingNumber = await this.shipping.createShipment(orderId, params.shippingAddress);

            // 7. 更新订单状态
            order.status = 'shipped';
            order.trackingNumber = trackingNumber;
            await this.orderRepo.update(order);

            // 8. 发送通知
            await this.notification.sendEmail(
                params.email,
                '订单确认',
                `您的订单 ${orderId} 已成功支付，物流单号: ${trackingNumber}`
            );
            await this.notification.sendSMS(params.phone, `订单已发货: ${trackingNumber}`);

            console.log('========== 订单处理完成 ==========\n');

            return {
                success: true,
                orderId,
                message: '订单创建成功',
            };

        } catch (error) {
            console.error('订单处理失败:', error);
            return { success: false, message: '系统错误，请稍后重试' };
        }
    }

    // 统一取消订单接口
    async cancelOrder(orderId: string): Promise<{ success: boolean; message: string }> {
        console.log('\n========== 取消订单 ==========');

        try {
            // 1. 查询订单
            const order = await this.orderRepo.findById(orderId);
            if (!order) {
                return { success: false, message: '订单不存在' };
            }

            if (order.status === 'shipped') {
                return { success: false, message: '已发货的订单无法取消' };
            }

            // 2. 退款
            if (order.status === 'paid') {
                await this.payment.refundPayment(order.transactionId);
            }

            // 3. 更新状态
            order.status = 'cancelled';
            await this.orderRepo.update(order);

            console.log('========== 取消完成 ==========\n');

            return { success: true, message: '订单已取消' };

        } catch (error) {
            console.error('取消失败:', error);
            return { success: false, message: '系统错误' };
        }
    }
}
```

```typescript
function useApi() {
    const [loading, setLoading] = useState(false);
    const [error, setError] = useState<string | null>(null);

        setLoading(true);
        setError(null);
        try {
            return await fn();
        } catch (e: any) {
            setError(e.message);
            return null;
        } finally {
            setLoading(false);
        }
    };

    return { loading, error, api, wrap };
}
```

## 7. 享元模式（Flyweight Pattern）

### 7.1 生活类比：打印店模式

```
生活场景：打印店打印文件

问题分析：
- 100个学生同时打印各自的论文
- 每个论文虽然内容不同，但页眉页脚相同
- 如果每个学生都创建新的页眉页脚模板，内存浪费严重

解决方案（享元模式）：
- 共享相同的页眉页脚模板
- 只创建一份模板，多个文档共享
- 每个文档只保存独特的部分（内容）

享元模式核心思想：
运用共享技术有效地支持大量细粒度的对象
将对象中不变的部分共享，减少内存占用
```

### 7.2 解决的问题

```
享元模式解决的核心问题：大量相似对象的内存优化

典型场景：
1. 文本编辑器
   - 大量字符共享字体样式
   - 每个字符只存储位置和字符值

2. 游戏开发
   - 大量子弹共享纹理
   - 大量树木共享模型

3. 图形系统
   - SVG图标复用
   - 地图marker共享

4. 缓存系统
   - 热点数据缓存
   - 共享计算结果
```

```typescript
// ========== 场景：文本编辑器字符对象 ==========

// 字符样式（享元的内部状态，可共享）
interface CharacterStyle {
    fontFamily: string;
    fontSize: number;
    color: string;
    bold: boolean;
    italic: boolean;
}

// 具体享元：字符样式
class ConcreteCharacterStyle implements CharacterStyle {
    constructor(
        public fontFamily: string,
        public fontSize: number,
        public color: string,
        public bold: boolean,
        public italic: boolean
    ) {}

    equals(other: CharacterStyle): boolean {
        return (
            this.fontFamily === other.fontFamily &&
            this.fontSize === other.fontSize &&
            this.color === other.color &&
            this.bold === other.bold &&
            this.italic === other.italic
        );
    }
}

// 享元工厂：管理样式对象
class CharacterStyleFactory {
    private styles: Map<string, CharacterStyle> = new Map();

    // 获取样式key
    private getStyleKey(style: CharacterStyle): string {
        return `${style.fontFamily}_${style.fontSize}_${style.color}_${style.bold}_${style.italic}`;
    }

    // 获取或创建样式
    getStyle(style: CharacterStyle): CharacterStyle {
        const key = this.getStyleKey(style);

        if (this.styles.has(key)) {
            console.log(`[享元工厂] 复用已有样式: ${key}`);
            return this.styles.get(key)!;
        }

        console.log(`[享元工厂] 创建新样式: ${key}`);
        this.styles.set(key, style);
        return style;
    }

    // 获取样式数量
    getStyleCount(): number {
        return this.styles.size;
    }
}

// 享元对象：单个字符（外部状态：位置，内部状态：样式）
interface Character {
    char: string;
    style: CharacterStyle;
    position: { x: number; y: number };
}

// 具体享元：字符
class CharacterFlyweight {
    // 内部状态：字符值和样式
    private char: string;
    private style: CharacterStyle;

    constructor(char: string, style: CharacterStyle) {
        this.char = char;
        this.style = style;
    }

    // 在指定位置渲染
    render(position: { x: number; y: number }): string {
        const boldTag = this.style.bold ? '<b>' : '';
        const boldEndTag = this.style.bold ? '</b>' : '';
        const italicTag = this.style.italic ? '<i>' : '';
        const italicEndTag = this.style.italic ? '</i>' : '';

        return `<span style="
            position:absolute;
            left:${position.x}px;
            top:${position.y}px;
            font-family:${this.style.fontFamily};
            font-size:${this.style.fontSize}px;
            color:${this.style.color};
        ">${boldTag}${italicTag}${this.char}${italicEndTag}${boldEndTag}</span>`;
    }

    getChar(): string {
        return this.char;
    }

    getStyle(): CharacterStyle {
        return this.style;
    }
}

// 享元管理器
class TextEditor {
    private characters: CharacterFlyweight[] = [];
    private styleFactory: CharacterStyleFactory;

    constructor() {
        this.styleFactory = new CharacterStyleFactory();
    }

    // 添加字符
    addChar(char: string, style: CharacterStyle, position: { x: number; y: number }): void {
        // 获取或创建样式（共享）
        const sharedStyle = this.styleFactory.getStyle(style);

        // 创建享元字符
        const charFlyweight = new CharacterFlyweight(char, sharedStyle);
        this.characters.push(charFlyweight);
    }

    // 渲染文本
    render(): string {
        return this.characters.map(c => c.render({ x: 0, y: 0 })).join('');
    }

    // 获取统计信息
    getStats(): { charCount: number; styleCount: number; savedMemory: number } {
        const uniqueStyles = this.styleFactory.getStyleCount();
        // 估算节省的内存：假设每个样式对象100字节
        const savedMemory = (this.characters.length - uniqueStyles) * 100;

        return {
            charCount: this.characters.length,
            styleCount: uniqueStyles,
            savedMemory,
        };
    }
}

    editor.addChar('l', defaultStyle, { x: 40, y: 0 });

    // 大量使用相同默认样式的字符
```

```typescript
// ========== 场景：游戏中的子弹系统 ==========

// 子弹配置（享元的内部状态）
interface BulletConfig {
    texture: string;
    speed: number;
    damage: number;
    size: { width: number; height: number };
    color: string;
}

// 子弹实例（外部状态）
interface BulletInstance {
    id: string;
    x: number;
    y: number;
    directionX: number;
    directionY: number;
    createdAt: number;
}

// 享元工厂
class BulletFactory {
    private bullets: Map<string, BulletConfig> = new Map();

    registerBulletType(type: string, config: BulletConfig): void {
        if (!this.bullets.has(type)) {
            this.bullets.set(type, config);
            console.log(`[BulletFactory] 注册子弹类型: ${type}`);
        }
    }

    getBulletConfig(type: string): BulletConfig | undefined {
        return this.bullets.get(type);
    }

    getBulletTypeCount(): number {
        return this.bullets.size;
    }
}

// 子弹管理器（享元客户端）
class BulletManager {
    private factory: BulletFactory;
    private activeBullets: Map<string, BulletInstance> = new Map();
    private bulletInstances: Map<string, { config: BulletConfig; count: number }> = new Map();

    constructor(factory: BulletFactory) {
        this.factory = factory;
    }

    // 创建子弹
    createBullet(type: string, x: number, y: number, dirX: number, dirY: number): string | null {
        const config = this.factory.getBulletConfig(type);
        if (!config) {
            console.error(`[BulletManager] 未知子弹类型: ${type}`);
            return null;
        }

        const id = `bullet_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`;

        const instance: BulletInstance = {
            id,
            x,
            y,
            directionX: dirX,
            directionY: dirY,
            createdAt: Date.now(),
        };

        this.activeBullets.set(id, instance);

        // 统计
        const key = `${type}`;
        if (this.bulletInstances.has(key)) {
            this.bulletInstances.get(key)!.count++;
        } else {
            this.bulletInstances.set(key, { config, count: 1 });
        }

        return id;
    }

    // 更新子弹位置
    updateBullet(id: string, deltaTime: number): void {
        const bullet = this.activeBullets.get(id);
        if (!bullet) return;

        const config = this.getBulletConfig(id);
        if (!config) return;

        bullet.x += bullet.directionX * config.speed * deltaTime;
        bullet.y += bullet.directionY * config.speed * deltaTime;
    }

    // 获取子弹配置
    private getBulletConfig(bulletId: string): BulletConfig | undefined {
        for (const [, { config }] of this.bulletInstances) {
            return config;
        }
        return undefined;
    }

    // 移除子弹
    removeBullet(id: string): void {
        this.activeBullets.delete(id);
    }

    // 获取统计
    getStats(): { activeCount: number; typeCount: number; memorySaving: number } {
        const activeCount = this.activeBullets.size;
        const typeCount = this.factory.getBulletTypeCount();

        // 假设每个子弹配置500字节，实例数据100字节
        const totalInstanceMemory = activeCount * 100;
        const totalConfigMemory = typeCount * 500;
        // 如果不使用享元，内存占用
        const withoutFlyweight = activeCount * (500 + 100);
        // 节省内存
        const memorySaving = withoutFlyweight - (totalInstanceMemory + totalConfigMemory);

        return { activeCount, typeCount, memorySaving };
    }
}
```

## 8. 代理模式（Proxy Pattern）

### 8.1 生活类比：房产中介

```
生活场景：租房找房产中介

问题分析：
```

#### 8.3.2 保护代理：权限验证

```typescript
// ========== 场景：保护代理权限控制 ==========

// 用户接口
interface User {
    getName(): string;
    getRole(): string;
    hasPermission(permission: string): boolean;
}

// 真实用户
class RealUser implements User {
    private name: string;
    private role: string;
    private permissions: string[];

    constructor(name: string, role: string, permissions: string[]) {
        this.name = name;
        this.role = role;
        this.permissions = permissions;
    }

    getName(): string {
        return this.name;
    }

    getRole(): string {
        return this.role;
    }

    hasPermission(permission: string): boolean {
        return this.permissions.includes(permission);
    }
}

// 资源接口
interface Resource {
    read(): void;
    write(): void;
    delete(): void;
    admin(): void;
}

// 真实资源
class RealResource implements Resource {
    private resourceId: string;

    constructor(resourceId: string) {
        this.resourceId = resourceId;
    }

    read(): void {
        console.log(`[RealResource] 读取资源 ${this.resourceId}`);
    }

    write(): void {
        console.log(`[RealResource] 写入资源 ${this.resourceId}`);
    }

    delete(): void {
        console.log(`[RealResource] 删除资源 ${this.resourceId}`);
    }

    admin(): void {
        console.log(`[RealResource] 管理员操作资源 ${this.resourceId}`);
    }
}

// 保护代理：权限验证
class ProtectedResource implements Resource {
    private realResource: RealResource;
    private user: User | null;

    constructor(resourceId: string) {
        this.realResource = new RealResource(resourceId);
        this.user = null;
    }

    // 设置当前用户
    setUser(user: User | null): void {
        console.log(`[ProtectedResource] 用户设置: ${user?.getName() || '未登录'}`);
        this.user = user;
    }

    private checkPermission(action: string): boolean {
        if (!this.user) {
            console.log(`[ProtectedResource] 未登录，无权${action}`);
            return false;
        }

        const permissionMap: Record<string, string> = {
            read: 'read',
            write: 'write',
            delete: 'delete',
            admin: 'admin',
        };

        const requiredPermission = permissionMap[action];
        if (!requiredPermission) {
            return true;
        }

        if (this.user.hasPermission(requiredPermission)) {

            return true;
        }

        console.log(`[ProtectedResource] 权限不足: ${this.user.getName()} 不可${action}`);
        return false;
    }

    read(): void {
        if (!this.checkPermission('read')) return;
        this.realResource.read();
    }

    write(): void {
        if (!this.checkPermission('write')) return;
        this.realResource.write();
    }

    delete(): void {
        if (!this.checkPermission('delete')) return;
        this.realResource.delete();
    }

    admin(): void {
        if (!this.checkPermission('admin')) return;
        this.realResource.admin();
    }
}
```

```typescript
class CachedDataService implements DataService {
    private realService: RealDataService;
    private cache: Map<string, { data: any; timestamp: number }> = new Map();
    private ttl: number; // 缓存有效期（毫秒）

    constructor(realService: RealDataService, ttl: number = 5000) {
        this.realService = realService;
        this.ttl = ttl;

        const cached = this.cache.get(key);

        // 检查缓存是否存在且有效
        if (cached && Date.now() - cached.timestamp < this.ttl) {
            console.log(`[CachedDataService] 从缓存获取: ${key}`);
            return cached.data;
        }

        // 缓存不存在或已过期，从真实服务获取
        console.log(`[CachedDataService] 缓存未命中，获取: ${key}`);
        const data = await this.realService.fetchData(key);

        // 更新缓存
        this.cache.set(key, {
            data,
            timestamp: Date.now(),
        });

        return data;
    }

    clearCacheItem(key: string): void {
        this.cache.delete(key);
        console.log(`[CachedDataService] 清除缓存: ${key}`);
    }

    // 清除所有缓存
    clearCache(): void {
        this.cache.clear();
        console.log('[CachedDataService] 清除所有缓存');
    }

    // 获取缓存统计
    getStats(): { size: number; keys: string[] } {
        return {
            size: this.cache.size,
            keys: Array.from(this.cache.keys()),
        };
```

```typescript
// 验证代理
function createValidator<T extends object>(target: T, validators: Record<string, (value: any) => boolean>): T {
    return new Proxy(target, {
        set(obj, prop, value) {
            if (validators[prop as string]) {
                if (!validators[prop as string](value)) {
                    throw new TypeError(`属性 ${String(prop)} 的值无效: ${value}`);
                }
            }
            obj[prop as keyof T] = value;
            return true;
        },
        get(obj, prop) {
            if (prop in obj) {
                return obj[prop as keyof T];
            }
            throw new ReferenceError(`属性 ${String(prop)} 不存在`);
        },
    });
}

// 懒加载代理
function createLazyLoader<T extends object>(initializer: () => T): T {
    let initialized = false;
    let instance: any;

    return new Proxy({} as T, {
        get(_target, prop) {
            if (!initialized) {
                initialized = true;
                instance = initializer();
            }
            return (instance as any)[prop];
        },
    });
}

// 只读代理
function createReadonly<T extends object>(target: T): T {
    return new Proxy(target, {
        set() {
            throw new Error('只读对象，不能修改');
        },
        deleteProperty() {
            throw new Error('只读对象，不能删除');
        },
        get(target, prop) {
            const value = (target as any)[prop];
            // 如果属性是对象，也返回只读代理
            if (value && typeof value === 'object') {
                return createReadonly(value);
            }
            return value;
        },
    });
}

            return Reflect.set(obj, prop, value, receiver);
        },
    });
}
```

```typescript
// 数据获取Hook代理
function useDataProxy<T>(url: string, options?: { useCache?: boolean; refetchInterval?: number }) {
    const [data, setData] = useState<T | null>(null);
    const [loading, setLoading] = useState(true);
    const [error, setError] = useState<Error | null>(null);

    const fetchData = useCallback(async () => {
        try {
            setLoading(true);
            const result = await fetchFromAPI<T>(url);
            setData(result);
            setError(null);
        } catch (e) {
            setError(e as Error);
        } finally {
            setLoading(false);
        }
    }, [url]);

    useEffect(() => {
        fetchData();

        // 定时刷新
        if (options?.refetchInterval) {
            const interval = setInterval(fetchData, options.refetchInterval);
            return () => clearInterval(interval);
        }
    }, [fetchData, options?.refetchInterval]);

    return { data, loading, error, refetch: fetchData };
}

    if (error) return <div className="error">错误: {error.message}</div>;

    return <>{children(data, loading, error)}</>;
}

        this.cache.clear();
    }
}
```

```typescript
// 模拟简化版Express
class Express {
    private middlewares: Array<(req: any, res: any, next: Function) => void> = [];

    use(fn: (req: any, res: any, next: Function) => void) {
        this.middlewares.push(fn);
        return this; // 支持链式调用（装饰器模式）
    }

        let index = 0;

        const next = async () => {
            if (index >= this.middlewares.length) {
                return;
            }
            const fn = this.middlewares[index++];
            await fn(req, res, next);
        };

        await next();
    }
}
```

### 结构型模式核心要点

```
结构型模式学习总结：

1. 适配器模式（Adapter）
   - 解决：接口不兼容问题
   - 关键：将现有接口转换为期望接口
   - 应用：第三方库集成、多数据源整合

2. 桥接模式（Bridge）
   - 解决：多维度独立变化问题
   - 关键：分离抽象与实现
   - 应用：跨平台开发、多渠道通知

3. 组合模式（Composite）
   - 解决：树形结构统一处理问题
   - 关键：整体与部分一致性
   - 应用：文件系统、组织架构、UI组件树

4. 装饰器模式（Decorator）
   - 解决：动态添加职责问题
   - 关键：包装而非继承
   - 应用：中间件、组件增强、类方法增强

5. 外观模式（Facade）
   - 解决：复杂接口简化问题
   - 关键：提供统一入口
   - 应用：系统封装、API网关、微服务聚合

6. 享元模式（Flyweight）
   - 解决：大量相似对象内存问题
   - 关键：共享内部状态
   - 应用：文本编辑器、游戏对象、图标系统

7. 代理模式（Proxy）
   - 解决：对象访问控制问题
   - 关键：对象包装
   - 应用：延迟加载、缓存、权限控制
```
