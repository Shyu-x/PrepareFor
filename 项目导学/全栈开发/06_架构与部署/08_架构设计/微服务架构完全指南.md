## 1. 微服务架构基础

### 1.1 什么是微服务

微服务架构（Microservices Architecture）是一种软件架构风格，它将应用程序构建为一组小型、自治的服务，每个服务运行在独立的进程中，通过轻量级通信机制（通常是HTTP API或消息队列）进行交互。

**微服务的核心特征：**

- **独立性**：每个服务可以独立开发、部署、扩展和维护
- **单一职责**：每个服务专注于完成单一功能
- **分散治理**：每个服务可以使用不同的技术栈
- **自动化部署**：通过CI/CD流水线实现自动化部署
- **基础设施自动化**：使用容器和编排工具管理基础设施

```typescript
// 传统的单体架构示例
// 所有功能都在一个大型应用中
class MegaApplication {
  // 用户管理模块
  async createUser(user: User): Promise<void> { /* ... */ }
  async authenticate(credentials: Credentials): Promise<Token> { /* ... */ }

  // 订单管理模块
  async createOrder(order: Order): Promise<void> { /* ... */ }
  async processPayment(payment: Payment): Promise<void> { /* ... */ }

  // 库存管理模块
  async updateStock(productId: string, quantity: number): Promise<void> { /* ... */ }
  async checkStock(productId: string): Promise<number> { /* ... */ }

  // 物流管理模块
  async shipOrder(orderId: string): Promise<void> { /* ... */ }
  async trackShipment(trackingId: string): Promise<ShipmentStatus> { /* ... */ }
}

// 微服务架构示例
// 每个服务独立，只负责特定功能

// 用户服务 - 专注用户管理和认证
class UserService {
  async createUser(user: User): Promise<void> { /* 只处理用户创建 */ }
  async authenticate(credentials: Credentials): Promise<Token> { /* 只处理认证 */ }
}

// 订单服务 - 专注订单处理
class OrderService {
  async createOrder(order: Order): Promise<void> { /* 只处理订单创建 */ }
  async getOrder(orderId: string): Promise<Order> { /* 只获取订单信息 */ }
}

// 库存服务 - 专注库存管理
class InventoryService {
  async updateStock(productId: string, quantity: number): Promise<void> { /* 只更新库存 */ }
  async checkStock(productId: string): Promise<number> { /* 只查询库存 */ }
}

// 物流服务 - 专注物流配送
class LogisticsService {
  async shipOrder(orderId: string): Promise<void> { /* 只处理发货 */ }
  async trackShipment(trackingId: string): Promise<ShipmentStatus> { /* 只追踪物流 */ }
}
```

### 1.2 为什么需要微服务

随着业务规模的增长，单体架构会面临诸多挑战：

**单体架构的问题：**

| 问题类型 | 具体表现 | 影响 |
|----------|----------|------|
| **复杂度高** | 代码库庞大，模块间耦合严重 | 维护困难，修改风险高 |
| **部署周期长** | 任何小改动都需要重新部署整个应用 | 敏捷性降低 |
| **扩展性差** | 无法针对瓶颈模块单独扩展 | 资源浪费 |
| **技术锁定** | 所有模块必须使用相同技术栈 | 无法利用新技术 |
| **故障扩散** | 一个模块的故障可能导致整个系统崩溃 | 可用性降低 |
| **团队协作难** | 多团队在同一代码库上工作冲突多 | 开发效率低 |

**微服务架构的优势：**

```typescript
// 微服务架构优势示例

// 优势1：独立扩展
// 当订单量增长时，只需要扩展订单服务
class ScalingExample {
  // 在Kubernetes中，可以针对特定服务进行扩展
  // 订单服务 - 高峰期扩展到10个实例
  // kubectl scale deployment order-service --replicas=10

  // 用户服务 - 平时保持3个实例
  // kubectl scale deployment user-service --replicas=3

  // 库存服务 - 低流量，1个实例即可
  // kubectl scale deployment inventory-service --replicas=1
}

// 优势2：技术多样性
// 不同服务可以使用最适合的技术栈
const technologyStack = {
  // 高性能计算服务 - 使用Rust
  'analytics-service': { language: 'Rust', framework: 'Actix' },

  // AI/ML服务 - 使用Python
  'ml-service': { language: 'Python', framework: 'FastAPI' },

  // Web API服务 - 使用Node.js
  'api-gateway': { language: 'Node.js', framework: 'Express' },

  // 实时通信服务 - 使用Go
  'websocket-service': { language: 'Go', framework: 'Gin' },

  // 批处理服务 - 使用Java
  'batch-service': { language: 'Java', framework: 'Spring Boot' }
};

// 优势3：故障隔离
// 单个服务故障不会导致整个系统崩溃
class FaultIsolationExample {
  // 当库存服务故障时
  async processOrder(order: Order) {
    try {
      // 可以优雅降级，提示用户库存服务暂时不可用
      const stock = await this.inventoryService.checkStock(order.productId);

      if (stock <= 0) {
        // 降级处理：允许用户下单但延迟发货
        return { status: 'pending', message: '库存不足，等待补货' };
      }

      await this.orderService.createOrder(order);
      return { status: 'success' };
    } catch (error) {
      // 即使库存服务故障，订单服务仍可继续
      if (error.service === 'inventory') {
        // 记录日志，发送告警，但不影响核心流程
        this.logWarning('库存服务故障，进入降级模式');
        return { status: 'degraded', message: '部分功能暂停' };
      }
      throw error;
    }
  }
}
```

### 1.4 微服务设计原则

微服务架构需要遵循一系列设计原则来确保系统的可维护性、可扩展性和可靠性。

**核心设计原则：**

```typescript
// 原则1：高内聚低耦合
// 高内聚：相关功能放在一起，每个服务职责单一
// 低耦合：服务间依赖最小化，通过接口通信

/**
 * 高内聚示例 - 每个服务职责单一
 */

// 用户服务 - 只负责用户相关业务
class UserService {
  // 用户注册、登录、信息管理
  async register(userData: UserData): Promise<User> { /* ... */ }
  async login(credentials: Credentials): Promise<AuthToken> { /* ... */ }
  async updateProfile(userId: string, profile: Profile): Promise<void> { /* ... */ }
  async getUserById(userId: string): Promise<User> { /* ... */ }
}

// 订单服务 - 只负责订单相关业务
class OrderService {
  // 订单创建、查询、状态管理
  async createOrder(orderData: OrderData): Promise<Order> { /* ... */ }
  async getOrder(orderId: string): Promise<Order> { /* ... */ }
  async cancelOrder(orderId: string): Promise<void> { /* ... */ }
  async getUserOrders(userId: string): Promise<Order[]> { /* ... */ }
}

/**
 * 低耦合示例 - 服务间通过接口通信，不直接依赖
 */

// 定义服务接口（契约）
interface UserServiceInterface {
  getUserById(userId: string): Promise<User>;
  validateToken(token: string): Promise<boolean>;
}

// 订单服务不直接依赖用户服务，而是依赖接口
class OrderServiceLowCoupling {
  // 通过依赖注入，订单服务不直接创建用户服务实例
  constructor(private userService: UserServiceInterface) {}

  // 订单创建时验证用户，但不直接调用用户服务的具体实现
  async createOrder(orderData: OrderData): Promise<Order> {
    // 通过接口调用，隐藏具体实现
    const isValid = await this.userService.validateToken(orderData.token);

    if (!isValid) {
      throw new Error('无效的认证令牌');
    }

    return this.orderRepository.create(orderData);
  }
}

// 原则2：单一职责原则（SRP）
// 每个服务只负责一个业务领域

/**
 * 单一职责示例 - 避免服务功能膨胀
 */

// 错误：订单服务承担了太多职责
class FatOrderService {
  async createOrder() { /* 创建订单 */ }
  async processPayment() { /* 处理支付 - 应该是支付服务的职责 */ }
  async sendEmail() { /* 发送邮件 - 应该是通知服务的职责 */ }
  async generateInvoice() { /* 生成发票 - 应该是财务服务的职责 */ }
  async updateInventory() { /* 更新库存 - 应该是库存服务的职责 */ }
}

// 正确：每个服务职责单一
class OrderServiceSRP {
  async createOrder() { /* 只创建订单 */ }
  async getOrder() { /* 只获取订单 */ }
  async cancelOrder() { /* 只取消订单 */ }
}

class PaymentService {
  async processPayment() { /* 只处理支付 */ }
  async refundPayment() { /* 只处理退款 */ }
  async getPaymentStatus() { /* 只查询支付状态 */ }
}

class NotificationService {
  async sendEmail() { /* 只发送邮件 */ }
  async sendSMS() { /* 只发送短信 */ }
  async sendPush() { /* 只发送推送 */ }
}

class InventoryService {
  async updateStock() { /* 只更新库存 */ }
  async checkStock() { /* 只查询库存 */ }
  async reserveStock() { /* 只预留库存 */ }
}

// 原则3：无状态服务
// 服务不保存客户端会话状态，每个请求都包含所需信息

/**
 * 无状态服务示例
 */

// 错误：有状态服务 - 在内存中保存会话
class StatefulUserService {
  // 在内存中存储用户会话 - 这会导致无法扩展
  private userSessions: Map<string, UserSession> = new Map();

  async login(username: string, password: string): Promise<string> {
    const sessionId = this.generateSessionId();
    // 将会话存储在服务内存中
    this.userSessions.set(sessionId, {
      userId: this.getUserId(username),
      loginTime: Date.now()
    });
    return sessionId;
  }

  async getSession(sessionId: string): Promise<UserSession> {
    // 依赖内存存储，实例重启后会话丢失
    return this.userSessions.get(sessionId);
  }
}

// 正确：无状态服务 - 会话存储在外部
class StatelessUserService {
  // 使用Redis等外部存储保存会话
  constructor(private redisClient: RedisClient) {}

  async login(username: string, password: string): Promise<string> {
    const sessionId = this.generateSessionId();
    const user = await this.authenticate(username, password);

    // 将会话存储在Redis中，而非服务内存
    await this.redisClient.setex(
      `session:${sessionId}`,
      86400, // 24小时过期
      JSON.stringify({ userId: user.id, loginTime: Date.now() })
    );

    return sessionId;
  }

  async getSession(sessionId: string): Promise<UserSession | null> {
    // 从Redis获取会话，任何服务实例都可以访问
    const sessionData = await this.redisClient.get(`session:${sessionId}`);
    return sessionData ? JSON.parse(sessionData) : null;
  }
}
```

### 1.5 服务拆分策略

服务拆分是微服务架构设计的核心问题，合理的拆分策略能够最大化微服务的优势。

### 2.2 同步通信：gRPC

gRPC是Google开源的高性能RPC框架，使用Protocol Buffers作为序列化协议，适合高性能场景。

```typescript
/**
 * Protocol Buffers 定义文件

// user.proto - 用户服务Protobuf定义
// syntax指定使用proto3语法
syntax = "proto3";

// 包名，避免命名冲突

// go_package指定生成的Go代码的包路径

// 用户服务定义
service UserService {

/**
 * Node.js gRPC 服务端实现
 */

import * as grpc from '@grpc/grpc-js';
import * as protoLoader from '@grpc/proto-loader';

// 加载Protobuf定义
const PROTO_PATH = './proto/user.proto';

const packageDefinition = protoLoader.loadSync(PROTO_PATH, {
  keepCase: true,           // 保留字段名大小写
  longs: String,            // 将long类型转为String
  enums: String,            // 将枚举转为String
  defaults: true,           // 使用默认值
  oneofs: true              // 支持oneof
});

// 加载包定义
const userProto = grpc.loadPackageDefinition(packageDefinition) as any;

/**
 * 用户服务实现
 */
class UserServiceImpl {
  // 创建用户实现
  async createUser(
    call: grpc.ServerUnaryCall<CreateUserRequest, CreateUserResponse>,
    callback: grpc.sendUnaryData<CreateUserResponse>
  ) {
    try {
      const { username, email, password } = call.request;

      // 创建用户逻辑
      const user: User = {
        id: this.generateUserId(),
        username,
        email,
        createdAt: new Date().toISOString(),
        status: 'USER_STATUS_ACTIVE'
      };

      // 保存到数据库
      await this.userRepository.create(user);

      callback(null, { user });
    } catch (error) {
      callback({
        code: grpc.status.INTERNAL,
        message: error.message
      });
    }
  }

  // 获取用户实现
  async getUser(
    call: grpc.ServerUnaryCall<GetUserRequest, GetUserResponse>,
    callback: grpc.sendUnaryData<GetUserResponse>
  ) {
    try {
      const { userId } = call.request;

      const user = await this.userRepository.findById(userId);

      if (!user) {
        callback({
          code: grpc.status.NOT_FOUND,
          message: '用户不存在'
        });
        return;
      }

      callback(null, { user });
    } catch (error) {
      callback({
        code: grpc.status.INTERNAL,
        message: error.message
      });
    }
  }

  // 批量获取用户 - 服务器流式
  async batchGetUsers(call: grpc.ServerWritableStream<BatchGetUsersRequest, GetUserResponse>) {
    const { userIds } = call.request;

    // 遍历用户ID，逐个发送
    for (const userId of userIds) {
      const user = await this.userRepository.findById(userId);
      if (user) {
        call.write({ user });
      }
    }

    // 结束流
    call.end();
  }

  // 双向流式处理
  async processUserUpdates(call: grpc.ServerDuplexStream<ProcessUserRequest, ProcessUserResponse>) {
    call.on('data', async (request: ProcessUserRequest) => {
      // 处理每个请求
      const result = await this.processRequest(request);
      call.write(result);
    });

    call.on('end', () => {
      call.end();
    });

    call.on('error', (error) => {
      console.error('流处理错误:', error);
    });
  }
}

          else resolve(response.user);
        }
      );
    });
  }

  async batchGetUsers(userIds: string[]): Promise<User[]> {

    // 创建流
    const stream = this.client.batchGetUsers({ userIds });

    // 收集流数据
    return new Promise((resolve, reject) => {
      stream.on('data', (response: GetUserResponse) => {
        users.push(response.user);
      });

      stream.on('end', () => {
        resolve(users);
      });

      stream.on('error', reject);
    });
  }

  // 双向流调用
  async* processUserUpdates(requests: ProcessUserRequest[]): AsyncGenerator<ProcessUserResponse> {
    const stream = this.client.processUserUpdates();

    for (const request of requests) {
      stream.write(request);
    }

    stream.end();

    for await (const response of stream) {
      yield response;
    }
  }
```

### 2.3 异步通信：消息队列

消息队列是微服务间异步通信的核心组件，用于解耦服务、提高系统吞吐量、实现最终一致性。

```typescript
// 消息队列通信示例

/**
 * 消息队列基础组件
 */

// 消息接口定义
interface Message {
  id: string;           // 消息唯一ID
  type: string;        // 消息类型
  payload: any;        // 消息内容
  timestamp: number;   // 时间戳
  headers: Record<string, string>; // 消息头
}

// 消息处理器接口
interface MessageHandler {
  handle(message: Message): Promise<void>;
}

/**
 * RabbitMQ 消息队列实现
 */

import amqp, { Connection, Channel, ConsumeMessage } from 'amqplib';

class RabbitMQService {
  private connection: Connection | null = null;
  private channel: Channel | null = null;

  // 连接到RabbitMQ服务器
  async connect(url: string): Promise<void> {
    this.connection = await amqp.connect(url);
    this.channel = await this.connection.createChannel();

    // 确保交换机存在
    await this.channel.assertExchange('order-exchange', 'topic', {
      durable: true  // 持久化
    });

    // 确保队列存在
    await this.channel.assertQueue('order-created-queue', {
      durable: true,  // 队列持久化
      arguments: {
        'x-message-ttl': 86400000  // 消息24小时过期
      }
    });

    console.log('已连接到RabbitMQ');
  }

  // 发布消息
  async publish(routingKey: string, message: Message): Promise<void> {
    if (!this.channel) {
      throw new Error('未连接到RabbitMQ');
    }

    const content = Buffer.from(JSON.stringify(message));

    // 发布消息到交换机
    this.channel.publish('order-exchange', routingKey, content, {
      persistent: true,     // 消息持久化
      contentType: 'application/json',
      headers: message.headers
    });

    console.log(`消息已发布: ${routingKey}`, message.id);
  }

  // 订阅消息
  async subscribe(
    queueName: string,
    routingKey: string,
    handler: MessageHandler
  ): Promise<void> {
    if (!this.channel) {
      throw new Error('未连接到RabbitMQ');
    }

    // 绑定队列到交换机
    await this.channel.bindQueue(queueName, 'order-exchange', routingKey);

    // 设置预取数量 - 一次最多处理的消息数
    await this.channel.prefetch(10);

    // 开始消费
    await this.channel.consume(queueName, async (msg: ConsumeMessage | null) => {
      if (!msg) return;

      try {
        const message: Message = JSON.parse(msg.content.toString());
        await handler.handle(message);

        // 确认消息处理成功
        this.channel!.ack(msg);
      } catch (error) {
        console.error('消息处理失败:', error);

        // 消息处理失败，拒绝并重新入队
        this.channel!.nack(msg, false, true);
      }
    });

    console.log(`已开始监听队列: ${queueName}`);
  }
}

/**
 * 库存服务 - 订阅订单事件
 */
class InventoryServiceWithMQ {
  constructor(private rabbitmq: RabbitMQService) {}

  // 初始化订阅
  async initialize(): Promise<void> {
    // 订阅订单创建事件 - 扣减库存
    await this.rabbitmq.subscribe(
      'inventory-order-queue',
      'order.created',
      {
        handle: async (message: Message) => {
          const { orderId, items } = message.payload;

          console.log(`收到订单创建事件: ${orderId}`);

          // 扣减库存
          for (const item of items) {
            await this.reserveStock(item.productId, item.quantity);
          }

          // 发布库存预留成功事件
          await this.rabbitmq.publish('inventory.reserved', {
            id: this.generateMessageId(),
            type: 'INVENTORY_RESERVED',
            payload: {
              orderId,
              items: items.map((i: any) => ({
                productId: i.productId,
                quantity: i.quantity
              })),
              reservedAt: Date.now()
            },
            timestamp: Date.now(),
            headers: {
              'correlation-id': orderId
            }
          });

          console.log(`库存预留完成: ${orderId}`);
        }
      }
    );

    // 订阅订单取消事件 - 释放库存
    await this.rabbitmq.subscribe(
      'inventory-release-queue',
      'order.cancelled',
      {
        handle: async (message: Message) => {
          const { orderId } = message.payload;

          console.log(`收到订单取消事件: ${orderId}`);

          // 查询预留的库存
          const reservations = await this.inventoryRepository.findByOrderId(orderId);

          // 释放库存
          for (const reservation of reservations) {
            await this.releaseStock(
              reservation.productId,
              reservation.quantity
            );
          }

          console.log(`库存已释放: ${orderId}`);
        }
      }
    );
  }

  // 预留库存
  private async reserveStock(productId: string, quantity: number): Promise<void> {
    const product = await this.productRepository.findById(productId);

    if (!product) {
      throw new Error(`产品不存在: ${productId}`);
    }

    if (product.stock < quantity) {
      throw new Error(`库存不足: ${productId}`);
    }

    // 扣减库存
    product.stock -= quantity;
    await this.productRepository.update(product);

    // 记录库存预留
    await this.inventoryRepository.create({
      productId,
      quantity,
      reservedAt: Date.now(),
      status: 'reserved'
    });
  }

  // 释放库存
  private async releaseStock(productId: string, quantity: number): Promise<void> {
    const product = await this.productRepository.findById(productId);

    // 增加库存
    product.stock += quantity;
    await this.productRepository.update(product);

    // 更新预留记录状态
    await this.inventoryRepository.updateStatus(productId, 'released');
  }
}
```

```typescript
    // 过滤出指定服务名称且健康的服务

      if (service.Service === serviceName) {
        // 检查服务健康状态
        const checks = await this.consul.agent.check.list();

        for (const [checkId, check] of Object.entries(checks as any)) {
          if (check.ServiceID === id && check.Status === 'passing') {
            healthyInstances.push({
              address: service.Address,
              port: service.Port
            });
            break;
          }
        }
      }
    }

    return healthyInstances;
  }

  // 监听服务变化
  async watchService(
    serviceName: string,
    callback: (services: Array<{ address: string; port: number }>) => void
  ): Promise<void> {
    let currentInstances: Array<{ address: string; port: number }> = [];

    // 定时检查服务列表
    const interval = setInterval(async () => {

      }
    }, 5000);

    // 返回停止监听的函数
    return () => clearInterval(interval);
  }

class ClientSideLoadBalancer {
  private serviceInstances: Map<string, Array<{
    address: string;
    port: number;
    weight: number;
  }>> = new Map();

  // 更新服务实例列表
  updateInstances(serviceName: string, instances: Array<{
    address: string;
    port: number;
  }>): void {
    this.serviceInstances.set(serviceName, instances.map(i => ({
      ...i,
      weight: 1  // 默认权重
    })));
  }

  // 选择服务实例 - 轮询策略
  selectRoundRobin(serviceName: string): { address: string; port: number } | null {
    const instances = this.serviceInstances.get(serviceName);
    if (!instances || instances.length === 0) return null;

    // 简单的轮询实现
    const index = Math.floor(Math.random() * instances.length);
    return {
      address: instances[index].address,
      port: instances[index].port
    };
  }

  // 选择服务实例 - 加权轮询策略
  selectWeightedRoundRobin(serviceName: string): { address: string; port: number } | null {
    const instances = this.serviceInstances.get(serviceName);
    if (!instances || instances.length === 0) return null;

    // 计算总权重
    const totalWeight = instances.reduce((sum, i) => sum + i.weight, 0);

    // 随机选择
    let random = Math.random() * totalWeight;

    for (const instance of instances) {
      random -= instance.weight;
      if (random <= 0) {
        return {
          address: instance.address,
          port: instance.port
        };
      }
    }

    // 兜底返回第一个
    return {
      address: instances[0].address,
      port: instances[0].port
    };
  }

  // 选择服务实例 - 最少连接策略
  private connectionCounts: Map<string, Map<string, number>> = new Map();

  selectLeastConnections(serviceName: string): { address: string; port: number } | null {
    const instances = this.serviceInstances.get(serviceName);
    if (!instances || instances.length === 0) return null;

    let minConnections = Infinity;
    let selectedInstance: { address: string; port: number } | null = null;

    for (const instance of instances) {
      const key = `${instance.address}:${instance.port}`;
      const connections = this.connectionCounts.get(serviceName)?.get(key) || 0;

      if (connections < minConnections) {
        minConnections = connections;
        selectedInstance = instance;
      }
    }

    return selectedInstance;
  }

  // 记录连接
  recordConnection(serviceName: string, address: string, port: number): void {
    const key = `${address}:${port}`;

    if (!this.connectionCounts.has(serviceName)) {
      this.connectionCounts.set(serviceName, new Map());
    }

    const current = this.connectionCounts.get(serviceName)!.get(key) || 0;
    this.connectionCounts.get(serviceName)!.set(key, current + 1);
  }

  // 释放连接
  releaseConnection(serviceName: string, address: string, port: number): void {
    const key = `${address}:${port}`;
    const current = this.connectionCounts.get(serviceName)?.get(key) || 1;

    this.connectionCounts.get(serviceName)?.set(key, Math.max(0, current - 1));
  }
}
```

### 2.5 熔断与降级

熔断器模式用于防止级联故障，降级机制在服务不可用时提供备用响应。

```typescript
// 熔断器与降级实现

/**
 * 熔断器状态
 */
enum CircuitState {
  CLOSED = 'closed',     // 关闭状态 - 正常调用
  OPEN = 'open',          // 打开状态 - 快速失败，不调用服务
  HALF_OPEN = 'half_open' // 半开状态 - 尝试调用，探测服务是否恢复
}

/**
 * 熔断器配置
 */
interface CircuitBreakerConfig {
  failureThreshold: number;      // 失败次数阈值，达到后打开熔断器
  successThreshold: number;     // 成功次数阈值，达到后关闭熔断器
  timeout: number;              // 超时时间（毫秒）
  halfOpenMaxCalls: number;    // 半开状态下的最大调用数
}

/**
 * 熔断器实现
 */
class CircuitBreaker {
  private state: CircuitState = CircuitState.CLOSED;
  private failureCount: number = 0;
  private successCount: number = 0;
  private lastFailureTime: number = 0;
  private halfOpenCalls: number = 0;

  constructor(private config: CircuitBreakerConfig) {}

  // 获取当前状态
  getState(): CircuitState {
    // 如果熔断器打开，检查是否超过超时时间
    if (this.state === CircuitState.OPEN) {
      const now = Date.now();
      if (now - this.lastFailureTime >= this.config.timeout) {
        // 转换为半开状态
        this.state = CircuitState.HALF_OPEN;
        this.halfOpenCalls = 0;
        console.log('熔断器从OPEN转换为HALF_OPEN');
      }
    }
    return this.state;
  }

  // 执行调用
  async execute<T>(
    operation: () => Promise<T>,
    fallback?: () => Promise<T>
  ): Promise<T> {
    const state = this.getState();

    // 熔断器打开状态，快速失败
    if (state === CircuitState.OPEN) {
      console.log('熔断器处于OPEN状态，快速失败');

      // 如果有降级逻辑，执行降级
      if (fallback) {
        console.log('执行降级逻辑');
        return fallback();
      }

      throw new Error('服务暂时不可用（熔断器打开）');
    }

    // 半开状态，检查是否允许调用
    if (state === CircuitState.HALF_OPEN) {
      if (this.halfOpenCalls >= this.config.halfOpenMaxCalls) {
        throw new Error('熔断器处于HALF_OPEN状态，已达到最大调用数');
      }
      this.halfOpenCalls++;
    }

    try {
      // 执行实际调用
      const result = await operation();

      // 调用成功
      this.onSuccess();

      return result;
    } catch (error) {
      // 调用失败
      this.onFailure();

      // 如果有降级逻辑，执行降级
      if (fallback) {
        console.log('执行降级逻辑');
        return fallback();
      }

      throw error;
    }
  }

  // 记录成功
  private onSuccess(): void {
    this.failureCount = 0;

    if (this.state === CircuitState.HALF_OPEN) {
      this.successCount++;

      // 半开状态下达到成功次数阈值，关闭熔断器
      if (this.successCount >= this.config.successThreshold) {
        this.state = CircuitState.CLOSED;
        this.successCount = 0;
        console.log('熔断器从HALF_OPEN转换为CLOSED');
      }
    }
  }

  // 记录失败
  private onFailure(): void {
    this.failureCount++;
    this.lastFailureTime = Date.now();

    if (this.state === CircuitState.HALF_OPEN) {
      // 半开状态下失败，重新打开熔断器
      this.state = CircuitState.OPEN;
      console.log('熔断器从HALF_OPEN转换为OPEN');
    } else if (this.failureCount >= this.config.failureThreshold) {
      // 达到失败阈值，打开熔断器
      this.state = CircuitState.OPEN;
      console.log('熔断器从CLOSED转换为OPEN');
    }
  }

  // 重置熔断器
  reset(): void {
    this.state = CircuitState.CLOSED;
    this.failureCount = 0;
    this.successCount = 0;
    this.lastFailureTime = 0;
    this.halfOpenCalls = 0;
  }
}

/**
 * 使用熔断器包装服务调用
 */

class ResilientOrderService {

  constructor(private paymentService: PaymentService) {

// 限流算法接口
interface RateLimiter {
  // 检查是否允许请求通过
  tryAcquire(): boolean;
  // 获取剩余令牌数
  getAvailablePermits(): number;
}

/**
 * 令牌桶限流器
 */
class TokenBucketRateLimiter implements RateLimiter {
  private tokens: number;
  private lastRefillTime: number;

  constructor(
    private capacity: number,    // 桶容量
    private refillRate: number   // 每秒补充的令牌数
  ) {
    this.tokens = capacity;
    this.lastRefillTime = Date.now();
  }

  // 补充令牌
  private refill(): void {
    const now = Date.now();
    const elapsed = (now - this.lastRefillTime) / 1000; // 转换为秒

    // 计算应该补充的令牌数
    const tokensToAdd = elapsed * this.refillRate;

    this.tokens = Math.min(this.capacity, this.tokens + tokensToAdd);
    this.lastRefillTime = now;
  }

  // 尝试获取令牌
  tryAcquire(): boolean {
    this.refill();

    if (this.tokens >= 1) {
      this.tokens -= 1;
      return true;
    }

    return false;
  }

  // 获取可用令牌数
  getAvailablePermits(): number {
    this.refill();
    return Math.floor(this.tokens);
  }
}

/**
 * 滑动窗口限流器
 */
class SlidingWindowRateLimiter implements RateLimiter {
  private requests: number[] = [];

  constructor(
    private windowSize: number,    // 窗口大小（毫秒）
    private maxRequests: number    // 窗口内最大请求数
  ) {}

  // 清理过期的请求记录
  private cleanOldRequests(): void {
    const now = Date.now();
    const windowStart = now - this.windowSize;

    // 移除窗口外的请求
    this.requests = this.requests.filter(timestamp => timestamp > windowStart);
  }

  // 尝试获取令牌
  tryAcquire(): boolean {
    this.cleanOldRequests();

    if (this.requests.length < this.maxRequests) {
      this.requests.push(Date.now());
      return true;
    }

    return false;
  }

  // 获取可用请求数
  getAvailablePermits(): number {
    this.cleanOldRequests();
    return this.maxRequests - this.requests.length;
  }
}

/**
 * 使用限流器保护服务
 */

class RateLimitedUserService {
  private rateLimiter: RateLimiter;

  constructor() {
    // 创建限流器：每秒100个请求，突发最多200个
    this.rateLimiter = new TokenBucketRateLimiter(200, 100);
  }

  // API调用，使用限流保护
  async getUser(userId: string): Promise<User> {
    if (!this.rateLimiter.tryAcquire()) {
      throw new Error('请求过于频繁，请稍后重试');
    }

    return await this.userRepository.findById(userId);
  }

  // 批量操作，使用更严格的限流
  async batchGetUsers(userIds: string[]): Promise<User[]> {
    // 批量操作每次消耗更多令牌
    const tokensNeeded = Math.ceil(userIds.length / 10);

    for (let i = 0; i < tokensNeeded; i++) {
      if (!this.rateLimiter.tryAcquire()) {
        throw new Error('请求过于频繁，请稍后重试');
      }
    }

    return await this.userRepository.findByIds(userIds);
  }
}
```

### 3.1 API网关架构

```typescript
// API网关实现

/**
 * API网关核心功能
 */

import express, { Request, Response, NextFunction } from 'express';
import axios from 'axios';

class APIGateway {
  private app: express.Application;
  private routes: Map<string, RouteConfig> = new Map();
  private authService: AuthService;
  private rateLimiter: RateLimiter;

  constructor() {
    this.app = express();
    this.authService = new AuthService();
    this.rateLimiter = new TokenBucketRateLimiter(1000, 500); // 1000容量，500/秒
  }

  // 注册路由
  registerRoute(config: RouteConfig): void {
    this.routes.set(config.path, config);
    console.log(`路由已注册: ${config.path} -> ${config.service}`);
  }

  // 启动网关
  async start(port: number): Promise<void> {
    // 全局中间件
    this.app.use(express.json());
    this.app.use(this.requestLogger.bind(this));
    this.app.use(this.errorHandler.bind(this));

    // 健康检查
    this.app.get('/health', (req, res) => {
      res.json({ status: 'healthy', timestamp: Date.now() });
    });

    // 路由处理
    this.app.use('/api/*', this.routeHandler.bind(this));

    await new Promise<void>((resolve) => {
      this.app.listen(port, () => {
        console.log(`API网关启动，端口: ${port}`);
        resolve();
      });
    });
  }

  // 请求日志
  private async requestLogger(req: Request, res: Response, next: NextFunction): Promise<void> {
    const startTime = Date.now();

    console.log(`[${new Date().toISOString()}] ${req.method} ${req.path}`);

    // 响应后记录耗时
    res.on('finish', () => {
      const duration = Date.now() - startTime;
      console.log(`[${new Date().toISOString()}] ${req.method} ${req.path} - ${res.statusCode} (${duration}ms)`);
    });

    next();
  }

  // 错误处理
  private errorHandler(err: Error, req: Request, res: Response, next: NextFunction): void {
    console.error('网关错误:', err);

    res.status(500).json({
      success: false,
      error: '内部服务器错误',
      message: process.env.NODE_ENV === 'development' ? err.message : undefined
    });
  }

  // 路由处理
  private async routeHandler(req: Request, res: Response): Promise<void> {
    const path = req.path.replace('/api', '');
    const route = this.findRoute(path);

    if (!route) {
      res.status(404).json({
        success: false,
        error: '路由未找到'
      });
      return;
    }

    try {
      // 限流检查
      if (!this.rateLimiter.tryAcquire()) {
        res.status(429).json({
          success: false,
          error: '请求过于频繁'
        });
        return;
      }

      // 认证检查
      if (route.requiresAuth) {
        const authResult = await this.authService.validateRequest(req);

        if (!authResult.valid) {
          res.status(401).json({
            success: false,
            error: '未授权'
          });
          return;
        }

        // 将用户信息附加到请求
        req.user = authResult.user;
      }

      // 路由转发
      await this.forwardRequest(req, res, route);
    } catch (error) {
      console.error('路由处理错误:', error);
      res.status(502).json({
        success: false,
        error: '网关错误'
      });
    }
  }

  // 查找匹配的路由
  private findRoute(path: string): RouteConfig | undefined {
    // 支持路径参数，如 /users/:id
    for (const [pattern, config] of this.routes) {
      if (this.matchPath(pattern, path)) {
        return config;
      }
    }
    return undefined;
  }

  // 路径匹配
  private matchPath(pattern: string, path: string): boolean {
    const patternParts = pattern.split('/');
    const pathParts = path.split('/');

    if (patternParts.length !== pathParts.length) {
      return false;
    }

    for (let i = 0; i < patternParts.length; i++) {
      if (patternParts[i].startsWith(':')) {
        continue; // 路径参数
      }
      if (patternParts[i] !== pathParts[i]) {
        return false;
      }
    }

    return true;
  }

  // 转发请求到后端服务
  private async forwardRequest(
    req: Request,
    res: Response,
    route: RouteConfig
  ): Promise<void> {
    // 构建目标URL
    const targetUrl = `${route.service}${route.path}${req.url.includes('?') ? '?' + req.url.split('?')[1] : ''}`;

    // 构建请求头
    const headers: Record<string, string> = {
      'X-Request-ID': req.headers['x-request-id'] as string || this.generateUUID(),
      'X-Forwarded-For': req.ip || '',
      'X-Forwarded-Proto': req.protocol
    };

    // 添加用户信息到请求头
    if ((req as any).user) {
      headers['X-User-ID'] = (req as any).user.id;
      headers['X-User-Role'] = (req as any).user.role;
    }

    try {
      const response = await axios({
        method: req.method,
        url: targetUrl,
        data: req.body,
        headers,
        timeout: route.timeout || 30000
      });

      res.status(response.status).json(response.data);
    } catch (error: any) {
      if (error.response) {
        res.status(error.response.status).json(error.response.data);
      } else {
        throw error;
      }
    }
  }

  // 生成UUID
  private generateUUID(): string {
    return 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, (c) => {
      const r = Math.random() * 16 | 0;
      const v = c === 'x' ? r : (r & 0x3 | 0x8);
      return v.toString(16);
    });
  }
}

/**
 * 认证服务
 */
class AuthService {
  // 验证请求的认证信息
  async validateRequest(req: Request): Promise<{
    valid: boolean;
    user?: User;
  }> {
    const token = req.headers.authorization?.replace('Bearer ', '');

    if (!token) {
      return { valid: false };
    }

    try {
      // 验证JWT令牌
      const user = await this.verifyToken(token);
      return { valid: true, user };
    } catch {
      return { valid: false };
    }
  }

  // 验证令牌
  private async verifyToken(token: string): Promise<User> {
    // JWT验证逻辑
    const jwt = require('jsonwebtoken');
    const secret = process.env.JWT_SECRET;

    const decoded = jwt.verify(token, secret);

    return {
      id: decoded.sub,
      username: decoded.username,
      role: decoded.role
    };
  }
}
```

## 4. 分布式数据管理

分布式数据管理是微服务架构中最复杂的部分，涉及数据一致性、分布式事务等问题。

### 4.1 CAP定理与BASE理论

```typescript
// CAP定理与BASE理论

/**
 * CAP定理
 *
 * 分布式系统最多只能同时满足以下三个特性中的两个：
 * - Consistency (一致性): 所有节点在同一时刻看到相同的数据
 * - Availability (可用性): 每个请求都能得到响应
 * - Partition tolerance (分区容错): 系统在网络分区时仍能运行
 */

enum CAPChoice {
  // CP系统 - 优先保证一致性
  CP = 'CP',
  // AP系统 - 优先保证可用性
  AP = 'AP'
}

/**
 * 一致性级别
 */
enum ConsistencyLevel {
  STRONG = 'strong',        // 强一致性 - 所有节点实时同步
  EVENTUAL = 'eventual',    // 最终一致性 - 允许短暂不一致
  CAUSAL = 'causal',        // 因果一致性 - 保持因果顺序
  READ_YOUR_WRITES = 'ryw'  // 写后读一致性
}

/**
 * BASE理论
 *
 * - Basically Available: 基本可用 - 允许系统部分功能不可用
 * - Soft state: 软状态 - 状态可以在一段时间内不同步
 * - Eventually consistent: 最终一致性 - 系统最终达到一致状态
 */

/**
 * BASE理论应用示例 - 订单服务
 */

class OrderServiceBASE {
  private db: Database;

  // 创建订单 - 采用最终一致性
  async createOrder(orderData: CreateOrderDTO): Promise<Order> {
    // 1. 创建订单，状态为"PENDING"
    const order: Order = {
      id: this.generateOrderId(),
      ...orderData,
      status: 'PENDING',
      createdAt: Date.now(),
      updatedAt: Date.now()
    };

    await this.db.insert('orders', order);

    // 2. 异步发送消息通知库存服务扣减库存
    await this.messageQueue.publish('order.created', {
      orderId: order.id,
      items: orderData.items
    });

    // 3. 立即返回订单（不等待库存确认）
    return order;
    // 4. 库存服务会异步处理，并通过消息队列反馈结果
    // 5. 订单状态最终会更新为"CONFIRMED"或"FAILED"
  }

  // 获取订单状态 - 允许返回稍微过期的数据
  async getOrder(orderId: string): Promise<Order | null> {
    // 优先从本地缓存读取
    const cached = await this.cache.get(`order:${orderId}`);

    if (cached) {
      return cached;
    }

    // 缓存未命中，从数据库读取
    const order = await this.db.findOne('orders', { id: orderId });

    if (order) {
      // 写入缓存，设置较短TTL
      await this.cache.setex(`order:${orderId}`, 30, order);
    }

    return order;
  }

  // 处理库存服务反馈
  async handleInventoryUpdate(data: {
    orderId: string;
    success: boolean;
    reason?: string;
  }): Promise<void> {
    const order = await this.db.findOne('orders', { id: data.orderId });

    if (!order) {
      console.error('订单不存在:', data.orderId);
      return;
    }

    // 更新订单状态
    if (data.success) {
      order.status = 'CONFIRMED';
    } else {
      order.status = 'INVENTORY_FAILED';
      order.failureReason = data.reason;
    }

    order.updatedAt = Date.now();

    await this.db.update('orders', { id: data.orderId }, order);
  }
}

/**
```
```typescript
 * 两阶段提交（2PC）实现

// 协调者节点
class TwoPhaseCommitCoordinator {
  private participants: Map<string, Participant> = new Map();
  private transactionLog: TransactionLog;

  // 开启事务
  async beginTransaction(transactionId: string): Promise<void> {
    console.log(`事务开始: ${transactionId}`);
    await this.transactionLog.create(transactionId, 'INIT');
  }

  // 阶段1：准备阶段 - 所有参与者投票
  async prepare(transactionId: string): Promise<boolean> {
    console.log(`阶段1 - 准备阶段: ${transactionId}`);

    const results = await Promise.all(
      Array.from(this.participants.entries()).map(async ([name, participant]) => {
        try {
          await participant.prepare();
          return { name, success: true };
        } catch (error) {
          console.error(`参与者 ${name} 准备失败:`, error);
          return { name, success: false };
        }
      })
    );

    // 检查所有参与者是否都准备好
    const allPrepared = results.every(r => r.success);

    if (allPrepared) {
      await this.transactionLog.update(transactionId, 'PREPARED');
      console.log(`所有参与者准备完成: ${transactionId}`);
    } else {
      await this.transactionLog.update(transactionId, 'PREPARE_FAILED');
      console.error(`部分参与者准备失败: ${transactionId}`);
    }

    return allPrepared;
  }

  // 阶段2：提交阶段 - 协调者发送提交命令
  async commit(transactionId: string): Promise<void> {
    console.log(`阶段2 - 提交阶段: ${transactionId}`);

    await Promise.all(
      Array.from(this.participants.entries()).map(async ([name, participant]) => {
        try {
          await participant.commit();
          console.log(`参与者 ${name} 提交完成`);
        } catch (error) {
          console.error(`参与者 ${name} 提交失败:`, error);
          // 提交失败需要处理
        }
      })
    );

    await this.transactionLog.update(transactionId, 'COMMITTED');
    console.log(`事务提交完成: ${transactionId}`);
  }

  // 回滚
  async rollback(transactionId: string): Promise<void> {
    console.log(`事务回滚: ${transactionId}`);

    await Promise.all(
      Array.from(this.participants.entries()).map(async ([name, participant]) => {
        try {
          await participant.rollback();
          console.log(`参与者 ${name} 回滚完成`);
        } catch (error) {
          console.error(`参与者 ${name} 回滚失败:`, error);
        }
      })
    );

    await this.transactionLog.update(transactionId, 'ROLLED_BACK');
  }

  // 注册参与者
  registerParticipant(name: string, participant: Participant): void {
    this.participants.set(name, participant);
  }
}

// Saga协调器
class SagaCoordinator {
  private steps: SagaStep[] = [];
  private executedSteps: SagaStep[] = [];

  // 添加步骤
  addStep(step: SagaStep): this {
    this.steps.push(step);
    return this;
  }

  // 执行Saga
  async execute(): Promise<void> {
    console.log('Saga开始执行');

    for (const step of this.steps) {
      try {
        await step.execute();
        this.executedSteps.push(step);
        console.log(`Saga步骤 ${step.name} 执行完成`);
      } catch (error) {
        console.error(`Saga步骤 ${step.name} 执行失败，开始补偿`);

        // 逆向补偿已执行的步骤
        await this.compensate();

        throw error;
      }
    }

    console.log('Saga执行完成');
  }

  // 补偿操作
  private async compensate(): Promise<void> {
    console.log('开始Saga补偿');

    // 逆序补偿已执行的步骤
    for (const step of this.executedSteps.reverse()) {
      try {
        await step.compensate();
        console.log(`Saga步骤 ${step.name} 补偿完成`);
      } catch (error) {
        console.error(`Saga步骤 ${step.name} 补偿失败:`, error);
        // 继续补偿其他步骤
      }
    }

    console.log('Saga补偿完成');
  }
}
```

### 4.3 分布式ID生成

在分布式系统中，需要生成全局唯一的ID来标识数据。

```typescript
// 分布式ID生成算法

/**
 * 雪花算法（Snowflake）实现
 *
 * ID结构（共64位）：
 * - 第1位：符号位，固定为0
 * - 第2-42位：时间戳（毫秒），约69年
 * - 第43-52位：机器ID（10位），可支持1024个节点
 * - 第53-64位：序列号（12位），每毫秒可生成4096个ID
 */

class SnowflakeIdGenerator {
  // 起始时间戳：2020-01-01
  private static readonly EPOCH = 1609459200000;

  // 各部分位数
  private static readonly TIMESTAMP_BITS = 41;
  private static readonly MACHINE_ID_BITS = 10;
  private static readonly SEQUENCE_BITS = 12;

  // 各部分最大值
  private static readonly MAX_MACHINE_ID = ~(~0 << SnowflakeIdGenerator.MACHINE_ID_BITS);
  private static readonly MAX_SEQUENCE = ~(~0 << SnowflakeIdGenerator.SEQUENCE_BITS);

  // 各部分左移位数
  private static readonly TIMESTAMP_SHIFT = SnowflakeIdGenerator.MACHINE_ID_BITS + SnowflakeIdGenerator.SEQUENCE_BITS;
  private static readonly MACHINE_ID_SHIFT = SnowflakeIdGenerator.SEQUENCE_BITS;

  private machineId: number;
  private sequence: number = 0;
  private lastTimestamp: number = -1;

  constructor(machineId: number) {
    if (machineId < 0 || machineId > SnowflakeIdGenerator.MAX_MACHINE_ID) {
      throw new Error(`机器ID必须在0到${SnowflakeIdGenerator.MAX_MACHINE_ID}之间`);
    }
    this.machineId = machineId;
  }

  // 生成下一个ID
  nextId(): bigint {
    let timestamp = this.getCurrentTimestamp();

    // 时钟回拨处理
    if (timestamp < this.lastTimestamp) {
      console.error('时钟回拨检测到，等待追赶');
      timestamp = this.lastTimestamp;
    }

    // 同一毫秒内，序列号递增
    if (timestamp === this.lastTimestamp) {
      this.sequence = (this.sequence + 1) & SnowflakeIdGenerator.MAX_SEQUENCE;

      // 序列号溢出，等待下一毫秒
      if (this.sequence === 0) {
        timestamp = this.waitNextMillis(timestamp);
      }
    } else {
      // 新毫秒，序列号重置
      this.sequence = 0;
    }

    this.lastTimestamp = timestamp;

    // 构建ID
    const id = (
      BigInt(timestamp - SnowflakeIdGenerator.EPOCH) << SnowflakeIdGenerator.TIMESTAMP_SHIFT |
      BigInt(this.machineId) << SnowflakeIdGenerator.MACHINE_ID_SHIFT |
      BigInt(this.sequence)
    );

    return id;
  }

  // 获取当前时间戳
  private getCurrentTimestamp(): number {
    return Date.now();
  }

  // 等待下一毫秒
  private waitNextMillis(currentTimestamp: number): number {
    let timestamp = this.getCurrentTimestamp();

    while (timestamp <= currentTimestamp) {
      timestamp = this.getCurrentTimestamp();
    }

    return timestamp;
  }

  // 从ID解析出各部分信息
  static parse(id: bigint): {
    timestamp: number;
    machineId: number;
    sequence: number;
    date: Date;
  } {
    const timestamp = Number(id >> BigInt(SnowflakeIdGenerator.TIMESTAMP_SHIFT)) + SnowflakeIdGenerator.EPOCH;
    const machineId = Number((id >> BigInt(SnowflakeIdGenerator.MACHINE_ID_SHIFT)) & BigInt(SnowflakeIdGenerator.MAX_MACHINE_ID));
    const sequence = Number(id & BigInt(SnowflakeIdGenerator.MAX_SEQUENCE));

    return {
      timestamp,
      machineId,
      sequence,
      date: new Date(timestamp)
    };
  }
}

/**
 * 改良版雪花算法 - 支持更长时间和更多机器
 */
class SnowflakeIdGeneratorV2 {
  // 使用更大位数的时间戳
  private static readonly CUSTOM_EPOCH = 1577836800000; // 2020-01-01 00:00:00

  // 优化后的位数分配
  private static readonly TIMESTAMP_BITS = 40;     // 可用34年+
  private static readonly MACHINE_ID_BITS = 12;    // 可支持4096个节点
  private static readonly SEQUENCE_BITS = 11;      // 每毫秒2048个序列

  private machineId: number;
  private sequence: number = 0;
  private lastTimestamp: number = -1;

  constructor(machineId: number) {
    this.machineId = machineId;
  }

  nextId(): bigint {
    let timestamp = Date.now();

    if (timestamp < this.lastTimestamp) {
      throw new Error('时钟回拨不允许');
    }

    if (timestamp === this.lastTimestamp) {
      this.sequence = (this.sequence + 1) & ((1 << SnowflakeIdGeneratorV2.SEQUENCE_BITS) - 1);

      if (this.sequence === 0) {
        timestamp = this.waitNextMillis(timestamp);
      }
    } else {
      this.sequence = 0;
    }

    this.lastTimestamp = timestamp;

    return (
      BigInt(timestamp - SnowflakeIdGeneratorV2.CUSTOM_EPOCH) <<
      BigInt(SnowflakeIdGeneratorV2.MACHINE_ID_BITS + SnowflakeIdGeneratorV2.SEQUENCE_BITS) |
      BigInt(this.machineId) <<
      BigInt(SnowflakeIdGeneratorV2.SEQUENCE_BITS) |
      BigInt(this.sequence)
    );
  }

  private waitNextMillis(currentTimestamp: number): number {
    let timestamp = Date.now();
    while (timestamp <= currentTimestamp) {
      timestamp = Date.now();
    }
    return timestamp;
  }
}

/**
 * UUID v4 实现（备选方案）

class UUIDGenerator {
  // 生成UUID v4格式
  static generate(): string {
    // 使用随机数生成UUID
    return 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, (c) => {
      const r = Math.random() * 16 | 0;
      const v = c === 'x' ? r : (r & 0x3 | 0x8);
      return v.toString(16);
    });
  }

  // 验证UUID格式
  static validate(uuid: string): boolean {
    const pattern = /^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i;
    return pattern.test(uuid);
  }
}
```

```typescript
// 根据用户ID分片
class UserIdShardingStrategy implements ShardingStrategy {
  // 计算分片编号
  calculateShard(userId: string): number {
    // 使用哈希取模
    const hash = this.hash(userId);
    return hash % this.shardCount;
  }

  // 获取表名
  getTableName(userId: string): string {
    const shardId = this.calculateShard(userId);
    return `${this.tableName}_${shardId}`;
  }

  private hash(str: string): number {
    let hash = 0;
    for (let i = 0; i < str.length; i++) {
      const char = str.charCodeAt(i);
      hash = ((hash << 5) - hash) + char;
      hash = hash & hash;
    }
    return Math.abs(hash);
  }
}

// 根据时间分片
class TimeBasedShardingStrategy implements ShardingStrategy {
  constructor(
    private tableName: string,
    private shardCount: number = 12  // 按月分片
  ) {}

  // 根据时间戳计算分片
  calculateShard(timestamp: number): number {
    const date = new Date(timestamp);
    const year = date.getFullYear();
    const month = date.getMonth();

    // 计算距离基准年份的月数
    const baseYear = 2020;
    const monthsDiff = (year - baseYear) * 12 + month;

    return monthsDiff % this.shardCount;
  }

  // 获取表名
  getTableName(timestamp: number): string {
    const shardId = this.calculateShard(timestamp);
    return `${this.tableName}_${shardId}`;
  }

  // 获取日期范围表名（如 orders_2024_01）
  getTableNameByDate(year: number, month: number): string {
    return `${this.tableName}_${year}_${String(month).padStart(2, '0')}`;
  }
}

/**
 * 分片路由器
 */
class ShardRouter {
  private strategies: Map<string, ShardingStrategy> = new Map();

  // 注册分片策略
  registerStrategy(tableName: string, strategy: ShardingStrategy): void {
    this.strategies.set(tableName, strategy);
  }

  // 路由查询
  async routeQuery<T>(
    tableName: string,
    query: Query,
    shardKey: string
  ): Promise<T[]> {
    const strategy = this.strategies.get(tableName);

    if (!strategy) {
      throw new Error(`未找到表 ${tableName} 的分片策略`);
    }

    const table = strategy.getTableName(shardKey);
    return this.executeQuery<T>(table, query);
  }

  // 跨分片查询（广播到所有分片）
  async broadcastQuery<T>(
    tableName: string,
    query: Query
  ): Promise<T[]> {
    const strategy = this.strategies.get(tableName);
    const results: T[] = [];

    // 查询所有分片
    for (let i = 0; i < strategy.shardCount; i++) {
      const table = `${tableName}_${i}`;
      const shardResults = await this.executeQuery<T>(table, query);
      results.push(...shardResults);
    }

    return results;
  }

  private async executeQuery<T>(table: string, query: Query): Promise<T[]> {
    // 执行查询的逻辑
    console.log(`在表 ${table} 上执行查询:`, query);
    return [];
  }
}
```

### 7.3 用户服务完整实现

```typescript
// services/user-service/src/index.ts - 用户服务主入口

import express from 'express';
import cors from 'cors';
import helmet from 'helmet';
import { ConsulService } from './consul';
import { DatabaseService } from './database';
import { RedisService } from './redis';
import { UserController } from './controllers/user.controller';
import { AuthController } from './controllers/auth.controller';
import { errorHandler } from './middleware/error.middleware';
import { requestLogger } from './middleware/logger.middleware';

/**
 * 用户服务主类
 * 负责任务：
 * 1. 用户注册、登录、信息管理
 * 2. JWT令牌生成和验证
 * 3. 用户会话管理
 */
class UserService {
  public app: express.Application;
  private consul: ConsulService;
  private db: DatabaseService;
  private redis: RedisService;
  private port: number;

  constructor() {
    this.app = express();
    this.port = parseInt(process.env.PORT || '3001', 10);

    // 初始化依赖
    this.consul = new ConsulService();
    this.db = new DatabaseService();
    this.redis = new RedisService();

    // 配置中间件
    this.setupMiddleware();

    // 配置路由
    this.setupRoutes();

    // 配置错误处理
    this.app.use(errorHandler);
  }

  /**
   * 配置中间件
   */
  private setupMiddleware(): void {
    // CORS跨域资源共享
    this.app.use(cors({
      origin: process.env.ALLOWED_ORIGINS?.split(',') || '*',
      credentials: true
    }));

    // 安全头
    this.app.use(helmet());

    // 请求体解析
    this.app.use(express.json());
    this.app.use(express.urlencoded({ extended: true }));

    // 请求日志
    this.app.use(requestLogger);

    // 请求ID跟踪
```

```typescript
// services/order-service/src/index.ts - 订单服务主入口

import express from 'express';
import { DatabaseService } from './database';
import { RabbitMQService } from './rabbitmq';
import { OrderController } from './controllers/order.controller';
import { OrderSaga } from './sagas/order.saga';
import { errorHandler } from './middleware/error.middleware';

/**
 * 订单服务主类
 * 负责任务：
 * 1. 订单创建、查询、取消
 * 2. 订单状态管理
 * 3. 协调库存、支付等下游服务
 * 4. 使用Saga模式保证分布式事务一致性
 */
class OrderService {
  public app: express.Application;
  private db: DatabaseService;
  private rabbitmq: RabbitMQService;
  private orderSaga: OrderSaga;
  private port: number;

  constructor() {
    this.app = express();
    this.port = parseInt(process.env.PORT || '3003', 10);

    // 初始化依赖
    this.db = new DatabaseService();
    this.rabbitmq = new RabbitMQService();
    this.orderSaga = new OrderSaga(this.db, this.rabbitmq);

    // 配置中间件
    this.setupMiddleware();

    // 配置路由
    this.setupRoutes();

    // 配置消息消费者
    this.setupConsumers();

    // 配置错误处理
    this.app.use(errorHandler);
  }

  /**
   * 配置中间件
   */
  private setupMiddleware(): void {
    this.app.use(express.json());

    // 验证用户身份
    this.app.use('/api/orders', this.verifyUser.bind(this));
  }

  /**
   * 用户验证中间件
   */
  private verifyUser(req: express.Request, res: express.Response, next: express.NextFunction): void {
    const token = req.headers.authorization?.replace('Bearer ', '');

    if (!token) {
      return res.status(401).json({ error: '未授权访问' });
    }

    // 验证令牌逻辑
    // 简化处理，实际应调用用户服务验证
    try {
      const user = this.verifyToken(token);
      (req as any).user = user;
      next();
    } catch {
      res.status(401).json({ error: '无效的令牌' });
    }
  }

  /**
   * 验证JWT令牌
   */
  private verifyToken(token: string): { id: string; role: string } {
    const jwt = require('jsonwebtoken');
    const secret = process.env.JWT_SECRET || 'secret';
    return jwt.verify(token, secret);
  }

  /**
   * 配置路由
   */
  private setupRoutes(): void {
    // 健康检查
    this.app.get('/health', (req, res) => {
      res.json({
        status: 'healthy',
        service: 'order-service',
        timestamp: new Date().toISOString()
      });
    });

    // 订单控制器
    const orderController = new OrderController(this.db, this.orderSaga);

    // 订单路由
    this.app.post('/api/orders', orderController.create.bind(orderController));
    this.app.get('/api/orders/:id', orderController.getById.bind(orderController));
    this.app.get('/api/orders', orderController.list.bind(orderController));
    this.app.put('/api/orders/:id/cancel', orderController.cancel.bind(orderController));
    this.app.get('/api/orders/:id/status', orderController.getStatus.bind(orderController));
  }

  /**
   * 配置消息消费者
   */
  private setupConsumers(): void {
    // 监听支付成功事件
    this.rabbitmq.consume('payment.completed', async (message) => {
      console.log('收到支付成功事件:', message);
      await this.handlePaymentCompleted(message);
    });

    // 监听支付失败事件
    this.rabbitmq.consume('payment.failed', async (message) => {
      console.log('收到支付失败事件:', message);
      await this.handlePaymentFailed(message);
    });

    // 监听库存不足事件
    this.rabbitmq.consume('inventory.insufficient', async (message) => {
      console.log('收到库存不足事件:', message);
      await this.handleInventoryInsufficient(message);
    });

    // 监听物流状态更新
    this.rabbitmq.consume('logistics.status.updated', async (message) => {
      console.log('收到物流状态更新:', message);
      await this.handleLogisticsStatusUpdate(message);
    });
  }

  /**
   * 处理支付完成事件
   */
  private async handlePaymentCompleted(message: any): Promise<void> {
    const { orderId } = message;

    // 更新订单状态为已支付
    await this.db.query(
      'UPDATE orders SET status = $1, paid_at = $2 WHERE id = $3',
      ['PAID', new Date(), orderId]
    );

    // 发布订单已支付事件，触发物流
    await this.rabbitmq.publish('order.paid', {
      orderId,
      paidAt: new Date()
    });

    console.log(`订单 ${orderId} 已支付`);
  }

  /**
   * 处理支付失败事件
   */
  private async handlePaymentFailed(message: any): Promise<void> {
    const { orderId, reason } = message;

    // 更新订单状态为支付失败
    await this.db.query(
      'UPDATE orders SET status = $1, failure_reason = $2 WHERE id = $3',
      ['PAYMENT_FAILED', reason, orderId]
    );

    // 释放预留的库存
    await this.rabbitmq.publish('inventory.release', {
      orderId
    });

    console.log(`订单 ${orderId} 支付失败: ${reason}`);
  }

  /**
   * 处理库存不足事件
   */
  private async handleInventoryInsufficient(message: any): Promise<void> {
    const { orderId, productId, requested, available } = message;

    // 更新订单状态
    await this.db.query(
      'UPDATE orders SET status = $1, failure_reason = $2 WHERE id = $3',
      ['INVENTORY_SHORTAGE', `商品${productId}库存不足（请求${requested}，可用${available}）`, orderId]
    );

    // 取消支付
    await this.rabbitmq.publish('payment.cancel', { orderId });

    console.log(`订单 ${orderId} 库存不足`);
  }

  /**
   * 处理物流状态更新
   */
  private async handleLogisticsStatusUpdate(message: any): Promise<void> {
    const { orderId, status, trackingNumber } = message;

    // 更新订单的物流信息
    await this.db.query(
      'UPDATE orders SET logistics_status = $1, tracking_number = $2 WHERE id = $3',
      [status, trackingNumber, orderId]
    );

    // 如果已发货，更新订单状态
    if (status === 'SHIPPED') {
      await this.db.query(
        'UPDATE orders SET status = $1, shipped_at = $2 WHERE id = $3',
        ['SHIPPED', new Date(), orderId]
      );
    }

    console.log(`订单 ${orderId} 物流状态更新: ${status}`);
  }

  /**
   * 启动服务
   */
  async start(): Promise<void> {
    try {
      // 连接数据库
      await this.db.connect();
      console.log('数据库连接成功');

      // 连接RabbitMQ
      await this.rabbitmq.connect();
      console.log('RabbitMQ连接成功');

      // 设置消息消费者
      await this.setupConsumers();
      console.log('消息消费者已设置');

      // 启动HTTP服务器
      this.app.listen(this.port, () => {
        console.log(`订单服务启动成功，端口: ${this.port}`);
      });
    } catch (error) {
      console.error('服务启动失败:', error);
      process.exit(1);
    }
  }
}
```

### 7.5 Saga模式订单协调器

```typescript
// services/order-service/src/sagas/order.saga.ts - 订单Saga协调器

import { DatabaseService } from '../database';
import { RabbitMQService } from '../rabbitmq';

/**
 * Saga步骤接口
 * 每个步骤包含正向操作和补偿操作
 */
interface SagaStep {
  name: string;                    // 步骤名称
  execute: () => Promise<void>;    // 正向操作
  compensate: () => Promise<void>; // 补偿操作
}

/**
 * 订单Saga协调器
 * 负责协调订单创建过程中的多个服务调用
 * 使用Saga模式保证分布式事务一致性
 */
export class OrderSaga {
  private executedSteps: SagaStep[] = [];

  constructor(
    private db: DatabaseService,
    private rabbitmq: RabbitMQService
  ) {}

  /**
   * 执行订单创建Saga
   *
   * 流程：
   * 1. 验证用户 - 确保用户存在且状态正常
   * 2. 验证商品 - 确保商品存在且价格正确
   * 3. 预留库存 - 冻结相应库存
   * 4. 创建订单 - 创建待支付订单
   * 5. 发起支付 - 调用支付服务
   *
   * 补偿流程（失败时逆向执行）：
   * - 如果支付失败，释放库存，取消订单
   * - 如果库存预留失败，取消订单
   */
  async executeCreateOrder(orderData: {
    userId: string;
    items: Array<{ productId: string; quantity: number }>;
    shippingAddress: string;
    paymentMethod: string;
  }): Promise<{ orderId: string; status: string }> {
    this.executedSteps = [];

    const { userId, items, shippingAddress, paymentMethod } = orderData;

    try {
      // ========== 步骤1：验证用户 ==========
      console.log('步骤1：验证用户');
      await this.validateUser(userId);
      this.executedSteps.push({
        name: 'validateUser',
        execute: () => this.validateUser(userId),
        compensate: () => Promise.resolve() // 用户验证无需补偿
      });

      // ========== 步骤2：验证商品并计算价格 ==========
      console.log('步骤2：验证商品');
      const productValidation = await this.validateProducts(items);
      this.executedSteps.push({
        name: 'validateProducts',
        execute: () => this.validateProducts(items),
        compensate: () => Promise.resolve() // 商品验证无需补偿
      });

      // ========== 步骤3：预留库存 ==========
      console.log('步骤3：预留库存');
      await this.reserveInventory(items);
      this.executedSteps.push({
        name: 'reserveInventory',
        execute: () => this.reserveInventory(items),
        compensate: () => this.releaseInventory(items)
      });

      // ========== 步骤4：创建订单 ==========
      console.log('步骤4：创建订单');
      const orderId = await this.createOrder({
        userId,
        items,
        totalAmount: productValidation.totalAmount,
        shippingAddress
      });
      this.executedSteps.push({
        name: 'createOrder',
        execute: () => this.createOrder({ userId, items, totalAmount: productValidation.totalAmount, shippingAddress }),
        compensate: () => this.cancelOrder(orderId)
      });

      // ========== 步骤5：发起支付 ==========
      console.log('步骤5：发起支付');
      await this.initiatePayment(orderId, productValidation.totalAmount, paymentMethod);
      this.executedSteps.push({
        name: 'initiatePayment',
        execute: () => this.initiatePayment(orderId, productValidation.totalAmount, paymentMethod),
        compensate: () => this.cancelPayment(orderId)
      });

      console.log('订单Saga执行成功:', orderId);
      return { orderId, status: 'PENDING_PAYMENT' };

    } catch (error) {
      console.error('订单Saga执行失败，开始补偿:', error.message);

      // 逆向执行补偿操作
      await this.compensate();

      throw error;
    }
  }

  /**
   * 验证用户
   */
  private async validateUser(userId: string): Promise<void> {
    // 发送消息到用户服务验证用户
    // 这里简化处理，实际应该调用用户服务API
    const response = await this.rabbitmq.request(
      'user.validate',
      { userId },
      'user.validation.response'
    );

    if (!response.valid) {
      throw new Error(`用户验证失败: ${response.reason}`);
    }

    console.log(`用户 ${userId} 验证通过`);
  }

  /**
   * 验证商品
   * 返回验证结果和总金额
   */
  private async validateProducts(
    items: Array<{ productId: string; quantity: number }>
  ): Promise<{ products: any[]; totalAmount: number }> {
    const products: any[] = [];
    let totalAmount = 0;

    for (const item of items) {
      // 发送消息到商品服务获取商品信息
      const product = await this.rabbitmq.request(
        'product.get',
        { productId: item.productId },
        'product.response'
      );

      if (!product) {
        throw new Error(`商品不存在: ${item.productId}`);
      }

      if (product.stock < item.quantity) {
        throw new Error(`商品库存不足: ${product.name}`);
      }

      products.push(product);
      totalAmount += product.price * item.quantity;
    }

    console.log(`商品验证通过，总金额: ${totalAmount}`);
    return { products, totalAmount };
  }

  /**
   * 预留库存
   */
  private async reserveInventory(
    items: Array<{ productId: string; quantity: number }>
  ): Promise<void> {
    // 发送库存预留请求
    await this.rabbitmq.publish('inventory.reserve', {
      items,
      reservedFor: 'order' // 预留类型
    });

    // 等待库存服务确认
    // 这里简化处理，实际应该使用消息确认机制
    console.log(`库存预留请求已发送`);
  }

  /**
   * 释放库存（补偿操作）
   */
  private async releaseInventory(
    items: Array<{ productId: string; quantity: number }>
  ): Promise<void> {
    await this.rabbitmq.publish('inventory.release', {
      items,
      reason: 'order_cancelled'
    });

    console.log(`库存释放请求已发送`);
  }

  /**
   * 创建订单
   */
  private async createOrder(data: {
    userId: string;
    items: Array<{ productId: string; quantity: number }>;
    totalAmount: number;
    shippingAddress: string;
  }): Promise<string> {
    const orderId = `ORD-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`;

    await this.db.query(
      `INSERT INTO orders (id, user_id, items, total_amount, status, shipping_address, created_at)
       VALUES ($1, $2, $3, $4, $5, $6, $7)`,
      [
        orderId,
        data.userId,
        JSON.stringify(data.items),
        data.totalAmount,
        'PENDING',
        data.shippingAddress,
        new Date()
      ]
    );

    console.log(`订单已创建: ${orderId}`);
    return orderId;
  }

  /**
   * 取消订单（补偿操作）
   */
  private async cancelOrder(orderId: string): Promise<void> {
    await this.db.query(
      'UPDATE orders SET status = $1, cancelled_at = $2 WHERE id = $3',
      ['CANCELLED', new Date(), orderId]
    );

    console.log(`订单已取消: ${orderId}`);
  }

  /**
   * 发起支付
   */
  private async initiatePayment(
    orderId: string,
    amount: number,
    paymentMethod: string
  ): Promise<void> {
    await this.rabbitmq.publish('payment.initiate', {
      orderId,
      amount,
      paymentMethod,
      callbackQueue: 'payment.result'
    });

    console.log(`支付请求已发起: ${orderId}`);
  }

  /**
   * 取消支付（补偿操作）
   */
  private async cancelPayment(orderId: string): Promise<void> {
    await this.rabbitmq.publish('payment.cancel', { orderId });
    console.log(`支付已取消: ${orderId}`);
  }

  /**
   * 执行补偿操作
   * 逆向执行已完成的步骤的补偿操作
   */
  private async compensate(): Promise<void> {
    console.log('开始执行补偿操作');

    // 逆序执行已成功步骤的补偿操作
    for (const step of this.executedSteps.reverse()) {
      try {
        console.log(`执行补偿: ${step.name}`);
        await step.compensate();
        console.log(`补偿完成: ${step.name}`);
      } catch (error) {
        // 补偿失败继续执行其他补偿操作
        console.error(`补偿失败: ${step.name}`, error);
      }
    }

    console.log('补偿操作执行完成');
  }
}
```
