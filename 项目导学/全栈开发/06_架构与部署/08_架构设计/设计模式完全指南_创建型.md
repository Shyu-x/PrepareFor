# 设计模式完全指南：创建型模式

## 概述

设计模式是软件开发中经过验证的解决方案，它们并非具体的代码，而是一种解决问题的思想和方法。创建型模式是设计模式中最基础的一类，主要关注**"如何创建对象"**这个问题。

在日常开发中，我们经常需要创建各种对象：用户对象、配置对象、数据库连接、API客户端等。如果没有好的创建方式，代码会变得混乱、难以维护。创建型模式正是为了解决这些问题而诞生的。

本章我们将学习5种最常用的创建型设计模式：

| 模式 | 核心思想 | 一句话理解 |
|------|----------|------------|
| 单例模式 | 一个类只有一个实例 | 全局唯一，像身份证号 |
| 工厂模式 | 统一创建对象 | 像工厂流水线生产产品 |
| 抽象工厂模式 | 创建一系列相关对象 | 像不同品牌的手机工厂 |
| 建造者模式 | 分步构建复杂对象 | 像组装汽车 |
| 原型模式 | 克隆已有对象 | 像复印机复印文件 |

## 一、单例模式（Singleton Pattern）

### 1.1 生活类比

**就像一个人的身份证号码**——无论你在哪里、做什么事情，你的身份证号码始终是同一个，不会出现第二个相同的号码。单例模式确保一个类只有一个实例，并提供一个全局访问点。

### 1.2 解决的问题

在开发中，有些对象理论上应该只有一个实例：

- **数据库连接池**：如果每次都创建新的连接，数据库压力会很大
- **全局配置管理器**：整个应用只需要一份配置
- **日志记录器**：所有地方都写入同一个日志文件
- **缓存管理器**：全局只需要一个缓存实例

如果没有单例模式，可能出现以下问题：

```typescript
// ❌ 问题示例：多次创建数据库连接
class Database {
  constructor() {
    console.log("创建新的数据库连接");
  }
}

const db1 = new Database(); // 输出: 创建新的数据库连接
const db2 = new Database(); // 输出: 创建新的数据库连接
// 问题：创建了多个连接，浪费资源！
```

### 1.3 代码示例

#### 基础单例模式（TypeScript）

```typescript
/**
 * 单例模式基础实现
 * 核心原理：通过闭包和静态属性确保只有一个实例
 */
class Singleton {
  // 保存唯一实例
  private static instance: Singleton;

  // 私有构造函数，防止外部直接 new
  private constructor() {
    console.log("单例实例被创建");
  }

  // 获取实例的静态方法
  public static getInstance(): Singleton {
    // 如果实例不存在，则创建；否则返回已有实例
    if (!Singleton.instance) {
      Singleton.instance = new Singleton();
    }
    return Singleton.instance;
  }

  // 示例方法
  public query(sql: string): void {
    console.log(`执行SQL: ${sql}`);
  }
}

// 测试
const instance1 = Singleton.getInstance();
const instance2 = Singleton.getInstance();

console.log(instance1 === instance2); // 输出: true，说明是同一个实例
```

#### 线程安全的单例模式（Node.js环境）

```typescript
/**
 * 线程安全的单例模式
 * 使用双重检查锁定，确保在多线程环境下也能正确工作
 */
class ThreadSafeSingleton {
  private static instance: ThreadSafeSingleton | null = null;
  private static lock = false;

  private constructor() {
    // 模拟耗时操作
    this.init();
  }

  private init(): void {
    console.log("初始化耗时资源...");
  }

  public static getInstance(): ThreadSafeSingleton {
    // 第一次检查
    if (this.instance === null) {
      // 第二次检查（配合锁）
      if (!this.lock) {
        this.lock = true;
        this.instance = new ThreadSafeSingleton();
      }
    }
    return this.instance!;
  }
}
```

#### 简化版：使用Proxy代理

```typescript
/**
 * 使用Proxy实现更简洁的单例
 * 原理：拦截构造函数调用，返回同一个实例
 */
function singleton<T extends { new (...args: any[]): T }>(TargetClass: T): T {
  let instance: T | null = null;

  return new Proxy(TargetClass, {
    // 拦截构造函数调用
    construct(target, args) {
      if (!instance) {
        instance = new target(...args);
      }
      return instance;
    }
  });
}

// 使用
@singleton
class Database {
  constructor(host: string, port: number) {
    console.log(`连接数据库: ${host}:${port}`);
  }
}

const db1 = new Database("localhost", 3306);
const db2 = new Database("localhost", 3306);

console.log(db1 === db2); // true
```

#### 案例2：React中的单例——全局状态管理

```typescript
/**
 * React中使用单例模式管理全局状态
 * 类似Zustand的实现原理
 */
class GlobalStateManager {
  private static instance: GlobalStateManager;
  private state: Record<string, any> = {};
  private listeners: Set<() => void> = new Set();

  private constructor() {}

  public static getInstance(): GlobalStateManager {
    if (!GlobalStateManager.instance) {
      GlobalStateManager.instance = new GlobalStateManager();
    }
    return GlobalStateManager.instance;
  }

  // 获取状态
  public getState<T>(key: string): T {
    return this.state[key];
  }

  // 设置状态
  public setState<T>(key: string, value: T): void {
    this.state[key] = value;
    // 通知所有监听者
    this.listeners.forEach(listener => listener());
  }

  // 订阅状态变化
  public subscribe(listener: () => void): () => void {
    this.listeners.add(listener);
    return () => this.listeners.delete(listener);
  }
}

// 使用
export const globalState = GlobalStateManager.getInstance();
```

#### 案例3：Node.js中的单例——数据库连接池

```typescript
/**
 * Node.js中使用单例模式管理数据库连接池
 * 确保整个应用只有一个连接池实例
 */
import mysql from 'mysql2/promise';

class DatabasePool {
  private static instance: DatabasePool;
  private pool: mysql.Pool | null = null;

  // 私有构造函数
  private constructor() {}

  // 获取实例
  public static getInstance(): DatabasePool {
    if (!DatabasePool.instance) {
      DatabasePool.instance = new DatabasePool();
    }
    return DatabasePool.instance;
  }

  // 初始化连接池
  public async init(config: mysql.PoolOptions): Promise<void> {
    if (!this.pool) {
      this.pool = mysql.createPool({
        ...config,
        waitForConnections: true,
        connectionLimit: 10,
        queueLimit: 0
      });
      console.log("数据库连接池初始化完成");
    }
  }

  // 获取连接
  public async getConnection(): Promise<mysql.PoolConnection> {
    if (!this.pool) {
      throw new Error("数据库连接池未初始化，请先调用init()");
    }
    return this.pool.getConnection();
  }

  // 执行查询
  public async query<T>(sql: string, params?: any[]): Promise<T> {
    const connection = await this.getConnection();
    try {
      const [results] = await connection.query(sql, params);
      return results as T;
    } finally {
      connection.release();
    }
  }
}

// 使用
const db = DatabasePool.getInstance();
await db.init({
  host: 'localhost',
  user: 'root',
  password: 'password',
  database: 'myapp'
});

// 任何地方使用都是同一个连接池
const users = await db.query("SELECT * FROM users");
```

### 1.5 单例模式的优缺点

**优点：**

- 全局只有一个实例，节省内存资源
- 避免了资源的重复创建
- 便于统一管理和配置
- 实现了全局共享

**缺点：**

- 单例职责过重，违反单一职责原则
- 隐藏了类之间的依赖关系
- 难以进行单元测试
- 在多线程环境下需要小心处理

## 二、工厂模式（Factory Pattern）

### 2.1 生活类比

**就像一家奶茶店点单**——你只需要说"一杯珍珠奶茶"，奶茶店会根据配方制作，你不需要关心茶叶怎么泡、珍珠怎么煮。工厂模式就是这样的"点单"机制：告诉工厂你想要什么，工厂负责生产。

### 2.2 解决的问题

考虑一个场景：创建一个在线教育平台，需要支持多种会员类型（免费会员、VIP会员、企业会员）。

**不使用工厂模式的问题：**

```typescript
// ❌ 问题：创建逻辑散落在各处
class MemberService {
  createMember(type: string, name: string) {
    if (type === 'free') {
      const member = new FreeMember(name);
      member.sendWelcomeEmail();
      member.initPermissions();
      return member;
    } else if (type === 'vip') {
      const member = new VipMember(name);
      member.sendWelcomeEmail();
      member.initPermissions();
      member.activatePremiumFeatures();
      return member;
    } else if (type === 'enterprise') {
      const member = new EnterpriseMember(name);
      member.sendWelcomeEmail();
      member.initPermissions();
      member.setupEnterpriseFeatures();
      member.assignDedicatedManager();
      return member;
    }
    // 每增加一个类型，都需要修改这个方法！
  }
}

// 问题：创建逻辑重复、难以扩展、违背开闭原则
```

### 2.3 代码示例

#### 基础工厂模式

```typescript
/**
 * 工厂模式基础实现
 * 核心思想：将对象创建集中到工厂中，客户端不需要知道具体类名
 */

// 定义产品接口
interface Member {
  name: string;
  type: string;
  getPermissions(): string[];
  getLevel(): number;
}

// 具体产品类：免费会员
class FreeMember implements Member {
  constructor(public name: string) {
    console.log(`创建免费会员: ${name}`);
  }

  get type(): string {
    return 'free';
  }

  getPermissions(): string[] {
    return ['read', 'comment'];
  }

  getLevel(): number {
    return 1;
  }
}

// 具体产品类：VIP会员
class VipMember implements Member {
  constructor(public name: string) {
    console.log(`创建VIP会员: ${name}`);
  }

  get type(): string {
    return 'vip';
  }

  getPermissions(): string[] {
    return ['read', 'comment', 'download', 'premium_content'];
  }

  getLevel(): number {
    return 3;
  }
}

// 具体产品类：企业会员
class EnterpriseMember implements Member {
  constructor(public name: string) {
    console.log(`创建企业会员: ${name}`);
  }

  get type(): string {
    return 'enterprise';
  }

  getPermissions(): string[] {
    return ['read', 'comment', 'download', 'premium_content', 'team_management', 'analytics'];
  }

  getLevel(): number {
    return 5;
  }
}

/**
 * 工厂类
 * 核心作用：根据类型字符串创建对应的产品实例
 */
class MemberFactory {
  // 静态工厂方法
  static create(type: 'free' | 'vip' | 'enterprise', name: string): Member {
    switch (type) {
      case 'free':
        return new FreeMember(name);
      case 'vip':
        return new VipMember(name);
      case 'enterprise':
        return new EnterpriseMember(name);
      default:
        throw new Error(`不支持的会员类型: ${type}`);
    }
  }
}

// 客户端代码
const freeUser = MemberFactory.create('free', '张三');
const vipUser = MemberFactory.create('vip', '李四');
const enterpriseUser = MemberFactory.create('enterprise', '王五');

console.log(freeUser.getPermissions()); // ['read', 'comment']
console.log(vipUser.getPermissions());   // ['read', 'comment', 'download', 'premium_content']
```

#### 工厂方法模式（更灵活的实现）

```typescript
/**
 * 工厂方法模式
 * 将工厂抽象化，每个产品有对应的工厂
 */

// 产品接口
interface Notification {
  send(message: string): void;
  getType(): string;
}

// 邮件通知
class EmailNotification implements Notification {
  constructor(private email: string) {}

  send(message: string): void {
    console.log(`[邮件] 发送到 ${this.email}: ${message}`);
  }

  getType(): string {
    return 'email';
  }
}

// 短信通知
class SMSNotification implements Notification {
  constructor(private phone: string) {}

  send(message: string): void {
    console.log(`[短信] 发送到 ${this.phone}: ${message}`);
  }

  getType(): string {
    return 'sms';
  }
}

// 推送通知
class PushNotification implements Notification {
  constructor(private deviceId: string) {}

  send(message: string): void {
    console.log(`[推送] 发送到设备 ${this.deviceId}: ${message}`);
  }

  getType(): string {
    return 'push';
  }
}

/**
 * 抽象工厂接口
 * 每个工厂负责创建一种产品
 */
interface NotificationFactory {
  createNotification(contact: string): Notification;
}

/**
 * 具体工厂类
 */
class EmailFactory implements NotificationFactory {
  createNotification(contact: string): Notification {
    // 可以添加邮箱格式验证
    if (!contact.includes('@')) {
      throw new Error('无效的邮箱地址');
    }
    return new EmailNotification(contact);
  }
}

class SMSFactory implements NotificationFactory {
  createNotification(contact: string): Notification {
    // 可以添加手机号验证
    if (!/^\d{11}$/.test(contact)) {
      throw new Error('无效的手机号');
    }
    return new SMSNotification(contact);
  }
}

class PushFactory implements NotificationFactory {
  createNotification(contact: string): Notification {
    return new PushNotification(contact);
  }
}

/**
 * 通知服务（使用工厂）
 */
class NotificationService {
  private factories: Map<string, NotificationFactory> = new Map();

  constructor() {
    // 注册所有工厂
    this.factories.set('email', new EmailFactory());
    this.factories.set('sms', new SMSFactory());
    this.factories.set('push', new PushFactory());
  }

  // 发送通知
  send(type: string, contact: string, message: string): void {
    const factory = this.factories.get(type);
    if (!factory) {
      throw new Error(`不支持的通知类型: ${type}`);
    }

    const notification = factory.createNotification(contact);
    notification.send(message);
  }

  // 添加新工厂（扩展方便）
  registerFactory(type: string, factory: NotificationFactory): void {
    this.factories.set(type, factory);
  }
}

// 使用
const service = new NotificationService();
service.send('email', 'user@example.com', '您的订单已发货');
service.send('sms', '13800138000', '您的验证码是123456');
service.send('push', 'device_abc123', '您有新的消息');
```

```typescript
/**

// 定义UI组件类型
type ButtonVariant = 'primary' | 'secondary' | 'danger';
type InputType = 'text' | 'password' | 'email';

// 按钮组件工厂
function createButton(
  variant: ButtonVariant,
  text: string,
  onClick: () => void
) {
  const baseClasses = 'px-4 py-2 rounded font-medium transition-colors';

  const variantClasses: Record<ButtonVariant, string> = {

    danger: 'bg-red-500 text-white hover:bg-red-600'

  return {
    type: 'button' as const,
    props: {
      className: `${baseClasses} ${variantClasses[variant]}`,
      onClick,
      children: text
    }
  };
}

// 表单输入框工厂
function createInput(
  type: InputType,
  placeholder: string,
  value: string,
  onChange: (value: string) => void
) {
  return {
    type: 'input' as const,
    props: {
      type,
      placeholder,
      value,
      onChange: (e: React.ChangeEvent<HTMLInputElement>) => onChange(e.target.value),
      className: 'w-full px-3 py-2 border border-gray-300 rounded focus:outline-none focus:ring-2'
    }
  };
}

});
```

#### 案例2：Node.js中的工厂模式——中间件创建

```typescript
/**
 * Node.js/Express中的工厂模式
 * 用于创建统一的中间件
 */

// 中间件工厂函数
function createAuthMiddleware(options: {
  required: boolean;
  roles?: string[];
}) {
  return async (req: Request, res: Response, next: NextFunction) => {
    // 从请求头获取token
    const token = req.headers.authorization?.replace('Bearer ', '');

    if (!token) {
      if (options.required) {
        return res.status(401).json({ error: '需要登录' });
      }
      return next();
    }

    try {
      // 验证token
      const user = await verifyToken(token);
      req.user = user;

      // 检查角色权限
      if (options.roles && !options.roles.includes(user.role)) {
        return res.status(403).json({ error: '权限不足' });
      }

      next();
    } catch (error) {
      res.status(401).json({ error: 'token无效' });
    }
  };
}

// 日志中间件工厂
function createLoggerMiddleware(level: 'info' | 'warn' | 'error') {
  const logMethods = {
    info: console.log,
    warn: console.warn,
    error: console.error
  };

  return (req: Request, res: Response, next: NextFunction) => {
    const start = Date.now();

    res.on('finish', () => {
      const duration = Date.now() - start;
      logMethods[level](
        `[${new Date().toISOString()}] ${req.method} ${req.path} - ${res.statusCode} (${duration}ms)`
      );
    });

    next();
  };
}

// 使用
app.use(createLoggerMiddleware('info'));
app.get('/admin', createAuthMiddleware({ required: true, roles: ['admin'] }));
app.get('/user', createAuthMiddleware({ required: true }));
```

#### 案例3：Vue中的工厂模式——表单组件创建

```typescript
/**
 * Vue中的工厂模式应用
 * 用于动态创建表单组件
 */

interface FormComponent {
  type: string;
  label: string;
  value: any;
  options?: { label: string; value: any }[];
}

// 表单组件工厂
function createFormComponent(config: {
  type: 'input' | 'select' | 'checkbox' | 'datepicker';
  label: string;
  modelValue?: any;
  options?: { label: string; value: any }[];
}): FormComponent {
  const base = {
    type: config.type,
    label: config.label,
    value: config.modelValue ?? null
  };

  switch (config.type) {
    case 'select':
      return { ...base, options: config.options || [] };
    case 'input':
    case 'checkbox':
    case 'datepicker':
      return base;
    default:
      throw new Error(`不支持的组件类型: ${config.type}`);
  }
}

// 使用
const formConfig = [
  createFormComponent({ type: 'input', label: '用户名', modelValue: '' }),
  createFormComponent({ type: 'input', label: '邮箱', modelValue: '' }),
  createFormComponent({
    type: 'select',
    label: '角色',
    modelValue: 'user',
    options: [
      { label: '普通用户', value: 'user' },
      { label: '管理员', value: 'admin' }
    ]
  })
];
```

### 2.5 工厂模式的优缺点

**优点：**

- 将对象的创建与使用分离，降低耦合度
- 符合开闭原则，新增产品类型不需要修改已有代码
- 集中管理对象的创建，便于维护
- 隐藏了具体类的实现细节

**缺点：**

- 类的数量会增加（每个产品都需要一个具体类）
- 引入工厂类增加了系统复杂度
- 简单场景下使用可能过度设计

## 三、抽象工厂模式（Abstract Factory Pattern）

### 3.1 生活类比

**就像不同品牌的手机工厂**——小米工厂生产小米手机、小米充电器、小米耳机；华为工厂生产华为手机、华为充电器、华为耳机。每个工厂都能生产一整套相关的产品，但具体产品不同。

抽象工厂模式就是创建**一系列相关对象**，而这些对象必须是一个产品族。

### 3.2 解决的问题

工厂模式适合创建单个对象，但如果需要创建**一套相关对象**，工厂模式就力不从心了。

例如：一款游戏需要同时支持Windows和Mac两个平台，每个平台都需要窗口、按钮、图标等UI组件：

```typescript
// ❌ 问题：每个平台都要单独创建
class WindowsWindow {}
class WindowsButton {}
class MacWindow {}
class MacButton {}

// 如果用普通工厂，需要分别创建
const windowFactory = {
  createWindow: (platform: string) => {
    if (platform === 'windows') return new WindowsWindow();
    if (platform === 'mac') return new MacWindow();
  }
  // ...
};

// 问题：无法保证一套产品的配套性
const winWindow = windowFactory.createWindow('windows');
```

```typescript
    const values = Object.values(data);
    const sql = `INSERT INTO ${table} (${keys.join(',')}) VALUES (${keys.map(() => '?').join(',')})`;

    await this.connection.query(`UPDATE ${table} SET ${sets} WHERE id = ?`, values);

    await this.connection.query(`DELETE FROM ${table} WHERE id = ?`, [id]);
```

### 3.5 抽象工厂模式的优缺点

**优点：**

- 确保一系列产品是配套的（同一产品族）
- 符合开闭原则，新增产品族不需要修改已有代码
- 客户端代码与具体产品解耦
- 方便切换整个产品族

**缺点：**

- 新增产品类型困难（需要修改抽象工厂接口）
- 类数量较多，增加系统复杂度
- 所有产品必须使用同一个产品族

## 四、建造者模式（Builder Pattern）

### 4.1 生活类比

**就像买电脑**——你可以告诉销售员"我要一台i7处理器、16GB内存、512GB固态硬盘、RTX3060显卡的电脑"，而不是自己一个个去组装。建造者模式就是让你一步步构建复杂对象，而不需要关心内部细节。

### 4.2 解决的问题

考虑创建一个用户对象的场景：

```typescript
 * 建造者模式基础实现
 * 使用链式调用构建对象，可读性强

// 产品类：User
class User {
  // 私有属性
  private constructor(
    private name: string,
    private email: string,
    private age: number | undefined,
    private phone: string | undefined,
    private address: Address | undefined,
    private avatar: string | undefined,
    private bio: string | undefined
  ) {}

  // 显示用户信息
  public toString(): string {
    return JSON.stringify({
      name: this.name,
      email: this.email,
      age: this.age,
      phone: this.phone,
      address: this.address,
      avatar: this.avatar,
      bio: this.bio
    }, null, 2);
  }
}

// 地址类（复杂对象的组成部分）
class Address {
  constructor(
    public country: string,
    public city: string,
    public street: string,
    public zipCode: string
  ) {}
}

// 建造者接口
interface UserBuilder {
  setName(name: string): UserBuilder;
  setEmail(email: string): UserBuilder;
  setAge(age: number): UserBuilder;
  setPhone(phone: string): UserBuilder;
  setAddress(country: string, city: string, street: string, zipCode: string): UserBuilder;
  setAvatar(avatar: string): UserBuilder;
  setBio(bio: string): UserBuilder;
  build(): User;
}

/**
 * 具体建造者
 * 负责一步步构建User对象
 */
class ConcreteUserBuilder implements UserBuilder {
  private name: string = '';
  private email: string = '';
  private age: number | undefined;
  private phone: string | undefined;
  private address: Address | undefined;
  private avatar: string | undefined;
  private bio: string | undefined;

  setName(name: string): UserBuilder {
    this.name = name;
    return this; // 返回this，支持链式调用
  }

  setEmail(email: string): UserBuilder {
    this.email = email;
    return this;
  }

  setAge(age: number): UserBuilder {
    this.age = age;
    return this;
  }

  setPhone(phone: string): UserBuilder {
    this.phone = phone;
    return this;
  }

  setAddress(country: string, city: string, street: string, zipCode: string): UserBuilder {
    this.address = new Address(country, city, street, zipCode);
    return this;
  }

  setAvatar(avatar: string): UserBuilder {
    this.avatar = avatar;
    return this;
  }

  setBio(bio: string): UserBuilder {
    this.bio = bio;
    return this;
  }

  /**
   * 最终构建方法
   * 验证必填字段，返回完整对象
   */
  build(): User {
    // 验证必填字段
    if (!this.name || !this.email) {
      throw new Error('姓名和邮箱是必填字段');
    }

    // 使用私有构造函数创建对象
    return new User(
      this.name,
      this.email,
      this.age,
      this.phone,
      this.address,
      this.avatar,
      this.bio
    );
  }
}
```

```typescript
class QueryBuilder {
  constructor(
    private table: string = '',
    private fields: string[] = ['*'],
    private conditions: Record<string, any> = {},
    private orderBy: string = '',
    private limitCount: number = 0,
    private limitOffset: number = 0
  ) {}

  // 转为SQL字符串
  toSQL(): string {
    let sql = `SELECT ${this.fields.join(', ')} FROM ${this.table}`;

    const whereClauses = Object.entries(this.conditions)
      .map(([key, value]) => `${key} = ${typeof value === 'string' ? `'${value}'` : value}`);
    if (whereClauses.length > 0) {
      sql += ` WHERE ${whereClauses.join(' AND ')}`;
    }

    if (this.orderBy) {
      sql += ` ORDER BY ${this.orderBy}`;
    }

    if (this.limitCount > 0) {
      sql += ` LIMIT ${this.limitCount}`;
      if (this.limitOffset > 0) {
        sql += ` OFFSET ${this.limitOffset}`;
      }
    }

    return sql;
  }
}

// 建造者接口
interface QueryBuilderBuilder {
  from(table: string): QueryBuilderBuilder;
  select(...fields: string[]): QueryBuilderBuilder;
  where(conditions: Record<string, any>): QueryBuilderBuilder;
  orderBy(field: string): QueryBuilderBuilder;
  limit(count: number, offset?: number): QueryBuilderBuilder;
  build(): QueryBuilder;
}

/**
 * 具体建造者
 */
class ConcreteQueryBuilder implements QueryBuilderBuilder {
  private query = new QueryBuilder();

  from(table: string): QueryBuilderBuilder {
    this.query = new QueryBuilder(table, this.query.fields, this.query.conditions, this.query.orderBy, this.query.limitCount, this.query.limitOffset);
    return this;
  }

  select(...fields: string[]): QueryBuilderBuilder {
    const currentFields = this.query.fields[0] === '*' ? [] : [...this.query.fields];
    const newFields = fields.length > 0 ? [...currentFields, ...fields] : ['*'];
    this.query = new QueryBuilder(this.query.table, newFields, this.query.conditions, this.query.orderBy, this.query.limitCount, this.query.limitOffset);
    return this;
  }

  where(conditions: Record<string, any>): QueryBuilderBuilder {
    this.query = new QueryBuilder(this.query.table, this.query.fields, { ...this.query.conditions, ...conditions }, this.query.orderBy, this.query.limitCount, this.query.limitOffset);
    return this;
  }

  orderBy(field: string): QueryBuilderBuilder {
    this.query = new QueryBuilder(this.query.table, this.query.fields, this.query.conditions, field, this.query.limitCount, this.query.limitOffset);
    return this;
  }

  limit(count: number, offset: number = 0): QueryBuilderBuilder {
    this.query = new QueryBuilder(this.query.table, this.query.fields, this.query.conditions, this.query.orderBy, count, offset);
    return this;
  }

  build(): QueryBuilder {
    return this.query;
  }
}

// ============ 使用示例 ============

const sql = new ConcreteQueryBuilder()
  .from('users')
  .select('id', 'name', 'email')
  .where({ status: 'active', role: 'admin' })
  .orderBy('created_at')
  .limit(10, 0)
  .build()
  .toSQL();

console.log('生成的SQL:', sql);
// 输出: SELECT id, name, email FROM users WHERE status = 'active' AND role = 'admin' ORDER BY created_at LIMIT 10
```

```typescript
/**
 * Node.js中使用建造者模式构建SQL查询
 * 类似 TypeORM 的 QueryBuilder
 */

class SQLQueryBuilder {
  private query: {
    type: 'SELECT' | 'INSERT' | 'UPDATE' | 'DELETE' | null;
    table: string;
    columns: string[];
    values: Record<string, any>;
    conditions: string[];
    orderBy: { column: string; direction: 'ASC' | 'DESC' }[];
    limit: number | null;
    offset: number | null;
    joins: string[];
  } = {
    type: null,
    table: '',
    columns: [],
    values: {},
    conditions: [],
    orderBy: [],
    limit: null,
    offset: null,
    joins: []
  };

  // SELECT 查询
  select(...columns: string[]): this {
    this.query.type = 'SELECT';
    this.query.columns = columns;
    return this;
  }

  // 指定表
  from(table: string): this {
    this.query.table = table;
    return this;
  }

  // WHERE 条件
  where(condition: string): this;
  where(column: string, value: any): this;
  where(column: string, operator: string, value: any): this;
  where(arg1: string, arg2?: any, arg3?: any): this {
    if (arg3 !== undefined) {
      // where('column', 'operator', value)
      this.query.conditions.push(`${arg1} ${arg2} ${typeof arg3 === 'string' ? `'${arg3}'` : arg3}`);
    } else if (arg2 !== undefined) {
      // where('column', value)
      this.query.conditions.push(`${arg1} = ${typeof arg2 === 'string' ? `'${arg2}'` : arg2}`);
    } else {
      // where('condition string')
      this.query.conditions.push(arg1);
    }
    return this;
  }

  // AND 条件
  and(condition: string): this;
  and(column: string, value: any): this;
  and(column: string, operator: string, value: any): this;
  and(arg1: string, arg2?: any, arg3?: any): this {
    if (arg3 !== undefined) {
      this.query.conditions.push(`AND ${arg1} ${arg2} ${typeof arg3 === 'string' ? `'${arg3}'` : arg3}`);
    } else if (arg2 !== undefined) {
      this.query.conditions.push(`AND ${arg1} = ${typeof arg2 === 'string' ? `'${arg2}'` : arg2}`);
    } else {
      this.query.conditions.push(`AND ${arg1}`);
    }
    return this;
  }

  // OR 条件
  or(condition: string): this;
  or(column: string, value: any): this;
  or(arg1: string, arg2?: any): this {
    if (arg2 !== undefined) {
      this.query.conditions.push(`OR ${arg1} = ${typeof arg2 === 'string' ? `'${arg2}'` : arg2}`);
    } else {
      this.query.conditions.push(`OR ${arg1}`);
    }
    return this;
  }

  // JOIN
  join(table: string, condition: string): this {
    this.query.joins.push(`JOIN ${table} ON ${condition}`);
    return this;
  }

  leftJoin(table: string, condition: string): this {
    this.query.joins.push(`LEFT JOIN ${table} ON ${condition}`);
    return this;
  }

  // ORDER BY
  orderBy(column: string, direction: 'ASC' | 'DESC' = 'ASC'): this {
    this.query.orderBy.push({ column, direction });
    return this;
  }

  // LIMIT
  limit(count: number): this {
    this.query.limit = count;
    return this;
  }

  // OFFSET
  offset(count: number): this {
    this.query.offset = count;
    return this;
  }

  // INSERT
  insert(values: Record<string, any>): this {
    this.query.type = 'INSERT';
    this.query.values = values;
    return this;
  }

  // UPDATE
  update(values: Record<string, any>): this {
    this.query.type = 'UPDATE';
    this.query.values = values;
    return this;
  }

  // DELETE
  delete(): this {
    this.query.type = 'DELETE';
    return this;
  }

  // 编译成SQL字符串
  toSQL(): string {
    if (!this.query.type || !this.query.table) {
      throw new Error('无效的查询');
    }

    switch (this.query.type) {
      case 'SELECT': {
        const columns = this.query.columns.join(', ') || '*';
        let sql = `SELECT ${columns} FROM ${this.query.table}`;

        if (this.query.joins.length > 0) {
          sql += ' ' + this.query.joins.join(' ');
        }

        if (this.query.conditions.length > 0) {
          sql += ' WHERE ' + this.query.conditions.join(' ');
        }

        if (this.query.orderBy.length > 0) {
          sql += ' ORDER BY ' + this.query.orderBy.map(o => `${o.column} ${o.direction}`).join(', ');
        }

        if (this.query.limit !== null) {
          sql += ` LIMIT ${this.query.limit}`;
        }

        if (this.query.offset !== null) {
          sql += ` OFFSET ${this.query.offset}`;
        }

        return sql;
      }

      case 'INSERT': {
        const columns = Object.keys(this.query.values).join(', ');
        const values = Object.values(this.query.values).map(v =>
          typeof v === 'string' ? `'${v}'` : v
        ).join(', ');
        return `INSERT INTO ${this.query.table} (${columns}) VALUES (${values})`;
      }

      case 'UPDATE': {
        const sets = Object.entries(this.query.values)
          .map(([k, v]) => `${k} = ${typeof v === 'string' ? `'${v}'` : v}`)
          .join(', ');
        let sql = `UPDATE ${this.query.table} SET ${sets}`;

        if (this.query.conditions.length > 0) {
          sql += ' WHERE ' + this.query.conditions.join(' ');
        }

        return sql;
      }

      case 'DELETE': {
        let sql = `DELETE FROM ${this.query.table}`;

        if (this.query.conditions.length > 0) {
          sql += ' WHERE ' + this.query.conditions.join(' ');
        }

        return sql;
      }

      default:
        throw new Error('未知的查询类型');
    }
  }

  // 清空状态
  clear(): void {
    this.query = {
      type: null,
      table: '',
      columns: [],
      values: {},
      conditions: [],
      orderBy: [],
      limit: null,
      offset: null,
      joins: []
    };
  }
}

// 复杂查询

  .select('id', 'name', 'email', 'status')
  .from('users')

  .where('status', 'active')

  .orderBy('created_at', 'DESC')
  .limit(20)
  .offset(0)
  .toSQL();

console.log('查询1:', sql1);

console.log('插入:', sql2);
```

### 4.5 建造者模式的优缺点

**优点：**

- 代码可读性强，使用链式调用
- 可以逐步构建对象，精细控制创建过程
- 可以创建不可变对象
- 同一构建过程可以创建不同表示的对象
- 符合单一职责原则

**缺点：**

- 类数量增加（每个产品需要对应的Builder）
- 相对于简单构造函数，代码更复杂
- 需要保证Builder的线程安全

## 五、原型模式（Prototype Pattern）

### 5.1 生活类比

**就像使用复印机**——你有一份文件需要多份副本，不需要重新打字，直接复印就行。原型模式就是通过"克隆"现有对象来创建新对象，而不是从头开始创建。

```typescript
/**
 * 原型模式基础实现
 * 通过克隆已有对象来创建新对象
 */

// 原型接口
interface Prototype<T> {
  clone(): T;
}

/**
 * 游戏怪物原型
 */
class Monster implements Prototype<Monster> {
  constructor(
    public name: string,
    public health: number,
    public attack: number,
    public defense: number,
    public skills: string[],
    public drops: { item: string; rate: number }[],
    public position?: { x: number; y: number }
  ) {}

  /**
   * 克隆方法
   * 创建一个新的实例，复制所有属性
   */
  clone(): Monster {
    // 深拷贝：确保引用类型的属性也被正确复制
    return new Monster(
      this.name,
      this.health,
      this.attack,
      this.defense,
      [...this.skills], // 浅拷贝基本类型数组
      this.drops.map(d => ({ ...d })), // 深拷贝对象数组
      this.position ? { ...this.position } : undefined
    );
  }

  /**
   * 另一种克隆方式：创建变体
   * 在克隆的基础上修改某些属性
   */
  cloneWithVariation(options: Partial<Omit<Monster, 'clone'>>): Monster {
    const clone = this.clone();

    if (options.name !== undefined) clone.name = options.name;
    if (options.health !== undefined) clone.health = options.health;
    if (options.attack !== undefined) clone.attack = options.attack;
    if (options.defense !== undefined) clone.defense = options.defense;
    if (options.skills !== undefined) clone.skills = [...options.skills];
    if (options.drops !== undefined) clone.drops = options.drops.map(d => ({ ...d }));
    if (options.position !== undefined) clone.position = { ...options.position };

    return clone;
  }
}
```

```typescript
interface Prototype<T> {

class Document implements Prototype<Document> {
  constructor(
    public title: string,
    public content: string,
    public metadata: Record<string, any>,
    public blocks: Block[],
    public collaborators: string[]
  ) {}

  /**
   * 使用JSON序列化实现深拷贝

  clone(): Document {
    const jsonStr = JSON.stringify(this);
    const cloned = JSON.parse(jsonStr);
    return new Document(
      cloned.title,
      cloned.content,
      cloned.metadata,
      cloned.blocks.map(b => new Block(b.type, b.content, b.props)),
      cloned.collaborators
    );
  }

  /**
   * 创建修改后的副本
   */
  cloneWithChanges(changes: Partial<Pick<Document, 'title' | 'content'>>): Document {
    const clone = this.clone();

    if (changes.title) clone.title = changes.title;
    if (changes.content) clone.content = changes.content;

    return clone;
  }
}

class Block {
  constructor(
    public type: string,
    public content: string,
    public props: Record<string, any>
  ) {}
}

// ============ 使用示例 ============

const originalDoc = new Document(
  '项目文档',
  '这是文档内容...',
  { author: '张三', createdAt: '2024-01-01', version: 1 },
  [
    new Block('heading', '标题', { level: 1 }),
    new Block('paragraph', '段落内容', { className: 'text-lg' }),
    new Block('code', 'const x = 1;', { language: 'javascript' })
  ],
  ['user1', 'user2']
);

// 克隆文档
const docCopy = originalDoc.clone();
docCopy.title = '项目文档（副本）';
docCopy.collaborators.push('user3');

console.log('原始文档协作者:', originalDoc.collaborators); // ['user1', 'user2']
console.log('副本文档协作者:', docCopy.collaborators);     // ['user1', 'user2', 'user3']

// 创建修改后的副本
const docV2 = originalDoc.cloneWithChanges({
  title: '项目文档 V2',
  content: '更新后的内容...'
});
```

```typescript
/**
 * Node.js中使用原型模式处理配置对象
 * 适合需要基于默认配置创建自定义配置的场景
 */

interface RequestConfig {
  url: string;
  method: 'GET' | 'POST' | 'PUT' | 'DELETE';
  headers: Record<string, string>;
  timeout: number;
  retries: number;
  data?: any;
}

// 默认配置原型
const defaultConfig: RequestConfig = {
  url: '',
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
    'Accept': 'application/json'
  },
  timeout: 30000,
  retries: 3
};

/**
 * 请求配置工厂（原型模式应用）
 */
class RequestConfigFactory {
  constructor(private prototype: RequestConfig) {}

  /**
   * 创建GET请求配置
   */
  createGet(url: string, headers?: Record<string, string>): RequestConfig {
    return this.create({ url, method: 'GET', headers });
  }

  /**
   * 创建POST请求配置
   */
  createPost(url: string, data: any, headers?: Record<string, string>): RequestConfig {
    return this.create({
      url,
      method: 'POST',
      data,
      headers: {
        ...this.prototype.headers,
        ...headers
      }
    });
  }

  /**
   * 创建PUT请求配置
   */
  createPut(url: string, data: any, headers?: Record<string, string>): RequestConfig {
    return this.create({
      url,
      method: 'PUT',
      data,
      headers: {
        ...this.prototype.headers,
        ...headers
      }
    });
  }

  /**
   * 创建DELETE请求配置
   */
  createDelete(url: string, headers?: Record<string, string>): RequestConfig {
    return this.create({ url, method: 'DELETE', headers });
  }

  /**
   * 通用创建方法
   */
  create(options: Partial<RequestConfig>): RequestConfig {
    // 深拷贝原型
    const config: RequestConfig = {
      url: this.prototype.url,
      method: this.prototype.method,
      headers: { ...this.prototype.headers },
      timeout: this.prototype.timeout,
      retries: this.prototype.retries
    };

    // 合并选项
    if (options.url !== undefined) config.url = options.url;
    if (options.method !== undefined) config.method = options.method;
    if (options.headers !== undefined) config.headers = { ...config.headers, ...options.headers };
    if (options.timeout !== undefined) config.timeout = options.timeout;
    if (options.retries !== undefined) config.retries = options.retries;
    if (options.data !== undefined) config.data = options.data;

    return config;
  }
}

// ============ 使用示例 ============

const factory = new RequestConfigFactory(defaultConfig);

// 创建各种请求配置
const getUserConfig = factory.createGet('/api/users/123', {
  'Authorization': 'Bearer token123'
});

const createPostConfig = factory.createPost('/api/posts', {
  title: '新文章',
  content: '文章内容...',
  authorId: 1
}, {
  'X-Custom-Header': 'custom-value'
});

const updateConfig = factory.create({
  url: '/api/posts/456',
  method: 'PUT',
  data: { title: '更新后的标题' }
});

const deleteConfig = factory.createDelete('/api/comments/789');

// 模拟发送请求

  console.log(`发送 ${config.method} 请求到 ${config.url}`);
  console.log('请求头:', config.headers);
  console.log('超时:', config.timeout);
  console.log('重试次数:', config.retries);
  if (config.data) {
    console.log('请求数据:', config.data);
  }
}

await sendRequest(getUserConfig);
await sendRequest(createPostConfig);
```

```typescript
/**
 * Vue中使用原型模式创建表单字段
 * 从模板批量创建相似字段
 */

interface FormField {
  type: 'text' | 'email' | 'password' | 'select' | 'checkbox';
  name: string;
  label: string;
  placeholder?: string;
  options?: { label: string; value: string }[];
  rules?: ((value: any) => boolean | string)[];
  value?: any;
  disabled?: boolean;
}

/**
 * 表单字段原型工厂
 */
class FormFieldFactory {
  private prototypes: Map<string, FormField> = new Map();

  /**
   * 注册原型
   */
  registerPrototype(type: string, field: FormField): void {
    this.prototypes.set(type, field);
  }

  /**
   * 创建字段（从原型克隆）
   */
  create(type: string, name: string, label: string, overrides?: Partial<FormField>): FormField | null {
    const prototype = this.prototypes.get(type);
    if (!prototype) {
      console.error(`未找到类型为 ${type} 的原型`);
      return null;
    }

    // 克隆原型
    const field: FormField = {
      type: prototype.type,
      name,
      label,
      placeholder: prototype.placeholder,
      options: prototype.options ? [...prototype.options] : undefined,
      rules: prototype.rules ? [...prototype.rules] : undefined,
      value: undefined,
      disabled: prototype.disabled
    };

    // 应用覆盖
    if (overrides) {
      Object.assign(field, overrides);
      // 深拷贝options
      if (overrides.options) {
        field.options = overrides.options.map(o => ({ ...o }));
      }
    }

    return field;
  }

  /**
   * 批量创建文本字段
   */
  createTextFields(fields: Array<{ name: string; label: string; placeholder?: string }>): FormField[] {
    return fields.map(f =>
      this.create('text', f.name, f.label, { placeholder: f.placeholder })!
    );
  }

  /**
   * 批量创建必填字段
   */
  createRequiredFields(fields: Array<{ type: string; name: string; label: string }>): FormField[] {
    return fields.map(f => {
      const field = this.create(f.type, f.name, f.label)!;
      field.rules = [(value: any) => !!value || `${f.label}不能为空`];
      return field;
    });
  }
}

factory.registerPrototype('select', {
```

### 5.5 原型模式的优缺点

**优点：**

- 简化对象创建过程，提高性能（避免重复初始化）
- 通过深拷贝避免引用共享问题
- 可以创建复杂对象的副本
- 降低类的数量（不需要为每种组合创建类）

**缺点：**

- 深拷贝实现复杂，容易出错
- 原型克隆需要类是可拷贝的
- 过度使用会增加系统复杂度

## 六、总结与对比

### 6.1 五种创建型模式对比

| 模式 | 核心思想 | 复杂度 | 适用场景 | 一句话总结 |
|------|----------|--------|----------|------------|
| **单例模式** | 全局唯一实例 | 低 | 全局共享资源 | 一个类只有一个对象 |
| **工厂模式** | 统一创建对象 | 中 | 创建同类不同种对象 | 工厂流水线生产产品 |
| **抽象工厂模式** | 创建产品族 | 高 | 一系列配套对象 | 品牌工厂生产全套产品 |
| **建造者模式** | 分步构建复杂对象 | 中 | 参数多、构建步骤复杂 | 一步步组装产品 |
| **原型模式** | 克隆现有对象 | 低 | 大量相似对象 | 复印机复印文件 |

### 6.2 选择指南

```
需要创建对象吗？
├── 需要一个唯一实例吗？
│   └── 是 → 单例模式
├── 需要统一创建流程吗？
│   └── 是 → 工厂模式
│       └── 需要创建产品族吗？
│           └── 是 → 抽象工厂模式
├── 构建过程复杂吗？
│   └── 是 → 建造者模式
├── 已有类似对象吗？
│   └── 是 → 原型模式
└── 以上都不是 → 直接new
```

### 6.3 实际项目中的应用建议

**Vue项目推荐使用：**

- 单例模式：Pinia/Vuex状态管理、全局配置
- 工厂模式：组件库按类型创建组件、表单字段
- 建造者模式：复杂表单配置、SQL查询构建
- 原型模式：组件模板复用、表单字段模板

**React项目推荐使用：**

- 单例模式：全局状态管理（如Zustand store）
- 工厂模式：HOC工厂、组件工厂
- 抽象工厂模式：主题系统、多语言组件
- 建造者模式：表单构建、查询构建器
- 原型模式：组件模板、数据克隆

**Node.js项目推荐使用：**

- 单例模式：数据库连接池、日志服务
- 工厂模式：中间件工厂、验证器工厂
- 抽象工厂模式：多数据库支持、多租户系统
- 建造者模式：SQL Query Builder、复杂配置对象
- 原型模式：配置对象模板、请求模板
