# 生产环境与开发环境核心区别

> 很多人都有过这样的经历：在自己电脑上跑得好好的代码，一部署到服务器就出问题了。这篇文章会详细解释为什么，以及如何避免这些问题。我会像一个经验丰富的老工程师一样，把踩过的坑都告诉你。

## 一、先理解一个核心理念

### 1.1 开发环境是"家"，生产环境是"战场"

想象一下：

```
开发环境 = 在自己家做菜
├── 厨房用品齐全（IDE、调试工具）
├── 食材随叫随到（本地数据库）
├── 失败了没关系（可以无限次重试）
├── 只有你自己吃（团队1-2人）
└── 锅烧糊了也没事（重启电脑解决）

生产环境 = 在万人食堂做大锅饭
├── 厨房标准化（固定配置）
├── 食材有供应链（远程数据库）
├── 失败会影响很多人（成千上万用户）
├── 有人专门检查（监控、告警）
└── 出了问题必须快速解决（没有重试的机会）
```

### 1.2 最常见的"本地能跑，生产不行"问题

| 问题类型 | 开发环境表现 | 生产环境表现 |
|---------|-------------|-------------|
| 路径问题 | `C:\Users\Me\project` | `/var/www/app` |
| 大小写敏感 | Windows不区分大小写 | Linux严格区分大小写 |
| 文件权限 | 管理员权限随便写 | www-data用户受限 |
| 环境变量 | hardcode也能跑 | 必须用环境变量 |
| 并发处理 | 单请求没问题 | 大量并发出问题 |
| 数据库 | 本地MySQL | 远程PostgreSQL |
| 网络延迟 | localhost无延迟 | 远程服务器有延迟 |

## 二、配置差异详解

### 2.1 文件路径和结构

这是最容易出错的地方。Windows和Mac使用反斜杠`\`，Linux使用正斜杠`/`：

```typescript
// ❌ 错误的写法：硬编码路径
const UPLOAD_PATH = 'C:\\Users\\Developer\\uploads';
const CONFIG_PATH = 'C:\\Users\\Developer\\config.json';

// ✅ 正确的写法：使用相对路径或环境变量
const UPLOAD_PATH = path.join(process.cwd(), 'uploads');
const CONFIG_PATH = path.join(process.cwd(), 'config', 'app.json');

// ✅ 更好的写法：完全使用环境变量
const UPLOAD_PATH = process.env.UPLOAD_PATH || '/var/www/uploads';
const CONFIG_PATH = process.env.CONFIG_PATH || '/etc/app/config.json';

/**
 * 为什么？
 *
 * 开发时，你的代码在 C:\project\ 下
 * 生产时，代码在 /var/www/app/ 下
 * 如果hardcode路径，上线后系统会找不到文件
 */
```

### 2.2 大小写敏感问题

这是Windows开发者最容易踩的坑：

```bash
# 场景：用户上传了一张图片

# Windows系统（不区分大小写）
# 这三种写法都能找到同一个文件：
photo.jpg
Photo.jpg
PHOTO.JPG

# Linux系统（严格区分大小写）
# 这三种写法指向不同的文件！
photo.jpg   # 文件1
Photo.jpg   # 文件2（不存在，返回404）
PHOTO.JPG   # 文件3（不存在，返回404）
```

```typescript
// 实际项目中的问题场景

// 用户访问文档，URL是 /document/myPhoto.jpg
// 但实际存储的文件名是 /document/myphoto.jpg

// Windows开发环境：能找到 ✓
// Linux生产环境：找不到，报404 ✗

// 解决方案：统一使用小写文件名存储
function saveUploadFile(file: File): string {
  // 获取文件扩展名
  const ext = path.extname(file.name).toLowerCase();

  // 生成统一的小写文件名
  const baseName = path.basename(file.name, ext).toLowerCase();

  // 使用时间戳+随机数避免冲突
  const fileName = `${baseName}_${Date.now()}_${Math.random().toString(36).substr(2, 9)}${ext}`;

  const savePath = path.join(UPLOAD_DIR, fileName);

  // 写入文件
  fs.writeFileSync(savePath, file.buffer);

  // 返回统一小写的文件路径
  return fileName;  // 返回相对路径，便于在不同环境下使用
}
```

### 2.3 文件权限问题

生产环境的权限管理比开发环境严格得多：

```bash
# ============================================
# Linux文件权限详解（用比喻理解）
# ============================================

# Linux的文件权限分为三组：
# 所有者(Owner) | 用户组(Group) | 其他用户(Others)
# 每组又分为：读(4) | 写(2) | 执行(1)

# 举例：rwxr-xr-x
# rwx (7) = 所有者有读、写、执行权限
# r-x (5) = 用户组有读、执行权限，没有写权限
# r-x (5) = 其他用户有读、执行权限，没有写权限

# 常见权限组合
chmod 755 file  # rwxr-xr-x  最常见，文件可执行
chmod 644 file  # rw-r--r--  普通文件
chmod 600 file  # rw-------  敏感文件，只有所有者可读写
chmod 700 dir   # rwx------  目录，只有所有者可访问

# ============================================
# 常见权限问题场景
# ============================================

# 场景1：Node.js应用无法写入日志
# 问题：日志目录权限不对
# 解决：
mkdir -p /var/log/myapp
chown -R www-data:www-data /var/log/myapp  # 改成运行应用的用户

# 场景2：无法创建上传文件
# 问题：上传目录没有写权限
# 解决：
mkdir -p /var/www/myapp/uploads
chown -R www-data:www-data /var/www/myapp/uploads
chmod 755 /var/www/myapp/uploads  # 允许写入

# 场景3：npm安装全局包失败
# 问题：没有sudo权限
# 解决：
# 方案1：配置npm使用用户目录
npm config set prefix ~/.npm-global
export PATH=~/.npm-global/bin:$PATH

# 方案2：使用nvm管理Node版本
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install --lts

# ============================================
# PM2的权限处理
# ============================================

# 使用PM2运行时，确保以正确用户启动
# 创建ecosystem.config.js
module.exports = {
  apps: [{
    name: 'myapp',
    script: 'dist/main.js',
    // 指定运行用户（需要在配置中设置）
    // 如果不指定，可能以root运行，会有安全隐患
    instances: 1,
    autorestart: true,
    // 日志
    error_file: '/var/log/myapp/error.log',
    out_file: '/var/log/myapp/out.log',
    time: true
  }]
};

# 以www-data用户启动（安全做法）
su - www-data -s /bin/bash -c 'pm2 start ecosystem.config.js'
```

### 2.4 环境变量配置

环境变量是连接开发和生产的桥梁，必须理解清楚：

```bash
# ============================================
# 环境变量的基础知识
# ============================================

# 1. 查看当前所有环境变量
env
printenv

# 2. 查看特定变量
echo $PATH
echo $HOME
echo $NODE_ENV

# 3. 临时设置环境变量（只对当前终端有效）
export NODE_ENV=development
node index.js

# 4. 永久设置环境变量
# Linux下编辑 ~/.bashrc 或 ~/.bash_profile
echo 'export NODE_ENV=production' >> ~/.bashrc
source ~/.bashrc

# ============================================
# Node.js中读取环境变量
# ============================================

// process.env 是Node.js内置的对象
// 包含了所有的环境变量

console.log(process.env.NODE_ENV);      // development / production
console.log(process.env.PORT);           // 服务器端口
console.log(process.env.DATABASE_URL);   // 数据库连接字符串

// 读取时指定默认值
const PORT = process.env.PORT || 3000;
const DB_URL = process.env.DATABASE_URL || 'postgresql://localhost:5432/mydb';

// 读取布尔值（环境变量都是字符串，需要转换）
const ENABLE_CACHE = process.env.ENABLE_CACHE === 'true';
const IS_DEBUG = process.env.NODE_ENV !== 'production';

// ============================================
# 生产环境变量文件（.env.production）
# ============================================

# 生产环境绝不能把敏感信息提交到Git！
# .env.production 应该只放在服务器上

# 创建生产环境变量文件
cat > /var/www/myapp/.env.production << 'EOF'
# 应用配置
NODE_ENV=production
PORT=3000
LOG_LEVEL=error

# 数据库配置（生产数据库地址）
DATABASE_URL=postgresql://prod_user:password@prod-db.example.com:5432/mydb

# Redis配置
REDIS_URL=redis://prod-redis.example.com:6379

# JWT密钥（必须是非常复杂的字符串！）
JWT_SECRET=your-super-secret-jwt-key-here-change-in-production
JWT_EXPIRES_IN=7d

# 第三方服务密钥
STRIPE_SECRET_KEY=sk_live_xxxxxxxxxxxxxxxxxxxxx
SENDGRID_API_KEY=SG.xxxxxxxxxxxxxxxxxxxxxx

# AWS S3配置
AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
AWS_REGION=us-east-1
AWS_BUCKET_NAME=myapp-production

# 上传文件大小限制
MAX_FILE_SIZE=10485760
EOF

# 权限设置为600（只有所有者可读写）
chmod 600 /var/www/myapp/.env.production

# ============================================
# Next.js的环境变量处理
# ============================================

# Next.js的环境变量有特殊规则：
#
# 1. NEXT_PUBLIC_ 前缀的变量会被嵌入到客户端代码中
# 2. 没有前缀的变量只会在服务端使用
#
# 原理：
# - 服务端变量：构建时替换，客户端拿不到
# - NEXT_PUBLIC_变量：构建时替换，客户端可以访问

# .env.local（开发环境）
NEXT_PUBLIC_API_URL=http://localhost:3000/api
DATABASE_URL=postgresql://localhost:5432/mydb_dev

# .env.production（生产环境）
NEXT_PUBLIC_API_URL=https://api.example.com/api
DATABASE_URL=postgresql://prod-db.example.com:5432/mydb

# 在代码中使用
// 只在服务端使用，不会泄露给客户端
const dbPassword = process.env.DATABASE_PASSWORD;

// 客户端和服务端都能用，但值会嵌入到构建产物中
const apiUrl = process.env.NEXT_PUBLIC_API_URL;
```

## 三、安全考虑差异

### 3.1 开发环境 vs 生产环境的安全策略

```
┌─────────────────────────────────────────────────────────┐
│                    安全策略对比                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  开发环境                                                │
│  ├── 可以开启详细错误日志                                │
│  ├── 可以使用测试密钥（测试Stripe等）                    │
│  ├── 可以允许CORS从任何来源请求                          │
│  ├── 可以关闭HTTPS验证                                  │
│  ├── 可以用简单的测试密码                                │
│  └── 可以开启调试模式                                   │
│                                                         │
│  生产环境                                                │
│  ├── 错误信息必须隐藏（不能让用户看到堆栈信息）           │
│  ├── 必须使用真实的SSL证书                              │
│  ├── CORS必须精确配置允许的来源                          │
│  ├── HTTPS不能关闭验证                                  │
│  ├── 密码必须足够复杂，定期更换                          │
│  └── 调试模式必须关闭                                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 敏感信息处理

```typescript
// ============================================
// 敏感信息处理最佳实践
// ============================================

// ❌ 绝对不能在代码中硬编码敏感信息
const API_KEY = 'sk_test_1234567890abcdef';  // 危险！
const DB_PASSWORD = 'my_secret_password';     // 危险！

// ❌ 不能把敏感信息提交到Git
// .env 文件应该加入 .gitignore
// .env
// .env.local
// .env.production

// ✅ 正确做法：使用环境变量
const apiKey = process.env.STRIPE_SECRET_KEY;

// ✅ 敏感配置应该放在服务器上，通过环境变量注入
// 生产服务器上的 .env.production 文件由运维管理
// 开发者在本地使用 .env.local（不提交到Git）

// ============================================
// 配置敏感信息的最佳实践
// ============================================

// 创建配置模块（config.ts）
class AppConfig {
  // 单例模式，确保配置只加载一次
  private static instance: AppConfig;

  // 数据库配置
  readonly database: {
    host: string;
    port: number;
    database: string;
    username: string;
    password: string;
    ssl: boolean;
  };

  // API配置
  readonly api: {
    url: string;
    timeout: number;
    retries: number;
  };

  // JWT配置
  readonly jwt: {
    secret: string;
    expiresIn: string;
  };

  // S3配置
  readonly s3: {
    bucket: string;
    region: string;
    accessKeyId: string;
    secretAccessKey: string;
  };

  private constructor() {
    // 从环境变量加载配置
    // 如果环境变量不存在，抛出错误（生产环境必须有这些值）

    const required = (name: string, value?: string): string => {
      if (!value) {
        throw new Error(`Missing required environment variable: ${name}`);
      }
      return value;
    };

    const optional = (name: string, defaultValue?: string): string | undefined => {
      return process.env[name] || defaultValue;
    };

    this.database = {
      host: required('DB_HOST'),
      port: parseInt(optional('DB_PORT', '5432')!),
      database: required('DB_NAME'),
      username: required('DB_USER'),
      password: required('DB_PASSWORD'),
      // 生产环境必须启用SSL
      ssl: process.env.NODE_ENV === 'production',
    };

    this.api = {
      url: required('API_URL'),
      timeout: parseInt(optional('API_TIMEOUT', '30000')!),
      retries: parseInt(optional('API_RETRIES', '3')!),
    };

    this.jwt = {
      secret: required('JWT_SECRET'),
      expiresIn: optional('JWT_EXPIRES_IN', '7d')!,
    };

    this.s3 = {
      bucket: required('AWS_BUCKET_NAME'),
      region: required('AWS_REGION'),
      accessKeyId: required('AWS_ACCESS_KEY_ID'),
      secretAccessKey: required('AWS_SECRET_ACCESS_KEY'),
    };
  }

  // 获取单例实例
  static getInstance(): AppConfig {
    if (!AppConfig.instance) {
      AppConfig.instance = new AppConfig();
    }
    return AppConfig.instance;
  }

  // 验证配置是否完整
  validate(): void {
    const errors: string[] = [];

    // 检查数据库连接
    if (!this.database.host) errors.push('Database host is required');
    if (!this.database.password) errors.push('Database password is required');

    // 检查JWT密钥强度
    if (this.jwt.secret.length < 32) {
      errors.push('JWT secret must be at least 32 characters long');
    }

    // 生产环境额外检查
    if (process.env.NODE_ENV === 'production') {
      if (!this.database.ssl) errors.push('Production database must use SSL');
      if (this.api.url.startsWith('http://')) {
        errors.push('Production API URL must use HTTPS');
      }
    }

    if (errors.length > 0) {
      throw new Error(`Configuration validation failed:\n${errors.join('\n')}`);
    }
  }
}

// 使用配置
const config = AppConfig.getInstance();
config.validate();

// 导出给其他模块使用
export const database = config.database;
export const api = config.api;
export const jwt = config.jwt;
export const s3 = config.s3;
```

### 3.3 生产环境的错误处理

```typescript
// ============================================
// 生产环境错误处理策略
// ============================================

// ❌ 开发环境：显示详细错误（便于调试）
// 生产环境：隐藏错误细节，只显示通用信息

// 全局错误处理中间件
export const errorHandler = (
  err: Error,
  req: Request,
  res: Response,
  next: NextFunction
) => {
  // 记录详细错误日志（管理员可以看到）
  console.error('Error occurred:', {
    message: err.message,
    stack: err.stack,
    url: req.url,
    method: req.method,
    ip: req.ip,
    userAgent: req.get('user-agent'),
    userId: req.user?.id,
    timestamp: new Date().toISOString(),
  });

  // 判断是开发环境还是生产环境
  if (process.env.NODE_ENV === 'development') {
    // 开发环境：返回详细错误信息
    res.status(500).json({
      success: false,
      error: {
        message: err.message,
        stack: err.stack,
        details: err
      }
    });
  } else {
    // 生产环境：只返回通用错误信息
    // 错误详情记录到日志，不返回给用户

    // 发送错误通知（邮件、钉钉等）
    sendErrorNotification({
      message: err.message,
      stack: err.stack,
      url: req.url,
      timestamp: new Date().toISOString(),
    });

    res.status(500).json({
      success: false,
      error: {
        // 用户友好的错误信息
        message: '服务器内部错误，请稍后重试',
        // 错误码，便于追踪
        code: 'INTERNAL_ERROR',
        // 错误ID，可以用来查询日志
        errorId: generateErrorId()
      }
    });
  }
};

// ============================================
// 自定义错误类
// ============================================

class AppError extends Error {
  constructor(
    message: string,
    public statusCode: number = 500,
    public code: string = 'INTERNAL_ERROR',
    public isOperational: boolean = true  // 是否是业务错误（而非编程错误）
  ) {
    super(message);
    Object.setPrototypeOf(this, AppError.prototype);
    Error.captureStackTrace(this, this.constructor);
  }
}

// 常见的业务错误类型
class ValidationError extends AppError {
  constructor(message: string, public fields?: Record<string, string>) {
    super(message, 400, 'VALIDATION_ERROR');
  }
}

class AuthenticationError extends AppError {
  constructor(message: string = '请先登录') {
    super(message, 401, 'AUTHENTICATION_ERROR');
  }
}

class AuthorizationError extends AppError {
  constructor(message: string = '没有权限访问此资源') {
    super(message, 403, 'AUTHORIZATION_ERROR');
  }
}

class NotFoundError extends AppError {
  constructor(resource: string = '资源') {
    super(`${resource}不存在`, 404, 'NOT_FOUND');
  }
}

class RateLimitError extends AppError {
  constructor(message: string = '请求过于频繁，请稍后重试') {
    super(message, 429, 'RATE_LIMIT_ERROR');
  }
}

// 使用示例
function getUserById(id: string) {
  if (!id) {
    // 业务错误：参数验证失败
    throw new ValidationError('用户ID不能为空');
  }

  const user = db.findUser(id);

  if (!user) {
    // 业务错误：资源不存在
    throw new NotFoundError('用户');
  }

  return user;
}

// 路由中使用
app.get('/users/:id', async (req, res, next) => {
  try {
    const user = getUserById(req.params.id);
    res.json({ success: true, data: user });
  } catch (error) {
    // 只处理AppError，其他错误按默认方式处理
    next(error);
  }
});
```

### 3.4 CORS配置差异

```typescript
// ============================================
// CORS（跨域资源共享）配置
// ============================================

// CORS用来控制哪些网站可以访问你的API
// 开发环境和生产环境的配置完全不同

// ❌ 开发环境：允许所有来源（方便调试）
app.use(cors({
  origin: '*',  // 允许任何来源访问
  credentials: true
}));

// ✅ 生产环境：精确配置允许的来源
app.use(cors({
  // 生产环境只允许特定域名
  origin: [
    'https://www.example.com',           // 主站
    'https://app.example.com',           // 子域名应用
    'https://admin.example.com',         // 管理后台
  ],
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization'],
  // 暴露响应头给客户端
  exposedHeaders: ['X-Total-Count', 'X-Page-Count'],
  // 预检请求缓存时间
  maxAge: 86400  // 24小时
}));

// ============================================
// Nginx层面的CORS配置
// ============================================

// 如果使用Nginx反向代理，也需要在Nginx中配置CORS

/*
# /etc/nginx/conf.d/api.conf

server {
    listen 80;
    server_name api.example.com;

    # CORS配置
    add_header 'Access-Control-Allow-Origin' 'https://www.example.com' always;
```

### 4.3 缓存策略差异

```typescript
// ============================================
// 缓存策略（开发 vs 生产）
// ============================================

// 开发环境：缓存可以随意使用，方便调试
// 生产环境：缓存策略直接影响性能和一致性

// 1. 内存缓存（适合单机部署）
const memoryCache = new Map<string, { data: any; expire: number }>();

function getFromMemoryCache<T>(key: string): T | null {
  const item = memoryCache.get(key);
  if (!item) return null;

  // 检查是否过期
  if (Date.now() > item.expire) {
    memoryCache.delete(key);
    return null;
  }

  return item.data as T;
}

function setMemoryCache<T>(key: string, data: T, ttlSeconds: number = 300): void {
  memoryCache.set(key, {
    data,
    expire: Date.now() + ttlSeconds * 1000
  });
}

// 2. Redis缓存（适合分布式部署）
async function getFromRedisCache<T>(key: string): Promise<T | null> {
  const data = await redis.get(key);
  if (!data) return null;

  try {
    return JSON.parse(data) as T;
  } catch {
    return null;
  }
}

async function setRedisCache<T>(key: string, data: T, ttlSeconds: number = 300): Promise<void> {
  await redis.setex(key, ttlSeconds, JSON.stringify(data));
}

// 3. 缓存层级策略（生产环境推荐）
class CacheStrategy {
  // 多级缓存：内存 -> Redis -> 数据库

  async get<T>(
    key: string,
    options: {
      fetchFromDb: () => Promise<T>;
      ttl: number;
      memoryTtl?: number;  // 内存缓存时间更短
    }
  ): Promise<T> {
    const { fetchFromDb, ttl, memoryTtl = 60 } = options;

    // 第一步：尝试从内存缓存获取（最快）
    const memoryData = getFromMemoryCache<T>(key);
    if (memoryData !== null) {
      console.log(`[Cache] 内存命中: ${key}`);
      return memoryData;
    }

    // 第二步：尝试从Redis获取（较快）
    const redisData = await getFromRedisCache<T>(key);
    if (redisData !== null) {
      console.log(`[Cache] Redis命中: ${key}`);
      // 回填到内存缓存
      setMemoryCache(key, redisData, memoryTtl);
      return redisData;
    }

    // 第三步：从数据库获取（最慢）
    console.log(`[Cache] 缓存未命中，从数据库获取: ${key}`);
    const dbData = await fetchFromDb();

    // 写入Redis和内存缓存
    await setRedisCache(key, dbData, ttl);
    setMemoryCache(key, dbData, memoryTtl);

    return dbData;
  }

  // 缓存失效（删除）
  async invalidate(key: string): Promise<void> {
    memoryCache.delete(key);
    await redis.del(key);
  }

  // 模式匹配删除（如删除某个用户的所有缓存）
  async invalidatePattern(pattern: string): Promise<void> {
    // 先删除Redis中匹配的所有key
    const keys = await redis.keys(pattern);
    if (keys.length > 0) {
      await redis.del(...keys);
    }

    // 清理内存缓存（简单遍历删除）
    for (const key of memoryCache.keys()) {
      if (key.includes(pattern.replace('*', ''))) {
        memoryCache.delete(key);
      }
    }
  }
}

// 使用示例
const cache = new CacheStrategy();

// 获取用户信息（缓存5分钟）
const user = await cache.get(
  `user:${userId}`,
  {
    fetchFromDb: () => db.getUser(userId),
    ttl: 300,        // Redis缓存5分钟
    memoryTtl: 60,   // 内存缓存1分钟
  }
);
```

```typescript
// 生产环境的WebSocket需要处理更多边界情况

class WebSocketClient {
  private ws: WebSocket | null = null;
  private reconnectAttempts = 0;
  private maxReconnectAttempts = 5;
  private reconnectDelay = 1000;
  private heartbeatInterval: NodeJS.Timeout | null = null;

  constructor(private url: string) {}

  connect() {
    // 生产环境使用WSS（WebSocket Secure）
    const wsUrl = this.url.startsWith('https')
      ? this.url.replace('https', 'wss')
      : this.url.startsWith('http')
        ? this.url.replace('http', 'ws')
        : this.url;

    this.ws = new WebSocket(wsUrl);

    // 连接成功
    this.ws.onopen = () => {
      console.log('[WebSocket] 连接成功');
      this.reconnectAttempts = 0;

      // 开始心跳
      this.startHeartbeat();

      // 发送认证信息
      this.sendAuth();
    };

    // 收到消息
    this.ws.onmessage = (event) => {
      try {
        const data = JSON.parse(event.data);
        this.handleMessage(data);
      } catch {
        console.error('[WebSocket] 消息解析失败:', event.data);
      }
    };

    // 连接关闭
    this.ws.onclose = (event) => {
      console.log(`[WebSocket] 连接关闭: ${event.code} ${event.reason}`);
      this.stopHeartbeat();

      // 尝试重连（生产环境必须）
      if (this.reconnectAttempts < this.maxReconnectAttempts) {
        this.reconnectAttempts++;
        const delay = this.reconnectDelay * Math.pow(2, this.reconnectAttempts - 1);
        console.log(`[WebSocket] ${delay}ms后尝试第${this.reconnectAttempts}次重连...`);
        setTimeout(() => this.connect(), delay);
      } else {
        console.error('[WebSocket] 重连次数用尽，请手动刷新页面');
        // 发送通知给用户
        showError('连接已断开，请刷新页面重新连接');
      }
    };

    // 连接错误
    this.ws.onerror = (error) => {
      console.error('[WebSocket] 连接错误:', error);
    };
  }

  private startHeartbeat() {
    // 每30秒发送一次心跳
    this.heartbeatInterval = setInterval(() => {
      if (this.ws?.readyState === WebSocket.OPEN) {
        this.ws.send(JSON.stringify({ type: 'ping' }));
      }
    }, 30000);
  }

  private stopHeartbeat() {
    if (this.heartbeatInterval) {
      clearInterval(this.heartbeatInterval);
      this.heartbeatInterval = null;
    }
  }

  private sendAuth() {
    const token = getAuthToken();
    if (token) {
      this.ws?.send(JSON.stringify({
        type: 'auth',
        token
      }));
    }
  }

  private handleMessage(data: any) {
    console.log('[WebSocket] 收到消息:', data);
    // 处理不同类型的消息
  }

  // 发送消息
  send(data: any) {
    if (this.ws?.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify(data));
    } else {
      console.warn('[WebSocket] 连接未就绪，消息未发送');
    }
  }

  // 关闭连接
  disconnect() {
    this.stopHeartbeat();
    this.ws?.close();
    this.ws = null;
  }
}
```

```typescript
        stack,  // 错误堆栈单独字段
      });
    } else {
      // 开发环境用彩色格式
      return `${timestamp} [${level.toUpperCase()}]: ${message}${stack ? '\n' + stack : ''}`;
    }
  })
);

// 传输器配置
const transports = [
  // 控制台输出
  new winston.transports.Console({
    format: winston.format.combine(
      winston.format.colorize(),  // 彩色输出
      format
    ),
  }),
];

// 生产环境添加文件输出
if (process.env.NODE_ENV === 'production') {
  // 错误日志单独文件
  transports.push(
    new DailyRotateFile({
      filename: 'logs/error-%DATE%.log',
      datePattern: 'YYYY-MM-DD',
      level: 'error',  // 只记录error级别
      maxSize: '100m',  // 单文件最大100MB
      maxFiles: '30d', // 保留30天
      format,
    })
  );

  // 所有日志文件
  transports.push(
    new DailyRotateFile({
      filename: 'logs/combined-%DATE%.log',
      datePattern: 'YYYY-MM-DD',
      maxSize: '200m',
      maxFiles: '14d',  // 保留14天
      format,
    })
  );

  // HTTP请求日志（单独文件，便于分析）
  transports.push(
    new DailyRotateFile({
      filename: 'logs/http-%DATE%.log',
      datePattern: 'YYYY-MM-DD',
      level: 'http',
      maxSize: '100m',
      maxFiles: '7d',  // HTTP日志保留7天
      format,
    })
  );
}

// 创建logger实例
const logger = winston.createLogger({
  level,
  levels,
  format,
  transports,
  // 关闭默认的异常处理（使用自定义）
  exitOnError: false,
});

// 扩展logger添加额外方法
export const loggerStream = {
  write: (message: string) => {
    // 用于将Express的morgan日志写入winston
    logger.http(message.trim());
  }
};

// 使用示例
export default logger;

// 使用方法

logger.warn('这是一个警告', { reason: 'token即将过期' });
logger.info('这是一条信息', { count: 5 });
logger.http('HTTP请求', { method: 'GET', path: '/api/users', status: 200 });
logger.debug('调试信息', { variable: 'value' });

// ============================================
// 日志脱敏处理
// ============================================

// 敏感信息必须脱敏后记录

function maskSensitiveData(obj: any, depth = 0): any {
  // 防止递归过深
  if (depth > 10) return '[MAX_DEPTH]';

  if (obj === null || obj === undefined) {
    return obj;
  }

  if (typeof obj !== 'object') {
    return obj;
  }

  if (Array.isArray(obj)) {
    return obj.map(item => maskSensitiveData(item, depth + 1));
  }

  const masked: any = {};
  for (const [key, value] of Object.entries(obj)) {
    // 需要脱敏的字段
    const sensitiveFields = [
      'password', 'token', 'secret', 'key', 'authorization',
      'credential', 'ssn', 'creditCard', 'phone', 'email', 'address', 'name'
    ];

    const isSensitive = sensitiveFields.some(field =>
      key.toLowerCase().includes(field.toLowerCase())
    );

    if (isSensitive && typeof value === 'string') {
      // 脱敏处理
      if (value.length <= 4) {
        masked[key] = '****';
      } else {
        masked[key] = value.slice(0, 2) + '****' + value.slice(-2);
      }
    } else if (typeof value === 'object') {
      masked[key] = maskSensitiveData(value, depth + 1);
    } else {
      masked[key] = value;
    }
  }

  return masked;
}

// 使用示例
logger.info('用户登录', {
  user: maskSensitiveData({
    id: '123',
    email: 'user@example.com',
    password: 'secret123',
    name: '张三'
  })
});
// 输出: { user: { id: '123', email: 'us****om', password: '****', name: '张****' } }
```
