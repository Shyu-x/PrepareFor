# 前端错误处理与容灾完全指南

## 概述

前端错误处理与容灾是保证 Web 应用稳定性和用户体验的关键技术体系。在现代 Web 应用中，用户期望获得流畅、无间断的服务体验，然而网络波动、服务器异常、代码 bug 等问题时刻威胁着应用的可用性。一个成熟的前端应用必须具备完善的错误处理机制和容灾能力，在问题发生时能够优雅地降级、快速地恢复，最大程度地保障用户的正常使用。

本指南将从错误分类开始，深入讲解全局错误捕获、错误上报、错误告警等核心技术，并重点介绍接口容错、UI 降级、数据容错、网络容灾等实战方案。我们将构建一个完整的前端容灾体系，覆盖从错误检测到恢复的全链路。最终通过电商案例和最佳实践，帮助读者掌握企业级前端容灾技术的全部要点。

## 一、错误分类

### 1.1 JS 运行时错误

JavaScript 运行时错误是最常见的错误类型，这类错误在 JavaScript 引擎执行代码时触发。运行时错误包括类型错误、引用错误、语法错误等，它们通常由代码逻辑问题或意外的数据状态引起。

**类型错误（TypeError）** 是最频繁出现的运行时错误。当代码尝试对非法的数据类型执行操作时会触发这种错误。例如，调用一个 null 对象的属性、读取 undefined 的属性、或者对非函数类型调用函数等场景都会导致 TypeError。来看一个典型的例子：

```typescript
// 类型错误示例
function processUserData(user) {
  // 如果 user 是 undefined 或 null，这里会抛出 TypeError
  return user.name.toUpperCase();
}

// 更安全的写法
function processUserDataSafe(user) {
  if (!user) {
    return 'Anonymous';
  }
  return (user.name || 'Anonymous').toUpperCase();
}
```

**引用错误（ReferenceError）** 发生在尝试引用一个不存在的变量时。这通常是因为变量名拼写错误、变量作用域问题，或者在变量声明之前就尝试使用它。

```typescript
// 引用错误示例
function calculateTotal() {
  // totalPrice 未定义，引用错误
  return totalPrice * quantity;
}

// 正确做法
function calculateTotal() {
  const totalPrice = getPrice();
  const quantity = getQuantity();
  return totalPrice * quantity;
}
```

**范围错误（RangeError）** 当数值超出合法范围时触发，常见于数组索引、字符串长度等操作。

```typescript
// 范围错误示例
const arr = [1, 2, 3];
// 尝试访问不存在的索引
console.log(arr[100]); // undefined，不会报错
// 但如果是递归调用导致栈溢出
function recursiveSum(n) {
  if (n <= 0) return 0;
  return n + recursiveSum(n - 1);
}
recursiveSum(1000000); // RangeError: Maximum call stack exceeded
```

运行时错误的捕获对于应用的稳定性至关重要。我们需要区分不同类型的错误，并采取适当的处理策略。对于一些预期内的运行时错误，应该通过代码逻辑提前预防；对于意外的错误，必须建立完善的捕获和上报机制。

### 1.2 资源加载错误

资源加载错误涵盖了页面中各类外部资源（如图片、脚本、样式表、字体、iframe 等）加载失败的情况。这类错误不会触发 JavaScript 的异常机制，但会影响页面的完整性和用户体验。资源加载错误需要通过特定的事件监听器来捕获。

**图片加载失败** 是最常见的资源加载错误。图片可能因为 URL 错误、服务器返回 404、网络中断等原因加载失败。

```typescript
// 图片加载错误处理
class ImageErrorHandler {
  private fallbackUrl: string;

  constructor(fallbackUrl: string = '/images/placeholder.png') {
    this.fallbackUrl = fallbackUrl;
  }

  // 创建带错误处理的图片元素
  createSafeImage(src: string, alt: string): HTMLImageElement {
    const img = document.createElement('img');
    img.alt = alt;
    img.src = src;

    // 图片加载失败时的处理
    img.onerror = () => {
      console.error(`图片加载失败: ${src}`);
      img.src = this.fallbackUrl;
      img.dataset.error = 'true';
    };

    // 图片加载成功
    img.onload = () => {
      img.dataset.loaded = 'true';
    };

    return img;
  }

  // React 组件中的使用
  static SafeImage = ({ src, alt, fallback }: SafeImageProps) => {
    const [error, setError] = useState(false);
    const [loading, setLoading] = useState(true);

    const handleError = () => {
      setError(true);
      setLoading(false);
    };

    const handleLoad = () => {
      setLoading(false);
    };

    if (error) {
      return fallback ? (
        typeof fallback === 'string' ? (
          <img src={fallback} alt={alt} />
        ) : (
          fallback
        )
      ) : (
        <div className="image-error-placeholder">{alt}</div>
      );
    }

    return (
      <img
        src={src}
        alt={alt}
        onError={handleError}
        onLoad={handleLoad}
        style={{ opacity: loading ? 0 : 1, transition: 'opacity 0.3s' }}
      />
    );
  };
}
```

**脚本加载错误** 可能导致依赖的 JavaScript 库无法加载，进而影响页面功能。使用 `defer` 和 `async` 属性加载的脚本，以及动态创建的脚本，都需要错误处理。

```typescript
// 脚本加载错误处理
class ScriptLoader {
  private loadedScripts: Map<string, boolean> = new Map();
  private errorCallbacks: Array<(url: string, error: Error) => void> = [];

  // 加载脚本
  async loadScript(url: string): Promise<void> {
    // 检查是否已加载
    if (this.loadedScripts.get(url)) {
      return Promise.resolve();
    }

    // 检查是否正在加载
    if (document.querySelector(`script[src="${url}"]`)) {
      return new Promise((resolve, reject) => {
        const interval = setInterval(() => {
          if (this.loadedScripts.get(url)) {
            clearInterval(interval);
            resolve();
          }
        }, 50);
        // 超时处理
        setTimeout(() => {
          clearInterval(interval);
          reject(new Error(`脚本加载超时: ${url}`));
        }, 10000);
      });
    }

    return new Promise((resolve, reject) => {
      const script = document.createElement('script');
      script.src = url;
      script.async = true; // 使用 async 避免阻塞

      script.onload = () => {
        this.loadedScripts.set(url, true);
        resolve();
      };

      script.onerror = (event) => {
        const error = new Error(`脚本加载失败: ${url}`);
        this.errorCallbacks.forEach(cb => cb(url, error));
        reject(error);
      };

      document.head.appendChild(script);
    });
  }

  // 批量加载脚本
  async loadScripts(urls: string[]): Promise<void[]> {
    return Promise.all(urls.map(url => this.loadScript(url)));
  }

  // 监听脚本错误
  onScriptError(callback: (url: string, error: Error) => void): void {
    this.errorCallbacks.push(callback);
  }
}
```

**样式表加载错误** 可能导致页面布局崩溃。由于 CSS 加载失败通常不会产生可见的错误，我们需要通过检测 link 元素的 stylesheet 属性来判断是否加载成功。

```typescript
// 样式表加载检测
function checkStylesheetLoaded(href: string): boolean {
  const links = document.styleSheets;
  for (let i = 0; i < links.length; i++) {
    const sheet = links[i] as CSSStyleSheet;
    if (sheet.href === href) {
      return true;
    }
  }
  return false;
}

// 动态加载 CSS 并处理错误
async function loadStylesheet(href: string): Promise<void> {
  return new Promise((resolve, reject) => {
    const link = document.createElement('link');
    link.rel = 'stylesheet';
    link.href = href;

    link.onload = () => resolve();
    link.onerror = () => {
      // CSS 加载失败，尝试使用内联样式作为降级方案
      console.warn(`CSS 加载失败: ${href}，使用内联样式降级`);
      const style = document.createElement('style');
      style.textContent = getInlineCriticalStyles();
      document.head.appendChild(style);
      resolve();
    };

    document.head.appendChild(link);
  });
}
```

### 1.3 Promise Rejection

Promise rejection 发生在 Promise 被拒绝但没有被 `.catch()` 或 `try...catch` 捕获时。自 ECMAScript 2015 引入 Promise 以来，未处理的 Promise 拒绝已经成为前端应用中最常见的错误类型之一。浏览器会对未处理的 Promise 拒绝发出警告，但不会自动捕获它们。

**同步捕获 Promise 拒绝** 需要使用 `async/await` 配合 `try...catch`，或者使用 Promise 的 `.catch()` 方法。

```typescript
// async/await 错误处理
async function fetchUserData(userId: string): Promise<UserData> {
  try {
    const response = await fetch(`/api/users/${userId}`);
    if (!response.ok) {
      throw new Error(`HTTP 错误: ${response.status}`);
    }
    const data = await response.json();
    return data;
  } catch (error) {
    // 分类处理不同类型的错误
    if (error instanceof TypeError) {
      // 网络错误
      console.error('网络连接失败:', error);
      throw new NetworkError('无法连接到服务器');
    } else if (error instanceof Error) {
      // HTTP 错误
      console.error('请求失败:', error);
      throw new ServerError(error.message);
    }
    throw error;
  }
}

// Promise chain 错误处理
function fetchUserDataPromise(userId: string): Promise<UserData> {
  return fetch(`/api/users/${userId}`)
    .then(response => {
      if (!response.ok) {
        throw new Error(`HTTP 错误: ${response.status}`);
      }
      return response.json();
    })
    .catch(error => {
      // 统一错误处理
      console.error('获取用户数据失败:', error);
      return defaultUserData; // 返回默认值而不是抛出错误
    });
}
```

**全局 Promise rejection 监听** 是捕获未处理拒绝的标准方式。通过监听 `unhandledrejection` 事件，我们可以在 Promise 被拒绝且没有处理器时执行特定的操作，比如上报错误日志。

```typescript
// 全局 Promise rejection 监听
if (typeof window !== 'undefined') {
  window.addEventListener('unhandledrejection', (event: PromiseRejectionEvent) => {
    const error = event.reason;

    // 区分不同类型的 rejection
    if (error instanceof Error) {
      // 标准 Error 对象
      console.error('未处理的 Promise 拒绝:', {
        message: error.message,
        stack: error.stack,
        name: error.name
      });

      // 上报到错误监控系统
      errorTracker.captureException(error, {
        type: 'unhandledrejection',
        timestamp: Date.now()
      });
    } else {
      // 非 Error 对象的 rejection
      console.error('未处理的 Promise 拒绝（非 Error 对象）:', error);
      errorTracker.captureMessage(
        `Unhandled Promise Rejection: ${String(error)}`,
        'warning'
      );
    }

    // 阻止默认行为（浏览器会打印警告）
    // event.preventDefault(); // 取消这一行以保持浏览器默认警告
  });

  // Promise rejection 被处理时的监听（可选）
  window.addEventListener('rejectionhandled', (event: PromiseRejectionEvent) => {
    console.log('Promise 拒绝已被处理:', event.reason);
  });
}
```

**Promise.all 的错误处理** 需要特别注意，因为任何一个 Promise 失败都会导致整个 `Promise.all` 失败。

```typescript
// Promise.all 错误处理策略

// 策略一：快速失败（默认行为）
async function quickFailStrategy() {
  const results = await Promise.all([
    fetchUser(1),
    fetchUser(2),
    fetchUser(3)
  ]);
  return results;
}

// 策略二：容忍部分失败
async function tolerantStrategy() {
  const promises = [
    fetchUser(1).then(u => ({ success: true, data: u })).catch(e => ({ success: false, error: e })),
    fetchUser(2).then(u => ({ success: true, data: u })).catch(e => ({ success: false, error: e })),
    fetchUser(3).then(u => ({ success: true, data: u })).catch(e => ({ success: false, error: e }))
  ];

  const results = await Promise.all(promises);
  const successful = results.filter(r => r.success);
  const failed = results.filter(r => !r.success);

  if (failed.length > 0) {
    console.warn(`${failed.length} 个请求失败`);
  }

  return successful.map(r => r.data);
}

// 策略三：使用 Promise.allSettled（ES2020+）
async function settledStrategy() {
  const results = await Promise.allSettled([
    fetchUser(1),
    fetchUser(2),
    fetchUser(3)
  ]);

  results.forEach((result, index) => {
    if (result.status === 'fulfilled') {
      console.log(`用户 ${index + 1} 获取成功:`, result.value);
    } else {
      console.error(`用户 ${index + 1} 获取失败:`, result.reason);
    }
  });

  // 提取成功的结果
  return results
    .filter(r => r.status === 'fulfilled')
    .map(r => (r as PromiseFulfilledResult<User>).value);
}
```

### 1.4 业务逻辑错误

业务逻辑错误是指应用代码在执行预期业务逻辑时产生的错误，这类错误不一定是代码本身的 bug，而可能是业务规则验证失败、数据状态不符合预期、外部系统返回异常等原因。业务逻辑错误通常需要根据具体的业务场景来定义和处理。

**数据验证错误** 是最常见的业务逻辑错误类型。用户在表单中输入了不符合要求的数据，或者后端返回的数据格式与前端预期不符，都会导致验证错误。

```typescript
// 数据验证错误处理
interface ValidationError {
  field: string;
  message: string;
  code: string;
}

class ValidationError extends Error {
  constructor(public errors: ValidationError[]) {
    super('数据验证失败');
    this.name = 'ValidationError';
  }
}

// 验证用户注册数据
interface RegisterForm {
  username: string;
  email: string;
  password: string;
  confirmPassword: string;
}

function validateRegistration(form: RegisterForm): void {
  const errors: ValidationError[] = [];

  // 用户名验证
  if (!form.username || form.username.length < 3) {
    errors.push({
      field: 'username',
      message: '用户名至少需要3个字符',
      code: 'USERNAME_TOO_SHORT'
    });
  }
  if (form.username && form.username.length > 20) {
    errors.push({
      field: 'username',
      message: '用户名不能超过20个字符',
      code: 'USERNAME_TOO_LONG'
    });
  }
  if (!/^[a-zA-Z0-9_]+$/.test(form.username)) {
    errors.push({
      field: 'username',
      message: '用户名只能包含字母、数字和下划线',
      code: 'USERNAME_INVALID_CHARS'
    });
  }

  // 邮箱验证
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!emailRegex.test(form.email)) {
    errors.push({
      field: 'email',
      message: '请输入有效的邮箱地址',
      code: 'EMAIL_INVALID'
    });
  }

  // 密码验证
  if (form.password.length < 8) {
    errors.push({
      field: 'password',
      message: '密码至少需要8个字符',
      code: 'PASSWORD_TOO_SHORT'
    });
  }
  if (!/[A-Z]/.test(form.password)) {
    errors.push({
      field: 'password',
      message: '密码必须包含至少一个大写字母',
      code: 'PASSWORD_NO_UPPERCASE'
    });
  }
  if (!/[a-z]/.test(form.password)) {
    errors.push({
      field: 'password',
      message: '密码必须包含至少一个小写字母',
      code: 'PASSWORD_NO_LOWERCASE'
    });
  }
  if (!/[0-9]/.test(form.password)) {
    errors.push({
      field: 'password',
      message: '密码必须包含至少一个数字',
      code: 'PASSWORD_NO_DIGIT'
    });
  }

  // 确认密码验证
  if (form.password !== form.confirmPassword) {
    errors.push({
      field: 'confirmPassword',
      message: '两次输入的密码不一致',
      code: 'PASSWORD_MISMATCH'
    });
  }

  if (errors.length > 0) {
    throw new ValidationError(errors);
  }
}
```

**业务状态错误** 是指在业务流程执行过程中，由于业务规则限制而导致的错误。例如，用户尝试执行一个不符合当前业务状态的操作，或者业务数据不满足执行条件。

```typescript
// 业务状态错误处理
enum BusinessErrorCode {
  ORDER_NOT_FOUND = 'ORDER_NOT_FOUND',
  ORDER_ALREADY_PAID = 'ORDER_ALREADY_PAID',
  ORDER_CANCELLED = 'ORDER_CANCELLED',
  INSUFFICIENT_STOCK = 'INSUFFICIENT_STOCK',
  INSUFFICIENT_BALANCE = 'INSUFFICIENT_BALANCE',
  PERMISSION_DENIED = 'PERMISSION_DENIED'
}

class BusinessError extends Error {
  constructor(
    public code: BusinessErrorCode,
    message: string,
    public details?: Record<string, any>
  ) {
    super(message);
    this.name = 'BusinessError';
  }
}

// 订单支付业务逻辑
class OrderPaymentService {
  async payOrder(orderId: string, paymentMethod: string): Promise<PaymentResult> {
    // 获取订单
    const order = await this.orderRepository.findById(orderId);

    if (!order) {
      throw new BusinessError(
        BusinessErrorCode.ORDER_NOT_FOUND,
        '订单不存在'
      );
    }

    // 检查订单状态
    if (order.status === 'paid') {
      throw new BusinessError(
        BusinessErrorCode.ORDER_ALREADY_PAID,
        '订单已支付，请勿重复操作',
        { paidAt: order.paidAt }
      );
    }

    if (order.status === 'cancelled') {
      throw new BusinessError(
        BusinessErrorCode.ORDER_CANCELLED,
        '订单已取消，无法支付'
      );
    }

    // 检查商品库存
    for (const item of order.items) {
      const stock = await this.stockService.getStock(item.productId);
      if (stock < item.quantity) {
        throw new BusinessError(
          BusinessErrorCode.INSUFFICIENT_STOCK,
          `商品 "${item.productName}" 库存不足`,
          { productId: item.productId, available: stock, required: item.quantity }
        );
      }
    }

    // 检查用户余额
    const userBalance = await this.userService.getBalance(order.userId);
    if (userBalance < order.totalAmount) {
      throw new BusinessError(
        BusinessErrorCode.INSUFFICIENT_BALANCE,
        '余额不足，请充值后再试',
        { balance: userBalance, required: order.totalAmount }
      );
    }

    // 执行支付逻辑
    return this.executePayment(order, paymentMethod);
  }
}
```

**第三方服务错误** 是指调用外部 API 或服务时产生的错误。这类错误需要特别处理，因为它们通常不受我们控制，而且可能影响用户的实际操作。

```typescript
// 第三方服务错误处理
class ThirdPartyServiceError extends Error {
  constructor(
    public service: string,
    public originalError: Error
  ) {
    super(`第三方服务 ${service} 调用失败: ${originalError.message}`);
    this.name = 'ThirdPartyServiceError';
  }
}

// 第三方服务调用封装
class ThirdPartyService {
  private adapters: Map<string, ServiceAdapter> = new Map();

  // 注册服务适配器
  registerAdapter(service: string, adapter: ServiceAdapter): void {
    this.adapters.set(service, adapter);
  }

  // 调用第三方服务
  async call<T>(
    service: string,
    method: string,
    params: Record<string, any>,
    options: CallOptions = {}
  ): Promise<T> {
    const adapter = this.adapters.get(service);
    if (!adapter) {
      throw new Error(`未注册的服务: ${service}`);
    }

    const timeout = options.timeout || 30000;
    const retries = options.retries || 3;
    const retryDelay = options.retryDelay || 1000;

    let lastError: Error;

    for (let attempt = 0; attempt <= retries; attempt++) {
      try {
        return await this.callWithTimeout(
          adapter,
          method,
          params,
          timeout
        );
      } catch (error) {
        lastError = error as Error;

        // 检查是否是可重试的错误
        if (this.isRetryableError(error) && attempt < retries) {
          console.warn(`第 ${attempt + 1} 次尝试失败，${retryDelay}ms 后重试...`);
          await this.delay(retryDelay);
          continue;
        }

        // 不可重试的错误，直接抛出
        break;
      }
    }

    throw new ThirdPartyServiceError(service, lastError!);
  }

  private async callWithTimeout<T>(
    adapter: ServiceAdapter,
    method: string,
    params: any,
    timeout: number
  ): Promise<T> {
    return Promise.race([
      adapter.call(method, params),
      new Promise<never>((_, reject) =>
        setTimeout(() => reject(new Error('服务调用超时')), timeout)
      )
    ]) as Promise<T>;
  }

  private isRetryableError(error: Error): boolean {
    // 网络错误、超时错误可重试
    if (error.message.includes('timeout') || error.message.includes('network')) {
      return true;
    }
    // 5xx 服务器错误可重试
    if (error.message.includes('500') || error.message.includes('502') || error.message.includes('503')) {
      return true;
    }
    return false;
  }

  private delay(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
}
```

### 1.5 实战：错误分类处理系统

下面我们构建一个完整的错误分类处理系统，统一管理和处理各类前端错误。

```typescript
// 错误分类枚举
enum ErrorCategory {
  RUNTIME = 'runtime',           // JS 运行时错误
  RESOURCE = 'resource',         // 资源加载错误
  PROMISE = 'promise',           // Promise 拒绝
  BUSINESS = 'business',          // 业务逻辑错误
  NETWORK = 'network',            // 网络错误
  THIRD_PARTY = 'third_party',    // 第三方服务错误
  UNKNOWN = 'unknown'             // 未知错误
}

// 错误严重程度
enum ErrorSeverity {
  CRITICAL = 'critical',  // 致命错误，应用无法继续运行
  ERROR = 'error',        // 一般错误，某些功能受影响
  WARNING = 'warning',    // 警告，不影响核心功能
  INFO = 'info'           // 信息，仅用于日志记录
}

// 错误信息结构
interface ErrorInfo {
  id: string;                    // 错误唯一标识
  category: ErrorCategory;       // 错误分类
  severity: ErrorSeverity;       // 严重程度
  message: string;               // 错误消息
  stack?: string;                // 错误堆栈
  timestamp: number;             // 发生时间
  context: ErrorContext;         // 错误上下文
  metadata: Record<string, any>; // 附加数据
}

interface ErrorContext {
  url: string;                   // 页面 URL
  userAgent: string;             // 用户代理
  viewport: string;              // 视口大小
  userId?: string;               // 用户 ID（如果已登录）
  sessionId: string;             // 会话 ID
  action?: string;               // 导致错误的用户操作
}

// 错误分类器
class ErrorClassifier {
  // 根据错误类型分类
  static classify(error: unknown): ErrorCategory {
    if (error instanceof TypeError || error instanceof ReferenceError ||
        error instanceof RangeError || error instanceof SyntaxError ||
        error instanceof URIError || error instanceof EvalError) {
      return ErrorCategory.RUNTIME;
    }

    if (error instanceof PromiseRejectionEvent) {
      return ErrorCategory.PROMISE;
    }

    if (error instanceof ValidationError || error instanceof BusinessError) {
      return ErrorCategory.BUSINESS;
    }

    if (error instanceof ThirdPartyServiceError || error instanceof NetworkError) {
      return ErrorCategory.NETWORK;
    }

    if (this.isResourceError(error)) {
      return ErrorCategory.RESOURCE;
    }

    return ErrorCategory.UNKNOWN;
  }

  // 判断是否为资源加载错误
  private static isResourceError(error: unknown): boolean {
    if (error instanceof Error && error.message) {
      // 资源错误通常有特定的 message 模式
      const resourcePatterns = [
        /Failed to fetch/i,
        /NetworkError/i,
        /net::ERR/i,
        /加载失败/i,
        /Loading failed/i
      ];
      return resourcePatterns.some(pattern => pattern.test(error.message));
    }
    return false;
  }

  // 根据错误类型和上下文确定严重程度
  static determineSeverity(
    category: ErrorCategory,
    error: unknown,
    context?: Partial<ErrorContext>
  ): ErrorSeverity {
    // 致命错误：运行时错误（非预期）
    if (category === ErrorCategory.RUNTIME) {
      const runtimeError = error as Error;
      // 检查是否是栈溢出等致命错误
      if (runtimeError.message?.includes('Maximum call stack')) {
        return ErrorSeverity.CRITICAL;
      }
      return ErrorSeverity.ERROR;
    }

    // 网络错误通常是警告级别（可重试）
    if (category === ErrorCategory.NETWORK) {
      return ErrorSeverity.WARNING;
    }

    // 业务错误根据具体类型判断
    if (category === ErrorCategory.BUSINESS) {
      const businessError = error as BusinessError;
      switch (businessError.code) {
        case BusinessErrorCode.PERMISSION_DENIED:
        case BusinessErrorCode.ORDER_ALREADY_PAID:
          return ErrorSeverity.WARNING;
        default:
          return ErrorSeverity.INFO;
      }
    }

    return ErrorSeverity.ERROR;
  }
}

// 错误处理器
class ErrorHandler {
  private errorQueue: ErrorInfo[] = [];
  private maxQueueSize = 100;
  private flushInterval = 5000; // 5秒上报一次
  private sessionId: string;

  constructor() {
    this.sessionId = this.generateSessionId();
    this.setupGlobalHandlers();
    this.startPeriodicFlush();
  }

  // 生成会话 ID
  private generateSessionId(): string {
    return `${Date.now()}-${Math.random().toString(36).substr(2, 9)}`;
  }

  // 处理错误
  handle(error: unknown, context: Partial<ErrorContext> = {}): void {
    const category = ErrorClassifier.classify(error);
    const severity = ErrorClassifier.determineSeverity(category, error, context);

    const errorInfo: ErrorInfo = {
      id: this.generateErrorId(),
      category,
      severity,
      message: this.extractMessage(error),
      stack: this.extractStack(error),
      timestamp: Date.now(),
      context: {
        url: typeof window !== 'undefined' ? window.location.href : '',
        userAgent: typeof navigator !== 'undefined' ? navigator.userAgent : '',
        viewport: typeof window !== 'undefined'
          ? `${window.innerWidth}x${window.innerHeight}`
          : '',
        sessionId: this.sessionId,
        ...context
      },
      metadata: this.extractMetadata(error)
    };

    // 添加到队列
    this.errorQueue.push(errorInfo);

    // 控制队列大小
    if (this.errorQueue.length > this.maxQueueSize) {
      this.errorQueue.shift();
    }

    // 根据严重程度决定处理方式
    if (severity === ErrorSeverity.CRITICAL) {
      this.handleCriticalError(errorInfo);
    }
  }

  // 提取错误消息
  private extractMessage(error: unknown): string {
    if (error instanceof Error) {
      return error.message;
    }
    return String(error);
  }

  // 提取错误堆栈
  private extractStack(error: unknown): string | undefined {
    if (error instanceof Error && error.stack) {
      return error.stack;
    }
    return undefined;
  }

  // 提取元数据
  private extractMetadata(error: unknown): Record<string, any> {
    if (error instanceof Error) {
      return {
        name: error.name,
        ...(error as any).metadata
      };
    }
    return {};
  }

  // 生成错误 ID
  private generateErrorId(): string {
    return `err_${Date.now()}_${Math.random().toString(36).substr(2, 6)}`;
  }

  // 处理致命错误
  private handleCriticalError(errorInfo: ErrorInfo): void {
    console.error('致命错误发生:', errorInfo);

    // 可以在这里添加更多处理逻辑
    // 比如：显示错误提示、尝试保存用户数据、跳转到错误页面等
  }

  // 设置全局错误处理器
  private setupGlobalHandlers(): void {
    if (typeof window === 'undefined') return;

    // window.onerror
    window.onerror = (message, source, lineno, colno, error) => {
      this.handle(error || new Error(String(message)), {
        action: 'window.onerror'
      });
      return false; // 允许浏览器默认处理
    };

    // unhandledrejection
    window.addEventListener('unhandledrejection', (event) => {
      this.handle(event.reason, {
        action: 'unhandledrejection'
      });
    });

    // 资源加载错误
    window.addEventListener('error', (event) => {
      if (event.target !== window) {
        // 资源加载错误
        const target = event.target as HTMLElement;
        const resourceUrl = (target as any).src || (target as any).href;
        console.error(`资源加载失败: ${resourceUrl}`);
      }
    }, true);
  }

  // 启动定期上报
  private startPeriodicFlush(): void {
    setInterval(() => {
      if (this.errorQueue.length > 0) {
        this.flush();
      }
    }, this.flushInterval);
  }

  // 上报错误
  private async flush(): Promise<void> {
    if (this.errorQueue.length === 0) return;

    const errors = [...this.errorQueue];
    this.errorQueue = [];

    try {
      // 发送到错误收集服务
      await fetch('/api/errors/batch', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ errors }),
        keepalive: true // 确保页面关闭时也能发送
      });
    } catch (error) {
      // 如果发送失败，将错误放回队列
      console.error('错误上报失败:', error);
      this.errorQueue.unshift(...errors);
    }
  }
}

// 创建全局错误处理器实例
export const errorHandler = new ErrorHandler();
```

## 二、全局错误捕获

### 2.1 window.onerror

`window.onerror` 是 JavaScript 中最原始的错误处理机制，它能捕获所有未被 try-catch 捕获的 JavaScript 运行时错误。当页面中发生 JavaScript 错误时，浏览器会调用这个全局错误处理函数。

`window.onerror` 的签名包含五个参数：错误消息、发生错误的 URL、行号、列号和错误对象。通过这些参数，我们可以获取关于错误的详细信息。返回值也很重要：返回 `true` 表示错误已被处理，浏览器不会显示默认的错误控制台信息；返回 `false` 则允许浏览器显示默认错误信息。

```typescript
// window.onerror 基本用法
window.onerror = function(
  message: string | Event,
  source?: string,
  lineno?: number,
  colno?: number,
  error?: Error
): boolean {
  // 收集错误信息
  const errorInfo = {
    message: typeof message === 'string' ? message : String(message),
    source,
    lineno,
    colno,
    stack: error?.stack,
    timestamp: Date.now()
  };

  // 打印错误详情
  console.error('全局错误捕获:', errorInfo);

  // 上报错误
  errorTracker?.capture?.(errorInfo);

  // 返回 true 阻止浏览器默认错误处理
  // 返回 false 允许浏览器显示默认错误信息
  return true;
};

// 使用箭头函数
window.onerror = (
  message,
  source,
  lineno,
  colno,
  error
) => {
  const errorInfo = {
    message,
    source,
    lineno,
    colno,
    errorType: error?.constructor?.name,
    stack: error?.stack
  };

  // 错误上报
  reportError(errorInfo);

  // 返回 true 阻止默认行为
  return true;
};
```

在实际项目中，`window.onerror` 存在一些局限性。它无法捕获以下类型的错误：资源加载错误（如图片、脚本、样式表加载失败）、Promise rejection（需要监听 `unhandledrejection` 事件）、以及跨域脚本错误（通常只返回 "Script error." 消息）。

为了解决跨域脚本错误的问题，我们需要在所有外部脚本的 `<script>` 标签上添加 `crossorigin="anonymous"` 属性，并在对应的服务器上设置正确的 CORS 头部。这样，当跨域脚本发生错误时，浏览器才能将详细的错误信息传递给 `window.onerror`。

```typescript
// 解决跨域脚本错误
// 1. HTML 中的 script 标签需要添加 crossorigin 属性
// <script src="https://cdn.example.com/external-script.js" crossorigin="anonymous"></script>

// 2. 服务器端需要设置 CORS 头部
// Access-Control-Allow-Origin: *

// 3. 错误处理时判断是否是跨域错误
window.onerror = (message, source, lineno, colno, error) => {
  // 跨域错误通常 message 是 "Script error."
  if (message === 'Script error.') {
    console.warn('跨域脚本错误，无法获取详细信息');
    // 可以通过其他方式监控（如 Performance API）
    return true;
  }

  // 本域脚本错误，可以获取完整信息
  console.error('脚本错误:', {
    message,
    source,
    lineno,
    colno,
    stack: error?.stack
  });

  return true;
};
```

### 2.2 window.addEventListener('error')

除了 `window.onerror`，我们还可以使用 `window.addEventListener('error')` 来监听错误事件。这两种方式的主要区别在于：`window.onerror` 是早期的错误处理方式，而 `addEventListener` 则提供了更灵活的事件处理机制。

`addEventListener` 的一个重要优势是可以在捕获阶段监听错误，这对于捕获资源加载错误特别有用。通过将第三个参数设置为 `true`，我们可以在事件冒泡之前捕获事件，从而更早地处理错误。

```typescript
// 监听 JavaScript 错误（冒泡阶段）
window.addEventListener('error', (event: ErrorEvent) => {
  const error = event.error;
  const message = event.message;
  const filename = event.filename;
  const lineno = event.lineno;
  const colno = event.colno;

  console.error('JavaScript 错误:', {
    message,
    filename,
    lineno,
    colno,
    error,
    stack: error?.stack
  });

  // 错误上报
  reportError({
    type: 'javascript',
    message,
    filename,
    lineno,
    colno,
    stack: error?.stack,
    timestamp: Date.now()
  });
});

// 监听资源加载错误（捕获阶段）
window.addEventListener('error', (event: ErrorEvent) => {
  const target = event.target;

  // 只处理资源加载错误，不处理 window 错误
  if (target !== window && target instanceof HTMLElement) {
    const resourceUrl = (target as any).src || (target as any).href;
    const resourceType = this.getResourceType(target);

    console.error('资源加载失败:', {
      url: resourceUrl,
      type: resourceType,
      element: target.tagName
    });

    // 资源错误上报
    reportError({
      type: 'resource',
      url: resourceUrl,
      resourceType,
      element: target.tagName,
      timestamp: Date.now()
    });

    // 阻止事件冒泡
    event.stopPropagation();
  }
}, true); // 使用捕获阶段

// 判断资源类型
private getResourceType(element: HTMLElement): string {
  if (element instanceof HTMLImageElement) return 'image';
  if (element instanceof HTMLScriptElement) return 'script';
  if (element instanceof HTMLLinkElement) return 'stylesheet';
  if (element instanceof HTMLIFrameElement) return 'iframe';
  return 'unknown';
}
```

`addEventListener` 还支持添加多个监听器，这使得我们可以将不同类型的错误处理逻辑解耦到不同的模块中。每个监听器都可以独立工作，互不干扰。

```typescript
// 模块化错误监听器
class ErrorListenerManager {
  private listeners: Map<string, EventListener> = new Map();

  // JavaScript 错误监听
  registerJSErrors(handler: (error: ErrorEvent) => void): void {
    const listener = (event: ErrorEvent) => {
      handler(event);
    };
    window.addEventListener('error', listener);
    this.listeners.set('js-error', listener);
  }

  // 资源加载错误监听
  registerResourceErrors(handler: (error: ErrorEvent) => void): void {
    const listener = (event: ErrorEvent) => {
      if (event.target !== window) {
        handler(event);
      }
    };
    window.addEventListener('error', listener, true);
    this.listeners.set('resource-error', listener);
  }

  // 未处理 Promise rejection 监听
  registerUnhandledRejections(handler: (event: PromiseRejectionEvent) => void): void {
    window.addEventListener('unhandledrejection', handler);
    this.listeners.set('unhandled-rejection', handler);
  }

  // 卸载所有监听器
  unregisterAll(): void {
    this.listeners.forEach((listener, key) => {
      if (key.includes('rejection')) {
        window.removeEventListener('unhandledrejection', listener);
      } else {
        window.removeEventListener('error', listener);
      }
    });
    this.listeners.clear();
  }
}
```

### 2.3 unhandledrejection

`unhandledrejection` 事件在 Promise 被拒绝且没有 `.catch()` 处理器时触发。这个事件对于监控未处理的异步错误非常重要，因为这类错误如果不被捕获，可能会导致静默失败，用户和开发者都无法察觉问题。

```typescript
// unhandledrejection 事件监听
window.addEventListener('unhandledrejection', (event: PromiseRejectionEvent) => {
  const reason = event.reason;

  // reason 可能是 Error 对象，也可能是任意值
  let errorInfo: Record<string, any> = {
    timestamp: Date.now(),
    type: 'unhandledrejection'
  };

  if (reason instanceof Error) {
    errorInfo = {
      ...errorInfo,
      message: reason.message,
      name: reason.name,
      stack: reason.stack,
      errorType: reason.constructor.name
    };
  } else {
    // 非 Error 对象的 rejection
    errorInfo = {
      ...errorInfo,
      message: String(reason),
      reasonType: typeof reason
    };
  }

  console.error('未处理的 Promise 拒绝:', errorInfo);

  // 上报错误
  reportError(errorInfo);

  // 阻止浏览器默认的警告行为（可选）
  // event.preventDefault();
});

// 区分已处理和未处理的 rejection
let pendingRejections = new Map<Promise<any>, { reason: any; time: number }>();

// 追踪所有 Promise rejections
window.addEventListener('unhandledrejection', (event) => {
  pendingRejections.set(event.promise, {
    reason: event.reason,
    time: Date.now()
  });
});

window.addEventListener('rejectionhandled', (event) => {
  // 当 rejection 被处理时，从 Map 中移除
  pendingRejections.delete(event.promise);

  console.log('Promise 拒绝已被处理');
});

// 定期检查未处理的 rejections
setInterval(() => {
  const now = Date.now();
  const timeout = 5000; // 5 秒视为未处理

  pendingRejections.forEach((value, promise) => {
    if (now - value.time > timeout) {
      console.warn('长时间未处理的 Promise 拒绝:', value.reason);
    }
  });
}, 10000);
```

在实际开发中，为了避免 `unhandledrejection` 警告，我们应该在创建 Promise 时就确保有正确的错误处理。对于 `async/await` 函数，始终使用 `try-catch` 包裹可能抛出错误的代码。

```typescript
// 最佳实践：始终处理 async/await 错误
async function fetchData(url: string): Promise<Data> {
  try {
    const response = await fetch(url);
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }
    return await response.json();
  } catch (error) {
    // 分类处理错误
    if (error instanceof TypeError) {
      // 网络错误
      throw new NetworkError('无法连接到服务器', error);
    }
    if (error instanceof Error) {
      // 服务器错误
      throw new ServerError(error.message, error);
    }
    // 未知错误
    throw new UnknownError('未知错误', error);
  }
}

// 如果确实不需要错误处理，可以使用这个模式
// 创建不会有 rejection 的 Promise
const noRejectionPromise = new Promise<Data>((resolve, reject) => {
  fetch(url)
    .then(resolve)
    .catch(reject); // reject 会正常触发，而不是 unhandled rejection
});

// 或者使用 defer pattern
function deferredPromise<T>(): {
  promise: Promise<T>;
  resolve: (value: T) => void;
  reject: (error: any) => void;
} {
  let resolve!: (value: T) => void;
  let reject!: (error: any) => void;

  const promise = new Promise<T>((res, rej) => {
    resolve = res;
    reject = rej;
  });

  return { promise, resolve, reject };
}
```

### 2.4 Vue/React 错误边界

错误边界是 React 16 引入的概念，用于捕获子组件树中的 JavaScript 错误，记录错误日志，并显示降级的 UI 而不是崩溃的组件树。错误边界本身是普通的 React 组件，但它们使用 `componentDidCatch` 或 `getDerivedStateFromError` 来处理错误。

错误边界的工作原理是：当子组件在渲染过程中发生错误时，React 会沿着组件树向上查找最近的错误边界组件，并将错误传递给它的 `componentDidCatch` 或 `getDerivedStateFromError` 生命周期方法。

```typescript
// React 错误边界组件
import { Component, ReactNode } from 'react';

interface ErrorBoundaryState {
  hasError: boolean;
  error?: Error;
  errorInfo?: ErrorInfo;
}

interface ErrorBoundaryProps {
  children: ReactNode;
  fallback?: ReactNode | ((error: Error, reset: () => void) => ReactNode);
  onError?: (error: Error, errorInfo: ErrorInfo) => void;
  level?: 'page' | 'section' | 'component';
}

class ErrorBoundary extends Component<ErrorBoundaryProps, ErrorBoundaryState> {
  constructor(props: ErrorBoundaryProps) {
    super(props);
    this.state = { hasError: false };
  }

  // 静态方法，用于在渲染前更新状态
  static getDerivedStateFromError(error: Error): Partial<ErrorBoundaryState> {
    // 返回新的 state，触发重新渲染
    return { hasError: true, error };
  }

  // 捕获错误并记录日志
  componentDidCatch(error: Error, errorInfo: ErrorInfo): void {
    // 记录错误日志
    console.error('错误边界捕获错误:', {
      error: error.message,
      stack: error.stack,
      componentStack: errorInfo.componentStack
    });

    // 将错误信息保存到 state
    this.setState({
      error,
      errorInfo
    });

    // 调用错误回调
    if (this.props.onError) {
      this.props.onError(error, {
        componentStack: errorInfo.componentStack || ''
      });
    }
  }

  // 重置错误状态
  resetError = (): void => {
    this.setState({ hasError: false, error: undefined, errorInfo: undefined });
  };

  render(): ReactNode {
    const { hasError, error, errorInfo } = this.state;
    const { children, fallback, level = 'section' } = this.props;

    if (hasError) {
      // 如果有自定义 fallback
      if (fallback) {
        if (typeof fallback === 'function') {
          return fallback(error!, this.resetError);
        }
        return fallback;
      }

      // 默认 fallback UI
      return (
        <div
          data-error-boundary={level}
          data-error-message={error?.message}
          style={{
```

### 2.5 实战：捕获实现

下面我们构建一个完整的全局错误捕获系统，整合所有错误处理机制。

```typescript
// 全局错误捕获系统
class GlobalErrorCapture {
  private static instance: GlobalErrorCapture;
  private isInitialized = false;
  private errorHandlers: Array<(error: ErrorInfo) => void> = [];
  private resourceErrorHandlers: Array<(info: ResourceErrorInfo) => void> = [];

  private constructor() {}

  // 单例模式
  static getInstance(): GlobalErrorCapture {
    if (!GlobalErrorCapture.instance) {
      GlobalErrorCapture.instance = new GlobalErrorCapture();
    }
    return GlobalErrorCapture.instance;
  }

  // 初始化所有错误监听
  initialize(): void {
    if (this.isInitialized || typeof window === 'undefined') return;

    this.isInitialized = true;

    // 1. JavaScript 运行时错误
    window.addEventListener('error', this.handleJSErrors.bind(this));

    // 2. 资源加载错误（捕获阶段）
    window.addEventListener('error', this.handleResourceErrors.bind(this), true);

    // 3. Promise rejection
    window.addEventListener('unhandledrejection', this.handlePromiseRejections.bind(this));

    // 4. Vue 错误处理（如果使用 Vue）
    this.setupVueErrorHandler?.();

    // 5. React 错误边界（需要用户自己添加组件）

    // 6. 页面卸载时确保错误上报
    window.addEventListener('beforeunload', this.flushErrors.bind(this));

    console.log('全局错误捕获系统已初始化');
  }

  // 处理 JavaScript 错误
  private handleJSErrors(event: ErrorEvent): void {
    // 跳过资源加载错误
    if (event.target !== window) return;

    const error = event.error;
    const errorInfo: ErrorInfo = {
      id: this.generateErrorId(),
      category: ErrorCategory.RUNTIME,
      severity: this.determineSeverity(error),
      message: event.message || String(error),
      stack: error?.stack,
      timestamp: Date.now(),
      context: this.getErrorContext(),
      metadata: {
        filename: event.filename,
        lineno: event.lineno,
        colno: event.colno,
        errorType: error?.constructor?.name
      }
    };

    this.notifyHandlers(errorInfo);
  }

  // 处理资源加载错误
  private handleResourceErrors(event: ErrorEvent): void {
    const target = event.target;

    // 只处理 HTML 元素，不处理 window 的 error 事件
    if (target instanceof HTMLElement) {
      const element = target as HTMLElement;
      const resourceInfo: ResourceErrorInfo = {
        id: this.generateErrorId(),
        url: (element as any).src || (element as any).href || '',
        resourceType: this.getResourceType(element),
        element: element.tagName,
        timestamp: Date.now(),
        context: this.getErrorContext()
      };

      this.notifyResourceHandlers(resourceInfo);

      // 触发降级处理
      this.handleResourceFallback(element, resourceInfo);
    }
  }

  // 处理 Promise rejection
  private handlePromiseRejections(event: PromiseRejectionEvent): void {
    const reason = event.reason;

    let errorInfo: ErrorInfo;

    if (reason instanceof Error) {
      errorInfo = {
        id: this.generateErrorId(),
        category: ErrorCategory.PROMISE,
        severity: ErrorSeverity.ERROR,
        message: reason.message,
        stack: reason.stack,
        timestamp: Date.now(),
        context: this.getErrorContext(),
        metadata: {
          errorType: reason.constructor.name,
          name: reason.name
        }
      };
    } else {
      errorInfo = {
        id: this.generateErrorId(),
        category: ErrorCategory.PROMISE,
        severity: ErrorSeverity.WARNING,
        message: String(reason),
        timestamp: Date.now(),
        context: this.getErrorContext(),
        metadata: {
          reasonType: typeof reason
        }
      };
    }

    this.notifyHandlers(errorInfo);

    // 标记为已处理的 rejection
    event.promise.catch(() => {}); // 防止 Unhandled rejection 警告
  }

  // 获取资源类型
  private getResourceType(element: HTMLElement): string {
    if (element instanceof HTMLImageElement) return 'image';
    if (element instanceof HTMLScriptElement) return 'script';
    if (element instanceof HTMLLinkElement) return 'stylesheet';
    if (element instanceof HTMLIFrameElement) return 'iframe';
    if (element instanceof HTMLVideoElement) return 'video';
    if (element instanceof HTMLAudioElement) return 'audio';
    return 'unknown';
  }

  // 资源加载失败时的降级处理
  private handleResourceFallback(element: HTMLElement, info: ResourceErrorInfo): void {
    // 图片降级
    if (element instanceof HTMLImageElement) {
      const fallbackUrl = '/images/placeholder.png';
      if (element.src !== fallbackUrl) {
        console.warn(`图片加载失败，降级到占位图: ${element.src}`);
        element.src = fallbackUrl;
      }
    }

    // 脚本加载失败
    if (element instanceof HTMLScriptElement) {
      console.error(`脚本加载失败: ${info.url}`);
      // 可以尝试重新加载或加载备用脚本
    }

    // 样式加载失败
    if (element instanceof HTMLLinkElement) {
      console.error(`样式加载失败: ${info.url}`);
      // 可以尝试内联关键样式
    }
  }

  // 确定错误严重程度
  private determineSeverity(error: Error | undefined): ErrorSeverity {
    if (!error) return ErrorSeverity.ERROR;

    const message = error.message || '';
    const fatalPatterns = [
      'Maximum call stack',
      'Out of memory',
      'Script error.'
    ];

    if (fatalPatterns.some(p => message.includes(p))) {
      return ErrorSeverity.CRITICAL;
    }

    return ErrorSeverity.ERROR;
  }

  // 获取错误上下文
  private getErrorContext(): ErrorContext {
    return {
      url: window.location.href,
      userAgent: navigator.userAgent,
      viewport: `${window.innerWidth}x${window.innerHeight}`,
      sessionId: this.getSessionId(),
      referrer: document.referrer
    };
  }

  // 获取或创建会话 ID
  private getSessionId(): string {
    let sessionId = sessionStorage.getItem('error_session_id');
    if (!sessionId) {
      sessionId = `sess_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`;
      sessionStorage.setItem('error_session_id', sessionId);
    }
    return sessionId;
  }

  // 生成错误 ID
  private generateErrorId(): string {
    return `err_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`;
  }

  // 通知错误处理器
  private notifyHandlers(errorInfo: ErrorInfo): void {
    this.errorHandlers.forEach(handler => {
      try {
        handler(errorInfo);
      } catch (e) {
        console.error('错误处理器执行失败:', e);
      }
    });
  }

  // 通知资源错误处理器
  private notifyResourceHandlers(info: ResourceErrorInfo): void {
    this.resourceErrorHandlers.forEach(handler => {
      try {
        handler(info);
      } catch (e) {
        console.error('资源错误处理器执行失败:', e);
      }
    });
  }

  // 注册错误处理器
  onError(handler: (error: ErrorInfo) => void): void {
    this.errorHandlers.push(handler);
  }

  // 注册资源错误处理器
  onResourceError(handler: (info: ResourceErrorInfo) => void): void {
    this.resourceErrorHandlers.push(handler);
  }

  // 刷新错误队列
  private flushErrors(): void {
    // 在页面卸载前确保关键错误已上报
    console.log('页面卸载，刷新错误队列');
  }
}

// 初始化全局错误捕获
export function initGlobalErrorCapture(): void {
  const capture = GlobalErrorCapture.getInstance();
  capture.initialize();

  // 注册默认错误处理器
  capture.onError((errorInfo) => {
    // 上报到服务器
    console.log('错误信息:', errorInfo);
  });

  capture.onResourceError((info) => {
    console.log('资源错误:', info);
  });
}
```

### 3.1 错误堆栈

错误堆栈是调试错误的关键信息，它记录了错误发生时的调用链路。理解错误堆栈的格式和内容，能够帮助开发者快速定位问题根源。

JavaScript Error 对象的 `stack` 属性包含了错误的调用栈信息，其格式因浏览器而异。标准的 V8 引擎（Chrome、Node.js）堆栈格式如下：

```
Error: Something went wrong
    at FunctionName (file.js:10:15)
    at Anonymous Function (file.js:20:5)
    at Module._compile (module.js:255:25)
```

```typescript
// 解析错误堆栈
interface StackFrame {
  functionName?: string;
  fileName?: string;
  lineNumber?: number;
  columnNumber?: number;
  isNative?: boolean;
  isConstructor?: boolean;
}

function parseStackTrace(stack: string): StackFrame[] {
  if (!stack) return [];

  // V8 引擎格式: at FunctionName (file.js:10:15)
  const v8Pattern = /at (?:(.+?)\s+\()?(.+?):(\d+):(\d+)\)?/g;

  const frames: StackFrame[] = [];
  let match;

  while ((match = v8Pattern.exec(stack)) !== null) {
    frames.push({
      functionName: match[1] || undefined,
      fileName: match[2],
      lineNumber: parseInt(match[3], 10),
      columnNumber: parseInt(match[4], 10)
    });
  }

  return frames;
}

// 简化堆栈信息（用于显示）
function simplifyStackTrace(stack: string, maxFrames = 5): string {
  const frames = parseStackTrace(stack);

  return frames
    .slice(0, maxFrames)
    .map((frame, index) => {
      const func = frame.functionName || '<anonymous>';
      const location = frame.fileName
        ? `${frame.fileName}:${frame.lineNumber}:${frame.columnNumber}`
        : '';
      return `  at ${func} (${location})`;
    })
    .join('\n');
}

// 收集完整错误信息
interface ErrorSnapshot {
  message: string;
  name: string;
  stack: string;
  parsedStack: StackFrame[];
  timestamp: number;
  userAgent: string;
  url: string;
}

function captureErrorSnapshot(error: Error): ErrorSnapshot {
  const stack = error.stack || '';

  return {
    message: error.message,
    name: error.name,
    stack,
    parsedStack: parseStackTrace(stack),
    timestamp: Date.now(),
    userAgent: navigator.userAgent,
    url: window.location.href
  };
}
```

### 3.2 用户信息

在错误上报中，用户信息的收集需要特别注意隐私保护。我们应该只收集对问题诊断有帮助的匿名化信息。

```typescript
// 用户信息收集器
interface UserInfo {
  userId?: string;
  userType: 'anonymous' | 'registered';
  sessionId: string;
  viewport: string;
  language: string;
  timezone: string;
  firstVisit?: number;
  visitCount?: number;
}

class UserInfoCollector {
  private sessionId: string;

  constructor() {
    this.sessionId = this.getOrCreateSessionId();
  }

  private getOrCreateSessionId(): string {
    const key = 'app_session_id';
    let sessionId = localStorage.getItem(key);

    if (!sessionId) {
      sessionId = `sess_${Date.now()}_${this.generateRandomId()}`;
      localStorage.setItem(key, sessionId);
    }

    return sessionId;
  }

  private generateRandomId(): string {
    return Math.random().toString(36).substr(2, 9);
  }

  // 收集用户信息（隐私保护版本）
  collect(isAuthenticated: boolean, authenticatedUserId?: string): UserInfo {
    return {
      userId: isAuthenticated ? this.hashUserId(authenticatedUserId) : undefined,
      userType: isAuthenticated ? 'registered' : 'anonymous',
      sessionId: this.sessionId,
      viewport: `${window.innerWidth}x${window.innerHeight}`,
      language: navigator.language,
      timezone: Intl.DateTimeFormat().resolvedOptions().timeZone,
      firstVisit: this.getFirstVisit(),
      visitCount: this.getVisitCount()
    };
  }

  // 哈希用户 ID（脱敏处理）
  private hashUserId(userId?: string): string {
    if (!userId) return 'anonymous';

    // 简单的哈希处理，实际生产中应使用更安全的方法
    let hash = 0;
    for (let i = 0; i < userId.length; i++) {
      const char = userId.charCodeAt(i);
      hash = ((hash << 5) - hash) + char;
      hash = hash & hash;
    }
    return `usr_${Math.abs(hash).toString(36)}`;
  }

  private getFirstVisit(): number {
    const key = 'first_visit';
    let firstVisit = sessionStorage.getItem(key);

    if (!firstVisit) {
      firstVisit = Date.now().toString();
      sessionStorage.setItem(key, firstVisit);
    }

    return parseInt(firstVisit, 10);
  }

  private getVisitCount(): number {
    const key = 'visit_count';
    let count = parseInt(sessionStorage.getItem(key) || '0', 10);
    count++;
    sessionStorage.setItem(key, count.toString());
    return count;
  }
}
```

### 3.3 环境信息

环境信息对于重现和诊断错误至关重要。这些信息包括浏览器版本、操作系统、网络状态等。

```typescript
// 环境信息收集器
interface EnvironmentInfo {
  // 浏览器信息
  browser: {
    name: string;
    version: string;
    major: string;
  };
  // 操作系统
  os: {
    name: string;
    version: string;

    };

  };
  // 应用信息
  app: {
    version: string;
    buildNumber: string;
    environment: 'development' | 'production' | 'test';
  };
}

class EnvironmentCollector {
  collect(appVersion: string, buildNumber: string, env: string): EnvironmentInfo {
    return {
      browser: this.detectBrowser(),
      os: this.detectOS(),
      device: this.detectDevice(),
      network: this.collectNetworkInfo(),
      performance: this.collectPerformanceInfo(),
      app: {
        version: appVersion,
        buildNumber,
        environment: env as any
      }
    };
  }

  // 浏览器检测
  private detectBrowser(): { name: string; version: string; major: string } {
    const ua = navigator.userAgent;
    const match = ua.match(/(Chrome|Firefox|Safari|Edge|Opera|MSIE|Trident(?=\/))\/?\s*(\d+)/i);

    if (!match) {
      return { name: 'Unknown', version: '0', major: '0' };
    }

    let name = match[1];
    let version = match[2];

    // 处理 IE 和 Trident
    if (name === 'MSIE' || name === 'Trident') {
      name = 'IE';
      // IE 11 的版本号在不同 UA 中
      const ieMatch = ua.match(/\b(?:MSIE |rv:)(\d+)/);
      version = ieMatch ? ieMatch[1] : '11';
    }

    // Edge 基于 Chromium
    if (name === 'Edge' && ua.includes('Edg/')) {
      const edgeMatch = ua.match(/Edg\/(\d+)/);
      if (edgeMatch) {
        name = 'Edge (Chromium)';
        version = edgeMatch[1];
      }
    }

    // Opera 基于 Chromium
    if (name === 'Opera' && ua.includes('OPR')) {
      const operaMatch = ua.match(/OPR\/(\d+)/);
      if (operaMatch) {
        name = 'Opera';
        version = operaMatch[1];
      }
    }

    return {
      name,
      version,
      major: version.split('.')[0]
    };
  }

  // 操作系统检测
  private detectOS(): { name: string; version: string } {
    const ua = navigator.userAgent;

    const osPatterns: Array<[RegExp, string]> = [
      [/Windows NT 10/, 'Windows 10'],
      [/Windows NT 6\.3/, 'Windows 8.1'],
      [/Windows NT 6\.2/, 'Windows 8'],
      [/Windows NT 6\.1/, 'Windows 7'],
      [/Windows Phone/, 'Windows Phone'],
      [/Mac OS X/, 'macOS'],
      [/Android/, 'Android'],
      [/iPhone|iPad|iPod/, 'iOS'],
      [/Linux/, 'Linux'],
      [/CrOS/, 'Chrome OS']
    ];

    for (const [pattern, name] of osPatterns) {
      if (pattern.test(ua)) {
        // 提取详细版本
        let version = '';
        if (name === 'macOS' || name === 'iOS') {
          const versionMatch = ua.match(/Mac OS X (\d+[._]\d+)/);
          if (versionMatch) {
            version = versionMatch[1].replace('_', '.');
          }
        } else if (name.startsWith('Windows')) {
          const winMatch = ua.match(/Windows NT (\d+\.\d+)/);
          if (winMatch) {
            const winVersions: Record<string, string> = {
              '10.0': '10',
              '6.3': '8.1',
              '6.2': '8',
              '6.1': '7'
            };
            version = winVersions[winMatch[1]] || winMatch[1];
          }
        }

        return { name, version };
      }
    }

    return { name: 'Unknown', version: '' };
  }

  // 设备类型检测
  private detectDevice(): { type: string; brand?: string; model?: string } {
    const ua = navigator.userAgent;

    if (/tablet|ipad/i.test(ua)) {
      return { type: 'tablet' };
    }

    if (/mobile|android|iphone|ipod/i.test(ua)) {
      // 尝试检测品牌和型号
      const mobileInfo = this.parseMobileInfo(ua);
      return {
        type: 'mobile',
        brand: mobileInfo.brand,
        model: mobileInfo.model
      };
    }

    return { type: 'desktop' };
  }

  private parseMobileInfo(ua: string): { brand?: string; model?: string } {
    // 简化版移动设备信息解析
    const knownBrands = ['Samsung', 'Huawei', 'Xiaomi', 'OPPO', 'Vivo', 'OnePlus', 'Google'];

    for (const brand of knownBrands) {
      if (ua.includes(brand)) {
        return { brand };
      }
    }

    return {};
  }

  // 网络信息
  private collectNetworkInfo(): { online: boolean; effectiveType?: string; downlink?: number; rtt?: number } {
    const connection = (navigator as any).connection ||
                       (navigator as any).mozConnection ||
                       (navigator as any).webkitConnection;

    return {
      online: navigator.onLine,
      effectiveType: connection?.effectiveType,
      downlink: connection?.downlink,
      rtt: connection?.rtt
    };
  }

  // 性能信息
  private collectPerformanceInfo(): { memory?: any; timing?: any } {
    const perf = performance as any;

    return {
      memory: perf.memory ? {
        usedJSHeapSize: perf.memory.usedJSHeapSize,
        totalJSHeapSize: perf.memory.totalJSHeapSize,
        jsHeapSizeLimit: perf.memory.jsHeapSizeLimit
      } : undefined,
      timing: performance.timing ? {
        navigationStart: performance.timing.navigationStart,
        domContentLoaded: performance.timing.domContentLoadedEventEnd,
        loadComplete: performance.timing.loadEventEnd
      } : undefined
    };
  }
}
```

浏览器信息是环境信息的重要组成部分。我们已经在环境信息收集器中包含了浏览器检测，这里再补充一些浏览器特定的处理。

```typescript
  static getBrowserSpecificErrorHandling(): Record<string, () => void> {
```

### 3.5 实战：上报 SDK

下面构建一个完整的上报 SDK，整合所有错误收集和上报功能。

```typescript
// 错误上报 SDK
interface ReportConfig {
  dsn: string;                    // 上报地址
  appKey: string;                 // 应用标识
  environment: string;            // 环境
  version: string;               // 版本
  release: string;               // 发布版本
  enabled: boolean;              // 是否启用
  maxQueueSize: number;          // 最大队列大小
  flushInterval: number;         // 上报间隔
  sampleRate: number;            // 采样率 (0-1)
  ignoreErrors: RegExp[];        // 忽略的错误模式
  whitelistErrors: RegExp[];     // 白名单错误（一定上报）
}

interface ErrorPayload {
  id: string;
  timestamp: number;
  type: string;
  level: string;
  message: string;
  stack?: string;
  metadata: Record<string, any>;
  context: ErrorContext;
  environment: EnvironmentInfo;
}

class ErrorReporter {
  private config: ReportConfig;
  private queue: ErrorPayload[] = [];
  private timer: number | null = null;
  private isFlushing = false;

  constructor(config: Partial<ReportConfig> = {}) {
    // 默认配置
    this.config = {
      dsn: config.dsn || '/api/errors',
      appKey: config.appKey || 'default',
      environment: config.environment || 'development',
      version: config.version || '1.0.0',
      release: config.release || '1.0.0',
      enabled: config.enabled !== false,
      maxQueueSize: config.maxQueueSize || 100,
      flushInterval: config.flushInterval || 5000,
      sampleRate: config.sampleRate || 1,
      ignoreErrors: config.ignoreErrors || [],
      whitelistErrors: config.whitelistErrors || []
    };

    if (this.config.enabled) {
      this.startFlushTimer();
    }
  }

  // 设置用户上下文
  setUserContext(userId?: string, userType?: string): void {
    sessionStorage.setItem('error_user_id', userId || '');
    sessionStorage.setItem('error_user_type', userType || 'anonymous');
  }

  // 清除用户上下文
  clearUserContext(): void {
    sessionStorage.removeItem('error_user_id');
    sessionStorage.removeItem('error_user_type');
  }

  // 上报错误
  captureException(error: Error | unknown, context: Record<string, any> = {}): void {
    if (!this.config.enabled) return;

    // 采样率控制
    if (Math.random() > this.config.sampleRate) return;

    // 检查是否忽略
    const message = error instanceof Error ? error.message : String(error);
    if (this.shouldIgnore(message)) return;

    const payload = this.buildPayload(error, context);
    this.queue.push(payload);

    // 控制队列大小
    if (this.queue.length > this.config.maxQueueSize) {
      this.queue.shift();
    }

    // 如果队列达到一定长度，立即上报
    if (this.queue.length >= 10) {
      this.flush();
    }
  }

  // 上报消息
  captureMessage(message: string, level: 'info' | 'warning' | 'error' = 'info'): void {
    if (!this.config.enabled) return;

    if (this.shouldIgnore(message)) return;

    const payload: ErrorPayload = {
      id: this.generateId(),
      timestamp: Date.now(),
      type: 'message',
      level,
      message,
      metadata: {},
      context: this.getContext(),
      environment: this.getEnvironment()
    };

    this.queue.push(payload);
  }

  // 检查是否应忽略
  private shouldIgnore(message: string): boolean {
    // 检查白名单
    for (const pattern of this.config.whitelistErrors) {
      if (pattern.test(message)) return false;
    }

    // 检查忽略列表
    for (const pattern of this.config.ignoreErrors) {
      if (pattern.test(message)) return true;
    }

    return false;
  }

  // 构建 payload
  private buildPayload(error: Error | unknown, context: Record<string, any>): ErrorPayload {
    const isError = error instanceof Error;

    return {
      id: this.generateId(),
      timestamp: Date.now(),
      type: isError ? error.constructor.name : 'UnknownError',
      level: 'error',
      message: isError ? error.message : String(error),
      stack: isError ? error.stack : undefined,
      metadata: context,
      context: this.getContext(),
      environment: this.getEnvironment()
    };
  }

  // 获取上下文
  private getContext(): ErrorContext {
    return {
      url: window.location.href,
      userAgent: navigator.userAgent,
      viewport: `${window.innerWidth}x${window.innerHeight}`,
      sessionId: sessionStorage.getItem('error_session_id') || '',
      userId: sessionStorage.getItem('error_user_id') || undefined,
      userType: sessionStorage.getItem('error_user_type') as any || 'anonymous'
    };
  }

  // 获取环境信息
  private getEnvironment(): EnvironmentInfo {
    const collector = new EnvironmentCollector();
    return collector.collect(
      this.config.version,
      this.config.release,
      this.config.environment
    );
  }

  // 生成唯一 ID
  private generateId(): string {
    return `${Date.now()}-${Math.random().toString(36).substr(2, 9)}`;
  }

  // 启动定时上报
  private startFlushTimer(): void {
    if (this.timer) clearInterval(this.timer);

    this.timer = window.setInterval(() => {
      this.flush();
    }, this.config.flushInterval);
  }

  // 立即上报
  async flush(): Promise<void> {
    if (this.isFlushing || this.queue.length === 0) return;

    this.isFlushing = true;

    const payload = [...this.queue];
    this.queue = [];

    try {
      await this.sendToServer(payload);
    } catch (error) {
      // 发送失败，将数据放回队列
      console.error('错误上报失败:', error);
      this.queue.unshift(...payload);
    } finally {
      this.isFlushing = false;
    }
  }

  // 发送到服务器
  private async sendToServer(payload: ErrorPayload[]): Promise<void> {
    const response = await fetch(this.config.dsn, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'X-App-Key': this.config.appKey,
        'X-Environment': this.config.environment
      },
      body: JSON.stringify({
        payload,
        appKey: this.config.appKey,
        environment: this.config.environment,
        version: this.config.version
      }),
      keepalive: true
    });

    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }
  }

  // 手动设置配置
  updateConfig(config: Partial<ReportConfig>): void {
    this.config = { ...this.config, ...config };

    if (config.flushInterval !== undefined || config.enabled !== undefined) {
      this.startFlushTimer();
    }
  }
}
```

## 四、错误告警

### 4.1 告警级别：Critical/Warning/Info

错误告警系统需要根据错误的严重程度采取不同的响应策略。合理的告警分级可以确保关键问题得到及时处理，同时避免过多的无效告警造成干扰。

```typescript
// 告警级别定义
enum AlertLevel {
  CRITICAL = 'critical',  // 致命，需要立即处理
  WARNING = 'warning',    // 警告，需要关注
  INFO = 'info'           // 信息，记录即可
}

// 告警规则
interface AlertRule {
  id: string;
  name: string;
  level: AlertLevel;
  condition: (error: ErrorInfo) => boolean;
  threshold?: number;      // 触发阈值
  timeWindow?: number;     // 时间窗口（毫秒）
  cooldown?: number;       // 冷却时间（避免重复告警）
  enabled: boolean;
}

// 告警历史记录
interface AlertRecord {
  ruleId: string;
  errorCount: number;
  firstOccurrence: number;
  lastOccurrence: number;
  acknowledged: boolean;
  resolvedAt?: number;
}

// 告警管理器
class AlertManager {
  private rules: Map<string, AlertRule> = new Map();
  private history: Map<string, AlertRecord> = new Map();
  private cooldowns: Map<string, number> = new Map();
  private listeners: Map<AlertLevel, Array<(alert: Alert) => void>> = new Map();

  // 构造函数中设置默认规则
  constructor() {
    this.setupDefaultRules();
  }

  // 设置默认告警规则
  private setupDefaultRules(): void {
    // 致命错误规则
    this.addRule({
      id: 'fatal-error',
      name: '致命错误',
      level: AlertLevel.CRITICAL,
      condition: (error) => error.severity === ErrorSeverity.CRITICAL,
      threshold: 1,
      cooldown: 0, // 立即告警
      enabled: true
    });

    // JS 运行时错误规则
    this.addRule({
      id: 'runtime-error',
      name: '运行时错误',
      level: AlertLevel.WARNING,
      condition: (error) => error.category === ErrorCategory.RUNTIME,
      threshold: 5,
      timeWindow: 60000, // 1 分钟内超过 5 次
      cooldown: 300000,   // 冷却 5 分钟
      enabled: true
    });

    // Promise rejection 规则
    this.addRule({
      id: 'promise-rejection',
      name: 'Promise 拒绝',
      level: AlertLevel.WARNING,
      condition: (error) => error.category === ErrorCategory.PROMISE,
      threshold: 10,
      timeWindow: 60000,
      cooldown: 300000,
      enabled: true
    });

    // 资源加载错误规则
    this.addRule({
      id: 'resource-error',
      name: '资源加载错误',
      level: AlertLevel.INFO,
      condition: (error) => error.category === ErrorCategory.RESOURCE,
      threshold: 20,
      timeWindow: 60000,
      cooldown: 600000,
      enabled: true
    });

    // 错误率规则（需要外部提供错误率数据）
    this.addRule({
      id: 'high-error-rate',
      name: '高错误率',
      level: AlertLevel.CRITICAL,
      condition: (error) => error.metadata?.errorRate > 0.05, // 5% 错误率
      threshold: 1,
      cooldown: 60000,
      enabled: true
    });
  }

  // 添加告警规则
  addRule(rule: AlertRule): void {
    this.rules.set(rule.id, rule);
  }

  // 移除告警规则
  removeRule(ruleId: string): void {
    this.rules.delete(ruleId);
  }

  // 处理错误
  processError(error: ErrorInfo): void {
    for (const [ruleId, rule] of this.rules) {
      if (!rule.enabled) continue;

      // 检查冷却期
      if (this.isInCooldown(ruleId)) continue;

      // 检查触发条件
      if (!rule.condition(error)) continue;

      // 更新历史记录
      const record = this.getOrCreateRecord(ruleId);
      record.errorCount++;
      record.lastOccurrence = Date.now();

      // 检查是否达到阈值
      if (record.errorCount >= (rule.threshold || 1)) {
        // 检查时间窗口
        if (rule.timeWindow) {
          const elapsed = record.lastOccurrence - record.firstOccurrence;
          if (elapsed <= rule.timeWindow) {
            this.triggerAlert(rule, record);
          }
        } else {
          this.triggerAlert(rule, record);
        }

        // 重置计数
        record.errorCount = 0;
        record.firstOccurrence = Date.now();

        // 设置冷却期
        if (rule.cooldown) {
          this.cooldowns.set(ruleId, Date.now() + rule.cooldown);
        }
      }
    }
  }

  // 获取或创建历史记录
  private getOrCreateRecord(ruleId: string): AlertRecord {
    let record = this.history.get(ruleId);

    if (!record) {
      record = {
        ruleId,
        errorCount: 0,
        firstOccurrence: Date.now(),
        lastOccurrence: Date.now(),
        acknowledged: false
      };
      this.history.set(ruleId, record);
    }

    return record;
  }

  // 检查是否在冷却期
  private isInCooldown(ruleId: string): boolean {
    const cooldownEnd = this.cooldowns.get(ruleId);
    if (cooldownEnd && Date.now() < cooldownEnd) {
      return true;
    }
    this.cooldowns.delete(ruleId);
    return false;
  }

  // 触发告警
  private triggerAlert(rule: AlertRule, record: AlertRecord): void {
    const alert: Alert = {
      id: `alert_${Date.now()}`,
      ruleId: rule.id,
      ruleName: rule.name,
      level: rule.level,
      message: this.buildAlertMessage(rule, record),
      timestamp: Date.now(),
      record
    };

    // 通知监听器
    const listeners = this.listeners.get(rule.level);
    if (listeners) {
      listeners.forEach(listener => {
        try {
          listener(alert);
        } catch (e) {
          console.error('告警监听器执行失败:', e);
        }
      });
    }

    // 发送告警通知
    this.sendAlertNotification(alert);
  }

  // 构建告警消息
  private buildAlertMessage(rule: AlertRule, record: AlertRecord): string {
    return `${rule.name}告警：${record.errorCount}次错误（阈值：${rule.threshold}）`;
  }

  // 发送告警通知
  private sendAlertNotification(alert: Alert): void {
    // 根据告警级别选择通知方式
    switch (alert.level) {
      case AlertLevel.CRITICAL:
        // 致命告警：短信、电话、邮件、钉钉/飞书全部通知
        this.notifyViaAllChannels(alert);
        break;
      case AlertLevel.WARNING:
        // 警告：邮件、钉钉/飞书通知
        this.notifyViaIM(alert);
        break;
      case AlertLevel.INFO:
        // 信息：仅记录
        console.info('告警信息:', alert);
        break;
    }
  }

  // 全部渠道通知
  private async notifyViaAllChannels(alert: Alert): Promise<void> {
    await Promise.all([
      this.sendSMS(alert),
      this.sendEmail(alert),
      this.sendIM(alert)
    ]);
  }

  // IM 通知
  private async notifyViaIM(alert: Alert): Promise<void> {
    await this.sendIM(alert);
  }

  // 短信通知
  private async sendSMS(alert: Alert): Promise<void> {
    console.log('发送短信告警:', alert);
  }

  // 邮件通知
  private async sendEmail(alert: Alert): Promise<void> {
    console.log('发送邮件告警:', alert);
  }

  // IM 通知（钉钉/飞书）
  private async sendIM(alert: Alert): Promise<void> {
    console.log('发送 IM 告警:', alert);
  }

  // 注册告警监听器
  onAlert(level: AlertLevel, listener: (alert: Alert) => void): void {
    const listeners = this.listeners.get(level) || [];
    listeners.push(listener);
    this.listeners.set(level, listeners);
  }
}
```

### 4.2 聚合分析

错误聚合分析是将大量相似的错误归类到一起，以便更好地理解问题的本质和影响范围。通过聚合，我们可以识别出哪些是根因问题，哪些是衍生问题。

```typescript
// 错误聚合器
class ErrorAggregator {
  // 错误签名：用于识别相同类型的错误
  private getErrorSignature(error: ErrorInfo): string {
    // 使用错误消息的哈希作为签名
    // 对于有堆栈的错误，可以包含堆栈的前几行
    const msg = error.message;
    const stack = error.stack || '';

    // 提取堆栈中的函数名
    const funcNames = stack
      .split('\n')
      .slice(0, 3)
      .map(line => {
        const match = line.match(/at\s+(.+?)\s+\(/);
        return match ? match[1] : '';
      })
      .filter(Boolean)
      .join('|');

    return `${msg}|${funcNames}`;
  }

  // 聚合错误
  aggregate(errors: ErrorInfo[]): AggregatedError[] {
    const groups = new Map<string, ErrorInfo[]>();

    // 按签名分组
    for (const error of errors) {
      const signature = this.getErrorSignature(error);
      const existing = groups.get(signature);

      if (existing) {
        existing.push(error);
      } else {
        groups.set(signature, [error]);
      }
    }

    // 转换为聚合结果
    const results: AggregatedError[] = [];

    for (const [signature, errorList] of groups) {
      const firstError = errorList[0];
      const latestError = errorList[errorList.length - 1];

      results.push({
        id: signature,
        signature,
        message: firstError.message,
        category: firstError.category,
        severity: this.calculateGroupSeverity(errorList),
        count: errorList.length,
        firstOccurrence: firstError.timestamp,
        lastOccurrence: latestError.timestamp,
        affectedUsers: this.countAffectedUsers(errorList),
        errors: errorList,
        stack: firstError.stack
      });
    }

    // 按严重程度和频率排序
    return results.sort((a, b) => {
      // 首先按严重程度
      const severityOrder = { critical: 0, error: 1, warning: 2, info: 3 };
      const severityDiff = severityOrder[a.severity] - severityOrder[b.severity];
      if (severityDiff !== 0) return severityDiff;

      // 然后按影响用户数
      return b.affectedUsers - a.affectedUsers;
    });
  }

  // 计算组内最高严重程度
  private calculateGroupSeverity(errors: ErrorInfo[]): ErrorSeverity {
    const severityOrder = { critical: 0, error: 1, warning: 2, info: 3 };

    return errors.reduce((max, error) => {
      const current = severityOrder[error.severity];
      const maxSeverity = severityOrder[max];
      return current < maxSeverity ? error.severity : max;
    }, 'info' as ErrorSeverity);
  }

  // 统计影响用户数
  private countAffectedUsers(errors: ErrorInfo[]): number {
    const userIds = new Set<string>();

    for (const error of errors) {
      if (error.context.userId) {
        userIds.add(error.context.userId);
      }
    }

    return userIds.size;
  }
}

// 聚合错误结果
interface AggregatedError {
  id: string;
  signature: string;
  message: string;
  category: ErrorCategory;
  severity: ErrorSeverity;
  count: number;
  firstOccurrence: number;
  lastOccurrence: number;
  affectedUsers: number;
  errors: ErrorInfo[];
  stack?: string;
}

// 错误分析报告
interface ErrorAnalysisReport {
  generatedAt: number;
  period: { start: number; end: number };
  summary: {
    totalErrors: number;
    totalUniqueErrors: number;
    totalAffectedUsers: number;
    criticalCount: number;
    errorCount: number;
    warningCount: number;
  };
  topErrors: AggregatedError[];
  errorsByCategory: Record<ErrorCategory, number>;
  errorsByHour: number[];
  recommendations: string[];
}
```

### 4.3 告警收敛

告警收敛是避免在短时间内发送大量重复告警的机制。当一个根因错误导致大量衍生错误时，我们应该只通知根因问题，而不是每个衍生错误都发送告警。

```typescript
// 告警收敛器
class AlertDeduplicator {
  private recentAlerts: Map<string, AlertRecord> = new Map();
  private deduplicationWindow = 60000; // 1 分钟内相同告警去重
  private rootCauseMap: Map<string, string> = new Map(); // 衍生关系

  constructor() {
    // 建立错误类型之间的关系
    this.setupRootCauseRelations();
  }

  // 设置根因关系
  private setupRootCauseRelations(): void {
    // 如果是 "Cannot read property" 错误，可能是因为 "fetch failed"
    this.rootCauseMap.set('TypeError', 'NetworkError');
    this.rootCauseMap.set('ReferenceError', 'NetworkError');
    this.rootCauseMap.set('SyntaxError', 'ParseError');
  }

  // 检查是否可以发送告警
  shouldSendAlert(alert: Alert): { send: boolean; reason?: string } {
    const key = this.getAlertKey(alert);
    const existing = this.recentAlerts.get(key);

    if (existing) {
      const elapsed = Date.now() - existing.timestamp;
      if (elapsed < this.deduplicationWindow) {
        return {
          send: false,
          reason: `相同告警在 ${Math.ceil((this.deduplicationWindow - elapsed) / 1000)} 秒后可重发`
        };
      }
    }

    // 检查是否是根因告警的衍生告警
    if (this.isDerivedAlert(alert)) {
      const rootAlert = this.findRootAlert(alert);
      if (rootAlert) {
        return {
          send: false,
          reason: `衍生告警，已由根因告警覆盖`
        };
      }
    }

    // 记录这次告警
    this.recentAlerts.set(key, {
      ...alert,
      timestamp: Date.now()
    });

    return { send: true };
  }

  // 获取告警 key
  private getAlertKey(alert: Alert): string {
    return `${alert.ruleId}:${alert.level}`;
  }

  // 判断是否是衍生告警
  private isDerivedAlert(alert: Alert): boolean {
    return this.rootCauseMap.has(alert.ruleId);
  }

  // 查找根因告警
  private findRootAlert(alert: Alert): Alert | null {
    const rootId = this.rootCauseMap.get(alert.ruleId);
    if (!rootId) return null;

    const key = `${rootId}:${alert.level}`;
    return this.recentAlerts.get(key) || null;
  }

  // 清理过期记录
  cleanup(): void {
    const now = Date.now();

    for (const [key, record] of this.recentAlerts) {
      if (now - record.timestamp > this.deduplicationWindow * 10) {
        this.recentAlerts.delete(key);
      }
    }
  }
}
```

```typescript
    for (const error of errors) {

      errorsByHour[hourIndex]++;

    const errorsByCategory = {} as Record<ErrorCategory, number>;
    for (const error of errors) {
```

### 5.1 超时处理

接口调用必须设置合理的超时时间，以避免请求无限等待导致用户界面冻结。超时处理是接口容错的第一道防线。

```typescript
// 超时控制工具
class TimeoutController {
  private timeoutId: number | null = null;
  private timeout: number;

  constructor(timeout: number = 30000) {
    this.timeout = timeout;
  }

  // 执行带超时的操作
  async execute<T>(operation: () => Promise<T>): Promise<T> {
    return new Promise((resolve, reject) => {
      this.timeoutId = window.setTimeout(() => {
        reject(new TimeoutError(`操作超时: ${this.timeout}ms`));
      }, this.timeout);

      operation()
        .then(result => {
          this.clear();
          resolve(result);
        })
        .catch(error => {
          this.clear();
          reject(error);
        });
    });
  }

  // 清除超时
  clear(): void {
    if (this.timeoutId !== null) {
      clearTimeout(this.timeoutId);
      this.timeoutId = null;
    }
  }
}

async function fetchWithTimeout(
  url: string,
  options: RequestInit & { timeout?: number } = {}
): Promise<Response> {
  const timeout = options.timeout || 30000;

    if (error instanceof DOMException && error.name === 'AbortError') {
      throw new TimeoutError(`请求超时: ${url}`);
    }

    throw error;
  }
}
```

### 5.2 重试机制

合理的重试机制可以提高接口的可靠性，但过度重试可能导致服务器压力和用户体验下降。我们需要实现智能的重试策略。

```typescript
// 重试配置
interface RetryConfig {
  maxRetries: number;        // 最大重试次数
  initialDelay: number;       // 初始延迟（毫秒）
  maxDelay: number;           // 最大延迟（毫秒）
  backoffFactor: number;      // 退避因子
  retryCondition?: (error: RetryError) => boolean; // 重试条件
  shouldRetry?: (error: RetryError) => boolean;      // 是否应该重试
}

// 重试错误
interface RetryError {
  error: Error;
  attempt: number;
  totalRetries: number;
}

// 重试装饰器
function withRetry<T extends (...args: any[]) => Promise<any>>(
  fn: T,
  config: Partial<RetryConfig> = {}
): T {
  const {
    maxRetries = 3,
    initialDelay = 1000,
    maxDelay = 10000,
    backoffFactor = 2,
    shouldRetry = defaultShouldRetry
  } = config;

  return (async (...args: any[]) => {
    let lastError: Error;
    let delay = initialDelay;

    for (let attempt = 0; attempt <= maxRetries; attempt++) {
      try {
        return await fn(...args);
      } catch (error) {
        lastError = error as Error;

        if (attempt === maxRetries || !shouldRetry(lastError)) {
          throw lastError;
        }

        console.log(`请求失败，第 ${attempt + 1} 次重试，${delay}ms 后...`);

        // 等待后重试
        await sleep(delay);

        // 计算下次延迟（指数退避）
        delay = Math.min(delay * backoffFactor, maxDelay);
      }
    }

    throw lastError!;
  }) as T;
}

// 默认重试条件
function defaultShouldRetry(error: Error): boolean {
  // 网络错误、超时、5xx 服务器错误可以重试
  if (error instanceof TimeoutError) return true;

  const message = error.message.toLowerCase();
  if (message.includes('network') || message.includes('fetch')) return true;
  if (message.includes('500') || message.includes('502') || message.includes('503')) return true;

  return false;
}

// 睡眠工具
function sleep(ms: number): Promise<void> {
  return new Promise(resolve => setTimeout(resolve, ms));
}

// API 请求封装（带重试）
class APIClient {
  private baseURL: string;
  private retryConfig: RetryConfig;

  constructor(baseURL: string, retryConfig: Partial<RetryConfig> = {}) {
    this.baseURL = baseURL;
    this.retryConfig = {
      maxRetries: 3,
      initialDelay: 1000,
      maxDelay: 10000,
      backoffFactor: 2,
      ...retryConfig
    };
  }

  // GET 请求
  async get<T>(path: string, params?: Record<string, string>): Promise<T> {
    const url = new URL(path, this.baseURL);
    if (params) {
      url.search = new URLSearchParams(params).toString();
    }

    return this.request<T>(url.toString(), { method: 'GET' });
  }

  // POST 请求
  async post<T>(path: string, data?: any): Promise<T> {
    return this.request<T>(`${this.baseURL}${path}`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    });
  }

  // 核心请求方法
  private async request<T>(url: string, options: RequestInit): Promise<T> {
    const requestWithRetry = withRetry(
      async () => {
        const response = await fetchWithTimeout(url, {
          ...options,
          timeout: 30000
        });

        if (!response.ok) {
          throw new HTTPError(response.status, response.statusText, url);
        }

        return response.json() as Promise<T>;
      },
      {
        shouldRetry: (error) => {
          // 只有服务器错误才重试，客户端错误不重试
          if (error instanceof HTTPError && error.status < 500) {
            return false;
          }
          return true;
        }
      }
    );

    return requestWithRetry();
  }
}

// HTTP 错误

  constructor(
    public status: number,
    public statusText: string,
    public url: string
  ) {
    super(`HTTP ${status}: ${statusText} (${url})`);

  }
}
```

### 5.3 降级处理

降级处理是在主要功能不可用时，提供一个次优但可用的替代方案。降级策略的目标是保证核心功能可用，即使部分功能暂时受损。

```typescript
// 降级处理策略
interface DegradeConfig<T> {
  // 原始数据源
  primary: () => Promise<T>;
  // 降级数据源
  fallback: () => Promise<T>;
  // 是否使用缓存作为降级
  useCache?: boolean;
  // 缓存 key
  cacheKey?: string;
  // 缓存有效期
  cacheTTL?: number;
}

// 降级执行器
class DegradeExecutor {
  // 执行带降级的操作
  static async execute<T>(
    config: DegradeConfig<T>
  ): Promise<{ data: T; source: 'primary' | 'fallback' | 'cache' }> {
    const { primary, fallback, useCache, cacheKey, cacheTTL = 3600000 } = config;

    // 1. 尝试主要数据源
    try {
      const data = await Promise.race([
        primary(),
        new Promise<null>((_, reject) =>
          setTimeout(() => reject(new TimeoutError('主要数据源超时')), 5000)
        )
      ]);

      if (data !== null) {
        // 如果有缓存，更新缓存
        if (cacheKey) {
          this.setCache(cacheKey, data, cacheTTL);
        }
        return { data, source: 'primary' };
      }
    } catch (primaryError) {
      console.warn('主要数据源失败:', primaryError);
    }

    // 2. 尝试缓存
    if (useCache && cacheKey) {
      const cached = this.getCache<T>(cacheKey);
      if (cached) {
        console.info('使用缓存数据降级');
        return { data: cached, source: 'cache' };
      }
    }

    // 3. 尝试降级数据源
    try {
      const fallbackData = await fallback();
      return { data: fallbackData, source: 'fallback' };
    } catch (fallbackError) {
      console.error('降级数据源也失败:', fallbackError);
      throw new Error('所有数据源都不可用');
    }
  }

  // 获取缓存
  private static getCache<T>(key: string): T | null {
    const cached = localStorage.getItem(`cache_${key}`);
    if (!cached) return null;

    try {
      const { data, expiry } = JSON.parse(cached);
      if (Date.now() > expiry) {
        localStorage.removeItem(`cache_${key}`);
        return null;
      }
      return data as T;
    } catch {
      return null;
    }
  }

  // 设置缓存
  private static setCache<T>(key: string, data: T, ttl: number): void {
    const cacheItem = {
      data,
      expiry: Date.now() + ttl
    };
    localStorage.setItem(`cache_${key}`, JSON.stringify(cacheItem));
  }
}

// 降级组件 HOC
function withDegradedUI<P extends object>(
  WrappedComponent: React.ComponentType<P>,
  fallbackUI: React.ReactNode,
  errorExtractor: (error: any) => boolean
) {
  return function DegradedComponent(props: P) {
    const [hasError, setHasError] = useState(false);

    const handleError = useCallback((error: any) => {
      if (errorExtractor(error)) {
        setHasError(true);
      }
    }, [errorExtractor]);

    if (hasError) {
      return <>{fallbackUI}</>;
    }

    return (
      <ErrorBoundary onError={handleError}>
        <WrappedComponent {...props} />
      </ErrorBoundary>
    );
  };
}
```

### 5.4 熔断器

熔断器模式借鉴自电路保护机制，当某个接口的失败率超过阈值时，熔断器会"跳闸"，后续对该接口的请求会直接返回降级响应，而不会真正发送到服务器。这样可以防止故障蔓延，给服务恢复提供时间。

```typescript
// 熔断器状态
enum CircuitState {
  CLOSED = 'closed',    // 关闭状态，正常请求
  OPEN = 'open',         // 打开状态，请求直接降级
  HALF_OPEN = 'half_open' // 半开状态，允许部分请求
}

// 熔断器配置
interface CircuitBreakerConfig {
  failureThreshold: number;      // 失败次数阈值
  successThreshold: number;     // 成功次数阈值（从半开恢复）
  timeout: number;              // 熔断持续时间
  halfOpenMaxCalls: number;    // 半开状态允许的请求数
}

// 熔断器
class CircuitBreaker {
  private state: CircuitState = CircuitState.CLOSED;
  private failureCount = 0;
  private successCount = 0;
  private nextAttempt = 0;
  private halfOpenCalls = 0;

  constructor(
    private name: string,
    private config: CircuitBreakerConfig
  ) {
    this.config = {
      failureThreshold: 5,
      successThreshold: 2,
      timeout: 60000,
      halfOpenMaxCalls: 3,
      ...config
    };
  }

  // 执行请求
  async execute<T>(
    operation: () => Promise<T>,
    fallback: () => Promise<T>
  ): Promise<T> {
    // 检查是否可以执行
    if (!this.canExecute()) {
      console.warn(`[CircuitBreaker:${this.name}] 熔断器打开，使用降级方案`);
      return fallback();
    }

    try {
      const result = await operation();
      this.onSuccess();
      return result;
    } catch (error) {
      this.onFailure();
      // 熔断器打开时，返回降级结果
      if (this.state === CircuitState.OPEN) {
        return fallback();
      }
      throw error;
    }
  }

  // 检查是否可以执行
  private canExecute(): boolean {
    if (this.state === CircuitState.CLOSED) {
      return true;
    }

    if (this.state === CircuitState.OPEN) {
      // 检查是否超过熔断时间
      if (Date.now() >= this.nextAttempt) {
        this.toHalfOpen();
        return true;
      }
      return false;
    }

    // 半开状态
    if (this.halfOpenCalls < this.config.halfOpenMaxCalls) {
      this.halfOpenCalls++;
      return true;
    }
    return false;
  }

  // 成功回调
  private onSuccess(): void {
    this.failureCount = 0;

    if (this.state === CircuitState.HALF_OPEN) {
      this.successCount++;
      if (this.successCount >= this.config.successThreshold) {
        this.toClosed();
      }
    }
  }

  // 失败回调
  private onFailure(): void {
    this.failureCount++;
    this.successCount = 0;

    if (this.state === CircuitState.HALF_OPEN) {
      this.toOpen();
    } else if (this.failureCount >= this.config.failureThreshold) {
      this.toOpen();
    }
  }

  // 转为打开状态
  private toOpen(): void {
    this.state = CircuitState.OPEN;
    this.nextAttempt = Date.now() + this.config.timeout;
    console.warn(`[CircuitBreaker:${this.name}] 熔断器打开`);
  }

  // 转为半开状态
  private toHalfOpen(): void {
    this.state = CircuitState.HALF_OPEN;
    this.halfOpenCalls = 0;
    this.successCount = 0;
    console.info(`[CircuitBreaker:${this.name}] 熔断器进入半开状态`);
  }

  // 转为关闭状态
  private toClosed(): void {
    this.state = CircuitState.CLOSED;
    this.failureCount = 0;
    this.successCount = 0;
    console.info(`[CircuitBreaker:${this.name}] 熔断器关闭，恢复正常`);
  }

  // 获取状态
  getState(): CircuitState {
    return this.state;
  }

  // 重置
  reset(): void {
    this.toClosed();
  }
}

// 熔断器管理器
class CircuitBreakerManager {
  private breakers: Map<string, CircuitBreaker> = new Map();

  // 获取或创建熔断器
  getBreaker(name: string, config?: Partial<CircuitBreakerConfig>): CircuitBreaker {
    let breaker = this.breakers.get(name);

    if (!breaker) {
      breaker = new CircuitBreaker(name, config || {});
      this.breakers.set(name, breaker);
    }

    return breaker;

    this.breakers.delete(name);
  }

  // 获取所有熔断器状态
  getAllStates(): Record<string, CircuitState> {
    const states: Record<string, CircuitState> = {};

    this.breakers.forEach((breaker, name) => {
      states[name] = breaker.getState();
    });

    return states;
  }
```

### 5.5 实战：容错实现

整合所有容错机制，实现一个健壮的 API 调用层。

```typescript
// 完整容错 API 客户端
class ResilientAPIClient {
  private baseURL: string;
  private circuitBreakerManager: CircuitBreakerManager;
  private defaultFallback: () => Promise<any>;

  constructor(baseURL: string) {
    this.baseURL = baseURL;
    this.circuitBreakerManager = new CircuitBreakerManager();

    // 默认降级处理
    this.defaultFallback = async () => {
      throw new Error('服务暂时不可用，请稍后重试');
    };
  }

  // 创建带容错的请求
  async request<T>(
    path: string,
    options: {
      method?: string;
      body?: any;
      retries?: number;
      timeout?: number;
      circuitBreaker?: boolean;
      fallback?: () => Promise<T>;
    } = {}
  ): Promise<T> {
    const {
      method = 'GET',
      body,
      retries = 3,
      timeout = 30000,
      circuitBreaker = true,
      fallback
    } = options;

    const url = `${this.baseURL}${path}`;
    const operation = () => this.executeRequest<T>(url, method, body, timeout);
    const executeFallback = fallback || this.defaultFallback;

    // 1. 带重试的执行
    if (retries > 0) {
      return this.executeWithRetry(operation, retries, executeFallback);
    }

    // 2. 带熔断器的执行
    if (circuitBreaker) {
      const breaker = this.circuitBreakerManager.getBreaker(path);
      return breaker.execute(operation, executeFallback);
    }

    // 3. 直接执行
    try {
      return await operation();
    } catch (error) {
      return executeFallback();
    }
  }

  // 执行 HTTP 请求
  private async executeRequest<T>(
    url: string,
    method: string,
    body: any,
    timeout: number
  ): Promise<T> {
    const controller = new AbortController();
    const timeoutId = setTimeout(() => controller.abort(), timeout);

    try {
      const response = await fetch(url, {
        method,
        headers: {
          'Content-Type': 'application/json'
        },
        body: body ? JSON.stringify(body) : undefined,
        signal: controller.signal
      });

      clearTimeout(timeoutId);

      if (!response.ok) {
        throw new HTTPError(response.status, response.statusText, url);
      }

      return response.json();
    } catch (error) {
      clearTimeout(timeoutId);

      if (error instanceof DOMException && error.name === 'AbortError') {
        throw new TimeoutError(`请求超时: ${url}`);
      }

      throw error;
    }
  }

  // 带重试的执行
  private async executeWithRetry<T>(
    operation: () => Promise<T>,
    maxRetries: number,
    fallback: () => Promise<T>
  ): Promise<T> {
    let lastError: Error;
    let delay = 1000;

    for (let attempt = 0; attempt <= maxRetries; attempt++) {
      try {
        return await operation();
      } catch (error) {
        lastError = error as Error;

        // 检查是否是可重试的错误
        if (!this.isRetryableError(error)) {
          throw error;
        }

        // 达到最大重试次数
        if (attempt === maxRetries) {
          break;
        }

        console.warn(`请求失败，第 ${attempt + 1} 次重试...`);
        await this.sleep(delay);
        delay = Math.min(delay * 2, 10000);
      }
    }

    // 所有重试都失败，返回降级结果
    console.warn('所有重试失败，执行降级处理');
    return fallback();
  }

  // 判断是否是可重试的错误
  private isRetryableError(error: Error): boolean {
    if (error instanceof TimeoutError) return true;
    if (error instanceof HTTPError && error.status >= 500) return true;

    const message = error.message.toLowerCase();
    return message.includes('network') || message.includes('fetch');
  }

  private sleep(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }

  // 获取熔断器状态
  getCircuitBreakerStates(): Record<string, CircuitState> {
    return this.circuitBreakerManager.getAllStates();
  }
}

// 创建客户端实例
const apiClient = new ResilientAPIClient('/api');

    circuitBreaker: true,

      // 降级：返回缓存的用户数据

      }
      // 降级：返回默认数据
```

```typescript
class DegradationManager {
```

```typescript
  // 添加乐观更新
  addUpdate(update: OptimisticUpdate): string {
    const id = `opt_${Date.now()}_${Math.random().toString(36).substr(2, 6)}`;

    this.pendingUpdates.set(id, {
      ...update,
      id,
      status: 'pending',
      createdAt: Date.now()
    });

    // 通知监听器
    this.notifyListeners(update);

    return id;
  }

  // 确认更新成功
  confirmUpdate(id: string): void {
    const update = this.pendingUpdates.get(id);
    if (update) {
      update.status = 'confirmed';
      update.confirmedAt = Date.now();
      this.pendingUpdates.delete(id);
      this.notifyListeners(update);
    }
  }

  // 回滚更新
  rollbackUpdate(id: string, error?: Error): void {
    const update = this.pendingUpdates.get(id);
    if (update) {
      update.status = 'rolled-back';
      update.rolledBackAt = Date.now();
      update.error = error;
      this.pendingUpdates.delete(id);

      // 执行回滚回调
      if (update.rollback) {
        update.rollback();
      }

      this.notifyListeners(update);
    }
  }

  // 获取待处理的更新
  getPendingUpdates(): OptimisticUpdate[] {
    return Array.from(this.pendingUpdates.values());
  }

  // 订阅更新变化
  subscribe(listener: (update: OptimisticUpdate) => void): () => void {
    this.listeners.add(listener);
    return () => this.listeners.delete(listener);
  }

  private notifyListeners(update: OptimisticUpdate): void {
    this.listeners.forEach(listener => {
      try {
        listener(update);
      } catch (e) {
        console.error('乐观更新监听器执行失败:', e);
      }
    });
  }
}

interface OptimisticUpdate {
  id: string;
  type: string;
  previousValue: any;
  newValue: any;
  status: 'pending' | 'confirmed' | 'rolled-back';
  createdAt: number;
  confirmedAt?: number;
  rolledBackAt?: number;
  error?: Error;
  rollback?: () => void;
}

// React 乐观更新 Hook
function useOptimisticUpdate<T>(
  initialValue: T
): {
  value: T;
  optimisticValue: T;
  isPending: boolean;
  update: (newValue: T, rollback?: () => void) => Promise<void>;
  confirm: () => void;
  rollback: (error?: Error) => void;
} {
  const [value, setValue] = useState<T>(initialValue);
  const [pendingUpdate, setPendingUpdate] = useState<T | null>(null);

  const update = async (newValue: T, rollback?: () => void) => {
    // 立即更新 UI
    setPendingUpdate(newValue);

    try {
      // 等待服务器确认
      // 这里假设 updateFn 会发起服务器请求
      // 实际上应该返回一个 Promise
      await new Promise(resolve => setTimeout(resolve, 1000));

      // 确认更新
      setValue(newValue);
      setPendingUpdate(null);
    } catch (error) {
      // 回滚
      if (rollback) rollback();
      setPendingUpdate(null);
      throw error;
    }
  };

  return {
    value,
    optimisticValue: pendingUpdate !== null ? pendingUpdate : value,
    isPending: pendingUpdate !== null,
    update,
    confirm: () => {
      if (pendingUpdate !== null) {
        setValue(pendingUpdate);
        setPendingUpdate(null);
      }
    },
    rollback: (error?: Error) => {
      setPendingUpdate(null);
      if (error) console.error('乐观更新失败:', error);
    }
  };
}
```

### 7.2 回滚机制

回滚机制是在操作失败时恢复到之前状态的能力。它是数据容错的核心组成部分。

```typescript
// 回滚管理器
class RollbackManager {
  private history: Map<string, VersionedValue[]> = new Map();
  private maxHistorySize = 50;

  // 保存值的历史版本
  saveVersion(key: string, value: any): void {
    const versions = this.history.get(key) || [];

    versions.push({
      value: this.deepClone(value),
      timestamp: Date.now()
    });

    // 限制历史版本数量
    if (versions.length > this.maxHistorySize) {
      versions.shift();
    }

    this.history.set(key, versions);
  }

  // 获取上一个版本
  getPreviousVersion(key: string): any | null {
    const versions = this.history.get(key);
    if (!versions || versions.length < 2) return null;

    // 返回倒数第二个版本（当前版本是最后一个）
    return versions[versions.length - 2].value;
  }

  // 执行回滚
  rollback(key: string): { success: boolean; value?: any; error?: string } {
    const previousValue = this.getPreviousVersion(key);

    if (previousValue === null) {
      return {
        success: false,
        error: `没有找到 ${key} 的历史版本`
      };
    }

    // 从历史中移除最新版本
    const versions = this.history.get(key)!;
    versions.pop();

    return {
      success: true,
      value: previousValue
    };
  }

  // 深度克隆
  private deepClone<T>(value: T): T {
    return JSON.parse(JSON.stringify(value));
  }
}

// 带回滚的数据操作
class RollbackableData<T> {
  private data: T;
  private rollbackManager: RollbackManager;
  private key: string;

  constructor(key: string, initialData: T) {
    this.key = key;
    this.data = initialData;
    this.rollbackManager = new RollbackManager();
    this.rollbackManager.saveVersion(key, initialData);
  }

  // 更新数据
  update(updater: (current: T) => T): { success: boolean; error?: Error } {
    try {
      const previousData = this.data;
      const newData = updater(this.data);

      // 保存当前版本到历史
      this.rollbackManager.saveVersion(this.key, previousData);
      this.data = newData;

      return { success: true };
    } catch (error) {
      return { success: false, error: error as Error };
    }
  }

  // 回滚到上一个版本
  rollback(): { success: boolean; error?: string } {
    const result = this.rollbackManager.rollback(this.key);

    if (result.success) {
      this.data = result.value!;
    }

    return result;
  }

  // 获取当前数据
  getData(): T {
    return this.data;
  }
}
```

### 7.3 本地缓存

本地缓存可以在服务端数据不可用时提供降级数据，是数据容错的重要组成部分。

```typescript
// 本地缓存管理器
class LocalCacheManager {
  private prefix = 'app_cache_';
  private defaultTTL = 3600000; // 1 小时

  // 设置缓存
  set<T>(key: string, value: T, ttl?: number): void {
    const cacheItem: CacheItem<T> = {
      value,
      expiry: Date.now() + (ttl || this.defaultTTL)
    };

    try {
      localStorage.setItem(
        `${this.prefix}${key}`,
        JSON.stringify(cacheItem)
      );
    } catch (error) {
      // 如果 localStorage 满了，尝试清理
      this.cleanup();
      try {
        localStorage.setItem(
          `${this.prefix}${key}`,
          JSON.stringify(cacheItem)
        );
      } catch {
        console.error('本地缓存写入失败:', error);
      }
    }
  }

  // 获取缓存
  get<T>(key: string): T | null {
    try {
      const item = localStorage.getItem(`${this.prefix}${key}`);
      if (!item) return null;

      const cacheItem: CacheItem<T> = JSON.parse(item);

      // 检查是否过期
      if (Date.now() > cacheItem.expiry) {
        this.remove(key);
        return null;
      }

      return cacheItem.value;
    } catch {
      return null;
    }
  }

  // 删除缓存
  remove(key: string): void {
    localStorage.removeItem(`${this.prefix}${key}`);
  }

  // 清空所有缓存
  clear(): void {
    const keys = this.getAllKeys();
    keys.forEach(key => localStorage.removeItem(key));
  }

  // 获取所有缓存 key
  private getAllKeys(): string[] {
    const keys: string[] = [];
    for (let i = 0; i < localStorage.length; i++) {
      const key = localStorage.key(i);
      if (key?.startsWith(this.prefix)) {
        keys.push(key.slice(this.prefix.length));
      }
    }
    return keys;
  }

  // 清理过期缓存
  private cleanup(): void {
    const keys = this.getAllKeys();
    keys.forEach(key => {
      this.get(key); // 如果过期会自动删除
    });
  }
}

interface CacheItem<T> {
  value: T;
  expiry: number;
}

// 创建缓存实例
const localCache = new LocalCacheManager();

// 缓存装饰器
function cached<T extends (...args: any[]) => any>(
  keyGenerator: (...args: Parameters<T>) => string,
  ttl?: number
): (target: any, propertyKey: string, descriptor: PropertyDescriptor) => void {
  return function(
    target: any,
    propertyKey: string,
    descriptor: PropertyDescriptor
  ) {
    const originalMethod = descriptor.value;

    descriptor.value = function(...args: Parameters<T>) {
      const cacheKey = keyGenerator(...args);
      const cachedValue = localCache.get<ReturnType<T>>(cacheKey);

      if (cachedValue !== null) {
        return cachedValue;
      }

      const result = originalMethod.apply(this, args);

      // 如果是 Promise，等待结果后缓存
      if (result instanceof Promise) {
        return result.then((value: ReturnType<T>) => {
          localCache.set(cacheKey, value, ttl);
          return value;
        });
      }

      localCache.set(cacheKey, result, ttl);
      return result;
    };
  };
}

// 使用示例
class UserService {
  @cached((userId: string) => `user_${userId}`, 300000) // 5 分钟缓存
  async getUser(userId: string): Promise<User> {
    const response = await fetch(`/api/users/${userId}`);
    return response.json();
  }
```

```typescript
// 数据校验器
class DataValidator {
  // 校验数据类型
  static validateType(value: any, expectedType: string): boolean {
    return typeof value === expectedType;
  }

  // 校验必需字段
  static validateRequired(obj: Record<string, any>, requiredFields: string[]): ValidationResult {
    const missing: string[] = [];

    for (const field of requiredFields) {
      if (obj[field] === undefined || obj[field] === null || obj[field] === '') {
        missing.push(field);
      }
    }

    return {
      valid: missing.length === 0,
      errors: missing.map(field => ({
        field,
        message: `字段 ${field} 是必需的`
      }))
    };
  }

  // 校验数据范围
  static validateRange(
    value: number,
    options: { min?: number; max?: number }
  ): ValidationResult {
    const errors: ValidationError[] = [];

    if (options.min !== undefined && value < options.min) {
      errors.push({
        field: 'value',
        message: `值不能小于 ${options.min}`
      });
    }

    if (options.max !== undefined && value > options.max) {
      errors.push({
        field: 'value',
        message: `值不能大于 ${options.max}`
      });
    }

    return {
      valid: errors.length === 0,
      errors
    };
  }

  // 校验字符串格式
  static validateFormat(
    value: string,
    pattern: RegExp
  ): ValidationResult {
    if (!pattern.test(value)) {
      return {
        valid: false,
        errors: [{
          field: 'value',
          message: '格式不正确'
        }]
      };
    }

    return { valid: true, errors: [] };
  }

  // 校验复杂对象
  static validateObject<T extends Record<string, any>>(
    obj: T,
    schema: ValidationSchema
  ): ValidationResult {
    const allErrors: ValidationError[] = [];

    for (const [field, rules] of Object.entries(schema)) {
      const value = obj[field];

      // 必需检查
      if (rules.required && (value === undefined || value === null)) {
        allErrors.push({ field, message: `字段 ${field} 是必需的` });
        continue;
      }

      // 类型检查
      if (value !== undefined && value !== null && rules.type) {
        if (!this.validateType(value, rules.type)) {
          allErrors.push({ field, message: `字段 ${field} 类型错误` });
        }
      }

      // 范围检查
      if (rules.min !== undefined || rules.max !== undefined) {
        const rangeResult = this.validateRange(value, {
          min: rules.min,
          max: rules.max
        });
        allErrors.push(...rangeResult.errors.map(e => ({ ...e, field })));
      }

      // 格式检查
      if (rules.pattern && value !== undefined && value !== null) {
        if (!this.validateFormat(String(value), rules.pattern).valid) {
          allErrors.push({ field, message: `字段 ${field} 格式不正确` });
        }
      }

      // 自定义校验
      if (rules.validator && value !== undefined) {
        const customResult = rules.validator(value);
        if (!customResult.valid) {
          allErrors.push(...customResult.errors.map(e => ({ ...e, field })));
        }
      }
    }

    return {
      valid: allErrors.length === 0,
      errors: allErrors
    };
  }
}

interface ValidationResult {
  valid: boolean;
  errors: ValidationError[];
}

interface ValidationError {
  field: string;
  message: string;
}

interface ValidationSchema {
  [field: string]: {
    required?: boolean;
    type?: string;
    min?: number;
    max?: number;
    pattern?: RegExp;
    validator?: (value: any) => ValidationResult;
  };
}
```

```typescript
// 完整数据容错层
class DataFaultToleranceLayer<T> {
  private cache: LocalCacheManager;
  private rollbackManager: RollbackManager;
  private optimisticManager: OptimisticUpdateManager;

  private data: T;
  private key: string;
  private syncFn?: (data: T) => Promise<void>;

  constructor(
    key: string,
    initialData: T,
    options: {
      cacheTTL?: number;
      syncFn?: (data: T) => Promise<void>;
    } = {}
  ) {
    this.key = key;
    this.data = initialData;
    this.syncFn = options.syncFn;
    this.cache = new LocalCacheManager();
    this.rollbackManager = new RollbackManager();
    this.optimisticManager = new OptimisticUpdateManager();

    // 加载缓存数据作为降级
    this.loadFromCache();
  }

  // 从缓存加载
  private loadFromCache(): void {
    const cached = this.cache.get<T>(`data_${this.key}`);
    if (cached !== null) {
      this.data = cached;
    }
  }

  // 乐观更新
  async optimisticUpdate(
    updater: (current: T) => T,
    rollback?: () => void
  ): Promise<{ success: boolean; error?: Error }> {
    const previousData = this.data;

    // 创建乐观更新
    const updateId = this.optimisticManager.addUpdate({
      type: 'data-update',
      previousValue: previousData,
      newValue: updater(previousData),
      rollback: () => {
        this.data = previousData;
        rollback?.();
      }
    });

    try {
      // 立即更新本地数据
      this.data = updater(previousData);

      // 保存版本用于回滚
      this.rollbackManager.saveVersion(this.key, previousData);

      // 同步到服务器
      if (this.syncFn) {
        await this.syncFn(this.data);
      }

      // 确认更新
      this.optimisticManager.confirmUpdate(updateId);

      // 更新缓存
      this.cache.set(`data_${this.key}`, this.data);

      return { success: true };
    } catch (error) {
      // 回滚
      this.optimisticManager.rollbackUpdate(updateId, error as Error);
      this.data = previousData;

      return { success: false, error: error as Error };
    }
  }

  // 纯回滚
  rollback(): { success: boolean; error?: string } {
    const result = this.rollbackManager.rollback(this.key);

    if (result.success) {
      this.data = result.value!;
      this.cache.set(`data_${this.key}`, this.data);
    }

    return result;
  }

  // 获取当前数据
  getData(): T {
    return this.data;
  }

  // 获取数据副本
  getDataSnapshot(): T {
    return JSON.parse(JSON.stringify(this.data));
  }
}

  const userData = new DataFaultToleranceLayer<User>(
    'current_user',
    { id: '', name: '', email: '' },
    {

        await fetch('/api/users/current', {
          method: 'PUT',
          body: JSON.stringify(data)
        });
      }
    }
  );

  // 乐观更新用户名
  const result = await userData.optimisticUpdate(
    user => ({ ...user, name: 'New Name' }),
    () => console.log('回滚到旧名称')
  );

  if (!result.success) {
    console.error('更新失败:', result.error);
  }
}
```

### 8.2 请求重试

网络请求的重试需要考虑网络状态，避免在弱网环境下过度重试。

```typescript
// 智能请求重试器
class SmartRetryHandler {
  private maxRetries = 3;
  private baseDelay = 1000;
  private maxDelay = 10000;

  // 根据网络状况调整重试策略
  async executeWithRetry<T>(
    fn: () => Promise<T>,
    options: {
      retries?: number;
      networkAware?: boolean;
      shouldRetry?: (error: any) => boolean;
    } = {}
  ): Promise<T> {
    const {
      retries = this.maxRetries,
      networkAware = true,
      shouldRetry = () => true
    } = options;

    let lastError: Error;
    let delay = this.baseDelay;

    for (let attempt = 0; attempt <= retries; attempt++) {
      try {
        return await fn();
      } catch (error) {
        lastError = error as Error;

        // 检查是否应该重试
        if (attempt === retries || !shouldRetry(error)) {
          throw lastError;
        }

        // 计算延迟时间
        let currentDelay = delay;

        // 如果启用网络感知，根据网络状况调整延迟
        if (networkAware) {
          const networkStatus = navigator.onLine;
          const connection = (navigator as any).connection;
          const isSlowNetwork = connection?.effectiveType === 'slow-2g' || connection?.effectiveType === '2g';

          if (!networkStatus || isSlowNetwork) {
            // 弱网环境下增加延迟
            currentDelay = Math.min(delay * 3, this.maxDelay);
          }
        }

        console.log(`请求失败，${currentDelay}ms 后第 ${attempt + 1} 次重试...`);
        await this.sleep(currentDelay);

        delay = Math.min(delay * 2, this.maxDelay);
      }
    }

    throw lastError!;
  }

  private sleep(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
}
```

```typescript
class CDNFailoverManager {
  private cdnDomains: string[];

  constructor(cdnDomains: string[]) {

// CDN 资源加载器
class CDNResourceLoader {
  private failOverManager: CDNFailoverManager;
  private resourceCache: Map<string, boolean> = new Map();

  constructor(cdnDomains: string[]) {
    this.failOverManager = new CDNFailoverManager(cdnDomains);
  }

  // 加载脚本
  async loadScript(src: string): Promise<void> {
    const url = this.failOverManager.buildURL(src);

    try {
      await this.loadScriptWithTimeout(url);
      this.resourceCache.set(src, true);
    } catch (error) {
      // 尝试容灾
      console.warn(`CDN ${this.failOverManager.getCurrentDomain()} 加载失败，尝试容灾...`);
      const failoverDomain = this.failOverManager.failover();

      try {
        await this.loadScriptWithTimeout(`${failoverDomain}${src}`);
        this.resourceCache.set(src, true);
      } catch (retryError) {
        console.error(`CDN 资源加载最终失败: ${src}`);
        throw retryError;
      }
    }
  }

  private loadScriptWithTimeout(src: string, timeout = 10000): Promise<void> {
    return new Promise((resolve, reject) => {
      const script = document.createElement('script');
      script.src = src;
      script.async = true;

      const timeoutId = setTimeout(() => {
        reject(new Error(`脚本加载超时: ${src}`));
      }, timeout);

      script.onload = () => {
        clearTimeout(timeoutId);
        resolve();
      };

      script.onerror = () => {
        clearTimeout(timeoutId);
        reject(new Error(`脚本加载失败: ${src}`));
      };

      document.head.appendChild(script);
    });
  }
}
```

```typescript
// 网络容错请求钩子
function useNetworkFaultTolerantRequest<T>(
  requestFn: () => Promise<T>,
  options: {
    offlineFallback?: T;
    retryCount?: number;
    cacheDuration?: number;
  } = {}
) {
  const { offlineFallback, retryCount = 3, cacheDuration } = options;

  const [state, setState] = useState<RequestState<T>>({
    loading: false,
    data: null,
    error: null
  });

  const networkStatus = useNetworkStatus();
  const cache = useRef(new Map<string, T>());

  const execute = useCallback(async () => {
    setState(prev => ({ ...prev, loading: true, error: null }));

    // 检查缓存
    const cacheKey = requestFn.toString();
    const cached = cache.current.get(cacheKey);
    if (cached && cacheDuration) {
      const cachedTime = (cache as any).get(`${cacheKey}_time`) || 0;
      if (Date.now() - cachedTime < cacheDuration) {
        setState({ loading: false, data: cached, error: null });
        return cached;
      }
    }

    // 离线环境
    if (!networkStatus.online) {
      if (offlineFallback !== undefined) {
        setState({ loading: false, data: offlineFallback, error: null });
        return offlineFallback;
      }
      setState({
        loading: false,
        data: null,
        error: new Error('网络已断开')
      });
      return;
    }

    // 带重试的执行
    const retryHandler = new SmartRetryHandler();

    try {
      const data = await retryHandler.executeWithRetry(
        requestFn,
        { retries: retryCount }
      );

      // 更新缓存
      cache.current.set(cacheKey, data);
      (cache as any).set(`${cacheKey}_time`, Date.now());

      setState({ loading: false, data, error: null });
      return data;
    } catch (error) {
      setState({
        loading: false,
        data: null,
        error: error as Error
      });
      throw error;
    }
  }, [requestFn, offlineFallback, retryCount, cacheDuration, networkStatus.online]);

  return {
    ...state,
    execute,
    refetch: execute
  };
}
```

### 9.2 自建错误监控

对于需要完全控制数据的企业，自建错误监控系统是必要的选择。

```typescript
// 自建错误监控系统
class SelfHostedErrorMonitor {
  private endpoint: string;
  private queue: ErrorPayload[] = [];
  private flushInterval = 5000;
  private maxQueueSize = 100;

  constructor(endpoint: string) {
    this.endpoint = endpoint;
    this.startFlushTimer();
  }

  // 捕获错误
  capture(error: Error | unknown, context?: ErrorContext): void {
    const payload = this.buildPayload(error, context);
    this.queue.push(payload);

    if (this.queue.length >= this.maxQueueSize) {
      this.flush();
    }
  }

  // 构建错误 payload
  private buildPayload(error: Error | unknown, context?: ErrorContext): ErrorPayload {
    return {
      id: this.generateId(),
      timestamp: Date.now(),
      type: error instanceof Error ? error.name : 'UnknownError',
      message: error instanceof Error ? error.message : String(error),
      stack: error instanceof Error ? error.stack : undefined,
      context: {
        url: window.location.href,
        userAgent: navigator.userAgent,
        viewport: `${window.innerWidth}x${window.innerHeight}`,
        ...context
      },
      environment: {
        appVersion: process.env.APP_VERSION || 'unknown',
        buildNumber: process.env.GIT_COMMIT || 'unknown'
      }
    };
  }

  // 发送错误到服务器
  private async flush(): Promise<void> {
    if (this.queue.length === 0) return;

    const payload = [...this.queue];
    this.queue = [];

    try {
      await fetch(this.endpoint, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ errors: payload }),
        keepalive: true
      });
    } catch (error) {
      // 发送失败，放回队列
      this.queue.unshift(...payload);
      console.error('错误上报失败:', error);
    }
  }

  // 启动定时发送
  private startFlushTimer(): void {
    setInterval(() => this.flush(), this.flushInterval);
  }

  private generateId(): string {
    return `${Date.now()}_${Math.random().toString(36).substr(2, 9)}`;
  }
}
```

```typescript
      error: error ? {
        name: error.name,
        message: error.message,
        stack: error.stack
      } : undefined
    });
  }

  // 订阅日志

    this.listeners.push(listener);
    return () => {
      this.listeners = this.listeners.filter(l => l !== listener);
    };
  }

      try {
        listener(log);
      } catch (e) {
        console.error('日志监听器执行失败:', e);
      }
    });
  }

    level?: LogLevel;
    source?: string;
    startTime?: number;
    endTime?: number;
    limit?: number;

    }
    if (options?.endTime) {
      filtered = filtered.filter(log => log.timestamp <= options.endTime!);
    }

    if (options?.limit) {
      filtered = filtered.slice(-options.limit);
    }

    return filtered;
  }

  // 清空日志
  clear(): void {
    this.logs = [];
  }

  private generateId(): string {
    return `log_${Date.now()}_${Math.random().toString(36).substr(2, 6)}`;
  }
```

### 9.4 链路追踪

链路追踪用于追踪一个请求在整个系统中的完整路径，帮助定位性能瓶颈和故障点。

```typescript
// 链路追踪
class TraceManager {
  private traces: Map<string, Trace> = new Map();
  private currentTraceId: string | null = null;

  // 开始追踪
  startTrace(name: string): string {
    const traceId = `trace_${Date.now()}_${Math.random().toString(36).substr(2, 6)}`;
    const trace: Trace = {
      id: traceId,
      name,
      startTime: Date.now(),
      spans: []
    };

    this.traces.set(traceId, trace);
    this.currentTraceId = traceId;

    return traceId;
  }

  // 结束追踪
  endTrace(traceId?: string): Trace | null {
    const id = traceId || this.currentTraceId;
    if (!id) return null;

    const trace = this.traces.get(id);
    if (!trace) return null;

    trace.endTime = Date.now();
    trace.duration = trace.endTime - trace.startTime;
    this.currentTraceId = null;

    return trace;
  }

  // 添加跨度
  addSpan(traceId: string | null, name: string, fn: () => any): any {
    const id = traceId || this.currentTraceId;
    if (!id) return fn();

    const span: Span = {
      id: `span_${Date.now()}_${Math.random().toString(36).substr(2, 6)}`,
      name,
      startTime: Date.now()
    };

    try {
      const result = fn();
      span.endTime = Date.now();
      span.duration = span.endTime - span.startTime;
      span.status = 'ok';
      return result;
    } catch (error) {
      span.endTime = Date.now();
      span.duration = span.endTime - span.startTime;
      span.status = 'error';
      span.error = error as Error;
      throw error;
    } finally {
      const trace = this.traces.get(id);
      if (trace) {
        trace.spans.push(span);
      }
    }
  }

  // 获取追踪结果
  getTrace(traceId: string): Trace | null {
    return this.traces.get(traceId) || null;
  }

  // 获取所有追踪
  getAllTraces(): Trace[] {
    return Array.from(this.traces.values());
  }
}

  endTime?: number;
  duration?: number;
  spans: Span[];
}
```

```typescript
class ImageDegradation {
  // 创建带降级的图片
  static createDegradableImage(
    src: string,
    options: {
      fallbackSrc?: string;
      placeholder?: string;
      onError?: (img: HTMLImageElement) => void;

        img.src = fallbackSrc;

        img.src = placeholder;
      }

  }

    src: string;
    fallbackSrc?: string;
    placeholder?: string;
    alt: string;
    className?: string;

    const [hasError, setHasError] = useState(false);

    useEffect(() => {
      setCurrentSrc(placeholder || '');
      setHasError(false);

      const img = new Image();
      img.src = src;

      img.onload = () => setCurrentSrc(src);
      img.onerror = () => {
        setHasError(true);
        if (fallbackSrc && src !== fallbackSrc) {
          setCurrentSrc(fallbackSrc);
        }
      };
    }, [src, fallbackSrc, placeholder]);

      <img
        src={currentSrc}
        alt={alt}
        className={className}
        data-error={hasError}
      />
    );
```

### 10.3 功能开关

功能开关允许在运行时动态启用或禁用某些功能，是容灾的重要手段。

```typescript
// 功能开关管理器
class FeatureFlagManager {
  private flags: Map<string, FeatureFlag> = new Map();
  private listeners: Map<string, Set<() => void>> = new Map();

  constructor() {
    this.loadDefaultFlags();
  }

  // 加载默认开关
  private loadDefaultFlags(): void {
    this.flags.set('dark_mode', {
      key: 'dark_mode',
      enabled: false,
      description: '深色模式'
    });

    this.flags.set('new_editor', {
      key: 'new_editor',
      enabled: true,
      description: '新编辑器'
    });

    this.flags.set('video_player', {
      key: 'video_player',
      enabled: true,
      description: '视频播放器'
    });
  }

  // 获取开关状态
  isEnabled(key: string): boolean {
    const flag = this.flags.get(key);
    return flag?.enabled ?? false;
  }

  // 启用开关
  enable(key: string): void {
    const flag = this.flags.get(key);
    if (flag) {
      flag.enabled = true;
      this.notifyListeners(key);
    }
  }

  // 禁用开关
  disable(key: string): void {
    const flag = this.flags.get(key);
    if (flag) {
      flag.enabled = false;
      this.notifyListeners(key);
    }
  }

  // 批量更新开关
  updateFlags(updates: Record<string, boolean>): void {
    Object.entries(updates).forEach(([key, enabled]) => {
      const flag = this.flags.get(key);
      if (flag) {
        flag.enabled = enabled;
      }
    });
    // 通知所有监听器
    Object.keys(updates).forEach(key => this.notifyListeners(key));
  }

  // 订阅开关变化
  subscribe(key: string, callback: () => void): () => void {
    const listeners = this.listeners.get(key) || new Set();
    listeners.add(callback);
    this.listeners.set(key, listeners);

    return () => listeners.delete(callback);
  }

  private notifyListeners(key: string): void {
    const listeners = this.listeners.get(key);
    if (listeners) {
      listeners.forEach(cb => cb());
    }
  }

  // 从服务器加载开关
  async loadFromServer(endpoint: string): Promise<void> {
    try {
      const response = await fetch(endpoint);
      const data = await response.json();
      this.updateFlags(data.flags);
    } catch (error) {
      console.error('加载功能开关失败:', error);
    }
  }
}

interface FeatureFlag {
  key: string;
  enabled: boolean;
  description: string;
}

// React Hook for 功能开关
function useFeatureFlag(key: string): boolean {
  const [enabled, setEnabled] = useState(() => featureFlagManager.isEnabled(key));

  useEffect(() => {
    return featureFlagManager.subscribe(key, () => {
      setEnabled(featureFlagManager.isEnabled(key));
    });
  }, [key]);

  return enabled;
}
```

### 10.4 灰度发布

灰度发布允许在正式发布前先让一小部分用户试用新功能，降低风险。

```typescript
// 灰度发布管理器
class灰度发布Manager {
  private rules: GrayReleaseRule[] = [];

  // 添加灰度规则
  addRule(rule: GrayReleaseRule): void {
    this.rules.push(rule);
  }

  // 检查用户是否在灰度范围内
  isInGrayRelease(featureKey: string, userId: string): boolean {
    const rule = this.rules.find(r => r.featureKey === featureKey);
    if (!rule) return false;

    // 基于用户 ID 哈希
    const hash = this.hashUserId(userId);
    const percentage = hash % 100;

    return percentage < rule.percentage;
  }

  // 获取用户 ID 哈希
  private hashUserId(userId: string): number {
    let hash = 0;
    for (let i = 0; i < userId.length; i++) {
      const char = userId.charCodeAt(i);
      hash = ((hash << 5) - hash) + char;
      hash = hash & hash;
    }
    return Math.abs(hash);
  }

  // 获取灰度用户的百分比
  getGrayPercentage(featureKey: string): number {
    const rule = this.rules.find(r => r.featureKey === featureKey);
    return rule?.percentage || 0;
  }
}

interface GrayReleaseRule {
  featureKey: string;
  percentage: number; // 0-100
  startTime?: number;
  endTime?: number;
  condition?: (userId: string) => boolean;
}

function useGrayRelease(featureKey: string, userId?: string): boolean {
  const grayManager = useRef(new GrayReleaseManager());

  return useMemo(() => {
    if (!userId) return false;
```

### 10.5 我的思考：容灾是系统的底线

容灾不是锦上添花，而是系统的最后一道防线。即使在所有服务都正常的情况下，容灾机制也应该存在，因为它能够在极端情况下保护用户数据和体验。

**容灾设计的核心原则：**

1. **冗余设计**：关键系统必须有备份，不能存在单点故障
2. **优雅降级**：当主系统不可用时，能够自动切换到降级方案
3. **快速恢复**：问题发生后能够快速恢复正常服务
4. **可观测性**：能够及时发现问题，而不是等到用户投诉

**容灾的成本与收益平衡：**

容灾系统本身也会带来成本，包括开发成本、运维成本和资源成本。因此，需要根据业务的重要性和风险程度来合理设计容灾方案。对于核心业务功能，应该投入更多资源建设容灾；对于边缘功能，可以采用简单的降级策略。

### 11.4 购物车容错

购物车是电商的核心功能，需要特别可靠的容错处理。

```typescript
// 购物车容错管理
class CartFaultTolerance {
  private optimisticManager = new OptimisticUpdateManager();
  private rollbackManager = new RollbackManager();

  // 购物车状态
  private cart: Cart = {
    items: [],
    totalAmount: 0,
    itemCount: 0
  };

  // 添加商品（乐观更新）
  async addItem(
    productId: string,
    quantity: number,
    price: number
  ): Promise<{ success: boolean; error?: string }> {
    const previousCart = this.deepClone(this.cart);
    const itemId = `item_${Date.now()}`;

    // 乐观更新：立即添加
    this.cart.items.push({
      id: itemId,
      productId,
      quantity,
      price,
      subtotal: price * quantity,
      status: 'pending'
    });
    this.recalculateTotal();

    // 保存版本用于回滚
    this.rollbackManager.saveVersion('cart', previousCart);

    try {
      // 调用 API
      const result = await this.syncToServer();

      if (result.success) {
        // 确认更新
        const item = this.cart.items.find(i => i.id === itemId);
        if (item) {
          item.status = 'confirmed';
          item.serverId = result.itemId;
        }

        // 保存到本地缓存
        localCache.set('cart', this.cart);

        return { success: true };
      } else {
        throw new Error(result.error);
      }
    } catch (error) {
      // 回滚
      this.cart = previousCart;
      return {
        success: false,
        error: error instanceof Error ? error.message : '添加失败'
      };
    }
  }

  // 更新数量
  async updateQuantity(itemId: string, quantity: number): Promise<void> {
    const previousCart = this.deepClone(this.cart);

    // 乐观更新
    const item = this.cart.items.find(i => i.id === itemId);
    if (item) {
      item.quantity = quantity;
      item.subtotal = item.price * quantity;
      this.recalculateTotal();
    }

    this.rollbackManager.saveVersion('cart', previousCart);

    try {
      await this.syncToServer();
      localCache.set('cart', this.cart);
    } catch {
      // 回滚
      this.cart = previousCart;
    }
  }

  // 删除商品
  async removeItem(itemId: string): Promise<void> {
    const previousCart = this.deepClone(this.cart);

    // 乐观更新
    this.cart.items = this.cart.items.filter(i => i.id !== itemId);
    this.recalculateTotal();

    this.rollbackManager.saveVersion('cart', previousCart);

    try {
      await this.syncToServer();
      localCache.set('cart', this.cart);
    } catch {
      // 回滚
      this.cart = previousCart;
    }
  }

  // 同步到服务器
  private async syncToServer(): Promise<{ success: boolean; itemId?: string; error?: string }> {
    try {
      const response = await fetch('/api/cart/sync', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(this.cart)
      });

      if (!response.ok) {
        throw new Error('同步失败');
      }

      return await response.json();
    } catch (error) {
      throw error;
    }
  }

  // 重试未同步的项
  async retryPendingItems(): Promise<void> {
    const pendingItems = this.cart.items.filter(i => i.status === 'pending');

    for (const item of pendingItems) {
      try {
        await this.syncToServer();
        item.status = 'confirmed';
      } catch {
        // 保持 pending 状态
      }
    }
  }

  // 从本地缓存恢复
  restoreFromCache(): void {
    const cached = localCache.get<Cart>('cart');
    if (cached) {
      this.cart = cached;
    }
  }

  private recalculateTotal(): void {
    this.cart.totalAmount = this.cart.items.reduce(
      (sum, item) => sum + item.subtotal,
      0
    );
    this.cart.itemCount = this.cart.items.reduce(
      (count, item) => count + item.quantity,
      0
    );
  }

  private deepClone<T>(obj: T): T {
    return JSON.parse(JSON.stringify(obj));
  }
}

interface Cart {
  items: CartItem[];
  totalAmount: number;
  itemCount: number;
}

interface CartItem {
  id: string;
  serverId?: string;
  productId: string;
  quantity: number;
  price: number;
  subtotal: number;
  status: 'pending' | 'confirmed' | 'error';
}
```

### 11.5 我的思考：用户体验是第一位

电商容灾的核心目标是保障用户体验。即使在系统出现故障时，我们也应该让用户能够继续浏览和购物，而不是直接显示"系统维护"。

**关键原则：**

1. **永远不让用户看到空白页面**：即使所有数据都加载失败，也要显示有意义的占位内容
2. **乐观更新提升感知性能**：用户操作应该立即得到反馈，而不是等待服务器响应
3. **降级要彻底**：如果某个功能不可用，要有完整的降级方案，而不是半成品
4. **错误消息要友好**：告诉用户发生了什么，能做什么，而不是显示技术术语

**容灾的反面案例：**

- 显示"加载中..."然后一直转圈
- 显示"系统错误"没有任何操作选项
- 降价功能不可用时让用户无法下单
- 图片加载失败时留下破碎的图片图标

### 12.2 告警阈值

合理的告警阈值可以确保关键问题得到及时处理，同时避免告警疲劳。

```typescript
// 告警阈值配置
const AlertThresholds = {
  // 错误率告警（5分钟内）
  errorRate: {
    warning: 0.01,    // 1% 错误率
    critical: 0.05   // 5% 错误率
  },

  // 响应时间告警
  responseTime: {
    p95: {
      warning: 500,   // P95 响应时间超过 500ms
      critical: 1000  // P95 响应时间超过 1s
    },
    p99: {
      warning: 1000,
      critical: 2000
    }
  },

  // 错误数量告警（5分钟内）
  errorCount: {
    warning: 10,
    critical: 50
  },

  // 特定错误类型告警
  specificErrors: {
    // 致命错误立即告警
    fatalError: {
      threshold: 1,
      window: 60000,  // 1 分钟内任何致命错误
      cooldown: 300000 // 冷却 5 分钟
    },

    // JS 运行时错误
    runtimeError: {
      threshold: 5,
      window: 60000,
      cooldown: 300000
    },

    // API 错误
    apiError: {
      threshold: 20,
      window: 60000,
      cooldown: 600000
    }
  }
};

// 告警评估器
class AlertEvaluator {
  evaluateMetrics(metrics: SystemMetrics): Alert[] {
    const alerts: Alert[] = [];

    // 检查错误率
    if (metrics.errorRate >= AlertThresholds.errorRate.critical) {
      alerts.push({
        level: AlertLevel.CRITICAL,
        message: `错误率过高: ${(metrics.errorRate * 100).toFixed(2)}%`,
        metric: 'errorRate'
      });
    } else if (metrics.errorRate >= AlertThresholds.errorRate.warning) {
      alerts.push({
        level: AlertLevel.WARNING,
        message: `错误率偏高: ${(metrics.errorRate * 100).toFixed(2)}%`,
        metric: 'errorRate'
      });
    }

    // 检查响应时间
    if (metrics.p95ResponseTime >= AlertThresholds.responseTime.p95.critical) {
      alerts.push({
        level: AlertLevel.CRITICAL,
        message: `P95 响应时间过长: ${metrics.p95ResponseTime}ms`,
        metric: 'p95ResponseTime'
      });
    }

    return alerts;
  }
}
```

### 12.3 快速响应

快速响应机制确保问题发生后能够及时处理。

```typescript
// 快速响应流程
class IncidentResponse {
  // 创建事故
  static createIncident(alert: Alert): Incident {
    return {
      id: `inc_${Date.now()}`,
      alertId: alert.id,
      level: alert.level,
      status: 'created',
      createdAt: Date.now(),
      assignee: null,
      timeline: [
        {
          event: 'incident_created',
          timestamp: Date.now(),
          description: '事故已创建'
        }
      ]
    };
  }

  // 响应升级
  static escalate(incident: Incident): Incident {
    const escalationLevels = ['L1', 'L2', 'L3', 'on-call'];
    const currentIndex = escalationLevels.indexOf(incident.escalationLevel || 'L1');
    const nextLevel = escalationLevels[Math.min(currentIndex + 1, escalationLevels.length - 1)];

    return {
      ...incident,
      escalationLevel: nextLevel,
      timeline: [
        ...incident.timeline,
        {
          event: 'escalated',
          timestamp: Date.now(),
          description: `升级到 ${nextLevel}`
        }
      ]
    };
  }

  // 自动通知
  static async notifyStakeholders(incident: Incident): Promise<void> {
    const channels = [];

    if (incident.level === 'critical') {
      channels.push('sms', 'phone'); // 严重问题需要电话通知
    } else {
      channels.push('email', 'slack');
    }

    for (const channel of channels) {
      await this.sendNotification(channel, incident);
    }
  }
}
```

### 12.5 我的思考：错误是最好的老师

在软件工程中，错误和故障是不可避免的，但我们应该把它们视为学习和改进的机会，而不是单纯的负面事件。

**错误管理的正确心态：**

1. **错误是正常现象**：没有任何系统是完美无缺的，错误是软件生命周期的一部分
2. **从错误中学习**：每次错误都应该有复盘，找出根因并改进
3. **预防胜于治疗**：建立完善的监控和预警机制，在问题发生前发现苗头
4. **容灾是底线**：即使预防措施失效，容灾机制也能保证用户基本体验

**持续改进的循环：**

```
监控发现异常 → 告警通知 → 快速响应 → 问题修复 → 复盘总结 → 改进预防措施
```

这个循环不断迭代，系统会变得越来越稳定，用户的体验也会越来越好。

4. **告警要合理**：避免告警过多或过少，确保关键问题得到及时处理
5. **容错是底线**：通过重试、降级、熔断等机制，在故障时保证核心功能可用
6. **降级要彻底**：从 UI、数据、网络等多个层面提供降级方案
