# 前端性能优化完全指南

前端性能优化是提升用户体验、降低跳出率、提高转化率的关键技术手段。本指南将从页面加载性能、运行时性能、React性能优化、网络性能、性能监控与度量、实战案例六个维度，全面深入地讲解前端性能优化的原理、方法与最佳实践。

---

## 一、页面加载性能

页面加载性能直接影响用户的第一印象和留存率。根据Google的研究，当页面加载时间超过3秒时，超过50%的用户会选择离开。因此，优化页面加载性能是前端开发的首要任务。

### 1.1 关键渲染路径（Critical Rendering Path）

关键渲染路径是指浏览器从接收HTML文档到将像素绘制到屏幕所经历的步骤序列。理解这一过程是优化页面加载性能的基础。

**关键渲染路径的七个步骤：**

```
1. 构建DOM树（Parsing HTML）→ 2. 构建CSSOM树（Parsing CSS）
   → 3. 执行JavaScript（JavaScript Execution）
   → 4. 生成渲染树（Render Tree）
   → 5. 布局计算（Layout）
   → 6. 分层绘制（Paint）
   → 7. 合成渲染（Composite）
```

**DOM树构建过程：**

浏览器从HTML文档的顶部开始，自上而下解析HTML标签。每当遇到`<script>`标签时，会暂停DOM构建，执行JavaScript代码，然后再继续。这是因为JavaScript可能会修改DOM结构。

```html
<!-- HTML解析被script阻塞的示例 -->
<html>
<head>
    <!-- 在这里浏览器开始解析HTML -->
</head>
<body>
    <div id="app"></div>
    <!-- 遇到script标签，DOM构建暂停 -->
    <script src="large-library.js"></script>
    <!-- script执行完毕后，DOM构建继续 -->
    <div class="content">
        <!-- 后续内容 -->
    </div>
</body>
</html>
```

**CSSOM树构建的特点：**

CSSOM（CSS Object Model）的构建是**渲染阻塞资源**（render-blocking）。与HTML不同，CSS必须完全解析后才能生成CSSOM，因为后续的渲染树生成依赖完整的CSS规则。

```html
<!-- CSS是渲染阻塞资源 -->
<head>
    <!-- 浏览器必须等待所有CSS加载并解析完成 -->
    <link rel="stylesheet" href="styles.css">
    <!-- 即使这段CSS在body内容不需要样式之前定义 -->
</head>
<body>
    <p>内容</p>
</body>
```

**渲染树生成规则：**

渲染树只包含可见元素。`display: none`的元素不会出现在渲染树中，但`visibility: hidden`的元素会保留（占用空间）。

```css
/* 不会出现在渲染树中 - 不占用任何空间 */
.hidden {
    display: none;
}

/* 会出现在渲染树中 - 占用空间但不可见 */
.invisible {
    visibility: hidden;
}
```

**关键渲染路径优化策略：**

```html
<!-- 策略一：异步加载JavaScript -->
<script async src="analytics.js"></script>
<!-- async属性：下载不阻塞HTML解析，执行时阻塞HTML解析 -->

<!-- 策略二：延迟执行JavaScript -->
<script defer src="app.js"></script>
<!-- defer属性：下载不阻塞HTML解析，在HTML解析完成后执行 -->

<!-- 策略三：内联关键JavaScript -->
<script>
    // 将首屏必须的关键JS内联
    document.getElementById('app').innerHTML = '<h1>Hello</h1>';
</script>

<!-- 策略四：非阻塞CSS加载 -->
<link rel="stylesheet" href="non-critical.css" media="print" onload="this.media='all'">
```

### 1.2 资源加载优化

#### CSS与JavaScript的加载策略

CSS和JavaScript的加载位置直接影响页面的渲染时间和用户体验。合理的资源加载策略可以显著提升首屏渲染速度。

**CSS放置位置的最佳实践：**

```html
<head>
    <!-- 关键CSS必须内联或同步加载 -->
    <style>
        /* 首屏渲染必需的CSS */
        .critical-header { font-size: 24px; font-weight: bold; }
        .critical-nav { display: flex; gap: 16px; }
        .critical-content { margin: 20px; }
    </style>

    <!-- 非关键CSS异步加载 -->
    <link rel="preload" href="styles.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
    <noscript><link rel="stylesheet" href="styles.css"></noscript>
</head>
<body>
    <!-- 内容 -->
</body>
```

**JavaScript位置策略对比：**

```html
<!DOCTYPE html>
<html>
<head>
    <!-- 现代项目推荐：将所有script放在body末尾 -->
</head>
<body>
    <div id="app"></div>

    <!-- 第三方库（使用defer） -->
    <script defer src="vendor.js"></script>

    <!-- 应用代码（使用defer） -->
    <script defer src="app.js"></script>

    <!-- 关键脚本可以使用async -->
    <!-- <script async src="analytics.js"></script> -->
</body>
</html>
```

**同步vs异步加载对比：**

| 加载方式 | HTML解析 | 执行时机 | 使用场景 |
|----------|----------|----------|----------|
| 普通script | 阻塞 | 立即执行 | 关键依赖，必须在渲染前执行 |
| async | 不阻塞 | 下载完成后立即执行 | 独立脚本（分析、统计） |
| defer | 不阻塞 | HTML解析完成后执行 | 应用代码，按顺序执行 |
| 内联script | 阻塞 | 立即执行 | 极小的关键代码 |

#### 预加载与预取技术

预加载（preload）和预取（prefetch）是现代浏览器提供的资源提示技术，可以显著提升页面加载和导航性能。

**preload预加载 - 当前会话必需的资源：**

```html
<head>
    <!-- 预加载关键字体 -->
    <link rel="preload" href="fonts/Roboto-Bold.woff2" as="font" type="font/woff2" crossorigin>

    <!-- 预加载关键CSS -->
    <link rel="preload" href="critical.css" as="style">

    <!-- 预加载关键JavaScript -->
    <link rel="preload" href="bundle.js" as="script">

    <!-- 预加载图片资源 -->
    <link rel="preload" href="hero-image.webp" as="image" type="image/webp">
</head>
```

**prefetch预取 - 下一个会话可能需要的资源：**

```html
<head>
    <!-- 预取下一个页面可能需要的资源 -->
    <link rel="prefetch" href="next-page.html" as="document">
    <link rel="prefetch" href="next-page-bundle.js" as="script">

    <!-- 预取DNS域名（跨域资源） -->
    <link rel="dns-prefetch" href="https://api.example.com">

    <!-- 预连接（建立TCP连接、TLS握手） -->
    <link rel="preconnect" href="https://cdn.example.com" crossorigin>
</head>
```

**预加载与预取的核心区别：**

```
preload（高优先级，当前页面必需）：
┌─────────────────────────────────────────────────────────┐
│  页面加载 ──────────────────────────────────────→       │
│            ↑ preload开始  ↑ 立即加载  ↑ 用于当前页面    │
└─────────────────────────────────────────────────────────┘

prefetch（低优先级，下一导航可能需要）：
┌─────────────────────────────────────────────────────────┐
│  页面A ─────→ 页面B ──────────────────────────────────→  │
│              ↑ prefetch开始  ↑ 空闲时下载  ↑ 用于下一页面│
└─────────────────────────────────────────────────────────┘
```

**实际项目中的预加载应用：**

### 1.3 字体优化

字体文件通常较大（几百KB到几MB），字体加载不当会导致"Flash of Invisible Text"（FOIT）和"Flash of Unstyled Text"（FOUT）问题。

**font-display属性详解：**

```css
/* font-display的五个取值 */

/* block - 字体未加载前隐藏文本，等待期过后使用fallback */
@font-face {
    font-family: 'CustomFont';
    src: url('custom-font.woff2') format('woff2');
    font-display: block; /* 适用于关键字体 */
}

/* swap - 字体未加载前使用fallback，之后交换 */
@font-face {
    font-family: 'CustomFont';
    src: url('custom-font.woff2') format('woff2');
    font-display: swap; /* 适用于正文字体 */
}

/* fallback - 短等待期后使用fallback，不再交换 */
@font-face {
    font-family: 'CustomFont';
    src: url('custom-font.woff2') format('woff2');
    font-display: fallback; /* 适用于装饰性字体 */
}

/* optional - 完全由浏览器决定，可能永远使用fallback */
@font-face {
    font-family: 'CustomFont';
    src: url('custom-font.woff2') format('woff2');
    font-display: optional; /* 适用于非关键字体 */
}

/* auto - 浏览器自动决定（通常等同于block） */
@font-face {
    font-family: 'CustomFont';
    src: url('custom-font.woff2') format('woff2');
    font-display: auto;
}
```

**字体子集化技术：**

只加载字体中实际使用的字符，大幅减少字体文件体积。

```css
/* 使用unicode-range选择特定字符集 */
@font-face {
    font-family: 'ChineseFont';
    src: url('chinese-subset.woff2') format('woff2');
    /* 只加载基本拉丁字符和常用汉字 */
    unicode-range: U+0000-00FF, U+4E00-9FFF, U+3000-303F;
}

/* 使用font-size-adjust调整fallback字体大小 */
.custom-font {
    font-family: 'CustomFont', 'Arial', sans-serif;
    /* 确保fallback字体与自定义字体大小匹配 */
    font-size-adjust: 0.5;
}
```

**现代字体优化策略：**

```html
<!-- 使用CSS Font Loading API进行细粒度控制 -->
<script>
    // 检测字体加载状态
    async function loadFonts() {
        const font = new FontFace(
            'MyFont',
            'url(https://fonts.example.com/myfont.woff2) format("woff2")',
            { weight: '400 700' }
        );

        try {
            await font.load();
            document.fonts.add(font);
            document.documentElement.classList.add('fonts-loaded');
        } catch (error) {
            console.error('字体加载失败:', error);
        }
    }

    loadFonts();

    // 监听字体加载完成事件
    document.fonts.onloadingdone = (fontFaceSetEvent) => {
        console.log('字体加载完成:', fontFaceSetEvent.fontfaces);
    };
</script>

<!-- 使用预连接加速字体加载 -->
<head>
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
    <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;700&display=swap" rel="stylesheet">
</head>
```

### 1.4 图片优化

图片通常是网页体积最大的资源，优化图片可以显著减少页面加载时间。

**图片格式选择指南：**

| 格式 | 特点 | 适用场景 | 压缩方式 |
|------|------|----------|----------|
| JPEG | 有损压缩，体积小 | 照片、复杂图像 | 质量压缩 |
| PNG | 无损压缩，支持透明 | 图标、logo、需要透明的图像 | 索引色压缩 |
| WebP | 现代格式，体积小40-70% | 通用场景（现代浏览器） | 有损/无损 |
| AVIF | 最新格式，体积最小 | 通用场景（浏览器支持有限） | 有损/无损 |
| SVG | 矢量，体积小 | 图标、logo、简单图形 | gzip压缩 |

**WebP与AVIF转换示例：**

```bash
# 使用cwebp转换图片为WebP
cwebp -q 80 -resize 800 0 original.jpg -o optimized.webp

# 使用avif编码器转换图片为AVIF
avifenc -q 80 -y 420 -s 8 original.jpg -o optimized.avif

# 批量转换脚本
#!/bin/bash
for file in *.jpg; do
    cwebp -q 75 "$file" -o "${file%.jpg}.webp"
done
```

**响应式图片与srcset属性：**

```html
<!-- 使用srcset提供多个分辨率 -->
<img
    src="image-800.jpg"
    srcset="
        image-400.jpg 400w,
        image-800.jpg 800w,
        image-1200.jpg 1200w
    "
    sizes="
        (max-width: 600px) 400px,
        (max-width: 1200px) 800px,
        1200px
    "
    alt="响应式图片示例"
    loading="lazy"
    decoding="async"
>

<!-- 使用picture元素提供不同格式 -->
<picture>
    <!-- 优先使用AVIF -->
    <source type="image/avif" srcset="image.avif">

    <!-- 其次使用WebP -->
    <source type="image/webp" srcset="image.webp">

    <!-- 最后使用JPEG作为fallback -->
    <img src="image.jpg" alt="图片描述" width="800" height="600">
</picture>

<!-- 使用loading属性实现懒加载 -->
<img src="placeholder.jpg" data-src="real-image.jpg" loading="lazy" class="lazy-image">

<!-- 使用decoding属性异步解码 -->
<img src="photo.jpg" alt="照片" loading="lazy" decoding="async">
```

### 1.5 代码分割

代码分割是将代码库拆分为多个小块，按需加载的技术，可以显著减少首屏加载时间。

**路由级代码分割（Next.js App Router）：**

### 1.6 Tree Shaking与产物压缩

Tree Shaking是ES6模块系统带来的特性，通过静态分析移除未使用的代码。产物压缩则进一步减小文件体积。

**Tree Shaking的工作原理：**

```
ES6模块特点（支持Tree Shaking）：
┌─────────────────────────────────────────┐
│  import { a, b } from 'module';         │
│  ↑ 静态分析，编译时确定导入              │
│  ↓ 未使用的导出可以被安全移除            │
└─────────────────────────────────────────┘

CommonJS模块特点（不支持Tree Shaking）：
┌─────────────────────────────────────────┐
│  const module = require('module');     │
│  ↑ 动态分析，运行时确定导入              │
│  ↓ 无法确定哪些导出被使用                │
└─────────────────────────────────────────┘
```

**在JavaScript中编写Tree Shaking友好的代码：**

```javascript
// module.ts - 导出多个函数
// 方式一：使用命名导出（推荐，支持Tree Shaking）
export function usedFunction() {
    return 'result';
}

export function unusedFunction() {
    return 'unused';
}

// 方式二：禁止副作用
// package.json中标记无副作用
{
    "sideEffects": false
}

// 方式三：标记有副作用的文件
{
    "sideEffects": [
        "src/styles.css",
        "src/polyfills.js"
    ]
}

// 在代码中使用（确保被使用）
import { usedFunction } from './module';
console.log(usedFunction());
// unusedFunction会被Tree Shaking移除
```

**Tree Shaking的常见误区：**

```javascript
// 误区一：动态导入无法Tree Shaking
// ❌ 错误写法
const module = require('./module' + variable);

// ✅ 正确写法
import { usedFunction } from './module';

// 误区二：表达式条件导入
// ❌ 错误写法
const utils = condition ? require('./utils-a') : require('./utils-b');

// 误区三：使用复杂表达式
// ❌ 错误写法
import { map } from 'lodash';
const result = map([1, 2, 3], item => item * 2);

// ✅ 正确写法 - 按需导入
import map from 'lodash/map';
// 或使用ES模块版本
import { map } from 'lodash-es';
```

## 二、运行时性能

运行时性能是指页面加载完成后，在用户交互过程中保持流畅的性能。与加载性能不同，运行时性能更关注页面响应速度和动画流畅度。

### 2.1 重排与重绘

重排（Reflow）和重绘（Repaint）是浏览器渲染引擎的两个核心概念，理解它们对于优化页面性能至关重要。

**重排（Reflow）与触发条件：**

重排是指当DOM元素的**几何属性**（位置、大小）发生变化时，浏览器需要重新计算文档几何信息的过程。

```javascript
// 触发重排的操作示例

element.style.height = '200px';

// ❌ 连续修改多个几何属性

element.style.height = '200px';
element.style.margin = '10px';  // 三次重排
```

### 2.2 事件委托与节流防抖

事件委托利用事件冒泡机制，将子元素的事件绑定到父元素上，减少事件监听器数量。节流和防抖是控制高频事件执行频率的技术。

**事件委托的实现原理：**

```
DOM事件流：
┌─────────────────────────────────────────────────────┐
│  Window → Document → html → body → parent → child  │
│                                                     │
│  捕获阶段（capturing）: Window → ... → target      │
│  目标阶段（target）: 事件发生在目标元素             │
│  冒泡阶段（bubbling）: target → ... → Window       │
└─────────────────────────────────────────────────────┘

事件委托示例：
<ul id="list">
    <li data-id="1">项目1</li>      ← 点击事件冒泡到<ul>
    <li data-id="2">项目2</li>      ← 点击事件冒泡到<ul>
    <li data-id="3">项目3</li>      ← 点击事件冒泡到<ul>
</ul>
```


**节流（Throttle）实现：**

节流是确保函数在指定时间间隔内最多执行一次。

```javascript
// 节流基础实现
function throttle(func, wait) {
    let timeout = null;
    let previous = 0;  // 上次执行时间

    return function(...args) {
        const now = Date.now();
        const remaining = wait - (now - previous);

        if (remaining <= 0 || remaining > wait) {
            // 超过等待时间，执行
            if (timeout) {
                clearTimeout(timeout);
                timeout = null;
            }
            previous = now;
            func.apply(this, args);
        } else if (!timeout) {
            // 最后一次触发（尾部执行）
            timeout = setTimeout(() => {
                previous = Date.now();
                timeout = null;
                func.apply(this, args);
            }, remaining);
        }
    };
}

// 节流使用示例
function SearchInput() {
    const [query, setQuery] = useState('');

    const handleSearch = useCallback(
        throttle((value) => {
            // 实际搜索逻辑（最多每300ms执行一次）
            fetch(`/api/search?q=${value}`)
                .then(res => res.json())
                .then(data => setResults(data));
        }, 300),
        []
    );

    return (
        <input
            type="search"
            value={query}
            onChange={(e) => {
                setQuery(e.target.value);
                handleSearch(e.target.value);
            }}
        />
    );
}
```

**防抖（Debounce）实现：**

防抖是确保函数在最后一次触发后等待指定时间才执行。

```javascript
// 防抖基础实现
function debounce(func, wait, immediate = false) {
    let timeout = null;

    return function(...args) {
        const context = this;

        if (timeout) clearTimeout(timeout);

        if (immediate && !timeout) {
            // 立即执行模式
            func.apply(context, args);
        }

        timeout = setTimeout(() => {
            if (!immediate) {
                func.apply(context, args);
            }
            timeout = null;
        }, wait);
    };
}

// 防抖使用示例 - 搜索输入
function SearchInput() {
    const [query, setQuery] = useState('');

    // 防抖：用户停止输入500ms后才执行搜索
    const debouncedSearch = useMemo(
        () => debounce((value) => {
            // 实际搜索逻辑
            fetch(`/api/search?q=${value}`)
                .then(res => res.json())
                .then(data => setResults(data));
        }, 500),
        []
    );

    return (
        <input
            type="search"
            value={query}
            onChange={(e) => {
                setQuery(e.target.value);
                debouncedSearch(e.target.value);
            }}
        />
    );
}

// 防抖使用示例 - 窗口调整
const handleResize = debounce(() => {
    // 窗口调整完成后的处理
    console.log('窗口尺寸:', window.innerWidth, window.innerHeight);
}, 250);

window.addEventListener('resize', handleResize);

// lodash debounce使用
import { debounce } from 'lodash';

const debouncedSave = debounce((data) => {
    localStorage.setItem('draft', JSON.stringify(data));
}, 1000);

// 表单自动保存
<textarea onChange={(e) => debouncedSave({ content: e.target.value })} />
```

**节流与防抖的选择：**

| 场景 | 推荐 | 原因 |
|------|------|------|
| 滚动事件 | 节流 | 持续触发，需要平滑响应 |
| 窗口调整 | 防抖 | 用户停止调整后才处理 |
| 搜索输入 | 防抖 | 用户停止输入后才搜索 |
| 按钮点击 | 节流/不限制 | 防止重复提交 |
| 鼠标移动 | 节流 | 持续触发，降低频率 |
| 键盘输入 | 防抖 | 输入过程中不搜索 |

### 2.3 虚拟滚动

虚拟滚动是一种处理大量数据渲染的技术，只渲染可见区域的DOM元素，大幅降低内存占用和渲染时间。

**手写虚拟滚动实现：**

```typescript
// hooks/useVirtualScroll.ts
import { useState, useRef, useEffect, useCallback } from 'react';

interface VirtualScrollOptions<T> {
    items: T[];
    itemHeight: number;  // 固定高度，如果为0则使用动态高度
    overscan?: number;   // 超出可见区域的项目数
}

function useVirtualScroll<T>({
    items,
    itemHeight,
    overscan = 3
}: VirtualScrollOptions<T>) {
    const containerRef = useRef<HTMLDivElement>(null);
    const [scrollTop, setScrollTop] = useState(0);
    const [containerHeight, setContainerHeight] = useState(0);

    // 容器尺寸监听
    useEffect(() => {
        const container = containerRef.current;
        if (!container) return;

        const resizeObserver = new ResizeObserver((entries) => {
            for (const entry of entries) {
                setContainerHeight(entry.contentRect.height);
            }
        });

        resizeObserver.observe(container);
        setContainerHeight(container.clientHeight);

        return () => resizeObserver.disconnect();
    }, []);

    // 滚动事件处理
    const handleScroll = useCallback((event: React.UIEvent<HTMLDivElement>) => {
        setScrollTop(event.currentTarget.scrollTop);
    }, []);

    // 计算可见范围
    const visibleRange = useCallback(() => {
        const startIndex = Math.max(
            0,
            Math.floor(scrollTop / itemHeight) - overscan
        );

        const visibleCount = Math.ceil(containerHeight / itemHeight);
        const endIndex = Math.min(
            items.length - 1,
            Math.floor(scrollTop / itemHeight) + visibleCount + overscan
        );

        return { startIndex, endIndex };
    }, [scrollTop, containerHeight, itemHeight, items.length, overscan]);

    // 渲染可见项目
    const visibleItems = useCallback(() => {
        const { startIndex, endIndex } = visibleRange();

        return items.slice(startIndex, endIndex + 1).map((item, index) => ({
            item,
            index: startIndex + index,
            style: {
                position: 'absolute',
                top: (startIndex + index) * itemHeight,
                height: itemHeight,
                width: '100%',
            }
        }));
    }, [items, visibleRange, itemHeight]);

    // 总高度（用于生成滚动条）
    const totalHeight = items.length * itemHeight;

    // 偏移量（用于定位起始项目）
    const offsetY = Math.floor(scrollTop / itemHeight) * itemHeight;

    return {
        containerRef,
        handleScroll,
        totalHeight,
        visibleItems: visibleItems(),
        containerHeight,
    };
}

// 使用虚拟滚动的组件
function VirtualScrollComponent({ data }) {
    const {
        containerRef,
        handleScroll,
        totalHeight,
        visibleItems,
    } = useVirtualScroll({
        items: data,
        itemHeight: 50,
        overscan: 3,
    });

    return (
        <div
            ref={containerRef}
            onScroll={handleScroll}
            style={{
                height: '600px',
                overflow: 'auto',
                position: 'relative',
            }}
        >
            {/* 内层容器设置总高度 */}
            <div style={{ height: totalHeight, position: 'relative' }}>
                {/* 渲染可见项目 */}
                {visibleItems.map(({ item, index, style }) => (
                    <div key={item.id} style={style} className="item">
                        <span>{item.name}</span>
                        <span>{item.description}</span>
                    </div>
                ))}
            </div>
        </div>
    );
}
```

**主线程与Worker通信：**

```typescript
// main.ts - 主线程文件
class DataProcessor {
    private worker: Worker | null = null;
    private pendingCallbacks: Map<string, (result: any) => void> = new Map();

    constructor() {
        this.initWorker();
    }

    private initWorker() {
        // 创建Worker
        this.worker = new Worker(new URL('./worker.js', import.meta.url));

        // 监听Worker消息
        this.worker.addEventListener('message', (event) => {
            const { type, data } = event.data;

            // 处理Worker返回的结果
            if (type === 'RESULT') {
                const callback = this.pendingCallbacks.get('PROCESS_DATA');
                if (callback) {
                    callback(data);
                    this.pendingCallbacks.delete('PROCESS_DATA');
                }
            }

            if (type === 'SORTED') {
                const callback = this.pendingCallbacks.get('SORT_ARRAY');
                if (callback) {
                    callback(data);
                    this.pendingCallbacks.delete('SORT_ARRAY');
                }
            }
        });

        // 错误处理
        this.worker.addEventListener('error', (error) => {
            console.error('Worker执行错误:', error);
        });
    }

    // 发送任务到Worker
    processData(data: any[]): Promise<any[]> {
        return new Promise((resolve) => {
            this.pendingCallbacks.set('PROCESS_DATA', resolve);
            this.worker?.postMessage({
                type: 'PROCESS_DATA',
                data: data,
            });
        });
    }

    // 排序数组
    sortArray(array: number[]): Promise<number[]> {
        return new Promise((resolve) => {
            this.pendingCallbacks.set('SORT_ARRAY', resolve);
            this.worker?.postMessage({
                type: 'SORT_ARRAY',
                data: { array },
            });
        });
    }

    // 终止Worker
    terminate() {
        this.worker?.terminate();
        this.worker = null;
    }
}

// 使用示例

// 处理大数据

    console.log('处理完成:', result);
```

```javascript
    // API请求：网络优先，失败时回退缓存

        event.respondWith(networkFirstStrategy(request, API_CACHE_NAME));
        return;

    // 静态资源：缓存优先，失败时回退网络

        event.respondWith(cacheFirstStrategy(request, CACHE_NAME));

    }

    // HTML页面：网络优先
    if (request.mode === 'navigate') {
        event.respondWith(networkFirstStrategy(request, CACHE_NAME));
        return;
    }

// 网络优先策略
async function networkFirstStrategy(request, cacheName) {
    try {
        const networkResponse = await fetch(request);

        // 如果网络成功，缓存响应
        if (networkResponse.ok) {
            const cache = await caches.open(cacheName);
            cache.put(request, networkResponse.clone());
        }

        return networkResponse;
    } catch (error) {
        // 网络失败，返回缓存
        const cachedResponse = await caches.match(request);
        if (cachedResponse) {
            return cachedResponse;
        }

        // 如果是API请求，返回错误JSON
        if (request.url.includes('/api/')) {
            return new Response(
                JSON.stringify({ error: '离线状态' }),
                { status: 503, headers: { 'Content-Type': 'application/json' } }
            );
        }

        // 返回离线页面
        return caches.match('/offline.html');
    }
}

// 缓存优先策略
async function cacheFirstStrategy(request, cacheName) {
    // 先从缓存查找
    const cachedResponse = await caches.match(request);

    if (cachedResponse) {
        // 返回缓存，同时更新缓存（后台更新）
        updateCache(request, cacheName);
        return cachedResponse;
    }

    // 缓存未命中，从网络获取
    try {
        const networkResponse = await fetch(request);

        if (networkResponse.ok) {
            const cache = await caches.open(cacheName);
            cache.put(request, networkResponse.clone());
        }

        return networkResponse;
    } catch (error) {
        console.error('获取失败:', error);
    }
}

// 后台更新缓存
async function updateCache(request, cacheName) {
    try {
        const networkResponse = await fetch(request);
        if (networkResponse.ok) {
            const cache = await caches.open(cacheName);
            cache.put(request, networkResponse);
        }
    } catch (error) {
        // 忽略错误
    }
}
```

```typescript
// hooks/useServiceWorker.ts
import { useEffect, useState } from 'react';

type ServiceWorkerStatus = 'registered' | 'ready' | 'updatefound' | 'error';

interface ServiceWorkerRegistration {
    status: ServiceWorkerStatus;
    registration: ServiceWorkerRegistration | null;
    update: () => void;
}

export function useServiceWorker(): ServiceWorkerRegistration {
    const [status, setStatus] = useState<ServiceWorkerStatus>('registered');
    const [registration, setRegistration] = useState<ServiceWorkerRegistration | null>(null);

    useEffect(() => {
        if (typeof window === 'undefined' || !('serviceWorker' in navigator)) {
            setStatus('error');
            return;
        }

        // 注册Service Worker
        navigator.serviceWorker
            .register('/sw.js')
            .then((reg) => {
                setRegistration(reg);
                setStatus('registered');

                // 检查更新
                reg.addEventListener('updatefound', () => {
                    setStatus('updatefound');
                    const newWorker = reg.installing;

                    if (newWorker) {
                        newWorker.addEventListener('statechange', () => {
                            if (newWorker.state === 'installed' && navigator.serviceWorker.controller) {
                                // 新的Service Worker可用
                                console.log('有新版本可用');
                            }
                        });
                    }
                });

                // 当前的Service Worker已就绪
                if (reg.active) {
                    setStatus('ready');
                }
            })
            .catch((error) => {
                console.error('Service Worker注册失败:', error);
                setStatus('error');
            });

        // 监听控制器变化（页面刷新后新SW接管）
        navigator.serviceWorker.addEventListener('controllerchange', () => {
            window.location.reload();
        });
    }, []);

    const update = () => {
        if (registration?.update) {
            registration.update();
        }
    };

    return { status, registration, update };
}
```

```typescript
// hooks/useLazyLoad.ts
import { useState, useEffect, useRef } from 'react';

interface LazyLoadOptions {
    root?: Element | null;
    rootMargin?: string;
    threshold?: number | number[];
}

export function useLazyLoad(options: LazyLoadOptions = {}) {
    const [isLoaded, setIsLoaded] = useState(false);
    const [isInView, setIsInView] = useState(false);
    const elementRef = useRef<HTMLDivElement>(null);

    useEffect(() => {
        const element = elementRef.current;
        if (!element) return;

        // 检查是否原生支持loading属性
        if ('loading' in HTMLImageElement.prototype) {
            setIsInView(true);
            return;
        }

        // 使用Intersection Observer
        const observer = new IntersectionObserver(
            (entries) => {
                entries.forEach((entry) => {
                    if (entry.isIntersecting) {
                        setIsInView(true);
                        observer.disconnect();  // 加载后停止观察
                    }
                });
            },
            {
                root: options.root || null,
                rootMargin: options.rootMargin || '100px',  // 提前100px开始加载
                threshold: options.threshold || 0,
            }
        );

        observer.observe(element);

        return () => observer.disconnect();
    }, [options.root, options.rootMargin, options.threshold]);

    useEffect(() => {
        if (isInView) {
            // 开始加载图片
            setIsLoaded(true);
        }
    }, [isInView]);

    return { elementRef, isLoaded, isInView };
}

// 图片懒加载组件
function LazyImage({ src, alt, placeholder }) {
    const { elementRef, isLoaded } = useLazyLoad({ rootMargin: '200px' });

    return (
        <div ref={elementRef} style={{ position: 'relative' }}>
            {isLoaded ? (
                <img
                    src={src}
                    alt={alt}
                    style={{ opacity: 1, transition: 'opacity 0.3s' }}
                />
            ) : (
                placeholder || <div className="placeholder" />
            )}
        </div>
    );
}
```

```tsx
                setPage(p => p + 1);

        } finally {
            setIsLoading(false);
        }

                {items.map(item => (
                    <ItemCard key={item.id} item={item} />

        const newItems = await response.json();

            setHasMore(false);
        } else {
            setItems(prev => [...prev, ...newItems]);
            setPage(p => p + 1);
        }
    }, [page, hasMore, fetchUrl]);

    useEffect(() => {

            (entries) => {
                if (entries[0].isIntersecting && hasMore) {
                    loadMore();
                }
            },
            { rootMargin: '100px' }
        );

        if (loaderRef.current) {
            observer.observe(loaderRef.current);
        }

        return () => observer.disconnect();
    }, [loadMore, hasMore]);

    return (
        <div>
            {items.map(item => (
                <ItemCard key={item.id} item={item} />
            ))}
            <div ref={loaderRef} className="loader">
                {hasMore ? <LoadingSpinner /> : '没有更多了'}
            </div>
        </div>
    );
```

## 四、网络性能

网络性能优化涉及减少网络请求次数、减小请求体积、利用缓存等多个方面。

### 4.1 HTTP缓存策略

HTTP缓存是提升网络性能最有效的方式，分为强缓存和协商缓存。

**强缓存（Cache-Control/Expires）：**

```
首次请求：
┌─────────┐     ┌──────────┐     ┌─────────┐
│ 浏览器   │────▶│  服务器   │────▶│ 返回资源 │
└─────────┘     └──────────┘     └─────────┘
                                           │
                                           ▼
                                    ┌──────────────┐
                                    │ 缓存(200 OK) │
                                    └──────────────┘

后续请求（未过期）：
┌─────────┐     ┌──────────────┐
│ 浏览器   │────▶│ 直接从缓存读取 │
└─────────┘     └──────────────┘
                      │
                      ▼
                 速度快，不向服务器发请求
```

```http
# 强缓存响应头示例

# HTTP/1.1 200 OK
# 方式一：Cache-Control（推荐）
Cache-Control: max-age=31536000, public, immutable
# max-age: 缓存有效期（秒）
# public: 可被任何缓存存储
# immutable: 资源永不变化

# 方式二：Expires（HTTP/1.0，已过时）
Expires: Wed, 21 Oct 2027 07:28:00 GMT
# 绝对时间，服务器时间与客户端时间必须一致
```

**协商缓存（Last-Modified/ETag）：**

```
协商缓存流程：
┌─────────┐     ┌──────────┐     ┌──────────────┐
│ 浏览器   │────▶│  服务器   │───▶│ 资源未变化    │
└─────────┘     └──────────┘     │ 返回304 Not   │
     │                               Modified    │
     │                               └──────────────┘
     │              ┌──────────┐
     └─────────────▶│ 304 使用  │
                    │ 缓存资源   │
                    └──────────┘

首次请求：
┌─────────┐     ┌──────────┐
│ 浏览器   │────▶│ 服务器    │────▶返回资源 + ETag/Last-Modified
└─────────┘     └──────────┘
                        │
                        ▼
                 ┌────────────────┐
                 │ ETag: "abc123" │
                 │ 或Last-Modified│
                 │ 时间戳         │
                 └────────────────┘

后续请求（带缓存标识）：
┌─────────┐     ┌──────────┐
│ 浏览器   │────▶│ If-None-Match: "abc123"  │
│ (带ETag) │     │ 或If-Modified-Since      │
└─────────┘     └──────────┘
                      │
                      ▼
                 ┌────────────────────────┐
                 │ 服务器检查ETag/时间戳   │
                 │ 匹配 → 返回304          │
                 │ 不匹配 → 返回新资源+新ETag│
                 └────────────────────────┘
```

**缓存策略配置（Nginx）：**


**Service Worker完整缓存策略：**

```javascript
// sw.js中的缓存策略实现
const CACHE_VERSION = 'v2';
const STATIC_CACHE = `static-${CACHE_VERSION}`;
const API_CACHE = `api-${CACHE_VERSION}`;

// 静态资源 - 缓存优先，fallback到网络
async function staticAssetsHandler(event) {
    const cache = await caches.open(STATIC_CACHE);
    const cachedResponse = await cache.match(event.request);

    if (cachedResponse) {
        // 返回缓存，同时后台更新
        event.waitUntil(updateCache(event.request));
        return cachedResponse;
    }

    // 缓存未命中，尝试网络
    try {
        const networkResponse = await fetch(event.request);
        if (networkResponse.ok) {
            cache.put(event.request, networkResponse.clone());
        }
        return networkResponse;
    } catch (error) {
        // 网络失败，返回离线资源
        return cache.match('/offline.html');
    }
}

// API请求 - 网络优先，fallback到缓存
async function apiHandler(event) {
    try {
        const networkResponse = await fetch(event.request);

        if (networkResponse.ok) {
            const cache = await caches.open(API_CACHE);
            cache.put(event.request, networkResponse.clone());
        }

        return networkResponse;
    } catch (error) {
        const cache = await caches.open(API_CACHE);
        const cachedResponse = await cache.match(event.request);

        if (cachedResponse) {
            return cachedResponse;
        }

        // API失败，返回错误JSON
        return new Response(
            JSON.stringify({ error: '网络请求失败' }),
            {
                status: 503,
                headers: { 'Content-Type': 'application/json' }
            }
        );
    }
}
```

**HTTP/1.1 vs HTTP/2对比：**

```
HTTP/1.1 - 管线化阻塞：
请求1 ───▶                           ◀─── 响应1
请求2 ──────────────────▶          ◀─── 响应2
请求3 ─────────────────────────▶    ◀─── 响应3
          ↑
     所有请求串行，需要创建多个连接

HTTP/2 - 多路复用：
          ┌──────────────────────────────┐
          │     单个TCP连接               │
请求1 ───▶◀── 响应1                       │
请求2 ───▶◀────────── 响应2              │
请求3 ───▶◀────────────────── 响应3      │
          └──────────────────────────────┘
     所有请求并行，共享连接
```

**HTTP/2优化配置：**

```nginx

    listen 443 ssl http2;
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # HTTP/2服务器推送

    # 静态资源
    location ~* \.(js|css|jpg|png|svg)$ {
        # 启用HTTP/2
        http2 on;
    }

```

### 4.5 WebSocket vs HTTP

WebSocket提供全双工通信，适合实时应用；HTTP适合请求-响应模式。

**连接特性对比：**

| 特性 | HTTP | WebSocket |
|------|------|-----------|
| 连接方式 | 短连接，每次请求新建连接 | 长连接，握手后保持 |
| 通信方向 | 半双工（请求-响应） | 全双工（双方可主动发送） |
| 头部开销 | 每次请求都需要header | 仅握手时需要header |
| 实时性 | 轮询/长轮询，有延迟 | 实时推送 |
| 适用场景 | RESTful API | 聊天、游戏、实时协作 |

**WebSocket心跳保活：**

```javascript
class WebSocketClient {
    constructor(url) {
        this.url = url;
        this.ws = null;
        this.reconnectAttempts = 0;
        this.maxReconnectAttempts = 5;
        this.reconnectDelay = 1000;
        this.heartbeatInterval = 30000;
        this.heartbeatTimer = null;
    }

    connect() {
        this.ws = new WebSocket(this.url);

        this.ws.onopen = () => {
            console.log('WebSocket连接已建立');
            this.reconnectAttempts = 0;
            this.startHeartbeat();
        };

        this.ws.onmessage = (event) => {
            if (event.data === 'pong') {
                console.log('心跳响应');
                return;
            }
            this.handleMessage(event.data);
        };

        this.ws.onerror = (error) => {
            console.error('WebSocket错误:', error);
        };

        this.ws.onclose = () => {
            console.log('WebSocket连接关闭');
            this.stopHeartbeat();
            this.reconnect();
        };
    }

    startHeartbeat() {
        this.heartbeatTimer = setInterval(() => {
            if (this.ws.readyState === WebSocket.OPEN) {
                this.ws.send('ping');
            }
        }, this.heartbeatInterval);
    }

    stopHeartbeat() {
        if (this.heartbeatTimer) {
            clearInterval(this.heartbeatTimer);
            this.heartbeatTimer = null;
        }
    }

    reconnect() {
        if (this.reconnectAttempts >= this.maxReconnectAttempts) {
            console.error('最大重连次数已达上限');
            return;
        }

        this.reconnectAttempts++;
        const delay = this.reconnectDelay * Math.pow(2, this.reconnectAttempts - 1);

        console.log(`${delay}ms后尝试重连...`);

        setTimeout(() => {
            console.log(`第${this.reconnectAttempts}次重连...`);
            this.connect();
        }, delay);
    }

    send(data) {
        if (this.ws.readyState === WebSocket.OPEN) {
            this.ws.send(JSON.stringify(data));
        } else {
            console.error('WebSocket未连接');
        }
    }

    handleMessage(data) {
        console.log('收到消息:', data);
    }

    close() {
        this.stopHeartbeat();
        if (this.ws) {
            this.ws.close();
        }
    }
}
```

```typescript
// 自定义RUM实现
class RealUserMonitor {
    private endpoint: string;
    private sessionId: string;
    private userId?: string;

    constructor(endpoint: string) {
        this.endpoint = endpoint;
        this.sessionId = this.generateSessionId();
        this.init();
    }

    private generateSessionId(): string {
        return `${Date.now()}-${Math.random().toString(36).substr(2, 9)}`;

    private init() {
        // 监控Web Vitals
        this.measureLCP();
        this.measureFID();
        this.measureCLS();
        this.measureTTFB();
        this.measureFCP();

        // 监控页面加载
        window.addEventListener('load', () => {
            this.trackTiming('page_load', performance.now());
        });

        // 监控AJAX请求
        this.injectXHRTracker();

        // 监控错误
        window.addEventListener('error', (event) => {
            this.track('js_error', {
                message: event.message,
                filename: event.filename,
                lineno: event.lineno,
                colno: event.colno,
            });
        });
    }

    private measureLCP() {
        new PerformanceObserver((list) => {
            const entries = list.getEntries();
            const lastEntry = entries[entries.length - 1] as LargestContentfulPaint;

            this.track('web_vital', {
                name: 'LCP',
                value: lastEntry.startTime,
                rating: lastEntry.startTime < 2500 ? 'good' : 'needs-improvement',
            });
        }).observe({ type: 'largest-contentful-paint', buffered: true });
    }

    private measureFID() {
        new PerformanceObserver((list) => {
            for (const entry of list.getEntries()) {
                const fid = (entry as FirstInput).processingStart - entry.startTime;

                this.track('web_vital', {
                    name: 'FID',
                    value: fid,
                    rating: fid < 100 ? 'good' : 'needs-improvement',
                });
            }
        }).observe({ type: 'first-input', buffered: true });
    }

    private measureCLS() {
        let clsValue = 0;

        new PerformanceObserver((list) => {
            for (const entry of list.getEntries()) {
                if (!(entry as LayoutShift).hadRecentInput) {
                    clsValue += (entry as LayoutShift).value;
                }
            }
        }).observe({ type: 'layout-shift', buffered: true });

        // CLS在页面卸载时上报
        document.addEventListener('visibilitychange', () => {
            if (document.visibilityState === 'hidden') {
                this.track('web_vital', {
                    name: 'CLS',
                    value: clsValue,
                    rating: clsValue < 0.1 ? 'good' : 'needs-improvement',
                });
            }
        });
    }

    private measureTTFB() {
        const ttfb = performance.getEntriesByType('navigation')[0] as PerformanceNavigationTiming;
        this.track('timing', {
            name: 'TTFB',
            value: ttfb.responseStart - ttfb.requestStart,
        });
    }

    private measureFCP() {
        new PerformanceObserver((list) => {
            const entries = list.getEntries();
            const fcp = entries[0] as PerformancePaintTiming;

            if (fcp.name === 'first-contentful-paint') {
                this.track('web_vital', {
                    name: 'FCP',
                    value: fcp.startTime,
                    rating: fcp.startTime < 1800 ? 'good' : 'needs-improvement',
                });
            }
        }).observe({ type: 'paint', buffered: true });
    }

    private injectXHRTracker() {
        const originalXHROpen = XMLHttpRequest.prototype.open;
        const originalXHRSend = XMLHttpRequest.prototype.send;
        const self = this;

        XMLHttpRequest.prototype.open = function(method, url, async) {
            (this as any)._xhrUrl = url;
            (this as any)._xhrMethod = method;
            return originalXHROpen.call(this, method, url, async);
        };

        XMLHttpRequest.prototype.send = function(data) {
            const startTime = Date.now();

            this.addEventListener('load', function() {
                const duration = Date.now() - startTime;
                self.track('xhr', {
                    url: (this as any)._xhrUrl,
                    method: (this as any)._xhrMethod,
                    status: this.status,
                    duration,
                });
            });

            return originalXHRSend.call(this, data);
        };
    }

    track(name: string, data: Record<string, any>) {
        const payload = {
            name,
            data,
            timestamp: Date.now(),
            sessionId: this.sessionId,
            userId: this.userId,
            url: window.location.href,
            userAgent: navigator.userAgent,
        };

        // 批量上报
        console.log('[RUM]', payload);

        // 实际发送到服务器
        if (navigator.sendBeacon) {
            navigator.sendBeacon(this.endpoint, JSON.stringify(payload));
        }
    }
}
```

```typescript
    const [containerWidth, setContainerWidth] = useState(1200);

            for (const entry of entries) {
                setContainerHeight(entry.contentRect.height);
                setContainerWidth(entry.contentRect.width);
            }

        resizeObserver.observe(container);
        setContainerHeight(container.clientHeight);
        setContainerWidth(container.clientWidth);

        return () => resizeObserver.disconnect();

    // 计算每行显示的商品数量（根据容器宽度）
    const productsPerRow = Math.max(1, Math.floor(containerWidth / 280));

    const rowHeight = 380;

            .filter((product) => {

                if (filters.category !== 'all' && product.category !== filters.category) {
                    return false;
                }
                // 价格过滤
                if (product.price < filters.priceRange[0] || product.price > filters.priceRange[1]) {
                    return false;
                }
                // 评分过滤
                if (filters.rating > 0 && product.rating < filters.rating) {
                    return false;
                }
                return true;
            })
            .sort((a, b) => {
                switch (filters.sortBy) {
                    case 'price-asc':
                        return a.price - b.price;
                    case 'price-desc':
                        return b.price - a.price;
                    case 'rating':
                        return b.rating - a.rating;
                    case 'newest':
                    default:
                        return 0; // 假设已按最新排序
                }
            });
    }, [products, filters]);

    const categories = useCallback(() => {
        return ['all', ...new Set(products.map((p) => p.category))];

        onProductClick,
```

### 6.3 大数据表格渲染优化

**场景：数据分析后台，需要渲染100万行数据，包含复杂单元格。**

```typescript
// 虚拟滚动大数据表格实现
// components/VirtualizedTable.tsx
import { useState, useCallback, useRef, useEffect, memo, useMemo } from 'react';
import { FixedSizeList as List } from 'react-window';

// 表格列配置
interface Column<T> {
    key: keyof T | string;
    title: string;
    width: number;
    render?: (value: any, row: T, index: number) => React.ReactNode;
    sortable?: boolean;
    align?: 'left' | 'center' | 'right';
}

// 表格行数据
interface TableRow {
    id: string;
    [key: string]: any;
}

// 表头组件
const TableHeader = memo(function TableHeader({
    columns,
    sortColumn,
    sortDirection,
    onSort,
}: {
    columns: Column<TableRow>[];
    sortColumn: string | null;
    sortDirection: 'asc' | 'desc' | null;
    onSort: (column: string) => void;
}) {
    return (
        <div className="table-header">
            {columns.map((column) => (
                <div
                    key={String(column.key)}
                    className={`table-header-cell ${column.align || 'left'}`}
                    style={{ width: column.width, minWidth: column.width }}
                    onClick={() => column.sortable && onSort(String(column.key))}
                >
                    <span>{column.title}</span>
                    {column.sortable && sortColumn === column.key && (
                        <span className="sort-indicator">
                            {sortDirection === 'asc' ? ' ▲' : ' ▼'}
                        </span>
                    )}
                </div>
            ))}
        </div>
    );
});

// 表格行组件
const TableRow = memo(function TableRow({
    index,
    style,
    data,
}: {
    index: number;
    style: React.CSSProperties;
    data: {
        rows: TableRow[];
        columns: Column<TableRow>[];
        onRowClick?: (row: TableRow) => void;
        selectedIds: Set<string>;
    };
}) {
    const row = data.rows[index];
    const isSelected = data.selectedIds.has(row.id);

    return (
        <div
            style={style}
            className={`table-row ${isSelected ? 'selected' : ''}`}
            onClick={() => data.onRowClick?.(row)}
        >
            {data.columns.map((column) => {
                const value = row[column.key as keyof TableRow];
                return (
                    <div
                        key={String(column.key)}
                        className={`table-cell ${column.align || 'left'}`}
                        style={{ width: column.width, minWidth: column.width }}
                    >
                        {column.render ? column.render(value, row, index) : value}
                    </div>
                );
            })}
        </div>
    );
});

// 主表格组件
interface VirtualizedTableProps {
    data: TableRow[];
    columns: Column<TableRow>[];
    height?: number;
    rowHeight?: number;
    onRowClick?: (row: TableRow) => void;
    onSelectionChange?: (selectedRows: TableRow[]) => void;
}

export default function VirtualizedTable({
    data,
    columns,
    height = 600,
    rowHeight = 48,
    onRowClick,
    onSelectionChange,
}: VirtualizedTableProps) {
    const containerRef = useRef<HTMLDivElement>(null);
    const [containerWidth, setContainerWidth] = useState(1200);
    const [sortColumn, setSortColumn] = useState<string | null>(null);
    const [sortDirection, setSortDirection] = useState<'asc' | 'desc' | null>(null);
    const [selectedIds, setSelectedIds] = useState<Set<string>>(new Set());
    const [selectAll, setSelectAll] = useState(false);

    // 容器宽度监听
    useEffect(() => {
        const container = containerRef.current;
        if (!container) return;

        const resizeObserver = new ResizeObserver((entries) => {
            for (const entry of entries) {
                setContainerWidth(entry.contentRect.width);
            }
        });

        resizeObserver.observe(container);
        setContainerWidth(container.clientWidth);

        return () => resizeObserver.disconnect();
    }, []);

    // 排序处理
    const handleSort = useCallback((column: string) => {
        setSortColumn((prev) => {
            if (prev === column) {
                setSortDirection((dir) => (dir === 'asc' ? 'desc' : 'asc'));
                return column;
            }
            setSortDirection('asc');
            return column;
        });
    }, []);

    // 排序后的数据
    const sortedData = useMemo(() => {
        if (!sortColumn || !sortDirection) return data;

        return [...data].sort((a, b) => {
            const aVal = a[sortColumn];
            const bVal = b[sortColumn];

            if (aVal === bVal) return 0;
            if (aVal === null || aVal === undefined) return 1;
            if (bVal === null || bVal === undefined) return -1;

            const comparison = aVal < bVal ? -1 : 1;
            return sortDirection === 'asc' ? comparison : -comparison;
        });
    }, [data, sortColumn, sortDirection]);

    // 选择处理
    const handleRowClick = useCallback(
        (row: TableRow) => {
            onRowClick?.(row);
        },
        [onRowClick]
    );

    const handleSelectAll = useCallback(() => {
        if (selectAll) {
            setSelectedIds(new Set());
            onSelectionChange?.([]);
        } else {
            const allIds = new Set(sortedData.map((row) => row.id));
            setSelectedIds(allIds);
            onSelectionChange?.(sortedData);
        }
        setSelectAll(!selectAll);
    }, [selectAll, sortedData, onSelectionChange]);

    const handleSelectRow = useCallback(
        (id: string) => {
            setSelectedIds((prev) => {
                const newSet = new Set(prev);
                if (newSet.has(id)) {
                    newSet.delete(id);
                } else {
                    newSet.add(id);
                }
                onSelectionChange?.(sortedData.filter((row) => newSet.has(row.id)));
                return newSet;
            });
            setSelectAll(false);
        },
        [sortedData, onSelectionChange]
    );

    // 行数据
    const itemData = useMemo(
        () => ({
            rows: sortedData,
            columns,
            onRowClick: handleRowClick,
            selectedIds,
        }),
        [sortedData, columns, handleRowClick, selectedIds]
    );

    // 总列宽
    const totalWidth = columns.reduce((sum, col) => sum + col.width, 0);

    return (
        <div className="virtualized-table">
            {/* 表头 */}
            <div className="table-header-wrapper">
                <TableHeader
                    columns={[
                        {
                            key: 'checkbox',
                            title: (
                                <input
                                    type="checkbox"
                                    checked={selectAll}
                                    onChange={handleSelectAll}
                                />
                            ),
                            width: 50,
                        },
                        ...columns,
                    ]}
                    sortColumn={sortColumn}
                    sortDirection={sortDirection}
                    onSort={handleSort}
                />
            </div>

            {/* 表体 */}
            <div ref={containerRef} className="table-body" style={{ height }}>
                <List
                    height={height}
                    width={Math.max(containerWidth, totalWidth)}
                    itemCount={sortedData.length}
                    itemSize={rowHeight}
                    itemData={itemData}
                    overscanCount={10}
                >
                    {({ index, style }) => {
                        const row = sortedData[index];
                        const isSelected = selectedIds.has(row.id);

                        return (
                            <div
                                style={{
                                    ...style,
                                    display: 'flex',
                                    borderBottom: '1px solid #eee',
                                }}
                                className={`table-row ${isSelected ? 'selected' : ''}`}
                            >
                                <div
                                    style={{ width: 50, minWidth: 50 }}
                                    className="table-cell center"
                                    onClick={(e) => {
                                        e.stopPropagation();
                                        handleSelectRow(row.id);
                                    }}
                                >
                                    <input
                                        type="checkbox"
                                        checked={isSelected}
                                        onChange={() => handleSelectRow(row.id)}
                                    />
                                </div>
                                {columns.map((column) => {
                                    const value = row[column.key as keyof TableRow];
                                    return (
                                        <div
                                            key={String(column.key)}
                                            className={`table-cell ${column.align || 'left'}`}
                                            style={{ width: column.width, minWidth: column.width }}
                                            onClick={() => onRowClick?.(row)}
                                        >
                                            {column.render
                                                ? column.render(value, row, index)
                                                : value}
                                        </div>
                                    );
                                })}
                            </div>
                        );
                    }}
                </List>
            </div>

            {/* 统计信息 */}
            <div className="table-footer">
                <span>
                    共 {sortedData.length.toLocaleString()} 行 | 已选择{' '}
                    {selectedIds.size.toLocaleString()} 行
                </span>
            </div>

    );
```

## 总结

前端性能优化是一个系统工程，需要从多个层面综合考虑：

**页面加载性能优化要点：**
- 理解关键渲染路径，消除阻塞
- 合理使用preload/prefetch
- 图片格式选择与懒加载
- 代码分割与Tree Shaking

**运行时性能优化要点：**
- 避免重排重绘，使用transform/opacity
- 事件委托减少监听器
- 虚拟滚动处理大数据
- Web Worker分离主线程

**React性能优化要点：**
- 合理使用memo、useMemo、useCallback
- 列表渲染优化与key选择
- 状态管理与Context分离
- 组件卸载清理

**网络性能优化要点：**
- HTTP缓存策略配置
- CDN与DNS预解析
- HTTP/2多路复用

**性能监控要点：**
- Core Web Vitals监控
- Performance API使用
- Lighthouse审计
- RUM真实用户监控

持续的性能监控和迭代优化是保持应用高性能的关键。建议建立性能预算制度，在CI/CD流程中集成性能测试，确保每次发布都不会引入性能退化。
