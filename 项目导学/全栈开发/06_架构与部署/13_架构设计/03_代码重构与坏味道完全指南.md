# 代码重构与坏味道完全指南

## 前言：为什么要重构？

想象你住在一间房子里。刚搬进来时，一切都井井有条。但随着时间推移，你随手放置物品，渐渐地东西越来越多、越来越乱。厨房的调料瓶堆在一起找不到，衣柜里的衣服乱七八糟，抽屉里的杂物堆积如山。这时你会意识到：需要整理一下了。代码也是一样的——重构就是"整理代码"的过程。

重构不是为了添加新功能，而是让现有的代码更易于理解、更容易修改。就像整理房间不会改变你拥有的东西，只是让东西更容易找到。重构不改变代码的外在行为，只是让内部结构更合理。

本文档将带你深入理解代码的"坏味道"、重构手法和自动化工具，让你的代码从"能跑就行"变成"优雅可维护"。

---

## 一、代码坏味道概述

### 1.1 什么是代码坏味道？

代码坏味道（Code Smell）是指代码中潜在的问题模式，这些模式可能表明代码需要重构，但不一定意味着代码有bug。坏味道是代码中的一些"症状"，提示开发者这里可能需要改进。

**生活类比**：就像身体的"亚健康"状态。你可能没有生病，但有些信号在提醒你需要注意：比如经常疲劳、睡眠不好、食欲不振。这些"小问题"积累多了，可能会导致大病。代码坏味道也是一样，单个可能没问题，但积累多了会让代码变得难以维护。

### 1.2 为什么要识别坏味道？

1. **提高可维护性**：坏味道往往意味着代码难以理解和修改
2. **降低技术债务**：及时处理坏味道，避免债务积累
3. **提高代码质量**：让代码更清晰、更健壮
4. **便于团队协作**：统一的代码标准让协作更顺畅

## 二、常见代码坏味道详解

### 2.1 重复代码（Duplicated Code）

**描述**：同一个代码逻辑在多个地方重复出现。

**问题**：
- 修改时需要改多处，容易遗漏
- 增加代码量
- 违反DRY原则（Don't Repeat Yourself）

**生活类比**：就像复印文件。如果原件改了，你需要重新复印所有副本，否则就会有过期版本在流通。

**识别方法**：在IDE中搜索重复代码块，或者使用代码相似度检测工具。

### 2.2 过长函数（Long Method）

**描述**：函数包含太多行代码，承担了太多职责。

**问题**：
- 难以理解和阅读
- 难以测试和维护
- 违反单一职责原则

**生活类比**：就像一本没有章节的书，所有内容都挤在一起。一本好的小说会分章节，每章讲述一个相对独立的故事。函数也应该像书的章节一样，每个函数讲述一个"故事"。

```typescript
/**
 * 重构前：过长函数
 */

// 处理订单的函数，有100多行
function processOrder(orderId: string): void {
  // 1. 验证订单
  console.log('开始处理订单...');

  // 从数据库获取订单
  const order = database.findOrder(orderId);
  if (!order) {
    console.log('订单不存在');
    return;
  }

  // 2. 验证订单状态
  if (order.status !== 'pending') {
    console.log('订单状态不正确');
    return;
  }

  // 3. 检查库存
  console.log('检查库存...');
  for (const item of order.items) {
    const stock = database.getStock(item.productId);
    if (stock < item.quantity) {
      console.log(`商品 ${item.productId} 库存不足`);
      return;
    }
  }

  // 4. 计算价格
  console.log('计算价格...');
  let total = 0;
  for (const item of order.items) {
    total += item.price * item.quantity;
  }

  // 应用折扣
  if (total > 1000) {
    total = total * 0.9; // 9折
  } else if (total > 500) {
    total = total * 0.95; // 95折
  }

  // 5. 处理支付
  console.log('处理支付...');
  if (order.paymentMethod === 'credit_card') {
    // 信用卡支付逻辑
    console.log('使用信用卡支付');
    const paymentResult = paymentGateway.charge(order.userId, total);
    if (!paymentResult.success) {
      console.log('支付失败');
      return;
    }
  } else if (order.paymentMethod === 'alipay') {
    // 支付宝支付逻辑
    console.log('使用支付宝支付');
    const paymentResult = alipayGateway.pay(orderId, total);
    if (!paymentResult.success) {
      console.log('支付失败');
      return;
    }
  } else if (order.paymentMethod === 'wechat') {
    // 微信支付逻辑
    console.log('使用微信支付');
    const paymentResult = wechatPayGateway.pay(orderId, total);
    if (!paymentResult.success) {
      console.log('支付失败');
      return;
    }
  }

  // 6. 更新库存
  console.log('更新库存...');
  for (const item of order.items) {
    database.reduceStock(item.productId, item.quantity);
  }

  // 7. 更新订单状态
  console.log('更新订单状态...');
  order.status = 'completed';
  order.paidAmount = total;
  order.paidAt = new Date();
  database.saveOrder(order);

  // 8. 发送通知
  console.log('发送通知...');
  const user = database.getUser(order.userId);
  emailService.send(user.email, '订单完成', '您的订单已完成支付');
  smsService.send(user.phone, '您的订单已完成支付');

  // 9. 记录日志
  console.log(`[${new Date()}] 订单处理完成: ${orderId}`);
}

/**
 * 重构后：拆分为多个小函数
 */

class OrderProcessor {
  /**
   * 主处理函数：协调各子步骤
   */
  processOrder(orderId: string): void {
    console.log('开始处理订单...');

    // 获取并验证订单
    const order = this.fetchAndValidateOrder(orderId);
    if (!order) return;

    // 检查库存
    if (!this.checkInventory(order)) return;

    // 处理支付
    const totalAmount = this.calculateTotal(order);
    if (!this.processPayment(order, totalAmount)) return;

    // 更新库存和订单
    this.updateInventory(order);
    this.completeOrder(order, totalAmount);

    // 发送通知并记录日志
    this.notifyAndLog(order);
  }

  /**
   * 获取并验证订单
   */
  private fetchAndValidateOrder(orderId: string): Order | null {
    const order = database.findOrder(orderId);
    if (!order) {
      console.log('订单不存在');
      return null;
    }

    if (order.status !== 'pending') {
      console.log('订单状态不正确');
      return null;
    }

    return order;
  }

  /**
   * 检查库存
   */
  private checkInventory(order: Order): boolean {
    console.log('检查库存...');
    for (const item of order.items) {
      const stock = database.getStock(item.productId);
      if (stock < item.quantity) {
        console.log(`商品 ${item.productId} 库存不足`);
        return false;
      }
    }
    return true;
  }

  /**
   * 计算总价
   */
  private calculateTotal(order: Order): number {
    console.log('计算价格...');
    let total = order.items.reduce(
      (sum, item) => sum + item.price * item.quantity,
      0
    );

    // 应用折扣
    if (total > 1000) {
      total = total * 0.9;
    } else if (total > 500) {
      total = total * 0.95;
    }

    return total;
  }

  /**
   * 处理支付
   */
  private processPayment(order: Order, totalAmount: number): boolean {
    console.log('处理支付...');

    const paymentStrategies: Record<string, () => PaymentResult> = {
      credit_card: () => paymentGateway.charge(order.userId, totalAmount),
      alipay: () => alipayGateway.pay(order.id, totalAmount),
      wechat: () => wechatPayGateway.pay(order.id, totalAmount)
    };

    const paymentStrategy = paymentStrategies[order.paymentMethod];
    if (!paymentStrategy) {
      console.log('未知的支付方式');
      return false;
    }

    const result = paymentStrategy();
    if (!result.success) {
      console.log('支付失败');
      return false;
    }

    return true;
  }

  /**
   * 更新库存
   */
  private updateInventory(order: Order): void {
    console.log('更新库存...');
    for (const item of order.items) {
      database.reduceStock(item.productId, item.quantity);
    }
  }

  /**
   * 完成订单
   */
  private completeOrder(order: Order, totalAmount: number): void {
    console.log('更新订单状态...');
    order.status = 'completed';
    order.paidAmount = totalAmount;
    order.paidAt = new Date();
    database.saveOrder(order);
  }

  /**
   * 发送通知并记录日志
   */
  private notifyAndLog(order: Order): void {
    console.log('发送通知...');
    const user = database.getUser(order.userId);
    emailService.send(user.email, '订单完成', '您的订单已完成支付');
    smsService.send(user.phone, '您的订单已完成支付');

    console.log(`[${new Date()}] 订单处理完成: ${order.id}`);
  }
}
```

### 2.3 过大类（Large Class）

**描述**：一个类承担了太多职责，代码行数过多。

**问题**：
- 难以理解和维护
- 违反单一职责原则
- 难以测试

**生活类比**：就像一个"全能型"员工，什么都做——前台接待、销售、财务、客服。时间久了，这个人会精疲力尽，而且每件事都做不好。应该让专业的人做专业的事。

```typescript
 * 重构后：拆分为多个类
 */

// 1. 用户数据类（纯数据）
interface UserData {
  id: string;
  name: string;
  email: string;
  phone: string;
  password: string;
  createdAt: Date;
  updatedAt: Date;
}

// 2. 用户验证类
class UserValidator {
  validateEmail(email: string): boolean {
    return email.includes('@');
  }

  validatePhone(phone: string): boolean {
    return /^1\d{10}$/.test(phone);
  }

  validatePassword(password: string): boolean {
    return password.length >= 6;
  }
}

// 3. 用户业务逻辑类
class UserBusinessLogic {
  constructor(private userData: UserData) {}

  calculateAge(): number {
    const now = new Date();
    const birthDate = new Date(this.userData.createdAt);
    return now.getFullYear() - birthDate.getFullYear();
  }

  isAdult(): boolean {
    return this.calculateAge() >= 18;
  }

  canAccessPremiumFeature(): boolean {
    return this.userData.membershipLevel === 'premium';
  }
}

// 4. 用户仓储类
class UserRepository {
  constructor(private userData: UserData) {}

  save(): void {
    console.log('保存用户到数据库');
  }

  delete(): void {
    console.log('从数据库删除用户');
  }

  static findById(id: string): UserData {
    console.log(`从数据库查询用户 ${id}`);
    return { id, name: '', email: '', phone: '', password: '', createdAt: new Date(), updatedAt: new Date() };
  }

  static findByEmail(email: string): UserData {
    console.log(`从数据库查询用户 ${email}`);
    return { id: '', name: '', email: '', phone: '', password: '', createdAt: new Date(), updatedAt: new Date() };
  }
}

// 5. 用户缓存类
class UserCache {
  private cache = new Map<string, UserData>();

  cacheUser(user: UserData): void {
    this.cache.set(user.id, user);
    console.log(`缓存用户 ${user.id}`);
  }

  invalidateCache(userId: string): void {
    this.cache.delete(userId);
    console.log(`清除缓存 ${userId}`);
  }

  getFromCache(userId: string): UserData | null {
    return this.cache.get(userId) || null;
  }
}

// 6. 用户序列化类
class UserSerializer {
  toJSON(user: UserData): string {
    return JSON.stringify(user);
  }

  fromJSON(json: string): UserData {
    return JSON.parse(json);
  }
}

// 7. 用户门面类（组合以上所有类，提供简单接口）
class UserFacade {
  private validator = new UserValidator();
  private cache = new UserCache();

  async createUser(data: UserData): Promise<UserData> {
    // 验证数据
    if (!this.validator.validateEmail(data.email)) {
      throw new Error('无效的邮箱');
    }
    if (!this.validator.validatePhone(data.phone)) {
      throw new Error('无效的手机号');
    }

    // 保存到数据库
    const repository = new UserRepository(data);
    repository.save();

    // 缓存
    this.cache.cacheUser(data);

    return data;
  }

  async getUser(userId: string): Promise<UserData | null> {
    // 先查缓存
    const cached = this.cache.getFromCache(userId);
    if (cached) {
      return cached;
    }

    // 查数据库
    const user = UserRepository.findById(userId);
    if (user) {
      this.cache.cacheUser(user);
    }

    return user;
  }
}
```

### 2.5 依恋情结（Feature Envy）

**描述**：一个类的方法对另一个类的依赖过多，总是调用另一个类的数据。

**问题**：
- 违反单一职责原则
- 类之间耦合度高
- 难以独立测试

**生活类比**：就像一个总是打听邻居家事情的八卦爱好者。他不关心自己家的事，反而对邻居家的生活了如指掌。代码中的"依恋情结"是指一个类总是"关心"另一个类的数据，而不是自己的。

```typescript
/**
 * 重构前：依恋情结
 */

class Order {
  items: { productId: string; quantity: number; price: number }[];
  customerId: string;
}

class OrderPrinter {
  // 这个方法对Order类有"依恋"，总是获取Order的数据
  printOrder(order: Order): void {
    console.log('========== 订单信息 ==========');
    console.log(`订单号: ${order.customerId}-${order.items.length}`);

    // 计算总价（属于Order的职责）
    let total = 0;
    for (const item of order.items) {
      total += item.price * item.quantity;
    }

    console.log(`商品数量: ${order.items.length}`);
    console.log(`总价: ¥${total.toFixed(2)}');
    console.log('============================');
  }
}

/**
 * 重构后：将计算逻辑移到Order类
 */

class OrderRefactored {
  items: { productId: string; quantity: number; price: number }[];
  customerId: string;

  // 把计算逻辑移到Order类内部
  calculateTotal(): number {
    return this.items.reduce(
      (sum, item) => sum + item.price * item.quantity,
      0
    );
  }

  // 添加打印方法到Order类
  print(): void {
    console.log('========== 订单信息 ==========');
    console.log(`订单号: ${this.customerId}-${this.items.length}`);
    console.log(`商品数量: ${this.items.length}`);
    console.log(`总价: ¥${this.calculateTotal().toFixed(2)}`);
    console.log('============================');
  }

```

### 2.6 Switch语句（Switch Statements）

**描述**：过多使用switch语句处理不同情况。

**问题**：
- 违反开闭原则（添加新case需要修改原有代码）
- 难以扩展
- 重复代码

**生活类比**：就像一个餐厅的菜单，如果有100种菜，每种菜都有一个独立的switch分支。当厨师要加一个新菜时，需要在所有地方都加一条case。更好的做法是用"菜谱"来代替这种分支结构。

```typescript
/**
 * 重构前：过多的switch语句
 */

class Employee {
  type: 'engineer' | 'manager' | 'sales';
}

// 计算员工奖金的函数
function calculateBonus(employee: Employee, amount: number): number {
  let bonus = 0;

  // 根据员工类型计算奖金
  switch (employee.type) {
    case 'engineer':
      bonus = amount * 1.5;
      break;
    case 'manager':
      bonus = amount * 2.0;
      break;
    case 'sales':
      bonus = amount * 2.5;
      break;
  }

  // 根据员工类型计算提成
  let commission = 0;
  switch (employee.type) {
    case 'engineer':
      commission = 0; // 工程师没有提成
      break;
    case 'manager':
      commission = amount * 0.1;
      break;
    case 'sales':
      commission = amount * 0.2;
      break;
  }

  return bonus + commission;
}

/**
 * 重构后：使用策略模式或对象映射
 */

// 奖金策略接口
interface BonusStrategy {
  calculateBonus(amount: number): number;
  calculateCommission(amount: number): number;
}

// 工程师策略
class EngineerStrategy implements BonusStrategy {
  calculateBonus(amount: number): number {
    return amount * 1.5;
  }

  calculateCommission(amount: number): number {
    return 0;
  }
}

// 经理策略
class ManagerStrategy implements BonusStrategy {
  calculateBonus(amount: number): number {
    return amount * 2.0;
  }

  calculateCommission(amount: number): number {
    return amount * 0.1;
  }
}

// 销售策略
class SalesStrategy implements BonusStrategy {
  calculateBonus(amount: number): number {
    return amount * 2.5;
  }

  calculateCommission(amount: number): number {
    return amount * 0.2;
  }
}

// 策略工厂
class EmployeeStrategyFactory {
  private static strategies = {
    engineer: new EngineerStrategy(),
    manager: new ManagerStrategy(),
    sales: new SalesStrategy()
  };

  static getStrategy(type: string): BonusStrategy {
    return this.strategies[type as keyof typeof this.strategies];
  }
}

// 重构后的计算函数
function calculateBonusRefactored(employee: Employee, amount: number): number {
  const strategy = EmployeeStrategyFactory.getStrategy(employee.type);
  return strategy.calculateBonus(amount) + strategy.calculateCommission(amount);
}
```

### 2.7 平行继承体系（Parallel Inheritance Hierarchies）

**描述**：每当为一个类添加子类时，必须同时为另一个相关的类添加子类。

**问题**：
- 类数量爆炸
- 难以维护
- 违反DRY原则

**生活类比**：就像一个公司的组织架构，每个部门都有一个"正式员工"和"实习生"版本。如果新增一个部门，就需要同时创建这两个版本。这种情况应该合并成一个类。

```typescript
/**
 * 重构前：平行继承体系
 */

// 左侧继承体系
class LinuxButton {}
class LinuxMenu {}
class LinuxDialog {}

// 右侧继承体系 - 每新增一个操作系统，就需要新增一套类
class WindowsButton {}
class WindowsMenu {}
class WindowsDialog {}

// 使用时需要根据操作系统选择
function createUI(os: 'linux' | 'windows') {
  if (os === 'linux') {
    return new LinuxDialog();
  } else {
    return new WindowsDialog();
  }
}

/**
 * 重构后：合并为一个类，使用属性区分
 */

// 统一的UI组件类
class Button {
  constructor(private os: string) {}

  render(): void {
    console.log(`渲染 ${this.os} 风格的按钮`);
  }
}

class Menu {
  constructor(private os: string) {}

  render(): void {
    console.log(`渲染 ${this.os} 风格的菜单`);
  }
}

class Dialog {
  constructor(private os: string) {}

  render(): void {
    console.log(`渲染 ${this.os} 风格的对话框`);
  }
}

// UI工厂
class UIFactory {
  constructor(private os: string) {}

  createButton(): Button {
    return new Button(this.os);
  }

  createMenu(): Menu {
    return new Menu(this.os);
  }

  createDialog(): Dialog {
    return new Dialog(this.os);
  }
}

// 使用
function createUI(os: string) {
  return new UIFactory(os);
}

const linuxUI = createUI('linux');
linuxUI.createDialog().render();
```

### 2.8 冗余类（Lazy Class）

**描述**：一个类做得太少，几乎没有什么实际的职责。

**问题**：
- 增加系统复杂度
- 浪费资源
- 难以理解

**生活类比**：就像一个公司里有一些"闲人"，他们做的事情很少，但公司需要花时间管理他们。如果一个类的存在价值不大，应该删除它。

```typescript
/**
 * 重构前：冗余类
 */

// 这个类几乎没有实际功能
class OrderUtil {
  static calculateTotal(order: Order): number {
    return order.items.reduce((sum, item) => sum + item.price * item.quantity, 0);
  }
}

// 使用时
const total = OrderUtil.calculateTotal(order);

// 如果这个类只有这一个方法，而且只用了一次
// 完全可以把这个方法移到Order类中

/**
 * 重构后：删除冗余类
 */

class Order {
  items: { price: number; quantity: number }[];

  // 直接把工具方法移到Order类中
  getTotal(): number {
    return this.items.reduce((sum, item) => sum + item.price * item.quantity, 0);
  }
}

// 使用时
const total = order.getTotal();
```

### 2.9 可疑注释（Speculative Generality）

**描述**：为了将来可能的需求写了注释或代码，但这些"扩展性"从未被使用。

**问题**：
- 增加复杂度
- 造成困惑
- 浪费资源

**生活类比**：就像为了"将来可能开车"提前考了个驾照，但你从来不打算开车。这不仅浪费了时间和金钱，还占用了抽屉的空间。代码中的"可疑注释"也是一样的。

```typescript
/**
 * 重构前：可疑注释
 */

/**
 * 这个类用于处理订单
 *
 * TODO: 将来可能需要支持批量订单处理
 * TODO: 可能需要添加订单取消功能
 * TODO: 可能需要添加订单合并功能
 *
 * @deprecated 请使用 OrderHandlerV2
 */
class OrderHandler {
  process(order: Order): void {
    // 只处理单个订单
  }
}

/**
 * 重构后：删除未使用的注释和代码
 */

class OrderHandler {
  process(order: Order): void {
    // 处理单个订单
  }
}
```

### 2.10 临时字段（Temporary Field）

**描述**：类的某个字段只有在特定情况下才有值，其他时候都是空的。

**问题**：
- 代码难以理解
- 可能导致空指针异常
- 违反单一职责原则

**生活类比**：就像一个抽屉，大部分时候是空的，但偶尔会放一些"临时物品"。更好的做法是为这些临时物品准备一个专门的盒子。

```typescript
/**
 * 重构前：临时字段
 */

class Order {
  items: OrderItem[];
  status: string;

  // 临时字段：只有在订单"处理中"状态时才有用
  processingStartTime: Date | null = null;
  processingEndTime: Date | null = null;
  processingResult: string | null = null;

  // 问题：如果在订单状态为"已创建"时访问这些字段，会得到null
}

/**
 * 重构后：使用对象或状态模式
 */

// 创建一个独立的处理状态对象
class ProcessingState {
  constructor(
    public startTime: Date,
    public endTime?: Date,
    public result?: string
  ) {}

  isCompleted(): boolean {
    return this.endTime !== undefined;
  }
}

class Order {
  items: OrderItem[];
  status: string;

  // 使用单独的Map存储与状态相关的数据
  private states: Map<string, ProcessingState> = new Map();

  // 设置处理状态
  setProcessingState(state: ProcessingState): void {
    this.states.set('processing', state);
  }

  // 获取处理状态
  getProcessingState(): ProcessingState | undefined {
    return this.states.get('processing');
  }
}
```

## 三、重构手法详解

### 3.1 提取函数（Extract Function）

**目的**：将长函数拆分为多个小函数。

**操作步骤**：
1. 识别函数中可以独立存在的代码块
2. 给新函数起一个描述性的名字
3. 将代码块提取到新函数
4. 将原代码替换为函数调用
5. 处理变量作用域

```typescript
/**
 * 重构前
 */
function printOwing(invoice: Invoice): void {
  let outstanding = 0;

  // 打印标题
  console.log('========== 账单 ==========');
  console.log(`客户: ${invoice.customer}`);

  // 计算欠款
  for (const order of invoice.orders) {
    outstanding += order.amount;
  }

  // 打印欠款
  console.log(`欠款: ¥${outstanding.toFixed(2)}`);
  console.log('========================');
}

/**
 * 重构后：提取多个函数
 */
function printOwing(invoice: Invoice): void {
  printBanner();
  const outstanding = calculateOutstanding(invoice);
  printDetails(invoice, outstanding);
}

function printBanner(): void {
  console.log('========== 账单 ==========');
  console.log(`客户: ${invoice.customer}`);
  console.log('========================');
}

function calculateOutstanding(invoice: Invoice): number {
  return invoice.orders.reduce((sum, order) => sum + order.amount, 0);
}

function printDetails(invoice: Invoice, outstanding: number): void {
  console.log(`欠款: ¥${outstanding.toFixed(2)}`);
}
```

### 3.2 提取变量（Extract Variable）

**目的**：将复杂表达式提取为有意义的变量。

```typescript
/**
 * 重构前
 */
function calculateOrderTotal(order: Order): number {
  // 复杂的表达式，难以理解
  return order.subtotal * (1 - order.discount) * (1 + order.taxRate) + order.shippingFee;
}

/**
 * 重构后
 */
function calculateOrderTotal(order: Order): number {
  const afterDiscount = order.subtotal * (1 - order.discount);
  const withTax = afterDiscount * (1 + order.taxRate);
  return withTax + order.shippingFee;
}
```

### 3.3 内联函数（Inline Function）

**目的**：将过于简单的函数合并到调用处。

```typescript
/**
 * 重构前：过于简单的函数
 */
function getAdjustedPrice(price: number): number {
  return price * 0.9;
}

function printPrice(): void {
  const price = 100;
  const adjustedPrice = getAdjustedPrice(price);
  console.log(`价格: ${adjustedPrice}`);
}

/**
 * 重构后：内联
 */
function printPrice(): void {
  const price = 100;
  const adjustedPrice = price * 0.9; // 直接计算，更清晰
  console.log(`价格: ${adjustedPrice}`);
}
```

### 3.4 引入参数对象（Introduce Parameter Object）

**目的**：将一组相关参数封装为对象。

### 3.5 函数组合成类（Combine Functions into Class）

**目的**：将一组相关的函数组织到一个类中。

```typescript
/**
 * 重构前：散落的函数
 */
function baseCharge(usage: number): number {
  const base = 100;
  const perUnit = 0.5;
  return base + usage * perUnit;
}

function chargeModifier(usage: number): number {
  if (usage > 100) {
    return usage * 0.1;
  }
  return 0;
}

function calculateTotalCharge(usage: number): number {
  return baseCharge(usage) + chargeModifier(usage);
}

/**
 * 重构后：组合成类
 */
class ElectricityBill {
  constructor(private usage: number) {}

  private get baseCharge(): number {
    const base = 100;
    const perUnit = 0.5;
    return base + this.usage * perUnit;
  }

  private get chargeModifier(): number {
    if (this.usage > 100) {
      return this.usage * 0.1;
    }
    return 0;
  }

  get totalCharge(): number {
    return this.baseCharge + this.chargeModifier;
  }
}
```

### 3.6 拆分阶段（Split Phase）

**目的**：将一个函数分为两个独立的阶段。

```typescript
/**
 * 重构前：混合了两个阶段
 */
function priceOrder(product: Product, quantity: number, discountRate: number): number {
  // 第一阶段：计算原价
  const basePrice = product.price * quantity;

  // 第二阶段：应用折扣
  const discount = basePrice * discountRate;

  // 第三阶段：计算税费
  const tax = (basePrice - discount) * 0.1;

  return basePrice - discount + tax;
}

/**
 * 重构后：拆分为两个函数
 */
function calculateBasePrice(product: Product, quantity: number): number {
  return product.price * quantity;
}

function applyPricing(basePrice: number, discountRate: number): { discounted: number; tax: number } {
  const discount = basePrice * discountRate;
  const tax = (basePrice - discount) * 0.1;
  return { discounted: basePrice - discount, tax };
}

function priceOrder(product: Product, quantity: number, discountRate: number): number {
  const basePrice = calculateBasePrice(product, quantity);
  const { discounted, tax } = applyPricing(basePrice, discountRate);
  return discounted + tax;
}
```

### 3.7 移动函数（Move Function）

**目的**：将函数移动到更合适的类中。

```typescript
/**
 * 重构前：函数位置不当

/**
 * 重构后：移动到正确的类

  name: string;
  orders: Order[];

  getTotalSpend(): number {
    return this.orders.reduce((sum, order) => sum + order.getTotal(), 0);
  }
}

```

### 3.8 搬移语句到函数（Move Statements into Function）

**目的**：将重复的语句移动到函数内部。


### 3.9 用函数替换临时变量（Replace Temp with Query）

**目的**：将临时变量替换为函数调用。

```typescript
/**
 * 重构前
 */
function calculateOrderTotal(order: Order): number {
  // 临时变量
  const basePrice = order.items.reduce((sum, item) => sum + item.price * item.quantity, 0);
  const discount = basePrice > 1000 ? basePrice * 0.1 : 0;
  const shipping = basePrice > 500 ? 0 : 50;

  return basePrice - discount + shipping;
}

/**
 * 重构后：使用函数
 */
function calculateOrderTotal(order: Order): number {
  const basePrice = this.getBasePrice(order);
  const discount = this.getDiscount(basePrice);
  const shipping = this.getShippingFee(basePrice);

  return basePrice - discount + shipping;
}

private getBasePrice(order: Order): number {
  return order.items.reduce((sum, item) => sum + item.price * item.quantity, 0);
}

private getDiscount(basePrice: number): number {
  return basePrice > 1000 ? basePrice * 0.1 : 0;
}

private getShippingFee(basePrice: number): number {
  return basePrice > 500 ? 0 : 50;
}
```

### 3.10 拆分循环（Split Loop）

**目的**：将包含多个功能的循环拆分为多个循环。

```typescript
/**
 * 重构前：循环做了太多事
 */
function processOrders(orders: Order[]): void {
  let totalAmount = 0;
  let itemCount = 0;
  let expensiveOrders = 0;

  for (const order of orders) {
    // 计算总额
    totalAmount += order.total;

    // 计算商品数
    itemCount += order.items.length;

    // 统计高价订单
    if (order.total > 1000) {
      expensiveOrders++;
    }
  }

  console.log(`总额: ${totalAmount}, 商品数: ${itemCount}, 高价订单: ${expensiveOrders}`);
}

/**
 * 重构后：拆分为多个循环
 */
function processOrders(orders: Order[]): void {
  const totalAmount = calculateTotalAmount(orders);
  const itemCount = calculateItemCount(orders);
  const expensiveOrders = countExpensiveOrders(orders);

  console.log(`总额: ${totalAmount}, 商品数: ${itemCount}, 高价订单: ${expensiveOrders}`);
}

function calculateTotalAmount(orders: Order[]): number {
  return orders.reduce((sum, order) => sum + order.total, 0);
}

function calculateItemCount(orders: Order[]): number {
  return orders.reduce((count, order) => count + order.items.length, 0);
}

function countExpensiveOrders(orders: Order[]): number {
  return orders.filter(order => order.total > 1000).length;
}
```

### 4.4 测试驱动重构流程

```typescript
/**

 * 1. 确保有测试覆盖
 * 2. 小步重构
 * 3. 每步后运行测试

// 示例：重构一个计算函数
describe('重构流程', () => {
  // 第一步：确保有测试覆盖

    const order = {
      items: [
        { price: 10, quantity: 2 },
        { price: 5, quantity: 3 }
      ]
    };

    // 原始实现

    expect(total).toBe(35);
  });

  // 第二步：提取为函数
  test('calculateTotal 重构后仍然正确', () => {
    const order = {

        { price: 5, quantity: 3 }

});

function calculateTotal(order: { items: { price: number; quantity: number }[] }): number {
```

```typescript
/**
 * 订单实体
 */
class Order {
  readonly id: string;
  readonly customerId: string;
  private _status: OrderStatus;
  private _items: OrderItem[];
  private _total: number;
  readonly createdAt: Date;
  private updatedAt: Date;
  private paidAt: Date | null;
  private cancelledAt: Date | null;
  private cancelReason: string | null;
  private paymentId: string | null;

  constructor(id: string, customerId: string) {
    this.id = id;
    this.customerId = customerId;
    this._status = OrderStatus.CREATED;
    this._items = [];
    this._total = 0;
    this.createdAt = new Date();
    this.updatedAt = new Date();
    this.paidAt = null;
    this.cancelledAt = null;
    this.cancelReason = null;
    this.paymentId = null;
  }

  get status(): OrderStatus { return this._status; }
  get items(): OrderItem[] { return [...this._items]; }
  get total(): number { return this._total; }

  /**
   * 添加订单项
   */
  addItem(item: OrderItem): void {
    if (this._status !== OrderStatus.CREATED) {
      throw new Error('只能向创建状态的订单添加商品');
    }
    this._items.push(item);
    this.recalculateTotal();
  }

  /**
   * 支付订单
   */
  pay(paymentId: string): void {
    if (this._status !== OrderStatus.CREATED) {
      throw new Error('订单状态不正确');
    }
    if (this._items.length === 0) {
      throw new Error('订单不能为空');
    }

    this._status = OrderStatus.PAID;
    this.paymentId = paymentId;
    this.paidAt = new Date();
    this.updatedAt = new Date();
  }

  /**
   * 取消订单
   */
  cancel(reason: string): void {
    if (this._status === OrderStatus.SHIPPED || this._status === OrderStatus.COMPLETED) {
      throw new Error('已发货或完成的订单不能取消');
    }

    this._status = this._status === OrderStatus.PAID
      ? OrderStatus.REFUNDING
      : OrderStatus.CANCELLED;
    this.cancelledAt = new Date();
    this.cancelReason = reason;
    this.updatedAt = new Date();
  }

  private recalculateTotal(): void {
    this._total = this._items.reduce((sum, item) => sum + item.subtotal, 0);
  }
}

class OrderApplicationService {
  constructor(
    private orderRepository: IOrderRepository,
    private paymentService: IPaymentService,
    private notificationService: INotificationService,
    private logger: ILogger
  ) {}

  /**
   * 创建订单
   */
  async createOrder(command: CreateOrderCommand): Promise<OrderDTO> {
    this.logger.info('创建订单', { customerId: command.customerId });

    // 创建订单
    const orderId = this.generateOrderId();
    const order = new Order(orderId, command.customerId);

    // 添加订单项
    for (const item of command.items) {
      order.addItem(item);
    }

    // 持久化
    await this.orderRepository.save(order);

    // 发送通知
    await this.notificationService.notify(
      order.customerId,
      '您的订单已创建'
    );

    this.logger.info('订单创建成功', { orderId });

    return this.toDTO(order);
  }

  /**
   * 支付订单
   */
  async payOrder(orderId: string, customerId: string, paymentMethod: string): Promise<OrderDTO> {
    this.logger.info('支付订单', { orderId, paymentMethod });

    const order = await this.orderRepository.findById(orderId);
    if (!order) {
      throw new Error('订单不存在');
    }

    if (order.customerId !== customerId) {
      throw new Error('无权支付此订单');
    }

    // 处理支付
    const paymentResult = await this.paymentService.processPayment(
      order.total,
      paymentMethod,
      orderId
    );

    // 更新订单
    order.pay(paymentResult.paymentId);
    await this.orderRepository.save(order);

    // 发送通知
    await this.notificationService.notify(
      order.customerId,
      '您的订单已支付成功'
    );

    this.logger.info('订单支付成功', { orderId, paymentId: paymentResult.paymentId });

    return this.toDTO(order);
  }

  /**
   * 取消订单
   */
  async cancelOrder(orderId: string, customerId: string, reason: string): Promise<OrderDTO> {
    this.logger.info('取消订单', { orderId, reason });

    const order = await this.orderRepository.findById(orderId);
    if (!order) {
      throw new Error('订单不存在');
    }

    if (order.customerId !== customerId) {
      throw new Error('无权取消此订单');

    order.cancel(reason);
    await this.orderRepository.save(order);

    // 如果已支付，需要处理退款
    if (order.status === OrderStatus.REFUNDING) {

    // 发送通知
    await this.notificationService.notify(
      order.customerId,
      '您的订单已取消'
    );

    this.logger.info('订单取消成功', { orderId });

    return this.toDTO(order);

      status: order.status,

      total: order.total,
```

## 七、重构与性能

### 7.1 不要过早优化

> "过早优化是万恶之源" —— Donald Knuth

重构时不要过分追求性能优化，除非性能已经成为问题。应该优先保证代码的可读性和可维护性。

### 7.2 性能敏感场景

在性能敏感的场景中，需要注意：

1. **循环优化**：避免在循环中重复计算
2. **缓存优化**：使用记忆化技术
3. **数据结构**：选择合适的数据结构

```typescript
/**
 * 性能优化示例
 */

// 不好的写法：每次调用都重新计算
function getTotal(order: Order): number {
  return order.items.reduce((sum, item) => sum + item.price * item.quantity, 0);
}

// 优化后的写法：使用缓存
class CachedOrder {
  private _total: number | null = null;

  constructor(
    private order: Order
  ) {}

  get total(): number {
    if (this._total === null) {
      this._total = this.order.items.reduce(
        (sum, item) => sum + item.price * item.quantity,
        0
      );
    }
    return this._total;
  }
}
```
