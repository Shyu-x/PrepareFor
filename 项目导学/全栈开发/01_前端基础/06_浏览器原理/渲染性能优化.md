## 1. 浏览器渲染原理

### 1.1 渲染流水线概述

浏览器渲染页面是一个复杂的过程，涉及多个阶段的协同工作。理解这个流水线对于优化渲染性能至关重要。

```
┌─────────────────────────────────────────────────────────────────────┐
│                        浏览器渲染流水线                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. 解析HTML                                                        │
│     ├── 构建DOM树（Document Object Model）                            │
│     └── 构建CSSOM树（CSS Object Model）                              │
│                                                                     │
│  2. 渲染树构建                                                       │
│     ├── DOM + CSSOM = 渲染树（Render Tree）                          │
│     └── 只包含可见节点                                               │
│                                                                     │
│  3. 布局（Layout）                                                   │
│     ├── 计算每个节点的位置和大小                                       │
│     ├── 回流（Reflow）- 几何属性变化时触发                            │
│     └── 布局计算从根节点开始                                          │
│                                                                     │
│  4. 绘制（Paint）                                                    │
│     ├── 将渲染树绘制到多个图层                                         │
│     ├── 重绘（Repaint）- 外观变化时触发                               │
│     └── 不改变元素位置和大小                                           │
│                                                                     │
│  5. 合成（Composite）                                                │
│     ├── 将多个图层合成为最终画面                                       │
│     ├── GPU参与处理                                                   │
│     └── 使用合成层可以避免重排重绘                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 渲染关键路径

渲染关键路径是指从接收HTML到首屏绘制的完整过程。优化这个路径可以显著提升用户体验。

```javascript
// 分析渲染关键路径
function analyzeCriticalRenderingPath() {
  const timing = performance.getEntriesByType('navigation')[0];

  const criticalPath = {
    // DNS解析时间
    dns: timing.domainLookupEnd - timing.domainLookupStart,

    // TCP连接时间
    tcp: timing.connectEnd - timing.connectStart,

    // 请求发送时间
    request: timing.requestStart - timing.requestEnd,

    // TTFB（首字节时间）
    ttfb: timing.responseStart - timing.requestStart,

    // 内容下载时间
    download: timing.responseEnd - timing.responseStart,

    // DOM解析时间
    domParse: timing.domInteractive - timing.responseEnd,

    // 渲染树构建
    renderTree: timing.domContentLoadedEventEnd - timing.domInteractive,

    // 首次绘制
    firstPaint: getFirstPaint(),

    // 首次内容绘制
    fcp: getFCP(),

    // 首次有意义绘制
    fmp: getFMP(),

    // 可交互时间
    tti: timing.domContentLoadedEventEnd,
  };

  return criticalPath;
}

// 获取首次绘制时间
function getFirstPaint() {
  const entries = performance.getEntriesByType('paint');
  const firstPaint = entries.find(entry => entry.name === 'first-paint');
  return firstPaint ? firstPaint.startTime : null;
}

// 获取首次内容绘制时间
function getFCP() {
  const entries = performance.getEntriesByType('paint');
  const fcp = entries.find(entry => entry.name === 'first-contentful-paint');
  return fcp ? fcp.startTime : null;
}

// 监听 Largest Contentful Paint
function getFMP() {
  return new Promise((resolve) => {
    const observer = new PerformanceObserver((list) => {
      const entries = list.getEntries();
      const lastEntry = entries[entries.length - 1];
      resolve(lastEntry.startTime);
    });

    observer.observe({ type: 'largest-contentful-paint', buffered: true });
  });
}
```

## 2. 重排与重绘

### 2.1 什么是重排和重绘

重排（Reflow）和重绘（Repaint）是浏览器渲染过程中两个关键的性能问题。理解它们的区别和触发条件对于优化渲染性能至关重要。

```javascript
// 重排（Reflow）- 也称为回流
// 几何属性发生变化时触发，包括：
// - 元素的位置、尺寸（width, height, padding, margin）
// - 元素的位置变化（top, left, right, bottom）
// - 字体大小变化
// - 添加或删除可见的DOM元素
// - 浏览器窗口大小变化

// 重绘（Repaint）- 不改变布局
// 外观属性发生变化时触发，包括：
// - 颜色变化（color, background-color）
// - 边框样式变化
// - 可见性变化（visibility, opacity）

// 示例：区分重排和重绘
function demonstrateReflowAndRepaint() {
  const element = document.getElementById('box');

  // 重排 + 重绘：改变尺寸或位置
  element.style.width = '200px';  // 触发重排
  element.style.height = '200px'; // 触发重排

  // 仅重绘：改变外观
  element.style.backgroundColor = 'blue'; // 仅触发重绘
  element.style.color = 'white'; // 仅触发重绘

  // 仅重绘：visibility
  element.style.visibility = 'hidden'; // 仅触发重绘

  // 触发重排 + 重绘：display
  element.style.display = 'none'; // 触发重排 + 重绘
}
```

### 2.2 强制同步布局

强制同步布局（Forced Synchronous Layout）是性能问题的常见原因。JavaScript读取布局信息时会强制浏览器在JavaScript执行期间计算布局。

```javascript
// 强制同步布局问题

// ❌ 错误：在循环中反复读取和写入布局属性
function badPattern() {
  const elements = document.querySelectorAll('.item');

  elements.forEach((el) => {
    // 读取：强制浏览器计算布局
    const height = el.offsetHeight;

    // 写入：触发重排
    el.style.height = height + 10 + 'px';
  });
}

// ✅ 正确：先读取所有值，再批量写入
function goodPattern() {
  const elements = document.querySelectorAll('.item');

  // 批量读取：只触发一次布局计算
  const heights = Array.from(elements).map(el => el.offsetHeight);

  // 批量写入：只触发一次重排
  elements.forEach((el, i) => {
    el.style.height = heights[i] + 10 + 'px';
  });
}

// ❌ 错误：读写交替导致强制同步布局
function veryBadPattern() {
  const box = document.getElementById('box');
  const heights = [100, 200, 300, 400, 500];

  heights.forEach((h) => {
    // 每次都触发强制同步布局
    box.style.height = h + 'px';
    console.log(box.offsetHeight); // 读取触发计算
  });
}

// ✅ 正确：读写分离
function veryGoodPattern() {
  const box = document.getElementById('box');
  const heights = [100, 200, 300, 400, 500];

  // 先写入
  heights.forEach((h) => {
    box.style.height = h + 'px';
  });

  // 再读取（只触发一次布局）
  console.log(box.offsetHeight);
}

// 使用 requestAnimationFrame 优化
function optimizedPattern() {
  const elements = document.querySelectorAll('.item');

  requestAnimationFrame(() => {
    const heights = Array.from(elements).map(el => el.offsetHeight);

    requestAnimationFrame(() => {
      elements.forEach((el, i) => {
        el.style.height = heights[i] + 10 + 'px';
      });
    });
  });
}
```

### 2.3 批量DOM操作

批量DOM操作是避免频繁重排重绘的关键策略。

```javascript
// 批量DOM操作优化

// 1. 使用文档片段（DocumentFragment）
function useDocumentFragment() {
  const fragment = document.createDocumentFragment();
  const items = ['Item 1', 'Item 2', 'Item 3', 'Item 4', 'Item 5'];

  items.forEach((text) => {
    const li = document.createElement('li');
    li.textContent = text;

  // 只触发一次重排
  document.getElementById('list').appendChild(fragment);

// 2. 使用 display:none 隐藏元素后操作
function useDisplayNone() {
  const element = document.getElementById('container');
  element.style.display = 'none';

  // 多次修改
  for (let i = 0; i < 100; i++) {
    const div = document.createElement('div');
    div.textContent = `Item ${i}`;
    element.appendChild(div);
  }

  // 显示元素，只触发一次重排
  element.style.display = 'block';
}

// 3. 使用 cloneNode 克隆后替换
function useCloneNode() {
  const element = document.getElementById('container');
  const clone = element.cloneNode(true);

  // 在克隆上操作
  for (let i = 0; i < 100; i++) {
    const div = document.createElement('div');
    div.textContent = `Item ${i}`;
    clone.appendChild(div);
  }

}

// 4. 使用 CSS 类切换
function useClassToggle() {
  const element = document.getElementById('container');

  // 添加包含所有变更的类
  element.classList.add('batch-update');

  // 在CSS中处理
  // .batch-update .item { ... }
}

// 5. 使用 requestAnimationFrame 批量更新
class BatchUpdater {
  constructor() {
    this.pending = false;
    this.queue = [];
  }

  add(callback) {
    this.queue.push(callback);
    if (!this.pending) {
      this.pending = true;
      requestAnimationFrame(() => this.flush());
    }
  }

  flush() {
    this.queue.forEach(callback => callback());
    this.queue = [];
    this.pending = false;
  }
}

const batchUpdater = new BatchUpdater();

function optimizedUpdate(fn) {
  batchUpdater.add(fn);
}
```

### 3.3 useMemo与useCallback

useMemo和useCallback是React提供的两个Hook，用于缓存计算结果和函数引用，避免不必要的计算和渲染。

```javascript
import { useMemo, useCallback, useState } from 'react';

// 1. useMemo：缓存计算结果
function UseMemoExample({ data, filterText }) {
  // 依赖项变化时才重新计算
  const filteredData = useMemo(() => {
    console.log('计算过滤数据');
    return data.filter(item =>
      item.name.toLowerCase().includes(filterText.toLowerCase())
    );
  }, [data, filterText]);

  // 依赖项为[]，只在首次渲染时计算
  const sortedData = useMemo(() => {
    console.log('排序数据');
    return [...filteredData].sort((a, b) => a.name.localeCompare(b.name));
  }, [filteredData]);

  // 依赖项为[]，memoizedValue永远不变
  const config = useMemo(() => ({
    pageSize: 10,
    maxResults: 100,
    timeout: 5000,
  }), []);

  return <List data={sortedData} config={config} />;
}

// 2. useCallback：缓存函数引用
function UseCallbackExample({ onItemClick }) {
  const [count, setCount] = useState(0);

  // ❌ 错误：每次渲染都创建新函数
  const handleClick1 = () => {
    console.log('clicked');
  };

  // ✅ 正确：使用useCallback缓存函数
  const handleClick2 = useCallback(() => {
    console.log('clicked');
  }, []);

  // 依赖count的回调
  const handleClickWithCount = useCallback(() => {
    console.log('count:', count);
    onItemClick(count);
  }, [count, onItemClick]);

  return (
    <div>
      <button onClick={handleClick1}>Button 1</button>
      <button onClick={handleClick2}>Button 2</button>
      <button onClick={handleClickWithCount}>Button 3</button>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
    </div>
  );
}

// 3. useMemo在大型列表中的应用
function LargeListOptimization({ items }) {
  // 缓存expensiveSort函数的结果
  const sortedItems = useMemo(() => {
    console.log('执行expensiveSort...');
    return items.slice().sort((a, b) => {
      // 假设这是一个昂贵的排序操作
      return a.name.localeCompare(b.name);
    });
  }, [items]);

  // 缓存分组结果
  const groupedItems = useMemo(() => {
    const groups = {};
    sortedItems.forEach(item => {
      const firstLetter = item.name[0].toUpperCase();
      if (!groups[firstLetter]) {
        groups[firstLetter] = [];
      }
      groups[firstLetter].push(item);
    });
    return groups;
  }, [sortedItems]);

  return (
    <div>
      {Object.entries(groupedItems).map(([letter, group]) => (
        <div key={letter}>
          <h3>{letter}</h3>
          {group.map(item => (
            <div key={item.id}>{item.name}</div>
          ))}
        </div>
      ))}
    </div>
  );
}
```

### 3.4 状态批量更新

React 18引入了自动批量更新（Automatic Batching），显著提升了性能。

```javascript
import { useState, useEffect } from 'react';

// 1. React 18之前的批量更新
// 在React 17中，只有事件处理函数中的多次setState才会批量更新
class LegacyComponent extends React.Component {
  state = { count: 0, flag: true };

  handleClick = () => {
    // 触发两次渲染
    this.setState({ count: this.state.count + 1 }); // 渲染1
    this.setState({ flag: !this.state.flag }); // 渲染2
  };
}

// 2. React 18的自动批量更新
function ModernComponent() {
  const [count, setCount] = useState(0);
```

```javascript
function ChunkedList({ items, chunkSize = 100 }) {
  const [renderedCount, setRenderedCount] = useState(chunkSize);

  useEffect(() => {
    if (renderedCount < items.length) {
      // 使用requestIdleCallback在空闲时间渲染更多项目
      const timeoutId = requestIdleCallback ?
        requestIdleCallback(() => {
          setRenderedCount(c => Math.min(c + chunkSize, items.length));
        }) :
        setTimeout(() => {
          setRenderedCount(c => Math.min(c + chunkSize, items.length));
        }, 0);

      return () => {
        if (timeoutId && typeof timeoutId !== 'number') {
          cancelIdleCallback(timeoutId);
        } else {
          clearTimeout(timeoutId);
        }
      };
    }
  }, [renderedCount, items.length, chunkSize]);

      {items.slice(0, renderedCount).map(item => (

      ))}
```

## 7. 面试高频问题

### 问题1：什么是重排和重绘？如何避免？

**答案：**

重排（Reflow）和重绘（Repaint）是浏览器渲染过程中的两个关键概念：

**重排（Reflow）：**
- 当元素的**几何属性**（位置、大小）发生变化时触发
- 成本较高，因为它需要重新计算整个页面的布局
- 触发条件：改变元素的width、height、margin、padding、position等

**重绘（Repaint）：**
- 当元素的**外观属性**变化但不影响布局时触发
- 成本比重排低，因为不需要计算布局
- 触发条件：改变color、background、visibility、opacity等

**避免方法：**

```javascript
// 1. 批量DOM操作
const fragment = document.createDocumentFragment();
items.forEach(item => {
  const li = document.createElement('li');
  li.textContent = item;
  fragment.appendChild(li);
});
document.getElementById('list').appendChild(fragment);

// 2. 使用CSS类切换
element.classList.add('visible');

// 3. 避免强制同步布局
// ❌ 错误
elements.forEach(el => {
  el.style.height = el.offsetHeight + 10 + 'px'; // 触发重排
});

// ✅ 正确
const heights = elements.map(el => el.offsetHeight);
elements.forEach((el, i) => {
  el.style.height = heights[i] + 10 + 'px';
});

// 4. 使用transform和opacity
// ❌ 错误
element.style.left = '100px'; // 触发重排
element.style.opacity = '0.5'; // 触发重绘

// ✅ 正确
element.style.transform = 'translateX(100px)'; // 不触发重排重绘
element.style.opacity = '0.5'; // 不触发重排
```

### 问题3：CSS动画性能优化有哪些方法？

**答案：**

1. **使用transform和opacity**
   - 这两个属性不需要触发重排重绘
   - 浏览器会将其提升到GPU合成层

2. **使用will-change提示浏览器**
   ```css
   .animated {
     will-change: transform, opacity;
   }
   ```

3. **避免动画width、height等布局属性**
   - 使用transform: scale() 代替 width/height变化

4. **使用contain隔离**
   ```css
   .isolated {
     contain: layout paint;
   }
   ```

5. **减少合成层数量**
   - 过多合成层会消耗大量GPU内存

### 问题4：什么是合成层？如何创建？

**答案：**

合成层（Compositing Layer）是浏览器将页面内容分割后独立绘制的图层。

**创建合成层的方法：**

1. **3D transforms**
   ```css
   .layer {
     transform: translateZ(0);
   }
   ```

2. **will-change属性**
   ```css
   .layer {
     will-change: transform;
   }
   ```

3. **CSS animations on transform/opacity**
   ```css
   @keyframes slide {
     from { transform: translateX(0); }
     to { transform: translateX(100px); }
   }
   ```

4. **video、canvas元素**

**注意事项：**
- 不要过度使用合成层
- 动画结束后移除will-change
- 合成层会增加GPU内存消耗
