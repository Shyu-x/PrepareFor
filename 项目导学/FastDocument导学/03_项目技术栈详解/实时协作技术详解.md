# 实时协作技术详解

## 一、概述

FastDocument 采用 **多层实时协作架构**，支持多人同时编辑文档，实时同步所有更改。本文档将深入分析实际项目源码，讲解设计原理和实现细节。

### 核心技术

| 技术 | 用途 | 版本 |
|------|------|------|
| **Socket.io** | 前后端实时通信 | 4.8.3 |
| **NestJS WebSocket** | 后端 WebSocket Gateway | NestJS 11.x |
| **Yjs** | CRDT 协作编辑 | 13.6.29 |
| **LiveKit** | 视频会议 | 2.17.2 |
| **Redis** | 高速缓存和会话管理 | ioredis 5.9.3 |

## 三、Socket.io 实现详解

### 3.1 前端 Socket.io Client 深度解析

**文件路径**：`src/lib/socket.ts`

#### 3.1.1 设计原理分析

Socket.io 客户端采用**单例模式**设计，这是一种常见的设计模式，主要考虑点如下：

1. **单例模式的优点**：
   - 全局唯一连接：避免多个 Socket 连接造成资源浪费
   - 状态共享：所有组件共享同一个连接状态
   - 简化管理：连接和断开只需在一处处理

2. **为什么要封装成类**：
   - 封装内部实现细节，外部只需调用 API
   - 统一管理事件监听器，避免内存泄漏
   - 提供统一的错误处理和重连机制

#### 3.1.2 源码逐行分析

```typescript
import { io, Socket } from "socket.io-client";

/**
 * 后端服务地址，优先使用环境变量
 * 设计原理：支持多环境配置，开发/生产环境使用不同地址
 */
const SOCKET_URL = process.env.NEXT_PUBLIC_BACKEND_URL || "http://localhost:5555";

/**
 * Socket.io 客户端包装类
 * 用于管理连接、房间加入及消息收发
 * 核心设计：单例模式 + 观察者模式
 */
class SocketClient {
  // 私有属性：确保只有一个 Socket 实例
  private socket: Socket | null = null;
  // 消息监听器数组：支持多个组件同时监听同一事件
  private messageListeners: Function[] = [];
  private usersListeners: Function[] = [];

  /**
   * 建立连接并绑定基础状态监听
   *
   * 设计原理：惰性连接（Lazy Connection）
   * - 只有在首次调用时才创建连接
   * - 避免页面加载时就建立连接，减少资源消耗
   *
   * 为什么要用 onStatusChange 回调？
   * - 允许调用者自定义连接状态变化的处理逻辑
   * - 实现关注点分离
   */
  connect(onStatusChange?: (online: boolean) => void) {
    // 双重检查锁定：避免重复创建连接
    if (!this.socket) {
      // io() 创建 Socket.io 客户端实例
      this.socket = io(SOCKET_URL, {
        reconnectionAttempts: 5,    // 断线自动重连次数
        timeout: 10000,            // 连接超时时间 10 秒
      });

      // 连接成功事件
      this.socket.on("connect", () => {
        console.log("成功连接至 WebSocket 服务器");
        if (onStatusChange) onStatusChange(true);
      });

      // 断开连接事件
      this.socket.on("disconnect", () => {
        console.warn("与 WebSocket 服务器断开连接");
        if (onStatusChange) onStatusChange(false);
      });

      // 聊天消息监听（使用发布订阅模式）
      this.socket.on("chatMessage", (data) => {
        // 广播给所有订阅者
        this.messageListeners.forEach(listener => listener(data));
      });

      // 在线用户更新监听
      this.socket.on("onlineUsersUpdate", (data) => {
        this.usersListeners.forEach(listener => listener(data));
      });
    }
    return this.socket;
  }

  /**
   * 加入特定的文档协作房间
   * @param docId 文档 ID
   * @param userName 当前用户名
   *
   * 设计原理：房间（Room）隔离
   * - 每个文档是一个独立的房间
   * - 消息只在同一房间内广播
   * - 实现文档级别的权限隔离
   */
  joinDocument(docId: string, userName: string) {
    if (this.socket) {
      // 发送加入房间请求
      this.socket.emit("joinDocument", { docId, userName });
    }
  }
```

#### 3.1.3 批量更新机制深度分析

```typescript
  /**
   * 批量更新块（增量同步优化）
   * 减少网络往返次数，提高大文件编辑性能
   *
   * 为什么要批量更新？
   * - 单次更新触发一次网络往返
   * - 频繁更新会导致网络拥塞
   * - 批量更新将 N 次往返合并为 1 次
   *
   * 实际应用场景：
   * - 用户快速输入时，每字符发送会导致卡顿
   * - 批量更新收集多个变更，统一发送
   */
  updateBlocksBatch(docId: string, blocks: { id: string; content: string; type?: string; properties?: any }[]) {
    if (this.socket) {
      this.socket.emit("updateBlocksBatch", { docId, blocks });
    }
  }

  /**
   * 监听批量块更新
   *
   * 设计原理：订阅-发布模式
   * - 返回清理函数，方便 React useEffect cleanup
   * - 防止组件卸载后继续监听导致内存泄漏
   */
  onBlocksUpdated(callback: (data: { docId: string; blocks: any[] }) => void) {
    if (this.socket) {
      this.socket.on("blocksUpdated", callback);
    }
    // 返回清理函数
    return () => {
      if (this.socket) {
        this.socket.off("blocksUpdated", callback);
      }
    };
  }
```

```typescript
  /**
   * 请求分块加载（大型文档优化）
   *
   * 解决的问题：
   * - 大型文档可能包含数千个块
   * - 一次性加载所有数据会导致：

    }
  }
```

#### 3.1.5 在线状态管理分析

```typescript
  /**
   * 监听在线用户列表
   *
   * 设计亮点：多监听器支持
   * - usersListeners 数组支持多个组件同时监听
   * - 每个组件都能收到独立的回调
   *
   * 为什么要返回清理函数？
   * - React 组件可能频繁挂载/卸载
   * - 不清理会导致僵尸监听器
   * - 内存泄漏的常见原因
   */
  onUsers(callback: (users: { id: string; name: string; docId?: string }[]) => void) {
    this.usersListeners.push(callback);
    // 监听后端的 onlineUsersUpdate 事件
    if (this.socket) {
      this.socket.on('onlineUsersUpdate', (data) => {
        this.usersListeners.forEach(listener => listener(data));
      });
    }
    // 返回清理函数
    return () => {
      this.usersListeners = this.usersListeners.filter(l => l !== callback);
    };
  }
```

### 3.2 后端 WebSocket Gateway 深度解析

**文件路径**：`backend/src/documents/documents.gateway.ts`

#### 3.2.1 设计原理分析

后端 Gateway 采用 NestJS 的装饰器模式，核心设计思想：

1. **声明式路由**：通过装饰器定义事件处理
2. **依赖注入**：通过构造函数注入服务
3. **Rooms 隔离**：Socket.io 的 room 机制实现消息隔离

#### 3.2.2 源码逐行分析

```typescript
import {
  WebSocketGateway,
  SubscribeMessage,
  MessageBody,
  ConnectedSocket,
  WebSocketServer,
  OnGatewayInit,
  OnGatewayConnection,
  OnGatewayDisconnect,
} from '@nestjs/websockets';
import { Server, Socket } from 'socket.io';
import { DocumentsService } from './documents.service';
import Redis from 'ioredis';

/**
 * 实时协作网关 (实时通信 + 高速缓存)
 * 集成多端内容同步、光标状态追踪、多端写冲突检测
 *
 * 为什么要使用 Redis？
 * - 光标位置等高频更新数据不适合每次都写数据库
 * - Redis 内存数据库，读写速度极快
 * - 支持过期自动清理
 */
@WebSocketGateway({
  cors: {
    origin: '*',
    methods: ['GET', 'POST'],
    credentials: true,
  },
})
export class DocumentsGateway implements OnGatewayInit, OnGatewayConnection, OnGatewayDisconnect {
  @WebSocketServer()
  server: Server;

  // Redis 客户端：用于缓存高频访问的数据
  private redis: Redis;

  constructor(private readonly documentsService: DocumentsService) {
    // 初始化高速缓存连接，用于缓存实时状态（如光标位置）
    // 从环境变量读取 Redis 配置，支持 Docker 映射端口
    // 为什么用环境变量？支持不同部署环境使用不同配置
    const redisPort = parseInt(process.env.REDIS_PORT || '16379', 10);
    this.redis = new Redis({ host: process.env.REDIS_HOST || 'localhost', port: redisPort });
  }

  afterInit(server: Server) {
    console.log('协作网关已初始化，集成高速缓存');
  }

  // 内存中的房间用户列表（生产环境建议用 Redis）
  private roomUsers: Record<string, { id: string; name: string }[]> = {};

  /**
   * 连接处理
   * 记录新连接，用于监控和调试
   */
  handleConnection(client: Socket) {
    console.log(`用户连接: ${client.id}`);
  }

  /**
   * 断开连接处理
   * 为什么要清理 Redis 和房间状态？
   * - 防止僵尸数据占用内存
   * - 保持在线用户列表准确性
   */
  async handleDisconnect(client: Socket) {
    const userId = client.id;
    // 清理 Redis 中的光标缓存
    await this.redis.del(`cursor:${userId}`);

    // 从所有记录的房间中移除该用户
    for (const docId in this.roomUsers) {
      const initialLength = this.roomUsers[docId].length;
      this.roomUsers[docId] = this.roomUsers[docId].filter(u => u.id !== userId);

      // 如果房间人数发生变化，向该房间广播最新的用户列表
      if (this.roomUsers[docId].length !== initialLength) {
        this.server.to(docId).emit('onlineUsersUpdate', this.roomUsers[docId]);
      }
    }

    console.log(`用户断开: ${userId}`);
  }

  /**
   * 加入房间处理
   *
   * 核心逻辑：
   * 1. 将客户端加入 Socket.io 房间
   * 2. 维护房间用户列表
   * 3. 发送当前文档内容
   * 4. 广播用户加入事件
   */
  @SubscribeMessage('joinDocument')
  async handleJoinRoom(
    @MessageBody() data: { docId: string; userName: string },
    @ConnectedSocket() client: Socket,
  ) {
    const { docId, userName } = data;

    // 1. 将客户端加入文档房间
    client.join(docId);

    // 2. 初始化房间用户列表
    if (!this.roomUsers[docId]) {
      this.roomUsers[docId] = [];
    }

    // 3. 记录用户信息（避免重复添加）
    if (!this.roomUsers[docId].find(u => u.id === client.id)) {
      this.roomUsers[docId].push({ id: client.id, name: userName || '匿名用户' });
    }

    // 4. 从数据库加载文档内容
    const doc = await this.documentsService.findOne(docId);
    if (doc) {
      // 发送文档内容给请求的客户端
      client.emit('documentLoaded', doc);
    }

    // 5. 向房间内所有人广播当前在线用户列表
    this.server.to(docId).emit('onlineUsersUpdate', this.roomUsers[docId]);
    console.log(`用户 ${userName} 加入房间 ${docId}`);
  }

  /**
   * 处理块更新事件
   *
   * 设计亮点：异步持久化
   * - 先广播变更（低延迟）
   *   后持久化到数据库
   * - 用户感知延迟最小化
   * - 数据库操作不影响实时性
   */
  @SubscribeMessage('updateBlock')
  async handleUpdateBlock(
    @MessageBody() data: { docId: string; blockId: string; content: string; type?: string; properties?: any },
    @ConnectedSocket() client: Socket,
  ) {
    // 1. 立即广播变更给房间内的其他用户（实现低延迟体验）
    // 注意：发给 others，不包括发送者自己
    client.to(data.docId).emit('blockUpdated', data);

    // 2. 异步将变更持久化到数据库
    // await 等待完成，确保数据不丢失
    await this.documentsService.updateBlock(data.docId, data.blockId, data.content, data.type, data.properties);
  }

  /**
   * 批量更新块（增量同步优化）
   * 减少网络往返次数，提高大文件编辑性能
   */
  @SubscribeMessage('updateBlocksBatch')
  async handleUpdateBlocksBatch(
    @MessageBody() data: { docId: string; blocks: { id: string; content: string; type?: string; properties?: any }[] },
    @ConnectedSocket() client: Socket,
  ) {
    // 1. 立即广播变更给房间内的其他用户
    client.to(data.docId).emit('blocksUpdated', { docId: data.docId, blocks: data.blocks });

    // 2. 异步批量持久化到数据库
    await this.documentsService.updateBlocksBatch(data.docId, data.blocks);
  }

  /**
   * 请求分块加载（大型文档优化）
   *
   * 为什么要限制单次加载数量？
   * - 防止客户端请求过多数据导致内存溢出
   * - 保护数据库查询性能
   */
  @SubscribeMessage('loadBlocksChunked')
  async handleLoadBlocksChunked(
    @MessageBody() data: { docId: string; offset: number; limit: number },
    @ConnectedSocket() client: Socket,
  ) {
    const result = await this.documentsService.findBlocksChunked(data.docId, {
      offset: data.offset,
      // 安全限制：单次最多加载 200 条
      limit: Math.min(data.limit, 200),
      includeTotal: true,
    });
    client.emit('blocksChunked', { docId: data.docId, ...result });
  }

  /**
   * 增量同步请求
   *
   * 解决的问题：
   * - 网络不稳定时，用户可能离线一段时间
   * - 重连后需要获取离线期间的变更
   * - 不需要重新下载整个文档
   */
  @SubscribeMessage('syncBlocks')
  async handleSyncBlocks(
    @MessageBody() data: { docId: string; since: string },
    @ConnectedSocket() client: Socket,
  ) {
    const lastSyncTime = new Date(data.since);
    const result = await this.documentsService.findBlocksSince(data.docId, lastSyncTime);
    client.emit('blocksSynced', { docId: data.docId, ...result });
  }

  /**
   * 正在输入状态扩散
   *
   * 设计亮点：使用 Redis 缓存存储临时状态
   * - 不落地数据库（输入状态是临时的）
   * - Redis 内存存储，速度快
   * - 设置过期时间，自动清理
   */
  @SubscribeMessage('typing')
  async handleTyping(
    @MessageBody() data: { docId: string; userId: string; userName: string },
    @ConnectedSocket() client: Socket,
  ) {
    // 广播输入状态，但不落地数据库
    // 直接广播，不存储（输入状态不需要持久化）
    client.to(data.docId).emit('userTyping', data);
  }

  /**
   * 处理聊天消息
   */
  @SubscribeMessage('chatMessage')
  handleChatMessage(
    @MessageBody() data: { docId: string; userId: string; userName: string; content: string; timestamp: number; mentions?: string[] },
    @ConnectedSocket() client: Socket,
  ) {
    // 广播消息给同一文档房间内的所有用户（包括自己）
    this.server.to(data.docId).emit('chatMessage', data);
  }

  /**
   * 高频光标移动同步
   *
   * 优化策略：
   * - 光标移动非常频繁，每秒可能几十次
   * - 可以配合 Redis 做节流（throttle）
   * - 当前实现直接广播，简单直接
   */
  @SubscribeMessage('cursorMove')
  handleCursorMove(
    @MessageBody() data: { docId: string; userId: string; position: { x: number; y: number } },
    @ConnectedSocket() client: Socket,
  ) {
    client.to(data.docId).emit('cursorMoved', data);
  }
}
```

## 四、Yjs CRDT 实现深度解析

### 4.1 Yjs 协作管理器源码分析

**文件路径**：`src/lib/yjs.ts`

#### 4.1.1 设计原理分析

Yjs 协作管理器采用**管理器模式**，其核心设计思想：

1. **多文档管理**：
   - 使用 Map 存储多个文档的协作状态
   - 支持同时打开多个文档

2. **离线优先**：
   - IndexedDB 本地持久化
   - 网络断开也能继续编辑
   - 网络恢复后自动同步

3. **优雅降级**：
   - WebSocket 失败时仍可使用本地模式
   - 不影响用户基本操作

#### 4.1.2 源码逐行分析

```typescript
/**
 * Yjs CRDT 协作服务
 * 提供基于 CRDT (Conflict-free Replicated Data Type) 的实时协作功能
 *
 * CRDT 的核心优势：
 * - 无需中心服务器解决冲突
 * - 支持离线编辑
 * - 最终一致性保证
 */

import * as Y from "yjs";
import { WebsocketProvider } from "y-websocket";
import { IndexeddbPersistence } from "y-indexeddb";

// 文档类型定义
export interface CRDTDocument {
  id: string;
  doc: Y.Doc;
  provider: WebsocketProvider | null;
  persistence: IndexeddbPersistence | null;
}

// 协作状态
export interface CollaborationState {
  connected: boolean;
  users: AwarenessUser[];
  syncing: boolean;
}

// 在线用户信息（用于光标显示）
export interface AwarenessUser {
  id: string;
  name: string;
  color: string;
  cursor?: {
    anchor: number;
    head: number;
  };
}

// 块操作类型
export type BlockOperationType = "add" | "update" | "delete" | "move";

/**
 * Yjs 协作管理器
 * 管理多个文档的 CRDT 协作连接
 *
 * 设计亮点：单例模式 + 工厂模式
 * - 全局唯一实例，避免重复连接
 * - 按需创建文档连接
 */
class YjsCollaborationManager {
  // 使用 Map 存储多个文档的协作状态
  // 为什么要用 Map？支持高效查找和遍历
  private documents: Map<string, CRDTDocument> = new Map();
  private currentDocId: string | null = null;

  // 服务器地址和用户信息
  private serverUrl: string;
  private userId: string;
  private userName: string;
  private userColor: string;

  constructor() {
    // 从环境变量或默认值获取 WebSocket 服务器地址
    // 为什么用环境变量？支持不同环境使用不同服务器
    this.serverUrl = process.env.NEXT_PUBLIC_COLLABORATION_WS_URL || "ws://localhost:1234";
    this.userId = "";
    this.userName = "";
    // 生成随机颜色，用于区分不同用户的光标
    this.userColor = this.generateRandomColor();
  }

  /**
   * 初始化用户信息
   * 为什么要分离初始化？因为用户信息可能异步获取
   */
  initUser(userId: string, userName: string) {
    this.userId = userId;
    this.userName = userName;
  }

  /**
   * 生成随机颜色
   * 用于在协作者的光标上显示不同颜色
   *
   * 为什么要预定义颜色池？
   * - 确保颜色对比度足够
   * - 避免生成难以辨认的颜色
   */
  private generateRandomColor(): string {
    const colors = [
      "#0D9488", // Teal
      "#0EA5E9", // Sky Blue
      "#8B5CF6", // Violet
      "#EC4899", // Pink
      "#F59E0B", // Amber
      "#10B981", // Emerald
      "#6366F1", // Indigo
      "#F43F5E", // Rose
    ];
    return colors[Math.floor(Math.random() * colors.length)];
  }

  /**
   * 连接文档协作
   *
   * 核心流程：
   * 1. 检查是否已有连接，有则复用
   * 2. 创建 Yjs 文档
   * 3. 创建 IndexedDB 持久化
   * 4. 创建 WebSocket 同步
   * 5. 设置用户感知
   *
   * 设计亮点：离线优先
   * - 即使网络不可用，也能编辑
   * - 网络恢复后自动同步
   */
  async connectDocument(
    docId: string,
    onSync: () => void,
    onAwarenessChange: (users: AwarenessUser[]) => void
  ): Promise<CRDTDocument> {
    // 1. 如果已存在连接，直接返回（避免重复连接）
    const existing = this.documents.get(docId);
    if (existing) {
      return existing;
    }

    // 2. 创建新的 Yjs 文档
    // Y.Doc 是 CRDT 的核心数据结构
    const doc = new Y.Doc();

    // 3. 创建 IndexedDB 持久化 (离线支持)
    // 为什么用 IndexedDB？
    // - 支持大容量存储
    // - 浏览器原生支持
    // - 即使关闭浏览器也不丢失数据
    const persistence = new IndexeddbPersistence(docId, doc);

    // 4. 等待 IndexedDB 加载完成
    // 为什么要等待？确保本地数据已加载后再连接服务器
    await new Promise<void>((resolve) => {
      persistence.once("synced", () => {
        console.log("IndexedDB 内容已加载");
        resolve();
      });
    });

    // 5. 创建 WebSocket 提供者 (实时协作)
    let provider: WebsocketProvider | null = null;

    try {
      // 连接到 Yjs 服务器
      // 注意：这里使用独立的 Yjs 服务器，不是主业务服务器
      provider = new WebsocketProvider(this.serverUrl, docId, doc, {
        connect: true,
      });

      // 6. 设置用户信息（Awareness）
      // Awareness 用于同步：
      // - 光标位置
      // - 选中区域
      // - 用户名称和颜色
      provider.awareness.setLocalState({
        user: {
          id: this.userId,
          name: this.userName,
          color: this.userColor,
        },
      });

      // 7. 监听同步事件
      // sync 事件表示本地数据与服务器数据一致
      provider.on("sync", (isSynced: boolean) => {
        if (isSynced) {
          console.log("文档已与服务器同步");
          onSync();
        }
      });

      // 8. 监听在线用户变化
      // 当有用户加入或离开时触发
      provider.awareness.on("change", () => {
        const states = provider?.awareness.getStates();
        const users: AwarenessUser[] = [];

        // 遍历所有用户状态，排除自己
        states?.forEach((state, clientId) => {
          if (state.user && clientId !== doc.clientID) {
            users.push({
              id: state.user.id || String(clientId),
              name: state.user.name || "未知用户",
              color: state.user.color || "#666",
              cursor: state.cursor,
            });
          }
        });

        onAwarenessChange(users);
      });

      // 9. 监听连接状态变化
      provider.on("status", (event: { status: string }) => {
        console.log("协作连接状态:", event.status);
      });
    } catch (error) {
      // 10. 优雅降级：WebSocket 失败时仍返回本地文档
      console.warn("WebSocket 连接失败，将仅使用本地离线模式:", error);
    }

    // 11. 组装并返回 CRDT 文档对象
    const crdtDoc: CRDTDocument = {
      id: docId,
      doc,
      provider,
      persistence,
    };

    this.documents.set(docId, crdtDoc);
    this.currentDocId = docId;

    return crdtDoc;
  }

  /**
   * 断开文档协作
   *
   * 为什么要正确清理？
   * - 释放 WebSocket 连接
   * - 释放 IndexedDB 资源
   * - 销毁 Yjs 文档
   * - 防止内存泄漏
   */
  disconnectDocument(docId: string) {
    const crdtDoc = this.documents.get(docId);
    if (!crdtDoc) return;

    // 断开 WebSocket 连接
    if (crdtDoc.provider) {
      crdtDoc.provider.disconnect();
      crdtDoc.provider.destroy();
    }

    // 销毁持久化
    if (crdtDoc.persistence) {
      crdtDoc.persistence.destroy();
    }

    // 销毁文档
    crdtDoc.doc.destroy();

    this.documents.delete(docId);

    if (this.currentDocId === docId) {
      this.currentDocId = null;
    }
  }
}
```

#### 4.1.3 CRDT 操作深度分析

```typescript
  /**
   * 获取当前文档的 Y.Array (用于块列表)
   *
   * 为什么要用 Y.Array？
   * - 支持多人同时添加/删除元素
   * - 保持元素顺序的一致性
   * - 高效的增量同步
   */
  getBlocksArray(): Y.Array<any> | null {
    const doc = this.getCurrentDoc();
    if (!doc) return null;
    return doc.getArray("blocks");
  }

  /**
   * 获取 Y.Map (用于文档元数据)
   *
   * 适用场景：
   * - 文档标题、封面等元信息
   * - 键值对形式的数据
   */
  getMetaMap(): Y.Map<any> | null {
    const doc = this.getCurrentDoc();
    if (!doc) return null;
    return doc.getMap("meta");
  }

  /**
   * 添加块
   *
   * CRDT 的插入操作：
   * - insert(index, items) 在指定位置插入
   * - 操作会被自动同步到其他客户端
   */
  addBlock(index: number, block: any) {
    const blocks = this.getBlocksArray();
    if (!blocks) return;

    blocks.insert(index, [block]);
  }

  /**
   * 更新块
   *
   * CRDT 的更新策略：
   * - Yjs 没有直接的"更新"操作
   * - 需要：删除旧值 + 插入新值
   * - 这两个操作会被作为原子操作同步
   */
  updateBlock(blockId: string, updates: Partial<any>) {
    const blocks = this.getBlocksArray();
    if (!blocks) return;

    // 遍历查找目标块
    blocks.forEach((block, index) => {
      if (block.id === blockId) {
        // 构造新块：合并旧值和新值
        const newBlock = { ...block, ...updates };
        // 删除旧块，插入新块
        blocks.delete(index, 1);
        blocks.insert(index, [newBlock]);
      }
    });
  }

  /**
   * 删除块
   *
   * CRDT 的删除操作：
   * - delete(index, length) 删除指定位置的元素
   * - 删除操作会被同步到所有客户端
   */
  deleteBlock(blockId: string) {
    const blocks = this.getBlocksArray();
    if (!blocks) return;

    blocks.forEach((block, index) => {
      if (block.id === blockId) {
        blocks.delete(index, 1);
      }
    });
  }

  /**
   * 移动块
   *
   * CRDT 的移动操作：
   * - 先删除，再插入
   * - 注意：目标位置需要调整（删除后索引会变化）
   */
  moveBlock(fromIndex: number, toIndex: number) {
    const blocks = this.getBlocksArray();
    if (!blocks) return;

    const block = blocks.get(fromIndex);
    if (block) {
      blocks.delete(fromIndex, 1);
      // 如果目标位置在原位置之后，需要减1
      blocks.insert(toIndex > fromIndex ? toIndex - 1 : toIndex, [block]);
    }
  }

  /**
   * 观察块变化
   *
   * 观察者模式：
   * - 当有远程变更时触发回调
   * - 用于更新 UI
   */
  observeBlocks(callback: (event: Y.YArrayEvent<any>) => void) {
    const blocks = this.getBlocksArray();
    if (!blocks) return;

    blocks.observe(callback);
  }
```

### 4.2 CRDT 核心概念深度解析

#### 4.2.1 为什么选择 CRDT？

| 特性 | 传统 OT | CRDT (Yjs) |
|------|---------|-------------|
| 冲突解决 | 需要中心服务器 | 分布式，无服务器 |
| 离线支持 | 困难 | 原生支持 |
| 复杂度 | 高 | 中等 |
| 扩展性 | 受限于服务器 | 理论上无限 |

#### 4.2.2 CRDT 工作原理图解

```
用户 A 编辑                    用户 B 编辑
    |                            |
    v                            v
[Y.Doc 本地副本]              [Y.Doc 本地副本]
    |                            |
    v                            v
  编辑操作                    编辑操作
(insert/delete)            (insert/delete)
    |                            |
    v                            v
 编码为二进制                编码为二进制
(Update V2)                (Update V2)
    |                            |
    v                            v
  WebSocket 发送              WebSocket 发送
    |                            |
    +------- 交换更新 --------+
    |                            |
    v                            v
Yjs 自动合并                Yjs 自动合并
    |                            |
    v                            v
   最终一致                   最终一致
```

#### 4.2.3 Yjs 数据类型详解


## 五、实时协作功能实现

### 5.1 文档同步

**流程：**

```
1. 用户 A 加入文档房间
   ↓
2. 服务器发送文档内容
   ↓
3. 用户 A 渲染文档
   ↓
4. 用户 A 修改块
   ↓
5. 发送 updateBlock 到服务器
   ↓
6. 服务器持久化到数据库
   ↓
7. 服务器广播 blockUpdated 到房间
   ↓
8. 用户 B 接收 blockUpdated
   ↓
9. 用户 B 更新本地状态
```

**代码示例：**

```typescript
// 前端：发送块更新
const { updateBlock } = useDocumentStore();

// 当用户编辑时
const handleBlockChange = (blockId: string, newContent: string) => {
  // 1. 更新本地状态（乐观更新）
  updateBlock(blockId, newContent);

  // 2. 发送到服务器
  socketClient.updateBlock(docId, blockId, newContent);
};

// 监听远程更新
useEffect(() => {
  const unsubscribe = socketClient.onBlockUpdated((data) => {
    if (data.blockId !== blockId) {
      // 更新其他用户的块
      remoteUpdateBlock(data.blockId, data.content);
    }
  });

  return unsubscribe;
}, []);
```

```typescript
// 聊天面板
function ChatDrawer() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [inputValue, setInputValue] = useState('');

  // 发送消息
  const handleSend = () => {
    if (!inputValue.trim()) return;

    const message: Message = {
      userId: currentUser.id,
      userName: currentUser.name,
      content: inputValue,
      timestamp: Date.now(),
    };

    // 1. 添加到本地
    setMessages((prev) => [...prev, message]);

    // 2. 发送到服务器
    socketClient.sendChat(docId, currentUser.id, currentUser.name, inputValue);

    // 3. 清空输入
    setInputValue('');
  };

  // 监听消息
  useEffect(() => {
    const unsubscribe = socketClient.onChatMessage((data) => {
      setMessages((prev) => [...prev, data]);
    });

    return unsubscribe;
  }, []);

  return (
    <div className="chat-drawer">
      <div className="messages">
        {messages.map((msg, index) => (
          <div key={index} className="message">
            <span className="sender">{msg.userName}:</span>
            <span className="content">{msg.content}</span>
          </div>
        ))}
      </div>

      <div className="input-area">
        <input
          type="text"
          value={inputValue}
          onChange={(e) => setInputValue(e.target.value)}
          onKeyDown={(e) => {
            if (e.key === 'Enter') {
              handleSend();
            }
          }}
        />
        <button onClick={handleSend}>发送</button>
      </div>
    </div>
  );
}
```

## 六、性能优化

### 6.1 批量更新

**问题**：每次块更新都发送单独的网络请求，性能差。

**解决方案**：批量收集更新，定时发送。

```typescript
// store/documentStore.ts
export const useDocumentStore = create<DocumentState>()((set, get) => {
  const pendingUpdates = new Map<string, { content: string; properties?: any }>();

  // 启动批量更新定时器
  const startBatchUpdate = () => {
    setInterval(() => {
      if (pendingUpdates.size > 0) {
        const blocks = Array.from(pendingUpdates.values());
        pendingUpdates.clear();

        // 发送批量更新
        socketClient.updateBlocksBatch(docId, blocks);
      }
    }, 500); // 每 500ms 批量发送一次
  };

  return {
    updateBlock: (id: string, content: string, properties?: any) => {
      // 立即更新本地
      set((state) => ({
        blocks: state.blocks.map(b =>
          b.id === id ? { ...b, content, ...(properties && { properties }) } : b
        ),
      }));

      // 收集批量更新
      pendingUpdates.set(id, { content, properties });
    },
  };
});
```

### 6.2 分块加载

**问题**：大文档一次性加载所有块，导致内存占用高和加载慢。

**解决方案**：分块加载，只加载可见区域。

```typescript
// store/documentStore.ts
export const useDocumentStore = create<DocumentState>()((set, get) => {
  return {
    // 加载文档块
    loadBlocksChunked: async (offset: number, limit: number) => {
      const { docId, chunkedLoading } = get();

      set({ isLoadingChunk: true });

      try {
        const res = await api.get(`${API_URL}/${docId}/blocks`, {
          params: { offset, limit, includeTotal: true },
        });

        const newChunks = new Map(chunkedLoading.loadedChunks);
        newChunks.set(offset, res.data.blocks);

        set({
          chunkedLoading: {
            ...chunkedLoading,
            loadedChunks: newChunks,
            totalBlocks: res.data.total || chunkedLoading.totalBlocks,
          },
        });
      } catch (error) {
        console.error('加载块失败:', error);
      } finally {
        set({ isLoadingChunk: false });
      }
    },

    // 检查是否还有更多块
    hasMoreBlocks: () => {
      const { chunkedLoading } = get();
      const loadedCount = chunkedLoading.loadedChunks.size * chunkedLoading.chunkSize;
      return loadedCount < chunkedLoading.totalBlocks;
    },
  };
});
```

### 6.3 增量同步

**问题**：每次离线重连后都需要下载整个文档。

**解决方案**：记录最后同步时间，只同步增量数据。


#### 7.1.2 源码逐行分析

```typescript
/**
 * LiveKit 客户端封装类
 * 视频会议功能（简化版）
 *
 * 为什么要封装？
 * - 简化 API 调用
 * - 统一错误处理
 * - 复用媒体控制逻辑
 */

// 使用 any 类型以简化实现（生产环境建议用具体类型）
type Room = any;
type RemoteParticipant = any;
type LocalParticipant = any;

/**
 * LiveKit 房间连接选项
 */
interface RoomConnectOptions {
  audio?: boolean;
  video?: boolean;
  screenShare?: boolean;
}

/**
 * LiveKit 客户端封装类
 * 用于管理视频会议房间连接及事件处理
 *
 * 设计亮点：观察者模式
 * - 事件回调机制
 * - 解耦业务逻辑
 */
export class LiveKitClient {
  // 私有属性：确保只有一个房间实例
  private room: Room | null = null;
  private localParticipant: LocalParticipant | null = null;

  // 事件回调：使用可选属性实现观察者模式
  // 为什么要用回调？解耦音视频事件和业务逻辑
  onParticipantJoined?: (participant: RemoteParticipant) => void;
  onParticipantLeft?: (participant: RemoteParticipant) => void;
  onTrackSubscribed?: (track: any, participant: RemoteParticipant) => void;
  onTrackUnsubscribed?: (track: any, participant: RemoteParticipant) => void;
  onConnectionStateChange?: (state: string) => void;
  onSpeakingChanged?: (participant: RemoteParticipant, isSpeaking: boolean) => void;

  /**
   * 连接到 LiveKit 房间
   *
   * 核心流程：
   * 1. 动态导入 livekit-client（按需加载）
   * 2. 创建房间实例
   * 3. 连接到服务器
   * 4. 设置事件监听
   *
   * 设计亮点：动态导入
   * - 减少首屏加载时间
   * - 用户不一定要用视频会议
   */
  async connect(url: string, token: string, _options?: RoomConnectOptions): Promise<Room | null> {
    try {
      // 1. 动态导入 livekit-client
      // 为什么动态导入？
      // - WebRTC 库体积较大
      // - 只有进入会议室时才需要
      const livekit = await import("livekit-client");

      // 2. 创建房间实例
      this.room = new livekit.Room();

      // 3. 连接房间
      // token 由后端生成，包含用户身份和权限信息
      await this.room.connect(url, token);

      // 4. 设置房间事件
      this.setupRoomEvents();

      // 5. 获取本地参与者
      // 用于控制自己的音视频
      this.localParticipant = this.room?.localParticipant;

      return this.room;
    } catch (error) {
      console.error("连接 LiveKit 房间失败:", error);
      return null;
    }
  }

  /**
   * 设置房间事件
   *
   * 为什么要动态获取 RoomEvent？
   * - livekit-client 可能未完全加载
   * - 确保事件类型正确
   */
  private setupRoomEvents() {
    if (!this.room) return;

    // 动态获取 RoomEvent
    import("livekit-client").then(livekit => {
      const RoomEvent = livekit.RoomEvent;

      // 参与者加入
      this.room?.on(RoomEvent.ParticipantConnected, (participant: RemoteParticipant) => {
        this.onParticipantJoined?.(participant);
      });

      // 参与者离开
      this.room?.on(RoomEvent.ParticipantDisconnected, (participant: RemoteParticipant) => {
        this.onParticipantLeft?.(participant);
      });

      // 轨道订阅（有人开启音视频）
      this.room?.on(RoomEvent.TrackSubscribed, (track: any, _publication: any, participant: RemoteParticipant) => {
        this.onTrackSubscribed?.(track, participant);
      });

      // 轨道取消订阅（有人关闭音视频）
      this.room?.on(RoomEvent.TrackUnsubscribed, (track: any, _publication: any, participant: RemoteParticipant) => {
        this.onTrackUnsubscribed?.(track, participant);
      });

      // 连接状态变化
      this.room?.on(RoomEvent.ConnectionStateChanged, (state: any) => {
        this.onConnectionStateChange?.(state);
      });
    });
  }

  /**
   * 切换麦克风
   *
   * 为什么要遍历所有轨道？
   * - 一个用户可能同时有多个音频轨道
   * - 需要统一控制
   */
  async toggleMicrophone(enabled: boolean) {
    if (!this.localParticipant) return;

    try {
      const audioTracks = (this.localParticipant as any).audioTracks;
      if (audioTracks) {
        for (const [, publication] of audioTracks) {
          if (publication.track) {
            if (enabled) {
              await publication.track.enable();
            } else {
              await publication.track.disable();
            }
          }
        }
      }
    } catch (error) {
      console.error("切换麦克风失败:", error);
    }
  }

  /**
   * 切换摄像头
   */
  async toggleCamera(enabled: boolean) {
    if (!this.localParticipant) return;

    try {
      const videoTracks = (this.localParticipant as any).videoTracks;
      if (videoTracks) {
        for (const [, publication] of videoTracks) {
          if (publication.track) {
            if (enabled) {
              await publication.track.enable();
            } else {
              await publication.track.disable();
            }
          }
        }
      }
    } catch (error) {
      console.error("切换摄像头失败:", error);
    }
  }

  /**
   * 屏幕共享
   *
   * 使用浏览器原生 API：
   * navigator.mediaDevices.getDisplayMedia
   */
  async startScreenShare() {
    if (!this.localParticipant) return;

    try {
      await (this.localParticipant as any).setScreenShareEnabled(true);
    } catch (error) {
      console.error("开始屏幕共享失败:", error);
    }
  }

  /**
   * 停止屏幕共享
   */
  async stopScreenShare() {
    if (!this.localParticipant) return;

    try {
      await (this.localParticipant as any).setScreenShareEnabled(false);
    } catch (error) {
      console.error("停止屏幕共享失败:", error);
    }
  }

  /**
   * 静音指定参与者
   *
   * 应用场景：主持人控制
   * 注意：这只是本地静音，不影响对方
   */
  async muteParticipant(participantSid: string) {
    if (!this.room) return;

    const participants = (this.room as any).participants;
    if (!participants) return;

    const participant = participants.get(participantSid);
    if (!participant) return;

    try {
      const audioTracks = (participant as any).audioTracks;
      if (audioTracks) {
        for (const [, publication] of audioTracks) {
          if (publication.track) {
            await publication.track.disable();
          }
        }
      }
    } catch (error) {
      console.error("静音参与者失败:", error);
    }
  }

  /**
   * 获取房间参与者
   */
  getParticipants(): Map<string, RemoteParticipant> {
    const participants = (this.room as any)?.participants;
    return participants || new Map();
  }

  /**
   * 获取房间连接状态
   */
  getConnectionState(): string {
    return (this.room as any)?.state || "disconnected";
  }

  /**
   * 获取房间实例
   * 供外部访问
   */
  getRoom(): Room | null {
    return this.room;
  }

  /**
   * 断开连接
   * 清理所有资源
   */
  async disconnect() {
    if (this.room) {
      await this.room.disconnect();
      this.room = null;
      this.localParticipant = null;
    }
  }
}

 * 导出 LiveKit 客户端单例
 */
```

## 八、性能优化深度分析

### 8.1 批量更新机制详解

#### 8.1.1 为什么需要批量更新？

**问题分析：**
```
用户快速输入时，每秒可能产生数十次更新：
- 每次按键触发一次 updateBlock
- 每次 updateBlock 触发一次网络请求
- 结果：网络拥塞、延迟增加、服务器压力增大
```

**解决方案：批量更新**
```
用户输入 → 收集到队列 → 定时批量发送

时间线：
0ms    用户输入 "H"
50ms   用户输入 "e"
100ms  用户输入 "l"
150ms  用户输入 "l"
200ms  用户输入 "o"
       ↓
250ms  批量发送：{ "H", "e", "l", "l", "o" }
```

#### 8.1.2 批量更新实现

```typescript
// store/documentStore.ts
export const useDocumentStore = create<DocumentState>()((set, get) => {
  // 待发送的更新队列
  const pendingUpdates = new Map<string, { content: string; properties?: any }>();

  // 启动批量更新定时器
  // 为什么要用定时器？
  // - 平衡延迟和性能
  // - 500ms 是经验值
  const startBatchUpdate = () => {
    setInterval(() => {
      if (pendingUpdates.size > 0) {
        // 将 Map 转换为数组
        const blocks = Array.from(pendingUpdates.values());
        // 清空队列
        pendingUpdates.clear();

        // 发送批量更新
        socketClient.updateBlocksBatch(docId, blocks);
      }
    }, 500); // 每 500ms 批量发送一次
  };

  return {
    updateBlock: (id: string, content: string, properties?: any) => {
      // 1. 立即更新本地状态（乐观更新）
      // 乐观更新：先更新 UI，再同步服务器
      // 目的：让用户感觉响应很快
      set((state) => ({
        blocks: state.blocks.map(b =>
          b.id === id ? { ...b, content, ...(properties && { properties }) } : b
        ),
      }));

      // 2. 收集到批量队列
      // 不立即发送，而是等待定时器
      pendingUpdates.set(id, { content, properties });
    },
  };
});
```

### 8.2 分块加载机制详解

#### 8.2.1 大型文档的性能挑战

```
假设文档有 10000 个块：
- 一次性加载：10MB+ 数据，5+ 秒加载时间
- 内存占用：高
- 渲染时间：长

分块加载：
- 每页加载 100 个块
- 按需加载：用户看到哪就加载哪
- 内存占用：低
- 加载时间：快
```

#### 8.2.2 分块加载实现

```typescript
// store/documentStore.ts
export const useDocumentStore = create<DocumentState>()((set, get) => {
  return {
    // 加载文档块（分块）
    loadBlocksChunked: async (offset: number, limit: number) => {
      const { docId, chunkedLoading } = get();

      set({ isLoadingChunk: true });

      try {
        // 请求指定范围的块
        const res = await api.get(`${API_URL}/${docId}/blocks`, {
          params: { offset, limit, includeTotal: true },
        });

        // 存储到已加载块 Map
        const newChunks = new Map(chunkedLoading.loadedChunks);
        newChunks.set(offset, res.data.blocks);

        set({
          chunkedLoading: {
            ...chunkedLoading,
            loadedChunks: newChunks,
            totalBlocks: res.data.total || chunkedLoading.totalBlocks,
          },
        });
      } catch (error) {
        console.error('加载块失败:', error);
      } finally {
        set({ isLoadingChunk: false });
      },
    },

    // 检查是否还有更多块
    hasMoreBlocks: () => {
      const { chunkedLoading } = get();
      const loadedCount = chunkedLoading.loadedChunks.size * chunkedLoading.chunkSize;
      return loadedCount < chunkedLoading.totalBlocks;
    },
  };
});

// 使用：滚动加载
const handleScroll = () => {
  // 计算滚动位置
  const scrollTop = window.scrollY;
  const windowHeight = window.innerHeight;
  const docHeight = document.body.offsetHeight;

  // 接近底部时加载更多
  if (scrollTop + windowHeight > docHeight - 500) {
    if (store.hasMoreBlocks() && !store.isLoadingChunk) {
      const nextOffset = store.getNextOffset();
      store.loadBlocksChunked(nextOffset, 100);
    }
  }
};
```

### 8.3 增量同步机制详解

#### 8.3.1 离线场景分析

```
场景：用户网络断开 1 小时
问题：重新连接后需要同步所有数据？
```
