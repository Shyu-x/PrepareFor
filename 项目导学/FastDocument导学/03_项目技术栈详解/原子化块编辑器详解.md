# 原子化块编辑器详解

## 一、原子化设计理念

### 1.1 什么是原子化块编辑器？

原子化块编辑器（Atomic Block Editor）是一种文档编辑范式，它将文档内容拆分为多个独立的"块"（Block），每个块都是原子级别的操作单元。

**核心概念：**

| 概念 | 说明 |
|------|------|
| **块（Block）** | 文档的最小编辑单位，如文本段落、标题、图片、代码块等 |
| **原子操作** | 对单个块的增删改查操作，不影响其他块 |
| **块类型** | 每个块都有特定类型，决定其渲染和行为 |
| **统一接口** | 所有块共享相同的数据结构和 API |

### 1.2 对比传统富文本编辑器

| 特性 | 传统富文本编辑器 | 原子化块编辑器 |
|------|-----------------|----------------|
| **数据结构** | HTML DOM 树 | JSON �数组 |
| **编辑单位** | 文本/节点 | 独立的块 |
| **扩展性** | 困难扩展 | 轻松添加新块类型 |
| **版本控制** | 困难追踪 | 每个块独立追踪 |
| **实时协作** | 文本光标同步 | 块级状态同步 |

```typescript
// 文本块渲染
const renderBlockContent = () => {
  switch (block.type) {
    case 'text':
      return (
        <div
          ref={inputRef}
          contentEditable
          onInput={handleInput}
          onFocus={handleFocus}
          onKeyDown={handleKeyDown}
          className="text-block"
          data-block-id={block.id}
        />
      );
  }
};

// 文本块事件处理
const handleInput = useCallback((e: React.FormEvent<HTMLDivElement>) => {
  const content = e.currentTarget.innerText || "";
  updateBlock(block.id, content);

  // 检查 Markdown 触发器（如 # 标题、- 待办）
  if (content.length <= 5) {
    checkMarkdownTrigger(content);
  }
}, [block.id, updateBlock]);

// 键盘事件
const handleKeyDown = useCallback((e: React.KeyboardEvent<HTMLDivElement>) => {
  // Enter：创建新块
  if (e.key === 'Enter' && !e.shiftKey) {
    e.preventDefault();
    addBlock('text', block.id);
  }

  // Backspace：删除空块
  if (e.key === 'Backspace' && !block.content) {
    e.preventDefault();
    removeBlock(block.id);
  }
}, [block.id, block.content, addBlock, removeBlock]);
```

**特性：**
- 使用 `contentEditable` 实现可编辑
- 支持 Markdown 快捷语法触发
- 键盘快捷键：Enter 创建新块，Backspace 删除空块
- 自动同步到服务器

### 3.2 待办事项块（TodoBlock）

**文件路径**：`src/components/TodoBlock.tsx`

**实现要点：**

```typescript
"use client";

import React, { useState } from "react";
import { Checkbox, Dropdown, message } from "antd";
import { CheckSquare, Calendar, Flag, MoreVertical } from "lucide-react";
import { useDocumentStore, Block } from "@/store/documentStore";
import { useTheme } from "@/components/ThemeProvider";
import dayjs from "dayjs";

// 优先级配置
const PRIORITY_CONFIG = {
  low: { label: "低", color: "#6B7280", bgColor: "#F3F4F6" },
  medium: { label: "中", color: "#F59E0B", bgColor: "#FEF3C7" },
  high: { label: "高", color: "#EF4444", bgColor: "#FEE2E2" },
  urgent: { label: "紧急", color: "#DC2626", bgColor: "#FECACA" },
};

interface TodoBlockProps {
  block: Block;
  onUpdate: (content: string, properties?: Block["properties"]) => void;
  onRemove: () => void;
}

export const TodoBlock: React.FC<TodoBlockProps> = ({ block, onUpdate, onRemove }) => {
  const { isMounted, theme } = useTheme();
  const [isEditing, setIsEditing] = useState(false);
  const [editContent, setEditContent] = useState(block.content);
  const [showDatePicker, setShowDatePicker] = useState(false);

  const isDark = isMounted && theme.isDark;
  const isChecked = block.properties?.checked || false;
  const priority = block.properties?.priority || "medium";
  const dueDate = block.properties?.dueDate;
  const priorityConfig = PRIORITY_CONFIG[priority as keyof typeof PRIORITY_CONFIG];

  // 处理复选框变更
  const handleCheckChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    onUpdate(block.content, {
      ...block.properties,
      checked: e.target.checked
    });
  };

  // 处理内容编辑
  const handleContentChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    setEditContent(e.target.value);
  };

  // 处理内容提交
  const handleContentSubmit = () => {
    if (editContent.trim()) {
      onUpdate(editContent, block.properties);
    }
    setIsEditing(false);
  };

  // 处理优先级变更
  const handlePriorityChange = (newPriority: string) => {
    onUpdate(block.content, {
      ...block.properties,
      priority: newPriority as 'low' | 'medium' | 'high' | 'urgent'
    });
  };

  // 处理日期变更
  const handleDateChange = (date: dayjs.Dayjs | null) => {
    onUpdate(block.content, {
      ...block.properties,
      dueDate: date ? date.valueOf() : undefined
    });
    setShowDatePicker(false);
  };

  // 删除截止日期
  const handleClearDate = () => {
    onUpdate(block.content, {
      ...block.properties,
      dueDate: undefined
    });
  };

  // 获取截止日期状态
  const getDueDateStatus = () => {
    if (!dueDate) return null;
    const now = dayjs();
    const due = dayjs(dueDate);
    const diff = due.diff(now, "day");

    if (isChecked) return { status: "completed", text: "已完成", color: "text-emerald-500" };
    if (diff < 0) return { status: "overdue", text: "已过期", color: "text-red-500" };
    if (diff === 0) return { status: "today", text: "今天到期", color: "text-amber-500" };
    if (diff === 1) return { status: "tomorrow", text: "明天到期", color: "text-amber-500" };
    if (diff <= 3) return { status: "soon", text: `${diff}天内到期`, color: "text-amber-500" };
    return { status: "normal", text: due.format("MM月DD日"), color: isDark ? "text-gray-400" : "text-gray-500" };
  };

  const dueDateStatus = getDueDateStatus();

  // 工具栏菜单
  const menuItems = [
    {
      key: "priority",
      label: "优先级",
      icon: <Flag size={14} />,
      children: [
        { key: "low", label: "低", onClick: () => handlePriorityChange("low") },
        { key: "medium", label: "中", onClick: () => handlePriorityChange("medium") },
        { key: "high", label: "高", onClick: () => handlePriorityChange("high") },
        { key: "urgent", label: "紧急", onClick: () => handlePriorityChange("urgent") },
      ],
    },
    {
      key: "date",
      label: dueDate ? "修改截止日期" : "设置截止日期",
      icon: <Calendar size={14} />,
      children: [
        { key: "today", label: "今天", onClick: () => handleDateChange(dayjs()) },
        { key: "tomorrow", label: "明天", onClick: () => handleDateChange(dayjs().add(1, "day")) },
        { key: "next-week", label: "下周", onClick: () => handleDateChange(dayjs().add(1, "week")) },
        { key: "next-month", label: "下月", onClick: () => handleDateChange(dayjs().add(1, "month")) },
        { type: "divider" },
        { key: "clear", label: "清除截止日期", danger: true, onClick: handleClearDate },
      ],
    },
    { type: "divider" },
    {
      key: "delete",
      label: "删除任务",
      icon: <MoreVertical size={14} />,
      danger: true,
      onClick: onRemove,
    },
  ];

  return (
    <div className={`group relative flex items-start gap-3 py-2 px-3 rounded-lg transition-colors ${
      isDark ? 'hover:bg-[#1F2937]/50' : 'hover:bg-gray-50'
    } ${isChecked ? 'opacity-60' : ''}`}>
      {/* 优先级指示器 */}
      <div className="flex-shrink-0 pt-0.5">
        <div
          className={`p-1 rounded transition-colors`}
          style={{
            backgroundColor: priorityConfig.bgColor,
            color: priorityConfig.color
          }}
        >
          <Flag size={16} />
        </div>
      </div>

      {/* 复选框 */}
      <div className="flex-shrink-0 pt-0.5">
        <input
          type="checkbox"
          checked={isChecked}
          onChange={handleCheckChange}
          className="w-5 h-5 rounded border-2 cursor-pointer accent-[#0D9488]"
          style={{
            borderColor: isChecked ? "#0D9488" : (isDark ? "#4B5563" : "#D1D5DB"),
            backgroundColor: isChecked ? "#0D9488" : "transparent"
          }}
        />
      </div>

      {/* 任务内容 */}
      <div className="flex-1 min-w-0">
        {isEditing ? (
          <input
            type="text"
            value={editContent}
            onChange={handleContentChange}
            onBlur={handleContentSubmit}
            onKeyDown={(e) => e.key === "Enter" && handleContentSubmit()}
            className={`w-full bg-transparent border-none outline-none ${
              isChecked ? "line-through text-gray-400" : (isDark ? "text-gray-200" : "text-gray-800")
            }`}
            autoFocus
          />
        ) : (
          <span
            onDoubleClick={() => setIsEditing(true)}
            className={`cursor-text ${isChecked ? "line-through text-gray-400" : (isDark ? "text-gray-200" : "text-gray-800")}`}
          >
            {block.content || "双击编辑任务内容..."}
          </span>
        )}

        {/* 元信息行 */}
        <div className="flex items-center gap-3 mt-1.5">
          {/* 截止日期 */}
          {dueDate && (
            <div className={`flex items-center gap-1 text-xs ${dueDateStatus?.color}`}>
              <Calendar size={12} />
              <span>{dueDateStatus?.text}</span>
            </div>
          )}

          {/* 未设置截止日期的提示 */}
          {!dueDate && !isChecked && (
            <button
              onClick={() => setShowDatePicker(true)}
              className={`flex items-center gap-1 text-xs transition-colors ${
                isDark ? 'text-gray-500 hover:text-gray-400' : 'text-gray-400 hover:text-gray-500'
              }`}
            >
              <Calendar size={12} />
              <span>设置截止日期</span>
            </button>
          )}
        </div>
      </div>

      {/* 操作菜单 */}
      <div className="flex-shrink-0 opacity-0 group-hover:opacity-100 transition-opacity">
        <Dropdown
          menu={{ items: menuItems }}
          trigger={["click"]}
          open={showDatePicker}
          onOpenChange={setShowDatePicker}
        >
          <button className="p-1.5 rounded-lg transition-colors bg-transparent hover:bg-gray-100">
            <MoreVertical size={16} />
          </button>
        </Dropdown>
      </div>

      {/* 日期选择器弹窗 */}
      <Modal
        title="设置截止日期"
        open={showDatePicker}
        onOk={() => setShowDatePicker(false)}
        onCancel={() => setShowDatePicker(false)}
        footer={null}
      >
        <div className="space-y-4">
          <DatePicker
            value={dueDate ? dayjs(dueDate) : null}
            onChange={handleDateChange}
            className="w-full"
            placeholder="选择日期"
            format="YYYY年MM月DD日"
          />
          <div className="flex gap-2">
            <button
              onClick={() => handleDateChange(dayjs())}
              className="px-3 py-1.5 text-sm rounded bg-gray-100 text-gray-600 hover:bg-gray-200"
            >
              今天
            </button>
            <button
              onClick={() => handleDateChange(dayjs().add(1, "day"))}
              className="px-3 py-1.5 text-sm rounded bg-gray-100 text-gray-600 hover:bg-gray-200"
            >
              明天
            </button>
            <button
              onClick={() => handleDateChange(dayjs().add(1, "week"))}
              className="px-3 py-1.5 text-sm rounded bg-gray-100 text-gray-600 hover:bg-gray-200"
            >
              一周后
            </button>
            <button
              onClick={() => handleDateChange(dayjs().add(1, "month"))}
              className="px-3 py-1.5 text-sm rounded bg-gray-100 text-gray-600 hover:bg-gray-200"
            >
              一个月后
            </button>
          </div>
        </div>
      </Modal>
    </div>
  );
};
```

**特性：**
- 复选框状态可视化
- 优先级颜色标识（低/中/高/紧急）
- 截止日期设置和状态显示
- 工具菜单（优先级、日期、删除）
- 双击编辑任务内容
- 自动同步所有属性变更

addBlock: (type: Block["type"], afterId?: string) => {
  const { blocks, focusedBlockId } = get();
  const newBlock: Block = {
    id: generateId(),
    type,
    content: "",
  };

  let newBlocks: Block[];
  if (afterId) {

    newBlocks = [
      ...blocks.slice(0, index + 1),
      newBlock,
      ...blocks.slice(index + 1),
    ];

    newBlocks = [
      ...blocks.slice(0, index + 1),
      newBlock,
      ...blocks.slice(index + 1),
    ];
  } else {
    newBlocks = [...blocks, newBlock];
  }

  set({ blocks: newBlocks });
  get().saveHistory();
},

// 更新块
updateBlock: (id: string, content: string, properties?: Block["properties"], type?: Block["type"]) => {
  const { blocks, focusedBlockId } = get();
  const index = blocks.findIndex((b) => b.id === id);
  if (index === -1) return;

  const updatedBlocks = [...blocks];
  updatedBlocks[index] = {
    ...updatedBlocks[index],
    content,
    ...(properties && { properties }),
    ...(type && { type }),
  };

  set({ blocks: updatedBlocks });

  // 发送到服务器
  socketClient.updateBlock(get().id, id, content, type, properties);

  // 更新历史
  if (id === focusedBlockId) {
    get().saveHistory();
  }
},

// 删除块
removeBlock: (id: string) => {
  const { blocks } = get();
  const newBlocks = blocks.filter((b) => b.id !== id);
  set({ blocks: newBlocks });
  get().saveHistory();
},

// 移动块（拖拽）
moveBlock: (fromIndex: number, toIndex: number) => {
  const { blocks } = get();
  const newBlocks = [...blocks];
  const [removed] = newBlocks.splice(fromIndex, 1);
  newBlocks.splice(toIndex, 0, removed);
  set({ blocks: newBlocks });
  get().saveHistory();
},

// 转换块类型
transformBlock: (id: string, newType: Block["type"]) => {
  const { blocks } = get();
  const newBlocks = blocks.map((b) =>
    b.id === id ? { ...b, type: newType } : b
  );
  set({ blocks: newBlocks });
  get().saveHistory();
},

## 五、虚拟滚动优化

### 5.1 性能问题

**问题**：当文档包含大量块（如 1000+）时，直接渲染所有块会导致：
1. DOM 节点过多，浏览器卡顿
2. 内存占用过高
3. 滚动不流畅

### 5.2 虚拟滚动实现

**文件路径**：`src/components/VirtualEditor.tsx`

```typescript
export default function VirtualEditor() {
  const { blocks } = useDocumentStore();
  const [visibleBlocks, setVisibleBlocks] = useState<Block[]>([]);
  const [scrollOffset, setScrollOffset] = useState(0);

  const BLOCK_HEIGHT = 80; // 假设每个块平均高度 80px
  const BUFFER_SIZE = 5; // 缓冲区：额外渲染前后各 5 个块

  // 虚拟滚动处理
  const handleScroll = useCallback((e: React.UIEvent<HTMLDivElement>) => {
    const scrollTop = e.currentTarget.scrollTop;
    const viewportHeight = e.currentTarget.clientHeight;

    // 计算可见区域的起始和结束索引
    const startIndex = Math.floor(scrollTop / BLOCK_HEIGHT);
    const visibleCount = Math.ceil(viewportHeight / BLOCK_HEIGHT) + BUFFER_SIZE * 2;
    const endIndex = Math.min(startIndex + visibleCount, blocks.length);

    setScrollOffset(startIndex);

    // 只渲染可见块（加缓冲）
    setVisibleBlocks(blocks.slice(startIndex, endIndex));
  }, [blocks, BLOCK_HEIGHT, BUFFER_SIZE]);

  // 初始渲染
  useEffect(() => {
    setVisibleBlocks(blocks.slice(0, 20)); // 初始渲染前 20 个块
  }, [blocks]);

  // 处理拖拽重排序
  const handleReorder = useCallback((newOrder: string[]) => {
    const newBlocks = newOrder.map((id) =>
      blocks.find((b) => b.id === id)!
    ).filter((Boolean));

    useDocumentStore.getState().setBlocks(newBlocks);
  }, [blocks]);

  return (
    <div className="virtual-editor-container" onScroll={handleScroll}>
      <AnimatePresence mode="popLayout">
        <Reorder.Group
          axis="="y"
          values={visibleBlocks.map((b) => b.id)}
          onReorder={handleReorder}
        >
          {visibleBlocks.map((block, index) => (
            <Reorder.Item key={block.id} value={block.id}>
              <BlockRenderer block={block} index={index} />
            </Reorder.Item>
          ))}
        </Reorder.Group>
      </AnimatePresence>
    </div>
  );
}
```

**优化效果：**
- 无论文档有多少块，只渲染可见区域的块
- 滚动流畅，DOM 节点数量稳定
- 内存占用显著降低
- 拖拽排序仍然正常工作

  ↓

  ↓

  ↓
服务器持久化块 X
  ↓
服务器广播 blockUpdated(X) 给所有用户
  ↓
用户 B 接收 blockUpdated(X)
  ↓
调用 remoteUpdateBlock(X) 更新本地状态
  ↓
检查发送者，避免循环更新
  ↓
如果不是发送者，则更新

3. **批量更新**
   - 收集多个更新，批量发送到服务器
   - 减少网络往返次数

4. **选择性订阅**
   - 只订阅需要的 Store 状态
   - 使用选择器函数避免不必要重渲染

5. **事件防抖**
   - 输入事件使用防抖
   - 滚动事件使用节流

---

## 八、总结

原子化块编辑器的优势：

1. **清晰的数据结构**
   - 每个块都有统一的数据格式
   - 易于序列化和持久化
   - 易于版本控制

2. **灵活的扩展性**
   - 添加新块类型只需实现新组件
   - 不影响现有代码
   - 模块化设计

3. **优秀的性能**
   - 虚拟滚动支持大文档
   - 细粒级更新减少重渲染
   - 批量操作优化网络请求

4. **良好的协作体验**
   - 块级状态同步
   - 避免循环更新
   - 操作可交换、幂等

5. **开发友好**
   - 类型安全
   - 易于调试
   - 统一的 API

FastDocument 项目通过原子化块编辑器，实现了一个功能强大、性能优秀、易于扩展的文档编辑系统。
