## 1. 开发环境问题

### 1.1 Node.js 版本兼容性

**问题**
项目需要特定版本的 Node.js，但当前版本不匹配。

**症状**
```
error: Your engine version (14.x.x) does not match the required version (>=18.x.x)
```

**解决方案**

```bash
# 检查当前版本
node --version

# 使用 nvm 切换版本（推荐）
nvm install 20
nvm use 20

# 或使用 n 管理器
npm install -g n
n 20
```

### 1.2 依赖安装问题

**问题**
依赖安装失败或版本冲突。

**症状**
```
ERESOLVE unable to resolve dependency tree
ENOENT: no such file or directory
```

**解决方案**

```bash
# 清理缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install

# 或使用 yarn
yarn install
```

### 1.3 端口冲突

**问题**
开发服务器端口被占用。

**症状**
```
Error: listen EADDRINUSE: address already in use :::13000
```

**解决方案**

```bash
# 查找占用进程
lsof -i :13000   # macOS/Linux
netstat -ano | findstr :13000  # Windows

# 终止进程
kill -9 <PID>

# 或启动时指定新端口
npm run dev -- -p 3001
```

## 2. 代码运行问题

### 2.1 React 组件渲染错误

**问题**
组件抛出异常导致白屏。

**症状**
```
Error: Element type is invalid: expected a string or class/function
```

**常见原因与解决**

```typescript
// 1. 导入方式错误
// ❌ 错误
import MyComponent from './MyComponent';

// ✅ 正确 - 命名导出
import { MyComponent } from './MyComponent';

// 2. 循环依赖
// ❌ 错误 - A 导入 B，B 又导入 A
// file A.ts
import { B } from './B';
export class A {}

// file B.ts
import { A } from './A';  // 循环！
export class B {}

// ✅ 正确 - 使用延迟导入或重构
```

### 2.2 Hooks 规则违反

**问题**
Hooks 在非函数组件中调用或顺序不一致。

**症状**
```
Error: Invalid hook call. Hooks can only be called inside of the body of a function component.
```

**解决方案**

```typescript
// ❌ 错误：在条件语句中调用 Hook
function MyComponent({ show }) {
  if (show) {
    const [state, setState] = useState(0);  // 错误！
  }
  return null;
}

// ✅ 正确：始终在顶层调用
function MyComponent({ show }) {
  const [state, setState] = useState(0);

  if (show) {
    // 可以在条件语句中使用值，但不能调用 Hook
    return <div>{state}</div>;
  }
  return null;
}

// ❌ 错误：在 useEffect 外部调用
const [state, setState] = useState(0);
function MyComponent() {
  useEffect(() => { ... }, []);
}

// ✅ 正确：在组件内部调用
function MyComponent() {
  const [state, setState] = useState(0);
  useEffect(() => { ... }, []);
}
```

### 2.3 状态更新问题

**问题**
状态更新后 UI 未同步更新。

**症状**
点击后数据变了但界面没更新。

**解决方案**

```typescript
// ❌ 错误：直接修改状态
function MyComponent() {
  const [items, setItems] = useState([]);

  const addItem = () => {
    items.push(newItem);  // 直接修改！
    setItems(items);       // 可能不会触发更新
  };
}

// ✅ 正确：创建新引用
function MyComponent() {
  const [items, setItems] = useState([]);

  const addItem = () => {
    setItems([...items, newItem]);  // 展开创建新数组
  };
}

// ✅ 正确：使用函数式更新
function MyComponent() {
  const [count, setCount] = useState(0);

  const increment = () => {
    setCount(c => c + 1);  // 使用函数获取最新值
  };
}
```

### 2.4 异步数据获取问题

**问题**
数据获取后组件已卸载。

**症状**
```
Warning: Can't perform a React state update on an unmounted component.
```

**解决方案**

```typescript
import { useEffect, useState, useRef } from 'react';

function MyComponent() {
  const [data, setData] = useState(null);
  const isMounted = useRef(true);

  useEffect(() => {
    // 标记组件已挂载
    isMounted.current = true;

    fetchData()
      .then(result => {
        // 检查组件是否仍然挂载
        if (isMounted.current) {
          setData(result);
        }
      });

    // 清理函数
    return () => {
      isMounted.current = false;
    };
  }, []);

  // ✅ 更好的方案：使用 AbortController
  useEffect(() => {
    const controller = new AbortController();

    fetch(url, { signal: controller.signal })
      .then(response => response.json())
      .then(data => {
        if (!controller.signal.aborted) {
          setData(data);
        }
      })
      .catch(err => {
        if (!controller.signal.aborted) {
          console.error(err);
        }
      });

    return () => controller.abort();
  }, []);
}
```

## 3. 性能问题

### 3.1 不必要的重新渲染

**问题**
组件频繁重新渲染导致性能下降。

**症状**
React DevTools 显示组件渲染次数异常高。

**解决方案**

```typescript
// 1. 使用 React.memo 包装纯组件
const BlockItem = memo(function BlockItem({ block, onUpdate }: BlockItemProps) {
  return <div onClick={() => onUpdate(block.content)}>{block.content}</div>;
});

// 2. 使用 useMemo 缓存计算结果
function ExpensiveComponent({ items }) {
  const sortedItems = useMemo(
    () => items.sort((a, b) => a.order - b.order),
    [items]
  );
  return <List items={sortedItems} />;
}

// 3. 使用 useCallback 缓存回调
function ParentComponent() {
  const handleClick = useCallback((id: string) => {
    console.log('Clicked:', id);
  }, []);  // 空依赖数组表示回调永远不变

  return <Child onClick={handleClick} />;
}
```

### 3.2 内存泄漏

**问题**
定时器或订阅未清理导致内存泄漏。

**症状**
使用一段时间后页面变慢，控制台无明显错误。

**解决方案**

```typescript
useEffect(() => {
  // ❌ 错误：未清理定时器
  const timer = setInterval(() => {
    // ...
  }, 1000);

  // ✅ 正确：清理定时器
  return () => clearInterval(timer);
}, []);

useEffect(() => {
  // ❌ 错误：未清理订阅
  const subscription = events.on('message', handler);

  // ✅ 正确：清理订阅
  return () => subscription.unsubscribe();
}, []);
```

### 3.3 大列表渲染性能

**问题**
渲染大量列表项导致卡顿。

**解决方案**

```typescript
// 1. 使用虚拟列表
import { useVirtualizer } from '@tanstack/react-virtual';

function VirtualList({ items }) {
  const parentRef = useRef(null);

  const virtualizer = useVirtualizer({
    count: items.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 50,
  });

  return (
    <div ref={parentRef} style={{ height: '400px', overflow: 'auto' }}>
      <div style={{ height: virtualizer.getTotalSize() }}>
        {virtualizer.getVirtualItems().map((virtualItem) => (
          <div
            key={virtualItem.key}
            style={{
              position: 'absolute',
              top: virtualItem.start,
              height: virtualItem.size,
            }}
          >
            {items[virtualItem.index].content}
          </div>
        ))}
      </div>
    </div>
  );
}

// 2. 分页加载
function PaginatedList() {
  const [page, setPage] = useState(1);
  const { data, hasMore } = useQuery(['items', page], () => fetchItems(page));

  return (
    <>
      {data.map(item => <Item key={item.id} item={item} />)}
      {hasMore && <button onClick={() => setPage(p => p + 1)}>加载更多</button>}
    </>
  );
}
```

## 6. 常见错误模式

### 6.1 类型错误

```typescript
// ❌ 错误：类型不匹配
const count: number = '5';

// ✅ 正确：类型转换
const count: number = parseInt('5', 10);

// ❌ 错误：可能为 null
const name = user.name.toUpperCase();

// ✅ 正确：可选链 + 空值合并
const name = user?.name?.toUpperCase() ?? '';
```

### 6.2 异步错误

```typescript
// ❌ 错误：未处理 Promise 拒绝
fetchData().then(data => console.log(data));
// 如果失败会静默失败

// ✅ 正确：添加 catch
fetchData()
  .then(data => console.log(data))
  .catch(error => console.error(error));

// ✅ 更好：使用 async/await
try {
  const data = await fetchData();
  console.log(data);
} catch (error) {
  console.error(error);
}
```

### 6.3 闭包问题

```typescript
// ❌ 错误：循环中的闭包
for (var i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 100);  // 输出: 5,5,5,5,5
}

// ✅ 正确：使用 let
for (let i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 100);  // 输出: 0,1,2,3,4
}

// ✅ 正确：使用闭包捕获
for (var i = 0; i < 5; i++) {
  ((j) => setTimeout(() => console.log(j), 100))(i);
}
```
