### 2.1 问题3：UNO 游戏规则如何实现？

**考察点：** 游戏状态机、规则引擎

**参考答案：**

#### 核心规则
1. **出牌条件**：颜色匹配、类型匹配、数字匹配
2. **功能牌效果**：
   - Skip：跳过下家
   - Reverse：反转方向
   - Draw Two：下家摸2张
   - Wild：变色
   - Wild Draw Four：下家摸4张并变色
3. **UNO 机制**：剩1张时必须喊UNO

#### 实现方式
```typescript
isPlayable(card: Card, topCard: Card, currentColor: CardColor): boolean {
  return card.color === CardColor.WILD ||
         card.color === currentColor ||
         card.type === topCard.type;
}
```

### 问题 4：多人游戏状态如何同步？

**考察点：** WebSocket、实时通信

**参考答案：**

1. **服务端为真源**：游戏状态存储在后端
2. **WebSocket 广播**：状态变更实时广播
3. **乐观更新**：本地先更新，再等待服务器确认

```typescript
// 客户端
socket.on('gameStateUpdate', (state) => {
  useGameStore.getState().setGameState(state);
});

// 服务端
broadcastState(roomId: string) {
  this.server.to(roomId).emit('gameStateUpdate', gameState);
}
```

### 3.1 问题5：AI 对手如何实现？

**考察点：** AI 算法、决策系统

**参考答案：**

#### 三种难度

- **简单（EASY）**：随机选择可出卡牌
- **中等（MEDIUM）**：优先打出颜色最多的普通牌，保留万能牌
- **困难（HARD）**：攻击策略 + 综合评估

```typescript
getBestMove(hand: Card[], topCard: Card, difficulty: AIDifficulty): Card {
  const playable = hand.filter(c => this.isPlayable(c, topCard));

  if (difficulty === 'EASY') {
    return random(playable);
  }

  if (difficulty === 'MEDIUM') {
    const colorCount = this.countColors(playable);
    const bestColor = maxBy(colorCount);
    return playable.find(c => c.color === bestColor) || playable[0];
  }

  // HARD: 攻击下家
  if (hand.length <= 3) {
    const actionCards = playable.filter(c => c.type !== 'NUMBER');
    if (actionCards.length) return actionCards[0];
  }
  // ...
}
```

我在 UnoThree 负责多人实时对战链路与 2D/3D 双渲染体验，核心工作是把"可玩"升级为"稳定可运营"。我重点推进了游戏规则引擎、状态广播优化、弱网重连和跨端响应式，解决了多人并发下的回合同步与动画一致性问题，最终实现了更稳定的对局体验与更可控的线上维护成本。

### 8.2 3 分钟深挖版本

1. **目标定义**
   - 体验目标：操作反馈快、动画自然、规则可解释
   - 稳定目标：断线可恢复、房间不"僵尸化"、状态不分叉
   - 运营目标：可观测、可回放、可快速定位争议对局

2. **架构关键点**
   - 前端采用 2D/3D 双通道渲染：弱机型保底 2D，高性能设备可切 3D
   - 游戏状态以服务端为真源，客户端保留局部乐观动画，避免状态漂移
   - WebSocket 事件按"动作-校验-广播-落盘"流程组织

3. **难点与解决**
   - 难点 A：特殊牌（+4、反转、跳过）叠加后状态复杂
     - 解决：规则引擎分层（合法性判定、效果执行、回合推进）
   - 难点 B：多人并发动作导致竞争
     - 解决：服务端回合锁 + 操作幂等键 + 超时处理
   - 难点 C：动画与真实状态错位
     - 解决：动画只做表现层，最终状态由服务端快照驱动
