### 5.2 JavaScript 核心基础

#### 5.2.1 数据类型与判断

| 类型 | 原始类型 | 说明 |
|------|---------|------|
| 原始类型 | undefined, null, Boolean, String, Number, Symbol, BigInt | 存放在栈中 |
| 引用类型 | Object (Array, Function, Date等) | 存放在堆中，栈中存引用 |

**类型判断**：

```javascript
typeof 123        // "number"
typeof "str"       // "string"
typeof null        // "object" - 历史bug
typeof []          // "object"
typeof {}          // "object"
typeof function    // "function"

Array.isArray([])  // true - 推荐
Object.prototype.toString.call([]) // "[object Array]"
```

**null vs undefined**：

| 区别 | null | undefined |
|------|------|-----------|
| 含义 | 空对象 | 未定义 |
| 类型 | object | undefined |
| 用途 | 主动设置 | 自动生成 |
| 转为数字 | 0 | NaN |

#### 5.2.2 原型与继承

**原型链**：
- 每个对象都有 `__proto__` 属性
- 构造函数有 `prototype` 属性
- 原型链顶端是 `Object.prototype`
- `Object.prototype.__proto__` 是 null

**继承实现方式**：

1. **原型链继承**：`Child.prototype = new Parent()`
2. **构造函数继承**：`Parent.call(this, args)`
3. **组合继承**：原型链 + 构造函数
4. **ES6 class**：`class Child extends Parent`

#### 5.2.3 作用域与闭包

**作用域类型**：
- 全局作用域
- 函数作用域
- 块级作用域 (let, const)

**闭包**：
- 定义：函数能够访问外部作用域变量的现象
- 应用场景：防抖、节流、模块化、私有变量

#### 5.2.4 事件循环

**宏任务和微任务**：

| 类型 | 例子 |
|------|------|
| 宏任务 | setTimeout, setInterval, I/O, UI渲染 |
| 微任务 | Promise, MutationObserver, process.nextTick |

**事件循环顺序**：
1. 执行同步代码
2. 执行微任务队列
3. 执行宏任务队列

### 5.3 高频手写代码题

#### 5.3.1 防抖与节流

```javascript
// 防抖 - debounce
function debounce(fn, delay) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流 - throttle
function throttle(fn, delay) {
  let last = 0;
  return function(...args) {
    const now = Date.now();
    if (now - last >= delay) {
      last = now;
      fn.apply(this, args);
    }
  };
}
```

#### 5.3.2 深拷贝

```javascript
function deepClone(obj, cache = new WeakMap()) {
  if (obj === null || typeof obj !== 'object') return obj;
  if (obj instanceof Date) return new Date(obj);
  if (obj instanceof RegExp) return new RegExp(obj);
  if (cache.has(obj)) return cache.get(obj);

  const allKeys = Reflect.ownKeys(obj);
  const cloneObj = Array.isArray(obj) ? [] : {};
  cache.set(obj, cloneObj);

  allKeys.forEach(key => {
    cloneObj[key] = deepClone(obj[key], cache);
  });

  return cloneObj;
}
```

#### 5.3.3 Promise实现

```javascript
class MyPromise {
  constructor(executor) {
    this.status = 'pending';
    this.value = undefined;
    this.reason = undefined;
    this.onFulfilledCallbacks = [];
    this.onRejectedCallbacks = [];

    const resolve = (value) => {
      if (this.status === 'pending') {
        this.status = 'fulfilled';
        this.value = value;
        this.onFulfilledCallbacks.forEach(cb => cb(this.value));
      }
    };

    const reject = (reason) => {
      if (this.status === 'pending') {
        this.status = 'rejected';
        this.reason = reason;
        this.onRejectedCallbacks.forEach(cb => cb(this.reason));
      }
    };

    try {
      executor(resolve, reject);
    } catch (err) {
      reject(err);
    }
  }

  then(onFulfilled, onRejected) {
    return new MyPromise((resolve, reject) => {
      if (this.status === 'fulfilled') {
        try {
          resolve(onFulfilled(this.value));
        } catch (err) {
          reject(err);
        }
      } else if (this.status === 'rejected') {
        try {
          resolve(onRejected(this.reason));
        } catch (err) {
          reject(err);
        }
      } else {
        this.onFulfilledCallbacks.push(value => {
          try {
            resolve(onFulfilled(value));
          } catch (err) {
            reject(err);
          }
        });
        this.onRejectedCallbacks.push(reason => {
          try {
            resolve(onRejected(reason));
          } catch (err) {
            reject(err);
          }
        });
      }
    });
  }
}
```

### 5.4 高频追问速答

#### 5.4.1 性能优化相关

**问题：首屏加载慢，怎么优化？**

**回答要点**：
1. 资源体积：Code Splitting、Tree Shaking、图片压缩
2. 网络请求：CDN、缓存、预加载
3. 渲染：SSR/SSG、关键CSS内联
4. 数据：接口合并、预加载、骨架屏

**问题：列表卡顿，怎么处理？**

**回答要点**：
1. 虚拟列表：只渲染可见区域
2. 防抖/节流：减少不必要的计算
3. WebWorker：复杂计算移出主线程
4. React.memo：避免不必要的重渲染

#### 5.4.2 框架相关

**问题：React Hooks的原理？**

**回答要点**：
1. Hooks基于链表存储
2. 每次渲染按调用顺序执行
3. useState使用useReducer实现
4. useEffect异步执行，在DOM更新后

**问题：Vue响应式原理？**

**回答要点**：
1. Vue2：Object.defineProperty，递归监听
2. Vue3：Proxy，更好的性能
3. 依赖收集：getter中收集，setter中触发
4. 数组特殊处理：重写数组方法

#### 5.4.3 网络相关

**问题：HTTP缓存？**

**回答要点**：
1. 强缓存：Cache-Control、Expires
2. 协商缓存：Last-Modified/If-Modified-Since、Etag/If-None-Match
3. 缓存位置：Memory Cache、Disk Cache
4. 实际应用：静态资源用强缓存，接口用协商缓存

**问题：HTTPS握手过程？**

**回答要点**：
1. 客户端发送支持的加密算法
2. 服务器选择算法，返回证书
3. 客户端验证证书，生成随机数
4. 服务器验证客户端，完成握手
