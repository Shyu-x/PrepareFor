### 1.1 TCP 三次握手与四次挥手

**参考答案：**

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓  
┃                    TCP 三次握手                                  ┃ 
┃                                                                  ┃ 
┃    客户端                              服务端                     ┃
┃      ┃                                  ┃                        ┃ 
┃      ┃  ━━━━━━  SYN = 1, seq = x  ━━━━▶ ┃                       ┃  
┃      ┃                                  ┃                        ┃ 
┃      ┃  ◀━━━━━  SYN = 1, ACK = 1    ━━━━┃                       ┃  
┃      ┃          seq = y, ack = x + 1    ┃                        ┃ 
┃      ┃                                  ┃                        ┃ 
┃      ┃  ━━━━━━  ACK = 1, seq = x + 1  ━━▶┃                       ┃ 
┃      ┃          ack = y + 1             ┃                        ┃ 
┃      ┃                                  ┃                        ┃ 
┃      ┃         [连接建立]                ┃                        ┃
┃                                                                  ┃ 
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛  

```

**三次握手过程**：

| 步骤 | 客户端 | 服务端 | 说明 |
| :--- | :--- | :--- | :--- |
| 1 | SYN=1, seq=x | | 客户端发送连接请求 |
| 2 | | SYN=1, ACK=1, seq=y, ack=x+1 | 服务端确认并发送 SYN |
| 3 | ACK=1, seq=x+1, ack=y+1 | | 客户端确认，连接建立 |

**四次挥手过程**：

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓  
┃                    TCP 四次挥手                                  ┃ 
┃                                                                  ┃ 
┃    客户端                              服务端                     ┃
┃      ┃                                  ┃                        ┃ 
┃      ┃  ━━━━━━  FIN = 1, seq = u   ━━━━▶ ┃   客户端发送断开请求   ┃
┃      ┃                                  ┃                        ┃ 
┃      ┃  ◀━━━━━  ACK = 1, ack = u + 1 ━━━┃   服务端确认           ┃ 
┃      ┃                                  ┃                        ┃ 
┃      ┃                                  ┃   (服务端可能还有数据)  ┃
┃      ┃                                  ┃                        ┃ 
┃      ┃  ◀━━━━━  FIN = 1, seq = v   ━━━━┃   服务端发送断开请求   ┃  
┃      ┃                                  ┃                        ┃ 
┃      ┃  ━━━━━━  ACK = 1, ack = v + 1 ━━▶┃   客户端确认           ┃ 
┃      ┃                                  ┃                        ┃ 
┃      ┃    [等待 2MSL 后关闭]            ┃                        ┃ 
┃                                                                  ┃ 
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛  

```

**为什么需要三次握手**：
- 防止已失效的连接请求报文被服务器接收，导致错误建立连接
- 确保双方都能收发数据

**为什么需要四次挥手**：
- TCP 是全双工通信，需要双方分别发送 FIN
- 确保数据全部传输完毕

### 1.2 TCP 与 UDP 的区别

**参考答案：**

| 特性 | TCP | UDP |
| :--- | :--- | :--- |
| **连接性** | 面向连接 | 无连接 |
| **可靠性** | 可靠（确认、重传、排序） | 不可靠 |
| **传输方式** | 字节流 | 报文 |
| **效率** | 较低 | 较高 |
| **头部大小** | 20-60 字节 | 8 字节 |
| **拥塞控制** | 有 | 无 |
| **流量控制** | 有 | 无 |
| **使用场景** | HTTP、HTTPS、FTP、SMTP | DNS、VoIP、视频流 |

### 1.3 HTTP/1.0 / 1.1 / 2 / 3 区别

**参考答案：**

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓ 
┃                      HTTP 协议演进                               ┃
┃                                                                  ┃
┃  HTTP/1.0                          HTTP/1.1                    ┃  
┃  ┣━ 短连接（每次请求建连）         ┣━ 持久连接（Keep-Alive）    ┃ 
┃  ┣━ 不支持断点续传                 ┣━ 管线化（ pipelining）    ┃  
┃  ┗━ 无虚拟主机支持                 ┣━ 支持断点续传             ┃  
┃                                       ┣━ 虚拟主机支持           ┃ 
┃                                       ┗━ 缓存更新               ┃ 
┃                                                                  ┃
┃  HTTP/2                              HTTP/3                     ┃ 
┃  ┣━ 二进制分帧                     ┣━ QUIC 协议（UDP）         ┃  
┃  ┣━ 多路复用（单一连接）           ┣━ 0-RTT 连接               ┃  
┃  ┣━ 头部压缩（HPACK）             ┣━ 无队头阻塞               ┃   
┃  ┣━ 服务端推送                     ┣━ 连接迁移                 ┃  
┃  ┗━ 优先级设置                     ┗━ 更好的拥塞控制           ┃  
┃                                                                  ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛ 

```

**HTTP/1.1 特性**：
- 持久连接（Keep-Alive）：复用 TCP 连接
- 管道化（Pipelining）：并行发送请求（但响应需按序）
- 断点续传（Range 请求）
- 虚拟主机（Host 头）

**HTTP/2 特性**：
- 二进制分帧：所有传输分割为更小的帧
- 多路复用：一个 TCP 连接可并行多个请求/响应
- HPACK 头部压缩：静态表 + 动态表
- 服务端推送：主动推送资源
- 优先级：可设置请求优先级

**HTTP/3 特性**：
- 基于 QUIC（UDP）：更快的连接建立
- 0-RTT：缓存密钥，快速恢复连接
- 无队头阻塞：流之间相互独立
- 连接迁移：切换网络时保持连接

### 1.6 HTTP 状态码详解

**参考答案：**

**1xx - 信息响应**：

| 状态码 | 说明 |
| :--- | :--- |
| 100 | Continue（继续） |
| 101 | Switching Protocols（切换协议） |

**2xx - 成功响应**：

| 状态码 | 说明 |
| :--- | :--- |
| 200 | OK（请求成功） |
| 201 | Created（已创建） |
| 204 | No Content（无内容） |
| 206 | Partial Content（部分内容） |

**3xx - 重定向**：

| 状态码 | 说明 |
| :--- | :--- |
| 301 | Moved Permanently（永久重定向） |
| 302 | Found（临时重定向） |
| 304 | Not Modified（未修改，缓存） |
| 307 | Temporary Redirect（临时重定向） |

**4xx - 客户端错误**：

| 状态码 | 说明 |
| :--- | :--- |
| 400 | Bad Request（请求错误） |
| 401 | Unauthorized（未授权） |
| 403 | Forbidden（禁止访问） |
| 404 | Not Found（未找到） |
| 429 | Too Many Requests（请求过多） |

**5xx - 服务器错误**：

| 状态码 | 说明 |
| :--- | :--- |
| 500 | Internal Server Error（内部错误） |
| 502 | Bad Gateway（网关错误） |
| 503 | Service Unavailable（服务不可用） |
| 504 | Gateway Timeout（网关超时） |

### 1.7 HTTP 缓存机制

**参考答案：**

**缓存头部**：

| 头部 | 说明 |
| :--- | :--- |
| **Expires** | 过期时间（绝对时间） |
| **Cache-Control** | 缓存控制（相对时间） |
| **Last-Modified** | 资源最后修改时间 |
| **ETag** | 资源唯一标识（哈希） |
| **If-None-Match** | 条件请求（匹配 ETag） |
| **If-Modified-Since** | 条件请求（匹配 Last-Modified） |

**Cache-Control 值**：

| 值 | 说明 |
| :--- | :--- |
| `public` | 可被任何缓存存储 |
| `private` | 仅浏览器缓存 |
| `no-cache` | 需验证后使用 |
| `no-store` | 不缓存 |
| `max-age=3600` | 缓存有效时间 |

### 1.8 Cookie 与 Session

**参考答案：**

**Cookie**：

```http
Set-Cookie: name=value; expires=Thu, 01 Jan 2025 00:00:00 GMT; path=/; domain=.example.com; secure; HttpOnly; SameSite=Strict
```

| 属性 | 说明 |
| :--- | :--- |
| `name=value` | 键值对 |
| `expires` | 过期时间 |
| `path` | 生效路径 |
| `domain` | 生效域名 |
| `secure` | 仅 HTTPS 传输 |
| `HttpOnly` | 无法通过 JS 访问 |
| `SameSite` | CSRF 防护（Strict/Lax/None） |

**对比**：

| 特性 | Cookie | Session |
| :--- | :--- | :--- |
| 存储位置 | 浏览器 | 服务器 |
| 安全性 | 较低（可加密） | 较高 |
| 容量 | 4KB | 无限制 |
| 性能 | 每次请求携带 | 服务端查询 |

### 1.9 HTTP 与 WebSocket 区别

**参考答案：**

| 特性 | HTTP | WebSocket |
| :--- | :--- | :--- |
| 连接方式 | 请求-响应 | 全双工 |
| 通信模式 | 半双工 | 全双工 |
| 头部大小 | 大（约 200B） | 小（约 2B） |

### 2.3 TCP 拥塞控制

**参考答案：**

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓ 
┃                    TCP 拥塞控制算法                              ┃
┃                                                                  ┃
┃  1. 慢启动 (Slow Start)                                         ┃ 
┃     ━━ 指数增长 ━━                                              ┃ 
┃     cwnd: 1 → 2 → 4 → 8 → 16 → ...                             ┃  
┃           ┃                                                     ┃ 
┃           ▼                                                     ┃ 
┃  2. 拥塞避免 (Congestion Avoidance)                             ┃ 
┃     ━━ 线性增长 ━━                                              ┃ 
┃     cwnd: +1 每个 RTT                                           ┃ 
┃           ┃                                                     ┃ 
┃           ▼                                                     ┃ 
┃  3. 快重传 (Fast Retransmit)                                    ┃ 
┃     ━━ 3 个重复 ACK ━━                                          ┃ 
┃     立即重传丢失的包                                             ┃
┃           ┃                                                     ┃ 
┃           ▼                                                     ┃ 
┃  4. 快恢复 (Fast Recovery)                                       ┃
┃     ━━ 调整 cwnd ━━                                             ┃ 
┃     cwnd = cwnd / 2 + 3                                         ┃ 
┃                                                                  ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛ 

```

### 3.2 IP 地址分类

**参考答案：**

| 分类 | 范围 | 说明 |
| :--- | :--- | :--- |
| A | 0.0.0.0 - 127.255.255.255 | 大型网络（2^24-2 主机） |
| B | 128.0.0.0 - 191.255.255.255 | 中型网络（2^16-2 主机） |
| C | 192.0.0.0 - 223.255.255.255 | 小型网络（2^8-2 主机） |
| D | 224.0.0.0 - 239.255.255.255 | 组播 |
| E | 240.0.0.0 - 255.255.255.255 | 保留 |

**特殊 IP**：

| IP | 说明 |
| :--- | :--- |
| 127.0.0.1 | 本地回环 |
| 0.0.0.0 | 任意地址 |
| 255.255.255.255 | 广播 |
| 10.x.x.x | 私有地址 |
| 192.168.x.x | 私有地址 |

**IPv4 vs IPv6**：

| 特性 | IPv4 | IPv6 |
| :--- | :--- | :--- |
| 地址长度 | 32 位 | 128 位 |
| 表示方式 | 点分十进制 | 冒号十六进制 |
| 地址数量 | ~40 亿 | 3.4×10^38 |
