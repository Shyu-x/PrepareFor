## 第4章 计算机网络

### 4.1 XSS 攻击与防御

**参考答案：**

**XSS（Cross-Site Scripting）**：跨站脚本攻击，注入恶意脚本。

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓ 
┃                      XSS 攻击原理                                ┃
┃                                                                  ┃
┃  1. 存储型 XSS                                                  ┃ 
┃     攻击者 ━━▶ 提交恶意脚本 ━━▶ 服务器存储 ━━▶ 用户访问 ━━▶ 执行 ┃
┃                                                                  ┃
┃  2. 反射型 XSS                                                  ┃ 
┃     攻击者 ━━▶ 构造恶意 URL ━━▶ 用户点击 ━━▶ 服务器返回 ━━▶ 执行 ┃
┃                                                                  ┃
┃  3. DOM 型 XSS                                                  ┃ 
┃     攻击者 ━━▶ 构造恶意 URL ━━▶ 前端 JS ━━▶ innerHTML ━━▶ 执行   ┃
┃                                                                  ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛ 

```

**防御措施**：

```javascript
// 1. 输入过滤
function escapeHTML(str) {
  return str.replace(/[&<>"']/g, char => ({
    '&': '&amp;',
    '<': '&lt;',
    '>': '&gt;',
    '"': '&quot;',
    "'": '&#39;'
  }[char]));
}

// 2. CSP 内容安全策略
// Content-Security-Policy: script-src 'self'; style-src 'self' 'unsafe-inline'

// 3. HttpOnly Cookie
// Set-Cookie: session=xxx; HttpOnly

// 4. 使用 React/Vue 等框架（自动转义）
```

### 4.2 CSRF 攻击与防御

**参考答案：**

**CSRF（Cross-Site Request Forgery）**：跨站请求伪造。

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓   
┃                      CSRF 攻击原理                              ┃   
┃                                                                  ┃  
┃    用户登录正常网站 A                                            ┃  
┃           ┃                                                     ┃   
┃           ▼                                                     ┃   
┃    用户被诱导访问恶意网站 B                                       ┃ 
┃           ┃                                                     ┃   
┃           ▼                                                     ┃   
┃    网站 B 自动发送请求到 A                                       ┃  
┃    GET: /transfer?to=攻击者&amount=10000                        ┃   
┃           ┃                                                     ┃   
┃           ▼                                                     ┃   
┃    浏览器携带 A 的 Cookie 发请求                                 ┃  
┃           ┃                                                     ┃   
┃           ▼                                                     ┃   
┃    服务器认为是用户操作，执行转账                                  ┃
┃                                                                  ┃  
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛   

```

**防御措施**：

| 措施 | 说明 |
| :--- | :--- |
| **SameSite Cookie** | `SameSite=Strict/Lax` |
| **CSRF Token** | 服务端生成随机 token，表单中携带 |
| **Referer 检查** | 检查请求来源 |
| **双重提交 Cookie** | Cookie 中设置 token，表单中也提交 |
| **验证码** | 短信/邮箱验证 |

### 4.3 HTTPS 安全机制

**参考答案：**

| 机制 | 说明 |
| :--- | :--- |
| **加密传输** | 对称加密（数据）+ 非对称加密（密钥交换） |
| **身份验证** | 数字证书验证服务器身份 |
| **完整性保护** | MAC（Message Authentication Code）校验 |
| **TLS 协议** | 记录协议 + 握手协议 |

**证书类型**：

| 类型 | 说明 |
| :--- | :--- |
| DV | 域名验证（个人） |
| OV | 组织验证（企业） |
| EV | 扩展验证（最高安全） |

### 4.4 SQL 注入与防御

**参考答案：**

**SQL 注入**：通过在用户输入中注入恶意 SQL 代码来攻击数据库。

**防御措施**：

```sql
-- 使用参数化查询
PREPARE stmt FROM 'SELECT * FROM users WHERE username = ?';
SET @username = 'john';
EXECUTE stmt USING @username;

-- 使用 ORM 框架
-- 使用输入验证
-- 最小权限原则
```

### 4.5 中间人攻击与防御

**参考答案：**

**中间人攻击（MITM）**：攻击者拦截并可能篡改通信双方之间的通信。

**防御措施**：
- 使用 HTTPS
- 验证证书
- 使用 HSTS（HTTP Strict Transport Security）
- 使用公钥固定（Public Key Pinning）

---

### 4.6 DDOS 攻击与防御

**参考答案：**

**DDoS（分布式拒绝服务）**：通过大量请求使目标服务不可用。

**防御措施**：
- 使用 CDN
- 流量清洗
- 限制请求速率
- 使用 Web 应用防火墙（WAF）
- 增加服务器资源

---

### 4.7 点击劫持与防御

**参考答案：**

**点击劫持（Clickjacking）**：攻击者通过透明 iframe 诱导用户点击。

**防御措施**：

```http
X-Frame-Options: DENY
X-Frame-Options: SAMEORIGIN
Content-Security-Policy: frame-ancestors 'self'
```

---

### 4.8 JSONP 攻击与防御

**参考答案：**

**JSONP 漏洞**：通过 JSONP 回调泄露敏感信息。

**防御措施**：
- 使用 CORS 代替 JSONP
- 验证 Referer
- 使用 CSRF Token

---

### 4.9 文件上传漏洞与防御

**参考答案：**

**文件上传漏洞**：上传恶意文件执行代码。

**防御措施**：
- 限制文件类型
- 重命名上传文件
- 存储在 Web 根目录外
- 验证文件内容
- 设置文件执行权限
