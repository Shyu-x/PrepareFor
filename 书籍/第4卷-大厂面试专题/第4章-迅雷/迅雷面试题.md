# 第4卷-大厂面试专题

## 第4章 迅雷

---

## 一、 面试概况
- **技术栈偏好**：Vue (为主), React, Electron (桌面端), WebRTC/P2P (传输相关)。
- **核心关注点**：JavaScript 基础深度、浏览器原理、网络协议 (TCP/UDP/P2SP)、性能优化、桌面端跨端方案。

---

## 二、 一面：技术基础 (45-60 min)

这一轮面试侧重于对前端基础知识的“刨根问底”，面试官通常会从一个基础点引申出底层原理。

### 2.1 JavaScript 深度
- **事件机制**：
  - `e.target` 与 `e.currentTarget` 的精确区别及应用场景（如事件委托）。
  - 如何手写实现自定义事件（如模拟 `click`）。
- **对象与函数**：
  - 构造函数如果手动 `return` 一个对象会发生什么？如果 `return` 原始类型呢？
  - `typeof` 的实现原理是什么？为什么 `typeof null === 'object'`？
- **异步编程**：
  - `Promise` 与 `async/await` 的底层联系，为什么有了 Promise 还需要 async/await？
  - 手写实现 `Promise.all` 或 `Promise` 限制并发数。
- **内存管理**：
  - 浏览器的垃圾回收机制（引用计数 vs 标记清除），V8 引擎如何优化（新生代与老生代）。

### 2.2 CSS 与 移动端
- **布局单位**：在不使用插件的情况下，如何利用 `vw` 和 `vh` 实现移动端自适应？
- **盒模型**：`padding-top: 50%;` 是相对于父元素的宽度还是高度？（答案：宽度）。
- **渲染性能**：
  - CSS 加载会阻塞 JS 加载吗？会阻塞渲染吗？
  - 如何实现丝滑动画？`requestAnimationFrame` 的优势。

### 2.3 浏览器与网络
- **缓存策略**：`ETag` 和 `Last-Modified` 同时存在时，哪一个优先级更高？（答案：ETag）。
- **跨标签通信**：除了监听 `storage` 事件，还有哪些方式可以实现浏览器多标签页通信？（如 `postMessage`、`SharedWorker`、`BroadcastChannel`）。
- **生命周期**：`window.DOMContentLoaded` 和 `window.onload` 的精确触发时机差异。

---

## 三、 二面：工程化与项目深挖 (45-60 min)

二面通常由资深开发或 Leader 主持，侧重于解决问题的思路和对工具底层原理的理解。

### 3.1 性能与稳定性
- **手写 vs 库**：在项目中，为什么你会倾向于自己写节流（throttle）或防抖（debounce）函数，而不是直接引用 Lodash？（考察对包体积、按需加载的敏感度）。
- **网络优化**：`webpack proxy` 的底层原理是什么？它是如何解决跨域的？在生产环境下又该如何处理？

### 3.2 桌面端专项 (针对迅雷客户端团队)
- **Electron 架构**：
  - 主进程与渲染进程的隔离性。如何安全地通过 `preload.js` 暴露 API？
  - `contextIsolation` 和 `nodeIntegration` 的安全性考虑。
- **渲染优化**：在 Electron 中，如何利用 `desktopCapturer` 或 `WebRTC` 实现高效的屏幕捕获或音视频传输？

### 3.3 业务场景
- **权限管理**：为什么不推荐使用指令（Directive）来实现按钮权限？（考察对安全性和动态渲染的理解）。

---

## 四、 迅雷特色考点：传输与协议

由于迅雷的业务特性，面试中常会出现与网络传输相关的深度问题。

- **P2SP 技术**：对 Peer-to-Server and Peer 架构的理解。
- **UDP 穿透**：STUN/TURN 服务器如何帮助 WebRTC 实现 NAT 穿透？
- **TCP 握手/挥手**：经典的 TCP 三次握手和四次挥手过程，以及为什么需要四次挥手。
- **HTTP/2.0**：相比 HTTP/1.1，2.0 做了哪些重大改进（多路复用、头部压缩等）？
