### 6.12.2 Vitest 单元测试

**参考答案：**

**配置：**

```bash
npm install -D vitest @vitejs/plugin-react
```

```tsx
// vitest.config.ts
import { defineConfig } from 'vitest/config'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  test: {
    environment: 'jsdom',
    globals: true,
  },
})
```

**测试示例：**

```tsx
// src/utils/format.test.ts
import { describe, it, expect } from 'vitest'
import { formatDate } from './format'

describe('formatDate', () => {
  it('should format date correctly', () => {
    const date = new Date('2024-01-01')
    expect(formatDate(date)).toBe('2024-01-01')
  })

  it('should handle invalid date', () => {
    expect(formatDate(null)).toBe('')
  })
})
```

### 6.12.3 React Testing Library

**参考答案：**

**组件测试：**

```tsx
// src/components/Counter.test.tsx
import { render, screen, fireEvent } from '@testing-library/react'
import { describe, it, expect, vi } from 'vitest'
import { Counter } from './Counter'

describe('Counter', () => {
  it('should render initial count', () => {
    render(<Counter />)
    expect(screen.getByText('Count: 0')).toBeInTheDocument()
  })

  it('should increment count', () => {
    render(<Counter />)
    const button = screen.getByText('Increment')

    fireEvent.click(button)

    expect(screen.getByText('Count: 1')).toBeInTheDocument()
  })
})
```

### 6.12.4 Playwright E2E测试

**参考答案：**

**安装：**

```bash
npm init playwright@latest
```

**测试示例：**

```tsx
// tests/homepage.spec.ts
import { test, expect } from '@playwright/test'

test('homepage has title', async ({ page }) => {
  await page.goto('/')

  await expect(page).toHaveTitle(/My App/)
})

test('can navigate to about page', async ({ page }) => {
  await page.goto('/')

  await page.click('text=About')

  await expect(page).toHaveURL('/about')
  await expect(page.locator('h1')).toContainText('About')
})
```

**组件测试：**

```tsx
// tests/counter.spec.ts
import { test, expect } from '@playwright/test'

test('counter increments', async ({ page }) => {
  await page.goto('/counter')

  await expect(page.locator('text=Count: 0')).toBeVisible()

  await page.click('text=Increment')

  await expect(page.locator('text=Count: 1')).toBeVisible()
})
```

### 6.12.5 Server Component 测试

**参考答案：**

```tsx
// src/app/posts/page.test.tsx
import { render, screen, waitFor } from '@testing-library/react'
import { HttpResponse, http } from 'msw/node'
import { setupServer } from 'msw/node'
import PostsPage from '@/app/posts/page'

const server = setupServer(
  http.get('/api/posts', () => {
    return HttpResponse.json([
      { id: 1, title: 'Post 1' },
      { id: 2, title: 'Post 2' },
    ])
  })
)

beforeAll(() => server.listen())
afterAll(() => server.close())

describe('PostsPage', () => {
  it('should render posts', async () => {
    render(<PostsPage />)

    await waitFor(() => {
      expect(screen.getByText('Post 1')).toBeInTheDocument()
    })
  })
})
```

### 6.12.7 面试常见问题

**Q1: 如何测试 Server Components？**

- 直接测试组件渲染结果
- 使用 MSW 模拟 API
- 可以使用 jest 或 vitest

**Q2: E2E 测试 vs 单元测试如何选择？**

- 单元测试：业务逻辑、工具函数
- 集成测试：组件交互
- E2E 测试：关键用户路径

**Q3: Playwright vs Cypress 怎么选？**

| 特性 | Playwright | Cypress |
|:---|:---|:---|
| 并行测试 | ✅ | ✅ |
| 多标签页 | ✅ | 有限 |
| iframe | ✅ | ✅ |
| 社区 | 增长快 | 成熟 |

---

> **面试提示**：测试是企业级应用的重要部分，需要理解不同测试方法的适用场景和最佳实践。
