### 6.4.2 Cache Components (Next.js 16 新特性)

**参考答案：**

Next.js 16 引入了显式缓存机制，这是与之前版本最大的区别。

**核心概念：**

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓     
┃                    Cache Components vs 传统缓存                      ┃
┃                                                                  ┃    
┃  Next.js 15 及之前 (隐式缓存)                                      ┃  
┃  ┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓   ┃     
┃  ┃  fetch 默认缓存                                            ┃   ┃   
┃  ┃  组件默认不缓存                                            ┃   ┃   
┃  ┃  缓存行为难以预测                                          ┃   ┃   
┃  ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛   ┃     
┃                                                                  ┃    
┃  Next.js 16 (显式缓存)                                           ┃    
┃  ┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓   ┃     
┃  ┃  'use cache' 指令声明缓存                                  ┃   ┃   
┃  ┃  动态代码默认每次执行                                       ┃   ┃  
┃  ┃  缓存行为完全可控                                          ┃   ┃   
┃  ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛   ┃     
┃                                                                  ┃    
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛     

```

**基本用法：**

```tsx
// src/app/page.tsx

// 缓存整个组件
'use cache'

export default async function CachedPage() {
  const data = await fetchData()
  return <div>{data.content}</div>
}

// 缓存组件函数
async function fetchData() {
  'use cache'
  const res = await fetch('https://api.example.com/data')
  return res.json()
}

// 缓存带标签的数据
async function fetchPosts() {
  'use cache'
  cacheTag('posts')
  const res = await fetch('https://api.example.com/posts')
  return res.json()
}

// 缓存带过期时间
async function fetchUser() {
  'use cache'
  cacheTag('user')
  const res = await fetch('https://api.example.com/user', {
    next: { revalidate: 3600 }  // 1小时后过期
  })
  return res.json()
}
```

**缓存配置选项：**

```tsx
// 基础缓存 - 永久缓存
'use cache'
export default function StaticPage() { }

// 带标签的缓存 - 可按需失效
async function getData() {
  'use cache'
  cacheTag('posts', 'featured')
  // ...
}

// 带过期时间 - ISR
async function getData() {
  'use cache'
  cacheTag('products')
  // 1小时后重新验证
}

// 动态数据 - 不缓存
// 在 Next.js 16 中，所有不带 'use cache' 的代码默认动态执行
export default async function DynamicPage() {
  // 每次请求都会执行
  const data = await fetchDynamicData()
  return <div>{data}</div>
}
```

**按需失效：**

```tsx
// src/app/api/revalidate/route.ts
import { revalidateTag } from 'next/cache'
import { NextRequest, NextResponse } from 'next/server'

export async function POST(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')

  // 失效指定标签的缓存
  revalidateTag(tag!)

  return NextResponse.json({
    revalidated: true,
    tag,
    timestamp: Date.now()
  })
}
```

**与 fetch 缓存的关系：**

```tsx
// Next.js 16 中两种缓存方式可以共存

// 方式1: fetch 缓存
const data1 = await fetch('https://api.example.com/data', {
  next: { tags: ['my-data'] }
})

// 方式2: Cache Components
async function getCachedData() {
  'use cache'
  cacheTag('my-data')
  const res = await fetch('https://api.example.com/data2')
  return res.json()
}

// 两者可以使用相同的标签，便于统一管理
```

**面试考点：**

| 问题 | 答案要点 |
|:---|:---|
| Cache Components 与 fetch 缓存的区别？ | Cache Components 是组件级缓存，fetch 是请求级缓存 |
| 为什么要引入显式缓存？ | 之前隐式缓存行为难以预测，显式缓存更可控 |
| 如何选择使用哪种缓存？ | 静态内容用 Cache Components，外部 API 用 fetch 缓存 |

### 6.4.3 Fetch API 缓存策略

**参考答案：**

```tsx
// src/app/page.tsx

export default async function Page() {
  // 1. 静态数据（默认）- 类似 getStaticProps
  const staticData = await fetch('https://api.example.com/data', {
    cache: 'force-cache',  // 默认，可以省略
  })

  // 2. 动态数据 - 类似 getServerSideProps
  const dynamicData = await fetch('https://api.example.com/data', {
    cache: 'no-store',
  })

  // 3. ISR - 每10秒重新验证 - 类似 revalidate
  const revalidatedData = await fetch('https://api.example.com/data', {
    next: { revalidate: 10 },
  })

  // 4. 基于标签的缓存
  const taggedData = await fetch('https://api.example.com/data', {
    next: { tags: ['posts'] },
  })

  return <div>{/* render data */}</div>
}
```

**缓存策略对比：**

| 策略 | cache 选项 | next 选项 | 效果 |
|:---|:---|:---|:---|
| 静态 | `force-cache` | - | 永久缓存，构建时确定 |
| 动态 | `no-store` | - | 每次请求获取 |
| ISR | `force-cache` | `{ revalidate: 60 }` | 定时重新验证 |
| 标签 | `force-cache` | `{ tags: ['x'] }` | 按标签重新验证 |

### 6.4.4 重新验证（Revalidating）

**参考答案：**

**按时间重新验证：**

```tsx
// 方法1: fetch 配置
const data = await fetch('https://api.example.com/posts', {
  next: { revalidate: 60 },  // 60秒
})

// 方法2: 路由配置
export const revalidate = 60  // 整个路由
```

**按标签重新验证：**

```tsx
// 获取数据时添加标签
const posts = await fetch('https://api.example.com/posts', {
  next: { tags: ['posts'] },
})

// 后端 API 中手动重新验证
// src/app/api/revalidate/route.ts
import { revalidateTag } from 'next/cache'
import { NextRequest, NextResponse } from 'next/server'

export async function POST(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')

  if (tag) {
    revalidateTag(tag)
    return NextResponse.json({ revalidated: true, tag })
  }

  return NextResponse.json({ error: 'Missing tag' }, { status: 400 })
}
```

**revalidatePath：**

```tsx
'use server'

import { revalidatePath } from 'next/cache'

export async function updatePost(id: string, data: any) {
  await db.post.update({
    where: { id },
    data,
  })

  // 重新验证指定路径
  revalidatePath('/posts')
  revalidatePath(`/posts/${id}`)
}
```

### 6.4.5 数据更新（Updating Data）

**参考答案：**

**使用 Server Actions 更新数据：**

```tsx
// src/app/actions.ts
'use server'

import { db } from '@/lib/db'
import { revalidatePath } from 'next/cache'

export async function createPost(formData: FormData) {
  const title = formData.get('title') as string
  const content = formData.get('content') as string

  const post = await db.post.create({
    data: { title, content },
  })

  // 重新验证页面
  revalidatePath('/posts')
  revalidatePath(`/posts/${post.id}`)

  return { success: true, post }
}

export async function updatePost(id: string, data: Partial<Post>) {
  const post = await db.post.update({
    where: { id },
    data,
  })

  revalidatePath('/posts')
  revalidatePath(`/posts/${id}`)

  return { success: true, post }
}

export async function deletePost(id: string) {
  await db.post.delete({
    where: { id },
  })

  revalidatePath('/posts')
}
```

**表单中的数据更新：**

```tsx
// src/app/posts/new/page.tsx

      </label>
```

### 6.4.6 use() API（Next.js 15+）

**参考答案：**

React 19 的 use() API 可用于在组件中读取 Promise。

**在 Server Component 中使用：**

```tsx
// src/app/page.tsx
import { getPosts } from '@/lib/posts'

export default async function Page() {
  // 直接 await promise
  const posts = await getPosts()

  return <PostList posts={posts} />
}
```

**在 Client Component 中使用 use()：**

```tsx
// src/components/PostList.tsx
'use client'

import { use } from 'react'

export function PostList({ postsPromise }: { postsPromise: Promise<Post[]> }) {
  const posts = use(postsPromise)

  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

```tsx
// src/app/page.tsx
import { PostList } from '@/components/PostList'
import { getPosts } from '@/lib/posts'

export default function Page() {
  const postsPromise = getPosts()

  return (
    <main>
      <h1>Blog Posts</h1>
      <PostList postsPromise={postsPromise} />
    </main>
  )
}
```

**与 Suspense 配合：**

```tsx
// src/components/DeferredPostList.tsx
'use client'

import { use, Suspense } from 'react'

function PostContent({ postsPromise }: { postsPromise: Promise<Post[]> }) {
  const posts = use(postsPromise)
  return <ul>{posts.map(p => <li key={p.id}>{p.title}</li>)}</ul>
}

export function DeferredPostList({ postsPromise }: Props) {
  return (
    <Suspense fallback={<div>加载中...</div>}>
      <PostContent postsPromise={postsPromise} />
    </Suspense>
  )
}
```

### 6.4.7 数据请求优化

**参考答案：**

**并行请求：**

```tsx
// src/app/page.tsx

// 串行（慢）
const user = await getUser()
const posts = await getPosts()

// 并行（快）
const [user, posts] = await Promise.all([
  getUser(),
  getPosts()
])
```

**串行依赖请求：**

```tsx
// 需要顺序获取
const user = await getUser()
const settings = await getUserSettings(user.id)  // 依赖 user

// 优化：可以合并 API 或使用并行
```

**预加载数据：**

```tsx
// src/lib/preload.ts
import { cache } from 'react'

export const getUser = cache(async () => {
  const res = await fetch('/api/user')
  return res.json()
})

export const getUserPosts = cache(async (userId: string) => {
  const res = await fetch(`/api/users/${userId}/posts`)
  return res.json()
})

// 使用 - 预加载
getUser()           // 预加载
getUserPosts(id)    // 预加载

// 页面中
export default async function Page() {
  const user = await getUser()       // 使用缓存
  const posts = await getUserPosts(user.id)  // 使用缓存
}
```

### 6.4.8 面试常见问题

**Q1: Next.js 的缓存机制是怎样的？**

| 缓存类型 | 说明 |
|:---|:---|
| Data Cache | fetch 响应的缓存 |
| Full Route Cache | 渲染输出的 HTML |
| Router Cache | 预加载的路由段 |
| Request Memoization | 请求级别的内存缓存 |

**Q2: ISR 和 SSR 有什么区别？**

- ISR: 静态生成，定期重新验证，兼顾性能和新鲜度
- SSR: 每次请求动态渲染，保证最新但速度慢

**Q3: 如何实现按需重新验证？**

1. 使用 revalidateTag 标签
2. 在 Server Action 中调用 revalidatePath
3. 通过 API Route 调用 revalidateTag

**Q4: Next.js 15 的缓存有什么变化？**

- fetch 请求默认不缓存
- 需要显式设置 cache: 'force-cache'
- 这与之前的默认行为不同
