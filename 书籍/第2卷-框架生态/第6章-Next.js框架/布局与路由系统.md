## 6.2 布局与路由系统

### 6.2.1 布局系统（Layouts）

**参考答案：**

Next.js 使用嵌套布局系统，每个路由可以有自己的布局。

**根布局（Root Layout）：**

```tsx
// src/app/layout.tsx
import './globals.css'
import type { Metadata } from 'next'
import { Inter } from 'next/font/google'

const inter = Inter({ subsets: ['latin'] })

export const metadata: Metadata = {
  title: {
    template: '%s | 我的网站',
    default: '我的网站',
  },
  description: 'Next.js 学习指南',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="zh-CN">
      <body className={inter.className}>
        <header>
          <nav>导航栏</nav>
        </header>
        <main>
          {children}
        </main>
        <footer>
          <p>© 2024 我的网站</p>
        </footer>
      </body>
    </html>
  )
}
```

**嵌套布局：**

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓  
┃                    布局嵌套层次                                   ┃
┃                                                                  ┃ 
┃  app/layout.tsx (根布局)                                         ┃ 
┃  ┣━━ <html>, <body>                                            ┃   
┃  ┣━━ <Header> (全局)                                            ┃  
┃  ┗━━ app/(marketing)/layout.tsx (路由组布局)                     ┃ 
┃      ┣━━ <MarketingNav>                                         ┃  
┃      ┗━━ app/(marketing)/about/page.tsx                         ┃  
┃                                                                  ┃ 
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛  

```

```tsx
// src/app/dashboard/layout.tsx
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <div className="dashboard">
      <aside className="sidebar">
        <ul>
          <li><a href="/dashboard">概览</a></li>
          <li><a href="/dashboard/settings">设置</a></li>
        </ul>
      </aside>
      <div className="content">
        {children}
      </div>
    </div>
  )
}
```

**路由组布局：**

```tsx

  return (
```

### 6.2.2 页面系统（Pages）

**参考答案：**

**基础页面：**

```tsx
// src/app/page.tsx
export default function Page() {
  return <h1>首页</h1>
}
```

**动态页面：**

```tsx
// src/app/blog/[category]/[slug]/page.tsx
interface PageProps {
  params: Promise<{
    category: string
    slug: string
  }>
}

export async function generateStaticParams() {
  const posts = [
    { category: 'tech', slug: 'nextjs' },
    { category: 'life', slug: 'hello' },
  ]
  return posts.map((post) => ({
    category: post.category,
    slug: post.slug,
  }))
}

export default async function BlogPost({ params }: PageProps) {
  const { category, slug } = await params

  return (
    <article>
      <h1>{slug}</h1>
      <p>分类: {category}</p>
    </article>
  )
}
```

**可选动态参数：**

```tsx
// src/app/blog/[[...slug]]/page.tsx
// 匹配: /blog, /blog/a, /blog/a/b, /blog/a/b/c

interface PageProps {
  params: Promise<{
    slug?: string[]
  }>
}

export default async function BlogPage({ params }: PageProps) {
  const { slug } = await params
  const path = slug?.join('/') || ''

  return <h1>博客路径: {path || '首页'}</h1>
}
```

### 6.2.3 链接与导航（Linking and Navigating）

**参考答案：**

**Link 组件：**

```tsx
import Link from 'next/link'

export default function Nav() {
  return (
    <nav>
      <Link href="/">首页</Link>
      <Link href="/about">关于</Link>
      <Link href="/blog/nextjs">博客文章</Link>

      {/* 动态链接 */}
      <Link href={`/user/${userId}>用户主页</Link>

      {/* 带查询参数 */}
      <Link href="/dashboard?tab=overview">仪表盘</Link>

      {/* 替换历史记录 */}
      <Link href="/login" replace>登录</Link>
    </nav>
  )
}
```

**编程式导航：**

```tsx
'use client'

import { useRouter, usePathname, useSearchParams } from 'next/navigation'

export function NavigationExample() {
  const router = useRouter()
  const pathname = usePathname()
  const searchParams = useSearchParams()

  const handleNavigation = () => {
    // 跳转到指定页面
    router.push('/dashboard')

    // 替换当前页面（不产生历史记录）
    router.replace('/login')

    // 刷新当前页面
    router.refresh()

    // 预加载指定页面
    router.prefetch('/about')
  }

  // 获取查询参数
  const tab = searchParams.get('tab')

  return (
    <div>
      <p>当前路径: {pathname}</p>
      <p>当前tab: {tab}</p>
      <button onClick={handleNavigation}>导航</button>
    </div>
  )
}
```

**useParams Hook：**

```tsx
'use client'

import { useParams } from 'next/navigation'

export function UserProfile() {
  const params = useParams()

  // 返回: { id: '123' }
  const userId = params.id

  return <h1>用户ID: {userId}</h1>
}
```

**useSearchParams Hook（需要 Suspense）：**

```tsx
'use client'

import { useSearchParams } from 'next/navigation'
import { Suspense } from 'react'

function SearchContent() {
  const searchParams = useSearchParams()
  const query = searchParams.get('q')

  return <p>搜索关键词: {query}</p>
}

export default function SearchPage() {
  return (
    <Suspense fallback={<p>加载中...</p>}>
      <SearchContent />
    </Suspense>
  )
}
```

### 6.2.4 路由组（Route Groups）

**参考答案：**

路由组使用圆括号 `(folderName)` 创建，不影响 URL 路径。

**组织结构：**

```
app/
┣━━ (marketing)/           # 路由组 - 不影响 URL
┃   ┣━━ layout.tsx       # 营销页面布局
┃   ┣━━ page.tsx         # / (首页)
┃   ┗━━ about/
┃       ┗━━ page.tsx     # /about
┃
┣━━ (shop)/               # 另一个路由组
┃   ┣━━ layout.tsx       # 商店布局
┃   ┣━━ products/
┃   ┃   ┗━━ page.tsx    # /products
┃   ┗━━ cart/
┃       ┗━━ page.tsx    # /cart
┃
┗━━ layout.tsx            # 根布局
```

**使用场景：**

1. **不同布局**：营销页面和后台使用不同布局
2. **分组管理**：按功能模块组织路由
3. **版本控制**：创建 `/v1/`, `/v2/` 路由组

### 6.2.5 并行路由（Parallel Routes）

**参考答案：**

并行路由使用 `@folder` 语法，可以在同一页面同时渲染多个路由。

**语法：**

```
app/
┣━━ @modal/               # 并行路由（槽位）
┃   ┗━━ /(.)login/
┃       ┗━━ page.tsx    # 模态框
┣━━ @sidebar/            # 另一个槽位
┃   ┗━━ page.tsx       # 侧边栏
┣━━ login/
┃   ┗━━ page.tsx       # 实际登录页
┗━━ page.tsx           # 使用槽位的主页
```

**使用槽位：**

```tsx
// src/app/page.tsx
import { Modal } from '@modal/(.)login/page'
import { Sidebar } from '@sidebar/page'

export default function Page() {
  return (
    <div>
      <Sidebar />
      <main>主内容</main>
      {/* 模态框会在条件满足时渲染 */}
      <Modal />
    </div>
  )
}
```

**拦截路由：**

```tsx
// @modal/(.)login/page.tsx
// 当访问 /login 时在模态框中显示

import Link from 'next/link'

export default function LoginModal() {
  return (
    <div className="modal">
      <h1>登录</h1>
      <Link href="/dashboard">进入仪表盘</Link>
    </div>
  )
}
```

### 6.2.6 拦截路由（Intercepting Routes）

**参考答案：**

拦截路由使用 `(.)folder` 语法，可以在特定场景下拦截路由。

**使用场景：**

```tsx
// 照片浏览应用
// 当点击照片时，模态框显示，同时 URL 更新
// 直接访问 URL 时，显示完整页面

app/
┣━━ photos/                                            
┃   ┗━━ [id]/                                          
┃       ┗━━ page.tsx        # 完整页面 /photos/123     
┃━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┃
┗━━ @modal/                  # 并行路由                
    ┗━━ (.)photos/                                     
        ┗━━ [id]/                                      
            ┗━━ page.tsx    # 模态框 /photos/123 (拦截)
```

**拦截模式：**

| 模式 | 语法 | 说明 |
|:---|:---|:---|
| 同级 | `(.)photos` | 拦截同级别 |
| 向上 | `(..)photos` | 拦截上级 |
| 根 | `(...)photos` | 拦截根路由 |

### 6.2.7 路由配置（Route Segment Config）

**参考答案：**

每个路由段可以导出配置来控制渲染行为。

```tsx
// src/app/posts/page.tsx

// 强制动态渲染
export const dynamic = 'force-dynamic'

// 强制静态渲染
export const dynamic = 'force-static'

// 设置 ISR 重新验证时间（秒）
export const revalidate = 60

// 禁用未预渲染的动态路由
export const dynamicParams = false

// 配置缓存策略
export const fetchCache = 'default-no-store'

// 运行时
export const runtime = 'nodejs' // 或 'edge'

// 预渲染参数
export async function generateStaticParams() {
  const posts = await getPosts()
  return posts.map((post) => ({
    slug: post.slug,
  }))
}

export default async function PostsPage() {
  const posts = await getPosts()
  return <PostList posts={posts} />
}
```

**配置选项详解：**

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓   
┃                    路由段配置选项                                  ┃
┃                                                                  ┃  
┃  dynamic:                                                        ┃  
┃    - 'auto'         默认，根据数据获取自动决定                    ┃ 
┃    - 'force-dynamic' 强制动态渲染，每次请求重新渲染               ┃ 
┃    - 'force-static'  强制静态渲染，部署时确定                    ┃  
┃    - 'error'        动态参数未预渲染时抛出错误                    ┃ 
┃                                                                  ┃  
┃  dynamicParams:                                                   ┃ 
┃    - true (默认)    允许未预渲染的动态参数                        ┃ 
┃    - false         不允许，返回 404                              ┃  
┃                                                                  ┃  
┃  revalidate:                                                      ┃ 
┃    - number        ISR 重新验证时间（秒）                         ┃ 
┃    - false         禁用重新验证                                  ┃  
┃    - 0            每次请求都重新验证                             ┃  
┃                                                                  ┃  
┃  fetchCache:                                                     ┃  
┃    - 'auto'        默认                                          ┃  
┃    - 'default-cache' 使用默认缓存                                ┃  
┃    - 'only-cache'   只使用缓存，失败则报错                       ┃  
┃    - 'force-cache'  强制使用缓存                                 ┃  
┃    - 'force-no-store' 强制不缓存                                 ┃  
┃    - 'default-no-store' 默认不缓存                              ┃   
┃                                                                  ┃  
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛   

```

### 6.2.8 面试常见问题

**Q1: App Router 和 Pages Router 的布局有什么区别？**

| 特性 | App Router | Pages Router |
|:---|:---|:---|
| 布局定义 | layout.tsx | _app.tsx |
| 文档结构 | layout.tsx | _document.tsx |
| 布局嵌套 | 自动嵌套 | 手动包裹 |
| 布局类型 | Server Component | Client Component |

**Q2: 如何实现模态框和路由联动？**

使用并行路由 + 拦截路由：
1. 创建 @modal 槽位存放模态框
2. 使用 (.) 前缀创建拦截路由
3. 用户点击时显示模态框并更新 URL
4. 直接访问 URL 时显示完整页面

**Q3: 路由组和普通文件夹有什么区别？**

- 路由组 `(folder)` 不影响 URL 路径
- 可以为不同组设置不同布局
- 常用于区分不同业务模块或版本

---

> **面试提示**：布局和路由是 Next.js 最核心的概念，特别是 App Router 的嵌套布局、路由组和并行路由是高级面试题的重点。
