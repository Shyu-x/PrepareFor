## 6.5 错误处理与边界

### 6.5.1 错误边界（Error Boundaries）

**参考答案：**

Next.js 提供了错误处理机制来处理渲染错误。

**error.tsx - 客户端错误边界：**

```tsx
// src/app/error.tsx
'use client'

import { useEffect } from 'react'

export default function Error({
  error,
  reset,
}: {
  error: Error & { digest?: string }
  reset: () => void
}) {
  useEffect(() => {
    // 记录错误到日志服务
    console.error(error)
  }, [error])

  return (
    <div className="error-page">
      <h2>出现问题！</h2>
      <p>发生了错误，请稍后重试。</p>
      <button
        onClick={
          // 尝试恢复渲染
          () => reset()
        }
      >
        重试
      </button>
    </div>
  )
}
```

**错误边界层级：**

```
app/
┣━━ error.tsx           # 根错误边界，处理全局错误
┣━━ dashboard/
┃   ┣━━ error.tsx     # 仪表盘错误边界
┃   ┗━━ page.tsx
┣━━ blog/
┃   ┗━━ error.tsx     # 博客错误边界
┗━━ page.tsx
```

**错误状态管理：**

```tsx
'use client'

import { useState } from 'react'

export default function Error({
  error,
  reset,
}: {
  error: Error
  reset: () => void
}) {
  const [isRetrying, setIsRetrying] = useState(false)

  const handleRetry = async () => {
    setIsRetrying(true)
    reset()
    setIsRetrying(false)
  }

  return (
    <div>
      <h2>发生错误</h2>
      <p>{error.message}</p>
      <button onClick={handleRetry} disabled={isRetrying}>
        {isRetrying ? '重试中...' : '重试'}
      </button>
    </div>
  )
}
```

### 6.5.2 not-found.tsx

**参考答案：**

处理 404 错误。

**基础用法：**

```tsx
// src/app/not-found.tsx
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>无法找到请求的页面</p>
      <Link href="/">返回首页</Link>
    </div>
  )
}
```

**在页面中使用 notFound() 函数：**

```tsx
// src/app/blog/[slug]/page.tsx
import { notFound } from 'next/navigation'

export default async function BlogPost({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  const post = await getPost(slug)

  if (!post) {
    notFound()
  }

  return <article>{post.content}</article>
}
```

**自定义 not-found 页面内容：**

```tsx
// src/app/blog/[slug]/page.tsx
import { notFound } from 'next/navigation'

export default async function BlogPost({ params }: Props) {
  const post = await getPost(params.slug)

  if (!post) {
    notFound()
  }

  return <article>{post.content}</article>
}
```

```tsx
// src/app/blog/[slug]/not-found.tsx
export default function NotFound() {
  return (
    <div>
      <h2>文章未找到</h2>
      <p>抱歉，您访问的文章不存在或已被删除。</p>
    </div>
  )
}
```

### 6.5.3 forbidden.tsx 和 unauthorized.tsx

**参考答案：**

处理权限相关的错误。

**forbidden.tsx - 禁止访问：**

```tsx
// src/app/forbidden.tsx
export default function Forbidden() {
  return (
    <div>
      <h2>禁止访问</h2>
      <p>您没有权限访问此页面。</p>
    </div>
  )
}
```

**unauthorized.tsx - 未授权：**

```tsx
// src/app/unauthorized.tsx
import Link from 'next/link'

export default function Unauthorized() {
  return (
    <div>
      <h2>请先登录</h2>
      <p>您需要登录才能访问此页面。</p>
      <Link href="/login">登录</Link>
    </div>
  )
}
```

**使用 forbidden() 和 unauthorized()：**

```tsx
import { forbidden, unauthorized } from 'next/navigation'

export async function getData(userId: string) {
  const user = await getUser(userId)

  if (!user) {
    unauthorized()
  }

  if (!user.hasPermission) {
    forbidden()
  }

  return user.data
}
```

### 6.5.4 加载状态（loading.tsx）

**参考答案：**

**基础加载组件：**

```tsx
// src/app/dashboard/loading.tsx
export default function Loading() {
  return (
    <div className="loading-skeleton">
      <div className="skeleton-header" />

          <div key={i} className="skeleton-line" />
        ))}

```

**骨架屏组件：**

```tsx
// src/components/Skeleton.tsx
export function PostSkeleton() {

        <div className="skeleton-line" />

```

### 6.5.5 模板（template.tsx）

**参考答案：**

template.tsx 会在每次导航时重新创建，与 layout.tsx 不同。

**template.tsx 用法：**

```tsx
// src/app/template.tsx
export default function Template({ children }: { children: React.ReactNode }) {
  return (
    <div className="page-transition">
      {children}
    </div>
  )
}
```

**layout vs template：**

| 特性 | layout.tsx | template.tsx |
|:---|:---|:---|
| 导航时重新创建 | 否 | 是 |
| 状态保持 | 是 | 否 |
| 适用场景 | 共享布局 | 页面过渡动画 |

### 6.5.6 面试常见问题

**Q1: Next.js 的错误处理机制有哪些？**

| 组件 | 作用 |
|:---|:---|
| error.tsx | 客户端错误边界 |
| not-found.tsx | 404 页面 |
| forbidden.tsx | 403 禁止访问 |
| unauthorized.tsx | 401 未授权 |

**Q2: error.tsx 必须是客户端组件吗？**

是的，error.tsx 必须是客户端组件（'use client'）。

**Q3: layout.tsx 和 template.tsx 有什么区别？**

- layout: 导航时保持状态，不重新创建
- template: 每次导航时重新创建，适合动画
