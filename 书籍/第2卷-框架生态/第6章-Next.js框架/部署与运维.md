## 6.9 部署与运维

### 6.9.1 部署到 Vercel

**参考答案：**

Vercel 是 Next.js 的官方部署平台，提供开箱即用的支持。

**部署流程：**

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓   
┃                    Vercel 部署流程                                 ┃
┃                                                                  ┃  
┃  1. 推送代码到 GitHub/GitLab/Bitbucket                          ┃   
┃  2. 在 Vercel 导入项目                                          ┃   
┃  3. 自动检测 Next.js 并配置                                      ┃  
┃  4. 自动部署                                                     ┃  
┃  5. 自动生成预览部署（PR）                                       ┃  
┃                                                                  ┃  
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛   

```

**预览部署：**

- 每个 PR 自动创建预览 URL
- 可以实时预览变更
- 预览链接永久保存

**环境变量配置：**

```
Vercel Dashboard → Project → Settings → Environment Variables

DATABASE_URL=...
API_KEY=...
NEXT_PUBLIC_API_URL=...
```

### 6.9.2 Docker 部署

**参考答案：**

**Dockerfile：**

```dockerfile
# 阶段1: 构建
FROM node:18-alpine AS builder

WORKDIR /app

# 安装依赖
COPY package.json yarn.lock ./
RUN yarn install --frozen-lockfile

# 复制源码
COPY . .

# 构建
RUN yarn build

# 阶段2: 运行
FROM node:18-alpine AS runner

WORKDIR /app

# 设置环境
ENV NODE_ENV=production

# 复制必要文件
COPY --from=builder /app/public ./public
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static

EXPOSE 3000

CMD ["node", "server.js"]
```

**standalone 输出配置：**

```tsx
// next.config.ts
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  output: 'standalone',
}

export default nextConfig
```

### 6.9.3 自托管（Self-Hosting）

**参考答案：**

**使用 Node.js：**

```bash
# 构建
npm run build

# 启动
npm start
```

**使用 PM2：**

```bash
# 安装 PM2
npm install -g pm2

# 启动
pm2 start npm --name "nextjs" -- start

# 开机自启
pm2 startup
pm2 save
```

**Nginx 配置：**

```nginx
upstream nextjs {
    server 127.0.0.1:3000;
}

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://nextjs;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 6.9.4 环境变量

**参考答案：**

**环境变量文件：**

```
.env                # 所有环境（不提交）
.env.local          # 本地开发（不提交）
.env.development    # 开发环境
.env.production     # 生产环境
.env.test           # 测试环境
```

**使用环境变量：**

```tsx
// 服务器端
const dbUrl = process.env.DATABASE_URL!

// 客户端（需要 NEXT_PUBLIC_ 前缀）
const apiUrl = process.env.NEXT_PUBLIC_API_URL
```

**TypeScript 类型：**

```tsx
// src/types/env.d.ts
namespace NodeJS {
  interface ProcessEnv {
    DATABASE_URL: string
    API_KEY: string
    NEXT_PUBLIC_API_URL: string
  }
}
```

### 6.9.5 性能监控

**参考答案：**

**Vercel Analytics：**

```tsx
// src/app/layout.tsx
import { Analytics } from '@vercel/analytics/react'

export default function RootLayout({ children }: Props) {
  return (
    <html>
      <body>
        {children}
        <Analytics />
      </body>
    </html>
  )
}
```

**Web Vitals：**

```tsx
// src/app/components/WebVitals.tsx
'use client'

import { useReportWebVitals } from 'next/web-vitals'

export function WebVitals() {
  useReportWebVitals((metric) => {
    console.log(metric)
    // 上报到分析服务
  })

  return null
}
```

```tsx
// src/app/layout.tsx
import { WebVitals } from '@/components/WebVitals'

export default function RootLayout({ children }: Props) {
  return (
    <html>
      <body>
        {children}
        <WebVitals />
      </body>
    </html>
  )
}
```

### 6.9.6 面试常见问题

**Q1: Vercel 部署有哪些优势？**

- 零配置部署
- 自动预览部署
- 全球 CDN
- Edge Functions
- Vercel Analytics

**Q2: 如何选择部署方式？**

| 场景 | 推荐 |
|:---|:---|
| 新项目 | Vercel |
| 企业/私有云 | Docker + 自托管 |
| 已有基础设施 | Node.js + PM2 |

**Q3: Next.js 的缓存如何配置？**

- Vercel: 自动配置 CDN 缓存
- 自托管: 需要配置 Nginx/负载均衡器

---

> **面试提示**：部署和运维是高级面试常考的内容，需要理解不同部署方案的优缺点和适用场景。
