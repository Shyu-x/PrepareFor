### 6.13.1 从 Pages Router 迁移

**参考答案：**

**迁移步骤：**

```

┃                                                                  ┃
```

### 6.13.2 核心迁移对照

**参考答案：**

| Pages Router | App Router |
|:---|:---|
| `_app.tsx` | `layout.tsx` |
| `_document.tsx` | `layout.tsx` |
| `pages/api/*` | `app/api/*/route.ts` |
| `getStaticProps` | async/await |
| `getServerSideProps` | async/await |
| `getStaticPaths` | `generateStaticParams` |
| `Link` | `Link` (相同) |
| `useRouter` | `useRouter`, `usePathname`, `useSearchParams` |

### 6.13.3 数据获取迁移

**参考答案：**

**Pages Router：**

```tsx
// pages/posts/[id].tsx
export async function getStaticProps({ params }) {
  const post = await getPost(params.id)
  return { props: { post } }
}

export async function getStaticPaths() {
  const posts = await getPosts()
  return {
    paths: posts.map((post) => ({ params: { id: post.id } })),
    fallback: 'blocking',
  }
}

export default function Post({ post }) {
  return <h1>{post.title}</h1>
}
```

**App Router：**

```tsx
// app/posts/[id]/page.tsx

export async function generateStaticParams() {
  const posts = await getPosts()
  return posts.map((post) => ({ id: post.id }))
}

export default async function Post({ params }: { params: Promise<{ id: string }> }) {
  const { id } = await params
  const post = await getPost(id)

  return <h1>{post.title}</h1>
}
```

### 6.13.4 布局迁移

**参考答案：**

**Pages Router：**

```tsx
// pages/_app.tsx
export default function App({ Component, pageProps }) {
  return (
    <Layout>
      <Component {...pageProps} />
    </Layout>
  )
}
```

**App Router：**

```tsx
// app/layout.tsx
export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <Layout>
          {children}
        </Layout>
      </body>
    </html>
  )
}
```

### 6.13.5 API 迁移

**参考答案：**

**Pages Router：**

```tsx
// pages/api/posts/index.ts
export default function handler(req, res) {
  if (req.method === 'GET') {
    const posts = getPosts()
    res.status(200).json(posts)
  } else if (req.method === 'POST') {
    const post = createPost(req.body)
    res.status(201).json(post)
  }
}
```

**App Router：**

```tsx
// app/api/posts/route.ts
import { NextRequest, NextResponse } from 'next/server'

export async function GET() {
  const posts = getPosts()
  return NextResponse.json(posts)
}

export async function POST(request: NextRequest) {
  const body = await request.json()
  const post = createPost(body)
  return NextResponse.json(post, { status: 201 })
}
```

### 6.13.6 useRouter 迁移

**参考答案：**

```tsx
// Pages Router
import { useRouter } from 'next/router'

export default function Page() {
  const router = useRouter()
  const { id } = router.query

  return <p>ID: {id}</p>
}
```

```tsx
// App Router
'use client'

import { useRouter, useParams, usePathname, useSearchParams } from 'next/navigation'

export default function Page() {
```

### 6.13.7 面试常见问题

**Q1: Pages Router 和 App Router 可以共存吗？**

可以，但建议逐步迁移到 App Router。

**Q2: 迁移过程中需要注意什么？**

1. Server/Client Components 边界
2. 数据获取方式变化
3. API 路由格式
4. 布局系统差异

**Q3: 为什么不直接重写？**

渐进式迁移可以：
- 降低风险
- 逐步验证
- 团队适应

---

> **面试提示**：迁移是实际项目中常见的需求，需要理解两套方案的差异和最佳实践。
